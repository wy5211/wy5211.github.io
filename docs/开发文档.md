# 个人博客系统开发文档（渐进式）

## 1. 文档目标

本文件用于从 0 到 1 搭建一个可持续迭代的个人博客系统，采用“先可用、再完善、后优化”的渐进式开发方式。

你可以先完成阶段 0～2 得到 MVP，再按阶段继续增强功能与质量。

---

## 2. 产品愿景与范围

### 2.1 愿景

构建一个**轻量、可维护、可扩展**的个人博客系统，支持内容创作、发布、阅读与基础运营。

### 2.2 首期范围（MVP）

- 博客文章的静态展示（Markdown/MDX 渲染）
- 标签与分类筛选
- SEO 基础能力（SSG 生成、sitemap）
- 响应式页面（移动端可阅读）
- 评论系统（基于 GitHub Discussions）
- 自动化部署（GitHub Actions -> GitHub Pages）

### 2.3 非目标（首期不做）

- 在线后台管理系统（改用 Git + IDE 本地管理）
- 实时协作编辑
- 用户登录注册系统

---

## 3. 技术方案（调整版：静态生成）

### 3.1 推荐技术栈

- 框架：Next.js (SSG 模式) + TypeScript
- 样式：Tailwind CSS + Shadcn UI
- 内容源：本地 Markdown/MDX 文件
- 内容处理：Contentlayer 或 Gray-matter + remark/rehype
- 评论：Giscus (基于 GitHub Discussions)
- 部署：GitHub Pages (配合 GitHub Actions)

### 3.2 选择理由

- **零成本运维**：完全利用 GitHub 免费资源，无需购买服务器与数据库。
- **高性能**：预渲染静态 HTML，加载速度极快，SEO 友好。
- **GitOps**：内容即代码，享受 Git 版本控制的所有优势。
- **安全**：无后端逻辑与数据库，几乎没有被攻击面。

---

## 4. 系统架构（逻辑）

### 4.1 模块划分

- **展示层**：
  - 首页（最近文章列表）
  - 文章详情页（Markdown 渲染 + 代码高亮 + 目录）
  - 归档页（按年份/分类/标签）
  - 关于页
- **构建层**：
  - 内容解析器（Markdown -> JSON/HTML）
  - 静态页面生成器（Next.js build）
  - 资源优化（图片压缩、CSS Purge）
- **数据层**：
  - 文件系统（`content/posts/*.mdx`）
  - 站点配置（`site.config.ts`）

### 4.2 核心数据流

1. **本地创作**：在 `content/posts` 目录下新建 Markdown 文件写作。
2. **提交推送**：`git push` 到 GitHub 仓库。
3. **自动构建**：触发 GitHub Actions，执行 `npm run build`。
4. **静态部署**：产物部署到 `gh-pages` 分支，站点更新。

---

## 5. 数据模型（Frontmatter 定义）

文章元数据直接定义在 Markdown 文件的头部（YAML Frontmatter）。

### 5.1 Post Frontmatter

```yaml
---
title: "我的第一篇博客"
date: "2023-10-01"
summary: "这是文章摘要..."
tags: ["Next.js", "SSG"]
category: "技术"
cover: "/images/cover.jpg"
draft: false
---
```

### 5.2 Site Config

全局配置，存放于代码常量中。

- siteUrl
- title
- description
- author
- socialLinks

## 6. 渐进式开发路线图

## 阶段 0：项目初始化

### 目标

搭建支持 SSG 的 Next.js 工程，完成基础目录与配置。

### 任务

- 初始化 Next.js + TypeScript + Tailwind 项目
- 配置 `next.config.js` 开启 `output: 'export'`
- 配置 ESLint/Prettier
- 建立 `content/`、`src/` 基础目录

### 验收标准

- `npm run build` 可生成静态 HTML 文件（out 目录）
- 无水合错误（Hydration Error）

---

## 阶段 1：内容解析与列表展示

### 目标

实现从文件系统读取 Markdown 并展示列表。

### 任务

- 引入 Markdown 解析库（推荐 Contentlayer 或 gray-matter）
- 定义 Post 数据结构与类型
- 实现首页文章列表（按时间倒序）
- 实现分页逻辑（静态分页）

### 验收标准

- 新增 `.md` 文件后，重启/热更新可显示在列表
- 列表包含标题、摘要、日期、标签

---

## 阶段 2：文章详情与渲染

### 目标

实现文章详情页的高质量渲染。

### 任务

- 实现动态路由 `posts/[slug]`
- 配置 Markdown 渲染器（支持 GitHub Flavored Markdown）
- 增加代码高亮（Prism.js / Shiki）
- 增加文章目录（Table of Contents）

### 验收标准

- 文章页 URL 友好（如 `/posts/hello-world`）
- 代码块高亮正常，图片自适应

---

## 阶段 3：分类、标签与搜索

### 目标

增强内容的可发现性。

### 任务

- 生成分类/标签聚合页
- 实现本地搜索（基于 Fuse.js 的前端搜索，无后端）
- 增加“上一篇/下一篇”导航

### 验收标准

- 点击标签可查看相关所有文章
- 搜索框可实时过滤文章标题

---

## 阶段 4：评论与部署

### 目标

完成社区交互与自动化上线。

### 任务

- 集成 Giscus 评论系统
- 配置 `sitemap.xml` 与 `robots.txt` 生成
- 编写 GitHub Actions 部署脚本
- 绑定自定义域名（可选）

### 验收标准

- 推送代码后，5分钟内站点自动更新
- 评论功能可用

---

## 8. 关键技术点备忘

- **图片处理**：SSG 模式下 `next/image` 的优化功能受限（需配置 unoptimized 或使用第三方加载器），建议优先使用标准 `<img>` 或配置 Cloudinary。
- **动态路由**：必须使用 `generateStaticParams` 预定义所有路径。

---

## 9. 风险与应对

- 风险：需求扩散导致延期  
  应对：严格以阶段目标验收，阶段内不扩 scope

- 风险：SEO 表现不稳定  
  应对：首期优先 SSR/静态化策略与规范化元数据

- 风险：后台安全薄弱  
  应对：最小权限、鉴权中间件、管理路由保护

---

## 10. 第一轮执行建议（现在就做）

按以下顺序推进最稳妥：

1. 完成阶段 0（项目初始化与规范）
2. 完成阶段 1（前台只读展示）
3. 完成阶段 2（后台管理闭环）

达到阶段 2 后，即得到可上线的 MVP 版本。

---

## 11. 里程碑定义

- M1：工程可运行，目录与规范完成
- M2：前台可浏览文章（只读）
- M3：后台可发布文章（MVP）
- M4：阅读体验与搜索增强
- M5：质量保障与自动化完善
