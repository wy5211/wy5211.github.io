## 前言

在上一篇文章中，我们创建了一个简单的 Todo 数据模型。但在实际应用中，我们需要处理更复杂的数据结构：用户关系、权限管理、状态枚举、时间追踪等等。

Prisma Schema 是定义这些数据模型的核心文件，它不仅描述了数据库结构，还决定了 Prisma Client 生成的类型。掌握 Schema 的语法，是构建类型安全应用的基础。

## Schema 文件结构

让我们先看一个完整的 Schema 文件示例：

```prisma
// prisma/schema.prisma

// 生成器配置
generator client {
  provider        = "prisma-client-js"
  previewFeatures = ["postgresqlExtensions"]
  binaryTargets   = ["native", "linux-musl-openssl-3.0.x"]
}

// 数据源配置
datasource db {
  provider   = "postgresql"
  url        = env("DATABASE_URL")
  extensions = [pg_trgm(prefix: "")]
}

// 模型定义
model User {
  id        String   @id @default(uuid())
  email     String   @unique
  name      String?
  role      Role     @default(USER)
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt
}

// 枚举定义
enum Role {
  USER
  ADMIN
  MODERATOR
}
```

### Generator 部分

控制 Prisma Client 的生成方式：

```prisma
generator client {
  provider           = "prisma-client-js"  // 生成 JavaScript/TypeScript 客户端
  output             = "../src/generated/client"  // 自定义输出路径
  previewFeatures    = ["fullTextSearch", "fullTextIndex"]  // 启用预览功能
  binaryTargets      = ["native", "linux-musl-openssl-3.0.x"]  // 目标平台
}
```

**常用 previewFeatures**：

- `fullTextSearch`：全文搜索
- `fullTextIndex`：全文索引
- `postgresqlExtensions`：PostgreSQL 扩展
- `mongoDb`：MongoDB 支持

### Datasource 部分

配置数据库连接：

```prisma
datasource db {
  provider = "postgresql"  // postgresql | mysql | sqlite | sqlserver | mongodb
  url      = env("DATABASE_URL")  // 从环境变量读取
  provider = "postgresql"
  url      = env("DATABASE_URL")
  schema   = "my_schema"  // 可选：指定 schema（PostgreSQL）
  // PostgreSQL 扩展
  extensions = [pg_trgm(prefix: "")]
}
```

## 字段类型详解

Prisma Schema 支持丰富的字段类型，涵盖了各种数据库的原生类型。

### 标准类型

```prisma
model StandardTypes {
  // 字符串类型
  id      String   @id
  title   String
  slug    String   @unique
  content String?  // 可选字段

  // 整数类型
  count   Int
  rating  Int      @default(0)

  // 浮点数类型
  price   Float
  score   Float    @default(0.0)

  // 布尔类型
  active  Boolean  @default(true)
  deleted Boolean  @default(false)

  // 日期时间类型
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt
  publishAt DateTime?
  expiresAt DateTime?

  // JSON 类型（PostgreSQL、MySQL）
  metadata Json?
  settings Json      @default("{}")

  // Bytes 类型（二进制数据）
  file     Bytes?
}
```

### 数据库特定类型

```prisma
model DatabaseSpecific {
  // PostgreSQL 特定类型
  ipAddress  Unsupported("inet")?
  macAddress Unsupported("macaddr")?
  xmlData    Unsupported("xml")?
  uuid       String   @default(uuid())

  // MySQL 特定类型
  tinyInt    Int      @db.TinyInt
  bigInt     Int      @db.BigInt
  decimal    Decimal  @db.Decimal(10, 2)

  // SQLite 特定类型
  text       String   @db.Text

  // 时间类型（PostgreSQL）
  time       DateTime @db.Time(8)
  timestamp  DateTime @db.Timestamp(8)
}
```

### Decimal 类型（精确数值）

```prisma
model Product {
  id    String  @id
  name  String
  price Decimal @db.Decimal(10, 2)  // 总共 10 位，小数点后 2 位
  // 例如：12345678.99
}
```

## 约束与默认值

### 主键（Primary Key）

```prisma
// 单字段主键
model User {
  id String @id
}

// 复合主键
model UserRole {
  userId    String
  roleId    String

  @@id([userId, roleId])
}

// 自增主键
model Post {
  id Int @id @default(autoincrement())
}

// UUID 主键
model Comment {
  id String @id @default(uuid())
}
```

### 唯一约束

```prisma
model User {
  id    String @id
  email String @unique  // 单字段唯一
  username String @unique
}

// 复合唯一约束
model Enrollment {
  courseId String
  studentId String

  @@unique([courseId, studentId])
}
```

### 默认值

```prisma
model Post {
  // UUID 默认值
  id        String   @id @default(uuid())

  // 自增默认值
  sequence  Int      @default(autoincrement())

  // 布尔默认值
  published Boolean  @default(false)

  // 时间默认值
  createdAt DateTime @default(now())

  // 随机默认值
  token     String   @default(cuid())

  // 常量字符串
  status    String   @default("DRAFT")

  // 数值计算
  order     Int      @default(0)
}
```

### 可选字段

```prisma
model User {
  id       String  @id
  email    String  // 必填
  name     String? // 可选（可为 null）
  bio      String? @db.Text
  avatar   String?
  birthday DateTime?
}
```

## 索引策略

合理的索引设计能大幅提升查询性能。

### 单字段索引

```prisma
model User {
  id    String @id
  email String @unique  // 唯一索引
  name  String

  @@index([email])
  @@index([name])
}
```

### 复合索引

```prisma
model Post {
  id        String   @id
  authorId  String
  status    String
  createdAt DateTime

  // 复合索引用于常见查询组合
  @@index([authorId, createdAt])
  @@index([status, publishedAt])
  @@index([status, createdAt])
}
```

### 多字段索引

```prisma
model Article {
  id        String   @id
  title     String
  category  String
  published Boolean
  createdAt DateTime

  // 三字段索引
  @@index([category, published, createdAt(sort: Desc)])
}
```

### 全文索引（PostgreSQL）

```prisma
model Post {
  id      String @id
  title   String
  content String @db.Text

  @@fulltext([title, content])
}
```

### 索引类型（PostgreSQL）

```prisma
model User {
  id    String @id
  email String @unique
  name  String

  // Hash 索引
  @@index([email], type: Hash)

  // BTree 索引（默认）
  @@index([name], type: BTree)

  // GiST 索引
  @@index([location], type: GiST)
}
```

### 命名索引

```prisma
model User {
  id    String @id
  email String
  name  String

  @@index([email], name: "user_email_index")
  @@index([name], map: "user_name_idx")
}
```

## 枚举类型

枚举让你的数据模型更清晰、更安全。

### 定义枚举

```prisma
enum UserRole {
  USER
  ADMIN
  MODERATOR
}

enum PostStatus {
  DRAFT
  PUBLISHED
  ARCHIVED
}

enum OrderStatus {
  PENDING
  PROCESSING
  SHIPPED
  DELIVERED
  CANCELLED
}
```

### 使用枚举

```prisma
model User {
  id    String    @id
  email String
  role  UserRole  @default(USER)
}

model Post {
  id     String     @id
  title  String
  status PostStatus @default(DRAFT)
}

// 在查询中使用
const admins = await prisma.user.findMany({
  where: {
    role: 'ADMIN', // TypeScript 自动补全
  },
})
```

### 枚举数据库映射

```prisma
enum Role {
  USER       @map("user_role")
  ADMIN      @map("admin_role")
  MODERATOR  @map("mod_role")
}

model User {
  id   String @id
  role Role
}

// 在数据库中，role 字段存储为 'user_role'、'admin_role' 等
```

## 时间戳管理

### 自动时间戳

```prisma
model Post {
  id        String   @id
  title     String

  // 创建时自动设置
  createdAt DateTime @default(now())

  // 更新时自动更新
  updatedAt DateTime @updatedAt

  // 创建和更新时都自动更新
  updatedAt DateTime @updatedAt
}
```

### 自定义时间戳

```prisma
model User {
  id           String    @id
  email        String
  lastLoginAt  DateTime?
  deletedAt    DateTime? // 软删除

  // 手动设置
  createdAt    DateTime  @default(now())
}
```

### 时间戳查询

```typescript
// 查询最近创建的记录
const recentPosts = await prisma.post.findMany({
  where: {
    createdAt: {
      gte: new Date(Date.now() - 7 * 24 * 60 * 60 * 1000), // 最近 7 天
    },
  },
  orderBy: {
    createdAt: "desc",
  },
});

// 查询软删除的记录
const deletedUsers = await prisma.user.findMany({
  where: {
    deletedAt: {
      not: null, // 或使用 `isSet: true`
    },
  },
});
```

## 完整示例：博客系统 Schema

让我们通过一个完整的博客系统示例，综合运用所有学到的知识：

```prisma
// prisma/schema.prisma

generator client {
  provider        = "prisma-client-js"
  previewFeatures = ["postgresqlExtensions", "fullTextSearch"]
}

datasource db {
  provider   = "postgresql"
  url        = env("DATABASE_URL")
  extensions = [pg_trgm(prefix: "")]
}

// 枚举定义
enum UserRole {
  USER
  ADMIN
  MODERATOR
}

enum PostStatus {
  DRAFT
  PUBLISHED
  ARCHIVED
}

enum CommentStatus {
  PENDING
  APPROVED
  SPAM
  DELETED
}

// 用户模型
model User {
  id        String    @id @default(uuid())
  email     String    @unique
  username  String    @unique
  password  String    // 应该存储哈希值
  role      UserRole  @default(USER)
  avatar    String?
  bio       String?   @db.Text
  website   String?
  location  String?
  isActive  Boolean   @default(true)
  lastLoginAt DateTime?
  createdAt DateTime  @default(now())
  updatedAt DateTime  @updatedAt
  deletedAt DateTime? // 软删除

  // 关系
  posts    Post[]
  comments Comment[]
  likes    Like[]

  // 索引
  @@index([email])
  @@index([username])
  @@index([isActive, createdAt])
  @@map("users")
}

// 文章模型
model Post {
  id          String     @id @default(uuid())
  title       String
  slug        String     @unique
  content     String     @db.Text
  excerpt     String?    @db.Text
  coverImage  String?
  status      PostStatus @default(DRAFT)
  viewCount   Int        @default(0)
  likeCount   Int        @default(0)
  commentCount Int       @default(0)

  // SEO
  metaTitle   String?
  metaDescription String?
  keywords    String[]

  // 作者
  authorId    String
  author      User       @relation(fields: [authorId], references: [id], onDelete: Cascade)

  // 时间
  publishedAt DateTime?
  createdAt   DateTime   @default(now())
  updatedAt   DateTime   @updatedAt
  deletedAt   DateTime?

  // 关系
  comments    Comment[]
  likes       Like[]
  tags        PostTag[]

  // 索引
  @@index([authorId])
  @@index([status, publishedAt])
  @@index([slug])
  @@index([createdAt(sort: Desc)])
  @@fulltext([title, content])
  @@map("posts")
}

// 评论模型
model Comment {
  id        String        @id @default(uuid())
  content   String        @db.Text
  status    CommentStatus @default(PENDING)

  // 关系
  authorId  String
  author    User          @relation(fields: [authorId], references: [id], onDelete: Cascade)
  postId    String
  post      Post          @relation(fields: [postId], references: [id], onDelete: Cascade)
  parentId  String?       // 嵌套评论
  parent    Comment?      @relation("CommentReplies", fields: [parentId], references: [id])
  replies   Comment[]     @relation("CommentReplies")

  // 时间
  createdAt DateTime      @default(now())
  updatedAt DateTime      @updatedAt
  deletedAt DateTime?

  // 索引
  @@index([postId])
  @@index([authorId])
  @@index([parentId])
  @@index([status, createdAt])
  @@map("comments")
}

// 点赞模型
model Like {
  id        String   @id @default(uuid())
  userId    String
  user      User     @relation(fields: [userId], references: [id], onDelete: Cascade)
  postId    String
  post      Post     @relation(fields: [postId], references: [id], onDelete: Cascade)
  createdAt DateTime @default(now())

  @@unique([userId, postId])
  @@index([postId])
  @@map("likes")
}

// 标签模型
model Tag {
  id        String   @id @default(uuid())
  name      String   @unique
  slug      String   @unique
  color     String?  // 十六进制颜色
  createdAt DateTime @default(now())

  // 关系
  posts     PostTag[]

  @@index([slug])
  @@map("tags")
}

// 文章-标签关联表（多对多）
model PostTag {
  postId   String
  tagId    String

  post     Post @relation(fields: [postId], references: [id], onDelete: Cascade)
  tag      Tag  @relation(fields: [tagId], references: [id], onDelete: Cascade)

  @@id([postId, tagId])
  @@map("post_tags")
}
```

## Schema 最佳实践

### 1. 使用一致的命名约定

```prisma
// ✅ 好的做法
model UserProfile {
  id        String   @id
  firstName String
  lastName  String
  createdAt DateTime @default(now())
}

// ❌ 避免
model user_profile {
  id        String   @id
  first_name String
  last_name  String
  created_at DateTime @default(now())
}
```

### 2. 合理使用索引

```prisma
model Post {
  id        String   @id
  title     String
  authorId  String
  status    String
  createdAt DateTime

  // ✅ 为常用查询创建索引
  @@index([authorId, createdAt])
  @@index([status, publishedAt])

  // ❌ 不要为所有字段都创建索引（影响写入性能）
  // @@index([title])
}
```

### 3. 使用软删除

```prisma
model User {
  id        String   @id
  email     String
  deletedAt DateTime? // 软删除标记

  @@index([deletedAt])
}

// 查询时自动过滤
const users = await prisma.user.findMany({
  where: {
    deletedAt: null,
  },
})
```

### 4. 使用枚举提高类型安全

```prisma
// ✅ 使用枚举
enum OrderStatus {
  PENDING
  SHIPPED
  DELIVERED
}

model Order {
  status OrderStatus
}

// ❌ 避免使用字符串
model Order {
  status String // "PENDING" | "SHIPPED" | "DELIVERED"
}
```

### 5. 合理设置默认值

```prisma
model Post {
  id        String   @id
  title     String
  status    String   @default("DRAFT")
  published Boolean  @default(false)
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt
}
```

## 属性完整参考

### 字段属性

| 属性          | 描述           | 示例                                  |
| ------------- | -------------- | ------------------------------------- |
| `@id`         | 主键           | `id String @id`                       |
| `@default()`  | 默认值         | `createdAt DateTime @default(now())`  |
| `@unique`     | 唯一约束       | `email String @unique`                |
| `@relation()` | 关系定义       | `author User @relation(...)`          |
| `@map()`      | 字段名映射     | `firstName String @map("first_name")` |
| `@updatedAt`  | 自动更新时间戳 | `updatedAt DateTime @updatedAt`       |
| `@db.`        | 数据库类型     | `data String @db.Text`                |

### 模型属性

| 属性           | 描述         | 示例                           |
| -------------- | ------------ | ------------------------------ |
| `@@id()`       | 复合主键     | `@@id([userId, postId])`       |
| `@@unique()`   | 复合唯一约束 | `@@unique([email, username])`  |
| `@@index()`    | 索引         | `@@index([email])`             |
| `@@fulltext()` | 全文索引     | `@@fulltext([title, content])` |
| `@@map()`      | 表名映射     | `@@map("users")`               |

## 常见问题

**Q: 如何修改现有字段类型？**

A: 使用迁移命令：

```bash
npx prisma migrate dev --name change_field_type
```

**Q: 如何重命名表或字段？**

A: 在 Schema 中使用 `@map` 或 `@@map`：

```prisma
model User {
  id   String @id
  name String @map("user_name")
  @@map("users")
}
```

**Q: 枚举可以添加新值吗？**

A: 可以，但需要创建迁移。删除或重命名枚举值需要手动处理迁移文件。

**Q: 如何设置字段长度？**

A: 使用数据库特定类型：

```prisma
model Post {
  title String @db.VarChar(200)
}
```
