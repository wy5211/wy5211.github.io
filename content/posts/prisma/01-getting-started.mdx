## 前言

在现代 Web 开发中，与数据库的交互是必不可少的一环。传统的 SQL 查询虽然强大，但在处理复杂业务逻辑时容易出现类型不安全、SQL 注入风险、以及代码维护困难等问题。

Prisma 作为新一代的 TypeScript-first ORM，通过自动类型生成、声明式 Schema 和强大的工具链，让数据库操作变得既安全又高效。

**为什么选择 Prisma？**

- 🔥 **端到端类型安全**：从数据库到代码，全程类型保护
- 🎯 **直观的 Schema**：用类似 TypeScript 的语法定义数据模型
- 🚀 **开发体验极佳**：自动补全、重构支持、错误提示
- 📊 **可视化工具**：Prisma Studio 让数据库管理变得简单
- 🔌 **多数据库支持**：PostgreSQL、MySQL、SQLite、MongoDB 等

## Prisma 核心概念

在开始编码前，我们需要理解 Prisma 的三大核心组件：

### 1. Prisma Schema

`schema.prisma` 文件是整个数据模型的定义中心，使用声明式语法描述你的数据库结构：

```prisma
model User {
  id    String @id @default(uuid())
  email String @unique
  name  String?
}
```

### 2. Prisma Client

根据 Schema 自动生成的类型安全客户端，提供完整的 TypeScript 类型推断：

```typescript
const user = await prisma.user.create({
  data: {
    email: "user@example.com",
    name: "John Doe",
  },
});
// user 的类型自动推断为 User
```

### 3. Prisma Migrate

数据库迁移工具，管理数据库结构的版本控制和演进：

```bash
npx prisma migrate dev --name init
```

## 环境搭建

### 第一步：创建项目

让我们创建一个新的项目目录并初始化：

```bash
# 创建项目目录
mkdir prisma-todo-app
cd prisma-todo-app

# 初始化 package.json
npm init -y

# 安装 Prisma CLI
npm install prisma --save-dev

# 初始化 Prisma
npx prisma init
```

执行 `prisma init` 后，会生成以下文件结构：

```
prisma-todo-app/
├── prisma/
│   └── schema.prisma    # Prisma Schema 文件
├── .env                 # 环境变量文件
└── package.json
```

### 第二步：配置数据库连接

编辑 `.env` 文件，设置数据库连接字符串。本教程使用 PostgreSQL：

```env
# .env
# 使用本地 PostgreSQL
DATABASE_URL="postgresql://postgres:password@localhost:5432/todo_db?schema=public"

# 或者使用 Supabase（推荐新手）
# DATABASE_URL="postgresql://postgres:[YOUR-PASSWORD]@db.[YOUR-PROJECT-REF].supabase.co:5432/postgres"

# 或者使用 SQLite（最简单，无需安装数据库）
# DATABASE_URL="file:./dev.db"
```

> 💡 **提示**：如果你不想安装数据库，可以直接使用 SQLite，只需将连接字符串改为 `file:./dev.db` 即可。

### 第三步：定义数据模型

编辑 `prisma/schema.prisma` 文件，定义我们的 Todo 数据模型：

```prisma
// prisma/schema.prisma

generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

model Todo {
  id        String   @id @default(uuid())
  title     String
  content   String?
  completed Boolean  @default(false)
  priority  Int      @default(0)
  dueDate   DateTime?
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt
}
```

**Schema 语法解释**：

- `@id`：定义主键
- `@default(uuid())`：默认值使用 UUID
- `@default(now())`：默认使用当前时间戳
- `@updatedAt`：自动更新时间戳
- `String?`：可选字段（可为 null）
- `@default(false)`：布尔值默认为 false

## 第一个 Prisma 应用

### 第一步：创建数据库迁移

在终端运行以下命令：

```bash
npx prisma migrate dev --name init
```

这个命令会：

1. 创建数据库（如果不存在）
2. 根据 Schema 创建迁移文件
3. 执行迁移，创建表结构
4. 生成 Prisma Client

输出示例：

```

Datasource "db": PostgreSQL database "todo_db" at "localhost:5432"

Applying migration `20240321000000_init`

The following migration(s) have been created and applied from new schema changes:

migrations/
  └─ 20240321000000_init/
    └─ migration.sql

Applying migration `20240321000000_init`

Generated Prisma Client (5.0.0) to ./node_modules/.prisma/client in 1s
```

### 第二步：安装 @prisma/client

```bash
npm install @prisma/client
```

### 第三步：创建操作脚本

创建 `src/index.ts` 文件：

```typescript
// src/index.ts
import { PrismaClient } from "@prisma/client";

const prisma = new PrismaClient();

async function main() {
  console.log("开始操作数据库...\n");

  // 1. 创建 Todo
  console.log("1. 创建 Todo");
  const todo = await prisma.todo.create({
    data: {
      title: "学习 Prisma",
      content: "完成 Prisma 快速入门教程",
      priority: 1,
      dueDate: new Date("2026-03-25"),
    },
  });
  console.log("创建成功:", todo);
  console.log();

  // 2. 查询单个 Todo
  console.log("2. 查询 Todo");
  const foundTodo = await prisma.todo.findUnique({
    where: { id: todo.id },
  });
  console.log("查询结果:", foundTodo);
  console.log();

  // 3. 创建多个 Todos
  console.log("3. 批量创建");
  await prisma.todo.createMany({
    data: [
      {
        title: "完成项目文档",
        content: "编写项目的 README 和 API 文档",
        priority: 2,
      },
      {
        title: "代码审查",
        content: "审查团队成员的 Pull Request",
        priority: 3,
        completed: true,
      },
      {
        title: "准备演示",
        content: "准备周五的项目演示",
        priority: 1,
        dueDate: new Date("2026-03-24"),
      },
    ],
  });
  console.log("批量创建完成");
  console.log();

  // 4. 查询所有 Todos
  console.log("4. 查询所有 Todos");
  const allTodos = await prisma.todo.findMany({
    orderBy: [{ priority: "desc" }, { createdAt: "desc" }],
  });
  console.log(`找到 ${allTodos.length} 个 Todos:`);
  allTodos.forEach((todo) => {
    console.log(
      `  - [${todo.completed ? "✓" : " "}] ${todo.title} (优先级: ${todo.priority})`
    );
  });
  console.log();

  // 5. 查询未完成的 Todos
  console.log("5. 查询未完成的 Todos");
  const incompleteTodos = await prisma.todo.findMany({
    where: { completed: false },
  });
  console.log(`未完成: ${incompleteTodos.length} 个`);
  console.log();

  // 6. 更新 Todo
  console.log("6. 更新 Todo");
  const updatedTodo = await prisma.todo.update({
    where: { id: todo.id },
    data: {
      completed: true,
    },
  });
  console.log("更新后:", updatedTodo);
  console.log();

  // 7. 统计数据
  console.log("7. 统计数据");
  const stats = await prisma.todo.aggregate({
    _count: { id: true },
    _where: { completed: false },
  });
  console.log(`总计: ${stats._count.id} 个 Todos`);
  console.log();

  // 8. 删除 Todo
  console.log("8. 删除已完成的 Todo");
  const deleteResult = await prisma.todo.deleteMany({
    where: { completed: true },
  });
  console.log(`删除了 ${deleteResult.count} 个 Todos`);
}

main()
  .catch((e) => {
    console.error("错误:", e);
    process.exit(1);
  })
  .finally(async () => {
    // 关闭数据库连接
    await prisma.$disconnect();
  });
```

## 使用 Prisma Studio

Prisma Studio 是一个可视化的数据库管理工具，让你可以像使用电子表格一样查看和编辑数据。

启动 Prisma Studio：

```bash
npx prisma studio
```

浏览器会自动打开 `http://localhost:5555`，你将看到：

- 所有表的列表
- 记录的详细信息
- 可以直接添加、编辑、删除记录
- 支持筛选和排序

## 最佳实践

### 1. 使用单例模式

为了避免创建多个数据库连接实例，推荐使用单例模式：

```typescript
// lib/prisma.ts
import { PrismaClient } from "@prisma/client";

const globalForPrisma = globalThis as unknown as {
  prisma: PrismaClient | undefined;
};

export const prisma = globalForPrisma.prisma ?? new PrismaClient();

if (process.env.NODE_ENV !== "production") globalForPrisma.prisma = prisma;
```

### 2. 始终断开连接

在脚本结束时，记得调用 `prisma.$disconnect()`：

```typescript
process.on("beforeExit", async () => {
  await prisma.$disconnect();
});
```

### 3. 启用查询日志（开发环境）

```typescript
const prisma = new PrismaClient({
  log: ["query", "info", "warn", "error"],
});
```

### 4. 使用环境变量

永远不要在代码中硬编码敏感信息：

```env
# .env
DATABASE_URL="postgresql://..."
```

```typescript
// prisma/schema.prisma
datasource db {
  url = env("DATABASE_URL")
}
```

## 常见问题

**Q: Prisma 支持哪些数据库？**

A: Prisma 支持 PostgreSQL、MySQL、SQLite、SQL Server 和 MongoDB。

**Q: 如何切换数据库？**

A: 只需修改 `schema.prisma` 中的 `provider` 和 `.env` 中的 `DATABASE_URL`，然后重新运行迁移。

**Q: Prisma 免费吗？**

A: Prisma 开源版完全免费，还有付费的 Prisma Cloud 提供额外功能。

**Q: 可以在现有数据库上使用 Prisma 吗？**

A: 可以！使用 `npx prisma db pull` 从现有数据库生成 Schema。
