## WHERE 条件详解

### 字段比较运算符

Prisma 提供了丰富的字段比较运算符：

```typescript
// 数值比较
const posts = await prisma.post.findMany({
  where: {
    views: {
      equals: 100, // 等于
      not: 50, // 不等于
      gt: 100, // 大于 (greater than)
      gte: 100, // 大于等于 (greater than or equal)
      lt: 1000, // 小于 (less than)
      lte: 1000, // 小于等于 (less than or equal)
    },
  },
});

// 日期比较
const posts = await prisma.post.findMany({
  where: {
    createdAt: {
      gte: new Date("2024-01-01"),
      lt: new Date("2025-01-01"),
    },
    publishedAt: {
      not: null, // 不为 null
    },
  },
});

// 字符串比较
const users = await prisma.user.findMany({
  where: {
    username: {
      equals: "john", // 完全匹配
      in: ["john", "jane"], // 在数组中
      notIn: ["admin"], // 不在数组中
    },
  },
});
```

### 字符串搜索运算符

```typescript
// PostgreSQL 的模式匹配（默认不区分大小写）
const users = await prisma.user.findMany({
  where: {
    username: {
      contains: "john", // 包含
      startsWith: "john", // 以...开头
      endsWith: "doe", // 以...结尾
    },
  },
});

// 显式指定区分大小写
const users = await prisma.user.findMany({
  where: {
    username: {
      contains: "John",
      mode: "insensitive", // insensitive | sensitive | default
    },
  },
});

// 数组字段查询
const posts = await prisma.post.findMany({
  where: {
    tags: {
      has: "JavaScript", // 数组包含某个值
      hasSome: ["JS", "TS"], // 数组包含任意一个
      hasEvery: ["JS", "TS"], // 数组包含所有值
      isEmpty: true, // 数组为空
    },
  },
});
```

### JSON 字段查询（PostgreSQL）

```typescript
const users = await prisma.user.findMany({

    metadata: {
      path: ["role"], // JSON 路径
```

## 逻辑运算符

### AND 运算符

所有条件都必须满足：

```typescript
const posts = await prisma.post.findMany({
  where: {
    AND: [
      { status: "PUBLISHED" },
      { publishedAt: { lte: new Date() } },
      { views: { gt: 100 } },
    ],
  },
});

// 简化写法（AND 可以省略）
const posts = await prisma.post.findMany({
  where: {
    status: "PUBLISHED",
    publishedAt: { lte: new Date() },
    views: { gt: 100 },
  },
});
```

### OR 运算符

任一条件满足即可：

```typescript
const posts = await prisma.post.findMany({
  where: {
    OR: [
      { title: { contains: "Prisma" } },
      { content: { contains: "Prisma" } },
      { tags: { has: "Prisma" } },
    ],
  },
});

// 结合 AND 使用
const posts = await prisma.post.findMany({
  where: {
    AND: [
      { status: "PUBLISHED" },
      {
        OR: [
          { title: { contains: "Prisma" } },
          { content: { contains: "Prisma" } },
        ],
      },
    ],
  },
});
```

### NOT 运算符

条件取反：

```typescript
const posts = await prisma.post.findMany({
  where: {
    NOT: {
      status: "ARCHIVED",
    },
  },
});

// 复杂 NOT
const posts = await prisma.post.findMany({
  where: {
    NOT: [{ status: "DRAFT" }, { deletedAt: { not: null } }],
  },
});
```

### 组合逻辑运算符

```typescript
const posts = await prisma.post.findMany({
  where: {
    AND: [
      { status: "PUBLISHED" },
      {
        OR: [
          { title: { contains: "TypeScript" } },
          {
            AND: [{ tags: { has: "JavaScript" } }, { views: { gt: 1000 } }],
          },
        ],
      },
      {
        NOT: {
          author: {
            email: { endsWith: "@spam.com" },
          },
        },
      },
    ],
  },
});
```

## 排序和分页

### 排序（orderBy）

```typescript
// 单字段排序
const posts = await prisma.post.findMany({
  orderBy: {
    createdAt: "desc", // 'asc' | 'desc'
  },
});

// 多字段排序
const posts = await prisma.post.findMany({
  orderBy: [
    { featured: "desc" }, // 先按特色排序
    { createdAt: "desc" }, // 再按创建时间排序
  ],
});

// 关系字段排序
const users = await prisma.user.findMany({
  orderBy: {
    posts: {
      _count: "desc",
    },
  },
});

// NULL 值排序（PostgreSQL）
const posts = await prisma.post.findMany({
  orderBy: {
    publishedAt: [
      { sort: "asc" },
      { nulls: "last" }, // 'first' | 'last'
    ],
  },
});
```

### 偏移分页（Offset-based）

```typescript
async function getPosts(page: number, limit: number) {
  const skip = (page - 1) * limit;

  const [posts, total] = await Promise.all([
    prisma.post.findMany({
      skip,
      take: limit,
      orderBy: { createdAt: "desc" },
    }),
    prisma.post.count(),
  ]);

  return {
    data: posts,
    meta: {
      page,
      limit,
      total,
      totalPages: Math.ceil(total / limit),
    },
  };
}

// 使用
const result = await getPosts(2, 10); // 第 2 页，每页 10 条
```

### 游标分页（Cursor-based）

更适合大数据量和不稳定数据集：

```typescript
async function getPosts(cursor: string | null, limit: number) {
  const posts = await prisma.post.findMany({
    take: limit,
    skip: cursor ? 1 : 0, // 跳过游标本身
    cursor: cursor ? { id: cursor } : undefined,
    orderBy: { createdAt: "desc" },
  });

  return {
    data: posts,
    nextCursor: posts.length === limit ? posts[posts.length - 1].id : null,
  };
}

// 使用
let cursor = null;
const page1 = await getPosts(cursor, 10);
cursor = page1.nextCursor;

const page2 = await getPosts(cursor, 10);
```

### 游标分页优势

```typescript
// ✅ 游标分页：性能稳定
const posts = await prisma.post.findMany({
  take: 10,
  cursor: { id: lastPostId },
  orderBy: { createdAt: "desc" },
});
// SQL: SELECT * FROM posts WHERE id > ? ORDER BY created_at DESC LIMIT 10

// ❌ 偏移分页：大数据量性能差
const posts = await prisma.post.findMany({
  take: 10,
  skip: 10000,
  orderBy: { createdAt: "desc" },
});
// SQL: SELECT * FROM posts ORDER BY created_at DESC LIMIT 10 OFFSET 10000
```

### 查询缓存

```typescript
// 简单的内存缓存实现
const cache = new Map<string, { data: any; expiry: number }>();

async function cachedQuery<T>(
  key: string,
  query: () => Promise<T>,
  ttl: number = 60000
): Promise<T> {
  const cached = cache.get(key);

  if (cached && cached.expiry > Date.now()) {
    return cached.data as T;
  }

  const data = await query();

  cache.set(key, {
    data,
    expiry: Date.now() + ttl,
  });

  return data;
}

// 使用
const posts = await cachedQuery(
  "posts:published",
  () => prisma.post.findMany({ where: { status: "PUBLISHED" } }),
  60000 // 缓存 1 分钟
);
```

## 原始 SQL 的使用场景

虽然 Prisma 的查询 API 很强大，但某些场景下原始 SQL 更合适。

### 复杂聚合查询

```typescript
// 统计每个月的文章数量
const stats = await prisma.$queryRaw<Array<{ month: string; count: bigint }>>`
  SELECT
    TO_CHAR(created_at, 'YYYY-MM') as month,
    COUNT(*) as count
  FROM posts
  WHERE status = 'PUBLISHED'
  GROUP BY TO_CHAR(created_at, 'YYYY-MM')
  ORDER BY month DESC
`;
```

### 窗口函数

```typescript
// 获取每个作者的最新文章
const latestPosts = await prisma.$queryRaw`
  SELECT DISTINCT ON (author_id)
    id,
    title,
    author_id,
    created_at
  FROM posts
  ORDER BY author_id, created_at DESC
`;
```

### CTE（Common Table Expressions）

```typescript
// 递归查询（评论树）
const commentTree = await prisma.$queryRaw`
  WITH RECURSIVE comment_tree AS (
    SELECT id, content, parent_id, 0 as level
    FROM comments
    WHERE parent_id IS NULL
    AND post_id = ${postId}

    UNION ALL

    SELECT c.id, c.content, c.parent_id, ct.level + 1
    FROM comments c
    JOIN comment_tree ct ON c.parent_id = ct.id
  )
  SELECT * FROM comment_tree
  ORDER BY level, id
`;
```

### 性能关键查询

```typescript
// 复杂的 JOIN 查询
const results = await prisma.$queryRaw`
  SELECT
    p.id,
    p.title,
    u.username as author_name,
    COUNT(c.id) as comment_count
  FROM posts p
  JOIN users u ON p.author_id = u.id
  LEFT JOIN comments c ON c.post_id = p.id
  WHERE p.status = 'PUBLISHED'
  GROUP BY p.id, u.username
  HAVING COUNT(c.id) > 10
  ORDER BY comment_count DESC
  LIMIT 20
`;
```

## 完整示例：搜索和筛选 API

```typescript
import { PrismaClient } from "@prisma/client";

const prisma = new PrismaClient();

interface SearchParams {
  keyword?: string;
  authorId?: string;
  tags?: string[];
  status?: string;
  dateFrom?: string;
  dateTo?: string;
  sortBy?: string;
  sortOrder?: "asc" | "desc";
  page?: number;
  limit?: number;
}

export async function searchPosts(params: SearchParams) {
  const {
    keyword,
    authorId,
    tags,
    status,
    dateFrom,
    dateTo,
    sortBy = "createdAt",
    sortOrder = "desc",
    page = 1,
    limit = 10,
  } = params;

  // 构建 WHERE 条件
  const where: any = {};

  // 关键词搜索
  if (keyword) {
    where.OR = [
      { title: { contains: keyword, mode: "insensitive" } },
      { content: { contains: keyword, mode: "insensitive" } },
      { excerpt: { contains: keyword, mode: "insensitive" } },
    ];
  }

  // 作者筛选
  if (authorId) {
    where.authorId = authorId;
  }

  // 标签筛选
  if (tags && tags.length > 0) {
    where.tags = {
      some: {
        id: { in: tags },
      },
    };
  }

  // 状态筛选
  if (status) {
    where.status = status;
  }

  // 日期范围筛选
  if (dateFrom || dateTo) {
    where.publishedAt = {};
    if (dateFrom) {
      where.publishedAt.gte = new Date(dateFrom);
    }
    if (dateTo) {
      where.publishedAt.lte = new Date(dateTo);
    }
  }

  // 排序
  const orderBy: any = {};
  orderBy[sortBy] = sortOrder;

  // 分页
  const skip = (page - 1) * limit;

  // 并行查询数据和总数
  const [posts, total] = await Promise.all([
    prisma.post.findMany({
      where,
      skip,
      take: limit,
      orderBy,
      include: {
        author: {
          select: {
            id: true,
            username: true,
            avatar: true,
          },
        },
        tags: {
          select: {
            id: true,
            name: true,
            slug: true,
          },
        },
        _count: {
          select: {
            comments: true,
            likes: true,
          },
        },
      },
    }),
    prisma.post.count({ where }),
  ]);

  return {
    data: posts,
    meta: {
      page,
      limit,
      total,
      totalPages: Math.ceil(total / limit),
      hasNext: page * limit < total,
      hasPrev: page > 1,
    },
  };
}

// 使用示例
const results = await searchPosts({
  keyword: "TypeScript",
  tags: ["tag-id-1", "tag-id-2"],
  status: "PUBLISHED",
  dateFrom: "2024-01-01",
  dateTo: "2024-12-31",
  sortBy: "publishedAt",
  sortOrder: "desc",
  page: 1,
  limit: 20,
});
```

## 常见问题

**Q: 如何实现模糊搜索？**

A: 使用 `contains` 运算符：

```typescript
const posts = await prisma.post.findMany({
  where: {
    OR: [{ title: { contains: keyword } }, { content: { contains: keyword } }],
  },
});
```

**Q: 游标分页和偏移分页如何选择？**

A: 游标分页适合大数据量和实时数据，偏移分页适合小数据量和静态数据。

**Q: 如何优化慢查询？**

A: 1. 检查是否使用了索引 2. 只查询需要的字段 3. 使用 EXPLAIN 分析查询计划 4. 考虑使用原始 SQL。

**Q: 全文搜索支持中文吗？**

A: PostgreSQL 支持中文全文搜索，需要配置正确的 text search configuration：

