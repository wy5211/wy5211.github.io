## 前言

CRUD（Create、Read、Update、Delete）是应用程序与数据库交互的四个基本操作。无论是简单的 Todo 应用，还是复杂的电商系统，本质上都是对数据的增删改查。

Prisma 提供了一套直观、类型安全的 API，让这些操作变得简单而强大。通过 Prisma Client，你可以用 TypeScript 的方式思考数据库操作，而不需要编写复杂的 SQL。

**本文将涵盖**：

- Create：创建数据的多种方式
- Read：读取数据的各种查询
- Update：更新数据的技巧
- Delete：删除数据的策略
- Upsert：存在则更新，否则创建
- 批量操作优化
- 计数和聚合查询
- 数据选择和排除

## Create：创建数据

### 创建单条记录

最基础的创建操作：

```typescript
const user = await prisma.user.create({
  data: {
    email: "user@example.com",
    username: "john",
    password: "hashed_password",
  },
});
// 返回类型：User
```

### 创建带嵌套关系

同时创建关联数据：

```typescript
// 创建文章时同时创建作者
const post = await prisma.post.create({
  data: {
    title: "我的第一篇文章",
    slug: "my-first-post",
    content: "文章内容...",
    author: {
      create: {
        email: "author@example.com",
        username: "author",
        password: "hashed",
      },
    },
  },
  include: {
    author: true,
  },
});
```

### 创建带关系的记录

连接到现有记录：

```typescript
// 为现有作者创建文章
const post = await prisma.post.create({
  data: {
    title: "Prisma 教程",
    slug: "prisma-tutorial",
    content: "...",
    author: {
      connect: { id: authorId },
    },
    tags: {
      connect: [{ id: tagId1 }, { id: tagId2 }],
    },
  },
});
```

### 批量创建

```typescript
// 批量创建多条记录
const result = await prisma.post.createMany({
  data: [
    {
      title: "文章 1",
      slug: "post-1",
      content: "...",
      authorId: authorId,
    },
    {
      title: "文章 2",
      slug: "post-2",
      content: "...",
      authorId: authorId,
    },
  ],
  skipDuplicates: true, // 跳过重复记录（基于唯一约束）
});

console.log(`创建了 ${result.count} 条记录`);
```

### 嵌套创建复杂关系

```typescript
const user = await prisma.user.create({
  data: {
    email: "user@example.com",
    username: "john",
    posts: {
      create: [
        {
          title: "第一篇文章",
          slug: "first-post",
          content: "...",
          comments: {
            create: {
              content: "第一条评论",
              author: {
                connect: { id: commenterId },
              },
            },
          },
        },
      ],
    },
  },
  include: {
    posts: {
      include: {
        comments: true,
      },
    },
  },
});
```

## Read：读取数据

### findUnique：查询单条唯一记录

根据主键或唯一字段查询：

```typescript
// 根据 ID 查询
const user = await prisma.user.findUnique({
  where: {
    id: userId,
  },
});

// 根据唯一字段查询
const user = await prisma.user.findUnique({
  where: {
    email: "user@example.com",
  },
});

// 根据复合唯一约束查询
const enrollment = await prisma.enrollment.findUnique({
  where: {
    studentId_courseId: {
      studentId: "student-1",
      courseId: "course-1",
    },
  },
});
```

### findFirst：查询第一条匹配记录

```typescript
// 查询第一条已发布的文章
const post = await prisma.post.findFirst({
  where: {
    status: "PUBLISHED",
  },
  orderBy: {
    createdAt: "desc",
  },
});

// 查询最后一条
const lastPost = await prisma.post.findFirst({
  where: {
    authorId: authorId,
  },
  orderBy: {
    createdAt: "desc",
  },
});
```

### findMany：查询多条记录

```typescript
// 查询所有已发布的文章
const posts = await prisma.post.findMany({
  where: {
    status: "PUBLISHED",
  },
  orderBy: {
    createdAt: "desc",
  },
});

// 分页查询
const posts = await prisma.post.findMany({
  skip: 0, // 跳过前 0 条
  take: 10, // 获取 10 条
  orderBy: {
    createdAt: "desc",
  },
});

// 第二页
const page2 = await prisma.post.findMany({
  skip: 10,
  take: 10,
});
```

### findFirstOrThrow 和 findUniqueOrThrow

如果未找到则抛出错误：

```typescript
try {
  const user = await prisma.user.findUniqueOrThrow({
    where: { id: userId },
  });
  // user 存在
} catch (error) {
  // 用户不存在
  if (error.code === "P2025") {
    console.log("用户不存在");
  }
}
```

### 选择特定字段

使用 `select` 只查询需要的字段：

```typescript
const user = await prisma.user.findUnique({
  where: { id: userId },
  select: {
    id: true,
    email: true,
    username: true,
    password: false, // 不查询密码
  },
});
```

### 包含关联数据

使用 `include` 加载关系：

```typescript
const post = await prisma.post.findUnique({
  where: { id: postId },
  include: {
    author: {
      select: {
        id: true,
        username: true,
        email: true,
      },
    },
    comments: {
      where: {
        parentId: null, // 只查询顶级评论
      },
      orderBy: {
        createdAt: "desc",
      },
      take: 10,
    },
    tags: true,
  },
});
```

### 嵌套查询

```typescript
const user = await prisma.user.findUnique({
  where: { id: userId },
  include: {
    posts: {
      where: {
        status: "PUBLISHED",
      },
      include: {
        comments: {
          include: {
            author: {
              select: {
                username: true,
              },
            },
          },
        },
      },
      orderBy: {
        createdAt: "desc",
      },
    },
  },
});
```

### 关系计数

```typescript
const user = await prisma.user.findUnique({
  where: { id: userId },
  include: {
    _count: {
      select: {
        posts: true,
        comments: true,
        likes: true,
      },
    },
  },
});

// 返回：{ id, email, ..., _count: { posts: 10, comments: 25, likes: 50 } }
```

### 事务性查询

一次查询多个模型：

```typescript
const [user, postCount, recentPosts] = await Promise.all([
  prisma.user.findUnique({ where: { id: userId } }),
  prisma.post.count({ where: { authorId: userId } }),
  prisma.post.findMany({
    where: { authorId: userId },
    take: 5,
    orderBy: { createdAt: "desc" },
  }),
]);
```

```typescript
// 重新分配文章作者
const post = await prisma.post.update({
  where: { id: postId },
  data: {
    author: {
      connect: { id: newAuthorId }, // 连接到新作者
    },
  },
});

// 添加标签
const post = await prisma.post.update({
  where: { id: postId },
  data: {
    tags: {
      connect: [{ id: tagId }],
    },
  },
});

// 移除标签
const post = await prisma.post.update({
  where: { id: postId },
  data: {
    tags: {
      disconnect: [{ id: tagId }],
    },
  },
});

// 替换所有标签
const post = await prisma.post.update({
  where: { id: postId },
  data: {
    tags: {
      set: [{ id: tagId1 }, { id: tagId2 }],
    },
  },
});

// 删除所有标签
const post = await prisma.post.update({
  where: { id: postId },
  data: {
    tags: {
      deleteMany: {},
    },
  },
});
```

### 批量更新

```typescript
// 将所有草稿标记为已归档
const result = await prisma.post.updateMany({

    },
  },
  data: {
    status: "ARCHIVED",

```

### 更新嵌套关系

```typescript
const user = await prisma.user.update({
```

### 增量更新

## Upsert：存在则更新，否则创建

Upsert 是 "update" + "insert" 的缩写，非常适合同步场景：

```typescript
const user = await prisma.user.upsert({
  where: {
    email: "user@example.com",
  },
  update: {
    // 如果存在，更新这些字段
    username: "updated_username",
    bio: "更新后的简介",
  },
  create: {
    // 如果不存在，创建新记录
    email: "user@example.com",
    username: "new_user",
    password: "hashed_password",
  },
});
```

### 实际应用场景

```typescript
// 从外部 API 同步数据
async function syncProduct(productData: ProductData) {
  return await prisma.product.upsert({
    where: {
      sku: productData.sku,
    },
    update: {
      name: productData.name,
      price: productData.price,
      stock: productData.stock,
      updatedAt: new Date(),
    },
    create: {
      sku: productData.sku,
      name: productData.name,
      price: productData.price,
      stock: productData.stock,
    },
  });
}

// 批量 Upsert
const results = await Promise.all(
  products.map((product) =>
    prisma.product.upsert({
      where: { sku: product.sku },
      update: { ...product },
      create: { ...product },
    })
  )
);
```

## 高级技巧

### 事务性操作

```typescript
// 使用事务 API
const transfer = await prisma.$transaction(async (tx) => {
  // 扣除发送者余额
  const sender = await tx.user.update({
    where: { id: senderId },
    data: {
      balance: {
        decrement: amount,
      },
    },
  });

  // 检查余额
  if (sender.balance < 0) {
    throw new Error("余额不足");
  }

  // 增加接收者余额
  const receiver = await tx.user.update({
    where: { id: receiverId },
    data: {
      balance: {
        increment: amount,
      },
    },
  });

  // 创建交易记录
  const transaction = await tx.transaction.create({
    data: {
      senderId,
      receiverId,
      amount,
    },
  });

  return transaction;
});

// 批量事务
await prisma.$transaction([
  prisma.user.update({
    where: { id: "1" },
    data: { balance: { decrement: 100 } },
  }),
  prisma.user.update({
    where: { id: "2" },
    data: { balance: { increment: 100 } },
  }),
]);
```

### 原始 SQL 查询

```typescript
// 执行原始 SQL
```

## 完整示例：博客 API

```typescript
import { PrismaClient } from "@prisma/client";

const prisma = new PrismaClient();

// 创建文章
export async function createPost(data: {
  title: string;
  content: string;
  authorId: string;
  tags?: string[];
}) {
  return await prisma.post.create({
    data: {
      title: data.title,
      slug: data.title.toLowerCase().replace(/\s+/g, "-"),
      content: data.content,
      author: {
        connect: { id: data.authorId },
      },
      tags: data.tags
        ? {
            connectOrCreate: data.tags.map((tag) => ({
              where: { name: tag },
              create: { name: tag, slug: tag.toLowerCase() },
            })),
          }
        : undefined,
    },
    include: {
      author: {
        select: {
          id: true,
          username: true,
        },
      },
      tags: true,
    },
  });
}

// 获取文章列表
export async function getPosts(params: {
  page?: number;
  limit?: number;
  status?: string;
  authorId?: string;
}) {
  const { page = 1, limit = 10, status, authorId } = params;

  const where = {
    ...(status && { status: status as any }),
    ...(authorId && { authorId }),
  };

  const [posts, total] = await Promise.all([
    prisma.post.findMany({
      where,
      skip: (page - 1) * limit,
      take: limit,
      orderBy: { createdAt: "desc" },
      include: {
        author: {
          select: {
            id: true,
            username: true,
          },
        },
        tags: true,
        _count: {
          select: {
            comments: true,
            likes: true,
          },
        },
      },
    }),
    prisma.post.count({ where }),
  ]);

  return {
    data: posts,
    meta: {
      page,
      limit,
      total,
      totalPages: Math.ceil(total / limit),
    },
  };
}

// 更新文章
export async function updatePost(
  id: string,
  data: {
    title?: string;
    content?: string;
    status?: string;
  }
) {
  return await prisma.post.update({
    where: { id },
    data: {
      ...data,
      ...(data.title && {
        slug: data.title.toLowerCase().replace(/\s+/g, "-"),
      }),
    },
  });
}

// 删除文章
export async function deletePost(id: string) {
  return await prisma.post.delete({
    where: { id },
  });
}

// 统计数据
export async function getStats() {
  const [users, posts, comments] = await Promise.all([
    prisma.user.count(),
    prisma.post.count(),
    prisma.comment.count(),
  ]);

  return { users, posts, comments };
}
```

## 最佳实践

### 1. 始终使用类型安全

```typescript
// ✅ 类型安全
type UserCreateInput = Prisma.UserCreateInput;

const user = await prisma.user.create({
  data: userData as UserCreateInput,
});

// ❌ 避免使用 any
const user = await prisma.user.create({
  data: userData as any,
});
```

### 2. 合理使用 select 和 include

```typescript
// ✅ 只查询需要的字段
const users = await prisma.user.findMany({
  select: {
    id: true,
    username: true,
    password: false, // 不查询敏感信息
  },
});

// ❌ 避免查询所有字段
const users = await prisma.user.findMany();
```

### 3. 使用批量操作优化性能

```typescript
// ✅ 批量创建
await prisma.post.createMany({
  data: posts,
});

// ❌ 避免循环创建
for (const post of posts) {
  await prisma.post.create({ data: post });
}
```

### 4. 处理边界情况

```typescript
// ✅ 使用 findFirstOrThrow
const user = await prisma.user.findFirstOrThrow({
  where: { id: userId },
});

// ❌ 不处理 null
const user = await prisma.user.findUnique({ where: { id: userId } });
user.username; // 可能为 null，导致错误
```

## 常见问题

**Q: update 和 updateMany 有什么区别？**

A: `update` 更新单条记录并返回更新后的对象，`updateMany` 更新多条记录并返回影响的行数。

**Q: 如何实现软删除？**

A: 添加 `deletedAt` 字段，查询时过滤 `deletedAt: null` 的记录。

**Q: 批量操作会影响性能吗？**

A: 不会。Prisma 会将批量操作优化为单个 SQL 语句。

**Q: 如何处理更新冲突？**

A: 使用事务或重试逻辑：

```typescript
let retries = 3
while (retries > 0) {
  try {
    return await prisma.user.update({ ... })
  } catch (error) {
    if (error.code === 'P2034') {
      retries--
      continue
    }
    throw error
  }
}
```
