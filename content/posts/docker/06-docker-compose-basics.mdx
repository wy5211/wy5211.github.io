## 前言

随着应用复杂度的增加，单个容器已经无法满足需求。典型的 Web 应用通常包含：前端、后端 API、数据库、缓存、消息队列等多个组件。手动管理这些容器既繁琐又容易出错。

Docker Compose 是定义和运行多容器应用的工具，通过 YAML 配置文件一键启动整个应用栈。

**学习目标**：

- 理解 Docker Compose 的核心概念
- 掌握 docker-compose.yml 的编写
- 学会常用命令：up、down、ps、logs
- 能够编排典型的多容器应用

## 什么是 Docker Compose

### 核心概念

Docker Compose 通过一个 YAML 文件定义多服务应用：

```yaml
# docker-compose.yml
services:
  web:
    build: .
    ports:
      - "3000:3000"
  db:
    image: postgres:15
    environment:
      POSTGRES_PASSWORD: secret
```

**一键启动**：

```bash
docker compose up
```

### 优势

| 特性       | 手动命令       | Docker Compose |
| ---------- | -------------- | -------------- |
| 启动速度   | 慢（逐个启动） | 快（并行启动） |
| 配置管理   | 分散（命令行） | 集中（YAML）   |
| 环境一致性 | 难保证         | 完全一致       |
| 网络配置   | 手动创建       | 自动配置       |
| 团队协作   | 需文档         | 代码即文档     |

## docker-compose.yml 结构

### 基本格式

```yaml
version: "3.8" # Compose 文件版本
services: # 定义服务
  web: # 服务名称
    build: . # 构建配置
    ports: # 端口映射
      - "3000:3000"
    depends_on: # 依赖关系
      - db
  db: # 另一个服务
    image: postgres:15 # 使用镜像
    environment: # 环境变量
      POSTGRES_PASSWORD: secret
volumes: # 数据卷
  db-data:
networks: # 网络
  frontend:
```

### 常用字段

| 字段          | 说明         | 示例                                    |
| ------------- | ------------ | --------------------------------------- |
| `image`       | 指定镜像     | `image: nginx:latest`                   |
| `build`       | 构建镜像     | `build: .` 或 `build: {context: ./dir}` |
| `command`     | 覆盖默认命令 | `command: npm run dev`                  |
| `ports`       | 端口映射     | `ports: ["80:80", "443:443"]`           |
| `environment` | 环境变量     | `environment: {NODE_ENV: dev}`          |
| `volumes`     | 数据卷挂载   | `volumes: ["./data:/app/data"]`         |
| `depends_on`  | 依赖关系     | `depends_on: ["db", "redis"]`           |
| `networks`    | 网络配置     | `networks: ["frontend", "backend"]`     |
| `restart`     | 重启策略     | `restart: unless-stopped`               |

## 实战演练 1：Web 应用栈

### 全栈应用配置

```yaml
version: "3.8"

services:
  # 前端服务
  frontend:
    build: ./frontend
    ports:
      - "80:80"
    depends_on:
      - backend
    networks:
      - frontend

  # 后端 API
  backend:
    build: ./backend
    ports:
      - "3000:3000"
    environment:
      DATABASE_URL: postgres://postgres:secret@db:5432/myapp
      REDIS_URL: redis://redis:6379
    depends_on:
      - db
      - redis
    networks:
      - frontend
      - backend

  # PostgreSQL 数据库
  db:
    image: postgres:15-alpine
    environment:
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: secret
      POSTGRES_DB: myapp
    volumes:
      - postgres-data:/var/lib/postgresql/data
    networks:
      - backend

  # Redis 缓存
  redis:
    image: redis:7-alpine
    networks:
      - backend

volumes:
  postgres-data:

networks:
  frontend:
  backend:
```

### 启动应用

```bash
# 后台启动所有服务
docker compose up -d

# 查看运行状态
docker compose ps

# 输出：
# NAME            SERVICE     STATUS
# app-frontend-1 frontend    running
# app-backend-1   backend     running
# app-db-1        db          running
# app-redis-1     redis       running

# 查看日志
docker compose logs -f backend

# 停止所有服务
docker compose down
```

## 实战演练 2：开发环境

### Node.js 开发环境

```yaml
version: "3.8"

services:
  app:
    build:
      context: .
      dockerfile: Dockerfile.dev
    volumes:
      # 热重载：挂载源代码
      - ./src:/app/src
      - ./public:/app/public
      # 防止 node_modules 被覆盖
      - /app/node_modules
    ports:
      - "3000:3000"
      - "9229:9229" # 调试端口
    environment:
      - NODE_ENV=development
      - CHOKIDAR_USEPOLLING=true # 文件监听
    command: npm run dev
```

**启动开发环境**：

```bash
docker compose up
# 源代码修改会自动重载
```

## 实战演练 3：生产环境

### 生产环境配置

```yaml
version: "3.8"

services:
  web:
    image: myapp:1.0.0
    ports:
      - "80:80"
      - "443:443"
    environment:
      - NODE_ENV=production
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:80/health"]
      interval: 30s
      timeout: 3s
      retries: 3
    deploy:
      replicas: 3
      resources:
        limits:
          cpus: "0.5"
          memory: 512M
        reservations:
          cpus: "0.25"
          memory: 256M

  nginx:
    image: nginx:alpine
    ports:
      - "80:80"
    volumes:
      - ./nginx.conf:/etc/nginx/nginx.conf:ro
    depends_on:
      - web
    restart: unless-stopped
```

## 常用命令

### 生命周期命令

```bash
# 创建并启动服务
docker compose up

# 后台启动
docker compose up -d

# 强制重新构建
docker compose up -d --build

# 停止服务
docker compose stop

# 停止并删除容器
docker compose down

# 删除容器和数据卷
docker compose down -v
```

### 查看命令

```bash
# 查看运行状态
docker compose ps

# 查看服务日志
docker compose logs

# 跟踪特定服务日志
docker compose logs -f web

# 查看最近 100 行
docker compose logs --tail=100 web
```

### 执行命令

```bash
# 在服务中执行命令
docker compose exec web /bin/bash

# 运行一次性命令
docker compose run --rm web npm test

# 进入特定容器
docker compose exec db psql -U postgres myapp
```

### 管理命令

```bash
# 启动已停止的服务
docker compose start

# 重启服务
docker compose restart

# 暂停服务
docker compose pause

# 恢复服务
docker compose unpause

# 查看资源使用
docker compose top
```

## 环境变量

### 使用 .env 文件

```bash
# .env
POSTGRES_USER=postgres
POSTGRES_PASSWORD=secret
POSTGRES_DB=myapp
REDIS_PASSWORD=secret
```

```yaml
# docker-compose.yml
version: "3.8"

services:
  db:
    image: postgres:15
    environment:
      POSTGRES_USER: ${POSTGRES_USER}
      POSTGRES_PASSWORD: ${POSTGRES_PASSWORD}
      POSTGRES_DB: ${POSTGRES_DB}

  redis:
    image: redis:7
    command: redis-server --requirepass ${REDIS_PASSWORD}
```

### 不同环境配置

```bash
# 开发环境
docker compose -f docker-compose.yml -f docker-compose.dev.yml up

# 生产环境
docker compose -f docker-compose.yml -f docker-compose.prod.yml up
```

```yaml
# docker-compose.dev.yml
version: '3.8'

services:
  web:
    build:
      target: development
    volumes:
      - ./src:/app/src
    environment:
      - NODE_ENV=development

# docker-compose.prod.yml
version: '3.8'

services:
  web:
    image: myapp:latest
    environment:
      - NODE_ENV=production
    restart: unless-stopped
```

## 服务依赖

### depends_on

```yaml
version: "3.8"

services:
  web:
    image: nginx
    depends_on:
      - app
  app:
    build: .
    depends_on:
      - db
  db:
    image: postgres
```

**注意**：`depends_on` 只控制启动顺序，不等待服务就绪。

### 健康检查等待

```yaml
version: "3.8"

services:
  app:
    image: myapp:latest
    depends_on:
      db:
        condition: service_healthy
  db:
    image: postgres:15
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U postgres"]
      interval: 5s
      timeout: 3s
      retries: 5
```

## 网络配置

### 自定义网络

```yaml
version: "3.8"

services:
  frontend:
    image: nginx
    networks:
      - frontend

  backend:
    build: .
    networks:
      - frontend
      - backend

  db:
    image: postgres
    networks:
      - backend

networks:
  frontend:
  backend:
    driver: bridge
```

### 网络别名

```yaml
version: "3.8"

services:
  web:
    image: nginx
    networks:
      app_net:
        aliases:
          - nginx
          - web-server

networks:
  app_net:
```

## 最佳实践

### 1. 命名规范

```yaml
# ✅ 推荐：使用有意义的服务名
services:
  frontend:
    image: nginx
  backend:
    build: .
  database:
    image: postgres
```

### 2. 版本固定

```yaml
# ✅ 推荐：使用固定版本
image: postgres:15.3-alpine

# ❌ 避免：使用 latest
image: postgres:latest
```

### 3. 资源限制

```yaml
# ✅ 推荐：设置资源限制
services:
  app:
    image: myapp
    deploy:
      resources:
        limits:
          cpus: "0.5"
          memory: 512M
```

### 4. 健康检查

```yaml
# ✅ 推荐：配置健康检查
services:
  web:
    image: nginx
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost"]
      interval: 30s
      timeout: 3s
```

## 常见问题

### Q1: 如何查看详细日志？

**答**：

```bash
docker compose logs --tail=100 -f web
```

### Q2: 如何重启单个服务？

**答**：

```bash
docker compose restart web
```

### Q3: 如何进入运行中的容器？

**答**：

```bash
docker compose exec web /bin/bash
```

### Q4: 如何重新构建镜像？

**答**：

```bash
docker compose build --no-cache web
```

### Q5: 如何复制文件到容器？

**答**：

```bash
docker compose cp /path/to/file web:/path/in/container
```
