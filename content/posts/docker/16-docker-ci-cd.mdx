## 前言

手动构建和部署镜像效率低下且容易出错。将 Docker 集成到 CI/CD 流水线，可以实现自动化构建、测试和部署。

## GitHub Actions 配置

### 完整工作流

```yaml
name: Docker CI/CD

on:
  push:
    branches: [main]
    tags: ["v*"]
  pull_request:

env:
  REGISTRY: ghcr.io
  IMAGE_NAME: ${{ github.repository }}

jobs:
  build-and-test:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v3

      - name: Set up Docker Buildx
        uses: docker/setup-buildx-action@v2

      - name: Build image
        run: |
          docker build -t $IMAGE_NAME:test .

      - name: Run tests
        run: |
          docker run --rm $IMAGE_NAME:test npm test
          docker run --rm $IMAGE_NAME:test npm run test:integration

  build-and-push:
    needs: build-and-test
    runs-on: ubuntu-latest
    if: github.event_name != 'pull_request'
    steps:
      - name: Checkout
        uses: actions/checkout@v3

      - name: Set up Docker Buildx
        uses: docker/setup-buildx-action@v2

      - name: Login to Registry
        uses: docker/login-action@v2
        with:
          registry: ${{ env.REGISTRY }}
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}

      - name: Extract metadata
        id: meta
        uses: docker/metadata-action@v4
        with:
          images: ${{ env.REGISTRY }}/${{ github.repository }}
          tags: |
            type=ref,event=branch
            type=semver,pattern={{version}}
            type=raw,value=latest,enable={{is_default_branch}}

      - name: Build and push
        uses: docker/build-push-action@v4
        with:
          context: .
          push: true
          tags: ${{ steps.meta.outputs.tags }}
          labels: ${{ steps.meta.outputs.labels }}
          cache-from: type=gha
          cache-to: type=gha,mode=max

      - name: Trivy scan
        uses: aquasecurity/trivy-action@master
        with:
          image-ref: ${{ steps.meta.outputs.tags }}
          format: "sarif"
          output: "trivy-results.sarif"

      - name: Upload scan results
        uses: github/codeql-action/upload-sarif@v2
        with:
          sarif_file: trivy-results.sarif
```

## 多环境部署

### 开发环境

```yaml
deploy-dev:
  runs-on: ubuntu-latest
  if: github.ref == 'refs/heads/develop'
  steps:
    - uses: actions/checkout@v3

    - name: Deploy to Dev
      run: |
        echo "Deploying to dev environment"
        # SSH 连接到开发服务器
        ssh user@dev-server "cd /app && docker compose pull && docker compose up -d"
```

### 生产环境

```yaml
deploy-prod:
  runs-on: ubuntu-latest
  if: startsWith(github.ref, 'refs/tags/v')
  steps:
    - uses: actions/checkout@v3

    - name: Deploy to Prod
      run: |
        echo "Deploying ${{ github.ref_name }} to production"
        # SSH 连接到生产服务器
        ssh user@prod-server "cd /app && \
          docker-compose pull && \
          docker-compose up -d && \
          docker-compose run --rm db npm run migrate"
```

## GitLab CI/CD

### 配置示例

```yaml
# .gitlab-ci.yml
stages:
  - build
  - test
  - deploy

variables:
  DOCKER_DRIVER: overlay2
  DOCKER_TLS_CERTDIR: "/certs/client"
  DOCKER_TLS_VERIFY: 0
  IMAGE_TAG: $CI_REGISTRY_IMAGE:$CI_COMMIT_SHORT_SHA

build:
  stage: build
  image: docker:24.0
  services:
    - docker:dind
  script:
    - docker login -u $CI_REGISTRY_USER -p $CI_REGISTRY_PASSWORD $CI_REGISTRY
    - docker build -t $IMAGE_TAG .
    - docker push $IMAGE_TAG

test:
  stage: test
  image: docker:24.0
  services:
    - docker:dind
  script:
    - docker pull $IMAGE_TAG
    - docker run $IMAGE_TAG npm test

deploy:
  stage: deploy
  image: docker:24.0
  script:
    - ssh $DEPLOY_SERVER "cd /app && docker compose pull && docker compose up -d"
  only:
    - main
```

## Dockerfile 优化

### 多阶段构建

```dockerfile
# 构建阶段
FROM golang:1.21-alpine AS builder
WORKDIR /app
COPY go.* ./
RUN go mod download
COPY . .
RUN CGO_ENABLED=0 go build -o app

# 运行阶段
FROM alpine:3.19
RUN apk --no-cache add ca-certificates
WORKDIR /app
COPY --from=builder /app/app .
RUN addgroup -g 1001 appuser && adduser -D -u 1001 appuser
USER appuser
CMD ["./app"]
```

### 缓存优化

```bash
# 在 CI 中使用 BuildKit cache
export DOCKER_BUILDKIT=1
docker build \
  --mount=type=cache,target=/go/pkg \
  --mount=type=cache,target=/go/.cache \
  -t myapp:latest .
```

## 测试自动化

### 单元测试

```yaml
test:
  runs-on: ubuntu-latest
  steps:
    - uses: actions/checkout@v3
    - run: docker build -t myapp:test .

    - name: Unit tests
      run: docker run myapp:test npm test

    - name: Upload coverage
      uses: codecov/codecov-action@v3
      with:
        file: ./coverage/lcov.info
```

### 集成测试

```yaml
integration-test:
  runs-on: ubuntu-latest
  steps:
    - uses: actions/checkout@v3
    - run: docker compose -f docker-compose.test.yml up -d

    - name: Run tests
      run: docker compose -f docker-compose.test.yml run --rm test

    - name: Cleanup
      run: docker compose -f docker-compose.test.yml down
```

## 安全扫描

### 漏洞扫描

```yaml
security:
  runs-on: ubuntu-latest
  steps:
    - name: Build image
      run: docker build -t $IMAGE_TAG .

    - name: Run Trivy
      uses: aquasecurity/trivy-action@master
      with:
        image-ref: $IMAGE_TAG
        format: "sarif"
        output: "trivy-results.sarif"
        severity: "HIGH,CRITICAL"

    - name: Upload results
      uses: github/codeql-action/upload-sarif@v2
      with:
        sarif_file: trivy-results.sarif
```

## 版本管理

### 语义化版本

```yaml
- name: Generate version
  id: version
  run: |
    VERSION=$(git describe --tags --abbrev=0)
    echo "VERSION=$VERSION" >> $GITHUB_ENV

- name: Build and push
  uses: docker/build-push-action@v4
  with:
    tags: |
      $REGISTRY/$IMAGE_NAME:${{ env.VERSION }}
      $REGISTRY/$IMAGE_NAME:latest
```

## 回滚策略

```yaml
- name: Deploy with fallback
  run: |
    # 部署新版本
    ssh server "cd /app && docker compose pull $NEW_VERSION && docker compose up -d"

    # 等待健康检查
    sleep 30

    # 如果失败，回滚到上一个版本
    if ! curl -f http://server/health; then
      ssh server "cd /app && docker compose pull $OLD_VERSION && docker compose up -d"
    fi
```

## 最佳实践

1. **自动化测试**：构建后自动测试
2. **安全扫描**：部署前漏洞检测
3. **多环境**：dev/staging/prod 分离
4. **版本管理**：语义化版本
5. **快速失败**：测试失败立即停止
6. **监控反馈**：部署状态实时通知
