## 前言

容器安全是生产部署的关键。本篇文章将全面介绍 Docker 安全的最佳实践。

## 镜像安全

### 扫描漏洞

```bash
# Docker Scout
docker scout quickscan nginx:latest

# Trivy
trivy image --severity HIGH,CRITICAL nginx:latest

# 定期扫描
docker scout cves
```

### 使用固定版本

```dockerfile
# ✅ 推荐
FROM node:18.17.0-alpine

# ❌ 避免
FROM node:latest
```

### 最小化镜像

```dockerfile
# distroless：无 shell，无包管理器
FROM gcr.io/distroless/nodejs18-debian11

COPY . .
CMD ["node", "server.js"]
```

## 运行时安全

### 非 root 用户

```dockerfile
FROM node:18-alpine

# 创建用户
RUN addgroup -g 1001 -S nodejs && \
    adduser -S nodejs -u 1001

# 设置权限
WORKDIR /app
RUN chown -R nodejs:nodejs /app

USER nodejs

CMD ["node", "server.js"]
```

### 只读文件系统

```dockerfile
FROM node:18-alpine
COPY . /
CMD ["node", "server.js"]

# 运行时设置只读
docker run --read-only --tmpfs /tmp app
```

### 能力限制

```bash
# 移除所有能力
docker run --cap-drop=ALL nginx

# 只添加需要的
docker run --cap-drop=ALL --cap-add=NET_BIND_SERVICE nginx

# 查看所需能力
docker inspect nginx | grep CapAdd
```

## 网络安全

### 隔离网络

```yaml
version: "3.8"

services:
  web:
    networks:
      - frontend

  db:
    networks:
      - backend
    # 无需暴露到前端

networks:
  frontend:
  backend:
```

### 加密通信

```yaml
# TLS/SSL 配置
services:
  web:
    image: nginx:alpine
    volumes:
      - ./certs:/etc/nginx/certs:ro
      - ./nginx.conf:/etc/nginx/nginx.conf:ro
    ports:
      - "443:443"
```

## 资源限制

### CPU 和内存限制

```yaml
services:
  app:
    image: myapp
    deploy:
      resources:
        limits:
          cpus: "0.5"
          memory: 512M
        reservations:
          cpus: "0.25"
          memory: 256M
```

## 总结

Docker 安全核心：

- **镜像层**：扫描、固定版本、最小化
- **运行层**：非root、只读、能力限制
- **网络层**：隔离、加密
