## 前言

容器是临时的，容器删除后，容器内的数据也会丢失。对于数据库、应用日志等需要持久化的数据，我们必须使用 Docker 的存储机制。本篇文章将详细介绍 Docker 的数据持久化方案。

**学习目标**：

- 理解容器文件系统的特性
- 掌握 Docker 数据卷的使用
- 学会绑定挂载的实际应用
- 了解存储驱动的选择和优化

## 为什么需要数据持久化

### 容器文件系统特性

```
┌─────────────────────────┐
│  容器可写层             │  ← 临时数据，容器删除后丢失
├─────────────────────────┤
│  镜像层（只读）         │  ← 镜像数据，只读
└─────────────────────────┘
```

**问题场景**：

- 🗄️ **数据库**：容器重启后数据丢失
- 📝 **应用日志**：容器删除后日志消失
- 📁 **用户上传**：用户上传的文件需要保存
- ⚙️ **配置文件**：应用配置需要持久化

## Docker 数据卷（Volumes）

### 什么是数据卷

数据卷是由 Docker 管理的宿主机文件系统的一部分，完全独立于容器的生命周期。

**特点**：

- ✅ **持久化**：容器删除后数据仍保留
- ✅ **共享性**：多个容器可共享同一卷
- ✅ **安全性**：由 Docker 管理，不依赖宿主机目录结构
- ✅ **跨平台**：Windows、macOS、Linux 通用

### 创建和管理数据卷

```bash
# 创建数据卷
docker volume create my-data

# 查看所有卷
docker volume ls

# 查看卷详情
docker volume inspect my-data

# 删除卷
docker volume rm my-data

# 删除未使用的卷
docker volume prune
```

### 使用数据卷

```bash
# 运行容器并挂载卷
docker run -d \
  --name postgres \
  -v postgres-data:/var/lib/postgresql/data \
  -e POSTGRES_PASSWORD=secret \
  postgres:15

# 查看卷的使用情况
docker volume ls

# 输出：
# DRIVER    VOLUME NAME
# local     postgres-data
```

### 卷的生命周期

```bash
# 删除容器时保留卷
docker rm postgres

# 重新使用已有卷
docker run -d \
  --name postgres-new \
  -v postgres-data:/var/lib/postgresql/data \
  -e POSTGRES_PASSWORD=secret \
  postgres:15

# 数据不会丢失！
```

## 绑定挂载（Bind Mounts）

### 什么是绑定挂载

将宿主机文件系统中的目录或文件挂载到容器中。

**特点**：

- ✅ **实时同步**：宿主机修改立即反映到容器
- ✅ **开发友好**：编辑器直接修改代码
- ⚠️ **依赖路径**：不同系统路径可能不同
- ⚠️ **权限问题**：需要注意文件权限

### 基本用法

```bash
# 挂载当前目录
docker run -d \
  --name my-app \
  -v $(pwd):/app \
  node:18

# 挂载单个文件
docker run -d \
  --name nginx \
  -v $(pwd)/nginx.conf:/etc/nginx/nginx.conf:ro \
  nginx:alpine

# 只读挂载
docker run -d \
  -v /host/path:/container/path:ro \
  nginx
```

### 开发环境实战

```bash
# 创建 Node.js 项目
mkdir myapp && cd myapp
npm init -y
echo "console.log('Hello from Docker!')" > index.js

# 运行容器并挂载代码
docker run -d \
  --name mynode \
  -v $(pwd):/app \
  -w /app \
  node:18 \
  node index.js

# 修改代码
echo "console.log('Updated!')" >> index.js

# 重启容器查看效果
docker restart mynode
docker logs mynode
```

## 挂载方式对比

### 卷 vs 绑定挂载

| 特性     | 数据卷 (Volumes) | 绑定挂载 (Bind Mounts) |
| -------- | ---------------- | ---------------------- |
| 位置     | Docker 管理      | 用户指定               |
| 性能     | 高               | 中等                   |
| 可移植性 | 优秀             | 依赖宿主机             |
| 实时同步 | 否               | 是                     |
| 适用场景 | 生产环境         | 开发环境               |

### 匿名卷 vs 命名卷

```bash
# 匿名卷（每次生成随机名称）
docker run -v /data nginx

# 命名卷（可读性强）
docker run -v app-data:/data nginx
```

**选择建议**：✅ 优先使用命名卷

## 实战场景

### 场景 1：数据库持久化

```bash
# 1. 创建卷
docker volume create mysql-data

# 2. 运行 MySQL
docker run -d \
  --name mysql \
  -v mysql-data:/var/lib/mysql \
  -e MYSQL_ROOT_PASSWORD=secret \
  -e MYSQL_DATABASE=myapp \
  -p 3306:3306 \
  mysql:8

# 3. 测试持久化
docker exec -i mysql mysql -uroot -psecret -e "CREATE TABLE test (id INT);"
docker stop mysql
docker rm mysql
docker run -d \
  --name mysql-new \
  -v mysql-data:/var/lib/mysql \
  -e MYSQL_ROOT_PASSWORD=secret \
  mysql:8
# 数据仍然存在！
```

### 场景 2：应用日志

```bash
# 日志目录持久化
docker run -d \
  --name app \
  -v app-logs:/app/logs \
  -v app-uploads:/app/uploads \
  myapp:latest

# 查看日志
ls /var/lib/docker/volumes/app-logs/_data/
```

### 场景 3：开发环境热重载

```yaml
version: "3.8"

services:
  app:
    build: .
    volumes:
      # 挂载源代码（热重载）
      - ./src:/app/src
      # 挂载 node_modules（避免被覆盖）
      - /app/node_modules
    ports:
      - "3000:3000"
    environment:
      - NODE_ENV=development
```

### 场景 4：配置文件管理

```bash
# 配置文件在宿主机
cat > nginx.conf <<'EOF'
server {
    listen 80;
    location / {
        root /usr/share/nginx/html;
    }
}
EOF

# 挂载配置文件
docker run -d \
  --name nginx \
  -v $(pwd)/nginx.conf:/etc/nginx/nginx.conf:ro \
  -p 8080:80 \
  nginx:alpine
```

## 数据卷管理

### 查看卷占用

```bash
# 查看卷占用的磁盘空间
docker system df -v

# 输出示例：
# VOLUME NAME   LINKS     SIZE
# postgres-data 1         156MB
# mysql-data    0         89MB
# app-logs      2         1.2GB
```

### 备份和恢复

```bash
# 备份卷
docker run --rm \
  -v postgres-data:/data \
  -v $(pwd):/backup \
  alpine tar czf /backup/postgres-backup.tar.gz /data

# 恢复卷
docker run --rm \
  -v postgres-data-new:/data \
  -v $(pwd):/backup \
  alpine tar xzf /backup/postgres-backup.tar.gz -C /
```

### 迁移数据

```bash
# 从容器复制数据到宿主机
docker cp my-container:/app/data ./backup

# 从宿主机复制数据到容器
docker cp ./backup my-container:/app/data

# 使用 tar 流式传输
docker exec my-container tar czf - /data | tar xzf - ./data-backup
```

## 存储驱动

### 查看当前驱动

```bash
docker info | grep "Storage Driver"

# 输出：
# Storage Driver: overlay2
```

### 常用存储驱动

| 驱动     | 性能 | 稳定性 | 适用场景               |
| -------- | ---- | ------ | ---------------------- |
| overlay2 | 高   | 高     | 默认首选（Linux 4.0+） |
| aufs     | 中   | 高     | 旧版 Linux 内核        |
| btrfs    | 高   | 中     | 需要文件系统快照       |
| zfs      | 高   | 高     | 需要高级存储功能       |

### 配置存储驱动

```json
// /etc/docker/daemon.json
{
  "storage-driver": "overlay2",
  "storage-opts": ["overlay2.override_kernel_check=true"]
}
```

## 最佳实践

### 1. 生产环境使用卷

```bash
# ✅ 推荐：数据卷
docker volume create app-data
docker run -v app-data:/app/data app

# ❌ 避免：使用容器存储
docker run app  # 数据会丢失
```

### 2. 开发环境使用绑定挂载

```bash
# ✅ 推荐：绑定挂载实现热重载
docker run -v $(pwd):/app app

# ✅ 推荐：docker-compose 开发配置
```

### 3. 命名卷

```bash
# ✅ 推荐：使用命名卷
docker run -v app-data:/app/data app

# ❌ 避免：使用匿名卷
docker run -v /app/data app
```

### 4. 权限管理

```bash
# 以非 root 用户运行
docker run -v data:/app --user 1000:1000 app

# 修改卷权限
docker run --rm \
  -v app-data:/data \
  alpine chown -R 1000:1000 /data
```

## 常见问题

### Q1: 容器内文件修改在宿主机看不到？

**答**：检查挂载点是否正确

```bash
docker inspect container | grep -A 10 Mounts
```

### Q2: Windows 路径问题？

**答**：使用 WSL2 路径

```bash
# ✅ 正确
docker run -v /c/Users/path:/app app

# 或使用
docker run -v /mnt/c/path:/app app
```

### Q3: 权限被拒绝？

**答**：调整文件权限

```bash
# Linux
sudo chown -R $USER:$USER ./data

# 或在容器内修改
docker exec -u root app chown -R node:node /app/data
```
