## 前言

容器是 Docker 的运行实体，是从镜像创建的轻量级、隔离的运行环境。如果说镜像是"菜谱"，那么容器就是"做出来的菜"。本篇文章将详细介绍 Docker 容器的完整生命周期管理，帮助你熟练掌握容器的各种操作。

**学习目标**：

- 理解容器的生命周期和状态转换
- 掌握容器的创建、启动、停止、删除
- 学会进入容器和执行命令
- 掌握日志查看和资源监控

## 容器生命周期

### 状态转换图

```
┌─────────┐
│ Created │──[docker start]──→ Running
└─────────┘                      │
     ↑                          ↓
     └────────[docker stop]──────┘
     │                          ↓
     └────────[docker pause]──→ Paused
     │                          │
     └────────[docker unpause]──┘
     │
     └───[docker rm]────→ Deleted
```

### 状态说明

| 状态         | 说明           | docker ps 显示       |
| ------------ | -------------- | -------------------- |
| `created`    | 已创建，未启动 | 不显示               |
| `running`    | 正在运行       | ✓ 显示               |
| `paused`     | 暂停中         | ✓ 显示（Paused）     |
| `restarting` | 重启中         | ✓ 显示（Restarting） |
| `exited`     | 已停止         | 不显示（加 -a 显示） |
| `dead`       | 崩溃或无法停止 | 不显示（加 -a 显示） |

## 创建并运行容器

### docker run 命令

**基本语法**：

```bash
docker run [OPTIONS] IMAGE [COMMAND] [ARG...]
```

**最简单的示例**：

```bash
# 运行 Hello World（输出后立即退出）
docker run hello-world

# 迓出：
# Hello from Docker!
# This message shows that your installation appears to be working correctly.
```

### 常用选项详解

```bash
# 完整示例
docker run \
  -d \                          # 后台运行
  --name my-nginx \              # 指定容器名称
  -p 8080:80 \                  # 端口映射（宿主机:容器）
  -v /host/path:/container/path \ # 挂载数据卷
  -e ENV_VAR=value \             # 设置环境变量
  --restart=unless-stopped \     # 重启策略
  --memory="512m" \              # 内存限制
  --cpus="1.5" \                 # CPU 限制
  nginx:latest                   # 镜像名称
```

### 交互式运行容器

```bash
# -i: 保持 STDIN 打开
# -t: 分配伪终端
docker run -it ubuntu:22.04 /bin/bash

# 现在你进入了容器内部
root@abc123:/# pwd
/
root@abc123:/# whoami
root
root@abc123:/# exit

# 容器退出后会停止
```

### 后台运行容器

```bash
# -d: detached（后台运行）
docker run -d --name web-server nginx

# 查看运行中的容器
docker ps

# 输出：
# CONTAINER ID   IMAGE     COMMAND                  CREATED         STATUS         PORTS                 NAMES
# abc123def456   nginx     "/docker-entrypoint.…"   5 seconds ago   Up 4 seconds   0.0.0.0:80->80/tcp   web-server
```

## 查看容器

### docker ps 命令

```bash
# 查看运行中的容器
docker ps

# 查看所有容器（包括已停止的）
docker ps -a

# 只显示容器 ID
docker ps -q

# 显示完整的容器 ID（64 位）
docker ps --no-trunc

# 显示容器大小
docker ps -s
```

### 输出字段说明

```
CONTAINER ID   IMAGE      COMMAND                  CREATED        STATUS                    PORTS                    NAMES
abc123def456   nginx      "/docker-entrypoint.…"   5 minutes ago  Up 5 minutes              0.0.0.0:8080->80/tcp      web-server
│               │           │                       │              │                          │                         │
│               │           │                       │              │                          │                         └─ 容器名称
│               │           │                       │              │                          └─────────────────────────── 端口映射
│               │           │                       │              └──────────────────────────────────────── 状态（运行时长）
│               │           │                       └─────────────────────────────────────────────────── 创建时间
│               │           └────────────────────────────────────────────────────────────────────── 启动命令
│               └───────────────────────────────────────────────────────────────────────────────── 使用的镜像
└────────────────────────────────────────────────────────────────────────────────────────────── 容器 ID（前12位）
```

### 查看容器详情

```bash
# 查看容器详细信息（JSON 格式）
docker inspect web-server

# 查看特定字段
docker inspect --format='{{.State.Status}}' web-server
docker inspect --format='{{.NetworkSettings.IPAddress}}' web-server
docker inspect --format='{{range $p, $conf := .NetworkSettings.Ports}}{{$p}} -> {{(index $conf 0).HostPort}}{{end}}' web-server
```

### 实时监控容器

```bash
# 实时查看容器资源使用情况
docker stats

# 输出：
# CONTAINER ID   NAME      CPU %     MEM USAGE / LIMIT   MEM %     NET I/O     BLOCK I/O    PIDS
# abc123def456   web-server 0.02%     2.5MiB / 512MiB      0.49%     1.2kB / 0B  0B / 0B      2

# 只显示特定容器
docker stats web-server

# 不显示流式输出
docker stats --no-stream
```

## 容器管理操作

### 启动容器

```bash
# 启动已停止的容器
docker start web-server

# 启动多个容器
docker start web-server database cache

# 启动并附加到容器（前台运行）
docker start -a web-server
```

### 停止容器

```bash
# 优雅停止（发送 SIGTERM 信号）
docker stop web-server

# 等待 10 秒后强制停止
docker stop -t 10 web-server

# 强制停止（发送 SIGKILL 信号）
docker kill web-server

# 停止所有运行中的容器
docker stop $(docker ps -q)
```

### 重启容器

```bash
# 重启容器
docker restart web-server

# 等待 20 秒后重启
docker restart -t 20 web-server

# 重启所有容器
docker restart $(docker ps -q)
```

### 暂停和恢复

```bash
# 暂停容器（冻结进程）
docker pause web-server

# 恢复容器
docker unpause web-server

# 暂停所有容器
docker pause $(docker ps -q)
```

## 删除容器

### docker rm 命令

```bash
# 删除已停止的容器
docker rm web-server

# 强制删除运行中的容器
docker rm -f web-server

# 删除多个容器
docker rm web-server database cache

# 删除所有已停止的容器
docker container prune

# 删除所有容器（包括运行中的）
docker rm -f $(docker ps -aq)
```

### 删除前确认

```bash
# 交互式删除
docker ps -a
docker rm web-server  # 确认后手动删除

# 安全脚本：删除所有已停止的容器
docker container prune --filter "until=24h"
```

## 进入容器

### docker exec 命令

**进入正在运行的容器**：

```bash
# 分配伪终端并进入
docker exec -it web-server /bin/bash

# 或使用 sh（alpine 镜像）
docker exec -it web-server /bin/sh

# 现在你在容器内部
root@abc123:/# ls
bin  boot  dev  etc  home  lib  ...
root@abc123:/# exit
```

### 执行单个命令

```bash
# 在容器中执行命令（不进入）
docker exec web-server ls /usr/share/nginx/html

# 执行多个命令
docker exec web-server bash -c "cd /tmp && ls"

# 以特定用户执行
docker exec -u postgres db-container psql -U postgres

# 在后台执行命令
docker exec -d web-server apt-get update
```

### 实用场景

```bash
# 1. 查看容器日志目录
docker exec web-server ls /var/log/nginx

# 2. 查看配置文件
docker exec web-server cat /etc/nginx/nginx.conf

# 3. 重启容器内的服务
docker exec web-server nginx -s reload

# 4. 在容器中安装包
docker exec web-server apt-get update && docker exec web-server apt-get install -y curl

# 5. 复制文件到容器
echo "hello" | docker exec -i web-server tee /tmp/test.txt
```

## 容器日志

### docker logs 命令

```bash
# 查看容器日志
docker logs web-server

# 实时跟踪日志（类似 tail -f）
docker logs -f web-server

# 显示最后 100 行
docker logs --tail 100 web-server

# 显示最近 10 分钟的日志
docker logs --since 10m web-server

# 显示特定时间段的日志
docker logs --since 2024-01-01T00:00:00 --until 2024-01-01T23:59:59 web-server

# 显示时间戳
docker logs -t web-server
```

### 日志管理

```bash
# 查看日志大小
docker inspect web-server --format='{{.LogPath}}' | xargs du -h

# 限制日志文件大小（在 daemon.json 中配置）
{
  "log-driver": "json-file",
  "log-opts": {
    "max-size": "10m",
    "max-file": "3"
  }
}

# 清空容器日志
docker logs web-server 2>/dev/null | wc -l  # 查看日志行数
sudo truncate -s 0 $(docker inspect --format='{{.LogPath}}' web-server)
```

## 资源限制

### 内存限制

```bash
# 限制容器最多使用 512MB 内存
docker run -d --memory="512m" nginx

# 限制内存和交换空间
docker run -d --memory="512m" --memory-swap="1g" nginx

# 禁用交换空间
docker run -d --memory="512m" --memory-swap="512m" nginx

# 限制内存预留（至少使用 256MB）
docker run -d --memory-reservation="256m" nginx
```

### CPU 限制

```bash
# 限制容器最多使用 1.5 个 CPU
docker run -d --cpus="1.5" nginx

# 限制容器只能使用 CPU 0 和 CPU 1
docker run -d --cpuset-cpus="0,1" nginx

# 设置 CPU 权重（相对权重，默认 1024）
docker run -d --cpu-shares=512 nginx
```

### 磁盘限制

```bash
# 限制磁盘读取速度为 10MB/s
docker run -d --device-read-bps /dev/sda:10mb nginx

# 限制磁盘写入速度为 5MB/s
docker run -d --device-write-bps /dev/sda:5mb nginx
```

### 综合示例

```bash
docker run -d \
  --name limited-app \
  --memory="512m" \
  --memory-swap="1g" \
  --cpus="1.0" \
  --pids-limit 100 \
  --restart=unless-stopped \
  nginx
```

## 端口映射

### 基本映射

```bash
# 映射单个端口（宿主机:容器）
docker run -d -p 8080:80 nginx

# 映射多个端口
docker run -d \
  -p 80:80 \
  -p 443:443 \
  nginx

# 映射到随机端口
docker run -d -p 80 nginx
# 宿主机端口会随机分配（如 49153）
```

### 高级映射

```bash
# 绑定到特定接口
docker run -d -p 127.0.0.1:8080:80 nginx    # 只监听 localhost
docker run -d -p 0.0.0.0:8080:80 nginx       # 监听所有接口

# 映射 UDP 端口
docker run -d -p 53:53/udp nginx

# 映射所有端口（不推荐）
docker run -d -P nginx
```

### 查看端口映射

```bash
# 查看容器的端口映射
docker port web-server

# 输出：
# 80/tcp -> 0.0.0.0:8080
# 443/tcp -> 0.0.0.0:8443

# 使用 inspect 查看
docker inspect --format='{{range $p, $conf := .NetworkSettings.Ports}}{{$p}} -> {{(index $conf 0).HostPort}}{{"\n"}}{{end}}' web-server
```

## 自动重启

### 重启策略

```bash
# no: 不自动重启（默认）
docker run --restart=no nginx

# on-failure: 只有在错误退出时重启
docker run --restart=on-failure nginx

# on-failure:5: 最多重启 5 次
docker run --restart=on-failure:5 nginx

# always: 总是重启（除非手动停止）
docker run --restart=always nginx

# unless-stopped: 总是重启（除非手动停止或 Docker 守护进程重启）
docker run --restart=unless-stopped nginx
```

### 使用场景

```bash
# Web 服务器（需要持续运行）
docker run -d --restart=unless-stopped --name web nginx

# 一次性任务（不重启）
docker run --restart=no my-script

# 数据库（需要持续运行）
docker run -d --restart=unless-stopped --name db postgres:15
```

## 数据卷管理

### 创建和使用卷

```bash
# 创建命名卷
docker volume create my-data

# 挂载卷到容器
docker run -d -v my-data:/app/data nginx

# 查看卷详情
docker volume inspect my-data
```

### 绑定挂载

```bash
# 挂载宿主机目录到容器
docker run -d -v /host/path:/container/path nginx

# 只读挂载
docker run -d -v /host/path:/container/path:ro nginx

# 挂载单个文件
docker run -d -v /host/config.conf:/etc/nginx/nginx.conf nginx
```

## 实战演练

### 场景 1：运行 Nginx Web 服务器

```bash
# 1. 拉取镜像
docker pull nginx:alpine

# 2. 运行容器
docker run -d \
  --name my-web \
  -p 8080:80 \
  -v $(pwd)/html:/usr/share/nginx/html:ro \
  --restart=unless-stopped \
  nginx:alpine

# 3. 创建测试页面
mkdir -p html
echo "<h1>Hello Docker!</h1>" > html/index.html

# 4. 访问测试
curl http://localhost:8080

# 5. 查看日志
docker logs -f my-web

# 6. 停止和删除
docker stop my-web
docker rm my-web
```

### 场景 2：运行 PostgreSQL 数据库

```bash
# 1. 创建数据卷
docker volume create postgres-data

# 2. 运行容器
docker run -d \
  --name postgres-db \
  -e POSTGRES_PASSWORD=mypassword \
  -e POSTGRES_DB=myapp \
  -v postgres-data:/var/lib/postgresql/data \
  -p 5432:5432 \
  --restart=unless-stopped \
  postgres:15-alpine

# 3. 等待启动
sleep 10

# 4. 连接数据库
docker exec -it postgres-db psql -U postgres -d myapp

# 5. 执行 SQL
CREATE TABLE users (id SERIAL PRIMARY KEY, name TEXT);
INSERT INTO users (name) VALUES ('Alice'), ('Bob');
SELECT * FROM users;

# 6. 退出
\q
```

### 场景 3：容器间通信

```bash
# 1. 创建自定义网络
docker network create my-app-network

# 2. 运行数据库
docker run -d \
  --name db \
  --network my-app-network \
  -e POSTGRES_PASSWORD=secret \
  postgres:15

# 3. 运行应用
docker run -d \
  --name app \
  --network my-app-network \
  -e DATABASE_URL=postgres://postgres:secret@db:5432/myapp \
  myapp:latest

# 4. 测试连接
docker exec app ping db
```

## 最佳实践

### 1. 容器命名

```bash
# ✅ 推荐：使用有意义的名称
docker run -d --name web-server nginx
docker run -d --name api-gateway nginx
docker run -d --name cache-db redis

# ❌ 避免：让 Docker 自动生成
docker run -d nginx  # 生成类似 nostalgic_brown
```

### 2. 资源限制

```bash
# ✅ 推荐：设置资源限制
docker run -d \
  --memory="512m" \
  --cpus="1.0" \
  nginx

# ❌ 避免：不限制资源
docker run -d nginx  # 可能占满宿主机资源
```

### 3. 日志管理

```bash
# ✅ 推荐：配置日志轮转
docker run -d \
  --log-driver json-file \
  --log-opt max-size=10m \
  --log-opt max-file=3 \
  nginx
```

### 4. 数据持久化

```bash
# ✅ 推荐：使用卷管理数据
docker run -d -v db-data:/var/lib/data postgres

# ❌ 避免：数据存储在容器内
docker run -d postgres  # 容器删除后数据丢失
```

### 5. 安全配置

```bash
# ✅ 推荐：以非 root 用户运行
docker run -d --user 1000:1000 nginx

# ✅ 推荐：只读文件系统
docker run -d --read-only nginx

# ✅ 推荐：限制能力
docker run -d --cap-drop=ALL --cap-add=NET_BIND_SERVICE nginx
```

## 常见问题

### Q1: 容器启动后立即退出？

**答**：检查以下几点：

1. 容器的主进程是否正常运行
2. 查看容器日志：`docker logs <container>`
3. 尝试交互式运行：`docker run -it <image> /bin/bash`

### Q2: 如何在宿主机和容器间复制文件？

**答**：

```bash
# 从宿主机复制到容器
docker cp file.txt container:/path/

# 从容器复制到宿主机
docker cp container:/path/file.txt ./
```

### Q3: 如何查看容器的 IP 地址？

**答**：

```bash
docker inspect --format='{{.NetworkSettings.IPAddress}}' container

# 或使用 inspect 查看完整网络配置
docker inspect container | grep -A 20 "NetworkSettings"
```

### Q4: 容器内如何访问宿主机服务？

**答**：

```bash
# Linux: 使用 host.docker.internal
# 或使用宿主机网关 IP
docker run -e HOST_HOSTNAME=host.docker.internal myapp

# 查看宿主机 IP
docker inspect bridge | grep Gateway
```

### Q5: 如何查看容器的实时资源使用？

**答**：

```bash
# 实时监控
docker stats

# 查看单个容器
docker stats container

# 不显示流式输出
docker stats --no-stream
```

**重要概念**：

- 容器生命周期：created → running → stopped → deleted
- 资源限制：内存、CPU、磁盘
- 数据持久化：卷和绑定挂载
- 网络配置：端口映射和网络模式
