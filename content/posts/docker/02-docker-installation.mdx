## 前言

在深入使用 Docker 之前，我们需要先在系统上安装它。Docker 支持多种操作系统，每个平台的安装方式略有不同。本篇文章将详细介绍在不同平台上安装 Docker 的方法，并配置一个高效、稳定的开发环境。

**学习目标**：

- 在 Linux/macOS/Windows 上安装 Docker
- 配置国内镜像加速器
- 理解 Docker 的用户权限管理
- 掌握常用配置选项

## 系统要求

### Linux 系统

**支持的发行版**：

- ✅ Ubuntu 20.04/22.04/24.04 LTS
- ✅ Debian 11/12
- ✅ CentOS 7/8/9 Stream
- ✅ Fedora 38/39
- ✅ RHEL 8/9

**最低要求**：

- CPU：64 位处理器
- 内存：2 GB RAM（推荐 4 GB）
- 磁盘：20 GB 可用空间
- 内核：3.10 或更高版本

### macOS 系统

**支持的版本**：

- ✅ macOS 11 Big Sur 或更高
- ✅ Apple Silicon (M1/M2/M3) 或 Intel 芯片

**要求**：

- 内存：4 GB RAM（推荐 8 GB）
- 磁盘：10 GB 可用空间

### Windows 系统

**支持的版本**：

- ✅ Windows 11 64-bit：专业版、企业版、教育版
- ✅ Windows 10 64-bit：专业版、企业版、教育版（Build 19041 或更高）

**要求**：

- 启用 Hyper-V 和容器功能
- 启用 WSL 2 后端
- 内存：4 GB RAM（推荐 8 GB）

## Linux 安装

### Ubuntu 安装（推荐方式）

```bash
# 1. 更新包索引
sudo apt-get update

# 2. 安装必要的依赖包
sudo apt-get install -y \
    ca-certificates \
    curl \
    gnupg \
    lsb-release

# 3. 添加 Docker 官方 GPG 密钥
sudo mkdir -m 0755 -p /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg

# 4. 设置 Docker 仓库
echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu \
  $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# 5. 更新包索引
sudo apt-get update

# 6. 安装 Docker Engine
sudo apt-get install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

# 7. 验证安装
sudo docker run hello-world
```

**输出示例**：

```
Hello from Docker!
This message shows that your installation appears to be working correctly.
```

### CentOS/RHEL 安装

```bash
# 1. 安装 yum-utils
sudo yum install -y yum-utils

# 2. 添加 Docker 仓库
sudo yum-config-manager \
    --add-repo \
    https://download.docker.com/linux/centos/docker-ce.repo

# 3. 安装 Docker
sudo yum install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

# 4. 启动 Docker
sudo systemctl start docker
sudo systemctl enable docker

# 5. 验证安装
sudo docker run hello-world
```

### 便捷安装脚本（仅用于测试）

```bash

# ⚠️ 注意：
# - 仅用于开发和测试环境
# - 生产环境不推荐使用
# - 脚本需要 root 权限
```

## Windows 安装

### WSL 2 安装步骤

```powershell
# 1. 以管理员身份打开 PowerShell
# 启用 WSL 功能
dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart

# 启用虚拟机平台
dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart

# 2. 重启计算机
Restart-Computer

# 3. 下载并安装 WSL 2 Linux 内核更新
# 访问：https://aka.ms/wsl2kernel
# 下载并运行安装包

# 4. 设置 WSL 2 为默认版本
wsl --set-default-version 2
```

### Docker Desktop 安装

1. **下载安装包**：
   - 访问：https://www.docker.com/products/docker-desktop/
   - 下载 Windows 版本

2. **安装**：

   - 确保勾选"Use WSL 2 instead of Hyper-V"

3. **配置 WSL 2**：

```powershell
# 查看 WSL 状态

# 输出示例：

```

4. **验证安装**：

```bash
docker --version
```


## 镜像加速器配置

由于网络原因，从 Docker Hub 拉取镜像可能很慢。配置国内镜像加速器可以大幅提升速度。

### 阿里云镜像加速器（推荐）

**获取加速地址**：

1. 访问阿里云容器镜像服务：https://cr.console.aliyun.com/
2. 注册/登录账号
3. 获取专属加速地址（格式：`https://xxxxxx.mirror.aliyuncs.com`）

**配置方式**：

**Linux/macOS**：

```bash
# 1. 创建或修改 Docker 配置文件
sudo mkdir -p /etc/docker
sudo tee /etc/docker/daemon.json <<-'EOF'
{
  "registry-mirrors": [
    "https://xxxxxx.mirror.aliyuncs.com"
  ],
  "log-driver": "json-file",
  "log-opts": {
    "max-size": "100m"
  }
}
EOF

# 2. 重启 Docker
sudo systemctl daemon-reload
sudo systemctl restart docker

# 3. 验证配置
docker info | grep -A 10 "Registry Mirrors"
```

**Docker Desktop（macOS/Windows）**：

1. 打开 Docker Desktop
2. 进入 Settings → Docker Engine
3. 添加配置：

```json
{
  "registry-mirrors": ["https://xxxxxx.mirror.aliyuncs.com"]
}
```

4. 点击"Apply & Restart"

### 其他国内镜像源

```json
{
  "registry-mirrors": [
    "https://docker.1panel.live",
    "https://dockerhub.icu",
    "https://docker.chenby.cn",
    "https://docker.awsl9527.cn"
  ]
}
```

### 测试镜像加速

```bash
# 拉取大镜像测试速度
time docker pull nginx:latest

# 对比前后速度差异
# 配置前：可能需要 5-10 分钟
# 配置后：通常 10-30 秒
```

## 用户权限管理

### 问题：sudo 权限

默认情况下，只有 root 用户和 docker 组的用户才能访问 Docker 引擎。

```bash
# 非 root 用户运行会报错
docker run hello-world
# 输出：
# permission denied while trying to connect to the Docker daemon socket
```

### 解决方案 1：将用户添加到 docker 组

```bash
# 1. 创建 docker 组（如果不存在）
sudo groupadd docker

# 2. 将当前用户添加到 docker 组
sudo usermod -aG docker $USER

# 3. 激活组的更改
newgrp docker

# 4. 验证（无需 sudo）
docker run hello-world
```

**⚠️ 安全警告**：

- docker 组与 root 用户权限等同
- 只将可信用户添加到 docker 组
- 生产环境需谨慎使用

### 解决方案 2：使用 sudo（更安全）

```bash
# 为常用命令创建别名
alias docker='sudo docker'
alias docker-compose='sudo docker compose'

# 添加到 ~/.bashrc 或 ~/.zshrc
echo "alias docker='sudo docker'" >> ~/.bashrc
source ~/.bashrc
```

### Rootless Docker（更安全）

Rootless 模式允许非 root 用户运行 Docker，无需 sudo 权限。

```bash
# 1. 安装相关工具
sudo apt-get install -y uidmap

# 2. 运行 rootless 安装脚本
curl -fsSL https://get.docker.com/rootless | sh

# 3. 添加环境变量
echo 'export PATH=/home/$USER/bin:$PATH' >> ~/.bashrc
source ~/.bashrc

# 4. 启用 systemd 服务
systemctl --user start docker.service
systemctl --user enable docker.service

# 5. 验证
docker run hello-world
```

## 常用配置选项

### daemon.json 完整配置示例

```json
{
  "registry-mirrors": ["https://xxxxxx.mirror.aliyuncs.com"],
  "log-driver": "json-file",
  "log-opts": {
    "max-size": "100m",
    "max-file": "3"
  },
  "storage-driver": "overlay2",
  "data-root": "/var/lib/docker",
  "live-restore": true,
  "default-ulimits": {
    "nofile": {
      "Name": "nofile",
      "Hard": 64000,
      "Soft": 64000
    }
  },
  "max-concurrent-downloads": 10,
  "max-concurrent-uploads": 5,
  "debug": false,
  "experimental": false
}
```

### 配置项说明

| 配置项                     | 说明                     | 推荐值        |
| -------------------------- | ------------------------ | ------------- |
| `registry-mirrors`         | 镜像加速器               | 国内必配      |
| `log-max-size`             | 单个日志文件最大大小     | 100m          |
| `log-max-file`             | 日志文件数量             | 3             |
| `storage-driver`           | 存储驱动                 | overlay2      |
| `live-restore`             | 守护进程重启时不停止容器 | true          |
| `max-concurrent-downloads` | 最大并发下载数           | 10            |
| `debug`                    | 调试模式                 | false（生产） |

## 故障排查

### 问题 1：容器无法访问外网

**症状**：

```bash
docker run ubuntu apt-get update
# 无法连接到 archive.ubuntu.com
```

**解决**：

```bash
# 1. 检查 DNS 配置
cat /etc/resolv.conf

# 2. 在 daemon.json 中配置 DNS
sudo tee /etc/docker/daemon.json <<-'EOF'
{
  "dns": ["8.8.8.8", "8.8.4.4"]
}
EOF

# 3. 重启 Docker
sudo systemctl restart docker
```

### 问题 2：容器启动失败

**症状**：

```bash
docker run hello-world
# Error response from daemon: ...
```

**解决**：

```bash
# 1. 查看 Docker 状态
sudo systemctl status docker

# 2. 查看 Docker 日志
sudo journalctl -u docker -n 50

# 3. 清理并重启
sudo systemctl stop docker
sudo rm -rf /var/lib/docker/*
sudo systemctl start docker
```

### 问题 3：WSL 2 问题（Windows）

**症状**：

```
error during connect: This error may indicate that the docker daemon is not running
```

**解决**：

```powershell
# 1. 检查 WSL 版本
wsl --list --verbose

# 2. 确保 WSL 2 已启用
wsl --set-default-version 2

# 3. 重启 WSL
wsl --shutdown
# 然后重新打开终端

# 4. 重置 Docker Desktop
# 在 Docker Desktop 中：Troubleshoot → Clean/Purge data
```

### 问题 4：权限被拒绝

**症状**：

```bash
docker ps
# Got permission denied while trying to connect to the Docker daemon socket
```

**解决**：

```bash
# 方案 1：添加到 docker 组
sudo usermod -aG docker $USER
newgrp docker

# 方案 2：修复 socket 权限
sudo chmod 666 /var/run/docker.sock
```

## 最佳实践

### 1. 定期清理

```bash

# 清理未使用的容器
```
