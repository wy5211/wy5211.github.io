## 前言

学习 Docker 后，你可能会听到 Kubernetes（K8s）。本篇文章将对比 Docker 和 K8s，帮助你理解它们的关系和选择标准。

## 核心区别

### Docker vs Kubernetes

```
Docker：容器引擎
├─ 构建镜像
├─ 运行容器
├─ 单机管理
└─ 适合：开发环境、小型项目

Kubernetes：容器编排平台
├─ 管理多个 Docker 容器
├─ 跨主机集群管理
├─ 自动扩缩容、自愈
└─ 适合：生产环境、大规模应用
```

| 特性           | Docker            | Kubernetes       |
| -------------- | ----------------- | ---------------- |
| **定位**       | 容器引擎          | 容器编排平台     |
| **管理范围**   | 单机              | 多机集群         |
| **自动重启**   | 支持（--restart） | 支持，更智能     |
| **负载均衡**   | 手动配置          | 内置 Service     |
| **自动扩缩容** | 不支持            | 支持（HPA）      |
| **滚动更新**   | 手动              | 自动             |
| **服务发现**   | 不支持            | 内置             |
| **存储管理**   | Volume            | PV/PVC           |
| **配置管理**   | env_file          | ConfigMap/Secret |
| **复杂度**     | 低                | 高               |

## Docker Compose vs Kubernetes

### Docker Compose 配置

```yaml
version: "3.8"

services:
  web:
    image: nginx:alpine
    ports:
      - "80:80"
    deploy:
      replicas: 3
      resources:
        limits:
          cpus: "0.5"
          memory: 512M

  api:
    image: myapp:latest
    environment:
      - DATABASE_URL=postgres://db:5432/app
    depends_on:
      - db

  db:
    image: postgres:15
    volumes:
      - postgres-data:/var/lib/postgresql/data

volumes:
  postgres-data:
```

### Kubernetes 配置

```yaml
# Deployment
apiVersion: apps/v1
kind: Deployment
metadata:
  name: web
spec:
  replicas: 3
  selector:
    matchLabels:
      app: web
  template:
    metadata:
      labels:
        app: web
    spec:
      containers:
        - name: nginx
          image: nginx:alpine
          ports:
            - containerPort: 80
          resources:
            limits:
              cpu: "500m"
              memory: "512Mi"

---
# Service
apiVersion: v1
kind: Service
metadata:
  name: web
spec:
  selector:
    app: web
  ports:
    - port: 80
      targetPort: 80
  type: LoadBalancer

---
# ConfigMap
apiVersion: v1
kind: ConfigMap
metadata:
  name: app-config
data:
  DATABASE_URL: "postgres://db:5432/app"

---
# PersistentVolumeClaim
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: postgres-data
spec:
  accessModes:
    - ReadWriteOnce
  resources:
    requests:
      storage: 10Gi
```

## 适用场景

### 使用 Docker 的场景

#### 1. 本地开发

```bash
# 快速启动开发环境
docker compose up -d
```

#### 2. 小型项目

```yaml
# 单机部署，简单应用
version: "3.8"
services:
  web:
    image: myapp:latest
    ports:
      - "80:80"
```

#### 3. 测试环境

```bash
# 隔离测试环境
docker compose -f docker-compose.test.yml up
```

#### 4. CI/CD 流水线

```yaml
# 构建和测试
docker build -t myapp:test .
docker run myapp:test npm test
```

### 使用 Kubernetes 的场景

#### 1. 大规模生产环境

```yaml
# 管理数千个容器
apiVersion: apps/v1
kind: Deployment
metadata:
  name: web
spec:
  replicas: 100
```

#### 2. 高可用要求

```yaml
# 跨节点部署，自动故障转移
apiVersion: apps/v1
kind: Deployment
spec:
  replicas: 5
  strategy:
    type: RollingUpdate
    rollingUpdate:
      maxSurge: 1
      maxUnavailable: 0
```

#### 3. 自动扩缩容

```yaml
# HPA：根据 CPU/内存自动扩容
apiVersion: autoscaling/v2
kind: HorizontalPodAutoscaler
metadata:
  name: web
spec:
  scaleTargetRef:
    apiVersion: apps/v1
    kind: Deployment
    name: web
  minReplicas: 2
  maxReplicas: 10
  metrics:
    - type: Resource
      resource:
        name: cpu
        target:
          type: Utilization
          averageUtilization: 80
```

#### 4. 复杂微服务架构

```yaml
# 服务网格、流量管理
apiVersion: networking.istio.io/v1beta1
kind: VirtualService
metadata:
  name: reviews
spec:
  hosts:
    - reviews
  http:
    - match:
        - headers:
            end-user:
              exact: jason
      route:
        - destination:
            host: reviews
            subset: v2
    - route:
        - destination:
            host: reviews
            subset: v1
```

## 功能对比

### 服务发现

#### Docker

```bash
# 容器间通过服务名访问
docker exec web ping api
```

#### Kubernetes

```yaml
# 自动 DNS
apiVersion: v1
kind: Service
metadata:
  name: api
spec:
  selector:
    app: api
  ports:
    - port: 80
      targetPort: 8080

# 其他 Pod 访问：http://api:80
```

### 负载均衡

#### Docker

```yaml
# 需要外部 Nginx
services:
  nginx:
    image: nginx:alpine
    ports:
      - "80:80"
    volumes:
      - ./nginx.conf:/etc/nginx/nginx.conf
```

#### Kubernetes

```yaml
# 内置 Service
apiVersion: v1
kind: Service
metadata:
  name: web
spec:
  selector:
    app: web
  ports:
    - port: 80
      targetPort: 8080
  type: LoadBalancer # 自动分配外部 IP
```

### 配置管理

#### Docker

```yaml
# 使用 env_file
services:
  app:
    env_file:
      - .env.production
```

#### Kubernetes

```yaml
# ConfigMap
apiVersion: v1
kind: ConfigMap
metadata:
  name: app-config
data:
  DATABASE_URL: "postgres://db:5432/app"
  REDIS_URL: "redis://redis:6379"

---
# Secret
apiVersion: v1
kind: Secret
metadata:
  name: app-secret
type: Opaque
data:
  API_KEY: YWJjZGVmZ2hpams=
```

### 存储管理

#### Docker

```yaml
# 简单 Volume
services:
  db:
    volumes:
      - postgres-data:/var/lib/postgresql/data

volumes:
  postgres-data:
```

#### Kubernetes

```yaml
# PV：存储资源
apiVersion: v1
kind: PersistentVolume
metadata:
  name: postgres-pv
spec:
  capacity:
    storage: 10Gi
  accessModes:
    - ReadWriteOnce
  persistentVolumeReclaimPolicy: Retain
  hostPath:
    path: /data/postgres

---
# PVC：存储声明
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: postgres-pvc
spec:
  accessModes:
    - ReadWriteOnce
  resources:
    requests:
      storage: 10Gi
```

## 学习路径

### 第一阶段：Docker（1-2 周）

✅ **你已经完成**：

- Docker 核心概念
- 镜像和容器管理
- Dockerfile 编写
- Docker Compose 编排
- 网络和存储
- 生产部署

### 第二阶段：Kubernetes 基础（2-4 周）

```bash
# 本地 K8s 环境
minikube start
kubectl create deployment hello --image=nginx
kubectl expose deployment hello --port=80
```

**学习内容**：

- 核心概念：Pod、Deployment、Service
- 配置管理：ConfigMap、Secret
- 存储：PV、PVC、StorageClass
- 网络：Service、Ingress

### 第三阶段：Kubernetes 进阶（4-8 周）

**学习内容**：

- 高级控制器：StatefulSet、DaemonSet
- 自动扩缩容：HPA、VPA
- 滚动更新和回滚
- 监控和日志：Prometheus、Loki
- 服务网格：Istio、Linkerd

### 第四阶段：云原生生态（持续学习）

- Helm：包管理器
- ArgoCD：GitOps
- Prometheus + Grafana：监控
- Fluentd/ Loki：日志
- Jenkins/GitLab CI：CI/CD

## 迁移指南

### 从 Docker Compose 到 Kubernetes

#### 1. 安装工具

```bash
# 安装 kubectl
curl -LO "https://dl.k8s.io/release/$(curl -L -s https://dl.k8s.io/release/stable.txt)/bin/darwin/amd64/kubectl"

# 安装 kind（本地 K8s）
brew install kind

# 创建集群
kind create cluster --name mycluster
```

#### 2. 转换配置

```bash
# 使用 Kompose 自动转换
kompose convert

# 生成文件：
# - web-deployment.yaml
# - web-service.yaml
# - db-deployment.yaml
# - db-service.yaml
```

#### 3. 部署到 K8s

```bash
kubectl apply -f .
kubectl get pods
kubectl get services
```

## 实战建议

### 何时选择 Docker

- ✅ 个人项目或小型团队
- ✅ 单机部署
- ✅ 快速原型开发
- ✅ 测试环境
- ✅ 简单的微服务（< 5 个服务）

### 何时选择 Kubernetes

- ✅ 大规模生产环境（> 10 个服务）
- ✅ 需要高可用和自动扩缩容
- ✅ 多团队协作
- ✅ 复杂的微服务架构
- ✅ 需要蓝绿部署、金丝雀发布

## 最佳实践

### Docker 最佳实践

1. **使用多阶段构建**：减小镜像体积
2. **固定版本标签**：避免意外更新
3. **健康检查**：确保容器健康
4. **资源限制**：防止资源耗尽
5. **日志管理**：输出到 stdout/stderr

### Kubernetes 最佳实践

1. **资源限制**：设置 requests 和 limits
2. **健康检查**：liveness 和 readiness probe
3. **标签选择器**：规范的标签管理
4. **命名空间**：环境隔离
5. **监控告警**：Prometheus + Grafana

## 常用命令对比

### Docker vs Kubectl

| 操作     | Docker              | Kubectl                    |
| -------- | ------------------- | -------------------------- |
| 运行容器 | `docker run`        | `kubectl run`              |
| 查看容器 | `docker ps`         | `kubectl get pods`         |
| 查看日志 | `docker logs`       | `kubectl logs`             |
| 进入容器 | `docker exec`       | `kubectl exec`             |
| 删除容器 | `docker rm`         | `kubectl delete pod`       |
| 查看镜像 | `docker images`     | `kubectl get pods -o wide` |
| 查看网络 | `docker network ls` | `kubectl get services`     |

## 总结

Docker 和 Kubernetes 的关系：

- **Docker**：容器运行时，负责构建和运行容器
- **Kubernetes**：容器编排平台，管理多个 Docker 容器

**选择建议**：

- 学习阶段：先 Docker，后 K8s
- 开发环境：Docker Compose
- 生产环境：Kubernetes
