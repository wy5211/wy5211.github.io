## 前言

使用 Docker 过程中难免会遇到各种问题。本篇文章将系统介绍 Docker 故障排查的方法和工具。

## 诊断工具

### docker inspect

```bash
# 查看容器详细信息
docker inspect web

# 查看特定字段
docker inspect -f '{{.State.Status}}' web
docker inspect -f '{{.NetworkSettings.IPAddress}}' web

# 查看挂载点
docker inspect -f '{{.Mounts}}' web
```

### docker events

```bash
# 实时查看 Docker 事件
docker events

# 查看最近的事件
docker events --since '1h'

# 过滤特定事件
docker events --filter container=web
```

### docker logs

```bash
# 查看日志
docker logs web

# 实时跟踪
docker logs -f web

# 查看最后 100 行
docker logs --tail 100 web

# 带时间戳
docker logs -t web

# 查看特定时间范围
docker logs --since 1h web
```

## 容器启动失败

### 常见原因

#### 1. 端口冲突

```bash
# 错误信息
Error: bind: address already in use

# 查看端口占用
netstat -tulpn | grep 80
lsof -i :80

# 解决方案：使用其他端口
docker run -p 8080:80 nginx
```

#### 2. 镜像不存在

```bash
# 错误信息
Unable to find image 'myapp:latest' locally

# 解决方案：拉取镜像
docker pull myapp:latest

# 或构建镜像
docker build -t myapp:latest .
```

#### 3. 命令不存在

```bash
# 错误信息
exec: "npm": executable file not found

# 检查 Dockerfile
FROM node:18-alpine
WORKDIR /app
COPY package*.json ./
RUN npm install  # 确保这里安装了依赖
COPY . .
CMD ["npm", "start"]
```

#### 4. 权限问题

```bash
# 错误信息
permission denied

# 解决方案 1：使用正确的用户
USER nodejs

# 解决方案 2：调整卷权限
docker run -v $(pwd)/data:/app/data:Z nginx

# 解决方案 3：使用 --user
docker run --user 1000:1000 myapp
```

## 容器运行异常

### 容器反复重启

```bash
# 查看重启次数
docker ps -a

# 查看退出码
docker inspect -f '{{.State.ExitCode}}' web

# 常见退出码
# 0: 正常退出
# 1: 一般错误
# 125: Docker daemon 错误
# 126: 命令不可执行
# 127: 命令未找到
# 137: SIGKILL (内存不足)
# 139: Segmentation fault
```

### 内存不足

```bash
# 检查内存限制
docker inspect -f '{{.HostConfig.Memory}}' web

# 临时增加内存
docker run -m 1g myapp

# 检查 OOM Killer
dmesg | grep -i kill
```

### CPU 限制

```bash
# 检查 CPU 使用
docker stats

# 调整 CPU 限制
docker update --cpus 2 web
```

## 网络问题

### 容器无法访问外部网络

```bash
# 检查 DNS
docker exec web cat /etc/resolv.conf

# 自定义 DNS
docker run --dns 8.8.8.8 myapp

# 检查防火墙
iptables -L -n

# 使用 host 网络
docker run --network host myapp
```

### 容器间无法通信

```bash
# 检查网络
docker network inspect mynetwork

# 确保在同一网络
docker network connect mynetwork web
docker network connect mynetwork db

# 测试连通性
docker exec web ping db
```

### 端口映射不生效

```bash
# 检查端口映射
docker port web

# 绑定到特定接口
docker run -p 0.0.0.0:80:80 myapp

# 检查防火墙
sudo ufw status
```

## 存储问题

### 磁盘空间不足

```bash
# 查看磁盘使用
docker system df

# 清理未使用的资源
docker system prune -a --volumes

# 清理镜像
docker image prune -a

# 清理容器
docker container prune

# 清理卷
docker volume prune
```

### 卷挂载失败

```bash
# 错误信息
ERROR: for db  Cannot create container for service db:
invalid mount path for volume

# 检查路径
docker inspect -f '{{.Mounts}}' db

# 创建目录
mkdir -p ./data

# 检查 SELinux
chcon -Rt svirt_sandbox_file_t ./data
```

### 数据持久化问题

```bash
# 检查卷
docker volume ls
docker volume inspect mydata

# 进入容器检查
docker exec -it myapp sh
ls -la /app/data

# 备份数据
docker run --rm \
  -v mydata:/data \
  -v $(pwd):/backup \
  alpine tar czf /backup/data.tar.gz /data
```

## 镜像问题

### 镜像拉取失败

```bash
# 网络问题
# 解决方案：使用镜像加速器
sudo tee /etc/docker/daemon.json <<-'EOF'
{
  "registry-mirrors": [
    "https://mirror.ccs.tencentyun.com"
  ]
}
EOF

sudo systemctl restart docker

# 认证问题
docker login registry.example.com
```

### 镜像构建失败

```bash
# 缓存问题
docker build --no-cache -t myapp .

# 查看构建详情
docker build --progress=plain -t myapp .

# 层缓存优化
COPY package*.json ./
RUN npm install
COPY . .
```

### 镜像体积过大

```bash
# 查看层大小
docker history myapp

# 多阶段构建
FROM golang:1.21 AS builder
WORKDIR /app
COPY . .
RUN go build -o app

FROM alpine:3.19
COPY --from=builder /app/app /app
```

## 性能问题

### 容器启动慢

```bash
# 优化启动时间
# 1. 使用更小的基础镜像
FROM alpine:3.19

# 2. 减少层数
RUN apt-get update && apt-get install -y \
    curl \
    vim \
    && rm -rf /var/lib/apt/lists/*

# 3. 利用缓存
COPY package*.json ./
RUN npm install
COPY . .
```

### 运行缓慢

```bash
# 性能分析
docker stats --no-stream

# 资源限制
docker run -m 512m --cpus 1.0 myapp

# I/O 性能
# 使用 volume 而不是 bind mount
docker run -v mydata:/app/data myapp
```

## 调试技巧

### 进入运行容器

```bash
# 交互式 shell
docker exec -it web sh

# 以 root 用户进入
docker exec -u 0 -it web sh

# 运行单个命令
docker exec web ps aux
docker exec web cat /etc/hosts
```

### 调试基础镜像

```bash
# 修改入口点
docker run -it --entrypoint sh myapp

# 覆盖命令
docker run -it myapp /bin/bash
```

### 查看容器进程

```bash
# 宿主机上查看
ps aux | grep docker

# 容器内查看
docker exec web ps aux

# 使用特权模式
docker run --privileged -it myapp
```

## 日志调试

### 应用日志

```dockerfile
# 确保应用输出到 stdout/stderr
FROM node:18-alpine

# 正确
CMD ["node", "server.js"]

# 错误（日志写文件）
CMD ["node", "server.js", "> /app/app.log"]

# 如果需要日志文件，同时输出到 stdout
CMD ["sh", "-c", "node server.js | tee /app/app.log"]
```

### Docker 守护进程日志

```bash
# systemd
journalctl -u docker.service

# 查看最近日志
journalctl -u docker.service -n 100

# 实时查看
journalctl -u docker.service -f
```

## 健康检查调试

### 健康检查失败

```bash
# 查看健康状态
docker inspect -f '{{.State.Health.Status}}' web

# 查看健康检查日志
docker inspect -f '{{range .State.Health.Log}}{{.Output}}{{end}}' web

# 手动测试
docker exec web curl http://localhost/health

# 修改健康检查
docker update --health-cmd "curl -f http://localhost/health || exit 1" web
docker update --health-interval 5s web
```

## 常用排查命令

### 快速诊断脚本

```bash
#!/bin/bash
# diagnose.sh

echo "=== Docker 版本 ==="
docker version

echo -e "\n=== 容器列表 ==="
docker ps -a

echo -e "\n=== 镜像列表 ==="
docker images

echo -e "\n=== 磁盘使用 ==="
docker system df

docker network ls

echo -e "\n=== 卷列表 ==="
docker volume ls

echo -e "\n=== 最近的容器 ==="
docker ps -a --format "table {{.Names}}\t{{.Status}}\t{{.Image}}"
```

## 故障排查流程

```
问题发生
    │
    ├─→ 容器状态检查 (docker ps)
    │       │
    │       ├─→ 未运行 → 查看日志 (docker logs)
    │       │             │
    │       │             ├─→ 启动错误 → 检查配置/镜像
    │       │             └─→ 运行时错误 → 检查应用
    │       │
    │       └─→ 运行中 → 健康检查 (docker inspect)
    │                   │
    │                   ├─→ 健康问题 → 资源检查 (docker stats)
    │                   └─→ 网络问题 → 网络诊断 (docker network inspect)
    │
    ├─→ 镜像问题 → 重新构建/拉取
    │
    └─→ 系统问题 → 检查磁盘/内存/日志
```

## 最佳实践

1. **日志集中**：所有日志输出到 stdout/stderr
2. **健康检查**：配置合适的健康检查
3. **资源限制**：设置 CPU 和内存限制
4. **监控告警**：使用 Prometheus + Grafana
5. **日志轮转**：配置日志大小和数量
6. **定期清理**：清理未使用的资源
7. **版本固定**：使用固定版本标签
8. **备份恢复**：重要数据定期备份

## 总结

故障排查关键：

- **日志优先**：先看日志定位问题
- **工具辅助**：inspect、events、stats
- **系统思维**：从容器到镜像到宿主机
- **预防为主**：健康检查、监控、告警
