图片通常是网页中最大的资源，占据了页面加载时间的"半壁江山"。本文将从前端开发者的角度，系统性地讲解图片优化的各种策略，帮助你显著提升页面性能。

## 为什么图片优化如此重要？

根据 HTTP Archive 的统计数据，图片平均占网页总重量的 **50% 以上**。未优化的图片会导致：

- **加载时间增加**：用户可能因为等待而离开
- **带宽消耗**：移动用户流量快速消耗
- **Core Web Vitals 受损**：LCP（Largest Contentful Paint）指标下降
- **SEO 排名下降**：Google 将页面速度作为排名因素

## 一、选择正确的图片格式

### 1.1 现代图片格式对比

| 格式 | 压缩类型  | 透明度 | 动画 | 浏览器支持 | 适用场景               |
| ---- | --------- | ------ | ---- | ---------- | ---------------------- |
| JPEG | 有损      | ❌     | ❌   | 全平台     | 照片、复杂图像         |
| PNG  | 无损      | ✅     | ❌   | 全平台     | 图标、需要透明度的图像 |
| WebP | 有损/无损 | ✅     | ✅   | 97%+       | 通用场景               |
| AVIF | 有损/无损 | ✅     | ✅   | 92%+       | 追求极致压缩           |
| GIF  | 无损      | ✅     | ✅   | 全平台     | 简单动画（已过时）     |

### 1.2 WebP：当前的最佳选择

WebP 是目前性价比最高的选择：

```html
&lt;!-- 使用 picture 元素提供降级方案 --&gt; &lt;picture&gt; &lt;source
srcset="image.avif" type="image/avif"&gt; &lt;source srcset="image.webp"
type="image/webp"&gt; &lt;img src="image.jpg" alt="描述文字" loading="lazy"&gt;
&lt;/picture&gt;
```

**压缩效果对比**（同等质量下）：

- WebP 比 PNG 小 **26%**
- WebP 比 JPEG 小 **25-34%**
- AVIF 比 WebP 再小 **20-30%**

### 1.3 AVIF：未来的格式

AVIF 是最新的图片格式，基于 AV1 视频编码：

```bash
# 使用 sharp 转换为 AVIF
npx sharp-cli input.png -o output.avif --format avif --quality 50
```

**AVIF 的优势**：

- 支持 HDR 和广色域
- 压缩率更高
- 支持alpha 通道

**兼容性处理**：

## 二、响应式图片

### 2.1 srcset 和 sizes 属性

让浏览器根据设备选择最合适的图片：

```html
&lt;img src="hero-800.jpg" srcset=" hero-400.jpg 400w, hero-800.jpg 800w,
hero-1200.jpg 1200w, hero-1600.jpg 1600w, hero-2000.jpg 2000w " sizes="
(max-width: 600px) 100vw, (max-width: 1200px) 50vw, 800px " alt="Hero 图片" &gt;
```

**sizes 属性解析**：

- `(max-width: 600px) 100vw`：窄屏下图片占满视口宽度
- `(max-width: 1200px) 50vw`：中等屏幕下图片占一半视口宽度
- `800px`：默认宽度 800px

### 2.2 art direction（艺术指导）

不同设备显示不同裁剪版本：

```html
&lt;picture&gt; &lt;!-- 移动端：竖版裁剪 --&gt; &lt;source media="(max-width:
600px)" srcset="hero-mobile.webp" type="image/webp" &gt; &lt;!-- 平板：中等裁剪
--&gt; &lt;source media="(max-width: 1200px)" srcset="hero-tablet.webp"
type="image/webp" &gt; &lt;!-- 桌面：完整版本 --&gt; &lt;img
src="hero-desktop.jpg" alt="Hero 图片" loading="lazy"&gt; &lt;/picture&gt;
```

### 2.3 实战：React 响应式图片组件

```tsx
interface ResponsiveImageProps {
  src: string;
  alt: string;
  widths?: number[];
  className?: string;
}

export function ResponsiveImage({
  src,
  alt,
  widths = [400, 800, 1200, 1600],
  className
}: ResponsiveImageProps) {
  const basePath = src.replace(/\.[^.]+$/, '');
  const ext = src.split('.').pop();

  const generateSrcSet = (format: string) =&gt; {
    return widths
      .map((w) =&gt; `${basePath}-${w}.${format} ${w}w`)
      .join(', ');
  };

  return (
    &lt;picture className={className}&gt;
      &lt;source type="image/avif" srcSet={generateSrcSet('avif')} /&gt;
      &lt;source type="image/webp" srcSet={generateSrcSet('webp')} /&gt;
      &lt;img
        src={src}
        alt={alt}
        srcSet={generateSrcSet(ext || 'jpg')}
        sizes="100vw"
        loading="lazy"
        decoding="async"
      /&gt;
    &lt;/picture&gt;
  );
}
```

## 三、懒加载策略

### 3.1 原生懒加载

最简单的方式，浏览器原生支持：

```html
&lt;img src="image.jpg" loading="lazy" alt="描述"&gt;
```

**loading 属性值**：

- `lazy`：延迟加载，直到图片接近视口
- `eager`：立即加载（默认值）

### 3.2 懒加载阈值控制

使用 CSS 和 JS 实现更精细的控制：

```javascript
// 使用 Intersection Observer
const lazyImages = document.querySelectorAll('img[data-src]');

const imageObserver = new IntersectionObserver(
  (entries) =&gt; {
    entries.forEach((entry) =&gt; {
      if (entry.isIntersecting) {
        const img = entry.target as HTMLImageElement;
        img.src = img.dataset.src!;
        img.removeAttribute('data-src');
        imageObserver.unobserve(img);
      }
    });
  },
  {
    rootMargin: '200px 0px', // 提前 200px 开始加载
    threshold: 0.01
  }
);

lazyImages.forEach((img) =&gt; imageObserver.observe(img));
```

### 3.3 首屏图片优先加载

首屏图片不应懒加载，而是预加载：

```html
&lt;head&gt; &lt;!-- 预加载关键图片 --&gt; &lt;link rel="preload" as="image"
href="hero.webp" type="image/webp"&gt; &lt;/head&gt; &lt;body&gt; &lt;!--
首屏图片使用 fetchpriority --&gt; &lt;img src="hero.webp" fetchpriority="high"
alt="Hero"&gt; &lt;!-- 非首屏图片懒加载 --&gt; &lt;img src="gallery-1.webp"
loading="lazy" alt="Gallery 1"&gt; &lt;/body&gt;
```

## 四、占位图策略

### 4.1 LQIP（Low Quality Image Placeholder）

先加载一个极低质量的模糊图，再替换为高清图：

```tsx
import { useState } from 'react';

interface LQIPImageProps {
  placeholder: string; // 极小尺寸的模糊图（10-20px）
  src: string;
  alt: string;
}

export function LQIPImage({ placeholder, src, alt }: LQIPImageProps) {
  const [loaded, setLoaded] = useState(false);

  return (
    &lt;div className="relative overflow-hidden"&gt;
      {/* 模糊占位图 */}
      &lt;img
        src={placeholder}
        alt={alt}
        className="absolute inset-0 w-full h-full object-cover blur-xl scale-110"
        aria-hidden="true"
      /&gt;
      {/* 真实图片 */}
      &lt;img
        src={src}
        alt={alt}
        className={`transition-opacity duration-300 ${loaded ? 'opacity-100' : 'opacity-0'}`}
        onLoad={() =&gt; setLoaded(true)}
      /&gt;
    &lt;/div&gt;
  );
}
```

### 4.2 SQIP（SVG-based Quality Image Placeholder）

用 SVG 生成占位图，文件更小：

```bash
# 安装 sqip
npm install sqip -D

# 生成 SVG 占位图
npx sqip input.jpg -o output.svg
```

### 4.3 颜色占位图

提取主色调作为背景：

```tsx
interface ColorPlaceholderImageProps {
  src: string;
  alt: string;
  dominantColor: string; // 主色调，如 '#4A90D9'
}

export function ColorPlaceholderImage({
  src,
  alt,
  dominantColor
}: ColorPlaceholderImageProps) {
  return (
    &lt;div
      className="relative"
      style={{ backgroundColor: dominantColor }}
    &gt;
      &lt;img
        src={src}
        alt={alt}
        className="w-full h-full object-cover"
        loading="lazy"
      /&gt;
    &lt;/div&gt;
  );
}
```

## 五、图片尺寸优化

### 5.1 合理的图片尺寸

不同场景下的推荐尺寸：

| 场景    | 推荐宽度    | 说明       |
| ------- | ----------- | ---------- |
| 缩略图  | 150-300px   | 列表页小图 |
| 内容图  | 600-800px   | 文章配图   |
| Hero 图 | 1200-2000px | 首屏大图   |
| 背景图  | 1920-2560px | 全屏背景   |

### 5.2 自动生成多尺寸

使用构建工具自动生成：

```javascript
// vite.config.js
import { defineConfig } from "vite";
import viteImagemin from "vite-plugin-imagemin";

export default defineConfig({
  plugins: [
    viteImagemin({
      gifsicle: { optimizationLevel: 3 },
      optipng: { optimizationLevel: 7 },
      mozjpeg: { quality: 80 },
      svgo: {
        plugins: [{ name: "removeViewBox", active: false }],
      },
      webp: { quality: 80 },
      avif: { quality: 50 },
    }),
  ],
});
```

### 5.3 Next.js Image 组件

Next.js 提供了开箱即用的图片优化：

```tsx
import Image from 'next/image';

export function HeroImage() {
  return (
    &lt;Image
      src="/hero.jpg"
      alt="Hero 图片"
      width={1200}
      height={600}
      priority // 首屏图片
      placeholder="blur"
      blurDataURL="data:image/jpeg;base64,/9j/4AAQ..." // 模糊占位
    /&gt;
  );
}
```

**Next.js Image 的优势**：

- 自动选择最佳格式（WebP/AVIF）
- 按需生成响应式尺寸
- 内置懒加载
- 防止布局偏移（CLS）

### 6.2 图片 CDN

使用 CDN 动态调整图片：

```javascript
// Cloudinary 示例
function getImageUrl(publicId, options = {}) {
  const params = new URLSearchParams({
    w: options.width || "auto",
    q: options.quality || "auto",
    f: options.format || "auto",
    c: options.crop || "fill",
  });

  return `https://res.cloudinary.com/your-cloud/image/upload/${params}/${publicId}`;
```

### 6.3 图片懒渲染

对于大量图片（如相册），使用虚拟列表：

```tsx
import { useVirtualizer } from '@tanstack/react-virtual';

function ImageGallery({ images }: { images: string[] }) {
  const parentRef = useRef&lt;HTMLDivElement&gt;(null);

  const rowVirtualizer = useVirtualizer({
    count: images.length,
    getScrollElement: () =&gt; parentRef.current,
    estimateSize: () =&gt; 300,
    overscan: 3
  });

  return (
    &lt;div ref={parentRef} className="h-screen overflow-auto"&gt;
      &lt;div style={{ height: rowVirtualizer.getTotalSize() }}&gt;
        {rowVirtualizer.getVirtualItems().map((virtualRow) =&gt; (
          &lt;div
            key={virtualRow.key}
            style={{
              position: 'absolute',
              top: 0,
              left: 0,
              width: '100%',
              height: virtualRow.size,
              transform: `translateY(${virtualRow.start}px)`
            }}
          &gt;
            &lt;img
              src={images[virtualRow.index]}
              alt=""
              loading="lazy"
            /&gt;
          &lt;/div&gt;
        ))}
      &lt;/div&gt;
    &lt;/div&gt;
  );
}
```

## 总结

图片优化是前端性能优化中最"性价比"的工作之一。通过选择正确的格式、响应式图片、懒加载和占位图策略，你可以显著提升页面加载速度和用户体验。

**核心要点**：

1. **格式选择**：优先 WebP，有条件支持 AVIF
2. **响应式**：使用 srcset + sizes + picture
3. **懒加载**：原生 loading="lazy" + Intersection Observer
4. **占位图**：LQIP/SQIP/颜色占位防止布局抖动
5. **工具化**：使用 Next.js Image 或图片 CDN 简化流程

记住：**没有银弹**，根据实际场景选择合适的优化策略，并通过性能监控持续改进。
