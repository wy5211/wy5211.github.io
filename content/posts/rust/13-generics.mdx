## 前言

假设你要写一个函数来找出两个数中的最大值。如果是 `i32`，你写一个版本；如果是 `f64`，你又要写一个几乎相同的版本。这太浪费了。

**泛型（Generics）** 允许你编写不指定具体类型的代码，一套代码适用于多种类型。Rust 的泛型在编译时进行**单态化（Monomorphization）**，确保运行时没有任何性能开销——这就是"零成本抽象"。

## 为什么需要泛型？

```rust
// ❌ 没有泛型：为每种类型写一个函数
fn largest_i32(list: &[i32]) -> i32 {
    let mut largest = list[0];
    for &item in list.iter().skip(1) {
        if item > largest { largest = item; }
    }
    largest
}

fn largest_f64(list: &[f64]) -> f64 {
    let mut largest = list[0];
    for &item in list.iter().skip(1) {
        if item > largest { largest = item; }
    }
    largest
}

// ✅ 使用泛型：一个函数搞定
fn largest<T: PartialOrd + Copy>(list: &[T]) -> T {
    let mut largest = list[0];
    for &item in list.iter().skip(1) {
        if item > largest { largest = item; }
    }
    largest
}
```

## 泛型函数

### 基本语法

在函数名后面用 `<T>` 声明泛型参数，然后在参数和返回值中使用：

```rust
fn print_value<T: std::fmt::Display>(value: T) {
    println!("值: {}", value);
}

fn main() {
    print_value(42);          // T = i32
    print_value(3.14);        // T = f64
    print_value("hello");     // T = &str
    print_value(true);        // T = bool
}
```

### 多个泛型参数

```rust
fn pair<T, U>(first: T, second: U) -> (T, U) {
    (first, second)
}

fn main() {
    let p1 = pair(1, "hello");       // (i32, &str)
    let p2 = pair("rust", 3.14);     // (&str, f64)
    let p3 = pair(vec![1, 2], true); // (Vec<i32>, bool)

    println!("{:?}", p1);
    println!("{:?}", p2);
    println!("{:?}", p3);
}
```

### 泛型与约束

泛型参数不能随便使用任何方法，需要通过 trait bound 约束其行为：

```rust
// T 必须实现 PartialOrd（可以比较大小）
fn largest<T: PartialOrd>(list: &[T]) -> &T {
    let mut largest = &list[0];
    for item in list.iter().skip(1) {
        if item > largest {
            largest = item;
        }
    }
    largest
}

// T 必须实现 Display（可以格式化显示）
fn describe<T: std::fmt::Display>(item: T) -> String {
    format!("值是: {}", item)
}

// 多个 trait 约束
fn print_and_compare<T: std::fmt::Display + PartialOrd>(a: T, b: T) {
    println!("a = {}, b = {}", a, b);
    if a > b {
        println!("a 更大");
    } else if a < b {
        println!("b 更大");
    } else {
        println!("相等");
    }
}

fn main() {
    let numbers = vec![34, 50, 25, 100, 65];
    println!("最大值: {}", largest(&numbers));

    println!("{}", describe(42));
    print_and_compare(10, 20);
}
```

## 泛型结构体

```rust
struct Point<T> {
    x: T,
    y: T,
}

fn main() {
    let integer_point = Point { x: 5, y: 10 };        // Point<i32>
    let float_point = Point { x: 1.5, y: 4.0 };       // Point<f64>
    let string_point = Point { x: "a", y: "b" };      // Point<&str>

    println!("整数: ({}, {})", integer_point.x, integer_point.y);
    println!("浮点: ({}, {})", float_point.x, float_point.y);
}
```

### 多类型参数的结构体

```rust
struct Pair<T, U> {
    first: T,
    second: U,
}

impl<T, U> Pair<T, U> {
    fn new(first: T, second: U) -> Self {
        Self { first, second }
    }

    fn get_first(&self) -> &T {
        &self.first
    }

    fn get_second(&self) -> U
    where
        U: Copy,
    {
        self.second
    }
}

fn main() {
    let p = Pair::new(42, "hello");
    println!("第一个: {}", p.get_first());
    println!("第二个: {}", p.get_second());
}
```

## 泛型枚举

我们已经用过 `Option<T>` 和 `Result<T, E>`，它们都是泛型枚举：

```rust
enum Option<T> {
    Some(T),
    None,
}

enum Result<T, E> {
    Ok(T),
    Err(E),
}
```

自定义泛型枚举：

```rust
enum Container<T> {
    Empty,
    Single(T),
    Multiple(Vec<T>),
}

impl<T: std::fmt::Display> Container<T> {
    fn describe(&self) {
        match self {
            Container::Empty => println!("空容器"),
            Container::Single(v) => println!("单个值: {}", v),
            Container::Multiple(v) => println!("多个值: {} 个", v.len()),
        }
    }
}

fn main() {
    let c1: Container<i32> = Container::Empty;
    let c2: Container<&str> = Container::Single("hello");
    let c3: Container<i32> = Container::Multiple(vec![1, 2, 3]);

    c1.describe();
    c2.describe();
    c3.describe();
}
```

## 泛型的性能：单态化

Rust 泛型的"零成本"来自**单态化**——编译器为每个具体类型生成一份专用代码：

```
┌─────────────────────────────────────────────────────────────┐
│                    单态化过程                                 │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  你写的代码：                                                │
│                                                             │
│  fn largest<T>(list: &[T]) -> &T { ... }                   │
│                                                             │
│  largest(&[1, 2, 3]);   // T = i32                         │
│  largest(&[1.0, 2.0]);  // T = f64                         │
│                                                             │
│  编译器生成的代码：                                          │
│                                                             │
│  fn largest_i32(list: &[i32]) -> &i32 { ... }              │
│  fn largest_f64(list: &[f64]) -> &f64 { ... }              │
│                                                             │
│  结果：运行时没有泛型开销，和手写每个版本一样快               │
│                                                             │
│  代价：编译后的二进制文件可能更大（代码膨胀）                │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 实战演练

### 示例 1：通用最大值函数

```rust
fn largest<T: PartialOrd>(list: &[T]) -> Option<&T> {
    if list.is_empty() {
        return None;
    }

    let mut largest = &list[0];
    for item in &list[1..] {
        if item > largest {
            largest = item;
        }
    }
    Some(largest)
}

fn main() {
    let numbers = vec![34, 50, 25, 100, 65];
    println!("最大数字: {:?}", largest(&numbers));

    let chars = vec!['y', 'm', 'a', 'q'];
    println!("最大字符: {:?}", largest(&chars));

    let words = vec!["apple", "banana", "cherry"];
    println!("最大单词: {:?}", largest(&words));

    println!("空列表: {:?}", largest::<i32>(&[]));
}
```

### 示例 2：泛型栈实现

```rust
#[derive(Debug)]
struct Stack<T> {
    items: Vec<T>,
}

impl<T> Stack<T> {
    fn new() -> Self {
        Self { items: Vec::new() }
    }

    fn push(&mut self, item: T) {
        self.items.push(item);
    }

    fn pop(&mut self) -> Option<T> {
        self.items.pop()
    }

    fn peek(&self) -> Option<&T> {
        self.items.last()
    }

    fn is_empty(&self) -> bool {
        self.items.is_empty()
    }

    fn size(&self) -> usize {
        self.items.len()
    }
}

fn main() {
    // 整数栈
    let mut int_stack = Stack::new();
    int_stack.push(1);
    int_stack.push(2);
    int_stack.push(3);
    println!("栈: {:?}", int_stack.items);
    println!("栈顶: {:?}", int_stack.peek());
    println!("弹出: {:?}", int_stack.pop());
    println!("弹出: {:?}", int_stack.pop());

    // 字符串栈
    let mut str_stack = Stack::new();
    str_stack.push("hello");
    str_stack.push("world");
    println!("字符串栈: {:?}", str_stack.items);
}
```

### 示例 3：泛型缓存

```rust
use std::collections::HashMap;

struct Cache<K, V> {
    store: HashMap<K, V>,
    max_size: usize,
}

impl<K: std::hash::Hash + Eq, V> Cache<K, V> {
    fn new(max_size: usize) -> Self {
        Self {
            store: HashMap::new(),
            max_size,
        }
    }

    fn get(&self, key: &K) -> Option<&V> {
        self.store.get(key)
    }

    fn set(&mut self, key: K, value: V) -> Option<V> {
        if self.store.len() >= self.max_size {
            // 简化：不实现淘汰策略
        }
        self.store.insert(key, value)
    }

    fn contains(&self, key: &K) -> bool {
        self.store.contains_key(key)
    }

    fn size(&self) -> usize {
        self.store.len()
    }

    fn clear(&mut self) {
        self.store.clear();
    }
}

fn main() {
    let mut cache = Cache::new(100);

    cache.set("key1", 100);
    cache.set("key2", 200);
    cache.set("key3", 300);

    println!("key1: {:?}", cache.get(&"key1"));  // Some(100)
    println!("key4: {:?}", cache.get(&"key4"));  // None
    println!("包含 key2: {}", cache.contains(&"key2"));  // true
    println!("缓存大小: {}", cache.size());  // 3
}
```

## 常见错误与修复

### 错误 1：泛型类型缺少 trait 约束

```rust
// ❌ T 没有约束，不能比较
// fn largest<T>(a: T, b: T) -> T {
//     if a > b { a } else { b }
// }

// ✅ 添加 PartialOrd 约束
fn largest<T: PartialOrd>(a: T, b: T) -> T {
    if a > b { a } else { b }
}
```

### 错误 2：结构体字段类型不一致

```rust
struct Point<T> {
    x: T,
    y: T,
}

fn main() {
    // ❌ x 和 y 类型必须相同
    // let p = Point { x: 5, y: 1.5 };

    // ✅ 使用两个泛型参数
    // struct Point<T, U> { x: T, y: U }
    // let p = Point { x: 5, y: 1.5 };
}
```

## 最佳实践

1. **只在需要时使用泛型**：如果只有一个类型用到，不需要泛型
2. **trait 约束尽量精确**：`T: Display` 比 `T: Any` 更好
3. **使用 `impl Trait` 简化**：简单情况用 `impl Trait` 代替 `<T: Trait>`
4. **避免过度泛化**：保持代码可读性

## 常见问题

**Q: Rust 泛型和 C++ 模板有什么区别？**

A: 概念类似，都是编译时单态化。但 Rust 的 trait bound 更明确，编译错误信息通常更清晰。

**Q: 泛型会影响编译速度吗？**

A: 会，因为编译器需要为每个具体类型生成代码。但运行时没有额外开销。

**Q: `impl Trait` 和 `<T: Trait>` 有什么区别？**

A: 功能上类似。`impl Trait` 更简洁，但只能有一个类型参数时使用。

## 总结

本文学习了 Rust 的泛型系统：

**泛型语法**

- 函数泛型：`fn foo<T>(x: T)`
- 结构体泛型：`struct Point<T>`
- 枚举泛型：`enum Option<T>`

**Trait 约束**

- `<T: Trait>` 限制泛型类型的行为
- 多个约束用 `+`
- `where` 子句用于复杂约束

**零成本抽象**

- 单态化：编译时生成具体版本的代码
- 运行时无性能开销
- 代价是二进制文件可能更大
