## 前言

到目前为止，所有权系统已经帮我们安全地管理了内存。但有些场景下，默认的所有权规则太严格了：

- 递归类型需要编译时已知大小
- 多个地方需要共享同一个数据
- 需要在运行时改变不可变数据的内部值

**智能指针（Smart Pointer）** 就是解决这些问题的工具。它们在拥有数据所有权的同时，提供了额外的能力。

## 智能指针概览

```
┌─────────────────────────────────────────────────────────────┐
│                    常用智能指针                               │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  Box<T>        堆上分配，单一所有权                          │
│  Rc<T>         引用计数，单线程共享                          │
│  Arc<T>        原子引用计数，多线程共享                      │
│  RefCell<T>    运行时借用检查，内部可变性                    │
│  Weak<T>       弱引用，不增加引用计数                        │
│                                                             │
│  选择指南：                                                  │
│                                                             │
│  需要堆分配？           → Box<T>                            │
│  单线程共享数据？       → Rc<T>                             │
│  多线程共享数据？       → Arc<T>                            │
│  需要修改不可变引用？   → RefCell<T>                        │
│  打破循环引用？         → Weak<T>                           │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## Box\<T\>：堆上分配

### 基本用法

`Box<T>` 将数据分配在堆上，栈上只保留一个指向堆的指针：

```rust
fn main() {
    let b = Box::new(5);
    println!("b = {}", *b);  // 解引用访问值
}
```

### 使用场景 1：编译时未知大小的类型

递归类型在编译时大小未知，需要用 Box 打破无限大小：

```rust
// ❌ 编译错误：Cons 的大小无法确定
// enum List {
//     Cons(i32, List),
//     Nil,
// }

// ✅ 使用 Box
enum List {
    Cons(i32, Box<List>),
    Nil,
}

use List::{Cons, Nil};

fn main() {
    let list = Cons(1, Box::new(Cons(2, Box::new(Cons(3, Box::new(Nil))))));
    println!("链表创建成功");
}
```

### 使用场景 2：大数据避免栈拷贝

```rust
#[derive(Debug)]
struct LargeData {
    data: [u8; 10000],
}

fn process(data: Box<LargeData>) -> Box<LargeData> {
    // Box 在栈上只有一个指针，传递开销很小
    println!("处理 {} 字节", data.data.len());
    data
}

fn main() {
    let data = Box::new(LargeData { data: [0u8; 10000] });
    let result = process(data);
    println!("{:p}", result);  // 打印堆地址
}
```

### 使用场景 3：Trait 对象

```rust
trait Animal {
    fn speak(&self);
}

struct Dog;
struct Cat;

impl Animal for Dog {
    fn speak(&self) { println!("汪汪！"); }
}

impl Animal for Cat {
    fn speak(&self) { println!("喵喵！"); }
}

fn main() {
    let animals: Vec<Box<dyn Animal>> = vec![
        Box::new(Dog),
        Box::new(Cat),
    ];

    for animal in &animals {
        animal.speak();
    }
}
```

## Rc\<T\>：引用计数

### 基本用法

当多个地方需要**共享**同一个数据，但都是只读时，使用 `Rc<T>`：

```rust
use std::rc::Rc;

fn main() {
    let a = Rc::new(String::from("hello"));

    // clone 不会深拷贝数据，只增加引用计数
    let b = Rc::clone(&a);
    let c = Rc::clone(&a);

    println!("a: {}", a);  // hello
    println!("b: {}", b);  // hello
    println!("c: {}", c);  // hello

    // 引用计数
    println!("引用计数: {}", Rc::strong_count(&a));  // 3
}
```

### Rc 与所有权

```rust
use std::rc::Rc;

enum List {
    Cons(i32, Rc<List>),
    Nil,
}

use List::{Cons, Nil};

fn main() {
    // 多个列表共享尾部
    let a = Rc::new(Cons(5, Rc::new(Cons(10, Rc::new(Nil)))));
    let b = Cons(3, Rc::clone(&a));
    let c = Cons(4, Rc::clone(&a));

    // a、b、c 共享 (5, 10, Nil) 这部分
    println!("a 引用计数: {}", Rc::strong_count(&a));  // 3
}
```

**注意**：`Rc` 只能在**单线程**中使用。

## RefCell\<T\>：内部可变性

### 为什么需要 RefCell？

Rust 的借用规则在编译时检查，但有时你需要在运行时"绕过"它：

```rust
use std::cell::RefCell;

fn main() {
    let data = RefCell::new(vec![1, 2, 3]);

    // 编译时 data 是不可变的，但运行时可以修改
    data.borrow_mut().push(4);

    println!("{:?}", data.borrow());  // [1, 2, 3, 4]
}
```

### RefCell 的 API

```rust
use std::cell::RefCell;

fn main() {
    let data = RefCell::new(String::from("hello"));

    // borrow()：获取不可变引用（运行时检查）
    let r1 = data.borrow();
    // let r2 = data.borrow_mut();  // ❌ panic！已存在不可变借用

    println!("{}", r1);
    drop(r1);  // 释放借用

    // borrow_mut()：获取可变引用（运行时检查）
    let mut r3 = data.borrow_mut();
    r3.push_str(" world");
    println!("{}", r3);
}
```

### RefCell 的规则（运行时检查）

```
┌─────────────────────────────────────────────────────────────┐
│                RefCell 借用规则                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  与编译时借用规则相同，但在运行时检查：                       │
│                                                             │
│  ✅ 可以有多个不可变借用（borrow()）                         │
│  ✅ 可以有一个可变借用（borrow_mut()）                       │
│  ❌ 不能同时有不可变和可变借用                                │
│                                                             │
│  违反规则时：panic（而不是编译错误）                         │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### Rc + RefCell 组合

这是非常常见的组合——多个所有者共享可变数据：

```rust
use std::cell::RefCell;
use std::rc::Rc;

fn main() {
    let shared_data = Rc::new(RefCell::new(vec![1, 2, 3]));

    let data1 = Rc::clone(&shared_data);
    let data2 = Rc::clone(&shared_data);

    // data1 修改
    data1.borrow_mut().push(4);

    // data2 可以看到修改
    println!("data2: {:?}", data2.borrow());  // [1, 2, 3, 4]
}
```

## Arc\<T\>：原子引用计数

`Arc` 是 `Rc` 的线程安全版本，名称来自 **A**tomic **R**eference **C**ounted：

```rust
use std::sync::Arc;
use std::thread;

fn main() {
    let data = Arc::new(vec![1, 2, 3, 4, 5]);

    let mut handles = vec![];

    for i in 0..3 {
        let data = Arc::clone(&data);
        let handle = thread::spawn(move || {
            let sum: i32 = data.iter().sum();
            println!("线程 {} 计算: {}", i, sum);
        });
        handles.push(handle);
    }

    for handle in handles {
        handle.join().unwrap();
    }
}
```

### Arc + Mutex

多线程共享可变数据的标准组合：

```rust
use std::sync::{Arc, Mutex};
use std::thread;

fn main() {
    let counter = Arc::new(Mutex::new(0));
    let mut handles = vec![];

    for _ in 0..10 {
        let counter = Arc::clone(&counter);
        let handle = thread::spawn(move || {
            let mut num = counter.lock().unwrap();
            *num += 1;
        });
        handles.push(handle);
    }

    for handle in handles {
        handle.join().unwrap();
    }

    println!("结果: {}", *counter.lock().unwrap());  // 10
}
```

## 实战演练

### 示例 1：递归数据结构——二叉树

```rust
use std::rc::Rc;

#[derive(Debug)]
enum Tree {
    Leaf(i32),
    Node {
        value: i32,
        left: Rc<Tree>,
        right: Rc<Tree>,
    },
}

impl Tree {
    fn sum(&self) -> i32 {
        match self {
            Tree::Leaf(v) => *v,
            Tree::Node { value, left, right } => {
                value + left.sum() + right.sum()
            }
        }
    }

    fn depth(&self) -> usize {
        match self {
            Tree::Leaf(_) => 1,
            Tree::Node { left, right, .. } => {
                1 + left.depth().max(right.depth())
            }
        }
    }
}

fn main() {
    let leaf1 = Rc::new(Tree::Leaf(3));
    let leaf2 = Rc::new(Tree::Leaf(7));
    let leaf3 = Rc::new(Tree::Leaf(1));
    let leaf4 = Rc::new(Tree::Leaf(9));

    let node1 = Rc::new(Tree::Node {
        value: 5,
        left: Rc::clone(&leaf1),
        right: Rc::clone(&leaf2),
    });

    let node2 = Rc::new(Tree::Node {
        value: 8,
        left: Rc::clone(&leaf3),
        right: Rc::clone(&leaf4),
    });

    let root = Rc::new(Tree::Node {
        value: 10,
        left: node1,
        right: node2,
    });

    println!("总和: {}", root.sum());    // 43
    println!("深度: {}", root.depth());  // 3
}
```

### 示例 2：可观察模式（Observer Pattern）

```rust
use std::cell::RefCell;
use std::rc::Rc;

trait Observer {
    fn update(&self, value: i32);
}

struct ConsoleObserver;

impl Observer for ConsoleObserver {
    fn update(&self, value: i32) {
        println!("控制台收到更新: {}", value);
    }
}

struct Subject {
    value: Rc<RefCell<i32>>,
    observers: Rc<RefCell<Vec<Box<dyn Observer>>>>,
}

impl Subject {
    fn new() -> Self {
        Self {
            value: Rc::new(RefCell::new(0)),
            observers: Rc::new(RefCell::new(Vec::new())),
        }
    }

    fn attach(&self, observer: Box<dyn Observer>) {
        self.observers.borrow_mut().push(observer);
    }

    fn set_value(&self, value: i32) {
        *self.value.borrow_mut() = value;
        for observer in self.observers.borrow().iter() {
            observer.update(value);
        }
    }
}

fn main() {
    let subject = Subject::new();
    subject.attach(Box::new(ConsoleObserver));

    subject.set_value(42);  // 控制台收到更新: 42
    subject.set_value(100); // 控制台收到更新: 100
}
```

## 常见错误与修复

### 错误 1：RefCell 借用冲突

```rust
use std::cell::RefCell;

fn main() {
    let data = RefCell::new(vec![1, 2, 3]);

    let r1 = data.borrow();
    // let r2 = data.borrow_mut();  // ❌ panic！

    drop(r1);  // 释放后再借
    let r2 = data.borrow_mut();  // ✅
}
```

### 错误 2：Rc 跨线程

```rust
use std::rc::Rc;
use std::thread;

fn main() {
    let data = Rc::new(5);

    // ❌ Rc 不是 Send 的
    // thread::spawn(move || {
    //     println!("{}", data);
    // });

    // ✅ 使用 Arc
    use std::sync::Arc;
    let data = Arc::new(5);
    let data2 = Arc::clone(&data);
    thread::spawn(move || {
        println!("{}", data2);
    }).join().unwrap();
}
```

## 最佳实践

1. **优先使用普通引用**：智能指针有运行时开销
2. **`Box` 用于堆分配和递归类型**：最轻量的智能指针
3. **`Rc` 用于单线程共享**：配合 `RefCell` 实现可变共享
4. **`Arc` 用于多线程共享**：配合 `Mutex` 实现线程安全可变
5. **`RefCell` 谨慎使用**：运行时 panic 不如编译时错误友好

## 常见问题

**Q: Box 和 Rc 有什么区别？**

A: `Box` 是单一所有权，`Rc` 是共享所有权（多个 Rc 指向同一数据）。

**Q: RefCell 什么时候用？**

A: 当编译器的借用检查太严格但你确信代码安全时。常见于图结构、观察者模式等。

**Q: 为什么不直接用 RefCell 代替普通引用？**

A: RefCell 的检查在运行时，违反规则会 panic。普通引用在编译时检查，更安全。

## 总结

本文学习了 Rust 的智能指针：

**Box\<T\>**

- 堆上分配，单一所有权
- 递归类型、Trait 对象、大数据传递

**Rc\<T\> / Arc\<T\>**

- 引用计数，共享所有权
- Rc 用于单线程，Arc 用于多线程

**RefCell\<T\>**

- 运行时借用检查
- 实现内部可变性
- 配合 Rc 使用

**常用组合**

- `Rc<RefCell<T>>`：单线程共享可变数据
- `Arc<Mutex<T>>`：多线程共享可变数据
