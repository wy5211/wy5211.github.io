## 为什么需要异步编程？

上一章我们学习了基于操作系统的多线程并发。但在构建分布式任务处理系统时，我们遇到了新的问题：系统需要处理大量 I/O 操作——等待数据库查询、等待网络响应、等待文件读写。

如果用多线程处理这些 I/O 操作，每个线程在等待期间都会阻塞，占用大量内存。对于成千上万个并发连接，创建同样数量的线程是不现实的。

**异步编程**解决了这个问题——一个线程可以同时处理成千上万个任务。在等待 I/O 时，线程自动切换到其他任务，而不是傻傻等待。这让系统能用少量线程处理大量并发连接，大大提高资源利用率。

## 同步 vs 异步

理解同步和异步的区别很重要。同步操作会阻塞当前线程，直到操作完成；异步操作会立即返回一个 Future，实际操作在后台进行。

```rust
use std::thread;
use std::time::Duration;

// 同步版本：每个请求阻塞一个线程
fn sync_fetch_all() {
    let urls = vec!["url1", "url2", "url3"];

    for url in urls {
        println!("同步请求: {}", url);
        thread::sleep(Duration::from_secs(1));  // 模拟网络请求
    }
    // 总耗时：3 秒
}

// 异步版本：并发执行多个请求
// async fn async_fetch_all() {
//     let urls = vec!["url1", "url2", "url3"];
//
//     let f1 = fetch(url1);  // 立即返回 Future
//     let f2 = fetch(url2);  // 立即返回 Future
//     let f3 = fetch(url3);  // 立即返回 Future
//
//     let (r1, r2, r3) = tokio::join!(f1, f2, f3);  // 并发执行
//     // 总耗时：约 1 秒（最慢的那个请求）
// }

fn main() {
    println!("同步版本开始");
    sync_fetch_all();
    println!("同步版本完成");
}
```

异步编程的核心优势在于：等待 I/O 时不占用线程，线程可以做其他事情。这对于 I/O 密集型应用（Web 服务、数据库客户端、网络爬虫）特别重要。

## async 函数和 .await

`async fn` 定义异步函数，它返回一个 Future 而不是直接返回值。`.await` 等待 Future 完成。

```rust
// 异步函数定义
async fn hello() {
    println!("Hello");
}

async fn world() {
    println!("World");
}

// 注意：async fn main 需要 Tokio 运行时
// #[tokio::main]
// async fn main() {
//     hello().await;
//     world().await;
// }
```

`async fn` 本质上是语法糖，编译器会将其转换为返回 Future 的普通函数。`.await` 也是语法糖，它会暂停当前 async 函数的执行，等待 Future 完成。

```rust
// async fn 的等价写法
use std::future::Future;

// async fn 定义的函数
async fn add(a: u32, b: u32) -> u32 {
    a + b
}

// 编译器实际生成的代码（简化版）
// fn add(a: u32, b: u32) -> impl Future<Output = u32> {
//     async move { a + b }
// }
```

## Future trait

`Future` 是异步的核心 trait，定义在 `std::future` 模块中。

```rust
use std::future::Future;
use std::pin::Pin;
use std::task::{Context, Poll};

pub trait Future {
    type Output;
    fn poll(self: Pin<&mut Self>, cx: &mut Context<'_>) -> Poll<Self::Output>;
}

pub enum Poll<T> {
    Ready(T),   // Future 已完成
    Pending,    // Future 还未完成
}
```

Future 是惰性的——创建 Future 不会执行任何操作，只有通过 `.await` 或交给运行时执行才会运行。这是异步编程高性能的关键：只有在真正需要结果时才会执行。

```rust
use std::future::Future;
use std::pin::Pin;
use std::task::{Context, Poll};
use std::time::{Duration, Instant};

struct Delay {
    when: Instant,
}

impl Future for Delay {
    type Output = &'static str;

    fn poll(mut self: Pin<&mut Self>, cx: &mut Context<'_>) -> Poll<Self::Output> {
        if Instant::now() >= self.when {
            Poll::Ready("done")
        } else {
            // 告诉运行时在何时唤醒
            cx.waker().wake_by_ref();
            Poll::Pending
        }
    }
}

// 注意：实际使用需要 Tokio 运行时
// #[tokio::main]
// async fn main() {
//     let delay = Delay {
//         when: Instant::now() + Duration::from_secs(1),
//     };
//     let out = delay.await;
//     println!("{}", out);
// }
```

## Tokio 运行时

Tokio 是 Rust 最流行的异步运行时，提供了事件循环、定时器、I/O 操作等基础设施。

```toml
# Cargo.toml
[dependencies]
tokio = { version = "1", features = ["full"] }
```

Tokio 的 `full` feature 包含了所有常用功能：任务调度、网络 I/O、定时器、文件系统操作等。

```rust
use tokio::time::{sleep, Duration};

#[tokio::main]  // 启动 Tokio 运行时
async fn main() {
    println!("开始");

    // 异步 sleep，不会阻塞整个线程
    sleep(Duration::from_secs(1)).await;
    println!("1 秒后");

    sleep(Duration::from_secs(1)).await;
    println!("2 秒后");
}
```

`#[tokio::main]` 宏会将 async fn main 转换为同步的 fn main，并在内部创建 Tokio 运行时。

## 并发执行

异步编程的真正威力在于并发——同时运行多个任务。Tokio 提供了多种方式来实现并发。

### join!：等待全部完成

`tokio::join!` 宏可以同时运行多个异步任务，等待它们全部完成。

```rust
use tokio::time::{sleep, Duration};

async fn task1() -> &'static str {
    sleep(Duration::from_millis(100)).await;
    "任务1完成"
}

async fn task2() -> &'static str {
    sleep(Duration::from_millis(200)).await;
    "任务2完成"
}

#[tokio::main]
async fn main() {
    let start = std::time::Instant::now();

    // 并发执行两个任务
    let (r1, r2) = tokio::join!(task1(), task2());

    let elapsed = start.elapsed();
    println!("{} {}", r1, r2);
    println!("总耗时: {:?}", elapsed);  // 约 200ms，而非 300ms
}
```

### select!：等待最先完成

`tokio::select!` 宏等待多个异步任务中**最先完成**的一个，这在处理超时、取消等场景很有用。

```rust
use tokio::sync::mpsc;
use tokio::time::{sleep, Duration};

#[tokio::main]
async fn main() {
    let (tx, mut rx) = mpsc::channel::<i32>(10);

    // 启动一个发送任务
    tokio::spawn(async move {
        sleep(Duration::from_millis(100)).await;
        tx.send(42).await.unwrap();
    });

    // 等待消息或超时，哪个先发生就处理哪个
    tokio::select! {
        val = rx.recv() => {
            println!("收到消息: {:?}", val);
        }
        _ = sleep(Duration::from_secs(1)) => {
            println!("超时");
        }
    }
}
```

### spawn：后台执行

`tokio::spawn` 创建一个在后台运行的异步任务，立即返回一个 JoinHandle。

```rust
use tokio::time::{sleep, Duration};

#[tokio::main]
async fn main() {
    let handle1 = tokio::spawn(async {
        sleep(Duration::from_millis(100)).await;
        println!("任务 1 完成");
    });

    let handle2 = tokio::spawn(async {
        sleep(Duration::from_millis(200)).await;
        println!("任务 2 完成");
    });

    // 主线程可以继续做其他事情
    println!("主线程继续执行");

    // 等待所有任务完成
    handle1.await.unwrap();
    handle2.await.unwrap();
    println!("所有任务完成");
}
```

## 实战：异步任务处理系统

现在让我们用异步编程构建一个高效的任务处理系统。这个系统能同时处理多个 I/O 密集型任务，而不会阻塞线程。

```rust
use tokio::time::{sleep, Duration};
use std::time::Instant;

#[derive(Debug, Clone)]
struct Task {
    id: u32,
    name: String,
    duration_ms: u64,
}

#[derive(Debug)]
struct TaskResult {
    task_id: u32,
    duration_ms: u64,
    status: String,
}

// 模拟异步 I/O 操作（如网络请求、数据库查询）
async fn process_task_io(task: Task) -> TaskResult {
    println!("开始处理任务: {} (预计 {}ms)", task.name, task.duration_ms);

    // 模拟异步 I/O 等待
    sleep(Duration::from_millis(task.duration_ms)).await;

    TaskResult {
        task_id: task.id,
        duration_ms: task.duration_ms,
        status: "完成".to_string(),
    }
}

#[tokio::main]
async fn main() {
    let start = Instant::now();

    // 定义任务列表
    let tasks = vec![
        Task { id: 1, name: String::from("数据库查询"), duration_ms: 200 },
        Task { id: 2, name: String::from("API 请求"), duration_ms: 150 },
        Task { id: 3, name: String::from("文件读取"), duration_ms: 100 },
        Task { id: 4, name: String::from("缓存更新"), duration_ms: 80 },
        Task { id: 5, name: String::from("日志写入"), duration_ms: 50 },
    ];

    println!("开始处理 {} 个任务", tasks.len());

    // 方式 1：顺序执行（慢）
    // let mut results = vec![];
    // for task in tasks {
    //     let result = process_task_io(task).await;
    //     results.push(result);
    // }
    // 总耗时：200 + 150 + 100 + 80 + 50 = 580ms

    // 方式 2：并发执行（快）
    let mut handles = vec![];
    for task in tasks {
        let handle = tokio::spawn(async move {
            process_task_io(task).await
        });
        handles.push(handle);
    }

    let mut results = vec![];
    for handle in handles {
        let result = handle.await.unwrap();
        println!("任务 {} 完成，耗时: {}ms", result.task_id, result.duration_ms);
        results.push(result);
    }
    // 总耗时：约 200ms（最慢的那个任务）

    let elapsed = start.elapsed();
    println!("\n所有任务完成，总耗时: {:?}", elapsed);
    println!("处理效率提升: {:.1}x",
        580.0 / elapsed.as_millis() as f64);
}
```

这个例子展示了异步编程的威力：通过并发执行，总耗时从 580ms 降低到约 200ms，提升了近 3 倍。实际应用中，随着任务数量增加，提升会更明显。

### 异步 HTTP 客户端

异步编程最常见的应用是处理大量网络请求。让我们看一个实际的 HTTP 客户端例子。

```toml
# Cargo.toml
[dependencies]
tokio = { version = "1", features = ["full"] }
reqwest = { version = "0.11", features = ["json"] }
serde = { version = "1", features = ["derive"] }
serde_json = "1"
```

```rust
use serde::Deserialize;
use reqwest::Error;

#[derive(Debug, Deserialize)]
struct User {
    id: u32,
    name: String,
    email: String,
}

// 异步获取用户信息
async fn fetch_user(id: u32) -> Result<User, Error> {
    let url = format!("https://jsonplaceholder.typicode.com/users/{}", id);
    let resp = reqwest::get(&url).await?;
    let user: User = resp.json().await?;
    Ok(user)
}

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    let start = std::time::Instant::now();

    // 并发获取多个用户
    let user1 = fetch_user(1);
    let user2 = fetch_user(2);
    let user3 = fetch_user(3);

    let (u1, u2, u3) = tokio::join!(user1, user2, user3);

    println!("用户 1: {}", u1?.name);
    println!("用户 2: {}", u2?.name);
    println!("用户 3: {}", u3?.name);

    println!("总耗时: {:?}", start.elapsed());
    Ok(())
}
```

## 常见陷阱

### 陷阱 1：在异步代码中阻塞

异步代码中避免阻塞操作，否则会阻塞整个线程。

```rust
use tokio::time::{sleep, Duration};
use std::thread;

#[tokio::main]
async fn main() {
    // 错误：阻塞异步运行时
    // thread::sleep(Duration::from_secs(1));

    // 正确：使用异步 sleep
    sleep(Duration::from_secs(1)).await;

    // 或者用 spawn_blocking 运行阻塞代码
    let result = tokio::task::spawn_blocking(|| {
        // 阻塞操作
        thread::sleep(Duration::from_secs(1));
        "done"
    }).await.unwrap();

    println!("{}", result);
}
```

### 陷阱 2：忘记 .await

```rust
async fn fetch_data() -> String {
    "data".to_string()
}

#[tokio::main]
async fn main() {
    // 错误：没有 .await，Future 不会执行
    // let data = fetch_data();

    // 正确：加上 .await
    let data = fetch_data().await;
    println!("{}", data);
}
```

## 权衡与考虑

异步编程的权衡：

**优势：**

- 高并发 I/O 处理能力
- 低内存占用（少量线程处理大量连接）
- 适合 Web 服务、网络爬虫、数据库客户端

**挑战：**

- 学习曲线陡峭
- 调试比同步代码困难
- 生态系统相比同步代码较小

**适用场景：**

- I/O 密集型应用
- 需要处理大量并发连接
- Web 服务、API 客户端、数据库操作

**不适用场景：**

- CPU 密集型计算（用多线程）
- 简单脚本（同步更直观）
- 团队不熟悉异步编程时
