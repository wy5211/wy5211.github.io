## 前言

随着项目变大，把所有代码放在一个文件里会变得难以维护。Rust 提供了**模块（Module）系统**来组织代码，让你可以把相关功能分组、控制可见性、避免命名冲突。

理解模块系统需要先搞清三个概念：**包（Package）**、**Crate**、**模块（Module）**。

## 包、Crate、模块的关系

```
┌─────────────────────────────────────────────────────────────┐
│                    Rust 代码组织层级                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  Package（包）                                               │
│  ├── Cargo.toml     定义包的元信息和依赖                     │
│  ├── src/                                                  │
│  │   ├── main.rs    二进制 Crate 的入口                     │
│  │   └── lib.rs     库 Crate 的入口（可选）                 │
│  └── tests/         集成测试                                │
│                                                             │
│  一个 Package 包含：                                        │
│  - 一个或多个 Crate                                         │
│  - 最多一个库 Crate                                         │
│  - 任意数量的二进制 Crate                                    │
│                                                             │
│  Crate（箱）                                                │
│  - 二进制 Crate：可执行程序，有 main() 函数                  │
│  - 库 Crate：可供其他代码引用，没有 main() 函数              │
│                                                             │
│  Module（模块）                                              │
│  - Crate 内的代码组织单元                                   │
│  - 可以嵌套                                                 │
│  - 控制可见性（私有/公开）                                   │
│                                                             │
│  Package → Crate → Module → 函数/结构体/枚举                │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 定义模块

### 内联模块

```rust
mod network {
    pub fn connect() {
        println!("连接到网络");
    }

    pub fn disconnect() {
        println!("断开网络");
    }

    fn internal_check() {
        // 私有函数，外部不可访问
        println!("内部检查");
    }
}

fn main() {
    network::connect();
    network::disconnect();
    // network::internal_check();  // ❌ 私有，无法访问
}
```

默认情况下，模块中的项是**私有**的。使用 `pub` 关键字使其公开。

### 分文件模块

随着模块变大，通常把模块移到单独的文件中：

```
src/
├── main.rs
├── network.rs       # network 模块
└── database.rs      # database 模块
```

`main.rs`:

```rust
mod network;
mod database;

fn main() {
    network::connect();
    database::save("hello");
}
```

`network.rs`:

```rust
pub fn connect() {
    println!("连接到网络");
}

pub fn disconnect() {
    println!("断开网络");
}
```

`database.rs`:

```rust
pub fn save(data: &str) {
    println!("保存数据: {}", data);
}
```

### 目录模块

当模块变复杂时，可以升级为目录：

```
src/
├── main.rs
└── network/
    ├── mod.rs        # network 模块声明
    ├── client.rs     # 子模块
    └── server.rs     # 子模块
```

`src/network/mod.rs`:

```rust
pub mod client;
pub mod server;

pub fn connect() {
    println!("连接到网络");
}
```

`src/network/client.rs`:

```rust
pub fn request(url: &str) {
    println!("请求: {}", url);
}
```

`src/network/server.rs`:

```rust
pub fn start(port: u16) {
    println!("服务器启动在端口 {}", port);
}
```

`src/main.rs`:

```rust
mod network;

fn main() {
    network::connect();
    network::client::request("https://example.com");
    network::server::start(8080);
}
```

## 可见性规则

### pub 关键字

`pub` 是控制可见性的核心：

```rust
mod my_module {
    pub fn public_function() {
        println!("公开函数");
    }

    fn private_function() {
        println!("私有函数");
    }

    pub mod inner {
        pub fn inner_public() {
            println!("内部公开函数");
        }

        fn inner_private() {
            println!("内部私有函数");
        }
    }
}

fn main() {
    my_module::public_function();     // ✅
    // my_module::private_function();  // ❌ 私有

    my_module::inner::inner_public();     // ✅
    // my_module::inner::inner_private(); // ❌ 私有
}
```

### 结构体的可见性

```rust
mod auth {
    pub struct User {
        pub username: String,  // 公开字段
        pub email: String,     // 公开字段
        password_hash: String,  // 私有字段
    }

    impl User {
        pub fn new(username: &str, email: &str, password: &str) -> Self {
            Self {
                username: username.to_string(),
                email: email.to_string(),
                password_hash: hash_password(password),
            }
        }

        pub fn verify_password(&self, password: &str) -> bool {
            self.password_hash == hash_password(password)
        }
    }

    fn hash_password(password: &str) -> String {
        format!("hash({})", password)  // 简化演示
    }
}

fn main() {
    let user = auth::User::new("alice", "a@b.com", "123456");

    // ✅ 可以访问公开字段
    println!("用户名: {}", user.username);

    // ❌ 不能访问私有字段
    // println!("{}", user.password_hash);

    // ✅ 可以调用公开方法
    println!("密码正确: {}", user.verify_password("123456"));
}
```

**要点**：即使结构体是 `pub` 的，它的字段默认仍然是私有的。需要每个字段单独标记 `pub`。

### 枚举的可见性

枚举和结构体不同，`pub enum` 的所有变体都是公开的：

```rust
mod shapes {
    pub enum Shape {
        Circle { radius: f64 },
        Rectangle { width: f64, height: f64 },
        Triangle { base: f64, height: f64 },
    }
}

fn main() {
    let s = shapes::Shape::Circle { radius: 5.0 };  // ✅
    // 所有变体都可以使用
}
```

### pub(crate) 和其他可见性

```rust
mod utils {
    // pub(crate) 表示在整个包内可见
    pub(crate) fn internal_helper() {
        println!("包内工具函数");
    }

    // pub(super) 表示在父模块内可见
    pub(super) fn parent_helper() {
        println!("父模块可见");
    }

    // pub(in path::to::module) 表示在指定路径内可见
    pub(in crate) fn crate_helper() {
        println!("crate 内可见");
    }
}
```

## use 关键字

`use` 用于将模块路径引入当前作用域，减少重复的完整路径：

```rust
mod network {
    pub mod client {
        pub fn request(url: &str) {
            println!("请求: {}", url);
        }
    }

    pub mod server {
        pub fn start(port: u16) {
            println!("启动服务器: {}", port);
        }
    }
}

// 使用完整路径
fn main() {
    network::client::request("https://example.com");
    network::server::start(8080);
}

// 使用 use 简化
use network::client;
use network::server;

fn main2() {
    client::request("https://example.com");
    server::start(8080);
}
```

### use 引入到当前作用域

```rust
use network::client::request;  // 直接引入函数

fn main() {
    request("https://example.com");  // 不需要模块前缀
}
```

### as 重命名

```rust
use network::client as net_client;
use network::server as net_server;

fn main() {
    net_client::request("https://example.com");
    net_server::start(8080);
}
```

### 嵌套路径

```rust
// 重复写前缀
// use network::client;
// use network::server;

// 简写
use network::{client, server};

// 更复杂的嵌套
use std::io::{self, Read, Write};
// 等价于
// use std::io;
// use std::io::Read;
// use std::io::Write;

// 使用 self
use std::io::self;
```

### glob 引入

```rust
use std::collections::*;  // 引入所有公开项（谨慎使用）

fn main() {
    let mut map = HashMap::new();
    map.insert("key", "value");
    println!("{:?}", map);
}
```

## 实战演练

### 示例 1：多模块项目

```
src/
├── main.rs
├── models.rs
├── services/
│   ├── mod.rs
│   ├── user_service.rs
│   └── auth_service.rs
└── utils.rs
```

`src/models.rs`:

```rust
pub struct User {
    pub id: u64,
    pub username: String,
    pub email: String,
}

pub enum UserRole {
    Admin,
    Editor,
    Viewer,
}
```

`src/utils.rs`:

```rust
pub fn greet(name: &str) -> String {
    format!("Hello, {}!", name)
}

pub fn format_user_info(id: u64, username: &str, email: &str) -> String {
    format!("ID: {}, 用户名: {}, 邮箱: {}", id, username, email)
}
```

`src/services/mod.rs`:

```rust
pub mod user_service;
pub mod auth_service;
```

`src/services/user_service.rs`:

```rust
use crate::models::User;

pub fn create_user(id: u64, username: &str, email: &str) -> User {
    User {
        id,
        username: username.to_string(),
        email: email.to_string(),
    }
}

pub fn display_user(user: &User) {
    println!("用户信息: ID={}, 用户名={}, 邮箱={}", user.id, user.username, user.email);
}
```

`src/services/auth_service.rs`:

```rust
use crate::models::UserRole;

pub fn check_permission(role: &UserRole, action: &str) -> bool {
    match role {
        UserRole::Admin => true,
        UserRole::Editor => action == "edit" || action == "view",
        UserRole::Viewer => action == "view",
    }
}
```

`src/main.rs`:

```rust
mod models;
mod utils;
mod services;

use models::{User, UserRole};
use services::{user_service, auth_service};

fn main() {
    // 创建用户
    let user = user_service::create_user(1, "alice", "alice@example.com");
    user_service::display_user(&user);

    // 权限检查
    let role = UserRole::Editor;
    println!("编辑权限: {}", auth_service::check_permission(&role, "edit"));
    println!("删除权限: {}", auth_service::check_permission(&role, "delete"));

    // 工具函数
    println!("{}", utils::greet("Rust"));
}
```

### 示例 2：库与二进制分离

```
my_project/
├── Cargo.toml
├── src/
│   ├── main.rs       # 二进制入口
│   └── lib.rs        # 库入口
```

`src/lib.rs`:

```rust
pub mod math {
    pub fn add(a: i32, b: i32) -> i32 {
        a + b
    }

    pub fn multiply(a: i32, b: i32) -> i32 {
        a * b
    }

    pub fn factorial(n: u64) -> u64 {
        if n <= 1 { 1 } else { n * factorial(n - 1) }
    }
}

pub mod string_utils {
    pub fn reverse(s: &str) -> String {
        s.chars().rev().collect()
    }

    pub fn is_palindrome(s: &str) -> bool {
        let cleaned: String = s.chars()
            .filter(|c| c.is_alphanumeric())
            .map(|c| c.to_ascii_lowercase())
            .collect();
        cleaned == cleaned.chars().rev().collect::<String>()
    }

    pub fn word_count(s: &str) -> usize {
        s.split_whitespace().count()
    }
}
```

`src/main.rs`:

```rust
use my_project::{math, string_utils};

fn main() {
    println!("加法: {}", math::add(3, 5));
    println!("阶乘: {}", math::factorial(10));
    println!("反转: {}", string_utils::reverse("hello"));
    println!("回文: {}", string_utils::is_palindrome("racecar"));
    println!("单词数: {}", string_utils::word_count("hello world rust"));
}
```

## 常见错误与修复

### 错误 1：忘记 pub

```rust
mod my_mod {
    // fn helper() {}     // ❌ 默认私有
    pub fn helper() {}    // ✅ 公开
}
```

### 错误 2：use 路径错误

```rust
// ❌ 路径不正确
// use my_module::helper;

// ✅ 检查模块是否正确声明
mod my_module {
    pub fn helper() {}
}
use my_module::helper;
```

## 最佳实践

1. **按功能组织模块**：而不是按类型（不要把所有结构体放一个文件）
2. **最小化 pub**：只暴露必要的接口，其余保持私有
3. **用 `use` 简化长路径**：但不要过度使用 glob `*`
4. **合理使用 `pub(crate)`**：包内共享但外部不可见的工具函数
5. **大项目用目录模块**：模块超过 100 行考虑拆分为目录

## 常见问题

**Q: `mod.rs` 和同名文件有什么区别？**

A: Rust 2018 后两种写法都支持。`src/network/mod.rs` 等价于 `src/network.rs`。目录模块推荐使用 `mod.rs`。

**Q: 什么时候需要 `pub use`？**

A: `pub use` 可以重新导出项，让外部使用者看到更简洁的 API 路径。

**Q: 如何引用外部 crate？**

A: 在 `Cargo.toml` 中添加依赖，然后用 `use crate_name::module` 引用。

## 总结

本文学习了 Rust 的模块与包管理系统：

**层级关系**

- Package：Cargo 管理的项目
- Crate：编译单元（二进制/库）
- Module：代码组织单元

**可见性**

- 默认私有
- `pub` 公开
- `pub(crate)` 包内可见
- 结构体字段需要单独 pub

**路径与引用**

- `use` 引入模块路径
- `as` 重命名
- `crate::` 从包根开始
- `super::` 从父模块开始
