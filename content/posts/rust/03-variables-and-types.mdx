## 为什么需要特殊的变量系统

在深入学习 Rust 的变量和数据类型之前，有必要理解为什么 Rust 的变量系统与其他语言不同。这个设计的核心目的是在编译期捕获更多错误，提高代码的安全性和可维护性。

传统语言中，变量默认是可变的，这在带来灵活性的同时也引入了很多难以追踪的 bug。例如，在大型代码库中，一个变量在多个地方被修改，要确定某个时刻的值变得非常困难。更严重的是，在并发环境下，可变状态容易导致数据竞争。

Rust 采取了一个激进但明智的设计决策：**变量默认不可变**。这意味着当你声明一个变量后，它的值不能被改变，除非显式声明为可变。这个简单的设计带来了几个重要好处：

**减少错误**：很多 bug 源于意外的变量修改。如果变量默认不可变，编译器会阻止意外的修改，让错误在编译期就被发现。

**提高可维护性**：不可变变量的行为更容易预测和推理。你不需要追踪变量的变化历史，因为它的值永远不会改变。

**并发安全**：不可变数据天然是线程安全的，可以安全地在多个线程之间共享，无需担心数据竞争。

**编译器优化**：不可变变量让编译器可以进行更激进的优化，因为它可以假设变量的值不会改变。

当然，Rust 也提供了可变性的机制，但要求开发者显式声明。这种"安全默认，显式选择"的设计哲学贯穿了整个 Rust 语言。

## 变量声明和可变性

### 基本变量声明

在 Rust 中，使用 `let` 关键字声明变量：

```rust
fn main() {
    // 声明一个变量，编译器会根据值推断类型
    let x = 5;

    // 声明变量并显式指定类型
    let y: i32 = 10;

    // 使用变量
    println!("x = {}, y = {}", x, y);
}
```

Rust 的变量命名遵循 snake_case 规范：小写字母，单词之间用下划线分隔。这是 Rust 社区的约定，遵守这个约定可以让代码更易读。

### 不可变性

如前所述，Rust 的变量默认是不可变的：

```rust
fn main() {
    let x = 5;
    // x = 10;  // 编译错误：不能对不可变变量重新赋值

    println!("x = {}", x);
}
```

如果你尝试编译这段代码，会得到类似以下的错误：

```
error[E0384]: cannot assign twice to immutable variable `x`
 --> src/main.rs:3:5
  |
2 |     let x = 5;
  |         - first assignment to `x`
3 |     x = 10;
  |     ^^^^^^ cannot assign twice to immutable variable
  |
  = note: consider making this binding mutable: `mut x`
```

这个错误信息非常清晰，甚至给出了修复建议。这种友好的错误提示是 Rust 编译器的一大特点。

### 可变变量

当你确实需要修改变量的值时，使用 `mut` 关键字：

```rust
fn main() {
    // 使用 mut 声明可变变量
    let mut x = 5;
    println!("初始值: x = {}", x);  // 输出: x = 5

    // 可以修改可变变量
    x = 10;
    println!("修改后: x = {}", x);  // 输出: x = 10

    // 可以多次修改
    x = x + 1;
    x *= 2;
    println!("最终: x = {}", x);  // 输出: x = 22
}
```

可变性的使用需要权衡。虽然可变变量提供了灵活性，但也带来了复杂性。**最佳实践是默认使用不可变变量，只有确实需要修改时才添加 `mut`**。

让我们看一个实际的例子，说明何时应该使用可变性：

```rust
fn main() {
    // 场景1：累积结果，适合使用可变变量
    let mut sum = 0;
    for i in 1..=10 {
        sum += i;
    }
    println!("1 到 10 的和: {}", sum);

    // 场景2：计算过程中的临时值，使用不可变变量更清晰
    let radius = 5.0;
    let pi = 3.14159;
    let area = pi * radius * radius;
    println!("半径为 {} 的圆面积: {}", radius, area);

    // 场景3：不需要修改原始值，使用遮蔽（下面会介绍）
    let x = 5;
    let x = x + 1;  // 遮蔽，而不是修改
    let x = x * 2;
    println!("x = {}", x);  // 输出: x = 12
}
```

### 变量遮蔽

Rust 允许声明一个与之前变量同名的新变量，这叫做**变量遮蔽**（Shadowing）。遮蔽与可变性不同，它创建了一个新变量，而不是修改现有变量：

```rust
fn main() {
    let x = 5;
    let x = x + 1;     // 创建新的 x，值为 6

    {
        let x = x * 2; // 在内部作用域中创建新的 x，值为 12
        println!("内部 x: {}", x);  // 输出: x = 12
    }

    println!("外部 x: {}", x);  // 输出: x = 6

    // 遮蔽可以改变变量的类型
    let spaces = "   ";           // &str 类型
    let spaces = spaces.len();    // usize 类型
    println!("空格数: {}", spaces);  // 输出: 空格数: 3
}
```

遮蔽的一个重要特性是**可以改变变量的类型**，这是使用 `mut` 无法做到的：

```rust
fn main() {
    // 使用遮蔽：类型可以改变
    let spaces = "   ";           // &str 类型
    let spaces = spaces.len();    // usize 类型，这是合法的

    // 使用 mut：类型不能改变
    let mut spaces = "   ";       // &str 类型
    // spaces = spaces.len();     // 编译错误：类型不匹配
}
```

遮蔽的主要用途包括：

1. **类型转换后保留相同的变量名**：如上面的例子，将字符串转换为长度后，仍使用 `spaces` 这个名字
2. **修改变量但不希望后续代码意外修改**：创建新变量而不是修改原变量
3. **在作用域内临时改变值**：在内部作用域中遮蔽外部变量，作用域结束后自动恢复

## 常量和静态变量

除了普通的 `let` 变量，Rust 还提供了常量和静态变量，它们有不同的用途和特性。

### 常量

使用 `const` 关键字声明常量：

```rust
// 常量必须显式标注类型
// 常量名约定使用大写字母和下划线分隔
const MAX_POINTS: u32 = 100_000;  // 数字中的下划线用于提高可读性
const PI: f64 = 3.14159;

fn main() {
    println!("最大分数: {}", MAX_POINTS);
    println!("圆周率: {}", PI);

    // 常量可以在任何作用域中使用，包括全局
    // 常量在编译期内联到使用位置，没有运行时开销
}
```

常量与不可变量的区别：

1. **必须显式标注类型**：常量不能依赖类型推断，必须明确指定类型
2. **只能设置为常量表达式**：不能在运行时计算，只能是编译期可计算的值
3. **可以在任何作用域声明**：包括全局作用域
4. **不能被遮蔽**：声明后不能再次声明同名的常量

```rust
// 正确的常量声明
const MAX_SIZE: usize = 100;
const VERSION: &str = "1.0.0";

// 错误的常量声明
// const VALUE: u32 = get_value();  // 错误：不能使用函数调用
// const MUT_VALUE: i32 = 5;        // 错误：不能使用 mut

fn get_value() -> u32 {
    42
}
```

### 静态变量

使用 `static` 关键字声明静态变量：

```rust
// 静态变量在程序运行期间一直存在
static GREETING: &str = "Hello, world!";

// 可变静态变量需要 unsafe 才能修改
static mut COUNTER: u32 = 0;

fn main() {
    println!("{}", GREETING);

    // 修改可变静态变量需要 unsafe 代码块
    unsafe {
        COUNTER += 1;
        println!("计数器: {}", COUNTER);
    }
}
```

静态变量的特点：

1. **有固定的内存地址**：静态变量在程序的整个生命周期内都有固定的地址
2. **可以是可变的**：但需要 `unsafe` 代码块来修改
3. **可以是引用类型**：但必须是 `'static` 生命周期的引用

常量和静态变量的选择：

- **使用 `const`**：对于编译时常量，如配置值、物理常数等
- **使用 `static`**：对于需要全局唯一地址的数据，或者需要可变的全局状态

## 基本数据类型

Rust 是静态类型语言，每个变量在编译期都必须有确定的类型。这不同于 Python、JavaScript 等动态类型语言，但也提供了类型推断来减少繁琐的类型标注。

### 整数类型

Rust 提供了多种整数类型，可以分为有符号整数和无符号整数两大类：

**有符号整数**（可以表示正数和负数）：

- `i8`：8 位，范围 -128 到 127
- `i16`：16 位，范围 -32,768 到 32,767
- `i32`：32 位，范围 -2³¹ 到 2³¹-1（默认整数类型）
- `i64`：64 位，范围 -2⁶³ 到 2⁶³-1
- `i128`：128 位，范围 -2¹²⁷ 到 2¹²⁷-1
- `isize`：指针大小，32 位系统为 32 位，64 位系统为 64 位

**无符号整数**（只能表示非负数）：

- `u8`：8 位，范围 0 到 255
- `u16`：16 位，范围 0 到 65,535
- `u32`：32 位，范围 0 到 2³²-1
- `u64`：64 位，范围 0 到 2⁶⁴-1
- `u128`：128 位，范围 0 到 2¹²⁸-1
- `usize`：指针大小，常用于索引和计数

```rust
fn main() {
    // 整数字面量的各种形式
    let decimal = 98_222;        // 十进制，下划线提高可读性
    let hex = 0xff;              // 十六进制，255
    let octal = 0o77;            // 八进制，63
    let binary = 0b1111_0000;    // 二进制，240
    let byte = b'A';             // 字节（仅限 u8），65

    // 类型后缀
    let x = 57u8;                // u8 类型
    let y = 1000i32;             // i32 类型

    // 默认类型是 i32
    let z = 42;                  // 推断为 i32

    println!("十进制: {}", decimal);
    println!("十六进制: {}", hex);
}
```

**整数溢出**：在 debug 模式下，整数溢出会导致 panic；在 release 模式下，会执行回绕（二进制补码回绕）：

```rust
fn main() {
    let mut x: u8 = 255;

    // Debug 模式下会 panic
    // x = x + 1;

    // 显式处理溢出
    x = x.wrapping_add(1);       // 回绕：0
    println!("wrapping: {}", x);

    x = x.checked_add(1).unwrap_or(0);  // 检查溢出，返回 None 或 Some
    x = x.saturating_add(1);            // 饱和：保持最大值
    x = x.overflowing_add(1);           // 类似 wrapping，但返回溢出标志
}
```

### 浮点类型

Rust 有两种浮点类型：`f32`（单精度）和 `f64`（双精度）：

```rust
fn main() {
    let x = 2.0;        // f64（默认）
    let y: f32 = 3.0;   // f32

    // 浮点运算
    let sum = x + y as f64;
    let difference = 95.5 - 4.3;
    let product = 4.0 * 30.0;
    let quotient = 56.7 / 32.2;
    let remainder = 43.0 % 5.0;

    println!("加法: {}", sum);
    println!("取余: {}", remainder);
}
```

需要注意浮点数的精度问题：

```rust
fn main() {
    let x = 0.1 + 0.2;
    // x != 0.3，由于浮点精度问题
    println!("0.1 + 0.2 = {}", x);

    // 正确的比较方式是使用容差
    let diff = (x - 0.3).abs();
    if diff < 1e-10 {
        println!("近似相等");
    }
}
```

### 布尔类型

布尔类型只有两个可能的值：`true` 和 `false`：

```rust
fn main() {
    let t = true;
    let f: bool = false;

    // 布尔运算
    let and = t && f;     // 逻辑与
    let or = t || f;      // 逻辑或
    let not = !t;         // 逻辑非

    println!("true && false = {}", and);   // false
    println!("true || false = {}", or);    // true
    println!("!true = {}", not);           // false

    // 主要用于条件判断
    if t {
        println!("条件为真");
    }
}
```

### 字符类型

Rust 的 `char` 类型是 Unicode 字符，占 4 字节：

```rust
fn main() {
    let c = 'z';
    let z = 'ℤ';
    let cat = '😻';
    let chinese = '中';

    println!("字符: {} {} {} {}", c, z, cat, chinese);

    // char 的方法
    println!("'中' 是字母吗? {}", chinese.is_alphabetic());
    println!("'5' 是数字吗? {}", '5'.is_numeric());
    println!("'a' 的 ASCII 值: {}", 'a' as u8);
}
```

需要注意的是，`char` 和字符串是不同的类型：

```rust
fn main() {
    let c: char = 'a';      // char，4 字节，单个 Unicode 字符
    let s: &str = "a";      // &str，字符串切片，可以是多个字符

    // 它们不能直接比较或转换
    // println!("{}", c == s);  // 编译错误

    // 正确的转换方式
    let s_from_c: String = c.to_string();
    let c_from_s: char = s.chars().next().unwrap();
}
```

## 复合类型

复合类型可以将多个值组合成一个类型。Rust 提供了两种基本的复合类型：元组和数组。

### 元组

元组可以将多个不同类型的值组合成一个复合类型：

```rust
fn main() {
    // 创建元组
    let tup: (i32, f64, u8) = (500, 6.4, 1);

    // 解构元组：将元组的值分解到多个变量
    let (x, y, z) = tup;
    println!("y 的值: {}", y);  // 输出: y 的值: 6.4

    // 通过索引访问元组元素
    let five_hundred = tup.0;
    let six_point_four = tup.1;
    let one = tup.2;

    println!("{}, {}, {}", five_hundred, six_point_four, one);
}
```

元组的常见用途：

1. **从函数返回多个值**：

```rust
fn calculate(s: &str) -> (usize, usize) {
    (s.len(), s.split_whitespace().count())
}

fn main() {
    let text = "Hello world";
    let (length, words) = calculate(text);
    println!("长度: {}, 单词数: {}", length, words);
}
```

2. **临时组合相关数据**：

```rust
fn main() {
    let person = ("Alice", 30, "Engineer");

    // 访问元组的字段
    println!("姓名: {}, 年龄: {}, 职业: {}", person.0, person.1, person.2);
}
```

3. **单元类型 `()`**：空元组，表示"无值"：

```rust
fn main() {
    let unit = ();  // 单元类型，占用 0 字节

    // 没有返回值的函数实际返回 ()
    let result = do_something();
    println!("单元类型: {:?}", result);
}

fn do_something() {  // 实际返回 ()
    println!("做点什么");
}
```

### 数组

数组是固定长度的同类型元素集合：

```rust
fn main() {
    // 创建数组
    let a = [1, 2, 3, 4, 5];

    // 指定类型和长度
    let b: [i32; 5] = [1, 2, 3, 4, 5];

    // 重复初始化：创建包含 5 个 3 的数组
    let c = [3; 5];  // 等同于 [3, 3, 3, 3, 3]

    // 访问元素
    let first = a[0];
    let second = a[1];

    println!("第一个元素: {}", first);
    println!("数组 c: {:?}", c);

    // 数组长度
    println!("数组 a 的长度: {}", a.len());

    // 遍历数组
    for element in a.iter() {
        println!("元素: {}", element);
    }

    // 带索引遍历
    for (index, element) in a.iter().enumerate() {
        println!("a[{}] = {}", index, element);
    }
}
```

**数组越界**：Rust 会在运行时检查数组边界，越界访问会导致 panic：

```rust
fn main() {
    let a = [1, 2, 3, 4, 5];

    // 运行时 panic
    // let element = a[10];
}
```

数组的适用场景：

1. **长度固定且已知**：如一年有 12 个月
2. **数据存储在栈上**：性能要求高，不需要堆分配
3. **编译期确定大小**：大小作为类型的一部分

对于动态长度的集合，应该使用 `Vec<T>`（向量），它是堆分配的可增长数组。

## 类型推断和转换

### 类型推断

Rust 有强大的类型推断系统，在很多情况下不需要显式标注类型：

```rust
fn main() {
    // 编译器根据值推断类型
    let x = 5;              // 推断为 i32
    let y = 1.0;            // 推断为 f64

    // 根据后续使用推断类型
    let mut v = Vec::new(); // 此时类型未定
    v.push(1);              // 推断为 Vec<i32>
    v.push(2);

    // 有时需要显式标注
    let z: i64 = 100;       // 显式指定为 i64
    let pi: f32 = 3.14;     // 显式指定为 f32 而不是默认的 f64
}
```

### 类型转换

Rust 不支持隐式类型转换，必须显式进行：

```rust
fn main() {
    let x: i32 = 5;
    // let y: i64 = x;  // 错误：不允许隐式转换

    // 使用 as 关键字显式转换
    let y: i64 = x as i64;

    // 数值类型转换
    let a: i32 = 42;
    let b: f64 = a as f64;

    // 字符转数字
    let digit = '9';
    let number = digit.to_digit(10).unwrap();  // Some(9)

    // 字符串转数字
    let num: i32 = "42".parse().expect("不是有效的数字");

    println!("转换后的值: {}, {}, {}", y, number, num);
}
```

使用 `as` 进行类型转换时需要注意，它不会检查溢出，可能导致意外的行为：

```rust
fn main() {
    let x: u8 = 300;       // 300 超出了 u8 的范围（0-255）
    let y: u8 = x;         // 实际存储的值是 300 % 256 = 44

    println!("y = {}", y);  // 输出: y = 44
}
```

对于需要检查溢出的转换，应该使用 `try_from` 或 `checked_` 系列方法：

```rust
fn main() {
    use std::convert::TryFrom;

    // 安全的转换，返回 Result
    let x: i32 = 300;
    match u8::try_from(x) {
        Ok(val) => println!("转换成功: {}", val),
        Err(e) => println!("转换失败: {}", e),
    }

    // 使用 checked 方法
    let y: i32 = 100;
    if let Some(val) = u8::checked_cast(y) {
        println!("安全转换: {}", val);
    }
}
```

## 实际应用示例

### 温度转换器

让我们用所学的知识编写一个实用的温度转换程序：

```rust
/// 将摄氏度转换为华氏度
fn celsius_to_fahrenheit(celsius: f64) -> f64 {
    celsius * 1.8 + 32.0
}

/// 将华氏度转换为摄氏度
fn fahrenheit_to_celsius(fahrenheit: f64) -> f64 {
    (fahrenheit - 32.0) / 1.8
}

fn main() {
    // 使用遮蔽进行多次转换
    let temp = 0.0;  // 摄氏度
    let temp = celsius_to_fahrenheit(temp);
    println!("0°C = {}°F", temp);  // 输出: 0°C = 32°F

    let temp = 100.0;  // 华氏度
    let temp = fahrenheit_to_celsius(temp);
    println!("100°F = {:.1}°C", temp);  // 输出: 100°F = 37.8°C

    // 使用元组存储温度对照表
    let temperatures: [(f64, f64); 3] = [
        (0.0, celsius_to_fahrenheit(0.0)),
        (100.0, celsius_to_fahrenheit(100.0)),
        (-40.0, celsius_to_fahrenheit(-40.0)),
    ];

    println!("\n温度对照表:");
    for (c, f) in temperatures.iter() {
        println!("{:6.1}°C = {:6.1}°F", c, f);
    }
}
```

这个例子展示了：

1. 函数定义和类型标注

## 权衡分析

在学习 Rust 的变量和类型系统时，需要在几个方面做出权衡：

**不可变性 vs 可变性**：

不可变性带来的好处：

- **更安全的代码**：编译器防止意外的修改，减少了 bug
- **更好的并发支持**：不可变数据可以安全地在多个线程间共享
- **更简单的推理**：不需要追踪变量的变化历史

可变性的适用场景：

- **性能优化**：避免不必要的数据复制
- **累积操作**：如循环中的计数器、累加器
- **算法需要**：某些算法（如原地排序）需要修改数据

**类型推断 vs 显式标注**：

类型推断的好处：

- **减少代码冗余**：不需要重复写显而易见的类型
- **提高可读性**：让代码更专注于逻辑而不是类型

显式标注的适用场景：

- **文档作用**：让代码的意图更明确
- **复杂类型**：编译器可能推断出复杂的类型，显式标注更清晰
- **类型转换**：确保获得正确的类型

**不同整数类型的选择**：

较小的类型（如 `i8`、`u8`）：

- **优势**：节省内存，提高缓存效率
- **劣势**：容易溢出，需要额外的检查

较大的类型（如 `i64`、`i128`）：

- **优势**：不容易溢出，减少边界检查
- **劣势**：占用更多内存，可能降低缓存效率

**默认选择**：

- **整数**：默认使用 `i32`，这是最平衡的选择
- **浮点数**：默认使用 `f64`，精度更高且在现代 CPU 上性能不差
- **索引**：使用 `usize`，这是平台的自然大小
- **字节**：使用 `u8`，这是字节的准确表示

理解这些权衡可以帮助你做出更好的设计决策，写出既安全又高效的代码。

## 总结

本文介绍了 Rust 的变量系统和基本数据类型，主要内容包括：

**变量系统**：

- 变量默认不可变，需要修改时使用 `mut`
- 变量遮蔽可以创建同名的新变量，甚至改变类型
- 常量和静态变量用于编译期已知的值

**基本数据类型**：

- 整数类型：有符号和无符号，各种大小的整数
- 浮点类型：`f32` 和 `f64`，需要注意精度问题
- 布尔类型：`true` 和 `false`
- 字符类型：Unicode 字符，占 4 字节

**复合类型**：

- 元组：固定长度，可以包含不同类型
- 数组：固定长度，同类型元素，栈上分配

**类型系统**：

- 强大的类型推断，减少冗余的类型标注
- 不支持隐式类型转换，必须显式转换
- 提供了安全的类型转换方法
