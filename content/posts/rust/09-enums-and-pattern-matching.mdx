## 前言

如果你用过 Java 或 C++，可能觉得枚举不过是几个命名常量。但 Rust 的枚举远比这强大——**枚举变体可以携带数据**，这使得 Rust 的枚举更像是一种"带标签的联合体"。

与枚举配合使用的**模式匹配（Pattern Matching）** 是 Rust 中处理分支逻辑的利器，比传统的 if-else 更安全、更强大。

## 定义枚举

### 基本枚举

```rust
enum Direction {
    Up,
    Down,
    Left,
    Right,
}

fn main() {
    let player_dir = Direction::Up;

    // 使用 match 处理枚举
    match player_dir {
        Direction::Up => println!("向上移动"),
        Direction::Down => println!("向下移动"),
        Direction::Left => println!("向左移动"),
        Direction::Right => println!("向右移动"),
    }
}
```

### 携带数据的枚举

Rust 枚举的每个变体可以携带不同类型和数量的数据：

```rust
enum Message {
    Quit,                       // 不携带数据
    Move { x: i32, y: i32 },    // 携带命名字段（类似结构体）
    Write(String),              // 携带一个 String
    ChangeColor(u8, u8, u8),    // 携带三个 u8
}

fn main() {
    let msg1 = Message::Quit;
    let msg2 = Message::Move { x: 10, y: 20 };
    let msg3 = Message::Write(String::from("hello"));
    let msg4 = Message::ChangeColor(255, 0, 0);

    process_message(msg1);
    process_message(msg2);
    process_message(msg3);
    process_message(msg4);
}

fn process_message(msg: Message) {
    match msg {
        Message::Quit => println!("退出"),
        Message::Move { x, y } => println!("移动到 ({}, {})", x, y),
        Message::Write(text) => println!("写入: {}", text),
        Message::ChangeColor(r, g, b) => println!("颜色: RGB({}, {}, {})", r, g, b),
    }
}
```


## 为枚举实现方法

和结构体一样，枚举也可以用 `impl` 定义方法：

```rust
#[derive(Debug)]
enum Message {
    Quit,
    Move { x: i32, y: i32 },
    Write(String),
    ChangeColor(u8, u8, u8),
}

impl Message {
    // 关联函数作为构造函数
    fn new_move(x: i32, y: i32) -> Self {
        Message::Move { x, y }
    }

    // 方法
    fn description(&self) -> &str {
        match self {
            Message::Quit => "退出消息",
            Message::Move { .. } => "移动消息",
            Message::Write(_) => "写入消息",
            Message::ChangeColor(..) => "颜色消息",
        }
    }
}

fn main() {
    let msg = Message::new_move(10, 20);
    println!("{:?}", msg);
    println!("{}", msg.description());
}
```

## Option 枚举

`Option` 是 Rust 标准库中最重要的枚举之一，用于表示"一个值可能存在，也可能不存在"。

### 定义

```rust
enum Option<T> {
    Some(T),   // 有值
    None,      // 无值
}
```

Rust 没有 `null`，用 `Option` 代替。你不需要手动引入它，它在 Prelude（预导入）中。

### 基本使用

```rust
fn main() {
    let some_number: Option<i32> = Some(5);
    let some_string: Option<&str> = Some("hello");
    let absent_number: Option<i32> = None;

    println!("{:?}", some_number);    // Some(5)
    println!("{:?}", some_string);   // Some("hello")
    println!("{:?}", absent_number);  // None
}
```

### Option 强制你处理空值

```rust
fn main() {
    let x: Option<i32> = Some(5);

    // ❌ 不能直接把 Option 当作 i32 使用
    // let sum = x + 1;  // 编译错误！

    // ✅ 必须先处理 Option
    let sum = match x {
        Some(num) => num + 1,
        None => 0,  // 处理 None 的情况
    };
    println!("sum = {}", sum);  // 6
}
```

这就是 Rust 消除空指针异常的方式——**类型系统强制你考虑"值不存在"的情况**。

### Option 的常用方法

```rust
fn main() {
    let some_val: Option<i32> = Some(42);
    let none_val: Option<i32> = None;

    // is_some / is_none
    println!("有值: {}", some_val.is_some());    // true
    println!("无值: {}", none_val.is_none());    // true

    // unwrap：取出值，如果是 None 则 panic
    // let val = none_val.unwrap();  // ⚠️ 会 panic！

    // unwrap_or：提供默认值
    let val = none_val.unwrap_or(0);
    println!("默认值: {}", val);  // 0

    // unwrap_or_else：通过闭包计算默认值
    let val = none_val.unwrap_or_else(|| expensive_computation());
    println!("默认值: {}", val);

    // map：转换 Some 内部的值
    let doubled = some_val.map(|x| x * 2);
    println!("翻倍: {:?}", doubled);  // Some(84)

    // and_then：链式操作
    let result = some_val.and_then(|x| if x > 0 { Some(x * 2) } else { None });
    println!("结果: {:?}", result);  // Some(84)
}

fn expensive_computation() -> i32 {
    println!("执行了昂贵的计算...");
    0
}
```

## 模式匹配

### match 表达式

`match` 是 Rust 最强大的控制流运算符，它检查一个值是否匹配一系列模式：

```rust
fn main() {
    let number = 3;

    match number {
        1 => println!("一"),
        2 => println!("二"),
        3 => println!("三"),
        4 | 5 => println!("四或五"),  // | 表示"或"
        6..=10 => println!("六到十"),  // ..= 表示范围
        _ => println!("其他"),        // _ 匹配所有剩余情况
    }
}
```

**重要**：`match` 的每个分支必须覆盖所有可能的情况，否则编译错误。`_` 是通配符，匹配任何值。

### 匹配守卫（Match Guard）

在模式后面添加 `if` 条件：

```rust
fn main() {
    let num = Some(4);

    match num {
        Some(x) if x < 5 => println!("小于 5: {}", x),
        Some(x) => println!("大于等于 5: {}", x),
        None => println!("没有值"),
    }

    // 匹配偶数
    let x = 4;
    match x {
        n if n % 2 == 0 => println!("{} 是偶数", n),
        n => println!("{} 是奇数", n),
    }
}
```

### 绑定变量

使用 `@` 在模式中绑定变量：

```rust
fn main() {
    let age = 15;

    match age {
        0 => println!("刚出生"),
        n @ 1..=12 => println!("儿童: {} 岁", n),   // n 绑定匹配的值
        n @ 13..=19 => println!("青少年: {} 岁", n),
        n => println!("成年: {} 岁", n),
    }
}
```

### 匹配结构体和枚举

```rust
struct Point {
    x: i32,
    y: i32,
}

enum Shape {
    Circle { radius: f64 },
    Rectangle { width: f64, height: f64 },
    Triangle { base: f64, height: f64 },
}

fn main() {
    let p = Point { x: 0, y: 7 };

    // 匹配结构体
    match p {
        Point { x: 0, y } => println!("在 y 轴上，y = {}", y),
        Point { x, y: 0 } => println!("在 x 轴上，x = {}", x),
        Point { x, y } => println!("在 ({}, {})", x, y),
    }

    // 匹配枚举
    let shape = Shape::Rectangle { width: 10.0, height: 5.0 };

    let area = match shape {
        Shape::Circle { radius } => std::f64::consts::PI * radius * radius,
        Shape::Rectangle { width, height } => width * height,
        Shape::Triangle { base, height } => 0.5 * base * height,
    };
    println!("面积: {}", area);
}
```

### if let 语法糖

当只需要匹配一个变体，忽略其他情况时，用 `if let`：

```rust
fn main() {
    let some_val = Some(3);

    // match 写法（啰嗦）
    match some_val {
        Some(3) => println!("三"),
        _ => (),
    }

    // if let 写法（简洁）
    if let Some(3) = some_val {
        println!("三");
    }

    // if let 也可以用 else
    if let Some(x) = some_val {
        println!("值是 {}", x);
    } else {
        println!("没有值");
    }
}
```

### while let

`while let` 循环只要模式匹配就继续：

```rust
fn main() {
    let mut stack = Vec::new();
    stack.push(1);
    stack.push(2);
    stack.push(3);

    // 持续弹出直到 Vec 为空
    while let Some(top) = stack.pop() {
        println!("弹出: {}", top);
    }
    // 输出：3 2 1
}
```

### let-else

Rust 1.65 引入的 `let-else`，在模式不匹配时执行 else 分支：

```rust
fn main() {
    // 如果解析失败，直接返回
    let Some(value) = parse_number("42") else {
        println!("解析失败");
        return;
    };
    println!("解析结果: {}", value);
}

fn parse_number(s: &str) -> Option<i32> {
    s.parse().ok()
}
```

## 实战演练

### 示例 1：IP 地址枚举

```rust
#[derive(Debug)]
enum IpAddr {
    V4(u8, u8, u8, u8),
    V6(String),
}

impl IpAddr {
    fn is_loopback(&self) -> bool {
        match self {
            IpAddr::V4(127, _, _, _) => true,
            IpAddr::V6(s) => s == "::1",
            _ => false,
        }
    }

    fn description(&self) -> String {
        match self {
            IpAddr::V4(a, b, c, d) => format!("IPv4: {}.{}.{}.{}", a, b, c, d),
            IpAddr::V6(addr) => format!("IPv6: {}", addr),
        }
    }
}

fn main() {
    let home = IpAddr::V4(127, 0, 0, 1);
    let loopback = IpAddr::V6(String::from("::1"));

    println!("{}", home.description());
    println!("{}", loopback.description());
    println!("home 是回环地址: {}", home.is_loopback());
    println!("loopback 是回环地址: {}", loopback.is_loopback());
}
```

### 示例 2：简单表达式计算器

```rust
#[derive(Debug)]
enum Expr {
    Number(i32),
    Add(Box<Expr>, Box<Expr>),
    Sub(Box<Expr>, Box<Expr>),
    Mul(Box<Expr>, Box<Expr>),
}

impl Expr {
    fn eval(&self) -> i32 {
        match self {
            Expr::Number(n) => *n,
            Expr::Add(left, right) => left.eval() + right.eval(),
            Expr::Sub(left, right) => left.eval() - right.eval(),
            Expr::Mul(left, right) => left.eval() * right.eval(),
        }
    }
}

fn main() {
    // 表达式: (2 + 3) * 4
    let expr = Expr::Mul(
        Box::new(Expr::Add(
            Box::new(Expr::Number(2)),
            Box::new(Expr::Number(3)),
        )),
        Box::new(Expr::Number(4)),
    );

    println!("表达式: {:?}", expr);
    println!("结果: {}", expr.eval());  // 20
}
```

### 示例 3：文件操作结果处理

```rust
use std::fs;

fn main() {
    let filename = "Cargo.toml";

    // 使用 match 处理 Option
    let extension = get_extension(filename);
    match extension {
        Some(ext) => println!("文件扩展名: {}", ext),
        None => println!("没有扩展名"),
    }

    // 使用 if let
    if let Some(ext) = get_extension("readme.md") {
        println!("扩展名: {}", ext);
    }

    // 使用 match 处理文件读取
    match read_file_content(filename) {
        Ok(content) => println!("文件内容（前 50 字符）: {}...", &content[..50.min(content.len())]),
        Err(e) => println!("读取失败: {}", e),
    }
}

fn get_extension(filename: &str) -> Option<&str> {
    filename.rsplit('.').next()
}

fn read_file_content(filename: &str) -> Result<String, std::io::Error> {
    fs::read_to_string(filename)
}
```

## 常见错误与修复

### 错误 1：match 没有覆盖所有情况

```rust
fn main() {
    let x: Option<i32> = Some(5);

    // ❌ 没有处理 None
    // match x {
    //     Some(n) => println!("{}", n),
    // }

    // ✅ 处理所有情况
    match x {
        Some(n) => println!("{}", n),
        None => println!("无值"),
    }
}
```

### 错误 2：混淆 Option 和值

```rust
fn main() {
    let x: Option<i32> = Some(5);

    // ❌ 不能直接做运算
    // let y = x + 1;

    // ✅ 先解包
    let y = x.unwrap() + 1;
    println!("{}", y);
}
```

## 最佳实践

1. **优先用 `match` 而不是 `unwrap`**：`unwrap` 在 `None` 时会 panic，`match` 让你优雅处理
2. **用 `if let` 简化单一匹配**：只需要处理一种情况时，`if let` 比 `match` 更简洁
3. **枚举变体用 PascalCase**：`Some`、`None`、`Move` 等
4. **用 `#[derive(Debug)]` 方便调试**
5. **用 `Option` 代替 null**：永远不要用 unsafe 代码模拟 null

## 常见问题

**Q: Option 和 null 有什么区别？**

A: `Option<T>` 在类型层面表示值可能不存在，编译器强制你处理 `None` 的情况。`null` 可以赋给任何引用类型，容易被忽略，导致空指针异常。

**Q: match 必须穷举所有情况吗？**

A: 是的。如果不处理所有情况，编译器会报错。可以用 `_` 通配符匹配剩余情况。

**Q: if let 和 match 怎么选？**

A: 只关心一种情况用 `if let`；需要处理多种情况用 `match`。

## 总结

本文学习了 Rust 强大的枚举和模式匹配系统：

**枚举**

- 变体可以携带不同类型的数据
- 支持 `impl` 定义方法
- `Option<T>` 替代 null，保证安全

**模式匹配**

- `match` 表达式：穷举所有情况
- `if let`：简化单一匹配
- `while let`：循环匹配
- 匹配守卫、绑定变量、范围匹配

**核心思想**：通过类型系统和模式匹配，让编译器帮你检查所有可能的情况，减少运行时错误。
