## 问题场景

在上一篇文章中，我们用结构体定义了任务数据结构。但在实际的任务处理系统中，任务的状态会不断变化：待处理、进行中、完成、失败等。如何用类型系统来安全地表达这些状态呢？

如果我们用整数来表示状态：

```rust
fn main() {
    let status = 1;  // 1 表示"进行中"
    if status == 1 {
        println!("任务进行中");
    }
}
```

这种方式的问题是：

1. **不安全**：任何整数都可以赋值给 status，包括无效的状态码
2. **不清晰**：数字 1 的含义不明确，需要查看文档
3. **难维护**：修改状态值需要在多处修改，容易出错

在 Java 或 C++ 中，我们会用枚举来解决这个问题。但 Rust 的枚举远比这强大——**枚举变体可以携带数据**。

与枚举配合使用的**模式匹配（Pattern Matching）** 是 Rust 中处理分支逻辑的利器，比传统的 if-else 更安全、更强大。

## 定义枚举

### 基本枚举

```rust
// 定义任务状态枚举
enum TaskStatus {
    Pending,      // 待处理
    InProgress,   // 进行中
    Completed,    // 已完成
    Failed,       // 已失败
}

fn main() {
    let status = TaskStatus::InProgress;

    // 使用 match 处理枚举
    match status {
        TaskStatus::Pending => println!("任务待处理"),
        TaskStatus::InProgress => println!("任务进行中"),
        TaskStatus::Completed => println!("任务已完成"),
        TaskStatus::Failed => println!("任务失败"),
    }
}
```

### 携带数据的枚举

Rust 枚举的每个变体可以携带不同类型和数量的数据：

```rust
// 定义任务执行结果枚举
enum TaskResult {
    Success,                           // 成功，无额外数据
    PartialSuccess { completed: usize, total: usize },  // 部分成功，携带命名字段
    Error(String),                     // 错误，携带错误信息
    Timeout(u32),                      // 超时，携带超时时长（秒）
}

fn main() {
    let result1 = TaskResult::Success;
    let result2 = TaskResult::PartialSuccess { completed: 8, total: 10 };
    let result3 = TaskResult::Error(String::from("网络连接失败"));
    let result4 = TaskResult::Timeout(30);

    // 使用 match 处理不同情况
    process_result(result1);
    process_result(result2);
    process_result(result3);
    process_result(result4);
}

fn process_result(result: TaskResult) {
    match result {
        TaskResult::Success => println!("任务执行成功"),
        TaskResult::PartialSuccess { completed, total } => {
            println!("任务部分成功: {}/{}", completed, total);
        }
        TaskResult::Error(msg) => println!("任务执行失败: {}", msg),
        TaskResult::Timeout(seconds) => println!("任务执行超时: {}秒", seconds),
    }
}
```

### 枚举变体的三种形式

```rust
enum Message {
    Quit,                       // 单元变体（无数据）
    Move { x: i32, y: i32 },    // 结构体变体（命名字段）
    Write(String),              // 元组变体（位置字段）
}

fn main() {
    let msg1 = Message::Quit;
    let msg2 = Message::Move { x: 10, y: 20 };
    let msg3 = Message::Write(String::from("hello"));

    // 匹配不同形式的变体
    match msg1 {
        Message::Quit => println!("退出"),
        Message::Move { x, y } => println!("移动到 ({}, {})", x, y),
        Message::Write(text) => println!("写入: {}", text),
    }
}
```

## 为枚举实现方法

和结构体一样，枚举也可以用 `impl` 定义方法：

```rust
#[derive(Debug)]
enum TaskStatus {
    Pending,
    InProgress,
    Completed,
    Failed,
}

impl TaskStatus {
    // 获取状态的描述
    fn description(&self) -> &str {
        match self {
            TaskStatus::Pending => "待处理",
            TaskStatus::InProgress => "进行中",
            TaskStatus::Completed => "已完成",
            TaskStatus::Failed => "失败",
        }
    }

    // 判断是否可以执行
    fn can_execute(&self) -> bool {
        matches!(self, TaskStatus::Pending)
    }

    // 判断是否已完成（成功或失败）
    fn is_finished(&self) -> bool {
        matches!(self, TaskStatus::Completed | TaskStatus::Failed)
    }
}

fn main() {
    let status = TaskStatus::InProgress;
    println!("状态: {}", status.description());
    println!("可执行: {}", status.can_execute());
    println!("已完成: {}", status.is_finished());
}
```

## Option 枚举

`Option` 是 Rust 标准库中最重要的枚举之一，用于表示"一个值可能存在，也可能不存在"。

### 定义

```rust
// Rust 标准库中的定义
enum Option<T> {
    Some(T),   // 有值
    None,      // 无值
}
```

Rust 没有 `null`，用 `Option` 代替。你不需要手动引入它，它在 Prelude（预导入）中。

### 基本使用

```rust
fn main() {
    let some_number: Option<i32> = Some(5);
    let some_string: Option<&str> = Some("hello");
    let absent_number: Option<i32> = None;

    println!("{:?}", some_number);    // Some(5)
    println!("{:?}", some_string);   // Some("hello")
    println!("{:?}", absent_number);  // None
}
```

### Option 强制处理空值

```rust
fn main() {
    let x: Option<i32> = Some(5);

    // 错误：不能直接把 Option 当作 i32 使用
    // let sum = x + 1;  // 编译错误

    // 正确：必须先处理 Option
    let sum = match x {
        Some(num) => num + 1,
        None => 0,
    };
    println!("sum = {}", sum);  // 6
}
```

这就是 Rust 消除空指针异常的方式——**类型系统强制你考虑"值不存在"的情况**。

### 在任务处理系统中使用 Option

```rust
struct Task {
    id: u32,
    name: String,
    result: Option<String>,  // 任务结果可能存在，也可能不存在
    error_message: Option<String>,  // 错误信息可能存在
}

impl Task {
    fn new(id: u32, name: &str) -> Self {
        Self {
            id,
            name: name.to_string(),
            result: None,
            error_message: None,
        }
    }

    // 设置任务结果
    fn set_result(&mut self, result: String) {
        self.result = Some(result);
        self.error_message = None;  // 清除错误信息
    }

    // 设置错误信息
    fn set_error(&mut self, error: String) {
        self.error_message = Some(error);
        self.result = None;  // 清除结果
    }

    // 获取任务结果
    fn get_result(&self) -> Option<&String> {
        self.result.as_ref()
    }

    // 判断任务是否成功
    fn is_successful(&self) -> bool {
        self.result.is_some()
    }

    // 获取显示信息
    fn get_display_info(&self) -> String {
        match &self.result {
            Some(result) => format!("任务成功: {}", result),
            None => match &self.error_message {
                Some(error) => format!("任务失败: {}", error),
                None => format!("任务待执行"),
            }
        }
    }
}

fn main() {
    let mut task = Task::new(1, "数据处理");

    println!("{}", task.get_display_info());  // 任务待执行

    task.set_result("处理了 100 条数据".to_string());
    println!("{}", task.get_display_info());  // 任务成功

    task.set_error("内存不足".to_string());
    println!("{}", task.get_display_info());  // 任务失败
}
```

### Option 的常用方法

```rust
fn main() {
    let some_val: Option<i32> = Some(42);
    let none_val: Option<i32> = None;

    // is_some / is_none
    println!("有值: {}", some_val.is_some());    // true
    println!("无值: {}", none_val.is_none());    // true

    // unwrap：取出值，如果是 None 则 panic
    // let val = none_val.unwrap();  // 会 panic

    // unwrap_or：提供默认值
    let val = none_val.unwrap_or(0);
    println!("默认值: {}", val);  // 0

    // map：转换 Some 内部的值
    let doubled = some_val.map(|x| x * 2);
    println!("翻倍: {:?}", doubled);  // Some(84)

    // and_then：链式操作
    let result = some_val.and_then(|x| {
        if x > 0 {
            Some(x * 2)
        } else {
            None
        }
    });
    println!("结果: {:?}", result);  // Some(84)
}
```

## 模式匹配

### match 表达式

`match` 是 Rust 最强大的控制流运算符，它检查一个值是否匹配一系列模式：

```rust
fn main() {
    let number = 3;

    match number {
        1 => println!("一"),
        2 => println!("二"),
        3 => println!("三"),
        4 | 5 => println!("四或五"),  // | 表示"或"
        6..=10 => println!("六到十"),  // ..= 表示范围
        _ => println!("其他"),        // _ 匹配所有剩余情况
    }
}
```

**重要**：`match` 的每个分支必须覆盖所有可能的情况，否则编译错误。`_` 是通配符，匹配任何值。

### 匹配守卫（Match Guard）

在模式后面添加 `if` 条件：

```rust
fn main() {
    let num = Some(4);

    match num {
        Some(x) if x < 5 => println!("小于 5: {}", x),
        Some(x) => println!("大于等于 5: {}", x),
        None => println!("没有值"),
    }

    // 匹配偶数
    let x = 4;
    match x {
        n if n % 2 == 0 => println!("{} 是偶数", n),
        n => println!("{} 是奇数", n),
    }
}
```

### if let 语法糖

当只需要匹配一个变体，忽略其他情况时，用 `if let`：

```rust
fn main() {
    let some_val = Some(3);

    // match 写法（啰嗦）
    match some_val {
        Some(3) => println!("三"),
        _ => (),
    }

    // if let 写法（简洁）
    if let Some(3) = some_val {
        println!("三");
    }

    // if let 也可以用 else
    if let Some(x) = some_val {
        println!("值是 {}", x);
    } else {
        println!("没有值");
    }
}
```

### while let

`while let` 循环只要模式匹配就继续：

```rust
fn main() {
    let mut tasks = vec![1, 2, 3];

    // 持续弹出直到 Vec 为空
    while let Some(task_id) = tasks.pop() {
        println!("处理任务: {}", task_id);
    }
    // 输出：3 2 1
}
```

## 实战：任务处理系统的状态管理

让我们用枚举和模式匹配来实现一个完整的任务状态管理系统：

```rust
// 任务优先级
#[derive(Debug, Clone, Copy)]
enum Priority {
    Low,
    Medium,
    High,
}

// 任务状态
#[derive(Debug, Clone, Copy)]
enum TaskStatus {
    Pending,
    InProgress,
    Completed,
    Failed,
    Cancelled,
}

// 任务执行结果
enum ExecutionResult {
    Success(String),
    PartialSuccess { processed: usize, total: usize },
    Error(String),
    Timeout(u32),
}

// 任务结构体
struct Task {
    id: u32,
    name: String,
    priority: Priority,
    status: TaskStatus,
    result: Option<String>,
    error: Option<String>,
}

impl Task {
    fn new(id: u32, name: &str, priority: Priority) -> Self {
        Self {
            id,
            name: name.to_string(),
            priority,
            status: TaskStatus::Pending,
            result: None,
            error: None,
        }
    }

    // 执行任务，返回执行结果
    fn execute(&mut self) -> ExecutionResult {
        // 更新状态为进行中
        self.status = TaskStatus::InProgress;

        // 模拟任务执行
        let result = self.simulate_execution();

        // 根据执行结果更新状态
        match &result {
            ExecutionResult::Success(msg) => {
                self.status = TaskStatus::Completed;
                self.result = Some(msg.clone());
            }
            ExecutionResult::PartialSuccess { .. } => {
                self.status = TaskStatus::Completed;
                self.result = Some("部分成功".to_string());
            }
            ExecutionResult::Error(err) => {
                self.status = TaskStatus::Failed;
                self.error = Some(err.clone());
            }
            ExecutionResult::Timeout(_) => {
                self.status = TaskStatus::Failed;
                self.error = Some("执行超时".to_string());
            }
        }

        result
    }

    // 模拟任务执行（简化版）
    fn simulate_execution(&self) -> ExecutionResult {
        // 根据优先级模拟不同的结果
        match self.priority {
            Priority::High => ExecutionResult::Success("任务执行成功".to_string()),
            Priority::Medium => ExecutionResult::PartialSuccess {
                processed: 8,
                total: 10,
            },
            Priority::Low => ExecutionResult::Timeout(30),
        }
    }

    // 获取任务显示信息
    fn get_info(&self) -> String {
        let status_str = match self.status {
            TaskStatus::Pending => "待处理",
            TaskStatus::InProgress => "进行中",
            TaskStatus::Completed => "已完成",
            TaskStatus::Failed => "失败",
            TaskStatus::Cancelled => "已取消",
        };

        let priority_str = match self.priority {
            Priority::Low => "低",
            Priority::Medium => "中",
            Priority::High => "高",
        };

        let result_str = match &self.result {
            Some(msg) => format!("结果: {}", msg),
            None => match &self.error {
                Some(err) => format!("错误: {}", err),
                None => String::new(),
            }
        };

        format!(
            "任务[{}] {} - 优先级:{}, 状态:{} {}",
            self.id, self.name, priority_str, status_str, result_str
        )
    }
}

// 任务处理器
struct TaskProcessor {
    tasks: Vec<Task>,
}

impl TaskProcessor {
    fn new() -> Self {
        Self { tasks: Vec::new() }
    }

    fn add_task(&mut self, task: Task) {
        self.tasks.push(task);
    }

    // 执行所有待处理的任务
    fn execute_all(&mut self) {
        println!("开始执行任务...");

        for task in &mut self.tasks {
            if matches!(task.status, TaskStatus::Pending) {
                println!("执行任务: {}", task.name);
                let result = task.execute();

                // 打印执行结果
                match result {
                    ExecutionResult::Success(msg) => {
                        println!("  成功: {}", msg);
                    }
                    ExecutionResult::PartialSuccess { processed, total } => {
                        println!("  部分成功: {}/{}", processed, total);
                    }
                    ExecutionResult::Error(err) => {
                        println!("  错误: {}", err);
                    }
                    ExecutionResult::Timeout(seconds) => {
                        println!("  超时: {}秒", seconds);
                    }
                }
            }
        }
    }

    // 列出所有任务
    fn list_tasks(&self) {
        println!("\n=== 任务列表 ===");
        for task in &self.tasks {
            println!("{}", task.get_info());
        }
        println!("================\n");
    }
}

fn main() {
    let mut processor = TaskProcessor::new();

    // 添加不同优先级的任务
    processor.add_task(Task::new(1, "数据备份", Priority::High));
    processor.add_task(Task::new(2, "日志清理", Priority::Low));
    processor.add_task(Task::new(3, "报表生成", Priority::Medium));

    // 列出任务
    processor.list_tasks();

    // 执行所有任务
    processor.execute_all();

    // 再次列出任务，查看状态变化
    processor.list_tasks();
}
```

## 常见错误与解决

### 错误1：match 没有覆盖所有情况

```rust
fn main() {
    let x: Option<i32> = Some(5);

    // 错误：没有处理 None
    // match x {
    //     Some(n) => println!("{}", n),
    // }

    // 解决：处理所有情况
    match x {
        Some(n) => println!("{}", n),
        None => println!("无值"),
    }
}
```

### 错误2：混淆 Option 和值

```rust
fn main() {
    let x: Option<i32> = Some(5);

    // 错误：不能直接做运算
    // let y = x + 1;

    // 解决：先解包
    let y = x.unwrap() + 1;
    println!("{}", y);
}
```

## 枚举与模式匹配的权衡

### 优势

1. **类型安全**：编译器强制处理所有可能的情况
2. **表达力强**：枚举变体可以携带数据，表达复杂的状态
3. **可读性好**：模式匹配清晰地表达了不同的处理分支
4. **无空指针异常**：`Option` 枚举消除了 null 带来的问题

### 权衡

1. **学习曲线**：初学者需要时间理解模式匹配的语法
2. **代码冗长**：简单的 if-else 可能更简洁
3. **性能考虑**：模式匹配在某些情况下可能不如 if-else 高效

## 总结

枚举和模式匹配是 Rust 强大的类型系统特性。通过枚举，我们可以用类型来表达状态和分支逻辑；通过模式匹配，我们可以安全、清晰地处理不同的状态。

**核心要点：**

1. 枚举变体可以携带不同类型的数据
2. `Option<T>` 替代 null，保证空值安全
3. `match` 表达式必须覆盖所有可能的情况
4. `if let` 和 `while let` 是模式匹配的语法糖
5. 模式匹配让编译器帮你检查所有可能的情况
