## 前言

上一篇文章我们了解了"为什么选择 Rust"，现在是时候动手实践了！本文将带你搭建完整的 Rust 开发环境，并编写第一个 Rust 程序。

Rust 的工具链非常现代化，主要依赖两个工具：

- **rustup**：Rust 工具链管理器，负责安装和更新 Rust
- **Cargo**：Rust 的包管理器和构建系统，相当于 npm + webpack 的组合

## 安装 Rust

### 使用 rustup 安装（推荐）

rustup 是 Rust 官方的工具链管理器，可以方便地安装、更新和管理 Rust 版本。

**Windows 系统**：

1. 访问 [rustup.rs](https://rustup.rs/)
2. 下载 `rustup-init.exe`
3. 运行安装程序，按提示完成安装
4. 如果没有 C++ 构建工具，安装程序会提示你安装 Visual Studio Build Tools

**macOS / Linux 系统**：

打开终端，执行：

```bash
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
```

安装过程中会询问安装选项，直接按回车选择默认安装即可：

```
1) Proceed with installation (default)
2) Customize installation
3) Cancel installation
>1
```

安装完成后，重启终端或执行：

```bash
source $HOME/.cargo/env
```

### 验证安装

```bash
# 查看 Rust 版本
rustc --version

# 查看 Cargo 版本
cargo --version

# 查看 rustup 版本
rustup --version
```

输出示例：

```
rustc 1.77.0 (aedd173a2 2024-03-17)
cargo 1.77.0 (3b18e7dda 2024-03-05)
rustup 1.27.0 (bbb9276d2 2024-03-08)
```

### 更新 Rust

Rust 发布周期为 6 周，更新频繁。使用以下命令更新：

```bash
# 更新到最新版本
rustup update
```

### 国内用户加速

如果下载速度慢，可以配置国内镜像源。

**配置环境变量（Shell）**：

```bash
# 设置清华大学镜像源
export RUSTUP_DIST_SERVER="https://rsproxy.cn"
export RUSTUP_UPDATE_ROOT="https://rsproxy.cn/rustup"
```

将以上内容添加到 `~/.bashrc` 或 `~/.zshrc` 中永久生效。

**配置 Cargo 镜像**：

编辑 `~/.cargo/config.toml`（不存在则创建）：

```toml
[source.crates-io]
replace-with = 'rsproxy-sparse'

[source.rsproxy]
registry = "https://rsproxy.cn/crates.io-index"

[source.rsproxy-sparse]
registry = "sparse+https://rsproxy.cn/index/"

[registries.rsproxy]
index = "https://rsproxy.cn/crates.io-index"

[net]
git-fetch-with-cli = true
```

## 开发工具配置

### VSCode（推荐）

VSCode 是目前最受欢迎的 Rust 开发编辑器。

**安装步骤**：

1. 安装 VSCode：https://code.visualstudio.com/

2. 安装插件 **rust-analyzer**（必装）
   - 这是 Rust 官方维护的语言服务器
   - 提供代码补全、类型提示、错误诊断等功能
   - ⚠️ 不要安装旧的 "Rust" 插件，它已弃用

3. 推荐安装的其他插件：
   - **CodeLLDB**：调试支持
   - **Even Better TOML**：Cargo.toml 语法高亮
   - **crates**：依赖版本提示

**VSCode 配置建议**：

在项目根目录创建 `.vscode/settings.json`：

```json
{
  "rust-analyzer.check.command": "clippy",
  "rust-analyzer.cargo.features": "all",
  "editor.formatOnSave": true,
  "[rust]": {
    "editor.defaultFormatter": "rust-lang.rust-analyzer"
  }
}
```

### 其他 IDE 选择

- **RustRover**：JetBrains 出品的专用 Rust IDE（收费，有免费版）
- **CLion**：JetBrains 的 C++ IDE，配合 Rust 插件使用
- **Neovim + rust-analyzer**：适合 Vim 用户

## Cargo：你的好伙伴

Cargo 是 Rust 的包管理器和构建系统，它会伴随你的整个 Rust 之旅。

### 常用命令

```bash
# 创建新项目
cargo new project_name

# 创建库项目（而非可执行程序）
cargo new --lib my_lib

# 初始化当前目录为 Rust 项目
cargo init

# 编译项目（debug 模式）
cargo build

# 编译项目（release 模式，优化更激进）
cargo build --release

# 运行项目
cargo run

# 运行 release 版本
cargo run --release

# 检查代码（比 build 快，只做语法检查）
cargo check

# 运行测试
cargo test

# 生成文档
cargo doc

# 生成文档并在浏览器打开
cargo doc --open

# 格式化代码
cargo fmt

# 代码检查（比编译器更严格）
cargo clippy

# 添加依赖
cargo add serde

# 添加指定版本的依赖
cargo add serde@1.0

# 添加开发依赖（仅用于测试）
cargo add --dev tempfile

# 查看依赖树
cargo tree

# 清理构建产物
cargo clean
```

### Cargo.toml 配置文件

每个 Rust 项目都有一个 `Cargo.toml` 文件，它是项目的配置中心：

```toml
[package]
name = "my_project"        # 项目名称
version = "0.1.0"          # 版本号
edition = "2021"           # Rust 版本（推荐 2021）

[dependencies]
serde = "1.0"              # 来自 crates.io 的依赖
serde_json = "1.0"

[dev-dependencies]
tempfile = "3"             # 仅用于测试的依赖

[profile.release]
opt-level = 3              # 优化级别 0-3
lto = true                 # 链接时优化
```

## 第一个 Rust 程序

### 创建项目

```bash
# 创建新项目
cargo new hello_rust

# 进入项目目录
cd hello_rust
```

Cargo 会创建以下目录结构：

```
hello_rust/
├── Cargo.toml      # 项目配置文件
├── Cargo.lock      # 依赖锁定文件（自动生成）
└── src/
    └── main.rs     # 主程序文件
```

### 查看 Cargo.toml

```toml
[package]
name = "hello_rust"
version = "0.1.0"
edition = "2021"

[dependencies]
```

### 查看 main.rs

Cargo 已经为我们生成了一个 Hello World 程序：

```rust
fn main() {
    println!("Hello, world!");
}
```

### 运行程序

```bash
cargo run
```

输出：

```
   Compiling hello_rust v0.1.0 (/path/to/hello_rust)
    Finished dev [unoptimized + debuginfo] target(s) in 0.5s
     Running `target/debug/hello_rust`
Hello, world!
```

### 代码解析

让我们逐行理解这个程序：

```rust
// fn 是 function 的缩写，定义函数
// main 是特殊函数名，程序入口
// () 表示没有参数
// 没有返回值类型，意味着返回 unit 类型 ()
fn main() {
    // println! 是一个宏（注意感叹号）
    // 宏会在编译期展开，比函数更灵活
    // 字符串使用双引号
    // 语句以分号结尾
    println!("Hello, world!");
}
```

**关键点**：

1. `fn main()` - 程序入口函数
2. `println!` - 宏，不是函数（注意感叹号）
3. `;` - 语句以分号结尾
4. `{}` - 代码块使用花括号

### 编译 vs 运行

```bash
# 只编译，生成可执行文件
cargo build

# 可执行文件位置
# Linux/macOS: target/debug/hello_rust
# Windows: target/debug/hello_rust.exe

# 直接运行可执行文件
./target/debug/hello_rust
```

### Release 构建

```bash
# Release 模式编译（优化更激进，编译更慢，运行更快）
cargo build --release

# 可执行文件位置
# target/release/hello_rust
```

**Debug vs Release 对比**：

| 特性       | Debug    | Release  |
| ---------- | -------- | -------- |
| 编译速度   | 快       | 慢       |
| 运行速度   | 慢       | 快       |
| 二进制大小 | 大       | 小       |
| 调试信息   | 包含     | 不包含   |
| 适用场景   | 开发调试 | 生产部署 |

## 进阶示例

### 示例 1：用户交互

修改 `src/main.rs`：

```rust
// 导入标准库中的输入输出模块
use std::io;

fn main() {
    // 打印提示信息
    println!("猜数游戏！");
    println!("请输入你猜的数字：");

    // 创建一个可变的空字符串，用于存储用户输入
    // mut 关键字表示变量可变（Rust 默认变量不可变）
    let mut guess = String::new();

    // 读取用户输入
    // &mut guess 是可变引用
    // expect 方法处理可能的错误
    io::stdin()
        .read_line(&mut guess)
        .expect("无法读取输入");

    // 打印用户输入
    // {} 是占位符
    println!("你输入的数字是：{}", guess);
}
```

运行：

```bash
cargo run
```

输出：

```
猜数游戏！
请输入你猜的数字：
42
你输入的数字是：42
```

### 示例 2：使用外部依赖

让我们添加一个随机数库来改进猜数游戏。

**添加依赖**：

```bash
cargo add rand
```

这会在 `Cargo.toml` 中添加：

```toml
[dependencies]
rand = "0.8"
```

**修改 main.rs**：

```rust
use std::io;
// 导入 rand 库的 Rng trait
use rand::Rng;

fn main() {
    println!("=== 猜数游戏 ===");

    // 生成 1 到 100 之间的随机数
    let secret_number = rand::thread_rng().gen_range(1..=100);

    println!("我已经想好了一个 1 到 100 之间的数字");
    println!("请输入你的猜测：");

    let mut guess = String::new();

    io::stdin()
        .read_line(&mut guess)
        .expect("无法读取输入");

    // 去除字符串末尾的换行符
    let guess: u32 = guess
        .trim()                          // 去除空白字符
        .parse()                         // 解析为数字
        .expect("请输入有效的数字！");   // 错误处理

    println!("你猜的是：{}", guess);

    // 比较猜测和答案
    if guess < secret_number {
        println!("太小了！");
    } else if guess > secret_number {
        println!("太大了！");
    } else {
        println!("恭喜你，猜对了！");
    }

    // 打印答案（调试用）
    println!("（答案是：{}）", secret_number);
}
```

运行：

```bash
cargo run
```

### 示例 3：循环和多轮游戏

```rust
use std::io;
use rand::Rng;
use std::cmp::Ordering;  // 比较结果枚举

fn main() {
    println!("=== 猜数游戏 ===");

    let secret_number = rand::thread_rng().gen_range(1..=100);

    println!("我已经想好了一个 1 到 100 之间的数字");
    println!("开始猜吧！");

    // loop 创建无限循环
    loop {
        println!("\n请输入你的猜测：");

        let mut guess = String::new();

        io::stdin()
            .read_line(&mut guess)
            .expect("无法读取输入");

        // 使用 match 处理可能的错误
        let guess: u32 = match guess.trim().parse() {
            Ok(num) => num,           // 解析成功，返回数字
            Err(_) => {               // 解析失败
                println!("请输入有效的数字！");
                continue;             // 跳过本次循环，继续下一次
            }
        };

        println!("你猜的是：{}", guess);

        // 使用 cmp 方法进行比较
        match guess.cmp(&secret_number) {
            Ordering::Less => println!("太小了！"),
            Ordering::Greater => println!("太大了！"),
            Ordering::Equal => {
                println!("🎉 恭喜你，猜对了！");
                break;  // 退出循环
            }
        }
    }
}
```

## Rust 项目结构

随着项目变大，你需要组织代码结构：

```
my_project/
├── Cargo.toml           # 项目配置
├── Cargo.lock           # 依赖锁定（自动生成，不要手动编辑）
├── src/
│   ├── main.rs          # 程序入口（可执行程序）
│   ├── lib.rs           # 库入口（如果创建库）
│   ├── module1.rs       # 自定义模块
│   └── module2/
│       ├── mod.rs       # 模块入口
│       └── sub.rs       # 子模块
├── tests/               # 集成测试
│   └── integration_test.rs
├── benches/             # 性能测试
│   └── benchmark.rs
├── examples/            # 示例代码
│   └── example1.rs
└── target/              # 编译输出（自动生成）
    ├── debug/           # Debug 构建产物
    └── release/         # Release 构建产物
```

## 最佳实践

### 1. 使用 cargo check 加速开发

```bash
# 编写代码时，使用 check 代替 build
cargo check
```

`check` 只做语法检查，不生成可执行文件，速度更快。

### 2. 使用 clippy 提高代码质量

```bash
# 安装 clippy（如果没有）
rustup component add clippy

# 运行检查
cargo clippy
```

Clippy 会给出比编译器更严格的建议。

### 3. 使用 rustfmt 格式化代码

```bash
# 安装 rustfmt（如果没有）
rustup component add rustfmt

# 格式化代码
cargo fmt
```

统一的代码风格让团队协作更顺畅。

### 4. 善用 cargo doc

```bash
# 生成项目文档（包括依赖）
cargo doc --open
```

这会为你的代码和所有依赖生成 HTML 文档。

## 常见问题

**Q: cargo build 很慢怎么办？**

A:

- 使用 `cargo check` 代替 `cargo build` 进行开发
- 配置国内镜像源加速下载
- 考虑使用 `sccache` 缓存编译结果

**Q: 如何查看已安装的 Rust 版本？**

A:

```bash
rustup show
```

**Q: 如何切换 Rust 版本？**

A:

```bash
# 安装特定版本
rustup install 1.75.0

# 切换到特定版本
rustup default 1.75.0

# 使用 nightly 版本
rustup default nightly
```

**Q: Cargo.lock 需要提交到 Git 吗？**

A:

- 对于可执行项目：**需要提交**
- 对于库项目：**不需要提交**

**Q: 如何查看某个命令的详细帮助？**

A:

```bash
cargo --help
cargo build --help
```
