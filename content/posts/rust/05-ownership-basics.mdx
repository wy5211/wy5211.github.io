## 前言

恭喜你来到 Rust 学习的**最关键一章**！

所有权（Ownership）是 Rust 最独特的特性，它让 Rust 无需垃圾回收器（GC）就能保证内存安全。理解所有权是掌握 Rust 的关键——很多初学者在这一点上放弃，但一旦理解，你会发现这是一个优雅的设计。

## 为什么需要所有权？

### 内存管理的三种方式

在编程语言的历史上，内存管理主要有三种方式：

```
┌─────────────────────────────────────────────────────────────┐
│                    内存管理方式对比                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 手动管理（C/C++）                                        │
│     程序员负责 malloc/free                                   │
│     ✅ 性能可控                                              │
│     ❌ 容易出错（内存泄漏、悬垂指针、双重释放）               │
│                                                             │
│  2. 垃圾回收（Java/Go/Python）                               │
│     运行时自动回收不再使用的内存                              │
│     ✅ 安全方便                                              │
│     ❌ GC 停顿、内存占用大、性能不可控                        │
│                                                             │
│  3. 所有权系统（Rust）                                       │
│     编译期静态检查内存安全                                    │
│     ✅ 性能可控、内存安全                                     │
│     ❌ 学习曲线陡峭                                          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### Rust 的解决方案

Rust 选择了第三条路：**通过所有权系统，在编译期保证内存安全**。

这意味着：

- 不需要手动 `malloc/free`
- 不需要垃圾回收器
- 如果代码能编译通过，就不会有内存安全问题

## 栈与堆

理解所有权前，先要理解栈（Stack）和堆（Heap）的区别。

### 栈（Stack）

```
┌─────────────────────────────────────────────────────────────┐
│                         栈                                   │
├─────────────────────────────────────────────────────────────┤
│ 特点：                                                       │
│ - 后进先出（LIFO）                                           │
│ - 大小固定，编译期已知                                        │
│ - 分配和释放非常快                                            │
│ - 自动管理（函数结束时自动释放）                               │
│                                                             │
│ 适合：                                                       │
│ - 基本类型（i32, f64, bool, char）                           │
│ - 固定大小数组                                                │
│ - 函数局部变量                                                │
└─────────────────────────────────────────────────────────────┘
```

### 堆（Heap）

```
┌─────────────────────────────────────────────────────────────┐
│                         堆                                   │
├─────────────────────────────────────────────────────────────┤
│ 特点：                                                       │
│ - 动态分配，大小可变                                          │
│ - 分配需要找足够大的空间，较慢                                 │
│ - 需要手动管理或 GC                                           │
│ - 通过指针访问                                                │
│                                                             │
│ 适合：                                                       │
│ - 动态大小数据（String, Vec）                                 │
│ - 需要在函数间共享的数据                                       │
│ - 大型数据结构                                                │
└─────────────────────────────────────────────────────────────┘
```

### Rust 的处理方式

```rust
fn main() {
    // 栈上数据：大小固定，直接存储
    let x = 5;              // i32，4 字节，在栈上
    let y = 10;             // i32，4 字节，在栈上

    // 堆上数据：动态大小，栈上存指针
    let s1 = String::from("hello");  // String，堆上存数据，栈上存指针+长度+容量

    println!("x = {}, s1 = {}", x, s1);
}
```

**String 的内存布局**：

```
String::from("hello") 的内存布局：

栈上：                          堆上：
┌─────────────────┐            ┌───────────────────────┐
│ ptr ─────────────┼───────────►│ 'h' 'e' 'l' 'l' 'o'  │
│ len  = 5        │            └───────────────────────┘
│ cap  = 5        │
└─────────────────┘
```

## 所有权规则

Rust 的所有权系统遵循三条核心规则：

```
┌─────────────────────────────────────────────────────────────┐
│                    所有权三原则                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 每个值都有一个变量作为它的所有者（owner）                  │
│                                                             │
│  2. 同一时刻，只能有一个所有者                                │
│                                                             │
│  3. 当所有者离开作用域，值会被自动释放                        │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 规则 1：每个值都有一个所有者

```rust
fn main() {
    // s 是字符串 "hello" 的所有者
    let s = String::from("hello");

    println!("{}", s);
}
```

### 规则 2：同一时刻只有一个所有者

当把一个变量赋给另一个变量时，所有权会**转移（Move）**：

```rust
fn main() {
    let s1 = String::from("hello");
    let s2 = s1;  // 所有权从 s1 转移到 s2

    // println!("{}", s1);  // ❌ 编译错误！s1 不再有效
    println!("{}", s2);     // ✅ s2 是新的所有者
}
```

### 规则 3：作用域结束时自动释放

```rust
fn main() {
    {                       // s 在这里无效，尚未声明
        let s = String::from("hello");  // s 从此处开始有效

        // 使用 s
        println!("{}", s);
    }                       // 作用域结束，s 被自动释放（调用 drop）

    // println!("{}", s);   // ❌ 编译错误！s 已无效
}
```

## 移动语义（Move）

### 为什么需要移动？

看一个 C++ 的问题：

```cpp
// C++ 代码（危险！）
#include <string>
#include <iostream>

int main() {
    std::string s1 = "hello";
    std::string s2 = s1;  // 浅拷贝：s1 和 s2 指向同一块内存

    // 当 s1 和 s2 都离开作用域时，同一块内存会被释放两次！
    // 这就是"双重释放"问题
}
```

Rust 通过**移动语义**避免了这个问题：

```rust
fn main() {
    let s1 = String::from("hello");
    let s2 = s1;  // s1 被移动到 s2，s1 不再有效

    // println!("{}", s1);  // ❌ 编译错误！s1 已失效

    // 只有 s2 会被释放，不会双重释放
    println!("{}", s2);
}
```

**移动后的内存状态**：

```
移动前：
s1: [ptr, len, cap] ───► 堆: "hello"

移动后：
s1: [无效]              堆: "hello"
                         ▲
s2: [ptr, len, cap] ─────┘
```

### 哪些类型会移动？

**堆上分配的类型会移动**：String、Vec、Box 等

**栈上的类型会复制**：基本类型（i32, f64, bool, char, 元组等）

```rust
fn main() {
    // 栈上数据：复制（Copy）
    let x = 5;
    let y = x;  // x 的值被复制到 y

    println!("x = {}, y = {}", x, y);  // ✅ 两个都有效

    // 堆上数据：移动（Move）
    let s1 = String::from("hello");
    let s2 = s1;  // s1 被移动

    // println!("{}", s1);  // ❌ 编译错误
    println!("{}", s2);
}
```

### Copy Trait

实现了 `Copy` trait 的类型在赋值时会复制而不是移动：

**实现了 Copy 的类型**：

- 所有整数类型（i32, u64, 等）
- 浮点类型（f32, f64）
- 布尔类型（bool）
- 字符类型（char）
- 元组（当且仅当所有元素都实现了 Copy）
- 不可变引用（&T）

**没有实现 Copy 的类型**：

- String
- Vec
- Box
- 可变引用（&mut T）

```rust
fn main() {
    // Copy 类型
    let a: i32 = 5;
    let b = a;      // 复制，a 仍然有效
    let c = a;      // 再复制，a 还是有效

    // 非 Copy 类型
    let s1 = String::from("hello");
    let s2 = s1;    // 移动，s1 无效
    // let s3 = s1; // ❌ 编译错误
    let s3 = s2;    // 移动，s2 无效
    // let s4 = s2; // ❌ 编译错误

    println!("a = {}, b = {}, c = {}", a, b, c);
    println!("s3 = {}", s3);
}
```

## 克隆（Clone）

如果你确实需要深拷贝，使用 `clone` 方法：

```rust
fn main() {
    let s1 = String::from("hello");
    let s2 = s1.clone();  // 深拷贝，s1 仍然有效

    println!("s1 = {}, s2 = {}", s1, s2);  // ✅ 两个都有效
}
```

**克隆后的内存状态**：

```
克隆后：
s1: [ptr, len, cap] ───► 堆: "hello"
s2: [ptr, len, cap] ───► 堆: "hello"  (另一块内存)
```

**什么时候用 clone？**

- 确实需要独立的数据副本
- 性能不是关键时
- 初学时可以多用，理解所有权后再优化

```rust
fn main() {
    // 示例：需要保留原数据
    let original = String::from("Hello");
    let uppercase = original.clone().to_uppercase();

    println!("原始: {}", original);     // Hello
    println!("大写: {}", uppercase);    // HELLO
}
```

## 函数与所有权

### 传递值给函数

将值传递给函数也会转移所有权：

```rust
fn main() {
    let s = String::from("hello");  // s 进入作用域

    takes_ownership(s);             // s 的值移动到函数里
                                    // s 在这里不再有效

    // println!("{}", s);           // ❌ 编译错误！s 已失效

    let x = 5;                      // x 进入作用域

    makes_copy(x);                  // x 是 Copy 类型，复制到函数
                                    // x 仍然有效

    println!("x = {}", x);          // ✅ x 仍然有效
}

fn takes_ownership(some_string: String) {
    // some_string 进入作用域
    println!("{}", some_string);
}   // some_string 离开作用域，调用 drop，内存被释放

fn makes_copy(some_integer: i32) {
    // some_integer 进入作用域
    println!("{}", some_integer);
}   // some_integer 离开作用域，但因为是 Copy，不会有特殊操作
```

### 返回值与所有权

函数返回值也会转移所有权：

```rust
fn main() {
    let s1 = gives_ownership();         // 所有权转移到 s1

    let s2 = String::from("hello");     // s2 进入作用域
    let s3 = takes_and_gives_back(s2);  // s2 移动到函数，返回值移动到 s3

    // println!("{}", s2);              // ❌ s2 已失效
    println!("{}", s1);
    println!("{}", s3);
}

fn gives_ownership() -> String {
    let some_string = String::from("yours");  // 创建 String
    some_string                                // 返回，所有权移动给调用者
}

fn takes_and_gives_back(a_string: String) -> String {
    a_string  // 返回，所有权移动给调用者
}
```

### 所有权转移的完整示例

```rust
fn main() {
    // 场景：处理字符串

    // 1. 创建字符串（获取所有权）
    let s1 = String::from("hello");

    // 2. 传递给函数（失去所有权）
    print_string(s1);
    // s1 在这里已无效

    // 3. 如果还想用，需要克隆或返回
    let s2 = String::from("hello");
    let s2 = process_and_return(s2);  // 传进去再拿回来
    println!("处理后: {}", s2);

    // 4. 或者使用克隆
    let s3 = String::from("hello");
    print_string(s3.clone());  // 传副本
    println!("s3 还能用: {}", s3);  // s3 仍然有效
}

fn print_string(s: String) {
    println!("{}", s);
}  // s 被释放

fn process_and_return(mut s: String) -> String {
    s.push_str(" world");
    s  // 返回所有权
}
```

## 所有权流程图

```
┌─────────────────────────────────────────────────────────────┐
│                    所有权流转图                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  let s1 = String::from("hello");                            │
│                                                             │
│         ┌──────────┐                                        │
│         │   s1     │ ───► "hello" (堆)                      │
│         └──────────┘                                        │
│                                                             │
│  let s2 = s1;  // Move                                      │
│                                                             │
│         ┌──────────┐                                        │
│         │   s1     │ ───► ❌ 无效                           │
│         └──────────┘                                        │
│         ┌──────────┐                                        │
│         │   s2     │ ───► "hello" (堆)                      │
│         └──────────┘                                        │
│                                                             │
│  let s3 = s2.clone();  // Clone                             │
│                                                             │
│         ┌──────────┐                                        │
│         │   s2     │ ───► "hello" (堆1)                     │
│         └──────────┘                                        │
│         ┌──────────┐                                        │
│         │   s3     │ ───► "hello" (堆2)                     │
│         └──────────┘                                        │
│                                                             │
│  some_function(s2);  // Move to function                    │
│                                                             │
│         ┌──────────┐                                        │
│         │   s2     │ ───► ❌ 无效                           │
│         └──────────┘                                        │
│                                                             │
│  // 函数结束时，堆1被释放                                     │
│  // 作用域结束时，s3 被释放，堆2被释放                         │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 实战演练

### 示例 1：计算字符串长度

```rust
fn main() {
    let s1 = String::from("hello");

    // 方式 1：转移所有权
    let len = calculate_length_move(s1);
    // println!("{}", s1);  // ❌ s1 已失效
    println!("长度: {}", len);

    // 方式 2：克隆（稍后学习更好的方式：引用）
    let s2 = String::from("hello");
    let len = calculate_length_clone(s2.clone());
    println!("s2: {}, 长度: {}", s2, len);  // s2 仍有效
}

// 接收所有权
fn calculate_length_move(s: String) -> usize {
    s.len()
}  // s 被释放

// 接收克隆
fn calculate_length_clone(s: String) -> usize {
    s.len()
}  // s 的克隆被释放，原字符串不受影响
```

### 示例 2：修改字符串

```rust
fn main() {
    let s = String::from("hello");

    // 传递并返回
    let s = append_world(s);
    println!("{}", s);  // hello world

    // 更好的方式：先克隆再修改
    let original = String::from("hello");
    let modified = append_world(original.clone());
    println!("原始: {}", original);
    println!("修改: {}", modified);
}

fn append_world(mut s: String) -> String {
    s.push_str(" world");
    s
}
```

### 示例 3：处理多个字符串

```rust
fn main() {
    let words = vec![
        String::from("hello"),
        String::from("world"),
        String::from("rust"),
    ];

    // 处理每个字符串
    for word in words {
        // word 的所有权转移到循环体
        println!("{} (长度: {})", word, word.len());
    }  // 每个 word 在这里被释放

    // words 已无效（元素被移动）
    // println!("{:?}", words);  // ❌ 编译错误
}
```

### 示例 4：构建字符串

```rust
fn main() {
    let mut result = String::new();

    result = append_line(result, "第一行");
    result = append_line(result, "第二行");
    result = append_line(result, "第三行");

    println!("{}", result);
}

fn append_line(mut s: String, line: &str) -> String {
    s.push_str(line);
    s.push('\n');
    s  // 返回所有权
}
```

## 常见错误与修复

### 错误 1：使用已移动的值

```rust
fn main() {
    let s = String::from("hello");
    let s2 = s;

    // println!("{}", s);  // ❌ 错误：s 已被移动

    // 修复方式 1：使用 clone
    let s3 = String::from("hello");
    let s4 = s3.clone();
    println!("s3: {}, s4: {}", s3, s4);

    // 修复方式 2：使用引用（下一章学习）
}
```

### 错误 2：函数消耗了所有权

```rust
fn main() {
    let s = String::from("hello");
    print_string(s);
    // println!("{}", s);  // ❌ 错误：s 已被移动到函数

    // 修复：使用引用或返回
}

fn print_string(s: String) {
    println!("{}", s);
}
```

### 错误 3：向量元素移动

```rust
fn main() {
    let v = vec![String::from("a"), String::from("b")];
    let first = v[0];  // ❌ 错误：不能移动向量的元素

    // 修复：使用引用或 clone
    let first = v[0].clone();  // ✅
    let first_ref = &v[0];     // ✅
}
```

## 最佳实践

1. **优先使用引用**：如果只需要读取数据，用引用而不是转移所有权
2. **避免不必要的 clone**：clone 有性能开销，理解所有权后可以优化
3. **设计函数签名时考虑所有权**：
   - 只读：使用引用
   - 需要消费：转移所有权
   - 需要修改但保留原值：转移后返回

## 常见问题

**Q: 为什么 String 会移动但 i32 不会？**

A: 因为 String 在堆上分配，需要管理内存；i32 是固定大小，直接存储在栈上，复制成本很低。

**Q: 所有权会让代码变得复杂吗？**

A: 初期可能会觉得复杂，但它能避免很多运行时错误。习惯后，你会写出更安全的代码。

**Q: 什么时候用 clone？**

A: 当你需要独立的数据副本时。初学时可以多用，理解所有权后再优化。

**Q: 如何判断一个类型是 Copy 还是 Move？**

A: 基本类型和只包含基本类型的元组/数组是 Copy。String、Vec、Box 等堆分配类型是 Move。

## 总结

本文我们学习了 Rust 最核心的概念——所有权：

**所有权三原则**

1. 每个值都有一个所有者
2. 同一时刻只能有一个所有者
3. 所有者离开作用域时，值被自动释放

**栈与堆**

- 栈：固定大小，自动管理
- 堆：动态大小，需要所有权管理

**移动与复制**

- Move：堆上数据转移所有权
- Copy：栈上数据复制值
- Clone：显式深拷贝

**函数与所有权**

- 传递值会转移所有权
- 返回值会转移所有权
- Copy 类型传递时复制
