## 前言

这是 Rust 渐进式学习系列的最后一篇文章。在之前的文章中，我们构建了一个完整的任务处理 API 服务器。然而，在实际运维中，我们经常需要快速地查看任务状态、提交新任务或取消正在运行的任务。

虽然可以使用 `curl` 或 Postman 等工具与 API 交互，但一个专门设计的命令行工具（CLI）能提供更好的用户体验。Rust 非常适合开发 CLI 工具，编译出的单一二进制文件无需依赖，部署简单，启动迅速。

本文将使用 `clap` 库构建一个功能完善的任务管理 CLI 工具，它将与我们在第 23 篇文章中构建的 API 服务器交互。

## 为什么选择 Rust 开发 CLI

在开始之前，让我们理解为什么 Rust 是开发 CLI 工具的理想选择：

- **单一二进制**：编译后不需要运行时或依赖库，直接部署即可运行
- **启动速度快**：没有 JVM 启动那样的预热时间
- **内存占用小**：相比于 Node.js 或 Python，资源消耗更低
- **类型安全**：编译时检查减少运行时错误
- **跨平台**：一次编写，到处编译

## 命令行参数设计

使用 `clap` 定义命令行接口：

```rust
use clap::{Parser, Subcommand};
use colored::*;

/// 任务管理命令行工具
#[derive(Parser)]
#[command(name = "taskctl")]
#[command(author = "Your Name")]
#[command(version = "0.1.0")]
#[command(about = "分布式任务管理 CLI 工具", long_about = None)]
struct Cli {
    /// API 服务器地址
    #[arg(short, long, default_value = "http://localhost:3000", global = true)]
    pub api_url: String,

    /// 输出格式
    #[arg(short, long, default_value = "pretty", global = true)]
    pub output: String,

    /// 子命令
    #[command(subcommand)]
    pub command: Commands,
}

/// 可用子命令
#[derive(Subcommand)]
enum Commands {
    /// 创建新任务
    Create {
        /// 任务类型
        #[arg(short, long)]
        task_type: String,

        /// 任务描述
        #[arg(short, long)]
        description: String,

        /// 任务优先级
        #[arg(short, long, default_value = "normal")]
        priority: String,
    },

    /// 获取任务详情
    Get {
        /// 任务 ID
        task_id: String,
    },

    /// 列出所有任务
    List {
        /// 按状态过滤
        #[arg(short, long)]
        status: Option<String>,

        /// 按类型过滤
        #[arg(short = 't', long)]
        task_type: Option<String>,

        /// 页码
        #[arg(short, long, default_value = "1")]
        page: usize,

        /// 每页数量
        #[arg(short = 'P', long, default_value = "20")]
        per_page: usize,
    },

    /// 更新任务
    Update {
        /// 任务 ID
        task_id: String,

        /// 新的描述
        #[arg(short, long)]
        description: Option<String>,

        /// 新的优先级
        #[arg(short, long)]
        priority: Option<String>,
    },

    /// 取消任务
    Cancel {
        /// 任务 ID
        task_id: String,
    },

    /// 删除任务
    Delete {
        /// 任务 ID
        task_id: String,
    },

    /// 监控任务状态
    Watch {
        /// 任务 ID
        task_id: String,

        /// 刷新间隔（秒）
        #[arg(short, long, default_value = "2")]
        interval: u64,
    },

    /// 显示统计信息
    Stats,
}
```

## HTTP 客户端实现

实现与 API 服务器交互的客户端：

```rust
use reqwest::Client;
use serde::{Deserialize, Serialize};
use std::str::FromStr;
use uuid::Uuid;
use anyhow::{Result, Context};

/// 任务优先级
#[derive(Debug, Clone, Copy, Serialize, Deserialize)]
#[serde(rename_all = "lowercase")]
pub enum TaskPriority {
    Low,
    Normal,
    High,
    Urgent,
}

impl FromStr for TaskPriority {
    type Err = String;

    fn from_str(s: &str) -> Result<Self, Self::Err> {
        match s.to_lowercase().as_str() {
            "low" => Ok(TaskPriority::Low),
            "normal" => Ok(TaskPriority::Normal),
            "high" => Ok(TaskPriority::High),
            "urgent" => Ok(TaskPriority::Urgent),
            _ => Err(format!("无效的优先级: {}", s)),
        }
    }
}

/// 任务状态
#[derive(Debug, Clone, Copy, Serialize, Deserialize)]
#[serde(rename_all = "lowercase")]
pub enum TaskStatus {
    Pending,
    Running,
    Completed,
    Failed,
    Cancelled,
}

impl FromStr for TaskStatus {
    type Err = String;

    fn from_str(s: &str) -> Result<Self, Self::Err> {
        match s.to_lowercase().as_str() {
            "pending" => Ok(TaskStatus::Pending),
            "running" => Ok(TaskStatus::Running),
            "completed" => Ok(TaskStatus::Completed),
            "failed" => Ok(TaskStatus::Failed),
            "cancelled" => Ok(TaskStatus::Cancelled),
            _ => Err(format!("无效的状态: {}", s)),
        }
    }
}

/// 任务结构
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct Task {
    pub id: Uuid,
    pub task_type: String,
    pub description: String,
    pub status: TaskStatus,
    pub priority: TaskPriority,
    pub created_at: String,
    pub started_at: Option<String>,
    pub completed_at: Option<String>,
    pub result: Option<String>,
    pub error: Option<String>,
    pub progress: u8,
}

/// API 响应
#[derive(Debug, Deserialize)]
pub struct ApiResponse<T> {
    pub success: bool,
    pub data: Option<T>,
    pub error: Option<String>,
}

/// API 客户端
pub struct ApiClient {
    client: Client,
    base_url: String,
}

impl ApiClient {
    pub fn new(base_url: String) -> Self {
        Self {
            client: Client::new(),
            base_url,
        }
    }

    /// 创建任务
    pub async fn create_task(
        &self,
        task_type: String,
        description: String,
        priority: TaskPriority,
    ) -> Result<Task> {
        #[derive(Serialize)]
        struct CreateRequest {
            task_type: String,
            description: String,
            priority: TaskPriority,
        }

        let response = self
            .client
            .post(format!("{}/tasks", self.base_url))
            .json(&CreateRequest {
                task_type,
                description,
                priority,
            })
            .send()
            .await?;

        let api_response: ApiResponse<Task> = response.json().await?;

        api_response.data.ok_or_else(|| {
            anyhow::anyhow!(
                "创建任务失败: {}",
                api_response.error.unwrap_or_default()
            )
        })
    }

    /// 获取任务
    pub async fn get_task(&self, task_id: Uuid) -> Result<Task> {
        let response = self
            .client
            .get(format!("{}/tasks/{}", self.base_url, task_id))
            .send()
            .await?;

        let api_response: ApiResponse<Task> = response.json().await?;

        api_response.data.ok_or_else(|| {
            anyhow::anyhow!(
                "获取任务失败: {}",
                api_response.error.unwrap_or_default()
            )
        })
    }

    /// 列出任务
    pub async fn list_tasks(
        &self,
        status: Option<TaskStatus>,
        task_type: Option<String>,
        page: usize,
        per_page: usize,
    ) -> Result<Vec<Task>> {
        let mut url = format!("{}/tasks?", self.base_url);

        if let Some(status) = status {
            url.push_str(&format!("status={}&", status));
        }
        if let Some(task_type) = task_type {
            url.push_str(&format!("task_type={}&", task_type));
        }
        url.push_str(&format!("page={}&per_page={}", page, per_page));

        let response = self.client.get(&url).send().await?;
        let api_response: ApiResponse<Vec<Task>> = response.json().await?;

        api_response.data.ok_or_else(|| {
            anyhow::anyhow!(
                "获取任务列表失败: {}",
                api_response.error.unwrap_or_default()
            )
        })
    }

    /// 更新任务
    pub async fn update_task(
        &self,
        task_id: Uuid,
        description: Option<String>,
        priority: Option<TaskPriority>,
    ) -> Result<Task> {
        #[derive(Serialize)]
        struct UpdateRequest {
            description: Option<String>,
            priority: Option<TaskPriority>,
        }

        let response = self
            .client
            .put(format!("{}/tasks/{}", self.base_url, task_id))
            .json(&UpdateRequest {
                description,
                priority,
            })
            .send()
            .await?;

        let api_response: ApiResponse<Task> = response.json().await?;

        api_response.data.ok_or_else(|| {
            anyhow::anyhow!(
                "更新任务失败: {}",
                api_response.error.unwrap_or_default()
            )
        })
    }

    /// 取消任务
    pub async fn cancel_task(&self, task_id: Uuid) -> Result<Task> {
        let response = self
            .client
            .post(format!("{}/tasks/{}/cancel", self.base_url, task_id))
            .send()
            .await?;

        let api_response: ApiResponse<Task> = response.json().await?;

        api_response.data.ok_or_else(|| {
            anyhow::anyhow!(
                "取消任务失败: {}",
                api_response.error.unwrap_or_default()
            )
        })
    }

    /// 删除任务
    pub async fn delete_task(&self, task_id: Uuid) -> Result<()> {
        let response = self
            .client
            .delete(format!("{}/tasks/{}", self.base_url, task_id))
            .send()
            .await?;

        if response.status().is_success() {
            Ok(())
        } else {
            Err(anyhow::anyhow!("删除任务失败"))
        }
    }

    /// 获取统计信息
    pub async fn get_stats(&self) -> Result<Vec<Task>> {
        self.list_tasks(None, None, 1, 1000)
    }
}
```

## 输出格式化

实现美观的命令行输出：

```rust
use colored::*;

/// 格式化任务详情
pub fn format_task_detail(task: &Task) -> String {
    let status_color = match task.status {
        TaskStatus::Pending => "yellow",
        TaskStatus::Running => "blue",
        TaskStatus::Completed => "green",
        TaskStatus::Failed => "red",
        TaskStatus::Cancelled => "dim",
    };

    let priority_color = match task.priority {
        TaskPriority::Low => "dim",
        TaskPriority::Normal => "white",
        TaskPriority::High => "yellow",
        TaskPriority::Urgent => "red",
    };

    format!(
        "{}\n\
         {}\n\
         {}\n\
         {}\n\
         {}\n\
         {}\n\
         {}\n\
         {}\n\
         {}\n\
         {}",
        format!("任务 ID: {}", task.id).bold(),
        format!("类型: {}", task.task_type.cyan()),
        format!("描述: {}", task.description.white()),
        format!(
            "状态: {}",
            format!("{:?}", task.status).color(status_color).bold()
        ),
        format!(
            "优先级: {}",
            format!("{:?}", task.priority).color(priority_color)
        ),
        format!("进度: {}%", task.progress),
        format!("创建时间: {}", task.created_at.dim()),
        format!(
            "开始时间: {}",
            task.started_at.as_ref().map(|s| s.as_str()).unwrap_or("未开始").dim()
        ),
        format!(
            "完成时间: {}",
            task.completed_at.as_ref().map(|s| s.as_str()).unwrap_or("未完成").dim()
        ),
        if let Some(ref error) = task.error {
            format!("错误: {}", error.red().bold())
        } else {
            format!("错误: {}", "无".dim())
        }
    )
}

/// 格式化任务列表
pub fn format_task_list(tasks: &[Task]) -> String {
    if tasks.is_empty() {
        return "没有找到任务".dim().to_string();
    }

    let mut output = String::new();

    for task in tasks {
        let status_symbol = match task.status {
            TaskStatus::Pending => "○".yellow(),
            TaskStatus::Running => "◉".blue(),
            TaskStatus::Completed => "●".green(),
            TaskStatus::Failed => "✗".red(),
            TaskStatus::Cancelled => "⊘".dim(),
        };

        output.push_str(
            &format!(
                "{} {} - {} [{}]",
                status_symbol,
                task.id,
                task.description,
                format!("{}%", task.progress).dim()
            )
            .bold()
            .to_string(),
        );
        output.push('\n');
    }

    output
}

/// 格式化统计信息
pub fn format_stats(tasks: &[Task]) -> String {
    let total = tasks.len();
    let pending = tasks.iter().filter(|t| t.status == TaskStatus::Pending).count();
    let running = tasks.iter().filter(|t| t.status == TaskStatus::Running).count();
    let completed = tasks.iter().filter(|t| t.status == TaskStatus::Completed).count();
    let failed = tasks.iter().filter(|t| t.status == TaskStatus::Failed).count();
    let cancelled = tasks.iter().filter(|t| t.status == TaskStatus::Cancelled).count();

    format!(
        "\n\
         {}\n\
         {}\n\
         {}\n\
         {}\n\
         {}\n\
         {}\n",
        format!("总任务数: {}", total).bold().white(),
        format!("待执行: {}", pending).yellow(),
        format!("运行中: {}", running).blue(),
        format!("已完成: {}", completed).green(),
        format!("已失败: {}", failed).red(),
        format!("已取消: {}", cancelled).dim(),
    )
}
```

## 主程序实现

将所有组件组合起来：

```rust
use tokio::time::{sleep, Duration};
use std::str::FromStr;

#[tokio::main]
async fn main() -> Result<()> {
    let cli = Cli::parse();

    // 创建 API 客户端
    let client = ApiClient::new(cli.api_url);

    // 处理子命令
    match cli.command {
        Commands::Create {
            task_type,
            description,
            priority,
        } => {
            let priority = TaskPriority::from_str(&priority)?;

            println!("{} 创建任务中...", "→".blue().bold());

            match client
                .create_task(task_type, description, priority)
                .await
            {
                Ok(task) => {
                    println!("{} 任务创建成功", "✓".green().bold());
                    println!("\n{}", format_task_detail(&task));
                }
                Err(e) => {
                    eprintln!("{} 创建任务失败: {}", "✗".red().bold(), e);
                    std::process::exit(1);
                }
            }
        }

        Commands::Get { task_id } => {
            let task_id = Uuid::from_str(&task_id)?;

            match client.get_task(task_id).await {
                Ok(task) => {
                    println!("{}", format_task_detail(&task));
                }
                Err(e) => {
                    eprintln!("{} 获取任务失败: {}", "✗".red().bold(), e);
                    std::process::exit(1);
                }
            }
        }

        Commands::List {
            status,
            task_type,
            page,
            per_page,
        } => {
            let status = status
                .map(|s| TaskStatus::from_str(&s))
                .transpose()?;

            match client
                .list_tasks(status, task_type, page, per_page)
                .await
            {
                Ok(tasks) => {
                    println!("{}", format_task_list(&tasks));
                    println!("\n第 {} 页，显示 {} 个任务", page, tasks.len());
                }
                Err(e) => {
                    eprintln!("{} 获取任务列表失败: {}", "✗".red().bold(), e);
                    std::process::exit(1);
                }
            }
        }

        Commands::Update {
            task_id,
            description,
            priority,
        } => {
            let task_id = Uuid::from_str(&task_id)?;
            let priority = priority
                .map(|p| TaskPriority::from_str(&p))
                .transpose()?;

            println!("{} 更新任务中...", "→".blue().bold());

            match client.update_task(task_id, description, priority).await {
                Ok(task) => {
                    println!("{} 任务更新成功", "✓".green().bold());
                    println!("\n{}", format_task_detail(&task));
                }
                Err(e) => {
                    eprintln!("{} 更新任务失败: {}", "✗".red().bold(), e);
                    std::process::exit(1);
                }
            }
        }

        Commands::Cancel { task_id } => {
            let task_id = Uuid::from_str(&task_id)?;

            println!("{} 取消任务中...", "→".blue().bold());

            match client.cancel_task(task_id).await {
                Ok(task) => {
                    println!("{} 任务已取消", "✓".green().bold());
                    println!("\n{}", format_task_detail(&task));
                }
                Err(e) => {
                    eprintln!("{} 取消任务失败: {}", "✗".red().bold(), e);
                    std::process::exit(1);
                }
            }
        }

        Commands::Delete { task_id } => {
            let task_id = Uuid::from_str(&task_id)?;

            println!("{} 删除任务中...", "→".blue().bold());

            match client.delete_task(task_id).await {
                Ok(_) => {
                    println!("{} 任务已删除", "✓".green().bold());
                }
                Err(e) => {
                    eprintln!("{} 删除任务失败: {}", "✗".red().bold(), e);
                    std::process::exit(1);
                }
            }
        }

        Commands::Watch { task_id, interval } => {
            let task_id = Uuid::from_str(&task_id)?;

            println!("{} 监控任务状态（按 Ctrl+C 退出）\n", "→".blue().bold());

            loop {
                match client.get_task(task_id).await {
                    Ok(task) => {
                        // 清屏并显示任务状态
                        print!("{}[2J{}[0;0H", 27 as char, 27 as char); // ANSI 清屏
                        println!("{}", format_task_detail(&task));
                        println!("\n最后更新: {}", chrono::Local::now().format("%Y-%m-%d %H:%M:%S"));

                        // 如果任务已完成、失败或取消，停止监控
                        match task.status {
                            TaskStatus::Completed => {
                                println!("\n{} 任务已完成", "✓".green().bold());
                                break;
                            }
                            TaskStatus::Failed => {
                                println!("\n{} 任务失败", "✗".red().bold());
                                break;
                            }
                            TaskStatus::Cancelled => {
                                println!("\n{} 任务已取消", "⊘".dim().bold());
                                break;
                            }
                            _ => {}
                        }
                    }
                    Err(e) => {
                        eprintln!("{} 获取任务状态失败: {}", "✗".red().bold(), e);
                    }
                }

                sleep(Duration::from_secs(interval)).await;
            }
        }

        Commands::Stats => {
            match client.get_stats().await {
                Ok(tasks) => {
                    println!("{}", "任务统计".bold().cyan());
                    println!("{}", format_stats(&tasks));
                }
                Err(e) => {
                    eprintln!("{} 获取统计信息失败: {}", "✗".red().bold(), e);
                    std::process::exit(1);
                }
            }
        }
    }

    Ok(())
}
```

## Trade-off 分析

### CLI 工具的优势

1. **快速操作**：无需打开浏览器或 GUI，键盘操作更高效
2. **脚本化**：可以轻松集成到 Shell 脚本中
3. **资源友好**：相比 Web UI，CLI 工具资源消耗更少
4. **远程友好**：通过 SSH 远程操作更方便

### CLI 工具的局限

1. **学习曲线**：需要记住命令和参数
2. **可视化差**：难以展示复杂的图表和趋势
3. **批量操作**：处理多个任务时不如 GUI 直观

### 生产环境改进

在实际生产中，可以考虑以下改进：

1. **配置文件**：支持 `~/.taskctl.yaml` 配置文件
2. **自动补全**：生成 shell 自动补全脚本
3. **批量操作**：支持同时操作多个任务
4. **导出功能**：支持导出为 CSV/JSON 格式
5. **交互模式**：提供 REPL 交互式模式
