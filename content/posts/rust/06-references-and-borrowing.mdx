## 前言

在上一篇中，我们学习了所有权系统——Rust 的灵魂。你可能已经注意到一个问题：每次调用函数都要转移所有权，用完还得传回来，代码变得很啰嗦。

```rust
fn main() {
    let s = String::from("hello");
    let len = calculate_length(s);   // s 被移动走了
    // println!("{}", s);            // ❌ s 已失效
    println!("长度: {}", len);
}

fn calculate_length(s: String) -> usize {
    s.len()
}  // s 被释放
```

如果只是想计算长度，并不想放弃 `s` 的所有权，怎么办？克隆？那太浪费了。

Rust 的解决方案是**引用（Reference）**和**借用（Borrowing）**。

## 什么是引用？

引用就像一个"指针"，它指向某个值，但**不拥有**这个值。

```
┌─────────────────────────────────────────────────────────────┐
│                    引用 vs 所有权                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  所有权（转移）：                                             │
│                                                             │
│  let s1 = String::from("hello");                            │
│  let s2 = s1;   // s1 → ❌ 无效                              │
│                                                             │
│         ┌──────────┐                                        │
│         │   s1     │ ───► ❌                                 │
│         └──────────┘                                        │
│         ┌──────────┐                                        │
│         │   s2     │ ───► "hello" (拥有)                     │
│         └──────────┘                                        │
│                                                             │
│  引用（借用）：                                               │
│                                                             │
│  let s1 = String::from("hello");                            │
│  let r = &s1;    // s1 → ✅ 仍然有效                          │
│                                                             │
│         ┌──────────┐                                        │
│         │   s1     │ ───► "hello" (拥有)                     │
│         └──────────┘       ▲                                 │
│         ┌──────────┐       │                                 │
│         │    r     │ ──────┘ (借看)                          │
│         └──────────┘                                        │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 创建引用

使用 `&` 运算符创建引用：

```rust
fn main() {
    let s1 = String::from("hello");

    // &s1 创建一个指向 s1 的引用
    let len = calculate_length(&s1);

    // s1 仍然有效！
    println!("'{}' 的长度是 {}", s1, len);
}

// 参数类型是 &String（String 的引用）
fn calculate_length(s: &String) -> usize {
    s.len()
}  // s 离开作用域，但因为 s 不拥有所有权，所以什么也不会发生
```

`&s1` 语法创建了一个**指向 `s1` 的引用**，但**没有获取所有权**。函数参数 `&String` 表示"借用一个 String"。

这个过程叫做**借用（Borrowing）**——就像借书一样，你可以看，但书还是图书馆的。

### 引用的关键特性

```rust
fn main() {
    let s = String::from("hello");

    let r1 = &s;   // 借用 s
    let r2 = &s;   // 再次借用 s

    println!("r1: {}, r2: {}", r1, r2);
    println!("s 仍然有效: {}", s);  // ✅ s 一直有效
}
```

- 引用不拥有数据
- 引用离开作用域时，不会释放数据
- 创建引用不会移动原值

## 不可变引用

默认情况下，引用是**不可变**的——你只能读取数据，不能修改：

```rust
fn main() {
    let s = String::from("hello");

    let r = &s;        // 不可变引用
    println!("{}", r);  // ✅ 读取
}
```

### 不可变引用的典型用法

```rust
// 计算字符串长度（只读）
fn string_length(s: &String) -> usize {
    s.len()
}

// 打印字符串（只读）
fn print_string(s: &String) {
    println!("{}", s);
}

// 检查字符串是否为空（只读）
fn is_empty(s: &String) -> bool {
    s.is_empty()
}

// 查找子串（只读）
fn contains_hello(s: &String) -> bool {
    s.contains("hello")
}

fn main() {
    let s = String::from("hello world");

    println!("长度: {}", string_length(&s));
    print_string(&s);
    println!("是否为空: {}", is_empty(&s));
    println!("包含 hello: {}", contains_hello(&s));

    // s 一直有效，可以继续使用
    println!("原文: {}", s);
}
```

多个不可变引用可以同时存在：

```rust
fn main() {
    let s = String::from("hello world");

    let r1 = &s;  // ✅
    let r2 = &s;  // ✅ 多个不可变引用没问题
    let r3 = &s;  // ✅ 想要多少有多少

    println!("{} {} {}", r1, r2, r3);
}
```

## 可变引用

如果你需要修改借来的数据，使用 `&mut` 创建**可变引用**：

```rust
fn main() {
    let mut s = String::from("hello");

    // &mut s 创建可变引用
    change(&mut s);

    println!("{}", s);  // hello, world
}

fn change(s: &mut String) {
    s.push_str(", world");  // 通过可变引用修改数据
}
```

注意两个要点：

1. **被引用的变量本身必须是 `mut` 的**
2. **引用类型也要标注 `mut`**

```rust
fn main() {
    // ❌ 错误：s 不是 mut 的
    let s = String::from("hello");
    // change(&mut s);  // 编译错误

    // ✅ 正确：s 是 mut 的
    let mut s = String::from("hello");
    change(&mut s);
    println!("{}", s);
}

fn change(s: &mut String) {
    s.push_str(", world");
}
```

## 借用规则

这是引用与借用**最核心**的内容，请务必牢记：

```
┌─────────────────────────────────────────────────────────────┐
│                      借用的两大规则                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  规则 1：在同一时刻，你可以拥有                               │
│         ✓ 任意数量的不可变引用（&T）                          │
│         或者                                                  │
│         ✓ 恰好一个可变引用（&mut T）                          │
│                                                             │
│  规则 2：引用必须总是有效的                                    │
│         （不能出现悬垂引用）                                  │
│                                                             │
│  简单记忆：                                                  │
│                                                             │
│  多个读者 ✅    一个写者 ✅                                    │
│  同时读写 ❌    多个写者 ❌                                    │
│                                                             │
│  类比图书馆：                                                │
│  - 很多读者可以同时看一本书 ✅                                │
│  - 一个人可以拿去修改（涂改）✅                                │
│  - 但不能有人在看的时候别人去涂改 ❌                           │
│  - 不能两个人同时涂改 ❌                                      │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 规则 1 详解：不可变引用与可变引用互斥

```rust
fn main() {
    let mut s = String::from("hello");

    // ❌ 错误：不能同时拥有不可变引用和可变引用
    let r1 = &s;       // 不可变引用
    let r2 = &mut s;   // 可变引用 —— 编译错误！
    // println!("{} {}", r1, r2);
}
```

编译器会报错：

```
error[E0502]: cannot borrow `s` as mutable because it is also borrowed as immutable
 --> src/main.rs:5:14
  |
4 |     let r1 = &s;       // 不可变引用
  |              -- immutable borrow occurs here
5 |     let r2 = &mut s;   // 可变引用
  |              ^^^^^^ mutable borrow occurs here
6 |     println!("{} {}", r1, r2);
  |              -- immutable borrow later used here
```

**为什么这样设计？** 因为如果有人在读取数据时，另一个人修改了数据，读者可能会读到不一致的数据（数据竞争）。

### 规则 1：多个可变引用也不行

```rust
fn main() {
    let mut s = String::from("hello");

    let r1 = &mut s;
    let r2 = &mut s;   // ❌ 错误：不能同时有两个可变引用
    // println!("{} {}", r1, r2);
}
```

**为什么？** 因为两个可变引用可能同时修改数据，导致不可预期的结果。

### 非同时作用域是允许的

关键在于**同一时刻**。如果不可变引用不再被使用，就可以创建可变引用：

```rust
fn main() {
    let mut s = String::from("hello");

    let r1 = &s;   // 不可变引用
    println!("{}", r1);  // r1 最后一次使用

    // r1 的作用域已经结束（非词法作用域 NLL）
    let r2 = &mut s;  // ✅ 现在可以创建可变引用
    println!("{}", r2);
}
```

这在 Rust 2018 版本引入的**非词法生命周期（Non-Lexical Lifetimes, NLL）**后变得更加灵活。编译器能判断引用是否真的还在使用，而不是简单地按照大括号来划分作用域。

### 完整示例：借用规则实战

```rust
fn main() {
    let mut data = String::from("Rust");

    // ✅ 场景 1：多个不可变引用
    let r1 = &data;
    let r2 = &data;
    println!("读取 1: {}, 读取 2: {}", r1, r2);

    // ✅ 场景 2：r1、r2 不再使用后，创建可变引用
    let r3 = &mut data;
    r3.push_str(" is awesome");
    println!("修改后: {}", r3);

    // ✅ 场景 3：r3 不再使用后，再次创建不可变引用
    let r4 = &data;
    println!("再次读取: {}", r4);

    println!("原文: {}", data);  // data 全程有效
}
```

## 实战演练

### 示例 1：计算字符串长度（使用引用）

对比上一篇的方案，引用让代码更简洁：

```rust
fn main() {
    let s = String::from("hello world");

    // 使用引用，不需要转移所有权
    let len = calculate_length(&s);

    // s 仍然有效，可以继续使用
    println!("'{}' 的长度是 {}", s, len);
    println!("首字母: {}", s.chars().next().unwrap());
}

// 接收 String 的引用
fn calculate_length(s: &String) -> usize {
    s.len()
}
```

### 示例 2：追加文本（使用可变引用）

```rust
fn main() {
    let mut greeting = String::from("Hello");

    // 追加名字
    append_name(&mut greeting, "Rust");
    println!("{}", greeting);  // Hello, Rust!

    // 追加感叹号
    add_exclamation(&mut greeting);
    println!("{}", greeting);  // Hello, Rust!
}

fn append_name(s: &mut String, name: &str) {
    s.push_str(", ");
    s.push_str(name);
}

fn add_exclamation(s: &mut String) {
    s.push('!');
}
```

### 示例 3：在函数中同时读取和修改

```rust
fn main() {
    let mut text = String::from("hello");

    process_text(&mut text);
    println!("最终结果: {}", text);
}

fn process_text(s: &mut String) {
    // 先读取
    println!("原始文本: {}", s);

    // 再修改
    s.push_str(" world");

    // 再读取修改后的结果
    println!("修改后文本: {}", s);
}
```

注意：在同一个函数内部，可变引用 `s` 的读取和修改是顺序执行的，不会同时发生，所以没问题。

### 示例 4：单词统计器

```rust
fn main() {
    let text = String::from("hello world hello rust world rust rust");

    let word_count = count_words(&text);
    println!("单词统计:");
    for (word, count) in &word_count {
        println!("  '{}': {} 次", word, count);
    }
}

fn count_words(text: &String) -> Vec<(&str, usize)> {
    use std::collections::HashMap;

    let mut map = HashMap::new();

    for word in text.split_whitespace() {
        *map.entry(word).or_insert(0) += 1;
    }

    let mut result: Vec<_> = map.into_iter().collect();
    result.sort_by(|a, b| b.1.cmp(&a.1));  // 按次数降序
    result
}
```

### 示例 5：字符串处理管道

```rust
fn main() {
    let mut content = String::from("  Hello, Rust World!  ");

    // 逐步处理
    trim_spaces(&mut content);
    println!("去除空格: '{}'", content);

    to_lowercase(&mut content);
    println!("转小写: '{}'", content);

    let has_rust = contains_word(&content, "rust");
    println!("包含 'rust': {}", has_rust);

    let word_count = word_count(&content);
    println!("单词数: {}", word_count);
}

fn trim_spaces(s: &mut String) {
    let trimmed = s.trim().to_string();
    *s = trimmed;
}

fn to_lowercase(s: &mut String) {
    let lower = s.to_lowercase();
    *s = lower;
}

fn contains_word(s: &String, word: &str) -> bool {
    s.split_whitespace().any(|w| w == word)
}

fn word_count(s: &String) -> usize {
    s.split_whitespace().count()
}
```

## 悬垂引用

**规则 2：引用必须总是有效的。**

悬垂引用（Dangling Reference）是指一个引用指向了已经被释放的内存。在 C/C++ 中这是一个常见的 bug，但在 Rust 中，**编译器会阻止你创建悬垂引用**。

### Rust 如何防止悬垂引用

```rust
fn main() {
    let reference_to_nothing = dangle();
}

// ❌ 编译错误！
fn dangle() -> &String {
    let s = String::from("hello");  // s 创建

    &s  // 返回 s 的引用
}   // s 离开作用域被释放，引用指向无效内存！
```

编译器会直接报错：

```
error[E0106]: missing lifetime specifier
 --> src/main.rs:5:16
  |
5 | fn dangle() -> &String {
  |                ^ expected named lifetime parameter
```

### 修复方式：返回所有权

```rust
fn main() {
    let s = no_dangle();
    println!("{}", s);  // ✅ s 拥有数据
}

// ✅ 返回 String，所有权转移给调用者
fn no_dangle() -> String {
    let s = String::from("hello");
    s  // s 的所有权被移动出去，不会被释放
}
```

### 为什么 Rust 能防止悬垂引用？

```
┌─────────────────────────────────────────────────────────────┐
│                  悬垂引用的防止                               │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  C 语言：                                                   │
│                                                             │
│  char* dangle() {                                          │
│      char s[] = "hello";                                   │
│      return s;  // ⚠️ 返回局部变量地址                       │
│  }               // s 被销毁，返回的指针是悬垂的              │
│                                                             │
│  Rust：                                                     │
│                                                             │
│  fn dangle() -> &String {                                  │
│      let s = String::from("hello");                        │
│      &s  // ❌ 编译错误！编译器知道 s 会在这里被释放          │
│  }                                                          │
│                                                             │
│  编译器通过检查生命周期，确保引用不会                        │
│  比它指向的数据活得更久。                                    │
│                                                             │
│  这是 Rust 内存安全的核心保证之一。                          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 引用的常见用法

### 引用与切片

引用经常和切片一起使用：

```rust
fn main() {
    let s = String::from("hello world");

    // &String 引用整个字符串
    let whole = &s;
    println!("整个字符串: {}", whole);

    // &s[0..5] 引用字符串的一部分
    let hello = &s[0..5];
    println!("切片: {}", hello);
}
```

### 引用与函数参数

```rust
fn main() {
    let name = String::from("Rust");

    // 传引用：不转移所有权
    greet(&name);
    println!("name 仍然可用: {}", name);

    // 传值：转移所有权
    // consume(name);
    // println!("{}", name);  // ❌ name 已失效
}

fn greet(name: &String) {
    println!("Hello, {}!", name);
}
```

### 字符串字面量就是引用

```rust
fn main() {
    let s: &str = "hello";  // &str 是字符串切片的引用

    // "hello" 存储在程序的二进制文件中
    // s 是指向它的引用

    println!("{}", s);
}
```

`&str` 是不可变引用，因为字符串字面量存储在只读内存中。

## 常见错误与修复

### 错误 1：修改不可变引用

```rust
fn main() {
    let s = String::from("hello");
    let r = &s;
    // r.push_str(" world");  // ❌ 错误：不能通过不可变引用修改数据
}
```

### 错误 2：同时存在可变和不可变引用

```rust
fn main() {
    let mut s = String::from("hello");
    let r1 = &s;
    let r2 = &mut s;  // ❌ 错误：r1 还在使用
    println!("{} {}", r1, r2);
}

// 修复：确保不可变引用不再使用
fn main() {
    let mut s = String::from("hello");
    let r1 = &s;
    println!("{}", r1);  // r1 最后一次使用
    let r2 = &mut s;  // ✅ 现在可以
    println!("{}", r2);
}
```

### 错误 3：创建悬垂引用

```rust
fn main() {
    // let r = dangle();  // ❌ 编译错误
}

// fn dangle() -> &String {
//     let s = String::from("hello");
//     &s  // 返回局部变量的引用
// }

// 修复：返回值而不是引用
fn no_dangle() -> String {
    let s = String::from("hello");
    s
}
```

### 错误 4：对不可变变量创建可变引用

```rust
fn main() {
    let s = String::from("hello");
    // let r = &mut s;  // ❌ 错误：s 不是 mut 的

    // 修复
    let mut s = String::from("hello");
    let r = &mut s;  // ✅
    r.push_str(" world");
    println!("{}", r);
}
```

## 最佳实践

1. **优先使用不可变引用**：默认用 `&T`，只在需要修改时才用 `&mut T`
2. **引用优先于所有权转移**：函数参数尽量用引用，避免不必要的 move
3. **缩短可变引用的作用域**：让可变引用尽早结束，方便后续代码使用
4. **不要过度使用 `clone`**：理解借用规则后，你会发现很少需要 clone
5. **信任编译器**：借用检查器（borrow checker）是你的朋友，不是敌人

## 常见问题

**Q: 引用和指针有什么区别？**

A: 引用本质上是指针，但更安全。引用总是指向有效数据（编译器保证），不能为空（除非用 `Option<&T>`），且有借用规则约束。

**Q: 为什么 Rust 不允许多个可变引用？**

A: 为了防止数据竞争（data race）。数据竞争需要同时满足三个条件：两个或更多指针访问同一数据、至少一个在写入、没有同步机制。Rust 通过借用规则在编译期消除了数据竞争。

**Q: 什么时候用 `&T`，什么时候用 `&mut T`？**

A: 简单原则：只读用 `&T`，需要修改用 `&mut T`。大多数情况下用 `&T` 就够了。

**Q: `&String` 和 `&str` 有什么区别？**

A: `&String` 是 `String` 对象的引用，`&str` 是字符串切片的引用。函数参数更推荐使用 `&str`，因为它同时接受 `&String` 和 `&str`，更灵活。

**Q: 借用规则会不会让代码很难写？**

A: 初期可能会频繁遇到编译错误，但随着你理解规则，你会发现大多数情况下自然就能通过编译。编译器的错误信息也很有帮助，会告诉你该怎么修改。

## 总结

本文我们学习了引用与借用——Rust 所有权系统的另一半：

**引用**

- `&T` 创建不可变引用，只能读取数据
- `&mut T` 创建可变引用，可以修改数据
- 引用不拥有数据，离开作用域不会释放数据

**借用规则**

- 同一时刻：多个不可变引用 **或** 一个可变引用
- 引用必须总是有效的（不能悬垂）
- 被引用的变量需要是 `mut` 才能创建可变引用

**引用的好处**

- 不转移所有权，函数调用后原值仍可用
- 避免不必要的内存拷贝
- 编译器保证内存安全

┌─────────────────────────────────────────────────────────────┐
│                    引用与借用速查表                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  &T     不可变引用   读取数据     可以有多个                  │
│  &mut T 可变引用     读写数据     同时只能有一个              │
│                                                             │

│  ❌ 不可变引用 + 可变引用（同时存在）                         │
