# Rust 渐进式学习计划

## 背景

### 为什么学习 Rust？

Rust 是一门系统级编程语言，由 Mozilla 研发，连续多年在 Stack Overflow 开发者调查中被评为"最受喜爱的编程语言"。它解决了困扰编程界数十年的核心问题：**如何在保证高性能的同时实现内存安全？**

### Rust 存在的必要性

1. **内存安全无需 GC**：C/C++ 性能强悍但内存不安全（空指针、悬垂指针、数据竞争），Java/Go 有 GC 但有停顿风险。Rust 通过所有权系统在编译期保证内存安全，无需垃圾回收器。

2. **并发安全**：Rust 的类型系统在编译期就能防止数据竞争，"如果代码能编译通过，就不会有数据竞争"。

3. **零成本抽象**：Rust 的高级抽象在运行时没有任何性能开销，写起来像高级语言，跑起来像 C。

4. **现代工具链**：Cargo 包管理器、rustfmt 格式化、clippy 代码检查、完善的文档系统。

### Rust 解决的痛点

| 痛点       | C/C++        | Java/Go    | Rust       |
| ---------- | ------------ | ---------- | ---------- |
| 内存泄漏   | 经常         | 罕见（GC） | 编译期阻止 |
| 空指针异常 | 运行时崩溃   | 运行时异常 | 编译期阻止 |
| 数据竞争   | 运行时不确定 | 运行时检测 | 编译期阻止 |
| GC 停顿    | 无           | 有         | 无         |
| 学习曲线   | 陡峭         | 平缓       | 陡峭       |

### Rust 优缺点

**优点**：

- 内存安全无需 GC
- 零成本抽象，性能媲美 C/C++
- 并发安全，编译期防止数据竞争
- 现代化的工具链和包管理
- 强大的类型推断和模式匹配
- 活跃的社区和丰富的生态

**缺点**：

- 学习曲线陡峭（所有权、生命周期）
- 编译速度较慢
- 编译器报错信息有时难以理解（但在不断改进）
- 生态相比成熟语言还有差距
- 不适合快速原型开发
