## 前言

在学习了 Rust 的所有权、并发、异步编程等核心概念后，我们将通过构建一个实际的分布式任务处理系统来综合运用这些知识。

在实际的生产环境中，任务处理系统通常需要提供 HTTP API 供其他服务调用。比如，一个电商系统可能需要异步处理订单、发送通知、生成报表等耗时操作。如果这些操作都同步执行，会严重影响用户体验。

本文将使用 **Axum** 框架构建一个 RESTful API 服务器，实现任务的提交、查询、取消等功能。

## 为什么选择 Axum

在开始编码之前，我们需要理解为什么选择 Axum 而不是其他 Web 框架：

- **类型安全**：路由参数、请求体、响应体都有强类型检查，编译时就能发现错误
- **性能出色**：基于 Tokio 和 Hyper，与 Go 的 Gin、Java 的 Netty 性能相当
- **生态兼容**：可以与 Tower 中间件生态无缝集成
- **学习曲线平缓**：API 设计直观，符合 Rust 的惯用法

## 数据模型设计

定义任务处理系统的核心数据结构：

```rust
use serde::{Deserialize, Serialize};
use uuid::Uuid;
use chrono::{DateTime, Utc};

/// 任务状态枚举
#[derive(Debug, Clone, Copy, PartialEq, Eq, Serialize, Deserialize)]
#[serde(rename_all = "lowercase")]
pub enum TaskStatus {
    /// 任务已创建，等待执行
    Pending,
    /// 任务正在执行
    Running,
    /// 任务执行成功
    Completed,
    /// 任务执行失败
    Failed,
    /// 任务被取消
    Cancelled,
}

/// 任务优先级
#[derive(Debug, Clone, Copy, PartialEq, Eq, Serialize, Deserialize)]
#[serde(rename_all = "lowercase")]
pub enum TaskPriority {
    Low,
    Normal,
    High,
    Urgent,
}

/// 任务结构
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct Task {
    /// 任务唯一标识
    pub id: Uuid,
    /// 任务类型，用于路由到不同的处理器
    pub task_type: String,
    /// 任务描述
    pub description: String,
    /// 任务状态
    pub status: TaskStatus,
    /// 任务优先级
    pub priority: TaskPriority,
    /// 任务创建时间
    pub created_at: DateTime<Utc>,
    /// 任务开始执行时间
    pub started_at: Option<DateTime<Utc>>,
    /// 任务完成时间
    pub completed_at: Option<DateTime<Utc>>,
    /// 任务执行结果
    pub result: Option<String>,
    /// 错误信息
    pub error: Option<String>,
    /// 执行进度（0-100）
    pub progress: u8,
}

/// 创建任务请求
#[derive(Debug, Deserialize)]
pub struct CreateTaskRequest {
    /// 任务类型
    pub task_type: String,
    /// 任务描述
    pub description: String,
    /// 任务优先级
    #[serde(default)]
    pub priority: TaskPriority,
}

/// 更新任务请求
#[derive(Debug, Deserialize)]
pub struct UpdateTaskRequest {
    /// 更新描述
    pub description: Option<String>,
    /// 更新优先级
    pub priority: Option<TaskPriority>,
}

/// 任务查询参数
#[derive(Debug, Deserialize)]
pub struct TaskQuery {
    /// 按状态过滤
    pub status: Option<TaskStatus>,
    /// 按类型过滤
    pub task_type: Option<String>,
    /// 按优先级过滤
    pub priority: Option<TaskPriority>,
    /// 分页：页码
    #[serde(default = "default_page")]
    pub page: usize,
    /// 分页：每页数量
    #[serde(default = "default_per_page")]
    pub per_page: usize,
}

fn default_page() -> usize { 1 }
fn default_per_page() -> usize { 20 }

/// API 响应包装
#[derive(Debug, Serialize)]
pub struct ApiResponse<T> {
    pub success: bool,
    pub data: Option<T>,
    pub error: Option<String>,
}

impl<T> ApiResponse<T> {
    pub fn ok(data: T) -> Self {
        Self {
            success: true,
            data: Some(data),
            error: None,
        }
    }

    pub fn error(message: String) -> Self {
        Self {
            success: false,
            data: None,
            error: Some(message),
        }
    }
}
```

## 任务存储层

实现一个基于内存的任务存储：

```rust
use std::collections::HashMap;
use std::sync::{Arc, Mutex};
use uuid::Uuid;

/// 任务存储接口
pub trait TaskStore: Send + Sync {
    fn create_task(&self, request: CreateTaskRequest) -> Result<Task, String>;
    fn get_task(&self, id: Uuid) -> Result<Task, String>;
    fn list_tasks(&self, query: TaskQuery) -> Result<Vec<Task>, String>;
    fn update_task(&self, id: Uuid, request: UpdateTaskRequest) -> Result<Task, String>;
    fn delete_task(&self, id: Uuid) -> Result<Task, String>;
    fn update_status(&self, id: Uuid, status: TaskStatus) -> Result<Task, String>;
    fn update_progress(&self, id: Uuid, progress: u8) -> Result<Task, String>;
}

/// 内存任务存储实现
pub struct InMemoryTaskStore {
    tasks: Arc<Mutex<HashMap<Uuid, Task>>>,
}

impl InMemoryTaskStore {
    pub fn new() -> Self {
        Self {
            tasks: Arc::new(Mutex::new(HashMap::new())),
        }
    }
}

impl Default for InMemoryTaskStore {
    fn default() -> Self {
        Self::new()
    }
}

impl TaskStore for InMemoryTaskStore {
    fn create_task(&self, request: CreateTaskRequest) -> Result<Task, String> {
        let task = Task {
            id: Uuid::new_v4(),
            task_type: request.task_type,
            description: request.description,
            status: TaskStatus::Pending,
            priority: request.priority,
            created_at: Utc::now(),
            started_at: None,
            completed_at: None,
            result: None,
            error: None,
            progress: 0,
        };

        let mut tasks = self.tasks.lock()
            .map_err(|e| format!("获取锁失败: {}", e))?;

        tasks.insert(task.id, task.clone());
        Ok(task)
    }

    fn get_task(&self, id: Uuid) -> Result<Task, String> {
        let tasks = self.tasks.lock()
            .map_err(|e| format!("获取锁失败: {}", e))?;

        tasks.get(&id)
            .cloned()
            .ok_or_else(|| format!("任务不存在: {}", id))
    }

    fn list_tasks(&self, query: TaskQuery) -> Result<Vec<Task>, String> {
        let tasks = self.tasks.lock()
            .map_err(|e| format!("获取锁失败: {}", e))?;

        let mut filtered: Vec<Task> = tasks.values()
            .filter(|task| {
                if let Some(status) = query.status {
                    if task.status != status {
                        return false;
                    }
                }
                if let Some(ref task_type) = query.task_type {
                    if &task.task_type != task_type {
                        return false;
                    }
                }
                if let Some(priority) = query.priority {
                    if task.priority != priority {
                        return false;
                    }
                }
                true
            })
            .cloned()
            .collect();

        // 按创建时间倒序排序
        filtered.sort_by(|a, b| b.created_at.cmp(&a.created_at));

        // 分页
        let start = (query.page - 1) * query.per_page;
        let end = start + query.per_page;
        let paginated = filtered.into_iter()
            .skip(start)
            .take(query.per_page)
            .collect();

        Ok(paginated)
    }

    fn update_task(&self, id: Uuid, request: UpdateTaskRequest) -> Result<Task, String> {
        let mut tasks = self.tasks.lock()
            .map_err(|e| format!("获取锁失败: {}", e))?;

        let task = tasks.get_mut(&id)
            .ok_or_else(|| format!("任务不存在: {}", id))?;

        if let Some(description) = request.description {
            task.description = description;
        }
        if let Some(priority) = request.priority {
            task.priority = priority;
        }

        Ok(task.clone())
    }

    fn delete_task(&self, id: Uuid) -> Result<Task, String> {
        let mut tasks = self.tasks.lock()
            .map_err(|e| format!("获取锁失败: {}", e))?;

        tasks.remove(&id)
            .ok_or_else(|| format!("任务不存在: {}", id))
    }

    fn update_status(&self, id: Uuid, status: TaskStatus) -> Result<Task, String> {
        let mut tasks = self.tasks.lock()
            .map_err(|e| format!("获取锁失败: {}", e))?;

        let task = tasks.get_mut(&id)
            .ok_or_else(|| format!("任务不存在: {}", id))?;

        task.status = status;

        // 更新时间戳
        match status {
            TaskStatus::Running => {
                if task.started_at.is_none() {
                    task.started_at = Some(Utc::now());
                }
            }
            TaskStatus::Completed | TaskStatus::Failed | TaskStatus::Cancelled => {
                if task.completed_at.is_none() {
                    task.completed_at = Some(Utc::now());
                }
            }
            _ => {}
        }

        Ok(task.clone())
    }

    fn update_progress(&self, id: Uuid, progress: u8) -> Result<Task, String> {
        let mut tasks = self.tasks.lock()
            .map_err(|e| format!("获取锁失败: {}", e))?;

        let task = tasks.get_mut(&id)
            .ok_or_else(|| format!("任务不存在: {}", id))?;

        task.progress = progress.min(100);

        Ok(task.clone())
    }
}
```

## API 路由和处理器

实现 RESTful API 的路由和处理器：

```rust
use axum::{
    extract::{Path, Query, State},
    http::StatusCode,
    routing::{get, post, put, delete},
    Json, Router,
};
use std::sync::Arc;

/// 应用状态
#[derive(Clone)]
pub struct AppState {
    pub store: Arc<dyn TaskStore>,
}

/// 创建任务
async fn create_task(
    State(state): State<AppState>,
    Json(request): Json<CreateTaskRequest>,
) -> Result<Json<ApiResponse<Task>>, (StatusCode, Json<ApiResponse<()>>)> {
    // 验证输入
    if request.task_type.is_empty() {
        return Err((
            StatusCode::BAD_REQUEST,
            Json(ApiResponse::error("任务类型不能为空".to_string())),
        ));
    }

    if request.description.is_empty() {
        return Err((
            StatusCode::BAD_REQUEST,
            Json(ApiResponse::error("任务描述不能为空".to_string())),
        ));
    }

    // 创建任务
    match state.store.create_task(request) {
        Ok(task) => Ok(Json(ApiResponse::ok(task))),
        Err(e) => Err((
            StatusCode::INTERNAL_SERVER_ERROR,
            Json(ApiResponse::error(e)),
        )),
    }
}

/// 获取任务详情
async fn get_task(
    State(state): State<AppState>,
    Path(id): Path<Uuid>,
) -> Result<Json<ApiResponse<Task>>, (StatusCode, Json<ApiResponse<()>>)> {
    match state.store.get_task(id) {
        Ok(task) => Ok(Json(ApiResponse::ok(task))),
        Err(e) => Err((
            StatusCode::NOT_FOUND,
            Json(ApiResponse::error(e)),
        )),
    }
}

/// 获取任务列表
async fn list_tasks(
    State(state): State<AppState>,
    Query(query): Query<TaskQuery>,
) -> Result<Json<ApiResponse<Vec<Task>>>, (StatusCode, Json<ApiResponse<()>>)> {
    match state.store.list_tasks(query) {
        Ok(tasks) => Ok(Json(ApiResponse::ok(tasks))),
        Err(e) => Err((
            StatusCode::INTERNAL_SERVER_ERROR,
            Json(ApiResponse::error(e)),
        )),
    }
}

/// 更新任务
async fn update_task(
    State(state): State<AppState>,
    Path(id): Path<Uuid>,
    Json(request): Json<UpdateTaskRequest>,
) -> Result<Json<ApiResponse<Task>>, (StatusCode, Json<ApiResponse<()>>)> {
    match state.store.update_task(id, request) {
        Ok(task) => Ok(Json(ApiResponse::ok(task))),
        Err(e) => Err((
            StatusCode::NOT_FOUND,
            Json(ApiResponse::error(e)),
        )),
    }
}

/// 删除任务
async fn delete_task(
    State(state): State<AppState>,
    Path(id): Path<Uuid>,
) -> Result<StatusCode, (StatusCode, Json<ApiResponse<()>>)> {
    match state.store.delete_task(id) {
        Ok(_) => Ok(StatusCode::NO_CONTENT),
        Err(e) => Err((
            StatusCode::NOT_FOUND,
            Json(ApiResponse::error(e)),
        )),
    }
}

/// 取消任务
async fn cancel_task(
    State(state): State<AppState>,
    Path(id): Path<Uuid>,
) -> Result<Json<ApiResponse<Task>>, (StatusCode, Json<ApiResponse<()>>)> {
    match state.store.update_status(id, TaskStatus::Cancelled) {
        Ok(task) => Ok(Json(ApiResponse::ok(task))),
        Err(e) => Err((
            StatusCode::NOT_FOUND,
            Json(ApiResponse::error(e)),
        )),
    }
}

/// 健康检查
async fn health_check() -> &'static str {
    "OK"
}
```

## 应用启动

创建应用并启动服务器：

```rust
use tracing_subscriber::{layer::SubscriberExt, util::SubscriberInitExt};

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    // 初始化日志
    tracing_subscriber::registry()
        .with(
            tracing_subscriber::EnvFilter::try_from_default_env()
                .unwrap_or_else(|_| "task_api=debug,tower_http=debug,axum=trace".into()),
        )
        .with(tracing_subscriber::fmt::layer())
        .init();

    // 创建任务存储
    let store: Arc<dyn TaskStore> = Arc::new(InMemoryTaskStore::new());

    // 构建应用状态
    let state = AppState { store };

    // 构建路由
    let app = Router::new()
        // 健康检查
        .route("/health", get(health_check))
        // 任务路由
        .route("/tasks", get(list_tasks).post(create_task))
        .route("/tasks/:id", get(get_task).put(update_task).delete(delete_task))
        .route("/tasks/:id/cancel", post(cancel_task))
        .with_state(state);

    // 启动服务器
    let listener = tokio::net::TcpListener::bind("0.0.0.0:3000").await?;
    tracing::info!("任务 API 服务器运行在 http://0.0.0.0:3000");

    axum::serve(listener, app).await?;

    Ok(())
}
```

## 添加任务执行器

为了使系统更加完整，我们添加一个简单的任务执行器：

```rust
use tokio::time::{sleep, Duration};
use tracing::{info, error};

/// 任务执行器
pub struct TaskExecutor {
    store: Arc<dyn TaskStore>,
}

impl TaskExecutor {
    pub fn new(store: Arc<dyn TaskStore>) -> Self {
        Self { store }
    }

    /// 启动执行器
    pub async fn start(&self) {
        let store = self.store.clone();

        // 定期检查待执行的任务
        tokio::spawn(async move {
            let mut interval = tokio::time::interval(Duration::from_secs(5));

            loop {
                interval.tick().await;

                // 获取所有待执行的任务
                let query = TaskQuery {
                    status: Some(TaskStatus::Pending),
                    task_type: None,
                    priority: None,
                    page: 1,
                    per_page: 10,
                };

                if let Ok(tasks) = store.list_tasks(query) {
                    for task in tasks {
                        let store = store.clone();
                        let task_id = task.id;

                        // 异步执行任务
                        tokio::spawn(async move {
                            Self::execute_task(store, task_id).await;
                        });
                    }
                }
            }
        });
    }

    /// 执行单个任务
    async fn execute_task(store: Arc<dyn TaskStore>, task_id: Uuid) {
        // 更新状态为运行中
        if let Err(e) = store.update_status(task_id, TaskStatus::Running) {
            error!("更新任务状态失败: {}", e);
            return;
        }

        // 模拟任务执行
        for progress in (0..=100).step_by(10) {
            sleep(Duration::from_millis(500)).await;

            if let Err(e) = store.update_progress(task_id, progress) {
                error!("更新任务进度失败: {}", e);
            }
        }

        // 标记任务完成
        let result = format!("任务 {} 执行成功", task_id);
        if let Err(e) = store.update_status(task_id, TaskStatus::Completed) {
            error!("更新任务状态失败: {}", e);
        } else {
            info!("任务完成: {}", result);
        }
    }
}
```

## Trade-off 分析

### 使用内存存储的优势

1. **简单快速**：无需配置数据库，适合原型开发
2. **性能优异**：内存操作极快，适合低延迟场景
3. **易于测试**：不需要外部依赖

### 使用内存存储的局限

1. **数据易失**：服务重启后数据丢失
2. **扩展性差**：无法跨进程/机器共享
3. **容量受限**：受限于单机内存

### 生产环境改进

在生产环境中，建议进行以下改进：

1. **持久化存储**：使用 Redis 或 PostgreSQL 存储任务
2. **分布式锁**：使用 Redis 实现跨实例的任务分配
3. **消息队列**：使用 RabbitMQ 或 Kafka 进行任务分发
4. **监控告警**：添加 Prometheus 指标和 Grafana 面板
5. **限流熔断**：防止 API 被过度调用

### 性能考虑

| 优化方向   | 当前实现 | 改进方案             |
| ---------- | -------- | -------------------- |
| 并发处理   | 单机内存 | 分布式任务队列       |
| 数据持久化 | 无       | Redis/PostgreSQL     |
| 错误恢复   | 无       | 任务重试机制         |
| 监控       | 日志     | Prometheus + Grafana |

## 总结

本文通过构建分布式任务处理系统的 API 服务器，综合运用了 Rust 的多项核心概念：

**技术要点**

- Axum 框架：类型安全的 Web 框架
- 异步编程：使用 `async/await` 处理并发请求
- 状态管理：使用 `Arc` 和 `Mutex` 共享任务存储
- 错误处理：使用 `Result` 类型处理错误
- 数据序列化：使用 `serde` 处理 JSON

**架构设计**

- 分层架构：存储层、业务层、API 层
- RESTful API：标准的 CRUD 操作
- 任务执行器：异步执行后台任务

**最佳实践**

- 输入验证：在 API 层验证请求数据
- 错误处理：返回合适的 HTTP 状态码
- 日志记录：使用 `tracing` 记录关键操作

这个 API 服务器为任务处理系统提供了基础的 Web 接口。在最后一篇文章中，我们将开发一个 **CLI 工具**来管理和监控任务，完成整个分布式任务处理系统的构建。
