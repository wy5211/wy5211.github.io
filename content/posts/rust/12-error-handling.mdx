## 前言

大多数语言用异常（Exception）处理错误，但 Rust 不走寻常路——它**没有异常机制**。取而代之的是类型系统中的 `Result<T, E>` 和 `Option<T>`。

这种设计让错误处理显式化：你从函数签名就能知道可能出错的地方，编译器强制你处理错误。

## Rust 错误处理的两大分类

```
┌─────────────────────────────────────────────────────────────┐
│                    Rust 错误分类                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  不可恢复错误（Unrecoverable）                               │
│  - 程序遇到 bug，无法继续运行                                │
│  - 使用 panic! 宏                                           │
│  - 例：数组越界、除以零、断言失败                            │
│  - 程序默认会打印错误信息并退出                              │
│                                                             │
│  可恢复错误（Recoverable）                                   │
│  - 预期中可能发生的错误，程序可以处理                        │
│  - 使用 Result<T, E> 枚举                                   │
│  - 例：文件不存在、网络超时、解析失败                        │
│  - 函数返回 Result，调用者决定如何处理                       │
│                                                             │
│  其他语言对比：                                              │
│  - Java: 不可恢复 → Error, 可恢复 → checked Exception       │
│  - Go: 不可恢复 → panic, 可恢复 → error value               │
│  - Rust: 不可恢复 → panic!, 可恢复 → Result<T, E>           │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## panic!：不可恢复错误

### 什么时候用 panic!

`panic!` 表示程序遇到了无法处理的错误，会立即展开栈（默认）或中止（可配置）。

```rust
fn main() {
    panic!("程序崩溃了！");
}
```

### 常见的 panic 场景

```rust
fn main() {
    let v = vec![1, 2, 3];

    // 数组越界 → panic
    // let x = v[99];

    // 使用 get 安全访问
    if let Some(x) = v.get(99) {
        println!("{}", x);
    } else {
        println!("索引越界，安全退出");
    }
}
```

### 断言宏

断言宏在条件不满足时 panic：

```rust
fn main() {
    let x = 5;

    // 条件为 false 时 panic
    assert!(x > 0, "x 必须大于 0");

    // 断言相等
    assert_eq!(x, 5);
    assert_ne!(x, 0, "x 不能为 0");

    println!("所有断言通过");
}
```

断言在测试中特别常用（后面章节会详细讲）。

## Result：可恢复错误

### Result 枚举定义

```rust
enum Result<T, E> {
    Ok(T),   // 成功，包含返回值
    Err(E),  // 失败，包含错误信息
}
```

### 基本使用

```rust
use std::fs;
use std::io;

fn read_username_from_file() -> Result<String, io::Error> {
    let content = fs::read_to_string("username.txt")?;

    Ok(content.trim().to_string())
}

fn main() {
    match read_username_from_file() {
        Ok(username) => println!("用户名: {}", username),
        Err(e) => println!("读取失败: {}", e),
    }
}
```

### unwrap 和 expect

快速但危险的方式：

```rust
fn main() {
    let content = fs::read_to_string("config.txt");

    // unwrap：成功返回值，失败 panic
    // let content = content.unwrap();  // ⚠️ 文件不存在时 panic

    // expect：失败时显示自定义信息
    let content = content.expect("无法读取配置文件，请检查 config.txt 是否存在");
    println!("{}", content);
}
```

**何时使用**：在确定不会出错的地方（如测试代码、程序启动时加载必要文件），可以用 `expect`。在可能出错的地方，应该用 `match` 或 `?`。

### ? 运算符

`?` 是 Rust 错误处理的语法糖，让错误传播变得极其简洁：

```rust
use std::fs;
use std::io;

// 不使用 ?：手动 match 传播错误
fn read_config_v1() -> Result<String, io::Error> {
    let content = match fs::read_to_string("config.txt") {
        Ok(c) => c,
        Err(e) => return Err(e),
    };
    Ok(content)
}

// 使用 ?：自动传播错误
fn read_config_v2() -> Result<String, io::Error> {
    let content = fs::read_to_string("config.txt")?;
    Ok(content)
}

// ? 可以链式使用
fn read_and_process() -> Result<String, io::Error> {
    let content = fs::read_to_string("config.txt")?;
    let trimmed = content.trim();
    let upper = trimmed.to_uppercase();
    Ok(upper)
}

fn main() {
    match read_and_process() {
        Ok(result) => println!("结果: {}", result),
        Err(e) => eprintln!("错误: {}", e),
    }
}
```

### ? 运算符的工作原理

`?` 运算符做了以下事情：

1. 如果是 `Ok(value)`，返回 `value`
2. 如果是 `Err(e)`，使用 `From` trait 转换错误类型，然后 `return Err(e)`

```
┌─────────────────────────────────────────────────────────────┐
│                    ? 运算符流程                               │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  let result = some_operation()?;                            │
│                                                             │
│  等价于：                                                    │
│                                                             │
│  let result = match some_operation() {                      │
│      Ok(value) => value,                                    │
│      Err(e) => return Err(From::from(e)),                  │
│  };                                                         │
│                                                             │
│  关键特性：                                                  │
│  1. 自动解包 Ok                                             │
│  2. 自动 return Err                                         │
│  3. 自动转换错误类型（通过 From trait）                      │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 自定义错误类型

### 定义错误枚举

```rust
use std::fmt;

// 自定义错误类型
#[derive(Debug)]
enum AppError {
    IoError(std::io::Error),
    ParseError(String),
    NotFound(String),
}

// 实现 Display trait（错误消息）
impl fmt::Display for AppError {
    fn fmt(&self, f: &mut fmt::Formatter) -> fmt::Result {
        match self {
            AppError::IoError(e) => write!(f, "IO 错误: {}", e),
            AppError::ParseError(msg) => write!(f, "解析错误: {}", msg),
            AppError::NotFound(msg) => write!(f, "未找到: {}", msg),
        }
    }
}

// 实现 std::error::Error trait
impl std::error::Error for AppError {
    fn source(&self) -> Option<&(dyn std::error::Error + 'static)> {
        match self {
            AppError::IoError(e) => Some(e),
            _ => None,
        }
    }
}

// 从 io::Error 转换
impl From<std::io::Error> for AppError {
    fn from(e: std::io::Error) -> Self {
        AppError::IoError(e)
    }
}
```

### 使用自定义错误

```rust
fn read_config(path: &str) -> Result<String, AppError> {
    let content = std::fs::read_to_string(path)?;  // 自动转换为 AppError

    if content.trim().is_empty() {
        return Err(AppError::ParseError("配置文件为空".to_string()));
    }

    Ok(content)
}

fn main() {
    match read_config("config.txt") {
        Ok(config) => println!("配置: {}", config),
        Err(e) => {
            eprintln!("错误: {}", e);
            if let Some(source) = e.source() {
                eprintln!("原因: {}", source);
            }
        }
    }
}
```

### thiserror 库

手动实现错误类型比较繁琐，实际项目中推荐使用 `thiserror` 库：

```toml
# Cargo.toml
[dependencies]
thiserror = "1.0"
```

```rust
use thiserror::Error;

#[derive(Error, Debug)]
enum AppError {
    #[error("IO 错误: {0}")]
    IoError(#[from] std::io::Error),

    #[error("解析错误: {0}")]
    ParseError(String),

    #[error("未找到: {0}")]
    NotFound(String),

    #[error("无效参数: {name} 不能为 {value}")]
    InvalidParam { name: String, value: i32 },
}

fn main() {
    let err = AppError::InvalidParam {
        name: "age".to_string(),
        value: -1,
    };
    println!("{}", err);  // 无效参数: age 不能为 -1
}
```

`thiserror` 自动实现了 `Display`、`Error` 和 `From`，大幅减少样板代码。

## Option 与错误处理

`Option<T>` 也是错误处理的一部分，用于表示"值可能不存在"：

```rust
fn find_user(users: &[(&str, u32)], name: &str) -> Option<u32> {
    users.iter()
        .find(|&&(n, _)| n == name)
        .map(|&(_, age)| age)
}

fn main() {
    let users = [("Alice", 30), ("Bob", 25)];

    match find_user(&users, "Alice") {
        Some(age) => println!("Alice 的年龄: {}", age),
        None => println!("未找到用户"),
    }

    // ? 也可以用于 Option（在返回 Option 的函数中）
}
```

## 实战演练

### 示例 1：安全的文件读取与处理

```rust
use std::fs;
use std::io;
use std::path::Path;

fn count_words_in_file(path: &Path) -> Result<usize, io::Error> {
    let content = fs::read_to_string(path)?;
    let count = content.split_whitespace().count();
    Ok(count)
}

fn process_files(paths: &[&str]) -> Result<Vec<(String, usize)>, io::Error> {
    let mut results = Vec::new();

    for path_str in paths {
        let path = Path::new(path_str);
        let count = count_words_in_file(path)?;
        results.push((path_str.to_string(), count));
    }

    Ok(results)
}

fn main() {
    let files = ["Cargo.toml", "README.md"];

    match process_files(&files) {
        Ok(results) => {
            for (file, count) in results {
                println!("{}: {} 个单词", file, count);
            }
        }
        Err(e) => eprintln!("处理失败: {}", e),
    }
}
```

### 示例 2：配置解析器

```rust
use std::collections::HashMap;
use std::io;

#[derive(Debug)]
struct Config {
    database_url: String,
    port: u16,
    debug: bool,
}

fn parse_config(content: &str) -> Result<Config, String> {
    let mut map = HashMap::new();

    for line in content.lines() {
        let line = line.trim();
        if line.is_empty() || line.starts_with('#') {
            continue;
        }

        let parts: Vec<&str> = line.splitn(2, '=').collect();
        if parts.len() != 2 {
            return Err(format!("无效的配置行: {}", line));
        }

        map.insert(parts[0].trim(), parts[1].trim());
    }

    let database_url = map.get("database_url")
        .ok_or("缺少 database_url 配置")?
        .to_string();

    let port: u16 = map.get("port")
        .ok_or("缺少 port 配置")?
        .parse()
        .map_err(|e| format!("port 解析失败: {}", e))?;

    let debug = map.get("debug")
        .map(|v| v == "true")
        .unwrap_or(false);

    Ok(Config { database_url, port, debug })
}

fn main() {
    let config_str = r#"
        # 数据库配置
        database_url=postgres://localhost/mydb
        port=5432
        debug=true
    "#;

    match parse_config(config_str) {
        Ok(config) => println!("{:?}", config),
        Err(e) => eprintln!("配置错误: {}", e),
    }
}
```

### 示例 3：使用 thiserror 的完整示例

```rust
use std::fs;
use std::path::Path;
use thiserror::Error;

#[derive(Error, Debug)]
enum DataError {
    #[error("文件读取失败: {0}")]
    Io(#[from] std::io::Error),

    #[error("格式无效: {0}")]
    InvalidFormat(String),

    #[error("数据缺失: 缺少 '{field}' 字段")]
    MissingField { field: String },
}

fn parse_csv_line(line: &str) -> Result<Vec<String>, DataError> {
    let fields: Vec<String> = line.split(',').map(String::from).collect();

    if fields.len() < 3 {
        return Err(DataError::InvalidFormat(
            format!("每行至少需要 3 个字段，实际有 {} 个", fields.len())
        ));
    }

    Ok(fields)
}

fn load_data(path: &Path) -> Result<Vec<Vec<String>>, DataError> {
    let content = fs::read_to_string(path)?;
    let mut data = Vec::new();

    for (i, line) in content.lines().enumerate() {
        if line.trim().is_empty() {
            continue;
        }
        let fields = parse_csv_line(line)
            .map_err(|e| DataError::InvalidFormat(
                format!("第 {} 行: {}", i + 1, e)
            ))?;
        data.push(fields);
    }

    if data.is_empty() {
        return Err(DataError::MissingField { field: "data".to_string() });
    }

    Ok(data)
}

fn main() {
    match load_data(Path::new("data.csv")) {
        Ok(data) => {
            println!("加载了 {} 行数据", data.len());
            for row in &data {
                println!("{:?}", row);
            }
        }
        Err(e) => eprintln!("错误: {}", e),
    }
}
```

## 常见错误与修复

### 错误 1：用 unwrap 忽略错误

```rust
fn main() {
    // ❌ 生产代码中不要这样做
    // let content = fs::read_to_string("file.txt").unwrap();

    // ✅ 正确处理错误
    match fs::read_to_string("file.txt") {
        Ok(content) => println!("{}", content),
        Err(e) => eprintln!("读取失败: {}", e),
    }
}
```

### 错误 2：? 运算符的返回类型不匹配

```rust
use std::fs;

// ❌ 返回类型不包含 io::Error
// fn read_file() -> String {
//     let content = fs::read_to_string("file.txt")?;
//     content
// }

// ✅ 返回 Result
fn read_file() -> Result<String, std::io::Error> {
    let content = fs::read_to_string("file.txt")?;
    Ok(content)
}
```

## 最佳实践

1. **库代码返回 Result**：让调用者决定如何处理错误
2. **应用入口处理所有错误**：`main` 函数中集中处理
3. **使用 `?` 传播错误**：比手动 match 更简洁
4. **用 `thiserror` 定义错误类型**：减少样板代码
5. **`expect` 用于不可能失败的断言**：提供有意义的错误信息

## 常见问题

**Q: Rust 没有 try-catch 吗？**

A: 没有。Rust 用 `Result` 和 `?` 运算符替代了异常机制。这种方式更安全，因为编译器强制你处理错误。

**Q: 什么时候用 panic!?**

A: 只在程序遇到 bug（不应该发生的情况）时使用。对于预期可能发生的错误，用 `Result`。

**Q: ? 可以用在 main 函数里吗？**

A: Rust 1.56+ 支持 `main` 返回 `Result`：

```rust
fn main() -> Result<(), Box<dyn std::error::Error>> {
    let content = fs::read_to_string("config.txt")?;
    println!("{}", content);
    Ok(())
}
```

## 总结

本文学习了 Rust 的错误处理机制：

**两种错误**

- `panic!`：不可恢复，程序终止
- `Result<T, E>`：可恢复，返回错误给调用者

**Result 处理方式**

- `match`：完整处理所有情况
- `unwrap()`/`expect()`：快速但危险
- `?` 运算符：简洁的错误传播

**自定义错误**

- 手动实现 `Display` + `Error` trait
- 使用 `thiserror` 库简化

**核心思想**：错误是值，通过类型系统处理，而不是通过控制流。
