## 前言

并发编程是出了名的困难——数据竞争、死锁、竞态条件无处不在。大多数语言让开发者自己小心处理这些问题，但 Rust 选择了一条不同的路：**在编译期防止数据竞争**。

Rust 的所有权和类型系统在并发编程中发挥了巨大作用——如果代码能编译通过，就不会有数据竞争。这被称为**无畏并发（Fearless Concurrency）**。

## 创建线程

### 基本线程

```rust
use std::thread;
use std::time::Duration;

fn main() {
    // 创建线程
    let handle = thread::spawn(|| {
        for i in 1..=5 {
            println!("子线程: {}", i);
            thread::sleep(Duration::from_millis(10));
        }
    });

    // 主线程同时运行
    for i in 1..=3 {
        println!("主线程: {}", i);
        thread::sleep(Duration::from_millis(10));
    }

    // 等待子线程结束
    handle.join().unwrap();
}
```

### move 闭包

线程默认使用闭包捕获环境变量，但需要 `move` 来转移所有权：

```rust
use std::thread;

fn main() {
    let v = vec![1, 2, 3];

    // ❌ 编译错误：v 可能在线程使用前被释放
    // let handle = thread::spawn(|| {
    //     println!("{:?}", v);
    // });

    // ✅ 使用 move 转移所有权
    let handle = thread::spawn(move || {
        println!("子线程: {:?}", v);
    });

    handle.join().unwrap();
    // println!("{:?}", v);  // ❌ v 已被移动
}
```

## 消息传递

Rust 的并发哲学是：**不要通过共享内存来通信，而要通过通信来共享内存**。

### 基本 channel

```rust
use std::sync::mpsc;
use std::thread;

fn main() {
    // 创建 channel（返回发送者和接收者）
    let (tx, rx) = mpsc::channel();

    thread::spawn(move || {
        let val = String::from("hello");
        tx.send(val).unwrap();
        // println!("{}", val);  // ❌ val 已被移动到 channel
    });

    // 接收消息（阻塞）
    let received = rx.recv().unwrap();
    println!("收到: {}", received);
}
```

### 发送多个值

```rust
use std::sync::mpsc;
use std::thread;
use std::time::Duration;

fn main() {
    let (tx, rx) = mpsc::channel();

    thread::spawn(move || {
        let vals = vec![
            String::from("hi"),
            String::from("from"),
            String::from("the"),
            String::from("thread"),
        ];

        for val in vals {
            tx.send(val).unwrap();
            thread::sleep(Duration::from_millis(100));
        }
    });

    // rx 是迭代器，自动接收所有消息
    for received in rx {
        println!("收到: {}", received);
    }
}
```

### 多发送者

使用 `mpsc::channel()` 创建的发送者可以通过 `clone` 创建多个：

```rust
use std::sync::mpsc;
use std::thread;
use std::time::Duration;

fn main() {
    let (tx, rx) = mpsc::channel();

    let tx1 = tx.clone();
    let tx2 = tx.clone();

    thread::spawn(move || {
        let vals = vec!["from t1: a", "from t1: b"];
        for val in vals {
            tx1.send(val).unwrap();
            thread::sleep(Duration::from_millis(100));
        }
    });

    thread::spawn(move || {
        let vals = vec!["from t2: x", "from t2: y"];
        for val in vals {
            tx2.send(val).unwrap();
            thread::sleep(Duration::from_millis(100));
        }
    });

    for received in rx {
        println!("收到: {}", received);
    }
}
```

### try_recv 非阻塞接收

```rust
use std::sync::mpsc;
use std::thread;

fn main() {
    let (tx, rx) = mpsc::channel();

    thread::spawn(move || {
        tx.send("hello").unwrap();
    });

    // try_recv 不阻塞，立即返回
    loop {
        match rx.try_recv() {
            Ok(msg) => {
                println!("收到: {}", msg);
                break;
            }
            Err(mpsc::TryRecvError::Empty) => {
                println!("暂无消息...");
                std::thread::sleep(std::time::Duration::from_millis(100));
            }
            Err(mpsc::TryRecvError::Disconnected) => {
                println!("发送者已断开");
                break;
            }
        }
    }
}
```

## Mutex：互斥器

`Mutex<T>` 提供了**互斥访问**——同一时刻只允许一个线程访问数据：

```rust
use std::sync::{Arc, Mutex};
use std::thread;

fn main() {
    let counter = Arc::new(Mutex::new(0));
    let mut handles = vec![];

    for _ in 0..10 {
        let counter = Arc::clone(&counter);
        let handle = thread::spawn(move || {
            let mut num = counter.lock().unwrap();
            *num += 1;
            // lock 在这里自动释放
        });
        handles.push(handle);
    }

    for handle in handles {
        handle.join().unwrap();
    }

    println!("结果: {}", *counter.lock().unwrap());  // 10
}
```

### Mutex 的工作原理

```
┌─────────────────────────────────────────────────────────────┐
│                    Mutex 工作流程                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  let m = Mutex::new(5);                                    │
│                                                             │
│  线程 A:  let num = m.lock();  // 获取锁                    │
│           *num += 1;            // 修改数据                  │
│           // lock 自动释放                                 │
│                                                             │
│  线程 B:  let num = m.lock();  // 等待锁...                 │
│           // 线程 A 释放后获取锁                             │
│           *num += 1;            // 修改数据                  │
│           // lock 自动释放                                 │
│                                                             │
│  注意：lock() 返回 MutexGuard，实现了 Drop trait             │
│  离开作用域时自动释放锁                                     │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 注意死锁

```rust
use std::sync::{Arc, Mutex};
use std::thread;

fn main() {
    let data1 = Arc::new(Mutex::new(0));
    let data2 = Arc::new(Mutex::new(0));

    let d1 = Arc::clone(&data1);
    let d2_a = Arc::clone(&data2);
    let d2_b = Arc::clone(&data2);
    let d1_b = Arc::clone(&data1);

    // ❌ 可能死锁：线程 1 持有 data1 等待 data2，
    //    线程 2 持有 data2 等待 data1
    // 解决方案：总是以相同顺序获取锁
    let h1 = thread::spawn(move || {
        let _a = d1.lock().unwrap();
        let _b = d2_a.lock().unwrap();
    });

    let h2 = thread::spawn(move || {
        let _a = d2_b.lock().unwrap();
        let _b = d1_b.lock().unwrap();
    });

    h1.join().unwrap();
    h2.join().unwrap();
}
```

## Send 和 Sync trait

Rust 通过两个 marker trait 实现并发安全的编译时检查：

```
┌─────────────────────────────────────────────────────────────┐
│                    Send 和 Sync                               │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  Send：可以在线程间转移所有权                                │
│  - 几乎所有类型都自动实现 Send                               │
│  - Rc<T> 不是 Send（非线程安全）                            │
│  - Arc<T> 是 Send                                           │
│                                                             │
│  Sync：可以在线程间共享引用（通过 &T）                       │
│  - 如果 T 是 Sync，则 &T 是 Send                            │
│  - RefCell<T> 不是 Sync                                     │
│  - Mutex<T> 是 Sync                                         │
│                                                             │
│  手动实现 Send/Sync 需要 unsafe                              │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 实战演练

### 示例 1：多线程下载器

```rust
use std::sync::{Arc, Mutex};
use std::thread;
use std::time::Duration;

fn main() {
    let urls = vec![
        "https://example.com/file1",
        "https://example.com/file2",
        "https://example.com/file3",
    ];

    let progress = Arc::new(Mutex::new(0));
    let total = urls.len();
    let mut handles = vec![];

    for (i, url) in urls.iter().enumerate() {
        let progress = Arc::clone(&progress);
        let url = url.to_string();

        let handle = thread::spawn(move || {
            println!("[下载 {}] 开始: {}", i, url);

            // 模拟下载
            thread::sleep(Duration::from_millis(500 + (i as u64 * 200)));

            // 更新进度
            let mut p = progress.lock().unwrap();
            *p += 1;
            println!("[下载 {}] 完成 ({}/{})", i, *p, total);
        });

        handles.push(handle);
    }

    for handle in handles {
        handle.join().unwrap();
    }

    println!("全部下载完成！");
}
```

### 示例 2：生产者-消费者模型

```rust
use std::sync::mpsc;
use std::thread;
use std::time::Duration;

fn main() {
    let (tx, rx) = mpsc::channel();
    let rx = Arc::new(Mutex::new(rx));

    // 消费者
    let rx2 = Arc::clone(&rx);
    let consumer = thread::spawn(move || {
        loop {
            let rx = rx2.lock().unwrap();
            match rx.try_recv() {
                Ok(item) => println!("消费: {}", item),
                Err(mpsc::TryRecvError::Empty) => {
                    drop(rx);
                    thread::sleep(Duration::from_millis(50));
                    continue;
                }
                Err(mpsc::TryRecvError::Disconnected) => {
                    println!("生产者结束，消费者退出");
                    break;
                }
            }
        }
    });

    // 生产者
    let producer = thread::spawn(move || {
        for i in 1..=5 {
            tx.send(format!("任务 {}", i)).unwrap();
            println!("生产: 任务 {}", i);
            thread::sleep(Duration::from_millis(100));
        }
        println!("生产者结束");
    });

    producer.join().unwrap();
    consumer.join().unwrap();
}
```

### 示例 3：线程池

```rust
use std::sync::{mpsc, Arc, Mutex};
use std::thread;

type Job = Box<dyn FnOnce() + Send + 'static>;

struct ThreadPool {
    workers: Vec<Worker>,
    sender: mpsc::Sender<Job>,
}

impl ThreadPool {
    fn new(size: usize) -> Self {
        let (sender, receiver) = mpsc::channel();
        let receiver = Arc::new(Mutex::new(receiver));

        let mut workers = Vec::with_capacity(size);

        for id in 0..size {
            workers.push(Worker::new(id, Arc::clone(&receiver)));
        }

        Self { workers, sender }
    }

    fn execute<F>(&self, f: F)
    where
        F: FnOnce() + Send + 'static,
    {
        let job = Box::new(f);
        self.sender.send(job).unwrap();
    }
}

struct Worker {
    id: usize,
    thread: Option<thread::JoinHandle<()>>,
}

impl Worker {
    fn new(id: usize, receiver: Arc<Mutex<mpsc::Receiver<Job>>>) -> Self {
        let thread = thread::spawn(move || loop {
            let job = receiver.lock().unwrap().recv();

            match job {
                Ok(job) => {
                    println!("Worker {} 执行任务", id);
                    job();
                }
                Err(_) => break,
            }
        });

        Self { id, thread: Some(thread) }
    }
}

fn main() {
    let pool = ThreadPool::new(3);

    for i in 0..6 {
        pool.execute(move || {
            println!("  处理任务 {}", i);
            thread::sleep(std::time::Duration::from_millis(200));
        });
    }

    thread::sleep(std::time::Duration::from_secs(2));
    println!("所有任务完成");
}
```

## 常见错误与修复

### 错误 1：Send trait 不满足

```rust
use std::rc::Rc;
use std::thread;

fn main() {
    let data = Rc::new(5);

    // ❌ Rc 不是 Send
    // thread::spawn(move || { println!("{}", data); });

    // ✅ 使用 Arc
    use std::sync::Arc;
    let data = Arc::new(5);
    let data2 = Arc::clone(&data);
    thread::spawn(move || { println!("{}", data2); });
}
```

### 错误 2：忘记 join

```rust
use std::thread;

fn main() {
    // ❌ 不 join 的话，主线程可能先结束
    thread::spawn(|| {
        println!("子线程");
    });
    // 主线程可能在这里就退出了，子线程来不及执行

    // ✅ 等待子线程
    let handle = thread::spawn(|| {
        println!("子线程");
    });
    handle.join().unwrap();
}
```

## 最佳实践

1. **优先使用 channel**：消息传递比共享内存更安全
2. **需要共享时用 Arc + Mutex**：标准组合
3. **避免长时间持锁**：获取锁后尽快释放
4. **统一锁的获取顺序**：避免死锁
5. **使用 `thread::scope`**（Rust 1.63+）：更安全的 scoped 线程

## 常见问题

**Q: Rust 真的能完全防止数据竞争吗？**

A: 是的。通过 `Send` 和 `Sync` trait，编译器确保数据在线程间安全传递。`unsafe` 代码除外。

**Q: 什么时候用 channel，什么时候用 Mutex？**

A: 简单原则：如果多个线程需要独立处理数据，用 channel；如果需要共享修改同一份数据，用 Mutex。

**Q: thread::scope 是什么？**

A: 它创建的线程可以借用外部变量（不需要 `move`），并且保证所有线程在 scope 结束前完成。

## 总结

本文学习了 Rust 的并发编程：

**线程**

- `thread::spawn` 创建线程
- `move` 闭包转移所有权
- `join()` 等待线程结束

**消息传递**

- `mpsc::channel` 创建 channel
- 多发送者（clone tx）
- `recv()` 阻塞接收，`try_recv()` 非阻塞

**共享状态**

- `Mutex<T>` 互斥访问
- `Arc<T>` 原子引用计数
- `Arc<Mutex<T>>` 标准组合

**编译时保证**

- `Send`：可以跨线程转移
- `Sync`：可以跨线程共享引用
- 编译器阻止不安全的并发操作
