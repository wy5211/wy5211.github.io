## 前言

测试是保证代码质量的重要手段。Rust 内置了测试框架，不需要额外安装任何依赖。本章将学习单元测试、集成测试和文档测试的编写方法。

## 单元测试

### 基本测试

在 Rust 中，测试函数用 `#[test]` 标注，通常放在 `#[cfg(test)]` 模块中：

```rust
// src/lib.rs
pub fn add(a: i32, b: i32) -> i32 {
    a + b
}

pub fn multiply(a: i32, b: i32) -> i32 {
    a * b
}

#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn test_add() {
        assert_eq!(add(2, 3), 5);
    }

    #[test]
    fn test_multiply() {
        assert_eq!(multiply(3, 4), 12);
    }
}
```

运行测试：`cargo test`

### 断言宏

```rust
#[cfg(test)]
mod tests {
    // assert_eq!：判断相等
    #[test]
    fn test_equal() {
        assert_eq!(2 + 2, 4);
    }

    // assert_ne!：判断不等
    #[test]
    fn test_not_equal() {
        assert_ne!(2 + 2, 5);
    }

    // assert!：判断条件为 true
    #[test]
    fn test_condition() {
        let result = 10;
        assert!(result > 5);
    }

    // 自定义错误消息
    #[test]
    fn test_with_message() {
        let value = 3;
        assert!(value < 10, "值 {} 应该小于 10", value);
    }
}
```

### 测试 panic

使用 `#[should_panic]` 标注预期会 panic 的测试：

```rust
pub fn divide(a: f64, b: f64) -> f64 {
    if b == 0.0 {
        panic!("除数不能为零");
    }
    a / b
}

#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    #[should_panic(expected = "除数不能为零")]
    fn test_divide_by_zero() {
        divide(10.0, 0.0);
    }
}
```

### 使用 Result 的测试

```rust
#[cfg(test)]
mod tests {
    #[test]
    fn test_parse() -> Result<(), String> {
        let result: i32 = "42".parse().map_err(|_| "解析失败")?;
        assert_eq!(result, 42);
        Ok(())
    }
}
```

### 测试辅助函数

```rust
#[cfg(test)]
mod tests {
    // 辅助函数不需要 #[test]
    fn setup_test_data() -> Vec<i32> {
        vec![1, 2, 3, 4, 5]
    }

    #[test]
    fn test_sum() {
        let data = setup_test_data();
        let sum: i32 = data.iter().sum();
        assert_eq!(sum, 15);
    }
}
```

## 控制测试执行

### 过滤测试

```bash
# 只运行名字包含 "add" 的测试
cargo test add

# 运行某个模块的所有测试
cargo test tests::test_add
```

### 忽略测试

```rust
#[test]
#[ignore]  // 跳过
fn expensive_test() {
    // 耗时很长的测试
}

// 运行被忽略的测试
// cargo test -- --ignored
```

### 显示输出

```bash
# 默认不显示 println! 输出
# cargo test -- --nocapture
```

## 集成测试

集成测试放在 `tests/` 目录下，测试整个库的公开 API：

```
my_project/
├── Cargo.toml
├── src/
│   └── lib.rs
└── tests/
    └── integration_test.rs
```

`tests/integration_test.rs`:

```rust
use my_project::add;
use my_project::multiply;

#[test]
fn test_integration() {
    assert_eq!(add(10, 20), 30);
    assert_eq!(multiply(5, 6), 30);
}

#[test]
fn test_combined() {
    let a = add(2, 3);
    let b = multiply(a, 4);
    assert_eq!(b, 20);
}
```

### 集成测试的子模块

```
tests/
├── common/
│   └── mod.rs      # 共享辅助代码
└── integration_test.rs
```

`tests/common/mod.rs`:

```rust
pub fn create_test_data() -> Vec<String> {
    vec!["hello".to_string(), "world".to_string()]
}
```

`tests/integration_test.rs`:

```rust
mod common;

#[test]
fn test_with_common() {
    let data = common::create_test_data();
    assert_eq!(data.len(), 2);
}
```

## 文档测试

Rust 支持在文档注释中编写测试：

````rust
/// 将两个数相加。
///
/// # Examples
///
/// ```
/// use my_project::add;
///
/// let result = add(2, 3);
/// assert_eq!(result, 5);
/// ```
pub fn add(a: i32, b: i32) -> i32 {
    a + b
}
````

运行文档测试：`cargo test --doc`

## 实战演练

### 示例：为计算器编写完整测试

```rust
pub struct Calculator {
    value: f64,
}

impl Calculator {
    pub fn new() -> Self {
        Self { value: 0.0 }
    }

    pub fn add(&mut self, n: f64) -> &mut Self {
        self.value += n;
        self
    }

    pub fn subtract(&mut self, n: f64) -> &mut Self {
        self.value -= n;
        self
    }

    pub fn multiply(&mut self, n: f64) -> &mut Self {
        self.value *= n;
        self
    }

    pub fn divide(&mut self, n: f64) -> Result<&mut Self, String> {
        if n == 0.0 {
            return Err("除数不能为零".to_string());
        }
        self.value /= n;
        Ok(self)
    }

    pub fn result(&self) -> f64 {
        self.value
    }

    pub fn clear(&mut self) -> &mut Self {
        self.value = 0.0;
        self
    }
}

#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn test_new() {
        let calc = Calculator::new();
        assert_eq!(calc.result(), 0.0);
    }

    #[test]
    fn test_add() {
        let mut calc = Calculator::new();
        calc.add(5.0);
        assert_eq!(calc.result(), 5.0);
    }

    #[test]
    fn test_chain() {
        let mut calc = Calculator::new();
        let result = calc.add(10.0).subtract(3.0).multiply(2.0).result();
        assert_eq!(result, 14.0);
    }

    #[test]
    fn test_divide() {
        let mut calc = Calculator::new();
        calc.add(10.0);
        assert!(calc.divide(2.0).is_ok());
        assert_eq!(calc.result(), 5.0);
    }

    #[test]
    #[should_panic(expected = "除数不能为零")]
    fn test_divide_by_zero() {
        let mut calc = Calculator::new();
        calc.divide(0.0).unwrap();
    }

    #[test]
    fn test_clear() {
        let mut calc = Calculator::new();
        calc.add(100.0).clear();
        assert_eq!(calc.result(), 0.0);
    }
}
```

## 最佳实践

1. **测试命名清晰**：`test_` 前缀 + 描述行为
2. **测试公开 API**：集成测试只测试公开接口
3. **测试边界条件**：空输入、零、最大值等
4. **使用 `Result` 测试**：避免 `unwrap` 在测试中 panic
5. **运行 `cargo test` 作为 CI 的一部分**

## 总结

本文学习了 Rust 的测试体系：

- **单元测试**：`#[test]` + `#[cfg(test)]` 模块
- **断言宏**：`assert_eq!`、`assert_ne!`、`assert!`
- **预期 panic**：`#[should_panic]`
- **集成测试**：`tests/` 目录
- **文档测试**：文档注释中的代码块
