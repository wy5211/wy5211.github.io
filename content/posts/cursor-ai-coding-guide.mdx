# AI 辅助编程实战：Cursor 高效开发指南

Cursor 是目前最强大的 AI 代码编辑器，基于 VS Code 深度定制，集成了 GPT-4 和 Claude 等大模型。用了 Cursor 之后，我再也回不去纯手动写代码了。本文分享我的实战经验，帮你快速上手。

## 一、为什么选择 Cursor？

| 特性 | Cursor | GitHub Copilot | Codeium |
|------|--------|----------------|---------|
| AI 补全 | ✅ | ✅ | ✅ |
| AI Chat | ✅ 内置 | ❌ 需另装 | ❌ 需另装 |
| 多文件编辑 | ✅ Composer | ❌ | ❌ |
| 代码库理解 | ✅ Codebase | ❌ | ❌ |
| 免费额度 | 有 | 付费 | 有 |

**Cursor 的杀手锏**：它能理解整个代码库，在多文件间进行协调修改。这是其他工具做不到的。

## 二、核心功能详解

### 2.1 AI 代码补全（Tab）

Cursor 的补全比 Copilot 更智能，因为它能看到你的整个项目。

```typescript
// 只需要写注释，按 Tab 自动补全
// 获取用户信息，包含头像和最近订单
async function getUserWithOrders(userId: string) {
  // Cursor 会根据项目中的 User、Order 类型自动生成正确代码
}
```

**快捷键**：
- `Tab` - 接受补全
- `Esc` - 拒绝补全
- `Ctrl/Cmd →` - 接受部分补全（逐词）

### 2.2 AI Chat（Ctrl/Cmd + L）

内置的 AI 聊天窗口，可以：
- 询问代码问题
- 让 AI 解释代码
- 生成代码片段
- 添加代码上下文

**关键技巧**：

```
# 添加上下文
@Files        添加文件
@Codebase     搜索代码库
@Web          联网搜索
@Docs         引用文档
@Definitions  添加类型定义
```

**示例对话**：

```
你：@Codebase 这个项目里的用户认证是怎么实现的？

AI：根据你的代码，项目使用了 JWT 认证：
1. login.ts 中生成 token
2. middleware/auth.ts 中验证 token
3. 前端存储在 localStorage
...
```

### 2.3 Composer（Ctrl/Cmd + I）⭐ 最强功能

Composer 可以同时编辑多个文件，是 Cursor 的核心杀手锏。

**使用场景**：
- 创建新功能（自动创建多个文件）
- 重构（跨文件修改）
- 添加新 API（同时修改前后端）

**示例**：

```
你：帮我创建一个用户管理模块，包含：
- 列表页面（带搜索、分页）
- 新增/编辑弹窗
- 删除确认
- API 接口

参考项目现有的 Product 模块风格
```

Composer 会：
1. 分析现有 Product 模块的代码风格
2. 创建/修改多个文件
3. 保持代码风格一致

### 2.4 内联编辑（Ctrl/Cmd + K）

选中代码后按 `Ctrl/Cmd + K`，可以进行：
- 重构
- 添加注释
- 修复 Bug
- 优化性能

**示例**：

```typescript
// 选中这段代码，Ctrl+K
function fetchData(url) {
  return fetch(url).then(res => res.json())
}

// 输入：添加错误处理和重试逻辑
// Cursor 生成：
async function fetchData(url, retries = 3) {
  for (let i = 0; i < retries; i++) {
    try {
      const res = await fetch(url)
      if (!res.ok) throw new Error(`HTTP ${res.status}`)
      return await res.json()
    } catch (error) {
      if (i === retries - 1) throw error
      await new Promise(r => setTimeout(r, 1000 * (i + 1)))
    }
  }
}
```

## 三、实战技巧

### 3.1 写好 Prompt 的技巧

**坏 Prompt**：
```
帮我写个登录功能
```

**好 Prompt**：
```
帮我实现登录功能，要求：
1. 使用 React Hook Form 处理表单
2. 使用 Zod 做表单验证（邮箱、密码必填）
3. 调用 /api/auth/login 接口
4. 登录成功后跳转到 /dashboard
5. 参考项目中的 RegisterForm 组件风格
```

**技巧总结**：
- 明确技术栈和库
- 指定验证规则
- 参考现有代码风格
- 分步骤描述需求

### 3.2 让 AI 理解你的代码库

**方法一：使用 @Codebase**

```
@Codebase 这个项目使用什么状态管理方案？
```

**方法二：添加相关文件**

```
@Files src/store/*.ts 帮我添加一个新的用户状态
```

**方法三：在 Composer 中描述上下文**

```
参考 src/features/product 模块的结构，创建一个 order 模块
```

### 3.3 Debug 技巧

**方法一：直接问 AI**

```
这段代码报错 "Cannot read property 'map' of undefined"，怎么修复？
```

**方法二：使用 Chat 分析错误**

```
@终端输出 帮我分析这个错误，给出修复方案
```

**方法三：让 AI 添加调试日志**

```
选中代码 → Ctrl+K → 添加详细的调试日志
```

### 3.4 重构技巧

**提取公共逻辑**：

```
@Files src/utils/*.ts 这几个文件中有重复的日期格式化逻辑，
帮我提取成一个公共函数
```

**类型优化**：

```
选中代码 → Ctrl+K → 为这个函数添加完整的 TypeScript 类型，
使用项目中的现有类型定义
```

## 四、高效工作流

### 4.1 新功能开发流程

```
1. Ctrl+L 打开 Chat，描述需求，让 AI 设计方案
2. Ctrl+I 打开 Composer，让 AI 创建文件结构
3. 逐文件细化，使用 Ctrl+K 优化代码
4. Ctrl+L 让 AI 审查代码，提出改进建议
```

### 4.2 Bug 修复流程

```
1. 复制错误信息到 Chat
2. 使用 @Codebase 搜索相关代码
3. 让 AI 分析原因并给出修复方案
4. 使用 Ctrl+K 应用修复
5. 让 AI 写单元测试验证修复
```

### 4.3 代码审查流程

```
1. 选中修改的代码
2. Ctrl+L → "审查这段代码，指出潜在问题"
3. 根据建议逐项修复
4. 让 AI 检查是否遗漏边界情况
```

## 六、最佳实践

### 6.1 分块开发

不要一次性让 AI 写完整个功能，而是分步骤：

```
❌ 帮我写一个完整的电商购物车

✅ 分步骤：
   1. 先写数据类型定义
   2. 再写核心逻辑（添加、删除、计算总价）
   3. 然后写 UI 组件
   4. 最后写 API 集成
```

### 6.2 始终 Review AI 生成的代码

AI 也会犯错，特别是：
- 类型定义不准确
- 边界情况遗漏
- 使用过时的 API

**养成习惯**：每次 AI 生成代码后，花 30 秒检查一下。

### 6.3 保持上下文清晰

Chat 对话太长时，AI 会"忘记"前面的内容：
- 定期开启新对话
- 重要信息用 `@Files` 重新添加
- 复杂任务拆分成多个小对话

### 6.4 善用 .cursorrules 文件

在项目根目录创建 `.cursorrules` 文件，定义项目规范：

```
# .cursorrules 示例

## 技术栈
- React 18 + TypeScript
- Tailwind CSS
- React Query

## 代码规范
- 组件使用函数式 + Hooks
- 使用 arrow function
- 样式优先用 Tailwind
- API 响应类型放在 types/ 目录

## 命名规范
- 组件：PascalCase
- 函数：camelCase
- 常量：UPPER_SNAKE_CASE
```

Cursor 会自动遵循这些规则。

## 总结

Cursor 不是一个简单的补全工具，而是一个 AI 结对编程伙伴。

**核心价值**：
1. **理解代码库** - 不只是当前文件
2. **多文件编辑** - Composer 是杀手锏
3. **内置 Chat** - 无需切换工具
4. **持续学习** - 越用越智能

**上手建议**：
1. 第一周：只用 Tab 补全和 Chat
2. 第二周：开始用 Ctrl+K 内联编辑
3. 第三周：尝试 Composer 多文件编辑
4. 之后：探索高级技巧，定制工作流

用好了 Cursor，开发效率至少提升 3-5 倍。不要抗拒 AI，让它成为你的超能力。
