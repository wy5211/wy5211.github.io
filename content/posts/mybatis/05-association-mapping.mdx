## 前言

在实际业务中，对象之间往往存在关联关系。本文将讲解 MyBatis 如何处理一对一关联。

---

## 一对一关系示例

### 场景：用户与用户详情

```java
@Data
public class User {
    private Long id;
    private String username;
    private UserDetail detail;  // 一对一关系
}

@Data
public class UserDetail {
    private Long id;
    private Long userId;
    private String realName;
    private String phone;
    private String address;
}
```

---

## association 嵌套查询

### 方式 1：嵌套查询（两次查询）

```xml
<resultMap id="UserWithDetailMap" type="User">
    <id column="id" property="id"/>
    <result column="username" property="username"/>

    <!-- 嵌套查询 -->
    <association property="detail" column="id"
                select="selectDetailById" fetchType="lazy"/>
</resultMap>

<select id="selectById" resultMap="UserWithDetailMap">
    SELECT * FROM users WHERE id = #{id}
</select>

<select id="selectDetailById" resultType="UserDetail">
    SELECT * FROM user_details WHERE user_id = #{id}
</select>
```

### 方式 2：嵌套结果（一次查询）

```xml
<resultMap id="UserWithDetailMap2" type="User">
    <id column="id" property="id"/>
    <result column="username" property="username"/>

    <!-- 嵌套结果 -->
    <association property="detail" javaType="UserDetail">
        <id column="detail_id" property="id"/>
        <result column="real_name" property="realName"/>
        <result column="phone" property="phone"/>
        <result column="detail_address" property="address"/>
    </association>
</resultMap>

<select id="selectByIdWithDetail" resultMap="UserWithDetailMap2">
    SELECT
        u.id, u.username,
        d.id as detail_id, d.real_name, d.phone, d.address as detail_address
    FROM users u
    LEFT JOIN user_details d ON u.id = d.user_id
    WHERE u.id = #{id}
</select>
```

## 延迟加载

### 配置延迟加载

```xml
<settings>
    <!-- 开启延迟加载 -->
    <setting name="lazyLoadingEnabled" value="true"/>
    <!-- 按需加载 -->
    <setting name="aggressiveLazyLoading" value="false"/>
</settings>
```

### 使用示例

```java
// 查询用户时不会立即查询详情
User user = userMapper.selectById(1L);

// 访问详情时才会触发查询
System.out.println(user.getDetail().getRealName());
```

---

## 最佳实践

✅ 数据量小用嵌套结果
✅ 数据大或不是每次都需要用嵌套查询
✅ 合理使用延迟加载
