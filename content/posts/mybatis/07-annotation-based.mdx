## 前言

除了 XML，MyBatis 还支持注解方式开发。对于简单 SQL，注解方式更加简洁。

---

## 基础注解

### @Select 查询注解

```java
public interface UserMapper {
    @Select("SELECT * FROM users WHERE id = #{id}")
    User selectById(Long id);

    @Select("SELECT * FROM users")
    List<User> selectAll();
}
```

### @Insert 插入注解

```java
@Insert("INSERT INTO users (username, email, age) " +
        "VALUES (#{username}, #{email}, #{age})")
@Options(useGeneratedKeys = true, keyProperty = "id")
int insert(User user);
```

### @Update 更新注解

```java
@Update("UPDATE users SET " +
        "username = #{username}, " +
        "email = #{email}, " +
        "age = #{age} " +
        "WHERE id = #{id}")
int update(User user);
```

### @Delete 删除注解

```java
@Delete("DELETE FROM users WHERE id = #{id}")
int deleteById(Long id);
```

---

## @Results 结果映射

### 定义结果映射

```java
@Results(id = "UserResult", value = {
    @Result(property = "id", column = "id", id = true),
    @Result(property = "username", column = "username"),
    @Result(property = "createTime", column = "create_time")
})
@Select("SELECT * FROM users WHERE id = #{id}")
User selectById(Long id);

// 引用结果映射
@ResultMap("UserResult")
@Select("SELECT * FROM users WHERE username = #{username}")
User selectByUsername(String username);
```

## 动态 SQL 注解

### @SelectProvider

```java
// 提供 SQL 的类
class UserSqlProvider {
    public String selectUsersByCondition(Map<String, Object> params) {
        return new SQL() {{
            SELECT("*");
            FROM("users");
            if (params.get("username") != null) {
                WHERE("username = #{username}");
            }
            if (params.get("minAge") != null) {
                WHERE("age >= #{minAge}");
            }
        }}.toString();
    }
}

// 使用 Provider
@SelectProvider(type = UserSqlProvider.class, method = "selectUsersByCondition")
List<User> selectUsersByCondition(Map<String, Object> params);
```

## 注解 vs XML

| 特性     | 注解               | XML                |
| -------- | ------------------ | ------------------ |
| 简洁性   | ✅ 简单 SQL 更简洁 | ❌ 需要 XML 文件   |
| 复杂 SQL | ❌ 复杂 SQL 难维护 | ✅ 复杂 SQL 更清晰 |
| 动态 SQL | ⚠️ 需要 Provider   | ✅ 标签更直观      |
| 维护性   | ⚠️ SQL 混在代码中  | ✅ SQL 独立管理    |

**选择建议**：

- 简单查询 → 注解
- 复杂查询 → XML
- 混合使用 → 简单用注解，复杂用 XML
