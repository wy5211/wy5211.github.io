# 开发者提效工具箱：终端、Git、VS Code 效率翻倍指南

工欲善其事，必先利其器。作为开发者，每天和终端、Git、编辑器打交道，掌握高效工具和技巧，能让你的开发效率翻倍。本文分享我常用的提效工具和技巧。

## 一、终端神器

### 1.1 iTerm2（macOS）

**核心功能：**

- 分屏（Cmd+D 垂直，Cmd+Shift+D 水平）
- 热键窗口（Option+Space 快速唤出）
- 搜索（Cmd+F）
- 自动补全（Cmd+;）
- 历史命令（Cmd+Shift+;）

**配置建议：**

```
Preferences → Appearance → Theme: Minimal
Preferences → Profiles → Colors: Solarized Dark
Preferences → Profiles → Keys → Key Mappings: 添加快捷键
```

### 1.2 Oh My Zsh

```bash
# 安装
sh -c "$(curl -fsSL https://raw.githubusercontent.com/ohmyzsh/ohmyzsh/master/tools/install.sh)"

# 必装插件（~/.zshrc）
plugins=(
  git
  zsh-autosuggestions      # 命令自动建议
  zsh-syntax-highlighting  # 语法高亮
  z                        # 目录跳转
  docker
  npm
)

# 安装插件
git clone https://github.com/zsh-users/zsh-autosuggestions ${ZSH_CUSTOM:-~/.oh-my-zsh/custom}/plugins/zsh-autosuggestions
git clone https://github.com/zsh-users/zsh-syntax-highlighting ${ZSH_CUSTOM:-~/.oh-my-zsh/custom}/plugins/zsh-syntax-highlighting
```

## 二、Git 效率技巧

### 2.1 Git 别名

```bash
# ~/.gitconfig
[alias]
  co = checkout
  br = branch
  ci = commit
  st = status
  unstage = reset HEAD --
  last = log -1 HEAD
  visual = log --oneline --graph --all
  undo = checkout HEAD --
  amend = commit --amend --no-edit
```

### 2.2 常用快捷操作

```bash
# 快速提交
git add . && git commit -m "message"

# 撤销上次提交（保留修改）
git reset --soft HEAD~1

# 撤销上次提交（丢弃修改）
git reset --hard HEAD~1

# 修改上次提交信息
git commit --amend -m "new message"

# 只提交部分文件
git add -p

# 暂存当前修改
git stash
git stash pop

# 查看文件历史
git log --follow -p -- filename

# 查看某行代码的修改历史
git log -L 10,20:file.js

# 回退到某个提交
git reset --hard <commit-hash>

# 删除已合并的分支
git branch --merged | grep -v "\*" | xargs -n 1 git branch -d
```

### 2.3 Git 钩子

```bash
# .git/hooks/pre-commit
#!/bin/sh
npm run lint
if [ $? -ne 0 ]; then
  echo "Lint failed, please fix before commit"
  exit 1
fi
```

使用 Husky 自动化：

```bash
npm install husky -D
npx husky init
# 创建 .husky/pre-commit
npm run lint
```

## 四、高效工作流

### 4.1 项目模板

```bash
# 创建项目模板目录
mkdir -p ~/templates/react-ts

# 使用模板创建新项目
cp -r ~/templates/react-ts ~/Projects/new-project
cd ~/Projects/new-project
rm -rf .git && git init
```

使用 pnpm create：

```bash
# 创建模板
pnpm create vite my-app --template react-ts
```

### 4.2 代码片段

VS Code 用户片段：

```json
// JavaScript/TypeScript 代码片段
// .vscode/snippets.code-snippets
{
  "Console Log": {
    "prefix": "cl",
    "body": ["console.log($1)", "$0"],
    "scope": "javascript,typescript"
  },
  "React Component": {
    "prefix": "rfc",
    "body": [
      "import React from 'react'",
      "",
      "interface ${1:Component}Props {",
      "  $2",
      "}",
      "",
      "export const ${1:Component}: React.FC<${1:Component}Props> = (props) => {",
      "  return (",
      "    <div>",
      "      $0",
      "    </div>",
      "  )",
      "}"
    ]
  }
}
```

### 4.3 自动化脚本

```bash
#!/bin/bash
# ~/scripts/new-project.sh

PROJECT_NAME=$1
TEMPLATE=$2

if [ -z "$PROJECT_NAME" ]; then
  echo "Usage: new-project <name> [template]"
  exit 1
fi

case $TEMPLATE in
  "react")
    pnpm create vite $PROJECT_NAME --template react-ts
    ;;
  "next")
    pnpm create next-app $PROJECT_NAME --typescript
    ;;
  "node")
    mkdir $PROJECT_NAME && cd $PROJECT_NAME
    pnpm init
    pnpm add typescript -D
    ;;
  *)
    mkdir $PROJECT_NAME
    ;;
esac

echo "Project $PROJECT_NAME created!"
```

## 总结

效率提升是持续优化的过程。

**核心工具：**

1. **终端** - iTerm2 + Oh My Zsh + 必备工具
2. **Git** - 别名 + 快捷操作 + 钩子
3. **VS Code** - 必装插件 + 快捷键 + 配置
4. **工作流** - 模板 + 代码片段 + 自动化

**建议：**

- 每周学一个新工具或技巧
- 记录常用命令，整理成别名
- 分享给团队，统一效率工具

效率工具不在多，在于精通。把工具变成肌肉记忆，才能真正提升效率。
