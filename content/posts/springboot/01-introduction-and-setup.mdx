## 前言

SpringBoot 是目前 Java 企业级开发中最流行的框架，它简化了 Spring 应用的初始搭建和开发过程，让我们能够快速构建生产级别的应用。

**为什么选择 SpringBoot？**

- 🚀 **快速开发**：约定优于配置，开箱即用
- 📦 **简化配置**：自动配置，无需繁琐的 XML 配置
- 🎯 **生产就绪**：内嵌 Tomcat，提供监控、健康检查等功能
- 🔧 **生态完善**：与 Spring 生态系统无缝集成
- 🏢 **企业首选**：大多数公司采用的技术栈

## 什么是 SpringBoot

### SpringBoot 的诞生

在传统的 Spring 开发中，我们需要配置大量的 XML 文件，开发效率较低。SpringBoot 应运而生，它不是对 Spring 功能的增强，而是提供了一种快速使用 Spring 的方式。

```
传统 Spring 开发：
- 配置繁琐的 XML
- 依赖管理复杂
- 部署配置麻烦
- 开发效率低

SpringBoot 开发：
- 零 XML 配置
- 自动依赖管理
- 内嵌服务器
- 快速开发
```

### 核心特性

**1. 自动配置（Auto-Configuration）**

SpringBoot 会根据项目中的 jar 依赖自动配置 Spring 应用，大大减少了手动配置的工作。

**2. 起步依赖（Starter Dependencies）**

将常用的依赖分组，引入一个 starter 就能获得该场景所需的所有依赖。

```xml
<!-- Web 开发场景 -->
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-web</artifactId>
</dependency>

<!-- 会自动引入：Spring MVC、Jackson、Tomcat、Validator 等 -->
```

**3. 内嵌服务器**

内嵌 Tomcat、Jetty 或 Undertow，无需部署 WAR 文件。

**4. 生产特性**

提供健康检查、指标监控、外部化配置等生产级别的特性。

**5. 无代码生成**

不需要生成任何代码，也不需要 XML 配置。

## SpringBoot vs Spring

| 特性     | Spring            | SpringBoot           |
| -------- | ----------------- | -------------------- |
| 配置     | 大量 XML          | 零配置，自动配置     |
| 依赖     | 手动管理          | Starter 简化         |
| 部署     | 需要 Servlet 容器 | 内嵌容器，jar 包运行 |
| 开发效率 | 较低              | 快速开发             |
| 学习曲线 | 陡峭              | 平缓                 |

**核心区别**：SpringBoot 是对 Spring 的进一步封装，简化了 Spring 应用的初始搭建和开发过程。

## 环境准备

### 1. JDK 安装（Java 17）

SpringBoot 3.x 需要 Java 17 或更高版本。

**检查 Java 版本**：

```bash
java -version
# 应该显示：java version "17.x.x"
```

**安装 Java 17**：

```bash
# macOS (使用 Homebrew)
brew install openjdk@17

# Ubuntu/Debian
sudo apt update
sudo apt install openjdk-17-jdk

# Windows
# 下载并安装 Oracle JDK 17 或 OpenJDK 17
```

**配置 JAVA_HOME**：

```bash
# macOS/Linux
export JAVA_HOME=/path/to/java17
export PATH=$JAVA_HOME/bin:$PATH

# 添加到 ~/.bashrc 或 ~/.zshrc
echo 'export JAVA_HOME=/path/to/java17' >> ~/.zshrc
source ~/.zshrc
```

### 2. Maven 安装

Maven 是 Java 项目管理工具，用于依赖管理和项目构建。

```bash
# 检查 Maven 版本
mvn -version

# macOS
brew install maven

# Ubuntu/Debian
sudo apt install maven

# 配置镜像加速（可选）
# 编辑 ~/.m2/settings.xml，添加阿里云镜像
```

**Gradle 用户**：

```bash
# macOS
brew install gradle

# Ubuntu/Debian
sudo apt install gradle
```

### 3. IDE 选择

推荐使用 **IntelliJ IDEA**（旗舰版或社区版）。

**为什么选择 IntelliJ IDEA？**

- 对 SpringBoot 有最好的支持
- 代码智能提示和自动补全
- 内置数据库工具
- 强大的调试功能
- Maven/Gradle 集成

**下载地址**：[https://www.jetbrains.com/idea/](https://www.jetbrains.com/idea/)

## 创建第一个项目

### 方式一：Spring Initializr（Web）

1. 访问 [https://start.spring.io/](https://start.spring.io/)
2. 配置项目信息：
   - Project: Maven
   - Language: Java
   - Spring Boot: 3.2.0
   - Group: com.example
   - Artifact: springboot-demo
   - Name: springboot-demo
   - Package name: com.example.springbootdemo
   - Packaging: Jar
   - Java: 17
3. 选择依赖：
   - Spring Web
   - Spring Boot DevTools
4. 点击 "GENERATE" 下载项目
5. 解压并用 IDEA 打开

### 方式二：IDEA 创建（推荐）

1. 打开 IntelliJ IDEA
2. New Project → Spring Initializr
3. 配置项目信息（同上）
4. 选择依赖
5. 点击 Finish，等待依赖下载

### 方式三：命令行创建

```bash
# 使用 curl
curl https://start.spring.io/starter.zip \
  -d dependencies=web,devtools \
  -d type=maven-project \
  -d language=java \
  -d bootVersion=3.2.0 \
  -d baseDir=springboot-demo \
  -o springboot-demo.zip

# 或使用 SpringBoot CLI
spring init --dependencies=web,devtools springboot-demo
```

## 项目结构

创建完成后，项目结构如下：

```
springboot-demo/
├── src/
│   ├── main/
│   │   ├── java/
│   │   │   └── com/example/springbootdemo/
│   │   │       ├── SpringbootDemoApplication.java  # 主类
│   │   │       └── controller/                        # 控制器
│   │   └── resources/
│   │       ├── application.properties                  # 配置文件
│   │       ├── static/                                 # 静态资源
│   │       └── templates/                              # 模板文件
│   └── test/                                           # 测试代码
├── pom.xml                                             # Maven 配置
└── springboot-demo.iml                                # IDEA 配置
```

**核心文件说明**：

```java
// 主类：SpringbootDemoApplication.java
@SpringBootApplication
public class SpringbootDemoApplication {
    public static void main(String[] args) {
        SpringApplication.run(SpringbootDemoApplication.class, args);
    }
}
```

**@SpringBootApplication 注解**：

这是一个复合注解，包含：

- `@Configuration`：配置类
- `@EnableAutoConfiguration`：启用自动配置
- `@ComponentScan`：组件扫描

## 编写第一个 API

### 创建 Controller

```java
package com.example.springbootdemo.controller;

import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.PathVariable;
import org.springframework.web.bind.annotation.RequestParam;
import org.springframework.web.bind.annotation.RestController;

@RestController
@RequestMapping("/api")
public class HelloController {

    // 第一个接口：Hello World
    @GetMapping("/hello")
    public String hello() {
        return "Hello, SpringBoot!";
    }

    // 带路径参数
    @GetMapping("/users/{id}")
    public String getUser(@PathVariable Long id) {
        return "用户 ID: " + id;
    }

    // 带查询参数
    @GetMapping("/search")
    public String search(@RequestParam String keyword) {
        return "搜索关键词: " + keyword;
    }
}
```

**代码解释**：

- `@RestController`：组合了 `@Controller` 和 `@ResponseBody`，返回 JSON 数据
- `@RequestMapping`：定义类级别的请求映射
- `@GetMapping`：GET 请求映射
- `@PathVariable`：获取路径参数
- `@RequestParam`：获取查询参数

### 运行应用

**方式 1：IDEA 运行**

1. 打开 `SpringbootDemoApplication.java`
2. 点击 main 方法左侧的绿色三角
3. 选择 "Run 'SpringbootDemoApplication'"

**方式 2：Maven 命令**

```bash
cd springboot-demo
mvn spring-boot:run
```

**方式 3：打包后运行**

```bash
# 打包
mvn clean package -DskipTests

# 运行
java -jar target/springboot-demo-0.0.1-SNAPSHOT.jar
```

### 测试接口

应用启动后（默认端口 8080），访问：


或使用浏览器访问：

## 配置文件

### application.properties

```properties
# 服务器配置
server.port=8080
server.servlet.context-path=/api

# 应用名称
spring.application.name=springboot-demo

# 日志级别
logging.level.root=INFO
logging.level.com.example.springbootdemo=DEBUG

# 开发工具热部署
spring.devtools.restart.enabled=true
```

### application.yml（推荐）

```yaml
server:
  port: 8080
  servlet:
    context-path: /api

spring:
  application:
    name: springboot-demo
  devtools:
    restart:
      enabled: true

logging:
  level:
    root: INFO
    com.example.springbootdemo: DEBUG
```

**YAML 优势**：

- 层级清晰，易读
- 支持注释
- 比 properties 更简洁

## 热部署

SpringBoot DevTools 提供热部署功能，修改代码后自动重启。

### 启用热部署

```xml
<!-- pom.xml 中已包含（如果使用 Spring Initializr 创建） -->
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-devtools</artifactId>
    <scope>runtime</scope>
    <optional>true</optional>
</dependency>
```

### IDEA 配置

1. Settings → Build, Execution, Deployment → Compiler
   - 勾选 "Build project automatically"

2. Settings → Advanced Settings
   - 勾选 "Allow auto-make to start even if developed application is currently running"

3. 使用快捷键：`Ctrl + F9` (Windows/Linux) 或 `Cmd + F9` (macOS) 重新编译

## 常见问题

### Q1: 端口被占用？

**A:** 修改端口或关闭占用进程。

```yaml
# 修改端口
server:
  port: 8081
```

### Q2: 依赖下载慢？

**A:** 配置 Maven 镜像。

```xml
<!-- ~/.m2/settings.xml -->
<mirrors>
  <mirror>
    <id>aliyun</id>
    <mirrorOf>central</mirrorOf>
    <url>https://maven.aliyun.com/repository/public</url>
  </mirror>
</mirrors>
```

### Q3: 中文乱码？

**A:** 确保文件编码为 UTF-8。

```yaml
server:
  servlet:
    encoding:
      charset: UTF-8
      force: true
```
