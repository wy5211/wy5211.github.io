## 前言

依赖注入（DI）和自动配置是 SpringBoot 的两大核心特性。理解这些概念对于编写高质量的 SpringBoot 应用至关重要。

本文将深入讲解 IoC 容器、依赖注入的实现方式，以及自动配置的工作原理。

## IoC 和 DI 概念

### 什么是 IoC（控制反转）？

IoC（Inversion of Control）是一种设计思想，将对象的创建和管理权交给容器，而不是对象自己管理。

**传统方式**：

```java
// 对象自己创建依赖
public class UserService {
    private UserRepository userRepository = new UserRepository();
    // 代码耦合，难以测试
}
```

**IoC 方式**：

```java
// 容器负责创建和注入依赖
public class UserService {
    @Autowired
    private UserRepository userRepository;
    // 解耦，易于测试
}
```

### 什么是 DI（依赖注入）？

DI 是 IoC 的实现方式，容器自动将依赖对象注入到目标对象中。

## 依赖注入方式

### 1. 字段注入

```java
@Service
public class UserService {

    @Autowired
    private UserRepository userRepository;

    public User findById(Long id) {
        return userRepository.findById(id).orElse(null);
    }
}
```

**优点**：简单方便
**缺点**：

- 容易忘记注入（空指针）
- 无法使用 final
- 不推荐使用

### 2. Setter 注入

```java
@Service
public class UserService {

    private UserRepository userRepository;

    @Autowired
    public void setUserRepository(UserRepository userRepository) {
        this.userRepository = userRepository;
    }
}
```

**优点**：可以依赖可选
**缺点**：

- 无法使用 final
- 注入后可能被修改

### 3. 构造器注入（推荐）

```java
@Service
public class UserService {

    private final UserRepository userRepository;

    @Autowired
    public UserService(UserRepository userRepository) {
        this.userRepository = userRepository;
    }

    public User findById(Long id) {
        return userRepository.findById(id).orElse(null);
    }
}
```

**优点**：

- ✅ 可以使用 final（保证不可变）
- ✅ 确保依赖不为空
- ✅ 便于测试
- ✅ Spring 4.3+ 可以省略 @Autowired

**最佳实践**：

```java
// Spring 4.3+：单构造函数可以省略 @Autowired
@Service
public class UserService {

    private final UserRepository userRepository;

    public UserService(UserRepository userRepository) {
        this.userRepository = userRepository;
    }
}
```

## 组件注解

### @Component

通用组件注解，标记为 Spring Bean。

```java
@Component
public class EmailUtil {

    public void sendEmail(String to, String subject, String content) {
        // 发送邮件逻辑
    }
}
```

### 专用组件注解

```java
@Service  // 服务层
@Repository  // 数据访问层
@Controller  // 控制器层
@RestController  // REST 控制器
@Configuration  // 配置类
```

**本质**：都是 @Component 的元注解。

```java
@Component
public @interface Service {
    // ...
}
```

## Bean 定义

### @Bean 注解

用于配置类中，手动定义 Bean。

```java
@Configuration
public class AppConfig {

    @Bean
    public RestTemplate restTemplate() {
        return new RestTemplate();
    }

    @Bean
    public DataSource dataSource() {
        HikariDataSource dataSource = new HikariDataSource();
        dataSource.setJdbcUrl("jdbc:mysql://localhost:3306/demo");
        dataSource.setUsername("root");
        dataSource.setPassword("root123");
        return dataSource;
    }
}
```

### Bean 依赖

```java
@Configuration
public class AppConfig {

    @Bean
    public UserService userService(UserRepository userRepository) {
        return new UserService(userRepository);
    }

    @Bean
    public UserRepository userRepository(DataSource dataSource) {
        return new UserRepositoryImpl(dataSource);
    }
}
```

## 依赖注入示例

### 完整示例

```java
// Entity
@Entity
public class User {
    private Long id;
    private String username;
    // ...
}

// Repository
@Repository
public interface UserRepository extends JpaRepository<User, Long> {
}

// Service
@Service
public class UserService {

    private final UserRepository userRepository;
    private final EmailService emailService;

    // Spring 4.3+：单构造函数可以省略 @Autowired
    public UserService(UserRepository userRepository, EmailService emailService) {
        this.userRepository = userRepository;
        this.emailService = emailService;
    }

    public User create(UserCreateDTO dto) {
        User user = new User();
        user.setUsername(dto.getUsername());
        user = userRepository.save(user);

        // 发送欢迎邮件
        emailService.sendWelcomeEmail(user);

        return user;
    }
}

// Email Service
@Service
public class EmailService {

    private final JavaMailSender mailSender;

    public EmailService(JavaMailSender mailSender) {
        this.mailSender = mailSender;
    }

    public void sendWelcomeEmail(User user) {
        // 发送邮件逻辑
    }
}

// Controller
@RestController
@RequestMapping("/api/users")
public class UserController {

    private final UserService userService;

    public UserController(UserService userService) {
        this.userService = userService;
    }

    @PostMapping
    public Result<User> create(@RequestBody UserCreateDTO dto) {
        User user = userService.create(dto);
        return Result.success(user);
    }
}
```

## 自动配置原理

### @SpringBootApplication

```java
@SpringBootApplication
public class SpringbootDemoApplication {
    public static void main(String[] args) {
        SpringApplication.run(SpringbootDemoApplication.class, args);
    }
}
```

**@SpringBootApplication 是复合注解**：

```java
@SpringBootConfiguration  // 配置类
@EnableAutoConfiguration    // 启用自动配置
@ComponentScan             // 组件扫描
public @interface SpringBootApplication {
    // ...
}
```

### 自动配置流程

```
1. @EnableAutoConfiguration
   ↓
2. 加载 META-INF/spring.factories
   ↓
3. 读取配置类
   ↓
4. 根据条件注解决定是否加载
   ↓
5. 创建 Bean
```

### 条件注解

```java
// 类存在时配置
@Configuration
@ConditionalOnClass(DataSource.class)
class DataSourceAutoConfiguration {
    // ...
}

// 类不存在时配置
@ConditionalOnMissingClass(DataSource.class)
class EmbeddedDataSourceConfiguration {
    // ...
}

// Bean 存在时配置
@ConditionalOnBean(DataSource.class)
class JpaConfiguration {
    // ...
}

// Bean 不存在时配置
@ConditionalOnMissingBean(DataSource.class)
class DefaultDataSourceConfiguration {
    // ...
}

// 属性存在时配置
@ConditionalOnProperty(name = "spring.datasource.url",
                       matchIfMissing = false)
class DataSourceConfiguration {
    // ...
}

// 资源存在时配置
@ConditionalOnResource(resources = "META-INF/persistence.xml")
class JpaConfiguration {
    // ...
}
```

## 自定义配置

### @ConfigurationProperties

将配置文件中的属性映射到 Java 对象。

```yaml
# application.yml
app:
  name: SpringBoot Demo
  version: 1.0.0
  author: 张三
```

```java
@Component
@ConfigurationProperties(prefix = "app")
public class AppProperties {

    private String name;
    private String version;
    private String author;

    // Getter 和 Setter
    public String getName() { return name; }
    public void setName(String name) { this.name = name; }

    public String getVersion() { return version; }
    public void setVersion(String version) { this.version = version; }

    public String getAuthor() { return author; }
    public void setAuthor(String author) { this.author = author; }
}
```

### @Value 注解

直接注入配置值。

```java
@Component
public class AppConfig {

    @Value("${app.name}")
    private String appName;

    @Value("${app.version}")
    private String appVersion;

    @Value("${app.author}")
    private String appAuthor;

    public void printInfo() {
        System.out.println("应用名: " + appName);
        System.out.println("版本: " + appVersion);
        System.out.println("作者: " + appAuthor);
    }
}
```

## 外部化配置

### Profile 配置

```yaml
# application-dev.yml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/dev

spring:
  datasource:
```

```yaml
  profiles:
```

```bash

java -jar app.jar --spring.profiles.active=prod

# 覆盖配置
java -jar app.jar --server.port=9090
```

## 最佳实践

### 1. 优先使用构造器注入

```java
// ✅ 推荐
@Service
public class UserService {

    private final UserRepository userRepository;

    public UserService(UserRepository userRepository) {
        this.userRepository = userRepository;
    }
}

// ❌ 避免
@Service
public class UserService {

    @Autowired
    private UserRepository userRepository;
}
```

### 2. 使用 final 保证不可变

```java
// ✅ 推荐
private final UserRepository userRepository;

// ❌ 避免
private UserRepository userRepository;
```

### 3. 合理划分层级

```java
// Controller → Service → Repository
// 单向依赖，避免循环依赖
```
