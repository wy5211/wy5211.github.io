## 前言

性能优化是生产环境的关键环节。本文将讲解从 JVM 到应用层面的全方位优化策略。

## JVM 调优

### JVM 参数

```bash
java -jar app.jar \
  -Xms512m \
  -Xmx1024m \
  -XX:+UseG1GC \
  -XX:MaxGCPauseMillis=200 \
  -XX:+HeapDumpOnOutOfMemoryError \
  -XX:HeapDumpPath=/app/logs/heapdump.hprof
```

### Dockerfile 中配置

```dockerfile
FROM eclipse-temurin:17-jre-alpine
WORKDIR /app
COPY app.jar app.jar
EXPOSE 8080

ENV JAVA_OPTS="-Xms512m -Xmx1024m -XX:+UseG1GC"

ENTRYPOINT ["sh", "-c", "java $JAVA_OPTS -jar app.jar"]
```

### 连接池配置

```yaml
spring:
  datasource:
    hikari:
      # 连接池大小
      maximum-pool-size: 20
      minimum-idle: 5

      # 连接超时
      connection-timeout: 30000
      idle-timeout: 600000
      max-lifetime: 1800000

      # 性能优化
      connection-test-query: SELECT 1
      validation-timeout: 3000
```

## 缓存优化

### 多级缓存

```java
@Service
public class UserService {

    // L1：本地缓存
    @Cacheable(value = "users", key = "#id")
    public User findById(Long id) {
        return userRepository.findById(id).orElse(null);
    }

    // L2：分布式缓存
    @Cacheable(value = "userSessions", key = "#token")
    public UserSession findByToken(String token) {
        return sessionRepository.findByToken(token);
    }
}
```

### 缓存预热

```java
@Component
public class CacheWarmup implements ApplicationRunner {

    @Autowired
    private UserService userService;

    @Override
    public void run(ApplicationArguments args) {
        // 预热热点数据
        userService.loadHotData();
    }
}
```

## 接口优化

### 分页优化

```java
// ✅ 推荐：使用游标分页
@GetMapping("/users")
public Page<User> getUsers(
    @RequestParam(defaultValue = "0") int page,
    @RequestParam(defaultValue = "20") int size
) {
    return userRepository.findAll(
        PageRequest.of(page, size, Sort.by("id").descending())
    );
}
```

### 异步处理

```java
@Service
public class ReportService {

    @Async
    public CompletableFuture<Report> generateReport(Long id) {
        // 耗时操作异步执行
        Report report = buildReport(id);
        return CompletableFuture.completedFuture(report);
    }

```

## 监控与分析

### Actuator 配置

```yaml
management:
  endpoints:
    web:
      exposure:
        include: health,info,metrics,prometheus
  metrics:
    export:
      prometheus:
        enabled: true
```

### 自定义指标

```java
@Component
public class ApiMetrics {

    private final MeterRegistry meterRegistry;

    public void recordApiCall(String endpoint, long duration) {
        Timer.builder("api.call.duration")
            .tag("endpoint", endpoint)
            .register(meterRegistry)
            .record(duration, TimeUnit.MILLISECONDS);
    }
}
```

### 性能分析工具

```bash
# JVM 监控
jps -l
jstat -gcutil <pid> 1000

# 线程 dump
jstack <pid> > thread_dump.txt

# 堆 dump
jmap -dump:live,format=b,file=heap.hprof <pid>
```
