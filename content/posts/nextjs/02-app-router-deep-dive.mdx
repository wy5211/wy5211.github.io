## 文件系统路由约定

### 基础路由

App Router 使用 `app` 目录下的文件系统来定义路由：

```
app/
├── layout.tsx           → /
├── page.tsx             → /
├── about/
│   └── page.tsx         → /about
├── blog/
│   ├── page.tsx         → /blog
│   └── [slug]/          # 动态路由段
│       └── page.tsx     → /blog/hello-world
└── shop/
    ├── page.tsx         → /shop
    └── [category]/      # 动态路由段
        └── [id]/        # 嵌套动态路由
            └── page.tsx → /shop/clothing/123
```

### 特殊文件约定

Next.js 使用特殊的文件名来定义特定功能：

| 文件名          | 功能                          |
| --------------- | ----------------------------- |
| `page.tsx`      | 路由的唯一页面，定义路由的 UI |
| `layout.tsx`    | 多个页面共享的 UI             |
| `loading.tsx`   | 加载状态 UI                   |
| `error.tsx`     | 错误边界 UI                   |
| `not-found.tsx` | 404 页面 UI                   |
| `route.ts`      | API 端点                      |

### 路由优先级

路由匹配遵循以下优先级（从高到低）：

1. 预定义路由（如 `blog/new`）
2. 动态路由（如 `blog/[slug]`）
3. 捕获所有路由（如 `[...slug]`）

## 动态路由

### 基础动态路由

使用方括号 `[]` 创建动态路由段：

```typescript
// app/blog/[slug]/page.tsx
type Props = {
  params: { slug: string };
};

export default async function BlogPost({ params }: Props) {
  const { slug } = params;

  // 根据 slug 获取文章数据
  const post = await fetch(`https://api.example.com/posts/${slug}`)
    .then(res => res.json());

  return (
    <article>
      <h1>{post.title}</h1>
      <p>{post.content}</p>
    </article>
  );
}
```

**URL 示例**：

- `/blog/nextjs-intro` → `slug = "nextjs-intro"`
- `/blog/react-hooks` → `slug = "react-hooks"`

### 多段动态路由

```typescript
// app/shop/[category]/[id]/page.tsx
type Props = {
  params: {
    category: string;
    id: string;
  };
};

export default async function ProductPage({ params }: Props) {
  const { category, id } = params;

  const product = await fetch(
    `https://api.example.com/products/${category}/${id}`
  ).then(res => res.json());

  return (
    <div>
      <h1>{product.name}</h1>
      <p>分类: {category}</p>
    </div>
  );
}
```

**URL 示例**：

- `/shop/clothing/123` → `category = "clothing"`, `id = "123"`

### 捕获所有路由

使用 `[...param]` 捕获多层路径：

```typescript
// app/docs/[...slug]/page.tsx
type Props = {
  params: { slug: string[] };
};

export default function DocPage({ params }: Props) {
  // slug 是一个字符串数组
  const path = params.slug.join('/');

  return (
    <div>
      <h1>文档: {path}</h1>
    </div>
  );
}
```

**URL 示例**：

- `/docs/getting-started` → `slug = ["getting-started"]`
- `/docs/guides/deployment` → `slug = ["guides", "deployment"]`
- `/docs/api/routes/handlers` → `slug = ["api", "routes", "handlers"]`

### 可选的捕获所有路由

使用 `[[...param]]` 使路由段可选：

```typescript
// app/blog/[[slug]]/page.tsx
type Props = {
  params: { slug?: string[] };
};

export default function BlogPage({ params }: Props) {
  if (!params.slug) {
    // 显示所有博客列表
    return <div>所有博客</div>;
  }

  // 显示特定博客或分类
  return <div>博客: {params.slug.join('/')}</div>;
}
```

**URL 示例**：

- `/blog` → `slug = undefined`
- `/blog/nextjs` → `slug = ["nextjs"]`
- `/blog/2026/march` → `slug = ["2026", "march"]`

## 路由组

### 组织代码而不影响 URL

使用括号 `()` 创建路由组，用于组织代码而不影响 URL：

```
app/
├── (marketing)/
│   ├── about/
│   │   └── page.tsx         → /about
│   ├── blog/
│   │   └── page.tsx         → /blog
│   └── layout.tsx           # marketing 组的布局
└── (dashboard)/
    ├── analytics/
    │   └── page.tsx         → /analytics
    ├── settings/
    │   └── page.tsx         → /settings
    └── layout.tsx           # dashboard 组的布局
```

### 多个根布局

路由组允许多个根布局：

```typescript
// app/(marketing)/layout.tsx
export default function MarketingLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <div className="marketing-layout">
      <header>营销页面头部</header>
      {children}
      <footer>营销页面底部</footer>
    </div>
  );
}

// app/(dashboard)/layout.tsx
export default function DashboardLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <div className="dashboard-layout">
      <aside>侧边栏</aside>
      <main>{children}</main>
    </div>
  );
}
```

### 路由组最佳实践

```
app/
├── (auth)/
│   ├── login/
│   │   └── page.tsx         → /login
│   ├── register/
│   │   └── page.tsx         → /register
│   └── layout.tsx           # 认证页面布局（无导航）
├── (app)/
│   ├── dashboard/
│   │   └── page.tsx         → /dashboard
│   ├── profile/
│   │   └── page.tsx         → /profile
│   └── layout.tsx           # 应用布局（有导航和侧边栏）
└── (public)/
    ├── about/
    │   └── page.tsx         → /about
    └── layout.tsx           # 公共页面布局（简单的头部和底部）
```

## 并行路由

### 创建独立的内容区域

并行路由允许同时渲染多个独立页面：

```
app/
├── layout.tsx
├── @team/
│   └── page.tsx            # 团队信息
├── @analytics/
│   └── page.tsx            # 分析数据
└── page.tsx                # 主内容
```

```typescript
// app/layout.tsx
export default function RootLayout({
  children,
  team,
  analytics,
}: {
  children: React.ReactNode;
  team: React.ReactNode;
  analytics: React.ReactNode;
}) {
  return (
    <html>
      <body>
        <div className="grid">
          <main>{children}</main>
          <aside>{team}</aside>
          <aside>{analytics}</aside>
        </div>
      </body>
    </html>
  );
}
```

### 插槽（Slots）

使用 `@` 符号定义插槽：

```
app/
├── layout.tsx
├── page.tsx                → 默认插槽 (children)
├── @dashboard/
│   └── page.tsx            → dashboard 插槽
└── @settings/
    └── page.tsx            → settings 插槽
```

```typescript
// app/layout.tsx
export default function Layout({
  children,
  dashboard,
  settings,
}: {
  children: React.ReactNode;
  dashboard?: React.ReactNode;
  settings?: React.ReactNode;
}) {
  return (
    <div className="layout">
      <main>{children}</main>
      {dashboard && <aside id="dashboard">{dashboard}</aside>}
      {settings && <aside id="settings">{settings}</aside>}
    </div>
  );
}
```

**URL 示例**：

- `/` → 只显示主内容
- `/@dashboard/dashboard` → 显示主内容 + dashboard
- `/@settings/settings` → 显示主内容 + settings

### 使用并行路由构建仪表板

```
app/
├── (dashboard)/
│   ├── layout.tsx
│   ├── @overview/
│   │   └── page.tsx        → 概览
│   ├── @performance/
│   │   └── page.tsx        → 性能
│   └── @sales/
│       └── page.tsx        → 销售
```

```typescript
// app/(dashboard)/layout.tsx
export default function DashboardLayout({

  performance,

  overview: React.ReactNode;
  performance: React.ReactNode;

  return (
    <div className="dashboard">
      <nav>
        <a href="/dashboard">概览</a>
        <a href="/dashboard/performance">性能</a>
        <a href="/dashboard/sales">销售</a>
      </nav>
      <div className="content">
        {overview}
        {performance}

      </div>
    </div>
  );
}
```

## 拦截路由

### 模态框和预览

拦截路由允许在当前上下文中显示路由：

```
app/
├── feed/
│   └── page.tsx            → /feed
├── photo/
│   └── [id]/
│       └── page.tsx        → /photo/123
└── (..)photo/
    └── [id]/
        └── page.tsx        # 拦截 /photo/123，显示为模态框
```

**使用 `(..)` 语法**：

- `(.)` - 同级
- `(..)` - 上一级
- `(..)(..)` - 上两级

### 实现照片模态框

```
app/
├── gallery/
│   ├── page.tsx            → /gallery（相册列表）
│   └── [id]/
│       └── page.tsx        → /gallery/123（照片详情页）
└── (.)gallery/
    └── [id]/
        └── page.tsx        → 拦截并显示为模态框
```

```typescript
// app/gallery/page.tsx
export default async function GalleryPage() {
  const photos = await fetch('https://api.example.com/photos')
    .then(res => res.json());

  return (
    <div>
      <h1>相册</h1>
      <div className="grid">
        {photos.map((photo: any) => (
          <a key={photo.id} href={`/gallery/${photo.id}`}>
            <img src={photo.thumbnail} alt={photo.title} />
          </a>
        ))}
      </div>
    </div>
  );
}
```

```typescript
// app/(.)gallery/[id]/page.tsx
type Props = {
  params: { id: string };
};

export default async function PhotoModal({ params }: Props) {
  const photo = await fetch(`https://api.example.com/photos/${params.id}`)
    .then(res => res.json());

  return (
    <dialog open>
      <img src={photo.url} alt={photo.title} />
      <form method="dialog">
        <button>关闭</button>
      </form>
    </dialog>
  );
}
```

**效果**：

- 直接访问 `/gallery/123` → 显示完整照片页
- 从相册点击照片 → 在模态框中显示（拦截路由）

## 布局系统

### 嵌套布局

布局自动嵌套，子布局会继承父布局：

```
app/
├── layout.tsx              # 根布局
├── page.tsx                # / → 根布局 + 首页
└── dashboard/
    ├── layout.tsx          # Dashboard 布局
    ├── page.tsx            # /dashboard → 根布局 + Dashboard 布局 + 仪表板
    └── settings/
        ├── layout.tsx      # Settings 布局
        └── page.tsx        # /dashboard/settings → 根布局 + Dashboard 布局 + Settings 布局 + 设置页
```

```typescript
// app/layout.tsx
export default function RootLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <html>
      <body>
        <header>网站头部</header>
        {children}
        <footer>网站底部</footer>
      </body>
    </html>
  );
}

// app/dashboard/layout.tsx
export default function DashboardLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <div className="dashboard">
      <aside>
        <nav>
          <a href="/dashboard">仪表板</a>
          <a href="/dashboard/settings">设置</a>
        </nav>
      </aside>
      <main>{children}</main>
    </div>
  );
}
```

### 路由特定的布局

每个路由可以有自己独立的布局：

```typescript
// app/login/page.tsx
export default function LoginPage() {
  return (
    <div className="login-page">

// app/login/layout.tsx

  children,
}: {
  children: React.ReactNode;

  return (
    <div className="minimal-layout">
      {children}
    </div>
  );

```

## 模板

模板与布局类似，但有几个关键区别：

| 特性     | Layout       | Template             |
| -------- | ------------ | -------------------- |
| 重新渲染 | 不重新渲染   | 每次导航都重新渲染   |
| 状态保留 | 保留         | 不保留               |
| 适用场景 | 导航、侧边栏 | 需要重新初始化的内容 |

```typescript
// app/template.tsx
export default function Template({
  children,
}: {
  children: React.ReactNode;
}) {
  // 每次导航都会重新创建
  return (
    <div>
      <h1>每次导航都会重新渲染这个模板</h1>
      {children}
    </div>
  );
}
```

**使用场景**：

- 动画（每次进入重新播放）
- 数据获取（每次刷新数据）
- 状态重置（如表单）

## 实战：博客详情页

让我们创建一个完整的博客详情页，结合所学的路由知识：

```typescript
// app/blog/[slug]/page.tsx
import { notFound } from 'next/navigation';

type Props = {
  params: { slug: string };
};

async function getBlogPost(slug: string) {
  const res = await fetch(`https://api.example.com/posts/${slug}`, {
    next: { revalidate: 3600 },
  });

  if (!res.ok) {
    return null;
  }

  return res.json();
}

export default async function BlogPost({ params }: Props) {
  const post = await getBlogPost(params.slug);

  if (!post) {
    notFound();
  }

  return (
    <article className="blog-post">
      <header>
        <h1>{post.title}</h1>
        <div className="meta">
          <time>{post.date}</time>
          <span>作者: {post.author}</span>
        </div>
      </header>

      <div
        className="content"
        dangerouslySetInnerHTML={{ __html: post.content }}
      />

      <footer>
        <div className="tags">
          {post.tags.map((tag: string) => (
            <span key={tag} className="tag">
              {tag}
            </span>
          ))}
        </div>
      </footer>
    </article>
  );
}

// 生成元数据
export async function generateMetadata({ params }: Props) {
  const post = await getBlogPost(params.slug);

  if (!post) {
    return {
      title: '文章未找到',
    };
  }

  return {
    title: post.title,
    description: post.excerpt,
    openGraph: {
      title: post.title,
      description: post.excerpt,
      images: [post.coverImage],
    },
  };
}
```

### ✅ 推荐做法

1. **使用路由组组织代码**

```
app/
├── (public)/
├── (app)/
└── (auth)/
```

2. **为动态路由添加类型安全**

```typescript
type Props = {
  params: { slug: string };
};
```

3. **合理使用嵌套布局**

```typescript
// 根布局 → 通用布局 → 特定布局
```

4. **添加加载和错误状态**

```typescript
// loading.tsx
// error.tsx
// not-found.tsx
```

### ❌ 避免做法

1. **不要过度嵌套路由**

```
❌ app/a/b/c/d/e/page.tsx
✅ app/[category]/[id]/page.tsx
```

2. **不要在布局中使用 `use client` 除非必要**

```typescript
// ❌ 不必要的客户端布局
"use client";
export default function Layout({ children }) {
  return <div>{children}</div>;
}

// ✅ 服务端布局
export default function Layout({ children }) {
  return <div>{children}</div>;
}
```

3. **不要混淆 params 和 searchParams**

```typescript
// ❌ 错误：params 用于动态路由
type Props = {
  params: { page: string }; // 这是错误的
};

// ✅ 正确：searchParams 用于查询参数
type Props = {
  searchParams: { page?: string };
};
```

## 常见问题

**Q: 路由组和嵌套路由有什么区别？**

A:

- **路由组**：用于组织代码，不影响 URL
- **嵌套路由**：影响 URL 结构，创建 URL 层级

**Q: 什么时候使用模板而不是布局？**

A: 当你需要：

- 每次导航都重新渲染
- 重新播放动画
- 重置状态

**Q: 可以在运行时动态创建路由吗？**

A: 不可以。Next.js 路由是构建时确定的。动态内容使用动态路由处理。

**Q: 如何处理大量动态路由？**

A: 使用 `generateStaticParams` 预构建常见路由：

```typescript
export async function generateStaticParams() {
  const posts = await fetch("https://api.example.com/posts").then((res) =>
    res.json()
  );

  return posts.map((post: any) => ({
    slug: post.slug,
  }));
}
```

✅ **文件系统路由**

- 嵌套布局
- 路由特定布局
- 模板 vs 布局

✅ **加载和错误状态**

- Loading UI
- Error Boundary
- 404 页面
