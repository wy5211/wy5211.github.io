## 前言

React Server Components（RSC）是 React 18 引入的革命性特性，Next.js App Router 完全基于此构建。理解 Server Components 和 Client Components 的区别和使用场景，是构建高性能 Next.js 应用的关键。

**学习目标**：

- ✅ 理解 Server Components 和 Client Components 的区别
- ✅ 掌握 'use client' 指令的使用时机
- ✅ 学会组件组合模式
- ✅ 了解数据获取的最佳实践
- ✅ 掌握第三方库集成策略

## Server Components vs Client Components

### 核心区别

| 特性           | Server Components                  | Client Components |
| -------------- | ---------------------------------- | ----------------- |
| 渲染位置       | 服务器                             | 浏览器            |
| 默认类型       | App Router 默认                    | 需要 'use client' |
| 访问后端资源   | ✅ 可以直接访问数据库、文件系统    | ❌ 需要通过 API   |
| 使用 Hooks     | ❌ 不能使用（useState、useEffect） | ✅ 可以使用       |
| 使用浏览器 API | ❌ 不能使用（window、document）    | ✅ 可以使用       |
| 包大小         | 📦 不发送给客户端                  | 📦 发送给客户端   |
| 交互性         | ❌ 无交互性                        | ✅ 支持用户交互   |
| SEO            | ✅ 完全支持                        | ⚠️ 需要谨慎处理   |

### 渲染流程

```
Server Components 渲染流程:
┌─────────────────────────────────────────────────────────────┐
│  1. 服务器接收请求                                          │
│  2. 渲染 Server Components 为 React Element Tree           │
│  3. 直接访问数据库、文件系统等后端资源                      │
│  4. 序列化 Component Tree                                   │
│  5. 发送 HTML 和 JavaScript 给客户端                        │
└─────────────────────────────────────────────────────────────┘

Client Components 渲染流程:
┌─────────────────────────────────────────────────────────────┐
│  1. 浏览器接收 JavaScript Bundle                            │
│  2. hydrate React Tree                                      │
│  3. 渲染交互式 UI                                           │
│  4. 响应用户事件                                            │
└─────────────────────────────────────────────────────────────┘
```

### 数据获取对比

```typescript
// ❌ 传统 React SPA（Client Side）
// 所有数据都在客户端获取
function BlogPage() {
  const [posts, setPosts] = useState([]);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    fetch('/api/posts')
      .then(res => res.json())
      .then(data => {
        setPosts(data);
        setLoading(false);
      });
  }, []);

  if (loading) return <div>Loading...</div>;
  return <div>{/* 渲染 posts */}</div>;
}

// ✅ Next.js Server Components
// 数据在服务器获取，直接发送 HTML
async function BlogPage() {
  const posts = await db.posts.findMany();

  return (
    <div>
      {posts.map(post => (
        <article key={post.id}>{post.title}</article>
      ))}
    </div>
  );
}
```

**性能对比**：

| 指标              | SPA（Client）                | Server Components        |
| ----------------- | ---------------------------- | ------------------------ |
| 首屏加载          | 需要等待 JS 下载和执行       | 立即显示 HTML            |
| 数据获取          | 多次往返（HTML → JS → Data） | 一次往返（HTML + Data）  |
| JavaScript Bundle | 包含所有组件代码             | 只包含 Client Components |
| SEO               | 依赖客户端渲染               | 完整 HTML 可爬取         |

## 'use client' 指令

### 何时使用 'use client'

只有在以下情况才需要使用 'use client'：

1. **使用 React Hooks**

```typescript
"use client";

import { useState } from 'react';

export default function Counter() {
  const [count, setCount] = useState(0);
  return <button onClick={() => setCount(count + 1)}>{count}</button>;
}
```

2. **使用浏览器事件处理器**

```typescript
"use client";

export default function Form() {
  return (
    <form onSubmit={(e) => {
      e.preventDefault();
      // 处理表单提交
    }}>
      <button type="submit">提交</button>
    </form>
  );
}
```

3. **使用浏览器 API**

```typescript
"use client";

export default function Map() {
  useEffect(() => {
    const map = new window.google.maps.Map(/* ... */);
    return () => map.destroy();
  }, []);

  return <div id="map" />;
}
```

4. **使用需要 React Context 的状态管理**

```typescript
"use client";

import { useTheme } from '@/components/ThemeProvider';

export default function ThemedButton() {
  const { theme } = useTheme();
  return <button className={theme}>Click me</button>;
}
```

### 何时不需要 'use client'

以下情况**不需要**使用 'use client'：

1. **展示数据**

```typescript
// ✅ 正确：Server Component
async function UserList() {
  const users = await db.users.findMany();
  return (
    <ul>
      {users.map(user => <li key={user.id}>{user.name}</li>)}
    </ul>
  );
}
```

2. **使用链接导航**

```typescript
// ✅ 正确：Server Component
import Link from 'next/link';

export default function Navigation() {
  return (
    <nav>
      <Link href="/about">关于</Link>
      <Link href="/blog">博客</Link>
    </nav>
  );
}
```

3. **使用表单（不使用事件处理器）**

```typescript
// ✅ 正确：Server Component（使用 Server Actions）
import { addUser } from '@/app/actions';

export default function UserForm() {
  return (
    <form action={addUser}>
      <input name="name" type="text" />
      <button type="submit">添加</button>
    </form>
  );
}
```

## 组件组合模式

### 组件边界

最佳实践是将应用分解为 Server 和 Client Components：

```
Server Component（父）
├── Server Component（子）- 展示数据
├── Client Component（子）- 交互功能
│   ├── Server Component（孙）- 静态内容
│   └── Server Component（孙）- 数据展示
└── Server Component（子）- 更多数据
```

### 示例：博客文章页面

```typescript
// app/blog/[slug]/page.tsx（Server Component）
import { notFound } from 'next/navigation';
import { LikeButton } from './LikeButton';
import { CommentList } from './CommentList';
import { CommentForm } from './CommentForm';

async function getPost(slug: string) {
  const post = await db.posts.findUnique({ where: { slug } });
  return post;
}

export default async function BlogPost({ params }: { params: { slug: string } }) {
  const post = await getPost(params.slug);

  if (!post) {
    notFound();
  }

  return (
    <article>
      {/* Server Component：静态内容 */}
      <header>
        <h1>{post.title}</h1>
        <p>{post.excerpt}</p>
      </header>

      {/* Server Component：内容 */}
      <div>{post.content}</div>

      {/* Client Component：交互功能 */}
      <LikeButton postId={post.id} />

      {/* Server Component：评论列表 */}
      <CommentList postId={post.id} />

      {/* Client Component：评论表单 */}
      <CommentForm postId={post.id} />
    </article>
  );
}
```

```typescript
// components/LikeButton.tsx（Client Component）
"use client";

import { useState } from 'react';

export function LikeButton({ postId }: { postId: string }) {
  const [likes, setLikes] = useState(0);

  return (
    <button onClick={() => setLikes(likes + 1)}>
      ♥ {likes}
    </button>
  );
}
```

```typescript
// components/CommentList.tsx（Server Component）
import { db } from '@/lib/db';

export async function CommentList({ postId }: { postId: string }) {
  const comments = await db.comments.findMany({
    where: { postId },
    orderBy: { createdAt: 'desc' },
  });

  return (
    <section>
      <h2>评论 ({comments.length})</h2>
      <ul>
        {comments.map(comment => (
          <li key={comment.id}>
            <p>{comment.content}</p>
            <small>{comment.author}</small>
          </li>
        ))}
      </ul>
    </section>
  );
}
```

### 传递 Server 组件给 Client 组件

```typescript
// components/Modal.tsx（Client Component）
"use client";

import { useState } from 'react';

export function Modal({ children }: { children: React.ReactNode }) {
  const [isOpen, setIsOpen] = useState(false);

  return (
    <>
      <button onClick={() => setIsOpen(true)}>打开</button>
      {isOpen && (
        <dialog open>
          {children}
          <button onClick={() => setIsOpen(false)}>关闭</button>
        </dialog>
      )}
    </>
  );
}
```

```typescript
// app/page.tsx（Server Component）
import { Modal } from '@/components/Modal';
import { ServerContent } from '@/components/ServerContent';

export default function Page() {
  return (
    <Modal>
      {/* Server Component 作为 children 传递给 Client Component */}
      <ServerContent />
    </Modal>
  );
}
```

### 序列化限制

从 Server Component 传递给 Client Component 的数据必须是可序列化的：

```typescript
// ✅ 可序列化
export function ClientComponent({ data }: { data: { id: string; title: string } }) {
  return <div>{data.title}</div>;
}

// ❌ 不可序列化（函数、Date、Map、Set 等）
export function ClientComponent({
  onClick,  // ❌ 函数
  date,     // ❌ Date 对象
  map,      // ❌ Map
}) {
  return <div>...</div>;
}
```

## 数据获取最佳实践

### Server Components 中获取数据

```typescript
// ✅ 推荐：在 Server Component 中直接获取数据
async function BlogPage() {
  const posts = await db.posts.findMany();
  return <PostList posts={posts} />;
}

// ✅ 推荐：并行获取多个数据
async function Dashboard() {
  const [posts, comments, users] = await Promise.all([
    db.posts.findMany(),
    db.comments.findMany(),
    db.users.findMany(),
  ]);

  return (
    <div>
      <PostList posts={posts} />
      <CommentList comments={comments} />
      <UserList users={users} />
    </div>
  );
}
```

### 使用 fetch 缓存

```typescript
// ✅ 默认缓存（SSG）
async function Posts() {
  const posts = await fetch('https://api.example.com/posts');
  const data = await posts.json();
  return <PostList posts={data} />;
}

// ✅ 不缓存（SSR）
async function RealTimeData() {
  const data = await fetch('https://api.example.com/data', {
    cache: 'no-store',
  }).then(res => res.json());
  return <DataDisplay data={data} />;
}

// ✅ 定期重新验证（ISR）
async function Posts() {
  const posts = await fetch('https://api.example.com/posts', {
    next: { revalidate: 3600 }, // 1 小时
  }).then(res => res.json());
  return <PostList posts={posts} />;
}

// ✅ 按需重新验证
async function PostPage({ params }: { params: { slug: string } }) {
  const post = await fetch(`https://api.example.com/posts/${params.slug}`, {
    next: { tags: [`post-${params.slug}`] },
  }).then(res => res.json());

  return <PostDetail post={post} />;
}
```

### 数据获取层

创建数据获取层以提高代码可维护性：

```typescript
// lib/posts.ts
import { db } from "@/lib/db";
import { cache } from "react";

// 使用 cache 避免重复请求
export const getPosts = cache(async () => {
  return db.posts.findMany();
});

export const getPostBySlug = cache(async (slug: string) => {
  return db.posts.findUnique({ where: { slug } });
});

export const getPostsWithAuthor = cache(async () => {
  return db.posts.findMany({
    include: { author: true },
  });
});
```

```typescript
// app/blog/page.tsx
import { getPostsWithAuthor } from '@/lib/posts';

export default async function BlogPage() {
  const posts = await getPostsWithAuthor();

  return (

        <article key={post.id}>
          <h2>{post.title}</h2>
          <p>作者: {post.author.name}</p>
        </article>

    </div>
  );
}
```

## 第三方库集成策略

### 选择支持库

| 库类型    | Server Components                    | Client Components   |
| --------- | ------------------------------------ | ------------------- |
| UI 组件库 | ✅ Mantine、shadcn/ui                | ✅ 所有库           |
| 表单      | ✅ React Hook Form（Server Actions） | ✅ 所有库           |
| 数据获取  | ✅ 直接使用 fetch/ORM                | ✅ SWR、React Query |
| 状态管理  | ❌ 不需要                            | ✅ Zustand、Jotai   |
| 样式      | ✅ Tailwind、CSS Modules             | ✅ 所有库           |

### UI 库集成

```typescript
// ✅ 使用支持 Server Components 的库
import { Button } from '@/components/ui/button';

export default function ServerComponent() {
  return <Button>点击</Button>;

// ✅ 将客户端功能的 UI 包装为 Client Component
// components/DatePicker.tsx
"use client";

import { useState } from 'react';

export function DatePicker() {
  const [date, setDate] = useState<Date>();

    <DayPicker
      mode="single"
      selected={date}
      onSelect={setDate}
    />

}
```

```typescript
"use client";

import { useFormState, useFormStatus } from 'react-dom';
import { createPost } from '@/app/actions';

export function PostForm() {

  return (

      {pending ? '提交中...' : '提交'}
    </button>
  );
}
```

## 常见陷阱

### 陷阱 1：过度使用 'use client'

```typescript
// ❌ 错误：不需要交互也使用 'use client'
"use client";

export function StaticContent() {
  return <div>这是静态内容</div>;
}

// ✅ 正确：默认为 Server Component
export function StaticContent() {
  return <div>这是静态内容</div>;
}
```

### 陷阱 2：在 Server Component 中使用 Hooks

```typescript
// ❌ 错误：Server Component 中不能使用 Hooks
export default function Page() {
  const [count, setCount] = useState(0); // Error!
  return <div>{count}</div>;
}

// ✅ 正确：使用 Client Component
"use client";

export default function Counter() {
  const [count, setCount] = useState(0);
  return <div>{count}</div>;
}
```

### 陷阱 3：在 Server Component 中直接使用浏览器 API

```typescript
// ❌ 错误：window 未定义
export default function Page() {
  console.log(window.innerWidth); // Error!
  return <div>Hello</div>;
}

// ✅ 正确：在 Client Component 中使用
"use client";

export function WindowWidth() {
  const [width, setWidth] = useState(0);

  useEffect(() => {
    setWidth(window.innerWidth);
  }, []);

  return <div>宽度: {width}</div>;
}
```

### 陷阱 4：传递不可序列化的数据

```typescript
// ❌ 错误：函数不能传递给 Client Component
"use client";

export function ClientComponent({ handler }: { handler: () => void }) {
  return <button onClick={handler}>点击</button>;
}

export default function ServerComponent() {
  return (
    <ClientComponent
      handler={() => console.log('clicked')} // Error!
    />
  );
}

// ✅ 正确：在 Client Component 内部定义
"use client";

export function ClientComponent() {
  return (
    <button onClick={() => console.log('clicked')}>
      点击
    </button>
  );
}
```

## 最佳实践

### ✅ 推荐做法

1. **默认使用 Server Components**

```typescript
// ✅ 默认就是 Server Component

  return <div>Content</div>;
}
```

2. **只在必要时使用 Client Components**

```typescript
// ✅ 只在需要交互时添加

export function InteractiveButton() {
  const [count, setCount] = useState(0);
  return <button onClick={() => setCount(count + 1)}>{count}</button>;
}
```

3. **将交互逻辑隔离**

```typescript
// ✅ 将交互部分提取为 Client Component
export default function ServerComponent() {
  return (
    <div>
      <StaticContent />
      <InteractivePart />
    </div>
  );
}
```

4. **使用 Server Components 获取数据**

```typescript
// ✅ 在 Server Component 中获取数据
async function Page() {
  const data = await fetchData();
  return <DataDisplay data={data} />;

```

1. **不要为整个应用添加 'use client'**

```typescript
// ❌ 避免
"use client";

export default function App() {
  // 整个应用都是客户端

```

2. **不要在 Server Component 中使用浏览器 API**


3. **不要过度使用 Client Components**

## 常见问题

**Q: 如何决定使用 Server 还是 Client Component？**

A: 问自己以下问题：

- 需要使用 React Hooks 吗？ → Client
- 需要用户交互（onClick、onChange）吗？ → Client
- 需要访问浏览器 API 吗？ → Client
- 只是展示数据？ → Server
- 需要访问数据库？ → Server

**Q: Server Components 可以嵌套 Client Components 吗？**

A: 可以。Server Components 可以渲染 Client Components，Client Components 也可以渲染 Server Components（通过 children）。

**Q: Client Components 的性能如何？**

A: Client Components 与传统 React 组件性能相同。关键是减少不必要的 Client Components，让更多内容在服务器渲染。

**Q: 如何调试组件是 Server 还是 Client？**

A: 使用 DevTools 或在组件中添加 `console.log`。Server Component 的日志在服务器控制台，Client Component 的日志在浏览器控制台。
