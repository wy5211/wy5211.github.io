## Vercel 部署

### 连接 Git 仓库

1. **访问 Vercel Dashboard**
   - 登录 https://vercel.com
   - 点击 "Add New Project"

2. **导入 Git 仓库**
   - 选择 GitHub、GitLab 或 Bitbucket
   - 选择你的 Next.js 项目仓库

3. **配置项目**

   ```bash
   # Project Name: my-blog
   # Framework Preset: Next.js
   # Root Directory: ./
   # Build Command: npm run build
   # Output Directory: (默认为空，Next.js 会自动处理)
   ```

4. **部署**
   - 点击 "Deploy"
   - Vercel 会自动检测配置并部署

### 环境变量配置

```typescript
// .env.local（本地开发）
DATABASE_URL = "postgresql://localhost:5432/mydb";
NEXTAUTH_SECRET = "local-secret";
NEXTAUTH_URL = "http://localhost:3000";
```

在 Vercel Dashboard 中添加环境变量：

1. 进入项目 Settings → Environment Variables
2. 添加生产环境变量：

```env
DATABASE_URL="postgresql://production-db"
NEXTAUTH_SECRET="production-secret"
NEXTAUTH_URL="https://your-domain.com"
GITHUB_CLIENT_ID="your-github-client-id"
GITHUB_CLIENT_SECRET="your-github-client-secret"
```

### 自定义域名

1. **在 Vercel Dashboard**
   - Settings → Domains
   - 添加自定义域名

2. **配置 DNS**

   ```
   Type: CNAME
   Name: www
   Value: cname.vercel-dns.com
   ```

3. **自动 HTTPS**
   - Vercel 自动为自定义域名配置 SSL 证书

## CI/CD 配置

### GitHub Actions

```yaml
# .github/workflows/ci.yml
name: CI

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  lint-and-test:
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: "20"
          cache: "npm"

      - name: Install dependencies
        run: npm ci

      - name: Run linter
        run: npm run lint

      - name: Run type check
        run: npm run type-check

      - name: Run tests
        run: npm run test

      - name: Build
        run: npm run build
```

### 自动部署到 Vercel

Vercel 默认配置：

- 推送到 `main` 分支 → 生产环境
- 推送到其他分支 → 预览环境

自定义部署触发器：

```yaml
# .github/workflows/deploy-preview.yml
name: Deploy Preview

on:
  pull_request:
    types: [opened, synchronize]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Deploy to Vercel
        uses: amondnet/vercel-action@v25
        with:
          vercel-token: ${{ secrets.VERCEL_TOKEN }}
          vercel-org-id: ${{ secrets.VERCEL_ORG_ID }}
          vercel-project-id: ${{ secrets.VERCEL_PROJECT_ID }}
          vercel-args: "--preview"
```

### 错误边界

```typescript
// app/error.tsx
"use client";

import { useEffect } from 'react';
import * as Sentry from '@sentry/nextjs';

export default function Error({
  error,
  reset,
}: {
  error: Error;
  reset: () => void;
}) {
  useEffect(() => {
    Sentry.captureException(error);
  }, [error]);

  return (
    <div>
      <h2>出错了！</h2>
      <button onClick={reset}>重试</button>
    </div>
  );
}
```

## 日志管理

### 结构化日志

```typescript
// lib/logger.ts
export function log(
  level: "info" | "warn" | "error",
  message: string,
  meta?: any
) {
  const logEntry = {
    timestamp: new Date().toISOString(),
    level,
    message,
    ...meta,
  };

  // 开发环境：输出到控制台
  if (process.env.NODE_ENV === "development") {
    console.log(JSON.stringify(logEntry, null, 2));
  }

  // 生产环境：发送到日志服务
  if (process.env.NODE_ENV === "production") {
    fetch("/api/logs", {
      method: "POST",
      body: JSON.stringify(logEntry),
    });
  }
}
```

### ❌ 避免做法

1. **不要在代码中硬编码敏感信息**

```typescript
// ❌ 错误
const apiKey = "sk-1234567890";

// ✅ 正确
const apiKey = process.env.API_KEY;
```

2. **不要忽略错误**

```typescript
// ❌ 错误
try {
  await riskyOperation();
} catch (error) {
  // 忽略错误
}

// ✅ 正确
try {
  await riskyOperation();
} catch (error) {
  Sentry.captureException(error);
}
```

3. **不要在生产环境输出详细日志**

```typescript
// ❌ 错误
console.log("用户数据:", sensitiveData);

// ✅ 正确
if (process.env.NODE_ENV === "development") {
  console.log("用户数据:", sensitiveData);
}
```

## 常见问题

**Q: Vercel 免费版有什么限制？**

A:

- 100GB 带宽/月
- 1000 次构建/月
- 10000 次函数调用/天
- 500 次数据库连接/月

**Q: 如何选择部署平台？**

A:

- **Vercel**：最佳 Next.js 体验，推荐
- **Netlify**：功能类似，也不错
- **Railway**：适合全栈应用
- **自托管**：完全控制，但需要维护

**Q: 如何处理环境变量？**

A:

- 开发环境：`.env.local`
- 生产环境：Vercel Dashboard
- 使用 `@t3-oss/env-nextjs` 管理环境变量
