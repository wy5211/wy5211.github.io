## 前言

欢迎来到 Next.js 的世界！如果你已经熟悉 React 基础，那么 Next.js 将是你构建现代 Web 应用的理想选择。

**为什么选择 Next.js？**

传统的 React 单页应用（SPA）虽然强大，但在 SEO、首屏加载性能和用户体验方面存在一些固有的限制。Next.js 作为 React 的生产级框架，通过服务端渲染（SSR）、静态站点生成（SSG）等技术在保留 React 开发体验的同时，解决了这些痛点。

**本系列学习目标**：

- ✅ 掌握 Next.js App Router 的核心概念
- ✅ 理解 Server Components 和 Client Components 的区别
- ✅ 学会使用 Server Actions 处理数据修改
- ✅ 构建类型安全的全栈应用
- ✅ 实现完整的认证授权系统
- ✅ 掌握性能优化和部署策略

## Next.js 核心概念

### Next.js vs React SPA

让我们先理解传统 React SPA 和 Next.js 的区别：

```
┌─────────────────────────────────────────────────────────────────┐
│                     传统 React SPA                              │
├─────────────────────────────────────────────────────────────────┤
│  1. 加载 HTML（空壳）                                            │
│  2. 下载 JavaScript Bundle（通常很大）                           │
│  3. 执行 JavaScript，渲染页面                                    │
│  4. 用户看到内容（白屏时间长）                                   │
│                                                                 │
│  ❌ SEO 不友好（爬虫看到的是空壳）                               │
│  ❌ 首屏加载慢                                                   │
│  ❌ JavaScript Bundle 大                                        │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│                     Next.js 应用                                │
├─────────────────────────────────────────────────────────────────┤
│  1. 服务端渲染 HTML（内容已填充）                                │
│  2. 返回完整 HTML 给浏览器                                       │
│  3. 用户立即看到内容                                             │
│  4. JavaScript hydrate（增强交互）                               │
│                                                                 │
│  ✅ SEO 友好（爬虫看到完整内容）                                 │
│  ✅ 首屏加载快                                                   │
│  ✅ 更好的用户体验                                               │
└─────────────────────────────────────────────────────────────────┘
```

### 三大渲染模式

Next.js 提供三种渲染模式，每种模式适用于不同的场景：

#### 1. 服务端渲染（SSR - Server-Side Rendering）

每次请求都在服务器上动态生成 HTML：

```typescript
// SSR 示例
async function BlogPage() {
  // 每次请求都会重新获取数据
  const posts = await db.posts.findMany();

  return (
    <div>
      {posts.map(post => (
        <article key={post.id}>{post.title}</article>
      ))}
    </div>
  );
}
```

**适用场景**：

- 需要实时数据（新闻、股票价格）
- 个性化内容（用户仪表板）
- SEO 关键页面

#### 2. 静态站点生成（SSG - Static Site Generation）

**构建时**生成 HTML，并缓存在 CDN 上：

```typescript
// SSG 示例
export const revalidate = 3600; // 每小时重新验证

async function BlogPage() {
  // 构建时获取数据，并缓存
  const posts = await db.posts.findMany();

  return (
    <div>
      {posts.map(post => (
        <article key={post.id}>{post.title}</article>
      ))}
    </div>
  );
}
```

**适用场景**：

- 内容不经常变化（博客、文档）
- 追求最佳性能
- 高流量站点

#### 3. 增量静态再生（ISR - Incremental Static Regeneration）

结合 SSG 和 SSR 的优势：

```typescript
// ISR 示例
export const revalidate = 600; // 10 分钟后重新生成

async function BlogPage() {
  const posts = await db.posts.findMany();

  return (
    <div>
      {posts.map(post => (
        <article key={post.id}>{post.title}</article>
      ))}
    </div>
  );
}
```

**适用场景**：

- 内容更新有规律（每小时、每天）
- 需要静态性能，但不能完全重新构建
- 电商产品页面

### App Router vs Pages Router

Next.js 有两套路由系统：

| 特性              | Pages Router（旧版）                   | App Router（新版，推荐） |
| ----------------- | -------------------------------------- | ------------------------ |
| 目录位置          | `pages/`                               | `app/`                   |
| 布局系统          | 需要自定义 `_app.js`                   | 内置布局系统             |
| 数据获取          | `getServerSideProps`、`getStaticProps` | 直接在组件中 `async`     |
| Server Components | 不支持                                 | 完全支持                 |
| Streaming         | 有限支持                               | 原生支持                 |
| 文件约定          | 较复杂                                 | 更直观                   |

**本系列使用 App Router**，它是 Next.js 13+ 的推荐方式，基于 React Server Components 构建。

## 环境搭建

### 第一步：创建项目

使用 Next.js 16.1.6 创建新项目：

```bash
# 使用 npm
npm create next-app@latest my-blog --typescript --tailwind --app

# 或使用 pnpm（推荐）
pnpm create next-app my-blog --typescript --tailwind --app

# 或使用 bun（最快）
bun create next-app my-blog --typescript --tailwind --app
```

**交互式选项**：

```
✔ Would you like to use TypeScript? Yes
✔ Would you like to use ESLint? Yes
✔ Would you like to use Tailwind CSS? Yes
✔ Would you like to use `src/` directory? Yes
✔ Would you like to use App Router? Yes
✔ Would you like to customize the default import alias (@/*)? No
```

### 第二步：项目结构

创建完成后，项目结构如下：

```
my-blog/
├── src/
│   └── app/
│       ├── layout.tsx          # 根布局
│       ├── page.tsx            # 首页
│       ├── globals.css         # 全局样式
│       └── favicon.ico
├── public/                     # 静态资源
├── next.config.ts             # Next.js 配置
├── tailwind.config.ts         # Tailwind 配置
├── tsconfig.json              # TypeScript 配置
└── package.json
```

### 第三步：启动开发服务器

```bash
cd my-blog
pnpm dev
```

访问 `http://localhost:3000`，你将看到 Next.js 欢迎页面。

## App Router 基础

### 文件系统路由

App Router 使用文件系统来定义路由：

```
app/
├── layout.tsx           → https://example.com/
├── page.tsx             → https://example.com/
├── about/
│   └── page.tsx         → https://example.com/about
├── blog/
│   ├── page.tsx         → https://example.com/blog
│   └── [slug]/          # 动态路由
│       └── page.tsx     → https://example.com/blog/hello-world
└── api/
    └── hello/
        └── route.ts     → https://example.com/api/hello
```

### 布局系统

布局（Layout）是多个页面共享的 UI：

```typescript
// src/app/layout.tsx
import type { Metadata } from "next";

export const metadata: Metadata = {
  title: "我的博客",
  description: "使用 Next.js 构建的博客",
};

export default function RootLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <html lang="zh-CN">
      <body>
        <nav>
          <a href="/">首页</a>
          <a href="/about">关于</a>
          <a href="/blog">博客</a>
        </nav>
        <main>{children}</main>
        <footer>
          <p>© 2026 我的博客</p>
        </footer>
      </body>
    </html>
  );
}
```

### 嵌套布局

可以在子目录中创建嵌套布局：

```
app/
├── layout.tsx           # 根布局
├── page.tsx
└── blog/
    ├── layout.tsx       # 博客专用布局
    └── page.tsx
```

```typescript
// src/app/blog/layout.tsx
export default function BlogLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <div className="blog-layout">
      <aside>
        <h2>博客分类</h2>
        <ul>
          <li><a href="/blog/tech">技术</a></li>
          <li><a href="/blog/life">生活</a></li>
        </ul>
      </aside>
      <div className="content">
        {children}
      </div>
    </div>
  );
}
```

## 数据获取基础

### 使用 fetch

在 Server Components 中直接使用 `fetch`：

```typescript
// src/app/blog/page.tsx
async function getPosts() {
  const res = await fetch('https://api.example.com/posts', {
    cache: 'force-cache', // 或 'no-store'
  });
  if (!res.ok) throw new Error('Failed to fetch');
  return res.json();
}

export default async function BlogPage() {
  const posts = await getPosts();

  return (
    <div>
      <h1>博客文章</h1>
      {posts.map((post: any) => (
        <article key={post.id}>
          <h2>{post.title}</h2>
          <p>{post.excerpt}</p>
        </article>
      ))}
    </div>
  );
}
```

### 缓存配置

```typescript
// 默认缓存（SSG）
const data = await fetch("https://api.example.com/data");

// 不缓存（SSR）
const data = await fetch("https://api.example.com/data", {
  cache: "no-store",
});

// 重新验证（ISR）
const data = await fetch("https://api.example.com/data", {
  next: { revalidate: 3600 }, // 1 小时
});
```

## 最佳实践

### ✅ 推荐做法

1. **使用 Server Components 作为默认**

```typescript
// ✅ 默认使用 Server Component
export default async function Page() {
  const data = await fetchData();
  return <div>{data.title}</div>;
}
```

2. **只在需要时使用 Client Components**

```typescript
// ✅ 只在需要交互时添加 'use client'
"use client";

export default function Counter() {
  const [count, setCount] = useState(0);
  return <button onClick={() => setCount(count + 1)}>{count}</button>;
}
```

3. **合理使用缓存**

```typescript
// ✅ 静态数据使用缓存
const posts = await fetch("/api/posts", {
  next: { revalidate: 3600 },
});
```

### ❌ 避免做法

1. **不要在 Server Component 中使用 hooks**

```typescript
// ❌ 错误：Server Component 中不能使用 useState
export default function Page() {
  const [count, setCount] = useState(0); // Error!
  return <div>{count}</div>;
}
```

2. **不要过度使用 Client Components**

```typescript
// ❌ 不必要的 Client Component
"use client"; // 如果没有交互，不需要这个

export default function StaticContent() {
  return <div>这是静态内容</div>;
}
```

3. **不要在 Server Component 中直接使用浏览器 API**

```typescript
// ❌ 错误：window 未定义
export default function Page() {
  console.log(window.innerWidth); // Error!
  return <div>Hello</div>;
}
```

## 常见问题

**Q: Next.js 适合什么类型的项目？**

A: Next.js 适合几乎所有 Web 应用：

- 内容网站（博客、文档）
- 电商应用
- SaaS 产品
- 企业应用

**Q: App Router 和 Pages Router 可以混用吗？**

A: 可以，但不推荐。建议选择一种并坚持使用。新项目应该使用 App Router。

**Q: 如何选择渲染模式？**

A:

- **SSG**：内容不经常变化
- **SSR**：需要实时或个性化数据
- **ISR**：介于两者之间

**Q: Next.js 学习曲线陡峭吗？**

A: 如果你有 React 基础，学习 Next.js 会很平滑。本系列将循序渐进地介绍所有概念。
