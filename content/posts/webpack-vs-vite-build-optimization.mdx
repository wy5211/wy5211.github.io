# 前端构建优化：Webpack 与 Vite 对比实战

构建工具直接影响开发体验和打包效率。Webpack 是老牌霸主，Vite 是后起之秀。本文深入对比两者，帮你做出选择。

## 一、核心原理对比

### 1.1 Webpack：打包构建

```
源码 → 依赖分析 → 打包 → Bundle → 浏览器
```

**特点：**

- 启动时分析所有依赖
- 编译所有模块
- 生成 Bundle 文件
- 使用 Babel 转译

### 1.2 Vite：原生 ESM

```
源码 → 浏览器请求 → 按需编译 → 浏览器
```

**特点：**

- 开发环境不打包
- 利用浏览器原生 ESM
- 使用 esbuild（Go 语言）预构建依赖
- 极速冷启动

### 1.3 启动速度对比

| 项目规模 | Webpack | Vite   |
| -------- | ------- | ------ |
| 小型项目 | 3-5s    | &lt;1s |
| 中型项目 | 10-20s  | 1-2s   |
| 大型项目 | 30-60s  | 2-4s   |

## 二、Webpack 优化实战

### 2.1 基础配置

```javascript
// webpack.config.js
const path = require("path");
const HtmlWebpackPlugin = require("html-webpack-plugin");

module.exports = {
  entry: "./src/index.js",
  output: {
    path: path.resolve(__dirname, "dist"),
    filename: "[name].[contenthash].js",
    clean: true,
  },
  module: {
    rules: [
      {
        test: /\.js$/,
        exclude: /node_modules/,
        use: {
          loader: "babel-loader",
          options: {
            cacheDirectory: true, // 开启缓存
          },
        },
      },
      {

        use: ["style-loader", "css-loader"],
      },
    ],
  },
  plugins: [
    new HtmlWebpackPlugin({
      template: "./public/index.html",
    }),
  ],
};
```

### 2.2 构建速度优化

```javascript
// 1. 缩小搜索范围
module.exports = {
  module: {
    rules: [
      {
        test: /\.js$/,
        include: path.resolve(__dirname, "src"), // 只处理 src
        exclude: /node_modules/,
        use: "babel-loader",
      },
    ],
  },

  // 2. resolve 优化
  resolve: {
    extensions: [".js", ".jsx"], // 减少后缀尝试
    alias: {
      "@": path.resolve(__dirname, "src"),
    },
    modules: ["node_modules"], // 指定模块目录
    mainFiles: ["index"], // 入口文件名
  },
};

// 3. 多线程编译
const ThreadLoader = require("thread-loader");

module.exports = {
  module: {
    rules: [
      {
        test: /\.js$/,
        use: [
          {
            loader: "thread-loader",
            options: {
              workers: require("os").cpus().length - 1,
            },
          },
          "babel-loader",
        ],
      },
    ],
  },
};

// 4. 持久化缓存
module.exports = {
  cache: {
    type: "filesystem",
    buildDependencies: {
      config: [__filename],
    },
  },
};
```

### 2.3 产物优化

```javascript
const MiniCssExtractPlugin = require("mini-css-extract-plugin");
const CssMinimizerPlugin = require("css-minimizer-webpack-plugin");
const TerserPlugin = require("terser-webpack-plugin");

module.exports = {
  // 1. 代码分割
  optimization: {
    splitChunks: {
      chunks: "all",
      cacheGroups: {
        vendor: {
          test: /[\\/]node_modules[\\/]/,
          name: "vendors",
          chunks: "all",
        },
        common: {
          minChunks: 2,
          priority: -10,
          reuseExistingChunk: true,
        },
      },
    },

    // 2. 压缩
    minimize: true,
    minimizer: [
      new TerserPlugin({
        parallel: true,
        terserOptions: {
          compress: {
            drop_console: true, // 移除 console
          },
        },
      }),
      new CssMinimizerPlugin(),
    ],

    // 3. Tree Shaking
    usedExports: true,
    sideEffects: true,
  },

  plugins: [
    // 4. CSS 提取
    new MiniCssExtractPlugin({
      filename: "[name].[contenthash].css",
    }),
  ],
};
```

## 三、Vite 优化实战

### 3.1 基础配置

```javascript
// vite.config.js
import { defineConfig } from "vite";
import react from "@vitejs/plugin-react";
import path from "path";

export default defineConfig({
  plugins: [react()],

  resolve: {
    alias: {
      "@": path.resolve(__dirname, "src"),
    },
  },

  server: {
    port: 3000,
    open: true,
    cors: true,
  },
});
```

### 3.2 构建优化

```javascript
import { defineConfig } from "vite";
import react from "@vitejs/plugin-react";

export default defineConfig({
  plugins: [react()],

  build: {
    // 1. 代码分割
    rollupOptions: {
      output: {
        manualChunks: {
          vendor: ["react", "react-dom"],
          utils: ["lodash", "axios"],
        },
      },
    },

    // 2. 压缩配置
    minify: "terser",
    terserOptions: {
      compress: {
        drop_console: true,
        drop_debugger: true,
      },
    },

    // 3. chunk 大小警告阈值
    chunkSizeWarningLimit: 1000,

    // 4. CSS 代码分割
    cssCodeSplit: true,

    // 5. 构建目录
    outDir: "dist",
    assetsDir: "assets",
  },
});
```

### 3.3 预构建优化

```javascript
export default defineConfig({
  optimizeDeps: {
    // 1. 预构建包含
    include: ["react", "react-dom", "axios"],

    // 2. 预构建排除
    exclude: ["your-linked-package"],

    // 3. 强制重新构建
    force: false,
  },
});
```

### 3.4 插件推荐

```javascript
import { defineConfig } from "vite";
import react from "@vitejs/plugin-react";
import svgr from "vite-plugin-svgr";
import compression from "vite-plugin-compression";
import visualizer from "rollup-plugin-visualizer";

export default defineConfig({
  plugins: [
    react(),

    // SVG 转 React 组件
    svgr(),

    // Gzip 压缩
    compression({
      algorithm: "gzip",
      threshold: 10240, // >10KB 才压缩
    }),

    // 包分析
    visualizer({
      open: true,
      gzipSize: true,
    }),
  ],
});
```

## 四、迁移指南

### 4.1 从 Webpack 迁移到 Vite

**1. 入口文件调整**

```html
<!-- index.html 移到根目录 -->
<!DOCTYPE html>
<html>
  <head>
    <meta charset="UTF-8" />
    <title>App</title>
  </head>
  <body>
    <div id="root"></div>
    <!-- 直接引入入口脚本 -->
    <script type="module" src="/src/main.jsx"></script>
  </body>
</html>
```

**2. 环境变量**

```javascript
// Webpack
process.env.NODE_ENV;
process.env.API_URL;

// Vite
import.meta.env.MODE;
import.meta.env.VITE_API_URL; // 必须以 VITE_ 开头
```

**3. 动态导入**

```javascript
// Webpack
require.context("./dir", true, /\.js$/);

// Vite
const modules = import.meta.glob("./dir/**/*.js");
```

**4. CommonJS 转换**

```javascript
// Webpack (支持 CommonJS)
const lodash = require("lodash");

// Vite (推荐 ESM)
import lodash from "lodash";
```

### 4.2 兼容性处理

## 六、选择建议

### 6.1 选 Webpack

- 大型复杂项目
- 需要丰富的 loader/plugin
- 团队熟悉 Webpack
- 需要微前端等高级功能
- 需要兼容旧浏览器（IE）

### 6.2 选 Vite

- 新项目
- 追求开发体验
- 中小型项目
- 使用现代浏览器
- React/Vue 官方模板支持

## 总结

Webpack 和 Vite 各有优势。

**核心对比：**
| 特性 | Webpack | Vite |
|------|---------|------|
| 开发启动 | 慢 | 极快 |
| 配置复杂度 | 高 | 低 |
| 生态成熟度 | 高 | 中 |
| 学习曲线 | 陡峭 | 平缓 |

**迁移建议：**

- 新项目：直接用 Vite
- 老项目：不痛就不要改
- 大型项目：Webpack 仍可靠

工具只是手段，关键是写出好代码。
