Prisma 是新一代的 Node.js ORM 工具，通过类型安全的查询构建器和强大的开发工具链，彻底改变了我们与数据库交互的方式。相比传统的 TypeORM 或 Sequelize，Prisma 提供了更好的开发体验和更强大的类型推导能力。

## 为什么选择 Prisma

### 传统 ORM 的痛点

在 Prisma 出现之前，Node.js 开发者通常面临以下问题：

- **类型安全差**：运行时才能发现查询错误
- **API 复杂**：需要记忆大量装饰器和配置
- **调试困难**：生成的 SQL 难以追踪
- **迁移混乱**：数据库版本控制不直观

### Prisma 的优势

Prisma 通过三个核心组件解决了这些问题：

1. **Prisma Schema**：声明式的数据模型定义
2. **Prisma Client**：自动生成的类型安全查询 API
3. **Prisma Migrate**：强大的数据库迁移系统

```bash
# 安装 Prisma
npm install prisma --save-dev
npm install @prisma/client

# 初始化 Prisma
npx prisma init
```

## Schema 设计基础

### 基本模型定义

Prisma Schema 使用 PSL（Prisma Schema Language）语法定义数据模型：

```prisma
// prisma/schema.prisma

datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

generator client {
  provider = "prisma-client-js"
}

model User {
  id        Int      @id @default(autoincrement())
  email     String   @unique
  name      String?
  avatar    String?
  status    Status   @default(ACTIVE)
  posts     Post[]
  comments  Comment[]
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt

  @@map("users")
}

model Post {
  id          Int       @id @default(autoincrement())
  title       String
  content     String?   @db.Text
  published   Boolean   @default(false)
  viewCount   Int       @default(0)
  authorId    Int
  author      User      @relation(fields: [authorId], references: [id])
  comments    Comment[]
  tags        Tag[]
  createdAt   DateTime  @default(now())
  updatedAt   DateTime  @updatedAt

  @@index([authorId])
  @@index([published])
  @@map("posts")
}

model Comment {
  id        Int      @id @default(autoincrement())
  content   String   @db.Text
  postId    Int
  post      Post     @relation(fields: [postId], references: [id])
  userId    Int
  user      User     @relation(fields: [userId], references: [id])
  createdAt DateTime @default(now())

  @@index([postId])
  @@index([userId])
  @@map("comments")
}

model Tag {
  id    Int    @id @default(autoincrement())
  name  String @unique
  posts Post[]

  @@map("tags")
}

enum Status {
  ACTIVE
  INACTIVE
  BANNED
}
```

### 字段类型和属性

Prisma 支持丰富的字段类型和属性：

```prisma
model Example {
  // 主键
  id        Int     @id @default(autoincrement())
  uuid      String  @id @default(uuid())
  customId  String  @id

  // 可选字段
  nickname  String?

  // 唯一约束
  email     String  @unique

  // 默认值
  role      String  @default("user")
  count     Int     @default(0)
  createdAt DateTime @default(now())

  // 数据库特定类型
  bio       String  @db.Text
  price     Decimal @db.Decimal(10, 2)

  // 自动更新
  updatedAt DateTime @updatedAt

  // 索引
  @@index([email])
  @@unique([email, nickname])

  // 表名映射
  @@map("examples")
}
```

## 关系映射

### 一对一关系

```prisma
model User {
  id      Int      @id @default(autoincrement())
  email   String   @unique
  profile Profile?
}

model Profile {
  id     Int     @id @default(autoincrement())
  bio    String?
  userId Int     @unique
  user   User    @relation(fields: [userId], references: [id])
}
```

查询一对一关系：

```typescript
// 包含关联数据
const userWithProfile = await prisma.user.findUnique({
  where: { email: "test@example.com" },
  include: { profile: true },
});

// 只选择特定字段
const userWithBio = await prisma.user.findUnique({
  where: { email: "test@example.com" },
  select: {
    email: true,
    profile: {
      select: { bio: true },
    },
  },
});
```

### 一对多关系

```prisma
model User {
  id     Int     @id @default(autoincrement())
  posts  Post[]
}

model Post {
  id       Int  @id @default(autoincrement())
  authorId Int
  author   User @relation(fields: [authorId], references: [id])
}
```

查询一对多关系：

```typescript
// 获取用户及其所有文章
const userWithPosts = await prisma.user.findUnique({
  where: { id: 1 },
  include: {
    posts: {
      where: { published: true },
      orderBy: { createdAt: "desc" },
      take: 10,
    },
  },
});

// 嵌套创建
const newUser = await prisma.user.create({
  data: {
    email: "new@example.com",
    posts: {
      create: [
        { title: "First Post", content: "Hello World" },
        { title: "Second Post", content: "Another one" },
      ],
    },
  },
});
```

### 多对多关系

```prisma
model Post {
  id    Int    @id @default(autoincrement())
  tags  Tag[]  // 隐式多对多
}

model Tag {
  id    Int    @id @default(autoincrement())
  posts Post[]
}
```

或者显式定义中间表（需要额外字段时）：

```prisma
model Post {
  id          Int               @id @default(autoincrement())
  taggings    PostTagging[]
  tags        Tag[]             @relation("PostToTag")
}

model Tag {
  id       Int           @id @default(autoincrement())
  taggings PostTagging[]
  posts    Post[]        @relation("PostToTag")
}

model PostTagging {
  postId    Int
  tagId     Int
  createdAt DateTime @default(now())

  post      Post     @relation(fields: [postId], references: [id])
  tag       Tag      @relation(fields: [tagId], references: [id])

  @@id([postId, tagId])
}
```

多对多查询和操作：

```typescript
// 连接现有标签
const updatedPost = await prisma.post.update({
  where: { id: 1 },
  data: {
    tags: {
      connect: [{ id: 1 }, { id: 2 }],
    },
  },
});

// 创建并连接
const newPost = await prisma.post.create({
  data: {
    title: "Tagged Post",
    tags: {
      connectOrCreate: [
        {
          where: { name: "TypeScript" },
          create: { name: "TypeScript" },
        },
      ],
    },
  },
});

// 断开连接
await prisma.post.update({
  where: { id: 1 },
  data: {
    tags: {
      disconnect: [{ id: 1 }],
    },
  },
});
```

## 查询操作详解

### 基础 CRUD

```typescript
// 创建
const user = await prisma.user.create({
  data: {
    email: "test@example.com",
    name: "Test User",
  },
});

// 批量创建
const users = await prisma.user.createMany({
  data: [{ email: "user1@example.com" }, { email: "user2@example.com" }],
  skipDuplicates: true,
});

// 查询单条
const user = await prisma.user.findUnique({
  where: { email: "test@example.com" },
});

// 或通过唯一字段
const user = await prisma.user.findUniqueOrThrow({
  where: { id: 1 },
});

// 查询多条
const users = await prisma.user.findMany({
  where: { status: "ACTIVE" },
  select: { id: true, email: true, name: true },
});

// 更新
const updated = await prisma.user.update({
  where: { id: 1 },
  data: { name: "New Name" },
});

// 批量更新
const updatedMany = await prisma.user.updateMany({
  where: { status: "INACTIVE" },
  data: { status: "ACTIVE" },
});

// 删除
await prisma.user.delete({
  where: { id: 1 },
});

// 批量删除
await prisma.user.deleteMany({
  where: { status: "BANNED" },
});
```

### 高级查询条件

```typescript
// 组合条件
const posts = await prisma.post.findMany({
  where: {
    AND: [{ published: true }, { viewCount: { gt: 100 } }],
  },
});

// 或条件
const posts = await prisma.post.findMany({
  where: {
    OR: [{ title: { contains: "Prisma" } }, { content: { contains: "ORM" } }],
  },
});

// 非条件
const posts = await prisma.post.findMany({
  where: {
    NOT: {
      title: { contains: "deprecated" },
    },
  },
});

// 范围查询
const posts = await prisma.post.findMany({
  where: {
    createdAt: {
      gte: new Date("2024-01-01"),
      lt: new Date("2024-12-31"),
    },
  },
});

// IN 查询
const posts = await prisma.post.findMany({
  where: {
    authorId: { in: [1, 2, 3] },
  },
});

// 模糊搜索
const posts = await prisma.post.findMany({
  where: {
    OR: [
      { title: { contains: "prisma", mode: "insensitive" } },
      { content: { search: "orm database" } }, // 全文搜索（PostgreSQL）
    ],
  },
});

// 关联条件

    author: {

  },

```

### 排序、分页和聚合

```typescript
// 排序
const posts = await prisma.post.findMany({
  orderBy: [{ viewCount: "desc" }, { createdAt: "desc" }],
});

// 分页
const posts = await prisma.post.findMany({
  skip: 20,
  take: 10,
});

// 游标分页（大数据集推荐）
const posts = await prisma.post.findMany({
  take: 10,
  cursor: { id: 100 },

// 聚合
const stats = await prisma.post.aggregate({
  where: { published: true },
  _count: { id: true },
  _sum: { viewCount: true },
  _avg: { viewCount: true },
  _min: { createdAt: true },
  _max: { createdAt: true },
});

// 分组
const authorStats = await prisma.post.groupBy({
  by: ["authorId"],
  _count: { id: true },
  _sum: { viewCount: true },
  having: {
    viewCount: { _sum: { gt: 1000 } },
  },
});
```

## 事务处理

### 交互式事务

```typescript
// 交互式事务 - 可以在事务中执行任意逻辑
const result = await prisma.$transaction(async (tx) => {
  // 扣减用户余额
  const fromUser = await tx.user.update({
    where: { id: 1 },
    data: { balance: { decrement: 100 } },
  });

  // 增加目标用户余额
  const toUser = await tx.user.update({
    where: { id: 2 },
    data: { balance: { increment: 100 } },
  });

  // 创建转账记录
  const transfer = await tx.transfer.create({
    data: {
      fromUserId: 1,
      toUserId: 2,
      amount: 100,
    },
  });

  return { fromUser, toUser, transfer };
});
```

### 批量事务

```typescript
// 批量事务 - 按顺序执行多个操作
const [user, post] = await prisma.$transaction([
  prisma.user.create({
    data: { email: "test@example.com" },
  }),
  prisma.post.create({
    data: {
      title: "First Post",
      authorId: 1,
    },
  }),
]);
```

### 事务选项

```typescript
// 设置事务隔离级别和超时
await prisma.$transaction(
  async (tx) => {
    // 事务操作
  },
  {
    maxWait: 5000, // 最大等待时间
    timeout: 10000, // 事务超时时间
    isolationLevel: Prisma.TransactionIsolationLevel.Serializable,
  }
);
```

## 实战案例：构建博客 API

### 完整的 Service 层

```typescript
// services/post.service.ts
import { PrismaClient, Prisma } from "@prisma/client";

const prisma = new PrismaClient();

export class PostService {
  // 创建文章
  async create(authorId: number, data: CreatePostDto) {
    return prisma.post.create({
      data: {
        ...data,
        authorId,
        tags: {
          connectOrCreate: data.tags.map((name) => ({
            where: { name },
            create: { name },
          })),
        },
      },
      include: {
        author: { select: { id: true, name: true } },
        tags: true,
      },
    });
  }

  // 获取文章列表
  async findAll(params: {
    page?: number;
    limit?: number;
    tag?: string;
    search?: string;
  }) {
    const { page = 1, limit = 10, tag, search } = params;
    const skip = (page - 1) * limit;

    const where: Prisma.PostWhereInput = {
      published: true,
      ...(tag && {
        tags: { some: { name: tag } },
      }),
      ...(search && {
        OR: [
          { title: { contains: search, mode: "insensitive" } },
          { content: { contains: search, mode: "insensitive" } },
        ],
      }),
    };

    const [posts, total] = await Promise.all([
      prisma.post.findMany({
        where,
        skip,
        take: limit,
        orderBy: { createdAt: "desc" },
        include: {
          author: { select: { id: true, name: true, avatar: true } },
          tags: true,
          _count: { comments: true },
        },
      }),
      prisma.post.count({ where }),
    ]);

    return {
      posts,
      pagination: {
        page,
        limit,
        total,
        totalPages: Math.ceil(total / limit),
      },
    };
  }

  // 获取文章详情
  async findOne(id: number) {
    const [post, relatedPosts] = await Promise.all([
      prisma.post.findUnique({
        where: { id },
        include: {
          author: { select: { id: true, name: true, avatar: true } },
          tags: true,
          comments: {
            include: {
              user: { select: { id: true, name: true, avatar: true } },
            },
            orderBy: { createdAt: "desc" },
          },
        },
      }),
      prisma.post.findMany({
        where: {
          id: { not: id },
          tags: {
            some: {
              id: { in: await this.getTagIds(id) },
            },
          },
        },
        take: 5,
        select: { id: true, title: true },
      }),
    ]);

    // 增加浏览量
    await prisma.post.update({
      where: { id },
      data: { viewCount: { increment: 1 } },
    });

    return { ...post, relatedPosts };
  }

  private async getTagIds(postId: number) {
    const post = await prisma.post.findUnique({
      where: { id: postId },
      include: { tags: { select: { id: true } } },
    });
    return post?.tags.map((t) => t.id) ?? [];
  }

  // 更新文章
  async update(id: number, authorId: number, data: UpdatePostDto) {
    return prisma.post.update({
      where: { id, authorId },
      data: {
        ...data,
        tags: data.tags && {
          set: [], // 清除现有标签
          connectOrCreate: data.tags.map((name) => ({
            where: { name },
            create: { name },
          })),
        },
      },
      include: { tags: true },
    });
  }

  // 删除文章
  async delete(id: number, authorId: number) {
    return prisma.$transaction(async (tx) => {
      // 先删除关联的评论
      await tx.comment.deleteMany({
        where: { postId: id },
      });

      // 再删除文章
      return tx.post.delete({
        where: { id, authorId },
      });
    });
  }
}
```

### 与 Express 集成

```typescript
// app.ts
import express from "express";
import { PostService } from "./services/post.service";

const app = express();
app.use(express.json());

const postService = new PostService();

// 获取文章列表
app.get("/posts", async (req, res) => {
  const { page, limit, tag, search } = req.query;
  const result = await postService.findAll({
    page: Number(page),
    limit: Number(limit),
    tag: tag as string,
    search: search as string,
  });
  res.json(result);
});

// 获取文章详情
app.get("/posts/:id", async (req, res) => {
  const post = await postService.findOne(Number(req.params.id));
  res.json(post);
});

// 创建文章（需要认证）
app.post("/posts", authenticate, async (req, res) => {
  const post = await postService.create(req.user.id, req.body);
  res.status(201).json(post);
});
```

## 最佳实践总结

### 1. Schema 设计原则

- 使用 `@@map` 保持数据库命名规范（snake_case）
- 合理设置索引，避免过度索引
- 使用枚举类型替代魔法字符串
- 为时间字段添加 `@default(now())` 和 `@updatedAt`

### 2. 查询优化策略

- 避免 N+1 查询，使用 `include` 或 `select`
- 只选择需要的字段
- 大数据集使用游标分页
- 合理使用批量操作

### 3. 类型安全

```typescript
// 利用 Prisma 生成的类型
import { Post, Prisma } from "@prisma/client";

// Where 输入类型
type PostWhereInput = Prisma.PostWhereInput;

// Create 输入类型
type PostCreateInput = Prisma.PostCreateInput;

// 包含关联的完整类型
type PostWithAuthor = Post & {
  author: { id: number; name: string };
};
```

### 4. 错误处理

```typescript
import { Prisma } from "@prisma/client";

try {
  await prisma.user.create({ data: { email: "existing@example.com" } });
} catch (error) {
  if (error instanceof Prisma.PrismaClientKnownRequestError) {
    if (error.code === "P2002") {
      // 唯一约束冲突
      console.log("Email already exists");
    }
  }
}
```

### 5. 生产环境配置

```typescript
// 单例模式

  var prisma: PrismaClient | undefined;

export const prisma =

if (process.env.NODE_ENV !== "production") {
  globalThis.prisma = prisma;

// 优雅关闭

  await prisma.$disconnect();
});
```

## 总结

Prisma 通过类型安全的 API、直观的 Schema 定义和强大的迁移系统，大大提升了 Node.js 后端开发的效率和代码质量。掌握 Prisma 的核心概念和最佳实践，能帮助你构建更健壮、更易维护的数据层。

关键要点回顾：

1. **Schema 是核心**：好的 Schema 设计是一切的基础
