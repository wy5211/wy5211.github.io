## 前言

Vue 生态的开发者一直在寻找类似于 Next.js + tRPC 的类型安全解决方案。随着 Nuxt 3 的发布和 Composition API 的成熟，现在我们可以在 Vue 项目中获得同样出色的 tRPC 开发体验。

**Nuxt + tRPC 的优势**：

- ✅ 完整的类型安全（和 Next.js 一样）
- ✅ 服务端渲染（SSR）支持
- ✅ Vue Composition API 集成
- ✅ 自动导入组件和函数
- ✅ 优秀的开发者体验

今天，我们将学习：

- Nuxt 3 项目配置和 tRPC 集成
- Vue Composition API 中使用 tRPC
- 服务端渲染和客户端渲染
- 与 Next.js 的对比
- 完整的 Vue 应用实战

## Nuxt vs Next.js：快速对比

在深入代码之前，让我们先看看两个框架的相似之处和差异：

| 特性          | Next.js (React)       | Nuxt (Vue)              |
| ------------- | --------------------- | ----------------------- |
| **类型安全**  | ✅ 完整               | ✅ 完整                 |
| **SSR**       | ✅ Server Components  | ✅ useAsyncData         |
| **组件风格**  | JSX/TSX               | Template + Script       |
| **状态管理**  | React Query           | useAsyncData + useState |
| **自动导入**  | 需配置                | ✅ 原生支持             |
| **路由系统**  | App Router (文件系统) | 文件系统路由            |
| **tRPC 集成** | @trpc/next            | @trpc-nuxt/nuxt         |

**学习曲线**：如果你已经掌握了 Next.js + tRPC，学习 Nuxt + tRPC 会非常容易！

## 服务端配置

### 创建 tRPC Context

```typescript
// server/trpc/context.ts

import { CreateNuxtContextOptions } from "@trpc-nuxt/nuxt";
import { getServerSession } from "#auth";

interface User {
  id: string;
  name: string;
  email: string;
  avatar: string | null;
}

export interface Context {
  user: User | null;
  session: Session | null;
}

export async function createContext({
  event,
}: CreateNuxtContextOptions): Promise<Context> {
  // 获取会话（使用 NextAuth 或其他认证方案）
  const session = await getServerSession(event);

  return {
    user: session?.user || null,
    session,
  };
}
```

### 创建 tRPC 实例

```typescript
// server/trpc/index.ts

import { initTRPC, TRPCError } from "@trpc/server";
import { ZodError } from "zod";
import type { Context } from "./context";

const t = initTRPC.context<Context>().create({
  errorFormatter({ shape, error }) {
    return {
      ...shape,
      data: {
        ...shape.data,
        zodError:
          error.code === "BAD_REQUEST" && error.cause instanceof ZodError
            ? error.cause.flatten()
            : null,
      },
    };
  },
});

// 中间件：是否已登录
const isAuthed = t.middleware(({ ctx, next }) => {
  if (!ctx.user) {
    throw new TRPCError({
      code: "UNAUTHORIZED",
      message: "需要登录",
    });
  }
  return next({
    ctx: {
      ...ctx,
      user: ctx.user, // 非 null 断言
    },
  });
});

// 导出
export const router = t.router;
export const publicProcedure = t.procedure;
export const protectedProcedure = t.procedure.use(isAuthed);
```

### 创建 Routers

```typescript
// server/trpc/routers/posts.ts

import { z } from "zod";
import { prisma } from "@/lib/prisma";
import { router, protectedProcedure } from "../index";

export const postsRouter = router({
  // 获取文章列表
  list: protectedProcedure
    .input(
      z.object({
        page: z.number().default(1),
        limit: z.number().min(1).max(100).default(20),
        published: z.boolean().optional(),
      })
    )
    .query(async ({ input }) => {
      const { page, limit, published } = input;
      const skip = (page - 1) * limit;

      const [posts, total] = await Promise.all([
        prisma.post.findMany({
          where: published !== undefined ? { published } : undefined,
          skip,
          take: limit,
          orderBy: { createdAt: "desc" },
          include: {
            author: {
              select: {
                id: true,
                name: true,
                avatar: true,
              },
            },
            _count: {
              select: {
                comments: true,
                likes: true,
              },
            },
          },
        }),
        prisma.post.count({
          where: published !== undefined ? { published } : undefined,
        }),
      ]);

      return {
        posts,
        pagination: {
          page,
          limit,
          total,
          totalPages: Math.ceil(total / limit),
        },
      };
    }),

  // 根据 ID 获取文章
  byId: protectedProcedure
    .input(
      z.object({
        id: z.string(),
        includeComments: z.boolean().default(true),
      })
    )
    .query(async ({ input }) => {
      const post = await prisma.post.findUnique({
        where: { id: input.id },
        include: {
          author: {
            select: {
              id: true,
              name: true,
              email: true,
              avatar: true,
              bio: true,
            },
          },
          ...(input.includeComments && {
            comments: {
              include: {
                author: {
                  select: {
                    id: true,
                    name: true,
                    avatar: true,
                  },
                },
              },
              orderBy: { createdAt: "desc" },
            },
          }),
          likes: {
            select: {
              user: {
                select: {
                  id: true,
                  name: true,
                },
              },
            },
          },
        },
      });

      if (!post) {
        throw new TRPCError({
          code: "NOT_FOUND",
          message: "文章不存在",
        });
      }

      return post;
    }),

  // 创建文章
  create: protectedProcedure
    .input(
      z.object({
        title: z.string().min(1).max(200),
        content: z.string().min(1),
        published: z.boolean().default(false),
      })
    )
    .mutation(async ({ input, ctx }) => {
      const post = await prisma.post.create({
        data: {
          ...input,
          authorId: ctx.user.id,
        },
        include: {
          author: {
            select: {
              id: true,
              name: true,
              avatar: true,
            },
          },
        },
      });

      return post;
    }),

  // 更新文章
  update: protectedProcedure
    .input(
      z.object({
        id: z.string(),
        title: z.string().min(1).max(200).optional(),
        content: z.string().min(1).optional(),
        published: z.boolean().optional(),
      })
    )
    .mutation(async ({ input, ctx }) => {
      const { id, ...data } = input;

      const post = await prisma.post.findUnique({
        where: { id },
      });

      if (!post) {
        throw new TRPCError({
          code: "NOT_FOUND",
          message: "文章不存在",
        });
      }

      if (post.authorId !== ctx.user.id) {
        throw new TRPCError({
          code: "FORBIDDEN",
          message: "只能编辑自己的文章",
        });
      }

      const updated = await prisma.post.update({
        where: { id },
        data,
        include: {
          author: {
            select: {
              id: true,
              name: true,
              avatar: true,
            },
          },
        },
      });

      return updated;
    }),

  // 删除文章
  delete: protectedProcedure
    .input(z.object({ id: z.string() }))
    .mutation(async ({ input, ctx }) => {
      const post = await prisma.post.findUnique({
        where: { id: input.id },
      });

      if (!post) {
        throw new TRPCError({
          code: "NOT_FOUND",
          message: "文章不存在",
        });
      }

      if (post.authorId !== ctx.user.id) {
        throw new TRPCError({
          code: "FORBIDDEN",
          message: "只能删除自己的文章",
        });
      }

      await prisma.post.delete({
        where: { id: input.id },
      });

      return { success: true };
    }),
});

export type PostsRouter = typeof postsRouter;
```

### 无限滚动

```vue
<!-- app/components/PostsInfinite.vue -->

<script setup lang="ts">
const posts = ref<any[]>([]);
const cursor = ref<string | undefined>(undefined);
const hasMore = ref(true);
const loading = ref(false);

const loadMore = async () => {
  if (loading.value || !hasMore.value) return;

  loading.value = true;
  try {
    const result = await $trpc.posts.infinite.query({
      limit: 20,
      cursor: cursor.value,
    });

    posts.value.push(...result.items);
    cursor.value = result.nextCursor;
    hasMore.value = !!result.nextCursor;
  } finally {
    loading.value = false;
  }
};

// 初始加载
onMounted(() => {
  loadMore();
});

// 无限滚动
const observerTarget = ref<HTMLElement | null>(null);

onMounted(() => {
  const observer = new IntersectionObserver(
    (entries) => {
      if (entries[0].isIntersecting && hasMore.value) {
        loadMore();
      }
    },
    { threshold: 1 }
  );

  if (observerTarget.value) {
    observer.observe(observerTarget.value);
  }
});
</script>

<template>
  <div class="space-y-6">
    <article v-for="post in posts" :key="post.id" class="border rounded-lg p-6">
      <NuxtLink :to="`/posts/${post.id}`">
        <h2 class="text-2xl font-semibold mb-2">
          {{ post.title }}
        </h2>
      </NuxtLink>
      <p class="text-gray-600">
        {{ post.content }}
      </p>
    </article>

    <!-- 观察目标 -->
    <div ref="observerTarget" />

    <div v-if="loading" class="text-center py-8">加载中...</div>

    <div v-if="!hasMore" class="text-center py-8 text-gray-500">没有更多了</div>
  </div>
</template>
```

## 组合式函数

### 创建可复用的组合式函数

```typescript
// app/composables/usePosts.ts

export const usePosts = (page: Ref<number> = ref(1)) => {
  const { data, pending, error, refresh } = useAsyncData(
    computed(() => `posts-page-${page.value}`),
    () =>
      $trpc.posts.list.query({
        page: page.value,
        limit: 20,
      })
  );

  return {
    posts: computed(() => data.value?.posts || []),
    pagination: computed(() => data.value?.pagination),
    pending,
    error,
    refresh,
  };
};
```

### 使用组合式函数

```vue
<!-- app/pages/index.vue -->

<script setup lang="ts">
const page = ref(1);

  if (page.value < (pagination.value?.totalPages || 0)) {
    page.value++;

  if (page.value > 1) {
    page.value--;
```

```typescript
export const useAuth = () => {
  const {
    data: user,
    pending,
```

## 总结

### 核心要点

1. **Nuxt + tRPC 集成**
   - @trpc-nuxt/nuxt 模块配置
   - Nitro 路由处理器
   - 自动导入和类型推断

2. **Vue Composition API**
   - useAsyncData 数据获取
   - useMutation 数据变更
   - composables 代码复用

3. **服务端渲染**
   - 默认 SSR 支持
   - SEO 友好
   - 性能优化

4. **与 Next.js 对比**
   - 相同的类型安全
   - 不同的组件风格
   - 各有优劣

**选择 Next.js + tRPC 如果**：

- ✅ 团队更熟悉 React
- ✅ 需要更成熟的生态系统
- ✅ 使用 Server Components

**选择 Nuxt + tRPC 如果**：

- ✅ 团队更熟悉 Vue
- ✅ 喜欢模板语法
- ✅ 需要自动导入功能
