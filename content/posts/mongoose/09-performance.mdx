## 前言

性能优化是生产应用的关键。本文将学习索引设计、查询优化和最佳实践。

## 索引基础

### 创建索引

```typescript
// 单字段索引
postSchema.index({ title: 1 });

// 复合索引
postSchema.index({ author: 1, createdAt: -1 });

// 文本索引
postSchema.index({ title: "text", content: "text" });

// 唯一索引
postSchema.index({ email: 1 }, { unique: true });

// 稀疏索引
postSchema.index({ phone: 1 }, { sparse: true });

// TTL 索引
logSchema.index({ createdAt: 1 }, { expireAfterSeconds: 3600 });
```

## 性能优化技巧

### 1. 使用 lean()

```typescript
// ✅ lean() 返回普通对象（快）
const posts = await Post.find().lean();

// ❌ 默认返回 Mongoose Document（慢）
const posts = await Post.find();
```

### 2. 选择必要字段

```typescript
const posts = await Post.find().select("title views").lean();
```

### 3. 批量操作

```typescript
// ✅ 批量插入
await Post.insertMany([...posts]);

// ❌ 循环插入
for (const post of posts) {
  await Post.create(post);
}
```

## 索引策略

### ESR 规则

```
E - Equality (精确匹配)
S - Sort (排序字段)
R - Range (范围查询)

索引顺序: E → S → R
```

### 示例

```typescript
// ✅ 好的索引
postSchema.index({ status: 1, createdAt: -1 });

// 查询
Post.find({ status: "published" }).sort({ createdAt: -1 });

// ❌ 差的索引
postSchema.index({ createdAt: -1, status: 1 });

// 查询
Post.find({ createdAt: { $gte: date } }).sort({ createdAt: -1 });
```

## 性能监控

### 使用 explain()

```typescript
const explanation = await Post.find({ author: userId }).explain(
  "executionStats"
);

console.log(explanation);
```
