上一篇聊了 Hooks 自动化。Hooks 是"被动"的——在特定时机自动执行。但很多时候你需要"主动"调用特定的能力——比如让 Agent 按特定流程开发、按特定格式审查代码。

在 [Claude Code 系列第五篇](/posts/claude-code-tips/05-skills-extend-capabilities) 里，我讲过 Claude Code 的 Skills 生态——从 Superpowers 到 Context7，再到自制 Skill。Claude Code 用"Skills + MCP"两层来扩展能力：Skills 扩展"能做什么"，MCP 扩展"能连接什么"。

Kiro 用了一个不同的分层：**Powers → Subagents → Skills**。三层，从上到下越来越具体。

## 三层架构

### Powers：顶层能力

Powers 是 Kiro 内置的通用能力，不需要配置就能用：

- **Code Generation**：创建文件、实现功能
- **Refactoring**：重构代码、调整结构
- **Debugging**：定位和修复 bug
- **Testing**：生成和运行测试
- **Documentation**：生成文档和注释

这些是"开箱即用"的能力。你在 Kiro 里说"帮我实现用户注册"，Powers 里的 Code Generation 就会被激活。

Claude Code 没有显式的"Powers"概念——所有能力都统一在一个 Agent 里。Kiro 把它们显式化，好处是 Agent 可以根据任务类型选择不同的执行策略。比如 Debugging 和 Code Generation 的工作方式就很不同——前者需要先分析问题，后者需要先理解需求。

### Subagents：专业化 Agent

每个 Power 下面可以有专门的 Subagent。比如 Code Generation 下面有：

- **Frontend Agent**：React、Vue、Angular、样式
- **Backend Agent**：API 设计、数据库、业务逻辑
- **DevOps Agent**：CI/CD、基础设施、部署
- **Security Agent**：漏洞扫描、安全最佳实践
- **Testing Agent**：单元测试、集成测试、E2E 测试

Subagent 的价值在于**专业化**。一个 Frontend Agent 在处理 React 组件时比通用 Agent 更准确——它理解组件生命周期、状态管理模式、常见的前端陷阱。

这和 Claude Code 的 Workflow 模式有相似之处——在 [Claude Code 系列第七篇](/posts/claude-code-tips/07-workflow-multi-agent) 里，我展示过如何编排多个 Agent 分别审查不同维度。但 Claude Code 的多 Agent 是临时编排的（每次定义），Kiro 的 Subagent 是预定义的专业化角色。

### Skills：可复用的操作模板

Skills 是最具体的一层——预制的操作模板，定义了特定任务该怎么做。

比如"创建 React 组件"是一个 Skill：它定义了组件文件该放在哪里、用什么模板、包含哪些必要部分。你一说"创建一个用户卡片组件"，这个 Skill 就会被触发。

Kiro 的 Skills 和 Claude Code 的 Skills 概念上类似，但生态成熟度不同。Claude Code 有 claudeskills.info 上 658+ 个社区 Skill，还有 everything-claude-code（ECC）这样的工具包。Kiro 的 Skill 生态还在早期阶段——社区共享和 Skill 市场还在建设中。

## 和 Claude Code Skills + MCP 的对比

### 扩展模型

**Claude Code**：两层模型

```
Skills（扩展能力） + MCP（扩展连接）
```

Skills 处理"怎么做"（TDD 流程、代码审查框架），MCP 处理"连什么"（GitHub、数据库、Figma）。两者独立，可以单独使用。

**Kiro**：三层模型

```
Powers（通用能力） → Subagents（专业化角色） → Skills（操作模板）
```

三层是层级关系——Skills 在 Subagent 的框架内执行，Subagent 在 Powers 的范围内工作。更像一个组织架构：公司（Powers）→ 部门（Subagents）→ 岗位（Skills）。

### 灵活性 vs 结构性

Claude Code 更灵活——你可以自由组合任何 Skill 和任何 MCP 服务器，没有层级约束。比如你可以让一个 Skill 同时使用 GitHub MCP 和数据库 MCP。

Kiro 更结构化——Subagent 的专业化意味着它对特定领域有更深的理解，但跨领域的能力可能不如通用方案。比如 Frontend Agent 不太擅长处理数据库操作。

### 生态成熟度

**Claude Code 生态**：

- claudeskills.info：658+ 社区 Skill
- awesome-claude-skills：1000+ Skill 精选列表
- `npx skills add` 一键安装
- ECC 工具包：136+ 配置打包
- 热门 Skills：Superpowers、Context7、Task Master、Deep Research

**Kiro 生态**：

- 内置 Powers 和 Subagents 够用，不需要太多社区扩展
- Skills 市场还在建设中
- 社区贡献的 Skill 数量远少于 Claude Code
- AWS 官方提供了一些配套的 Skill

如果你依赖丰富的社区生态，Claude Code 目前有明显优势。如果你主要需要内置能力且用 AWS 技术栈，Kiro 的开箱即用体验更平滑。

## 创建自定义 Skill

Kiro 的 Skill 创建方式和 Claude Code 类似——写一个指令文件。但 Kiro 的 Skill 可以指定关联的 Subagent：

```markdown
---
name: blog-series-writer
description: 创建和撰写博客系列文章
subagent: frontend
---

# 博客系列写作 Skill

## 前置条件

确认系列主题、文章数量、文章类型、目标读者。

## 创建流程

### 第一步：创建目录和计划书

在 content/posts/{category}/ 下创建新目录，
创建 plan.md 描述系列规划。

### 第二步：逐篇撰写

按 frontmatter 格式创建 .mdx 文件，
每篇包含 title, date, summary, tags, category, series, seriesOrder。

### 第三步：构建验证

运行 npm run build，修复错误直到通过。
```

和 Claude Code Skill 的主要区别：

- 多了一个 `subagent` 字段，指定由哪个专业化 Agent 执行
- 指令内容可以引用 Steering Files 的约定（不需要重复写）
- Kiro 的 Skill 可以调用 Hook（比如在"构建验证"步骤自动触发 preCommit Hook）

## 三层架构的优势场景

### 大型团队

团队里的不同角色可以对应不同的 Subagent：前端开发用 Frontend Agent，后端开发用 Backend Agent，DevOps 用 DevOps Agent。每个人使用的 Skill 都在对应的 Subagent 框架内执行，确保专业性。

### 复杂项目

多层架构可以在不同粒度上管理 AI 的行为。Powers 控制大方向（代码生成还是调试），Subagents 控制具体领域（前端还是后端），Skills 控制具体操作（怎么创建组件）。

### 质量保障

Subagent 的专业化意味着它在自己的领域内犯错更少。一个 Security Agent 在做安全审查时比通用 Agent 更可靠——这和 Claude Code Workflow 里的"对抗验证"有异曲同工之妙。

## 三层架构的劣势场景

### 小项目

三层架构对小项目来说是过度设计。你只需要"帮我写个组件"，不需要管什么 Powers、Subagents、Skills。Claude Code 的扁平模型更适合——一个 Skill 就够了。

### 跨领域任务

如果你需要同时涉及前端和后端（比如实现一个完整的用户注册流程），Kiro 的 Subagent 分工反而可能造成摩擦——Frontend Agent 不了解后端逻辑，Backend Agent 不关心前端表现。

Claude Code 没有这个问题——一个通用 Agent 同时处理前后端。虽然不如专业化 Agent 在单一领域精准，但跨领域任务的上下文连贯性更好。

### 学习成本

Powers、Subagents、Skills 三层概念需要时间理解。Claude Code 只需要理解 Skills（和 MCP），上手更快。

## 实际使用建议

如果你同时用 Kiro 和 Claude Code：

- **Kiro** 负责：结构化的功能开发（SDD + Subagent）、AWS 相关工作、需要专业化的任务
- **Claude Code** 负责：快速原型、调试分析、跨领域任务、需要丰富社区生态的场景

下一篇聊 Kiro 的主场——AWS 原生集成。Claude Code 需要通过 MCP 服务器连接外部服务，Kiro 对 AWS 服务是原生集成的。如果你用 AWS，这是 Kiro 最大的优势。
