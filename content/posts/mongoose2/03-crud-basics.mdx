上一篇文章我们设计了完整的 User 和 Post Schema，定义了数据结构和方法。但光有 Schema 不够，我们还需要操作数据：创建文章、读取文章列表、更新文章内容、删除文章。

CRUD（Create, Read, Update, Delete）是数据库操作的核心。Mongoose 提供了丰富的 API 来执行这些操作，但方法很多，容易搞混。这篇文章我们围绕 CMS 文章管理的实际场景，系统讲解各种 CRUD 操作。

## 创建数据

创建数据有三种主要方式：new + save、create、insertMany。它们的区别是什么？

- **new + save**：先创建实例，可以修改后再保存，适合需要预处理数据的场景
- **create**：直接创建并保存，代码简洁
- **insertMany**：批量插入，性能最好

假设我们要为 CMS 创建文章：

```typescript
import Post from "../models/Post";
import User from "../models/User";

// 方式一：new + save
const post1 = new Post({
  title: "Mongoose 入门教程",
  content: "这是一篇关于 Mongoose 的详细教程...",
  author: "usr_8x9k2m",
  status: "draft",
});

// 可以在保存前修改
post1.tags = ["mongodb", "nodejs", "tutorial"];

await post1.save();

// 方式二：create（更简洁）
const post2 = await Post.create({
  title: "Node.js 实战",
  content: "Node.js 实战教程内容...",
  author: "usr_8x9k2m",
  status: "published",
  tags: ["nodejs", "javascript"],
});

// 方式三：批量创建
const posts = await Post.create([
  {
    title: "MongoDB 高级技巧",
    content: "...",
    author: "usr_8x9k2m",
    status: "published",
    tags: ["mongodb", "database"],
  },
  {
    title: "TypeScript 指南",
    content: "...",
    author: "usr_8x9k2m",
    status: "draft",
    tags: ["typescript", "javascript"],
  },
]);

// 方式四：insertMany（性能最好，适合大量数据）
const result = await Post.insertMany([
  { title: "文章1", content: "...", author: "usr_xxx" },
  { title: "文章2", content: "...", author: "usr_xxx" },
  { title: "文章3", content: "...", author: "usr_xxx" },
]);
```

什么时候用哪种方式？

- **少量数据且需要预处理**：用 new + save
- **少量数据且直接创建**：用 create
- **批量数据**：用 insertMany（性能好很多）

## 读取数据

读取数据是最常用的操作。Mongoose 提供了 find、findOne、findById 等方法，还支持条件筛选、排序、分页、投影等功能。

### 基础查询

```typescript
// 查询所有文章
const allPosts = await Post.find();

// 条件查询
const publishedPosts = await Post.find({ status: "published" });

// 查询单个文档
const firstPost = await Post.findOne({ status: "published" });

// 通过 ID 查询
const post = await Post.findById("post_a1b2c3d4");
```

### 条件筛选

CMS 中经常需要按条件筛选文章，比如按状态、按作者、按浏览量：

```typescript
// 比较运算符
const popularPosts = await Post.find({
  views: { $gte: 100 }, // 大于等于
});

// 组合条件
const recentPopularPosts = await Post.find({
  status: "published",
  views: { $gte: 50 },
  createdAt: { $gte: new Date(Date.now() - 7 * 24 * 60 * 60 * 1000) }, // 最近7天
});

// 逻辑运算符
const tagPosts = await Post.find({
  $or: [{ tags: "mongodb" }, { tags: "nodejs" }],
});
```

### 投影（选择字段）

有时候你不需要返回所有字段，比如文章列表页只需要标题、浏览量、创建时间，不需要完整内容：

```typescript
// 只返回指定字段
const postTitles = await Post.find({ status: "published" })
  .select("title views createdAt")
  .limit(3);

// 排除字段
const postsWithoutContent = await Post.find()
  .select("-content -metadata")
  .limit(2);
```

投影的好处是减少数据传输量，提升性能。

### 排序

CMS 需要按不同维度排序文章：

```typescript
// 单字段排序
const latestPosts = await Post.find({ status: "published" })
  .sort({ createdAt: -1 }) // -1 降序，1 升序
  .limit(3);

// 多字段排序
const sortedPosts = await Post.find({ status: "published" })
  .sort({ views: -1, createdAt: -1 }) // 先按浏览量降序，再按创建时间降序
  .limit(3);
```

### 分页

文章列表需要分页展示，避免一次加载太多数据：

```typescript
const page = 1;
const limit = 5;
const skip = (page - 1) * limit;

const paginatedPosts = await Post.find({ status: "published" })
  .skip(skip)
  .limit(limit)
  .sort({ createdAt: -1 });

const totalCount = await Post.countDocuments({ status: "published" });
const totalPages = Math.ceil(totalCount / limit);
```

这种方式叫 offset-based 分页，适合数据量不大的场景。数据量大时可以用 cursor-based 分页，性能更好。

### lean 提升性能

如果你只需要读取数据，不需要修改或调用文档方法，用 lean 返回普通 JS 对象，性能更好：

```typescript
// 普通查询（返回 Mongoose Document）
const posts1 = await Post.find();

// lean 查询（返回普通 JS 对象，性能更好）
const posts2 = await Post.find().lean();
```

lean 查询去掉 Mongoose 的内部包装，直接返回 MongoDB 的原始数据，查询速度更快、内存占用更少。

## 更新数据

更新数据也有多种方式：修改后保存、updateOne、findOneAndUpdate。它们的区别是什么？

- **document.save()**：先查询文档，修改后保存，会触发中间件和验证
- **updateOne()**：直接更新，不触发中间件，性能更好
- **findOneAndUpdate()**：查找并更新，返回更新后的文档

### 修改后保存

```typescript
const post = await Post.findById("post_a1b2c3d4");

post.title = "更新后的标题";
post.views = 100;

await post.save();
```

这种方式会触发 pre/post save 中间件，适合需要自动处理相关数据的场景。

### 直接更新

```typescript
// updateOne（不返回文档）
await Post.updateOne(
  { _id: "post_a1b2c3d4" },
  { $set: { title: "第二次更新的标题" } }
);

// findOneAndUpdate（返回更新后的文档）
const updatedPost = await Post.findOneAndUpdate(
  { _id: "post_a1b2c3d4" },
  { $set: { status: "published" } },
  { new: true } // 返回更新后的文档
);
```

### 更新运算符

Mongoose 支持丰富的更新运算符，用于不同场景：

```typescript
// $inc（增加/减少）
await Post.findByIdAndUpdate("post_a1b2c3d4", {
  $inc: { views: 10, likes: 5 },
});

// $mul（乘法）
await Post.findByIdAndUpdate("post_a1b2c3d4", {
  $mul: { views: 2 }, // 浏览量乘以2
});

// $rename（重命名字段）
await Post.findByIdAndUpdate("post_a1b2c3d4", {
  $rename: { "metadata.seoTitle": "metadata.title" },
});

// $unset（删除字段）
await Post.findByIdAndUpdate("post_a1b2c3d4", {
  $unset: { "metadata.seoDescription": "" },
});

// 数组更新运算符
await Post.findByIdAndUpdate("post_a1b2c3d4", {
  $push: { tags: "new-tag" }, // 添加到数组
  $addToSet: { tags: "mongodb" }, // 添加到集合，避免重复
  $pull: { tags: "test" }, // 从数组删除
});
```

### 批量更新

CMS 中需要批量操作，比如把所有草稿文章改为已发布：

```typescript
const result = await Post.updateMany(
  { status: "draft" },
  {
    $set: { status: "published" },
    $inc: { views: 1 },
  }
);

console.log("匹配文档数:", result.matchedCount);
console.log("修改文档数:", result.modifiedCount);
```

## 删除数据

删除数据有多种方式：deleteOne、deleteMany、findOneAndDelete。CMS 中还需要软删除，标记数据为已删除而不是真正删除。

### 硬删除

```typescript
// 方式一：document.deleteOne()
const post = await Post.findById("post_a1b2c3d4");
await post.deleteOne();

// 方式二：Model.deleteOne()
await Post.deleteOne({ _id: "post_a1b2c3d4" });

// 方式三：批量删除
await Post.deleteMany({ status: "draft" });

// 方式四：findOneAndDelete（返回被删除的文档）
const deletedPost = await Post.findOneAndDelete({ title: "待删除文章" });
```

### 软删除

硬删除会永久丢失数据，很多场景需要软删除，只是标记为已删除：

```typescript
// 添加 deletedAt 字段
await Post.findByIdAndUpdate("post_a1b2c3d4", {
  deletedAt: new Date(),
});

// 查询时过滤已删除文档
const activePosts = await Post.find({
  deletedAt: { $exists: false },
});
```

更好的做法是用中间件自动过滤，在 Schema 中添加：

```typescript
schema.pre(/^find/, function () {
  this.where({ deletedAt: { $exists: false } });
});
```

这样所有 find 查询都会自动过滤已删除文档。

## 综合示例：文章管理 API

把上面的知识整合起来，为 CMS 写一个文章管理类：

```typescript
import Post from "../models/Post";

class ArticleManager {
  // 创建文章
  async createArticle(data: any) {
    const post = await Post.create(data);
    return await Post.findById(post._id).populate("author", "name email");
  }

  // 获取文章列表（分页）
  async getArticles(page = 1, limit = 10, status = "published") {
    const skip = (page - 1) * limit;
    const [articles, total] = await Promise.all([
      Post.find({ status })
        .skip(skip)
        .limit(limit)
        .sort({ createdAt: -1 })
        .populate("author", "name avatar")
        .lean(),
      Post.countDocuments({ status }),
    ]);

    return {
      articles,
      pagination: {
        page,
        limit,
        total,
        totalPages: Math.ceil(total / limit),
      },
    };
  }

  // 获取文章详情
  async getArticleById(id: string) {
    const article = await Post.findById(id)
      .populate("author", "name email avatar bio")
      .populate("comments.author", "name avatar");

    if (!article) {
      throw new Error("文章不存在");
    }

    // 增加浏览量
    await article.incrementViews();

    return article;
  }

  // 更新文章
  async updateArticle(id: string, updates: any) {
    const article = await Post.findByIdAndUpdate(
      id,
      { $set: updates },
      {
        new: true,
        runValidators: true,
      }
    ).populate("author", "name email");

    if (!article) {
      throw new Error("文章不存在");
    }

    return article;
  }

  // 删除文章
  async deleteArticle(id: string) {
    const article = await Post.findByIdAndDelete(id);

    if (!article) {
      throw new Error("文章不存在");
    }

    return { message: "文章已删除", id };
  }

  // 搜索文章
  async searchArticles(keyword: string, page = 1, limit = 10) {
    const skip = (page - 1) * limit;

    const [articles, total] = await Promise.all([
      Post.find({
        status: "published",
        $or: [
          { title: { $regex: keyword, $options: "i" } },
          { content: { $regex: keyword, $options: "i" } },
          { tags: { $regex: keyword, $options: "i" } },
        ],
      })
        .skip(skip)
        .limit(limit)
        .sort({ createdAt: -1 })
        .populate("author", "name avatar")
        .lean(),
      Post.countDocuments({
        status: "published",
        $or: [
          { title: { $regex: keyword, $options: "i" } },
          { content: { $regex: keyword, $options: "i" } },
          { tags: { $regex: keyword, $options: "i" } },
        ],
      }),
    ]);

    return {
      articles,
      pagination: {
        page,
        limit,
        total,
        totalPages: Math.ceil(total / limit),
      },
    };
  }
}

// 使用示例
const manager = new ArticleManager();

// 创建文章
const newArticle = await manager.createArticle({
  title: "完整示例文章",
  content: "这是一篇完整的示例文章...",
  author: "usr_8x9k2m",
  status: "published",
  tags: ["example", "test"],
});

// 获取文章列表
const articles = await manager.getArticles(1, 5);

// 获取文章详情
const articleDetail = await manager.getArticleById(newArticle._id.toString());

// 搜索文章
const searchResults = await manager.searchArticles("示例", 1, 5);

// 更新文章
const updatedArticle = await manager.updateArticle(newArticle._id.toString(), {
  title: "更新后的文章标题",
});

// 删除文章
await manager.deleteArticle(newArticle._id.toString());
```

这个类封装了文章管理的所有 CRUD 操作，业务逻辑清晰，复用性强。

## 性能优化建议

### 使用 lean

对于只读查询，用 lean 返回普通 JS 对象：

```typescript
// API 返回（性能优先）
const posts = await Post.find().lean();
```

### 只选择需要的字段

```typescript
const posts = await Post.find().select("title views createdAt").lean();
```

### 使用索引字段查询

```typescript
const posts = await Post.find({ author: userId, status: "published" });
```

### 限制返回数量

```typescript
const posts = await Post.find().limit(20);
```

### 批量操作

```typescript
// 批量插入（高效）
await Post.insertMany([...posts]);

// 批量更新
await Post.updateMany({ status: "draft" }, { $set: { status: "published" } });
```

## 总结与思考

这篇文章我们掌握了完整的 CRUD 操作。你可能觉得方法太多了，记不住。其实常用的就那么几个：

- **创建**：create（单个）、insertMany（批量）
- **读取**：find（多个）、findOne（单个）、findById（单个）
- **更新**：save（修改后保存）、findOneAndUpdate（查找并更新）
- **删除**：deleteOne（单个）、deleteMany（批量）、软删除（标记删除）

关键是要理解每种方法的适用场景和性能特点。比如 insertMany 比 create 快很多，但只适合批量操作。lean 查询性能好，但失去了文档方法。
