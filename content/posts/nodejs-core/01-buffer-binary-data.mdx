## 一、Buffer 是什么？

### 1.1 为什么需要 Buffer？

**JavaScript 语言本身的局限**：

JavaScript 最初设计时，主要处理网页中的文本数据，只有字符串、数字、布尔值等基本类型。对于二进制数据（如图片、视频、PDF），原生 JavaScript 并没有很好的支持。

```javascript
// ❌ JavaScript 原生无法直接操作二进制数据
const binaryData = ??? // 没有合适的类型
```

**前端的历史演进**：

```javascript
// 早期：用字符串模拟二进制（非常低效）
const binary = "\x00\x01\x02"; // 每个字符占 2-3 字节

// 后来：ArrayBuffer + TypedArray
const buffer = new ArrayBuffer(1024); // 分配 1KB 内存
const view = new Uint8Array(buffer); // 8 位无符号整数视图
view[0] = 255;
```

**Node.js 的方案**：

Node.js 在 V8 引擎之外，实现了 **Buffer** 对象，用于处理二进制数据流。

```javascript
// ✅ Node.js 的 Buffer
const buffer = Buffer.from("hello", "utf-8");
console.log(buffer); // <Buffer 68 65 6c 6c 6f>
```

### 1.2 Buffer 的特点

**核心特性**：

1. **固定大小**：创建后大小不可变
2. **原始内存**：直接操作内存中的字节
3. **全局可用**：无需 require，直接使用
4. **堆外内存**：不受 V8 堆内存限制

```javascript
// Buffer 的基本使用
const buf1 = Buffer.alloc(1024); // 分配 1KB，初始化为 0
const buf2 = Buffer.allocUnsafe(1024); // 分配 1KB，不初始化（更快，但包含旧数据）
const buf3 = Buffer.from([0x01, 0x02, 0x03]); // 从数组创建
const buf4 = Buffer.from("hello", "utf-8"); // 从字符串创建

console.log(buf1.length); // 1024（字节）
```

## 二、与前端 JS 的对比

### 2.1 Buffer vs ArrayBuffer

| 特性           | Node.js Buffer                    | 浏览器 ArrayBuffer      |
| -------------- | --------------------------------- | ----------------------- |
| **位置**       | Node.js 全局对象                  | 浏览器全局对象          |
| **内存**       | 堆外内存（C++ 层面）              | V8 垃圾回收管理的堆内存 |
| **大小**       | 可通过 `length` 读写              | 大小固定，不可变        |
| **索引访问**   | `buf[0]` 返回数字（0-255）        | 需要 TypedArray 视图    |
| **字符串编码** | 内置多种编码（utf8、base64、hex） | 需要手动编码/解码       |
| **内存池**     | 有 Buffer 池（8KB 以下）          | 无内存池                |
| **性能**       | 更快（直接内存操作）              | 相对较慢                |

### 2.2 代码对比

**前端处理二进制数据**：

```javascript
// 浏览器环境
const response = await fetch("/api/data");
const arrayBuffer = await response.arrayBuffer();

// 需要 TypedArray 视图才能操作
const uint8Array = new Uint8Array(arrayBuffer);
const dataView = new DataView(arrayBuffer);

// 读取第一个字节
const firstByte = uint8Array[0];

// 读取一个 16 位整数
const int16 = dataView.getInt16(0, false); // big-endian

// 转换为字符串
const text = new TextDecoder().decode(arrayBuffer);
```

**Node.js 处理二进制数据**：

```javascript
// Node.js 环境
const buffer = Buffer.from([0x01, 0x02, 0x03, 0x04]);

// 直接索引访问
console.log(buffer[0]); // 1

// 读取多字节数据
const int16 = buffer.readUInt16BE(0); // 大端序
console.log(int16); // 258 (0x0102)

// 转换为字符串
console.log(buffer.toString("hex")); // '01020304'
console.log(buffer.toString("base64")); // 'AQIDBA=='

// 直接写入
buffer.writeUInt32BE(0x12345678, 0);
```

### 2.3 为什么前端不常接触？

1. **浏览器的安全限制**：不允许直接操作内存
2. **应用场景少**：前端主要处理 DOM、网络请求、UI 渲染
3. **封装层多**：`fetch`、`FileReader` 等已经封装了二进制处理

**Node.js 必须掌握的原因**：

1. **文件系统操作**：读取/写入文件都是二进制数据
2. **网络通信**：TCP/HTTP 传输的是二进制流
3. **加密操作**：加密库的输入输出都是 Buffer
4. **性能优化**：二进制处理比字符串更高效

## 三、Buffer 的内存机制

### 3.1 堆外内存是什么？

**V8 的内存结构**：

```
┌─────────────────────────────────────┐
│         V8 堆内存（托管）            │
│  ├─ Code 段（代码）                  │
│  ├─ Stack（调用栈）                  │
│  └─ Heap（对象、字符串等）            │
├─────────────────────────────────────┤
│      Buffer 堆外内存（非托管）         │
│  └─ 原始内存，V8 无法直接 GC          │
└─────────────────────────────────────┘
```

**关键点**：

- **堆内内存**：由 V8 管理，受垃圾回收（GC）控制
- **堆外内存**：由 C++ 层管理，V8 只持有引用

```javascript
// Buffer 在堆外分配内存
const buf = Buffer.alloc(1024); // 在 C++ 层分配 1KB

// V8 只有一个 JavaScript 对象包装
console.log(buf); // <Buffer 00 00 00 ...>
```

### 3.2 Buffer 池（8KB 以下）

**Node.js 的优化策略**：

为了提高性能，Node.js 为小 Buffer（≤ 8KB）实现了**内存池**。

```javascript
// 小 Buffer 使用内存池（预分配 8KB）
const buf1 = Buffer.alloc(1024); // 从池中分配
const buf2 = Buffer.alloc(2048); // 从池中分配

// 大 Buffer 直接分配
const buf3 = Buffer.alloc(10 * 1024); // 直接分配 10KB
```

**内存池的工作原理**：

```
Buffer Pool (8KB)
├── [已用 1KB][空闲 7KB]
├── [已用 2KB][空闲 6KB]
└── [已用 4KB][空闲 4KB]
```

**性能对比**：

```javascript
// 测试：分配 1000 次 1KB Buffer

console.time("Buffer.alloc(1024) x 1000");
for (let i = 0; i < 1000; i++) {
  Buffer.alloc(1024);
}
console.timeEnd("Buffer.alloc(1024) x 1000");
// 约 1-2ms（使用内存池）

console.time("Buffer.alloc(10KB) x 1000");
for (let i = 0; i < 1000; i++) {
  Buffer.alloc(10 * 1024);
}
console.timeEnd("Buffer.alloc(10KB) x 1000");
// 约 10-20ms（直接分配）
```

### 3.3 alloc vs allocUnsafe

**alloc（安全）**：

```javascript
const buf = Buffer.alloc(1024);
console.log(buf[0]); // 0（已初始化）
```

- 分配内存并初始化为 0
- 较慢，但安全
- 适合敏感数据（密码、密钥）

**allocUnsafe（快速）**：

```javascript
const buf = Buffer.allocUnsafe(1024);
console.log(buf[0]); // 可能是任意值（未初始化）
```

- 分配内存但不初始化
- 更快，但包含旧数据
- 需要手动覆盖：`buf.fill(0)`

**性能对比**：

```javascript
console.time("alloc");
for (let i = 0; i < 10000; i++) {
  Buffer.alloc(1024);
}
console.timeEnd("alloc");
// 约 15ms

console.time("allocUnsafe");
for (let i = 0; i < 10000; i++) {
  Buffer.allocUnsafe(1024);
}
console.timeEnd("allocUnsafe");
// 约 5ms（快 3 倍）
```

**使用建议**：

```javascript
// ✅ 敏感数据用 alloc
const passwordBuffer = Buffer.alloc(32);
crypto.randomFillSync(passwordBuffer);

// ✅ 性能敏感场景用 allocUnsafe + fill
const buf = Buffer.allocUnsafe(1024);
buf.fill(0); // 手动初始化

// ❌ 不要直接使用 allocUnsafe 的数据
const buf = Buffer.allocUnsafe(1024);
fs.writeSync(fd, buf); // 可能泄露旧数据！
```

## 四、Buffer 核心操作

### 4.1 创建 Buffer

```javascript
// 方式 1：指定大小
const buf1 = Buffer.alloc(1024); // 分配 1KB，初始化为 0
const buf2 = Buffer.allocUnsafe(1024); // 分配 1KB，不初始化

// 方式 2：从数组创建
const buf3 = Buffer.from([0x01, 0x02, 0x03, 0x04]);

// 方式 3：从字符串创建
const buf4 = Buffer.from("hello", "utf-8");

// 方式 4：从 ArrayBuffer 创建（与前端互操作）
const arrayBuffer = new ArrayBuffer(1024);
const buf5 = Buffer.from(arrayBuffer);

// 方式 5：从 Buffer 创建（浅拷贝）
const buf6 = Buffer.from(buf4); // 共享底层内存
```

### 4.2 读写数据

```javascript
const buf = Buffer.alloc(8);

// 写入不同类型的数据
buf.writeUInt8(0xff, 0); // 写入 8 位无符号整数
buf.writeUInt16BE(0x1234, 2); // 写入 16 位大端序整数
buf.writeUInt32LE(0x12345678, 4); // 写入 32 位小端序整数
buf.writeDoubleBE(3.14, 6); // 写入双精度浮点数

// 读取数据
console.log(buf.readUInt8(0)); // 255
console.log(buf.readUInt16BE(2)); // 4660
console.log(buf.readUInt32LE(4)); // 2018915346
console.log(buf.readDoubleBE(6)); // 3.14
```

**字节序（Endianness）**：

```javascript
const buf = Buffer.alloc(4);
const value = 0x12345678;

// 大端序（Big-Endian）：高位在前
buf.writeUInt32BE(value, 0);
console.log(buf); // <Buffer 12 34 56 78>

// 小端序（Little-Endian）：低位在前
buf.writeUInt32LE(value, 0);
console.log(buf); // <Buffer 78 56 34 12>
```

### 4.3 字符串编码

```javascript
const buf = Buffer.from("你好世界", "utf-8");

// 转换为不同编码
console.log(buf.toString("utf-8")); // '你好世界'
console.log(buf.toString("hex")); // 'e4bda0e5a5bde4b896e7958c'
console.log(buf.toString("base64")); // '5L2g5aW95LiW55WM'

// 其他编码
const buf2 = Buffer.from("hello", "utf-16le");
console.log(buf2.toString("hex")); // '680065006c006c006f00'

// Latin1（单字节编码）
const buf3 = Buffer.from("hello", "latin1");
console.log(buf3.length); // 5（每个字符 1 字节）
```

### 4.4 Buffer 切片与拼接

```javascript
const buf = Buffer.from("Hello World");

// 切片（浅拷贝，共享内存）
const slice = buf.slice(0, 5);
console.log(slice.toString()); // 'Hello'

// 修改切片会影响原 Buffer
slice[0] = 0x68; // 'H' -> 'h'
console.log(buf.toString()); // 'hello World'

// 拼接（深拷贝）
const buf1 = Buffer.from("Hello");
const buf2 = Buffer.from("World");
const concatenated = Buffer.concat([buf1, buf2]);
console.log(concatenated.toString()); // 'HelloWorld'
```

## 五、实战案例

### 5.1 文件上传的二进制处理

**场景**：用户上传图片，需要验证格式并提取元数据。

```javascript
const fs = require("fs");
const http = require("http");

const server = http.createServer((req, res) => {
  if (req.url === "/upload" && req.method === "POST") {
    const chunks = [];

    req.on("data", (chunk) => {
      chunks.push(chunk);
    });

    req.on("end", () => {
      const buffer = Buffer.concat(chunks);

      // 验证图片格式
      const fileType = detectImageType(buffer);

      if (fileType === "unknown") {
        res.writeHead(400);
        res.end("Invalid image format");
        return;
      }

      // 提取图片信息
      const info = extractImageInfo(buffer, fileType);

      // 保存文件
      fs.writeFileSync(`uploads/${Date.now()}.${fileType}`, buffer);

      res.writeHead(200, { "Content-Type": "application/json" });
      res.end(JSON.stringify(info));
    });
  }
});

server.listen(3000);

// 检测图片类型
function detectImageType(buffer) {
  // JPEG: FF D8 FF
  if (buffer[0] === 0xff && buffer[1] === 0xd8 && buffer[2] === 0xff) {
    return "jpg";
  }

  // PNG: 89 50 4E 47
  if (
    buffer[0] === 0x89 &&
    buffer[1] === 0x50 &&
    buffer[2] === 0x4e &&
    buffer[3] === 0x47
  ) {
    return "png";
  }

  // GIF: 47 49 46 38
  if (
    buffer[0] === 0x47 &&
    buffer[1] === 0x49 &&
    buffer[2] === 0x46 &&
    buffer[3] === 0x38
  ) {
    return "gif";
  }

  // WebP: 52 49 46 46 ... 57 45 42 50
  if (
    buffer[0] === 0x52 &&
    buffer[1] === 0x49 &&
    buffer[2] === 0x46 &&
    buffer[3] === 0x46 &&
    buffer[8] === 0x57 &&
    buffer[9] === 0x45 &&
    buffer[10] === 0x42 &&
    buffer[11] === 0x50
  ) {
    return "webp";
  }

  return "unknown";
}

// 提取图片信息
function extractImageInfo(buffer, type) {
  const info = {
    type,
    size: buffer.length,
  };

  if (type === "jpg") {
    // JPEG 尺寸信息在标记中
    let i = 2;
    while (i < buffer.length) {
      if (buffer[i] === 0xff && buffer[i + 1] === 0xc0) {
        info.height = buffer.readUInt16BE(i + 5);
        info.width = buffer.readUInt16BE(i + 7);
        break;
      }
      i += 2 + buffer.readUInt16BE(i + 2);
    }
  } else if (type === "png") {
    // PNG 尺寸在 IHDR chunk
    info.width = buffer.readUInt32BE(16);
    info.height = buffer.readUInt32BE(20);
  }

  return info;
}
```

### 5.2 TCP 数据包解析

**场景**：实现一个简单的 TCP 协议，处理二进制数据包。

```javascript
const net = require("net");

const server = net.createServer((socket) => {
  const buffer = Buffer.alloc(0);

  socket.on("data", (data) => {
    // 追加数据到缓冲区
    buffer = Buffer.concat([buffer, data]);

    // 尝试解析数据包
    while (buffer.length >= 4) {
      // 读取数据包长度（前 4 字节）
      const packetLength = buffer.readUInt32BE(0);

      // 检查是否收到完整数据包
      if (buffer.length < 4 + packetLength) {
        break; // 等待更多数据
      }

      // 提取数据包
      const packet = buffer.slice(4, 4 + packetLength);
      handlePacket(packet);

      // 移除已处理的数据
      buffer = buffer.slice(4 + packetLength);
    }
  });

  function handlePacket(packet) {
    // 解析数据包
    const messageType = packet.readUInt8(0);
    const payload = packet.slice(1);

    console.log("Received:", {
      type: messageType,
      data: payload.toString("utf-8"),
    });

    // 回复
    const response = Buffer.from(`Echo: ${payload.toString("utf-8")}`);
    const responsePacket = Buffer.alloc(4 + response.length);
    responsePacket.writeUInt32BE(response.length, 0);
    response.set(response, 4);
    socket.write(responsePacket);
  }
});

server.listen(8080);
```

### 5.3 图片处理（Buffer 与 Stream 结合）

```javascript
const fs = require("fs");
const sharp = require("sharp");

// 处理图片上传
async function processImage(inputPath, outputPath) {
  // 读取文件到 Buffer
  const inputBuffer = fs.readFileSync(inputPath);

  // 获取图片信息
  const metadata = await sharp(inputBuffer).metadata();
  console.log("Original:", metadata);

  // 处理图片
  const processedBuffer = await sharp(inputBuffer)
    .resize(800, 800, {
      fit: "inside",
      withoutEnlargement: true,
    })
    .jpeg({ quality: 85 })
    .toBuffer();

  // 写入文件
  fs.writeFileSync(outputPath, processedBuffer);

  console.log("Processed:", {
    originalSize: inputBuffer.length,
    processedSize: processedBuffer.length,
    ratio:
      ((processedBuffer.length / inputBuffer.length) * 100).toFixed(2) + "%",
  });

  return processedBuffer;
}

// 使用
processImage("input.jpg", "output.jpg");
```

### 5.4 加密操作

```javascript
const crypto = require("crypto");

// 密码哈希
function hashPassword(password, salt) {
  const hash = crypto.createHmac("sha256", salt);
  hash.update(password);
  return hash.digest("hex");
}

// AES 加密
function encrypt(text, key) {
  const iv = crypto.randomBytes(16);
  const cipher = crypto.createCipheriv("aes-256-cbc", Buffer.from(key), iv);

  let encrypted = cipher.update(text, "utf8", "hex");
  encrypted += cipher.final("hex");

  return {
    iv: iv.toString("hex"),
    data: encrypted,
  };
}

// AES 解密
function decrypt(encryptedData, key) {
  const iv = Buffer.from(encryptedData.iv, "hex");
  const encrypted = encryptedData.data;

  const decipher = crypto.createDecipheriv("aes-256-cbc", Buffer.from(key), iv);

  let decrypted = decipher.update(encrypted, "hex", "utf8");
  decrypted += decipher.final("utf8");

  return decrypted;
}

// 使用
const key = crypto.randomBytes(32); // 256 位密钥
const encrypted = encrypt("Hello World", key);
console.log("Encrypted:", encrypted);

const decrypted = decrypt(encrypted, key);
console.log("Decrypted:", decrypted);
```

## 六、Buffer 性能优化

### 6.1 避免频繁创建小 Buffer

```javascript
// ❌ 不好：频繁创建小 Buffer
for (let i = 0; i < 1000; i++) {
  const buf = Buffer.alloc(1024);
  // 使用 buf...
}

// ✅ 好：复用 Buffer
const buf = Buffer.alloc(1024);
for (let i = 0; i < 1000; i++) {
  buf.fill(0); // 清空
  // 使用 buf...
}
```

### 6.2 使用 Stream 处理大文件

```javascript
// ❌ 不好：一次性读取大文件到内存
const fs = require("fs");
const data = fs.readFileSync("large-file.txt"); // 可能导致内存溢出

// ✅ 好：使用 Stream 流式处理
const stream = fs.createReadStream("large-file.txt");
let hash = crypto.createHash("sha256");

stream.on("data", (chunk) => {
  hash.update(chunk); // 分块更新哈希
});

stream.on("end", () => {
  console.log("Hash:", hash.digest("hex"));
});
```

### 6.3 Buffer 池的陷阱

```javascript
// ⚠️ 陷阱：小 Buffer 共享内存池
const buf1 = Buffer.alloc(1024);
const buf2 = Buffer.alloc(1024);

// 修改 buf1 可能影响 buf2（如果它们共享同一个池）
// 但实际上 alloc 会清零，所以这个陷阱较少见

// 真正的陷阱：allocUnsafe
const buf1 = Buffer.allocUnsafe(1024);
const buf2 = Buffer.allocUnsafe(1024);

// 如果它们共享内存池，修改 buf1 会影响 buf2
```

## 七、常见问题

### Q1: Buffer 和 Uint8Array 的区别？

**A**: Buffer 继承自 Uint8Array，但有额外的方法：

```javascript
const buf = Buffer.from("hello");
const uint8 = new Uint8Array(buf);

console.log(buf instanceof Uint8Array); // true
console.log(buf.toString); // [Function: toString]
console.log(uint8.toString); // [Function: toString]
```

### Q2: 为什么 Buffer[0] 返回数字而不是字节？

**A**: 为了方便使用，Buffer 的索引访问返回数字（0-255）：

```javascript
const buf = Buffer.from([0xff]);
console.log(buf[0]); // 255（不是 -1）

// 如果需要无符号访问，使用 & 0xff
console.log(buf[0] & 0xff); // 255
```

### Q3: Buffer 有内存泄漏吗？

**A**: Buffer 本身不会泄漏，但引用会：

```javascript
// ❌ 全局引用导致泄漏
const cache = [];
function process(data) {
  cache.push(Buffer.from(data)); // 永远不释放
}

// ✅ 限制缓存大小
const cache = [];
const MAX_CACHE = 100;
function process(data) {
  cache.push(Buffer.from(data));
  if (cache.length > MAX_CACHE) {
    cache.shift(); // 移除最旧的
  }
}
```

### Q4: 如何调试 Buffer 问题？

**A**: 使用调试工具：

```javascript
// 打印 Buffer 内容
const buf = Buffer.from("hello");
console.log(buf); // <Buffer 68 65 6c 6c 6f>
console.log(buf.toString("hex")); // 68656c6c6f
console.log(buf.toString("utf8")); // hello

// 检查 Buffer 类型和大小
console.log(Buffer.isBuffer(buf)); // true
console.log(buf.length); // 5

// JSON 序列化
console.log(JSON.stringify(buf)); // {"type":"Buffer","data":[104,101,108,108,111]}
```
