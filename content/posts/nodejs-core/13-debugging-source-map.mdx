## 前言

在 Node.js 后端开发中，**调试**是解决问题的核心技能。理解 Inspector 协议、Source Map、远程调试等技术对于高效开发至关重要。

**核心内容**：

- Inspector 协议与 Chrome DevTools 集成
- node --inspect 与调试器连接
- Source Map 在 Node.js 中的应用
- 实战：远程调试 Docker 容器中的 Node.js 应用

---

## 一、Inspector 协议

### 1.1 启用调试

```bash
node --inspect app.js
```

### 1.2 Chrome DevTools 调试

1. 打开 `chrome://inspect`
2. 点击 "inspect"
3. 在 DevTools 中调试

---

## 二、VSCode 调试

### 2.1 launch.json

```json
{
  "version": "0.2.0",
  "configurations": [
    {
      "type": "node",
      "request": "launch",
      "name": "Debug",
      "program": "${workspaceFolder}/app.js"
    }
  ]
}
```

### 2.2 断点调试

```javascript
// app.js
const express = require("express");

const app = express();

app.get("/", (req, res) => {
  debugger; // 断点
  res.send("Hello");
});

app.listen(3000);
```

---

## 三、Source Map

### 3.1 生成 Source Map

```javascript
// tsconfig.json
{
  "compilerOptions": {
    "sourceMap": true
  }
}
```

### 3.2 启用 Source Map

```bash
node --enable-source-map app.js
```

---

## 四、远程调试

### 4.1 Docker 调试

```bash
docker run -p 9229:9229 node --inspect=0.0.0.0:9229 app.js
```

### 4.2 SSH 隧道

```bash
ssh -L 9229:localhost:9229 user@server
```
