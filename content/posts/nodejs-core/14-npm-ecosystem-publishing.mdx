## 前言

在 Node.js 后端开发中，**NPM 生态系统**是代码组织的基础。理解 package.json、依赖管理、monorepo 等概念对于构建可维护的项目至关重要。

**核心内容**：

- package.json 详细字段解析
- 依赖版本范围与锁定机制
- pnpm 的硬链接与符号链接
- monorepo 工具对比
- 实战：发布私有 npm 包、搭建私有 npm 镜像

---

## 一、package.json

### 1.1 核心字段

```json
{
  "name": "my-app",
  "version": "1.0.0",
  "description": "My application",
  "main": "index.js",
  "type": "module",
  "bin": {
    "my-cli": "./cli.js"
  },
  "scripts": {
    "start": "node index.js",
    "test": "jest"
  },
  "dependencies": {
    "express": "^4.18.0"
  },
  "devDependencies": {
    "jest": "^29.0.0"
  },
  "engines": {
    "node": ">=18.0.0"
  },
  "os": ["!win32"],
  "cpu": ["x64", "arm64"]
}
```

---

## 二、依赖管理

### 2.1 版本范围

```json
{
  "dependencies": {
    "express": "^4.18.0", // >=4.18.0 <5.0.0
    "lodash": "~4.17.0", // >=4.17.0 <4.18.0
    "moment": "2.29.0" // 精确版本
  }
}
```

### 2.2 pnpm

```bash
npm install -g pnpm

pnpm install
pnpm add express
```

---

## 三、monorepo

### 3.1 pnpm workspace

```json
// pnpm-workspace.yaml
packages:
  - 'packages/*'
```

```bash
pnpm install
```

---

## 四、发布 npm 包

### 4.1 准备发布

```bash
npm login
npm publish
```

### 4.2 私有包

```bash
npm publish --access=restricted
```

---

## 五、总结

**核心要点**：

1. package.json 定义项目元数据
2. pnpm 更高效的依赖管理
3. monorepo 管理多包项目
4. npm publish 发布包
