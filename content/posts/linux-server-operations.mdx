# Linux 服务器运维：常用命令与故障排查指南

无论是后端开发还是运维，Linux 服务器管理都是必备技能。本文整理了最常用的命令和故障排查技巧，帮你快速定位和解决问题。

## 一、系统信息

### 1.1 基本信息

```bash
# 系统版本
cat /etc/os-release
uname -a

# 内核版本
uname -r

# 主机名
hostname
hostnamectl

# 系统运行时间
uptime

# 当前用户
whoami
who
w
```

### 1.2 硬件信息

```bash
# CPU 信息
lscpu
cat /proc/cpuinfo | grep "model name" | head -1
nproc  # CPU 核心数

# 内存信息
free -h
cat /proc/meminfo

# 磁盘信息
lsblk
fdisk -l
df -h

# 网络接口
ip addr
ip link
```

## 二、进程管理

### 2.1 查看进程

```bash
# 查看所有进程
ps aux
ps -ef

# 查找进程
ps aux | grep nginx
pgrep nginx
pidof nginx

# 进程树
pstree
pstree -p

# 实时监控
top
htop  # 更友好

# 按资源排序
ps aux --sort=-%cpu | head -10  # CPU 占用前 10
ps aux --sort=-%mem | head -10  # 内存占用前 10
```

### 2.2 进程控制

```bash
# 后台运行
nohup ./server.sh &
./server.sh &

# 查看后台任务
jobs

# 前台恢复
fg %1

# 暂停进程
Ctrl+Z

# 终止进程
kill <pid>
kill -9 <pid>  # 强制终止
kill -15 <pid> # 正常终止（默认）

# 终止所有同名进程
killall nginx
pkill -f "node server.js"
```

### 2.3 Systemd 服务

```bash
# 启动服务
systemctl start nginx

# 停止服务
systemctl stop nginx

# 重启服务
systemctl restart nginx

# 重载配置
systemctl reload nginx

# 查看状态
systemctl status nginx

# 开机自启
systemctl enable nginx
systemctl disable nginx

# 查看日志
journalctl -u nginx
journalctl -u nginx -f  # 实时
journalctl --since "1 hour ago"
```

## 三、资源监控

### 3.1 CPU 监控

```bash
# 实时 CPU 使用
top
htop

# CPU 统计
mpstat 1 5  # 每秒采样，共 5 次

# 查看负载
uptime
cat /proc/loadavg

# 查看高 CPU 进程
top -c
ps -eo pid,ppid,cmd,%cpu,%mem --sort=-%cpu | head
```

### 3.2 内存监控

```bash
# 内存使用
free -h

# 详细内存信息
cat /proc/meminfo

# 内存使用排行
ps -eo pid,cmd,%mem --sort=-%mem | head

# 实时监控
vmstat 1

# 查看进程内存
pmap <pid>
smem -t -k -s rss
```

### 3.3 磁盘监控

```bash
# 磁盘使用
df -h

# 目录大小
du -sh /var/log
du -h --max-depth=1 /var

# 查找大文件
find / -type f -size +100M 2>/dev/null

# 磁盘 I/O
iostat -x 1
iotop

# 查看打开的文件
lsof | grep deleted  # 已删除但仍占用空间的文件
```

### 3.4 网络监控

```bash
# 网络连接
netstat -tulnp
ss -tulnp

# 网络流量
iftop
nethogs  # 按进程显示

# 连接统计
netstat -an | grep :80 | wc -l

# 实时连接
watch -n 1 'netstat -an | grep :80 | wc -l'
```

## 四、日志分析

### 4.1 系统日志

```bash
# 系统日志
/var/log/syslog      # Ubuntu/Debian
/var/log/messages    # CentOS/RHEL

# 查看日志
tail -f /var/log/syslog
less /var/log/syslog

# 搜索日志
grep "error" /var/log/syslog
grep -i "error\|fail" /var/log/syslog

# 按时间过滤
grep "Mar 27" /var/log/syslog
awk '/Mar 27 10:00/,/Mar 27 11:00/' /var/log/syslog
```

### 4.2 应用日志

```bash
# Nginx 日志
/var/log/nginx/access.log
/var/log/nginx/error.log

# 查看访问量前 10 的 IP
awk '{print $1}' /var/log/nginx/access.log | sort | uniq -c | sort -rn | head

# 查看状态码分布
awk '{print $9}' /var/log/nginx/access.log | sort | uniq -c | sort -rn

# 查看慢请求
awk '$NF > 1 {print $0}' /var/log/nginx/access.log

# 实时监控
tail -f /var/log/nginx/access.log | grep " 500 "
```

### 4.3 日志管理

```bash
# 日志轮转配置
/etc/logrotate.conf
/etc/logrotate.d/nginx

# 手动轮转
logrotate -f /etc/logrotate.d/nginx

# 清理旧日志
find /var/log -name "*.log.*" -mtime +30 -delete
```

## 五、网络排查

### 5.1 连通性测试

```bash
# Ping 测试
ping google.com
ping -c 5 google.com

# 端口连通
telnet example.com 80
nc -zv example.com 80

# DNS 解析
nslookup example.com
dig example.com
host example.com

# 路由追踪
traceroute example.com
mtr example.com
```

### 5.2 抓包分析

```bash
# 抓取 80 端口
tcpdump -i eth0 port 80

# 抓取特定主机
tcpdump -i eth0 host 192.168.1.100

# 保存到文件
tcpdump -i eth0 -w capture.pcap

# 读取文件
tcpdump -r capture.pcap

# HTTP 请求
tcpdump -i eth0 -A -s 0 'tcp port 80'
```

### 5.3 防火墙

```bash
# iptables
iptables -L -n
iptables -A INPUT -p tcp --dport 80 -j ACCEPT
iptables -A INPUT -p tcp --dport 22 -j DROP

# ufw (Ubuntu)
ufw status
ufw allow 80
ufw allow 443
ufw enable

# firewalld (CentOS)
firewall-cmd --list-all
firewall-cmd --add-port=80/tcp --permanent
firewall-cmd --reload
```

## 六、性能调优

### 6.1 系统参数

```bash
# 查看参数
sysctl -a | grep net.core
sysctl net.core.somaxconn

# 临时修改
sysctl -w net.core.somaxconn=65535

# 永久修改
echo "net.core.somaxconn = 65535" >> /etc/sysctl.conf
sysctl -p
```

常用参数：
```bash
# /etc/sysctl.conf
net.core.somaxconn = 65535
net.ipv4.tcp_max_syn_backlog = 65535
net.ipv4.tcp_tw_reuse = 1
net.ipv4.ip_local_port_range = 1024 65535
vm.swappiness = 10
```

### 6.2 文件描述符

```bash
# 查看限制
ulimit -n

# 临时修改
ulimit -n 65535

# 永久修改
# /etc/security/limits.conf
* soft nofile 65535
* hard nofile 65535
```

### 6.3 磁盘性能

```bash
# 磁盘性能测试
hdparm -Tt /dev/sda

# IO 调度
cat /sys/block/sda/queue/scheduler
echo noop > /sys/block/sda/queue/scheduler  # SSD 用 noop
```

## 七、故障排查案例

### 7.1 服务器响应慢

```bash
# 1. 检查负载
uptime
top

# 2. 检查 CPU
mpstat 1 5
ps -eo pid,cmd,%cpu --sort=-%cpu | head

# 3. 检查内存
free -h
vmstat 1

# 4. 检查磁盘 I/O
iostat -x 1
iotop

# 5. 检查网络
netstat -tulnp
ss -s
```

### 7.2 磁盘空间不足

```bash
# 1. 查看使用情况
df -h

# 2. 找大目录
du -h --max-depth=1 / | sort -hr

# 3. 找大文件
find / -type f -size +100M 2>/dev/null

# 4. 清理日志
find /var/log -name "*.log" -size +100M -exec truncate -s 0 {} \;

# 5. 清理包缓存
apt clean  # Ubuntu
yum clean all  # CentOS

# 6. 删除已卸载的文件
lsof | grep deleted
```

### 7.3 内存不足

```bash
# 1. 查看内存
free -h

# 2. 找内存占用高的进程
ps -eo pid,cmd,%mem --sort=-%mem | head

# 3. 释放缓存
sync && echo 3 > /proc/sys/vm/drop_caches

# 4. 检查 OOM
dmesg | grep -i "out of memory"
grep -i "out of memory" /var/log/syslog
```

### 7.4 端口被占用

```bash
# 1. 查看端口占用
lsof -i :80
netstat -tulnp | grep :80

# 2. 终止进程
kill -9 <pid>
```

## 八、常用脚本

### 8.1 系统监控

```bash
#!/bin/bash
# monitor.sh

ALERT=90
EMAIL="admin@example.com"

# CPU 使用率
CPU=$(top -bn1 | grep "Cpu(s)" | awk '{print $2}' | awk -F. '{print $1}')
if [ $CPU -gt $ALERT ]; then
  echo "CPU usage: $CPU%" | mail -s "CPU Alert" $EMAIL
fi

# 内存使用率
MEM=$(free | grep Mem | awk '{print ($3/$2)*100}' | awk -F. '{print $1}')
if [ $MEM -gt $ALERT ]; then
  echo "Memory usage: $MEM%" | mail -s "Memory Alert" $EMAIL
fi

# 磁盘使用率
DISK=$(df -h | grep -vE '^Filesystem|tmpfs|cdrom' | awk '{ print $5 }' | sed 's/%//g')
for d in $DISK; do
  if [ $d -gt $ALERT ]; then
    echo "Disk usage: $d%" | mail -s "Disk Alert" $EMAIL
  fi
done
```

### 8.2 自动备份

```bash
#!/bin/bash
# backup.sh

DATE=$(date +%Y%m%d)
BACKUP_DIR="/backup"
SOURCE="/var/www"

# 创建备份目录
mkdir -p $BACKUP_DIR

# 备份数据库
mysqldump -u root -p'password' mydb | gzip > $BACKUP_DIR/db_$DATE.sql.gz

# 备份文件
tar -czf $BACKUP_DIR/www_$DATE.tar.gz $SOURCE

# 保留最近 7 天
find $BACKUP_DIR -mtime +7 -delete

echo "Backup completed: $DATE"
```

### 8.3 日志清理

```bash
#!/bin/bash
# log-clean.sh

# 清理超过 30 天的日志
find /var/log -name "*.log.*" -mtime +30 -delete

# 清空超大日志
find /var/log -name "*.log" -size +100M -exec truncate -s 0 {} \;

echo "Log cleanup completed"
```

## 总结

Linux 运维需要持续学习和实践。

**核心技能：**
1. **系统信息** - 了解服务器配置
2. **进程管理** - 控制和监控进程
3. **资源监控** - CPU、内存、磁盘、网络
4. **日志分析** - 快速定位问题
5. **网络排查** - 连通性和抓包
6. **性能调优** - 系统参数优化
7. **故障排查** - 系统化解决问题

**排查思路：**
```
现象 → 检查资源 → 查看日志 → 定位进程 → 解决问题
```

多动手实践，遇到问题先看日志，善用 `man` 命令查阅文档。
