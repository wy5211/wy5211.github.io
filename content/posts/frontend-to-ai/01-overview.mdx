你最近是不是经常刷到这两种内容——

一边是「前端已死」「前端岗位暴跌」的焦虑帖：InfoQ 引用 1.8 亿条招聘信息分析说，2025 年前端岗位同比下降 9.89%，是所有软件岗位里降幅最大的；V2EX 上有人贴出公司主管群发的邮件，原话是「独立的前端工程师大部分会失业，取代你的是产品 / UI / 全栈」。

另一边是有人吃到了红利：idoubi（艾逗笔）从腾讯裸辞，一年半上线了 11 款 AI 出海产品；V2EX 上一个被裁的前端，去年一整年代码全让 AI 写，做出海产品 MRR 做到 2500 美元，宣布全职独立开发；更夸张的 Cal AI，两个高中生用 AI 做拍照识别卡路里的 App，两年做到 3000 万美元营收。

岗位在缩，红利在涨——这中间的差距到底在哪？

**不是谁更努力，而是谁在用新规则玩。**

这个系列就是帮你搞清楚这件事：作为前端，在 AI 这波里怎么找到自己的位置、怎么吃到红利、怎么不被碾过去。不灌鸡汤，不贩卖焦虑，全部用真实数据和一线案例说话。

## 先说结论：前端没死，死的是「只会写页面」的旧身份

InfoQ 大厂圆桌（淘天 / 快手 / 菜鸟的技术负责人同台）有句话，是整个系列最该先记住的：

> 「会 AI 不一定让你不被淘汰，但不会 AI 早晚会被淘汰。淘汰你的从来都是人，不是 AI。」

AI 把「会写代码」的门槛拉到了地板——但**不是所有写代码都贬值**：能被一句话生成的代码贬值了，而前端的另外一些手艺（结构化复杂状态、编排多步、流式渲染）恰恰和 Agent 架构同构，反而**更值钱**。于是能**定义问题、验证、闭环、把 AI 放大成产品**的人被推到天花板。而前端——离用户最近、技术栈最契合 Agent 生态（TypeScript）、又有独立开发的全栈潜力——恰恰是这一轮里**起点最好的身份之一**。

所以问题从来不是「前端还能不能干」，而是「**你想当哪种前端**」：

- 继续做「只会写页面」的那个旧身份——这条路确实在变窄
- 还是升级成「能用 AI 一个人把产品从 0 闭环到赚钱」的新身份——这条路正在变宽

这个系列讲的是后者。

## 看清现实：别被焦虑带偏，也别装睡

选路之前，先建立正确的现实坐标。三个数据摆这：

## 五条路：你可以这样吃到 AI 红利

我搜罗了一圈中英文社区的高赞讨论（掘金 / 知乎 / V2EX / Hacker News / Reddit），前端能走的路，高认同的就这么几条。每条这个系列都会单独开一篇深挖。

### 🥇 路径一：AI Agent 工程师（最高共识）

不做底层模型研发（那是算法工程师的战场），而是把 LLM、RAG、Function Calling、Agent 编排做成**产品功能**。海外有个专门的名字叫 **AI Engineer**。

为什么前端天然占优：Mastra 创始人（Gatsby 联创，⚠️ 他自己在做 TS Agent 框架）透露，YC 最新一批（X25）约 **60-70% 的 Agent 创业公司在用 TypeScript 而非 Python 构建**（趋势信号，非精确统计）。你不用重学语言，前端的组件化、状态管理、流式渲染，正是 LLM 应用 UI 最需要的能力。

👉 详见 [03 AI Agent 工程师](/posts/frontend-to-ai/03-ai-agent-engineer)

### 🥈 路径二：全栈 AI 工程师

借 AI 编码工具把 Node / 后端能力补齐，变成「一人多端」。以前补后端要啃两年，现在 Next.js 全栈框架 + Serverless + AI 辅助，让前后端边界消失。美团履约团队已经把部分前端转去做 Agent 全栈项目。

👉 详见 [04 全栈 AI 工程师](/posts/frontend-to-ai/04-fullstack-ai)

### 🥉 路径三：独立开发 / 出海 SaaS

用 AI 编码工具 + 极简全栈栈，一个人从 0 到赚钱闭环。这是「前端吃 AI 红利」最直接的答案——前端的**全栈能力 + 快速迭代 + 审美**，正是出海 SaaS 的核心竞争力。idoubi、Pieter Levels（Photo AI 年入 300 万美元）都是这条路。

👉 详见 [05 独立开发出海](/posts/frontend-to-ai/05-indie-overseas)

### 路径四：AI 增效型前端专家

不转岗、不创业，而是把 Cursor / Claude Code 用到极致，**让一个人产出过去两三个人的量**，从而成为公司「留下来的人」。转转技术团队实测：系统化用 Cursor 后，团队效率 **+21%**（188 工时 → 149 工时）。

👉 详见 [06 AI 增效型前端](/posts/frontend-to-ai/06-ai-amplified-frontend)

### 路径五：产品工程师

向上游走，做「产品经理 + 前端」。因为前端离用户最近。代表人物就是语雀创始人玉伯——他说「我大概率能经历前端从诞生到消亡的全过程」，然后**用脚投票**：辞职创业做 AI 原生办公产品 YouMind，两个月估值过亿。

## 护城河：把 80% 交给 AI，守住关键 20%

InfoQ 圆桌嘉宾有个判断：AI 像一个 3-4 岁智力的孩子，能完成 80% 工作量，但**剩下那 20% 才是关键，也正是人类的优势**。守住这四样，AI 永远干不掉你：

- **判断力 + 产品思维 + 架构**——AI 能生成代码，但理解不了业务副作用，也保障不了体验确定性
- **深度技术理解**——「细节迟早会从抽象里漏出来，能读懂漏在哪个塔、哪层、哪个房间的人，最值钱」
- **传统工程能力**——性能、稳定性、体验一致性，恰恰是 AI 生成不了的护城河
- **验证能力**——LLM 是「容易生成、难以验证」；强化 TypeScript 类型编程 = 容易生成且容易验证

MIT 有个实证研究：**顶尖科学家从 AI 获益近乎翻倍，下游三分之一受益甚微**。基本功越扎实，越能从 AI 获益。这是整个系列最重要的认知之一。

👉 详见 [08 护城河篇](/posts/frontend-to-ai/08-moat)

## 几个坑：别人花钱买的教训

转型路上最容易踩的坑，全是「别人花钱买的教训」：

- ❌ **盲目卷传统后端**——掘金 2.1 万赞热文的忠告：前端理解的全栈是「纸老虎」，独立开发 90% 精力耗在配环境、查后端 bug、修服务器
- ❌ **从底层原理学 AI**——知乎主流的「Python→高数→机器学习→训练模型」路线会直接劝退工程师
- ❌ **纯 AI Wrapper 套壳红利将尽**——白鲸出海判断，「套壳马上能赚钱」的窗口正在关闭
- ❌ **vibe coding 成瘾**——V2EX 真实案例：有人靠 Copilot 写代码飞快，面试时基本功崩盘
- ❌ **Agent 专职岗其实不好找**——JD 关键词死板，更现实的是把 AI 能力用于现有业务
