上一篇你具备了独立交付产品的能力。这一篇，我们把这股能力往**变现**推一步——前端怎么做 AI 产品、一个人赚到钱。

但先把丑话说前面：这条路是五条活法里**风险最高**的，也是被幸存者偏差包装得最狠的。你能看到的 idoubi、Pieter Levels 都是赚到钱的，但你看不到的是——**国内做独立开发，能赚钱的不超过 20%，MRR（月收入）超过 2000 美元的不超过 2%**。这篇不会给你打鸡血，而是帮你看清：赚钱的是哪 2%，赔钱的又是哪 80%，你属于哪一类。

## 一、先算账：你有多少 runway

在聊任何技术栈之前，先回答一个问题：**如果你接下来 12 个月零收入，你的积蓄撑得住吗？**

这不是吓你，这是独立开发者的头号死因。你以为失败是因为产品没做好，其实大多数人是**钱烧完了、扛不住压力、自己放弃的**。

给你一个决策框架：

> **runway < 6 个月，且没有第二收入源 → 不要裸辞，先在职做 MVP。**

独立开发从 0 到第一笔收入，典型周期是 3-9 个月；要做到稳定的 MRR，通常要 12-24 个月。也就是说，你得准备好「半年到两年不靠这个吃饭」。

所以现实地分成两条子路径：

- **在职试水（强烈推荐起步）**：保住工资，用业余时间做 MVP，验证有人付费了再考虑全职。idoubi 早期、大多数成功独立开发者都是这么起步的；
- **全职独立（高 runway 才行）**：只有当你的积蓄能撑 12 个月以上，或已有稳定副业收入，才考虑裸辞。

这里有个经常被忽略的前提：你能看到的成功案例，背后都有「退路」。idoubi 是**前腾讯裸辞**（有大厂背书 + 积蓄），V2EX 上那个被裁的前端 kulove（可能有补偿缓冲）。他们的「退路」从没在故事里点破，但你抄作业前得先问自己：我有吗？

## 二、为什么前端优势最大——但优势 ≠ 成功

先肯定一句：在所有身份里，前端做独立开发的优势确实是最大的。

- **全栈潜力**：借 AI + Serverless（上一篇讲的），你一个人能搞定前后端；
- **快速迭代**：你是离 UI 最近的人，改产品速度是后端的几倍；
- **审美**：独立产品，长得好不好看直接决定转化率，这正是前端的强项。

但这只是「起跑线优势」。**起跑线靠前，不等于能跑完。** 下面这三道墙，才是淘汰 80% 人的真正原因，而且它们一道比一道跟「技术」无关。

## 三、为什么 80% 赚不到钱：三道墙

### 🧱 第一道墙：英文 / 收款合规

这是国内独立开发者最先撞、也最常被低估的墙。

你要做海外市场，就得能收海外用户的钱。Stripe 是主流，但** Stripe 不支持中国大陆主体直接收款**——你得注册一家美国公司（通常走 Stripe Atlas，一次性约 $500，特拉华州年审约 $300/年），还要处理美国公司的报税义务。嫌麻烦想用 LemonSqueezy？它可以省掉税务（它是 Merchant of Record，替你收税），但抽成更高（约 5% + 50¢）。

光是「能把钱收进来」这一步，就劝退了一大半人。再加上产品文案、客服、社媒运营都得用英文——**英文能力门槛**比技术门槛更早出现。

### 🧱 第二道墙：流量

产品做出来了，没人用，等于零。

- **SEO 冷启动要 6-12 个月**：新站没有权重，谷歌不会给你流量，你得持续写内容、做外链；
- **社媒买量要看 ROI**：投广告获客（CAC）得低于用户终身价值（LTV），业内经验是 LTV/CAC > 3 才健康，很多人投着投着发现越投越亏；
- **idoubi 的解法是「天下武功唯快不破」**：靠抢热点（HuggingFace Space + Google Trends + GitHub trending）、抢注域名、vibe coding 快速上线，打的是短平快的流量——但这对执行力要求极高。

**流量这件事，比写代码难十倍，而且无法靠 AI 捷径。** 这是技术出身的人最容易栽的地方。

### 🧱 第三道墙：产品

最后才是产品本身。白鲸出海有个判断很清醒：

> 「纯 AI Wrapper 套壳『马上能赚钱』这件事大概率要结束了，接下来需要更深度的产品化。」

什么意思？2023-2024 年，你套个壳、给 GPT 套个界面就能赚钱。2026 年，这种红利已经关了。你得有真正的工作流、数据壁垒、或者用户网络效应——也就是说，**「会调 API」已经不值钱了，得回到「做产品」本身**。

## 四、出海硬成本清单：让你算账

别只盯着技术栈，先算清楚你每个月固定烧多少钱：

| 项目                | 月成本（约）      | 说明                                            |
| ------------------- | ----------------- | ----------------------------------------------- |
| 美国公司注册 / 年审 | $30-50            | Stripe Atlas ~$500 一次性 + 特拉华年审 ~$300/年 |
| 收款（Stripe）      | 2.9% + $0.30/笔   | 无月费；LemonSqueezy ~5% + 50¢                  |
| 域名                | $1-5              | 年化摊销                                        |
| Vercel / Cloudflare | $0-20             | 免费层通常够 MVP                                |
| LLM API             | $10-200+          | 随用量波动，**最大变量**                        |
| **合计（MVP 期）**  | **约 $50-300/月** | **不含你的个人生活成本**                        |

把这张表加上你的生活费（比如一线城市 1 万/月），再乘以 12——那就是你需要的最低 runway。算完你会对「要不要裸辞」清醒很多。

## 五、标准技术栈 + idoubi 的 SOP

如果你算完账决定上路，技术栈基本是标准化的：

**Next.js + Vercel + Supabase + Stripe + Cloudflare R2**，日常用 Cursor，大任务用 Claude Code。（这套和我们前几篇讲的一脉相承，所以全栈那篇是地基。）

至于「怎么找到能赚钱的点」，idoubi（艾逗笔，前腾讯，一年半做了 11 款出海产品）有一套可复制的 SOP：

> 发现热点（HuggingFace Space + Google Trends + GitHub trending）→ 抢注域名 → vibe coding 快速上线 → 获取流量（SEO + 社媒）→ 变现（订阅 + 谷歌广告）。

他的心法是：**「梦想可以很大，切入应该要小。先起飞，再加油。」** 这句话值得记住——独立开发最大的敌人是「想做完美的产品」，而赢家都在做「能上线的小东西」。

## 六、案例表：看清楚哪些能复制，哪些不能

最后给你一张案例表，但请注意——**这里的每一条都标了「代表性」，因为不是所有成功都能复制到你身上：**

| 人物 / 产品      | 背景              | 代表性                  | 成绩                                   |
| ---------------- | ----------------- | ----------------------- | -------------------------------------- |
| idoubi（艾逗笔） | 前腾讯，2023 裸辞 | ✅ 前端可复制           | 11 款产品，ShipAny 预售 4h 破 1 万美金 |
| kulove           | 被裁前端          | ✅ 前端可复制           | MRR ~$2500（正好压在 2% 临界线）       |
| Pieter Levels    | PHP solo 一人公司 | ⚠️ 非典型前端           | Photo AI 年入 $300 万                  |
| Cal AI           | 两名高中生        | ⚠️ 非前端、时机不可复制 | 两年 $3000 万 ARR                      |

idoubi 和 kulove 是你能学的——前端背景、近两年的时机、从 0 做起。**但 Pieter Levels 和 Cal AI 你学不来**：Levels 是积累了十年的个人品牌 + PHP 时代的老兵，Cal AI 是踩中了 2023 年卡路里识别风口的两个高中生。把他们当激励可以，当路径会害你。

## 七、毕业项目 + 这条路适合谁

毕业项目：

> **上线一个能收到 Stripe 测试支付的 MVP**（哪怕只有付费墙 + 一个核心 AI 功能）。验收：测试支付成功页截图 + 公开可访问的 URL。

注意，是「能收到钱」，不是「产品很完整」——**验证付费意愿，比打磨产品重要十倍。**

**这条路适合谁**：有积蓄缓冲（或可承担风险）、追求自由度的资深前端；想对冲打工风险的人。**不适合**：runway 不足 6 个月、无法承受 12 个月零收入的人——请先在职试水。
