作为开发者，我们每天都要在终端里敲命令。但你有没有想过，那些用了几十年的 `ls`、`grep`、`find` 命令，其实早就有了更好的替代品？

本文介绍一套现代命令行工具，它们更快、更美、更智能。花 10 分钟配置，换来每天几小时的时间节省。

## 🚀 工具总览

| 老工具   | 现代替代       | 提升点                      |
| -------- | -------------- | --------------------------- |
| `ls`     | `exa` / `lsd`  | 彩色输出、Git 状态、图标    |
| `cat`    | `bat`          | 语法高亮、行号、Git diff    |
| `grep`   | `ripgrep (rg)` | 快 10-100 倍、自动忽略 .git |
| `find`   | `fd`           | 更快、更简洁的语法          |
| `cd`     | `zoxide (z)`   | 智能跳转、记住历史          |
| `Ctrl+R` | `fzf`          | 模糊搜索、预览              |

## 安装方式

大多数工具可以用 Homebrew 一键安装：

```bash
# 核心工具全家桶
brew install exa bat ripgrep fd zoxide fzf

# 可选：更花哨的 ls 替代
brew install lsd
```

## 1. exa / lsd：彩色版的 ls

### exa

`exa` 是 `ls` 的现代替代品，支持彩色输出、Git 状态、文件图标。

```bash
# 基础用法
exa                    # 等同于 ls
exa -la                # 等同于 ls -la，但更好看
exa --tree             # 树形显示
exa --tree --level=2   # 限制层级
exa --git              # 显示 Git 状态
exa --icons            # 显示文件图标（需要 Nerd Font）
```

### lsd

如果你想要更炫酷的效果，`lsd` 支持更多图标和颜色主题：

```bash
lsd
lsd -la
lsd --tree
lsd --icon-theme fancy  # 更丰富的图标
```

### 配置别名

在 `~/.zshrc` 或 `~/.bashrc` 中添加：

```bash
# exa 别名
alias ls='exa'
alias ll='exa -la'
alias lt='exa --tree --level=2'
alias lg='exa -la --git'

# 或者用 lsd
# alias ls='lsd'
# alias ll='lsd -la'
```

## 2. bat：带语法高亮的 cat

`bat` 是 `cat` 的增强版，自带语法高亮、行号、Git 修改标记。

```bash
# 基础用法
bat filename.txt
bat package.json
bat src/index.ts

# 显示行号（默认开启）
bat --number filename.txt

# 不显示行号
bat --style=plain filename.txt

# 同时显示多个文件
bat src/*.ts

# 指定语言高亮
bat --language=json data.txt
```

### 配置别名

```bash
alias cat='bat'
```

### 实用技巧

```bash
# 查看所有支持的语言
bat --list-languages

# 查看所有主题
bat --list-themes

# 设置默认主题（添加到配置文件）
# ~/.config/bat/config
echo "--theme=Monokai Extended" > ~/.config/bat/config
```

## 3. ripgrep (rg)：比 grep 快 100 倍

`ripgrep` 是目前最快的搜索工具，自动忽略 `.git`、`node_modules` 等目录。

```bash
# 基础搜索
rg "function"              # 搜索当前目录
rg "useState" src/         # 指定目录
rg "TODO" --type js        # 只搜 JS 文件

# 显示行号和上下文
rg "error" -C 3            # 显示前后 3 行
rg "error" -A 5            # 显示后 5 行
rg "error" -B 2            # 显示前 2 行

# 正则搜索
rg "import.*from"          # 支持正则
rg -i "error"              # 忽略大小写

# 只显示文件名
rg -l "deprecated"         # 只列出包含的文件

# 统计匹配次数
rg -c "console.log"
```

### 与 grep 对比

```bash
# grep 写法
grep -r "function" --include="*.js" --exclude-dir=node_modules .

# rg 写法（自动处理）
rg "function" -t js
```

### 配置别名

```bash
alias grep='rg'
```

## 4. fd：比 find 更简单更快

`fd` 是 `find` 的现代替代品，语法更简洁，速度更快。

```bash
# 基础搜索
fd pattern                 # 搜索文件名包含 pattern 的文件
fd "test"                  # 搜索包含 test 的文件
fd "" --type f             # 列出所有文件

# 按类型搜索
fd --type f "config"       # 只搜文件
fd --type d "node_modules" # 只搜目录
fd --type x "node"         # 只搜可执行文件

# 按扩展名搜索
fd -e json                 # 所有 JSON 文件
fd -e ts -e js "api"       # TS 或 JS 文件

# 排除目录
fd "test" --exclude node_modules --exclude dist

# 正则搜索
fd "^api.*service$"

# 执行命令
fd -e jpg -x convert {} {.}.png  # 对每个 jpg 执行命令
```

### 与 find 对比

```bash
# find 写法
find . -name "*.js" -type f -not -path "*/node_modules/*"

# fd 写法
fd -e js --exclude node_modules
```

### 配置别名

```bash
alias find='fd'
```

## 5. zoxide：智能目录跳转

`zoxide` 记住你访问过的目录，支持模糊匹配跳转，告别漫长的 `cd` 路径。

```bash
# 初始化（添加到 ~/.zshrc）
eval "$(zoxide init zsh)"

# 基础用法
z project        # 跳转到包含 project 的目录
z blog           # 跳转到包含 blog 的目录
z src comp       # 跳转到路径包含 src 和 comp 的目录

# 查看历史
z -l             # 列出所有记录的目录

# 交互式选择（配合 fzf）
zi               # 弹出 fzf 选择目录
```

### 配置

```bash
# ~/.zshrc
eval "$(zoxide init zsh)"

# 可选：用 z 替代 cd
alias cd='z'
```

### 实际场景

```bash
# 传统方式
cd ~/Desktop/code/company/project/frontend/src/components

# zoxide 方式（第一次之后）
z front comp
# 或者更短
z f c
```

## 6. fzf：模糊搜索神器

`fzf` 是命令行模糊搜索工具，可以搜索文件、历史命令、进程等。

### 基础安装

```bash
brew install fzf

# 安装 Shell 集成
$(brew --prefix)/opt/fzf/install
```

### 历史命令搜索

按 `Ctrl+R` 后，输入模糊关键词：

```
> git push
  123  git push origin main
  456  git push --force-with-lease
  789  git push -u origin feature
```

### 文件搜索

```bash
# 模糊搜索文件
fzf

# 预览文件内容
fzf --preview 'bat --style=numbers --color=always {} | head -500'

# 搜索后打开
vim $(fzf)
code $(fzf)
```

### 进程搜索

```bash
# 杀进程
kill -9 $(ps aux | fzf | awk '{print $2}')
```

### 与其他工具组合

```bash
# 搜索文件内容
rg --line-number "" | fzf

# Git 分支切换
git checkout $(git branch | fzf)

# Docker 容器操作
docker exec -it $(docker ps | fzf | awk '{print $1}') sh
```

### 高级配置

```bash
# ~/.zshrc

# 默认预览
export FZF_DEFAULT_OPTS="--preview 'bat --style=numbers --color=always {} 2>/dev/null | head -500'"

# 使用 fd 作为搜索源
export FZF_DEFAULT_COMMAND="fd --type f --exclude node_modules --exclude .git"

# Ctrl+T 搜索文件
export FZF_CTRL_T_COMMAND="$FZF_DEFAULT_COMMAND"

# Alt+C 跳转目录
export FZF_ALT_C_COMMAND="fd --type d --exclude node_modules"
```

## 7. 组合技：工具联动

这些工具组合起来威力更大：

### 搜索文件并预览

```bash
# 用 fd 搜索，fzf 选择，bat 预览
fd -t f | fzf --preview 'bat --style=numbers --color=always {}'
```

### 搜索内容并编辑

```bash
# 用 rg 搜索，fzf 选择，直接打开
vim $(rg -l "pattern" | fzf)
```

### 快速跳转并执行

```bash
# 跳转到项目并启动
z project && npm run dev

# 交互式跳转
zi && npm run dev
```

### Git 增强

```bash
# 交互式 git add
git add $(git status --short | fzf | awk '{print $2}')

# 交互式 git checkout
git checkout $(git branch -a | fzf)

# 查看 git log 并选择
git log --oneline | fzf
```

## 8. 一键配置脚本

把以下内容添加到 `~/.zshrc`：

```bash
# ============ 现代命令行工具配置 ============

# 初始化 zoxide
eval "$(zoxide init zsh)"

# 初始化 fzf
[ -f ~/.fzf.zsh ] && source ~/.fzf.zsh

# 别名设置
alias ls='exa'
alias ll='exa -la'
alias lt='exa --tree --level=2'
alias cat='bat'
alias find='fd'
alias grep='rg'

# fzf 配置
export FZF_DEFAULT_OPTS="
  --height 60%
  --layout=reverse
  --border
  --preview 'bat --style=numbers --color=always {} 2>/dev/null | head -500'
"

export FZF_DEFAULT_COMMAND="fd --type f --exclude node_modules --exclude .git --exclude dist"
export FZF_CTRL_T_COMMAND="$FZF_DEFAULT_COMMAND"
export FZF_ALT_C_COMMAND="fd --type d --exclude node_modules"

# 快捷函数

# 搜索文件并用 VS Code 打开
fe() {
  local file
  file=$(fzf --query="$1")
  [ -n "$file" ] && code "$file"
}

# 搜索内容并打开
se() {
  local file
  file=$(rg -l "$1" | fzf)
  [ -n "$file" ] && code "$file"
}

# 快速 git commit
gc() {
  local msg
  msg=$(fzf --prompt="Commit message: ")
  [ -n "$msg" ] && git commit -m "$msg"
}
```

## 9. 进阶推荐

### delta：更好的 git diff

```bash
brew install git-delta

# ~/.gitconfig
[core]
    pager = delta

[delta]
    side-by-side = true
    line-numbers = true
```

### tldr：简化版 man

```bash
brew install tldr

tldr tar
tldr git
tldr curl
```

### htop：更好的 top

```bash
brew install htop
htop
```

### direnv：目录级环境变量

```bash
brew install direnv

# ~/.zshrc
eval "$(direnv hook zsh)"

# 项目目录下创建 .envrc
echo "export NODE_ENV=development" > .envrc
direnv allow
```
