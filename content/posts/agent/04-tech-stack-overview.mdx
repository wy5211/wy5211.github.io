# Agent 开发技术栈全景图

上一篇我们建立了 Agent 开发工程师的能力模型。你可能想问：这些能力对应什么具体技术？应该从哪里开始学？有哪些主流工具和框架？

这篇我们来梳理 Agent 开发的完整技术栈。

## 技术栈分层

Agent 开发技术栈可以分成四层：基础设施层、能力层、框架层、应用层。

基础设施层是地基，包括 LLM API、向量数据库、向量化服务、可观测工具。没有这些，上层无法运行。

能力层是通用能力的封装，包括 RAG（检索增强）、工具调用、记忆管理、任务规划。这些能力可以复用，避免重复开发。

框架层是开发工具，包括 LangChain、AutoGen、CrewAI 等。框架提供了便捷的 API，让你快速构建 Agent。

应用层是具体的 Agent 产品，比如聊天机器人、自动化工具、智能助手。

这个分层结构的好处是你可以按需学习。不是所有技术都需要精通，根据你的目标和场景选择合适的深度。

## 学习优先级

技术栈很广，但不要被吓到。有优先级的学习能更快见效。

P0 优先级是 LLM API 使用和 Prompt 工程。这两个是一切的基础，必须先掌握。没有 LLM API，你无法让 Agent 工作。没有 Prompt 工程，你无法让 Agent 理解任务。

P1 优先级是 Agent 框架和 RAG 技术。框架让你快速开发，RAG 让 Agent 能够利用私有知识。这两个是企业应用的核心能力。

P2 优先级是向量数据库和多 Agent 协作。向量数据库是 RAG 的基础设施，多 Agent 协作是高级应用场景。这两个在复杂项目中才需要。

P3 优先级是部署和运维。当 Agent 需要上线时才需要关注，初期学习可以延后。

## LLM 基础：Agent 的大脑

LLM 是 Agent 的核心决策引擎。选择合适的 LLM 对 Agent 的效果、成本、性能都有重大影响。

主流 LLM 提供商有几个阵营。OpenAI 的 GPT 系列生态最完善，工具调用能力强，但成本较高。Anthropic 的 Claude 系列有超长上下文（200K token），代码能力强，安全对齐好。国内有通义千问、文心一言、智谱 GLM、DeepSeek 等，价格有优势，中文理解好。开源有 Llama 3、Qwen、Mistral 等，可以自托管，数据隐私好。

选择时需要考虑：任务复杂度（简单任务用小模型，复杂任务用大模型）、成本预算（开发时用便宜模型，生产时用优质模型）、语言需求（中文任务优先考虑国内模型）、部署需求（数据敏感时考虑开源模型）。

```typescript
// LLM API 调用示例
import OpenAI from "openai";

const client = new OpenAI({
  apiKey: process.env.OPENAI_API_KEY,
});

// 基础对话
async function chat(message: string) {
  const response = await client.chat.completions.create({
    model: "gpt-4o",
    messages: [
      { role: "system", content: "你是一个有帮助的助手。" },
      { role: "user", content: message },
    ],
    temperature: 0.7,
    max_tokens: 1000,
  });

  return response.choices[0].message.content;
}

// 工具调用（Function Calling）
async function chatWithTools(message: string) {
  const tools = [
    {
      type: "function",
      function: {
        name: "get_weather",
        description: "获取指定城市的天气信息",
        parameters: {
          type: "object",
          properties: {
            city: {
              type: "string",
              description: "城市名称",
            },
          },
          required: ["city"],
        },
      },
    },
  ];

  const response = await client.chat.completions.create({
    model: "gpt-4o",
    messages: [{ role: "user", content: message }],
    tools,
  });

  const message = response.choices[0].message;

  if (message.tool_calls) {
    const toolCall = message.tool_calls[0];
    const args = JSON.parse(toolCall.function.arguments);

    // 执行工具
    const result = await getWeather(args.city);

    // 返回工具结果给 LLM
    const finalResponse = await client.chat.completions.create({
      model: "gpt-4o",
      messages: [
        { role: "user", content: message },
        message,
        {
          role: "tool",
          tool_call_id: toolCall.id,
          content: JSON.stringify(result),
        },
      ],
    });

    return finalResponse.choices[0].message.content;
  }

  return message.content;
}

async function getWeather(city: string) {
  // 调用实际天气 API
  return { city, temperature: 25, condition: "晴天" };
}
```

## Agent 框架：加速开发

Agent 框架封装了通用功能，让你专注于业务逻辑。不用从零实现 Agent 循环、工具管理、记忆系统，直接调用框架提供的 API。

主流框架各有特点。LangChain 是生态最完善的，Python 和 JS 都支持，文档丰富，社区活跃。Vercel AI SDK 对前端开发者最友好，与 Next.js 深度集成，边缘部署支持好。AutoGen 专注于多 Agent 对话，适合复杂协作场景。CrewAI 采用角色扮演模式，易于理解和配置。

如果你是前端开发者，推荐从 Vercel AI SDK 开始，简单场景直接用，复杂场景再引入 LangChain.js。如果你是后端开发者，推荐 LangChain.py，功能全面，生态成熟。

```typescript
// Vercel AI SDK 示例（前端友好）
import { openai } from "@ai-sdk/openai";
import { generateText, streamText, tool } from "ai";
import { z } from "zod";

// 简单文本生成
async function simpleGenerate() {
  const { text } = await generateText({
    model: openai("gpt-4o"),
    prompt: "用一句话解释什么是 AI Agent",
  });

  return text;
}

// 工具调用
async function generateWithTools() {
  const { text } = await generateText({
    model: openai("gpt-4o"),
    tools: {
      weather: tool({
        description: "获取城市天气",
        parameters: z.object({
          city: z.string().describe("城市名称"),
        }),
        execute: async ({ city }) => {
          // 调用天气 API
          return { city, temp: 25, condition: "晴天" };
        },
      }),
    },
    prompt: "北京今天天气怎么样？",
  });

  return text;
}

// Next.js API Route 集成
import { openai } from "@ai-sdk/openai";
import { streamText } from "ai";

export async function POST(req: Request) {
  const { messages } = await req.json();

  const result = streamText({
    model: openai("gpt-4o"),
    messages,
  });

  return result.toDataStreamResponse();
}
```

```typescript
// LangChain 示例（功能全面）
import { ChatOpenAI } from "@langchain/openai";
import { AgentExecutor, createOpenAIFunctionsAgent } from "langchain/agents";
import { DynamicTool } from "@langchain/core/tools";

// 定义工具
const tools = [
  new DynamicTool({
    name: "calculator",
    description: "计算数学表达式",
    func: async (input: string) => {
      try {
        const result = eval(input);
        return String(result);
      } catch {
        return "计算错误";
      }
    },
  }),
];

// 创建模型
const llm = new ChatOpenAI({
  modelName: "gpt-4o",
  temperature: 0,
});

// 创建 Agent
const agent = await createOpenAIFunctionsAgent({
  llm,
  tools,
  prompt: await pull("hwchase17/openai-functions-agent"),
});

// 创建执行器
const agentExecutor = new AgentExecutor({
  agent,
  tools,
  verbose: true,
  maxIterations: 10,
});

// 执行任务
const result = await agentExecutor.invoke({
  input: "帮我计算 (25 + 17) * 3 的结果",
});
```

## RAG 技术：给 Agent 知识

RAG（Retrieval-Augmented Generation，检索增强生成）让 Agent 能够利用外部知识。为什么需要？因为 LLM 的知识有局限：训练截止日期后的事件不知道、私有数据不知道、容易产生幻觉。

RAG 的思路是：给 LLM 提供相关文档，让它基于这些文档回答问题。这样 LLM 可以利用最新、最准确的信息，减少幻觉。

RAG 系统分两个阶段。离线索引阶段：收集文档、分块处理、向量化、存储到向量数据库。在线查询阶段：问题向量化、相似度检索、构建上下文、LLM 生成回答。

```typescript
// RAG 系统实现
import { OpenAI } from "openai";
import { Pinecone } from "@pinecone-database/pinecone";

const openai = new OpenAI();
const pinecone = new Pinecone({
  apiKey: process.env.PINECONE_API_KEY,
});

// 索引文档
async function indexDocuments(documents: string[]) {
  const index = pinecone.index("knowledge-base");

  for (const doc of documents) {
    const embedding = await createEmbedding(doc);

    await index.upsert([
      {
        id: generateId(),
        values: embedding,
        metadata: { text: doc },
      },
    ]);
  }
}

// 创建向量
async function createEmbedding(text: string): Promise<number[]> {
  const response = await openai.embeddings.create({
    model: "text-embedding-3-small",
    input: text,
  });

  return response.data[0].embedding;
}

// 检索相关文档
async function retrieveRelevantDocs(query: string, topK: number = 3) {
  const index = pinecone.index("knowledge-base");
  const queryEmbedding = await createEmbedding(query);

  const results = await index.query({
    vector: queryEmbedding,
    topK,
    includeMetadata: true,
  });

  return results.matches.map((match) => ({
    text: match.metadata?.text as string,
    score: match.score,
  }));
}

// RAG 查询
async function ragQuery(question: string) {
  const relevantDocs = await retrieveRelevantDocs(question);

  const context = relevantDocs
    .map((doc, i) => `[文档${i + 1}] ${doc.text}`)
    .join("\n\n");

  const response = await openai.chat.completions.create({
    model: "gpt-4o",
    messages: [
      {
        role: "system",
        content: `你是一个助手，请根据以下知识库内容回答问题。
如果知识库中没有相关信息，请诚实地说不知道。

知识库内容：
${context}`,
      },
      { role: "user", content: question },
    ],
  });

  return {
    answer: response.choices[0].message.content,
    sources: relevantDocs,
  };
}
```

## 向量数据库：RAG 的基础设施

向量数据库是 RAG 系统的核心组件，存储和检索文档向量。

主流向量数据库各有特点。Pinecone 是云服务，托管、易用、高性能，适合快速原型和生产环境。Chroma 是开源的，轻量级、Python 友好，适合开发测试。Qdrant 是开源的，Rust 实现、高效，适合自托管需求。Pgvector 是 PostgreSQL 扩展，与 SQL 结合，适合已有 PG 的项目。

选择时考虑：部署方式（云服务省心，开源可控）、数据规模（小数据用 Chroma，大数据用 Milvus）、查询需求（简单查询用基础库，复杂过滤用高级库）、成本预算（开源免费但需要运维，云服务收费但省事）。

```typescript
// Pinecone 使用示例
import { Pinecone } from "@pinecone-database/pinecone";

const pc = new Pinecone({
  apiKey: process.env.PINECONE_API_KEY,
});

// 创建索引
async function createIndex() {
  await pc.createIndex({
    name: "my-index",
    dimension: 1536,
    metric: "cosine",
    spec: {
      serverless: {
        cloud: "aws",
        region: "us-east-1",
      },
    },
  });
}

// 使用索引
async function useIndex() {
  const index = pc.index("my-index");

  // 插入向量
  await index.upsert([
    {
      id: "doc-1",
      values: new Array(1536).fill(0.1),
      metadata: {
        title: "文档标题",
        content: "文档内容...",
        category: "技术",
      },
    },
  ]);

  // 查询
  const results = await index.query({
    vector: new Array(1536).fill(0.1),
    topK: 10,
    includeMetadata: true,
    filter: {
      category: { $eq: "技术" },
    },
  });

  return results;
}
```

## 开发工具与调试

Agent 开发需要专门的工具来调试和优化。

LangSmith 是 LangChain 官方的调试平台，可以追踪 Agent 执行过程、可视化决策路径、分析 Token 消耗、A/B 测试不同 Prompt。Weights & Biages 用于实验追踪，管理 Prompt 实验、记录评估指标。Helicone 用于 API 监控，追踪 OpenAI 调用、分析成本和性能。Promptfoo 用于 Prompt 测试，自动化评估不同 Prompt 的效果。

这些工具不是必须的，但能显著提高开发效率。初期可以不用，等项目规模上来后再引入。

## 技术选型建议

根据场景选择合适的技术栈。

如果你的主要开发语言是 JavaScript/TypeScript，推荐 Vercel AI SDK + LangChain.js + Pinecone。Vercel AI SDK 前端友好，LangChain.js 功能全面，Pinecone 易用稳定。

如果是 Python，推荐 LangChain.py + Pinecone/Chroma。LangChain.py 生态最成熟，Pinecone 生产稳定，Chroma 开发便捷。

如果是个人项目或原型，推荐 Chroma + GPT-3.5。Chroma 本地运行无需配置，GPT-3.5 成本低。

如果是企业项目，推荐 Pinecone/Milvus + GPT-4 + LangSmith。Pinecone/Milvus 稳定可靠，GPT-4 效果好，LangSmith 便于调试。

如果需要多 Agent 协作，推荐 AutoGen 或 CrewAI。AutoGen 适合复杂对话场景，CrewAI 适合角色扮演场景。

## 核心权衡

技术栈选择有几个需要权衡的方面。

易用性 vs 灵活性。框架越高级，开发越快，但定制空间越小。LangChain 高层 API 易用但定制受限，底层 API 灵活但需要更多代码。

成本 vs 性能。GPT-4 效果好但成本高，GPT-3.5 成本低但效果差。需要根据任务复杂度选择合适的模型。

云服务 vs 自托管。Pinecone 等云服务省事但成本高，Chroma 等开源方案免费但需要运维。根据团队资源和项目规模选择。

学习曲线 vs 项目进度。LangChain 功能全面但学习曲线陡，Vercel AI SDK 简单但功能有限。根据时间要求和长期规划选择。
