## 前言

在上一篇《文件上传》中，我们实现了常规的单文件、多文件上传。但当文件较大（超过 100MB 甚至几个 GB）时，传统上传方式会遇到诸多问题：网络中断导致从头重传、服务器超时、内存占用过高等。

**切片上传**（Chunked Upload）是解决大文件上传的标准方案——将大文件切割成多个小分片，逐个上传，最后在服务端合并。

**本文将解决什么问题？**

你可能想知道：

- 切片上传的整体流程是怎样的？
- 前端如何对文件进行切片和计算哈希？
- 后端如何接收、存储和合并分片？
- 如何实现断点续传（网络中断后从断点继续）？
- 如何实现秒传（相同文件瞬间完成）？
- 如何追踪上传进度？

**本文将带你：**

1. 理解切片上传的完整架构
2. 前端实现文件切片、哈希计算、并发控制
3. 后端实现分片接收、合并、校验

5. 完整的前后端联调

## 一、切片上传整体架构

### 1.1 传统上传 vs 切片上传

```
传统上传：
┌──────────────┐                    ┌──────────────┐
│   浏览器     │ ──── 整个文件 ────→ │    服务器     │
│              │ ←── 失败？重来 ──── │              │
└──────────────┘                    └──────────────┘
问题：文件大时容易超时、网络中断需全部重传、内存压力大

切片上传：
┌──────────────┐                    ┌──────────────┐
│   浏览器     │ ──── 分片 1 ────→  │    服务器     │
│              │ ──── 分片 2 ────→  │   (暂存)     │
│              │ ──── 分片 3 ────→  │              │
│              │ ──── ... ──────→   │              │
│              │ ──── 合并请求 ──→   │   (合并文件)  │
│              │ ←── 上传成功 ────   │              │
└──────────────┘                    └──────────────┘
优势：失败只需重传该分片、可并发上传、支持断点续传
```

### 1.2 完整流程图

```
前端                                后端
 │                                   │
 │  1. 计算文件哈希（MD5/SHA-256）    │
 │                                   │
 │  2. 秒传检查 ──────────────────→  │  查询数据库：该哈希是否已存在？
 │  ←── 已存在，直接返回 URL ─────── │  ✅ 秒传成功
 │                                   │
 │  3. 断点续传检查 ──────────────→  │  查询已上传的分片列表
 │  ←── 返回已上传的分片索引 ─────── │
 │                                   │
 │  4. 逐片上传（跳过已上传的）───→   │  保存分片到临时目录
 │  ←── 分片接收确认 ──────────────  │
 │                                   │
 │  5. 发送合并请求 ──────────────→  │  合并所有分片 → 最终文件
 │  ←── 返回文件 URL ──────────────  │  ✅ 上传完成
```

### 1.3 核心概念

| 概念                 | 说明                                               |
| -------------------- | -------------------------------------------------- |
| **分片（Chunk）**    | 将大文件按固定大小（如 2MB）切分成多个小块         |
| **文件哈希（Hash）** | 文件的唯一指纹，用于秒传和断点续传判断             |
| **秒传**             | 哈希相同说明文件已存在，直接返回 URL，无需重复上传 |
| **断点续传**         | 记录已上传的分片，中断后只需继续上传剩余分片       |
| **合并**             | 所有分片上传完成后，服务端按顺序合并为完整文件     |

### 2.3 上传服务

```typescript
// src/upload/upload.service.ts
import { Injectable, BadRequestException, Logger } from "@nestjs/common";
import * as fs from "fs";
import * as path from "path";
import { PrismaService } from "../prisma/prisma.service";
import { CheckUploadDto } from "./dto/check-upload.dto";
import { MergeUploadDto } from "./dto/merge-upload.dto";

// 上传目录配置
const UPLOAD_DIR = path.resolve(process.cwd(), "uploads");
const CHUNK_DIR = path.resolve(UPLOAD_DIR, "chunks");
const MERGE_DIR = path.resolve(UPLOAD_DIR, "files");

@Injectable()
export class UploadService {
  private readonly logger = new Logger(UploadService.name);

  constructor(private prisma: PrismaService) {
    // 确保目录存在
    this.ensureDirectoryExists();
  }

  /**
   * 确保上传所需目录存在
   */
  private ensureDirectoryExists() {
    const dirs = [UPLOAD_DIR, CHUNK_DIR, MERGE_DIR];
    dirs.forEach((dir) => {
      if (!fs.existsSync(dir)) {
        fs.mkdirSync(dir, { recursive: true });
      }
    });
  }

  /**
   * 获取分片临时存储路径
   * 每个文件的分片存储在以 fileHash 命名的子目录中
   */
  private getChunkDir(fileHash: string): string {
    return path.resolve(CHUNK_DIR, fileHash);
  }

  /**
   * 获取单个分片文件路径
   */
  private getChunkPath(fileHash: string, chunkIndex: number): string {
    return path.resolve(this.getChunkDir(fileHash), `${chunkIndex}`);
  }

  /**
   * 秒传检查 + 断点续传检查
   * 返回：秒传结果 或 已上传的分片索引列表
   */
  async checkUpload(dto: CheckUploadDto) {
    const { fileHash, fileName } = dto;

    // 1. 秒传检查：查询数据库中是否已存在相同哈希的文件
    const existingFile = await this.prisma.uploadedFile.findUnique({
      where: { hash: fileHash },
    });

    if (existingFile) {
      return {
        uploaded: true, // 秒传：文件已存在
        url: existingFile.url,
        message: "秒传成功，文件已存在",
      };
    }

    // 2. 断点续传检查：查看已上传的分片
    const chunkDir = this.getChunkDir(fileHash);
    let uploadedChunks: number[] = [];

    if (fs.existsSync(chunkDir)) {
      const files = fs.readdirSync(chunkDir);
      uploadedChunks = files
        .filter((f) => !isNaN(Number(f)))
        .map((f) => Number(f))
        .sort((a, b) => a - b);
    }

    return {
      uploaded: false,
      uploadedChunks,
      message:
        uploadedChunks.length > 0
          ? `发现 ${uploadedChunks.length} 个已上传分片，可断点续传`
          : "新文件上传",
    };
  }

  /**
   * 保存单个分片
   */
  async saveChunk(
    fileHash: string,
    chunkIndex: number,
    buffer: Buffer
  ): Promise<{ success: boolean; chunkIndex: number }> {
    const chunkDir = this.getChunkDir(fileHash);

    // 确保分片目录存在
    if (!fs.existsSync(chunkDir)) {
      fs.mkdirSync(chunkDir, { recursive: true });
    }

    const chunkPath = this.getChunkPath(fileHash, chunkIndex);

    // 如果分片已存在，直接返回（幂等性）
    if (fs.existsSync(chunkPath)) {
      this.logger.log(`分片 ${chunkIndex} 已存在，跳过写入`);
      return { success: true, chunkIndex };
    }

    // 写入分片
    fs.writeFileSync(chunkPath, buffer);
    this.logger.log(
      `分片 ${chunkIndex} 保存成功，大小: ${buffer.length} bytes`
    );

    return { success: true, chunkIndex };
  }

  /**
   * 合并所有分片
   */
  async mergeChunks(dto: MergeUploadDto) {
    const { fileHash, fileName, totalChunks, fileSize } = dto;
    const chunkDir = this.getChunkDir(fileHash);

    // 1. 检查分片目录是否存在
    if (!fs.existsSync(chunkDir)) {
      throw new BadRequestException("分片目录不存在，请重新上传");
    }

    // 2. 检查分片数量是否完整
    const chunks = fs.readdirSync(chunkDir);
    if (chunks.length !== totalChunks) {
      throw new BadRequestException(
        `分片不完整，期望 ${totalChunks} 个，实际 ${chunks.length} 个`
      );
    }

    // 3. 生成最终文件名
    const ext = path.extname(fileName);
    const finalFileName = `${fileHash}${ext}`;
    const finalFilePath = path.resolve(MERGE_DIR, finalFileName);

    // 4. 如果最终文件已存在（并发场景），直接返回
    if (fs.existsSync(finalFilePath)) {
      const existingFile = await this.prisma.uploadedFile.findUnique({
        where: { hash: fileHash },
      });

      if (existingFile) {
        // 清理分片
        this.cleanChunks(chunkDir);
        return {
          url: existingFile.url,
          fileName: finalFileName,
          message: "文件已合并",
        };
      }
    }

    // 5. 按顺序合并分片
    const writeStream = fs.createWriteStream(finalFilePath);

    for (let i = 0; i < totalChunks; i++) {
      const chunkPath = this.getChunkPath(fileHash, i);

      if (!fs.existsSync(chunkPath)) {
        throw new BadRequestException(`分片 ${i} 不存在`);
      }

      // 使用同步读取确保顺序
      const chunkBuffer = fs.readFileSync(chunkPath);
      writeStream.write(chunkBuffer);
    }

    writeStream.end();

    // 等待写入完成
    await new Promise<void>((resolve, reject) => {
      writeStream.on("finish", resolve);
      writeStream.on("error", reject);
    });

    // 6. 校验文件大小
    const stats = fs.statSync(finalFilePath);
    if (stats.size !== fileSize) {
      // 大小不一致，删除合并后的文件
      fs.unlinkSync(finalFilePath);
      throw new BadRequestException(
        `文件大小校验失败，期望 ${fileSize} bytes，实际 ${stats.size} bytes`
      );
    }

    // 7. 写入数据库记录（用于秒传）
    const fileUrl = `/uploads/files/${finalFileName}`;
    await this.prisma.uploadedFile.upsert({
      where: { hash: fileHash },
      update: { url: fileUrl, size: fileSize, fileName },
      create: {
        hash: fileHash,
        url: fileUrl,
        size: fileSize,
        fileName,
      },
    });

    // 8. 清理分片文件
    this.cleanChunks(chunkDir);

    this.logger.log(
      `文件合并完成: ${finalFileName}，大小: ${stats.size} bytes`
    );

    return {
      url: fileUrl,
      fileName: finalFileName,
      size: stats.size,
      message: "文件上传并合并成功",
    };
  }

  /**
   * 清理分片文件
   */
  private cleanChunks(chunkDir: string) {
    if (fs.existsSync(chunkDir)) {
      fs.rmSync(chunkDir, { recursive: true, force: true });
      this.logger.log(`已清理分片目录: ${chunkDir}`);
    }
  }
}
```

```typescript
// utils/chunk-upload.ts

/** 默认分片大小：2MB */
export const DEFAULT_CHUNK_SIZE = 2 * 1024 * 1024;

/**
 * 将文件切割成多个分片
 * @param file 原始文件
 * @param chunkSize 分片大小（字节）
 * @returns Blob[] 分片数组
 */
export function createFileChunks(
  file: File,
  chunkSize: number = DEFAULT_CHUNK_SIZE
): Blob[] {
  const chunks: Blob[] = [];
  let cur = 0;

  while (cur < file.size) {
    // File 继承自 Blob，slice 方法用于切割
    const chunk = file.slice(cur, cur + chunkSize);
    chunks.push(chunk);
    cur += chunkSize;
  }

  return chunks;
}

/**
 * 计算文件的 MD5 哈希值
 * 使用 SparkMD5 库，分片读取避免内存溢出
 */
export async function calculateFileHash(
  file: File,
  chunkSize: number = DEFAULT_CHUNK_SIZE
): Promise<string> {
  // 动态导入 SparkMD5（需要安装：pnpm add spark-md5）
  const SparkMD5 = (await import("spark-md5")).default;

  return new Promise((resolve, reject) => {
    const spark = new SparkMD5.ArrayBuffer();
    const reader = new FileReader();
    const chunks = Math.ceil(file.size / chunkSize);
    let currentChunk = 0;

    reader.onload = (e) => {
      if (e.target?.result) {
        spark.append(e.target.result as ArrayBuffer);
        currentChunk++;

        if (currentChunk < chunks) {
          loadNext();
        } else {
          // 所有分片读取完毕，返回哈希
          resolve(spark.end());
        }
      }
    };

    reader.onerror = () => {
      reject(new Error("文件读取失败"));
    };

    function loadNext() {
      const start = currentChunk * chunkSize;
      const end = Math.min(start + chunkSize, file.size);
      // 注意：这里 slice 的对象是原始 File，不是 Blob 分片
      reader.readAsArrayBuffer(file.slice(start, end));
    }

    loadNext();
  });
}
```

### 3.2 上传服务类

```typescript
// services/chunk-upload.service.ts

import {
  createFileChunks,
  calculateFileHash,
  DEFAULT_CHUNK_SIZE,
} from "../utils/chunk-upload";

/** 上传进度回调 */
export type ProgressCallback = (progress: {
  uploadedChunks: number;
  totalChunks: number;
  percent: number;
}) => void;

/** 并发控制器 */
class ConcurrencyPool {
  private running = 0;
  private queue: (() => void)[] = [];

  constructor(private maxConcurrency: number) {}

  async run<T>(fn: () => Promise<T>): Promise<T> {
    // 如果当前运行数已达上限，排队等待
    while (this.running >= this.maxConcurrency) {
      await new Promise<void>((resolve) => this.queue.push(resolve));
    }

    this.running++;
    try {
      return await fn();
    } finally {
      this.running--;
      // 从队列中取出下一个任务执行
      const next = this.queue.shift();
      if (next) next();
    }
  }
}

export class ChunkUploadService {
  private chunkSize: number;
  private maxConcurrency: number;
  private baseURL: string;
  private abortController: AbortController | null = null;

  constructor(options?: {
    chunkSize?: number;
    maxConcurrency?: number;
    baseURL?: string;
  }) {
    this.chunkSize = options?.chunkSize ?? DEFAULT_CHUNK_SIZE;
    this.maxConcurrency = options?.maxConcurrency ?? 3; // 默认 3 个并发
    this.baseURL = options?.baseURL ?? "http://localhost:3000";
  }

  /**
   * 取消上传
   */
  cancel() {
    this.abortController?.abort();
    this.abortController = null;
  }

  /**
   * 完整上传流程
   */
  async upload(
    file: File,
    onProgress?: ProgressCallback
  ): Promise<{ url: string; hash: string }> {
    // 创建 AbortController 用于取消
    this.abortController = new AbortController();
    const signal = this.abortController.signal;

    // 第一步：计算文件哈希
    console.log("正在计算文件哈希...");
    const fileHash = await calculateFileHash(file, this.chunkSize);
    console.log(`文件哈希: ${fileHash}`);

    // 检查是否已取消
    if (signal.aborted) throw new Error("上传已取消");

    // 第二步：秒传检查
    const checkResult = await this.checkUpload(fileHash, file.name, signal);
    if (checkResult.uploaded) {
      console.log("秒传成功！");
      onProgress?.({
        uploadedChunks: 0,
        totalChunks: 0,
        percent: 100,
      });
      return { url: checkResult.url, hash: fileHash };
    }

    // 第三步：切片
    const chunks = createFileChunks(file, this.chunkSize);
    const totalChunks = chunks.length;

    // 第四步：过滤已上传的分片（断点续传）
    const uploadedChunks = new Set(checkResult.uploadedChunks || []);
    const pendingChunks = chunks.filter(
      (_, index) => !uploadedChunks.has(index)
    );

    console.log(
      `总分片: ${totalChunks}，已上传: ${uploadedChunks.size}，待上传: ${pendingChunks.length}`
    );

    // 第五步：并发上传分片
    let completedCount = uploadedChunks.size;
    const pool = new ConcurrencyPool(this.maxConcurrency);

    const uploadTasks = chunks.map((chunk, index) => {
      // 跳过已上传的分片
      if (uploadedChunks.has(index)) return Promise.resolve();

      return pool.run(async () => {
        if (signal.aborted) throw new Error("上传已取消");

        await this.uploadChunk(
          fileHash,
          index,
          totalChunks,
          chunk,
          file.name,
          signal
        );

        completedCount++;
        onProgress?.({
          uploadedChunks: completedCount,
          totalChunks,
          percent: Math.round((completedCount / totalChunks) * 100),
        });
      });
    });

    await Promise.all(uploadTasks);

    // 第六步：合并请求
    console.log("所有分片上传完成，正在合并...");
    const mergeResult = await this.mergeChunks(
      fileHash,
      file.name,
      totalChunks,
      file.size,
      signal
    );

    console.log("文件上传成功！", mergeResult.url);
    return { url: mergeResult.url, hash: fileHash };
  }

  /** 秒传 + 断点续传检查 */
  private async checkUpload(
    fileHash: string,
    fileName: string,
    signal: AbortSignal
  ) {
    const response = await fetch(
      `${this.baseURL}/upload/check?fileHash=${fileHash}&fileName=${encodeURIComponent(fileName)}`,
      { signal }
    );

    if (!response.ok) throw new Error("检查上传状态失败");
    return response.json();
  }

  /** 上传单个分片 */
  private async uploadChunk(
    fileHash: string,
    chunkIndex: number,
    totalChunks: number,
    chunk: Blob,
    fileName: string,
    signal: AbortSignal
  ) {
    const formData = new FormData();
    formData.append("chunk", chunk); // 分片数据
    formData.append("fileHash", fileHash); // 文件哈希
    formData.append("chunkIndex", String(chunkIndex)); // 当前分片索引
    formData.append("totalChunks", String(totalChunks)); // 总分片数
    formData.append("fileName", fileName); // 文件名

    const response = await fetch(`${this.baseURL}/upload/chunk`, {
      method: "POST",
      body: formData,
      signal,
    });

    if (!response.ok) throw new Error(`分片 ${chunkIndex} 上传失败`);
    return response.json();
  }

  /** 合并请求 */
  private async mergeChunks(
    fileHash: string,
    fileName: string,
    totalChunks: number,
    fileSize: number,
    signal: AbortSignal
  ) {
    const response = await fetch(`${this.baseURL}/upload/merge`, {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({ fileHash, fileName, totalChunks, fileSize }),
      signal,
    });

    if (!response.ok) throw new Error("合并请求失败");
    return response.json();
  }
}
```

## 四、流程详解

### 4.1 文件哈希计算

文件哈希是整个切片上传的核心标识。为什么用 MD5 而不是 SHA-256？因为 MD5 计算速度更快，对于文件去重场景足够使用。

**关键点：分片读取**

```typescript
// ❌ 错误：一次性读取整个大文件（可能内存溢出）
const buffer = await file.arrayBuffer();
const hash = spark.append(buffer).end();

// ✅ 正确：分片读取，逐步计算哈希
function loadNext() {
  const start = currentChunk * chunkSize;
  const end = Math.min(start + chunkSize, file.size);
  reader.readAsArrayBuffer(file.slice(start, end));
}
```

**为什么不直接对所有 Blob 分片计算哈希？**

`createFileChunks` 返回的是 `Blob[]`，而 `FileReader.readAsArrayBuffer` 接受 `Blob`。但为了保证哈希一致性（合并后的文件和原始文件哈希相同），我们应该对原始 `File` 对象的切片进行哈希计算，而不是对二次切片的结果。

### 4.2 秒传原理

```
第一次上传 file.mp4 (500MB)
  → 计算哈希：a1b2c3d4e5f6...
  → 上传所有分片 → 合并 → 保存记录到数据库
  → 数据库：{ hash: "a1b2c3d4e5f6...", url: "/uploads/files/a1b2c3d4.mp4" }

第二次上传相同的 file.mp4
  → 计算哈希：a1b2c3d4e5f6...（相同！）
  → 调用 /upload/check → 数据库中已存在
  → 直接返回 URL，0 字节传输 ✅
```

### 4.3 断点续传原理

```
上传 file.mp4，共 250 个分片
  → 上传到第 100 个分片时网络断开
  → 服务器已保存分片 0~99

恢复上传：
  → 调用 /upload/check → 返回 uploadedChunks: [0, 1, 2, ..., 99]
  → 跳过分片 0~99，从分片 100 开始继续上传
  → 上传完成后调用 /upload/merge 合并
```

### 4.4 分片存储结构

```
uploads/
├── chunks/                          # 分片临时目录
│   ├── a1b2c3d4e5f6.../            # 以文件哈希命名的子目录
│   │   ├── 0                        # 分片 0
│   │   ├── 1                        # 分片 1
│   │   ├── 2                        # 分片 2
│   │   └── ...                      # 更多分片
│   └── f7e8d9c0b1a2.../
│       └── ...
├── files/                           # 合并后的最终文件
│   ├── a1b2c3d4e5f6....mp4
│   └── f7e8d9c0b1a2....pdf
```

## 五、并发控制与错误重试

### 5.1 并发控制

同时发送太多请求会拖慢浏览器和网络，因此需要限制并发数：

```typescript
class ConcurrencyPool {
  private running = 0;
  private queue: (() => void)[] = [];

  constructor(private maxConcurrency: number) {}

  async run<T>(fn: () => Promise<T>): Promise<T> {
    // 超过最大并发数时排队等待
    while (this.running >= this.maxConcurrency) {
      await new Promise<void>((resolve) => this.queue.push(resolve));
    }

    this.running++;
    try {
      return await fn();
    } finally {
      this.running--;
      const next = this.queue.shift();
      if (next) next();
    }
  }
}

// 使用示例：3 个并发
const pool = new ConcurrencyPool(3);
await Promise.all(
  chunks.map((chunk, index) => pool.run(() => uploadChunk(index, chunk)))
);
```

### 5.2 分片上传重试

网络不稳定时，单个分片可能上传失败。添加自动重试：

```typescript
/**
 * 带重试的分片上传
 */
async function uploadChunkWithRetry(
  fileHash: string,
  chunkIndex: number,
  totalChunks: number,
  chunk: Blob,
  fileName: string,
  signal: AbortSignal,
  maxRetries: number = 3
): Promise<any> {
  let lastError: Error | null = null;

  for (let attempt = 1; attempt <= maxRetries; attempt++) {
    try {
      if (signal.aborted) throw new Error("上传已取消");

      return await uploadChunk(
        fileHash,
        chunkIndex,
        totalChunks,
        chunk,
        fileName,
        signal
      );
    } catch (err: any) {
      lastError = err;

      if (signal.aborted) throw err;

      console.warn(
        `分片 ${chunkIndex} 上传失败（第 ${attempt}/${maxRetries} 次尝试）`,
        err.message
      );

      // 指数退避：等待时间逐渐增加
      if (attempt < maxRetries) {
        const delay = Math.pow(2, attempt) * 1000; // 2s, 4s, 8s...
        await new Promise((resolve) => setTimeout(resolve, delay));
      }
    }
  }

  throw new Error(
    `分片 ${chunkIndex} 上传失败，已重试 ${maxRetries} 次: ${lastError?.message}`
  );
}
```

## 六、进阶优化

### 6.1 Web Worker 计算哈希

文件哈希计算是 CPU 密集型任务，放在主线程会导致页面卡顿。使用 Web Worker 在后台线程中计算：

```typescript
// workers/hash-worker.ts
import SparkMD5 from "spark-md5";

self.onmessage = (e: MessageEvent) => {
  const { fileChunkList } = e.data;
  const spark = new SparkMD5.ArrayBuffer();
  let percentage = 0;
  let count = 0;

  const loadNext = (index: number) => {
    const reader = new FileReader();
    reader.readAsArrayBuffer(fileChunkList[index]);

    reader.onload = (event) => {
      count++;
      spark.append(event.target!.result as ArrayBuffer);

      if (count === fileChunkList.length) {
        // 计算完成，发送结果
        self.postMessage({
          hash: spark.end(),
          percentage: 100,
        });
        self.close();
      } else {
        // 更新进度
        const newPercentage = Math.round((count / fileChunkList.length) * 100);
        if (newPercentage !== percentage) {
          percentage = newPercentage;
          self.postMessage({ percentage });
        }
        loadNext(count);
      }
    };
  };

  loadNext(0);
};
```

**前端使用 Worker：**

```typescript
function calculateHashInWorker(file: File, chunkSize: number): Promise<string> {
  return new Promise((resolve, reject) => {
    // 先切片（File 对象不能传递到 Worker，但 Blob 可以）
    const chunks: Blob[] = [];
    let cur = 0;
    while (cur < file.size) {
      chunks.push(file.slice(cur, cur + chunkSize));
      cur += chunkSize;
    }

    // 创建 Worker
    const worker = new Worker(
      new URL("../workers/hash-worker.ts", import.meta.url)
    );

    worker.onmessage = (e) => {
      if (e.data.hash) {
        resolve(e.data.hash);
      }
      // e.data.percentage 可以用来显示哈希计算进度
    };

    worker.onerror = (e) => {
      reject(new Error(`Worker 错误: ${e.message}`));
    };

    // 发送分片到 Worker
    worker.postMessage({ fileChunkList: chunks });
  });
}
```

### 6.2 使用流式写入优化合并

对于超大文件（几个 GB），`readFileSync` + `write` 会占用大量内存。使用流式写入更高效：

```typescript
/**
 * 使用 Stream 流式合并分片（内存友好）
 */
async mergeChunksStream(
  fileHash: string,
  totalChunks: number,
  finalFilePath: string
): Promise<void> {
  const chunkDir = this.getChunkDir(fileHash);

  return new Promise((resolve, reject) => {
    const writeStream = fs.createWriteStream(finalFilePath);

    // 逐个读取分片并 pipe 到写入流
    let currentIndex = 0;

    const appendChunk = () => {
      if (currentIndex >= totalChunks) {
        writeStream.end();
        return;
      }

      const chunkPath = path.resolve(chunkDir, `${currentIndex}`);
      const readStream = fs.createReadStream(chunkPath);

      readStream.on("end", () => {
        currentIndex++;
        appendChunk(); // 继续下一个分片
      });

      readStream.on("error", reject);
      readStream.pipe(writeStream, { end: false });
    };

    writeStream.on("finish", resolve);
    writeStream.on("error", reject);

    appendChunk();
  });
}
```

```typescript
export class UploadService {
  // ...

  /**
   * 每天凌晨 2 点清理超过 24 小时未合并的分片
   */
  @Cron("0 2 * * *")
  cleanExpiredChunks() {
    if (!fs.existsSync(CHUNK_DIR)) return;

    const dirs = fs.readdirSync(CHUNK_DIR);
    const now = Date.now();
    const maxAge = 24 * 60 * 60 * 1000; // 24 小时
    let cleaned = 0;

    for (const dir of dirs) {
      const dirPath = path.resolve(CHUNK_DIR, dir);
      const stat = fs.statSync(dirPath);

      if (now - stat.mtimeMs > maxAge) {
        fs.rmSync(dirPath, { recursive: true, force: true });
        cleaned++;
        this.logger.log(`清理过期分片: ${dir}`);
      }
    }

    if (cleaned > 0) {
      this.logger.log(`共清理 ${cleaned} 个过期分片目录`);
    }
  }
}
```

### 8.2 文件哈希抽样计算（性能优化）

对于超大文件，计算完整 MD5 可能需要几十秒。可以使用抽样哈希：取文件头部、中部、尾部各 2MB，加上文件大小，拼接后计算哈希：

```typescript
/**
 * 抽样哈希：牺牲少量准确性换取大幅速度提升
 * 适用于：视频、音频等大文件场景
 */
export async function calculateHashSample(file: File): Promise<string> {
  const SparkMD5 = (await import("spark-md5")).default;
  const spark = new SparkMD5.ArrayBuffer();
  const reader = new FileReader();

  // 文件大小
  const size = file.size;
  const offset = 2 * 1024 * 1024; // 每段取 2MB

  // 采样区域：头部 + 中部 + 尾部
  const chunks: Blob[] = [
    file.slice(0, offset), // 头部 2MB
    file.slice(Math.floor(size / 2), Math.floor(size / 2) + offset), // 中部 2MB
    file.slice(size - offset, size), // 尾部 2MB
  ];

  return new Promise((resolve) => {
    reader.onload = (e) => {
      spark.append(e.target!.result as ArrayBuffer);
      // 加入文件大小作为额外因子
      const sizeBuffer = new TextEncoder().encode(String(size));
      spark.append(sizeBuffer.buffer as ArrayBuffer);
      resolve(spark.end());
    };
    // 将三个区域拼接后计算
    const combinedBlob = new Blob(chunks);
    reader.readAsArrayBuffer(combinedBlob);
  });
}
```

## 九、常见问题

### Q1：切片上传和普通上传如何选择？

文件小于 50MB 时使用普通上传即可。切片上传的优势在大文件场景下才明显——网络中断恢复、并发加速、进度追踪等。对于小文件，切片反而增加了请求开销。

### Q2：哈希计算很慢怎么办？

两种优化方案：

1. 使用 Web Worker 在后台线程计算，不阻塞 UI
2. 使用抽样哈希，牺牲极少量准确性换取速度（见 8.2 节）

### Q3：合并后文件校验失败怎么办？

通常是某个分片上传不完整导致。解决方案：

- 合并前校验每个分片的大小
- 前端在上传时也计算分片的 MD5，传给后端做比对
- 合并失败后删除最终文件和已记录的分片，让前端重新上传

### Q4：多个用户同时上传相同文件怎么办？

没问题。相同哈希的分片写入同一目录，分片文件名就是索引号，不会冲突。合并时使用 `upsert` 操作，先完成的写入数据库记录，后来的发现已存在直接返回。

### Q5：分片上传中间可以关闭浏览器吗？

可以。只要服务端分片文件还在，下次打开页面重新上传相同文件时，会通过哈希匹配到已上传的分片，实现断点续传。需要注意服务端的分片过期清理时间（建议 24~48 小时）。

**本文我们完成了**：

- 理解切片上传的完整架构和流程
- 后端实现分片接收、合并、秒传、断点续传

- 进阶优化：Web Worker、流式合并、自动重试、定时清理

**核心流程回顾**：

```
计算哈希 → 秒传检查 → 断点续传检查 → 并发上传分片 → 合并
```

**关键设计决策**：

1. **分片大小 2MB** — 在请求数和传输效率之间取得平衡
2. **哈希使用 SparkMD5** — 分片读取避免内存溢出
3. **并发数 3** — 避免浏览器同域 6 连接限制导致的阻塞
4. **分片路径用哈希隔离** — 不同文件的分片互不干扰
5. **合并后清理分片** — 避免磁盘空间浪费
