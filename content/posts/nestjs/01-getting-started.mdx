## 一、NestJS 是什么？

### 1.1 什么是 NestJS？

**NestJS** 是一个用于构建高效、可扩展的 Node.js 服务器端应用程序的框架。它使用 **TypeScript** 构建，结合了 **面向对象编程（OOP）**、**函数式编程（FP）** 和 **函数响应式编程（FRP）** 的元素。

简单来说：

- **Express** 是一个简洁的 Web 框架，给你最大的自由度
- **NestJS** 在 Express 之上，提供了**结构化的架构**和**企业级开发体验**

### 1.2 NestJS 解决了什么问题？

使用 Express 或 Koa 等传统框架时，随着项目变大，你会遇到这些问题：

```javascript
// ❌ 传统 Express 项目容易变成这样
// app.js
const express = require("express");
const app = express();

// 所有路由都写在一个文件里
app.get("/users", (req, res) => {
  // 数据库查询逻辑
  // 业务逻辑
  // 响应格式化
  // 全部混在一起！
});

app.post("/users", (req, res) => {
  // 又是一堆逻辑...
});

// 当项目有 100+ 个路由时，这个文件可能有几千行...
```

**NestJS 的解决方案**：

```typescript
// ✅ NestJS 的组织方式
// users/users.controller.ts
@Controller("users")
export class UsersController {
  constructor(private usersService: UsersService) {}

  @Get()
  findAll() {
    return this.usersService.findAll();
  }

  @Post()
  create(@Body() createUserDto: CreateUserDto) {
    return this.usersService.create(createUserDto);
  }
}

// users/users.service.ts
@Injectable()
export class UsersService {
  constructor(private prisma: PrismaService) {}

  findAll() {
    return this.prisma.user.findMany();
  }

  create(dto: CreateUserDto) {
    return this.prisma.user.create({ data: dto });
  }
}
```

**核心优势**：

1. **结构化架构**：模块化设计，代码自动组织到不同目录
2. **依赖注入**：自动管理依赖，解耦业务逻辑
3. **TypeScript 原生支持**：完整的类型安全
4. **企业级模式**：内置装饰器、中间件、守卫、拦截器等
5. **可测试性**：每个组件都可以独立测试
6. **可扩展性**：轻松支持微服务、GraphQL、WebSocket 等

## 二、环境搭建

### 2.1 前置要求

在开始之前，请确保你的电脑已经安装：

- **Node.js** >= 18.0.0（推荐 20.x LTS）
- **npm** >= 9.0.0 或 **pnpm** >= 8.0.0
- **Git**（可选，但推荐）

**检查版本**：

```bash
node --version   # 应该显示 v18.x.x 或更高
npm --version    # 应该显示 9.x.x 或更高
```

如果你的版本过低，请访问 [nodejs.org](https://nodejs.org/) 下载最新 LTS 版本。

### 2.2 全局安装 Nest CLI

**Nest CLI** 是 NestJS 的命令行工具，它能帮你快速创建项目、生成文件等。

```bash
# 使用 npm 安装
npm install -g @nestjs/cli

# 或使用 pnpm 安装（更快，更节省空间）
pnpm install -g @nestjs/cli
```

**验证安装**：

```bash
nest --version
# 应该显示类似：10.3.2
```

### 2.3 创建第一个项目

使用 CLI 创建新项目：

```bash
# 方式一：使用 nest new 命令
nest new my-nestjs-blog

# 方式二：指定包管理器（推荐）
nest new my-nestjs-blog -p pnpm

# 或使用 npm
nest new my-nestjs-blog -p npm
```

CLI 会问你几个问题：

```
? Which package manager would you like to use?
  npm
  yarn
  pnpm    # 推荐选择 pnpm
```

选择后，CLI 会自动：

1. 创建项目目录
2. 初始化 `package.json`
3. 生成基础代码结构
4. 安装依赖
5. 初始化 Git 仓库

**预计时间**：1-3 分钟（取决于网络速度）

### 3.2 核心文件说明

#### **main.ts** - 应用入口

```typescript
import { NestFactory } from "@nestjs/core";
import { AppModule } from "./app.module";

async function bootstrap() {
  const app = await NestFactory.create(AppModule);
  await app.listen(3000);
}
bootstrap();
```

**这段代码做了什么？**

1. `NestFactory.create(AppModule)` - 创建 NestJS 应用实例
2. `app.listen(3000)` - 监听 3000 端口
3. `bootstrap()` - 启动应用程序

#### **app.module.ts** - 根模块

```typescript
import { Module } from "@nestjs/common";
import { AppController } from "./app.controller";
import { AppService } from "./app.service";

@Module({
  imports: [], // 导入其他模块
  controllers: [AppController], // 声明控制器
  providers: [AppService], // 声明服务提供者
})
export class AppModule {}
```

**@Module 装饰器**是 NestJS 的核心，它告诉 NestJS 如何组织应用：

- `imports`：导入其他模块（稍后详细讲解）
- `controllers`：处理 HTTP 请求的控制器
- `providers`：提供业务逻辑的服务

#### **app.controller.ts** - 根控制器

```typescript
import { Controller, Get } from "@nestjs/common";
import { AppService } from "./app.service";

@Controller()
export class AppController {
  constructor(private readonly appService: AppService) {}

  @Get()
  getHello(): string {
    return this.appService.getHello();
  }
}
```

**这段代码做了什么？**

1. `@Controller()` - 声明这是一个控制器类
2. `constructor` - 通过依赖注入获取 `AppService`
3. `@Get()` - 定义一个 GET 路由（对应根路径 `/`）
4. `getHello()` - 调用服务层的方法

#### **app.service.ts** - 根服务

```typescript
import { Injectable } from "@nestjs/common";

@Injectable()
export class AppService {
  getHello(): string {
    return "Hello World!";
  }
}
```

**这段代码做了什么？**

1. `@Injectable()` - 声明这是一个可以被注入的服务
2. `getHello()` - 提供业务逻辑（返回字符串）

### 5.2 查看生成的控制器

打开 `src/users/users.controller.ts`：

```typescript
import {
  Controller,
  Get,
  Post,
  Body,
  Patch,
  Param,
  Delete,
} from "@nestjs/common";
import { UsersService } from "./users.service";
import { CreateUserDto } from "./dto/create-user.dto";
import { UpdateUserDto } from "./dto/update-user.dto";

@Controller("users")
export class UsersController {
  constructor(private readonly usersService: UsersService) {}

  @Post()
  create(@Body() createUserDto: CreateUserDto) {
    return this.usersService.create(createUserDto);
  }

  @Get()
  findAll() {
    return this.usersService.findAll();
  }

  @Get(":id")
  findOne(@Param("id") id: string) {
    return this.usersService.findOne(id);
  }

  @Patch(":id")
  update(@Param("id") id: string, @Body() updateUserDto: UpdateUserDto) {
    return this.usersService.update(id, updateUserDto);
  }

  @Delete(":id")
  remove(@Param("id") id: string) {
    return this.usersService.remove(id);
  }
}
```

**这段代码做了什么？**

- `@Controller('users')` - 定义路由前缀为 `/users`
- `@Post()` - 创建 POST 路由 `/users`
- `@Get()` - 创建 GET 路由 `/users`
- `@Get(':id')` - 创建 GET 路由 `/users/:id`
- `@Patch(':id')` - 创建 PATCH 路由 `/users/:id`
- `@Delete(':id')` - 创建 DELETE 路由 `/users/:id`

### 5.3 实现简单的用户服务

修改 `src/users/users.service.ts`：

```typescript
import { Injectable, NotFoundException } from "@nestjs/common";
import { CreateUserDto } from "./dto/create-user.dto";
import { UpdateUserDto } from "./dto/update-user.dto";

@Injectable()
export class UsersService {
  // 临时使用内存存储，后面会换成数据库
  private users = [
    { id: "1", name: "张三", email: "zhangsan@example.com" },
    { id: "2", name: "李四", email: "lisi@example.com" },
  ];

  create(createUserDto: CreateUserDto) {
    const newUser = {
      id: String(Date.now()),
      ...createUserDto,
    };
    this.users.push(newUser);
    return newUser;
  }

  findAll() {
    return this.users;
  }

  findOne(id: string) {
    const user = this.users.find((u) => u.id === id);
    if (!user) {
      throw new NotFoundException(`用户 ID ${id} 不存在`);
    }
    return user;
  }

  update(id: string, updateUserDto: UpdateUserDto) {
    const userIndex = this.users.findIndex((u) => u.id === id);
    if (userIndex === -1) {
      throw new NotFoundException(`用户 ID ${id} 不存在`);
    }
    this.users[userIndex] = {
      ...this.users[userIndex],
      ...updateUserDto,
    };
    return this.users[userIndex];
  }

  remove(id: string) {
    const userIndex = this.users.findIndex((u) => u.id === id);
    if (userIndex === -1) {
      throw new NotFoundException(`用户 ID ${id} 不存在`);
    }
    this.users.splice(userIndex, 1);
    return { message: "删除成功" };
  }
}
```

## 六、NestJS 核心概念总结

通过第一个项目，你已经接触了 NestJS 的三大核心概念：

### 6.1 模块（Module）

**模块是 NestJS 应用的基本组织单元**。

```typescript
@Module({
  imports: [OtherModule], // 导入其他模块
  controllers: [UsersController], // 声明控制器
  providers: [UsersService], // 声明服务
  exports: [UsersService], // 导出服务供其他模块使用
})
export class UsersModule {}
```

**作用**：

- 组织应用结构
- 管理依赖关系
- 提供封装性

### 6.2 控制器（Controller）

**控制器负责处理 HTTP 请求**。

```typescript
@Controller('users')  // 路由前缀
export class UsersController {
  @Get()              // GET /users
  findAll() { ... }

  @Get(':id')         // GET /users/:id
  findOne(@Param('id') id: string) { ... }

  @Post()             // POST /users
  create(@Body() dto: CreateUserDto) { ... }
}
```

**作用**：

- 定义路由
- 接收请求参数
- 返回响应

### 6.3 服务（Service/Provider）

**服务包含业务逻辑**。

```typescript
@Injectable() // 声明为可注入的服务
export class UsersService {
  findAll() {
    // 业务逻辑
    return "数据";
  }
}
```

**作用**：

- 实现业务逻辑
- 数据处理
- 可被多个控制器复用

### 6.4 依赖注入（DI）

**NestJS 自动管理依赖关系**。

```typescript
@Controller("users")
export class UsersController {
  // 无需手动创建，NestJS 自动注入
  constructor(private usersService: UsersService) {}
}
```

**好处**：

- 解耦组件
- 便于测试
- 自动管理生命周期

## 八、常见问题

### Q1: 为什么要使用 TypeScript？

**A**: TypeScript 提供了：

- ✅ 类型安全，减少运行时错误
- ✅ 更好的 IDE 智能提示
- ✅ 更容易重构
- ✅ 更好的代码文档化

### Q2: NestJS 和 Express 的关系？

**A**: NestJS 底层默认使用 Express，但也支持 Fastify。

- Express = 底层 HTTP 服务器
- NestJS = 在 Express 之上的结构化框架

你可以这样理解：

```
Express (原始) → NestJS (封装) → 你的应用
```

### Q3: 如何调试 NestJS 应用？

**A**: 在 VSCode 中配置 `.vscode/launch.json`：

```json
{
  "version": "0.2.0",
  "configurations": [
    {
      "type": "node",
      "request": "launch",
      "name": "Debug NestJS",
      "runtimeExecutable": "pnpm",
      "runtimeArgs": ["run", "start:debug", "--", "--inspect-brk"],
      "console": "integratedTerminal",
      "restart": true,
      "protocol": "inspector"
    }
  ]
}
```

然后按 F5 启动调试。
