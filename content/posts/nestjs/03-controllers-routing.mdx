## 一、什么是控制器？

### 1.1 控制器的概念

**控制器（Controller）** 是 NestJS 中负责处理 HTTP 请求的组件，它：

- 定义路由路径
- 接收客户端请求
- 调用服务层处理业务逻辑
- 返回响应给客户端

**类比**：

想象一下餐厅的服务员：

```
顾客（客户端）
  ↓ 点菜
服务员（控制器） ← 负责接待顾客、传递订单
  ↓ 传达订单
厨房（服务层） ← 实际做菜的地方
  ↓ 做好菜
服务员（控制器） ← 端菜给顾客
  ↓
顾客（客户端） ← 享用美食
```

### 1.2 为什么需要控制器？

**没有控制器的问题**：

```typescript
// ❌ 将所有逻辑混在一个地方
import { Injectable } from "@nestjs/common";

@Injectable()
export class AppService {
  // 业务逻辑、HTTP 处理、数据验证全在一起
  async handleGetUser(id: string) {
    // 验证参数
    if (!id) throw new Error("Invalid ID");

    // 查询数据库
    const user = await this.db.findUser(id);

    // 格式化响应
    return { id: user.id, name: user.name };
  }
}
```

**使用控制器的好处**：

```typescript
// ✅ 职责分离
@Controller("users")
export class UsersController {
  constructor(private usersService: UsersService) {}

  @Get(":id")
  async findOne(@Param("id") id: string) {
    // 控制器只负责：接收请求、调用服务、返回响应
    return this.usersService.findOne(id);
  }
}

// users.service.ts
@Injectable()
export class UsersService {
  // 服务层专注业务逻辑
  async findOne(id: string) {
    return this.db.findUser(id);
  }
}
```

**核心优势**：

- ✅ 职责清晰：控制器处理 HTTP，服务处理业务
- ✅ 便于测试：可以独立测试控制器和服务
- ✅ 代码复用：服务可以被多个控制器使用
- ✅ 易于维护：修改路由不影响业务逻辑

## 二、创建控制器

### 2.1 使用 CLI 生成控制器

```bash
# 生成控制器
nest g controller users

# 或使用完整命令
nest generate controller users
```

CLI 会自动创建：

```
src/users/
├── users.controller.spec.ts    # 测试文件
├── users.controller.ts         # 控制器
└── users.module.ts             # 模块（如果不存在）
```

### 2.2 基础控制器结构

```typescript
import { Controller } from "@nestjs/common";

@Controller("users") // 路由前缀
export class UsersController {
  // 路由处理方法
}
```

**@Controller() 装饰器**：

- 接受一个字符串参数作为路由前缀
- 所有路由都会自动加上这个前缀

```typescript
@Controller("users")
export class UsersController {
  @Get()
  findAll() {} // 实际路由：GET /users

  @Get(":id")
  findOne() {} // 实际路由：GET /users/:id
}
```

## 三、HTTP 方法装饰器

NestJS 提供了对应所有 HTTP 方法的装饰器：

### 3.1 常用装饰器

| 装饰器       | HTTP 方法 | 用途               |
| ------------ | --------- | ------------------ |
| `@Get()`     | GET       | 获取资源           |
| `@Post()`    | POST      | 创建资源           |
| `@Put()`     | PUT       | 完整更新资源       |
| `@Patch()`   | PATCH     | 部分更新资源       |
| `@Delete()`  | DELETE    | 删除资源           |
| `@All()`     | 所有      | 处理所有 HTTP 方法 |
| `@Options()` | OPTIONS   | 处理 OPTIONS 请求  |
| `@Head()`    | HEAD      | 处理 HEAD 请求     |

### 3.2 完整示例

```typescript
import { Controller, Get, Post, Put, Patch, Delete } from "@nestjs/common";

@Controller("users")
export class UsersController {
  // GET /users - 获取所有用户
  @Get()
  findAll() {
    return ["user1", "user2", "user3"];
  }

  // GET /users/:id - 获取单个用户
  @Get(":id")
  findOne(@Param("id") id: string) {
    return `用户 ${id}`;
  }

  // POST /users - 创建用户
  @Post()
  create() {
    return "创建用户";
  }

  // PUT /users/:id - 完整更新用户
  @Put(":id")
  replace(@Param("id") id: string) {
    return `更新用户 ${id}`;
  }

  // PATCH /users/:id - 部分更新用户
  @Patch(":id")
  update(@Param("id") id: string) {
    return `部分更新用户 ${id}`;
  }

  // DELETE /users/:id - 删除用户
  @Delete(":id")
  remove(@Param("id") id: string) {
    return `删除用户 ${id}`;
  }
}
```

## 四、获取请求参数

NestJS 提供了多个装饰器来获取不同类型的请求参数：

### 4.1 参数装饰器一览

| 装饰器                   | 来源                 | 示例            |
| ------------------------ | -------------------- | --------------- |
| `@Req()` / `@Request()`  | 完整的 request 对象  | `req.headers`   |
| `@Res()` / `@Response()` | 完整的 response 对象 | `res.json()`    |
| `@Param(key?)`           | 路由参数             | `/users/:id`    |
| `@Query(key?)`           | 查询参数             | `/users?page=1` |
| `@Body(key?)`            | 请求体               | POST 数据       |
| `@Headers(key?)`         | 请求头               | `authorization` |
| `@Ip()`                  | 客户端 IP            | `192.168.1.1`   |
| `@HostParam()`           | 主机参数             | `subdomain`     |

### 4.2 路由参数 @Param()

**场景**：从 URL 路径中获取参数

```typescript
@Controller("users")
export class UsersController {
  // GET /users/:id
  @Get(":id")
  findOne(@Param("id") id: string) {
    return `用户 ID: ${id}`;
  }

  // GET /users/:id/posts/:postId
  @Get(":id/posts/:postId")
  findUserPost(@Param("id") id: string, @Param("postId") postId: string) {
    return `用户 ${id} 的文章 ${postId}`;
  }

  // 获取所有参数
  @Get(":id/profile/:section")
  getProfile(@Param() params: any) {
    // params = { id: '123', section: 'basic' }
    return params;
  }
}
```

### 4.3 查询参数 @Query()

**场景**：从 URL 查询字符串中获取参数

```typescript
@Controller("users")
export class UsersController {
  // GET /users?page=1&limit=10
  @Get()
  findAll(@Query() query: any) {
    // query = { page: '1', limit: '10' }
    const { page = 1, limit = 10 } = query;
    return `第 ${page} 页，每页 ${limit} 条`;
  }

  // 获取特定查询参数
  @Get("search")
  search(@Query("keyword") keyword: string) {
    return `搜索关键词: ${keyword}`;
  }

  // 多个查询参数
  @Get()
  findAll(
    @Query("page") page: string = "1",
    @Query("limit") limit: string = "10",
    @Query("sort") sort: string = "createdAt"
  ) {
    return {
      page: parseInt(page),
      limit: parseInt(limit),
      sort,
    };
  }
}
```

**实际应用示例**：

```typescript
@Controller("posts")
export class PostsController {
  @Get()
  async findAll(
    @Query("page") page: string = "1",
    @Query("limit") limit: string = "10",
    @Query("search") search?: string,
    @Query("category") category?: string
  ) {
    const pageNum = parseInt(page);
    const limitNum = parseInt(limit);
    const skip = (pageNum - 1) * limitNum;

    const where = {
      ...(search && {
        OR: [
          { title: { contains: search } },
          { content: { contains: search } },
        ],
      }),
      ...(category && { category }),
    };

    const [items, total] = await Promise.all([
      this.prisma.post.findMany({
        where,
        skip,
        take: limitNum,
        orderBy: { createdAt: "desc" },
      }),
      this.prisma.post.count({ where }),
    ]);

    return {
      items,
      total,
      page: pageNum,
      limit: limitNum,
      totalPages: Math.ceil(total / limitNum),
    };
  }
}
```

### 4.4 请求体 @Body()

**场景**：获取 POST/PUT/PATCH 请求的数据

```typescript
@Controller("users")
export class UsersController {
  // POST /users
  // Body: { "name": "张三", "email": "zhangsan@example.com" }
  @Post()
  create(@Body() userData: any) {
    // userData = { name: '张三', email: 'zhangsan@example.com' }
    return this.usersService.create(userData);
  }

  // 获取特定字段
  @Post()
  create(@Body("name") name: string, @Body("email") email: string) {
    return this.usersService.create({ name, email });
  }
}
```

**配合 DTO 使用（推荐）**：


### 4.5 请求头 @Headers()

```typescript
@Controller("users")
export class UsersController {
```

### 4.6 客户端 IP @Ip()

```typescript
@Controller("users")
export class UsersController {

  login(@Ip() ip: string, @Body() loginDto: LoginDto) {
```

### 4.7 综合示例

```typescript
@Controller("posts")
export class PostsController {
  // PUT /posts/:id/publish
  // Headers: Authorization: Bearer token123

  async publish(
    @Param("id") id: string, // 路由参数
    @Body("published") published: boolean, // 请求体字段
```

## 五、设计 RESTful API

### 5.1 RESTful 设计原则

**资源（Resource）**：一切都是资源

- 用户（users）
- 文章（posts）
- 评论（comments）

**HTTP 方法**：操作类型

- GET - 获取资源
- POST - 创建资源
- PUT/PATCH - 更新资源
- DELETE - 删除资源

**URL 设计**：名词而非动词

```
❌ 错误示例：
GET /getUsers
POST /createUser
PUT /updateUser
DELETE /deleteUser

✅ 正确示例：
GET /users
POST /users
PUT /users/:id
DELETE /users/:id
```

### 5.2 标准的 RESTful 路由设计

#### **用户管理 API**

```typescript
@Controller("users")
export class UsersController {
  // 获取所有用户（支持分页、搜索）
  @Get()
  findAll(
    @Query("page") page: string = "1",
    @Query("limit") limit: string = "10",
    @Query("search") search?: string
  ) {
    return this.usersService.findAll({ page, limit, search });
  }

  // 获取单个用户
  @Get(":id")
  findOne(@Param("id") id: string) {
    return this.usersService.findOne(id);
  }

  // 创建用户
  @Post()
  create(@Body() createUserDto: CreateUserDto) {
    return this.usersService.create(createUserDto);
  }

  // 完整更新用户
  @Put(":id")
  replace(@Param("id") id: string, @Body() updateUserDto: UpdateUserDto) {
    return this.usersService.replace(id, updateUserDto);
  }

  // 部分更新用户
  @Patch(":id")
  update(@Param("id") id: string, @Body() updateUserDto: UpdateUserDto) {
    return this.usersService.update(id, updateUserDto);
  }

  // 删除用户
  @Delete(":id")
  remove(@Param("id") id: string) {
    return this.usersService.remove(id);
  }

  // 获取用户的文章
  @Get(":id/posts")
  findUserPosts(
    @Param("id") userId: string,
    @Query("page") page: string = "1"
  ) {
    return this.postsService.findByUser(userId, { page });
  }

  // 获取用户的评论
  @Get(":id/comments")
  findUserComments(@Param("id") userId: string) {
    return this.commentsService.findByUser(userId);
  }
}
```

#### **文章管理 API**

```typescript
@Controller("posts")
export class PostsController {
  // 获取所有文章
  @Get()
  findAll(@Query() query: PostQueryDto) {
    return this.postsService.findAll(query);

  }

  // 删除文章
  @Delete(":id")
  remove(@Param("id") id: string) {
```

### 5.3 嵌套资源处理

对于嵌套资源（如用户的文章），有两种设计方式：

**方式一：扁平化（推荐）**

```typescript
// 获取用户的文章
GET /users/:userId/posts

// 获取文章的评论
GET /posts/:postId/comments
```

**方式二：完全嵌套**

```typescript
// 获取特定文章的特定评论
GET /users/:userId/posts/:postId/comments/:commentId
```

**实践建议**：

- 嵌套不超过 2 层
- 优先使用扁平化设计

## 六、响应处理

### 6.1 自动响应

NestJS 会自动序列化返回的对象为 JSON：

```typescript
@Get()
findAll() {
```

### 6.2 设置状态码

使用 `@HttpCode()` 装饰器：

```typescript
import { HttpCode, HttpStatus } from '@nestjs/common';

@Post()
@HttpCode(HttpStatus.CREATED)  // 201
create() {
  return '创建成功';
}

@Post('login')
@HttpCode(HttpStatus.OK)  // 200（POST 默认返回 201）
login() {
  return '登录成功';
}

@Delete(':id')
@HttpCode(HttpStatus.NO_CONTENT)  // 204
remove() {
  return;
}
```

### 6.5 完整响应示例

```typescript
import {
  Controller,
  Get,
  Post,
  Put,
  Delete,
  HttpCode,
  HttpStatus,
} from "@nestjs/common";

@Controller("posts")
export class PostsController {
  // 201 Created
  @Post()
  @HttpCode(HttpStatus.CREATED)
  async create(@Body() dto: CreatePostDto) {
    const post = await this.postsService.create(dto);
    return {
      success: true,
      data: post,
      message: "文章创建成功",
    };
  }

  // 200 OK
  @Get(":id")
  async findOne(@Param("id") id: string) {
    const post = await this.postsService.findOne(id);
    return {
      success: true,
      data: post,
    };
  }

  // 200 OK（部分更新）
  @Patch(":id")
  async update(@Param("id") id: string, @Body() dto: UpdatePostDto) {
    const post = await this.postsService.update(id, dto);
    return {
      success: true,
      data: post,
      message: "文章更新成功",
    };
  }

  // 204 No Content
  @Delete(":id")
  @HttpCode(HttpStatus.NO_CONTENT)
  async remove(@Param("id") id: string) {
    await this.postsService.remove(id);
    // 不返回任何内容
  }

  // 400 Bad Request（手动抛出异常）
  @Post()
  async createWithError(@Body() dto: CreatePostDto) {
    if (!dto.title) {
      throw new BadRequestException("标题不能为空");
    }
    return this.postsService.create(dto);
  }
}
```

## 七、高级路由技巧

### 7.1 路由通配符

```typescript
@Controller("users")
export class UsersController {
  // 匹配 /users/*（不包含子路径）
  @Get("*")
  wildcard() {
    return "通配符路由";
  }
}
```

### 7.2 路由优先级

路由匹配是按照定义顺序的：

```typescript
@Controller("users")
export class UsersController {
  @Get("special") // 优先匹配
  special() {
    return "特殊路由";
  }

  @Get(":id") // 然后匹配这个
  findOne() {
    return "用户详情";
  }

  @Get("*") // 最后匹配通配符
  wildcard() {
    return "未找到";
  }
}
```

### 7.3 路由版本控制

**方式一：URL 版本**

```typescript
@Controller("v1/users")
export class UsersV1Controller {
  @Get()
  findAll() {
    return "V1 API";
  }
}

@Controller("v2/users")
export class UsersV2Controller {
  @Get()
  findAll() {
    return "V2 API";
  }
}
```

**方式二：请求头版本**

```typescript
@Controller("users")
export class UsersController {

  }

  @Get()
  @Version("2")

```

客户端通过请求头指定版本：

```bash
curl -H "Accept-Version: 1" http://localhost:3000/users
```

## 八、实战：博客 API 完整示例

让我们实现一个完整的博客文章控制器：

```typescript
// posts/posts.controller.ts
import {
  Controller,
  Get,
  Post,
  Put,
  Patch,
  Delete,
  Param,
  Query,
  Body,
  HttpCode,
  HttpStatus,
  ParseIntPipe,
  DefaultValuePipe,
} from "@nestjs/common";
import { PostsService } from "./posts.service";
import { CreatePostDto } from "./dto/create-post.dto";
import { UpdatePostDto } from "./dto/update-post.dto";

@Controller("posts")
export class PostsController {
  constructor(private postsService: PostsService) {}

  /**
   * 获取所有文章
   * GET /posts?page=1&limit=10&search=keyword
   */
  @Get()
  async findAll(
    @Query("page", new DefaultValuePipe(1), ParseIntPipe) page: number,
    @Query("limit", new DefaultValuePipe(10), ParseIntPipe) limit: number,
    @Query("search") search?: string,
    @Query("category") category?: string
  ) {
    const result = await this.postsService.findAll({
      page,
      limit,
      search,
      category,
    });

    return {
      success: true,
      data: result.items,
      pagination: {
        page,
        limit,
        total: result.total,
        totalPages: Math.ceil(result.total / limit),
      },
    };
  }

  /**
   * 获取单篇文章
   * GET /posts/:id
   */
  @Get(":id")
  async findOne(@Param("id") id: string) {
    const post = await this.postsService.findOne(id);

    return {
      success: true,
      data: post,
    };
  }

  /**
   * 创建文章
   * POST /posts
   * Body: { title, content, category }
   */
  @Post()
  @HttpCode(HttpStatus.CREATED)
  async create(@Body() createPostDto: CreatePostDto) {
    const post = await this.postsService.create(createPostDto);

    return {
      success: true,
      data: post,
      message: "文章创建成功",
    };
  }

  /**
   * 完整更新文章
   * PUT /posts/:id
   */
  @Put(":id")
  async replace(@Param("id") id: string, @Body() updatePostDto: UpdatePostDto) {
    const post = await this.postsService.replace(id, updatePostDto);

    return {
      success: true,
      data: post,
      message: "文章更新成功",
    };
  }

  /**
   * 部分更新文章
   * PATCH /posts/:id
   */
  @Patch(":id")
  async update(@Param("id") id: string, @Body() updatePostDto: UpdatePostDto) {
    const post = await this.postsService.update(id, updatePostDto);

    return {
      success: true,
      data: post,
      message: "文章更新成功",
    };
  }

  /**
   * 删除文章
   * DELETE /posts/:id
   */
  @Delete(":id")
  @HttpCode(HttpStatus.NO_CONTENT)
  async remove(@Param("id") id: string) {
    await this.postsService.remove(id);
  }

  /**
   * 发布文章
   * POST /posts/:id/publish
   */
  @Post(":id/publish")
  @HttpCode(HttpStatus.OK)
  async publish(@Param("id") id: string) {
    const post = await this.postsService.publish(id);

    return {
      success: true,
      data: post,
      message: "文章发布成功",
    };
  }

  /**
   * 获取文章的评论
   * GET /posts/:id/comments
   */
  @Get(":id/comments")
  async getComments(
    @Param("id") postId: string,
    @Query("page", new DefaultValuePipe(1), ParseIntPipe) page: number
  ) {
    const comments = await this.postsService.getComments(postId, { page });

    return {
      success: true,
      data: comments,
    };
  }
}
```

## 九、常见问题

### Q1: @Param() 和 @Body() 有什么区别？

**A**:

- `@Param()` - 获取 URL 路径中的参数（如 `/users/:id`）
- `@Body()` - 获取请求体中的数据（POST/PUT 请求的数据）

```typescript
// URL: PUT /users/123
// Body: { "name": "李四" }

@Put(':id')
update(
  @Param('id') id: string,  // id = '123'
  @Body() data: any,         // data = { name: '李四' }
) {}
```

### Q2: 什么时候使用 PUT，什么时候使用 PATCH？

**A**:

- **PUT** - 完整替换资源（需要提供所有字段）
- **PATCH** - 部分更新资源（只提供需要修改的字段）

```typescript
// PUT - 完整更新
@Put(':id')
replace(@Param('id') id: string, @Body() data: UpdateUserDto) {
  // data 必须包含所有字段
  // { name: '张三', email: 'zhangsan@example.com', age: 25 }
}

// PATCH - 部分更新
@Patch(':id')
update(@Param('id') id: string, @Body() data: Partial<UpdateUserDto>) {
  // data 可以只包含部分字段
  // { age: 26 }
}
```

### Q3: 如何处理查询参数的类型转换？

**A**: 使用 `ParseIntPipe`、`ParseBoolPipe` 等内置管道：

```typescript
@Get()
findAll(
  @Query('page', ParseIntPipe) page: number,  // 自动转为数字
  @Query('published', ParseBoolPipe) published: boolean,  // 自动转为布尔值
) {}
```

或者使用默认值管道：

```typescript
@Query('page', new DefaultValuePipe(1), ParseIntPipe) page: number
```
