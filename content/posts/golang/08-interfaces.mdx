## 为什么需要接口

假设你正在编写一个任务处理系统。系统需要处理不同类型的任务：邮件任务、短信任务、推送任务等。如果为每种任务类型编写单独的处理代码，会导致代码重复且难以扩展。

接口提供了解决方案。它定义了一组行为规范，任何实现了这些行为的类型都可以被视为该接口的实现者。这使得代码更加灵活，可以轻松添加新的任务类型而无需修改现有代码。

Go 的接口是隐式实现的，不需要显式声明实现了某个接口。这种设计让接口的使用更加自然和灵活。

## 接口基础

### 定义和实现接口

```go
package main

import "fmt"

// 定义接口
type Processor interface {
    Process() error
    GetName() string
}

// 邮件任务
type EmailTask struct {
    To      string
    Subject string
    Body    string
}

// EmailTask 实现了 Processor 接口（隐式）
func (e *EmailTask) Process() error {
    fmt.Printf("发送邮件到: %s\n", e.To)
    fmt.Printf("  主题: %s\n", e.Subject)
    fmt.Printf("  内容: %s\n", e.Body)
    return nil
}

func (e *EmailTask) GetName() string {
    return fmt.Sprintf("邮件任务-%s", e.To)
}

// 短信任务
type SMSTask struct {
    Phone string
    Message string
}

// SMSTask 实现了 Processor 接口（隐式）
func (s *SMSTask) Process() error {
    fmt.Printf("发送短信到: %s\n", s.Phone)
    fmt.Printf("  内容: %s\n", s.Message)
    return nil
}

func (s *SMSTask) GetName() string {
    return fmt.Sprintf("短信任务-%s", s.Phone)
}

// 处理任意实现了 Processor 接口的任务
func executeTask(p Processor) error {
    fmt.Printf("执行任务: %s\n", p.GetName())
    return p.Process()
}

func main() {
    // 创建不同类型的任务
    email := &EmailTask{
        To:      "user@example.com",
        Subject: "欢迎使用",
        Body:    "感谢注册我们的服务",
    }

    sms := &SMSTask{
        Phone:   "13800138000",
        Message: "您的验证码是 123456",
    }

    // 使用统一的接口处理不同类型的任务
    tasks := []Processor{email, sms}

    fmt.Println("=== 任务处理系统 ===")
    for _, task := range tasks {
        if err := executeTask(task); err != nil {
            fmt.Printf("任务失败: %v\n", err)
        }
        fmt.Println()
    }
}
```

### 空接口

空接口 `interface{}` 没有任何方法要求，因此任何类型都实现了空接口。这在需要处理任意类型时很有用。

```go
package main

import "fmt"

// 打印任意类型的值
func printValue(v interface{}) {
    fmt.Printf("值: %v, 类型: %T\n", v, v)
}

// 存储不同类型的值
type Container struct {
    items []interface{}
}

func (c *Container) Add(item interface{}) {
    c.items = append(c.items, item)
}

func (c *Container) PrintAll() {
    for i, item := range c.items {
        fmt.Printf("索引 %d: %v (%T)\n", i, item, item)
    }
}

func main() {
    printValue(42)
    printValue("Hello")
    printValue(3.14)
    printValue([]int{1, 2, 3})

    fmt.Println("\n=== 容器示例 ===")
    container := &Container{}
    container.Add("字符串")
    container.Add(123)
    container.Add(3.14)
    container.Add([]string{"a", "b", "c"})
    container.PrintAll()
}
```

## 接口组合

Go 的接口可以组合其他接口，形成新的接口。这类似于结构体的嵌套。

```go
package main

import "fmt"

// Reader 接口
type Reader interface {
    Read() (string, error)
}

// Writer 接口
type Writer interface {
    Write(string) error
}

// ReadWriter 接口组合了 Reader 和 Writer
type ReadWriter interface {
    Reader
    Writer
}

// Closer 接口
type Closer interface {
    Close() error
}

// ReadWriteCloser 组合了多个接口
type ReadWriteCloser interface {
    Reader
    Writer
    Closer
}

// File 实现了所有接口
type File struct {
    content string
    closed  bool
}

func (f *File) Read() (string, error) {
    if f.closed {
        return "", fmt.Errorf("文件已关闭")
    }
    return f.content, nil
}

func (f *File) Write(data string) error {
    if f.closed {
        return fmt.Errorf("文件已关闭")
    }
    f.content = data
    return nil
}

func (f *File) Close() error {
    f.closed = true
    return nil
}

func main() {
    file := &File{}

    // 作为 ReadWriter 使用
    var rw ReadWriter = file
    rw.Write("Hello, World!")
    content, _ := rw.Read()
    fmt.Printf("内容: %s\n", content)

    // 作为 ReadWriteCloser 使用
    var rwc ReadWriteCloser = file
    rwc.Close()
    _, err := rwc.Read()
    fmt.Printf("读取关闭的文件: %v\n", err)
}
```

## 类型断言和类型选择

### 类型断言

类型断言用于将接口值转换为具体类型。

```go
package main

import "fmt"

func describeValue(i interface{}) {
    // 类型断言，返回值和是否成功的布尔值
    s, ok := i.(string)
    if ok {
        fmt.Printf("字符串: %q\n", s)
        return
    }

    n, ok := i.(int)
    if ok {
        fmt.Printf("整数: %d\n", n)
        return
    }

    fmt.Printf("未知类型: %T\n", i)
}

func main() {
    describeValue("Hello")
    describeValue(42)
    describeValue(3.14)
}
```

### 类型选择

类型选择是类型断言的更强大版本，可以处理多种类型。

```go
package main

import "fmt"

func processValue(i interface{}) string {
    // 类型选择
    switch v := i.(type) {
    case string:
        return fmt.Sprintf("字符串: %q", v)
    case int:
        return fmt.Sprintf("整数: %d", v)
    case float64:
        return fmt.Sprintf("浮点数: %.2f", v)
    case []int:
        return fmt.Sprintf("整数切片: %v", v)
    case nil:
        return "空值"
    default:
        return fmt.Sprintf("未知类型: %T", v)
    }
}

func main() {
    values := []interface{}{
        "Hello",
        42,
        3.14,
        []int{1, 2, 3},
        nil,
        true,
    }

    for _, v := range values {
        fmt.Println(processValue(v))
    }
}
```

## 标准接口

Go 标准库定义了一些广泛使用的接口。

### Stringer 接口

```go
package main

import "fmt"

// Stringer 接口在 fmt 包中定义
// type Stringer interface {
//     String() string
// }

type Person struct {
    Name string
    Age  int
}

func (p Person) String() string {
    return fmt.Sprintf("%s(%d岁)", p.Name, p.Age)
}

type Task struct {
    ID     int
    Name   string
    Status string
}

func (t Task) String() string {
    return fmt.Sprintf("Task[%d: %s - %s]", t.ID, t.Name, t.Status)
}

func main() {
    person := Person{Name: "张三", Age: 25}
    // fmt.Println 会自动调用 String() 方法
    fmt.Println(person)

    task := Task{ID: 1, Name: "发送邮件", Status: "完成"}
    fmt.Println(task)
}
```

### error 接口

```go
package main

import "fmt"

// 自定义错误类型
type ValidationError struct {
    Field   string
    Message string
}

func (e *ValidationError) Error() string {
    return fmt.Sprintf("验证失败: %s %s", e.Field, e.Message)
}

type NotFoundError struct {
    Resource string
    ID       int
}

func (e *NotFoundError) Error() string {
    return fmt.Sprintf("%s [ID:%d] 不存在", e.Resource, e.ID)
}

// 使用自定义错误
func validateEmail(email string) error {
    if email == "" {
        return &ValidationError{Field: "email", Message: "不能为空"}
    }
    return nil
}

func findUser(id int) error {
    if id > 100 {
        return &NotFoundError{Resource: "用户", ID: id}
    }
    return nil
}

func main() {
    if err := validateEmail(""); err != nil {
        fmt.Println("错误:", err)
    }

    if err := findUser(101); err != nil {
        fmt.Println("错误:", err)
    }
}
```

## 实战示例：任务队列 API 的接口设计

现在我们用接口来改进任务队列系统，使其更加灵活。

```go
package main

import (
    "fmt"
    "time"
)

// TaskStatus 任务状态类型
type TaskStatus string

const (
    StatusPending    TaskStatus = "待处理"
    StatusProcessing TaskStatus = "处理中"
    StatusCompleted  TaskStatus = "已完成"
    StatusFailed     TaskStatus = "失败"
)

// Task 接口定义了任务的基本行为
type Task interface {
    // GetID 返回任务 ID
    GetID() int
    // GetName 返回任务名称
    GetName() string
    // GetPriority 返回任务优先级
    GetPriority() int
    // GetStatus 返回任务状态
    GetStatus() TaskStatus
    // Execute 执行任务
    Execute() error
    // String 实现 Stringer 接口
    String() string
}

// BaseTask 提供任务的基本实现
type BaseTask struct {
    id        int
    name      string
    priority  int
    status    TaskStatus
    createdAt time.Time
    errorMsg  string
}

func NewBaseTask(id int, name string, priority int) *BaseTask {
    return &BaseTask{
        id:        id,
        name:      name,
        priority:  priority,
        status:    StatusPending,
        createdAt: time.Now(),
    }
}

func (bt *BaseTask) GetID() int          { return bt.id }
func (bt *BaseTask) GetName() string     { return bt.name }
func (bt *BaseTask) GetPriority() int    { return bt.priority }
func (bt *BaseTask) GetStatus() TaskStatus { return bt.status }
func (bt *BaseTask) SetStatus(status TaskStatus) { bt.status = status }
func (bt *BaseTask) SetError(msg string) { bt.errorMsg = msg }

func (bt *BaseTask) String() string {
    return fmt.Sprintf("Task[%d: %s - %s]", bt.id, bt.name, bt.status)
}

// EmailTask 邮件任务
type EmailTask struct {
    *BaseTask
    To      string
    Subject string
    Body    string
}

func NewEmailTask(id int, to, subject, body string, priority int) *EmailTask {
    return &EmailTask{
        BaseTask: NewBaseTask(id, fmt.Sprintf("邮件-%s", to), priority),
        To:       to,
        Subject:  subject,
        Body:     body,
    }
}

func (et *EmailTask) Execute() error {
    et.SetStatus(StatusProcessing)
    fmt.Printf("发送邮件到: %s\n", et.To)
    fmt.Printf("  主题: %s\n", et.Subject)

    // 模拟发送过程
    time.Sleep(time.Millisecond * 50)

    // 简单的失败逻辑
    if et.To == "fail@example.com" {
        et.SetStatus(StatusFailed)
        et.SetError("邮箱地址无效")
        return fmt.Errorf("发送失败: 邮箱地址无效")
    }

    et.SetStatus(StatusCompleted)
    return nil
}

// SMSTask 短信任务
type SMSTask struct {
    *BaseTask
    Phone   string
    Message string
}

func NewSMSTask(id int, phone, message string, priority int) *SMSTask {
    return &SMSTask{
        BaseTask: NewBaseTask(id, fmt.Sprintf("短信-%s", phone), priority),
        Phone:    phone,
        Message:  message,
    }
}

func (st *SMSTask) Execute() error {
    st.SetStatus(StatusProcessing)
    fmt.Printf("发送短信到: %s\n", st.Phone)
    fmt.Printf("  内容: %s\n", st.Message)

    // 模拟发送过程
    time.Sleep(time.Millisecond * 30)

    st.SetStatus(StatusCompleted)
    return nil
}

// ReportTask 报表任务
type ReportTask struct {
    *BaseTask
    ReportType string
    DateRange  string
}

func NewReportTask(id int, reportType, dateRange string, priority int) *ReportTask {
    return &ReportTask{
        BaseTask:  NewBaseTask(id, fmt.Sprintf("报表-%s", reportType), priority),
        ReportType: reportType,
        DateRange:  dateRange,
    }
}

func (rt *ReportTask) Execute() error {
    rt.SetStatus(StatusProcessing)
    fmt.Printf("生成报表: %s\n", rt.ReportType)
    fmt.Printf("  时间范围: %s\n", rt.DateRange)

    // 模拟生成过程
    time.Sleep(time.Millisecond * 100)

    rt.SetStatus(StatusCompleted)
    return nil
}

// TaskProcessor 任务处理器接口
type TaskProcessor interface {
    // ProcessTask 处理单个任务
    ProcessTask(task Task) error
    // GetProcessedCount 返回已处理的任务数
    GetProcessedCount() int
}

// SimpleProcessor 简单任务处理器
type SimpleProcessor struct {
    processedCount int
}

func NewSimpleProcessor() *SimpleProcessor {
    return &SimpleProcessor{}
}

func (sp *SimpleProcessor) ProcessTask(task Task) error {
    fmt.Printf("处理任务: %s\n", task.String())
    err := task.Execute()
    if err != nil {
        return fmt.Errorf("任务执行失败: %w", err)
    }
    sp.processedCount++
    return nil
}

func (sp *SimpleProcessor) GetProcessedCount() int {
    return sp.processedCount
}

// TaskQueue 任务队列
type TaskQueue struct {
    tasks     []Task
    processor TaskProcessor
}

func NewTaskQueue(processor TaskProcessor) *TaskQueue {
    return &TaskQueue{
        tasks:     make([]Task, 0),
        processor: processor,
    }
}

func (tq *TaskQueue) AddTask(task Task) {
    tq.tasks = append(tq.tasks, task)
}

func (tq *TaskQueue) ProcessAll() {
    fmt.Println("\n=== 开始处理任务 ===")
    for _, task := range tq.tasks {
        if err := tq.processor.ProcessTask(task); err != nil {
            fmt.Printf("错误: %v\n", err)
        }
    }
    fmt.Printf("\n处理完成，共处理 %d 个任务\n", tq.processor.GetProcessedCount())
}

func main() {
    // 创建任务处理器
    processor := NewSimpleProcessor()

    // 创建任务队列
    queue := NewTaskQueue(processor)

    // 添加不同类型的任务
    queue.AddTask(NewEmailTask(1, "user@example.com", "欢迎", "欢迎注册", 1))
    queue.AddTask(NewSMSTask(2, "13800138000", "验证码 123456", 2))
    queue.AddTask(NewReportTask(3, "销售报表", "2026-01", 3))
    queue.AddTask(NewEmailTask(4, "fail@example.com", "测试失败", "这会失败", 1))

    // 处理所有任务
    queue.ProcessAll()

    // 打印任务状态
    fmt.Println("\n=== 任务状态 ===")
    for _, task := range queue.tasks {
        fmt.Printf("%s: %s\n", task.GetName(), task.GetStatus())
    }
}
```

## 权衡与思考

### 接口设计的原则

**小接口原则**：

- 接口应该尽可能小，只包含必要的方法
- 小接口更容易实现，也更灵活
- 标准库的 io.Reader、io.Writer 都是很好的例子

**何时使用接口**：

- 需要处理多种不同的实现
- 希望代码更灵活、可测试
- 需要解耦调用者和实现者

**何时不使用接口**：

- 只有一种实现，且不太可能有其他实现
- 接口只是为了"面向对象"而设计
- 过早优化，增加了不必要的复杂性

### 接口 vs 结构体

**接口的优势**：

- 提供多态性
- 降低耦合
- 提高可测试性

**接口的代价**：

- 增加了一层间接性
- 可能有轻微的性能开销
- 过度使用会导致代码难以理解

### 类型断言的使用

**安全使用**：

- 总是使用 comma-ok 模式检查断言是否成功
- 使用类型选择处理多种类型
- 避免在核心路径上频繁使用类型断言

**性能考虑**：

- 类型断言有运行时开销
- 如果类型已知，避免使用空接口

## 总结

接口是 Go 语言实现抽象和多态的核心机制：

- 接口定义行为规范，不关心实现细节
- Go 的接口是隐式实现的，更加自然
- 接口可以组合，形成更复杂的接口
- 类型断言和类型选择用于处理接口值
- 小接口原则让代码更灵活
- 标准库定义了广泛使用的接口

在任务队列 API 中，接口让我们能够轻松添加新的任务类型，而不需要修改现有代码。这是接口带来的最大价值。
