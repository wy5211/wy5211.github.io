## 为什么需要理解变量声明

你已经在 Java 或 Python 中用过变量了，但 Go 的变量声明方式有点不一样。Go 是强类型语言，但能自动推断类型，这让你既能享受类型安全，又不用每次都写类型声明。

更重要的是，Go 的变量声明和初始化规则会影响你后续写代码的方式。理解了这些基础，你才能顺畅地学习函数、结构体、接口等更复杂的主题。这篇文章会用一个简单但实用的例子——任务状态追踪——来讲解 Go 的变量系统。

## 变量声明的多种方式

Go 提供了三种主要的变量声明方式，各有各的用途。

`var` 关键字是最标准的方式，可以声明一个或多个变量：

```go
package main

import "fmt"

func main() {
    // 标准声明：先声明类型，后赋值
    var taskName string
    taskName = "处理图片"
    fmt.Println(taskName)

    // 声明时直接初始化
    var taskCount int = 5
    fmt.Println(taskCount)

    // 类型推断：让编译器根据值推断类型（推荐）
    var priority = "high"  // 自动推断为 string
    fmt.Println(priority)
}
```

短变量声明 `:=` 是 Go 的特色，只能在函数内部使用：

```go
package main

import "fmt"

func main() {
    // 短变量声明：自动推断类型（最常用）
    taskID := 1001
    status := "pending"
    fmt.Printf("任务 %d 状态: %s\n", taskID, status)

    // 同时声明多个变量
    name, retries, timeout := "发送邮件", 3, 30
    fmt.Printf("任务: %s, 重试次数: %d, 超时: %d秒\n", name, retries, timeout)
}
```

批量声明可以保持代码整洁，特别适合定义相关的配置变量：

```go
package main

import "fmt"

func main() {
    // 批量声明多个变量
    var (
        maxWorkers    int     = 10
        queueSize     int     = 1000
        timeoutSecond float64 = 30.5
    )

    fmt.Printf("最大 Worker 数: %d\n", maxWorkers)
    fmt.Printf("队列大小: %d\n", queueSize)
    fmt.Printf("超时时间: %.1f秒\n", timeoutSecond)
}
```

## 基本数据类型

Go 的类型系统比 Python 严格，但比 Java 简洁。它提供了足够丰富的基本类型来处理各种场景。

### 布尔类型

布尔类型只有两个值：`true` 和 `false`。常用于条件判断：

```go
package main

import "fmt"

func main() {
    var isEnabled bool = true
    var isFailed bool = false

    fmt.Printf("任务启用: %t\n", isEnabled)
    fmt.Printf("任务失败: %t\n", isFailed)

    // 短变量声明
    isCompleted := false
    fmt.Printf("任务完成: %t\n", isCompleted)
}
```

### 数值类型

Go 提供了多种整型和浮点型，让你可以根据需求选择合适的精度和范围：

```go
package main

import "fmt"

func main() {
    // 有符号整型（int 的大小根据平台变化，通常是 64 位）
    var taskID int = 12345
    var retryCount int8 = 5        // -128 到 127
    var maxAttempts uint8 = 10     // 0 到 255

    fmt.Printf("任务 ID: %d\n", taskID)
    fmt.Printf("重试次数: %d\n", retryCount)
    fmt.Printf("最大尝试: %d\n", maxAttempts)

    // 浮点型
    var cpuUsage float32 = 75.5
    var memoryUsage float64 = 1024.256

    fmt.Printf("CPU 使用率: %.2f%%\n", cpuUsage)
    fmt.Printf("内存使用: %.3f MB\n", memoryUsage)
}
```

### 字符串类型

Go 的字符串是不可变的字节序列，支持 UTF-8 编码：

```go
package main

import "fmt"

func main() {
    // 字符串声明
    var taskName string = "数据备份"
    description := "每日凌晨 2 点执行全量备份"

    // 字符串拼接
    fullMessage := taskName + ": " + description
    fmt.Println(fullMessage)

    // 多行字符串（用反引号）
    logMessage := `
    任务启动时间: 2026-04-12 02:00:00
    任务状态: 运行中
    当前进度: 45%
    `
    fmt.Println(logMessage)

    // 字符串长度（字节数，不是字符数）
    fmt.Printf("消息长度: %d 字节\n", len(fullMessage))
}
```

## 常量和 iota

常量的值在编译时就确定了，运行时不能修改。`iota` 是 Go 的常量生成器，特别适合定义相关的枚举值：

```go
package main

import "fmt"

func main() {
    // 基本常量
    const maxRetries = 3
    const defaultTimeout = 30

    fmt.Printf("最大重试次数: %d\n", maxRetries)
    fmt.Printf("默认超时: %d秒\n", defaultTimeout)

    // 使用 iota 定义任务状态枚举
    const (
        StatusPending    = iota  // 0: 待处理
        StatusRunning            // 1: 运行中
        StatusCompleted          // 2: 已完成
        StatusFailed             // 3: 失败
    )

    fmt.Printf("待处理状态: %d\n", StatusPending)
    fmt.Printf("运行中状态: %d\n", StatusRunning)
    fmt.Printf("完成状态: %d\n", StatusCompleted)
    fmt.Printf("失败状态: %d\n", StatusFailed)

    // iota 的高级用法：位标志
    const (
        LogNone    = 1 << iota  // 1 (二进制: 0001)
        LogError                // 2 (二进制: 0010)
        LogWarn                 // 4 (二进制: 0100)
        LogInfo                 // 8 (二进制: 1000)
    )

    fmt.Printf("错误日志标志: %d\n", LogError)
    fmt.Printf("所有日志标志: %d\n", LogError|LogWarn|LogInfo)
}
```

## 类型转换

Go 不支持隐式类型转换，所有类型转换都必须显式声明。这避免了很多潜在的类型错误：

```go
package main

import (
    "fmt"
    "strconv"
)

func main() {
    // 基本类型转换
    var intValue int = 42
    var floatValue float64 = float64(intValue)
    fmt.Printf("整数: %d, 浮点数: %.2f\n", intValue, floatValue)

    // 浮点数转整数（注意：会直接截断小数部分）
    var pi float64 = 3.99
    var truncatedInt int = int(pi)
    fmt.Printf("原浮点数: %.2f, 转换后整数: %d\n", pi, truncatedInt)

    // 字符串和数字转换需要用 strconv 包
    strNumber := "100"
    num, err := strconv.Atoi(strNumber)
    if err != nil {
        fmt.Println("转换失败:", err)
    } else {
        fmt.Printf("字符串 '%s' 转为整数: %d\n", strNumber, num)
    }

    // 整数转字符串
    score := 95
    scoreStr := strconv.Itoa(score)
    fmt.Printf("整数 %d 转为字符串: '%s'\n", score, scoreStr)

    // 字符串转浮点数
    percentage := "85.5"
    percent, _ := strconv.ParseFloat(percentage, 64)
    fmt.Printf("字符串 '%s' 转为浮点数: %.1f%%\n", percentage, percent)
}
```

## 变量作用域和零值

理解变量的作用域和零值很重要，这能帮你避免很多常见错误。

Go 的变量有块级作用域，内层变量可以遮蔽外层同名变量：

```go
package main

import "fmt"

func main() {
    // 外层变量
    taskID := 100
    fmt.Printf("外层 taskID: %d\n", taskID)

    {
        // 内层作用域，声明新的 taskID（遮蔽外层）
        taskID := 200
        tempVar := "临时变量"
        fmt.Printf("内层 taskID: %d\n", taskID)
        fmt.Printf("内层临时变量: %s\n", tempVar)
    }

    // 外层 taskID 不受影响
    fmt.Printf("外层 taskID: %d\n", taskID)
    // fmt.Println(tempVar)  // 编译错误：tempVar 在这里不可访问
}
```

变量声明后会自动初始化为零值，不同类型的零值不同：

```go
package main

import "fmt"

func main() {
    var (
        i      int
        f      float64
        b      bool
        s      string
        ptr    *int
        slice  []int
        m      map[string]int
    )

    fmt.Printf("int 零值: %d\n", i)          // 0
    fmt.Printf("float64 零值: %f\n", f)      // 0.000000
    fmt.Printf("bool 零值: %t\n", b)         // false
    fmt.Printf("string 零值: '%s'\n", s)     // '' (空字符串)
    fmt.Printf("指针零值: %v\n", ptr)        // <nil>
    fmt.Printf("切片零值: %v\n", slice)      // []
    fmt.Printf("map 零值: %v\n", m)          // map[]
}
```

## 实战例子：任务状态检查器

把学到的知识整合起来，写一个简单的任务状态检查器：

```go
package main

import (
    "fmt"
    "strconv"
)

// 任务状态常量
const (
    StatusPending   = "pending"
    StatusRunning   = "running"
    StatusCompleted = "completed"
    StatusFailed    = "failed"
)

func main() {
    // 任务信息
    var taskName string = "图片处理任务"
    taskID := 1001
    retryCount := 0
    maxRetries := 3
    currentStatus := StatusPending

    fmt.Printf("=== 任务信息 ===\n")
    fmt.Printf("任务名称: %s\n", taskName)
    fmt.Printf("任务 ID: %d\n", taskID)
    fmt.Printf("当前状态: %s\n", currentStatus)

    // 模拟状态变化
    fmt.Printf("\n=== 任务执行 ===\n")
    currentStatus = StatusRunning
    fmt.Printf("状态更新: %s\n", currentStatus)

    // 模拟重试逻辑
    for retryCount < maxRetries {
        retryCount++
        fmt.Printf("尝试第 %d 次执行...\n", retryCount)

        // 假设第三次成功
        if retryCount == 3 {
            currentStatus = StatusCompleted
            fmt.Printf("任务执行成功！\n")
            break
        }
    }

    // 最终状态
    fmt.Printf("\n=== 最终结果 ===\n")
    fmt.Printf("任务名称: %s\n", taskName)
    fmt.Printf("最终状态: %s\n", currentStatus)
    fmt.Printf("执行次数: %d/%d\n", retryCount, maxRetries)

    // 类型转换示例：生成报告字符串
    reportID := strconv.Itoa(taskID)
    report := "任务 " + reportID + " 状态: " + currentStatus
    fmt.Printf("\n报告: %s\n", report)
}
```

## 权衡与考虑

Go 的类型系统在安全和简洁之间找到了平衡。显式类型转换虽然需要多写点代码，但避免了 JavaScript 那种隐式转换导致的诡异 bug。`iota` 机制让枚举值定义既简洁又类型安全。

短变量声明 `:=` 很方便，但要注意它只能在函数内使用。包级别的变量还是得用 `var`。变量遮蔽是 Go 的一个特性，但也可能导致代码难以理解，所以尽量避免在嵌套作用域里重用外层变量名。

零值机制让代码更安全，你不用担心变量未初始化的问题，但也要记住 map、slice、指针的零值是 `nil`，使用前需要检查。
