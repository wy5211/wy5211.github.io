## 为什么需要并发

假设你的任务队列 API 需要同时处理 1000 个任务。如果使用串行处理，每个任务耗时 100 毫秒，总共需要 100 秒。但如果使用并发处理，10 个工作线程同时工作，只需 10 秒就能完成。

并发编程是提高程序性能的重要手段。传统的线程模型存在以下问题：

- 线程创建和销毁开销大
- 线程栈内存占用高（通常几 MB）
- 线程切换成本高
- 线程数量受限于系统资源

Go 语言的 goroutine 解决了这些问题。它是轻量级线程，由 Go 运行时管理，具有以下特点：

- 创建和销毁开销极小
- 栈内存动态调整（初始几 KB）
- 切换成本低
- 可以轻松创建成千上万个 goroutine

## goroutine 基础

### 创建 goroutine

使用 `go` 关键字启动 goroutine。

```go
package main

import (
    "fmt"
    "time"
)

func sayHello(name string) {
    fmt.Printf("Hello, %s!\n", name)
}

func main() {
    // 普通函数调用
    sayHello("张三")

    // 启动 goroutine
    go sayHello("李四")
    go sayHello("王五")

    // 主 goroutine 需要等待子 goroutine 完成
    time.Sleep(time.Millisecond * 100)
}
```

### 匿名函数 goroutine

```go
package main

import (
    "fmt"
    "time"
)

func main() {
    // 使用匿名函数启动 goroutine
    go func() {
        fmt.Println("匿名 goroutine 1")
    }()

    // 带参数的匿名函数
    go func(msg string) {
        fmt.Println("消息:", msg)
    }("来自 goroutine")

    // 闭包捕获变量（注意：可能有问题）
    i := 0
    go func() {
        fmt.Println("闭包捕获的 i:", i)
    }()
    i = 100

    time.Sleep(time.Millisecond * 100)
}
```

### 闭包陷阱

在循环中创建 goroutine 时，需要注意变量捕获问题。

```go
package main

import (
    "fmt"
    "time"
)

func main() {
    // 错误示例：所有 goroutine 可能打印相同的值
    fmt.Println("=== 错误示例 ===")
    for i := 0; i < 3; i++ {
        go func() {
            fmt.Printf("i = %d\n", i)
        }()
    }
    time.Sleep(time.Millisecond * 100)

    // 正确示例 1：作为参数传递
    fmt.Println("\n=== 正确示例 1 ===")
    for i := 0; i < 3; i++ {
        go func(n int) {
            fmt.Printf("n = %d\n", n)
        }(i)
    }
    time.Sleep(time.Millisecond * 100)

    // 正确示例 2：在循环内部创建局部变量
    fmt.Println("\n=== 正确示例 2 ===")
    for i := 0; i < 3; i++ {
        i := i // 创建新的局部变量
        go func() {
            fmt.Printf("i = %d\n", i)
        }()
    }
    time.Sleep(time.Millisecond * 100)
}
```

## goroutine 同步

### WaitGroup

WaitGroup 用于等待一组 goroutine 完成。

```go
package main

import (
    "fmt"
    "sync"
    "time"
)

func worker(id int, wg *sync.WaitGroup) {
    defer wg.Done() // 完成时调用

    fmt.Printf("Worker %d 开始\n", id)
    time.Sleep(time.Second)
    fmt.Printf("Worker %d 完成\n", id)
}

func main() {
    var wg sync.WaitGroup

    // 启动 3 个 worker
    for i := 1; i <= 3; i++ {
        wg.Add(1) // 增加计数
        go worker(i, &wg)
    }

    wg.Wait() // 等待所有 worker 完成
    fmt.Println("所有 worker 完成")
}
```

### 互斥锁

当多个 goroutine 需要访问共享数据时，需要使用互斥锁。

```go
package main

import (
    "fmt"
    "sync"
)

type Counter struct {
    mu    sync.Mutex
    count int
}

func (c *Counter) Increment() {
    c.mu.Lock()
    defer c.mu.Unlock()
    c.count++
}

func (c *Counter) GetCount() int {
    c.mu.Lock()
    defer c.mu.Unlock()
    return c.count
}

func main() {
    counter := &Counter{}
    var wg sync.WaitGroup

    // 启动 1000 个 goroutine
    for i := 0; i < 1000; i++ {
        wg.Add(1)
        go func() {
            defer wg.Done()
            counter.Increment()
        }()
    }

    wg.Wait()
    fmt.Printf("最终计数值: %d\n", counter.GetCount())
}
```

## goroutine 通信

Go 推荐通过通信来共享内存，而不是通过共享内存来通信。

### 使用 channel

```go
package main

import "fmt"

// worker 从 jobs channel 接收任务，将结果发送到 results channel
func worker(id int, jobs <-chan int, results chan<- int) {
    for job := range jobs {
        fmt.Printf("Worker %d 处理任务 %d\n", id, job)
        // 模拟处理
        results <- job * 2
    }
}

func main() {
    jobs := make(chan int, 100)
    results := make(chan int, 100)

    // 启动 3 个 worker
    for w := 1; w <= 3; w++ {
        go worker(w, jobs, results)
    }

    // 发送 5 个任务
    for j := 1; j <= 5; j++ {
        jobs <- j
    }
    close(jobs)

    // 接收 5 个结果
    for j := 1; j <= 5; j++ {
        result := <-results
        fmt.Printf("收到结果: %d\n", result)
    }
}
```

## goroutine 调度

### Goroutine 调度器

Go 的调度器使用 M:N 调度模型，将 M 个 goroutine 映射到 N 个 OS 线程。

```go
package main

import (
    "fmt"
    "runtime"
    "time"
)

func main() {
    // 查看当前使用的 CPU 核数
    fmt.Printf("CPU 核数: %d\n", runtime.NumCPU())
    fmt.Printf("Goroutine 数量: %d\n", runtime.NumGoroutine())

    // 设置使用的 CPU 核数
    // runtime.GOMAXPROCS(runtime.NumCPU())

    // 启动大量 goroutine
    for i := 0; i < 100; i++ {
        go func(n int) {
            time.Sleep(time.Second)
            fmt.Printf("Goroutine %d 完成\n", n)
        }(i)
    }

    time.Sleep(time.Second * 2)
    fmt.Printf("最终 Goroutine 数量: %d\n", runtime.NumGoroutine())
}
```

## 实战示例：并发任务队列

现在我们用 goroutine 来实现真正的并发任务队列。

```go
package main

import (
    "fmt"
    "sync"
    "time"
)

// 任务状态
type TaskStatus string

const (
    StatusPending    TaskStatus = "待处理"
    StatusProcessing TaskStatus = "处理中"
    StatusCompleted  TaskStatus = "已完成"
    StatusFailed     TaskStatus = "失败"
)

// Task 接口
type Task interface {
    GetID() int
    GetName() string
    GetStatus() TaskStatus
    Execute() error
}

// BaseTask 基础任务实现
type BaseTask struct {
    id        int
    name      string
    status    TaskStatus
    error     error
    startTime time.Time
    endTime   time.Time
}

func NewBaseTask(id int, name string) *BaseTask {
    return &BaseTask{
        id:     id,
        name:   name,
        status: StatusPending,
    }
}

func (bt *BaseTask) GetID() int              { return bt.id }
func (bt *BaseTask) GetName() string         { return bt.name }
func (bt *BaseTask) GetStatus() TaskStatus   { return bt.status }
func (bt *BaseTask) SetStatus(s TaskStatus)  { bt.status = s }
func (bt *BaseTask) SetError(err error)      { bt.error = err }
func (bt *BaseTask) GetError() error         { return bt.error }
func (bt *BaseTask) SetStartTime(t time.Time) { bt.startTime = t }
func (bt *BaseTask) SetEndTime(t time.Time)   { bt.endTime = t }

// EmailTask 邮件任务
type EmailTask struct {
    *BaseTask
    To      string
    Subject string
}

func NewEmailTask(id int, to, subject string) *EmailTask {
    return &EmailTask{
        BaseTask: NewBaseTask(id, fmt.Sprintf("邮件-%s", to)),
        To:       to,
        Subject:  subject,
    }
}

func (et *EmailTask) Execute() error {
    et.SetStatus(StatusProcessing)
    et.SetStartTime(time.Now())

    fmt.Printf("[%s] 发送邮件到: %s\n", time.Now().Format("15:04:05.000"), et.To)

    // 模拟处理时间
    time.Sleep(time.Millisecond * 100)

    // 模拟失败
    if et.To == "fail@example.com" {
        et.SetStatus(StatusFailed)
        et.SetError(fmt.Errorf("邮箱地址无效"))
        et.SetEndTime(time.Now())
        return fmt.Errorf("发送失败: 邮箱地址无效")
    }

    et.SetStatus(StatusCompleted)
    et.SetEndTime(time.Now())
    return nil
}

// SMSTask 短信任务
type SMSTask struct {
    *BaseTask
    Phone   string
    Message string
}

func NewSMSTask(id int, phone, message string) *SMSTask {
    return &SMSTask{
        BaseTask: NewBaseTask(id, fmt.Sprintf("短信-%s", phone)),
        Phone:    phone,
        Message:  message,
    }
}

func (st *SMSTask) Execute() error {
    st.SetStatus(StatusProcessing)
    st.SetStartTime(time.Now())

    fmt.Printf("[%s] 发送短信到: %s\n", time.Now().Format("15:04:05.000"), st.Phone)

    // 模拟处理时间
    time.Sleep(time.Millisecond * 50)

    // 模拟失败
    if st.Phone == "error" {
        st.SetStatus(StatusFailed)
        st.SetError(fmt.Errorf("网络超时"))
        st.SetEndTime(time.Now())
        return fmt.Errorf("发送失败: 网络超时")
    }

    st.SetStatus(StatusCompleted)
    st.SetEndTime(time.Now())
    return nil
}

// Worker 工作线程
type Worker struct {
    id       int
    taskChan chan Task
    quit     chan bool
    wg       *sync.WaitGroup
}

func NewWorker(id int, taskChan chan Task, wg *sync.WaitGroup) *Worker {
    return &Worker{
        id:       id,
        taskChan: taskChan,
        quit:     make(chan bool),
        wg:       wg,
    }
}

func (w *Worker) Start() {
    w.wg.Add(1)
    go func() {
        defer w.wg.Done()
        fmt.Printf("Worker %d 启动\n", w.id)

        for {
            select {
            case task, ok := <-w.taskChan:
                if !ok {
                    fmt.Printf("Worker %d 停止\n", w.id)
                    return
                }
                // 处理任务
                if err := task.Execute(); err != nil {
                    fmt.Printf("Worker %d 处理任务失败: %v\n", w.id, err)
                }
            case <-w.quit:
                fmt.Printf("Worker %d 收到停止信号\n", w.id)
                return
            }
        }
    }()
}

func (w *Worker) Stop() {
    close(w.quit)
}

// TaskQueue 任务队列
type TaskQueue struct {
    workers   []*Worker
    taskChan  chan Task
    taskList  []Task
    workerWg  sync.WaitGroup
    mu        sync.Mutex
    nextID    int
}

func NewTaskQueue(workerCount int) *TaskQueue {
    taskChan := make(chan Task, 100)

    queue := &TaskQueue{
        workers:  make([]*Worker, workerCount),
        taskChan: taskChan,
        taskList: make([]Task, 0),
        nextID:   1,
    }

    // 创建并启动 workers
    for i := 0; i < workerCount; i++ {
        queue.workers[i] = NewWorker(i+1, taskChan, &queue.workerWg)
    }

    return queue
}

func (tq *TaskQueue) Start() {
    fmt.Println("=== 任务队列系统启动 ===")
    for _, worker := range tq.workers {
        worker.Start()
    }
}

func (tq *TaskQueue) AddTask(task Task) {
    tq.mu.Lock()
    defer tq.mu.Unlock()

    tq.taskList = append(tq.taskList, task)
    tq.taskChan <- task
    fmt.Printf("添加任务: %s (队列长度: %d)\n", task.GetName(), len(tq.taskChan))
}

func (tq *TaskQueue) Stop() {
    fmt.Println("\n=== 停止任务队列 ===")
    close(tq.taskChan)
    tq.workerWg.Wait()
}

func (tq *TaskQueue) PrintStats() {
    tq.mu.Lock()
    defer tq.mu.Unlock()

    completed := 0
    failed := 0
    pending := 0
    processing := 0

    for _, task := range tq.taskList {
        switch task.GetStatus() {
        case StatusCompleted:
            completed++
        case StatusFailed:
            failed++
        case StatusPending:
            pending++
        case StatusProcessing:
            processing++
        }
    }

    fmt.Println("\n=== 任务统计 ===")
    fmt.Printf("总任务数: %d\n", len(tq.taskList))
    fmt.Printf("已完成: %d\n", completed)
    fmt.Printf("失败: %d\n", failed)
    fmt.Printf("待处理: %d\n", pending)
    fmt.Printf("处理中: %d\n", processing)

    // 打印失败的任务
    if failed > 0 {
        fmt.Println("\n失败的任务:")
        for _, task := range tq.taskList {
            if task.GetStatus() == StatusFailed {
                fmt.Printf("  - %s: %v\n", task.GetName(), task.(*BaseTask).GetError())
            }
        }
    }
}

func main() {
    // 创建任务队列，3 个 worker
    queue := NewTaskQueue(3)
    queue.Start()

    // 添加任务
    fmt.Println("\n=== 添加任务 ===")
    queue.AddTask(NewEmailTask(1, "user1@example.com", "欢迎邮件"))
    queue.AddTask(NewEmailTask(2, "user2@example.com", "通知邮件"))
    queue.AddTask(NewSMSTask(3, "13800138001", "验证码 123456"))
    queue.AddTask(NewEmailTask(4, "fail@example.com", "这会失败"))
    queue.AddTask(NewSMSTask(5, "13800138002", "您的订单已发货"))
    queue.AddTask(NewEmailTask(6, "user3@example.com", "促销信息"))
    queue.AddTask(NewSMSTask(6, "error", "这也会失败"))
    queue.AddTask(NewEmailTask(7, "user4@example.com", "活动提醒"))
    queue.AddTask(NewSMSTask(8, "13800138003", "您的账户已激活"))

    // 等待所有任务完成
    time.Sleep(time.Second)

    // 停止队列
    queue.Stop()

    // 打印统计信息
    queue.PrintStats()
}
```

## goroutine 泄漏

### 常见泄漏场景

```go
package main

import (
    "fmt"
    "time"
)

// 泄漏场景 1：永远不会发送的 channel
func leak1() {
    ch := make(chan int)
    go func() {
        val := <-ch // 永远阻塞
        fmt.Println(val)
    }()
    // ch 从未发送数据，goroutine 永远阻塞
}

// 泄漏场景 2：忘记关闭 channel
func leak2() {
    ch := make(chan int)
    go func() {
        for val := range ch {
            fmt.Println(val)
        }
    }()
    ch <- 1
    // 忘记 close(ch)，goroutine 永远等待
}

// 泄漏场景 3：无限等待
func leak3() {
    go func() {
        for {
            time.Sleep(time.Second)
            // 没有退出条件
        }
    }()
}

// 正确的做法
func correct() {
    ch := make(chan int)
    go func() {
        select {
        case val := <-ch:
            fmt.Println(val)
        case <-time.After(time.Second):
            fmt.Println("超时退出")
        }
    }()
    time.Sleep(time.Second * 2)
}

func main() {
    fmt.Println("演示 goroutine 泄漏场景")
    leak1()
    leak2()
    leak3()
    correct()
    time.Sleep(time.Second * 2)
}
```

## 权衡与思考

### Goroutine vs 线程

**Goroutine 的优势**：

- 创建和销毁开销极小
- 内存占用小（初始几 KB）
- 切换成本低
- 可以轻松创建大量 goroutine

**Goroutine 的局限**：

- 不适合 CPU 密集型任务（会占用所有 CPU）
- 需要显式同步（WaitGroup、channel）
- 调试相对困难

**何时使用 goroutine**：

- I/O 密集型任务（网络、文件）
- 需要并发处理多个独立任务
- 需要提高响应速度

**何时不使用 goroutine**：

- 简单的顺序逻辑
- 任务之间有强依赖关系
- CPU 密集型任务（考虑使用 worker pool）

### 同步原语的选择

**Channel**：

- 适合传递数据
- 适合协调 goroutine
- 推荐：通过通信共享内存

**Mutex**：

- 适合保护共享状态
- 适合简单的互斥访问
- 性能开销比 channel 小

**WaitGroup**：

- 适合等待一组 goroutine 完成
- 不适合传递数据

### 并发级别

**如何确定并发度**：

- I/O 密集型：可以设置较高的并发度
- CPU 密集型：通常设置为 CPU 核数
- 混合型：需要根据实际情况调整

**过多 goroutine 的问题**：

- 调度开销增加
- 内存占用增加
- 可能导致性能下降

**建议**：

- 使用 worker pool 模式限制并发数
- 监控 goroutine 数量
- 根据实际情况调整并发度

## 总结

Goroutine 是 Go 并发编程的基础，它提供了轻量级的并发实现：

- 使用 `go` 关键字创建 goroutine
- WaitGroup 用于等待一组 goroutine 完成
- Channel 用于 goroutine 之间的通信
- Mutex 用于保护共享状态
- 注意避免 goroutine 泄漏

在任务队列 API 中，goroutine 让我们能够：

- 并发处理多个任务
- 充分利用系统资源
- 提高处理效率
- 实现真正的高并发系统
