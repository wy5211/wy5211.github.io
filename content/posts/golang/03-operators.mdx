## 为什么运算符很重要

你可能觉得运算符很基础，但它们是构建任何逻辑的原子。在任务队列 API 中，你需要用运算符来计算优先级、比较任务状态、判断重试条件、处理并发标志等。Go 的运算符和其他语言类似，但有一些细节差异需要注意。

理解运算符的优先级和短路求值能帮你写出更清晰、更高效的代码。这篇文章会用任务调度的实际场景来讲解各种运算符。

## 算术运算符

算术运算符是最基础的，用于数值计算。Go 的算术运算符和大多数语言一样，但有一个重要限制：`++` 和 `--` 只能作为语句，不能用作表达式。

```go
package main

import "fmt"

func main() {
    // 基本算术运算
    completedTasks := 15
    totalTasks := 20
    remainingTasks := totalTasks - completedTasks

    fmt.Printf("已完成任务: %d\n", completedTasks)
    fmt.Printf("剩余任务: %d\n", remainingTasks)

    // 整数除法会截断小数部分
    successRate := completedTasks / totalTasks
    fmt.Printf("成功率（整数除法）: %d\n", successRate)  // 输出 0，不是 0.75

    // 要得到精确结果，至少有一个操作数必须是浮点数
    preciseRate := float64(completedTasks) / float64(totalTasks)
    fmt.Printf("精确成功率: %.2f%%\n", preciseRate * 100)

    // 取模运算常用于循环和周期性任务
    retryCount := 7
    maxRetries := 3
    retryAttempt := retryCount % (maxRetries + 1)
    fmt.Printf("重试阶段: %d\n", retryAttempt)
}
```

自增和自减运算符在 Go 中只能作为独立语句，不能嵌入到表达式中：

```go
package main

import "fmt"

func main() {
    attempts := 0

    // 正确：作为独立语句
    attempts++
    fmt.Printf("尝试次数: %d\n", attempts)

    // 错误：不能用作表达式
    // count := attempts++  // 编译错误
    // if attempts++ > 5 {} // 编译错误

    // 正确的做法
    attempts++
    if attempts > 5 {
        fmt.Println("超过最大尝试次数")
    }
}
```

## 关系运算符

关系运算符用于比较两个值，返回布尔结果。在任务调度中，你经常需要比较任务优先级、检查是否超时、判断重试次数等。

```go
package main

import "fmt"

func main() {
    // 任务优先级比较
    taskPriority := 5
    highPriorityThreshold := 8
    lowPriorityThreshold := 3

    isHighPriority := taskPriority >= highPriorityThreshold
    isLowPriority := taskPriority <= lowPriorityThreshold

    fmt.Printf("任务优先级: %d\n", taskPriority)
    fmt.Printf("是否高优先级: %t\n", isHighPriority)
    fmt.Printf("是否低优先级: %t\n", isLowPriority)

    // 字符串按字典序比较
    currentStatus := "running"
    targetStatus := "completed"

    fmt.Printf("状态 '%s' == '%s': %t\n", currentStatus, targetStatus,
        currentStatus == targetStatus)
    fmt.Printf("状态 '%s' < '%s': %t\n", currentStatus, targetStatus,
        currentStatus < targetStatus)
}
```

## 逻辑运算符

逻辑运算符用于组合多个布尔表达式。Go 的逻辑运算符支持短路求值，这意味着如果第一个操作数就能决定结果，第二个操作数不会被计算。这在避免运行时错误时很有用。

```go
package main

import "fmt"

func main() {
    // 短路求值示例
    taskCount := 0
    maxTasks := 100
    hasPermission := false

    // 由于 hasPermission 是 false，第二个条件不会被计算
    // 所以不会出现除零错误
    canExecuteTask := hasPermission && (taskCount/maxTasks > 0)
    fmt.Printf("可以执行任务: %t\n", canExecuteTask)

    // 实际应用：检查任务是否可以执行
    shouldRetry := true
    attempts := 3
    maxAttempts := 5
    isFailed := false

    // 只有当任务失败且未超过最大重试次数时才重试
    if isFailed && attempts < maxAttempts {
        fmt.Println("准备重试任务")
    } else {
        fmt.Println("任务不需要重试")
    }

    // 使用逻辑或提供默认值
    var timeout int  // 零值为 0
    defaultTimeout := 30
    effectiveTimeout := timeout

    if timeout == 0 {
        effectiveTimeout = defaultTimeout
    }

    fmt.Printf("有效超时时间: %d秒\n", effectiveTimeout)
}
```

## 位运算符

位运算符直接对整数的二进制位进行操作。在系统编程中，位运算常用于标志位操作、权限控制等。在任务队列中，你可以用位运算来组合多个标志。

```go
package main

import "fmt"

func main() {
    // 定义任务标志位
    const (
        FlagHighPriority = 1 << iota  // 1 (二进制: 0001)
        FlagRetryable                  // 2 (二进制: 0010)
        FlagLogEnabled                 // 4 (二进制: 0100)
        FlagNotifyOnComplete           // 8 (二进制: 1000)
    )

    // 组合多个标志
    taskFlags := FlagHighPriority | FlagRetryable | FlagLogEnabled
    fmt.Printf("任务标志: %d (二进制: %b)\n", taskFlags, taskFlags)

    // 检查是否设置了特定标志
    isHighPriority := (taskFlags & FlagHighPriority) != 0
    isRetryable := (taskFlags & FlagRetryable) != 0
    shouldNotify := (taskFlags & FlagNotifyOnComplete) != 0

    fmt.Printf("是否高优先级: %t\n", isHighPriority)
    fmt.Printf("是否可重试: %t\n", isRetryable)
    fmt.Printf("完成时是否通知: %t\n", shouldNotify)

    // 清除特定标志
    newFlags := taskFlags &^ FlagRetryable
    fmt.Printf("移除重试标志后的值: %d\n", newFlags)

    // 切换标志（如果存在则移除，不存在则添加）
    toggledFlags := taskFlags ^ FlagLogEnabled
    fmt.Printf("切换日志标志后的值: %d\n", toggledFlags)
}
```

## 赋值运算符

Go 支持复合赋值运算符，让你能更简洁地更新变量值。多重赋值是 Go 的一个特色，可以在一行代码中同时给多个变量赋值，这在交换变量值时特别有用。

```go
package main

import "fmt"

func main() {
    // 复合赋值运算符
    retryCount := 0
    maxRetries := 3

    // 等价于 retryCount = retryCount + 1
    retryCount += 1
    fmt.Printf("重试次数: %d\n", retryCount)

    // 等价于 retryCount = retryCount * 2
    retryCount *= 2
    fmt.Printf("加倍后的重试次数: %d\n", retryCount)

    // 多重赋值：交换变量值（不需要临时变量）
    highPriority := 8
    lowPriority := 3
    fmt.Printf("交换前: 高优先级=%d, 低优先级=%d\n", highPriority, lowPriority)

    highPriority, lowPriority = lowPriority, highPriority
    fmt.Printf("交换后: 高优先级=%d, 低优先级=%d\n", highPriority, lowPriority)

    // 函数返回多重值时，多重赋值特别有用
    min, max := findMinMax(5, 10)
    fmt.Printf("最小值: %d, 最大值: %d\n", min, max)
}

func findMinMax(a, b int) (int, int) {
    if a < b {
        return a, b
    }
    return b, a
}
```

## 运算符优先级

运算符优先级决定了复杂表达式中运算的顺序。Go 的优先级规则和大多数语言类似，但最好用括号来明确意图，这样代码更易读。

从高到低的优先级：

```go
package main

import "fmt"

func main() {
    // 优先级示例
    x := 10
    y := 5
    z := 2

    // 先乘除后加减
    result := x + y*z  // 等价于 x + (y*z)
    fmt.Printf("%d + %d * %d = %d\n", x, y, z, result)

    // 比较运算符优先级高于逻辑运算符
    a := 5
    b := 10
    c := 15

    // 先比较，再进行逻辑运算
    condition := a < b && b < c  // true && true = true
    fmt.Printf("(%d < %d) && (%d < %d) = %t\n", a, b, b, c, condition)

    // 用括号明确意图（推荐）
    clearCondition := (a < b) && (b < c)
    fmt.Printf("用括号更清晰: %t\n", clearCondition)
}
```

## 实战例子：任务优先级计算器

把运算符知识整合起来，实现一个简单的任务优先级计算器。这个计算器会根据多个因素计算任务的最终优先级分数。

```go
package main

import "fmt"

// 任务标志位
const (
    FlagUrgent   = 1 << iota  // 1 (二进制: 0001)
    FlagRecurring              // 2 (二进制: 0010)
    FlagRequiresApproval       // 4 (二进制: 0100)
)

func main() {
    // 任务基本信息
    taskName := "数据同步任务"
    basePriority := 5       // 基础优先级 1-10
    timeFactor := 1.5       // 时间因子
    userImportance := 2     // 用户重要性权重

    // 任务标志
    taskFlags := FlagUrgent | FlagRecurring

    // 计算最终优先级分数
    finalScore := calculatePriorityScore(basePriority, timeFactor, userImportance, taskFlags)

    fmt.Printf("=== 任务优先级分析 ===\n")
    fmt.Printf("任务名称: %s\n", taskName)
    fmt.Printf("基础优先级: %d\n", basePriority)
    fmt.Printf("时间因子: %.1f\n", timeFactor)
    fmt.Printf("用户重要性: %d\n", userImportance)
    fmt.Printf("任务标志: %d\n", taskFlags)
    fmt.Printf("最终优先级分数: %.2f\n", finalScore)

    // 判断任务等级
    var priorityLevel string
    switch {
    case finalScore >= 20:
        priorityLevel = "紧急"
    case finalScore >= 15:
        priorityLevel = "高"
    case finalScore >= 10:
        priorityLevel = "中"
    default:
        priorityLevel = "低"
    }

    fmt.Printf("优先级等级: %s\n", priorityLevel)
}

func calculatePriorityScore(base int, timeFactor float64, userWeight int, flags int) float64 {
    // 基础分数
    score := float64(base) * timeFactor

    // 加上用户权重
    score += float64(userWeight)

    // 根据标志位调整分数
    if (flags & FlagUrgent) != 0 {
        score *= 1.5  // 紧急任务增加 50% 分数
    }

    if (flags & FlagRecurring) != 0 {
        score *= 1.2  // 周期性任务增加 20% 分数
    }

    if (flags & FlagRequiresApproval) != 0 {
        score *= 0.8  // 需要审批的任务降低 20% 分数（更复杂）
    }

    return score
}
```

## 权衡与考虑

Go 的运算符设计体现了简洁性和安全性。限制 `++` 和 `--` 为独立语句避免了 C 语言中常见的复杂表达式 bug。短路求值提高了性能并防止运行时错误。

位运算符提供了高效的标志位操作方式，但可读性不如布尔标志。在性能关键的代码中使用位运算是合理的，但在一般业务逻辑中，清晰的布尔变量更易于维护。

运算符优先级规则需要掌握，但不要依赖隐式优先级。用括号明确运算顺序能提高代码可读性，也减少了出错的可能性。多重赋值是 Go 的优雅特性，但过度使用可能让代码意图变得模糊。
