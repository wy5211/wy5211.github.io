## 问题场景

到目前为止，我们学习了 Rust 的基本类型和所有权系统。但在实际开发中，我们需要表示更复杂的概念，比如"用户"、"任务"、"订单"等。这些概念不能简单地用整数或字符串表示，它们包含多个相关联的属性。

例如，如果我们想表示一个任务，它可能包含：

- 任务 ID
- 任务名称
- 任务描述
- 优先级
- 创建时间
- 执行状态

如果用独立的变量来表示这些信息，代码会变得非常混乱：

```rust
fn main() {
    let task_id = 1;
    let task_name = String::from("数据处理");
    let task_desc = String::from("处理用户数据");
    let priority = 1;
    let created_at = "2024-01-01";

    // 需要传递很多参数
    process_task(task_id, task_name, task_desc, priority, created_at);
}

fn process_task(id: u32, name: String, desc: String, priority: u32, created_at: &str) {
    // ...
}
```

这种方式的问题：

1. 变量之间没有关联，容易传错顺序
2. 函数参数过多，难以维护
3. 代码可读性差，不知道这些变量属于同一个任务

Rust 提供了**结构体（Struct）**来解决这个问题。结构体允许我们将相关联的数据组合成一个有意义的整体。

## 定义结构体

### 基本语法

```rust
// 定义结构体
struct Task {
    id: u32,              // 任务 ID
    name: String,         // 任务名称
    description: String,  // 任务描述
    priority: u8,         // 优先级（0-10）
    status: TaskStatus,   // 任务状态（枚举类型）
}

// 定义任务状态枚举
enum TaskStatus {
    Pending,
    InProgress,
    Completed,
    Failed,
}

fn main() {
    // 创建结构体实例
    let task = Task {
        id: 1,
        name: String::from("数据处理"),
        description: String::from("处理用户数据"),
        priority: 5,
        status: TaskStatus::Pending,
    };

    // 访问结构体字段
    println!("任务 ID: {}", task.id);
    println!("任务名称: {}", task.name);
    println!("优先级: {}", task.priority);
}
```

### 字段初始化简写

当变量名和字段名相同时，可以简写：

```rust
fn main() {
    let id = 1;
    let name = String::from("数据处理");
    let description = String::from("处理用户数据");
    let priority = 5;

    // 完整写法
    let task1 = Task {
        id: id,
        name: name.clone(),
        description: description.clone(),
        priority: priority,
        status: TaskStatus::Pending,
    };

    // 简写：变量名和字段名相同时
    let task2 = Task {
        id,
        name,
        description,
        priority,
        status: TaskStatus::Pending,
    };
}
```

### 结构体更新语法

从已有结构体创建新结构体，使用 `..` 语法：

```rust
fn main() {
    let task1 = Task {
        id: 1,
        name: String::from("数据处理"),
        description: String::from("处理用户数据"),
        priority: 5,
        status: TaskStatus::Pending,
    };

    // 使用 ..task1 复制其余字段
    let task2 = Task {
        id: 2,
        name: String::from("数据备份"),
        description: String::from("备份用户数据"),
        ..task1  // priority 和 status 从 task1 复制
    };

    println!("task2 优先级: {}", task2.priority);  // 5（来自 task1）

    // 注意：task1 的 name 和 description 已被移动，不能再使用
    // 但 priority 和 status 是 Copy 类型，仍然可用
    println!("task1 优先级: {}", task1.priority);  // 可以
    // println!("{}", task1.name);  // 错误：已被移动
}
```

## 结构体的三种形式

### 1. 经典结构体（最常用）

```rust
struct Task {
    id: u32,
    name: String,
}

fn main() {
    let task = Task {
        id: 1,
        name: String::from("数据处理"),
    };
    println!("任务: {} - {}", task.id, task.name);
}
```

### 2. 元组结构体

有名字的元组，适合"整个结构体有意义但各字段没有独立名称"的场景：

```rust
// 表示不同类型的数值，避免混淆
struct Meters(f64);
struct Kilometers(f64);
struct Celsius(f64);
struct Fahrenheit(f64);

fn main() {
    let distance = Meters(1000.0);
    let temp = Celsius(25.0);

    println!("距离: {} 米", distance.0);
    println!("温度: {} 摄氏度", temp.0);
}
```

### 3. 单元结构体

没有任何字段的结构体，用于实现 trait：

```rust
// 用于标记类型的单元结构体
struct Processable;
struct Serializable;

fn main() {
    let marker = Processable;
    // 可以用来标记类型或实现某些行为
}
```

## 方法

方法是定义在结构体上的函数，第一个参数总是 `self`（或 `&self`、`&mut self`），表示方法被调用的实例。

### 定义方法

使用 `impl` 块定义方法：

```rust
impl Task {
    // &self 是 self: &Self 的简写，表示借用 self
    fn get_info(&self) -> String {
        format!("任务[{}]: {}", self.id, self.name)
    }

    // &mut self 可以修改结构体
    fn start(&mut self) {
        self.status = TaskStatus::InProgress;
        println!("任务 {} 开始执行", self.name);
    }

    // self 获取所有权（较少使用）
    fn consume(self) {
        println!("任务 {} 被消耗", self.name);
    }  // self 在这里被释放
}

fn main() {
    let mut task = Task {
        id: 1,
        name: String::from("数据处理"),
        description: String::from("处理用户数据"),
        priority: 5,
        status: TaskStatus::Pending,
    };

    // 调用方法
    println!("{}", task.get_info());

    // 修改结构体
    task.start();
    println!("任务状态: {:?}", task.status);

    // 消耗结构体
    task.consume();
    // task.get_info();  // 错误：task 已被移动
}
```

### 方法的自动引用和解引用

调用方法时，Rust 会自动添加 `&`、`&mut` 或移动：

```rust
impl Task {
    fn is_high_priority(&self) -> bool {
        self.priority >= 8
    }
}

fn main() {
    let task = Task {
        id: 1,
        name: String::from("数据处理"),
        description: String::from("处理用户数据"),
        priority: 9,
        status: TaskStatus::Pending,
    };

    // 这两种写法等价
    let result1 = task.is_high_priority();       // Rust 自动转换为 (&task).is_high_priority()
    let result2 = (&task).is_high_priority();    // 显式写法

    println!("高优先级: {}", result1);
}
```

## 关联函数

`impl` 块中不接收 `self` 参数的函数叫做**关联函数**，类似于其他语言中的"静态方法"：

```rust
impl Task {
    // 关联函数，用作"构造函数"
    fn new(id: u32, name: &str, description: &str, priority: u8) -> Self {
        Self {
            id,
            name: name.to_string(),
            description: description.to_string(),
            priority,
            status: TaskStatus::Pending,
        }
    }

    // 带默认值的构造函数
    fn default_task(id: u32) -> Self {
        Self {
            id,
            name: String::from("默认任务"),
            description: String::from("默认描述"),
            priority: 5,
            status: TaskStatus::Pending,
        }
    }
}

fn main() {
    // 使用 :: 调用关联函数
    let task1 = Task::new(1, "数据处理", "处理用户数据", 5);
    let task2 = Task::default_task(2);

    println!("任务1: {}", task1.name);
    println!("任务2: {}", task2.name);
}
```

## 实战：任务处理系统的结构体设计

让我们设计一个完整的任务处理系统的数据结构：

```rust
// 任务优先级枚举
#[derive(Debug, Clone, Copy)]
enum Priority {
    Low = 1,
    Medium = 5,
    High = 8,
    Critical = 10,
}

// 任务状态枚举
#[derive(Debug)]
enum TaskStatus {
    Pending,
    InProgress,
    Completed,
    Failed,
    Cancelled,
}

// 任务结构体
#[derive(Debug)]
struct Task {
    id: u32,
    name: String,
    description: String,
    priority: Priority,
    status: TaskStatus,
    created_at: String,
    completed_at: Option<String>,
}

impl Task {
    // 创建新任务
    fn new(id: u32, name: &str, description: &str, priority: Priority) -> Self {
        Self {
            id,
            name: name.to_string(),
            description: description.to_string(),
            priority,
            status: TaskStatus::Pending,
            created_at: Self::current_time(),
            completed_at: None,
        }
    }

    // 开始任务
    fn start(&mut self) {
        self.status = TaskStatus::InProgress;
        println!("任务 [{}] 开始执行", self.name);
    }

    // 完成任务
    fn complete(&mut self) {
        self.status = TaskStatus::Completed;
        self.completed_at = Some(Self::current_time());
        println!("任务 [{}] 执行完成", self.name);
    }

    // 失败任务
    fn fail(&mut self) {
        self.status = TaskStatus::Failed;
        println!("任务 [{}] 执行失败", self.name);
    }

    // 获取任务信息
    fn get_info(&self) -> String {
        format!(
            "任务[{}]: {} - 优先级: {:?}, 状态: {:?}",
            self.id, self.name, self.priority, self.status
        )
    }

    // 判断是否为高优先级
    fn is_high_priority(&self) -> bool {
        matches!(self.priority, Priority::High | Priority::Critical)
    }

    // 获取当前时间（简化版）
    fn current_time() -> String {
        "2024-01-01 00:00:00".to_string()
    }
}

// 任务处理器结构体
struct TaskProcessor {
    tasks: Vec<Task>,
    next_id: u32,
}

impl TaskProcessor {
    // 创建新的任务处理器
    fn new() -> Self {
        Self {
            tasks: Vec::new(),
            next_id: 1,
        }
    }

    // 添加任务
    fn add_task(&mut self, name: &str, description: &str, priority: Priority) -> u32 {
        let task = Task::new(self.next_id, name, description, priority);
        let id = task.id;
        self.tasks.push(task);
        self.next_id += 1;
        println!("添加任务: {}", name);
        id
    }

    // 获取任务
    fn get_task(&self, id: u32) -> Option<&Task> {
        self.tasks.iter().find(|t| t.id == id)
    }

    // 获取可变任务
    fn get_task_mut(&mut self, id: u32) -> Option<&mut Task> {
        self.tasks.iter_mut().find(|t| t.id == id)
    }

    // 执行任务
    fn execute_task(&mut self, id: u32) {
        if let Some(task) = self.get_task_mut(id) {
            task.start();
            // 模拟执行任务
            println!("正在执行: {}", task.description);
            task.complete();
        } else {
            println!("任务 {} 不存在", id);
        }
    }

    // 列出所有任务
    fn list_tasks(&self) {
        println!("\n=== 任务列表 ===");
        for task in &self.tasks {
            println!("{}", task.get_info());
        }
        println!("================\n");
    }

    // 获取高优先级任务
    fn get_high_priority_tasks(&self) -> Vec<&Task> {
        self.tasks
            .iter()
            .filter(|t| t.is_high_priority())
            .collect()
    }
}

fn main() {
    // 创建任务处理器
    let mut processor = TaskProcessor::new();

    // 添加任务
    processor.add_task("数据备份", "备份用户数据", Priority::High);
    processor.add_task("日志清理", "清理过期日志", Priority::Low);
    processor.add_task("系统监控", "监控系统状态", Priority::Critical);
    processor.add_task("报表生成", "生成月度报表", Priority::Medium);

    // 列出所有任务
    processor.list_tasks();

    // 执行任务
    processor.execute_task(1);
    processor.execute_task(3);

    // 查看任务状态
    if let Some(task) = processor.get_task(1) {
        println!("任务状态: {:?}", task.status);
    }

    // 获取高优先级任务
    let high_priority = processor.get_high_priority_tasks();
    println!("高优先级任务数量: {}", high_priority.len());
}
```

## Debug 输出

手动为每个结构体实现 `println!` 很麻烦，Rust 提供了 `Debug` trait 的派生宏：

```rust
// 添加 derive 宏自动实现 Debug
#[derive(Debug)]
struct Task {
    id: u32,
    name: String,
}

fn main() {
    let task = Task {
        id: 1,
        name: String::from("数据处理"),
    };

    // 使用 {:?} 格式化输出
    println!("{:?}", task);
    // Task { id: 1, name: "数据处理" }

    // 使用 {:#?} 美化输出
    println!("{:#?}", task);
    // Task {
    //     id: 1,
    //     name: "数据处理",
    // }
}
```

## 常见错误与解决

### 错误1：部分可变性

```rust
struct Task {
    name: String,
    status: TaskStatus,
}

fn main() {
    // 错误：Rust 不支持部分可变
    // let task = Task { name: String::from("数据处理"), status: TaskStatus::Pending };
    // task.status = TaskStatus::InProgress;  // 编译错误

    // 解决：整个结构体设为 mut
    let mut task = Task {
        name: String::from("数据处理"),
        status: TaskStatus::Pending,
    };
    task.status = TaskStatus::InProgress;
}
```

### 错误2：使用未初始化的结构体

```rust
fn main() {
    // 错误：结构体必须初始化所有字段
    // let task = Task { id: 1 };

    // 解决：初始化所有字段
    let task = Task {
        id: 1,
        name: String::from("数据处理"),
        description: String::from("处理用户数据"),
        priority: 5,
        status: TaskStatus::Pending,
    };
}
```

### 错误3：在方法签名中使用错误的 self

```rust
impl Task {
    // 错误：修改数据但用了 &self
    // fn start(&self) { self.status = TaskStatus::InProgress; }

    // 解决：需要修改用 &mut self
    fn start(&mut self) {
        self.status = TaskStatus::InProgress;
    }
}
```

## 结构体的权衡

### 结构体的优势

1. **数据组织**：将相关数据组合在一起，提高代码可读性
2. **类型安全**：通过类型系统确保数据的完整性
3. **方法封装**：将与数据相关的操作组织在一起
4. **所有权清晰**：明确数据的归属和生命周期

### 结构体的设计考虑

1. **字段数量**：字段过多可能表示结构体职责不单一
2. **嵌套深度**：过度嵌套会降低代码可读性
3. **Copy vs Clone**：简单的结构体可以实现 Copy trait
4. **可见性**：合理使用 pub 控制字段和方法的可见性

### 什么时候使用结构体

**适合使用结构体的场景：**

- 需要组合多个相关数据
- 数据之间有逻辑关联
- 需要对数据进行操作
- 需要明确的所有权语义

**不适合使用结构体的场景：**

- 数据之间没有语义关联（使用元组）
- 只需要一个标记（使用单元结构体）
- 需要联合类型（使用枚举）

## 总结

结构体是 Rust 中组织自定义数据的核心工具。通过结构体，我们可以将相关数据组合成一个有意义的整体，并通过方法为数据提供行为。

**核心要点：**

1. 结构体定义使用 `struct` 关键字，字段需要标注类型
2. 创建结构体实例时必须初始化所有字段
3. 使用 `impl` 块为结构体定义方法和关联函数
4. `&self` 用于读取数据，`&mut self` 用于修改数据
5. `#[derive(Debug)]` 可以快速实现调试输出
6. 字段初始化简写和结构体更新语法可以简化代码
