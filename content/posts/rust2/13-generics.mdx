## 问题引入

假设你在任务处理系统中需要实现一个"找最大值"的功能：

- 从任务队列中找到优先级最高的任务
- 从 worker 列表中找到负载最高的 worker
- 从分数列表中找到最高分

不使用泛型，你需要为每种类型写一个几乎相同的函数：

```rust
// 为任务找最大值
fn max_task(tasks: &[Task]) -> Option<&Task> {
    // ...
}

// 为 worker 找最大值
fn max_worker(workers: &[Worker]) -> Option<&Worker> {
    // ...
}

// 为分数找最大值
fn max_score(scores: &[i32]) -> Option<&i32> {
    // ...
}
```

这三个函数的逻辑完全相同，只是类型不同。**泛型（Generics）**允许你编写一套代码适用于多种类型，Rust 会在编译时为每个具体类型生成专用版本，确保运行时没有任何性能开销——这就是"零成本抽象"。

## 泛型函数

### 基本语法

在函数名后用 `<T>` 声明泛型参数，然后在参数和返回值中使用：

```rust
// 泛型函数：找出切片中的最大值
fn largest<T: PartialOrd>(list: &[T]) -> Option<&T> {
    if list.is_empty() {
        return None;
    }

    let mut largest = &list[0];
    for item in &list[1..] {
        if item > largest {
            largest = item;
        }
    }
    Some(largest)
}

// 使用同一个函数处理不同类型
fn main() {
    let numbers = vec![34, 50, 25, 100, 65];
    println!("最大数字: {:?}", largest(&numbers));  // Some(&100)

    let chars = vec!['y', 'm', 'a', 'q'];
    println!("最大字符: {:?}", largest(&chars));  // Some(&'y')

    let words = vec!["apple", "banana", "cherry"];
    println!("最大单词: {:?}", largest(&words));  // Some(&"cherry")
}
```

**`T: PartialOrd` 是什么**：这是 trait bound（trait 约束），表示 `T` 必须实现了 `PartialOrd` trait（可以比较大小）。没有这个约束，编译器不知道 `item > largest` 是否合法。

### 多个泛型参数

函数可以有多个泛型参数，用逗号分隔：

```rust
// 创建键值对
fn pair<T, U>(first: T, second: U) -> (T, U) {
    (first, second)
}

fn main() {
    let p1 = pair(1, "hello");          // (i32, &str)
    let p2 = pair("rust", 3.14);        // (&str, f64)
    let p3 = pair(vec![1, 2], true);    // (Vec<i32>, bool)

    println!("{:?}", p1);
    println!("{:?}", p2);
    println!("{:?}", p3);
}
```

### Trait 约束的详细语法

泛型参数不能随便使用任何操作，需要通过 trait bound 约束其行为：

```rust
use std::fmt::Display;

// 单个约束：T 必须能比较大小
fn largest<T: PartialOrd>(list: &[T]) -> Option<&T> {
    // ...
}

// 多个约束：T 必须能比较大小且能显示
fn print_largest<T: PartialOrd + Display>(list: &[T]) {
    match largest(list) {
        Some(value) => println!("最大值: {}", value),
        None => println!("列表为空"),
    }
}

// where 子句：更清晰的语法（推荐）
fn complex_function<T, U>(t: &T, u: &U) -> i32
where
    T: Display + Clone,
    U: Clone + PartialEq,
{
    // 函数体
    0
}
```

**何时使用 where 子句**：当约束较多或函数签名较长时，`where` 子句比 `<T: Trait1 + Trait2>` 更清晰。

## 泛型结构体

### 基本用法

```rust
// 泛型结构体：点
struct Point<T> {
    x: T,
    y: T,
}

fn main() {
    let integer = Point { x: 5, y: 10 };      // Point<i32>
    let float = Point { x: 1.0, y: 4.0 };     // Point<f64>

    println!("整数点: ({}, {})", integer.x, integer.y);
    println!("浮点点: ({}, {})", float.x, float.y);
}
```

**注意**：`Point<T>` 的 `x` 和 `y` 必须是相同类型。如果需要不同类型，用两个泛型参数：

```rust
struct Point<T, U> {
    x: T,
    y: U,
}

fn main() {
    let p = Point { x: 5, y: 1.5 };  // Point<i32, f64>
}
```

### 为泛型结构体实现方法

```rust
struct Point<T> {
    x: T,
    y: T,
}

// 为所有 Point<T> 实现方法
impl<T> Point<T> {
    fn new(x: T, y: T) -> Self {
        Self { x, y }
    }

    fn x(&self) -> &T {
        &self.x
    }
}

// 只为 Point<f64> 实现特定方法
impl Point<f64> {
    fn distance_from_origin(&self) -> f64 {
        (self.x.powi(2) + self.y.powi(2)).sqrt()
    }
}

fn main() {
    let p1 = Point::new(5, 10);
    println!("x 坐标: {}", p1.x());

    let p2 = Point::new(3.0, 4.0);
    println!("距离原点: {}", p2.distance_from_origin());  // 5.0

    // let p3 = Point::new(3, 4);
    // p3.distance_from_origin();  // 编译错误：只有 f64 类型有此方法
}
```

**关键点**：

- `impl<T>` 必须在 `impl` 后声明泛型参数
- `Point<T>` 和 `Point<f64>` 是不同的类型
- 可以为特定类型实现额外方法

### 泛型枚举

你已经用过的 `Option<T>` 和 `Result<T, E>` 就是泛型枚举：

```rust
enum Option<T> {
    Some(T),
    None,
}

enum Result<T, E> {
    Ok(T),
    Err(E),
}
```

自定义泛型枚举：

```rust
enum Container<T> {
    Empty,
    Single(T),
    Multiple(Vec<T>),
}

impl<T: std::fmt::Display> Container<T> {
    fn describe(&self) {
        match self {
            Container::Empty => println!("空容器"),
            Container::Single(v) => println!("单个值: {}", v),
            Container::Multiple(v) => println!("多个值: {} 个", v.len()),
        }
    }
}

fn main() {
    let c1: Container<i32> = Container::Empty;
    let c2: Container<&str> = Container::Single("hello");
    let c3: Container<i32> = Container::Multiple(vec![1, 2, 3]);

    c1.describe();  // 空容器
    c2.describe();  // 单个值: hello
    c3.describe();  // 多个值: 3 个
}
```

## 实战案例：泛型任务队列

让我们用泛型实现一个可以存储任意类型任务的队列：

```rust
use std::collections::HashMap;

// 任务优先级（必须能排序）
#[derive(Debug, Clone, Copy, PartialEq, Eq, PartialOrd, Ord)]
enum Priority {
    Low,
    Medium,
    High,
}

// 泛型任务结构
struct Task<T> {
    id: u64,
    priority: Priority,
    data: T,  // 任务数据可以是任意类型
}

impl<T: std::fmt::Display> Task<T> {
    fn describe(&self) -> String {
        format!("任务 {} (优先级 {:?}): {}", self.id, self.priority, self.data)
    }
}

// 泛型任务队列
struct TaskQueue<T> {
    tasks: Vec<Task<T>>,
    next_id: u64,
}

impl<T> TaskQueue<T> {
    fn new() -> Self {
        Self {
            tasks: Vec::new(),
            next_id: 1,
        }
    }

    // 添加任务，返回任务 ID
    fn push(&mut self, priority: Priority, data: T) -> u64 {
        let id = self.next_id;
        self.next_id += 1;

        let task = Task { id, priority, data };
        self.tasks.push(task);

        // 按优先级排序（高优先级在前）
        self.tasks.sort_by(|a, b| b.priority.cmp(&a.priority));

        id
    }

    // 取出下一个任务（最高优先级）
    fn pop(&mut self) -> Option<Task<T>> {
        if self.tasks.is_empty() {
            None
        } else {
            Some(self.tasks.remove(0))
        }
    }

    // 查看任务数量
    fn len(&self) -> usize {
        self.tasks.len()
    }

    // 判断是否为空
    fn is_empty(&self) -> bool {
        self.tasks.is_empty()
    }
}

// 泛型函数：打印队列状态
fn print_queue_status<T: std::fmt::Display>(queue: &TaskQueue<T>) {
    println!("队列中的任务数: {}", queue.len());
}

fn main() {
    // 存储字符串任务
    let mut string_queue = TaskQueue::new();
    string_queue.push(Priority::High, "处理紧急数据");
    string_queue.push(Priority::Low, "清理日志");
    string_queue.push(Priority::Medium, "生成报表");

    while let Some(task) = string_queue.pop() {
        println!("{}", task.describe());
    }

    println!();

    // 存储数值任务
    let mut number_queue = TaskQueue::new();
    number_queue.push(Priority::Low, 42);
    number_queue.push(Priority::High, 99);
    number_queue.push(Priority::Medium, 76);

    while let Some(task) = number_queue.pop() {
        println!("任务 ID: {}, 数据: {}", task.id, task.data);
    }
}
```

## 泛型的性能：单态化

你可能担心泛型会有性能开销（像 C++ 的模板或 Java 的泛型）。但 Rust 的泛型是**零成本抽象**，这得益于**单态化（Monomorphization）**。

### 什么是单态化

单态化是编译器在编译时为每个具体类型生成一份专用代码的过程：

```rust
// 你写的泛型代码
fn largest<T: PartialOrd>(list: &[T]) -> Option<&T> {
    // ...
}

// 你使用泛型函数
let numbers = vec![1, 2, 3];
largest(&numbers);  // T = i32

let floats = vec![1.0, 2.0];
largest(&floats);   // T = f64

// 编译器生成的代码（类似）：
fn largest_for_i32(list: &[i32]) -> Option<&i32> {
    // 与 largest<T> 相同的逻辑，但 T 替换为 i32
}

fn largest_for_f64(list: &[f64]) -> Option<&f64> {
    // 与 largest<T> 相同的逻辑，但 T 替换为 f64
}
```

**关键点**：

- 泛型函数在编译时被"展开"为多个具体版本
- 每个版本都是针对特定类型优化的
- 运行时没有任何动态分发的开销

### 单态化的代价

单态化不是免费的：

- **编译时间更长**：编译器需要生成和编译更多代码
- **二进制文件更大**：每个具体类型都有一份代码副本
- **代码膨胀**：如果泛型用于很多类型，会显著增加代码量

### 何时考虑不用泛型

如果单态化的代价太高，可以考虑以下替代方案：

```rust
// 方式 1：使用 trait 对象（动态分发）
fn process_items(items: &[&dyn Display]) {
    for item in items {
        println!("{}", item);
    }
}

// 方式 2：使用枚举
enum Item {
    Number(i32),
    Text(String),
}

fn process_items(items: &[Item]) {
    // ...
}
```

## impl Trait 语法

Rust 还提供了 `impl Trait` 语法，在某些场景下更简洁：

### 在参数中使用

```rust
// 泛型写法
fn print_all<T: Display>(items: &[T]) {
    for item in items {
        println!("{}", item);
    }
}

// impl Trait 写法（等价）
fn print_all(items: &[impl Display]) {
    for item in items {
        println!("{}", item);
    }
}
```

**限制**：`impl Trait` 在参数中只能用于单一类型，不能像 `<T: Trait>` 那样表示"调用者可以选择任何类型"。

### 在返回值中使用

```rust
// 返回实现了 Display 的类型（但具体类型固定）
fn get_displayable() -> impl Display {
    42  // 返回 i32
}

// 等价于：
fn get_displayable() -> i32 {
    42
}
```

**注意**：`impl Trait` 返回类型只能返回一种具体类型。如果需要返回多种类型，必须用 trait 对象：

```rust
// 错误：不能返回不同类型
fn get_item(condition: bool) -> impl Display {
    if condition {
        42  // i32
    } else {
        "hello"  // &str
    }
}

// 正确：使用 trait 对象
fn get_item(condition: bool) -> Box<dyn Display> {
    if condition {
        Box::new(42)
    } else {
        Box::new("hello")
    }
}
```

## 泛型的常见陷阱

### 1. 缺少 trait 约束

```rust
// 错误：不能比较没有实现 PartialOrd 的类型
fn largest<T>(list: &[T]) -> Option<&T> {
    let mut largest = &list[0];
    for item in &list[1..] {
        if item > largest {  // 编译错误
            largest = item;
        }
    }
    Some(largest)
}

// 正确：添加 trait 约束
fn largest<T: PartialOrd>(list: &[T]) -> Option<&T> {
    // ...
}
```

### 2. 浮点数不要实现 Eq

```rust
// 错误：f64 不实现 Eq（因为 NaN != NaN）
// fn max<T: Eq>(a: T, b: T) -> T { }

// 正确：使用 PartialEq
fn max<T: PartialEq>(a: T, b: T) -> T {
    // ...
}
```

### 3. 避免过度泛化

```rust
// 不推荐：不必要的泛型
struct Task {
    id: u64,
    // 如果 ID 总是 u64，不需要泛型
}

// 推荐：直接使用具体类型
struct Task {
    id: u64,
}
```

**原则**：只在真正需要代码复用时使用泛型。如果只有一个类型会用到，直接用具体类型更简单。

## Trade-off 分析

### 泛型 vs 宏

**泛型**：

- 类型安全，编译时检查
- 代码可读性好
- 受单态化限制

**宏**：

- 更灵活，可以操作语法树
- 可以减少重复代码
- 类型安全性较差，调试困难

### 泛型 vs trait 对象

**泛型（静态分发）**：

- 性能好（无虚函数开销）
- 编译时单态化
- 代码膨胀

**trait 对象（动态分发）**：

- 代码紧凑
- 运行时多态
- 有虚函数调用开销

**选择建议**：

- 性能关键代码：用泛型
- 需要存储多种类型：用 trait 对象
- 库代码：优先泛型（让用户选择）

## 总结

泛型是 Rust 类型系统的重要组成部分：

**核心概念**：

- 泛型允许编写适用于多种类型的代码
- 通过 trait bound 约束泛型类型的行为
- 单态化确保零运行时开销

**语法要点**：

- 函数泛型：`fn foo<T: Trait>(x: T)`
- 结构体泛型：`struct Point<T>`
- 枚举泛型：`enum Option<T>`
- `impl Trait`：更简洁的语法

**权衡考虑**：

- 泛型的代价是编译时间和二进制大小
- 考虑用 trait 对象避免代码膨胀
- 避免过度泛化

在任务处理系统中，我们可以用泛型编写通用的队列、缓存、存储结构，让它们能够处理不同类型的任务和数据。泛型为代码复用提供了类型安全、零成本的方式。
