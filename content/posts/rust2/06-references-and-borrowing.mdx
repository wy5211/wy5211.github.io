## 问题场景

在上一篇文章中，我们学习了所有权系统——Rust 通过所有权规则在编译期保证内存安全。但你可能很快遇到了一个实际问题：每次调用函数都要转移所有权，用完还得把值传回来，代码变得非常啰嗦。

```rust
fn main() {
    let s = String::from("hello");
    let len = calculate_length(s);
    // println!("{}", s);  // 编译错误：s 已失效
    println!("长度: {}", len);
}

fn calculate_length(s: String) -> usize {
    s.len()
}  // s 在这里被释放
```

这个问题的本质是：我们只是想借用数据查看一下，并不想获得它的所有权。如果在 C/C++ 中，我们会传指针；在 Java 中，我们会传引用。但 Rust 的所有权系统不允许我们简单地"借用"数据而不承担后果。

这个问题引出了 Rust 的另一个核心概念：**引用（Reference）**和**借用（Borrowing）**。

## 引用的基本概念

引用是一个指向某个值但不拥有该值的指针。当你创建一个引用时，你只是借用了数据的访问权，数据的所有权仍然属于原变量。

### 创建引用

使用 `&` 运算符创建引用：

```rust
fn main() {
    let s1 = String::from("hello");

    // &s1 创建一个指向 s1 的引用，不转移所有权
    let len = calculate_length(&s1);

    // s1 仍然有效，可以继续使用
    println!("'{}' 的长度是 {}", s1, len);
}

// 参数类型是 &String，表示"借用一个 String"
fn calculate_length(s: &String) -> usize {
    s.len()
}  // s 离开作用域，但因为 s 不拥有所有权，所以不会释放数据
```

### 引用与所有权的对比

```rust
fn main() {
    // 所有权转移：原变量失效
    let s1 = String::from("hello");
    let s2 = s1;
    // println!("{}", s1);  // 编译错误：s1 已失效

    // 引用：原变量仍然有效
    let s3 = String::from("world");
    let r = &s3;
    println!("{}", s3);  // s3 仍然有效
    println!("{}", r);   // r 是 s3 的引用
}
```

## 不可变引用

默认情况下，引用是不可变的——你只能读取数据，不能修改。

### 不可变引用的特点

```rust
fn main() {
    let s = String::from("hello");

    // 创建不可变引用
    let r1 = &s;
    let r2 = &s;

    // 可以有多个不可变引用
    println!("r1: {}, r2: {}", r1, r2);

    // s 仍然有效
    println!("原字符串: {}", s);
}
```

### 不可变引用的典型用法

```rust
// 计算字符串长度（只读操作）
fn string_length(s: &String) -> usize {
    s.len()
}

// 检查字符串是否为空（只读操作）
fn is_empty(s: &String) -> bool {
    s.is_empty()
}

// 查找子串（只读操作）
fn contains_hello(s: &String) -> bool {
    s.contains("hello")
}

fn main() {
    let s = String::from("hello world");

    // 可以多次调用不同的函数，因为都只是读取
    println!("长度: {}", string_length(&s));
    println!("是否为空: {}", is_empty(&s));
    println!("包含 hello: {}", contains_hello(&s));

    // s 一直有效，可以继续使用
    println!("原文: {}", s);
}
```

## 可变引用

如果你需要修改借用的数据，需要使用 `&mut` 创建可变引用。

### 创建可变引用

```rust
fn main() {
    // 变量本身必须是 mut 的才能创建可变引用
    let mut s = String::from("hello");

    // &mut s 创建可变引用
    change(&mut s);

    println!("{}", s);  // hello, world
}

fn change(s: &mut String) {
    s.push_str(", world");  // 通过可变引用修改数据
}
```

### 可变引用的限制

```rust
fn main() {
    let mut s = String::from("hello");

    // 不能同时存在多个可变引用
    let r1 = &mut s;
    let r2 = &mut s;  // 编译错误：不能同时有两个可变引用
    // println!("{} {}", r1, r2);
}
```

这个限制的原因是：如果有两个可变引用同时修改数据，会导致数据竞争（Data Race），产生不可预测的结果。

## 借用规则

Rust 的借用规则是保证内存安全的核心机制，请务必牢记：

### 规则一：引用的互斥性

在同一时刻，你可以拥有：

- 任意数量的不可变引用（`&T`），或者
- 恰好一个可变引用（`&mut T`）

但不能同时拥有不可变引用和可变引用。

```rust
fn main() {
    let mut s = String::from("hello");

    // 错误：不能同时拥有不可变引用和可变引用
    let r1 = &s;
    let r2 = &mut s;
    // println!("{} {}", r1, r2);  // 编译错误

    // 正确：不可变引用不再使用后，可以创建可变引用
    let r3 = &s;
    println!("{}", r3);  // r3 最后一次使用
    let r4 = &mut s;     // 现在可以创建可变引用
    r4.push_str(" world");
}
```

### 规则二：引用必须总是有效的

引用不能指向已释放的内存，这被称为"悬垂引用"（Dangling Reference）。Rust 编译器会在编译期阻止这种情况。

```rust
// 编译错误：返回局部变量的引用
fn dangle() -> &String {
    let s = String::from("hello");
    &s  // 错误：s 将在函数结束时被释放
}

// 正确：返回所有权
fn no_dangle() -> String {
    let s = String::from("hello");
    s  // s 的所有权被转移出去，不会被释放
}
```

## 借用规则的实际应用

### 示例：统计单词出现次数

```rust
use std::collections::HashMap;

fn main() {
    let text = String::from("hello world hello rust world rust rust");

    // 使用不可变借用，不转移所有权
    let word_count = count_words(&text);

    println!("单词统计:");
    for (word, count) in &word_count {
        println!("  '{}': {} 次", word, count);
    }

    // text 仍然有效，可以继续使用
    println!("原文: {}", text);
}

fn count_words(text: &String) -> Vec<(&str, usize)> {
    let mut map = HashMap::new();

    // 遍历单词，统计出现次数
    for word in text.split_whitespace() {
        *map.entry(word).or_insert(0) += 1;
    }

    // 转换为向量并排序
    let mut result: Vec<_> = map.into_iter().collect();
    result.sort_by(|a, b| b.1.cmp(&a.1));
    result
}
```

### 示例：字符串处理管道

```rust
fn main() {
    let mut content = String::from("  Hello, Rust World!  ");

    // 逐步处理字符串
    trim_and_process(&mut content);
    println!("最终结果: '{}'", content);
}

fn trim_and_process(s: &mut String) {
    // 先读取数据
    println!("原始文本: '{}'", s);

    // 再修改数据
    let trimmed = s.trim().to_string();
    *s = trimmed;

    // 再读取修改后的结果
    println!("去除空格后: '{}'", s);

    // 继续修改
    let lower = s.to_lowercase();
    *s = lower;

    println!("转小写后: '{}'", s);
}
```

在这个例子中，我们在同一个函数内先读取、再修改、再读取。因为这些操作是顺序执行的，不会同时发生，所以符合借用规则。

## 引用与函数参数

### 函数参数应该用什么类型？

```rust
// 方式1：转移所有权（不推荐）
fn consume(s: String) {
    println!("{}", s);
}  // s 被释放

// 方式2：不可变借用（推荐）
fn borrow(s: &String) {
    println!("{}", s);
}  // s 不拥有数据，不释放

// 方式3：可变借用（需要修改时）
fn modify(s: &mut String) {
    s.push_str("!");
}  // s 不拥有数据，但可以修改
```

选择原则：

- 只需要读取数据：用 `&T`
- 需要修改数据：用 `&mut T`
- 确实需要转移所有权：用 `T`

### &String vs &str

```rust
// 更灵活的写法：使用 &str
fn print_length(s: &str) {
    println!("'{}' 的长度是 {}", s, s.len());
}

fn main() {
    let s = String::from("hello");
    let literal = "world";

    // &str 可以同时接受 String 和字符串字面量
    print_length(&s);        // 传入 String 的引用
    print_length(literal);   // 传入字符串字面量
}
```

`&str` 比 `&String` 更通用，因为它可以接受字符串字面量和 `String` 的引用。

## 常见错误与解决

### 错误1：同时存在可变和不可变引用

```rust
fn main() {
    let mut s = String::from("hello");

    let r1 = &s;
    let r2 = &mut s;  // 编译错误
    // println!("{} {}", r1, r2);

    // 解决方案：确保不可变引用不再使用
    let r3 = &s;
    println!("{}", r3);  // r3 最后一次使用
    let r4 = &mut s;     // 现在可以创建可变引用
    r4.push_str(" world");
}
```

### 错误2：修改不可变引用

```rust
fn main() {
    let s = String::from("hello");
    let r = &s;
    // r.push_str(" world");  // 编译错误：不能修改不可变引用

    // 解决方案：使用可变引用
    let mut s = String::from("hello");
    let r = &mut s;
    r.push_str(" world");
    println!("{}", r);
}
```

### 错误3：创建悬垂引用

```rust
// 编译错误：返回局部变量的引用
fn dangle() -> &String {
    let s = String::from("hello");
    &s  // s 将在函数结束时被释放
}

// 解决方案：返回所有权而不是引用
fn no_dangle() -> String {
    let s = String::from("hello");
    s  // 所有权被转移，不会被释放
}
```

## 引用的优势与权衡

### 优势

1. **避免不必要的拷贝**：引用只是指针，不需要复制整个数据结构
2. **保持数据有效性**：借用数据后原变量仍然可用
3. **编译期安全检查**：编译器保证引用不会出现悬垂指针或数据竞争
4. **明确的意图表达**：引用类型清楚地表明函数是读取还是修改数据

### 权衡

1. **生命周期复杂度**：当返回引用时，需要考虑生命周期标注（后续文章会详细讲解）
2. **借用规则限制**：在某些场景下，借用规则可能会让代码变得复杂
3. **学习曲线**：初学者需要时间理解借用规则和所有权系统

### 何时使用引用

**适合使用引用的场景：**

- 函数只需要读取数据，不需要拥有数据
- 需要在多处访问同一份数据
- 数据结构较大，避免拷贝开销
- 需要修改数据但不想转移所有权

**不适合使用引用的场景：**

- 函数确实需要拥有数据的所有权
- 需要在多个线程间传递数据（需要使用 Arc 等并发原语）
- 数据的生命周期不明确（后续生命周期文章会详细讲解）

## 总结

引用和借用是 Rust 所有权系统的重要组成部分。通过引用，我们可以在不转移所有权的情况下访问数据，这大大提高了代码的灵活性和效率。

**核心要点：**

1. 引用不拥有数据，只是借用了访问权
2. 不可变引用（`&T`）只能读取，可变引用（`&mut T`）可以修改
3. 借用规则：同一时刻可以有多个不可变引用或一个可变引用
4. 引用必须总是有效的，编译器会阻止悬垂引用
5. 函数参数优先使用引用，特别是 `&str` 这样的通用引用类型
