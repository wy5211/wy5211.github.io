## 问题场景

随着项目变大，把所有代码放在一个文件里会变得难以维护。假设我们在开发任务处理系统，代码可能包括：

- 任务定义（Task 结构体）
- 任务处理器（TaskProcessor）
- 状态管理（状态枚举和处理逻辑）
- 工具函数（辅助函数）
- 错误处理（错误类型）

如果把这些代码都放在一个 `main.rs` 文件中，文件会很快变得庞大且难以维护。我们需要一种方式来组织代码，把相关功能分组、控制可见性、避免命名冲突。

Rust 提供了**模块（Module）系统**来解决这个问题。理解模块系统需要先搞清三个概念：**包（Package）**、**Crate**、**模块（Module）**。

## 包、Crate、模块的关系

### 层级关系

```
Package（包）
  ├── Cargo.toml     定义包的元信息和依赖
  ├── src/
  │   ├── main.rs    二进制 Crate 的入口
  │   └── lib.rs     库 Crate 的入口（可选）
  └── tests/         集成测试

一个 Package 包含：
  - 一个或多个 Crate
  - 最多一个库 Crate
  - 任意数量的二进制 Crate

Crate（箱）
  - 二进制 Crate：可执行程序，有 main() 函数
  - 库 Crate：可供其他代码引用，没有 main() 函数

Module（模块）
  - Crate 内的代码组织单元
  - 可以嵌套
  - 控制可见性（私有/公开）

Package → Crate → Module → 函数/结构体/枚举
```

### 实际示例

```bash
my_project/
├── Cargo.toml          # 包配置文件
├── src/
│   ├── main.rs         # 二进制 crate 入口
│   ├── lib.rs          # 库 crate 入口（可选）
│   ├── task.rs         # 任务模块
│   ├── processor.rs    # 处理器模块
│   └── models/
│       ├── mod.rs      # 模块声明
│       ├── status.rs   # 状态枚举
│       └── result.rs   # 结果类型
└── tests/              # 集成测试
    └── integration_test.rs
```

## 定义模块

### 内联模块

```rust
// 在 main.rs 中定义内联模块
mod task {
    // 结构体默认是私有的
    pub struct Task {
        pub id: u32,
        pub name: String,
        status: TaskStatus,  // 私有字段
    }

    // 枚举变体默认是公开的
    pub enum TaskStatus {
        Pending,
        InProgress,
        Completed,
    }

    // 函数默认是私有的
    pub fn create_task(id: u32, name: &str) -> Task {
        Task {
            id,
            name: name.to_string(),
            status: TaskStatus::Pending,
        }
    }

    // 私有函数
    fn validate_name(name: &str) -> bool {
        !name.is_empty()
    }
}

fn main() {
    // 使用完整路径访问模块
    let task = task::create_task(1, "数据处理");
    println!("任务: {}", task.name);

    // task.status  // 错误：status 是私有的
    // task::validate_name("test");  // 错误：validate_name 是私有的
}
```

### 分文件模块

当模块变大时，通常把模块移到单独的文件中：

```
src/
├── main.rs
├── task.rs       # task 模块
└── processor.rs  # processor 模块
```

`src/main.rs`:

```rust
mod task;      // 声明 task 模块，Rust 会查找 task.rs 文件
mod processor; // 声明 processor 模块

use task::Task;  // 引入 Task 类型

fn main() {
    let task = Task::new(1, "数据处理");
    processor::process_task(&task);
}
```

`src/task.rs`:

```rust
pub struct Task {
    pub id: u32,
    pub name: String,
    pub status: TaskStatus,
}

pub enum TaskStatus {
    Pending,
    InProgress,
    Completed,
}

impl Task {
    pub fn new(id: u32, name: &str) -> Self {
        Self {
            id,
            name: name.to_string(),
            status: TaskStatus::Pending,
        }
    }
}
```

`src/processor.rs`:

```rust
use crate::task::Task;  // 使用 crate:: 从包根开始引用

pub fn process_task(task: &Task) {
    println!("处理任务: {}", task.name);
}
```

### 目录模块

当模块变复杂时，可以升级为目录：

```
src/
├── main.rs
└── models/
    ├── mod.rs        # models 模块声明
    ├── task.rs       # 子模块
    └── status.rs     # 子模块
```

`src/models/mod.rs`:

```rust
pub mod task;    // 声明子模块
pub mod status;  // 声明子模块

// 可以在这里定义公共项
pub use task::Task;  // 重新导出 Task
pub use status::TaskStatus;  // 重新导出 TaskStatus
```

`src/models/task.rs`:

```rust
pub struct Task {
    pub id: u32,
    pub name: String,
    pub status: super::TaskStatus,  // 使用 super:: 引用父模块
}

impl Task {
    pub fn new(id: u32, name: &str) -> Self {
        Self {
            id,
            name: name.to_string(),
            status: super::TaskStatus::Pending,
        }
    }
}
```

`src/models/status.rs`:

```rust
pub enum TaskStatus {
    Pending,
    InProgress,
    Completed,
}
```

`src/main.rs`:

```rust
mod models;

use models::Task;  // 使用重新导出的类型

fn main() {
    let task = Task::new(1, "数据处理");
    println!("任务: {}", task.name);
}
```

## 可见性规则

### pub 关键字

```rust
mod my_module {
    // 公开函数
    pub fn public_function() {
        println!("公开函数");
    }

    // 私有函数
    fn private_function() {
        println!("私有函数");
    }

    // 公开子模块
    pub mod inner_module {
        pub fn inner_public() {
            println!("内部公开函数");
        }

        fn inner_private() {
            println!("内部私有函数");
        }
    }
}

fn main() {
    my_module::public_function();     // 可以
    // my_module::private_function();  // 错误：私有

    my_module::inner_module::inner_public();     // 可以
    // my_module::inner_module::inner_private(); // 错误：私有
}
```

### 结构体的可见性

```rust
mod auth {
    pub struct User {
        pub username: String,  // 公开字段
        email: String,         // 私有字段
        password_hash: String, // 私有字段
    }

    impl User {
        pub fn new(username: &str, email: &str, password: &str) -> Self {
            Self {
                username: username.to_string(),
                email: email.to_string(),
                password_hash: hash_password(password),
            }
        }

        pub fn verify_password(&self, password: &str) -> bool {
            self.password_hash == hash_password(password)
        }

        fn hash_password(password: &str) -> String {
            format!("hash({})", password)
        }
    }
}

fn main() {
    let user = auth::User::new("alice", "alice@example.com", "123456");

    println!("用户名: {}", user.username);  // 可以
    // println!("{}", user.email);  // 错误：私有字段
    // println!("{}", user.password_hash);  // 错误：私有字段

    println!("密码正确: {}", user.verify_password("123456"));  // 可以
}
```

**要点**：即使结构体是 `pub` 的，它的字段默认仍然是私有的。需要每个字段单独标记 `pub`。

### 枚举的可见性

枚举和结构体不同，`pub enum` 的所有变体都是公开的：

```rust
mod models {
    pub enum TaskStatus {
        Pending,
        InProgress,
        Completed,
    }
}

fn main() {
    let status = models::TaskStatus::Pending;  // 可以访问所有变体
}
```

### pub(crate) 和其他可见性

```rust
mod utils {
    // pub(crate) 表示在整个包内可见
    pub(crate) fn internal_helper() {
        println!("包内工具函数");
    }

    // pub(super) 表示在父模块内可见
    pub(super) fn parent_helper() {
        println!("父模块可见");
    }

    // pub(in path::to::module) 表示在指定路径内可见
    pub(in crate) fn crate_helper() {
        println!("crate 内可见");
    }
}
```

## use 关键字

`use` 用于将模块路径引入当前作用域，减少重复的完整路径：

```rust
mod models {
    pub mod task {
        pub struct Task {
            pub id: u32,
            pub name: String,
        }

        impl Task {
            pub fn new(id: u32, name: &str) -> Self {
                Self {
                    id,
                    name: name.to_string(),
                }
            }
        }
    }

    pub mod processor {
        use super::task::Task;  // 在模块内引入 Task

        pub fn process(task: &Task) {
            println!("处理任务: {}", task.name);
        }
    }
}

use models::task::Task;      // 引入 Task 类型
use models::processor;       // 引入 processor 模块

fn main() {
    let task = Task::new(1, "数据处理");
    processor::process(&task);
}
```

### use 引入到当前作用域

```rust
use models::task::Task;  // 引入类型
use models::task::Task::new;  // 引入关联函数

fn main() {
    let task = new(1, "数据处理");  // 直接使用函数名
    println!("任务: {}", task.name);
}
```

### as 重命名

```rust
use models::task::Task as TaskModel;  // 重命名避免冲突
use std::fmt::Result as FmtResult;  // 重命名

fn main() {
    let task = TaskModel::new(1, "数据处理");
}
```

### 嵌套路径

```rust
// 重复写前缀
// use models::task;
// use models::processor;

// 简写
use models::{task, processor};

// 更复杂的嵌套
use std::io::{self, Read, Write};
// 等价于
// use std::io;
// use std::io::Read;
// use std::io::Write;
```

### glob 引入

```rust
use std::collections::*;  // 引入所有公开项（谨慎使用）

fn main() {
    let mut map = HashMap::new();  // 不需要写 std::collections::HashMap
}
```

## 实战：任务处理系统的模块组织

让我们设计一个完整的任务处理系统的模块结构：

```
src/
├── main.rs           # 二进制入口
├── lib.rs            # 库入口
├── models/
│   ├── mod.rs        # 模块声明
│   ├── task.rs       # Task 结构体
│   ├── status.rs     # TaskStatus 枚举
│   └── result.rs     # ExecutionResult 枚举
├── processor/
│   ├── mod.rs        # 模块声明
│   └── executor.rs   # 任务执行器
└── utils/
    ├── mod.rs        # 模块声明
    └── time.rs       # 时间工具函数
```

`src/models/mod.rs`:

```rust
pub mod task;
pub mod status;
pub mod result;

// 重新导出常用类型
pub use task::Task;
pub use status::TaskStatus;
pub use result::ExecutionResult;
```

`src/models/task.rs`:

```rust
use crate::models::TaskStatus;

pub struct Task {
    pub id: u32,
    pub name: String,
    pub description: String,
    pub status: TaskStatus,
    pub result: Option<String>,
}

impl Task {
    pub fn new(id: u32, name: &str, description: &str) -> Self {
        Self {
            id,
            name: name.to_string(),
            description: description.to_string(),
            status: TaskStatus::Pending,
            result: None,
        }
    }

    pub fn get_info(&self) -> String {
        format!(
            "任务[{}]: {} - 状态: {:?}",
            self.id, self.name, self.status
        )
    }
}
```

`src/models/status.rs`:

```rust
#[derive(Debug, Clone, Copy)]
pub enum TaskStatus {
    Pending,
    InProgress,
    Completed,
    Failed,
}
```

`src/models/result.rs`:

```rust
pub enum ExecutionResult {
    Success(String),
    PartialSuccess { processed: usize, total: usize },
    Error(String),
}
```

`src/processor/mod.rs`:

`src/processor/executor.rs`:

```rust
use crate::models::{Task, TaskStatus, ExecutionResult};

pub struct TaskExecutor {
    tasks: Vec<Task>,
    next_id: u32,
}

impl TaskExecutor {
    pub fn new() -> Self {
        Self {
            tasks: Vec::new(),
            next_id: 1,
        }
    }

    pub fn add_task(&mut self, name: &str, description: &str) -> u32 {
        let task = Task::new(self.next_id, name, description);
        let id = task.id;
        self.tasks.push(task);
        self.next_id += 1;
        id
    }

    pub fn execute_task(&mut self, id: u32) -> ExecutionResult {
        if let Some(task) = self.tasks.iter_mut().find(|t| t.id == id) {
            task.status = TaskStatus::InProgress;

            // 模拟执行
            let result = ExecutionResult::Success("任务执行成功".to_string());

            match &result {
                ExecutionResult::Success(msg) => {
                    task.status = TaskStatus::Completed;
                    task.result = Some(msg.clone());
                }
                ExecutionResult::Error(err) => {
                    task.status = TaskStatus::Failed;
                    task.result = Some(err.clone());
                }
                _ => {}
            }

            result
        } else {
            ExecutionResult::Error("任务不存在".to_string())
        }
    }

    pub fn list_tasks(&self) {
        println!("\n=== 任务列表 ===");
        for task in &self.tasks {
            println!("{}", task.get_info());
        }
        println!("================\n");
    }
}
```

`src/utils/mod.rs`:

```rust

pub use time::current_time;

`src/utils/time.rs`:

`src/lib.rs`:

```

`src/main.rs`:

```rust
// 使用库
use task_processor::{TaskExecutor, ExecutionResult};

fn main() {
    let mut executor = TaskExecutor::new();

    // 添加任务
    executor.add_task("数据备份", "备份用户数据");
    executor.add_task("日志清理", "清理过期日志");

    // 列出任务
    executor.list_tasks();

    // 执行任务
    let result = executor.execute_task(1);
    match result {
        ExecutionResult::Success(msg) => println!("成功: {}", msg),
        ExecutionResult::Error(err) => println!("错误: {}", err),
        _ => {}
    }

    // 再次列出任务
    executor.list_tasks();
}
```

## 常见错误与解决

### 错误1：忘记 pub

```rust
mod my_mod {
    // fn helper() {}     // 错误：默认私有
    pub fn helper() {}    // 正确：公开
}
```

### 错误2：use 路径错误

```rust
// 错误：路径不正确
// use my_module::helper;

// 正确：检查模块是否正确声明
mod my_module {
    pub fn helper() {}
}
use my_module::helper;
```

### 错误3：循环依赖

```rust
// mod a.rs
// use crate::b::function_b;  // 错误：循环依赖
// pub fn function_a() {}

// mod b.rs
// use crate::a::function_a;
// pub fn function_b() {}

// 解决：重构模块结构
```

## 模块系统的权衡

### 优势

1. **代码组织**：将相关功能分组，提高代码可读性
2. **封装性**：通过可见性控制接口和实现
3. **避免命名冲突**：不同模块可以有同名项
4. **可重用性**：模块可以独立测试和重用

### 权衡

1. **复杂度**：大型项目的模块结构可能变得复杂
2. **路径长度**：可能需要写很长的路径
3. **学习成本**：需要理解可见性规则和路径系统

## 最佳实践

1. **按功能组织模块**：而不是按类型（不要把所有结构体放一个文件）
2. **最小化 pub**：只暴露必要的接口，其余保持私有
3. **用 `use` 简化长路径**：但不要过度使用 glob `*`
4. **合理使用 `pub(crate)`**：包内共享但外部不可见的工具函数
5. **大项目用目录模块**：模块超过 100 行考虑拆分为目录
6. **使用 `mod.rs` 或同名文件**：两种写法都支持，选择一致的风格

## 总结

模块系统是 Rust 组织代码的核心机制。通过模块，我们可以将代码按功能分组，控制可见性，避免命名冲突，提高代码的可维护性。

**核心要点：**

1. Package → Crate → Module → 函数/结构体/枚举
2. 默认私有，使用 `pub` 控制可见性
3. `use` 简化路径引用
4. `crate::` 从包根开始，`super::` 从父模块开始
5. 结构体字段需要单独 pub，枚举变体默认全部 pub
6. 支持多种可见性：`pub`、`pub(crate)`、`pub(super)` 等
