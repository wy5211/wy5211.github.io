## 为什么选择 Hono？

在深入之前，让我们先看看 Hono 在 Web 框架生态中的定位：

### 与主流框架对比

| 特性           | Hono          | Express     | NestJS      | Fastify     |
| -------------- | ------------- | ----------- | ----------- | ----------- |
| **包大小**     | ~13KB (gzip)  | ~200KB      | ~500KB+     | ~180KB      |
| **TypeScript** | ✅ 原生支持   | ⚠️ 需要配置 | ✅ 原生支持 | ✅ 原生支持 |
| **边缘运行**   | ✅ 完美支持   | ❌ 不支持   | ❌ 不支持   | ⚠️ 有限支持 |
| **学习曲线**   | 🟢 平缓       | 🟢 平缓     | 🔴 陡峭     | 🟡 中等     |
| **性能**       | 🚀 极快       | 🐢 较慢     | 🐢 较慢     | 🚀 快       |
| **装饰器**     | ❌ 无         | ❌ 无       | ✅ 大量     | ❌ 无       |
| **适用场景**   | 边缘计算、API | 传统服务    | 企业应用    | 高性能服务  |

### Hono 的核心优势

**1. 超轻量级**

```typescript
// Hono 核心包仅 13KB（gzip）
import { Hono } from "hono";

const app = new Hono();
app.get("/", (c) => c.text("Hello Hono!"));
```

**2. 极致性能**

基于 Web Standards 设计，充分利用现代 JavaScript 引擎优化。在基准测试中，Hono 的性能表现优异：

```
Frameworks (simple route)

Express:       38,502 req/sec
Fastify:       78,952 req/sec
Hono:          143,210 req/sec ⚡
```

**3. 边缘优先**

Hono 天生为边缘计算设计，支持：

- 🌐 **Cloudflare Workers**
- 🦕 **Deno Deploy**
- 🥟 **Bun**
- 🟢 **Node.js** (20+)
- 🟠 **Fastly Compute@Edge**
- 🔵 **AWS Lambda**

这意味着你的应用可以部署在全球数千个边缘节点，为用户提供毫秒级响应！

**4. TypeScript 原生**

100% TypeScript 编写，提供完整的类型推断和智能提示：

```typescript
import { Hono } from "hono";

const app = new Hono();

// 类型安全的路由
app.post("/users", async (c) => {
  const body = await c.req.json(); // 类型自动推断
  // body: any

  return c.json({ message: "User created" });
});
```

**5. 简单直观**

如果你用过 Express，你会觉得 Hono 非常熟悉：

```typescript
// Express
app.get("/users/:id", (req, res) => {
  const id = req.params.id;
  res.json({ id });
});

// Hono
app.get("/users/:id", (c) => {
  const id = c.req.param("id");
  return c.json({ id });
});
```

## 核心概念概览

Hono 的设计非常简洁，主要由以下几个核心概念组成：

### 1. 应用实例（App）

Hono 应用的核心是 `Hono` 类的实例：

```typescript
import { Hono } from "hono";

const app = new Hono();
```

### 2. 路由（Routes）

定义 API 端点：

```typescript
app.get("/", handler);
app.post("/users", handler);
app.put("/users/:id", handler);
```

### 3. 上下文（Context）

每个请求都有一个 `Context` 对象，包含了请求和响应的所有信息：

```typescript
app.get("/", (c) => {
  // c 是 Context 对象
  // c.req - 请求
  // c.json() - 响应方法
  return c.json({ message: "Hello" });
});
```

### 4. 中间件（Middleware）

在请求处理前后执行逻辑：

```typescript
app.use("*", logger()); // 日志中间件
app.use("*", cors()); // CORS 中间件
```

这就是 Hono 的全部核心概念！简单吧？

## 环境搭建

让我们开始搭建第一个 Hono 项目。

### 前置要求

- **Node.js** 20+ 或 **Deno** 或 **Bun**（任选其一）
- **包管理器**：pnpm（推荐）/ npm / yarn

### 使用 npm/pnpm 创建项目

```bash
# 创建项目目录
mkdir my-hono-app
cd my-hono-app

# 初始化项目
pnpm init

# 安装 Hono 和适配器
pnpm add hono
pnpm add @hono/node-server # Node.js 适配器
```

**为什么需要适配器？**

Hono 核心是运行时无关的，需要适配器才能在特定运行时上运行：

- `@hono/node-server` - Node.js
- `@hono/cloudflare-workers` - Cloudflare Workers
- Deno/Bun 无需额外适配器

### 项目结构

推荐的项目结构：

```
my-hono-app/
├── src/
│   └── index.ts        # 应用入口
├── package.json
└── tsconfig.json
```

### TypeScript 配置

创建 `tsconfig.json`：

```json
{
  "compilerOptions": {
    "target": "ES2022",
    "module": "ES2022",
    "lib": ["ES2022"],
    "moduleResolution": "bundler",
    "strict": true,
    "skipLibCheck": true,
    "esModuleInterop": true,
    "resolveJsonModule": true,
    "allowSyntheticDefaultImports": true
  },
  "include": ["src/**/*"],
  "exclude": ["node_modules"]
}
```

## 创建第一个 Hono 应用

### 基础版本

创建 `src/index.ts`：

```typescript
import { Hono } from "hono";
import { serve } from "@hono/node-server";

const app = new Hono();

// 定义路由

  return c.text("Hello Hono!");
});

console.log(`Server is running on http://localhost:${port}`);

serve({
  fetch: app.fetch,
  port,
});
```


```

```

### 添加更多路由

让我们创建一个更完整的示例：


## 多运行时支持

Hono 最强大的特性之一是"一次编写，到处运行"。同样的代码可以运行在多个运行时上！

### 1. Node.js

```typescript
// src/index.ts
import { Hono } from "hono";
import { serve } from "@hono/node-server";

const app = new Hono();

app.get("/", (c) => c.text("Running on Node.js!"));

export default app;

// 开发环境启动
if (import.meta.env.DEV) {
  serve({
    fetch: app.fetch,
    port: 3000,
  });
}
```

### 2. Deno

```typescript
// deno.json
{
  "tasks": {
    "dev": "deno run --watch --allow-net src/index.ts"
  }
}
```

```typescript
// src/index.ts (Deno)
import { Hono } from "https://deno.land/x/hono/mod.ts";
import { serve } from "https://deno.land/x/hono/server.ts";

const app = new Hono();

app.get("/", (c) => c.text("Running on Deno!"));

await serve(app.fetch, { port: 3000 });
```

### 3. Bun

```typescript
// src/index.ts (Bun)
```

## 完整示例：RESTful API

让我们创建一个完整的 RESTful API 示例：

```typescript
import { Hono } from "hono";
import { logger } from "hono/logger";
import { cors } from "hono/cors";
import { serve } from "@hono/node-server";

// 类型定义
type User = {
  id: string;
  name: string;
  email: string;
  createdAt: string;
};

// 模拟数据库
const users: User[] = [
  {
    id: "1",
    name: "Alice",
    email: "alice@example.com",
    createdAt: new Date().toISOString(),
  },
  {
    id: "2",
    name: "Bob",
    email: "bob@example.com",
    createdAt: new Date().toISOString(),
  },
];

const app = new Hono();

// 中间件
app.use("*", logger());
app.use("*", cors());

// 路由
// GET / - 首页
app.get("/", (c) => {
  return c.json({
    message: "Welcome to User API",
    version: "1.0.0",
    endpoints: {
      users: "/users",
      userById: "/users/:id",
    },
  });
});

// GET /users - 获取用户列表
app.get("/users", (c) => {
  return c.json({
    data: users,
    total: users.length,
  });
});

// GET /users/:id - 获取单个用户
app.get("/users/:id", (c) => {
  const id = c.req.param("id");
  const user = users.find((u) => u.id === id);

  if (!user) {
    return c.json({ error: "User not found" }, 404);
  }

  return c.json({ data: user });
});

// POST /users - 创建用户
app.post("/users", async (c) => {
  const body = await c.req.json();
  const { name, email } = body;

  if (!name || !email) {
    return c.json({ error: "Name and email are required" }, 400);
  }

  const newUser: User = {
    id: String(users.length + 1),
    name,
    email,
    createdAt: new Date().toISOString(),
  };

  users.push(newUser);

  return c.json({ data: newUser }, 201);
});

// PUT /users/:id - 更新用户
app.put("/users/:id", async (c) => {
  const id = c.req.param("id");
  const body = await c.req.json();

  const userIndex = users.findIndex((u) => u.id === id);

  if (userIndex === -1) {
    return c.json({ error: "User not found" }, 404);
  }

  users[userIndex] = {
    ...users[userIndex],
    ...body,
  };

  return c.json({ data: users[userIndex] });
});

// DELETE /users/:id - 删除用户
app.delete("/users/:id", (c) => {
  const id = c.req.param("id");
  const userIndex = users.findIndex((u) => u.id === id);

  if (userIndex === -1) {
    return c.json({ error: "User not found" }, 404);
  }

  users.splice(userIndex, 1);

  return c.json({ message: "User deleted successfully" });
});

// 启动服务器
const port = 3000;
console.log(`🚀 Server is running on http://localhost:${port}`);
console.log(`📚 API Documentation: http://localhost:${port}/`);

serve({
  fetch: app.fetch,
  port,
});
```

**测试 API**：

```bash
# 获取所有用户
curl http://localhost:3000/users

# 获取单个用户
curl http://localhost:3000/users/1

# 创建用户
curl -X POST http://localhost:3000/users \
  -H "Content-Type: application/json" \
  -d '{"name":"Charlie","email":"charlie@example.com"}'

# 更新用户
curl -X PUT http://localhost:3000/users/1 \
  -H "Content-Type: application/json" \
  -d '{"name":"Alice Updated"}'

# 删除用户
curl -X DELETE http://localhost:3000/users/1
```

## 最佳实践

### 1. 模块化路由

对于大型应用，建议将路由拆分到独立文件：

```typescript
// routes/users.ts
import { Hono } from "hono";

const users = new Hono();

users.get("/", listUsers);
users.get("/:id", getUser);
users.post("/", createUser);
users.put("/:id", updateUser);
users.delete("/:id", deleteUser);

export default users;

// index.ts
import { Hono } from "hono";
import users from "./routes/users";

const app = new Hono();
app.route("/api/users", users);
```

### 2. 环境变量

使用 `dotenv` 管理环境变量：

```typescript
import { config } from "dotenv";

config();

const app = new Hono();

app.get("/", (c) => {
  const env = c.env?.ENV || process.env.NODE_ENV || "development";
  return c.json({ environment: env });
});
```

### 3. 错误处理

添加全局错误处理：

```typescript
app.onError((err, c) => {
  console.error(err);
  return c.json(
    {
      success: false,
      message:
        c.env?.ENV === "production" ? "Internal Server Error" : err.message,
    },
    500
  );
});

app.notFound((c) => {
  return c.json(
    {
      success: false,
      message: "Route not found",
    },
    404
  );
});
```

### 4. 类型安全

为 Context 添加类型：

```typescript
type Env = {
  Bindings: {
    DB: D1Database;
    CACHE: KVNamespace;
  };
};

const app = new Hono<Env>();

app.get("/", (c) => {
  const db = c.env.DB; // 类型安全
  // ...
});
```

## 常见问题

### Q1: Hono 和 Express 该怎么选？

**A**:

- **选择 Hono**：需要边缘部署、追求极致性能、想要更好的 TypeScript 体验
- **选择 Express**：现有项目、团队熟悉、需要大量中间件生态

### Q2: Hono 适合大型应用吗？

**A**: 非常适合！Hono 的模块化设计让大型应用的组织变得简单。很多生产级应用都在使用 Hono。

### Q3: 如何从 Express 迁移到 Hono？

**A**: 迁移成本很低！API 非常相似，主要差异在于：

- `req.params` → `c.req.param()`
- `req.query` → `c.req.query()`
- `res.json()` → `c.json()`

### Q4: Hono 的中间件生态丰富吗？

**A**: Hono 有丰富的内置中间件和社区插件，涵盖 CORS、日志、认证、验证等常见需求。
