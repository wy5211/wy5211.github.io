## 前言

Deno Deploy 是 Deno 官方的无服务器边缘计算平台，以其出色的性能和简洁的开发体验而闻名。与 Cloudflare Workers 类似，Deno Deploy 也在全球边缘网络运行代码，但使用 Deno 运行时。

Hono 对 Deno 有原生支持，可以轻松部署到 Deno Deploy。

本篇文章将带你：

- 🦕 理解 Deno Deploy 的特点
- ⚙️ 配置 Deno 项目
- 🔧 设置环境变量
- 💾 使用 Deno KV
- 🚀 实现持续部署
- 📊 监控应用性能

## 应用代码

### 基础设置

```typescript
// src/main.ts
import { Hono } from "hono";
import { serve } from "https://deno.land/std@0.208.0/http/server.ts";

const app = new Hono();

app.get("/", (c) => {
  return c.json({
    message: "Hello from Deno Deploy!",
    timestamp: new Date().toISOString(),
  });
});

// Deno Deploy
export default app;

// 本地开发
if (import.meta.main) {
  await serve(app.fetch, {
    port: 8000,
    onListen: ({ port, hostname }) => {
      console.log(`Server running on http://${hostname}:${port}`);
    },
  });
}
```

### 环境变量

```typescript
// 类型安全的 Bindings
type Bindings = {
  KV: Deno.Kv;
  DATABASE_URL: string;
  API_KEY: string;
  ENVIRONMENT: "development" | "production";
};

const app = new Hono<{ Bindings: Bindings }>();

app.get("/", (c) => {
  return c.json({
    environment: c.env.ENVIRONMENT,
  });
});

app.get("/kv/:key", async (c) => {
  const { key } = c.req.param();
  const value = await c.env.KV.get(["cache", key]);

  return c.json({ key, value });
});
```

## Deno KV

### 创建 KV 数据库

```bash
# 本地开发
deno kv --allow-net --allow-read

# 初始化数据库
deno kv init
```

### KV 基本操作

```typescript
// 使用 KV
app.get("/kv", async (c) => {
  const kv = await Deno.openKv();

  // 设置值
  await kv.set(["users", "123"], {
    name: "John",
    email: "john@example.com",
  });

  // 获取值
  const user = await kv.get(["users", "123"]);

  // 删除值
  await kv.delete(["users", "123"]);

  // 列出所有键
  const entries = kv.list({ prefix: ["users"] });

  return c.json({ user: user.value });
});
```

### 原子操作

```typescript
// 原子递增
app.post("/posts/:id/like", async (c) => {
  const { id } = c.req.param();
  const kv = await Deno.openKv();

  const result = await kv
    .atomic()
    .mutate({
      type: "increment",
      key: ["posts", id, "likes"],
      value: new BigInt(1),
    })
    .commit();

  return c.json({ success: result.ok });
});
```

## 部署配置

### GitHub Actions

```yaml
# .github/workflows/deploy-deno.yml
name: Deploy to Deno Deploy

on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v3

      - name: Deploy to Deno Deploy
        uses: denoland/deployctl@v1
        with:
          project: my-hono-app
          entrypoint: src/main.ts
          github-token: ${{ secrets.GITHUB_TOKEN }}
```

### 手动部署

```bash
# 安装 deployctl
deno install --allow-net --allow-read https://deno.land/x/deployctl/deployctl.ts

# 登录
deployctl login

# 部署
deployctl deploy --project=my-hono-app src/main.ts
```

## 环境变量

### Deno Deploy 设置

在 Deno Deploy Dashboard 中设置：

- `KV`: KV 数据库绑定
- `DATABASE_URL`: 数据库连接字符串
- `API_KEY`: API 密钥
- `ENVIRONMENT`: 环境标识

### 代码中使用

```typescript
app.get("/env", (c) => {
  return c.json({
    kv: !!c.env.KV,
    databaseUrl: c.env.DATABASE_URL?.substring(0, 10) + "...",
    environment: c.env.ENVIRONMENT,
  });
});
```
