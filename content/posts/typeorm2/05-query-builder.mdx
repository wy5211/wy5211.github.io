在上一篇文章中，我们用 Repository 的 find 方法实现了任务的 CRUD 操作。find 方法配合 FindOptions 可以处理大部分查询场景，但当查询需求变得复杂时，比如"统计每个项目下各状态的任务数"或者"查询未完成任务数大于 10 的项目"，find 方法就不够用了。

这时就需要 QueryBuilder 出场了。QueryBuilder 提供了类似 SQL 的链式 API，让你可以构建任意复杂的查询，同时保持类型安全。这篇文章会带你深入了解 QueryBuilder 的各种用法，从基础的联表查询到复杂的统计聚合。

我们的项目管理平台需要一个 Dashboard 来展示各种统计数据：每个工作空间的项目数、每个项目的任务完成率、各状态的任务数分布。这些查询都离不开 QueryBuilder。

## QueryBuilder 基础

QueryBuilder 的 API 设计和 SQL 很像，如果你熟悉 SQL，上手会很快。

### 创建 QueryBuilder

```typescript
import { DataSource } from "typeorm";
import { Task } from "./entity/Task";

// 方式 1：通过 Repository 创建
const qb = taskRepo.createQueryBuilder("task");

// 方式 2：通过 DataSource 创建
const qb = dataSource.createQueryBuilder(Task, "task");

// 方式 3：通过 EntityManager 创建
const qb = entityManager.createQueryBuilder(Task, "task");
```

参数 "task" 是表的别名，后续所有引用都要用这个别名。

### 基本查询

```typescript
// SELECT 查询
const tasks = await taskRepo.createQueryBuilder("task").getMany();
// SELECT * FROM tasks task

// 查询单条
const task = await taskRepo
  .createQueryBuilder("task")
  .where("task.id = :id", { id: 1 })
  .getOne();
// SELECT * FROM tasks task WHERE task.id = 1

// 查询数量
const count = await taskRepo
  .createQueryBuilder("task")
  .where("task.status = :status", { status: "TODO" })
  .getCount();
// SELECT COUNT(*) FROM tasks task WHERE task.status = 'TODO'
```

### 选择字段

```typescript
// 选择所有字段
const tasks = await taskRepo.createQueryBuilder("task").select().getMany();

// 选择指定字段
const tasks = await taskRepo
  .createQueryBuilder("task")
  .select(["task.id", "task.title", "task.status"])
  .getMany();
// SELECT task.id, task.title, task.status FROM tasks task

// 使用 addSelect 追加字段
const tasks = await taskRepo
  .createQueryBuilder("task")
  .select("task.id")
  .addSelect("task.title")
  .getMany();

// 使用别名（返回原始数据）
const result = await taskRepo
  .createQueryBuilder("task")
  .select("task.title", "taskTitle")
  .addSelect("task.status", "taskStatus")
  .getRawMany();
// SELECT task.title AS "taskTitle", task.status AS "taskStatus" FROM tasks task
// 返回: [{ taskTitle: '修复导航栏', taskStatus: 'TODO' }, ...]
```

### WHERE 条件

```typescript
// 等值条件
const tasks = await taskRepo
  .createQueryBuilder("task")
  .where("task.id = :id", { id: 1 })
  .getOne();

// 多个条件（AND）
const tasks = await taskRepo
  .createQueryBuilder("task")
  .where("task.status = :status", { status: "IN_PROGRESS" })
  .andWhere("task.priority = :priority", { priority: "URGENT" })
  .getMany();
// WHERE task.status = 'IN_PROGRESS' AND task.priority = 'URGENT'

// OR 条件
const tasks = await taskRepo
  .createQueryBuilder("task")
  .where("task.status = :status1", { status1: "TODO" })
  .orWhere("task.status = :status2", { status2: "IN_PROGRESS" })
  .getMany();
// WHERE task.status = 'TODO' OR task.status = 'IN_PROGRESS'

// 复杂条件组合（使用 Brackets）
import { Brackets } from "typeorm";

const tasks = await taskRepo
  .createQueryBuilder("task")
  .where(
    new Brackets((qb) => {
      qb.where("task.status = :status", { status: "TODO" }).orWhere(
        "task.assignee IS NULL"
      );
    })
  )
  .andWhere("task.priority = :priority", { priority: "URGENT" })
  .getMany();
// WHERE (task.status = 'TODO' OR task.assignee_id IS NULL)
//   AND task.priority = 'URGENT'

// IN 条件
const tasks = await taskRepo
  .createQueryBuilder("task")
  .where("task.status IN (:...statuses)", {
    statuses: ["TODO", "IN_PROGRESS"],
  })
  .getMany();

// LIKE 条件
const tasks = await taskRepo
  .createQueryBuilder("task")
  .where("task.title LIKE :keyword", { keyword: "%导航栏%" })
  .getMany();

// BETWEEN 条件
const tasks = await taskRepo
  .createQueryBuilder("task")
  .where("task.createdAt BETWEEN :start AND :end", {
    start: new Date("2026-01-01"),
    end: new Date("2026-12-31"),
  })
  .getMany();

// IS NULL
const tasks = await taskRepo
  .createQueryBuilder("task")
  .where("task.assignee IS NULL")
  .getMany();
```

### 排序和分页

```typescript
const tasks = await taskRepo
  .createQueryBuilder("task")
  .where("task.projectId = :projectId", { projectId: 1 })
  .orderBy("task.priority", "DESC") // 按优先级降序
  .addOrderBy("task.createdAt", "ASC") // 同级按创建时间升序
  .skip(0) // 跳过 0 条（offset）
  .take(10) // 取 10 条（limit）
  .getMany();
```

## 联表查询（JOIN）

联表查询是 QueryBuilder 最强大的功能之一。在项目管理平台中，我们经常需要关联查询任务、项目、用户、标签等多张表。

### LEFT JOIN

```typescript
// 查询任务及其指派人（左连接）
const tasks = await taskRepo
  .createQueryBuilder("task")
  .leftJoinAndSelect("task.assignee", "user") // 关联并加载用户数据
  .where("task.projectId = :projectId", { projectId: 1 })
  .getMany();
// SELECT task.*, user.* FROM tasks task
// LEFT JOIN users user ON user.id = task.assignee_id
// 结果包含没有指派人的任务（user 为 null）

// 只用于条件过滤，不加载关联数据
const tasks = await taskRepo
  .createQueryBuilder("task")
  .leftJoin("task.assignee", "user") // 不加 AndSelect
  .where("user.name = :name", { name: "zhang_wei" })
  .getMany();
// 只返回指派给 zhang_wei 的任务，但不包含用户数据
```

### INNER JOIN

```typescript
// 内连接：只返回有匹配记录的行
const tasks = await taskRepo
  .createQueryBuilder("task")
  .innerJoinAndSelect("task.assignee", "user")
  .where("task.projectId = :projectId", { projectId: 1 })
  .getMany();
// 只返回有指派人的任务

// 带条件的内连接
const tasks = await taskRepo
  .createQueryBuilder("task")
  .innerJoinAndSelect(
    "task.assignee",
    "user",
    "user.isActive = :active", // 连接条件
    { active: true }
  )
  .where("task.projectId = :projectId", { projectId: 1 })
  .getMany();
// 只返回指派给活跃用户的任务
```

### 多表关联

```typescript
// 查询任务及其所属项目、工作空间、指派人、标签
const tasks = await taskRepo
  .createQueryBuilder("task")
  .leftJoinAndSelect("task.project", "project")
  .leftJoinAndSelect("project.workspace", "workspace")
  .leftJoinAndSelect("task.assignee", "user")
  .leftJoinAndSelect("task.tags", "tag")
  .where("workspace.id = :workspaceId", { workspaceId: 1 })
  .orderBy("task.priority", "DESC")
  .getMany();
```

### 选择关联表字段

```typescript
// 只选择关联表的部分字段
const tasks = await taskRepo
  .createQueryBuilder("task")
  .leftJoin("task.assignee", "user")
  .leftJoin("task.tags", "tag")
  .select([
    "task.id",
    "task.title",
    "task.status",
    "user.id",
    "user.name",
    "tag.id",
    "tag.name",
  ])
  .where("task.projectId = :projectId", { projectId: 1 })
  .getRawMany();
// 返回原始数据: [{ task_id: 1, task_title: '...', user_name: '...', tag_name: '...' }, ...]
```

## 分组聚合

分组聚合查询是 Dashboard 统计的基础。

### 基本聚合

```typescript
// 统计各状态的任务数
const stats = await taskRepo
  .createQueryBuilder("task")
  .select("task.status", "status")
  .addSelect("COUNT(*)", "count")
  .where("task.projectId = :projectId", { projectId: 1 })
  .groupBy("task.status")
  .getRawMany();
// SELECT task.status, COUNT(*) AS "count"
// FROM tasks task
// WHERE task.project_id = 1
// GROUP BY task.status
// 结果: [{ status: 'TODO', count: '15' }, { status: 'IN_PROGRESS', count: '8' }, ...]
```

### 多字段聚合

```typescript
// 统计每个项目各状态的任务数
const stats = await taskRepo
  .createQueryBuilder("task")
  .leftJoin("task.project", "project")
  .select("project.name", "projectName")
  .addSelect("task.status", "status")
  .addSelect("COUNT(*)", "count")
  .groupBy("project.id, task.status")
  .orderBy("project.name", "ASC")
  .addOrderBy("task.status", "ASC")
  .getRawMany();
// 结果: [
//   { projectName: '官网重构 v2.0', status: 'TODO', count: '5' },
//   { projectName: '官网重构 v2.0', status: 'IN_PROGRESS', count: '3' },
//   ...
// ]
```

### HAVING 条件

```typescript
// 查询未完成任务数大于 10 的项目
const projects = await taskRepo
  .createQueryBuilder("task")
  .leftJoin("task.project", "project")
  .select("project.id", "projectId")
  .addSelect("project.name", "projectName")
  .addSelect("COUNT(*)", "taskCount")
  .where("task.status != :status", { status: "COMPLETED" })
  .groupBy("project.id")
  .having("COUNT(*) > :min", { min: 10 })
  .orderBy("taskCount", "DESC")
  .getRawMany();
// SELECT project.id, project.name, COUNT(*) AS "taskCount"
// FROM tasks task
// LEFT JOIN projects project ON project.id = task.project_id
// WHERE task.status != 'COMPLETED'
// GROUP BY project.id
// HAVING COUNT(*) > 10
// ORDER BY "taskCount" DESC
```

### 时间聚合

```typescript
// 统计每天的任务完成数
import { Raw } from "typeorm";

const dailyStats = await taskRepo
  .createQueryBuilder("task")
  .select("DATE(task.completed_at)", "date")
  .addSelect("COUNT(*)", "count")
  .where("task.status = :status", { status: "COMPLETED" })
  .andWhere("task.completedAt IS NOT NULL")
  .groupBy("DATE(task.completed_at)")
  .orderBy("date", "DESC")
  .limit(30)
  .getRawMany();
// 结果: [
//   { date: '2026-03-15', count: '12' },
//   { date: '2026-03-14', count: '8' },
//   ...
// ]
```

## 子查询

子查询在复杂查询中非常有用，比如"查询有未完成任务的项目"。

### WHERE 中的子查询

```typescript
// 查询有未完成任务的项目
const projects = await projectRepo
  .createQueryBuilder("project")
  .where((qb) => {
    const subQuery = qb
      .subQuery()
      .select("task.projectId")
      .from(Task, "task")
      .where("task.status != :status", { status: "COMPLETED" })
      .getQuery();
    return "project.id IN " + subQuery;
  })
  .getMany();
// SELECT * FROM projects project
// WHERE project.id IN (
//   SELECT task.project_id FROM tasks task WHERE task.status != 'COMPLETED'
// )
```

### FROM 中的子查询

```typescript
// 统计每个项目的任务数，并按任务数排序
const result = await dataSource
  .createQueryBuilder()
  .select("summary.projectId", "projectId")
  .addSelect("summary.projectName", "projectName")
  .addSelect("summary.taskCount", "taskCount")
  .from((subQuery) => {
    return subQuery
      .select("project.id", "projectId")
      .addSelect("project.name", "projectName")
      .addSelect("COUNT(*)", "taskCount")
      .from(Task, "task")
      .leftJoin("task.project", "project")
      .groupBy("project.id")
      .getQuery();
  }, "summary")
  .orderBy("summary.taskCount", "DESC")
  .getRawMany();
```

### SELECT 中的子查询

```typescript
// 查询项目及其任务数
const projects = await projectRepo
  .createQueryBuilder("project")
  .leftJoinAndSelect("project.workspace", "workspace")
  .select(["project.id", "project.name"])
  .addSelect((subQuery) => {
    return subQuery
      .select("COUNT(*)")
      .from(Task, "task")
      .where("task.projectId = project.id")
      .andWhere("task.status != :status", { status: "COMPLETED" });
  }, "taskCount")
  .getRawMany();
// SELECT project.id, project.name,
//   (SELECT COUNT(*) FROM tasks task
//    WHERE task.project_id = project.id
//    AND task.status != 'COMPLETED') AS "taskCount"
// FROM projects project
```

### EXISTS 子查询

```typescript
// 查询有紧急任务的项目
const projects = await projectRepo
  .createQueryBuilder("project")
  .where((qb) => {
    const subQuery = qb
      .subQuery()
      .select("1")
      .from(Task, "task")
      .where("task.projectId = project.id")
      .andWhere("task.priority = :priority", { priority: "URGENT" })
      .getQuery();
    return `EXISTS ${subQuery}`;
  })
  .getMany();
// SELECT * FROM projects project
// WHERE EXISTS (
//   SELECT 1 FROM tasks task
//   WHERE task.project_id = project.id
//   AND task.priority = 'URGENT'
// )
```

## 计数字段

Query Builder 提供了一个便捷方法 loadRelationCountAndMap，用于计算关联关系的数量。

```typescript
// 查询项目及其任务数
const projects = await projectRepo
  .createQueryBuilder("project")
  .leftJoinAndSelect("project.workspace", "workspace")
  .loadRelationCountAndMap("project.taskCount", "project.tasks")
  .loadRelationCountAndMap(
    "project.completedTaskCount",
    "project.tasks",
    "task",
    (qb) => qb.where("task.status = :status", { status: "COMPLETED" })
  )
  .getMany();
// 结果: [
//   {
//     id: 1,
//     name: '官网重构 v2.0',
//     taskCount: 15,
//     completedTaskCount: 8,
//     workspace: { ... }
//   },
//   ...
// ]
```

这比子查询写法更简洁，而且性能也更好。

## 原始 SQL 查询

当 QueryBuilder 也无法满足需求时，可以直接执行原始 SQL。

```typescript
// 执行原始 SQL
const result = await dataSource.query(
  `SELECT
    p.id,
    p.name,
    COUNT(t.id) as task_count,
    COUNT(CASE WHEN t.status = 'COMPLETED' THEN 1 END) as completed_count
  FROM projects p
  LEFT JOIN tasks t ON t.project_id = p.id
  WHERE p.workspace_id = $1
  GROUP BY p.id
  ORDER BY task_count DESC`,
  [1]
);
```

但要注意，原始 SQL 失去了类型安全，而且容易引入 SQL 注入风险。只有在确实需要时才使用。

## 实战：Dashboard 查询服务

让我们整合上面的知识，实现一个 Dashboard 查询服务。

```typescript
import { Injectable } from "@nestjs/common";
import { InjectRepository } from "@nestjs/typeorm";
import { Repository, Brackets } from "typeorm";
import { Task, TaskStatus } from "./entity/Task";
import { Project } from "./entity/Project";
import { Workspace } from "./entity/Workspace";

interface DashboardStats {
  workspaceSummary: WorkspaceSummary[];
  projectSummary: ProjectSummary[];
  taskStats: TaskStats;
  recentTasks: Task[];
}

interface WorkspaceSummary {
  workspaceId: number;
  workspaceName: string;
  projectCount: number;
  taskCount: number;
}

interface ProjectSummary {
  projectId: number;
  projectName: string;
  totalTasks: number;
  completedTasks: number;
  completionRate: number;
}

interface TaskStats {
  byStatus: Record<string, number>;
  byPriority: Record<string, number>;
  overdueCount: number;
}

@Injectable()
export class DashboardService {
  constructor(
    @InjectRepository(Task)
    private taskRepo: Repository<Task>,
    @InjectRepository(Project)
    private projectRepo: Repository<Project>
  ) {}

  async getDashboardStats(workspaceId: number): Promise<DashboardStats> {
    const [workspaceSummary, projectSummary, taskStats, recentTasks] =
      await Promise.all([
        this.getWorkspaceSummary(workspaceId),
        this.getProjectSummary(workspaceId),
        this.getTaskStats(workspaceId),
        this.getRecentTasks(workspaceId),
      ]);

    return {
      workspaceSummary,
      projectSummary,
      taskStats,
      recentTasks,
    };
  }

  async getWorkspaceSummary(workspaceId: number) {
    const projects = await this.projectRepo
      .createQueryBuilder("project")
      .leftJoinAndSelect("project.workspace", "workspace")
      .select(["project.id", "project.name", "workspace.id"])
      .where("workspace.id = :workspaceId", { workspaceId })
      .getMany();

    const projectIds = projects.map((p) => p.id);

    // 统计每个项目的任务数
    const taskCounts = await this.taskRepo
      .createQueryBuilder("task")
      .select("task.projectId", "projectId")
      .addSelect("COUNT(*)", "taskCount")
      .where("task.projectId IN (:...projectIds)", { projectIds })
      .groupBy("task.projectId")
      .getRawMany();

    const taskCountMap = taskCounts.reduce(
      (acc, item) => {
        acc[item.projectId] = parseInt(item.taskCount);
        return acc;
      },
      {} as Record<number, number>
    );

    return projects.map((project) => ({
      workspaceId,
      workspaceName: project.workspace.name,
      projectId: project.id,
      projectName: project.name,
      projectCount: 1,
      taskCount: taskCountMap[project.id] || 0,
    }));
  }

  async getProjectSummary(workspaceId: number) {
    const stats = await this.taskRepo
      .createQueryBuilder("task")
      .leftJoin("task.project", "project")
      .select("project.id", "projectId")
      .addSelect("project.name", "projectName")
      .addSelect("COUNT(*)", "totalTasks")
      .addSelect(
        "COUNT(CASE WHEN task.status = :completed THEN 1 END)",
        "completedTasks"
      )
      .where("project.workspaceId = :workspaceId", { workspaceId })
      .setParameter("completed", TaskStatus.COMPLETED)
      .groupBy("project.id")
      .getRawMany();

    return stats.map((item) => {
      const total = parseInt(item.totalTasks);
      const completed = parseInt(item.completedTasks);
      const rate = total > 0 ? Math.round((completed / total) * 100) : 0;

      return {
        projectId: item.projectId,
        projectName: item.projectName,
        totalTasks: total,
        completedTasks: completed,
        completionRate: rate,
      };
    });
  }

  async getTaskStats(workspaceId: number) {
    // 按状态统计
    const byStatus = await this.taskRepo
      .createQueryBuilder("task")
      .leftJoin("task.project", "project")
      .select("task.status", "status")
      .addSelect("COUNT(*)", "count")
      .where("project.workspaceId = :workspaceId", { workspaceId })
      .groupBy("task.status")
      .getRawMany();

    const statusMap = byStatus.reduce(
      (acc, item) => {
        acc[item.status] = parseInt(item.count);
        return acc;
      },
      {} as Record<string, number>
    );

    // 按优先级统计
    const byPriority = await this.taskRepo
      .createQueryBuilder("task")
      .leftJoin("task.project", "project")
      .select("task.priority", "priority")
      .addSelect("COUNT(*)", "count")
      .where("project.workspaceId = :workspaceId", { workspaceId })
      .groupBy("task.priority")
      .getRawMany();

    const priorityMap = byPriority.reduce(
      (acc, item) => {
        acc[item.priority] = parseInt(item.count);
        return acc;
      },
      {} as Record<string, number>
    );

    // 统计逾期任务
    const overdueCount = await this.taskRepo
      .createQueryBuilder("task")
      .leftJoin("task.project", "project")
      .where("project.workspaceId = :workspaceId", { workspaceId })
      .andWhere("task.dueDate < :now", { now: new Date() })
      .andWhere("task.status != :status", { status: TaskStatus.COMPLETED })
      .getCount();

    return {
      byStatus: statusMap,
      byPriority: priorityMap,
      overdueCount,
    };
  }

  async getRecentTasks(workspaceId: number, limit = 10) {
    return this.taskRepo
      .createQueryBuilder("task")
      .leftJoinAndSelect("task.project", "project")
      .leftJoinAndSelect("task.assignee", "user")
      .leftJoinAndSelect("task.tags", "tag")
      .where("project.workspaceId = :workspaceId", { workspaceId })
      .orderBy("task.createdAt", "DESC")
      .limit(limit)
      .getMany();
  }
}
```

这个 Dashboard 查询服务展示了 QueryBuilder 的各种高级用法：JOIN、GROUP BY、CASE WHEN、子查询、计数等。它返回一个工作空间的完整统计数据：项目概览、任务统计、最近任务。

## Trade-off 分析

QueryBuilder vs Repository.find 如何选择？

Repository.find 简单直观，对于基础查询（条件、排序、分页、关联加载）完全够用。而且代码更简洁，可读性更好。

QueryBuilder 表达能力更强，可以处理任意复杂的查询（JOIN、GROUP BY、子查询），但代码会更冗长。一个实用的经验法则是：能用 find 解决的就用 find，find 写不出来的（比如统计查询）再用 QueryBuilder。

getMany vs getRawMany 如何选择？

getMany 返回实体对象，字段名是驼峰命名（task.title），适合需要继续操作实体的场景。

getRawMany 返回原始数据，字段名是查询中定义的别名（taskTitle），适合只读查询和统计场景。原始数据性能稍好，但失去了类型安全。

## 总结

这篇文章我们深入了 QueryBuilder 的各种用法：从基础的 WHERE、ORDER BY，到高级的 JOIN、GROUP BY、子查询。我们还实现了一个完整的 Dashboard 查询服务，展示了如何在实际项目中应用这些知识。

但还有一个重要问题没讲：当 Entity 定义改变后，数据库表结构怎么同步更新？直接用 synchronize: true 在生产环境是危险的。下一篇文章我们会讲数据库迁移，让你安全地管理数据库结构变更。
