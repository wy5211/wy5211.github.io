前面七篇文章我们学习了 TypeORM 的核心概念——从实体定义、关系映射、Repository 查询、QueryBuilder 高级查询、数据库迁移，到事务处理。但这些都是"知识点"，怎么把它们组装成一个真正能跑的项目呢？这一篇我们把这些知识整合到 NestJS 框架中，构建项目管理平台的任务模块。

## NestJS 与 TypeORM 的集成方式

NestJS 提供了 `@nestjs/typeorm` 包，让 TypeORM 的集成变得很简单。核心是两个模块配置：

- `TypeOrmModule.forRootAsync()`：配置数据库连接，在 AppModule 中全局注册
- `TypeOrmModule.forFeature()`：注册特定模块使用的 Entity 和 Repository

先看最基础的配置：

```typescript
// src/app.module.ts
import { Module } from "@nestjs/common";
import { ConfigModule, ConfigService } from "@nestjs/config";
import { TypeOrmModule } from "@nestjs/typeorm";

@Module({
  imports: [
    // 环境变量配置
    ConfigModule.forRoot({
      isGlobal: true,
      envFilePath: `.env.${process.env.NODE_ENV || "development"}`,
    }),

    // TypeORM 全局配置
    TypeOrmModule.forRootAsync({
      imports: [ConfigModule],
      inject: [ConfigService],
      useFactory: (configService: ConfigService) => ({
        type: "postgres",
        host: configService.get("DB_HOST", "localhost"),
        port: configService.get<number>("DB_PORT", 5432),
        username: configService.get("DB_USERNAME", "postgres"),
        password: configService.get("DB_PASSWORD", "postgres"),
        database: configService.get("DB_DATABASE", "project_management"),
        entities: [__dirname + "/**/*.entity{.ts,.js}"],
        synchronize: configService.get("NODE_ENV") === "development",
        logging: configService.get("NODE_ENV") === "development",
      }),
    }),
  ],
})
export class AppModule {}
```

`useFactory` 允许我们从环境变量动态读取配置，而不是硬编码。这样开发、测试、生产环境可以用不同的数据库连接。

## 定义 Entity

先定义任务相关的实体。还是用我们的项目管理平台案例，工作空间 `ws_fe_8x9k2m` 里有项目，项目下有任务：

```typescript
// src/tasks/entities/task.entity.ts
import {
  Entity,
  PrimaryGeneratedColumn,
  Column,
  CreateDateColumn,
  UpdateDateColumn,
  ManyToOne,
  ManyToMany,
  JoinTable,
  Index,
} from "typeorm";
import { Project } from "./project.entity";
import { User } from "./user.entity";
import { Tag } from "./tag.entity";

export enum TaskStatus {
  TODO = "TODO",
  IN_PROGRESS = "IN_PROGRESS",
  DONE = "DONE",
  CANCELLED = "CANCELLED",
}

export enum TaskPriority {
  LOW = "LOW",
  MEDIUM = "MEDIUM",
  HIGH = "HIGH",
  URGENT = "URGENT",
}

@Entity("tasks")
@Index(["projectId", "status"])
@Index(["assigneeId", "status"])
export class Task {
  @PrimaryGeneratedColumn()
  id: number;

  @Column({ length: 255 })
  title: string;

  @Column({ type: "text", nullable: true })
  description: string;

  @Column({ name: "project_id" })
  projectId: number;

  @ManyToOne(() => Project, { onDelete: "CASCADE" })
  project: Project;

  @Column({ name: "assignee_id", nullable: true })
  assigneeId: number;

  @ManyToOne(() => User, { onDelete: "SET NULL", nullable: true })
  assignee: User;

  @Column({
    type: "enum",
    enum: TaskStatus,
    default: TaskStatus.TODO,
  })
  status: TaskStatus;

  @Column({
    type: "enum",
    enum: TaskPriority,
    default: TaskPriority.MEDIUM,
  })
  priority: TaskPriority;

  @ManyToMany(() => Tag, { eager: true })
  @JoinTable({
    name: "task_tags",
    joinColumn: { name: "task_id", referencedColumnName: "id" },
    inverseJoinColumn: { name: "tag_id", referencedColumnName: "id" },
  })
  tags: Tag[];

  @Column({ name: "due_date", type: "timestamptz", nullable: true })
  dueDate: Date;

  @CreateDateColumn({ name: "created_at" })
  createdAt: Date;

  @UpdateDateColumn({ name: "updated_at" })
  updatedAt: Date;
}
```

注意 `@Index` 装饰器，我们在 `(projectId, status)` 和 `(assigneeId, status)` 上建了复合索引，这是为了优化常见的查询："查询某个项目下特定状态的任务"、"查询某人负责的特定状态的任务"。

## 创建模块和注册 Repository

接下来创建 Tasks 模块，注册 Entity 和注入 Repository：

```typescript
// src/tasks/tasks.module.ts
import { Module } from "@nestjs/common";
import { TypeOrmModule } from "@nestjs/typeorm";
import { TasksController } from "./tasks.controller";
import { TasksService } from "./tasks.service";
import { Task } from "./entities/task.entity";
import { Project } from "./entities/project.entity";
import { User } from "./entities/user.entity";
import { Tag } from "./entities/tag.entity";

@Module({
  imports: [TypeOrmModule.forFeature([Task, Project, User, Tag])],
  controllers: [TasksController],
  providers: [TasksService],
  exports: [TasksService],
})
export class TasksModule {}
```

`TypeOrmModule.forFeature()` 会为这些 Entity 创建 Repository，然后我们可以在 Service 中通过 `@InjectRepository()` 注入。

```typescript
// src/tasks/tasks.service.ts
import { Injectable, NotFoundException } from "@nestjs/common";
import { InjectRepository } from "@nestjs/typeorm";
import { Repository, In } from "typeorm";
import { Task, TaskStatus } from "./entities/task.entity";
import { CreateTaskDto } from "./dto/create-task.dto";
import { UpdateTaskDto } from "./dto/update-task.dto";

@Injectable()
export class TasksService {
  constructor(
    @InjectRepository(Task)
    private taskRepo: Repository<Task>
  ) {}

  async create(createTaskDto: CreateTaskDto): Promise<Task> {
    const task = this.taskRepo.create({
      ...createTaskDto,
      tags: createTaskDto.tagIds?.map((id) => ({ id })),
    });

    return this.taskRepo.save(task);
  }

  async findAll(projectId: number, status?: TaskStatus) {
    return this.taskRepo.find({
      where: { projectId, status },
      relations: ["assignee", "tags"],
      order: { createdAt: "DESC" },
    });
  }

  async findOne(id: number): Promise<Task> {
    const task = await this.taskRepo.findOne({
      where: { id },
      relations: ["project", "assignee", "tags"],
    });

    if (!task) {
      throw new NotFoundException(`任务 #${id} 不存在`);
    }

    return task;
  }

  async update(id: number, updateTaskDto: UpdateTaskDto): Promise<Task> {
    const task = await this.findOne(id);

    if (updateTaskDto.tagIds) {
      task.tags = updateTaskDto.tagIds.map((id) => ({ id })) as any;
    }

    Object.assign(task, updateTaskDto);

    return this.taskRepo.save(task);
  }

  async remove(id: number): Promise<void> {
    const task = await this.findOne(id);
    await this.taskRepo.remove(task);
  }
}
```

## DTO 验证

NestJS 推荐用 DTO 来验证请求数据，结合 `class-validator` 和 `class-transformer`，可以实现声明式的验证规则。

```typescript
// src/tasks/dto/create-task.dto.ts
import {
  IsString,
  IsNotEmpty,
  IsOptional,
  IsEnum,
  IsInt,
  IsArray,
  MaxLength,
  IsDateString,
} from "class-validator";
import { TaskStatus, TaskPriority } from "../entities/task.entity";

export class CreateTaskDto {
  @IsString()
  @IsNotEmpty()
  @MaxLength(255)
  title: string;

  @IsString()
  @IsOptional()
  description?: string;

  @IsInt()
  projectId: number;

  @IsInt()
  @IsOptional()
  assigneeId?: number;

  @IsEnum(TaskStatus)
  @IsOptional()
  status?: TaskStatus;

  @IsEnum(TaskPriority)
  @IsOptional()
  priority?: TaskPriority;

  @IsArray()
  @IsInt({ each: true })
  @IsOptional()
  tagIds?: number[];

  @IsDateString()
  @IsOptional()
  dueDate?: string;
}
```

然后在 `main.ts` 中全局启用 `ValidationPipe`：

```typescript
// src/main.ts
import { ValidationPipe } from "@nestjs/common";
import { NestFactory } from "@nestjs/core";
import { AppModule } from "./app.module";

async function bootstrap() {
  const app = await NestFactory.create(AppModule);

  app.useGlobalPipes(
    new ValidationPipe({
      whitelist: true, // 过滤掉 DTO 中没有定义的属性
      forbidNonWhitelisted: true, // 如果传入了未定义的属性，抛出错误
      transform: true, // 自动转换类型（比如字符串 "123" 转为数字 123）
    })
  );

  await app.listen(3000);
}
bootstrap();
```

这样如果客户端传入了 `{ title: '新任务', priority: 'INVALID' }`，ValidationPipe 会自动返回 400 错误，提示 `priority must be one of: LOW, MEDIUM, HIGH, URGENT`。

## Controller 层

Controller 负责处理 HTTP 请求，调用 Service，返回响应。

```typescript
// src/tasks/tasks.controller.ts
import {
  Controller,
  Get,
  Post,
  Put,
  Delete,
  Body,
  Param,
  Query,
  ParseIntPipe,
  HttpCode,
  HttpStatus,
} from "@nestjs/common";
import { TasksService } from "./tasks.service";
import { CreateTaskDto } from "./dto/create-task.dto";
import { UpdateTaskDto } from "./dto/update-task.dto";

@Controller("tasks")
export class TasksController {
  constructor(private readonly tasksService: TasksService) {}

  @Post()
  @HttpCode(HttpStatus.CREATED)
  async create(@Body() createTaskDto: CreateTaskDto) {
    const task = await this.tasksService.create(createTaskDto);
    return {
      success: true,
      data: task,
    };
  }

  @Get()
  async findAll(
    @Query("projectId", ParseIntPipe) projectId: number,
    @Query("status") status?: string
  ) {
    const tasks = await this.tasksService.findAll(projectId, status as any);
    return {
      success: true,
      data: tasks,
    };
  }

  @Get(":id")
  async findOne(@Param("id", ParseIntPipe) id: number) {
    const task = await this.tasksService.findOne(id);
    return {
      success: true,
      data: task,
    };
  }

  @Put(":id")
  async update(
    @Param("id", ParseIntPipe) id: number,
    @Body() updateTaskDto: UpdateTaskDto
  ) {
    const task = await this.tasksService.update(id, updateTaskDto);
    return {
      success: true,
      data: task,
    };
  }

  @Delete(":id")
  @HttpCode(HttpStatus.NO_CONTENT)
  async remove(@Param("id", ParseIntPipe) id: number) {
    await this.tasksService.remove(id);
  }
}
```

`ParseIntPipe` 会自动把路径参数从字符串转为数字，如果转换失败（比如 `/tasks/abc`）会返回 400 错误。

## 在事务中创建任务

上一篇文章我们学习了事务，现在把它集成到 NestJS 中。比如创建任务时需要关联标签、记录活动日志、更新项目统计：

```typescript
// src/tasks/tasks.service.ts
import { Injectable, NotFoundException } from "@nestjs/common";
import { InjectRepository } from "@nestjs/typeorm";
import { DataSource, Repository, In } from "typeorm";
import { Task } from "./entities/task.entity";
import { ActivityLog } from "./entities/activity-log.entity";
import { Project } from "./entities/project.entity";

@Injectable()
export class TasksService {
  constructor(
    @InjectRepository(Task)
    private taskRepo: Repository<Task>,
    private dataSource: DataSource // 注入 DataSource
  ) {}

  async createWithTransaction(createTaskDto: CreateTaskDto): Promise<Task> {
    return this.dataSource.transaction(async (manager) => {
      // 创建任务
      const task = manager.create(Task, {
        ...createTaskDto,
        tags: createTaskDto.tagIds?.map((id) => ({ id })),
      });
      await manager.save(task);

      // 记录活动日志
      await manager.save(ActivityLog, {
        workspaceId: createTaskDto.workspaceId,
        action: "TASK_CREATED",
        entityType: "TASK",
        entityId: task.id,
        userId: createTaskDto.creatorId,
      });

      // 更新项目统计
      await manager.increment(
        Project,
        { id: createTaskDto.projectId },
        "taskCount",
        1
      );

      return task;
    });
  }
}
```

注意这里注入了 `DataSource` 而不是 Repository，因为事务需要通过 `manager` 来操作，而不是普通的 Repository。

## 自定义 Repository

如果有一些通用的查询逻辑，可以封装到自定义 Repository 中：

```typescript
// src/tasks/repositories/task.repository.ts
import { EntityRepository, Repository } from "typeorm";
import { Task, TaskStatus } from "../entities/task.entity";

@EntityRepository(Task)
export class TaskRepository extends Repository<Task> {
  // 查询逾期任务
  async findOverdueTasks(projectId: number): Promise<Task[]> {
    return this.createQueryBuilder("task")
      .where("task.projectId = :projectId", { projectId })
      .andWhere("task.dueDate < :now", { now: new Date() })
      .andWhere("task.status NOT IN (:...completedStatuses)", {
        completedStatuses: [TaskStatus.DONE, TaskStatus.CANCELLED],
      })
      .leftJoinAndSelect("task.assignee", "assignee")
      .orderBy("task.dueDate", "ASC")
      .getMany();
  }

  // 统计各状态的任务数
  async countByStatus(projectId: number): Promise<Record<TaskStatus, number>> {
    const result = await this.createQueryBuilder("task")
      .select("task.status", "status")
      .addSelect("COUNT(*)", "count")
      .where("task.projectId = :projectId", { projectId })
      .groupBy("task.status")
      .getRawMany();

    return result.reduce(
      (acc, { status, count }) => {
        acc[status] = parseInt(count, 10);
        return acc;
      },
      {} as Record<TaskStatus, number>
    );
  }
}
```

然后在模块中注册自定义 Repository：

```typescript
// src/tasks/tasks.module.ts
import { Module } from "@nestjs/common";
import { TypeOrmModule } from "@nestjs/typeorm";
import { TaskRepository } from "./repositories/task.repository";

@Module({
  imports: [TypeOrmModule.forFeature([Task])],
  providers: [
    {
      provide: "TaskRepository",
      useClass: TaskRepository,
    },
    TasksService,
  ],
})
export class TasksModule {}
```

在 Service 中注入使用：

```typescript
// src/tasks/tasks.service.ts
@Injectable()
export class TasksService {
  constructor(
    @Inject("TaskRepository")
    private taskRepo: TaskRepository
  ) {}

  async getOverdueTasks(projectId: number) {
    return this.taskRepo.findOverdueTasks(projectId);
  }
}
```

不过要注意，TypeORM 0.3.x 版本推荐使用"扩展 Repository"的方式，而不是继承：

```typescript

@CustomRepository(Task)

  // 同上的方法
}

// src/tasks/decorators/custom-repository.decorator.ts

    // 注册自定义 Repository 的逻辑
  };
```

## Entity 到 DTO 的转换

直接把 Entity 返回给客户端不是好主意，可能会泄露内部字段（比如 deletedAt），或者包含不必要的关联数据。推荐用专门的响应 DTO。

```typescript
// src/tasks/dto/task-response.dto.ts
import { Task } from "../entities/task.entity";

export class TaskResponseDto {
  id: number;
  title: string;
  description: string | null;
  status: string;
  priority: string;
  dueDate: Date | null;
  assignee: {
    id: number;
    name: string;
    email: string;
  } | null;
  tags: Array<{
    id: number;
    name: string;
    color: string;
  }>;
  createdAt: Date;
  updatedAt: Date;

  static fromEntity(task: Task): TaskResponseDto {
    return {
      id: task.id,
      title: task.title,
      description: task.description,
      status: task.status,
      priority: task.priority,
      dueDate: task.dueDate,
      assignee: task.assignee
        ? {
            id: task.assignee.id,
            name: task.assignee.name,
            email: task.assignee.email,
          }
        : null,
      tags: task.tags.map((tag) => ({
        id: tag.id,
        name: tag.name,
        color: tag.color,
      })),
      createdAt: task.createdAt,
      updatedAt: task.updatedAt,
    };
  }
}
```

在 Service 中转换后再返回：

```typescript
// src/tasks/tasks.service.ts
async findOne(id: number): Promise<TaskResponseDto> {
  const task = await this.taskRepo.findOne({
    where: { id },
    relations: ['assignee', 'tags'],
  });

  if (!task) {
    throw new NotFoundException(`任务 #${id} 不存在`);
  }

  return TaskResponseDto.fromEntity(task);
}
```

或者用 `class-transformer` 的 `@Exclude()` 装饰器自动过滤字段：

```typescript
// src/tasks/entities/task.entity.ts
import { Entity, PrimaryGeneratedColumn, Column } from "typeorm";
import { Exclude } from "class-transformer";

@Entity("tasks")
export class Task {
  @PrimaryGeneratedColumn()
  id: number;

  @Column()
  title: string;

  @Exclude()
  @Column({ name: "internal_notes", nullable: true })
  internalNotes: string;
}
```

然后在 Controller 中用 `classToPlain` 转换：

```typescript
import { classToPlain } from 'class-transformer';

@Get(':id')
async findOne(@Param('id', ParseIntPipe) id: number) {
  const task = await this.tasksService.findOne(id);
  return classToPlain(task);
}
```

## 模块化设计

随着业务变复杂，单个模块会变得臃肿。可以按业务域拆分子模块：

```typescript
src/
├── tasks/
│   ├── tasks.module.ts
│   ├── tasks.controller.ts
│   ├── tasks.service.ts
│   ├── entities/
│   ├── dto/
│   └── repositories/
├── projects/
│   ├── projects.module.ts
│   ├── projects.controller.ts
│   └── ...
└── workspaces/
    ├── workspaces.module.ts
    ├── workspaces.controller.ts
    └── ...
```

模块之间通过 `imports` 和 `exports` 协作：

```typescript
// src/tasks/tasks.module.ts
@Module({
  imports: [ProjectsModule], // 如果需要访问 Project
  providers: [TasksService],
  exports: [TasksService], // 导出给其他模块使用
})
export class TasksModule {}
```

## Trade-off 分析

NestJS 和 TypeORM 的集成很自然，但也有一些设计选择需要权衡。

- **forFeature vs 手动注册**：`TypeOrmModule.forFeature()` 自动注册 Repository，简单但不够灵活；手动注册可以自定义 Repository，但需要额外代码。大多数场景用 forFeature 就够了。
- **Entity 直接返回 vs DTO 转换**：直接返回 Entity 简单快速，但可能泄露字段；DTO 转换更安全，但需要手写转换逻辑。可以用 `class-transformer` 简化这个过程。
- **全局 ValidationPipe vs 局部 Pipe**：全局 Pipe 统一验证，但可能影响所有接口；局部 Pipe 精细控制，但重复代码多。推荐全局启用，特殊情况用局部 Pipe 覆盖。
- **模块拆分粒度**：细粒度模块职责清晰，但模块间通信复杂；粗粒度模块简单，但容易变成大泥球。按业务域拆分通常是好的起点。

TypeORM 和 NestJS 的集成方案已经比较成熟，大部分场景照着官方文档做就行。关键是要理解背后的原理——依赖注入、模块系统、装饰器——这样遇到问题时才能快速定位。
