## 缓存模式对比

### Cache-Aside（旁路缓存）

```
读取：
1. 先查缓存
2. 缓存命中，返回
3. 缓存未命中，查数据库
4. 写入缓存
5. 返回

更新：
1. 更新数据库
2. 删除缓存
```

**适用场景**：读多写少

### Read-Through（读穿透）

```
读取：
1. 先查缓存
2. 缓存命中，返回
3. 缓存未命中，缓存层自动查数据库
4. 写入缓存
5. 返回
```

**适用场景**：统一数据访问层

### Write-Through（写穿透）

```
更新：
1. 先写缓存
2. 缓存层同步写数据库
3. 返回
```

**适用场景**：读写平衡，强一致性

### Write-Behind（异步写回）

```
更新：
1. 先写缓存
2. 异步写数据库
3. 返回
```

**适用场景**：写多读少，可接受短暂数据不一致

## Cache-Aside 实现

### 基础实现

```typescript
import Redis from "ioredis";

const redis = new Redis();

class CacheAside {
  constructor(
    private redis: Redis,
    private db: any,
    private defaultTTL: number = 3600
  ) {}

  // 读取数据
  async get(key: string) {
    // 1. 先查缓存
    const cached = await this.redis.get(key);
    if (cached) {
      console.log("缓存命中");
      return JSON.parse(cached);
    }

    // 2. 缓存未命中，查数据库
    console.log("缓存未命中");
    const data = await this.db.query(key);

    if (data) {
      // 3. 写入缓存
      await this.redis.setex(key, this.defaultTTL, JSON.stringify(data));
    }

    return data;
  }

  // 更新数据
  async set(key: string, data: any) {
    // 1. 更新数据库
    await this.db.update(key, data);

    // 2. 删除缓存
    await this.redis.del(key);
  }

  // 删除数据
  async delete(key: string) {
    // 1. 删除数据库
    await this.db.delete(key);

    // 2. 删除缓存
    await this.redis.del(key);
  }
}
```

### 带版本控制的缓存

```typescript
class VersionedCache {
  constructor(
    private redis: Redis,
    private db: any
  ) {}

  async get(key: string) {
    // 获取数据版本
    const version = await this.db.getVersion(key);
    const cacheKey = `${key}:${version}`;

    const cached = await this.redis.get(cacheKey);
    if (cached) {
      return JSON.parse(cached);
    }

    const data = await this.db.query(key);
    if (data) {
      await this.redis.setex(cacheKey, 3600, JSON.stringify(data));
    }

    return data;
  }

  async set(key: string, data: any) {
    // 更新数据，版本号自动增加
    await this.db.update(key, data);

    // 旧版本缓存会自动失效
  }
}
```

### 预热缓存

```typescript
async function warmupCache(keys: string[]) {
  for (const key of keys) {
    const data = await db.query(key);

      await redis.setex(key, 3600, JSON.stringify(data));
```

## 缓存更新策略

### 先更新数据库，后删除缓存

```typescript
async function updateData(key: string, data: any) {
  // 1. 更新数据库
  await db.update(key, data);

  // 2. 删除缓存
  await redis.del(key);
}
```

**问题**：可能删除失败导致数据不一致

### 延迟双删

```typescript
async function updateDataWithDelay(key: string, data: any) {
  // 1. 删除缓存
  await redis.del(key);

  // 2. 更新数据库
  await db.update(key, data);

  // 3. 延迟后再删除缓存
  setTimeout(async () => {
    await redis.del(key);
  }, 500);
}
```

### 使用消息队列保证最终一致性

```typescript
async function updateData(key: string, data: any) {
  // 1. 更新数据库
  await db.update(key, data);

  // 2. 发送删除缓存消息
  await queue.publish("cache:invalidate", { key });
}

// 消费者
queue.subscribe("cache:invalidate", async ({ key }) => {
  await redis.del(key);
});
```

## 最佳实践

### 1. 设置合理的过期时间

```typescript
// 热点数据：短过期时间
await redis.setex("hot:data", 60, data);

// 普通数据：中等过期时间
await redis.setex("normal:data", 3600, data);

// 冷数据：长过期时间
await redis.setex("cold:data", 86400, data);
```

### 2. 避免缓存穿透

```typescript
async function getWithNullCache(key: string) {
  let data = await redis.get(key);
  if (data) {
    if (data === "NULL") {
      return null;
    }
    return JSON.parse(data);
  }

  data = await db.query(key);

  if (!data) {
    // 缓存空值，防止缓存穿透
    await redis.setex(key, 60, "NULL");
    return null;
  }

  await redis.setex(key, 3600, JSON.stringify(data));
  return data;
}
```

### 3. 使用互斥锁防止缓存击穿

```typescript
async function getWithLock(key: string) {
  let data = await redis.get(key);
  if (data) {
    return JSON.parse(data);
  }

  const lockKey = `lock:${key}`;
  const lock = await redis.set(lockKey, "1", "NX", "EX", 10);

  if (lock) {
    try {
      data = await db.query(key);
      await redis.setex(key, 3600, JSON.stringify(data));
      return data;
    } finally {
      await redis.del(lockKey);
    }
  }

  // 等待并重试
  await new Promise((resolve) => setTimeout(resolve, 100));
  return await getWithLock(key);
}
```

## 常见陷阱

### 1. 缓存雪崩

**问题**：大量缓存同时过期

**解决**：

```typescript
// 过期时间加随机值
const ttl = 3600 + Math.random() * 600;
await redis.setex(key, ttl, data);
```

### 2. 缓存击穿

**问题**：热点数据过期时大量请求打到数据库

**解决**：使用互斥锁（见上文）

### 3. 缓存穿透

**问题**：查询不存在的数据

**解决**：缓存空值或使用布隆过滤器
