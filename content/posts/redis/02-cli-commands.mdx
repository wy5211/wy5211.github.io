## redis-cli 高级用法

### 基础连接

```bash
# 连接本地 Redis（默认 localhost:6379）
redis-cli

# 连接指定主机和端口
redis-cli -h 127.0.0.1 -p 6379

# 连接带密码的 Redis
redis-cli -a your_password

# 连接指定数据库（Redis 有 16 个数据库，编号 0-15）
redis-cli -n 2

# 组合使用
redis-cli -h 192.168.1.100 -p 6379 -a password -n 1
```

### 交互式 vs 非交互式模式

**交互式模式**：

```bash
$ redis-cli
127.0.0.1:6379> PING
PONG
127.0.0.1:6379> SET name "张三"
OK
127.0.0.1:6379> GET name
"张三"
127.0.0.1:6379> KEYS user:*
1) "user:1"
2) "user:2"
```

**非交互式模式**：

```bash
# 直接执行命令
redis-cli PING
# 输出：PONG

redis-cli SET name "张三"
# 输出：OK

redis-cli GET name
# 输出："张三"

# 执行多个命令
redis-cli SET name "张三" GET name
# 输出：
# OK
# "张三"
```

### 批量执行命令

**从文件执行**：

```bash
# 创建命令文件
cat > commands.txt << EOF
SET key1 "value1"
SET key2 "value2"
GET key1
GET key2
EOF

# 执行文件中的命令
redis-cli < commands.txt

# 或使用 --pipe 选项（更高效）
redis-cli --pipe < commands.txt
```

### 常用选项

```bash
# 输出原始格式（不格式化）
redis-cli --raw GET name

# 显示请求/响应时间
redis-cli --latency GET name

# 返回 Unix 时间戳格式
redis-cli --csv KEYS "*"

# 模拟从服务器
redis-cli --slave

# 发送多个命令（pipeline）
redis-cli -n 0 INFO KEYS "*" GET key
```

## String 类型专用命令

### 基础命令

```bash
# 设置和获取
SET key value
GET key
MSET key1 value1 key2 value2
MGET key1 key2

# 设置过期时间
SETEX key 3600 value    # 设置值并指定过期时间（秒）
SET key value EX 3600   # 另一种写法

# 仅当键不存在时设置
SETNX key value         # 返回 1 表示成功，0 表示失败
```

### 数值操作

```bash
# 初始化数值
SET count 0

# 自增 1
INCR count
# 返回：(integer) 1

# 自增指定值
INCRBY count 5
# 返回：(integer) 6

# 自减 1
DECR count
# 返回：(integer) 5

# 自减指定值
DECRBY count 2
# 返回：(integer) 3

# 浮点数自增
SET score 95.5
INCRBYFLOAT score 0.5
# 返回："96"

# INCRBYFLOAT 也支持负数
INCRBYFLOAT score -2.5
# 返回："93.5"
```

### 字符串操作

```bash
# 追加字符串
SET greeting "Hello"
APPEND greeting " World"
GET greeting
# 返回："Hello World"

# 获取字符串长度
STRLEN greeting
# 返回：(integer) 11

# 获取子字符串（闭区间）
SET msg "Hello Redis"
GETRANGE msg 0 4
# 返回："Hello"

GETRANGE msg 6 -1
# 返回："Redis"

# 设置子字符串
SETRANGE msg 6 "World"
GET msg
# 返回："Hello World"

# 同时设置和获取旧值
SET counter 100
GETSET counter 200
# 返回："100"（旧值）
GET counter
# 返回："200"（新值）
```

### 实际应用示例

**场景 1：计数器**

```bash
# 文章浏览次数
INCR article:123:views
# 返回：(integer) 1

INCR article:123:views
# 返回：(integer) 2

# 查看浏览次数
GET article:123:views
# 返回："2"

# 一次增加多次
INCRBY article:123:views 10
# 返回：(integer) 12
```

**场景 2：分布式锁**

```bash
# 尝试获取锁（30 秒过期）
SET lock:order:123 "unique_id" NX EX 30
# 返回：OK（成功）或 nil（失败）

# 释放锁
GET lock:order:123
# 返回："unique_id"

# 确认是自己的锁后再删除
EVAL "if redis.call('get', KEYS[1]) == ARGV[1] then return redis.call('del', KEYS[1]) else return 0 end" 1 lock:order:123 unique_id
```

**场景 3：限流**

```bash
# 每分钟限制 100 次请求
SET user:123:limit 100 EX 60

# 每次请求减 1
DECR user:123:limit
# 返回：(integer) 99

# 检查是否超过限制
GET user:123:limit
# 返回："99"

# 当值为 0 时拒绝请求
```

## Hash 类型专用命令

### 基础操作

```bash
# 设置单个字段
HSET user:1 name "张三"
HSET user:1 age 25
HSET user:1 email "zhangsan@example.com"

# 获取单个字段
HGET user:1 name
# 返回："张三"

# 获取多个字段
HMGET user:1 name age email
# 返回：
# 1) "张三"
# 2) "25"
# 3) "zhangsan@example.com"

# 获取所有字段和值
HGETALL user:1
# 返回：
# 1) "name"
# 2) "张三"
# 3) "age"
# 4) "25"
# 5) "email"
# 6) "zhangsan@example.com"

# 获取所有字段名
HKEYS user:1
# 返回：
# 1) "name"
# 2) "age"
# 3) "email"

# 获取所有值
HVALS user:1
# 返回：
# 1) "张三"
# 2) "25"
# 3) "zhangsan@example.com"
```

### 字段操作

```bash
# 检查字段是否存在
HEXISTS user:1 name
# 返回：(integer) 1

# 删除字段
HDEL user:1 email
# 返回：(integer) 1

# 获取字段数量
HLEN user:1
# 返回：(integer) 2

# 仅当字段不存在时设置
HSETNX user:1 city "北京"
# 返回：(integer) 1（成功）
HSETNX user:1 city "上海"
# 返回：(integer) 0（失败，已存在）
```

### 数值操作

```bash
# 数值自增
HSET user:1 score 80
HINCRBY user:1 score 5
# 返回：(integer) 85

HINCRBY user:1 score -10
# 返回：(integer) 75

# 浮点数自增
HINCRBYFLOAT user:1 score 2.5
# 返回："77.5"
```

### 实际应用示例

**场景：购物车**

```bash
# 添加商品到购物车
HSET cart:user:123 product:1 2    # 商品 ID 1，数量 2
HSET cart:user:123 product:5 1    # 商品 ID 5，数量 1

# 查看购物车
HGETALL cart:user:123
# 返回：
# 1) "product:1"
# 2) "2"
# 3) "product:5"
# 4) "1"

# 修改商品数量
HINCRBY cart:user:123 product:1 1
# 返回：(integer) 3

# 删除商品
HDEL cart:user:123 product:5
# 返回：(integer) 1

# 获取购物车商品数量
HLEN cart:user:123
# 返回：(integer) 1
```

## List 类型专用命令

### 基础操作

```bash
# 从左侧插入（头插）
LPUSH list "item1"
LPUSH list "item2"
LPUSH list "item3"

# 从右侧插入（尾插）
RPUSH list "item4"

# 查看列表所有元素（0 表示第一个，-1 表示最后一个）
LRANGE list 0 -1
# 返回：
# 1) "item3"
# 2) "item2"
# 3) "item1"
# 4) "item4"

# 获取列表长度
LLEN list
# 返回：(integer) 4
```

### 弹出操作

```bash
# 从左侧弹出（队列）
LPOP list
# 返回："item3"

# 从右侧弹出（栈）
RPOP list
# 返回："item4"

# 阻塞式弹出（如果没有元素会等待）
BLPOP queue 0      # 永久阻塞
BRPOP queue 5      # 阻塞 5 秒
```

### 索引操作

```bash
# 获取指定索引的元素
LINDEX list 0
# 返回："item2"

# 设置指定索引的元素
LSET list 0 "new_item2"

# 保留指定范围的元素
LTRIM list 0 1     # 只保留前 2 个元素
```

### 实际应用示例

**场景 1：消息队列（FIFO）**

```bash
# 生产者：推送消息
RPUSH queue:email "message1"
RPUSH queue:email "message2"
RPUSH queue:email "message3"

# 消费者：弹出消息
LPOP queue:email
# 返回："message1"

LPOP queue:email
# 返回："message2"

# 阻塞式消费
BLPOP queue:email 0
# 如果队列空，会阻塞等待
```

**场景 2：栈（LIFO）**

```bash
# 压栈
LPUSH stack "item1"
LPUSH stack "item2"

# 出栈
LPOP stack
# 返回："item2"
```

**场景 3：最新列表**

```bash
# 添加最新文章
LPUSH latest:articles "article:123"
LPUSH latest:articles "article:124"

# 只保留最新的 10 篇
LTRIM latest:articles 0 9

# 获取最新文章列表
LRANGE latest:articles 0 4
```

## Set 类型专用命令

### 基础操作

```bash
# 添加元素
SADD set1 "member1"
SADD set1 "member2"
SADD set1 "member3"
SADD set1 "member2"   # 重复添加会被忽略

# 获取所有元素
SMEMBERS set1
# 返回：
# 1) "member1"
# 2) "member2"
# 3) "member3"

# 检查元素是否存在
SISMEMBER set1 "member2"
# 返回：(integer) 1

# 获取集合元素数量
SCARD set1
# 返回：(integer) 3
```

### 集合运算

```bash
# 创建第二个集合
SADD set2 "member2"
SADD set2 "member3"
SADD set2 "member4"

# 交集
SINTER set1 set2
# 返回：
# 1) "member2"
# 2) "member3"

# 并集
SUNION set1 set2
# 返回：
# 1) "member1"
# 2) "member2"
# 3) "member3"
# 4) "member4"

# 差集（set1 中有但 set2 中没有的）
SDIFF set1 set2
# 返回：
# 1) "member1"
```

### 弹出和移动

```bash
# 随机弹出一个元素
SPOP set1
# 返回："member3"（随机）

# 随机获取但不删除
SRANDMEMBER set1
# 返回："member2"（不删除）

# 将元素从一个集合移动到另一个集合
SMOVE set1 set2 "member1"
# 返回：(integer) 1
```

### 实际应用示例

**场景：共同好友**

```bash
# 用户 1 的好友
SADD user:1:friends "user:2" "user:3" "user:4"

# 用户 2 的好友
SADD user:2:friends "user:3" "user:4" "user:5"

# 找出共同好友
SINTER user:1:friends user:2:friends
# 返回：
# 1) "user:3"
# 2) "user:4"

# 用户 2 独有的好友
SDIFF user:2:friends user:1:friends
# 返回：
# 1) "user:5"
```

## Sorted Set 类型专用命令

### 基础操作

```bash
# 添加元素（分数在前，元素在后）
ZADD leaderboard 100 "player1"
ZADD leaderboard 85 "player2"
ZADD leaderboard 95 "player3"

# 获取所有元素（按分数从低到高）
ZRANGE leaderboard 0 -1
# 返回：
# 1) "player2"
# 2) "player3"
# 3) "player1"

# 获取所有元素（按分数从高到低）
ZREVRANGE leaderboard 0 -1
# 返回：
# 1) "player1"
# 2) "player3"
# 3) "player2"

# 获取带分数的元素
ZRANGE leaderboard 0 -1 WITHSCORES
# 返回：
# 1) "player2"
# 2) "85"
# 3) "player3"
# 4) "95"
# 5) "player1"
# 6) "100"
```

### 分数操作

```bash
# 获取元素分数
ZSCORE leaderboard player1
# 返回："100"

# 增加元素分数
ZINCRBY leaderboard 5 player2
# 返回："90"

# 获取指定分数范围的元素
ZRANGEBYSCORE leaderboard 90 100
# 返回：
# 1) "player2"
# 2) "player3"
# 3) "player1"

# 获取排名（从 0 开始）
ZRANK leaderboard player1
# 返回：(integer) 2（第 3 名）

ZREVRANK leaderboard player1
# 返回：(integer) 0（倒数第 1 名）
```

### 统计操作

```bash
# 获取元素数量
ZCARD leaderboard
# 返回：(integer) 3

# 获取指定分数范围内的元素数量
ZCOUNT leaderboard 90 100
# 返回：(integer) 3
```

### 实际应用示例

**场景：排行榜 Top 10**

```bash
# 更新玩家分数
ZADD game:scores 1500 "player1"
ZADD game:scores 2000 "player2"
ZADD game:scores 1800 "player3"

# 获取 Top 10
ZREVRANGE game:scores 0 9 WITHSCORES
# 返回：
# 1) "player2"
# 2) "2000"
# 3) "player3"
# 4) "1800"
# 5) "player1"
# 6) "1500"

# 获取玩家排名
ZREVRANK game:scores player1
# 返回：(integer) 2（第 3 名）
```

**场景：延时队列**

```bash
# 添加任务（时间戳作为分数）
ZADD delay:queue 1672531200 "task1"  # 2023-01-01 00:00:00
ZADD delay:queue 1672534800 "task2"  # 2023-01-01 01:00:00

# 获取到期的任务
ZRANGEBYSCORE delay:queue 0 1672531200
# 返回：
# 1) "task1"

# 删除已处理的任务
ZREM delay:queue task1
```

## 服务器信息与监控

### INFO 命令

```bash
# 获取所有信息
INFO

# 获取特定部分信息
INFO server      # 服务器信息
INFO memory      # 内存信息
INFO persistence # 持久化信息
INFO stats       # 统计信息
INFO replication # 复制信息
INFO cpu         # CPU 信息

# 常用信息查看
redis-cli INFO memory | grep used_memory_human
# used_memory_human:256.00M

redis-cli INFO stats | grep instantaneous_ops_per_sec
# instantaneous_ops_per_sec:120
```

### 实时监控

```bash
# 实时显示 Redis 状态
redis-cli --stat

# 输出示例：
# ------- data ------ --------------------- load -------------------- - child -
# keys       mem      clients blocked requests            connections
# 100000     256.00M  2       0       120 ( ops/s)          1000
```

### 慢查询

```bash
# 获取慢查询（最近 10 条）

# 输出示例：
```

## 实用技巧

### 批量删除键

```bash
# 删除匹配的所有键
redis-cli KEYS "user:*" | xargs redis-cli DEL

# 使用 scan 更安全（不会阻塞）
redis-cli --scan --pattern "user:*" | xargs redis-cli DEL
```

### 导出数据

```bash
# 导出所有键值
redis-cli --raw dump.rdb

# 导出匹配的键
redis-cli --scan --pattern "user:*" > keys.txt
```

### 监控命令

```bash
# 监控所有执行的命令
redis-cli MONITOR

# 输出：
# 1672531200.123456 [0 127.0.0.1:50001] "GET" "name"
# 1672531201.234567 [0 127.0.0.1:50001] "SET" "age" "25"
```

## 最佳实践

### 1. 生产环境避免使用 KEYS

```bash
# ❌ 可能导致 Redis 阻塞
redis-cli KEYS "*"

# ✅ 使用 SCAN
redis-cli SCAN 0 MATCH "*" COUNT 100
```

### 2. 使用 Pipeline 减少网络往返

```bash
# ❌ 多次网络请求
redis-cli SET key1 value1
redis-cli SET key2 value2
redis-cli SET key3 value3

# ✅ 使用 Pipeline
echo -e "SET key1 value1\nSET key2 value2\nSET key3 value3" | redis-cli --pipe
```

### 3. 批量操作使用专用命令

```bash
# ❌ 多次调用
redis-cli GET key1
redis-cli GET key2
redis-cli GET key3

# ✅ 使用 MGET
redis-cli MGET key1 key2 key3
```

## 常见问题

### Q1: KEYS 命令为什么不推荐在生产环境使用？

**A**: KEYS 命令会遍历所有键，在键很多时会阻塞 Redis。使用 SCAN 命令替代：

```bash
# SCAN 不会阻塞，分批返回
SCAN 0 MATCH "user:*" COUNT 100
```

### Q2: 如何查看 Redis 占用多少内存？

**A**:

```bash
redis-cli INFO memory | grep used_memory_human
```

### Q3: 如何清空 Redis 数据？

**A**:

```bash
# 清空当前数据库
FLUSHDB

# 清空所有数据库
FLUSHALL
```
