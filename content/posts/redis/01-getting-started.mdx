## 前言

在现代 Web 应用中，性能是核心竞争力。你是否遇到过这样的问题：

- 数据库查询缓慢，影响用户体验？

**Redis** 就是为解决这些问题而生的！

Redis 是一个开源的内存数据结构存储系统，可以用作数据库、缓存和消息代理。它以其**极高的性能**（读速度可达 110000次/s，写速度可达 81000次/s）和**丰富的数据结构**而闻名。

## 为什么选择 Redis？

### Redis 核心优势

**1. 极致性能**

Redis 基于内存操作，所有数据都存储在内存中，避免了磁盘 I/O 的开销：

```
性能对比（QPS - 每秒查询次数）：

MySQL:      1,000 - 5,000   QPS
PostgreSQL: 5,000 - 10,000  QPS
MongoDB:    10,000 - 20,000 QPS
Redis:      100,000+        QPS ⚡
```

**2. 丰富的数据结构**

Redis 不仅支持简单的键值存储，还提供了多种数据结构：

```bash
String    - 字符串、整数、浮点数
Hash      - 键值对集合（适合对象存储）
List      - 双向链表（队列、栈）
Set       - 无序集合（去重、交集）
Sorted Set - 有序集合（排行榜、范围查询）
```

**3. 功能强大**

- ✅ **持久化**：支持 RDB 和 AOF 两种持久化方式
- ✅ **发布订阅**：实现消息广播
- ✅ **事务**：保证一组命令的原子性
- ✅ **Lua 脚本**：支持复杂逻辑
- ✅ **集群**：支持主从复制、Sentinel、Cluster

**4. 简单易用**

Redis 的 API 设计简洁直观，学习成本低：

```bash
# 设置值
SET user:1 "张三"

# 获取值
GET user:1

# 设置过期时间
EXPIRE user:1 3600
```

### Redis 应用场景

| 场景         | 使用的数据结构 | 实现方式          |
| ------------ | -------------- | ----------------- |
| **缓存**     | String         | Cache-Aside 模式  |
| **会话存储** | String         | 存储 Session 数据 |
| **计数器**   | String         | INCR/DECR 命令    |
| **排行榜**   | Sorted Set     | ZADD/ZRANGE 命令  |
| **消息队列** | List           | LPUSH/BRPOP 命令  |
| **关注系统** | Set            | SADD/SINTER 命令  |
| **购物车**   | Hash           | HSET/HGETALL 命令 |
| **分布式锁** | String         | SET NX EX 命令    |

## 环境搭建

### 方式一：使用 Docker（推荐）

Docker 是最简单快捷的方式：

```bash
# 拉取 Redis 镜像
docker pull redis:7-alpine

# 启动 Redis 容器
docker run -d \
  --name redis \
  -p 6379:6379 \
  redis:7-alpine

# 启动带持久化的 Redis
docker run -d \
  --name redis \
  -p 6379:6379 \
  -v redis-data:/data \
  redis:7-alpine \
  redis-server --appendonly yes

# 查看 Redis 日志
docker logs -f redis
```

**参数说明**：

- `-d`：后台运行
- `--name redis`：容器名称
- `-p 6379:6379`：端口映射（主机:容器）
- `-v redis-data:/data`：数据卷挂载
- `--appendonly yes`：启用 AOF 持久化

### 方式二：使用 Homebrew（macOS）

```bash
# 安装 Redis
brew install redis

# 启动 Redis 服务
brew services start redis

# 停止 Redis 服务
brew services stop redis

# 重启 Redis 服务
brew services restart redis
```

### 方式三：使用 apt/yum（Linux）

```bash
# Ubuntu/Debian
sudo apt update
sudo apt install redis-server

# CentOS/RHEL
sudo yum install redis

# 启动 Redis
sudo systemctl start redis
sudo systemctl enable redis
```

### 方式四：使用 Windows

Windows 用户推荐使用 Docker 或 WSL：

```bash
# 在 WSL 中安装
sudo apt update
sudo apt install redis-server
```

或者下载 Redis for Windows（非官方）。

## Redis 基本命令

### 1. 键值操作

```bash
# 设置键值
SET name "张三"
SET age 25

# 获取值
GET name    # "张三"
GET age     # "25"

# 设置键值并指定过期时间（秒）
SETEX session:abc 3600 "user_data"

# 仅当键不存在时设置
SETNX lock:resource 1  # 返回 1 表示成功，0 表示失败

# 批量设置
MSET key1 "value1" key2 "value2" key3 "value3"

# 批量获取
MGET key1 key2 key3
# 1) "value1"
# 2) "value2"
# 3) "value3"
```

### 2. 键操作

```bash
# 查看所有键
KEYS *

# 查看匹配的键
KEYS user:*

# 检查键是否存在
EXISTS name  # 1 表示存在，0 表示不存在

# 查看键的类型
TYPE name    # string

# 重命名键
RENAME old_key new_key

# 删除键
DEL key1 key2 key3

# 设置过期时间
EXPIRE name 60      # 60 秒后过期
EXPIREAT name 1672531200  # 在指定时间戳过期

# 查看剩余过期时间（秒）
TTL name           # 返回剩余秒数，-1 表示永不过期，-2 表示不存在

# 查看键的内存占用
MEMORY USAGE name
```

### 3. 数值操作

```bash
# 设置数值
SET count 0

# 自增 1
INCR count         # 1

# 自增指定值
INCRBY count 5     # 6

# 自减 1
DECR count         # 5

# 自减指定值
DECRBY count 2     # 3

# 浮点数自增
INCRBYFLOAT score 0.5
```

## Node.js 客户端使用

### 安装 ioredis

```bash
npm install ioredis
```

### 基本使用

```typescript
// redis-client.ts
import Redis from "ioredis";

// 创建 Redis 客户端
const redis = new Redis({
  host: "localhost",
  port: 6379,
  // password: "your_password", // 如果设置了密码
});

async function basicOperations() {
  // 1. 设置值
  await redis.set("name", "张三");
  console.log("设置 name = 张三");

  // 2. 获取值
  const name = await redis.get("name");
  console.log("获取 name:", name); // 张三

  // 3. 设置过期时间
  await redis.setex("session:123", 3600, "session_data");
  console.log("设置 session，过期时间 3600 秒");

  // 4. 检查键是否存在
  const exists = await redis.exists("name");
  console.log("name 是否存在:", exists === 1); // true

  // 5. 删除键
  await redis.del("name");
  console.log("删除 name");

  // 6. 数值操作
  await redis.set("count", 0);
  await redis.incr("count"); // 1
  await redis.incrby("count", 5); // 6
  const count = await redis.get("count");
  console.log("count:", count); // 6

  // 7. 批量操作
  await redis.mset("key1", "value1", "key2", "value2");
  const values = await redis.mget("key1", "key2");
  console.log("批量获取:", values); // ["value1", "value2"]

  // 8. 查看剩余过期时间
  const ttl = await redis.ttl("session:123");
  console.log("session:123 剩余时间:", ttl, "秒");

  // 9. 查看所有匹配的键
  const keys = await redis.keys("*");
  console.log("所有键:", keys);

  // 关闭连接
  await redis.quit();
}

basicOperations().catch(console.error);
```

### 实际应用示例：缓存用户信息

```typescript
// cache-example.ts
import Redis from "ioredis";

const redis = new Redis();

// 模拟数据库查询
async function getUserFromDB(userId: string) {
  // 模拟数据库查询耗时
  await new Promise((resolve) => setTimeout(resolve, 100));

  return {
    id: userId,
    name: "张三",
    email: "zhangsan@example.com",
    age: 25,
  };
}

// 使用缓存获取用户信息
async function getUser(userId: string) {
  const cacheKey = `user:${userId}`;

  // 1. 先查缓存
  const cached = await redis.get(cacheKey);
  if (cached) {
    console.log("从缓存获取用户");
    return JSON.parse(cached);
  }

  // 2. 缓存未命中，查询数据库
  console.log("从数据库获取用户");
  const user = await getUserFromDB(userId);

  // 3. 写入缓存，过期时间 1 小时
  await redis.setex(cacheKey, 3600, JSON.stringify(user));

  return user;
}

// 使用示例
async function main() {
  // 第一次查询：从数据库获取
  const user1 = await getUser("123");
  console.log(user1);

  // 第二次查询：从缓存获取
  const user2 = await getUser("123");
  console.log(user2);

  await redis.quit();
}

main().catch(console.error);
```

## Redis 配置基础

### 配置文件位置

```bash
# Docker 挂载配置文件
docker run -d \
  --name redis \
  -p 6379:6379 \
  -v /path/to/redis.conf:/usr/local/etc/redis/redis.conf \
  redis:7-alpine \
  redis-server /usr/local/etc/redis/redis.conf
```

### 常用配置项

```bash
# redis.conf

# 绑定地址（默认只允许本地访问）
bind 127.0.0.1

# 端口
port 6379

# 密码
requirepass your_password

# 数据库数量（默认 16 个）
databases 16

# RDB 持久化配置
save 900 1     # 900 秒内至少 1 个 key 变化
save 300 10    # 300 秒内至少 10 个 key 变化
save 60 10000  # 60 秒内至少 10000 个 key 变化

# AOF 持久化
appendonly yes
appendfsync everysec

# 最大内存
maxmemory 256mb

# 内存淘汰策略
maxmemory-policy allkeys-lru
```

### 内存淘汰策略

| 策略              | 说明                                  |
| ----------------- | ------------------------------------- |
| `noeviction`      | 不淘汰，内存满时返回错误              |
| `allkeys-lru`     | 淘汰最少使用的 key                    |
| `allkeys-lfu`     | 淘汰使用频率最低的 key                |
| `volatile-lru`    | 淘汰设置了过期时间的 key 中最少使用的 |
| `volatile-ttl`    | 淘汰即将过期的 key                    |
| `allkeys-random`  | 随机淘汰                              |
| `volatile-random` | 随机淘汰设置了过期时间的 key          |

## 最佳实践

### 1. 键命名规范

```typescript
// ✅ 好的命名
"user:123:profile";
"session:abc123";
"cache:product:456";
"lock:order:789";

// ❌ 避免的命名
"user123profile";
"User123Profile";
"key_1_2_3";
```

**命名规则**：

- 使用冒号 `:` 分隔层级
- 使用小写字母和数字
- 使用有意义的名称
- 保持一致性

### 2. 设置合理的过期时间

```typescript
// ❌ 永不过期（可能导致内存泄漏）
await redis.set("user:123", data);

// ✅ 设置过期时间
await redis.setex("user:123", 3600, data); // 1 小时
await redis.setex("user:123", 60, data); // 1 分钟（频繁变化的数据）
await redis.setex("user:123", 86400, data); // 1 天（不常变化的数据）
```

### 3. 使用连接池

```typescript
// ✅ 复用 Redis 连接
import Redis from "ioredis";

// 创建单例
const redis = new Redis({
  host: "localhost",
  port: 6379,
  maxRetriesPerRequest: 3,
  enableReadyCheck: true,
  lazyConnect: false,
});

export default redis;
```

### 4. 错误处理

```typescript
async function safeGet(key: string) {
  try {
    const value = await redis.get(key);
    return value;
  } catch (error) {
    console.error("Redis 获取失败:", error);
    // 降级到数据库或其他存储
    return null;
  }
}
```

### 5. 批量操作

```typescript
// ❌ 多次网络往返
for (const id of userIds) {
  const user = await redis.get(`user:${id}`);
}

// ✅ 使用 pipeline 或 mget
const pipeline = redis.pipeline();
for (const id of userIds) {
  pipeline.get(`user:${id}`);
}
const results = await pipeline.exec();
```

## 常见问题

### Q1: Redis 和 Memcached 有什么区别？

**A**:

| 特性     | Redis         | Memcached           |
| -------- | ------------- | ------------------- |
| 数据类型 | 丰富（5+ 种） | 仅 String           |
| 持久化   | 支持          | 不支持              |
| 集群     | 支持          | 不支持              |
| 单线程   | 是            | 是（多线程在 3.0+） |
| 内存使用 | 更高          | 更低                |

**选择建议**：

- 需要持久化 → Redis
- 需要复杂数据结构 → Redis
- 简单缓存 → 都可以
- 纯内存缓存 → Memcached

### Q2: Redis 是单线程的吗？

**A**: Redis 的网络请求和命令处理是单线程的，但：

- 现代版本支持多线程 I/O
- 持久化（RDB/AOF）使用子进程
- 单线程避免了锁竞争，性能反而更好

### Q3: Redis 数据会丢失吗？

**A**: 取决于持久化配置：

- **不持久化**：重启后数据丢失
- **RDB**：根据保存策略，可能丢失最近几分钟数据
- **AOF**：配置 `appendfsync always` 可以不丢数据，但性能较差
- **混合持久化**：RDB + AOF，推荐配置

### Q4: 如何监控 Redis 性能？

**A**: 使用以下命令：

```bash
# 查看 Redis 信息
redis-cli info

# 查看内存使用
redis-cli info memory

# 查看统计信息
redis-cli info stats

# 实时监控
redis-cli --stat

# 慢查询
redis-cli slowlog get 10
```
