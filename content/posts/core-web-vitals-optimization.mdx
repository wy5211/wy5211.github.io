# 前端性能优化实战：Core Web Vitals 优化指南

Google 在 2020 年推出了 Core Web Vitals（核心网页指标），作为衡量网页用户体验的重要标准。这三个指标直接影响 SEO 排名和用户留存。本文将深入解析每个指标，并提供可落地的优化方案。

## Core Web Vitals 三大指标

| 指标 | 全称 | 含义 | 良好标准 |
|------|------|------|----------|
| **LCP** | Largest Contentful Paint | 最大内容绘制时间 | ≤ 2.5s |
| **FID** | First Input Delay | 首次输入延迟 | ≤ 100ms |
| **CLS** | Cumulative Layout Shift | 累积布局偏移 | ≤ 0.1 |

### 1. LCP（最大内容绘制）

LCP 测量页面主要内容加载完成的时间。通常指的是：
- `<img>` 元素
- `<video>` 元素
- 块级元素内的文本节点
- 通过 `background-image` 加载的图片

### 2. FID（首次输入延迟）

FID 测量用户首次与页面交互（点击链接、按钮等）到浏览器响应的时间。反映了页面的交互响应能力。

> 注意：FID 已被 INP（Interaction to Next Paint）取代，但优化思路类似。

### 3. CLS（累积布局偏移）

CLS 测量页面视觉稳定性。当元素在渲染过程中发生位置变化，就会产生布局偏移。

## 一、LCP 优化实战

### 1.1 图片优化

图片通常是 LCP 元素，优化图片是提升 LCP 的关键。

#### 使用现代图片格式

```html
<picture>
  <source srcset="hero.avif" type="image/avif" />
  <source srcset="hero.webp" type="image/webp" />
  <img src="hero.jpg" alt="Hero image" loading="eager" fetchpriority="high" />
</picture>
```

#### 响应式图片

```html
<img
  srcset="image-320w.jpg 320w,
          image-640w.jpg 640w,
          image-1280w.jpg 1280w"
  sizes="(max-width: 640px) 100vw, 50vw"
  src="image-640w.jpg"
  alt="Responsive image"
/>
```

#### 图片懒加载（非关键图片）

```html
<img src="image.jpg" alt="..." loading="lazy" />
```

#### 预加载关键图片

```html
<link rel="preload" as="image" href="hero.jpg" />
```

### 1.2 字体优化

自定义字体加载会阻塞文本渲染，导致 LCP 延迟。

#### 使用 font-display

```css
@font-face {
  font-family: "CustomFont";
  src: url("/fonts/custom.woff2") format("woff2");
  font-display: swap; /* 关键：立即显示后备字体 */
}
```

#### 预加载关键字体

```html
<link rel="preload" href="/fonts/custom.woff2" as="font" type="font/woff2" crossorigin />
```

### 1.3 减少渲染阻塞资源

#### 内联关键 CSS

```html
<head>
  <style>
    /* 首屏关键 CSS 直接内联 */
    .header { height: 60px; background: #fff; }
    .hero { min-height: 400px; }
  </style>
  <link rel="preload" href="/styles/main.css" as="style" onload="this.onload=null;this.rel='stylesheet'">
</head>
```

#### 异步加载 JavaScript

```html
<script src="analytics.js" async></script>
<script src="non-critical.js" defer></script>
```

## 二、FID/INP 优化实战

### 2.1 减少 JavaScript 执行时间

#### 代码分割

```javascript
// 之前：一次性加载所有代码
import { heavyModule } from './heavy-module'

// 之后：按需加载
button.addEventListener('click', async () => {
  const { heavyModule } = await import('./heavy-module')
  heavyModule.run()
})
```

#### 使用 Webpack 代码分割

```javascript
// webpack.config.js
module.exports = {
  optimization: {
    splitChunks: {
      chunks: 'all',
      cacheGroups: {
        vendor: {
          test: /[\\/]node_modules[\\/]/,
          name: 'vendors',
          chunks: 'all',
        },
      },
    },
  },
}
```

### 2.2 优化长任务

任何超过 50ms 的任务都会阻塞主线程。

#### 使用 requestIdleCallback

```javascript
// 低优先级任务放在空闲时间执行
requestIdleCallback(() => {
  // 发送分析数据、预加载非关键资源等
  sendAnalytics()
})
```

#### 拆分长任务

```javascript
// 之前：一次性处理大量数据
function processLargeArray(array) {
  array.forEach(item => {
    // 处理每个 item
  })
}

// 之后：分批处理
async function processLargeArray(array, batchSize = 100) {
  for (let i = 0; i < array.length; i += batchSize) {
    const batch = array.slice(i, i + batchSize)
    batch.forEach(item => {
      // 处理每个 item
    })
    // 让出主线程
    await new Promise(resolve => setTimeout(resolve, 0))
  }
}
```

### 2.3 使用 Web Worker

将 CPU 密集型任务移到 Web Worker 中。

```javascript
// main.js
const worker = new Worker('worker.js')

worker.postMessage({ type: 'PROCESS_DATA', data: largeData })

worker.onmessage = (e) => {
  const result = e.data
  // 处理结果
}

// worker.js
self.onmessage = (e) => {
  if (e.data.type === 'PROCESS_DATA') {
    const result = heavyComputation(e.data.data)
    self.postMessage(result)
  }
}
```

## 三、CLS 优化实战

### 3.1 为图片和媒体设置尺寸

```html
<!-- 坏做法：没有尺寸，加载时会跳动 -->
<img src="hero.jpg" alt="Hero">

<!-- 好做法：明确设置尺寸 -->
<img src="hero.jpg" alt="Hero" width="800" height="450">
```

### 3.2 使用 aspect-ratio

```css
.image-container {
  width: 100%;
  aspect-ratio: 16 / 9;
  background: #f0f0f0; /* 占位背景色 */
}

.image-container img {
  width: 100%;
  height: 100%;
  object-fit: cover;
}
```

### 3.3 预留广告位

```css
.ad-container {
  min-height: 250px; /* 预留广告高度 */
  background: #f5f5f5;
}
```

### 3.4 避免在已渲染内容上方插入内容

```javascript
// 坏做法：在顶部插入内容
document.body.insertBefore(newElement, document.body.firstChild)

// 好做法：使用 transform 动画，不影响布局流
newElement.style.transform = 'translateY(-100%)'
newElement.style.transition = 'transform 0.3s'
newElement.style.transform = 'translateY(0)'
```

### 3.5 字体加载优化

使用 `size-adjust` 和 `ascent-override` 等属性，让自定义字体与后备字体大小匹配。

```css
@font-face {
  font-family: "CustomFont";
  src: url("/fonts/custom.woff2") format("woff2");
  font-display: swap;
  size-adjust: 100%;
  ascent-override: 90%;
  descent-override: 20%;
  line-gap-override: 0%;
}
```

## 四、性能监控工具

### 4.1 Lighthouse

Chrome 内置工具，提供全面的性能报告。

```bash
# 命令行运行
npx lighthouse https://example.com --output html --output-path ./report.html
```

### 4.2 WebPageTest

https://www.webpagetest.org/ 提供真实设备测试，包含瀑布图和视频录制。

### 4.3 Chrome DevTools

- **Performance 面板**：录制页面加载和交互过程
- **Lighthouse 面板**：一键生成性能报告
- **Coverage 面板**：查看未使用的 CSS/JS

### 4.4 web-vitals 库

在代码中采集真实用户数据。

```bash
npm install web-vitals
```

```javascript
import { getLCP, getFID, getCLS } from 'web-vitals'

function sendToAnalytics(metric) {
  const body = JSON.stringify({
    name: metric.name,
    value: metric.value,
    id: metric.id,
  })
  
  // 发送到分析服务
  navigator.sendBeacon('/analytics', body)
}

getLCP(sendToAnalytics)
getFID(sendToAnalytics)
getCLS(sendToAnalytics)
```

## 总结

Core Web Vitals 不仅影响 SEO 排名，更直接影响用户体验和转化率。优化时记住：

1. **LCP 关注加载速度**：优化图片、字体、减少阻塞资源
2. **FID/INP 关注交互响应**：减少 JS 执行时间，拆分长任务
3. **CLS 关注视觉稳定**：预留空间，避免布局跳动

性能优化是一个持续的过程，建议定期使用 Lighthouse 检测，并监控真实用户的性能数据。把性能作为产品的一部分，而不是事后补救。
