# 抽象类与接口

## 前言

抽象类和接口是 Java 面向对象设计的重要工具。它们定义了规范，让代码更加灵活和可扩展。理解两者的区别和应用场景，是成为优秀 Java 开发者的必经之路。

## 抽象类

### 什么是抽象类

抽象类是不能被实例化的类，通常包含抽象方法（没有实现的方法）。

```java
// 抽象类
public abstract class Animal {

    protected String name;
    protected int age;

    public Animal(String name, int age) {
        this.name = name;
        this.age = age;
    }

    // 具体方法
    public void sleep() {
        System.out.println(name + " 在睡觉");
    }

    // 抽象方法（没有实现）
    public abstract void makeSound();

    public abstract void eat();
}
```

### 抽象类特点

1. **不能实例化**：不能使用 `new` 创建对象
2. **可以包含抽象方法**：没有方法体的方法
3. **可以包含具体方法**：有完整实现的方法
4. **可以有构造方法**：供子类调用

### 抽象类示例

```java
public abstract class Shape {

    protected String color;

    public Shape(String color) {
        this.color = color;
    }

    // 具体方法
    public void setColor(String color) {
        this.color = color;
    }

    public String getColor() {
        return color;
    }

    // 抽象方法
    public abstract double getArea();

    public abstract double getPerimeter();
}

// 矩形
public class Rectangle extends Shape {

    private double width;
    private double height;

    public Rectangle(String color, double width, double height) {
        super(color);
        this.width = width;
        this.height = height;
    }

    @Override
    public double getArea() {
        return width * height;
    }

    @Override
    public double getPerimeter() {
        return 2 * (width + height);
    }
}

// 圆形
public class Circle extends Shape {

    private double radius;

    public Circle(String color, double radius) {
        super(color);
        this.radius = radius;
    }

    @Override
    public double getArea() {
        return Math.PI * radius * radius;
    }

    @Override
    public double getPerimeter() {
        return 2 * Math.PI * radius;
    }
}
```

## 接口

### 什么是接口

接口是抽象方法的集合，定义了类必须实现的行为规范。

```java
public interface Flyable {

    // 接口中的方法默认是 public abstract
    void fly();

    void takeOff();

    void land();
}

// 实现接口
public class Bird implements Flyable {

    @Override
    public void fly() {
        System.out.println("鸟在飞翔");
    }

    @Override
    public void takeOff() {
        System.out.println("鸟起飞");
    }

    @Override
    public void land() {
        System.out.println("鸟降落");
    }
}
```

### 接口特点

1. **所有方法默认是 public abstract**
2. **所有字段默认是 public static final**
3. **不能实例化**
4. **一个类可以实现多个接口**

### 接口演进

```java
// Java 8 之前：只有抽象方法
public interface OldInterface {
    void method1();
    void method2();
}

// Java 8：支持默认方法和静态方法
public interface NewInterface {

    // 抽象方法
    void abstractMethod();

    // 默认方法（Java 8+）
    default void defaultMethod() {
        System.out.println("默认方法实现");
    }

    // 静态方法（Java 8+）
    static void staticMethod() {
        System.out.println("静态方法");
    }

    // 私有方法（Java 9+）
    private void privateMethod() {
        System.out.println("私有方法");
    }
}
```

### 多接口实现

```java
public interface Flyable {
    void fly();
}

public interface Swimmable {
    void swim();
}

public interface Walkable {
    void walk();
}

// 一个类可以实现多个接口
public class Duck implements Flyable, Swimmable, Walkable {

    @Override
    public void fly() {
        System.out.println("鸭子在飞");
    }

    @Override
    public void swim() {
        System.out.println("鸭子在游泳");
    }

    @Override
    public void walk() {
        System.out.println("鸭子在走路");
    }

    public void quack() {
        System.out.println("鸭子嘎嘎叫");
    }
}
```

## 抽象类 vs 接口

| 特性             | 抽象类         | 接口                     |
| ---------------- | -------------- | ------------------------ |
| 实例化           | 不能           | 不能                     |
| 构造方法         | 有             | 无                       |
| 字段             | 各种类型       | 只有 public static final |
| 方法             | 可以有具体方法 | Java 8 前都是抽象的      |
| 单继承 vs 多实现 | 单继承         | 多实现                   |
| 设计目的         | 代码复用       | 定义规范                 |

### 选择建议

**使用抽象类**：

- 需要共享代码
- 需要定义字段
- 需要使用访问修饰符

**使用接口**：

- 定义规范
- 多重继承
- 不相关的类实现相同行为

## 实战演练

### 示例 1：支付系统接口

```java
// 支付接口
public interface Payment {

    boolean pay(double amount);

    boolean refund(String transactionId, double amount);

    double getBalance();
}

// 支付宝实现
public class Alipay implements Payment {

    private double balance;

    public Alipay(double initialBalance) {
        this.balance = initialBalance;
    }

    @Override
    public boolean pay(double amount) {
        if (balance >= amount) {
            balance -= amount;
            System.out.printf("支付宝支付 %.2f 元%n", amount);
            return true;
        }
        System.out.println("余额不足");
        return false;
    }

    @Override
    public boolean refund(String transactionId, double amount) {
        balance += amount;
        System.out.printf("退款 %.2f 元到支付宝%n", amount);
        return true;
    }

    @Override
    public double getBalance() {
        return balance;
    }
}
```

### 示例 2：回调接口

```java
// 回调接口
public interface Callback {

    void onSuccess(String result);

    void onFailure(String error);
}

// 网络请求类
public class NetworkRequest {

    public void fetchData(Callback callback) {
        new Thread(() -> {
            try {
                Thread.sleep(1000);
                callback.onSuccess("数据获取成功");
            } catch (Exception e) {
                callback.onFailure("网络错误");
            }
        }).start();
    }
}
```

### 示例 3：策略模式

```java
// 排序策略接口
public interface SortStrategy {

    void sort(int[] array);
}

// 冒泡排序
public class BubbleSort implements SortStrategy {

    @Override
    public void sort(int[] array) {
        // 冒泡排序实现
        System.out.println("使用冒泡排序");
    }
}

// 快速排序
public class QuickSort implements SortStrategy {

    @Override
    public void sort(int[] array) {
        // 快速排序实现
        System.out.println("使用快速排序");
    }
}
```

## 最佳实践

### 1. 接口命名

```java
// ✅ 好的做法：使用 -able 后缀
public interface Flyable { }
public interface Runnable { }
public interface Callable<V> { }
```

### 2. 接口隔离

```java
// ❌ 不好的做法：接口过大
public interface BigInterface {
    void method1();
    void method2();
    // ... 100 个方法
}

// ✅ 好的做法：拆分为多个小接口
public interface Interface1 { void method1(); }
public interface Interface2 { void method2(); }
```

### 3. 优先使用接口

```java
// ✅ 好的做法
List<String> list = new ArrayList<>();

// ❌ 不好的做法
ArrayList<String> list = new ArrayList<>();
```

## 常见问题

### Q1：抽象类和接口的选择？

**答**：需要代码复用用抽象类，定义规范用接口。

### Q2：一个类可以实现多个接口吗？

**答**：可以，这正是接口的优势。

### Q3：接口可以继承接口吗？

**答**：可以，使用 extends 关键字。

### Q4：默认方法会破坏现有实现吗？

**答**：不会，默认方法提供默认实现。
