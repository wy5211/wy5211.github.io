# 综合实战

20 篇文章，从"为什么学 Java"走到这里。如果只用一个词概括 Java 和 JS 的区别，那就是"显式"——Java 要求你把一切都说清楚：类型、结构、边界、异常。这种显式性在初期让人不适应，但在项目规模增长后，它就是你最可靠的防线。

这篇收官之作，我们用纯 Java（没有任何框架）把 CMS 做成一个可运行的控制台应用，把前面所有知识点串起来。

// PostStatus.java — 第 9 篇：枚举
public enum PostStatus {
    DRAFT("草稿"),
    PUBLISHED("已发布"),
    ARCHIVED("已归档");

    private final String label;
    PostStatus(String label) { this.label = label; }
    public String getLabel() { return label; }
}

// Content.java — 第 7 篇：继承基类
public abstract class Content {
    protected Long id;
    protected String title;
    protected LocalDateTime createdAt;

    protected Content(String title) {
        this.title = title;
        this.createdAt = LocalDateTime.now();
    }

    public abstract String getSummary();  // 第 8 篇：抽象方法
}

// Post.java — 第 5/7/8 篇：继承 + 接口
public class Post extends Content implements Comparable<Post> {
    private String content;        // Markdown 正文
    private PostStatus status;
    private Long authorId;
    private long viewCount;
    private List<Tag> tags = new ArrayList<>();

    public Post(String title, String content) {
        super(title);
        this.content = content;
        this.status = PostStatus.DRAFT;
    }

    @Override
    public String getSummary() {
        // 取前 100 个字符作为摘要
        if (content.length() <= 100) return content;
        return content.substring(0, 100) + "...";
    }

    // 第 8 篇：Comparable 接口
    @Override
    public int compareTo(Post other) {
        return other.createdAt.compareTo(this.createdAt);  // 按时间倒序
    }

    // 第 6 篇：封装
    public void addTag(Tag tag) {
        if (tags.size() >= 5) {
            throw new IllegalStateException("每篇文章最多 5 个标签");
        }
        if (tags.contains(tag)) return;  // 第 13 篇：Set 去重逻辑
        tags.add(tag);
    }

    public void publish() {
        if (content == null || content.isBlank()) {
            throw new IllegalArgumentException("文章内容不能为空");  // 第 10 篇：异常
        }
        this.status = PostStatus.PUBLISHED;
    }

    // getter 省略...
}

## 业务服务：集合、Stream、Lambda

文章管理服务，涉及第 11-15 篇和第 19 篇的知识：

```java
public class PostService {
    private final List<Post> posts = new ArrayList<>();

    public void addPost(Post post) {
        posts.add(post);
    }

    // 第 15 篇：Stream 过滤
    public List<Post> findPublished() {
        return posts.stream()
            .filter(p -> p.getStatus() == PostStatus.PUBLISHED)
            .sorted()
            .collect(Collectors.toList());
    }

    // 第 19 篇：接受 Predicate，灵活组合过滤条件
    public List<Post> find(Predicate<Post> condition) {
        return posts.stream()
            .filter(condition)
            .sorted()
            .collect(Collectors.toList());
    }

    // 第 13 篇：Set 去重标签
    public Set<String> allTagNames() {
        return posts.stream()
            .flatMap(p -> p.getTags().stream())
            .map(Tag::getName)
            .collect(Collectors.toSet());
    }

    // 第 15 篇：Stream 统计
    public Map<PostStatus, Long> countByStatus() {
        return posts.stream()
            .collect(Collectors.groupingBy(Post::getStatus, Collectors.counting()));
    }
}
```

## 评论服务：并发安全

评论计数器，涉及第 17-18 篇的多线程知识：

```java
public class CommentService {
    private final Map<Long, AtomicInteger> commentCounts = new ConcurrentHashMap<>();

    public void addComment(long postId) {
        commentCounts
            .computeIfAbsent(postId, k -> new AtomicInteger(0))
            .incrementAndGet();
    }

    public int getCount(long postId) {
        AtomicInteger counter = commentCounts.get(postId);
        return counter != null ? counter.get() : 0;
    }

    // 第 17 篇：多线程批量处理
    public Map<Long, Integer> batchCount(List<Long> postIds) {
        Map<Long, Integer> results = new HashMap<>();
        List<Thread> threads = new ArrayList<>();

        for (Long postId : postIds) {
            Thread t = new Thread(() -> {
                results.put(postId, getCount(postId));
            });
            threads.add(t);
            t.start();
        }

        for (Thread t : threads) {
            try { t.join(); } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        }
        return results;
    }
}
```

## 文件读写：IO 操作

Markdown 文件导入导出，涉及第 16 篇的 IO 知识：

```java
public class MarkdownIO {
    // 导出文章为 Markdown 文件
    public static void exportPost(Post post, String directory) throws IOException {
        Path dir = Paths.get(directory);
        Files.createDirectories(dir);

        String filename = post.getTitle().replaceAll("[^a-zA-Z0-9\\u4e00-\\u9fff]", "-")
            .toLowerCase() + ".md";
        Path filePath = dir.resolve(filename);

        StringBuilder sb = new StringBuilder();
        sb.append("# ").append(post.getTitle()).append("\n\n");
        sb.append("> 状态：").append(post.getStatus().getLabel()).append("\n\n");
        sb.append(post.getContent()).append("\n");

        Files.writeString(filePath, sb.toString(), StandardCharsets.UTF_8);
        return filePath;
    }

    // 批量导出，第 17 篇：多线程
    public static List<Path> batchExport(List<Post> posts, String directory) {
        List<Path> exported = Collections.synchronizedList(new ArrayList<>());

        posts.parallelStream().forEach(post -> {
            try {
                Path path = exportPost(post, directory);
                exported.add(path);
            } catch (IOException e) {
                System.err.println("导出失败：" + post.getTitle());
            }
        });

        return exported;
    }
}
```

## 控制台应用入口

把所有部分组合起来：

```java
import java.util.*;
import java.util.function.*;

public class CMSApplication {
    private static final Scanner scanner = new Scanner(System.in);
    private static final PostService postService = new PostService();
    private static final CommentService commentService = new CommentService();

    public static void main(String[] args) {
        // 初始化示例数据
        initData();

        // 主循环
        while (true) {
            printMenu();
            String choice = scanner.nextLine().trim();
            switch (choice) {
                case "1" -> listPosts();
                case "2" -> createPost();
                case "3" -> publishPost();
                case "4" -> searchPosts();
                case "5" -> showStats();
                case "6" -> exportPosts();
                case "0" -> { System.out.println("再见！"); return; }
                default -> System.out.println("无效选项");
            }
        }
    }

    private static void printMenu() {
        System.out.println("\n=== TechBlog CMS ===");
        System.out.println("1. 文章列表");
        System.out.println("2. 创建文章");
        System.out.println("3. 发布文章");
        System.out.println("4. 搜索文章");
        System.out.println("5. 数据统计");
        System.out.println("6. 导出文章");
        System.out.println("0. 退出");
        System.out.print("请选择：");
    }

    private static void listPosts() {
        List<Post> posts = postService.findPublished();
        if (posts.isEmpty()) {
            System.out.println("暂无已发布文章");
            return;
        }
        posts.forEach(p -> System.out.printf(
            "[%d] %s (浏览：%d，评论：%d)%n",
            p.getId(), p.getTitle(), p.getViewCount(),
            commentService.getCount(p.getId())
        ));
    }

    private static void createPost() {
        System.out.print("标题：");
        String title = scanner.nextLine();
        System.out.print("正文（输入 END 结束）：");
        StringBuilder content = new StringBuilder();
        String line;
        while (!(line = scanner.nextLine()).equals("END")) {
            content.append(line).append("\n");
        }
        Post post = new Post(title, content.toString());
        postService.addPost(post);
        System.out.println("草稿已保存，ID：" + post.getId());
    }

    private static void publishPost() {
        System.out.print("文章 ID：");
        long id = Long.parseLong(scanner.nextLine());
        try {
            postService.find(p -> p.getId().equals(id))
                .stream()
                .findFirst()
                .ifPresentOrElse(
                    post -> { post.publish(); System.out.println("发布成功"); },
                    () -> System.out.println("文章不存在")
                );
        } catch (IllegalArgumentException e) {
            System.out.println("发布失败：" + e.getMessage());  // 第 10 篇：异常处理
        }
    }

    private static void searchPosts() {
        System.out.print("关键词：");
        String keyword = scanner.nextLine();
        List<Post> results = postService.find(
            p -> p.getTitle().contains(keyword)
              || p.getContent().contains(keyword)
        );
        results.forEach(p -> System.out.printf("  - %s [%s]%n",
            p.getTitle(), p.getStatus().getLabel()));
    }

    private static void showStats() {
        System.out.println("--- 数据统计 ---");
        postService.countByStatus().forEach((status, count) ->
            System.out.printf("  %s：%d 篇%n", status.getLabel(), count)
        );
        System.out.println("标签数：" + postService.allTagNames().size());
    }

    private static void exportPosts() {
        List<Post> published = postService.findPublished();
        List<Path> exported = MarkdownIO.batchExport(published, "./exported");
        System.out.println("已导出 " + exported.size() + " 篇文章");
    }

    private static void initData() {
        Tag java = new Tag("Java", "java", "#E76F00");
        Tag spring = new Tag("Spring Boot", "spring-boot", "#6DB33F");
        Tag js = new Tag("JavaScript", "javascript", "#F7DF1E");

        Post p1 = new Post("Java 入门指南", "从零开始学习 Java 编程...");
        p1.addTag(java);
        p1.publish();

        Post p2 = new Post("Spring Boot 实战", "构建 RESTful API...");
        p2.addTag(java);
        p2.addTag(spring);
        p2.publish();

        Post p3 = new Post("JS vs Java", "两种语言的对比...");
        p3.addTag(js);

        postService.addPost(p1);
        postService.addPost(p2);
        postService.addPost(p3);

        commentService.addComment(p1.getId());
        commentService.addComment(p1.getId());
        commentService.addComment(p2.getId());
    }
}
```

## 全系列 JS vs Java 回顾

20 篇走完，是时候回顾一下那些关键的对比点了。

### 语法层面（01-04）

| 篇  | 知识点   | JS                           | Java                                  |
| --- | -------- | ---------------------------- | ------------------------------------- |
| 01  | 运行模型 | V8 解释执行，npm 生态        | JVM 编译+解释，Maven 生态             |
| 02  | 类型系统 | 动态类型，`1 + '1' === '11'` | 静态类型，编译期类型检查              |
| 03  | 控制流   | switch 穿透，弱等于          | switch 表达式，模式匹配（Java 17+）   |
| 04  | 字符串   | 模板字符串，可变             | `String` 不可变，`StringBuilder` 拼接 |

### OOP 层面（05-09）

| 篇  | 知识点 | JS                         | Java                                |
| --- | ------ | -------------------------- | ----------------------------------- |
| 05  | 类     | 语法糖，基于原型链         | 真正的类，编译时确定结构            |
| 06  | 封装   | `#` 私有字段（较新）       | `private/protected/public`，package |
| 07  | 继承   | `extends` + prototype      | `extends` + 方法重写 + `@Override`  |
| 08  | 接口   | TS 的 interface 编译后消失 | Java 的 interface 有运行时存在      |
| 09  | 枚举   | 对象常量 / union type      | 真正的 enum，可以有字段和方法       |

### 核心特性（10-14）

| 篇  | 知识点  | JS                                  | Java                                      |
| --- | ------- | ----------------------------------- | ----------------------------------------- |
| 10  | 异常    | `try-catch`，所有异常都是 unchecked | checked vs unchecked，强制处理            |
| 11  | 集合    | `Array` 一个类型打天下              | `List`/`Set`/`Map`/`Queue` 各有分工       |
| 12  | List    | 几乎只用 `Array`                    | `ArrayList` vs `LinkedList`，分场景选择   |
| 13  | Set/Map | `Set` + `Map`/`Object`              | `HashSet`/`HashMap` + `TreeSet`/`TreeMap` |
| 14  | 泛型    | TS 泛型有运行时类型信息             | Java 泛型是类型擦除，运行时消失           |

### 现代特性（15-20）

| 篇  | 知识点   | JS                         | Java                                   |
| --- | -------- | -------------------------- | -------------------------------------- |
| 15  | Stream   | `Array.prototype` 链式调用 | `Stream` 惰性求值，只能消费一次        |
| 16  | IO       | `fs` 模块，回调/Promise    | `Reader/Writer` + `try-with-resources` |
| 17  | 多线程   | 单线程 + 事件循环          | 真多线程，`Thread`/`Runnable`          |
| 18  | 并发安全 | 无竞态条件（主线程）       | `synchronized`/原子类/`volatile`       |
| 19  | Lambda   | 箭头函数，一等公民         | 必须依附函数式接口，effectively final  |
| 20  | 综合实战 | —                          | 无框架也能构建完整应用                 |

### 三个最本质的差异

1. **类型系统**。Java 的静态类型是双刃剑——编译期更安全，但开发效率在初期低于 JS。随着项目规模增长，这个 trade-off 会逐渐向 Java 倾斜。

2. **并发模型**。JS 的单线程模型简化了状态管理，代价是 CPU 密集任务需要 Worker。Java 的多线程更强大但也更复杂，并发安全是需要持续关注的领域。

3. **显式 vs 隐式**。Java 要求你显式声明一切：类型、异常、接口、修饰符。JS 倾向于隐式推断。显式性让代码更可预测，但也更冗长。

## 接下来：Spring Boot

纯 Java 能做的事很多，但你也会注意到：没有框架时，很多工作需要手动完成——路由、依赖注入、数据库连接、配置管理。这正是 Spring Boot 要解决的。

如果你跟着这 20 篇一路走过来，你已经具备了学习 Spring Boot 的全部基础：

- **OOP 和接口**：理解 Spring 的 IoC 容器和 Bean 依赖注入
- **注解**：`@Override`、`@FunctionalInterface` 为理解 `@RestController`、`@Autowired` 做了铺垫
- **异常处理**：Spring 的 `@ControllerAdvice` 全局异常处理和 checked exception 一脉相承
- **泛型**：`ResponseEntity<T>`、`Page<T>` 等泛型容器
- **Stream 和 Lambda**：Spring Data 的方法引用查询、Java 8 日期时间 API
- **IO**：`Resource` 抽象和文件上传处理
- **多线程**：`@Async` 异步方法、线程池配置

Java 的学习曲线比 JS 陡，但每一步都是坚实的。这 20 篇覆盖了 Java 语言本身的核心，接下来的 Spring Boot 系列将带你进入 Java 生态的主流开发方式。
