## 前言

数据模型是应用的基石。Drizzle ORM 使用**纯 TypeScript** 定义 Schema，提供了完整的类型安全和开发体验。

**核心优势**：

- ✅ **原生 TypeScript**：无需学习 DSL，直接使用 TS
- ✅ **完整类型推断**：自动推断表结构
- ✅ **代码提示友好**：IDE 自动补全
- ✅ **无代码生成**：定义即类型
- ✅ **数据库无关**：PostgreSQL、MySQL、SQLite 一套 API

今天，我们将深入学习 Drizzle ORM 的 Schema 定义系统。

## Schema 基础

### 什么是 Schema？

在 Drizzle ORM 中，Schema 是用 TypeScript 定义的表结构描述：

```typescript
// ❌ 传统 Prisma：使用 DSL（领域特定语言）
// schema.prisma:
model User {
  id    Int    @id
  email String @unique
  name  String?
}

// ✅ Drizzle ORM：使用原生 TypeScript
import { pgTable, serial, varchar, text } from 'drizzle-orm/pg-core';

export const users = pgTable('users', {
  id: serial('id').primaryKey(),
  email: varchar('email', { length: 255 }).notNull().unique(),
  name: text('name'),
});
```

### Schema 的作用

```typescript
// 1. 类型定义
export const users = pgTable("users", {
  id: serial("id").primaryKey(),
  email: varchar("email", { length: 255 }).notNull(),
});

// users.id 的类型：PgSerial<number>
// users.email 的类型：PgVarchar<255>

// 2. 查询类型推断
const result = await db.select().from(users);
// result 类型：{ id: number; email: string }[]

// 3. 插入类型验证
await db.insert(users).values({
  email: "test@example.com", // ✅ 类型正确
  // email: 123  // ❌ 类型错误：不能将 number 赋给 string
});

// 4. 生成 SQL
// 自动生成：CREATE TABLE users (id SERIAL PRIMARY KEY, email VARCHAR(255) NOT NULL, ...);
```

## PostgreSQL 数据类型映射

### 1. 整数类型

```typescript
import {
  pgTable,
  serial,
  bigserial,
  integer,
  bigint,
} from "drizzle-orm/pg-core";

export const numbers = pgTable("numbers", {
  // serial：自增整数（4 字节）
  id: serial("id").primaryKey(),
  // SQL：id SERIAL PRIMARY KEY

  // bigserial：自增大整数（8 字节）
  bigId: bigserial("big_id").primaryKey(),
  // SQL：big_id BIGSERIAL PRIMARY KEY

  // integer：普通整数（4 字节，-2^31 到 2^31-1）
  count: integer("count"),
  // SQL：count INTEGER

  // bigint：大整数（8 字节）
  balance: bigint("balance", { mode: "number" }),
  // SQL：balance BIGINT

  // smallint：小整数（2 字节）
  age: integer("age"),
  // SQL：age INTEGER
});
```

**使用场景：**

- `serial`：主键 ID
- `bigserial`：高并发主键
- `integer`：计数、数量
- `bigint`：金额、余额

### 2. 字符串类型

```typescript
import { pgTable, varchar, text, char } from "drizzle-orm/pg-core";

export const strings = pgTable("strings", {
  // varchar(n)：变长字符串，最大长度 n
  email: varchar("email", { length: 255 }).notNull(),
  // SQL：email VARCHAR(255) NOT NULL

  // text：无限长度字符串
  content: text("content"),
  // SQL：content TEXT

  // char(n)：定长字符串，不足补空格
  code: char("code", { length: 10 }),
  // SQL：code CHAR(10)
});
```

**使用场景：**

- `varchar(255)`：邮箱、用户名
- `text`：文章内容、评论
- `char(10)`：固定格式代码

### 3. 布尔类型

```typescript
import { pgTable, boolean } from "drizzle-orm/pg-core";

export const flags = pgTable("flags", {
  isActive: boolean("is_active").default(false),
  // SQL：is_active BOOLEAN DEFAULT FALSE

  isVerified: boolean("is_verified").notNull(),
  // SQL：is_verified BOOLEAN NOT NULL
});
```

### 4. 时间类型

```typescript
import { pgTable, timestamp, date, time, interval } from "drizzle-orm/pg-core";

export const times = pgTable("times", {
  // timestamp：日期和时间
  createdAt: timestamp("created_at").defaultNow(),
  // SQL：created_at TIMESTAMP DEFAULT NOW()

  // 带时区
  updatedAt: timestamp("updated_at", { withTimezone: true, mode: "date" }),
  // SQL：updated_at TIMESTAMPTZ

  // date：只有日期
  birthDate: date("birth_date"),
  // SQL：birth_date DATE

  // time：只有时间
  workTime: time("work_time"),
  // SQL：work_time TIME

  // interval：时间间隔
  duration: interval("duration"),
  // SQL：duration INTERVAL
});
```

**timestamp 的 mode 参数：**

```typescript
// mode: 'string'（默认）
timestamp("created_at", { mode: "string" });
// TypeScript 类型：string
// SQL 值：'2024-01-01 12:00:00'

// mode: 'date'
timestamp("created_at", { mode: "date" });
// TypeScript 类型：Date
// SQL 值：'2024-01-01 12:00:00'

// mode: 'number'
timestamp("created_at", { mode: "number" });
// TypeScript 类型：number（Unix 时间戳）
// SQL 值：1704067200
```

### 5. 数值类型

```typescript
import {
  pgTable,
  decimal,
  numeric,
  real,
  doublePrecision,
} from "drizzle-orm/pg-core";

export const decimals = pgTable("decimals", {
  // decimal：精确的小数（推荐用于金额）
  price: decimal("price", { precision: 10, scale: 2 }),
  // SQL：price DECIMAL(10, 2)
  // 存储：总共 10 位，小数点后 2 位
  // 范围：-99999999.99 到 99999999.99

  // numeric：decimal 的别名
  balance: numeric("balance", { precision: 15, scale: 2 }),
  // SQL：balance NUMERIC(15, 2)

  // real：单精度浮点数（6 位精度）
  temperature: real("temperature"),
  // SQL：temperature REAL

  // doublePrecision：双精度浮点数（15 位精度）
  weight: doublePrecision("weight"),
  // SQL：weight DOUBLE PRECISION
});
```

**使用场景：**

- `decimal(10, 2)`：商品价格、金额
- `real`：温度、湿度
- `doublePrecision`：体重、身高

### 6. JSON 类型

```typescript
import { pgTable, json, jsonb } from "drizzle-orm/pg-core";

export const jsons = pgTable("jsons", {
  // json：存储为文本，查询慢
  metadata: json("metadata"),
  // SQL：metadata JSON
  // TypeScript 类型：unknown

  // jsonb：二进制存储，查询快（推荐）
  settings: jsonb("settings").$type<{
    theme: "light" | "dark";
    notifications: boolean;
  }>(),
  // SQL：settings JSONB
  // TypeScript 类型：{ theme: 'light' | 'dark'; notifications: boolean }
});
```

**使用场景：**

- `jsonb`：用户设置、配置文件
- `json`：历史数据、不需要查询的 JSON

### 7. 数组类型

```typescript
import { pgTable, array, varchar } from "drizzle-orm/pg-core";

export const arrays = pgTable("arrays", {
  // 字符串数组
  tags: array("tags", varchar("tag", { length: 50 })),
  // SQL：tags VARCHAR(50)[]

  // 整数数组
  scores: array("scores", integer("score")),
  // SQL：scores INTEGER[]

  // 带默认值
  categories: array("categories", varchar("category", { length: 50 })).default(
    sql`'[]'`
  ),
  // SQL：categories VARCHAR(50)[] DEFAULT '[]'
});
```

**使用场景：**

- 文章标签
- 商品分类
- 用户权限列表

### 8. 二进制类型

```typescript
import { pgTable, bytea } from "drizzle-orm/pg-core";

export const binaries = pgTable("binaries", {
  // bytea：二进制数据
  avatar: bytea("avatar"),
  // SQL：avatar BYTEA

  // 存储图片、文件等
  file: bytea("file"),
  // SQL：file BYTEA
});
```

### 9. 枚举类型

```typescript
import { pgTable, pgEnum } from "drizzle-orm/pg-core";

// 1. 定义枚举
export const roleEnum = pgEnum("role", ["USER", "ADMIN", "MODERATOR"]);
// SQL：CREATE TYPE role AS ENUM ('USER', 'ADMIN', 'MODERATOR');

// 2. 使用枚举
export const users = pgTable("users", {
  id: serial("id").primaryKey(),
  role: roleEnum("role").default("USER"),
  // SQL：role ROLE DEFAULT 'USER'
});

// TypeScript 类型自动推断为：'USER' | 'ADMIN' | 'MODERATOR'

// 使用示例
await db.insert(users).values({
  role: "ADMIN", // ✅ 类型正确
  // role: 'INVALID'  // ❌ 类型错误
});
```

### 10. UUID 类型

```typescript

export const uuids = pgTable("uuids", {
```

## 表关系定义

### 1. 一对一关系

```typescript
import { pgTable, serial, integer, text } from "drizzle-orm/pg-core";

export const users = pgTable("users", {
  id: serial("id").primaryKey(),
  name: text("name").notNull(),
});

export const profiles = pgTable("profiles", {
  id: serial("id").primaryKey(),
  userId: integer("user_id").references(() => users.id, {
    onDelete: "cascade",
  }),
  bio: text("bio"),
});

// SQL：
// CREATE TABLE profiles (
//   id SERIAL PRIMARY KEY,
//   user_id INTEGER REFERENCES users(id) ON DELETE CASCADE,
//   bio TEXT
// );

// 查询关联数据
const result = await db
  .select()
  .from(users)
  .leftJoin(profiles, eq(users.id, profiles.userId));
```

**外键约束选项：**

```typescript
.references(() => users.id, {
  onDelete: 'cascade',    // 删除用户时，同时删除 profile
  // onDelete: 'set null',  // 删除用户时，profile.user_id 设为 NULL
  // onDelete: 'restrict',  // 删除用户时，如果有 profile 则禁止删除
  // onDelete: 'no action', // 不采取任何动作
  onUpdate: 'cascade'     // 更新用户 ID 时，同时更新 profile.user_id
})
```

### 2. 一对多关系

```typescript
export const authors = pgTable("authors", {
  id: serial("id").primaryKey(),
  name: text("name").notNull(),
});

export const posts = pgTable("posts", {
  id: serial("id").primaryKey(),
  title: text("title").notNull(),
  authorId: integer("author_id").references(() => authors.id),
  content: text("content"),
});

// 查询作者及其文章
const authorsWithPosts = await db
  .select()
  .from(authors)
  .leftJoin(posts, eq(authors.id, posts.authorId));

// 结果：
// [
//   { author: { id: 1, name: 'Alice' }, post: { id: 1, title: 'Hello' } },
//   { author: { id: 1, name: 'Alice' }, post: { id: 2, title: 'World' } },
//   { author: { id: 2, name: 'Bob' }, post: null }
// ]
```

### 3. 多对多关系

```typescript
export const posts = pgTable("posts", {
  id: serial("id").primaryKey(),
  title: text("title").notNull(),
});

export const tags = pgTable("tags", {
  id: serial("id").primaryKey(),
  name: text("name").notNull().unique(),
});

// 中间表（关联表）
export const postsToTags = pgTable("posts_to_tags", {
  postId: integer("post_id").references(() => posts.id, {
    onDelete: "cascade",
  }),
  tagId: integer("tag_id").references(() => tags.id, { onDelete: "cascade" }),
});

// 复合主键
export const postsToTagsPk = pgTable(
  "posts_to_tags_pk",
  {
    postId: integer("post_id").references(() => posts.id, {
      onDelete: "cascade",
    }),
    tagId: integer("tag_id").references(() => tags.id, { onDelete: "cascade" }),
  },
  (table) => ({
    pk: primaryKey({ columns: [table.postId, table.tagId] }),
  })
);

// 查询文章及其标签
const postsWithTags = await db
  .select()
  .from(posts)
  .innerJoin(postsToTags, eq(posts.id, postsToTags.postId))
  .innerJoin(tags, eq(postsToTags.tagId, tags.id));
```

## 索引和约束

### 1. 单列索引

```typescript
import { pgTable, serial, varchar, index } from "drizzle-orm/pg-core";

export const users = pgTable(
  "users",
  {
    id: serial("id").primaryKey(),
    email: varchar("email", { length: 255 }).notNull().unique(),
    name: varchar("name", { length: 100 }),
  },
  (table) => ({
    // 创建索引
    nameIdx: index("name_idx").on(table.name),
    // SQL：CREATE INDEX name_idx ON users(name)
  })
);
```

### 2. 复合索引

```typescript
export const posts = pgTable(
  "posts",
  {
    id: serial("id").primaryKey(),
    title: varchar("title", { length: 255 }),
    authorId: integer("author_id"),
    createdAt: timestamp("created_at").defaultNow(),
  },
  (table) => ({
    // 复合索引
    authorCreatedAtIdx: index("author_created_at_idx").on(
      table.authorId,
      table.createdAt
    ),
    // SQL：CREATE INDEX author_created_at_idx ON posts(author_id, created_at)
  })
);
```

### 3. 唯一索引

```typescript
export const users = pgTable(
  "users",
  {
    id: serial("id").primaryKey(),
    email: varchar("email", { length: 255 }).notNull(),
  },
  (table) => ({
    // 唯一索引
    emailUnique: unique("email_unique").on(table.email),
    // SQL：CREATE UNIQUE INDEX email_unique ON users(email)
  })
);
```

### 4. 全文搜索索引

```typescript
import { pgTable, serial, text, index } from "drizzle-orm/pg-core";

export const articles = pgTable(
  "articles",
  {
    id: serial("id").primaryKey(),
    title: text("title").notNull(),
    content: text("content").notNull(),
  },
  (table) => ({
    // 全文搜索索引
    titleContentIdx: index("title_content_idx").using(
      "gin",
      sql`to_tsvector('english', ${table.title} || ' ' || ${table.content})`
    ),
    // SQL：CREATE INDEX title_content_idx ON articles USING gin(to_tsvector('english', title || ' ' || content))
  })
);

// 使用全文搜索
const results = await db
  .select()
  .from(articles)
  .where(
    sql`to_tsvector('english', ${articles.title} || ' ' || ${articles.content}) @@ to_tsquery('english', 'search')`
  );
```

## 实战：博客系统数据模型

让我们构建一个完整的博客系统数据模型。

### 完整 Schema 定义

```typescript
// src/schema.ts

import {
  pgTable,
  serial,
  varchar,
  text,
  integer,
  timestamp,
  boolean,
  pgEnum,
} from "drizzle-orm/pg-core";
import { relations } from "drizzle-orm";
import { index, unique } from "drizzle-orm/pg-core";

// ========== 枚举定义 ==========

export const postStatusEnum = pgEnum("post_status", [
  "DRAFT",
  "PUBLISHED",
  "ARCHIVED",
]);
export const commentStatusEnum = pgEnum("comment_status", [
  "PENDING",
  "APPROVED",
  "SPAM",
]);

// ========== 用户表 ==========
export const users = pgTable(
  "users",
  {
    id: serial("id").primaryKey(),
    email: varchar("email", { length: 255 }).notNull().unique(),
    username: varchar("username", { length: 50 }).notNull().unique(),
    passwordHash: varchar("password_hash", { length: 255 }).notNull(),
    name: varchar("name", { length: 100 }),
    bio: text("bio"),
    avatar: varchar("avatar", { length: 500 }),
    isActive: boolean("is_active").default(true),
    createdAt: timestamp("created_at").defaultNow(),
    updatedAt: timestamp("updated_at").defaultNow(),
  },
  (table) => ({
    emailIdx: index("user_email_idx").on(table.email),
    usernameIdx: index("user_username_idx").on(table.username),
  })
);

// ========== 文章表 ==========
export const posts = pgTable(
  "posts",
  {
    id: serial("id").primaryKey(),
    title: varchar("title", { length: 255 }).notNull(),
    slug: varchar("slug", { length: 255 }).notNull().unique(),
    excerpt: varchar("excerpt", { length: 500 }),
    content: text("content").notNull(),
    coverImage: varchar("cover_image", { length: 500 }),
    authorId: integer("author_id").references(() => users.id, {
      onDelete: "cascade",
    }),
    categoryId: integer("category_id").references(() => categories.id),
    status: postStatusEnum("status").default("DRAFT"),
    viewCount: integer("view_count").default(0),
    isFeatured: boolean("is_featured").default(false),
    publishedAt: timestamp("published_at"),
    createdAt: timestamp("created_at").defaultNow(),
    updatedAt: timestamp("updated_at").defaultNow(),
  },
  (table) => ({
    slugIdx: unique("post_slug_unique").on(table.slug),
    authorIdx: index("post_author_idx").on(table.authorId),
    categoryIdx: index("post_category_idx").on(table.categoryId),
    statusIdx: index("post_status_idx").on(table.status),
    publishedAtIdx: index("post_published_at_idx").on(table.publishedAt),
    authorStatusIdx: index("post_author_status_idx").on(
      table.authorId,
      table.status
    ),
  })
);

// ========== 分类表 ==========
export const categories = pgTable(
  "categories",
  {
    id: serial("id").primaryKey(),
    name: varchar("name", { length: 100 }).notNull().unique(),
    slug: varchar("slug", { length: 100 }).notNull().unique(),
    description: text("description"),
    parentId: integer("parent_id").references(() => categories.id),
    createdAt: timestamp("created_at").defaultNow(),
  },
  (table) => ({
    slugIdx: unique("category_slug_unique").on(table.slug),
    parentIdIdx: index("category_parent_idx").on(table.parentId),
  })
);

// ========== 标签表 ==========
export const tags = pgTable(
  "tags",
  {
    id: serial("id").primaryKey(),
    name: varchar("name", { length: 50 }).notNull().unique(),
    slug: varchar("slug", { length: 50 }).notNull().unique(),
    createdAt: timestamp("created_at").defaultNow(),
  },
  (table) => ({
    slugUnique: unique("tag_slug_unique").on(table.slug),
  })
);

// ========== 文章-标签关联表 ==========
export const postsToTags = pgTable(
  "posts_to_tags",
  {
    postId: integer("post_id").references(() => posts.id, {
      onDelete: "cascade",
    }),
    tagId: integer("tag_id").references(() => tags.id, { onDelete: "cascade" }),
    createdAt: timestamp("created_at").defaultNow(),
  },
  (table) => ({
    pk: primaryKey({ columns: [table.postId, table.tagId] }),
    postIdIdx: index("posts_to_tags_post_idx").on(table.postId),
    tagIdIdx: index("posts_to_tags_tag_idx").on(table.tagId),
  })
);

// ========== 评论表 ==========
export const comments = pgTable(
  "comments",
  {
    id: serial("id").primaryKey(),
    postId: integer("post_id").references(() => posts.id, {
      onDelete: "cascade",
    }),
    authorId: integer("author_id").references(() => users.id, {
      onDelete: "set null",
    }),
    parentId: integer("parent_id").references(() => comments.id, {
      onDelete: "cascade",
    }),
    content: text("content").notNull(),
    status: commentStatusEnum("status").default("PENDING"),
    createdAt: timestamp("created_at").defaultNow(),
    updatedAt: timestamp("updated_at").defaultNow(),
  },
  (table) => ({
    postIdIdx: index("comment_post_idx").on(table.postId),
    authorIdIdx: index("comment_author_idx").on(table.authorId),
    parentIdIdx: index("comment_parent_idx").on(table.parentId),
    statusIdx: index("comment_status_idx").on(table.status),
  })
);

// ========== Drizzle Relations（可选）==========
export const usersRelations = relations(users, ({ many }) => ({
  posts: many(posts),
  comments: many(comments),
}));

export const postsRelations = relations(posts, ({ one, many }) => ({
  author: one(users, {
    fields: [posts.authorId],
    references: [users.id],
  }),
  category: one(categories, {
    fields: [posts.categoryId],
    references: [categories.id],
  }),
  comments: many(comments),
  tags: many(postsToTags),
}));
```

## 高级技巧

### 1. 生成列（Generated Columns）

```typescript
import { pgTable, serial, varchar, text, integer } from "drizzle-orm/pg-core";

export const users = pgTable("users", {
  id: serial("id").primaryKey(),
  firstName: varchar("first_name", { length: 50 }).notNull(),
  lastName: varchar("last_name", { length: 50 }).notNull(),
  fullName: text("full_name").generatedAlwaysAs(
    sql`${users.firstName} || ' ' || ${users.lastName}`
  ),
  // SQL：full_name TEXT GENERATED ALWAYS AS (first_name || ' ' || last_name) STORED
});

// fullName 是虚拟列，自动计算，无需手动设置
```

### 2. 默认值表达式

```typescript
export const posts = pgTable("posts", {
  id: serial("id").primaryKey(),
  title: varchar("title", { length: 255 }).notNull(),
  slug: varchar("slug", { length: 255 })
    .notNull()
    .default(sql`lower(replace(title, ' ', '-'))`),
  // SQL：slug VARCHAR(255) NOT NULL DEFAULT lower(replace(title, ' ', '-'))
});
```

### 3. 检查约束

```typescript
export const users = pgTable(
  "users",
  {
    id: serial("id").primaryKey(),
    age: integer("age"),
    email: varchar("email", { length: 255 }).notNull(),
  },
  (table) => ({
    // 年龄必须大于 0
    ageCheck: check("age_check", sql`${table.age} > 0`),
    // SQL：CHECK (age > 0)

    // 邮箱格式验证
    emailCheck: check(
      "email_check",
      sql`${table.email} ~ '^[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Za-z]{2,}$'`
    ),
  })
);
```

### 4. 表级约束

```typescript
export const products = pgTable(
  "products",
  {
    id: serial("id").primaryKey(),
    name: varchar("name", { length: 255 }).notNull(),
    price: integer("price").notNull(),
    stock: integer("stock").notNull(),
  },
  (table) => ({
    // 价格必须大于 0
    priceCheck: check("price_check", sql`${table.price} > 0`),

    // 库存必须大于等于 0
    stockCheck: check("stock_check", sql`${table.stock} >= 0`),
  })
);
```

## 最佳实践

### 1. 命名规范

```typescript
// ✅ 推荐：使用 snake_case（数据库命名）
export const users = pgTable("users", {
  userId: serial("user_id").primaryKey(), // ❌ 不一致
  id: serial("id").primaryKey(), // ✅ 一致
  firstName: varchar("first_name", { length: 50 }).notNull(), // ✅
  createdAt: timestamp("created_at").defaultNow(), // ✅
});

// ❌ 避免：使用 camelCase（数据库命名）
export const users = pgTable("users", {
  firstName: varchar("firstName", { length: 50 }), // 数据库列名是 firstName
  // SQL：firstName VARCHAR(50) -- 不是标准 SQL 命名
});
```

### 2. 字段长度

```typescript
// ✅ 推荐：根据实际需求设置长度
export const users = pgTable("users", {
  email: varchar("email", { length: 255 }), // ✅ 标准邮箱长度
  username: varchar("username", { length: 50 }), // ✅ 用户名通常较短
  bio: text("bio"), // ✅ 不确定长度用 text
});

// ❌ 避免：过长或不设置长度
export const users = pgTable("users", {
  email: varchar("email", { length: 10000 }), // ❌ 过度
  username: varchar("username"), // ❌ PostgreSQL varchar 必须指定长度
});
```

### 3. 索引策略

```typescript
// ✅ 推荐：只为经常查询的字段创建索引
export const posts = pgTable(
  "posts",
  {
    id: serial("id").primaryKey(),
    title: varchar("title", { length: 255 }),
    authorId: integer("author_id"),
    viewCount: integer("view_count"),
  },
  (table) => ({
    // 经常查询的字段
    authorIdIdx: index("author_idx").on(table.authorId), // ✅
    // 复合查询
    authorViewIdx: index("author_view_idx").on(table.authorId, table.viewCount), // ✅

    // ❌ 不要为每个字段都创建索引
    // viewCountIdx: index('view_count_idx').on(table.viewCount),  // 不必要
  })
);
```

### 4. 外键约束

```typescript
// ✅ 推荐：设置合适的 onDelete 行为
export const posts = pgTable("posts", {
  id: serial("id").primaryKey(),
  authorId: integer("author_id").references(() => users.id, {
    onDelete: "cascade", // ✅ 删除用户时，同时删除文章
  }),
  categoryId: integer("category_id").references(() => categories.id, {
    onDelete: "restrict", // ✅ 有关联文章时，禁止删除分类
  }),
});
```

## 总结

### 核心要点

1. **PostgreSQL 类型映射**
   - 整数：serial、integer、bigint
   - 字符串：varchar、text、char
   - 时间：timestamp、date、time
   - JSON：json、jsonb
   - 数组：array()

2. **表关系**
   - 一对一：外键 + unique
   - 一对多：外键
   - 多对多：中间表

3. **索引和约束**
   - 单列索引
   - 复合索引
   - 唯一索引
   - 检查约束

4. **最佳实践**
   - snake_case 命名
   - 合理的字段长度
   - 适当的索引策略
   - 正确的外键约束
