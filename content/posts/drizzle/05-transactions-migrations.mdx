## 一、数据库事务基础

### 1.1 什么是事务？

事务（Transaction）是数据库操作的逻辑单元，保证一组操作**要么全部成功，要么全部失败**。

**ACID 特性**：

- **A**tomicity（原子性）：事务中的操作要么全部完成，要么全部不完成
- **C**onsistency（一致性）：事务执行前后，数据库状态保持一致
- **I**solation（隔离性）：并发事务之间互不干扰
- **D**urability（持久性）：事务提交后，修改永久保存

### 1.2 基础事务

```typescript
import { db } from "./db";

// 简单事务
await db.transaction(async (tx) => {
  // 在事务中执行操作
  await tx.insert(users).values({
    email: "test@example.com",
    name: "Test User",
  });

  await tx.insert(posts).values({
    title: "First Post",
    content: "Hello World",
    authorId: 1, // 假设刚创建的用户 ID 是 1
  });

  // 如果发生错误，事务会自动回滚
  // 如果成功，事务会自动提交
});
```

### 1.3 事务回滚

```typescript
await db.transaction(async (tx) => {
  const [user] = await tx
    .insert(users)
    .values({
      email: "test@example.com",
      name: "Test User",
    })
    .returning();

  // 检查业务规则
  const existingPosts = await tx
    .select()
    .from(posts)
    .where(eq(posts.authorId, user.id));

  if (existingPosts.length >= 10) {
    // 抛出错误会触发回滚
    throw new Error("User has too many posts");
  }

  await tx.insert(posts).values({
    title: "New Post",
    content: "Content",
    authorId: user.id,
  });
});
```

## 二、实际事务场景

### 2.1 转账（原子操作）

```typescript
import { eq, and, sql } from "drizzle-orm";

export const accounts = pgTable("accounts", {
  id: serial("id").primaryKey(),
  userId: integer("user_id").references(() => users.id),
  balance: integer("balance").notNull().default(0),
});

async function transferMoney(
  fromUserId: number,
  toUserId: number,
  amount: number
) {
  await db.transaction(async (tx) => {
    // 检查余额
    const [fromAccount] = await tx
      .select()
      .from(accounts)
      .where(eq(accounts.userId, fromUserId))
      .for("update") // 锁定行
      .limit(1);

    if (!fromAccount) {
      throw new Error("Source account not found");
    }

    if (fromAccount.balance < amount) {
      throw new Error("Insufficient balance");
    }

    // 扣除金额
    await tx
      .update(accounts)
      .set({
        balance: sql`${accounts.balance} - ${amount}`,
      })
      .where(eq(accounts.userId, fromUserId));

    // 增加金额
    await tx
      .update(accounts)
      .set({
        balance: sql`${accounts.balance} + ${amount}`,
      })
      .where(eq(accounts.userId, toUserId));

    // 记录交易
    await tx.insert(transactions).values({
      fromUserId,
      toUserId,
      amount,
      type: "transfer",
    });
  });
}
```

### 2.2 订单创建（库存检查）

```typescript
export const products = pgTable("products", {
  id: serial("id").primaryKey(),
  name: varchar("name", { length: 255 }).notNull(),
  stock: integer("stock").notNull().default(0),
  price: integer("price").notNull(),
});

export const orders = pgTable("orders", {
  id: serial("id").primaryKey(),
  userId: integer("user_id").references(() => users.id),
  total: integer("total").notNull(),
  status: varchar("status", { length: 50 }).default("pending"),
  createdAt: timestamp("created_at").defaultNow(),
});

export const orderItems = pgTable("order_items", {
  id: serial("id").primaryKey(),
  orderId: integer("order_id").references(() => orders.id),
  productId: integer("product_id").references(() => products.id),
  quantity: integer("quantity").notNull(),
  price: integer("price").notNull(),
});

async function createOrder(
  userId: number,
  items: Array<{ productId: number; quantity: number }>
) {
  return await db.transaction(async (tx) => {
    // 检查库存
    for (const item of items) {
      const [product] = await tx
        .select()
        .from(products)
        .where(eq(products.id, item.productId))
        .for("update")
        .limit(1);

      if (!product) {
        throw new Error(`Product ${item.productId} not found`);
      }

      if (product.stock < item.quantity) {
        throw new Error(`Insufficient stock for ${product.name}`);
      }
    }

    // 创建订单
    const [order] = await tx
      .insert(orders)
      .values({
        userId,
        status: "pending",
        total: 0, // 稍后计算
      })
      .returning();

    // 创建订单项并扣减库存
    let total = 0;

    for (const item of items) {
      const [product] = await tx
        .select()
        .from(products)
        .where(eq(products.id, item.productId))
        .limit(1);

      const subtotal = product.price * item.quantity;
      total += subtotal;

      await tx.insert(orderItems).values({
        orderId: order.id,
        productId: item.productId,
        quantity: item.quantity,
        price: product.price,
      });

      // 扣减库存
      await tx
        .update(products)
        .set({
          stock: sql`${products.stock} - ${item.quantity}`,
        })
        .where(eq(products.id, item.productId));
    }

    // 更新订单总额
    await tx.update(orders).set({ total }).where(eq(orders.id, order.id));

    return order;
  });
}
```

## 三、事务隔离级别

### 3.1 隔离级别说明

PostgreSQL 支持的隔离级别：

```typescript
import { sql } from "drizzle-orm";

// READ UNCOMMITTED（未提交读）- PostgreSQL 不支持，会降级到 READ COMMITTED
// READ COMMITTED（提交读）- 默认级别
// REPEATABLE READ（可重复读）- PostgreSQL 实际上是 SERIALIZABLE 的快照隔离
// SERIALIZABLE（可序列化）- 最高隔离级别
```

### 3.2 设置隔离级别

```typescript
// 设置事务隔离级别
await db.transaction(
  async (tx) => {
    // 事务操作
    await tx.insert(users).values({ email: "test@example.com" });
  },
  {
    isolationLevel: "serializable", // 或 'read committed', 'repeatable read'
  }
);
```

### 3.3 行级锁

```typescript
// SELECT ... FOR UPDATE - 排他锁
const [account] = await tx
  .select()
  .from(accounts)
  .where(eq(accounts.id, 1))
  .for("update") // 锁定行，其他事务不能修改
  .limit(1);

// SELECT ... FOR SHARE - 共享锁
const [account] = await tx
  .select()
  .from(accounts)
  .where(eq(accounts.id, 1))
  .for("share") // 其他事务可以读取但不能修改
  .limit(1);
```

### 5.1 NOT NULL 约束

```typescript
export const users = pgTable("users", {
  id: serial("id").primaryKey(),
  email: varchar("email", { length: 255 })
    .notNull() // 不允许 NULL
    .unique(),
  name: text("name"), // 允许 NULL

```

### 5.2 UNIQUE 约束

```typescript
export const users = pgTable("users", {
  id: serial("id").primaryKey(),
  email: varchar("email", { length: 255 }).unique(),
  username: varchar("username", { length: 50 }).unique(),
});

// 组合唯一
export const votes = pgTable(
  "votes",
  {
    id: serial("id").primaryKey(),
    userId: integer("user_id").notNull(),
    postId: integer("post_id").notNull(),
  },
  (table) => ({
    // 一个用户对一篇文章只能投一票
    uniqueVote: unique("unique_vote").on(table.userId, table.postId),
  })

```

### 5.3 CHECK 约束

```typescript
import { pgTable, serial, integer, check } from "drizzle-orm/pg-core";

export const products = pgTable(
  "products",
  {
    id: serial("id").primaryKey(),
    name: varchar("name", { length: 255 }).notNull(),
    price: integer("price").notNull(),
    stock: integer("stock").notNull(),
  },
  (table) => ({
    // 价格必须大于 0
    priceCheck: check("price_check", sql`${table.price} > 0`),
    // 库存不能为负数
    stockCheck: check("stock_check", sql`${table.stock} >= 0`),
  })
);
```

### 5.4 外键约束

```typescript
export const posts = pgTable("posts", {
  id: serial("id").primaryKey(),
  title: varchar("title", { length: 255 }).notNull(),
  authorId: integer("author_id")
    .references(() => users.id, {
      onDelete: "cascade", // 删除用户时删除文章
      onUpdate: "cascade", // 更新用户 ID 时更新文章
    })
    .notNull(),
});
```

外键选项：

| 选项          | 说明                                      |
| ------------- | ----------------------------------------- |
| `CASCADE`     | 删除/更新父记录时，自动删除/更新子记录    |
| `RESTRICT`    | 如果有子记录，阻止删除/更新父记录（默认） |
| `SET NULL`    | 删除/更新父记录时，子记录的外键设为 NULL  |
| `SET DEFAULT` | 删除/更新父记录时，子记录的外键设为默认值 |
| `NO ACTION`   | 与 RESTRICT 类似，但检查时机不同          |

## 七、实战示例

### 7.1 电商订单处理

```typescript
async function processOrder(orderId: number) {
  return await db.transaction(async (tx) => {
    // 1. 锁定订单
    const [order] = await tx
      .select()
      .from(orders)
      .where(eq(orders.id, orderId))
      .for("update")
      .limit(1);

    if (!order) {
      throw new Error("Order not found");
    }

    if (order.status !== "pending") {
      throw new Error("Order already processed");
    }

    // 2. 获取订单项
    const items = await tx
      .select()
      .from(orderItems)
      .where(eq(orderItems.orderId, orderId));

    // 3. 检查并扣减库存
    for (const item of items) {
      const [product] = await tx
        .select()
        .from(products)
        .where(eq(products.id, item.productId))
        .for("update")
        .limit(1);

      if (!product || product.stock < item.quantity) {
        throw new Error(`Insufficient stock for product ${item.productId}`);
      }

      await tx
        .update(products)
        .set({
          stock: sql`${products.stock} - ${item.quantity}`,
        })
        .where(eq(products.id, item.productId));
    }

    // 4. 更新订单状态
    await tx
      .update(orders)
      .set({ status: "processing" })
      .where(eq(orders.id, orderId));

    // 5. 创建支付记录
    await tx.insert(payments).values({
      orderId,
      amount: order.total,
      status: "pending",
    });

    return order;
  });
}
```

## 八、最佳实践

### 8.1 事务边界

```typescript
// ✅ 好的做法：事务包含所有相关操作
await db.transaction(async (tx) => {
  await tx.insert(users).values({ email: "test@example.com" });
  await tx.insert(posts).values({ title: "Hello", authorId: 1 });
});

// ❌ 不好的做法：分散事务
await db.transaction(async (tx) => {
  await tx.insert(users).values({ email: "test@example.com" });
});
await db.transaction(async (tx) => {
  await tx.insert(posts).values({ title: "Hello", authorId: 1 });
});
```

### 8.2 锁定顺序

```typescript
// ✅ 始终按相同顺序锁定表（避免死锁）
await db.transaction(async (tx) => {
  await tx.select().from(users).for("update");
  await tx.select().from(posts).for("update");
});

// ❌ 不同顺序可能导致死锁
// 事务 1：users → posts
// 事务 2：posts → users（死锁！）
```

### 8.3 事务大小

```typescript
// ✅ 保持事务简短
await db.transaction(async (tx) => {
  // 只包含必要的数据库操作
  await tx.update(accounts).set({ balance: newBalance });
  await tx.insert(transactions).values({ amount });
});

// ❌ 避免在事务中执行耗时操作
await db.transaction(async (tx) => {
  await tx.update(accounts).set({ balance: newBalance });

  // 不要！发送邮件
  await sendEmail(user.email, "Transfer complete");

  await tx.insert(transactions).values({ amount });
});
```
