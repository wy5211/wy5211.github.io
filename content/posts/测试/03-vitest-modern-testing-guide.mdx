## 前言

如果你厌倦了 Jest 慢吞吞的启动速度和复杂的配置，那么 Vitest 绝对会让你眼前一亮。作为由 Vite 团队官方维护的测试框架，Vitest 以「极速」和「开箱即用」为核心设计理念，正在成为现代前端项目的首选测试工具。

本文将从零开始，带你全面掌握 Vitest 的使用方法和最佳实践。

## 为什么选择 Vitest？

### 1. 极速冷启动

Vitest 共享 Vite 的配置和转换管线，无需像 Jest 那样预编译整个项目：

```bash
# Jest 启动时间
Test Suites: 20, Time: 12.5s

# Vitest 启动时间
Test Suites: 20, Time: 1.2s
```

### 2. 原生 ESM 支持

无需配置 Babel 或 ts-jest，直接支持：

- ES Modules
- TypeScript
- JSX
- CSS 注入

### 3. 智能监听模式

Vitest 的 watch 模式只会重新运行与修改文件相关的测试，而非全部重跑。

### 4. 兼容 Jest API

大部分 Jest 测试可以直接迁移，学习成本极低。

## 快速开始

### 安装

```bash
# npm
npm install -D vitest

# pnpm
pnpm add -D vitest

# yarn
yarn add -D vitest
```

### 最简配置

在 `vite.config.ts` 中添加：

```typescript
import { defineConfig } from "vite";
import react from "@vitejs/plugin-react";

export default defineConfig({
  plugins: [react()],
  test: {
    // 测试配置
  },
});
```

### 第一个测试

创建 `sum.test.ts`：

```typescript
import { describe, it, expect } from "vitest";
import { sum } from "./sum";

describe("sum 函数", () => {
  it("应该正确相加两个数字", () => {
    expect(sum(1, 2)).toBe(3);
  });

  it("应该处理负数", () => {
    expect(sum(-1, -2)).toBe(-3);
  });
});
```

运行测试：

```bash
npx vitest
```

## 核心特性详解

### 断言 API

Vitest 兼容 Chai 断言库，同时扩展了 Jest 的 `expect` 语法：

```typescript
import { expect, test } from "vitest";

test("断言示例", () => {
  const obj = { name: "vitest", version: "1.0" };

  // 基础断言
  expect(1 + 1).toBe(2);
  expect(obj).toEqual({ name: "vitest", version: "1.0" });
  expect(obj).toHaveProperty("name");

  // 真值检查
  expect(true).toBeTruthy();
  expect(null).toBeFalsy();
  expect(undefined).toBeUndefined();

  // 数字比较
  expect(5).toBeGreaterThan(3);
  expect(5).toBeLessThanOrEqual(5);
  expect(0.1 + 0.2).toBeCloseTo(0.3);

  // 字符串匹配
  expect("hello vitest").toContain("vitest");
  expect("hello vitest").toMatch(/vitest/);

  // 数组检查
  expect([1, 2, 3]).toContain(2);
  expect([1, 2, 3]).toHaveLength(3);

  // 异常测试
  expect(() => {
    throw new Error("oops");
  }).toThrow("oops");
});
```

### Mock 函数

Vitest 内置了强大的 Mock 功能（基于 tinyspy）：

```typescript
import { vi, describe, it, expect, beforeEach, afterEach } from "vitest";

describe("Mock 示例", () => {
  beforeEach(() => {
    // 每个 test 前重置所有 mock
    vi.clearAllMocks();
  });

  it("应该追踪函数调用", () => {
    const callback = vi.fn();

    callback("hello");
    callback("world");

    expect(callback).toHaveBeenCalledTimes(2);
    expect(callback).toHaveBeenNthCalledWith(1, "hello");
    expect(callback).toHaveBeenNthCalledWith(2, "world");
  });

  it("应该返回模拟值", () => {
    const mockFn = vi.fn();
    mockFn.mockReturnValue("mocked");

    expect(mockFn()).toBe("mocked");
  });

  it("应该返回动态值", () => {
    const mockFn = vi.fn((x) => x * 2);

    expect(mockFn(5)).toBe(10);
    expect(mockFn(3)).toBe(6);
  });
});
```

### 模块 Mock

模拟整个模块或部分导出：

```typescript
// utils.ts
export const fetchData = async () => {
  const res = await fetch("/api/data");
  return res.json();
};

// utils.test.ts
import { vi, describe, it, expect } from "vitest";

// 模拟 fetch
global.fetch = vi.fn(() =>
  Promise.resolve({
    json: () => Promise.resolve({ data: "mocked" }),
  })
) as any;

describe("fetchData", () => {
  it("应该返回数据", async () => {
    const result = await fetchData();
    expect(result).toEqual({ data: "mocked" });
  });
});
```

部分模块 Mock：

```typescript
import { vi, describe, it, expect } from "vitest";
import * as utils from "./utils";

vi.mock("./utils", async (importOriginal) => {
  const mod = await importOriginal<typeof utils>();
  return {
    ...mod,
    // 只 mock 特定函数
    fetchData: vi.fn(() => Promise.resolve({ mocked: true })),
  };
});
```

### 定时器 Mock

控制 `setTimeout`、`setInterval` 等定时器：

```typescript
import { vi, describe, it, expect, beforeEach, afterEach } from "vitest";

describe("定时器测试", () => {
  beforeEach(() => {
    vi.useFakeTimers();
  });

  afterEach(() => {
    vi.useRealTimers();
  });

  it("应该在延迟后执行", () => {
    const callback = vi.fn();

    setTimeout(callback, 1000);

    // 快进 500ms - 还没执行
    vi.advanceTimersByTime(500);
    expect(callback).not.toHaveBeenCalled();

    // 再快进 500ms - 已执行
    vi.advanceTimersByTime(500);
    expect(callback).toHaveBeenCalled();
  });

  it("应该跳转到指定时间", () => {
    const callback = vi.fn();
    setInterval(callback, 100);

    vi.advanceTimersToNextTimer();
    expect(callback).toHaveBeenCalledTimes(1);

    vi.runAllTimers();
    // 无限循环会继续...
  });
});
```

## 高级特性

### UI 模式

Vitest 提供了漂亮的可视化界面：

```bash
npx vitest --ui
```

- 实时更新测试结果

### 覆盖率报告

内置 Istanbul 覆盖率支持：

```bash
npx vitest --coverage
```

配置覆盖率：

```typescript
// vite.config.ts
export default defineConfig({
  test: {
    coverage: {
      provider: "istanbul", // 或 'v8'
      reporter: ["text", "json", "html"],
      exclude: ["node_modules/", "test/"],
      lines: 80, // 行覆盖率阈值
      functions: 80, // 函数覆盖率阈值
      branches: 70, // 分支覆盖率阈值
      statements: 80, // 语句覆盖率阈值
    },
  },
});
```

### 测试隔离

每个测试文件在独立环境中运行：

```typescript
import { describe, test, expect } from "vitest";

describe.sequential("顺序执行", () => {
  // 这些测试会按顺序执行
  test("第一步", () => {});
  test("第二步", () => {});
});

describe.only("只运行这个", () => {
  test("重要测试", () => {});
});

describe.skip("跳过这些", () => {
  test("暂不测试", () => {});
});

test.todo("待实现的测试");
```

### 并行执行

Vitest 默认并行运行测试文件，可配置：

```typescript
export default defineConfig({
  test: {
    pool: "threads", // 'threads' | 'forks' | 'vmThreads'
    poolOptions: {
      threads: {
        singleThread: false,
        minThreads: 1,
        maxThreads: 4,
      },
    },
  },
});
```

### 基准测试（Benchmark）

Vitest 内置性能基准测试：

```typescript
// bench.sort.test.ts
import { bench, describe } from 'vitest'

describe('排序算法性能', () => {
  const arr = Array.from({ length: 10000 }, () => Math.random())

  bench('Array.sort()', () => {
    ;[...arr].sort((a, b) => a - b)
  })

  bench('冒泡排序', () => {
    const result = [...arr]
    for (let i = 0; i &lt; result.length; i++) {
      for (let j = 0; j &lt; result.length - i - 1; j++) {
        if (result[j] &gt; result[j + 1]) {
          ;[result[j], result[j + 1]] = [result[j + 1], result[j]]
        }
      }
    }
  })
})
```

运行基准测试：

```bash
npx vitest bench
```

Setup 文件：

```typescript

import "@testing-library/jest-dom";
```

测试组件：

```typescript
// Counter.test.tsx
import { render, screen, fireEvent } from '@testing-library/react'
import { describe, it, expect } from 'vitest'
import Counter from './Counter'

describe('Counter 组件', () => {
  it('应该渲染初始值', () => {
    render(&lt;Counter initialCount={5} /&gt;)
    expect(screen.getByText('5')).toBeInTheDocument()
  })

  it('点击按钮应该增加计数', () => {
    render(&lt;Counter initialCount={0} /&gt;)

    const button = screen.getByRole('button', { name: /increment/i })
    fireEvent.click(button)

    expect(screen.getByText('1')).toBeInTheDocument()
  })
})
```

## 从 Jest 迁移

### 兼容模式

Vitest 提供了 Jest 兼容模式：

```typescript
// vite.config.ts
export default defineConfig({
  test: {
    globals: true, // 全局 API
    environment: "jsdom",
    setupFiles: ["./setup.ts"],
  },
});
```

### 常见迁移问题

| Jest                   | Vitest                                 |
| ---------------------- | -------------------------------------- |
| `jest.fn()`            | `vi.fn()`                              |
| `jest.mock()`          | `vi.mock()`                            |
| `jest.spyOn()`         | `vi.spyOn()`                           |
| `jest.useFakeTimers()` | `vi.useFakeTimers()`                   |
| `jest.setTimeout()`    | `vi.setConfig({ testTimeout: 10000 })` |

如果大量使用全局 `jest` 对象，可以创建别名：

```typescript
// setup.ts
import { vi } from "vitest";

global.jest = vi as any;
```

## 最佳实践

### 1. 测试文件组织

```
src/
├── components/
│   ├── Button.tsx
│   └── Button.test.tsx    # 同目录放置测试
├── utils/
│   ├── format.ts
│   └── format.test.ts
└── __tests__/
    └── integration/        # 集成测试单独目录
```

### 2. 使用 describe 组织测试

```typescript
describe("UserService", () => {
  describe("createUser", () => {
    it("应该创建用户");
    it("应该验证邮箱");
    it("应该拒绝重复用户名");
  });

  describe("deleteUser", () => {
    it("应该删除用户");
    it("应该处理不存在的用户");
  });
});
```

### 3. 遵循 AAA 模式

```typescript
it("应该正确计算总价", () => {
  // Arrange - 准备
  const cart = [
    { price: 100, quantity: 2 },
    { price: 50, quantity: 1 },
  ];

  // Act - 执行
  const total = calculateTotal(cart);

  // Assert - 断言
  expect(total).toBe(250);
});
```

### 4. 测试边界条件

```typescript
describe("divide 函数", () => {
  it("应该正确除法", () => {
    expect(divide(10, 2)).toBe(5);
  });

  it("应该处理除以零", () => {
    expect(() => divide(10, 0)).toThrow("Division by zero");
  });

  it("应该处理浮点数", () => {
    expect(divide(1, 3)).toBeCloseTo(0.333, 2);
  });
});
```

### 5. 使用 test.each 减少重复

```typescript
test.each([
  [1, 1, 2],
  [1, 2, 3],
  [2, 2, 4],
])("add(%i, %i) = %i", (a, b, expected) => {
  expect(add(a, b)).toBe(expected);
});

// 或使用模板字符串
test.each`
  a    | b    | expected
  ${1} | ${1} | ${2}
  ${1} | ${2} | ${3}
  ${2} | ${2} | ${4}
`("add($a, $b) = $expected", ({ a, b, expected }) => {
  expect(add(a, b)).toBe(expected);
});
```

## 性能优化

### 1. 减少测试文件数量

合并相关测试到单个文件，减少 worker 启动开销。

### 2. 使用 --shard 进行分片

在 CI 中分片运行：

```bash
# CI Job 1
npx vitest --shard=1/3

# CI Job 2
npx vitest --shard=2/3

# CI Job 3
npx vitest --shard=3/3
```

### 3. 优化 mock

避免在全局 setup 中创建大量 mock：

```typescript
// ❌ 不好
// setup.ts
vi.mock("heavy-module");

// ✅ 好
// 在需要的测试文件中 mock
vi.mock("heavy-module", async () => {
  const mod = await import("heavy-module");
  return { ...mod, expensive: vi.fn() };
});
```

## 总结

Vitest 作为新一代前端测试框架，凭借以下优势成为现代项目的理想选择：

| 特性         | Vitest      | Jest       |
| ------------ | ----------- | ---------- |
| 启动速度     | ⚡ 极快     | 较慢       |
| ESM 支持     | ✅ 原生     | 需要 Babel |
| 配置复杂度   | 🎯 零配置   | 较复杂     |
| Watch 模式   | 🔄 智能增量 | 全量重跑   |
| 与 Vite 集成 | ✅ 完美     | 需额外配置 |
| UI 模式      | ✅ 内置     | 需插件     |
| 基准测试     | ✅ 内置     | 需额外工具 |

如果你的项目使用 Vite，Vitest 几乎是不二之选。即使是非 Vite 项目，Vitest 的速度和开发体验也值得一试。
