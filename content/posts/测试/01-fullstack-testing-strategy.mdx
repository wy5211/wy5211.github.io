# 全栈测试策略实战：从单元测试到 E2E 测试的完整指南

测试是保证代码质量的基石，但在全栈项目中，如何制定合理的测试策略却是一门艺术。测试太少，bug 频发；测试太多，开发效率下降。本文将带你建立一套高效、实用的全栈测试体系。

## 测试金字塔：理解测试层次

在开始之前，我们需要理解经典的**测试金字塔**模型：

```
        /\
       /E2\      ← 端到端测试（少量）
      /----\
     /集成测试\   ← 集成测试（适量）
    /----------\
   /   单元测试  \ ← 单元测试（大量）
  /--------------\
```

- **单元测试**：测试独立函数/组件，速度快，数量多
- **集成测试**：测试模块间协作，覆盖关键流程
- **E2E 测试**：模拟真实用户操作，覆盖核心业务

**黄金法则**：70% 单元测试 + 20% 集成测试 + 10% E2E 测试

## 一、单元测试：代码质量的基石

### 1.1 工具选择：Jest vs Vitest

| 特性       | Jest     | Vitest            |
| ---------- | -------- | ----------------- |
| 启动速度   | 较慢     | 极快（Vite 驱动） |
| ESM 支持   | 需配置   | 原生支持          |
| TypeScript | 需配置   | 开箱即用          |
| 生态成熟度 | 非常成熟 | 快速发展          |

**推荐**：新项目用 Vitest，遗留项目用 Jest

```typescript
// src/utils/calculations.ts
export function calculateDiscount(
  price: number,
  discountPercent: number
): number {
  if (price < 0) throw new Error("Price cannot be negative");
  if (discountPercent < 0 || discountPercent > 100) {
    throw new Error("Discount must be between 0 and 100");
  }
  return Math.round(price * (1 - discountPercent / 100) * 100) / 100;
}

export function formatCurrency(amount: number, currency = "CNY"): string {
  return new Intl.NumberFormat("zh-CN", {
    style: "currency",
    currency,
  }).format(amount);
}
```

```typescript
// src/hooks/useCounter.ts
import { useState, useCallback } from "react";

export function useCounter(initialValue = 0) {
  const [count, setCount] = useState(initialValue);

  const increment = useCallback(() => setCount((c) => c + 1), []);
  const decrement = useCallback(() => setCount((c) => c - 1), []);
  const reset = useCallback(() => setCount(initialValue), [initialValue]);

  return { count, increment, decrement, reset };
}
```

```typescript
// src/api/users.ts
import { z } from "zod";

const UserSchema = z.object({
  id: z.number(),
  name: z.string(),
  email: z.string().email(),
});

export type User = z.infer<typeof UserSchema>;

export async function fetchUser(id: number): Promise<User> {
  const response = await fetch(`/api/users/${id}`);

  if (!response.ok) {
    throw new Error(`Failed to fetch user: ${response.status}`);
  }

  const data = await response.json();
  return UserSchema.parse(data);
}

export async function updateUser(
  id: number,
  data: Partial<User>
): Promise<User> {
  const response = await fetch(`/api/users/${id}`, {
    method: "PATCH",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify(data),
  });

  if (!response.ok) {
    throw new Error(`Failed to update user: ${response.status}`);
  }

  return UserSchema.parse(await response.json());
}
```

### 2.2 数据库集成测试（后端）

```typescript
// src/services/user.service.ts
import { db } from "../db";
import { users } from "../db/schema";
import { eq } from "drizzle-orm";

export class UserService {
  async findById(id: number) {
    const result = await db.select().from(users).where(eq(users.id, id));
    return result[0] ?? null;
  }

  async create(data: { name: string; email: string }) {
    const result = await db.insert(users).values(data).returning();
    return result[0];
  }

  async update(id: number, data: Partial<{ name: string; email: string }>) {
    const result = await db
      .update(users)
      .set(data)
      .where(eq(users.id, id))
      .returning();
    return result[0] ?? null;
  }

  async delete(id: number) {
    const result = await db.delete(users).where(eq(users.id, id)).returning();
    return result.length > 0;
  }
}
```

```typescript
import type { User } from "../../api/users";

const users: User[] = [
  { id: 1, name: "John Doe", email: "john@example.com" },
  { id: 2, name: "Jane Smith", email: "jane@example.com" },
];

export const handlers = [
  http.get("/api/users/:id", async ({ params }) => {
    await delay(100); // 模拟网络延迟

    const id = Number(params.id);
    const user = users.find((u) => u.id === id);

    if (!user) {
      return new HttpResponse(null, { status: 404 });
    }

    return HttpResponse.json(user);
  }),

  http.patch("/api/users/:id", async ({ params, request }) => {
    const id = Number(params.id);
    const body = (await request.json()) as Partial<User>;

    const index = users.findIndex((u) => u.id === id);
    if (index === -1) {
      return new HttpResponse(null, { status: 404 });
    }

    users[index] = { ...users[index], ...body };
    return HttpResponse.json(users[index]);
  }),

  http.post("/api/auth/login", async ({ request }) => {
    const { email, password } = (await request.json()) as any;

    if (email === "test@example.com" && password === "password123") {

        token: "mock-jwt-token",
        user: { id: 1, name: "Test User", email },
      });
    }

    return new HttpResponse(
      JSON.stringify({ message: "Invalid credentials" }),
      { status: 401 }
    );
  }),
```

### 4.1 AAA 模式

```typescript
it("should calculate total with discount", () => {
  // Arrange（准备）
  const items = [
    { price: 100, quantity: 2 },
    { price: 50, quantity: 1 },
  ];
  const discount = 10;

  // Act（执行）
  const total = calculateTotal(items, discount);

  // Assert（断言）

});
```

### 4.2 测试命名规范


### 4.3 避免测试实现细节

```typescript

  expect(result.current.isLoading).toBe(true) // 内部状态

// ✅ 测试行为

  render(<UserProfile userId={1} />)

    expect(screen.queryByTestId('loading-indicator')).not.toBeInTheDocument()
  })

```

## 总结

一套好的测试策略应该是：

1. **分层合理**：遵循测试金字塔，单元测试为主
2. **速度快**：单元测试秒级，E2E 分钟级
3. **可维护**：避免测试实现细节，使用稳定选择器
4. **有价值**：测试关键业务逻辑，而非为了覆盖率

**记住**：测试的目的是给你信心重构，而不是束缚你的手脚。写有价值的测试，而不是写最多的测试。
