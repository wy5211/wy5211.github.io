## 前言

在现代 Web 开发中，端到端（E2E）测试是保证产品质量的关键环节。Playwright 作为微软开源的新一代测试框架，以其强大的跨浏览器支持、稳定的测试执行和出色的开发者体验，迅速成为前端测试领域的热门选择。

本文将从零开始，带你深入掌握 Playwright 的核心特性和最佳实践。

## 为什么选择 Playwright？

### Playwright vs Cypress vs Selenium

| 特性         | Playwright                 | Cypress                     | Selenium       |
| ------------ | -------------------------- | --------------------------- | -------------- |
| 跨浏览器支持 | Chrome/Firefox/Safari/Edge | Chrome/Firefox/Edge（有限） | 所有主流浏览器 |
| 测试速度     | ⭐⭐⭐⭐⭐                 | ⭐⭐⭐⭐                    | ⭐⭐⭐         |
| 并行执行     | 原生支持                   | 需付费                      | 支持           |
| 网络拦截     | 强大                       | 支持                        | 有限           |
| 学习曲线     | 中等                       | 简单                        | 较陡           |
| 生态成熟度   | 快速发展中                 | 成熟                        | 非常成熟       |

### Playwright 核心优势

1. **真正的跨浏览器** - 一套代码，所有浏览器
2. **自动等待机制** - 无需手动添加 sleep/wait
3. **强大的选择器引擎** - 支持 CSS、XPath、Text 等多种选择器
4. **网络拦截和 Mock** - 轻松模拟 API 响应
5. **内置测试工具** - Trace Viewer、Codegen、Inspector
6. **优秀的 TypeScript 支持** - 完整的类型提示

## 核心概念深入

### Page Object Model（页面对象模式）

POM 是测试代码组织的最佳实践，将页面逻辑与测试逻辑分离：

```typescript
// pages/LoginPage.ts
import { Page, Locator } from "@playwright/test";

export class LoginPage {
  readonly page: Page;
  readonly usernameInput: Locator;
  readonly passwordInput: Locator;
  readonly loginButton: Locator;
  readonly errorMessage: Locator;

  constructor(page: Page) {
    this.page = page;
    this.usernameInput = page.locator("#username");
    this.passwordInput = page.locator("#password");
    this.loginButton = page.locator('button[type="submit"]');
    this.errorMessage = page.locator(".error-message");
  }

  async goto() {
    await this.page.goto("/login");
  }

  async login(username: string, password: string) {
    await this.usernameInput.fill(username);
    await this.passwordInput.fill(password);
    await this.loginButton.click();
  }

  async expectError(message: string) {
    await expect(this.errorMessage).toHaveText(message);
  }
}
```

在测试中使用：

```typescript
// tests/auth/login.spec.ts
import { test, expect } from "@playwright/test";
import { LoginPage } from "../../pages/LoginPage";

test.describe("登录功能", () => {
  let loginPage: LoginPage;

  test.beforeEach(async ({ page }) => {
    loginPage = new LoginPage(page);
    await loginPage.goto();
  });

  test("成功登录", async ({ page }) => {
    await loginPage.login("testuser", "password123");
    await expect(page).toHaveURL(/\/dashboard/);
  });

  test("密码错误显示提示", async () => {
    await loginPage.login("testuser", "wrongpassword");
    await loginPage.expectError("用户名或密码错误");
  });
});
```

### 高级选择器技巧

Playwright 支持多种选择器策略：

```typescript
// CSS 选择器
await page.locator(".card").click();
await page.locator("#submit-button").click();

// 文本选择器
await page.getByText("欢迎回来").click();
await page.getByText(/^添加到购物车$/).click(); // 精确匹配

// 角色选择器（推荐 - 基于可访问性）
await page.getByRole("button", { name: "提交" }).click();
await page.getByRole("link", { name: "了解更多" }).click();
await page.getByRole("textbox", { name: "用户名" }).fill("test");

// 标签选择器
await page.getByLabel("邮箱地址").fill("test@example.com");

// 占位符选择器
await page.getByPlaceholder("请输入搜索内容").fill("Playwright");

// 测试 ID 选择器（最稳定）
await page.getByTestId("submit-button").click();

// 组合选择器
await page
  .locator("article")
  .filter({ hasText: "Playwright" })
  .getByRole("button", { name: "阅读更多" })
  .click();

// 链式选择器
await page
  .locator(".product-card")
  .locator(".price")
  .filter({ hasText: "¥99" });
```

### 等待策略

Playwright 的自动等待机制：

```typescript
// 自动等待 - 元素可操作时才执行
await page.click("button"); // 自动等待元素可见、可点击

// 显式等待状态
await expect(page.locator(".loading")).not.toBeVisible();
await expect(page.locator(".result")).toBeVisible();

// 等待特定条件
await page.waitForURL("**/dashboard");
await page.waitForResponse((resp) => resp.url().includes("/api/data"));

// 等待元素数量
await expect(page.locator(".item")).toHaveCount(5);

// 等待元素包含文本
await expect(page.locator(".status")).toContainText("成功");
```

## 网络请求处理

### API Mock 和拦截

```typescript
import { test, expect } from "@playwright/test";

test("Mock API 响应", async ({ page }) => {
  // 拦截并 Mock API 响应
  await page.route("**/api/users", async (route) => {
    const json = {
      users: [
        { id: 1, name: "张三" },
        { id: 2, name: "李四" },
      ],
    };
    await route.fulfill({ json });
  });

  await page.goto("/users");
  await expect(page.getByText("张三")).toBeVisible();
});

test("修改请求头", async ({ page }) => {
  await page.route("**/api/**", async (route) => {
    const headers = {
      ...route.request().headers(),
      Authorization: "Bearer test-token",
    };
    await route.continue({ headers });
  });

  await page.goto("/dashboard");
});

test("模拟网络错误", async ({ page }) => {
  await page.route("**/api/data", (route) => route.abort("failed"));

  await page.goto("/dashboard");
  await expect(page.locator(".error-message")).toHaveText("网络错误");
});

test("延迟响应模拟慢网络", async ({ page }) => {
  await page.route("**/api/data", async (route) => {
    await new Promise((resolve) => setTimeout(resolve, 3000));
    await route.continue();
  });

  await page.goto("/dashboard");
});
```

### API 测试

Playwright 也可用于纯 API 测试：

```typescript
import { test, expect } from "@playwright/test";

test.describe("API 测试", () => {
  test("获取用户列表", async ({ request }) => {
    const response = await request.get("/api/users");

    expect(response.ok()).toBeTruthy();

    const users = await response.json();
    expect(users.length).toBeGreaterThan(0);
  });

  test("创建新用户", async ({ request }) => {
    const response = await request.post("/api/users", {
      data: {
        name: "新用户",
        email: "new@example.com",
      },
    });

    expect(response.status()).toBe(201);

    const user = await response.json();
    expect(user.name).toBe("新用户");
  });

  test("需要认证的接口", async ({ request }) => {
    // 先登录获取 token
    const loginResponse = await request.post("/api/auth/login", {
      data: {
        username: "testuser",
        password: "password123",
      },
    });

    const { token } = await loginResponse.json();

    // 使用 token 访问受保护资源
    const response = await request.get("/api/profile", {
      headers: {
        Authorization: `Bearer ${token}`,
      },
    });

    expect(response.ok()).toBeTruthy();
  });
});
```

### 测试夹具（Fixtures）

Fixtures 用于共享测试设置和状态：

```typescript
// tests/fixtures.ts
import { test as base, Page } from "@playwright/test";

// 定义自定义夹具类型
type MyFixtures = {
  loggedInPage: Page;
  todoPage: TodoPage;
};

// 扩展 test 对象
export const test = base.extend<MyFixtures>({
  // 登录状态的页面
  loggedInPage: async ({ page }, use) => {
    await page.goto("/login");
    await page.fill("#username", "testuser");
    await page.fill("#password", "password123");
    await page.click('button[type="submit"]');
    await page.waitForURL("**/dashboard");

    await use(page);
  },

  // Todo 页面对象
  todoPage: async ({ page }, use) => {
    const todoPage = new TodoPage(page);
    await use(todoPage);
  },
});

// 使用自定义夹具
test("添加待办事项", async ({ loggedInPage, todoPage }) => {
  await todoPage.addTodo("学习 Playwright");
  await expect(todoPage.todoItem("学习 Playwright")).toBeVisible();
});
```

import { defineConfig, devices } from "@playwright/test";

export default defineConfig({
  // 测试目录

  fullyParallel: true,

  // CI 上失败时禁止 test.only
  forbidOnly: !!process.env.CI,

  retries: process.env.CI ? 2 : 0,

## 最佳实践总结

### 1. 选择器优先级

```
1. getByRole() - 基于可访问性，最稳定
2. getByLabel() / getByPlaceholder() - 表单元素
3. getByText() - 文本内容
4. getByTestId() - 当以上都不适用时
5. CSS/XPath - 最后的选择
```

### 4. 测试原则

- **独立性**：每个测试应独立运行，不依赖其他测试
- **可重复性**：多次运行结果一致
- **可读性**：测试代码即文档
- **速度**：合理使用 Mock 加速测试
- **稳定性**：避免依赖不稳定的外部因素
