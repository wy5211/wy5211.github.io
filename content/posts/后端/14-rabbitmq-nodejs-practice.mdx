## 为什么需要消息队列？

在微服务架构中，服务间通信是核心挑战之一。传统的 HTTP 同步调用存在明显问题：

- **耦合度高**：调用方必须等待响应，服务不可用直接影响业务
- **流量洪峰**：突发请求可能压垮下游服务
- **失败重试**：需要自行实现复杂的重试逻辑
- **扩展困难**：难以应对不均匀的负载

消息队列通过**异步解耦**解决这些问题：

```
生产者 → [消息队列] → 消费者
         ↓
   - 削峰填谷
   - 失败重试
   - 松耦合
   - 可扩展
```

<Callout type="info">
  RabbitMQ 是最流行的开源消息代理之一，支持 AMQP
  协议，具有可靠性高、功能丰富、社区成熟等优点，非常适合企业级应用。
</Callout>

## RabbitMQ 核心概念

### 基础架构

```bash
# Docker 快速启动
docker run -d --name rabbitmq \
  -p 5672:5672 \   # AMQP 协议端口
  -p 15672:15672 \ # 管理界面端口
  rabbitmq:3-management
```

访问 `http://localhost:15672`（默认账号：guest/guest）查看管理界面。

### 核心术语

| 概念            | 说明                   | 类比         |
| --------------- | ---------------------- | ------------ |
| **Producer**    | 消息生产者             | 寄信人       |
| **Exchange**    | 交换机，路由消息到队列 | 邮局分拣中心 |
| **Queue**       | 消息队列，存储消息     | 邮箱         |
| **Consumer**    | 消息消费者             | 收信人       |
| **Binding**     | 交换机与队列的绑定关系 | 投递规则     |
| **Routing Key** | 路由键，消息路由依据   | 邮编         |

### 消息流转流程

```
Producer → Exchange → [Binding + Routing Key] → Queue → Consumer
```

消息不会直接发送到队列，而是先发送到 Exchange，由 Exchange 根据路由规则分发到队列。

## Exchange 类型详解

RabbitMQ 提供 4 种 Exchange 类型：

### 1. Direct Exchange（直连）

精确匹配 routing key，适合点对点通信。

```typescript
// 场景：订单服务 → 库存服务
import * as amqp from "amqplib";

async function directExample() {
  const connection = await amqp.connect("amqp://localhost");
  const channel = await connection.createChannel();

  // 声明 direct exchange
  await channel.assertExchange("order.direct", "direct", { durable: true });

  // 声明队列
  await channel.assertQueue("inventory.update", { durable: true });

  // 绑定：routing key = 'order.created'
  await channel.bindQueue("inventory.update", "order.direct", "order.created");

  // 发送消息
  channel.publish(
    "order.direct",
    "order.created", // routing key
    Buffer.from(JSON.stringify({ orderId: "123", items: [] })),
    { persistent: true }
  );
}
```

### 2. Fanout Exchange（扇出）

广播到所有绑定队列，忽略 routing key，适合发布订阅。

```typescript
// 场景：订单创建后通知多个服务
async function fanoutExample() {
  const channel = await getChannel();

  await channel.assertExchange("order.fanout", "fanout", { durable: true });

  // 多个队列绑定到同一个 fanout exchange
  await channel.assertQueue("email.notification", { durable: true });
  await channel.assertQueue("sms.notification", { durable: true });
  await channel.assertQueue("analytics.track", { durable: true });

  await channel.bindQueue("email.notification", "order.fanout", "");
  await channel.bindQueue("sms.notification", "order.fanout", "");
  await channel.bindQueue("analytics.track", "order.fanout", "");

  // 一条消息，三个队列都会收到
  channel.publish(
    "order.fanout",
    "", // fanout 忽略 routing key
    Buffer.from(JSON.stringify({ orderId: "123" }))
  );
}
```

### 3. Topic Exchange（主题）

支持通配符匹配，适合灵活的路由场景。

```typescript
// 场景：日志收集系统
async function topicExample() {
  const channel = await getChannel();

  await channel.assertExchange("logs.topic", "topic", { durable: true });

  // * 匹配一个单词，# 匹配零或多个单词
  await channel.assertQueue("error.logs", { durable: true });
  await channel.assertQueue("all.logs", { durable: true });
  await channel.assertQueue("user.logs", { durable: true });

  await channel.bindQueue("error.logs", "logs.topic", "*.error"); // user.error, order.error
  await channel.bindQueue("all.logs", "logs.topic", "#"); // 所有日志
  await channel.bindQueue("user.logs", "logs.topic", "user.*"); // user.info, user.error

  // 发送日志
  channel.publish("logs.topic", "user.error", Buffer.from("User login failed"));
  channel.publish("logs.topic", "order.info", Buffer.from("Order created"));
}
```

### 4. Headers Exchange（头部）

根据消息头匹配，适合复杂条件路由（较少使用）。

```typescript
async function headersExample() {
  const channel = await getChannel();

  await channel.assertExchange("match.headers", "headers", { durable: true });

  await channel.assertQueue("premium.users", { durable: true });
  await channel.bindQueue("premium.users", "match.headers", "", {
    "x-match": "all", // 或 'any'
    "user-type": "premium",
    region: "cn",
  });

  // 只有同时满足 user-type=premium 且 region=cn 的消息才会被路由
  channel.publish("match.headers", "", Buffer.from("Premium content"), {
    headers: {
      "user-type": "premium",
      region: "cn",
    },
  });
}
```

### 共享队列模块

```typescript
// shared/queue.ts
import * as amqp from 'amqplib';
import type { Channel, Connection } from 'amqplib';

class RabbitMQConnection {
  private connection: Connection | null = null;
  private channel: Channel | null = null;
  private readonly url: string;
  private reconnectAttempts = 0;
  private readonly maxReconnectAttempts = 5;

  constructor(url = 'amqp://localhost') {
    this.url = url;
  }

  async connect(): Promise&lt;Channel&gt; {
    if (this.channel) return this.channel;

    try {
      this.connection = await amqp.connect(this.url);
      this.channel = await this.connection.createChannel();

      // 连接关闭处理
      this.connection.on('close', () => {
        console.log('RabbitMQ connection closed');
        this.handleReconnect();
      });

      this.connection.on('error', (err) => {
        console.error('RabbitMQ connection error:', err);
      });

      this.reconnectAttempts = 0;
      console.log('RabbitMQ connected successfully');
      return this.channel;
    } catch (error) {
      console.error('Failed to connect to RabbitMQ:', error);
      throw error;
    }
  }

  private async handleReconnect() {
    if (this.reconnectAttempts >= this.maxReconnectAttempts) {
      console.error('Max reconnect attempts reached');
      return;
    }

    this.reconnectAttempts++;
    const delay = Math.min(1000 * Math.pow(2, this.reconnectAttempts), 30000);

    console.log(`Reconnecting in ${delay}ms (attempt ${this.reconnectAttempts})`);
    await new Promise(resolve => setTimeout(resolve, delay));

    try {
      await this.connect();
    } catch (error) {
      console.error('Reconnect failed:', error);
    }
  }

  async close() {
    await this.channel?.close();
    await this.connection?.close();
    this.channel = null;
    this.connection = null;
  }
}

export const rabbitMQ = new RabbitMQConnection(
  process.env.RABBITMQ_URL || 'amqp://localhost'
);
```

### 订单服务（生产者）

```typescript
// services/order-service/src/index.ts
import { rabbitMQ } from '@shared/queue';
import express from 'express';

const app = express();
app.use(express.json());

// 定义消息类型
interface OrderMessage {
  eventType: 'created' | 'cancelled' | 'paid';
  orderId: string;
  userId: string;
  items: Array&lt;{ productId: string; quantity: number }&gt;;
  timestamp: number;
}

// 初始化 Exchange 和队列
async function setupQueues() {
  const channel = await rabbitMQ.connect();

  // 声明 exchanges
  await channel.assertExchange('order.events', 'topic', { durable: true });

  // 声明队列
  await channel.assertQueue('inventory.update', { durable: true });
  await channel.assertQueue('notification.send', { durable: true });
  await channel.assertQueue('order.dlq', { durable: true }); // 死信队列

  // 绑定队列
  await channel.bindQueue('inventory.update', 'order.events', 'order.created');
  await channel.bindQueue('inventory.update', 'order.events', 'order.cancelled');
  await channel.bindQueue('notification.send', 'order.events', 'order.#');
}

// 发布消息
async function publishOrderEvent(message: OrderMessage) {
  const channel = await rabbitMQ.connect();
  const routingKey = `order.${message.eventType}`;

  channel.publish(
    'order.events',
    routingKey,
    Buffer.from(JSON.stringify(message)),
    {
      persistent: true,
      contentType: 'application/json',
      timestamp: Date.now(),
      messageId: `${message.orderId}-${Date.now()}`
    }
  );

  console.log(`Published event: ${routingKey}`, message.orderId);
}

// API 路由
app.post('/orders', async (req, res) => {
  const { userId, items } = req.body;
  const orderId = `ORD-${Date.now()}`;

  // 1. 创建订单（省略数据库操作）

  // 2. 发布订单创建事件
  await publishOrderEvent({
    eventType: 'created',
    orderId,
    userId,
    items,
    timestamp: Date.now()
  });

  res.json({ orderId, status: 'created' });
});

app.post('/orders/:id/pay', async (req, res) => {
  const { id } = req.params;

  // 1. 更新订单状态

  // 2. 发布支付事件
  await publishOrderEvent({
    eventType: 'paid',
    orderId: id,
    userId: 'user-123',
    items: [],
    timestamp: Date.now()
  });

  res.json({ orderId: id, status: 'paid' });
});

// 启动服务
async function start() {
  await setupQueues();
  app.listen(3001, () => console.log('Order service running on :3001'));
}

start().catch(console.error);
```

### 库存服务（消费者）

```typescript
// services/inventory-service/src/index.ts
import { rabbitMQ } from "@shared/queue";

async function startConsumer() {
  const channel = await rabbitMQ.connect();

  // 设置预取数量（重要！防止消费者过载）
  await channel.prefetch(10);

  console.log("Inventory service waiting for messages...");

  channel.consume("inventory.update", async (msg) => {
    if (!msg) return;

    try {
      const content = JSON.parse(msg.content.toString());
      console.log("Received message:", content);

      // 处理逻辑
      if (content.eventType === "created") {
        await reserveInventory(content.items);
      } else if (content.eventType === "cancelled") {
        await releaseInventory(content.items);
      }

      // 手动确认（重要！确保消息不丢失）
      channel.ack(msg);
    } catch (error) {
      console.error("Processing error:", error);

      // 消息处理失败，拒绝并重新入队
      // 第三个参数 false = 重新入队
      channel.nack(msg, false, true);
    }
  });
}

async function reserveInventory(items: any[]) {
  // 实现库存预留逻辑
  console.log("Reserving inventory:", items);
  // 模拟处理
  await new Promise((resolve) => setTimeout(resolve, 100));
}

async function releaseInventory(items: any[]) {
  // 实现库存释放逻辑
  console.log("Releasing inventory:", items);
}

startConsumer().catch(console.error);
```

### 通知服务（消费者）

```typescript
// services/notification-service/src/index.ts
import { rabbitMQ } from "@shared/queue";

async function startConsumer() {
  const channel = await rabbitMQ.connect();
  await channel.prefetch(5);

  channel.consume("notification.send", async (msg) => {
    if (!msg) return;

    const content = JSON.parse(msg.content.toString());

    try {
      // 根据 routing key 决定通知类型
      const routingKey = msg.fields.routingKey;

      if (routingKey === "order.created") {
        await sendEmail(
          content.userId,
          "订单创建成功",
          `订单号: ${content.orderId}`
        );
      } else if (routingKey === "order.paid") {
        await sendEmail(
          content.userId,
          "支付成功",
          `订单号: ${content.orderId}`
        );
      } else if (routingKey === "order.cancelled") {
        await sendEmail(
          content.userId,
          "订单已取消",
          `订单号: ${content.orderId}`
        );
      }

      channel.ack(msg);
    } catch (error) {
      console.error("Notification failed:", error);
      // 通知失败不重试，直接确认（避免死循环）
      channel.ack(msg);
    }
  });
}

async function sendEmail(userId: string, subject: string, body: string) {
  console.log(`Sending email to user ${userId}: ${subject}`);
  // 实现邮件发送逻辑
}

startConsumer().catch(console.error);
```

## 高级特性

### 1. 消息持久化

确保消息在 RabbitMQ 重启后不丢失。

```typescript
// 1. 队列持久化
await channel.assertQueue("my.queue", { durable: true });

// 2. Exchange 持久化
await channel.assertExchange("my.exchange", "direct", { durable: true });

// 3. 消息持久化
channel.publish("my.exchange", "routing.key", Buffer.from("message"), {
  persistent: true, // 或 deliveryMode: 2
});
```

<Callout type="warning">
  持久化会影响性能，在高吞吐场景需权衡。建议仅对关键消息启用持久化。
</Callout>

### 2. 消息确认（ACK）

确保消息被正确处理。

```typescript
// 消费者确认模式
channel.consume(
  "queue",
  async (msg) => {
    try {
      await processMessage(msg);
      channel.ack(msg); // 确认成功
    } catch (error) {
      // channel.nack(msg, false, false); // 拒绝，不重新入队
      // channel.nack(msg, false, true);  // 拒绝，重新入队
      // channel.reject(msg, false);       // 拒绝单个消息

      // 实际场景：根据错误类型决定
      if (isRecoverableError(error)) {
        channel.nack(msg, false, true); // 重试
      } else {
        channel.ack(msg); // 跳过无法处理的消息
      }
    }
  },
  { noAck: false }
); // noAck: false 表示手动确认
```

### 3. 死信队列（DLQ）

处理无法消费的消息。

```typescript
// 声明死信队列
await channel.assertExchange("dlx.exchange", "direct", { durable: true });
await channel.assertQueue("dlq", { durable: true });
await channel.bindQueue("dlq", "dlx.exchange", "dlq");

// 主队列配置死信交换机
await channel.assertQueue("main.queue", {
  durable: true,
  deadLetterExchange: "dlx.exchange",
  deadLetterRoutingKey: "dlq",
  messageTtl: 60000, // 可选：消息过期时间
});

// 死信队列消费者
channel.consume("dlq", (msg) => {
  console.error("Dead letter received:", msg?.content.toString());
  // 记录日志、告警、人工处理
  channel.ack(msg!);
});
```

### 4. 延迟队列

实现定时任务和延迟重试。

```typescript
// 方案 1：使用 RabbitMQ 延迟插件（推荐）
// 安装插件：rabbitmq_delayed_message_exchange

await channel.assertExchange("delayed.exchange", "x-delayed-message", {
  durable: true,
  arguments: { "x-delayed-type": "direct" },
});

channel.publish("delayed.exchange", "routing.key", Buffer.from("message"), {
  headers: { "x-delay": 60000 }, // 延迟 60 秒
});

// 方案 2：使用 TTL + 死信队列
await channel.assertQueue("delay.queue", {
  durable: true,
  messageTtl: 60000, // 60 秒后过期
  deadLetterExchange: "work.exchange", // 过期后转发到工作队列
});
```

### 5. 优先级队列

高优先级消息优先消费。

```typescript
await channel.assertQueue("priority.queue", {
  durable: true,
  maxPriority: 10, // 0-10，数字越大优先级越高
});

// 发送高优先级消息
channel.publish("", "priority.queue", Buffer.from("urgent"), {
  priority: 10,
});

// 发送普通消息
channel.publish("", "priority.queue", Buffer.from("normal"), {
  priority: 1,
});
```

## 性能优化

### 1. 批量发送

```typescript
// 使用 confirm 模式批量发送
const channel = await connection.createConfirmChannel();

const messages = ["msg1", "msg2", "msg3"];
messages.forEach((msg) => {
  channel.sendToQueue("batch.queue", Buffer.from(msg));
});

// 等待所有消息确认
await new Promise((resolve, reject) => {
  channel.waitForConfirms((err) => {
    if (err) reject(err);
    else resolve(true);
  });
});
```

### 2. 合理设置 Prefetch

```typescript
// Prefetch 太小：吞吐量低
// Prefetch 太大：内存占用高、处理不均
await channel.prefetch(10); // 推荐值：10-100
```

### 3. 连接池管理

```typescript
// 多 channel 复用单 connection
class ChannelPool {
  private channels: Channel[] = [];
  private current = 0;

  constructor(private connection: Connection, private poolSize = 5) {}

  async init() {
    for (let i = 0; i &lt; this.poolSize; i++) {
      this.channels.push(await this.connection.createChannel());
    }
  }

  getChannel(): Channel {
    const channel = this.channels[this.current];
    this.current = (this.current + 1) % this.channels.length;
    return channel;
  }
}
```

### 关键指标

| 指标             | 说明       | 告警阈值   |

| Queue Depth      | 队列深度   | &gt; 10000 |

### 健康检查

```typescript
// HTTP API 检查队列状态
async function checkQueueHealth() {
  const response = await fetch('http://localhost:15672/api/queues', {
    headers: {
      'Authorization': 'Basic ' + Buffer.from('guest:guest').toString('base64')
    }
  });

  const queues = await response.json();

  for (const queue of queues) {
    if (queue.messages &gt; 10000) {
      console.error(`Queue ${queue.name} has too many messages: ${queue.messages}`);
      // 发送告警
    }

    if (queue.consumers === 0 &amp;&amp; queue.messages &gt; 0) {
      console.error(`Queue ${queue.name} has no consumers but ${queue.messages} messages`);
    }
  }
}
```

## 总结

RabbitMQ 是构建微服务通信的强大工具，核心优势在于：

1. **可靠性**：消息持久化、确认机制、死信队列
2. **灵活性**：多种 Exchange 类型、路由规则
3. **可扩展**：轻松增加消费者、支持集群部署
4. **成熟生态**：完善的监控、管理工具

**何时选择 RabbitMQ？**

- ✅ 需要可靠消息投递
- ✅ 复杂路由场景
- ✅ 任务队列、事件驱动架构
- ✅ 传统企业应用

**何时考虑其他方案？**

- 高吞吐场景（Kafka 更合适）
- 简单队列需求（Redis 足够）
- 云原生环境（云托管 MQ）

掌握 RabbitMQ，让你的微服务架构更加健壮和可扩展！
