# NestJS Guard、Interceptor、Pipe 深度实战：掌握 AOP 编程精髓

NestJS 的 Guard、Interceptor、Pipe 是三个最强大但常被误解的特性。它们本质上是 **AOP（面向切面编程）** 的实现，让你在不修改业务代码的情况下，横切关注点如权限、日志、验证等。

很多开发者会用 `@UseGuards()`，但不知道如何组合使用；知道 Pipe 可以验证，但不清楚如何自定义转换逻辑；听说过 Interceptor，却只在日志场景用过。

本文从原理到实战，帮你彻底掌握这三把"利剑"。

## 一、执行顺序：理解请求的生命周期

在深入每个组件前，必须理解它们的执行顺序。一个请求进入 NestJS 的完整生命周期：

```
Incoming Request
    ↓
1. Middleware（中间件）
    ↓
2. Guard（守卫）← 决定是否继续
    ↓
3. Interceptor - Before（拦截器前置）
    ↓
4. Pipe（管道）← 参数验证与转换
    ↓
5. Route Handler（路由处理器）
    ↓
6. Interceptor - After（拦截器后置）
    ↓
Response Sent
```

**关键点：**

- Guard 返回 `false` 直接抛 403，后续都不会执行
- Pipe 抛异常会中断流程
- Interceptor 的 `after` 部分即使出错，也能通过 `catchError` 捕获

## 二、Guard：守卫你的应用

Guard 的核心职责是**决定请求是否应该被处理**。最经典的场景是认证和授权。

### 2.1 基础：认证 Guard

```typescript
// auth.guard.ts
import { Injectable, CanActivate, ExecutionContext, UnauthorizedException } from '@nestjs/common';

@Injectable()
export class AuthGuard implements CanActivate {
  constructor(private readonly authService: AuthService) {}

  async canActivate(context: ExecutionContext): Promise&lt;boolean&gt; {
    const request = context.switchToHttp().getRequest();
    const token = this.extractToken(request);

    if (!token) {
      throw new UnauthorizedException('Token not found');
    }

    const user = await this.authService.validateToken(token);
    if (!user) {
      throw new UnauthorizedException('Invalid token');
    }

    // 将用户信息挂载到 request，后续可用
    request.user = user;
    return true;
  }

  private extractToken(request: any): string | null {
    const authHeader = request.headers.authorization;
    if (!authHeader?.startsWith('Bearer ')) {
      return null;
    }
    return authHeader.substring(7);
  }
}
```

### 2.2 进阶：基于角色的授权 Guard

```typescript
// roles.guard.ts
import { Injectable, CanActivate, ExecutionContext, ForbiddenException } from '@nestjs/common';
import { Reflector } from '@nestjs/core';

export const ROLES_KEY = 'roles';
export const Roles = (...roles: string[]) => SetMetadata(ROLES_KEY, roles);

@Injectable()
export class RolesGuard implements CanActivate {
  constructor(private reflector: Reflector) {}

  canActivate(context: ExecutionContext): boolean {
    // 从 Handler 和 Class 上获取所需的 roles
    const requiredRoles = this.reflector.getAllAndOverride&lt;string[]&gt;(
      ROLES_KEY,
      [context.getHandler(), context.getClass()],
    );

    if (!requiredRoles) {
      return true; // 没有角色要求，放行
    }

    const request = context.switchToHttp().getRequest();
    const user = request.user;

    if (!user) {
      throw new ForbiddenException('User not authenticated');
    }

    const hasRole = requiredRoles.some((role) =&gt; user.roles?.includes(role));
    if (!hasRole) {
      throw new ForbiddenException('Insufficient permissions');
    }

    return true;
  }
}
```

**使用示例：**

```typescript
@Controller("admin")
@UseGuards(AuthGuard, RolesGuard)
@Roles("admin")
export class AdminController {
  @Get("dashboard")
  getDashboard() {
    return { message: "Admin dashboard" };
  }

  @Get("users")
  @Roles("admin", "super-admin") // 覆盖 Class 级别的角色要求
  getUsers() {
    return { users: [] };
  }
}
```

### 2.3 高级：动态权限 Guard

更灵活的权限控制，支持资源级别的权限：

```typescript
// permission.guard.ts
import { Injectable, CanActivate, ExecutionContext, ForbiddenException } from '@nestjs/common';
import { Reflector } from '@nestjs/core';

export const PERMISSION_KEY = 'permission';

export const RequirePermission = (action: string, resource: string) =>
  SetMetadata(PERMISSION_KEY, { action, resource });

@Injectable()
export class PermissionGuard implements CanActivate {
  constructor(
    private reflector: Reflector,
    private readonly caslService: CaslService,
  ) {}

  async canActivate(context: ExecutionContext): Promise&lt;boolean&gt; {
    const permission = this.reflector.getAllAndOverride&lt;{ action: string; resource: string }&gt;(
      PERMISSION_KEY,
      [context.getHandler(), context.getClass()],
    );

    if (!permission) {
      return true;
    }

    const request = context.switchToHttp().getRequest();
    const user = request.user;

    // 使用 CASL 风格的权限检查
    const ability = await this.caslService.createAbility(user);
    const hasPermission = ability.can(permission.action, permission.resource);

    if (!hasPermission) {
      throw new ForbiddenException(`No permission to ${permission.action} ${permission.resource}`);
    }

    return true;
  }
}
```

**实战用法：**

```typescript
@Controller("articles")
@UseGuards(AuthGuard, PermissionGuard)
export class ArticleController {
  @Post()
  @RequirePermission("create", "Article")
  create() {
    // 只有有 'create Article' 权限的用户才能访问
  }

  @Delete(":id")
  @RequirePermission("delete", "Article")
  remove() {
    // 只有有 'delete Article' 权限的用户才能访问
  }
}
```

## 三、Interceptor：拦截器的魔力

Interceptor 是最灵活的组件，可以在**请求前后**都执行逻辑。典型场景：日志、缓存、响应转换、异常映射。

### 3.1 经典场景：请求日志 Interceptor

```typescript
// logging.interceptor.ts
import {
  Injectable,
  NestInterceptor,
  ExecutionContext,
  CallHandler,
  Logger,
} from '@nestjs/common';
import { Observable } from 'rxjs';
import { tap, catchError } from 'rxjs/operators';

@Injectable()
export class LoggingInterceptor implements NestInterceptor {
  private readonly logger = new Logger(LoggingInterceptor.name);

  intercept(context: ExecutionContext, next: CallHandler): Observable&lt;any&gt; {
    const request = context.switchToHttp().getRequest();
    const { method, url, ip } = request;
    const userAgent = request.headers['user-agent'];
    const now = Date.now();

    this.logger.log(`→ ${method} ${url} - ${ip} - ${userAgent}`);

    return next.handle().pipe(
      tap(() =&gt; {
        const duration = Date.now() - now;
        this.logger.log(`← ${method} ${url} - ${duration}ms`);
      }),
      catchError((error) =&gt; {
        const duration = Date.now() - now;
        this.logger.error(`✗ ${method} ${url} - ${duration}ms - ${error.message}`);
        throw error;
      }),
    );
  }
}
```

### 3.2 响应格式统一 Interceptor

企业级应用通常需要统一的响应格式：

```typescript
// transform.interceptor.ts
import {
  Injectable,
  NestInterceptor,
  ExecutionContext,
  CallHandler,
} from '@nestjs/common';
import { Observable } from 'rxjs';
import { map } from 'rxjs/operators';

export interface Response&lt;T&gt; {
  code: number;
  message: string;
  data: T;
  timestamp: number;
}

@Injectable()
export class TransformInterceptor&lt;T&gt; implements NestInterceptor&lt;T, Response&lt;T&gt;&gt; {
  intercept(context: ExecutionContext, next: CallHandler): Observable&lt;Response&lt;T&gt;&gt; {
    return next.handle().pipe(
      map((data) =&gt; ({
        code: 0,
        message: 'success',
        data,
        timestamp: Date.now(),
      })),
    );
  }
}
```

**效果：**

```typescript
// Controller 返回
return { id: 1, name: 'Alice' };

// 实际响应
{
  "code": 0,
  "message": "success",
  "data": { "id": 1, "name": "Alice" },
  "timestamp": 1710662400000
}
```

### 3.3 缓存 Interceptor

```typescript
// cache.interceptor.ts
import {
  Injectable,
  NestInterceptor,
  ExecutionContext,
  CallHandler,
} from '@nestjs/common';
import { Observable, of } from 'rxjs';
import { tap } from 'rxjs/operators';
import { RedisService } from './redis.service';

@Injectable()
export class CacheInterceptor implements NestInterceptor {
  constructor(private readonly redisService: RedisService) {}

  async intercept(context: ExecutionContext, next: CallHandler): Promise&lt;Observable&lt;any&gt;&gt; {
    const request = context.switchToHttp().getRequest();
    const cacheKey = `cache:${request.method}:${request.url}`;

    // 尝试从缓存获取
    const cached = await this.redisService.get(cacheKey);
    if (cached) {
      console.log(`🎯 Cache hit: ${cacheKey}`);
      return of(JSON.parse(cached));
    }

    // 缓存未命中，执行请求并缓存结果
    return next.handle().pipe(
      tap(async (response) =&gt; {
        // 缓存 5 分钟
        await this.redisService.setex(cacheKey, 300, JSON.stringify(response));
        console.log(`💾 Cached: ${cacheKey}`);
      }),
    );
  }
}
```

**使用时需要排除某些路由：**

```typescript
// cache.decorator.ts
import { SetMetadata } from '@nestjs/common';

export const NO_CACHE_KEY = 'no_cache';
export const NoCache = () => SetMetadata(NO_CACHE_KEY, true);

// cache.interceptor.ts (改进版)
@Injectable()
export class CacheInterceptor implements NestInterceptor {
  constructor(
    private readonly redisService: RedisService,
    private readonly reflector: Reflector,
  ) {}

  async intercept(context: ExecutionContext, next: CallHandler): Promise&lt;Observable&lt;any&gt;&gt; {
    const noCache = this.reflector.getAllAndOverride&lt;boolean&gt;(
      NO_CACHE_KEY,
      [context.getHandler(), context.getClass()],
    );

    if (noCache) {
      return next.handle();
    }

    // ... 缓存逻辑
  }
}
```

### 3.4 超时 Interceptor

防止请求长时间阻塞：

```typescript
// timeout.interceptor.ts
import {
  Injectable,
  NestInterceptor,
  ExecutionContext,
  CallHandler,
  RequestTimeoutException,
} from '@nestjs/common';
import { Observable, throwError, TimeoutError } from 'rxjs';
import { timeout, catchError } from 'rxjs/operators';

@Injectable()
export class TimeoutInterceptor implements NestInterceptor {
  constructor(private readonly timeoutMs: number) {}

  intercept(context: ExecutionContext, next: CallHandler): Observable&lt;any&gt; {
    return next.handle().pipe(
      timeout(this.timeoutMs),
      catchError((err) =&gt; {
        if (err instanceof TimeoutError) {
          return throwError(() =&gt; new RequestTimeoutException(
            `Request timeout after ${this.timeoutMs}ms`,
          ));
        }
        return throwError(() =&gt; err);
      }),
    );
  }
}

// 使用
@UseInterceptors(new TimeoutInterceptor(5000)) // 5 秒超时
```

## 四、Pipe：数据验证与转换

Pipe 的职责是**验证和转换**输入数据。NestJS 内置了多个 Pipe，但自定义 Pipe 才是发挥威力的地方。

### 4.1 内置 Pipe 速览

| Pipe               | 作用                          | 示例                                      |
| ------------------ | ----------------------------- | ----------------------------------------- |
| `ValidationPipe`   | 配合 class-validator 验证 DTO | `@UsePipes(ValidationPipe)`               |
| `ParseIntPipe`     | 将字符串转为整数              | `@Param('id', ParseIntPipe)`              |
| `ParseUUIDPipe`    | 验证 UUID 格式                | `@Param('id', ParseUUIDPipe)`             |
| `ParseBoolPipe`    | 将字符串转为布尔值            | `@Query('active', ParseBoolPipe)`         |
| `DefaultValuePipe` | 提供默认值                    | `@Query('page', new DefaultValuePipe(1))` |

### 4.2 ValidationPipe 高级配置

```typescript
// main.ts
import { ValidationPipe } from "@nestjs/common";

app.useGlobalPipes(
  new ValidationPipe({
    whitelist: true, // 过滤掉 DTO 中未定义的属性
    forbidNonWhitelisted: true, // 如果有未定义属性，抛出错误
    transform: true, // 自动转换类型（字符串 → 数字等）
    transformOptions: {
      enableImplicitConversion: true, // 隐式类型转换
    },
    disableErrorMessages: false, // 生产环境可设为 true
  })
);
```

### 4.3 自定义验证 Pipe

场景：验证手机号格式

```typescript
// phone.pipe.ts
import {
  PipeTransform,
  Injectable,
  ArgumentMetadata,
  BadRequestException,
} from '@nestjs/common';

@Injectable()
export class ParsePhonePipe implements PipeTransform&lt;string, string&gt; {
  transform(value: string, metadata: ArgumentMetadata): string {
    if (!value) {
      throw new BadRequestException('Phone number is required');
    }

    // 中国大陆手机号正则
    const phoneRegex = /^1[3-9]\d{9}$/;
    if (!phoneRegex.test(value)) {
      throw new BadRequestException('Invalid phone number format');
    }

    return value;
  }
}

// 使用
@Get('user/:phone')
getUserByPhone(@Param('phone', ParsePhonePipe) phone: string) {
  // phone 已经验证过格式
}
```

### 4.4 复合 Pipe：验证 + 转换

```typescript
// parse-int-or-default.pipe.ts
import {
  PipeTransform,
  Injectable,
  ArgumentMetadata,
} from '@nestjs/common';

@Injectable()
export class ParseIntOrDefaultPipe implements PipeTransform&lt;string, number&gt; {
  constructor(private readonly defaultValue: number) {}

  transform(value: string, metadata: ArgumentMetadata): number {
    if (!value) {
      return this.defaultValue;
    }

    const parsed = parseInt(value, 10);
    if (isNaN(parsed)) {
      return this.defaultValue;
    }

    return parsed;
  }
}

// 使用
@Get('articles')
getArticles(
  @Query('page', new ParseIntOrDefaultPipe(1)) page: number,
  @Query('pageSize', new ParseIntOrDefaultPipe(10)) pageSize: number,
) {
  // 如果没有传 page，默认为 1
  // 如果没有传 pageSize，默认为 10
}
```

### 4.5 实体查找 Pipe

一个常见模式：通过 ID 参数自动查找实体，如果不存在抛出 404。

```typescript
// entity-exists.pipe.ts
import {
  PipeTransform,
  Injectable,
  ArgumentMetadata,
  NotFoundException,
} from '@nestjs/common';

@Injectable()
export class EntityExistsPipe implements PipeTransform&lt;string, Promise&lt;any&gt;&gt; {
  constructor(
    private readonly repository: any,
    private readonly entityName: string,
  ) {}

  async transform(id: string, metadata: ArgumentMetadata): Promise&lt;any&gt; {
    const entity = await this.repository.findOne({ where: { id } });
    if (!entity) {
      throw new NotFoundException(`${this.entityName} with id ${id} not found`);
    }
    return entity;
  }
}

// 使用：创建一个工厂函数
export const UserExistsPipe = () =&gt; new EntityExistsPipe(userRepository, 'User');

@Controller('users')
export class UserController {
  @Get(':id')
  getUser(@Param('id', UserExistsPipe()) user: User) {
    // user 已经是查到的实体对象，不再是 id 字符串
    return user;
  }
}
```

## 五、组合使用：最佳实践

### 5.1 完整的 Controller 示例

```typescript
@Controller("articles")
@UseGuards(AuthGuard) // 全局认证
@UseInterceptors(LoggingInterceptor, TransformInterceptor) // 日志 + 响应格式化
export class ArticleController {
  constructor(
    private readonly articleService: ArticleService,
    private readonly cacheService: CacheService
  ) {}

  @Get()
  @UseInterceptors(CacheInterceptor) // 列表接口使用缓存
  @UsePipes(new ValidationPipe({ transform: true }))
  async findAll(@Query() query: QueryArticleDto) {
    return this.articleService.findAll(query);
  }

  @Get(":id")
  @UseInterceptors(CacheInterceptor)
  async findOne(@Param("id", ParseUUIDPipe) id: string) {
    return this.articleService.findOne(id);
  }

  @Post()
  @UseGuards(RolesGuard) // 额外的角色检查
  @Roles("author", "admin")
  async create(
    @Body() createArticleDto: CreateArticleDto,
    @Request() req: any
  ) {
    return this.articleService.create({
      ...createArticleDto,
      authorId: req.user.id,
    });
  }

  @Patch(":id")
  @UseGuards(ArticleOwnerGuard) // 只能修改自己的文章
  async update(
    @Param("id", ParseUUIDPipe) id: string,
    @Body() updateArticleDto: UpdateArticleDto
  ) {
    return this.articleService.update(id, updateArticleDto);
  }

  @Delete(":id")
  @UseGuards(RolesGuard)
  @Roles("admin")
  @NoCache() // 删除操作不清缓存
  async remove(@Param("id", ParseUUIDPipe) id: string) {
    await this.articleService.remove(id);
    return { message: "Article deleted" };
  }
}
```

### 5.2 全局注册

```typescript
// app.module.ts
import { Module } from "@nestjs/common";
import { APP_GUARD, APP_INTERCEPTOR, APP_PIPE } from "@nestjs/core";

@Module({
  providers: [
    // 全局 Guard
    {
      provide: APP_GUARD,
      useClass: AuthGuard,
    },
    {
      provide: APP_GUARD,
      useClass: RolesGuard,
    },
    // 全局 Interceptor
    {
      provide: APP_INTERCEPTOR,
      useClass: LoggingInterceptor,
    },
    {
      provide: APP_INTERCEPTOR,
      useClass: TransformInterceptor,
    },
    // 全局 Pipe
    {
      provide: APP_PIPE,
      useValue: new ValidationPipe({
        whitelist: true,
        transform: true,
      }),
    },
  ],
})
export class AppModule {}
```

**全局注册的注意点：**

- 多个 Guard 按注册顺序执行
- 全局 Interceptor 会应用到所有路由
- 如果需要跳过某个 Guard/Interceptor，使用 `@SkipGuards()` 或自定义装饰器

## 六、性能优化建议

### 6.1 Guard 优化

```typescript
// ❌ 每次都查询数据库
@Injectable()
export class AuthGuard implements CanActivate {
  async canActivate(context: ExecutionContext): Promise&lt;boolean&gt; {
    const token = this.extractToken(context);
    const user = await this.userService.findByToken(token); // 每次都查 DB
    return !!user;
  }
}

// ✅ 使用缓存
@Injectable()
export class AuthGuard implements CanActivate {
  constructor(private readonly redisService: RedisService) {}

  async canActivate(context: ExecutionContext): Promise&lt;boolean&gt; {
    const token = this.extractToken(context);
    const cacheKey = `user:token:${token}`;

    // 先查缓存
    let user = await this.redisService.get(cacheKey);
    if (user) {
      context.switchToHttp().getRequest().user = JSON.parse(user);
      return true;
    }

    // 缓存未命中，查数据库
    user = await this.userService.findByToken(token);
    if (user) {
      await this.redisService.setex(cacheKey, 3600, JSON.stringify(user));
      context.switchToHttp().getRequest().user = user;
      return true;
    }

    return false;
  }
}
```

### 6.2 Interceptor 优化

```typescript
// ❌ 所有响应都序列化两次
intercept(context: ExecutionContext, next: CallHandler) {
  return next.handle().pipe(
    map(data =&gt; JSON.stringify(data)), // 序列化
    map(json =&gt; JSON.parse(json)),    // 反序列化
  );
}

// ✅ 避免不必要的序列化
intercept(context: ExecutionContext, next: CallHandler) {
  return next.handle().pipe(
    map(data =&gt; ({
      code: 0,
      data,
      timestamp: Date.now(),
    })),
  );
}
```

### 6.3 Pipe 优化

```typescript
// ❌ 每个 Pipe 都创建新实例
@Get(':id')
getUser(@Param('id', new ParseIntPipe()) id: number) {}

// ✅ 复用 Pipe 实例
const intPipe = new ParseIntPipe();

@Get(':id')
getUser(@Param('id', intPipe) id: number) {}
```

## 七、常见陷阱

### 7.1 Guard 返回 false vs 抛异常

```typescript
// ❌ 返回 false 会抛 403 Forbidden
canActivate() {
  return false; // 403 Forbidden
}

// ✅ 明确抛出合适的异常
canActivate() {
  throw new UnauthorizedException('Please login first'); // 401
}
```

### 7.2 Interceptor 中的异常处理

```typescript
// ❌ 异常会导致 Interceptor 的 tap 不执行
intercept(context, next) {
  return next.handle().pipe(
    tap(() =&gt; {
      // 如果 Handler 抛异常，这里不会执行
      this.logger.log('Success');
    }),
  );
}

// ✅ 使用 finalize 确保清理逻辑执行
intercept(context, next) {
  return next.handle().pipe(
    tap(() =&gt; this.logger.log('Success')),
    catchError(err =&gt; {
      this.logger.error(`Error: ${err.message}`);
      throw err;
    }),
    finalize(() =&gt; {
      // 无论成功失败都会执行
      this.cleanup();
    }),
  );
}
```

### 7.3 Pipe 的执行时机

```typescript
// ❌ 误以为 Pipe 会验证整个对象
@Post()
create(@Body() body: any, @Body('email') email: string) {
  // email 参数会经过 ValidationPipe
  // 但 body 不会，因为是 any 类型
}

// ✅ 使用 DTO 类
@Post()
create(@Body() body: CreateArticleDto) {
  // body 会根据 DTO 的装饰器进行验证
}
```

## 八、总结

| 组件            | 职责         | 典型场景             | 返回值                |
| --------------- | ------------ | -------------------- | --------------------- |
| **Guard**       | 认证/授权    | JWT 验证、角色检查   | `boolean` 或抛异常    |
| **Interceptor** | 请求前后处理 | 日志、缓存、响应转换 | `Observable&lt;R&gt;` |
| **Pipe**        | 验证/转换    | DTO 验证、类型转换   | 转换后的值或抛异常    |

**核心原则：**

1. **Guard** 关注"能不能访问"
2. **Interceptor** 关注"请求怎么处理"
3. **Pipe** 关注"数据是否有效"

**最佳实践：**

- 全局注册基础 Guard 和 Interceptor
- 特殊场景使用装饰器覆盖
- Pipe 优先使用内置，复杂逻辑再自定义
- 性能敏感场景加缓存

掌握这三把利剑，你的 NestJS 代码将更加优雅、可维护、可扩展。AOP 的魅力在于：**业务逻辑与横切关注点分离**，让每个文件只做一件事。
