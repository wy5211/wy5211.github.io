## 为什么需要结构化日志？

传统的文本日志（如 `console.log('User logged in: userId=123')`）在小型应用中足够使用，但在生产环境面临以下问题：

**1. 难以查询和过滤**

```bash
# 如何找出所有 userId=123 的错误日志？
grep "userId=123" app.log | grep "ERROR"  # 不准确，可能误匹配
```

**2. 无法进行统计分析**

```bash
# 如何统计每个接口的错误率？
# 需要复杂的正则表达式和脚本
```

**3. 缺乏上下文**

```
ERROR: Database connection failed
```

缺少时间戳、请求ID、环境信息等关键上下文。

---

## 结构化日志的优势

结构化日志使用 JSON 格式存储，每条日志都是一个可查询的对象：

```json
{
  "level": "error",
  "time": "2026-03-18T10:30:45.123Z",
  "msg": "Database connection failed",
  "userId": 123,
  "requestId": "req-abc-123",
  "environment": "production",
  "service": "api-server",
  "err": {
    "type": "ConnectionError",
    "message": "ECONNREFUSED",
    "stack": "..."
  }
}
```

**优势：**

- ✅ 可通过日志平台（ELK、Grafana Loki、Datadog）快速查询
- ✅ 支持复杂过滤：`level:error AND userId:123`
- ✅ 易于统计分析：错误率、响应时间分布
- ✅ 完整上下文，便于问题排查

## Pino：高性能日志库

### 基础使用

```bash
npm install pino
```

```typescript
import pino from "pino";

// 创建 logger 实例
const logger = pino({
  level: "info", // 最低日志级别
  transport: {
    target: "pino-pretty", // 开发环境美化输出
    options: {
      colorize: true,
      translateTime: "SYS:standard",
      ignore: "pid,hostname",
    },
  },
});

// 基础日志
logger.info("Hello, Pino!");
logger.error({ err: new Error("Something went wrong") }, "Error occurred");

// 结构化数据
logger.info({ userId: 123, action: "login" }, "User logged in");
```

**输出示例：**

```json
{
  "level": 30,
  "time": 1710763845123,
  "pid": 12345,
  "hostname": "macbook-pro",
  "userId": 123,
  "action": "login",
  "msg": "User logged in"
}
```

### 日志级别

```typescript
logger.trace("Trace message"); // 10
logger.debug("Debug message"); // 20
logger.info("Info message"); // 30
logger.warn("Warning message"); // 40
logger.error("Error message"); // 50
logger.fatal("Fatal message"); // 60
```

### 子日志记录器（Child Logger）

为特定模块创建带预设上下文的子 logger：

```typescript
// 创建带 userId 上下文的子 logger
const userLogger = logger.child({ userId: 123, module: "auth" });

userLogger.info("Login successful");
userLogger.error({ err: error }, "Token validation failed");

// 输出自动包含 userId 和 module
```

### 与 Express 集成

```typescript
import express from "express";
import pino from "pino";
import pinoHttp from "pino-http";

const app = express();
const logger = pino();

// HTTP 请求日志中间件
app.use(
  pinoHttp({
    logger,
    // 自定义日志字段
    customProps: (req, res) => ({
      userId: req.user?.id,
      requestId: req.headers["x-request-id"],
    }),
    // 只记录错误响应
    autoLogging: {
      ignore: (req) => req.path === "/health",
    },
  })
);

// 在路由中使用
app.post("/api/users", (req, res) => {
  req.log.info({ body: req.body }, "Creating user");
  // ...
});
```

### 生产环境配置

```typescript
const logger = pino({
  level: process.env.LOG_LEVEL || "info",
  // 生产环境不使用 pino-pretty
  ...(process.env.NODE_ENV !== "production" && {
    transport: {
      target: "pino-pretty",
    },
  }),
  // 添加基础字段
  base: {
    service: "api-server",
    environment: process.env.NODE_ENV,
    version: process.env.npm_package_version,
  },
  // 时间戳格式
  timestamp: pino.stdTimeFunctions.isoTime,
  // 格式化错误对象
  formatters: {
    level: (label) => ({ level: label }),
  },
});
```

## Winston：企业级日志方案

### 基础使用

```bash
npm install winston
```

```typescript
import winston from "winston";

const logger = winston.createLogger({
  level: "info",
  format: winston.format.combine(
    winston.format.timestamp(),
    winston.format.errors({ stack: true }),
    winston.format.json()
  ),
  defaultMeta: { service: "api-server" },
  transports: [
    // 控制台输出
    new winston.transports.Console({
      format: winston.format.combine(
        winston.format.colorize(),
        winston.format.simple()
      ),
    }),
    // 文件输出
    new winston.transports.File({ filename: "logs/error.log", level: "error" }),
    new winston.transports.File({ filename: "logs/combined.log" }),
  ],
});

// 使用
logger.info("User logged in", { userId: 123 });
logger.error("Database error", { err: new Error("Connection failed") });
```

### 多传输（Transports）配置

Winston 支持多种输出目标：

```typescript
import DailyRotateFile from "winston-daily-rotate-file";

const logger = winston.createLogger({
  transports: [
    // 控制台
    new winston.transports.Console(),

    // 按日期轮转的文件日志
    new DailyRotateFile({
      filename: "logs/application-%DATE%.log",
      datePattern: "YYYY-MM-DD",
      maxSize: "20m",
      maxFiles: "14d", // 保留 14 天
      level: "info",
    }),

    // 错误日志单独存储
    new DailyRotateFile({
      filename: "logs/error-%DATE%.log",
      datePattern: "YYYY-MM-DD",
      level: "error",
    }),

    // 发送到远程日志服务（如 Loggly、Splunk）
    new winston.transports.Http({
      host: "logs.loggly.com",
      path: "/inputs/your-token",
      ssl: true,
    }),
  ],
});
```

### 自定义格式

```typescript
const customFormat = winston.format.printf(({ level, message, timestamp, ...metadata }) => {
  let msg = `${timestamp} [${level}]: ${message}`

  if (Object.keys(metadata).length &gt; 0) {
    msg += ` ${JSON.stringify(metadata)}`
  }

  return msg
})

const logger = winston.createLogger({
  format: winston.format.combine(
    winston.format.timestamp(),
    winston.format.colorize(),
    customFormat
  ),
  transports: [new winston.transports.Console()]
})
```

### 与 NestJS 集成

```typescript
// main.ts
import { NestFactory } from "@nestjs/core";
import { WinstonModule } from "nest-winston";
import * as winston from "winston";

async function bootstrap() {
  const app = await NestFactory.create(AppModule, {
    logger: WinstonModule.createLogger({
      transports: [
        new winston.transports.Console({
          format: winston.format.combine(
            winston.format.timestamp(),
            winston.format.colorize(),
            winston.format.printf(({ level, message, timestamp, context }) => {
              return `${timestamp} [${context}] ${level}: ${message}`;
            })
          ),
        }),
        new winston.transports.File({ filename: "logs/app.log" }),
      ],
    }),
  });

  await app.listen(3000);
}
```

## 最佳实践

### 1. 设计日志 Schema

定义标准化的日志字段：

```typescript
interface LogContext {
  // 用户相关
  userId?: string;
  userEmail?: string;

  // 请求相关
  requestId?: string;
  method?: string;
  path?: string;
  statusCode?: number;
  responseTime?: number;

  // 业务相关
  action?: string;
  resource?: string;
  resourceId?: string;

  // 错误相关
  err?: Error;
  stack?: string;

  // 其他
  [key: string]: any;
}
```

### 2. 使用请求 ID 追踪请求

```typescript
import { randomUUID } from "crypto";
import pino from "pino";

const logger = pino();

app.use((req, res, next) => {
  const requestId = req.headers["x-request-id"] || randomUUID();
  req.requestId = requestId;
  res.setHeader("X-Request-Id", requestId);

  // 为每个请求创建带 requestId 的子 logger
  req.log = logger.child({ requestId });
  next();
});

// 在路由中使用
app.post("/api/orders", (req, res) => {
  req.log.info({ userId: req.user.id, orderId: "123" }, "Creating order");
  // 所有日志自动包含 requestId
});
```

### 3. 敏感信息过滤

```typescript
import pino from "pino";

const logger = pino({
  redact: {
    paths: ["password", "token", "authorization", "*.password"],
    censor: "[REDACTED]",
  },
});

logger.info(
  {
    username: "john",
    password: "secret123", // 会被替换为 [REDACTED]
  },
  "User signup"
);
```

### 4. 错误日志最佳实践

```typescript
import pino from "pino";

const logger = pino();

try {

} catch (err) {
  logger.error(
    {
```

### 5. 性能追踪

```typescript
import pino from 'pino'

const logger = pino()

async function measurePerformance<T>(
  operation: string,
  fn: () => Promise&lt;T&gt;
): Promise&lt;T&gt; {
  const start = Date.now()

  try {
    const result = await fn()
    const duration = Date.now() - start

    logger.info({ operation, duration, success: true }, 'Operation completed')

    return result
  } catch (err) {
    const duration = Date.now() - start

    logger.error({ operation, duration, success: false, err }, 'Operation failed')
    throw err
  }
}

// 使用
const users = await measurePerformance('fetchUsers', () =>
  db.user.findMany()
)
```

### 6. 结构化错误码

定义标准化的错误码：

```typescript
enum ErrorCode {
  AUTH_INVALID_TOKEN = "AUTH_001",
  AUTH_EXPIRED_TOKEN = "AUTH_002",
  DB_CONNECTION_FAILED = "DB_001",
  DB_QUERY_ERROR = "DB_002",
  VALIDATION_ERROR = "VAL_001",
  NOT_FOUND = "GEN_001",
}

class AppError extends Error {
  constructor(
    public code: ErrorCode,
    message: string,
    public statusCode: number = 500,
    public details?: any
  ) {
    super(message);
  }
}

// 使用
try {
  // ...
} catch (err) {
  if (err instanceof AppError) {
    logger.error(
      {
        errorCode: err.code,
        statusCode: err.statusCode,
        message: err.message,
        details: err.details,
        userId: req.user.id,
      },
      "Application error"
    );
  }
}
```

```typescript
const logger = pino({
```

## 总结

**选择建议：**

- 🚀 **高性能应用**：选择 Pino（每秒处理数十万条日志）
- 🏢 **企业应用**：选择 Winston（功能全面，生态丰富）
- 🎯 **简单应用**：选择 Bunyan 或直接使用 Pino

**核心原则：**

1. ✅ 始终使用结构化 JSON 日志
2. ✅ 包含足够的上下文信息（userId、requestId、时间戳）
3. ✅ 过滤敏感信息（密码、token）
4. ✅ 使用不同的日志级别（debug/info/warn/error）
5. ✅ 生产环境配置日志轮转和归档
6. ✅ 集成日志聚合平台（ELK/Loki/Datadog）

结构化日志是可观测性的基础，投入时间优化日志系统，能大幅提升问题排查效率，是值得的长期投资。
