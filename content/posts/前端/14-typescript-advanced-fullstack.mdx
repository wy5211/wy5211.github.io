# TypeScript 高级类型系统与全栈实践

对于前端开发者来说，TypeScript 已经是日常开发的标准配置。但大多数开发者只使用了 TypeScript 的基础特性——接口、类型注解、简单的类型推断。当转向全栈开发时，你会发现自己需要更深入地理解 TypeScript 的类型系统，才能构建真正类型安全的应用程序。

本文将深入探讨 TypeScript 的高级类型特性，以及如何在整个全栈应用中实现端到端的类型安全。

## TypeScript 类型系统基础回顾

在深入高级特性之前，让我们快速回顾 TypeScript 类型系统的核心概念。

### 类型推断

TypeScript 能够自动推断变量的类型：

```typescript
// 类型推断示例
let name = "Alice"; // 推断为 string
let age = 30; // 推断为 number
let isActive = true; // 推断为 boolean
```

### 基础类型注解

```typescript
// 函数类型注解
function greet(name: string): string {
  return `Hello, ${name}!`;
}

// 对象类型注解
interface User {
  id: number;
  name: string;
  email: string;
  age?: number; // 可选属性
}

const user: User = {
  id: 1,
  name: "Alice",
  email: "alice@example.com",
};
```

### 联合类型与交叉类型

```typescript
// 联合类型：可以是多种类型之一
type Status = "pending" | "in_progress" | "completed";

// 交叉类型：同时具有多个类型的属性
type UserWithRole = User & { role: "admin" | "user" };

const admin: UserWithRole = {
  id: 1,
  name: "Alice",
  email: "alice@example.com",
  role: "admin",
};
```

这些基础特性可能已经满足前端开发的大部分需求，但在全栈应用中，我们需要更强大的类型系统来应对复杂的业务场景。

## 泛型编程实战

泛型（Generics）是 TypeScript 中最强大、最灵活的特性之一。它允许你编写可重用的组件，同时保持类型安全。

### 泛型基础

泛型允许你定义类型变量，使用这些变量创建"可变类型"的组件：

```typescript
// 泛型函数
function identity<T>(arg: T): T {
  return arg;
}

// 使用泛型
const num = identity<number>(42); // 类型为 number
const str = identity<string>("hello"); // 类型为 string

// 类型推断（可以省略类型参数）
const bool = identity(true); // 类型为 boolean
```

### 泛型接口

```typescript
// 泛型接口
interface Box<T> {
  value: T;
  get(): T;
  set(value: T): void;
}

// 使用泛型接口
const numberBox: Box<number> = {
  value: 42,
  get() {
    return this.value;
  },
  set(value: T) {
    this.value = value;
  },
};
```

### 泛型类

```typescript
// 泛型类
class Repository<T> {
  private items: T[] = [];

  add(item: T): void {
    this.items.push(item);
  }

  find(predicate: (item: T) => boolean): T | undefined {
    return this.items.find(predicate);
  }

  getAll(): T[] {
    return [...this.items];
  }
}

// 使用泛型类
interface Product {
  id: number;
  name: string;
  price: number;
}

const productRepo = new Repository<Product>();
productRepo.add({ id: 1, name: "Laptop", price: 999 });
productRepo.add({ id: 2, name: "Mouse", price: 29 });

const laptop = productRepo.find((p) => p.id === 1);
// laptop 的类型是 Product | undefined
```

### 泛型约束

有时候你需要限制泛型类型的范围，这时可以使用泛型约束：

```typescript
// 泛型约束：T 必须有 length 属性
function logLength<T extends { length: number }>(arg: T): void {
  console.log(`Length: ${arg.length}`);
}

logLength("hello"); // ✅
logLength([1, 2, 3]); // ✅
logLength({ length: 5 }); // ✅
logLength(42); // ❌ Error: Argument of type 'number' is not assignable
```

```typescript
// 更复杂的泛型约束
interface HasId {
  id: number;
}

function updateItem<T extends HasId>(item: T, updates: Partial<T>): T {
  return { ...item, ...updates };
}

interface User {
  id: number;
  name: string;
  email: string;
  role: "admin" | "user";
}

const user: User = {
  id: 1,
  name: "Alice",
  email: "alice@example.com",
  role: "user",
};

const updatedUser = updateItem(user, {
  name: "Alice Updated",
  email: "alice.updated@example.com",
  // id 不能更新，因为 Partial<User> 中 id 仍然是 number
  // 但由于我们的实现使用展开运算符，可以覆盖
  // 真实场景可能需要更严格的处理
});
```

### 多个泛型参数

```typescript
// 两个泛型参数
function pair<T, U>(first: T, second: U): [T, U] {
  return [first, second];
}

const result = pair<string, number>("age", 30);
// result: [string, number]

// 泛型默认值
interface ApiResponse<T, E = Error> {
  data: T;
  error: E | null;
}

const success: ApiResponse<string> = {
  data: "Success",
  error: null,
};

const failure: ApiResponse<null, string> = {
  data: null,
  error: "Something went wrong",
};
```

## 条件类型进阶

条件类型（Conditional Types）是 TypeScript 2.8 引入的强大特性，它允许你基于类型关系选择类型。

### 条件类型基础

条件类型使用 `T extends U ? X : Y` 的语法：

```typescript
// 条件类型示例
type IsString<T> = T extends string ? true : false;

type Test1 = IsString<string>; // true
type Test2 = IsString<number>; // false

// 使用条件类型
function process<T>(value: T): IsString<T> extends true ? string : number {
  if (typeof value === "string") {
    return value.toUpperCase() as any; // 类型守卫
  }
  return (value * 2) as any;
}

process("hello"); // 返回 string
process(42); // 返回 number
```

### 分布式条件类型

当条件类型作用于联合类型时，它会"分布"到每个成员：

```typescript
// 分布式条件类型
type ToArray<T> = T extends any ? T[] : never;

type Numbers = number | string;

type Result = ToArray<Numbers>;
// Result = number[] | string[]

// 实际应用：过滤联合类型
type NonNullable<T> = T extends null | undefined ? never : T;

type MaybeString = string | null | undefined;

type DefinitelyString = NonNullable<MaybeString>;
// DefinitelyString = string

// 实际应用：提取可调用类型
type FunctionPropertyNames<T> = {
  [K in keyof T]: T[K] extends Function ? K : never;
}[keyof T];

interface User {
  id: number;
  name: string;
  getName(): string;
  setName(name: string): void;
}

type FunctionNames = FunctionPropertyNames<User>;
// FunctionNames = "getName" | "setName"
```

### 条件类型与类型推断

使用 `infer` 关键字从类型中推断信息：

```typescript
// 从函数类型推断返回类型
type ReturnType<T> = T extends (...args: any[]) => infer R ? R : never;

function greet(): string {
  return "Hello";
}

type GreetReturn = ReturnType<typeof greet>;
// GreetReturn = string

// 从 Promise 类型推断解析类型
type PromiseType<T> = T extends Promise<infer U> ? U : never;

async function fetchData(): Promise<User> {
  return { id: 1, name: "Alice", email: "alice@example.com", age: 30 };
}

type DataType = PromiseType<ReturnType<typeof fetchData>>;
// DataType = User

// 从数组类型推断元素类型
type ElementType<T> = T extends (infer U)[]
  ? U
  : T extends Array<infer U>
    ? U
    : T;

type Numbers = ElementType<number[]>;
// Numbers = number

type UserFromList = ElementType<User[]>;
// UserFromList = User

// 实际应用：Unwrap 包装类型
type UnwrapPromise<T> = T extends Promise<infer U> ? U : T;
type UnwrapArray<T> = T extends Array<infer U> ? U : T;
type UnwrapNullable<T> = T extends null | undefined ? never : T;

// 链式解包
type DeepUnwrap<T> =
  T extends Promise<infer U>
    ? DeepUnwrap<U>
    : T extends Array<infer U>
      ? DeepUnwrap<U>
      : T;

type DeepResult = DeepUnwrap<Promise<User[]>>;
// DeepResult = User
```

## 映射类型

映射类型（Mapped Types）允许你基于现有类型创建新类型，通过映射属性的方式转换类型。

### 基础映射类型

```typescript
// 基础映射：使所有属性变为可选
type Partial<T> = {
  [P in keyof T]?: T[P];
};

interface User {
  id: number;
  name: string;
  email: string;
}

type PartialUser = Partial<User>;
// 等价于：
// interface PartialUser {
//   id?: number;
//   name?: string;
//   email?: string;
// }

// 使所有属性变为必需
type Required<T> = {
  [P in keyof T]-?: T[P]; // -? 移除可选标记
};

// 只读属性
type Readonly<T> = {
  readonly [P in keyof T]: T[P];
};

const readonlyUser: Readonly<User> = {
  id: 1,
  name: "Alice",
  email: "alice@example.com",
};

// readonlyUser.id = 2;  // ❌ Error: Cannot assign to 'id'
```

### 高级映射类型

```typescript
// 提取属性类型
type Pick<T, K extends keyof T> = {
  [P in K]: T[P];
};

type UserNameAndEmail = Pick<User, "name" | "email">;
// 等价于：
// interface UserNameAndEmail {
//   name: string;
//   email: string;
// }

// 排除属性
type Omit<T, K extends keyof T> = Pick<T, Exclude<keyof T, K>>;

type UserWithoutId = Omit<User, "id">;
// 等价于：
// interface UserWithoutId {
//   name: string;
//   email: string;
//   age?: number;
// }

// 修改属性类型
type Modify<T, R> = Omit<T, keyof R> & R;

interface UpdatedUser {
  name: string;
  email: string;
  role: string; // 之前没有 role
}

type ModifiedUser = Modify<User, UpdatedUser>;
// 等价于：
// interface ModifiedUser {
//   id: number;
//   name: string;
//   email: string;
//   age?: number;
//   role: string;
// }
```

### 模板字面量类型

TypeScript 4.1 引入了模板字面量类型，可以组合字符串字面量类型：

```typescript
// 模板字面量类型
type EventName<T extends string> = `on${Capitalize<T>}`;

type ClickEvent = EventName<"click">;
// ClickEvent = "onClick"

type ChangeEvent = EventName<"change">;
// ChangeEvent = "onChange"

// 实际应用：创建事件处理器类型
type EventHandler = {
  [E in EventName<"click" | "change" | "submit">]: (event: Event) => void;
};

const handlers: EventHandler = {
  onClick: (event) => console.log("Clicked"),
  onChange: (event) => console.log("Changed"),
  onSubmit: (event) => console.log("Submitted"),
};

// 更复杂的示例：API 路由类型
type HttpMethod = "GET" | "POST" | "PUT" | "DELETE";
type ApiPath = "/users" | "/posts" | "/comments";

type ApiRoute = `${HttpMethod} ${ApiPath}`;

type UserRoutes = ApiRoute;
// UserRoutes = "GET /users" | "POST /users" | "PUT /users" | "DELETE /users"
//            | "GET /posts" | "POST /posts" | "PUT /posts" | "DELETE /posts"
//            | "GET /comments" | "POST /comments" | "PUT /comments" | "DELETE /comments"
```

### 条件映射类型

```typescript
// 深度可选（递归映射类型）
type DeepPartial<T> = {
  [P in keyof T]?: T[P] extends object ? DeepPartial<T[P]> : T[P];
};

interface Config {
  server: {
    port: number;
    host: string;
    ssl: {
      enabled: boolean;
      cert: string;
      key: string;
    };
  };
  database: {
    url: string;
    timeout: number;
  };
}

const partialConfig: DeepPartial<Config> = {
  server: {
    port: 3000,
    // host, ssl 都是可选
  },
  // database 完全可选
};

// 深度只读
type DeepReadonly<T> = {
  readonly [P in keyof T]: T[P] extends object ? DeepReadonly<T[P]> : T[P];
};

const readonlyConfig: DeepReadonly<Config> = {
  server: {
    port: 3000,
    host: "localhost",
    ssl: {
      enabled: true,
      cert: "cert.pem",
      key: "key.pem",
    },
  },
  database: {
    url: "mongodb://localhost",
    timeout: 5000,
  },
};

// readonlyConfig.server.port = 4000;  // ❌ Error
// readonlyConfig.server.ssl.enabled = false;  // ❌ Error
```

## 前后端类型共享方案

在全栈应用中，前后端类型共享是类型安全的关键。如果前后端分别定义类型，很容易出现不一致的问题。

### 类型共享架构

理想的前后端类型共享架构：

```
┌─────────────────────────────────────────────────────────┐
│                    共享类型包                          │
│  ┌──────────────────────────────────────────────────┐  │
│  │  types/                                       │  │
│  │    - user.ts                                   │  │
│  │    - api.ts                                    │  │
│  │    - response.ts                               │  │
│  └──────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────┘
         ↑                              ↑
         │                              │
┌────────┴────────┐           ┌──────────┴──────────┐
│   前端应用       │           │    后端应用 (API)    │
│   - Next.js      │           │    - Express/Next.js  │
│   - React       │           │    - Prisma          │
└─────────────────┘           └─────────────────────┘
```

### Monorepo 类型共享

使用 Monorepo（如 pnpm workspace）实现类型共享：

```yaml
# pnpm-workspace.yaml
packages:
  - "apps/*"
  - "packages/*"
```

```
my-monorepo/
├── packages/
│   └── shared-types/        # 共享类型包
│       ├── package.json
│       ├── tsconfig.json
│       └── src/
│           ├── index.ts
│           ├── user.ts
│           └── api.ts
├── apps/
│   ├── frontend/            # 前端应用
│   │   ├── package.json
│   │   └── src/
│   └── backend/             # 后端应用
│       ├── package.json
│       └── src/
└── pnpm-workspace.yaml
```

**共享类型包**：

```typescript
// packages/shared-types/src/index.ts
export * from "./user";
export * from "./api";
export * from "./response";

// packages/shared-types/src/user.ts
export interface User {
  id: number;
  name: string;
  email: string;
  age?: number;
  role: "admin" | "user";
  createdAt: Date;
  updatedAt: Date;
}

export interface CreateUserDto {
  name: string;
  email: string;
  password: string;
  role?: "admin" | "user";
}

export interface UpdateUserDto {
  name?: string;
  email?: string;
  role?: "admin" | "user";
}

// packages/shared-types/src/api.ts
export interface PaginationParams {
  page: number;
  limit: number;
  sortBy?: string;
  sortOrder?: "asc" | "desc";
}

export interface PaginatedResponse<T> {
  data: T[];
  total: number;
  page: number;
  limit: number;
  totalPages: number;
}

export type HttpMethod = "GET" | "POST" | "PUT" | "DELETE" | "PATCH";

// packages/shared-types/src/response.ts
export interface ApiResponse<T> {
  success: boolean;
  data: T;
  message?: string;
  error?: {
    code: string;
    message: string;
    details?: Record<string, any>;
  };
}

export type ApiError = ApiResponse<null> & {
  success: false;
  error: {
    code: string;
    message: string;
    details?: Record<string, any>;
  };
};

export type ApiSuccess<T> = ApiResponse<T> & {
  success: true;
};
```

**前端应用使用共享类型**：

```typescript
// apps/frontend/src/api/user.ts
import {
  User,
  CreateUserDto,
  UpdateUserDto,
  ApiSuccess,
  ApiError,
} from "@shared-types";

const API_BASE = process.env.NEXT_PUBLIC_API_URL || "http://localhost:3000";

// 类型安全的 API 请求函数
async function fetchApi<T>(
  endpoint: string,
  options?: RequestInit
): Promise<ApiSuccess<T> | ApiError> {
  const response = await fetch(`${API_BASE}${endpoint}`, {
    headers: {
      "Content-Type": "application/json",
      ...options?.headers,
    },
    ...options,
  });

  return response.json();
}

// 获取用户列表
export async function getUsers(): Promise<User[]> {
  const result = await fetchApi<User[]>("/users");

  if (result.success) {
    return result.data;
  }

  throw new Error(result.error?.message || "Failed to fetch users");
}

// 获取单个用户
export async function getUser(id: number): Promise<User> {
  const result = await fetchApi<User>(`/users/${id}`);

  if (result.success) {
    return result.data;
  }

  throw new Error(result.error?.message || "Failed to fetch user");
}

// 创建用户
export async function createUser(dto: CreateUserDto): Promise<User> {
  const result = await fetchApi<User>("/users", {
    method: "POST",
    body: JSON.stringify(dto),
  });

  if (result.success) {
    return result.data;
  }

  throw new Error(result.error?.message || "Failed to create user");
}

// 更新用户
export async function updateUser(
  id: number,
  dto: UpdateUserDto
): Promise<User> {
  const result = await fetchApi<User>(`/users/${id}`, {
    method: "PUT",
    body: JSON.stringify(dto),
  });

  if (result.success) {
    return result.data;
  }

  throw new Error(result.error?.message || "Failed to update user");
}

// 删除用户
export async function deleteUser(id: number): Promise<void> {
  const result = await fetchApi<void>(`/users/${id}`, {
    method: "DELETE",
  });

  if (!result.success) {
    throw new Error(result.error?.message || "Failed to delete user");
  }
}
```

**后端应用使用共享类型**：

```typescript
// apps/backend/src/routes/users.ts
import {
  User,
  CreateUserDto,
  UpdateUserDto,
  ApiSuccess,
  ApiError,
} from "@shared-types";

// 模拟数据库
const users: User[] = [];

// GET /users
export async function GET(): Promise<Response> {
  return Response.json({
    success: true,
    data: users,
  } satisfies ApiSuccess<User[]>);
}

// GET /users/:id
export async function GET(
  request: Request,
  { params }: { params: { id: string } }
): Promise<Response> {
  const id = parseInt(params.id);
  const user = users.find((u) => u.id === id);

  if (!user) {
    return Response.json(
      {
        success: false,
        error: {
          code: "USER_NOT_FOUND",
          message: `User with id ${id} not found`,
        },
      } satisfies ApiError,
      { status: 404 }
    );
  }

  return Response.json({
    success: true,
    data: user,
  } satisfies ApiSuccess<User>);
}

// POST /users
export async function POST(request: Request): Promise<Response> {
  const dto: CreateUserDto = await request.json();

  // 类型安全的验证
  if (!dto.name || !dto.email || !dto.password) {
    return Response.json(
      {
        success: false,
        error: {
          code: "VALIDATION_ERROR",
          message: "Missing required fields",
          details: dto,
        },
      } satisfies ApiError,
      { status: 400 }
    );
  }

  const newUser: User = {
    id: users.length + 1,
    name: dto.name,
    email: dto.email,
    role: dto.role || "user",
    createdAt: new Date(),
    updatedAt: new Date(),
  };

  users.push(newUser);

  return Response.json(
    {
      success: true,
      data: newUser,
    } satisfies ApiSuccess<User>,
    { status: 201 }
  );
}

// PUT /users/:id
export async function PUT(
  request: Request,
  { params }: { params: { id: string } }
): Promise<Response> {
  const id = parseInt(params.id);
  const dto: UpdateUserDto = await request.json();

  const userIndex = users.findIndex((u) => u.id === id);

  if (userIndex === -1) {
    return Response.json(
      {
        success: false,
        error: {
          code: "USER_NOT_FOUND",
          message: `User with id ${id} not found`,
        },
      } satisfies ApiError,
      { status: 404 }
    );
  }

  // 类型安全的更新
  const updatedUser: User = {
    ...users[userIndex],
    ...dto,
    updatedAt: new Date(),
  };

  users[userIndex] = updatedUser;

  return Response.json({
    success: true,
    data: updatedUser,
  } satisfies ApiSuccess<User>);
}

// DELETE /users/:id
export async function DELETE(
  request: Request,
  { params }: { params: { id: string } }
): Promise<Response> {
  const id = parseInt(params.id);
  const userIndex = users.findIndex((u) => u.id === id);

  if (userIndex === -1) {
    return Response.json(
      {
        success: false,
        error: {
          code: "USER_NOT_FOUND",
          message: `User with id ${id} not found`,
        },
      } satisfies ApiError,
      { status: 404 }
    );
  }

  users.splice(userIndex, 1);

  return Response.json({
    success: true,
    data: null,
  } satisfies ApiSuccess<null>);
}
```

## Zod 运行时类型验证

TypeScript 只在编译时提供类型检查，无法保证运行时类型安全。Zod 是一个 TypeScript-first 的模式验证库，可以在运行时验证数据，并与 TypeScript 类型系统无缝集成。

### Zod 基础使用

```typescript
import { z } from "zod";

// 定义 schema
const userSchema = z.object({
  id: z.number(),
  name: z.string().min(2),
  email: z.string().email(),
  age: z.number().optional(),
  role: z.enum(["admin", "user"]),
});

// 推断类型
type User = z.infer<typeof userSchema>;
// type User = {
//   id: number;
//   name: string;
//   email: string;
//   age?: number | undefined;
//   role: "admin" | "user";
// }

// 验证数据
const data = {
  id: 1,
  name: "Alice",
  email: "alice@example.com",
  role: "user",
};

const result = userSchema.safeParse(data);

if (result.success) {
  // result.data 是类型安全的 User
  console.log(result.data.name);
} else {
  // result.error 包含验证错误
  console.error(result.error.errors);
}

// 直接解析（失败会抛出错误）
const user = userSchema.parse(data);
// user 的类型是 User
```

### Zod 在 API 中的应用

```typescript
import { z } from "zod";
import { NextRequest, NextResponse } from "next/server";

// 定义 DTO schema
const createUserSchema = z.object({
  name: z.string().min(2).max(100),
  email: z.string().email(),
  password: z
    .string()
    .min(8)
    .regex(/^(?=.*[a-z])(?=.*[A-Z])(?=.*\d)/, {
      message: "Password must contain uppercase, lowercase and number",
    }),
  role: z.enum(["admin", "user"]).default("user"),
});

type CreateUserDto = z.infer<typeof createUserSchema>;

// API 路由处理器
export async function POST(request: NextRequest) {
  try {
    // 解析请求体
    const body = await request.json();

    // 验证数据
    const dto = createUserSchema.parse(body);

    // dto 现在是完全类型安全的 CreateUserDto
    const user = await createUser(dto);

    return NextResponse.json(
      {
        success: true,
        data: user,
      },
      { status: 201 }
    );
  } catch (error) {
    if (error instanceof z.ZodError) {
      // Zod 验证错误
      return NextResponse.json(
        {
          success: false,
          error: {
            code: "VALIDATION_ERROR",
            message: "Invalid request data",
            details: error.errors,
          },
        },
        { status: 400 }
      );
    }

    // 其他错误
    return NextResponse.json(
      {
        success: false,
        error: {
          code: "INTERNAL_ERROR",
          message: "An unexpected error occurred",
        },
      },
      { status: 500 }
    );
  }
}

// Zod 验证中间件
export function withValidation<T extends z.ZodSchema>(
  schema: T,
  handler: (request: NextRequest, dto: z.infer<T>) => Promise<Response>
) {
  return async (request: NextRequest) => {
    try {
      const body = await request.json();
      const dto = schema.parse(body);
      return handler(request, dto);
    } catch (error) {
      if (error instanceof z.ZodError) {
        return NextResponse.json(
          {
            success: false,
            error: {
              code: "VALIDATION_ERROR",
              message: "Invalid request data",
              details: error.errors,
            },
          },
          { status: 400 }
        );
      }

      throw error;
    }
  };
}

// 使用中间件
export const POST = withValidation(createUserSchema, async (request, dto) => {
  const user = await createUser(dto);
  return NextResponse.json({ success: true, data: user }, { status: 201 });
});
```

### 类型安全的错误处理

```typescript
// 错误代码枚举
export enum ErrorCode {
  VALIDATION_ERROR = "VALIDATION_ERROR",
  NOT_FOUND = "NOT_FOUND",
  UNAUTHORIZED = "UNAUTHORIZED",
  FORBIDDEN = "FORBIDDEN",
  INTERNAL_ERROR = "INTERNAL_ERROR",
}

// 错误类
export class ApiError extends Error {
  constructor(
    public code: ErrorCode,
    message: string,
    public details?: Record<string, any>,
    public statusCode: number = 500
  ) {
    super(message);
    this.name = "ApiError";
  }
}

// 错误助手
export function validationError(
  message: string,
  details?: Record<string, any>
): ApiError {
  return new ApiError(ErrorCode.VALIDATION_ERROR, message, details, 400);
}

export function notFoundError(resource: string, identifier: string): ApiError {
  return new ApiError(
    ErrorCode.NOT_FOUND,
    `${resource} with identifier '${identifier}' not found`,
    undefined,
    404
  );
}

export function unauthorizedError(message: string = "Unauthorized"): ApiError {
  return new ApiError(ErrorCode.UNAUTHORIZED, message, undefined, 401);
}

// 错误处理中间件
export function withErrorHandler(
  handler: (request: NextRequest) => Promise<Response>
) {
  return async (request: NextRequest) => {
    try {
      return await handler(request);
    } catch (error) {
      if (error instanceof ApiError) {
        return NextResponse.json(
          {
            success: false,
            data: null,
            error: {
              code: error.code,
              message: error.message,
              details: error.details,
            },
          },
          { status: error.statusCode }
        );
      }

      // 未处理的错误
      console.error("Unhandled error:", error);
      return NextResponse.json(
        {
          success: false,
          data: null,
          error: {
            code: ErrorCode.INTERNAL_ERROR,
            message: "An unexpected error occurred",
          },
        },
        { status: 500 }
      );
    }
  };
}

// 使用
export const POST = withErrorHandler(async (request: NextRequest) => {
  const body = await request.json();

  if (!body.email || !body.password) {
    throw validationError("Email and password are required", body);
  }

  const user = await prisma.user.findUnique({
    where: { email: body.email },
  });

  if (!user) {
    throw notFoundError("User", body.email);
  }

});
```

```typescript
// apps/frontend/src/lib/api.ts
import {
  User,
  CreateUserDto,
  UpdateUserDto,
  ApiSuccess,
  ApiError,
} from "@shared-types";

const API_BASE = process.env.NEXT_PUBLIC_API_URL || "http://localhost:3001";

async function fetchApi<T>(
  endpoint: string,
  options?: RequestInit
): Promise<ApiSuccess<T> | ApiError> {
  const response = await fetch(`${API_BASE}${endpoint}`, {
    headers: {
      "Content-Type": "application/json",
      ...options?.headers,
    },
    credentials: "include",
    ...options,
  });

  return response.json();
}

export const userApi = {
  list: async (params?: { page?: number; limit?: number }) => {
    const query = new URLSearchParams(
      params as Record<string, string>
    ).toString();
    const result = await fetchApi<User[]>(
      `/api/users${query ? `?${query}` : ""}`
    );

    if (!result.success) {
      throw new Error(result.error?.message || "Failed to fetch users");
    }

    return result.data;
  },

  get: async (id: number) => {
    const result = await fetchApi<User>(`/api/users/${id}`);

    if (!result.success) {
      throw new Error(result.error?.message || "Failed to fetch user");
    }

    return result.data;
  },

  create: async (dto: CreateUserDto) => {
    const result = await fetchApi<User>("/api/users", {
      method: "POST",
      body: JSON.stringify(dto),
    });

    if (!result.success) {
      throw new Error(result.error?.message || "Failed to create user");
    }

    return result.data;
  },

  update: async (id: number, dto: UpdateUserDto) => {
    const result = await fetchApi<User>(`/api/users/${id}`, {
      method: "PUT",
      body: JSON.stringify(dto),
    });

    if (!result.success) {
      throw new Error(result.error?.message || "Failed to update user");
    }

    return result.data;
  },

  delete: async (id: number) => {
    const result = await fetchApi<void>(`/api/users/${id}`, {
      method: "DELETE",
    });

    if (!result.success) {
      throw new Error(result.error?.message || "Failed to delete user");
    }
  },
```

```typescript
  const handleUpdateUser = async (id: number, dto: UpdateUserDto) => {
    try {
      const updatedUser = await userApi.update(id, dto);
      setUsers(users.map(u => u.id === id ? updatedUser : u));
    } catch (err) {
      setError(err instanceof Error ? err.message : 'Failed to update user');
    }
  };

  const handleDeleteUser = async (id: number) => {
    if (!confirm('Are you sure?')) return;

    try {
      await userApi.delete(id);
      setUsers(users.filter(u => u.id !== id));
    } catch (err) {
      setError(err instanceof Error ? err.message : 'Failed to delete user');
    }
  };

  if (loading) return <div>Loading...</div>;

  if (error) return <div className="text-red-500">{error}</div>;

  return (
    <div className="p-4">
      <div className="flex justify-between items-center mb-4">
        <h1 className="text-2xl font-bold">Users</h1>
        <button
          onClick={() => setShowCreateModal(true)}
          className="px-4 py-2 bg-blue-500 text-white rounded"
        >
          Create User
        </button>
      </div>

      <table className="w-full">
        <thead>
          <tr>
            <th className="border p-2">ID</th>
            <th className="border p-2">Name</th>
            <th className="border p-2">Email</th>
            <th className="border p-2">Role</th>
            <th className="border p-2">Actions</th>
          </tr>
        </thead>
        <tbody>
          {users.map(user => (
            <tr key={user.id}>
              <td className="border p-2">{user.id}</td>
              <td className="border p-2">{user.name}</td>
              <td className="border p-2">{user.email}</td>
              <td className="border p-2">{user.role}</td>
              <td className="border p-2">
                <button
                  onClick={() => handleUpdateUser(user.id, { name: `${user.name} Updated` })}
                  className="px-2 py-1 bg-green-500 text-white rounded mr-2"
                >
                  Edit
                </button>
                <button
                  onClick={() => handleDeleteUser(user.id)}
                  className="px-2 py-1 bg-red-500 text-white rounded"
                >
                  Delete
                </button>
              </td>
            </tr>
          ))}
        </tbody>
      </table>

      {showCreateModal && (
        <CreateUserModal
          onClose={() => setShowCreateModal(false)}
          onCreate={handleCreateUser}
        />
      )}
    </div>
  );
}

function CreateUserModal({
```

## 总结

TypeScript 的高级类型系统为全栈开发提供了强大的类型安全保障。通过深入理解泛型、条件类型、映射类型等特性，并结合前后端类型共享、Zod 运行时验证和类型安全的 API 设计，你可以构建出真正可靠的全栈应用。

### 核心要点

1. **泛型编程**：编写可重用的类型安全组件
2. **条件类型**：基于类型关系选择类型
3. **映射类型**：转换和操作现有类型
4. **前后端类型共享**：使用 Monorepo 或类型生成工具
5. **运行时验证**：使用 Zod 保证运行时类型安全
6. **类型安全的 API**：统一响应格式和错误处理

### 最佳实践

- 使用 Monorepo 实现前后端类型共享
- 结合 Zod 实现 TypeScript 类型和运行时验证的统一
- 设计清晰的 API 响应和错误处理模式
- 利用 TypeScript 类型系统减少手动类型注解
- 保持类型定义的简洁和可维护性

TypeScript 的类型系统功能强大但复杂，掌握高级特性需要时间和实践。但一旦掌握，它将成为你构建高质量全栈应用的利器。
