# React 性能优化：从 memo 到 Concurrent Mode

React 应用卡顿？组件频繁重渲染？本文系统讲解 React 性能优化技巧，从基础到进阶。

## 一、React 渲染机制

### 1.1 触发渲染的原因

```javascript
// 1. 自身 state 变化
const [count, setCount] = useState(0)

// 2. 父组件传递的 props 变化
<Child value={value} />

// 3. 父组件重新渲染（子组件默认会跟着渲染）
function Parent() {
  const [count, setCount] = useState(0)
  return <Child />  // Child 会随 Parent 重渲染
}
```

### 1.2 使用 React DevTools 分析

```
1. 安装 React DevTools 浏览器扩展
2. 打开 DevTools → Profiler 标签
3. 点击录制按钮
4. 操作页面
5. 停止录制，分析渲染情况
```

## 二、避免不必要渲染

### 2.1 React.memo

```javascript
// ❌ 每次父组件更新，Child 都会重渲染
function Parent() {
  const [count, setCount] = useState(0);
  return (
    <>
      <button onClick={() => setCount((c) => c + 1)}>Count: {count}</button>
      <Child name="Alice" />
    </>
  );
}

function Child({ name }) {
  console.log("Child rendered");
  return <div>{name}</div>;
}

// ✅ 使用 memo，props 不变就不渲染
const Child = React.memo(function Child({ name }) {
  console.log("Child rendered");
  return <div>{name}</div>;
});
```

### 2.2 useMemo

```javascript
// ❌ 每次渲染都重新计算
function ExpensiveComponent({ list }) {
  const sorted = list.sort((a, b) => b.value - a.value); // 每次都排序
  return (
    <div>
      {sorted.map((item) => (
        <Item key={item.id} {...item} />
      ))}
    </div>
  );
}

// ✅ 使用 useMemo 缓存计算结果
function ExpensiveComponent({ list }) {
  const sorted = useMemo(() => {
    console.log("Sorting...");
    return list.sort((a, b) => b.value - a.value);
  }, [list]); // 只有 list 变化才重新计算

  return (
    <div>
      {sorted.map((item) => (
        <Item key={item.id} {...item} />
      ))}
    </div>
  );
}
```

### 2.3 useCallback

```javascript
// ❌ 每次渲染都创建新函数，导致子组件重渲染
function Parent() {
  const [count, setCount] = useState(0);

  const handleClick = () => {
    // 每次都是新函数
    console.log("clicked");
  };

  return <Child onClick={handleClick} />;
}

// ✅ 使用 useCallback 缓存函数
function Parent() {
  const [count, setCount] = useState(0);

  const handleClick = useCallback(() => {
    console.log("clicked");
  }, []); // 依赖为空，永远返回同一个函数

  return <Child onClick={handleClick} />;
}

// ✅ 带依赖的 useCallback
function Parent() {
  const [count, setCount] = useState(0);

  const handleIncrement = useCallback(() => {
    setCount((c) => c + 1); // 使用函数式更新，不需要依赖 count
  }, []);

  return <Child onIncrement={handleIncrement} />;
}
```

### 2.4 合理使用 memo/useMemo/useCallback

```javascript
// 不是所有地方都需要，过度使用反而降低性能

// ❌ 不需要 memo：组件很轻量
const SmallComponent = React.memo(({ text }) => <span>{text}</span>);

// ❌ 不需要 useMemo：计算很快
const doubled = useMemo(() => count * 2, [count]);

// ❌ 不需要 useCallback：没有传给子组件
const handleChange = useCallback((e) => {
  setValue(e.target.value);
}, []);

// ✅ 需要 memo：组件重、props 经常不变
const HeavyComponent = React.memo(({ data }) => {
  // 大量渲染逻辑
});

// ✅ 需要 useMemo：计算昂贵
const result = useMemo(() => heavyComputation(bigArray), [bigArray]);

// ✅ 需要 useCallback：传给 memo 子组件
const handleClick = useCallback(() => {
  // ...
}, [dep]);
```

## 三、列表优化

### 3.1 key 的正确使用

```javascript
// ❌ 使用 index 作为 key（会导致性能问题和 bug）
{
  items.map((item, index) => <Item key={index} {...item} />);
}

// ✅ 使用唯一 ID
{
  items.map((item) => <Item key={item.id} {...item} />);
}
```

### 3.3 分页加载

```javascript
function PaginatedList() {
  const [items, setItems] = useState([]);
  const [page, setPage] = useState(1);
  const [hasMore, setHasMore] = useState(true);
  const loaderRef = useRef();

  useEffect(() => {
    const observer = new IntersectionObserver((entries) => {
      if (entries[0].isIntersecting && hasMore) {
        loadMore();
      }
    });

    if (loaderRef.current) {
      observer.observe(loaderRef.current);
    }

    return () => observer.disconnect();
  }, [hasMore]);

  const loadMore = async () => {
    const newItems = await fetchItems(page);
    setItems((prev) => [...prev, ...newItems]);
    setPage((p) => p + 1);
    setHasMore(newItems.length > 0);
  };

  return (
    <>
      {items.map((item) => (
        <Item key={item.id} {...item} />
      ))}
      {hasMore && <div ref={loaderRef}>Loading...</div>}
    </>
  );
}
```

## 四、状态优化

### 4.1 状态下沉

```javascript
// ❌ 状态在顶层，导致整个树重渲染
function App() {
  const [value, setValue] = useState("");
  return (
    <>
      <Header />
      <Sidebar />
      <Content value={value} setValue={setValue} />
      <Footer />
    </>
  );
}

// ✅ 状态下移，只影响需要的组件
function App() {
  return (
    <>
      <Header />
      <Sidebar />
      <ContentWithState /> {/* 状态在内部 */}
      <Footer />
    </>
  );
}

function ContentWithState() {
  const [value, setValue] = useState("");
  return <Content value={value} setValue={setValue} />;
}
```

### 4.2 状态拆分

```javascript
// ❌ 单个大状态对象
const [state, setState] = useState({

// ✅ 拆分为独立状态
```

### 4.3 使用 useReducer

```javascript
// 复杂状态逻辑用 useReducer
function reducer(state, action) {
  switch (action.type) {
    case "SET_USER":
      return { ...state, user: action.payload };
    case "ADD_POST":
      return { ...state, posts: [...state.posts, action.payload] };
    case "SET_THEME":
      return { ...state, theme: action.payload };
    default:
      return state;
  }
}

function App() {
  const [state, dispatch] = useReducer(reducer, {
    user: null,
    posts: [],
    theme: "light",
  });

  return (
    <ThemeContext.Provider value={{ theme: state.theme, dispatch }}>
      <AppContent />
    </ThemeContext.Provider>
  );
}
```

## 五、Context 优化

### 5.1 拆分 Context

```javascript
// ❌ 单一大 Context，任何变化都会导致所有消费者重渲染
const AppContext = createContext();

function App() {
  const [user, setUser] = useState(null);
  const [theme, setTheme] = useState("light");

  return (
    <AppContext.Provider value={{ user, setUser, theme, setTheme }}>
      <App />
    </AppContext.Provider>
  );
}

// ✅ 拆分 Context
const UserContext = createContext();
const ThemeContext = createContext();

function App() {
  const [user, setUser] = useState(null);
  const [theme, setTheme] = useState("light");

  return (
    <UserContext.Provider value={{ user, setUser }}>
      <ThemeContext.Provider value={{ theme, setTheme }}>
        <App />
      </ThemeContext.Provider>
    </UserContext.Provider>
  );
}
```

### 5.2 使用 useMemo 优化 value

```javascript
// ❌ 每次渲染都创建新对象
function Provider({ children }) {
  const [user, setUser] = useState(null);

  return (
    <UserContext.Provider value={{ user, setUser }}>
      {children}
    </UserContext.Provider>
  );
}

// ✅ 使用 useMemo 缓存 value
function Provider({ children }) {
  const [user, setUser] = useState(null);

  const value = useMemo(() => ({ user, setUser }), [user]);

  return <UserContext.Provider value={value}>{children}</UserContext.Provider>;
}
```

## 六、并发模式（Concurrent Mode）

### 6.1 useTransition

```javascript
import { useTransition } from "react";

function SearchComponent() {
  const [isPending, startTransition] = useTransition();
  const [query, setQuery] = useState("");
  const [results, setResults] = useState([]);

  const handleChange = (e) => {
    // 立即更新输入框
    setQuery(e.target.value);

    // 标记为非紧急更新
    startTransition(() => {
      const filtered = filterResults(e.target.value);
      setResults(filtered);
    });
  };

  return (
    <>
      <input value={query} onChange={handleChange} />
      {isPending && <span>Loading...</span>}
      <Results items={results} />
    </>
  );
}
```

### 6.2 useDeferredValue

```javascript
import { useDeferredValue } from "react";

function SearchComponent({ query }) {
  // 延迟更新，让位给更紧急的更新
  const deferredQuery = useDeferredValue(query);

  const results = useMemo(() => {
    return filterResults(deferredQuery);
  }, [deferredQuery]);

  return <Results items={results} />;
}
```

### 6.3 Suspense

```javascript
import { Suspense } from "react";

// 数据加载组件
const Posts = React.lazy(() => import("./Posts"));

function App() {
  return (
    <Suspense fallback={<div>Loading posts...</div>}>

// 配合 React Query 等数据获取库
function Posts() {
```

## 总结

React 性能优化是系统工程。

**核心原则：**

1. **先测量** - 用 Profiler 找到瓶颈
2. **避免过度** - 不是所有地方都需要优化
3. **合理使用** - memo/useMemo/useCallback 适度
4. **关注重点** - 重组件、大列表、频繁更新

**优化顺序：**

```
测量定位 → 避免不必要渲染 → 计算缓存 → 虚拟列表 → 并发特性
```

记住：过早优化是万恶之源。先让它工作，再让它快。
