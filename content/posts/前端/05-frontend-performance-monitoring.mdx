# 前端性能监控实战：从指标采集到智能告警

性能优化不是一次性工作，而是持续的过程。没有监控，就无法知道优化是否有效，也无法及时发现性能退化。本文将带你从零构建一套完整的前端性能监控系统。

## 一、为什么需要性能监控？

### 1.1 常见问题场景

| 场景              | 没有监控的后果           | 有监控的好处           |
| ----------------- | ------------------------ | ---------------------- |
| 新版本上线        | 用户反馈慢，问题定位困难 | 实时告警，快速回滚     |
| 第三方 SDK 加载慢 | 不知道拖慢了哪个环节     | 精确到资源的性能数据   |
| 特定地区/设备慢   | 只能靠猜测               | 地理分布、设备分类统计 |
| 优化效果验证      | 无法量化收益             | 优化前后对比数据       |

### 1.2 监控的价值

```text
性能监控 = 问题发现 + 根因定位 + 效果验证 + 持续优化
```

- **问题发现**：在用户投诉前发现性能问题
- **根因定位**：精确到代码、资源、API 的性能瓶颈
- **效果验证**：量化优化收益，证明价值
- **持续优化**：建立性能基线，防止回归

### 2.2 Web Vitals 采集

```javascript
// 封装 Web Vitals 采集器
class WebVitalsCollector {
  constructor() {
    this.metrics = {}
    this.observers = []
  }

  // LCP - 最大内容绘制
  measureLCP() {
    const observer = new PerformanceObserver((list) =&gt; {
      const entries = list.getEntries()
      const lastEntry = entries[entries.length - 1]

      this.metrics.lcp = {
        value: lastEntry.startTime,
        element: lastEntry.element?.tagName,
        url: lastEntry.url || '',
        timestamp: Date.now()
      }
    })

    observer.observe({ type: 'largest-contentful-paint', buffered: true })
    this.observers.push(observer)
  }

  // FID - 首次输入延迟
  measureFID() {
    const observer = new PerformanceObserver((list) =&gt; {
      for (const entry of list.getEntries()) {
        this.metrics.fid = {
          value: entry.processingStart - entry.startTime,
          eventType: entry.name,
          timestamp: Date.now()
        }
      }
    })

    observer.observe({ type: 'first-input', buffered: true })
    this.observers.push(observer)
  }

  // CLS - 累积布局偏移
  measureCLS() {
    let clsValue = 0
    let sessionValue = 0
    let sessionEntries = []

    const observer = new PerformanceObserver((list) =&gt; {
      for (const entry of list.getEntries()) {
        // 只统计非用户输入导致的布局偏移
        if (!entry.hadRecentInput) {
          sessionValue += entry.value
          sessionEntries.push(entry)
        }
      }

      clsValue = sessionValue
      this.metrics.cls = {
        value: clsValue,
        entries: sessionEntries.length,
        timestamp: Date.now()
      }
    })

    observer.observe({ type: 'layout-shift', buffered: true })
    this.observers.push(observer)
  }

  // INP - 交互到下一次绘制（替代 FID 的新指标）
  measureINP() {
    const observer = new PerformanceObserver((list) =&gt; {
      const entries = list.getEntries()
      let maxDuration = 0

      entries.forEach(entry =&gt; {
        if (entry.interactionId) {
          const duration = entry.duration
          if (duration &gt; maxDuration) {
            maxDuration = duration
          }
        }
      })

      this.metrics.inp = {
        value: maxDuration,
        timestamp: Date.now()
      }
    })

    observer.observe({ type: 'event', buffered: true, durationThreshold: 16 })
    this.observers.push(observer)
  }

  // 获取所有指标
  getMetrics() {
    return this.metrics
  }

  // 清理观察者
  disconnect() {
    this.observers.forEach(observer =&gt; observer.disconnect())
    this.observers = []
  }
}

// 使用示例
const collector = new WebVitalsCollector()
collector.measureLCP()
collector.measureFID()
collector.measureCLS()
collector.measureINP()

// 页面加载完成后获取数据
window.addEventListener('load', () =&gt; {
  setTimeout(() =&gt; {
    console.log('Web Vitals:', collector.getMetrics())
  }, 1000)
})
```

## 三、数据上报策略

### 3.1 上报时机选择

```javascript
class PerformanceReporter {
  constructor(endpoint) {
    this.endpoint = endpoint
    this.queue = []
    this.maxQueueSize = 10
    this.maxRetryCount = 3
  }

  // 添加指标到队列
  addMetric(metric) {
    this.queue.push({
      ...metric,
      timestamp: Date.now(),
      url: location.href,
      userAgent: navigator.userAgent,
      connection: this.getConnectionInfo()
    })

    if (this.queue.length &gt;= this.maxQueueSize) {
      this.flush()
    }
  }

  // 获取网络连接信息
  getConnectionInfo() {
    const connection = navigator.connection || navigator.mozConnection || navigator.webkitConnection

    if (!connection) return null

    return {
      effectiveType: connection.effectiveType, // 4g, 3g, 2g
      downlink: connection.downlink, // 下行速度 Mbps
      rtt: connection.rtt, // 往返时间 ms
      saveData: connection.saveData // 省流模式
    }
  }

  // 批量上报
  async flush() {
    if (this.queue.length === 0) return

    const data = [...this.queue]
    this.queue = []

    try {
      await this.send(data)
    } catch (error) {
      console.error('性能数据上报失败:', error)
      // 失败后重新加入队列
      this.queue = [...data, ...this.queue]
    }
  }

  // 使用 sendBeacon 保证页面关闭时也能上报
  send(data) {
    return new Promise((resolve, reject) =&gt; {
      // 优先使用 sendBeacon
      if (navigator.sendBeacon) {
        const blob = new Blob([JSON.stringify(data)], { type: 'application/json' })
        const success = navigator.sendBeacon(this.endpoint, blob)

        if (success) {
          resolve()
          return
        }
      }

      // 降级使用 fetch + keepalive
      fetch(this.endpoint, {
        method: 'POST',
        body: JSON.stringify(data),
        headers: { 'Content-Type': 'application/json' },
        keepalive: true
      })
      .then(resolve)
      .catch(reject)
    })
  }
}
```

### 3.2 智能上报策略

```javascript
class SmartReporter extends PerformanceReporter {
  constructor(endpoint) {
    super(endpoint)
    this.setupVisibilityHandler()
    this.setupIdleHandler()
  }

  // 页面可见性变化时上报
  setupVisibilityHandler() {
    document.addEventListener('visibilitychange', () =&gt; {
      if (document.visibilityState === 'hidden') {
        this.flush()
      }
    })
  }

  // 使用 requestIdleCallback 在空闲时上报
  setupIdleHandler() {
    const reportWhenIdle = () =&gt; {
      if ('requestIdleCallback' in window) {
        requestIdleCallback(() =&gt; {
          if (this.queue.length &gt; 0) {
            this.flush()
          }
        }, { timeout: 5000 })
      } else {
        // 降级使用 setTimeout
        setTimeout(() =&gt; {
          if (this.queue.length &gt; 0) {
            this.flush()
          }
        }, 5000)
      }
    }

    // 页面加载完成后延迟上报
    window.addEventListener('load', () =&gt; {
      setTimeout(reportWhenIdle, 3000)
    })
  }

  // 采样上报（高流量场景）
  shouldReport(sampleRate = 0.1) {
    // 开发环境 100% 上报
    if (process.env.NODE_ENV === 'development') {
      return true
    }

    // 生产环境采样
    return Math.random() &lt; sampleRate
  }
}

// 性能数据聚合服务
class PerformanceAggregator {
  constructor() {
    this.data = {
      hourly: new Map(),
      daily: new Map(),
      weekly: new Map()
    }
  }

  // 聚合指标数据
  aggregate(metrics) {
    const hour = this.getHourKey()
    const day = this.getDayKey()
    const week = this.getWeekKey()

    // 小时级聚合
    this.aggregateToKey(this.data.hourly, hour, metrics)

    // 天级聚合
    this.aggregateToKey(this.data.daily, day, metrics)

    // 周级聚合
    this.aggregateToKey(this.data.weekly, week, metrics)
  }

  aggregateToKey(map, key, metrics) {
    if (!map.has(key)) {
      map.set(key, {
        count: 0,
        lcp: { sum: 0, values: [] },
        fid: { sum: 0, values: [] },
        cls: { sum: 0, values: [] }
      })
    }

    const entry = map.get(key)
    entry.count++

    if (metrics.lcp) {
      entry.lcp.sum += metrics.lcp.value
      entry.lcp.values.push(metrics.lcp.value)
    }

    if (metrics.fid) {
      entry.fid.sum += metrics.fid.value
      entry.fid.values.push(metrics.fid.value)
    }

    if (metrics.cls) {
      entry.cls.sum += metrics.cls.value
      entry.cls.values.push(metrics.cls.value)
    }
  }

  // 计算百分位数
  getPercentile(values, percentile) {
    const sorted = [...values].sort((a, b) =&gt; a - b)
    const index = Math.ceil((percentile / 100) * sorted.length) - 1
    return sorted[index]
  }

  // 生成报告
  generateReport(period = 'daily') {
    const map = this.data[period]
    const report = []

    map.forEach((data, key) =&gt; {
      report.push({
        period: key,
        count: data.count,
        lcp: {
          avg: data.lcp.sum / data.count,
          p50: this.getPercentile(data.lcp.values, 50),
          p95: this.getPercentile(data.lcp.values, 95),
          p99: this.getPercentile(data.lcp.values, 99)
        },
        fid: {
          avg: data.fid.sum / data.count,
          p50: this.getPercentile(data.fid.values, 50),
          p95: this.getPercentile(data.fid.values, 95)
        },
        cls: {
          avg: data.cls.sum / data.count,
          p50: this.getPercentile(data.cls.values, 50)
        }
      })
    })

    return report
  }

  getHourKey() {
    const now = new Date()
    return `${now.getFullYear()}-${now.getMonth() + 1}-${now.getDate()}-${now.getHours()}`
  }

  getDayKey() {
    const now = new Date()
    return `${now.getFullYear()}-${now.getMonth() + 1}-${now.getDate()}`
  }

  getWeekKey() {
    const now = new Date()
    const weekNumber = this.getWeekNumber(now)
    return `${now.getFullYear()}-W${weekNumber}`
  }

  getWeekNumber(date) {
    const d = new Date(Date.UTC(date.getFullYear(), date.getMonth(), date.getDate()))
    const dayNum = d.getUTCDay() || 7
    d.setUTCDate(d.getUTCDate() + 4 - dayNum)
    const yearStart = new Date(Date.UTC(d.getUTCFullYear(), 0, 1))
    return Math.ceil((((d - yearStart) / 86400000) + 1) / 7)
  }
}

// 告警规则引擎
class AlertEngine {
  constructor() {
    this.rules = []
    this.alerts = []
  }

  // 定义告警规则
  defineRule(rule) {
    this.rules.push({
      id: rule.id,
      name: rule.name,
      metric: rule.metric,
      condition: rule.condition, // 'greater', 'less', 'anomaly'
      threshold: rule.threshold,
      duration: rule.duration, // 持续时间窗口（分钟）
      severity: rule.severity, // 'critical', 'warning', 'info'
      channels: rule.channels, // ['email', 'slack', 'webhook']
      enabled: rule.enabled !== false
    })
  }

  // 检查是否触发告警
  check(metrics, context = {}) {
    const triggeredAlerts = []

    this.rules.forEach(rule =&gt; {
      if (!rule.enabled) return

      const value = this.getMetricValue(metrics, rule.metric)
      if (value === undefined) return

      const isTriggered = this.evaluateCondition(value, rule)

      if (isTriggered) {
        const alert = {
          rule: rule,
          value: value,
          timestamp: Date.now(),
          context: context
        }

        triggeredAlerts.push(alert)
        this.alerts.push(alert)
      }
    })

    return triggeredAlerts
  }

  // 获取指标值
  getMetricValue(metrics, metricPath) {
    const parts = metricPath.split('.')
    let value = metrics

    for (const part of parts) {
      if (value === undefined || value === null) return undefined
      value = value[part]
    }

    return typeof value === 'object' ? value.value : value
  }

  // 评估条件
  evaluateCondition(value, rule) {
    switch (rule.condition) {
      case 'greater':
        return value &gt; rule.threshold
      case 'less':
        return value &lt; rule.threshold
      case 'anomaly':
        return this.detectAnomaly(value, rule)
      default:
        return false
    }
  }

  // 异常检测（简单实现）

    return zScore &gt; 3 // 超过 3 个标准差视为异常

// 告警通知服务
class AlertNotifier {
  constructor() {
    this.channels = {
      email: new EmailChannel(),
      slack: new SlackChannel(),
      webhook: new WebhookChannel(),
      sms: new SMSChannel()
    }
  }

  async send(alert) {
    const { channels, severity } = alert.rule

    // 根据严重程度决定通知方式
    const enabledChannels = this.getEnabledChannels(severity, channels)

    const promises = enabledChannels.map(channel =&gt; {
      const notifier = this.channels[channel]
      if (!notifier) return Promise.resolve()

      return notifier.send({
        title: `[${severity.toUpperCase()}] ${alert.rule.name}`,
        message: this.formatMessage(alert),
        data: alert
      }).catch(error =&gt; {
        console.error(`告警发送失败 [${channel}]:`, error)
      })
    })

    await Promise.allSettled(promises)
  }

  getEnabledChannels(severity, configuredChannels) {
    // Critical 级别全渠道通知
    if (severity === 'critical') {
      return Object.keys(this.channels)
    }

    return configuredChannels
  }

  formatMessage(alert) {
    return `
性能告警：${alert.rule.name}

指标：${alert.rule.metric}
当前值：${alert.value}
阈值：${alert.rule.threshold}
时间：${new Date(alert.timestamp).toLocaleString('zh-CN')}
页面：${alert.context.url || '未知'}

请立即处理！
    `.trim()
  }
}

// performance-monitor.js
class PerformanceMonitor {
  constructor(config = {}) {
    this.config = {
      endpoint: config.endpoint || '/api/performance',
      sampleRate: config.sampleRate || 1,
      enableLongTask: config.enableLongTask !== false,
      enableResourceTiming: config.enableResourceTiming !== false,
      enableWebVitals: config.enableWebVitals !== false,
      ...config
    }

    this.reporter = new SmartReporter(this.config.endpoint)
    this.collector = new WebVitalsCollector()
    this.aggregator = new PerformanceAggregator()
    this.alertEngine = new AlertEngine()
    this.notifier = new AlertNotifier()

    this.init()
  }

  init() {
    // 采样检查
    if (!this.shouldSample()) {
      console.log('[Performance Monitor] 采样跳过')
      return
    }

    // 初始化各类监控
    if (this.config.enableWebVitals) {
      this.initWebVitals()
    }

    if (this.config.enableLongTask) {
      this.initLongTaskMonitoring()
    }

    if (this.config.enableResourceTiming) {
      this.initResourceMonitoring()
    }

    // 页面卸载时上报剩余数据
    window.addEventListener('beforeunload', () =&gt; {
      this.reporter.flush()
    })

    console.log('[Performance Monitor] 初始化完成')
  }

  shouldSample() {
    return Math.random() &lt; this.config.sampleRate
  }

  initWebVitals() {
    this.collector.measureLCP()
    this.collector.measureFID()
    this.collector.measureCLS()
    this.collector.measureINP()

    // 延迟上报
    window.addEventListener('load', () =&gt; {
      setTimeout(() =&gt; {
        const metrics = this.collector.getMetrics()
        this.reportMetrics(metrics)
      }, 1000)
    })
  }

  initLongTaskMonitoring() {
    const observer = new PerformanceObserver((list) =&gt; {
      for (const entry of list.getEntries()) {
        this.reportMetrics({
          type: 'longtask',
          duration: entry.duration,
          startTime: entry.startTime,
          name: entry.name
        })
      }
    })

    observer.observe({ entryTypes: ['longtask'] })
  }

  initResourceMonitoring() {
    window.addEventListener('load', () =&gt; {
      setTimeout(() =&gt; {
        const resources = performance.getEntriesByType('resource')

        // 统计慢资源
        const slowResources = resources.filter(r =&gt; r.duration &gt; 1000)

        if (slowResources.length &gt; 0) {
          this.reportMetrics({
            type: 'slow-resource',
            count: slowResources.length,
            resources: slowResources.map(r =&gt; ({
              name: r.name,
              duration: r.duration,
              type: r.initiatorType
            }))
          })
        }
      }, 2000)
    })
  }

  reportMetrics(metrics) {
    // 添加到上报队列
    this.reporter.addMetric(metrics)

    // 聚合数据
    this.aggregator.aggregate(metrics)

    // 检查告警
    const alerts = this.alertEngine.check(metrics, {
      url: location.href,
      userAgent: navigator.userAgent
    })

    // 发送告警
    alerts.forEach(alert =&gt; {
      this.notifier.send(alert)
    })
  }
}

// 自动初始化
if (typeof window !== 'undefined') {
  window.PerformanceMonitor = PerformanceMonitor
}
```

## 七、最佳实践与注意事项

### 7.1 性能监控的最佳实践

| 实践         | 说明                     | 收益                     |
| ------------ | ------------------------ | ------------------------ |
| **采样上报** | 高流量场景采样 1%-10%    | 降低服务器压力，节省成本 |
| **批量上报** | 积累多条数据后批量发送   | 减少网络请求，提升效率   |
| **空闲上报** | 使用 requestIdleCallback | 避免影响用户交互         |
| **压缩数据** | 使用 gzip 压缩上报数据   | 减少带宽消耗             |
| **异常处理** | 监控代码出错不影响业务   | 保证业务稳定性           |

### 7.2 常见陷阱

```javascript
// ❌ 错误：直接在主线程频繁上报
setInterval(() =&gt; {
  fetch('/api/performance', { body: JSON.stringify(metrics) })
}, 1000)

// ✅ 正确：使用 sendBeacon 和批量上报
navigator.sendBeacon('/api/performance', JSON.stringify(batchMetrics))

// ❌ 错误：监控代码报错影响业务
try {
  // 业务代码
} catch (error) {
  // 监控代码出错，业务中断
  reportError(error)
}

// ✅ 正确：监控代码隔离
try {
  // 业务代码
} catch (error) {
  // 监控代码包裹在 try-catch 中
  try {
    reportError(error)
  } catch (monitorError) {
    console.error('监控报错:', monitorError)
  }
}
```

### 7.3 隐私与合规

```javascript
// 脱敏处理用户数据
class DataSanitizer {
  sanitize(metrics) {
    const sanitized = { ...metrics }

    // 移除 URL 中的敏感参数
    if (sanitized.url) {
      sanitized.url = this.removeSensitiveParams(sanitized.url)
    }

    // 移除用户标识
    delete sanitized.userId
    delete sanitized.userName

    return sanitized
  }

  removeSensitiveParams(url) {
    const sensitiveParams = ['token', 'password', 'email', 'phone']
    const urlObj = new URL(url)

    sensitiveParams.forEach(param =&gt; {
      urlObj.searchParams.delete(param)
    })

    return urlObj.toString()
  }
}
```

## 八、总结

前端性能监控是一个系统工程，需要从指标采集、数据上报、可视化展示到智能告警形成完整闭环。

**关键要点：**

1. **选择合适的指标**：Web Vitals + 业务自定义指标
2. **优化上报策略**：采样、批量、空闲上报
3. **建立告警机制**：阈值告警 + 异常检测
4. **持续迭代优化**：监控数据驱动优化决策

**监控是优化的前提，没有监控，优化就是盲人摸象。**
