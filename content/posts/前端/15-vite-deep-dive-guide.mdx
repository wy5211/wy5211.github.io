## 前言

在 2020 年，Vue.js 作者尤雨溪创建了 Vite，一个革命性的前端构建工具。短短几年时间，Vite 已经成为前端开发的首选工具之一，被 React、Vue、Svelte 等主流框架广泛采用。

Vite 的核心优势在于：

- **极速的开发服务器启动**：利用浏览器原生 ESM 能力，无需打包即可启动
- **即时热更新**：模块级别的 HMR，无论项目多大都能快速响应
- **高效的生产构建**：基于 Rollup 的预配置构建，输出高度优化的静态资源
- **开箱即用**：TypeScript、JSX、CSS 预处理器等无需配置即可使用

本文将深入探讨 Vite 的各个方面，帮助你从入门到精通。

## Vite 的核心原理

### 开发环境 vs 生产环境

Vite 在开发环境和生产环境下采用不同的策略：

**开发环境**：

- 利用浏览器原生 ESM（ES Modules）能力
- 按需编译，只编译当前页面用到的文件
- 使用 esbuild 进行依赖预构建（比 JavaScript 工具快 10-100 倍）
- 源码通过原生 ESM 提供给浏览器

**生产环境**：

- 使用 Rollup 进行打包
- 代码分割、Tree-shaking、懒加载等优化
- 支持多种输出格式（ESM、CommonJS、UMD）

### 依赖预构建

Vite 会将项目的依赖项预构建成一个单独的文件，原因如下：

1. **CommonJS/UMD 兼容性**：将非 ESM 格式的依赖转换为 ESM
2. **性能优化**：将多个小文件合并，减少 HTTP 请求

预构建的文件存储在 `node_modules/.vite` 目录下，只有以下情况才会重新构建：

- package.json 的 dependencies 变化
- 包管理器的 lockfile 变化
- `vite.config.js` 中相关字段变化

### 模块热更新（HMR）

Vite 的 HMR 实现原理：

1. **客户端注入**：Vite 会自动注入 HMR 客户端代码
2. **模块依赖图**：Vite 维护一个模块依赖图，记录模块间的导入关系
3. **精确更新**：当文件变化时，Vite 只更新受影响的模块链
4. **边界处理**：如果模块没有 HMR 处理函数，向上查找最近的 HMR 边界

### 配置文件结构

```typescript
// vite.config.ts
import { defineConfig } from "vite";
import react from "@vitejs/plugin-react";
import path from "path";

export default defineConfig({
  // 插件
  plugins: [react()],

  // 开发服务器
  server: {
    host: "0.0.0.0",
    port: 3000,
    open: true,
    cors: true,
    proxy: {
      "/api": {
        target: "http://localhost:8080",
        changeOrigin: true,
        rewrite: (path) => path.replace(/^\/api/, ""),
      },
    },
  },

  // 构建配置
  build: {
    outDir: "dist",
    sourcemap: true,
    minify: "terser",
    rollupOptions: {
      output: {
        manualChunks: {
          vendor: ["react", "react-dom"],
          utils: ["lodash", "axios"],
        },
      },
    },
  },

  // 路径别名
  resolve: {
    alias: {
      "@": path.resolve(__dirname, "./src"),
    },
  },
});
```

### 环境变量

Vite 使用 `import.meta.env` 暴露环境变量：

```bash
# .env.development
VITE_API_URL=http://localhost:8080/api
VITE_APP_TITLE=开发环境

# .env.production
VITE_API_URL=https://api.example.com
VITE_APP_TITLE=生产环境
```

```typescript
// 使用环境变量
console.log(import.meta.env.VITE_API_URL);
console.log(import.meta.env.MODE); // 'development' | 'production'
console.log(import.meta.env.DEV); // true in development
console.log(import.meta.env.PROD); // true in production
```

**重要规则**：

- 环境变量必须以 `VITE_` 开头才会暴露给客户端
- `import.meta.env` 是不可变对象
- 支持 TypeScript 类型定义：

```typescript
// vite-env.d.ts
interface ImportMetaEnv {
  readonly VITE_API_URL: string;
  readonly VITE_APP_TITLE: string;
}

interface ImportMeta {
  readonly env: ImportMetaEnv;
}
```

### 构建优化

#### 代码分割策略

```typescript
export default defineConfig({
  build: {
    rollupOptions: {
      output: {
        // 手动分割代码块
        manualChunks(id) {
          // 将 node_modules 中的包分割到 vendor
          if (id.includes("node_modules")) {
            // 按包名分割
            const match = id.match(/node_modules\/([^/]+)/);
            if (match) {
              return `vendor-${match[1].replace("@", "")}`;
            }
            return "vendor";
          }

          // 将组件库单独分割
          if (id.includes("/components/")) {
            return "components";
          }

          // 将工具函数单独分割
          if (id.includes("/utils/")) {
            return "utils";
          }
        },
      },
    },
  },
});
```

## 插件开发

Vite 插件基于 Rollup 插件接口扩展而来，增加了 Vite 特有的钩子。

### 插件结构

```typescript
import type { Plugin } from "vite";

interface MyPluginOptions {
  // 插件选项
}

export default function myPlugin(options: MyPluginOptions = {}): Plugin {
  return {
    name: "vite-plugin-my-plugin",

    // 应用模式
    apply: "build", // 'build' | 'serve' | ((config, { command }) => boolean)

    // 执行顺序
    enforce: "pre", // 'pre' | 'post'

    // Rollup 钩子
    options(options) {
      // 修改 Rollup 配置
      return options;
    },

    buildStart(options) {
      // 构建开始
    },

    resolveId(source, importer) {
      // 自定义模块解析
      if (source === "virtual:my-module") {
        return source;
      }
      return null;
    },

    load(id) {
      // 自定义加载逻辑
      if (id === "virtual:my-module") {
        return 'export default "This is virtual module"';
      }
      return null;
    },

    transform(code, id) {
      // 转换代码
      if (id.endsWith(".custom")) {
        return {
          code: compiledCode,
          map: sourceMap,
        };
      }
    },

    buildEnd(error) {
      // 构建结束
    },

    // Vite 特有钩子
    config(config) {
      // 修改 Vite 配置
      return {
        define: {
          __MY_PLUGIN__: JSON.stringify(true),
        },
      };
    },

    configResolved(resolvedConfig) {
      // 配置解析完成
      console.log("Config resolved:", resolvedConfig);
    },

    configureServer(server) {
      // 配置开发服务器
      server.middlewares.use((req, res, next) => {
        // 自定义中间件
        next();
      });
    },

    transformIndexHtml(html) {
      // 转换 index.html
      return html.replace(
        /&lt;\/title&gt;/,
        `&lt;/title&gt;&lt;script&gt;console.log('injected')&lt;/script&gt;`
      );
    },

    handleHotUpdate({ file, server, modules }) {
      // 自定义 HMR 处理
      if (file.endsWith(".custom")) {
        server.ws.send({
          type: "custom",
          event: "custom-update",
          data: { file },
        });
        return [];
      }
    },
  };
}
```

### 实战案例：自动导入插件

```typescript
import type { Plugin } from 'vite'
import { parse } from '@babel/parser'
import traverse from '@babel/traverse'
import generate from '@babel/generator'

export default function autoImportPlugin(): Plugin {
  const autoImports = {
    React: 'import React from "react"',
    useState: 'import { useState } from "react"',
    useEffect: 'import { useEffect } from "react"',
    useMemo: 'import { useMemo } from "react"',
    useCallback: 'import { useCallback } from "react"'
  }

  return {
    name: 'vite-plugin-auto-import',
    enforce: 'pre',

    transform(code, id) {
      // 只处理 JSX/TSX 文件
      if (!/\.(t|j)sx$/.test(id)) return null

      const ast = parse(code, {
        sourceType: 'module',
        plugins: ['jsx', 'typescript']
      })

      const usedIdentifiers = new Set&lt;string&gt;()

      traverse(ast, {
        Identifier(path) {
          const name = path.node.name
          if (autoImports[name] && !path.scope.hasBinding(name)) {
            usedIdentifiers.add(name)
          }
        }
      })

      if (usedIdentifiers.size === 0) return null

      const imports = Array.from(usedIdentifiers)
        .map(name =&gt; autoImports[name])
        .join('\n')

      return {
        code: `${imports}\n${code}`,
        map: null
      }
    }
  }
}
```

### 实战案例：SVG 组件化插件

```typescript
import type { Plugin } from 'vite'
import fs from 'fs'
import path from 'path'

export default function svgComponentPlugin(): Plugin {
  return {
    name: 'vite-plugin-svg-component',
    enforce: 'pre',

    resolveId(source) {
      if (source.endsWith('.svg?component')) {
        return source
      }
      return null
    },

    load(id) {
      if (!id.endsWith('.svg?component')) return null

      const filePath = id.replace('?component', '')
      const svgContent = fs.readFileSync(filePath, 'utf-8')

      // 转换为 React 组件
      const code = `
        import React from 'react';

        export default function SvgComponent(props) {
          return (
            &lt;svg
              dangerouslySetInnerHTML={{ __html: ${JSON.stringify(svgContent)} }}
              {...props}
            /&gt;
          );
        }
      `

      return code
    }
  }
}

// 使用
import Logo from './logo.svg?component'
&lt;Logo className="logo" /&gt;
```

## 迁移指南

### 从 Webpack 迁移

**1. 配置迁移**

| Webpack           | Vite                                        |
| ----------------- | ------------------------------------------- |
| `entry`           | `build.rollupOptions.input`                 |
| `output.path`     | `build.outDir`                              |
| `output.filename` | `build.rollupOptions.output.entryFileNames` |
| `resolve.alias`   | `resolve.alias`                             |
| `module.rules`    | `plugins` + `css`                           |
| `devServer.proxy` | `server.proxy`                              |

**2. 语法迁移**

```javascript
// Webpack
const ENV = process.env.NODE_ENV;
require("./module");

// Vite
const ENV = import.meta.env.MODE;
import("./module");
```

**3. 需要注意的差异**

- Vite 不支持 CommonJS 模块的 `require()`
- 环境变量使用 `import.meta.env` 而非 `process.env`
- 静态资源处理方式不同
- HMR 实现机制不同


```typescript
  // 使用 web-vitals 库
  import("web-vitals").then(({ getCLS, getFID, getFCP, getLCP, getTTFB }) => {

    getFID(reportWebVitals);
    getFCP(reportWebVitals);
    getLCP(reportWebVitals);
    getTTFB(reportWebVitals);
  });
```
