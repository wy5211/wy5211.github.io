# 前端可访问性(A11y)实践指南：让每个人都能使用你的网站

可访问性（Accessibility，简称 A11y）不是锦上添花，而是前端开发的基本要求。全球约有15%的人口有某种形式的残疾，良好的可访问性意味着你的产品能够服务更广泛的用户群体。

本文系统讲解前端可访问性实践，帮你打造对所有人都友好的网站。

## 一、为什么可访问性很重要

### 1.1 用户群体

可访问性受益者包括：

- **视觉障碍用户** - 盲人、低视力、色盲
- **听觉障碍用户** - 聋人、重听
- **运动障碍用户** - 无法使用鼠标、颤抖
- **认知障碍用户** - 阅读障碍、注意力缺陷
- **临时障碍用户** - 手臂骨折、嘈杂环境
- **老年用户** - 视力下降、反应变慢

### 1.2 商业价值

```
✅ 法律合规 - 许多国家要求网站可访问
✅ 用户规模 - 覆盖全球15%的残疾人口
✅ SEO 优化 - 语义化结构利于搜索引擎
✅ 代码质量 - 更清晰的代码结构
✅ 品牌形象 - 展示企业社会责任
```

### 1.3 WCAG 标准

Web 内容可访问性指南（WCAG）是国际标准，遵循 **POUR** 原则：

| 原则                  | 说明             | 示例               |
| --------------------- | ---------------- | ------------------ |
| 可感知 Perceivable    | 信息必须能被感知 | 图片alt文本、字幕  |
| 可操作 Operable       | 界面必须能被操作 | 键盘导航、足够时间 |
| 可理解 Understandable | 内容必须能被理解 | 清晰语言、一致性   |
| 健壮 Robust           | 兼容各种工具     | 语义化HTML、ARIA   |

## 二、语义化 HTML：可访问性的基础

### 2.1 使用正确的标签

```html
<!-- ❌ 错误：滥用 div -->
<div onclick="navigate()">首页</div>
<div class="header">
  <div class="nav">...</div>
</div>

<!-- ✅ 正确：使用语义化标签 -->
<a href="/">首页</a>
<header>
  <nav aria-label="主导航">...</nav>
</header>
```

### 2.2 标题层级

正确的标题层级帮助屏幕阅读器用户理解页面结构：

```html
<!-- ✅ 正确的标题层级 -->
<h1>产品名称</h1>
<h2>功能介绍</h2>
<h3>核心功能</h3>
<h3>高级功能</h3>
<h2>价格方案</h2>
<h3>个人版</h3>
<h3>企业版</h3>

<!-- ❌ 错误：跳级或仅用于样式 -->
<h1>标题</h1>
<h3>跳过了 h2</h3>
<!-- 不要跳级 -->
<p class="h2-style">这只是个段落</p>
<!-- 不要用段落模拟标题 -->
```

### 2.3 地标角色

HTML5 语义化标签自动带有地标角色：

```html
<header role="banner">页头</header>
<nav role="navigation">导航</nav>
<main role="main">主要内容</main>
<aside role="complementary">侧边栏</aside>
<footer role="contentinfo">页脚</footer>
```

### 2.4 列表结构

```html
<!-- 无序列表 -->
<ul>
  <li>苹果</li>
  <li>香蕉</li>
  <li>橙子</li>
</ul>

<!-- 有序列表 -->
<ol>
  <li>第一步：打开设置</li>
  <li>第二步：选择语言</li>
  <li>第三步：保存</li>
</ol>

<!-- 描述列表 -->
<dl>
  <dt>HTML</dt>
  <dd>超文本标记语言</dd>
  <dt>CSS</dt>
  <dd>层叠样式表</dd>
</dl>
```

## 三、表单可访问性

### 3.1 标签关联

```html
<!-- ✅ 使用 label 包裹 -->
<label>
  用户名
  <input type="text" name="username" />
</label>

<!-- ✅ 使用 for 属性关联 -->
<label for="email">邮箱</label>
<input type="email" id="email" name="email" />

<!-- ❌ 错误：没有关联标签 -->
<div>用户名</div>
<input type="text" />
```

### 3.2 必填与错误提示

```html
<div class="form-group">
  <label for="phone">
    手机号
    <span aria-hidden="true">*</span>
    <span class="sr-only">（必填）</span>
  </label>
  <input
    type="tel"
    id="phone"
    name="phone"
    required
    aria-required="true"
    aria-invalid="false"
    aria-describedby="phone-error"
  />
  <span id="phone-error" class="error" role="alert">
    <!-- 错误信息将在这里显示 -->
  </span>
</div>
```

### 3.3 表单分组

```html
<fieldset>
  <legend>配送方式</legend>
  <label>
    <input type="radio" name="shipping" value="express" />
    快递配送
  </label>
  <label>
    <input type="radio" name="shipping" value="pickup" />
    到店自提
  </label>
</fieldset>
```

## 四、图片与媒体

### 4.1 图片替代文本

```html
<!-- 装饰性图片：使用空 alt -->
<img src="decoration.png" alt="" role="presentation" />

<!-- 信息性图片：描述性 alt -->
<img src="chart.png" alt="2024年销售额增长30%的柱状图" />

<!-- 复杂图片：使用 longdesc 或 aria-describedby -->
<img src="complex-chart.png" alt="销售趋势图" aria-describedby="chart-desc" />
<div id="chart-desc" class="sr-only">
  这张图表展示了从2020年到2024年的销售趋势...
</div>

<!-- 功能性图片：描述功能 -->
<button aria-label="搜索">
  <img src="search-icon.svg" alt="" />
</button>
```

### 4.2 视频与音频

```html
<video controls>
  <source src="video.mp4" type="video/mp4" />
  <track kind="captions" src="captions.vtt" srclang="zh" label="中文字幕" />
  <track kind="subtitles" src="subtitles.vtt" srclang="en" label="English" />
  您的浏览器不支持视频播放
</video>

<audio controls>
  <source src="audio.mp3" type="audio/mpeg" />
  <a href="transcript.html">查看文字稿</a>
</audio>
```

## 五、ARIA 属性详解

### 5.1 何时使用 ARIA

```
原生 HTML 能解决的，不要用 ARIA

第一法则：不要改变原生语义
第二法则：所有交互元素必须能键盘操作
```

```html
<!-- ❌ 不推荐：用 ARIA 模拟按钮 -->
<div role="button" tabindex="0" onclick="submit()">提交</div>

<!-- ✅ 推荐：使用原生按钮 -->
<button type="submit">提交</button>
```

### 5.2 常用 ARIA 属性

```html
<!-- aria-label：添加可访问名称 -->
<button aria-label="关闭对话框">
  <svg>...</svg>
</button>

<!-- aria-labelledby：引用其他元素作为标签 -->
<div id="dialog-title">确认删除</div>
<div role="dialog" aria-labelledby="dialog-title">确定要删除这个文件吗？</div>

<!-- aria-describedby：引用描述信息 -->
<input type="password" aria-describedby="password-hint" />
<span id="password-hint">密码至少8个字符</span>

<!-- aria-live：动态内容通知 -->
<div aria-live="polite" id="status">
  <!-- 状态更新将在这里宣布 -->
</div>

<!-- aria-expanded：展开/折叠状态 -->
<button aria-expanded="false" aria-controls="menu">展开菜单</button>
<ul id="menu" hidden>
  <li>选项1</li>
  <li>选项2</li>
</ul>
```

### 5.3 角色（Role）

```html
<!-- 对话框 -->
<div role="dialog" aria-modal="true" aria-labelledby="dialog-title">
  <h2 id="dialog-title">确认操作</h2>
  <button>确认</button>
  <button>取消</button>
</div>

<!-- 标签页 -->
<div role="tablist">
  <button role="tab" aria-selected="true" aria-controls="panel1">标签1</button>
  <button role="tab" aria-selected="false" aria-controls="panel2">标签2</button>
</div>
<div role="tabpanel" id="panel1">内容1</div>
<div role="tabpanel" id="panel2" hidden>内容2</div>

<!-- 提醒/警告 -->
<div role="alert">您的会话将在5分钟后过期</div>
```

## 六、键盘导航

### 6.1 焦点管理

```css
/* 可见的焦点样式 */
:focus {
  outline: 2px solid #005fcc;
  outline-offset: 2px;
}

/* 不要移除焦点样式 */
:focus {
  outline: none; /* ❌ 绝对不要这样做 */
}

/* 使用 :focus-visible 仅在键盘导航时显示 */
:focus:not(:focus-visible) {
  outline: none;
}

:focus-visible {
  outline: 2px solid #005fcc;
  outline-offset: 2px;
}
```

### 6.2 Tab 顺序

```html
<!-- 使用 tabindex -->
<button tabindex="0">可聚焦</button>
<div tabindex="-1">可编程聚焦，不在 Tab 顺序中</div>
<div tabindex="2">避免使用 &gt;0 的值</div>
<!-- &gt; 表示 > -->

<!-- 跳过导航链接 -->
<a href="#main-content" class="skip-link">跳到主要内容</a>
<nav>...</nav>
<main id="main-content">...</main>
```

```css
/* 跳过链接样式 */
.skip-link {
  position: absolute;
  top: -40px;
  left: 0;
  padding: 8px 16px;
  background: #000;
  color: #fff;
  z-index: 100;
}

.skip-link:focus {
  top: 0;
}
```

### 6.3 键盘事件处理

```javascript
// 自定义组件必须支持键盘操作
const menu = document.querySelector('[role="menu"]');

menu.addEventListener('keydown', (e) =&gt; {
  const items = menu.querySelectorAll('[role="menuitem"]');
  const currentIndex = Array.from(items).indexOf(document.activeElement);

  switch (e.key) {
    case 'ArrowDown':
      e.preventDefault();
      items[(currentIndex + 1) % items.length].focus();
      break;
    case 'ArrowUp':
      e.preventDefault();
      items[(currentIndex - 1 + items.length) % items.length].focus();
      break;
    case 'Escape':
      menu.previousElementSibling.focus();
      menu.hidden = true;
      break;
    case 'Enter':
    case ' ':
      e.preventDefault();
      document.activeElement.click();
      break;
  }
});
```

## 七、色彩与对比度

### 7.1 对比度要求

WCAG 对比度要求：

| 级别 | 正常文本 | 大文本（&gt;18px 或 粗体 &gt;14px） |
| ---- | -------- | ----------------------------------- |
| AA   | 4.5:1    | 3:1                                 |
| AAA  | 7:1      | 4.5:1                               |

```css
/* ✅ 良好对比度 */
.text {
  color: #333333;
  background: #ffffff;
  /* 对比度: 12.6:1 ✓ */
}

/* ❌ 对比度不足 */
.poor-contrast {
  color: #999999;
  background: #ffffff;
  /* 对比度: 2.8:1 ✗ */
}
```

### 7.2 不要仅依赖颜色

```html
<!-- ❌ 错误：仅用颜色表示错误 -->
<input class="error" style="border-color: red;" />

<!-- ✅ 正确：颜色 + 文字 + 图标 -->
<div class="form-group error">
  <input style="border-color: red;" />
  <span class="error-message">
    <svg aria-hidden="true">...</svg>
    请输入有效的邮箱地址
  </span>
</div>
```

### 7.3 焦点指示器

```css
/* 自定义焦点样式 */
:focus-visible {
  outline: none;
  box-shadow: 0 0 0 3px rgba(0, 95, 204, 0.5);
}

/* 高对比度模式支持 */
@media (prefers-contrast: high) {
  :focus-visible {
    outline: 3px solid #000;
  }
}
```

## 八、动态内容与动画

### 8.1 减少动画

```css
/* 尊重用户偏好 */
@media (prefers-reduced-motion: reduce) {
  *,
  *::before,
  *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
    scroll-behavior: auto !important;
  }
}

/* 默认动画 */
.element {
  transition: transform 0.3s ease;
}

@media (prefers-reduced-motion: no-preference) {
  .element:hover {
    transform: scale(1.05);
  }
}
```

### 8.2 动态内容通知

```html
<!-- 实时区域：通知内容变化 -->
<div aria-live="polite" aria-atomic="true" id="notifications">
  <!-- 新消息将在这里宣布 -->
</div>

<!-- 紧急通知 -->
<div role="alert" aria-live="assertive">系统将在5分钟后维护</div>

<!-- 状态更新 -->
<div aria-live="polite" aria-relevant="additions">
  <span>新评论已添加</span>
</div>
```

### 8.3 加载状态

```html
<!-- 加载指示器 -->
<div role="status" aria-live="polite">
  <span class="sr-only">加载中...</span>
  <svg aria-hidden="true" class="spinner">...</svg>

  aria-valuemax="100"

</div>
```

## 十、常见问题与解决方案

### 10.1 图标按钮

```html
<!-- ✅ 图标按钮 -->
<button aria-label="添加到收藏">
  <svg aria-hidden="true">
    <use href="#heart-icon"></use>
  </svg>
</button>

<!-- ✅ 图标 + 文字 -->
<button>
  <svg aria-hidden="true">...</svg>
  <span>收藏</span>
</button>
```

### 10.2 模态对话框

```javascript
function openModal(modal) {
  // 保存当前焦点
  const previousFocus = document.activeElement;

  // 显示模态框
  modal.hidden = false;
  modal.setAttribute("aria-hidden", "false");

  // 聚焦到模态框
  const firstFocusable = modal.querySelector("button, [href], input");
  firstFocusable?.focus();

  // 陷阱焦点
  const focusTrap = createFocusTrap(modal);

  function closeModal() {
    modal.hidden = true;
    modal.setAttribute("aria-hidden", "true");
    focusTrap.deactivate();
    previousFocus?.focus();
  }
}
```

### 10.3 无限滚动

```html
<div role="feed" aria-busy="false" aria-label="文章列表">
  <article aria-posinset="1" aria-setsize="-1">...</article>
  <article aria-posinset="2" aria-setsize="-1">...</article>
  <!-- 更多文章 -->
</div>

<button id="load-more">加载更多</button>
```

```javascript
// 加载状态
feed.setAttribute("aria-busy", "true");

// 加载完成
feed.setAttribute("aria-busy", "false");
```

## 十二、总结

可访问性不是额外负担，而是高质量代码的标志：

1. **语义化优先** - 用正确的 HTML 标签
2. **键盘友好** - 所有交互可键盘操作
3. **清晰反馈** - 状态变化要通知用户
4. **持续测试** - 自动化 + 手动测试
5. **用户为先** - 考虑各种能力水平的用户

```
记住：可访问性是功能，不是 Bug 修复
```
