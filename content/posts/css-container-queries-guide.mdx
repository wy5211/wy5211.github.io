## 引言

在传统响应式设计中，我们习惯使用媒体查询（Media Queries）来根据视口宽度调整布局。但这种方式存在一个根本性问题：**组件的样式依赖于整个页面的视口尺寸，而不是组件自身的容器**。

CSS容器查询（Container Queries）的出现彻底改变了这一局面，它允许我们根据**父容器的尺寸**来调整子元素的样式，让组件真正实现了独立性和可复用性。

## 为什么需要容器查询？

### 传统媒体查询的局限性

考虑一个场景：你开发了一个精美的卡片组件，在不同视口下有不同的布局：

```css
/* 传统方式 - 基于视口宽度 */
.card {
  display: flex;
  flex-direction: column;
}

@media (min-width: 768px) {
  .card {
    flex-direction: row;
  }
}
```

这个卡片在桌面端的侧边栏中使用时，即使侧边栏宽度只有300px，卡片仍然会呈现横向布局，导致布局错乱。

### 容器查询的优势

容器查询让组件的样式**自适应其容器**，而不是视口：

```css
/* 现代方式 - 基于容器宽度 */
.card-container {
  container-type: inline-size;
  container-name: card;
}

@container card (min-width: 400px) {
  .card {
    flex-direction: row;
  }
}
```

这样，无论组件被放置在页面的哪个位置，都能根据实际容器大小智能调整布局。

## 容器查询基础语法

### 1. 定义容器

使用 `container-type` 属性将元素声明为查询容器：

```css
.sidebar {
  container-type: inline-size;
}

.main-content {
  container-type: inline-size;
}
```

`container-type` 有三个值：

- **`inline-size`**：基于容器的行向尺寸（通常是宽度）进行查询（推荐）
- **`block-size`**：基于容器的块向尺寸（通常是高度）进行查询
- **`size`**：同时基于宽度和高度进行查询（性能影响较大，谨慎使用）

### 2. 命名容器（可选）

为容器命名可以让查询更精确：

```css
.sidebar {
  container-type: inline-size;
  container-name: sidebar-container;
}

/* 简写形式 */
.main-content {
  container: main / inline-size;
}
```

### 3. 编写容器查询

使用 `@container` 规则编写查询：

```css
/* 查询最近的容器 */
@container (min-width: 500px) {
  .widget {
    display: grid;
    grid-template-columns: 1fr 1fr;
  }
}

/* 查询指定名称的容器 */
@container sidebar-container (min-width: 300px) {
  .menu-item {
    font-size: 16px;
  }
}
```

## 实战案例：响应式卡片组件

下面是一个完整的响应式卡片组件示例，展示容器查询的强大之处：

```html
<div class="container">
  <div class="card">
    <img src="photo.jpg" alt="示例图片" class="card-image" />
    <div class="card-content">
      <h3 class="card-title">卡片标题</h3>
      <p class="card-description">
        这是卡片的描述文字，可以根据容器大小自动调整布局。
      </p>
      <button class="card-button">查看详情</button>
    </div>
  </div>
</div>
```

```css
/* 定义容器 */
.container {
  container-type: inline-size;
  container-name: card-wrapper;
}

/* 基础样式 - 小容器 */
.card {
  display: flex;
  flex-direction: column;
  background: white;
  border-radius: 8px;
  overflow: hidden;
  box-shadow: 0 2px 8px rgba(0, 0, 0, 0.1);
}

.card-image {
  width: 100%;
  height: 200px;
  object-fit: cover;
}

.card-content {
  padding: 16px;
}

.card-title {
  font-size: 18px;
  margin-bottom: 8px;
}

.card-description {
  font-size: 14px;
  color: #666;
  margin-bottom: 12px;
}

.card-button {
  width: 100%;
  padding: 10px 20px;
  background: #007bff;
  color: white;
  border: none;
  border-radius: 4px;
  cursor: pointer;
}

/* 中等容器 - 卡片横向布局 */
@container card-wrapper (min-width: 400px) {
  .card {
    flex-direction: row;
  }

  .card-image {
    width: 200px;
    height: auto;
  }

  .card-content {
    display: flex;
    flex-direction: column;
    justify-content: center;
    flex: 1;
  }

  .card-button {
    width: auto;
    align-self: flex-start;
  }
}

/* 大容器 - 网格布局 */
@container card-wrapper (min-width: 800px) {
  .card {
    flex-direction: column;
  }

  .card-image {
    width: 100%;
    height: 250px;
  }

  .card-content {
    padding: 24px;
  }

  .card-title {
    font-size: 24px;
  }

  .card-description {
    font-size: 16px;
  }
}
```

这个卡片组件会根据容器大小自动调整：

- **小容器（&lt; 400px）**：纵向布局，图片在上，按钮全宽
- **中等容器（400px - 800px）**：横向布局，图片在左侧，按钮自适应宽度
- **大容器（&gt; 800px）**：回到纵向布局，但内容更宽松，字体更大

## 容器查询单位

CSS还引入了基于容器的相对单位，类似于 `vw`、`vh`，但是基于容器尺寸：

| 单位    | 说明                      | 示例                |
| ------- | ------------------------- | ------------------- |
| `cqw`   | 容器宽度的 1%             | `width: 50cqw`      |
| `cqh`   | 容器高度的 1%             | `height: 100cqh`    |
| `cqmin` | `cqw` 和 `cqh` 中较小的值 | `font-size: 5cqmin` |
| `cqmax` | `cqw` 和 `cqh` 中较大的值 | `padding: 2cqmax`   |

### 实用示例：容器自适应字体

```css
.container {
  container-type: inline-size;
}

.headline {
  /* 容器越宽，字体越大 */
  font-size: clamp(16px, 5cqw, 48px);
  /* 或者直接使用 */
  font-size: 5cqw;
}
```

## 高级技巧

### 1. 嵌套容器查询

容器可以嵌套，内层容器会同时响应自身和外层的查询：

```css
.outer-container {
  container-type: inline-size;
  container-name: outer;
}

.inner-container {
  container-type: inline-size;
  container-name: inner;
}

/* 外层容器查询 */
@container outer (min-width: 1000px) {
  .inner-container {
    padding: 20px;
  }
}

/* 内层容器查询 */
@container inner (min-width: 400px) {
  .item {
    display: flex;
  }
}
```

### 2. 结合CSS Grid

容器查询与Grid布局是完美搭档：

```css
.grid-container {
  container-type: inline-size;
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(250px, 1fr));
  gap: 20px;
}

@container (min-width: 600px) {
  .grid-item {
    grid-column: span 2;
  }
}

@container (min-width: 900px) {
  .grid-item.featured {
    grid-column: span 3;
  }
}
```

### 3. 容器查询 + CSS变量

使用CSS变量让容器查询更灵活：

```css
:root {
  --card-padding: 16px;
  --card-font-size: 14px;
}

.card-container {
  container-type: inline-size;
}

@container (min-width: 500px) {
  .card {
    --card-padding: 24px;
    --card-font-size: 16px;
  }
}

.card-content {
  padding: var(--card-padding);
  font-size: var(--card-font-size);
}
```

### 4. 多条件查询

可以像媒体查询一样使用多个条件：

```css
@container (min-width: 400px) and (max-width: 800px) {
  .element {
    /* 容器宽度在400px到800px之间 */
  }
}

/* 或者使用 or 逻辑 */
@container (min-width: 400px) or (orientation: landscape) {
  .element {
    /* 容器宽度大于400px 或 横向排列 */
  }
}
```

### 兼容性降级方案

对于旧浏览器，可以使用以下策略：

```css
/* 基础样式 - 所有浏览器都支持 */
.card {
  display: flex;
  flex-direction: column;
}

/* 现代浏览器 - 容器查询 */
@supports (container-type: inline-size) {
  .container {
    container-type: inline-size;
  }

  @container (min-width: 400px) {
    .card {
      flex-direction: row;
    }
  }
}

/* 旧浏览器 - 媒体查询降级 */
@supports not (container-type: inline-size) {
  @media (min-width: 768px) {
    .card {
      flex-direction: row;
    }
  }
}
```

## 性能优化建议

### 1. 合理使用 container-type

- 优先使用 `inline-size` 而不是 `size`
- 只在需要查询的容器上声明 `container-type`
- 避免在频繁变化的容器上使用

### 2. 避免过度嵌套

```css
/* ❌ 不推荐 - 过度嵌套 */
.level-1 {
  container-type: inline-size;
}

.level-2 {
  container-type: inline-size;
}

.level-3 {
  container-type: inline-size;
}

/* ✅ 推荐 - 只在必要层级声明 */
.level-1 {
  container-type: inline-size;
}
```

### 3. 使用容器名称提高性能

```css
/* 命名容器更高效 */
@container specific-name (min-width: 500px) {
  /* 浏览器直接定位到指定容器 */
}

/* 未命名容器需要向上查找 */
@container (min-width: 500px) {
  /* 浏览器需要遍历DOM树查找最近的容器 */
}
```

## 实际应用场景

### 1. 可复用组件库

容器查询让组件真正实现了"即插即用"：

```css
/* 组件样式不依赖外部环境 */
.component-container {
  container-type: inline-size;
}

@container (min-width: 300px) {
  .component {
    /* 小容器样式 */
  }
}

@container (min-width: 600px) {
  .component {
    /* 大容器样式 */
  }
}
```

无论组件被放在侧边栏、主内容区还是弹窗中，都能自动适应。

### 2. 响应式表单

```css
.form-container {
  container-type: inline-size;
}

.form-row {
  display: flex;
  flex-direction: column;
  gap: 12px;
}

@container (min-width: 500px) {
  .form-row {
    flex-direction: row;
  }

  .form-field {
    flex: 1;
  }
}
```

### 3. 主题切换

容器查询可以用于主题切换：

```css
.theme-container {
  container-type: inline-size;
}

/* 容器宽度 &lt; 600px 时使用紧凑主题 */
@container (max-width: 600px) {
  .theme-container {
    --spacing: 8px;
    --font-size: 14px;
  }
}

/* 容器宽度 &gt;= 600px 时使用标准主题 */
@container (min-width: 600px) {
  .theme-container {
    --spacing: 16px;
    --font-size: 16px;
  }
}
```

## 常见陷阱与解决方案

### 陷阱1：忘记设置 container-type

```css
/* ❌ 错误 - 没有声明容器 */
.sidebar {
  width: 300px;
}

@container (min-width: 300px) {
  /* 这个查询永远不会生效 */
}

/* ✅ 正确 */
.sidebar {
  width: 300px;
  container-type: inline-size;
}
```

### 陷阱2：容器查询 vs 媒体查询混淆

```css
/* 容器查询用于组件内部布局 */
@container (min-width: 400px) {
  .component {
    /* 基于容器调整 */
  }
}

/* 媒体查询用于页面整体布局 */
@media (min-width: 768px) {
  .page-layout {
    /* 基于视口调整 */
  }
}
```

### 陷阱3：容器的继承问题

容器查询**不会继承**，必须显式声明：

```css
/* ❌ 子元素不会自动成为容器 */
.parent {
  container-type: inline-size;
}

.child {
  /* 需要显式声明 */
  container-type: inline-size;
}
```

## 总结

CSS容器查询是现代响应式设计的重要工具，它解决了传统媒体查询的根本性局限，让组件真正实现了**基于容器的自适应**。

**核心要点：**

1. 使用 `container-type` 声明查询容器
2. 使用 `@container` 规则编写容器查询
3. 优先使用 `inline-size` 而不是 `size`
4. 为容器命名提高查询精确度和性能
5. 结合容器查询单位（`cqw`、`cqh`）实现更灵活的布局

**最佳实践：**

- ✅ 让组件独立于页面布局
- ✅ 提高组件的可复用性
- ✅ 简化响应式逻辑
- ✅ 更好的代码组织和维护性

容器查询的普及标志着前端开发进入了一个新时代——**组件优先的响应式设计**。掌握这项技术，将让你的CSS代码更加优雅、可维护、可复用。
