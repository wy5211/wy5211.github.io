Function Calling 是大模型应用开发的核心能力之一，它让 AI 从"只会聊天"进化为"能干活的助手"。本文将深入探讨 Function Calling 的原理、最佳实践和生产级应用方案。

## 什么是 Function Calling？

Function Calling（函数调用）允许大模型在生成回复时，识别用户意图并调用预定义的外部函数或 API。这使得 AI 能够：

- 查询实时数据（天气、股价、数据库）
- 执行操作（发送邮件、创建订单、更新状态）
- 与外部系统集成（数据库、第三方 API、企业服务）

### 核心流程

```text
用户输入 → 模型理解意图 → 选择合适的函数 → 生成调用参数
    → 执行函数 → 返回结果 → 模型生成最终回复
```

export async function getWeather(city: string, unit: string = "celsius") {
  // 调用天气 API
  const response = await fetch(
    `https://api.openweathermap.org/data/2.5/weather?q=${city}&appid=${process.env.WEATHER_API_KEY}`

  };
}

### 3. Function Calling 执行器

```typescript
// executor.ts
import { toolRegistry, getToolSchemas } from "./tools/registry";

export class FunctionCallingExecutor {
  private openai: OpenAI;
  private maxIterations: number;

  constructor(maxIterations: number = 5) {
    this.openai = new OpenAI({ apiKey: process.env.OPENAI_API_KEY });
    this.maxIterations = maxIterations;
  }

  async execute(userMessage: string, conversationHistory: Message[] = []) {
    const messages: Message[] = [
      ...conversationHistory,
      { role: "user", content: userMessage }
    ];

    const tools = getToolSchemas();

    for (let i = 0; i &lt; this.maxIterations; i++) {
      // 调用 LLM
      const response = await this.openai.chat.completions.create({
        model: "gpt-4-turbo-preview",
        messages,
        tools,
        tool_choice: "auto"
      });

      const assistantMessage = response.choices[0].message;
      messages.push(assistantMessage);

      // 检查是否需要调用函数
      if (!assistantMessage.tool_calls?.length) {
        // 没有函数调用，返回最终回复
        return {
          message: assistantMessage.content,
          conversationHistory: messages
        };
      }

      // 执行所有函数调用
      for (const toolCall of assistantMessage.tool_calls) {
        const functionName = toolCall.function.name;
        const args = JSON.parse(toolCall.function.arguments);

        console.log(`[调用] ${functionName}`, args);

        try {
          // 执行函数
          const result = await this.executeFunction(functionName, args);

          // 将结果添加到对话历史
          messages.push({
            role: "tool",
            tool_call_id: toolCall.id,
            content: JSON.stringify(result)
          });

          console.log(`[结果] ${functionName}`, result);
        } catch (error) {
          // 错误处理
          messages.push({
            role: "tool",
            tool_call_id: toolCall.id,
            content: JSON.stringify({
              error: true,
              message: error.message
            })
          });
        }
      }
    }

    throw new Error("超过最大迭代次数");
  }

  private async executeFunction(name: string, args: Record&lt;string, any&gt;) {
    const tool = toolRegistry[name];
    if (!tool) {
      throw new Error(`未知的工具: ${name}`);
    }

    return await tool.handler(args);
  }
}
```

## 高级技巧

### 1. 并行调用优化

```typescript
// 当多个函数相互独立时，可以并行执行
async function executeToolCalls(toolCalls: ToolCall[]) {
  // 分组：独立的调用可以并行
  const independentCalls = toolCalls.filter(
    (call) => !hasDependency(call, toolCalls)
  );

  // 并行执行
  const results = await Promise.all(
    independentCalls.map((call) => executeFunction(call))
  );

  return results;
}
```

### 2. 函数调用缓存

```typescript
import NodeCache from "node-cache";

const cache = new NodeCache({ stdTTL: 300 }); // 5 分钟缓存

async function executeFunctionWithCache(name: string, args: any) {
  const cacheKey = `${name}:${JSON.stringify(args)}`;

  // 检查缓存
  const cached = cache.get(cacheKey);
  if (cached) {
    console.log(`[缓存命中] ${name}`);
    return cached;
  }

  // 执行函数
  const result = await executeFunction(name, args);

  // 存入缓存
  cache.set(cacheKey, result);

  return result;
}
```

### 3. 错误重试与降级

```typescript
async function executeWithRetry(
  name: string,
  args: any,
  maxRetries = 3
) {
  let lastError: Error;

  for (let i = 0; i &lt; maxRetries; i++) {
    try {
      return await executeFunction(name, args);
    } catch (error) {
      lastError = error;

      // 指数退避
      await sleep(Math.pow(2, i) * 1000);
    }
  }

  // 重试失败，返回降级响应
  return {
    error: true,
    message: `${name} 调用失败: ${lastError.message}`,
    fallback: true
  };
}
```

## 安全最佳实践

### 1. 权限控制

```typescript
// 基于用户角色的工具访问控制
const toolPermissions = {
  get_weather: ["user", "admin"],
  query_database: ["admin"],
  create_event: ["user", "admin"],
  delete_records: ["admin"],
};

function canUseTool(userId: string, toolName: string): boolean {
  const userRole = getUserRole(userId);
  const allowedRoles = toolPermissions[toolName] || [];
  return allowedRoles.includes(userRole);
}

// 在执行前检查
async function executeFunction(name: string, args: any, userId: string) {
  if (!canUseTool(userId, name)) {
    throw new Error(`无权限使用工具: ${name}`);
  }

  return await toolRegistry[name].handler(args);
}
```

### 2. SQL 注入防护

```typescript
function isSafeQuery(sql: string): boolean {
  // 只允许 SELECT 语句
  const normalizedSql = sql.trim().toLowerCase();

  if (!normalizedSql.startsWith("select")) {
    return false;
  }

  // 禁止危险关键字
  const dangerousKeywords = [
    "insert",
    "update",
    "delete",
    "drop",
    "truncate",
    "alter",
    "create",
    "exec",
    "execute",
  ];

  return !dangerousKeywords.some((kw) => normalizedSql.includes(kw));
}
```

function sanitizeResult(result: any): any {
  const sensitiveFields = [
    "password",
    "token",
    "apiKey",
    "secret",
    "ssn",
    "creditCard",
  ];

  if (typeof result !== "object" || result === null) {
    return result;
  }

  const sanitized = { ...result };

  for (const field of sensitiveFields) {
    if (field in sanitized) {
      sanitized[field] = "***REDACTED***";
    }
  }

  return sanitized;
}

## 性能优化

### 1. 流式输出 + 函数调用

```typescript
async function* executeStreaming(userMessage: string) {
  const stream = await openai.chat.completions.create({
    model: "gpt-4-turbo-preview",
    messages: [{ role: "user", content: userMessage }],
    tools,
    stream: true,
  });

  for await (const chunk of stream) {
    // 实时返回文本内容
    if (chunk.choices[0]?.delta?.content) {
      yield {
        type: "text",
        content: chunk.choices[0].delta.content,
      };
    }

    // 检测函数调用
    if (chunk.choices[0]?.delta?.tool_calls) {
      const toolCalls = chunk.choices[0].delta.tool_calls;

      for (const call of toolCalls) {
        yield {
          type: "tool_call",
          name: call.function?.name,
          args: call.function?.arguments,
        };
      }
    }
  }
}
```

### 2. 工具选择优化

```typescript
// 根据 context 动态选择相关工具
function selectRelevantTools(userMessage: string, allTools: Tool[]) {
  // 使用 embedding 相似度匹配
  const messageEmbedding = await getEmbedding(userMessage);

  const scoredTools = await Promise.all(
    allTools.map(async (tool) => ({
      tool,
      score: cosineSimilarity(
        messageEmbedding,
        await getEmbedding(tool.description)
      ),
    }))
  );

  // 只传递最相关的工具（减少 token 消耗）
  return scoredTools
    .filter((t) => t.score > 0.7)
    .sort((a, b) => b.score - a.score)
    .slice(0, 10)
    .map((t) => t.tool);
}
```

## 常见问题与解决方案

### Q1：函数调用失败怎么办？

**方案**：实现错误处理和降级机制，让 LLM 知道调用失败的原因并提供替代方案。

```typescript
// 返回结构化错误信息
{
  error: true,
  code: "API_TIMEOUT",
  message: "天气 API 超时，请稍后重试",
  fallback: "建议查看天气应用"
}
```

### Q2：如何处理复杂的嵌套调用？

**方案**：使用多轮对话模式，让 LLM 自己决定调用顺序。

```typescript
// LLM 会自动处理依赖关系
// "查询用户订单，然后获取每个产品的详细信息"
// 1. 调用 query_orders
// 2. 调用 get_product_details（多次）
```

### Q3：如何减少 token 消耗？

**方案**：

- 只传递相关的工具（相似度匹配）
- 简化工具描述
- 使用工具分组
- 缓存重复调用

### Q4：如何测试 Function Calling？

**方案**：使用 mock 工具进行单元测试

```typescript
// 测试用例
describe("Function Calling Executor", () => {
  it("应该正确调用天气 API", async () => {

    expect(result.message).toContain("25");
  });
});
```
