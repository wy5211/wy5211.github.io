上一篇我们讲了 Set 类型，适合处理需要唯一性和集合运算的场景。但有些场景不仅需要唯一性，还需要排序，比如排行榜、分数查询。这时候就需要 Sorted Set 类型。

Sorted Set 是一个有序的唯一集合，每个元素都有一个分数（score），按分数排序。它既保证了元素的唯一性，又支持按分数范围查询和排名查询。

## 为什么需要 Sorted Set

想象一个直播场景：主播收到礼物后，需要实时更新礼物榜，显示排名前 10 的观众。如果用普通的数据结构，每次更新分数后都要重新排序，性能很差。

Sorted Set 天生就是为这种场景设计的。它会自动按分数排序，更新分数时自动调整位置，查询排名和范围都非常高效。

## 礼物排行榜

让我们用 Sorted Set 实现一个礼物排行榜：

```typescript
import Redis from "ioredis";

const redis = new Redis();

class GiftLeaderboard {
  constructor(
    private redis: Redis,
    private roomId: string
  ) {}

  // 获取排行榜键
  private getKey(): string {
    return `livestream:room:${this.roomId}:gift_leaderboard`;
  }

  // 送礼物（增加积分）
  async sendGift(userId: string, points: number): Promise<number> {
    const key = this.getKey();
    // ZINCRBY 原子性地增加分数，如果用户不存在会创建
    return await this.redis.zincrby(key, points, userId);
  }

  // 获取用户排名（分数从高到低）
  async getRank(userId: string): Promise<number | null> {
    const key = this.getKey();
    // ZREVRANK 返回排名（从0开始），分数相同时按字典序
    const rank = await this.redis.zrevrank(key, userId);
    return rank !== null ? rank + 1 : null; // 转换为从1开始
  }

  // 获取用户分数
  async getScore(userId: string): Promise<number | null> {
    const key = this.getKey();
    const score = await this.redis.zscore(key, userId);
    return score !== null ? parseFloat(score) : null;
  }

  // 获取 Top N
  async getTopN(n: number): Promise<Array<{ userId: string; score: number }>> {
    const key = this.getKey();
    // ZREVRANGE 按分数从高到低获取，WITHSCORES 同时返回分数
    const results = await this.redis.zrevrange(key, 0, n - 1, "WITHSCORES");

    const members: Array<{ userId: string; score: number }> = [];
    for (let i = 0; i < results.length; i += 2) {
      members.push({
        userId: results[i],
        score: parseFloat(results[i + 1]),
      });
    }

    return members;
  }

  // 获取用户及其排名信息
  async getUserInfo(
    userId: string
  ): Promise<{ userId: string; rank: number; score: number } | null> {
    const key = this.getKey();

    const pipeline = this.redis.pipeline();
    pipeline.zrevrank(key, userId);
    pipeline.zscore(key, userId);

    const results = await pipeline.exec();
    if (!results || results[0][1] === null || results[1][1] === null) {
      return null;
    }

    return {
      userId,
      rank: (results[0][1] as number) + 1,
      score: parseFloat(results[1][1] as string),
    };
  }

  // 获取总人数
  async getTotalCount(): Promise<number> {
    const key = this.getKey();
    return await this.redis.zcard(key);
  }
}

// 使用示例
const leaderboard = new GiftLeaderboard(redis, "abc123");

// 送礼物（user_888 送了火箭，50000 积分）
await leaderboard.sendGift("user_888", 50000);

// 送礼物（user_999 送了跑车，30000 积分）
await leaderboard.sendGift("user_999", 30000);

// 获取 Top 10
const top10 = await leaderboard.getTopN(10);
console.log("排行榜:", top10);
// [
//   { userId: "user_888", score: 50000 },
//   { userId: "user_999", score: 30000 },
//   ...
// ]

// 获取 user_888 的排名
const rank = await leaderboard.getRank("user_888");
console.log("排名:", rank); // 1

// 获取用户详细信息
const info = await leaderboard.getUserInfo("user_888");
console.log("用户信息:", info);
// { userId: "user_888", rank: 1, score: 50000 }
```

ZINCRBY 是一个原子操作，如果用户不存在会创建并设置分数，如果存在则增加分数。这保证了并发送礼物时分数的准确性。

## 分页排行榜

当排行榜人数很多时，需要分页展示：

```typescript
class PaginatedLeaderboard {
  constructor(
    private redis: Redis,
    private roomId: string
  ) {}

  private getKey(): string {
    return `livestream:room:${this.roomId}:gift_leaderboard`;
  }

  // 获取分页数据
  async getPage(
    page: number = 1,
    pageSize: number = 10
  ): Promise<{
    data: Array<{ userId: string; score: number; rank: number }>;
    pagination: {
      page: number;
      pageSize: number;
      total: number;
      totalPages: number;
      hasNext: boolean;
      hasPrev: boolean;
    };
  }> {
    const key = this.getKey();
    const start = (page - 1) * pageSize;
    const end = start + pageSize - 1;

    // 获取当前页的数据
    const results = await this.redis.zrevrange(key, start, end, "WITHSCORES");

    // 获取总人数
    const total = await this.redis.zcard(key);
    const totalPages = Math.ceil(total / pageSize);

    // 格式化数据
    const data: Array<{ userId: string; score: number; rank: number }> = [];
    for (let i = 0; i < results.length; i += 2) {
      data.push({
        userId: results[i],
        score: parseFloat(results[i + 1]),
        rank: start + i / 2 + 1,
      });
    }

    return {
      data,
      pagination: {
        page,
        pageSize,
        total,
        totalPages,
        hasNext: page < totalPages,
        hasPrev: page > 1,
      },
    };
  }
}

// 使用示例
const paginated = new PaginatedLeaderboard(redis, "abc123");

const page1 = await paginated.getPage(1, 10);
console.log("第1页:", page1);

const page2 = await paginated.getPage(2, 10);
console.log("第2页:", page2);
```

分页的关键是计算 start 和 end 索引。ZRANGE/ZREVRANGE 的索引从 0 开始，所以第 1 页是 0-9，第 2 页是 10-19，以此类推。

## 分数范围查询

有时候需要查询某个分数范围内的用户，比如"积分在 10000 到 50000 之间的用户"：

```typescript
class RangeQuery {
  constructor(
    private redis: Redis,
    private roomId: string
  ) {}

  private getKey(): string {
    return `livestream:room:${this.roomId}:gift_leaderboard`;
  }

  // 按分数范围查询
  async getByScoreRange(
    minScore: number,
    maxScore: number
  ): Promise<Array<{ userId: string; score: number }>> {
    const key = this.getKey();
    // ZRANGEBYSCORE 按分数范围查询
    const results = await this.redis.zrangebyscore(
      key,
      minScore,
      maxScore,
      "WITHSCORES"
    );

    const members: Array<{ userId: string; score: number }> = [];
    for (let i = 0; i < results.length; i += 2) {
      members.push({
        userId: results[i],
        score: parseFloat(results[i + 1]),
      });
    }

    return members;
  }

  // 统计分数范围内的用户数
  async countByScoreRange(minScore: number, maxScore: number): Promise<number> {
    const key = this.getKey();
    return await this.redis.zcount(key, minScore, maxScore);
  }

  // 获取某个排名范围的用户
  async getByRankRange(
    startRank: number,
    endRank: number
  ): Promise<Array<{ userId: string; score: number; rank: number }>> {
    const key = this.getKey();
    // 排名从1开始，需要转换成从0开始的索引
    const start = startRank - 1;
    const end = endRank - 1;

    const results = await this.redis.zrevrange(key, start, end, "WITHSCORES");

    const members: Array<{ userId: string; score: number; rank: number }> = [];
    for (let i = 0; i < results.length; i += 2) {
      members.push({
        userId: results[i],
        score: parseFloat(results[i + 1]),
        rank: start + i / 2 + 1,
      });
    }

    return members;
  }
}

// 使用示例
const rangeQuery = new RangeQuery(redis, "abc123");

// 查询积分在 10000-50000 之间的用户
const users = await rangeQuery.getByScoreRange(10000, 50000);
console.log("符合条件的用户:", users);

// 统计人数
const count = await rangeQuery.countByScoreRange(10000, 50000);
console.log("人数:", count);

// 获取第 11-20 名
const rank11to20 = await rangeQuery.getByRankRange(11, 20);
console.log("第11-20名:", rank11to20);
```

ZRANGEBYSCORE 支持多种分数格式：

- 具体值：10000
- 无穷大：+inf, -inf
- 开区间：(10000 表示不包含 10000

## 时间段排行榜

直播平台通常需要多个榜单：日榜、周榜、总榜。可以用不同的键来实现：

```typescript
class TimeBasedLeaderboard {
  constructor(
    private redis: Redis,
    private roomId: string
  ) {}

  // 获取不同时间段的键
  private getKey(period: "daily" | "weekly" | "alltime"): string {
    if (period === "daily") {
      const today = new Date().toISOString().split("T")[0];
      return `livestream:room:${this.roomId}:gifts:daily:${today}`;
    } else if (period === "weekly") {
      const weekStart = this.getWeekStart();
      return `livestream:room:${this.roomId}:gifts:weekly:${weekStart}`;
    } else {
      return `livestream:room:${this.roomId}:gifts:alltime`;
    }
  }

  // 获取本周一的日期
  private getWeekStart(): string {
    const now = new Date();
    const day = now.getDay();
    const diff = now.getDate() - day + (day === 0 ? -6 : 1);
    const weekStart = new Date(now.setDate(diff));
    return weekStart.toISOString().split("T")[0];
  }

  // 送礼物（同时更新多个榜单）
  async sendGift(userId: string, points: number): Promise<void> {
    const periods: Array<"daily" | "weekly" | "alltime"> = [
      "daily",
      "weekly",
      "alltime",
    ];

    for (const period of periods) {
      const key = this.getKey(period);
      await this.redis.zincrby(key, points, userId);

      // 设置过期时间
      if (period === "daily") {
        await this.redis.expire(key, 86400 * 2); // 2天后过期
      } else if (period === "weekly") {
        await this.redis.expire(key, 86400 * 14); // 14天后过期
      }
    }
  }

  // 获取指定时间段的 Top N
  async getTopN(period: "daily" | "weekly" | "alltime", n: number = 10) {
    const key = this.getKey(period);
    const results = await this.redis.zrevrange(key, 0, n - 1, "WITHSCORES");

    const members: Array<{ userId: string; score: number }> = [];
    for (let i = 0; i < results.length; i += 2) {
      members.push({
        userId: results[i],
        score: parseFloat(results[i + 1]),
      });
    }

    return members;
  }

  // 获取用户在各个榜单的排名
  async getUserRanks(userId: string): Promise<{
    daily: number | null;
    weekly: number | null;
    alltime: number | null;
  }> {
    const periods: Array<"daily" | "weekly" | "alltime"> = [
      "daily",
      "weekly",
      "alltime",
    ];

    const pipeline = this.redis.pipeline();
    for (const period of periods) {
      const key = this.getKey(period);
      pipeline.zrevrank(key, userId);
    }

    const results = await pipeline.exec();

    return {
      daily: results[0][1] !== null ? (results[0][1] as number) + 1 : null,
      weekly: results[1][1] !== null ? (results[1][1] as number) + 1 : null,
      alltime: results[2][1] !== null ? (results[2][1] as number) + 1 : null,
    };
  }
}

// 使用示例
const timeLeaderboard = new TimeBasedLeaderboard(redis, "abc123");

// 送礼物（同时更新日榜、周榜、总榜）
await timeLeaderboard.sendGift("user_888", 50000);

// 获取日榜 Top 10
const dailyTop10 = await timeLeaderboard.getTopN("daily", 10);
console.log("日榜:", dailyTop10);

// 获取周榜 Top 10
const weeklyTop10 = await timeLeaderboard.getTopN("weekly", 10);
console.log("周榜:", weeklyTop10);

// 获取用户在各榜单的排名
const ranks = await timeLeaderboard.getUserRanks("user_888");
console.log("排名:", ranks);
// { daily: 1, weekly: 1, alltime: 5 }
```

时间段排行榜的关键是键的设计。键名中包含日期或周数，这样可以自然地分割时间段。同时设置过期时间，自动清理旧数据。

## 在线用户统计

Sorted Set 的分数不一定是整数，也可以是时间戳。这可以用来统计在线用户：

```typescript
class OnlineUserTracker {
  constructor(
    private redis: Redis,
    private roomId: string
  ) {}

  private getKey(): string {
    return `livestream:room:${this.roomId}:online_users`;
  }

  // 记录用户活跃（分数为最后活跃时间戳）
  async recordActivity(userId: string): Promise<void> {
    const key = this.getKey();
    const now = Date.now();
    await this.redis.zadd(key, now, userId);
  }

  // 获取在线用户数（最近 N 秒活跃）
  async getOnlineCount(seconds: number = 300): Promise<number> {
    const key = this.getKey();
    const threshold = Date.now() - seconds * 1000;
    // 统计分数（时间戳）大于阈值的用户数
    return await this.redis.zcount(key, threshold, Date.now());
  }

  // 获取在线用户列表
  async getOnlineUsers(seconds: number = 300): Promise<string[]> {
    const key = this.getKey();
    const threshold = Date.now() - seconds * 1000;
    return await this.redis.zrangebyscore(key, threshold, Date.now());
  }

  // 清理过期数据（删除很久没活动的用户）
  async cleanup(seconds: number = 3600): Promise<number> {
    const key = this.getKey();
    const threshold = Date.now() - seconds * 1000;
    return await this.redis.zremrangebyscore(key, 0, threshold);
  }

  // 移除用户
  async removeUser(userId: string): Promise<number> {
    const key = this.getKey();
    return await this.redis.zrem(key, userId);
  }
}

// 使用示例
const tracker = new OnlineUserTracker(redis, "abc123");

// 用户进入直播间或互动时记录
await tracker.recordActivity("usr_a1b2c3");

// 获取最近 5 分钟活跃的用户数
const count = await tracker.getOnlineCount(300);
console.log("在线人数:", count);

// 获取在线用户列表
const users = await tracker.getOnlineUsers(300);
console.log("在线用户:", users);

// 定期清理过期数据（比如每小时执行一次）
await tracker.cleanup(3600);
```

这个实现的巧妙之处在于用时间戳作为分数。ZCOUNT 可以统计"分数在某个范围内"的元素数，正好用来统计"最近 N 秒活跃"的用户数。

## 分数相同时的处理

Sorted Set 在分数相同时，按成员的字典序排序。如果需要控制顺序，可以在分数中加入小数：

```typescript
// 想让 user_888 排在 user_999 前面（即使分数相同）
await redis.zadd("leaderboard", 50000.001, "user_888");
await redis.zadd("leaderboard", 50000.002, "user_999");

// 或者用时间戳
await redis.zadd("leaderboard", 50000 + Date.now() * 0.000001, "user_888");
```

但这只是个 workaround。如果需要精确控制顺序，最好在分数上做文章，比如"50000 + 时间戳 / 1000000"。

## Sorted Set 的性能

Sorted Set 的核心操作时间复杂度：

- ZADD: O(log N)
- ZRANK: O(log N)
- ZRANGE: O(log N + M)，M 是返回的元素数量
- ZRANGEBYSCORE: O(log N + M)

N 是集合大小，M 是返回的元素数量。即使集合有几百万元素，查询 Top 10 也很快，因为 M 很小。

## 总结与 Trade-off

Sorted Set 是 Redis 中最强大的数据结构之一，非常适合实现排行榜和范围查询。

**Sorted Set 的优点：**

- 自动排序，更新分数时自动调整位置
- 支持按排名查询和按分数范围查询
- 元素唯一性，重复添加会更新分数
- 性能优秀，核心操作都是 O(log N)

**Sorted Set 的缺点：**

- 内存占用相对较高（要存储分数）
- 分数相同时按字典序排序，可能不符合预期
- 不支持复杂的查询条件（比如多字段排序）

**适用场景：**

- 排行榜（礼物榜、积分榜、销量榜）
- 实时统计（在线用户、活跃度）
- 范围查询（分数区间、排名区间）
- 延时队列（用时间戳作为分数）

**不适合的场景：**

- 需要按多个字段排序（用数据库）
- 需要复杂的条件查询（用搜索引擎）
- 分数更新非常频繁（考虑其他方案）
