# 大模型本地部署实战：从 Ollama 到 vLLM 的完整指南

随着大语言模型（LLM）的爆发式发展，越来越多的开发者希望在自己的机器上部署和运行模型。本地部署不仅能保护数据隐私，还能降低 API 调用成本，实现更灵活的定制化开发。本文将带你从零开始，掌握大模型本地部署的核心技术。

## 为什么选择本地部署大模型？

### 本地部署的优势

**数据隐私与安全**

- 敏感数据无需上传到第三方服务器
- 符合企业数据合规要求
- 适用于医疗、金融等隐私敏感领域

**成本控制**

- 无 API 调用费用，长期使用成本更低
- 可无限次调用，不受速率限制
- 适合高频调用场景

**定制化能力**

- 可以使用自定义数据微调模型
- 完全控制模型参数和行为
- 支持私有领域知识注入

**离线可用性**

- 无需网络连接即可使用
- 响应速度更快（本地推理）
- 适合边缘设备和离线环境

### 适用场景

| 场景          | 推荐方案           | 模型规模 |
| ------------- | ------------------ | -------- |
| 个人学习/实验 | Ollama + 7B 模型   | 7B-13B   |
| 原型开发      | LM Studio / Ollama | 7B-13B   |
| 中小团队生产  | vLLM + 量化模型    | 13B-70B  |
| 企业级应用    | vLLM + 分布式部署  | 70B+     |

## 硬件选型指南

### GPU 选择建议

本地部署大模型最关键的硬件是 GPU 显存。以下是不同规模模型的显存需求参考：

**7B 参数模型**

- FP16（半精度）：约 14GB 显存
- INT8（8位量化）：约 7GB 显存
- INT4（4位量化）：约 4GB 显存

**13B 参数模型**

- FP16：约 26GB 显存
- INT8：约 13GB 显存
- INT4：约 7GB 显存

**70B 参数模型**

- FP16：约 140GB 显存
- INT8：约 70GB 显存（需要多卡）
- INT4：约 35GB 显存（多卡或高端显卡）

### 推荐配置方案

**入门级（个人学习）**

```text
GPU: RTX 3060 12GB / RTX 4060 Ti 16GB
内存: 32GB DDR4/DDR5
存储: 500GB NVMe SSD
适用: 7B-13B 量化模型
```

**中端配置（小团队）**

```text
GPU: RTX 4090 24GB / RTX 3090 24GB
内存: 64GB DDR4/DDR5
存储: 1TB NVMe SSD
适用: 13B-34B 模型，多模型并行
```

**高端配置（企业生产）**

```text
GPU: A100 80GB × 2 或 H100 80GB × 2
内存: 256GB DDR5
存储: 2TB+ NVMe SSD RAID
适用: 70B+ 模型，高并发推理
```

### CPU 和内存要求

如果 GPU 显存不足，可以使用 CPU + 系统内存运行模型（速度较慢）：

- **CPU**: 支持 AVX2/AVX-512 指令集的现代 CPU
- **内存**: 至少为模型大小的 2-3 倍
- **推荐**: 32GB 内存起步，64GB 更稳妥

## Ollama：最简单的本地部署方案

### Ollama 简介

Ollama 是目前最易用的本地大模型运行工具，支持一键下载和运行模型，适合快速入门和原型开发。

### 安装 Ollama

**macOS / Linux**

```bash
# 官方安装脚本
curl -fsSL https://ollama.com/install.sh | sh

# 或使用 Homebrew (macOS)
brew install ollama
```

**Windows**

下载官方安装包：https://ollama.com/download

**Docker 方式**

```bash
docker run -d -v ollama:/root/.ollama -p 11434:11434 --name ollama ollama/ollama
```

### 基本使用

**运行模型**

```bash
# 下载并运行 Llama 3.2 3B
ollama run llama3.2

# 运行其他模型
ollama run mistral
ollama run codellama
ollama run qwen2.5
```

**常用命令**

```bash
# 列出已安装的模型
ollama list

# 查看模型信息
ollama show llama3.2

# 删除模型
ollama rm llama3.2

# 拉取模型（不运行）
ollama pull llama3.2

# 查看运行中的模型
ollama ps
```

### 模型量化与选择

Ollama 自动使用量化模型以节省显存。常用模型标签：

```bash
# 默认量化版本（通常是 4-bit）
ollama run llama3.2

# 指定量化级别
ollama run llama3.2:7b-q4_0    # 4-bit 量化
ollama run llama3.2:7b-q8_0    # 8-bit 量化
ollama run llama3.2:7b-fp16    # 半精度（无量化）
```

**量化级别对比**

| 量化方式 | 显存占用 | 推理速度 | 质量损失 |
| -------- | -------- | -------- | -------- |
| FP16     | 100%     | 基准     | 无       |
| INT8     | ~50%     | 更快     | 极小     |
| INT4     | ~25%     | 最快     | 轻微     |

## vLLM：生产级高性能推理引擎

### vLLM 简介

vLLM 是专为高吞吐量和低延迟设计的 LLM 推理引擎，采用创新的 PagedAttention 技术，适合生产环境部署。

### vLLM 核心优势

**PagedAttention 技术**

- 类似操作系统的虚拟内存管理
- 动态分配显存，减少碎片
- 支持更大的 batch size
- 提升显存利用率 2-4 倍

**性能对比**

| 指标         | HuggingFace Transformers | vLLM        |
| ------------ | ------------------------ | ----------- |
| 吞吐量       | 基准                     | 10-20x 提升 |
| 显存利用率   | 20-40%                   | 80-95%      |
| 并发请求支持 | 有限                     | 高并发      |

### 安装 vLLM

```bash
# 使用 pip 安装
pip install vllm

# 建议在虚拟环境中安装
python -m venv vllm-env
source vllm-env/bin/activate
pip install vllm
```

### 基本使用

**离线推理**

```python
from vllm import LLM, SamplingParams

# 初始化模型
llm = LLM(
    model="meta-llama/Llama-3.2-3B-Instruct",
    tensor_parallel_size=1,  # GPU 数量
    gpu_memory_utilization=0.9,
)

# 设置采样参数
sampling_params = SamplingParams(
    temperature=0.7,
    top_p=0.9,
    max_tokens=256,
)

# 生成文本
prompts = [
    "Explain machine learning in simple terms",
    "Write a Python function to sort a list",
]

outputs = llm.generate(prompts, sampling_params)

for output in outputs:
    print(f"Prompt: {output.prompt}")
    print(f"Generated: {output.outputs[0].text}\n")
```

## 实战：构建本地 AI 应用

### 场景 1：代码助手

使用本地模型构建代码补全和生成工具：

````python
from vllm import LLM, SamplingParams

class CodeAssistant:
    def __init__(self, model_path: str):
        self.llm = LLM(model=model_path)
        self.params = SamplingParams(
            temperature=0.3,  # 代码生成用较低温度
            top_p=0.95,
            max_tokens=512,
        )

    def complete(self, code: str, cursor_position: int) -> str:
        """代码补全"""
        prompt = f"""Complete the following code:

```python
{code[:cursor_position]}
````

Continue from where it left off:"""

        output = self.llm.generate([prompt], self.params)
        return output[0].outputs[0].text

    def explain(self, code: str) -> str:
        """代码解释"""
        prompt = f"""Explain what this code does:

```python
{code}
```

Explanation:"""

        output = self.llm.generate([prompt], self.params)
        return output[0].outputs[0].text

    def review(self, code: str) -> str:
        """代码审查"""
        prompt = f"""Review this code and suggest improvements:

```python
{code}
```

Review:"""

        output = self.llm.generate([prompt], self.params)
        return output[0].outputs[0].text

# 使用示例

assistant = CodeAssistant("codellama/CodeLlama-7b-Instruct-hf")
code = "def fibonacci(n):"
print(assistant.complete(code, len(code)))

````

### 场景 2：文档问答系统

结合向量数据库构建私有知识库问答：

```python
from vllm import LLM, SamplingParams
import chromadb
from chromadb.utils import embedding_functions

class DocumentQA:
    def __init__(self, model_path: str):
        self.llm = LLM(model=model_path)
        self.params = SamplingParams(
            temperature=0.1,
            max_tokens=512,
        )

        # 初始化向量数据库
        self.chroma = chromadb.Client()
        self.embedder = embedding_functions.SentenceTransformerEmbeddingFunction(
            model_name="all-MiniLM-L6-v2"
        )
        self.collection = self.chroma.create_collection(
            name="documents",
            embedding_function=self.embedder,
        )

    def add_documents(self, documents: list[str]):
        """添加文档到知识库"""
        self.collection.add(
            documents=documents,
            ids=[f"doc_{i}" for i in range(len(documents))],
        )

    def query(self, question: str, k: int = 3) -> str:
        """问答"""
        # 检索相关文档
        results = self.collection.query(
            query_texts=[question],
            n_results=k,
        )

        context = "\n\n".join(results['documents'][0])

        prompt = f"""Based on the following context, answer the question.

Context:
{context}

Question: {question}

Answer:"""

        output = self.llm.generate([prompt], self.params)
        return output[0].outputs[0].text

# 使用示例
qa = DocumentQA("meta-llama/Llama-3.2-3B-Instruct")
qa.add_documents([
    "The company was founded in 2020.",
    "Our main product is an AI-powered analytics platform.",
    "We have offices in San Francisco and New York.",
])

answer = qa.query("When was the company founded?")
print(answer)
````

### 场景 3：实时聊天机器人

构建支持上下文的聊天机器人：

```python
from fastapi import FastAPI
from fastapi.responses import StreamingResponse
from vllm import LLM, SamplingParams
import json

app = FastAPI()

# 全局模型实例
llm = LLM(model="meta-llama/Llama-3.2-3B-Instruct")

# 会话存储（生产环境应使用 Redis）
sessions = {}

@app.post("/chat")
async def chat(session_id: str, message: str):
    """对话接口"""
    # 获取或创建会话
    if session_id not in sessions:
        sessions[session_id] = []

    # 添加用户消息
    sessions[session_id].append({
        "role": "user",
        "content": message,
    })

    # 构建提示词
    prompt = build_prompt(sessions[session_id])

    # 生成回复
    params = SamplingParams(
        temperature=0.7,
        max_tokens=512,
    )
    output = llm.generate([prompt], params)
    reply = output[0].outputs[0].text

    # 添加助手回复
    sessions[session_id].append({
        "role": "assistant",
        "content": reply,
    })

    return {"reply": reply}

def build_prompt(messages: list[dict]) -> str:
    """构建对话提示词"""
    prompt = "You are a helpful assistant.\n\n"
    for msg in messages:
        role = msg["role"].capitalize()
        prompt += f"{role}: {msg['content']}\n"
    prompt += "Assistant:"
    return prompt

if __name__ == "__main__":
    import uvicorn
    uvicorn.run(app, host="0.0.0.0", port=8000)
```

## 最佳实践与注意事项

### 模型选择建议

**考虑因素**

1. **任务类型**
   - 通用对话：Llama 3.x, Mistral, Qwen
   - 代码生成：CodeLlama, DeepSeek Coder

   - 追求质量：FP16 或 INT8

   - 平衡选择：AWQ/GPTQ 量化

- 使用 RLHF/安全微调的模型

- 定期更新模型版本

**合规要求**

- 遵守模型许可证（Llama License, Apache 2.0 等）

## 故障排查

### 常见问题

**问题 1：显存不足（OOM）**

```bash
# 解决方案
1. 使用量化模型（INT4/INT8）
2. 减小 batch size
3. 降低 gpu_memory_utilization
4. 使用 CPU offload（慢但可行）
```

**问题 2：推理速度慢**

```bash
# 解决方案
1. 检查是否使用 GPU（nvidia-smi）
2. 启用 Flash Attention
3. 使用 vLLM 替代原生 Transformers
4. 升级到更快的 GPU
```

**问题 3：模型加载失败**

```bash
# 解决方案
1. 检查模型文件完整性
2. 确保有足够磁盘空间
3. 验证 HuggingFace token 权限
4. 尝试使用本地模型文件
```

### 应用方向

- **多模态模型**: 文本 + 图像 + 音频
- **长上下文**: 100K+ tokens 上下文窗口
- **Agent 系统**: 自主决策和工具使用
- **实时交互**: 语音对话、视频理解

## 总结

本地部署大模型已经从"技术探索"走向"生产可用"。从简单的 Ollama 到高性能的 vLLM，开发者可以根据实际需求选择合适的方案：

**快速入门选择 Ollama**

- 5 分钟即可开始使用
- 适合学习和原型开发
- 自动处理量化和优化

**生产部署选择 vLLM**

- 高吞吐量和低延迟
- 兼容 OpenAI API
- 丰富的优化选项

**关键要点回顾**

1. **硬件是基础**: 根据模型规模选择合适的 GPU
2. **量化是关键**: INT4 量化在质量和速度间取得最佳平衡
3. **工具要选对**: Ollama 简单，vLLM 强大
4. **监控不能少**: 生产环境必须监控性能和资源
5. **安全要重视**: 数据隐私和模型安全不可忽视

本地大模型部署打开了无限可能，从个人项目到企业应用，从代码助手到智能客服。掌握这些技术，你就能在自己的机器上运行最先进的 AI 模型，保护数据隐私的同时享受 AI 的强大能力。

现在就开始你的本地大模型之旅吧！从 `ollama run llama3.2` 这一行命令开始，探索 AI 的无限可能。
