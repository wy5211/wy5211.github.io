## 前言

在 AI Agent 开发中，一个核心挑战是：**如何让 AI 安全、标准化地与外部世界交互？**

传统方案各有局限：

- **Function Calling** - 需要为每个模型单独定义，不通用
- **插件系统** - 各家实现不同，无法复用
- **直接 API 集成** - 安全风险高，维护成本大

Anthropic 在 2024 年底推出了 **Model Context Protocol（MCP）**，这是一个开放标准，旨在解决 AI 与外部资源连接的标准化问题。

本文将深入解析 MCP 协议的原理、架构，并通过实战案例演示如何开发 MCP Server 和 Client。

## MCP 是什么？

### 核心概念

MCP（Model Context Protocol）是一个**开放协议**，定义了 AI 应用与外部数据源、工具之间的标准通信方式。

```
┌─────────────────┐     MCP Protocol      ┌─────────────────┐
│   MCP Client    │ &lt;───────────────────&gt; │   MCP Server    │
│  (Claude/AI)    │     JSON-RPC 2.0      │  (Resources)    │
└─────────────────┘                       └─────────────────┘
```

### 设计目标

1. **标准化** - 统一的接口定义，一次开发处处可用
2. **安全性** - 客户端控制权限，服务端验证请求
3. **可组合** - 多个 Server 可以组合使用
4. **语言无关** - 支持任何编程语言实现

### 核心能力

MCP 提供三类核心能力：

| 能力          | 说明         | 示例                           |
| ------------- | ------------ | ------------------------------ |
| **Resources** | 读取数据     | 文件内容、数据库记录、API 响应 |
| **Prompts**   | 预定义提示词 | 代码审查模板、文档生成提示     |
| **Tools**     | 执行操作     | 创建文件、发送请求、运行命令   |

## 架构解析

### 通信模型

MCP 基于 **JSON-RPC 2.0** 协议，支持两种传输方式：

#### 1. Stdio 传输（本地）

最简单的模式，适合本地工具集成：

```typescript
// MCP Client 通过 stdin/stdout 与 Server 通信
const client = new Client({ name: "my-client", version: "1.0.0" });
const transport = new StdioClientTransport({
  command: "node",
  args: ["mcp-server.js"],
});
await client.connect(transport);
```

#### 2. HTTP + SSE 传输（远程）

适合分布式部署：

```typescript
const transport = new SSEClientTransport(new URL("http://localhost:3000/sse"));
await client.connect(transport);
```

### 消息类型

```typescript
// 请求
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "tools/list",
  "params": {}
}

// 响应
{
  "jsonrpc": "2.0",
  "id": 1,
  "result": {
    "tools": [
      { "name": "read_file", "description": "读取文件内容" }
    ]
  }
}

// 通知（无需响应）
{
  "jsonrpc": "2.0",
  "method": "notifications/resources/updated",
  "params": { "uri": "file:///example.txt" }
}
```

## 实战：开发 MCP Server

让我们开发一个实用的 MCP Server，提供**文件操作**和**网页抓取**能力。

### 项目初始化

```bash
mkdir mcp-tools-server
cd mcp-tools-server
npm init -y
npm install @modelcontextprotocol/sdk zod
npm install -D typescript @types/node
```

### Server 实现

```typescript
// src/index.ts
import { Server } from "@modelcontextprotocol/sdk/server/index.js";
import { StdioServerTransport } from "@modelcontextprotocol/sdk/server/stdio.js";
import {
  CallToolRequestSchema,
  ListToolsRequestSchema,
  ListResourcesRequestSchema,
  ReadResourceRequestSchema,
} from "@modelcontextprotocol/sdk/types.js";
import fs from "fs/promises";
import path from "path";
import { z } from "zod";

// 创建 Server 实例
const server = new Server(
  { name: "tools-server", version: "1.0.0" },
  { capabilities: { tools: {}, resources: {} } }
);

// ==================== Tools 定义 ====================

const ReadFileSchema = z.object({
  path: z.string().describe("文件路径"),
});

const WriteFileSchema = z.object({
  path: z.string().describe("文件路径"),
  content: z.string().describe("文件内容"),
});

const FetchUrlSchema = z.object({
  url: z.string().url().describe("要抓取的 URL"),
});

// 注册 Tools 列表
server.setRequestHandler(ListToolsRequestSchema, async () => {
  return {
    tools: [
      {
        name: "read_file",
        description: "读取本地文件内容",
        inputSchema: {
          type: "object",
          properties: {
            path: { type: "string", description: "文件路径" },
          },
          required: ["path"],
        },
      },
      {
        name: "write_file",
        description: "写入内容到本地文件",
        inputSchema: {
          type: "object",
          properties: {
            path: { type: "string", description: "文件路径" },
            content: { type: "string", description: "文件内容" },
          },
          required: ["path", "content"],
        },
      },
      {
        name: "fetch_url",
        description: "抓取网页内容（返回纯文本）",
        inputSchema: {
          type: "object",
          properties: {
            url: { type: "string", description: "要抓取的 URL" },
          },
          required: ["url"],
        },
      },
      {
        name: "list_directory",
        description: "列出目录内容",
        inputSchema: {
          type: "object",
          properties: {
            path: { type: "string", description: "目录路径" },
          },
          required: ["path"],
        },
      },
    ],
  };
});

// 处理 Tool 调用
server.setRequestHandler(CallToolRequestSchema, async (request) => {
  const { name, arguments: args } = request.params;

  switch (name) {
    case "read_file": {
      const { path: filePath } = ReadFileSchema.parse(args);
      try {
        const content = await fs.readFile(filePath, "utf-8");
        return {
          content: [{ type: "text", text: content }],
        };
      } catch (error) {
        return {
          content: [{ type: "text", text: `读取失败: ${error.message}` }],
          isError: true,
        };
      }
    }

    case "write_file": {
      const { path: filePath, content } = WriteFileSchema.parse(args);
      try {
        await fs.mkdir(path.dirname(filePath), { recursive: true });
        await fs.writeFile(filePath, content, "utf-8");
        return {
          content: [{ type: "text", text: `文件已写入: ${filePath}` }],
        };
      } catch (error) {
        return {
          content: [{ type: "text", text: `写入失败: ${error.message}` }],
          isError: true,
        };
      }
    }

    case "fetch_url": {
      const { url } = FetchUrlSchema.parse(args);
      try {
        const response = await fetch(url);
        const html = await response.text();
        // 简单提取文本（实际可用 cheerio 等库）
        const text = html
          .replace(/&lt;script[^&gt;]*&gt;[\s\S]*?&lt;\/script&gt;/gi, "")
          .replace(/&lt;style[^&gt;]*&gt;[\s\S]*?&lt;\/style&gt;/gi, "")
          .replace(/&lt;[^&gt;]+&gt;/g, "")
          .replace(/\s+/g, " ")
          .trim();
        return {
          content: [{ type: "text", text: text.slice(0, 5000) }], // 限制长度
        };
      } catch (error) {
        return {
          content: [{ type: "text", text: `抓取失败: ${error.message}` }],
          isError: true,
        };
      }
    }

    case "list_directory": {
      const { path: dirPath } = z.object({ path: z.string() }).parse(args);
      try {
        const entries = await fs.readdir(dirPath, { withFileTypes: true });
        const list = entries
          .map((e) => `${e.isDirectory() ? "📁" : "📄"} ${e.name}`)
          .join("\n");
        return {
          content: [{ type: "text", text: list }],
        };
      } catch (error) {
        return {
          content: [{ type: "text", text: `列出失败: ${error.message}` }],
          isError: true,
        };
      }
    }

    default:
      throw new Error(`未知工具: ${name}`);
  }
});

// ==================== Resources 定义 ====================

// 注册 Resources 列表
server.setRequestHandler(ListResourcesRequestSchema, async () => {
  return {
    resources: [
      {
        uri: "file:///README.md",
        name: "项目说明文档",
        mimeType: "text/markdown",
        description: "项目的 README 文件",
      },
    ],
  };
});

// 处理 Resource 读取
server.setRequestHandler(ReadResourceRequestSchema, async (request) => {
  const { uri } = request.params;

  if (uri.startsWith("file:///")) {
    const filePath = uri.replace("file://", "");
    try {
      const content = await fs.readFile(filePath, "utf-8");
      return {
        contents: [
          {
            uri,
            mimeType: "text/plain",
            text: content,
          },
        ],
      };
    } catch (error) {
      throw new Error(`读取资源失败: ${error.message}`);
    }
  }

  throw new Error(`不支持的 URI: ${uri}`);
});

// 启动服务
async function main() {
  const transport = new StdioServerTransport();
  await server.connect(transport);
  console.error("MCP Server 已启动");
}

main().catch(console.error);
```

## 实战：开发 MCP Client

### 连接到 Server

```typescript
// client.ts
import { Client } from "@modelcontextprotocol/sdk/client/index.js";
import { StdioClientTransport } from "@modelcontextprotocol/sdk/client/stdio.js";

async function main() {
  // 创建 Client
  const client = new Client(
    { name: "my-client", version: "1.0.0" },
    { capabilities: {} }
  );

  // 连接到 Server
  const transport = new StdioClientTransport({
    command: "node",
    args: ["dist/index.js"],
  });
  await client.connect(transport);

  // 列出可用工具
  const { tools } = await client.request(
    { method: "tools/list" },
    ListToolsResultSchema
  );
  console.log(
    "可用工具:",
    tools.map((t) => t.name)
  );

  // 调用工具
  const result = await client.request(
    {
      method: "tools/call",
      params: {
        name: "read_file",
        arguments: { path: "/Users/test/example.txt" },
      },
    },
    CallToolResultSchema
  );
  console.log("文件内容:", result.content[0].text);

  await client.close();
}

main();
```

## 安全最佳实践

### 1. 权限控制

```typescript
// 在 Server 端验证权限
server.setRequestHandler(CallToolRequestSchema, async (request) => {
  const { name, arguments: args } = request.params;

  // 限制可访问的目录
  const allowedDirs = ["/Users/test/safe-dir"];
  const targetPath = args.path;

  if (!allowedDirs.some((dir) => targetPath.startsWith(dir))) {
    return {
      content: [{ type: "text", text: "权限拒绝：路径不在允许范围内" }],
      isError: true,
    };
  }

  // ... 执行操作
});
```

## MCP vs Function Calling

| 特性       | MCP         | Function Calling |
| ---------- | ----------- | ---------------- |
| **标准化** | ✅ 开放协议 | ❌ 各家不同      |
| **跨模型** | ✅ 通用     | ❌ 需适配        |
| **安全性** | ✅ 内置权限 | ⚠️ 需自行实现    |
| **调试**   | ✅ 标准工具 | ⚠️ 需自己搭建    |
| **生态**   | 🔄 发展中   | ✅ 成熟          |

**选择建议：**

- **使用 MCP**：构建通用 AI Agent、需要多模型支持、重视标准化
- **使用 Function Calling**：单一模型场景、快速原型、简单集成
