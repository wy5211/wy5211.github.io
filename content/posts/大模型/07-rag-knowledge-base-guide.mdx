## 前言

大语言模型（LLM）虽然强大，但存在两个致命问题：

1. **知识过时**：训练数据截止日期之后的信息完全不知道
2. **幻觉问题**：不知道的问题会一本正经地胡说八道

RAG（Retrieval-Augmented Generation，检索增强生成）技术完美解决了这两个问题。它的核心思路很简单：**先检索相关信息，再让大模型基于检索结果生成答案**。

本文将带你从零开始构建一个生产级的企业知识库问答系统。

## RAG 技术架构

### 基本原理

```text
用户问题 → 向量化 → 检索相似文档 → 构建提示词 → 大模型生成答案
                ↓
           向量数据库（存储文档向量）
```

### 核心组件

1. **文档处理**：文本分割、清洗、元数据提取
2. **向量模型**：将文本转换为向量表示
3. **向量数据库**：存储和检索向量
4. **检索策略**：相似度计算、重排序
5. **大模型**：基于上下文生成答案

## 技术选型

### 向量数据库对比

| 数据库       | 优势              | 劣势     | 适用场景             |
| ------------ | ----------------- | -------- | -------------------- |
| **Pinecone** | 全托管、性能强    | 收费贵   | 生产环境、无运维需求 |
| **Milvus**   | 开源、可扩展      | 部署复杂 | 大规模生产           |
| **Qdrant**   | Rust 实现、性能好 | 社区较小 | 中小规模应用         |
| **Chroma**   | 轻量、易上手      | 功能有限 | 原型开发、小规模     |
| **Weaviate** | GraphQL、混合检索 | 学习曲线 | 需要复杂查询         |

**推荐**：中小规模用 Qdrant，大规模用 Milvus，快速原型用 Chroma。

### 1. 文档处理与向量化

```typescript
// src/documentProcessor.ts
import { RecursiveCharacterTextSplitter } from "langchain/text_splitter";
import { OpenAIEmbeddings } from "@langchain/openai";
import { PDFLoader } from "langchain/document_loaders/fs/pdf";
import { TextLoader } from "langchain/document_loaders/fs/text";
import * as fs from "fs";
import * as path from "path";

export class DocumentProcessor {
  private embeddings: OpenAIEmbeddings;
  private splitter: RecursiveCharacterTextSplitter;

  constructor(openaiApiKey: string) {
    this.embeddings = new OpenAIEmbeddings({
      openAIApiKey: openaiApiKey,
      modelName: "text-embedding-3-small",
    });

    // 智能分割：按段落、句子、字符递归
    this.splitter = new RecursiveCharacterTextSplitter({
      chunkSize: 1000, // 每块最大字符数
      chunkOverlap: 200, // 块之间的重叠
      separators: ["\n\n", "\n", "。", "！", "？", ".", "!", "?", " "],
    });
  }

  /**
   * 加载文档（支持 PDF、TXT、MD）
   */
  async loadDocument(filePath: string) {
    const ext = path.extname(filePath).toLowerCase();

    let loader;
    switch (ext) {
      case ".pdf":
        loader = new PDFLoader(filePath);
        break;
      case ".txt":
      case ".md":
        loader = new TextLoader(filePath);
        break;
      default:
        throw new Error(`不支持的文件格式: ${ext}`);
    }

    const docs = await loader.load();

    // 添加元数据
    const fileName = path.basename(filePath);
    docs.forEach((doc) => {
      doc.metadata.source = fileName;
      doc.metadata.filePath = filePath;
      doc.metadata.timestamp = new Date().toISOString();
    });

    return docs;
  }

  /**
   * 分割文档为小块
   */
  async splitDocuments(documents: any[]) {
    const chunks = await this.splitter.splitDocuments(documents);

    console.log(`原始文档数: ${documents.length}`);
    console.log(`分割后块数: ${chunks.length}`);

    return chunks;
  }

  /**
   * 生成向量
   */
  async generateEmbeddings(texts: string[]) {
    return await this.embeddings.embedDocuments(texts);
  }

  /**
   * 生成单个查询向量
   */
  async embedQuery(query: string) {
    return await this.embeddings.embedQuery(query);
  }
}
```

### 2. 向量数据库集成

```typescript
// src/vectorStore.ts
import { QdrantVectorStore } from "@langchain/qdrant";
import { QdrantClient } from "@qdrant/js-client-rest";
import { Document } from "@langchain/core/documents";

export class VectorStoreManager {
  private client: QdrantClient;
  private collectionName: string;
  private embeddings: any;

  constructor(
    embeddings: any,
    collectionName: string = "knowledge_base",
    url: string = "http://localhost:6333"
  ) {
    this.client = new QdrantClient({ url });
    this.collectionName = collectionName;
    this.embeddings = embeddings;
  }

  /**
   * 创建集合
   */
  async createCollection(vectorSize: number = 1536) {
    try {
      await this.client.createCollection(this.collectionName, {
        vectors: {
          size: vectorSize,
          distance: "Cosine", // 余弦相似度
        },
        // 优化配置
        optimizers_config: {
          indexing_threshold: 10000,
        },
      });

      console.log(`集合 ${this.collectionName} 创建成功`);
    } catch (error: any) {
      if (error.message.includes("already exists")) {
        console.log(`集合 ${this.collectionName} 已存在`);
      } else {
        throw error;
      }
    }
  }

  /**
   * 添加文档到向量库
   */
  async addDocuments(documents: Document[]) {
    const vectorStore = await QdrantVectorStore.fromDocuments(
      documents,
      this.embeddings,
      {
        client: this.client,
        collectionName: this.collectionName,
      }
    );

    console.log(`成功添加 ${documents.length} 个文档块`);
    return vectorStore;
  }

  /**
   * 相似度搜索
   */
  async similaritySearch(query: string, k: number = 4) {
    const vectorStore = await QdrantVectorStore.fromExistingCollection(
      this.embeddings,
      {
        client: this.client,
        collectionName: this.collectionName,
      }
    );

    const results = await vectorStore.similaritySearch(query, k);

    return results;
  }

  /**
   * 带分数的相似度搜索
   */
  async similaritySearchWithScore(query: string, k: number = 4) {
    const vectorStore = await QdrantVectorStore.fromExistingCollection(
      this.embeddings,
      {
        client: this.client,
        collectionName: this.collectionName,
      }
    );

    const results = await vectorStore.similaritySearchWithScore(query, k);

    return results.map(([doc, score]) => ({
      document: doc,
      score: score,
    }));
  }

  /**
   * 删除集合
   */
  async deleteCollection() {
    await this.client.deleteCollection(this.collectionName);
    console.log(`集合 ${this.collectionName} 已删除`);
  }

  /**
   * 获取集合信息
   */
  async getCollectionInfo() {
    const info = await this.client.getCollection(this.collectionName);
    return info;
  }
}
```

### 3. RAG 核心引擎

```typescript
// src/ragEngine.ts
import { ChatOpenAI } from "@langchain/openai";
import { PromptTemplate } from "@langchain/core/prompts";
import { StringOutputParser } from "@langchain/core/output_parsers";
import { RunnableSequence } from "@langchain/core/runnables";
import { DocumentProcessor } from "./documentProcessor";
import { VectorStoreManager } from "./vectorStore";

export interface RAGConfig {
  openaiApiKey: string;
  model?: string;
  temperature?: number;
  retrievalCount?: number;
  minScore?: number;
}

export class RAGEngine {
  private llm: ChatOpenAI;
  private processor: DocumentProcessor;
  private vectorStore: VectorStoreManager;
  private config: Required&lt;RAGConfig&gt;;

  constructor(config: RAGConfig) {
    this.config = {
      model: "gpt-4o-mini",
      temperature: 0.3,
      retrievalCount: 5,
      minScore: 0.7,
      ...config,
    };

    this.llm = new ChatOpenAI({
      openAIApiKey: config.openaiApiKey,
      modelName: this.config.model,
      temperature: this.config.temperature,
    });

    this.processor = new DocumentProcessor(config.openaiApiKey);
    this.vectorStore = new VectorStoreManager(
      (this.processor as any).embeddings,
      "knowledge_base"
    );
  }

  /**
   * 初始化知识库
   */
  async initializeKnowledgeBase(documentPaths: string[]) {
    // 创建向量集合
    await this.vectorStore.createCollection();

    // 处理所有文档
    const allChunks: any[] = [];

    for (const filePath of documentPaths) {
      console.log(`处理文档: ${filePath}`);
      const docs = await this.processor.loadDocument(filePath);
      const chunks = await this.processor.splitDocuments(docs);
      allChunks.push(...chunks);
    }

    // 批量添加到向量库
    await this.vectorStore.addDocuments(allChunks);

    console.log(`知识库初始化完成，共 ${allChunks.length} 个文档块`);
  }

  /**
   * 检索相关文档
   */
  async retrieve(query: string) {
    const results = await this.vectorStore.similaritySearchWithScore(
      query,
      this.config.retrievalCount
    );

    // 过滤低分结果
    const filtered = results.filter(
      (r) => r.score &gt;= this.config.minScore
    );

    return filtered;
  }

  /**
   * 生成答案
   */
  async generateAnswer(query: string) {
    // 1. 检索相关文档
    const retrievedDocs = await this.retrieve(query);

    if (retrievedDocs.length === 0) {
      return {
        answer: "抱歉，我在知识库中没有找到相关信息。",
        sources: [],
        confidence: 0,
      };
    }

    // 2. 构建上下文
    const context = retrievedDocs
      .map((r, i) =&gt; `[${i + 1}] ${r.document.pageContent}`)
      .join("\n\n");

    // 3. 构建提示词
    const promptTemplate = PromptTemplate.fromTemplate(`
你是一个专业的知识库助手。请基于提供的上下文信息回答用户问题。

## 规则：
1. 只使用上下文中的信息，不要编造
2. 如果上下文中没有相关信息，明确告知用户
3. 回答要准确、简洁、有条理
4. 引用来源时标注 [1], [2] 等

## 上下文：
{context}

## 用户问题：
{question}

## 回答：
    `);

    // 4. 生成答案
    const chain = RunnableSequence.from([
      promptTemplate,
      this.llm,
      new StringOutputParser(),
    ]);

    const answer = await chain.invoke({
      context,
      question: query,
    });

    // 5. 计算置信度
    const avgScore =
      retrievedDocs.reduce((sum, r) =&gt; sum + r.score, 0) / retrievedDocs.length;

    return {
      answer,
      sources: retrievedDocs.map((r) =&gt; ({
        content: r.document.pageContent.substring(0, 200) + "...",
        source: r.document.metadata.source,
        score: r.score,
      })),
      confidence: avgScore,
    };
  }

  /**
   * 流式生成答案
   */
  async *generateAnswerStream(query: string) {
    const retrievedDocs = await this.retrieve(query);

    if (retrievedDocs.length === 0) {
      yield "抱歉，我在知识库中没有找到相关信息。";
      return;
    }

    const context = retrievedDocs
      .map((r, i) =&gt; `[${i + 1}] ${r.document.pageContent}`)
      .join("\n\n");

    const promptTemplate = PromptTemplate.fromTemplate(`
基于以下上下文回答问题：

上下文：
{context}

问题：{question}

回答：
    `);

    const chain = promptTemplate.pipe(this.llm).pipe(new StringOutputParser());

    const stream = await chain.stream({
      context,
      question: query,
    });

    for await (const chunk of stream) {
      yield chunk;
    }
  }
}
```

```typescript
// src/index.ts
import { RAGEngine } from "./ragEngine";
import * as path from "path";

async function main() {
  // 初始化 RAG 引擎

  const questions = [

    "如何重置密码？",
    "支持哪些支付方式？",
  ];

    const result = await rag.generateAnswer(question);

    console.log(`答案: ${result.answer}`);
    console.log(`\n置信度: ${(result.confidence * 100).toFixed(1)}%`);
    console.log(`\n来源文档:`);
    result.sources.forEach((s, i) =&gt; {
      console.log(`  [${i + 1}] ${s.source} (相关度: ${s.score.toFixed(2)})`);
    });
  }
}

main().catch(console.error);
```

### 5. API 服务封装

```typescript
// src/api.ts
import express from "express";
import { RAGEngine } from "./ragEngine";
import multer from "multer";

const app = express();
app.use(express.json());

const rag = new RAGEngine({
  openaiApiKey: process.env.OPENAI_API_KEY!,

// 文件上传配置
const upload = multer({ dest: "./uploads/" });

// 问答接口
app.post("/api/ask", async (req, res) =&gt; {
  try {
    const { question } = req.body;

    if (!question) {
      return res.status(400).json({ error: "问题不能为空" });
    }

    const result = await rag.generateAnswer(question);

    res.json({
      success: true,
      data: result,
    });
  } catch (error) {
    console.error("问答错误:", error);
    res.status(500).json({ error: "服务器错误" });
  }
});

// 流式问答接口
app.get("/api/ask/stream", async (req, res) =&gt; {
  const { question } = req.query;

  res.setHeader("Content-Type", "text/event-stream");
  res.setHeader("Cache-Control", "no-cache");
  res.setHeader("Connection", "keep-alive");

  try {
    for await (const chunk of rag.generateAnswerStream(question as string)) {
      res.write(`data: ${JSON.stringify({ chunk })}\n\n`);
    }

    res.write("data: [DONE]\n\n");
    res.end();
  } catch (error) {
    res.write(`data: ${JSON.stringify({ error: "生成失败" })}\n\n`);
    res.end();
  }
});

// 上传文档接口
app.post(
  "/api/documents/upload",
  upload.array("files"),
  async (req, res) =&gt; {
    try {
      const files = req.files as Express.Multer.File[];
      const filePaths = files.map((f) =&gt; f.path);

      await rag.initializeKnowledgeBase(filePaths);

      res.json({
        success: true,
        message: `成功上传 ${files.length} 个文档`,
      });
    } catch (error) {
      res.status(500).json({ error: "上传失败" });
    }
  }
);

app.listen(3000, () =&gt; {
  console.log("RAG API 服务运行在 http://localhost:3000");
});
```

## 进阶优化技巧

### 1. 混合检索（Hybrid Search）

结合关键词检索和向量检索，提高召回率：

```typescript
import { BM25Retriever } from "langchain/retrievers/bm25";
import { EnsembleRetriever } from "langchain/retrievers/ensemble";

async function createHybridRetriever(vectorStore: any, documents: any[]) {
  // BM25 关键词检索
  const bm25Retriever = new BM25Retriever.fromDocuments(documents);
  bm25Retriever.k = 5;

  // 向量检索
  const vectorRetriever = vectorStore.asRetriever(5);

  // 集成检索器
  const ensembleRetriever = new EnsembleRetriever({
    retrievers: [bm25Retriever, vectorRetriever],
    weights: [0.4, 0.6], // BM25 权重 40%，向量权重 60%
  });

  return ensembleRetriever;
}
```

```typescript
async function batchEmbed(texts: string[], batchSize = 100) {
  const results = [];

  for (let i = 0; i &lt; texts.length; i += batchSize) {
```

```typescript
  countTokens(text: string) {
    // 简单估算：中文约 1.5 字/token，英文约 4 字符/token
    return Math.ceil(text.length / 2);
  }

  trackUsage(prompt: string, completion: string) {
    this.usage.prompt += this.countTokens(prompt);
    this.usage.completion += this.countTokens(completion);

    // 计算成本（GPT-4o-mini 价格）
    const cost =
      (this.usage.prompt * 0.00015) / 1000 +
      (this.usage.completion * 0.0006) / 1000;

    return { usage: this.usage, cost };
  }
}
```

## 常见问题与解决方案

### Q1: 检索不到相关文档？

**原因**：

- 分块过大或过小
- Embedding 模型不适合业务场景
- 查询与文档表述差异大

**解决**：

- 调整 chunkSize（500-1500 之间）
- 使用领域特定的 Embedding 模型
- 实现查询改写和同义词扩展

### Q2: 回答出现幻觉？

**原因**：

- 检索文档不相关
- 提示词不够严格

**解决**：

- 提高相似度阈值
- 强化提示词约束："只使用上下文信息"
- 后处理：验证答案中的事实

### Q3: 响应速度慢？

**原因**：

- 向量检索慢
- LLM 生成慢

**解决**：

- 使用流式输出
- 向量库索引优化
- 引入缓存层

### Q4: 成本过高？

**解决方案**：

- 使用更便宜的模型（GPT-4o-mini）
- 缓存常见问题的答案
- 减少 retrievalCount
- 使用开源模型本地部署

## 总结

RAG 技术让大模型具备了"外挂知识库"的能力，是构建企业 AI 应用的核心技术。关键要点：

1. **文档处理**：选择合适的分块策略，保留语义完整性
2. **向量数据库**：根据规模选择 Qdrant（中小）或 Milvus（大）
3. **检索优化**：混合检索 + 重排序提升召回率
4. **提示工程**：明确约束，避免幻觉
5. **性能优化**：批量处理、并发控制、缓存

下一步可以探索：

- **多模态 RAG**：支持图片、表格检索
- **Agent RAG**：让 AI 自主决定何时检索
- **Graph RAG**：结合知识图谱，提升推理能力
