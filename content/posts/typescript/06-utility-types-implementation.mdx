## 前言

TypeScript 提供了许多内置的工具类型（Utility Types），如 `Partial`、`Readonly`、`Pick`、`Omit` 等。这些工具类型极大提升了我们的开发效率，但你是否真正理解它们的实现原理？

掌握工具类型的实现原理，不仅能够更好地使用它们，还能根据实际需求创建自己的工具类型。

## 核心概念

### 映射类型（Mapped Types）

映射类型是工具类型的基础，允许我们遍历类型的键：

```typescript
type Mapped<T> = {
  [K in keyof T]: T[K];
};

interface User {
  id: number;
  name: string;
  email: string;
}

type MappedUser = Mapped<User>;
// 等同于 User
```

### 映射修饰符

在映射类型中使用 `+`、`-`、`readonly`、`?` 等修饰符：

```typescript
// 添加 readonly
type Readonly<T> = {
  readonly [K in keyof T]: T[K];
};

// 添加可选
type Partial<T> = {
  [K in keyof T]?: T[K];
};

// 移除 readonly
type Mutable<T> = {
  -readonly [K in keyof T]: T[K];
};

// 移除可选
type Required<T> = {
  [K in keyof T]-?: T[K];
};
```

### Key Remapping

使用 `as` 子句重新映射键：

```typescript
type Getters<T> = {
  [K in keyof T as `get${Capitalize<string & K>}`]: () => T[K];
};

interface User {
  name: string;
  age: number;
}

type UserGetters = Getters<User>;
// {
//   getName: () => string
//   getAge: () => number
// }
```

## 内置工具类型实现

### 1. Partial<T>

将所有属性变为可选：

```typescript
// ✅ TypeScript 实现
type Partial<T> = {
  [P in keyof T]?: T[P];
};

// 使用示例
interface User {
  id: number;
  name: string;
  email: string;
}

type PartialUser = Partial<User>;
// {
//   id?: number
//   name?: string
//   email?: string
// }

function updateUser(id: number, updates: Partial<User>) {
  // 实现更新逻辑
}
```

### 2. Required<T>

将所有属性变为必需：

```typescript
// ✅ TypeScript 实现
type Required<T> = {
  [P in keyof T]-?: T[P];
};

// 使用示例
interface PartialUser {
  id?: number;
  name?: string;
  email?: string;
}

type CompleteUser = Required<PartialUser>;
// {
//   id: number
//   name: string
//   email: string
// }
```

### 3. Readonly<T>

将所有属性变为只读：

```typescript
// ✅ TypeScript 实现
type Readonly<T> = {
  readonly [P in keyof T]: T[P];
};

// 使用示例
interface User {
  id: number;
  name: string;
}

type ReadonlyUser = Readonly<User>;

const user: ReadonlyUser = { id: 1, name: "Alice" };
user.name = "Bob"; // ❌ 无法分配到 "name" ，因为它是只读属性
```

### 4. Record<K, T>

创建一个类型，其键为 K，值为 T：

```typescript
// ✅ TypeScript 实现
type Record<K extends keyof any, T> = {
  [P in K]: T;
};

// 使用示例
type UserRole = "admin" | "user" | "guest";

type Permissions = Record<UserRole, string[]>;
// {
//   admin: string[]
//   user: string[]
//   guest: string[]
// }

const permissions: Permissions = {
  admin: ["read", "write", "delete"],
  user: ["read", "write"],
  guest: ["read"],
};
```

### 5. Pick<T, K>

从 T 中选择一组属性 K：

```typescript
// ✅ TypeScript 实现
type Pick<T, K extends keyof T> = {
  [P in K]: T[P];
};

// 使用示例
interface User {
  id: number;
  name: string;
  email: string;
  age: number;
}

type UserSummary = Pick<User, "id" | "name">;
// {
//   id: number
//   name: string
// }

function getUserSummary(user: User): UserSummary {
  return {
    id: user.id,
    name: user.name,
  };
}
```

### 6. Omit<T, K>

从 T 中排除一组属性 K：

```typescript
// ✅ TypeScript 实现（新版本）
type Omit<T, K extends keyof any> = Pick<T, Exclude<keyof T, K>>;

// 等价于：
type Omit<T, K extends keyof any> = {
  [P in keyof T as P extends K ? never : P]: T[P];
};

// 使用示例
interface User {
  id: number;
  name: string;
  email: string;
  password: string;
}

type PublicUser = Omit<User, "password">;
// {
//   id: number
//   name: string
//   email: string
// }

function toPublicUser(user: User): PublicUser {
  const { password, ...publicUser } = user;
  return publicUser;
}
```

### 7. Extract<T, U>

从 T 中提取可以赋值给 U 的类型：

```typescript
// ✅ TypeScript 实现
type Extract<T, U> = T extends U ? T : never;

// 使用示例
type T = string | number | boolean;

type OnlyString = Extract<T, string>;
// string

type StringOrNumber = Extract<T, string | number>;
// string | number
```

### 8. Exclude<T, U>

从 T 中排除可以赋值给 U 的类型：

```typescript
// ✅ TypeScript 实现
type Exclude<T, U> = T extends U ? never : T;

// 使用示例
type T = string | number | boolean;

type NotString = Exclude<T, string>;
// number | boolean

type OnlyBoolean = Exclude<T, string | number>;
// boolean
```

### 9. ReturnType<T>

获取函数类型的返回值类型：

```typescript
// ✅ TypeScript 实现
type ReturnType<T extends (...args: any) => any> = T extends (
  ...args: any
) => infer R
  ? R
  : any;

// 使用示例
function getUser() {
  return { id: 1, name: "Alice" };
}

type User = ReturnType<typeof getUser>;
// { id: number; name: string }

async function fetchUser() {
  return { id: 1, name: "Alice" };
}

type FetchedUser = Awaited<ReturnType<typeof fetchUser>>;
// { id: number; name: string }
```

### 10. Parameters<T>

获取函数类型的参数类型：

```typescript
// ✅ TypeScript 实现
type Parameters<T extends (...args: any) => any> = T extends {
  (...args: infer P): any;
}
  ? P
  : never;

// 使用示例
function updateUser(id: number, name: string, email: string) {
  // ...
}

type UpdateUserParams = Parameters<typeof updateUser>;
// [number, string, string]
```

## 自定义工具类型

### 深度工具类型

```typescript
// 深度 Partial
type DeepPartial<T> = {
  [P in keyof T]?: T[P] extends object ? DeepPartial<T[P]> : T[P];
};

// 深度 Required
type DeepRequired<T> = {
  [P in keyof T]-?: T[P] extends object ? DeepRequired<T[P]> : T[P];
};

// 深度 Readonly
type DeepReadonly<T> = {
  readonly [P in keyof T]: T[P] extends object ? DeepReadonly<T[P]> : T[P];
};

// 深度 Mutable
type DeepMutable<T> = {
  -readonly [P in keyof T]: T[P] extends object ? DeepMutable<T[P]> : T[P];
};

// 使用示例
interface User {
  id: number;
  profile: {
    name: string;
    address: {
      city: string;
      country: string;
    };
  };
}

type PartialUser = DeepPartial<User>;
// 所有嵌套属性都是可选的
```

### 条件工具类型

```typescript
// 提取函数类型
type FunctionProperties<T> = {
  [K in keyof T]: T[K] extends Function ? K : never;
}[keyof T];

// 提取非函数类型
type NonFunctionProperties<T> = {
  [K in keyof T]: T[K] extends Function ? never : K;
}[keyof T];

// 使用示例
class UserService {
  id = 1;
  name = "Service";

  getUser() {
    return { id: 1, name: "Alice" };
  }
  updateUser(user: User) {}
}

type UserMethods = FunctionProperties<UserService>;
// 'getUser' | 'updateUser'

type UserServiceProps = NonFunctionProperties<UserService>;
// 'id' | 'name'
```

### 转换工具类型

```typescript
// 将所有属性变为大写键
type UppercaseKeys<T> = {
  [K in keyof T as Uppercase<string & K>]: T[K];
};

// 将所有属性变为小写键
type LowercaseKeys<T> = {
  [K in keyof T as Lowercase<string & K>]: T[K];
};

// 首字母大写
type CapitalizeKeys<T> = {
  [K in keyof T as Capitalize<string & K>]: T[K];
};

// 使用示例
interface User {
  firstName: string;
  lastName: string;
  email: string;
}

type CapitalizedUser = CapitalizeKeys<User>;
// {
//   FirstName: string
//   LastName: string
//   Email: string
// }
```

### 路径工具类型

```typescript
// 获取对象的所有路径
type Paths<T, K extends keyof T = keyof T> = K extends string | number
  ? T[K] extends object
    ? K | `${K}.${Paths<T[K]>}`
    : K
  : never;

// 使用示例
interface User {
  id: number;
  name: string;
  profile: {
    email: string;
    address: {
      city: string;
    };
  };
}

type UserPaths = Paths<User>;
// 'id' | 'name' | 'profile' | 'profile.email' | 'profile.address' | 'profile.address.city'

// 根据路径获取值类型
type PathValue<T, P extends Paths<T>> = P extends `${infer K}.${infer Rest}`
  ? K extends keyof T
    ? PathValue<T[K], Rest>
    : never
  : P extends keyof T
    ? T[P]
    : never;

type CityType = PathValue<User, "profile.address.city">;
// string
```

### 构造器工具类型

```typescript
// 获取实例类型
type InstanceType<T extends new (...args: any) => any> = T extends new (
  ...args: any
) => infer R
  ? R
  : any;

// 获取构造器参数类型
type ConstructorParameters<T extends new (...args: any) => any> =
  T extends new (...args: infer P) => any ? P : never;

// 使用示例
class User {
  constructor(
    public id: number,
    public name: string
  ) {}
}

type UserInstance = InstanceType<typeof User>;
// User

type UserConstructorParams = ConstructorParameters<typeof User>;
// [number, string]
```

## 高级应用

### 类型安全的表单字段

```typescript
// 提取所有字段
type Fields<T> = {
  [K in keyof T]: T[K];
};

// 提取所有字段的验证器类型
type FieldValidators<T> = {
  [K in keyof T]?: (value: T[K]) => boolean | string;
};

// 使用示例
interface User {
  name: string;
  email: string;
  age: number;
}

const userValidators: FieldValidators<User> = {
  name: (value) => value.length > 0 || "Name is required",
  email: (value) => /^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(value) || "Invalid email",
  age: (value) => value >= 18 || "Must be 18 or older",
};

function validateUser<T extends object>(
  data: T,
  validators: FieldValidators<T>
): boolean | string {
  for (const key in validators) {
    const validator = validators[key];
    if (validator) {
      const result = validator(data[key]);
      if (typeof result === "string") {
        return result;
      } else if (!result) {
        return `Validation failed for ${key}`;
      }
    }
  }
  return true;
}
```

### 状态机类型

```typescript
// 定义状态和事件
type State = "idle" | "loading" | "success" | "error";

type Event =
  | { type: "FETCH" }
  | { type: "SUCCESS"; data: any }
  | { type: "ERROR"; error: Error }
  | { type: "RESET" };

// 状态转换
type StateTransitions = {
  idle: Event[];
  loading: Event[];
  success: Event[];
  error: Event[];
};

// 类型安全的处理函数
type StateHandler = {
  [S in State]: (event: Extract<Event, { type: any }>) => State;
};

const stateHandlers: StateHandler = {
  idle: (event) => {
    if (event.type === "FETCH") return "loading";
    return "idle";
  },
  loading: (event) => {
    if (event.type === "SUCCESS") return "success";
    if (event.type === "ERROR") return "error";
    return "loading";
  },
  success: (event) => {
    if (event.type === "FETCH") return "loading";
    if (event.type === "RESET") return "idle";
    return "success";
  },
  error: (event) => {
    if (event.type === "FETCH") return "loading";
    if (event.type === "RESET") return "idle";
    return "error";
  },
};
```

## 实战应用

### 构建 API 响应类型系统

```typescript
// API 响应基础类型
type APIResponse<T, E = Error> =
  | { success: true; data: T }
  | { success: false; error: E };

// 提取成功数据
type ExtractData<T> = T extends { success: true; data: infer D } ? D : never;

// 提取错误信息
type ExtractError<T> = T extends { success: false; error: infer E } ? E : never;

// 使用示例
type UserResponse = APIResponse<User>;

async function fetchUser(id: number): Promise<UserResponse> {
  try {
    const user = await api.getUser(id);
    return { success: true, data: user };
  } catch (error) {
    return { success: false, error: error as Error };
  }
}

// 处理响应
const response = await fetchUser(1);

if (response.success) {
  // response.data: User
  console.log(response.data.name);
} else {
  // response.error: Error
  console.error(response.error.message);
}
```

### 构建类型安全的配置系统

```typescript
// 配置项定义
type ConfigItem<T = any> = {
  type: "string" | "number" | "boolean" | "object";
  default?: T;
  required?: boolean;
  validator?: (value: T) => boolean;
};

type ConfigSchema<T extends Record<string, ConfigItem>> = {
  [K in keyof T]: T[K]["default"];
};

// 创建配置验证器
function createConfigValidator<T extends Record<string, ConfigItem>>(
  schema: T
): (config: Partial<ConfigSchema<T>>) => ConfigSchema<T> {
  return (config) => {
    const result: any = {};

    for (const key in schema) {
      const item = schema[key];
      const value = config[key];

      if (value === undefined) {
        if (item.required) {
          throw new Error(`Missing required config: ${key}`);
        }
        result[key] = item.default;
      } else {
        if (item.validator && !item.validator(value)) {
          throw new Error(`Invalid config value for: ${key}`);
        }
        result[key] = value;
      }
    }

    return result;
  };
}

// 使用示例
const appConfigSchema = {
  apiUrl: {
    type: "string",
    default: "https://api.example.com",
    required: true,
  },
  timeout: { type: "number", default: 5000, required: false },
  debug: { type: "boolean", default: false, required: false },
  retries: {
    type: "number",
    default: 3,
    validator: (v) => v >= 0 && v <= 10,
  },
} satisfies Record<string, ConfigItem>;

const validateConfig = createConfigValidator(appConfigSchema);

const config = validateConfig({
  apiUrl: "https://production.api.com",
  timeout: 10000,
});
```

## 避坑指南

### ❌ 错误 1：忽略映射类型的分配

```typescript
// ❌ 可能导致意外的行为
type Wrapped<T> = { value: T };
type Distributed<T> = T extends any ? Wrapped<T> : never;

type Result = Distributed<string | number>;
// Wrapped<string> | Wrapped<number> (分发)

// ✅ 如果不想分发，使用元组包装
type NonDistributed<T extends any> = T extends any ? Wrapped<T> : never;
```

---

### ❌ 错误 2：Key Remapping 中的类型断言

```typescript
// ❌ 过度使用类型断言
type Getters<T> = {
  [K in keyof T as `get${Capitalize<string & K>}`]: T[K];
};

// ✅ 更安全的写法
type SafeGetters<T> = {
  [K in keyof T as K extends string ? `get${Capitalize<K>}` : never]: T[K];
};
```

---

### ❌ 错误 3：递归类型导致性能问题

```typescript
// ❌ 可能导致性能问题
type DeepObject<T> = {
  [K in keyof T]: T[K] extends object
    ? DeepObject<T[K]>
    : T[K]
}

// ✅ 添加深度限制
type DeepObjectLimited<T, Depth extends number = 5> =
  Depth extends 0
    ? T
    : T extends object
      ? { [K in keyof T]: DeepObjectLimited<T[K], Depth extends 10 ? 9 : Depth - 1> }
      : T
```

## 总结

### 核心要点

1. **映射类型**：遍历类型的键，创建新类型
2. **映射修饰符**：`readonly`、`?`、`-readonly`、`-?`
3. **Key Remapping**：使用 `as` 重新映射键
4. **条件类型**：实现复杂的类型逻辑
5. **递归类型**：处理嵌套结构（注意性能）

### 实用工具类型

```typescript
// 基础工具类型
type Partial<T> = { [P in keyof T]?: T[P] };
type Required<T> = { [P in keyof T]-?: T[P] };
type Readonly<T> = { readonly [P in keyof T]: T[P] };
type Pick<T, K extends keyof T> = { [P in K]: T[P] };
type Omit<T, K extends keyof any> = Pick<T, Exclude<keyof T, K>>;

// 条件类型
type Extract<T, U> = T extends U ? T : never;
type Exclude<T, U> = T extends U ? never : T;

// 函数类型
type ReturnType<T extends (...args: any) => any> = T extends (
  ...args: any
) => infer R
  ? R
  : any;

type Parameters<T extends (...args: any) => any> = T extends {
  (...args: infer P): any;
}
  ? P
  : never;
```
