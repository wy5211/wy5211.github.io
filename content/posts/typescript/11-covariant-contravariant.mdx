## 前言

TypeScript 的类型系统基于**结构化类型**（Structural Typing），但类型兼容性并不是简单的一对一关系。理解**协变**（Covariance）、**逆变**（Contravariance）和**双向协变**（Bivariance）对于理解 TypeScript 的类型系统至关重要。

这些概念会影响你的代码是否能够通过类型检查，特别是在函数参数和泛型中。

## 核心概念

### 类型层次结构

```typescript
// 类型层次：Animal 是父类型，Dog 是子类型
class Animal {
  name: string;
}

class Dog extends Animal {
  bark(): void {
    console.log("Woof!");
  }
}

// 子类型可以赋值给父类型（协变）
const animal: Animal = new Dog(); // ✅
const dog: Dog = new Animal(); // ❌
```

### 什么是协变和逆变？

- **协变**：类型方向一致（子类型 → 子类型）
- **逆变**：类型方向相反（父类型 → 子类型）
- **双向协变**：两个方向都允许

```typescript
// 协变示例（普通对象）
interface AnimalBox {
  value: Animal;
}

interface DogBox {
  value: Dog;
}

const animalBox: AnimalBox = { value: new Dog() }; // ✅ 协变

// 函数参数的逆变
type AnimalHandler = (animal: Animal) => void;
type DogHandler = (dog: Dog) => void;

const handleAnimal: AnimalHandler = (animal) => {};
const handleDog: DogHandler = (dog) => {};

handleDog = handleAnimal; // ✅ 逆变：父类型函数可以赋值给子类型
handleAnimal = handleDog; // ❌ 类型错误
```

## 协变与逆变详解

### 1. 对象类型的协变

对象的属性类型是协变的：

```typescript
interface Producer<T> {
  get(): T;
}

// Producer<Dog> 可以赋值给 Producer<Animal>
const dogProducer: Producer<Dog> = {
  get: () => new Dog(),
};

const animalProducer: Producer<Animal> = dogProducer; // ✅ 协变

// 原因：Producer<Dog> 返回 Dog，Dog 是 Animal 的子类型
// 所以 Producer<Dog> 返回的也是 Animal 的子类型
```

### 2. 函数参数的逆变

函数的参数类型是逆变的：

```typescript
interface Consumer<T> {
  consume(value: T): void;
}

// Consumer<Animal> 可以赋值给 Consumer<Dog>
const animalConsumer: Consumer<Animal> = {
  consume: (animal) => console.log(animal.name),
};

const dogConsumer: Consumer<Dog> = animalConsumer; // ✅ 逆变

// 原因：animalConsumer 可以处理 Animal，当然也能处理 Dog
// 所以 animalConsumer 可以安全地用作 dogConsumer
```

### 3. 函数返回值的协变

函数的返回值类型是协变的：

```typescript
interface Producer<T> {
  produce(): T;
}

const dogProducer: Producer<Dog> = {
  produce: () => new Dog(),
};

const animalProducer: Producer<Animal> = dogProducer; // ✅ 协变

// 原因：dogProducer.produce() 返回 Dog，Dog 是 Animal 的子类型
// 所以返回值是兼容的
```

## 高级应用

### 技巧 1：理解函数类型的类型关系

```typescript
// 函数类型的完整规则：
// 参数：逆变
// 返回值：协变

type F1 = (animal: Animal) => Dog;
type F2 = (dog: Dog) => Animal;

// F1 可以赋值给 F2？
// 参数：Animal 是 Dog 的父类型（逆变方向相反）
// 返回值：Dog 是 Animal 的子类型（协变方向一致）
// 结果：F1 可以赋值给 F2

const f1: F1 = (animal) => new Dog();
const f2: F2 = f1; // ✅

// 验证
f2(new Dog()); // 等价于 f1(new Dog())
// f1 接受 Animal，传入 Dog（子类型）✅
// f1 返回 Dog，f2 期望返回 Animal（Dog 是子类型）✅
```

---

### 技巧 2：数组的双向协变

在 TypeScript 中，数组是双向协变的（这不是类型安全的）：

```typescript
const animals: Animal[] = [];
const dogs: Dog[] = [];

// ⚠️ 双向协变（类型不安全）
animals = dogs; // ✅ 允许
dogs = animals; // ✅ 允许（不安全！）

// 为什么不安全？
dogs.push(new Animal()); // 运行时可能出错
```

**启用严格函数类型检查**：

```json
// tsconfig.json
{
  "compilerOptions": {
    "strictFunctionTypes": true
  }
}
```

### 技巧 3：泛型的协变与逆变

```typescript
// 默认情况下，泛型参数是不变的（既不协变也不逆变）
interface Box<T> {
  value: T;
}

const animalBox: Box<Animal> = { value: new Animal() };
const dogBox: Box<Dog> = { value: new Dog() };

// ❌ 严格模式下不允许
animalBox = dogBox;
dogBox = animalBox;

// 指定协变
interface CovariantBox<out T> {
  get(): T;
}

const covariantDogBox: CovariantBox<Dog> = {
  get: () => new Dog(),
};

const covariantAnimalBox: CovariantBox<Animal> = covariantDogBox; // ✅ 协变

// 指定逆变
interface ContravariantBox<in T> {
  set(value: T): void;
}

const contravariantAnimalBox: ContravariantBox<Animal> = {
  set: (animal) => {},
};

const contravariantDogBox: ContravariantBox<Dog> = contravariantAnimalBox; // ✅ 逆变
```

---

### 技巧 4：混合协变和逆变

```typescript
interface Processor<in T, out U> {
  process(value: T): U;
}

// T 是逆变的（输入）
// U 是协变的（输出）

const animalToDog: Processor<Animal, Dog> = {
  process: (animal) => new Dog(),
};

const dogToAnimal: Processor<Dog, Animal> = {
  process: (dog) => new Animal(),
};

// ✅ Processor<Animal, Dog> 可以赋值给 Processor<Dog, Animal>
// T: Animal → Dog（逆变方向相反，Animal 是父类型，正确）
// U: Dog → Animal（协变方向一致，Dog 是子类型，正确）
const processor: Processor<Dog, Animal> = animalToDog;
```

### 技巧 5：实用示例 - 事件处理器

```typescript
// 事件处理器需要逆变
type EventHandler<T> = (event: T) => void;

class EventEmitter<T> {
  private listeners: EventHandler<T>[] = [];

  on(handler: EventHandler<T>): void {
    this.listeners.push(handler);
  }

  emit(event: T): void {
    this.listeners.forEach((handler) => handler(event));
  }
}

// 使用示例
interface MouseEvent {
  x: number;
  y: number;
}

interface ClickEvent extends MouseEvent {
  button: number;
}

// ❌ 错误理解
const mouseEmitter = new EventEmitter<MouseEvent>();
const clickEmitter = new EventEmitter<ClickEvent>();

mouseEmitter = clickEmitter; // ❌ 类型错误

// ✅ 正确理解
// 处理 ClickEvent 的函数可以处理 MouseEvent 吗？不行
// 处理 MouseEvent 的函数可以处理 ClickEvent 吗？可以

const mouseHandler: EventHandler<MouseEvent> = (event) => {
  console.log(`Mouse at ${event.x}, ${event.y}`);
};

const clickHandler: EventHandler<ClickEvent> = (event) => {
  console.log(`Click at ${event.x}, ${event.y} with button ${event.button}`);
};

const clickEmitter = new EventEmitter<ClickEvent>();

// ✅ mouseHandler 可以处理 ClickEvent（ClickEvent 是 MouseEvent 的子类型）
clickEmitter.on(mouseHandler);

clickEmitter.emit({ x: 100, y: 200, button: 1 });
```

### 技巧 6：类型受控的数组操作

```typescript
// 只读数组（协变）
type ReadonlyArray<T> = {
  readonly [index: number]: T;
  length: number;
};

// 可写数组（逆变）
type WritableArray<T> = {
  [index: number]: T;
  length: number;
};

// ReadonlyArray<Dog> 可以赋值给 ReadonlyArray<Animal>
const readonlyDogs: ReadonlyArray<Dog> = [new Dog()];
const readonlyAnimals: ReadonlyArray<Animal> = readonlyDogs; // ✅

// WritableArray<Animal> 可以赋值给 WritableArray<Dog>（逆变）
// 但实际上这是不安全的，所以 TypeScript 不允许
```

## 实战应用

### 构建类型安全的可观测对象

```typescript
interface Observable<in T> {
  subscribe(observer: Observer<T>): Subscription;
  pipe<R>(operators: Operator<T, R>): Observable<R>;
}

interface Observer<in T> {
  next(value: T): void;
  error(err: Error): void;
  complete(): void;
}

interface Operator<in T, out R> {
  (observable: Observable<T>): Observable<R>;
}

interface Subscription {
  unsubscribe(): void;
}

// 使用逆变
class Subject<T> implements Observable<T> {
  private observers: Observer<T>[] = [];

  subscribe(observer: Observer<T>): Subscription {
    this.observers.push(observer);
    return {
      unsubscribe: () => {
        const index = this.observers.indexOf(observer);
        if (index > -1) {
          this.observers.splice(index, 1);
        }
      },
    };
  }

  next(value: T): void {
    this.observers.forEach((observer) => observer.next(value));
  }

  error(err: Error): void {
    this.observers.forEach((observer) => observer.error(err));
  }

  complete(): void {
    this.observers.forEach((observer) => observer.complete());
  }
}

// 使用示例
const animalSubject = new Subject<Animal>();

// ✅ Observer<Animal> 可以订阅 Subject<Animal>
const animalObserver: Observer<Animal> = {
  next: (animal) => console.log(animal.name),
  error: (err) => console.error(err),
  complete: () => console.log("Complete"),
};

animalSubject.subscribe(animalObserver);

// ✅ Observer<Dog> 也可以订阅 Subject<Animal>（逆变）
const dogObserver: Observer<Dog> = {
  next: (dog) => console.log(`${dog.name} barks!`),
  error: (err) => console.error(err),
  complete: () => console.log("Complete"),
};

animalSubject.subscribe(dogObserver);

animalSubject.next(new Dog());
```

### 构建类型安全的管道操作

```typescript
interface Pipe<in T, out R> {
  (value: T): R;
}

// 字符串处理管道
type StringPipe = Pipe<string, string>;

function toUpperCase(): StringPipe {
  return (value) => value.toUpperCase();
}

function toLowerCase(): StringPipe {
  return (value) => value.toLowerCase();
}

function trim(): StringPipe {
  return (value) => value.trim();
}

function pipe<T>(...fns: Array<Pipe<T, T>>): Pipe<T, T> {
  return (value) => fns.reduce((acc, fn) => fn(acc), value);
}

// 使用示例
const processor = pipe(trim(), toUpperCase());

const result = processor("  hello world  "); // 'HELLO WORLD'

// 类型安全的管道
type StringToNumberPipe = Pipe<string, number>;

function parseIntPipe(base?: number): StringToNumberPipe {
  return (value) => parseInt(value, base);
}

function pipe<T, R>(fn1: Pipe<T, any>, fn2: Pipe<any, R>): Pipe<T, R> {
  return (value) => fn2(fn1(value));
}

const stringToNumber = pipe<String, number>(trim(), parseIntPipe(10));

const number = stringToNumber("  42  "); // 42
```

## 避坑指南

### ❌ 错误 1：忽略函数参数的逆变

```typescript
// ❌ 错误理解
type Callback<T> = (value: T) => void;

function process(callback: Callback<Animal>) {
  callback(new Animal());
}

// 传入 Dog 的回调
process((dog: Dog) => console.log(dog.bark())); // ❌ 运行时错误

// ✅ 正确理解
// process 期望一个能处理 Animal 的回调
// 传入一个只能处理 Dog 的回调是不安全的
// 因为 process 可能传入一个不是 Dog 的 Animal
```

---

### ❌ 错误 2：双向协变的陷阱

```typescript
// ⚠️ 双向协变（不安全）
let animals: Animal[];
let dogs: Dog[];

animals = dogs; // ✅
dogs = animals; // ✅ （不安全！）

// 为什么不安全？
dogs.push(new Animal()); // 运行时错误

// ✅ 使用 readonly 避免这个问题
let readonlyAnimals: ReadonlyArray<Animal>;
let readonlyDogs: ReadonlyArray<Dog>;

readonlyAnimals = readonlyDogs; // ✅ 协变
readonlyDogs = readonlyAnimals; // ❌ 类型错误
```

---

### ❌ 错误 3：误用 out 和 in 修饰符

```typescript
// ❌ 错误：in 修饰符用于协变位置
interface Bad<in T> {
  get(): T; // 应该用 out
}

// ❌ 错误：out 修饰符用于逆变位置
interface Bad<out T> {
  set(value: T): void; // 应该用 in
}

// ✅ 正确
interface Good<in T, out U> {
  set(value: T): void;
  get(): U;
}
```

## 总结

### 核心要点

1. **协变**：类型方向一致（子类型 → 子类型）
2. **逆变**：类型方向相反（父类型 → 子类型）
3. **函数参数**：逆变（为了类型安全）
4. **函数返回值**：协变
5. **双向协变**：TypeScript 中数组默认行为（不安全）

### 实用规则

```typescript
// 生产者（返回值）：协变
interface Producer<out T> {
  produce(): T;
}

// 消费者（参数）：逆变
interface Consumer<in T> {
  consume(value: T): void;
}

// 转换者（输入 + 输出）：混合
interface Transformer<in T, out U> {
  transform(value: T): U;
}

// 默认：不变
interface Box<T> {
  value: T;
}
```
