## 券商 APP 上的数字够用吗

你打开券商 APP，看到沪深 300 ETF 的 PE 是 12.3 倍。然后呢？

12.3 倍是高还是低？和历史比在哪？是 TTM 还是静态 PE？成分股盈利有没有被一次性事件扭曲？这些信息，APP 上通常看不到。

更实际的场景：你持有 6 只 ETF，想每周快速扫描一遍估值状态。每次都打开 APP 一个一个查，太慢了。如果你能拿到原始数据，自己算分位数、做对比、画趋势图，效率会高很多。

这篇就解决一个问题：**估值数据从哪来。**

## 五个免费数据源，各有所长

### 乌龟量化（wglab.com）

最适合：**快速查看 A 股指数的历史估值分位数**

输入指数名称或代码，直接展示 PE/PB 的历史分位数曲线，支持 3 年/5 年/10 年/全部历史窗口切换。还能看到当前的百分位值——比如「沪深 300 当前 PE 处于近 10 年 52% 分位」。

优点：界面直观、完全免费、数据覆盖全 A 股主要指数和行业指数。

缺点：没有 API，不能自动化；数据有延迟（通常 T+1 更新）。

### 集思录（jisilu.cn）

最适合：**看实时 ETF 溢折价率和基本估值数据**

集思录的核心功能是 ETF 溢折价监控——场内价格和场外净值之间的差。但它的 ETF 数据页面也包含了 PE、PB 等估值指标，以及一些社区用户整理的估值排行。

优点：实时更新、社区氛围好、有 ETF 溢折价这个独特数据。

缺点：估值数据不如乌龟量化全面和直观；社区信息噪音多。

### 中证指数官网（csindex.com.cn）

最适合：**查官方权威的指数编制规则和成分股权重**

中证指数公司是沪深 300、中证 500、中证红利等 A 股主流指数的编制方。官网提供每个指数的详细编制方案、成分股名单和权重、历史估值数据下载。

优点：数据最权威——这是指数的「出生证明」；成分股和权重数据其他平台不一定有。

缺点：网站经常改版，估值数据的展示位置不太好找；没有 API，需要手动下载 Excel。

### Sina Finance API

最适合：**用代码拉取 ETF/指数的 K 线数据**

Sina 提供了一个简洁的 K 线 API，返回 JSON 格式的 OHLCV 数据，不需要登录、不需要 API Key：

```
https://money.finance.sina.com.cn/quotes_service/api/json_v2.php/
  CN_MarketData.getKLineData
  ?symbol=sh510300
  &scale=240      # 日K=240，周K=1680
  &ma=no
  &datalen=500    # 拉最近500个交易日
```

`symbol` 的格式是 `sh` + 代码（上交所）或 `sz` + 代码（深交所）。`scale=240` 是日线，`datalen` 控制返回条数。

优点：免费、不限流（实测连续调用几百次没问题）、JSON 格式直接解析。

缺点：只有价格数据（开盘、最高、最低、收盘、成交量），没有 PE/PB 估值数据——需要自己配合其他数据源计算。

### Yahoo Finance API

最适合：**拉取国际指数和美股 ETF 的历史数据**

Yahoo 的图表 API 覆盖全球市场：

```
https://query1.finance.yahoo.com/v8/finance/chart/
  ^GSPC          # 标普500
  ?range=5y      # 5年数据
  &interval=1mo  # 月度
```

国内指数用 `000300.SS`（沪深 300）、`399006.SZ`（创业板指）这样的格式。

优点：覆盖全球市场、数据时间长（可拉 10 年+）、免费。

缺点：需要挂梯子才能访问；返回数据格式比 Sina 复杂一些。

## 用 Python 自动拉数据：一个实际例子

光有工具还不够，关键是要能**自动化**。下面是一个用 Python + Sina API 拉取 ETF 月度 K 线的脚本——这段代码你可以直接跑，也可以改成定时任务：

```python
import json
import urllib.request

def fetch_kline(symbol: str, datalen: int = 60) -> list:
    """
    从 Sina Finance 拉取 ETF 月度 K 线数据
    symbol: 'sh510300' 或 'sz159915'
    datalen: 返回多少条月线数据
    """
    url = (
        "https://money.finance.sina.com.cn"
        "/quotes_service/api/json_v2.php/"
        "CN_MarketData.getKLineData"
        f"?symbol={symbol}&scale=240&ma=no&datalen={datalen}"
    )
    req = urllib.request.Request(url, headers={
        "User-Agent": "Mozilla/5.0"
    })
    with urllib.request.urlopen(req, timeout=10) as resp:
        return json.loads(resp.read().decode("utf-8"))

# 拉取沪深 300 ETF 最近 60 个月的数据
data = fetch_kline("sh510300", datalen=60)

# 最近一条就是最新数据
latest = data[-1]
print(f"日期: {latest['day']}, 收盘: {latest['close']}")

# 算一下当前价格在 60 个月中的分位
prices = [float(d["close"]) for d in data]
current = prices[-1]
percentile = sum(1 for p in prices if p < current) / len(prices) * 100
print(f"当前价格处于近 5 年 {percentile:.0f}% 分位")
```

这段代码的核心逻辑就三步：拉数据、解析 JSON、算分位。运行结果大概是这样的：

```
日期: 2026-06-25, 收盘: 5.048
当前价格处于近5年 92% 分位
```

沪深 300 ETF 当前价格在近 5 年 92% 分位——这意味着过去 5 年里，只有 8% 的时间价格比现在高。价格分位不等于估值分位（价格包含了利润增长），但可以作为快速参考。

## 自己算估值分位数：需要什么数据

上面只算了价格分位数。要算**估值分位数**（PE/PB 历史分位），还需要另一个数据源——指数或 ETF 的 PE/PB 时间序列。

乌龟量化和集思录会展示这个数据，但如果你想自己算，最靠谱的方式是从两个地方分别拉数据再合并：

1. **Sina API 拉价格数据**（月度 K 线）
2. **中证指数官网或乌龟量化拉估值数据**（PE/PB 时间序列）

或者更简单的方式——直接用乌龟量化的数据。它虽然没有 API，但它的估值历史数据是可以导出 CSV 的。导出后用 pandas 处理：

```python
import pandas as pd

# 假设从乌龟量化导出了 CSV
df = pd.read_csv("hs300_pe_history.csv", columns=["date", "pe"])

# 当前 PE 在历史中的分位
current_pe = df["pe"].iloc[-1]
percentile = (df["pe"] < current_pe).sum() / len(df) * 100
print(f"当前 PE: {current_pe:.1f}, 历史{percentile:.0f}%分位")
```

## 数据获取的几个坑

踩过的坑，帮你省时间：

**坑一：不同平台的 PE 口径不同。** 同一个指数，乌龟量化显示 PE 12 倍，中证官网可能显示 13 倍。差异来自计算方法——TTM PE（滚动 12 个月利润）、静态 PE（去年年报利润）、预测 PE（分析师预期利润）。比较时用同一个平台的同一个口径，不要跨平台混用。

**坑二：科创板的 PE 经常失真。** 科创 50 很多成分股还在亏损——PE 显示负数或异常高值（几百倍）。这时候 PE 没参考意义，要看 PS（市销率）或 PB。之前在 [PE 深入](/posts/valuation/03-pe-deep-dive) 里讲过这个问题。

**坑三：Yahoo Finance 需要梯子。** 在国内直连大概率超时。解决方式：通过代理访问，或者用 Sina API 替代（Sina 只覆盖 A 股和部分港股）。

**坑四：数据更新有延迟。** 大多数免费平台的估值数据是 T+1 或 T+2 更新。如果你在周三查到「当前 PE 12 倍」，这个数据可能是周一或周二的。对于长期投资决策来说，1-2 天的延迟无所谓。

## 一个实用的估值数据获取方案

综合上面的分析，如果你只想快速上手，我的建议是：

| 需求                          | 推荐方案                    |
| ----------------------------- | --------------------------- |
| 快速看一眼某指数估不值钱      | 乌龟量化，30 秒搞定         |
| 看 ETF 溢折价和实时数据       | 集思录                      |
| 查指数编制规则和成分股        | 中证指数官网                |
| 自动化拉价格数据写分析        | Sina Finance API + Python   |
| 拉美股/国际指数数据           | Yahoo Finance API（需梯子） |
| 深度估值分析（PE 分位数趋势） | 乌龟量化导出 CSV + Python   |

核心原则：**先用免费工具建立直觉，再用代码自动化提升效率。** 不需要一步到位搞一套完美的数据系统——从乌龟量化开始就够了。等你发现自己每周都要手动查好几只 ETF 的数据、想自动化的时候，再上 Python 也不迟。
