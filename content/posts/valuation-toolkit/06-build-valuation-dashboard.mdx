## 6 篇内容，一个表格

前面五篇分别讲了估值数据的获取、利率对估值的影响、如何识别估值陷阱、多因子评分模型、以及从估值判断到买卖决策的完整链条。

把这些知识落地成一个可执行的动作，最终产物很简单：**一张每周更新的估值追踪表。**

这篇就讲怎么搭出来。三个版本，从最轻量到全自动，按你的需求选。

## 版本一：手动版（每周 10 分钟）

适合：刚开始用估值思维做投资、不想折腾代码的人。

工具：乌龟量化 + 一个 Excel/Notion 表格。

每周一次，打开乌龟量化，依次查你的持仓 ETF：

| ETF          | 主力指标 | 当前值 | 近 5 年分位 | 上周分位 | 趋势 |
| ------------ | -------- | ------ | ----------- | -------- | ---- |
| 红利低波 ETF | PB       | 0.65   | 35%         | 33%      | →    |
| 科创 50 ETF  | PS       | 4.2    | 75%         | 72%      | ↑    |
| 沪深 300 ETF | PE       | 12.5   | 52%         | 50%      | →    |
| 纳指 100 ETF | PE       | 28     | 68%         | 65%      | ↑    |

手动版的核心价值不在于「精确」，在于**建立每周看估值的习惯**。大多数人只有在想买卖的时候才去看估值——但这时候情绪已经介入了，估值数据成了「给自己的决定找理由」的工具。

每周看一次的好处是：你在情绪平静的时候建立了一个「基准认知」。等到真正想操作的时候，你已经有了一个持续更新的估值背景——不容易被短期涨跌牵着走。

## 版本二：半自动版（Python + Excel）

适合：会一点 Python、想提升效率的人。

核心思路：用 Python 自动拉数据 + 算分位数，输出到 Excel，手动看结果做决策。

```python
import json
import urllib.request
from datetime import datetime

# 你的持仓 ETF 配置
# 每只 ETF 指定代码、名称、主力指标类型和权重
PORTFOLIO = [
    {"symbol": "sh563020", "name": "红利低波ETF",
     "type": "value", "code": "563020"},
    {"symbol": "sh588000", "name": "科创50ETF",
     "type": "growth", "code": "588000"},
    {"symbol": "sh510300", "name": "沪深300ETF",
     "type": "broad", "code": "510300"},
    {"symbol": "sh513100", "name": "纳指100ETF",
     "type": "growth_us", "code": "513100"},
]

def fetch_kline(symbol: str, months: int = 60) -> list:
    """从 Sina Finance 拉月度 K 线"""
    url = (
        "https://money.finance.sina.com.cn"
        "/quotes_service/api/json_v2.php/"
        "CN_MarketData.getKLineData"
        f"?symbol={symbol}&scale=240&ma=no&datalen={months}"
    )
    req = urllib.request.Request(url, headers={
        "User-Agent": "Mozilla/5.0"
    })
    with urllib.request.urlopen(req, timeout=10) as resp:
        return json.loads(resp.read().decode("utf-8"))

def calc_price_percentile(symbol: str) -> dict:
    """计算当前价格在近 5 年的分位"""
    data = fetch_kline(symbol, 60)  # 60 个月 ≈ 5 年
    if not data:
        return {"error": "no data"}

    prices = [float(d["close"]) for d in data]
    current = prices[-1]
    percentile = sum(1 for p in prices if p < current) / len(prices) * 100

    return {
        "date": data[-1]["day"],
        "price": current,
        "percentile_5y": round(percentile, 1),
        "data_points": len(prices),
    }

# 遍历持仓，生成追踪表
report = []
for etf in PORTFOLIO:
    result = calc_price_percentile(etf["symbol"])
    if "error" not in result:
        report.append({
            "name": etf["name"],
            **result,
            "note": ""  # 留给你手动填写备注
        })

# 打印报告
print(f"估值追踪报告 - {datetime.now().strftime('%Y-%m-%d')}")
print("-" * 60)
for r in report:
    flag = "⚠ 高估" if r["percentile_5y"] > 70 else (
           "✓ 低估" if r["percentile_5y"] < 30 else "— 正常")
    print(
        f"{r['name']:12s} "
        f"价格:{r['price']:>8.3f}  "
        f"5年分位:{r['percentile_5y']:>5.1f}%  "
        f"{flag}"
    )
```

运行后输出大概是这样的：

```
估值追踪报告 - 2026-06-26
------------------------------------------------------------
红利低波ETF    价格:    1.081  5年分位: 38.5%  — 正常
科创50ETF      价格:    2.178  5年分位: 95.2%  ⚠ 高估
沪深300ETF     价格:    5.048  5年分位: 89.1%  ⚠ 高估
纳指100ETF     价格:    2.208  5年分位: 72.3%  ⚠ 高估
```

这段代码拉的是**价格分位**，不是 PE/PB 分位。要算真正的估值分位数，需要额外拉 PE/PB 时间序列数据（来源：乌龟量化导出 CSV 或中证指数官网下载）。

价格分位和估值分位的区别：价格包含了利润增长的影响。如果 EPS 翻倍了但股价只涨了 50%，价格分位高但估值分位可能低。价格分位更适合作为**快速扫描**工具，估值分位适合做**精细分析**。

## 版本三：全自动版（定时运行 + 结果推送）

适合：持有 5 只以上 ETF、想彻底自动化的人。

在版本二的基础上，加两件事：

**一、用 Hermes 的 cron job 定时执行脚本。** 每周一早上自动拉数据、生成报告、推送结果。你只需要每周一收到一条消息，花 2 分钟扫一眼。

**二、加入估值评分计算。** 把上一篇 [多因子评分](/posts/valuation-toolkit/04-multi-factor-score) 的逻辑嵌入脚本，每只 ETF 直接输出综合评分。

全自动版的完整流程：

```
每周一早上 9:00
    ↓
cron job 触发 Python 脚本
    ↓
脚本拉取所有持仓 ETF 的价格数据
    ↓
计算价格分位数 + （如果数据源可用）估值分位数
    ↓
生成评分报告
    ↓
推送到微信
    ↓
你花 2 分钟看结果，决定是否操作
```

这个方案的核心价值不是「更准确」，是**强制纪律**。当追踪变成自动化流程，你就不会因为忙、因为懒、因为情绪不好而跳过每周的估值检查。而持续追踪是估值投资策略生效的前提——你不可能对不了解的东西做出好决策。

## 追踪体系的正确打开方式

不管你选哪个版本，有几个原则要遵守：

**原则一：评分变化不等于交易信号。** 评分从 40 变到 35，不意味着你必须立刻加仓。评分的意义在于提供一个**背景认知**——让你在操作的时候知道「估值环境是什么样的」。具体操作还需要结合 [买卖决策](/posts/valuation-toolkit/05-from-valuation-to-decision) 篇的仓位检查和趋势过滤。

**原则二：关注趋势变化，而不是单次评分。** 评分从 80 连续三周降到 65，比单次看到 65 更值得关注——这说明估值环境在系统性改善，可能正在从高估区向合理区过渡。反过来，评分从 25 连续三周升到 40，可能说明「低估」正在被修复（价格在涨）。

**原则三：极端值才值得行动。** 评分低于 15 或高于 85 时，考虑调整仓位。30-70 之间的波动，大概率不值得操作——频繁交易的手续费和摩擦成本会侵蚀你的收益。

**原则四：每周看一次，不要每天看。** 估值是长期变量，日波动没有参考意义。每天看估值不仅浪费精力，还容易因为短期波动做出情绪化操作。

## 这篇的核心结论

估值追踪体系的搭建分三个层次：

- **手动版**：乌龟量化 + Excel，每周 10 分钟，适合入门
- **半自动版**：Python 拉数据 + 算分位，输出评分表，适合进阶
- **全自动版**：cron job 定时执行 + 结果推送，适合持有 5 只以上 ETF 的人

无论哪个版本，核心原则：

- **每周看一次，不每天看**
- **关注趋势变化，不关注单次波动**
- **极端值才行动，中间值就持有**
- **追踪的意义是建立背景认知，不是寻找交易信号**
