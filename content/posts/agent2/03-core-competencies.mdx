# Agent 开发工程师的核心能力模型

上一篇我们聊了 Agent 开发工程师这个岗位。你可能想问：这个岗位到底需要什么能力？和我现在的技能有什么关系？如何系统提升？

这篇我们来构建一个能力模型，帮你理清学习方向。

## 六大核心能力

Agent 开发工程师需要的能力可以分成六个领域：AI/ML 基础、Prompt 工程、Agent 架构、工具开发、工程化能力、产品思维。

这六个能力不是孤立的，而是相互支撑的。AI/ML 基础是底层，支撑其他所有能力。Prompt 工程和工具开发是核心技能，直接决定 Agent 的效果。Agent 架构是系统设计能力，决定整体方案的可行性。工程化能力是保障，让 Agent 能可靠运行。产品思维是顶层，确保 Agent 真正解决用户问题。

让我逐一展开。

## AI/ML 基础能力

你不需要成为 AI 研究员，但需要理解 LLM 的基本原理和能力边界。为什么？因为如果你不理解 LLM 能做什么、不能做什么，就会设计出不切实际的 Agent，要么期待过高，要么浪费潜力。

理解 LLM 的本质很重要。LLM 本质上是"下一个词预测器"，它基于训练数据中的模式来生成文本。这决定了它的优势和局限。

LLM 擅长语言理解和生成、知识检索和整合、推理和规划（在上下文中）、代码生成和理解。但不擅长精确计算（可能出错）、实时信息（训练截止日期后）、长期记忆（超出上下文窗口）、执行物理操作。

理解这些，你才能合理设计 Agent。比如需要精确计算时，应该让 Agent 调用计算器工具而不是自己算。需要最新信息时，应该让 Agent 搜索网络而不是依赖训练数据。

```typescript
// 理解 LLM 的能力边界
class AgentCapabilityBoundary {
  async analyzeMarketTrend(query: string) {
    // ❌ 不好的设计：让 LLM 直接回答实时问题
    const result = await this.llm.ask(`当前${query}的市场趋势是什么？`);
    // LLM 可能给出过时信息

    // ✅ 好的设计：让 LLM 分析工具获取的实时数据
    const data = await this.tools.searchWeb(query);
    const analysis = await this.llm.ask(`
      以下是最新数据：
      ${JSON.stringify(data)}

      请分析这些数据的趋势和意义。
    `);
    return analysis;
  }
}
```

理解 Token 也很重要。Token 是 LLM 计费的基本单位，理解 Token 有助于成本优化。中文大约 1 个汉字 1-2 个 Token，英文大约 1 个单词 1 个 Token。你应该学会估算输入输出 Token 数，优化 Prompt 以减少消耗，选择合适的模型平衡成本和质量。

```typescript
// 成本优化示例
class TokenOptimizer {
  // ❌ 浪费 Token 的 Prompt
  wastefulPrompt = `
    你是一个代码分析专家，你有丰富的经验...
    [省略 500 字的背景介绍]

    请分析以下代码：
    ${code}
  `;

  // ✅ 精简的 Prompt
  efficientPrompt = `
    角色：代码分析专家
    任务：分析代码质量（复杂度、漏洞、性能、规范）
    代码：
    ${code}
  `;

  estimateTokens(text: string): number {
    const chineseChars = (text.match(/[\u4e00-\u9fa5]/g) || []).length;
    const englishWords = text.split(/\s+/).filter((w) => w).length;
    return Math.ceil(chineseChars * 1.5 + englishWords * 1);
  }
}
```

## Prompt 工程能力

Prompt 工程是 Agent 开发的核心技能。为什么？因为 Prompt 是你与 LLM 沟通的唯一方式，好的 Prompt 能让 LLM 准确理解任务、产生稳定输出、减少错误。

写 Prompt 像写代码，但比代码更微妙。代码是确定的，Prompt 有一定的模糊性。代码有明确的语法，Prompt 更像自然语言交流。需要清晰、完整、无歧义。

基础 Prompt 技巧包括：清晰的指令编写、充分的上下文设置、明确的角色定义、严格的格式约束。

```typescript
// 基础 Prompt 技巧
const basicPrompt = `
## 角色
你是一个代码审查专家，擅长发现潜在问题。

## 任务
审查以下代码，找出质量问题。

## 代码
${code}

## 输出要求
1. 列出发现的问题
2. 每个问题标注严重程度（高/中/低）
3. 提供具体的改进建议

## 输出格式
使用 JSON 格式：
{
  "issues": [
    {
      "severity": "高|中|低",
      "location": "文件名:行号",
      "description": "问题描述",
      "suggestion": "改进建议"
    }
  ]
}
`;
```

高级 Prompt 技巧包括：思维链（让 LLM 展示推理过程）、少样本学习（提供示例）、自一致性（多次采样取共识）、思维树（探索多个推理路径）。

```typescript
// 思维链示例
const chainOfThoughtPrompt = `
你是一个代码分析 Agent。分析代码时，请按以下步骤思考：

## 思考步骤
1. 理解问题：这段代码的目的是什么？
2. 分析逻辑：代码是如何实现目的的？
3. 识别问题：可能存在什么问题？
4. 验证问题：如何确认这些问题？
5. 给出结论：总结发现和建议

## 代码
${code}

请按上述步骤逐步分析。
`;

// 少样本学习示例
const fewShotPrompt = `
以下是如何分析代码复杂度的示例：

示例 1：
代码：function a() { if (x) { if (y) { return z; } } }
分析：嵌套深度 3，复杂度中等

示例 2：
代码：function b() { for (i) { for (j) { for (k) { ... } } } }
分析：嵌套深度 3，多层循环，复杂度高

现在请分析：
代码：${code}
分析：
`;
```

Agent 专用的 Prompt 技巧包括：工具调用提示（让 Agent 知道何时使用工具）、多轮对话管理（维护对话上下文）、错误恢复提示（出错时如何恢复）、输出格式控制（确保可解析）。

## Agent 架构能力

Agent 架构能力是设计整个系统的能力。为什么重要？因为单个 Prompt 无法解决复杂问题，你需要设计完整的系统架构：Agent 如何组织？工具如何集成？记忆如何管理？如何确保安全和可控？

最常用的是 ReAct 架构（Reasoning + Acting）。Agent 循环执行：思考当前情况、决定采取行动、执行行动、观察结果、继续循环直到完成。

```typescript
// ReAct Agent 实现
class ReActAgent {
  async run(task: string): Promise<string> {
    const steps: ReActStep[] = [];
    let isComplete = false;

    while (!isComplete) {
      // 思考阶段
      const thought = await this.think(task, steps);

      if (thought.includes("Task complete")) {
        isComplete = true;
        break;
      }

      // 行动阶段
      const action = await this.decideAction(thought);

      // 观察阶段
      const observation = await this.executeAction(action);

      steps.push({ thought, action, observation });

      if (steps.length > 20) {
        throw new Error("Max steps exceeded");
      }
    }

    return this.generateAnswer(steps);
  }

  private async think(task: string, history: ReActStep[]): Promise<string> {
    const prompt = `
      任务: ${task}

      历史:
      ${this.formatHistory(history)}

      请思考下一步该做什么。
    `;

    return await this.llm.generate(prompt);
  }
}
```

更复杂的是多 Agent 协作架构。不同 Agent 扮演不同角色，通过协作完成复杂任务。比如代码分析系统可以有：研究员 Agent（收集信息）、分析员 Agent（分析代码）、报告员 Agent（生成报告）。

```typescript
// 多 Agent 协作
class MultiAgentSystem {
  private agents: Map<string, Agent>;

  constructor() {
    this.agents = new Map([
      ["researcher", new ResearcherAgent()],
      ["analyzer", new AnalyzerAgent()],
      ["reporter", new ReporterAgent()],
    ]);
  }

  async analyzeRepository(repoUrl: string): Promise<Report> {
    // 研究员收集信息
    const info = await this.agents.get("researcher")!.run({
      task: "收集仓库信息",
      url: repoUrl,
    });

    // 分析员分析代码
    const analysis = await this.agents.get("analyzer")!.run({
      task: "分析代码质量",
      repositoryInfo: info,
    });

    // 报告员生成报告
    const report = await this.agents.get("reporter")!.run({
      task: "生成分析报告",
      analysis: analysis,
    });

    return report;
  }
}
```

## 工具开发能力

工具是 Agent 与外部世界交互的接口。工具开发能力为什么重要？因为 LLM 本身能力有限，需要通过工具扩展能力。工具质量直接决定 Agent 能做什么。

好的工具有几个特征：接口清晰（名称、描述、参数、返回值）、健壮性强（错误处理、重试机制）、性能优化（缓存、异步）、安全可控（参数验证、权限控制）。

```typescript
// 工具开发最佳实践
class GitHubRepoTool {
  definition = {
    name: "clone_github_repo",
    description: "克隆 GitHub 仓库到本地，用于代码分析",
    parameters: {
      type: "object",
      properties: {
        url: {
          type: "string",
          description: "GitHub 仓库 URL",
        },
        branch: {
          type: "string",
          description: "分支名称，默认 main",
          default: "main",
        },
      },
      required: ["url"],
    },
  };

  async execute(args: { url: string; branch?: string }) {
    // 参数验证
    if (!this.isValidGitHubUrl(args.url)) {
      return {
        success: false,
        error: "Invalid GitHub URL",
        code: "INVALID_URL",
      };
    }

    try {
      // 执行克隆
      const localPath = await this.cloneRepo(args.url, args.branch);

      return {
        success: true,
        data: {
          localPath,
          fileCount: await this.countFiles(localPath),
          languages: await this.detectLanguages(localPath),
        },
      };
    } catch (error) {
      // 错误处理
      return {
        success: false,
        error: error.message,
        code: "CLONE_FAILED",
      };
    }
  }

  private isValidGitHubUrl(url: string): boolean {
    return url.startsWith("https://github.com/") && url.split("/").length >= 4;
  }

  private async cloneRepo(
    url: string,
    branch: string = "main"
  ): Promise<string> {
    // 实际克隆逻辑
    const repoName = url.split("/").pop();
    const localPath = `/tmp/${repoName}`;
    await exec(`git clone -b ${branch} ${url} ${localPath}`);
    return localPath;
  }
}
```

工具设计需要考虑：接口设计（参数应该如何定义）、错误处理（如何优雅地失败）、性能优化（如何提高效率）、安全考虑（如何防止滥用）。

## 工程化能力

工程化能力是让 Agent 可靠运行的能力。为什么重要？因为原型 Agent 和生产 Agent 是两回事。原型可能不稳定、成本高、难维护，生产 Agent 需要稳定、可控、可优化。

测试是基础。Agent 测试与传统软件测试不同，因为 Agent 行为有随机性。你需要设计评估数据集（覆盖各种场景）、定义评估指标（成功率、准确率、成本）、建立回归测试（防止退化）。

```typescript
// Agent 测试框架
class AgentEvaluator {
  async evaluate(
    agent: Agent,
    testCases: TestCase[]
  ): Promise<EvaluationReport> {
    const results: TestResult[] = [];

    for (const testCase of testCases) {
      const startTime = Date.now();
      const result = await agent.run(testCase.input);
      const latency = Date.now() - startTime;

      results.push({
        testCase,
        result,
        evaluation: {
          passed: this.evaluateResult(result, testCase),
          latency,
          cost: result.cost,
        },
      });
    }

    return this.generateReport(results);
  }

  private evaluateResult(result: AgentResult, testCase: TestCase): boolean {
    // 检查输出是否符合预期
    if (testCase.expectedOutput) {
      const similarity = this.calculateSimilarity(
        result.output,
        testCase.expectedOutput
      );
      if (similarity < 0.8) return false;
    }

    // 检查是否使用了正确的工具
    if (testCase.expectedTools) {
      const usedTools = result.toolsUsed.map((t) => t.name);
      for (const expectedTool of testCase.expectedTools) {
        if (!usedTools.includes(expectedTool)) return false;
      }
    }

    // 检查成本和延迟
    if (testCase.maxCost && result.cost > testCase.maxCost) return false;
    if (testCase.maxLatency && result.latency > testCase.maxLatency)
      return false;

    return true;
  }
}
```

监控和运维是保障。你需要监控性能指标（响应时间、成功率）、追踪执行过程（决策路径、工具调用）、控制成本（Token 消耗、API 调用）、及时告警（异常检测、性能退化）。

## 产品思维与软技能

产品思维是确保 Agent 真正解决用户问题的能力。为什么重要？因为技术能力再强，如果解决的不是真实需求，就是浪费。Agent 开发需要你站在用户角度思考：用户遇到什么问题？Agent 如何帮助？体验如何设计？

```typescript
// 产品思维：从用户角度设计
const userCenteredDesign = {
  // ❌ 技术导向
  techFocused: {
    features: ["支持 50+ 种工具", "使用最新 GPT-4", "多轮对话"],
  },

  // ✅ 用户导向
  userFocused: {
    problems: [
      "用户需要频繁切换系统",
      "用户不知道如何表达需求",
      "用户担心 AI 出错",
    ],

    solutions: ["自动识别意图，减少交互", "提供智能引导", "关键操作前确认"],

    metrics: ["任务完成率 > 90%", "平均交互次数 < 3", "用户满意度 > 4.5/5"],
  },
};
```

沟通协作也很重要。Agent 开发往往需要跨团队协作：与产品对齐需求、与后端对接接口、与设计优化体验。你需要能够清晰地表达技术方案、理解业务需求、编写有效文档。

## 能力发展路径

这六个能力不是同等重要，也不是都需要精通。建议按优先级发展：

第一阶段（0-3个月），掌握 AI/ML 基础和 Prompt 工程。这两个是基础，不掌握后面无法进行。重点学习 LLM 原理、Token 概念、Prompt 编写技巧。

第二阶段（3-9个月），深入工具开发和 Agent 架构。这两个是核心技能，直接决定你能否构建有用的 Agent。重点学习工具设计、ReAct 架构、多 Agent 协作。

第三阶段（9-18个月），强化工程化能力。这是从原型到生产的关键。重点学习测试、监控、优化、运维。

第四阶段（18个月+），培养产品思维。这是从工程师到架构师的提升。重点学习用户研究、需求分析、系统设计。

不同背景的人可能有不同的起点。前端工程师可能工具开发入手快，后端工程师可能架构设计有优势，AI/ML 工程师可能 Prompt 工程上手快。但最终都需要全面发展，才能成为优秀的 Agent 开发工程师。

## 核心权衡

Agent 开发工程师的能力模型有一个特点：广度优先，深度次之。

这与传统软件工程不同。传统工程师可能在某个领域很专精（比如数据库专家、前端专家），Agent 开发工程师需要在多个领域都有一定基础：理解 AI、会写 Prompt、能设计架构、能开发工具、懂工程化、有产品思维。

广度的优势是能独立完成完整的 Agent 系统，不依赖其他角色。但代价是难以在某个领域达到顶级深度。

另一个权衡是理论学习 vs 实践经验。Agent 技术还在快速发展，理论体系不完善。过度研究理论可能跟不上技术演进，但完全依赖实践可能缺乏系统性思考。

建议是：快速建立基础认知（通过阅读和课程），然后通过大量实践积累经验，定期总结提炼形成自己的知识体系。
