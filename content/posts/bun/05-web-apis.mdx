上一篇我们直面了 Bun 的 Node.js 兼容性短板。这篇换个角度——Bun 不只是"兼容 Node.js"，它还有自己的增强：内置了浏览器标准的 Web API。

这个设计选择的含义是：你在浏览器里用 `fetch`、`WebSocket`、`FormData` 这些 API 写代码，到了 Bun 里不用装任何库，直接用。而在 Node.js 里，要么装第三方库，要么用实验性的内置实现。

用一个实时聊天服务来感受这种差异。

## 场景：一个实时聊天服务

我们要实现的功能：

1. HTTP API：注册、登录、获取聊天记录
2. WebSocket：实时消息推送
3. 文件上传：发送图片
4. 热重载：开发时代码修改自动生效

### Node.js 需要的依赖

```
express        — HTTP 服务器
ws             — WebSocket
multer         — 文件上传
dotenv         — 环境变量
nodemon        — 热重载
jsonwebtoken   — JWT
```

### Bun 需要的依赖

```
无（全部内置）
```

Bun 内置了 HTTP 服务器、WebSocket、FormData 解析、环境变量加载、热重载。JWT 不是 Web 标准 API，还是需要一个库（或者自己用 `crypto` 实现）。

## fetch API：不只是"能发请求"

### Node.js 的 fetch 现状

Node.js 18 开始内置了 `fetch`，但它基于 `undici` 库，有一些已知限制：

```typescript
// Node.js 22 的 fetch
const response = await fetch("https://api.example.com/users");
const data = await response.json();

// ⚠️ 一些限制：
// - 不自动处理 cookies
// - 某些边缘场景下的行为和浏览器不一致
// - AbortController 的某些用法支持有限
// - 重定向行为有差异
```

### Bun 的 fetch

Bun 的 `fetch` 用 Zig 原生实现，行为更接近浏览器：

```typescript
// Bun 的 fetch — 和浏览器行为一致
const response = await fetch("https://api.example.com/users", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({ name: "张三", role: "user" }),
});

const data = await response.json();

// 支持所有标准特性
const controller = new AbortController();
setTimeout(() => controller.abort(), 5000);

const res = await fetch("https://slow-api.example.com/data", {
  signal: controller.signal,
});
```

Bun 的 `fetch` 还支持一些实用的高级特性：

```typescript
// 自动重试（Bun 扩展）
const res = await fetch("https://api.example.com/data", {
  retry: 3,
  retryDelay: 1000,
});

// 自动跟随重定向，可以自定义最大次数
const res = await fetch("https://example.com/old-url", {
  redirect: "follow",
});

// TLS 配置（Node.js 的 fetch 不方便做）
const res = await fetch("https://self-signed.example.com", {
  tls: { rejectUnauthorized: false },
});
```

### 在聊天服务中使用

获取用户信息的 API 调用：

```typescript
// 调用外部认证服务验证 token
async function verifyToken(token: string) {
  const res = await fetch(`https://auth.internal/api/verify`, {
    method: "POST",
    headers: { Authorization: `Bearer ${token}` },
  });

  if (!res.ok) throw new Error("Token invalid");
  return res.json();
}
```

不需要装 `axios` 或 `node-fetch`，`fetch` 就是全局可用的。

## WebSocket：原生支持，不需要 ws 库

这是 Bun 相比 Node.js 优势最明显的 Web API。

### Node.js + ws 的写法

```typescript
// 先装 ws
// npm install ws @types/ws
import { WebSocketServer } from "ws";
import http from "http";

const server = http.createServer();
const wss = new WebSocketServer({ server });

const clients = new Set();

wss.on("connection", (ws) => {
  clients.add(ws);

  ws.on("message", (data) => {
    // 广播给所有客户端
    for (const client of clients) {
      if (client !== ws && client.readyState === 1) {
        client.send(data.toString());
      }
    }
  });

  ws.on("close", () => {
    clients.delete(ws);
  });
});

server.listen(3000);
```

这里有几个烦人的点：`ws` 是第三方库需要安装，`data` 是 `Buffer` 类型需要 `.toString()` 转换，还需要手动管理 `readyState` 检查。

### Bun 的 WebSocket 写法

```typescript
const clients = new Set();

const server = Bun.serve({
  port: 3000,
  fetch(req, server) {
    // 升级为 WebSocket 连接
    const success = server.upgrade(req);
    if (success) return; // WebSocket 升级成功，不返回 Response
    return new Response("Expected WebSocket", { status: 400 });
  },
  websocket: {
    open(ws) {
      clients.add(ws);
    },
    message(ws, message) {
      // message 直接是字符串，不需要 toString()
      for (const client of clients) {
        if (client !== ws) {
          client.send(message);
        }
      }
    },
    close(ws) {
      clients.delete(ws);
    },
  },
});
```

差别在哪？

1. **HTTP 和 WebSocket 是同一个服务器**。不需要先建 `http.createServer()` 再挂 `WebSocketServer`，`Bun.serve()` 同时处理两种协议
2. **message 直接是字符串**。不需要手动 `toString()`
3. **不需要检查 readyState**。Bun 保证 `send` 时连接是活的
4. **不需要装任何依赖**

### WebSocket 的发布/订阅

Bun 的 WebSocket 还内置了发布/订阅模式，不需要自己实现广播逻辑：

```typescript
const server = Bun.serve({
  fetch(req, server) {
    const url = new URL(req.url);
    const channel = url.searchParams.get("channel") || "general";
    server.upgrade(req, { data: { channel } });
    return new Response("Expected WebSocket", { status: 400 });
  },
  websocket: {
    open(ws) {
      // 订阅频道
      ws.subscribe(ws.data.channel);
    },
    message(ws, message) {
      // 发布到频道（包括自己）
      ws.publish(ws.data.channel, message);
    },
    close(ws) {
      // 取消订阅
      ws.unsubscribe(ws.data.channel);
    },
  },
});
```

`subscribe`/`publish`/`unsubscribe` 是内置方法。一个聊天室就是一个 channel，用户加入时 subscribe，发消息时 publish，离开时 unsubscribe。不需要自己维护 `clients` 集合。

这在多用户聊天场景下特别方便：

```typescript
// 用户加入 "project-alpha" 聊天室
ws.subscribe("project-alpha");

// 用户发消息，所有在该聊天室的人都能收到
ws.publish(
  "project-alpha",
  JSON.stringify({
    user: "张三",
    content: "新版部署完成了",
    timestamp: Date.now(),
  })
);
```

## FormData 和文件上传

### Node.js 处理文件上传

```typescript
// npm install multer
import multer from "multer";
import express from "express";

const upload = multer({ dest: "uploads/" });
const app = express();

app.post("/upload", upload.single("avatar"), (req, res) => {
  const file = req.file;
  // file.path, file.size, file.mimetype
  res.json({ url: `/uploads/${file.filename}` });
});
```

需要装 `multer`（或者 `formidable`、`busboy`），还要配置存储路径和文件名策略。

### Bun 处理文件上传

```typescript
Bun.serve({
  port: 3000,
  async fetch(req) {
    const url = new URL(req.url);

    if (url.pathname === "/upload" && req.method === "POST") {
      const formData = await req.formData();
      const file = formData.get("avatar") as File;

      // File 是 Web 标准 API
      const buffer = await file.arrayBuffer();
      await Bun.write(`uploads/${file.name}`, buffer);

      return Response.json({ url: `/uploads/${file.name}` });
    }

    return new Response("Not found", { status: 404 });
  },
});
```

不需要任何第三方库。`req.formData()` 是标准的 Web API，返回的 `File` 对象也是标准的。`Bun.write()` 是 Bun 特有的文件写入方法，比 `fs.writeFile` 更简洁。

### 在聊天服务中发图片

```typescript
// 上传图片
async fetch(req) {
  const formData = await req.formData()
  const image = formData.get("image") as File

  // 验证文件类型和大小
  if (!image.type.startsWith("image/")) {
    return Response.json({ error: "Only images allowed" }, { status: 400 })
  }
  if (image.size > 5 * 1024 * 1024) {
    return Response.json({ error: "Max 5MB" }, { status: 400 })
  }

  // 保存文件
  const filename = `img_${Date.now()}_${image.name}`
  await Bun.write(`uploads/${filename}`, image)

  // 通过 WebSocket 广播图片消息
  server.publish("general", JSON.stringify({
    type: "image",
    url: `/uploads/${filename}`,
    user: "张三",
  }))

  return Response.json({ url: `/uploads/${filename}` })
}
```

## Bun.serve()：内置 HTTP 服务器

前面已经用了很多次 `Bun.serve()`，来系统地看看它和 Express 的差异。

### 路由对比

```typescript
// Express
app.get("/users", listUsers);
app.post("/users", createUser);
app.get("/users/:id", getUser);
app.put("/users/:id", updateUser);
app.delete("/users/:id", deleteUser);

// Bun.serve() — 没有路由器，需要自己处理
Bun.serve({
  fetch(req) {
    const url = new URL(req.url);

    if (url.pathname === "/users" && req.method === "GET")
      return listUsers(req);
    if (url.pathname === "/users" && req.method === "POST")
      return createUser(req);
    if (url.pathname.startsWith("/users/") && req.method === "GET") {
      const id = url.pathname.split("/")[2];
      return getUser(req, id);
    }
    // ...更多路由

    return new Response("Not found", { status: 404 });
  },
});
```

`Bun.serve()` 没有内置路由器。对于简单项目，`if/else` 够用。但路由多了之后会变得难以维护。

解决方案：用支持 Bun 的路由框架。Hono 是目前最好的选择：

```typescript
import { Hono } from "hono";

const app = new Hono();

app.get("/users", listUsers);
app.post("/users", createUser);
app.get("/users/:id", getUser);
app.put("/users/:id", updateUser);
app.delete("/users/:id", deleteUser);

Bun.serve({
  fetch: app.fetch,
});
```

Hono 的路由 API 和 Express 几乎一样，但它原生支持 Bun（以及 Deno、Cloudflare Workers），不依赖 Node.js 的 `http` 模块。

### 中间件对比

Express 的中间件模式是 `(req, res, next) => {}`，Bun 原生没有中间件概念。但用 Hono 就有了：

```typescript
import { Hono } from "hono";
import { bearerAuth } from "hono/bearer-auth";
import { logger } from "hono/logger";
import { cors } from "hono/cors";

const app = new Hono();

app.use("*", logger());
app.use("/api/*", cors());
app.use(
  "/api/*",
  bearerAuth({ verifyToken: async (token) => token === process.env.API_KEY })
);

app.get("/api/users", async (c) => {
  // 不需要 Express 的 res.json()，直接 return
  return c.json({ users: await db.query("SELECT * FROM users") });
});
```

## 热重载：开发体验

### Node.js 的热重载

```bash
# 装 nodemon
npm install -D nodemon

# 启动
npx nodemon src/index.ts
```

nodemon 监听文件变化后**重启整个进程**。如果你的应用有数据库连接、WebSocket 连接，重启意味着所有连接断开重连。

### Bun 的热重载

Bun 提供两种模式：

```bash
# --watch：文件变化时重启进程（类似 nodemon，但更快）
bun run --watch src/index.ts

# --hot：文件变化时只替换修改的模块，不重启进程
bun run --hot src/index.ts
```

`--hot` 是 Bun 独有的。它不会重启整个进程，而是热替换变化的模块。这意味着：

- 数据库连接不断开
- WebSocket 连接保持
- 内存中的状态不丢失
- 替换速度比 `--watch` 快 10 倍以上

在开发聊天服务时，`--hot` 的体验非常好——修改路由逻辑后，浏览器端不需要刷新，WebSocket 连接不断，立即看到效果。

```
Node.js + nodemon：文件变化 → 重启进程（1-2s） → 重新连接 DB → 恢复状态
Bun --watch：      文件变化 → 重启进程（0.1s） → 完成
Bun --hot：        文件变化 → 热替换模块（0.01s） → 完成，状态保留
```

## 其他内置 Web API

Bun 还内置了一些不那么起眼但很实用的 Web API：

**`URL` 和 `URLSearchParams`**：全局可用，不需要 `import { URL } from "url"`

**`ReadableStream` / `WritableStream`**：Streams API，在处理大文件上传或流式响应时有用

**`TextEncoder` / `TextDecoder`**：全局可用，不需要 `import { TextEncoder } from "util"`

**`crypto.subtle`**：Web Crypto API，可以用标准接口做加密操作

```typescript
// 使用 Web Crypto API 做哈希（不需要 crypto 模块）
const hashBuffer = await crypto.subtle.digest(
  "SHA-256",
  new TextEncoder().encode("hello world")
);
const hashArray = Array.from(new Uint8Array(hashBuffer));
const hashHex = hashArray.map((b) => b.toString(16).padStart(2, "0")).join("");
```

**`structuredClone`**：深拷贝，不需要 `JSON.parse(JSON.stringify())` 或 `lodash.cloneDeep`

## 依赖对比：同一个聊天服务

| 功能        | Node.js 依赖        | Bun 内置                    |
| ----------- | ------------------- | --------------------------- |
| HTTP 服务器 | express             | `Bun.serve()`               |
| 路由        | express.Router      | Hono（推荐）                |
| WebSocket   | ws                  | 内置                        |
| 文件上传    | multer / formidable | `req.formData()`            |
| HTTP 客户端 | axios / node-fetch  | `fetch`                     |
| 环境变量    | dotenv              | 自动加载                    |
| 热重载      | nodemon             | `--hot`                     |
| 深拷贝      | lodash.cloneDeep    | `structuredClone`           |
| Cookie 解析 | cookie-parser       | `req.headers.get("cookie")` |

一个中等复杂度的项目，Node.js 版可能有 15-20 个依赖，Bun 版可能只有 3-5 个。
