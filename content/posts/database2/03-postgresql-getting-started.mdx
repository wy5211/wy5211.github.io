上一篇文章我们学习了 SQL 语言基础，知道了如何用 SQL 查询和操作数据。但光有语法还不够，我们需要一个实际的数据库来实践。在众多关系型数据库中，PostgreSQL 以其强大的功能、优异的性能和开源免费的特点，成为了越来越多企业的首选。

为什么选择 PostgreSQL？相比 MySQL 它有哪些优势？让我们先看看 PostgreSQL 的核心优势。

PostgreSQL 提供完整的 ACID 支持，保证数据的原子性、一致性、隔离性和持久性。它有丰富的数据类型，包括 JSONB、ARRAY、UUID、CIDR、HSTORE 等独特类型。强大的扩展性支持 PostGIS 地理空间数据扩展、pgcrypto 加密函数库、全文搜索、外部数据包装器等功能。MVCC 并发控制实现多版本并发，读写不阻塞，适合高并发场景。高度标准兼容，完全支持 SQL:2016 标准，跨平台支持 Linux、Windows、macOS。

PostgreSQL 和 MySQL 的对比很有意思。PostgreSQL 在复杂查询、数据类型、并发性能、扩展性方面表现更优秀，但学习曲线较陡。MySQL 在易用性方面更友好，学习曲线较平，适合简单的 Web 应用和快速开发。

如果不确定怎么选择，可以参考这些建议：复杂业务、数据分析、地理信息、高并发选择 PostgreSQL；简单 Web 应用、快速开发、广为人知选择 MySQL。

PostgreSQL 有几个核心概念需要理解。数据库集群是一个 PostgreSQL 服务器实例管理多个数据库，物理上共享同一数据目录。数据库是逻辑上的数据隔离，不同数据库之间表名可以重复。Schema 是数据库内部的命名空间，默认 schema 是 public，同一数据库内不同 schema 的表名可以重复。表是数据的实际存储单元，属于某个 schema。它们的层次关系是：Cluster 包含 Database，Database 包含 Schema，Schema 包含 Table。

现在让我们搭建 PostgreSQL 开发环境。使用 Docker Compose 是最简单、最推荐的方式。

首先创建项目目录：

```bash
mkdir postgres-learning
cd postgres-learning
```

然后创建 Docker Compose 配置文件：

```yaml
# docker-compose.yml
version: "3.8"

services:
  postgres:
    image: postgres:16-alpine
    container_name: postgres_dev
    environment:
      POSTGRES_USER: dev
      POSTGRES_PASSWORD: dev123
      POSTGRES_DB: dev_db
    ports:
      - "5432:5432"
    volumes:
      - postgres_data:/var/lib/postgresql/data
      - ./init.sql:/docker-entrypoint-initdb.d/init.sql
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U dev"]
      interval: 10s
      timeout: 5s
      retries: 5

  pgadmin:
    image: dpage/pgadmin4:latest
    container_name: pgadmin_dev
    environment:
      PGADMIN_DEFAULT_EMAIL: admin@example.com
      PGADMIN_DEFAULT_PASSWORD: admin
    ports:
      - "5050:80"
    depends_on:
      - postgres

volumes:
  postgres_data:
```

创建初始化脚本，这个脚本会在容器首次启动时自动执行：

```sql
-- init.sql
CREATE DATABASE ecommerce;

\c ecommerce;

CREATE TABLE users (
  id SERIAL PRIMARY KEY,
  email VARCHAR(255) UNIQUE NOT NULL,
  password_hash VARCHAR(255) NOT NULL,
  full_name VARCHAR(100),
  is_verified BOOLEAN DEFAULT FALSE,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

INSERT INTO users (email, password_hash, full_name) VALUES
  ('user1@example.com', 'hash1', '张三'),
  ('user2@example.com', 'hash2', '李四'),
  ('user3@example.com', 'hash3', '王五');

GRANT ALL PRIVILEGES ON DATABASE ecommerce TO dev;
```

启动服务：

```bash
docker-compose up -d
docker-compose logs -f postgres
docker-compose ps
```

连接数据库有几种方式。使用 psql 命令行工具：

```bash
docker exec -it postgres_dev psql -U dev -d dev_db
psql -h localhost -p 5432 -U dev -d dev_db
```

使用 pgAdmin 可视化工具，打开浏览器访问 http://localhost:5050，使用 admin@example.com / admin 登录，然后添加服务器连接，Host 填 postgres，Port 填 5432，Username 填 dev，Password 填 dev123。

psql 有很多常用命令。连接数据库后，可以用 \l 列出所有数据库，\c database_name 切换数据库，\dt 列出当前数据库的所有表，\d table_name 查看表结构，\du 列出所有用户，\conninfo 显示当前连接信息，\i file.sql 执行 SQL 文件，\o file.txt 将查询结果输出到文件，\q 退出。

数据库操作包括创建数据库、切换数据库、查看数据库大小、删除数据库、重命名数据库：

```sql
CREATE DATABASE myapp;
\c myapp;

SELECT
  pg_database.datname,
  pg_size_pretty(pg_database_size(pg_database.datname)) AS size
FROM pg_database
ORDER BY pg_database_size(pg_database.datname) DESC;

DROP DATABASE myapp;
ALTER DATABASE myapp RENAME TO newname;
```

表操作包括创建表、查看表结构、重命名表、删除表：

```sql
CREATE TABLE products (
  id SERIAL PRIMARY KEY,
  name VARCHAR(255) NOT NULL,
  price NUMERIC(10, 2) NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

\d products
ALTER TABLE products RENAME TO items;
DROP TABLE products;
DROP TABLE IF EXISTS products;
```

数据操作包括插入、查询、更新、删除：

```sql
INSERT INTO users (email, password_hash, full_name)
VALUES ('newuser@example.com', 'hash', '赵六');

SELECT * FROM users;
SELECT id, email, full_name FROM users;
SELECT * FROM users WHERE is_verified = TRUE;

UPDATE users
SET is_verified = TRUE
WHERE id = 1;

DELETE FROM users WHERE id = 1;
```

PostgreSQL 的权限系统非常灵活。创建用户：

```sql
CREATE USER app_user WITH PASSWORD 'secure_password';
CREATE USER readonly WITH PASSWORD 'readonly_pass';
CREATE USER admin WITH PASSWORD 'admin_pass' SUPERUSER;
```

授权：

```sql
GRANT CONNECT ON DATABASE ecommerce TO app_user;
GRANT USAGE ON SCHEMA public TO app_user;
GRANT SELECT, INSERT, UPDATE, DELETE ON ALL TABLES IN SCHEMA public TO app_user;
GRANT SELECT ON ALL TABLES IN SCHEMA public TO readonly;
GRANT USAGE, SELECT ON ALL SEQUENCES IN SCHEMA public TO app_user;

ALTER DEFAULT PRIVILEGES IN SCHEMA public
  GRANT SELECT, INSERT, UPDATE, DELETE ON TABLES TO app_user;
```

撤销权限：

```sql
REVOKE DELETE ON ALL TABLES IN SCHEMA public FROM app_user;
ALTER USER admin WITH NOSUPERUSER;
```

角色管理：

```sql
CREATE ROLE developer;
GRANT developer TO app_user;
GRANT ALL PRIVILEGES ON DATABASE ecommerce TO developer;
\du
```

PostgreSQL 的配置文件位于容器的 /var/lib/postgresql/data 目录。主配置文件 postgresql.conf 包含连接设置、内存设置、WAL 设置、日志设置：

```bash
listen_addresses = '*'
max_connections = 100
shared_buffers = 256MB
effective_cache_size = 1GB
work_mem = 16MB
maintenance_work_mem = 128MB
wal_buffers = 16MB
checkpoint_completion_target = 0.9
logging_collector = on
log_directory = 'log'
log_filename = 'postgresql-%Y-%m-%d.log'
log_statement = 'mod'
log_duration = on
```

认证配置文件 pg_hba.conf 控制客户端认证：

```bash
local   all   all   trust
local   all   all   md5
host    all   all   0.0.0.0/0   md5
host    ecommerce  app_user  192.168.1.0/24  md5
```

在高并发场景下，直接创建大量数据库连接会消耗很多资源，这时需要使用连接池。PgBouncer 是流行的连接池工具。在 docker-compose.yml 中添加 PgBouncer 服务：

```yaml
pgbouncer:
  image: edoburu/pgbouncer:latest
  container_name: pgbouncer_dev
  environment:
    DB_HOST: postgres
    DB_PORT: 5432
    DB_USER: dev
    DB_PASSWORD: dev123
    DATABASES: dev_db
    POOL_MODE: transaction
    MAX_CLIENT_CONN: 200
    DEFAULT_POOL_SIZE: 25
  ports:
    - "6432:6432"
  depends_on:
    - postgres
```

应用连接 PgBouncer 而不是直接连接 PostgreSQL：

```bash
psql -h localhost -p 6432 -U dev -d dev_db
```

在生产环境部署时，有一些安全性建议：使用强密码、限制监听地址、使用 SSL/TLS 加密连接、定期更新数据库版本、最小权限原则。性能优化建议：根据服务器内存调整 shared_buffers、定期执行 VACUUM ANALYZE、监控慢查询、合理设置索引。高可用建议：配置主从复制 Streaming Replication、定期备份、监控数据库状态。

开发环境建议使用 Docker Compose 快速搭建、数据持久化、方便团队协作。版本控制方面，使用迁移工具如 Prisma Migrate、Flyway，将 Schema 变更纳入版本控制，分离开发测试生产环境配置。数据管理方面，使用种子数据进行测试，定期清理测试数据，使用 .env 文件管理敏感信息。

监控与日志很重要。启用慢查询日志：

```sql
log_min_duration_statement = 1000
```

查看当前活动连接：

```sql
SELECT
  pid,
  usename,
  application_name,
  client_addr,
  state,
  query_start,
  state_change,
  query
FROM pg_stat_activity
WHERE state != 'idle';
```

忘记密码怎么办？先停止容器，然后以单用户模式启动，修改密码，最后正常启动容器：

```bash
docker-compose stop postgres
docker run -it --rm \
  -v postgres_data:/var/lib/postgresql/data \
  postgres:16-alpine bash
psql -U dev
ALTER USER dev WITH PASSWORD 'new_password';
\q
docker-compose start postgres
```

查看 PostgreSQL 版本：

```sql
SELECT version();
psql --version
SHOW server_version;
```

数据库连接失败时，检查清单包括：检查容器是否运行、检查端口是否被占用、查看日志、测试连接、检查防火墙。

清理测试数据有几种方式：删除所有数据保留表结构、删除并重建表、删除整个 schema：

```sql
TRUNCATE TABLE users, orders, products RESTART IDENTITY CASCADE;
DROP TABLE IF EXISTS users CASCADE;
CREATE TABLE users (...);
DROP SCHEMA public CASCADE;
CREATE SCHEMA public;
GRANT ALL ON SCHEMA public TO dev;
```

导出导入数据：

```bash
pg_dump -h localhost -U dev -d dev_db > dev_db.sql
pg_dump -h localhost -U dev -d dev_db -t users > users.sql
psql -h localhost -U dev -d dev_db < dev_db.sql
docker exec -i postgres_dev pg_dump -U dev dev_db > backup.sql
docker exec -i postgres_dev psql -U dev dev_db < backup.sql
```

PostgreSQL 还有一些特有功能值得了解。JSONB 类型可以高效存储和查询 JSON 数据，数组类型支持原生数组操作，枚举类型可以定义自定义枚举，全文搜索支持强大的文本搜索功能。
