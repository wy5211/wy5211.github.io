# Vite 渐进式学习计划

## 背景

Vite 改变了前端构建的游戏规则——利用浏览器原生 ESM，启动速度比 Webpack 快 10-100 倍。但现有的 Vite 学习路径往往有两个问题：一是内容分散——每篇文章独立举例，读者不知道在实际项目中如何配合；二是缺少真实项目驱动——堆砌 API 和配置，但不知道什么时候该用。

这个系列的目标是：**用一个真实的组件库项目从零到发布，串联起 Vite 的核心技能**。

### 技术栈

- Vite 5.x
- React + TypeScript
- Rollup（生产构建）
- Vitest（测试）

### 主线案例：开发 SparkUI 组件库

贯穿全系列的真实案例——一个类似 Ant Design 的 React 组件库，包含 Button、Input、Modal 等常用组件。案例从初始化项目开始，随着文章逐步演进：配置开发环境 → 添加 Storybook → 优化构建 → 发布到 npm。

选择这个案例的原因：

- 涉及多种场景（库模式、多入口、CSS 处理），适合展示 Vite 的能力
- 有真实的性能需求（包体积、构建速度）
- 从简单到复杂，自然需要 Vite 各项技术

## 学习阶段划分

### 基础篇（第 1-3 篇）：从原理到实践

让读者理解 Vite 为什么快，以及如何初始化一个符合最佳实践的项目。第 1 篇解释核心原理，第 2 篇搭建 SparkUI 项目，第 3 篇优化开发体验。

### 构建篇（第 4-5 篇）：从开发到生产

第 4 篇配置生产构建，第 5 篇处理 CSS 和静态资源。这是组件库最关键的两个环节。

### 进阶篇（第 6-8 篇）：走向专业

第 6 篇开发自定义插件，第 7 篇集成测试，第 8 篇从 Webpack 迁移实战。

## 每篇文章大纲

### 01. 为什么选择 Vite

**文件名**：`01-why-vite.mdx`

**核心问题**：Webpack 已经够用了，为什么还需要 Vite？

**主线案例**：一个 Webpack 项目启动要 30 秒，切换到 Vite 后 1 秒

**内容要点**：

1. **从一个真实场景开始**：你刚吃完午饭回来，Webpack 还在编译...
2. **ESM vs CommonJS**：为什么 Webpack 必须打包，Vite 不需要
3. **依赖预构建**：如何处理 node_modules 的 CommonJS 包
4. **HMR 原理**：Vite 如何做到毫秒级热更新
5. **双模式架构**：开发 No-bundle，生产 Rollup 打包
6. **适用场景**：什么时候选 Vite，什么时候还是 Webpack

**与前篇的关联**：无（系列第一篇）
