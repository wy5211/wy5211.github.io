## 前言

开发环境的极速体验固然重要，但生产构建的质量直接影响用户体验。本文将深入探讨 Vite 生产构建的优化策略，帮助你打造高性能的生产环境。

## Rollup 配置深入

### 理解 Rollup

Vite 生产环境使用 Rollup 打包，而非开发环境的 esbuild。

```
开发环境：esbuild（极速）
生产环境：Rollup（功能丰富）
```

**为什么生产用 Rollup？**

1. **更强大的 Tree Shaking**
2. **更灵活的代码分割**
3. **更成熟的插件生态**
4. **更好的输出格式支持**

### 基础配置

```typescript
// vite.config.ts
export default defineConfig({
  build: {
    // 构建目标
    target: "es2015",

    // 输出目录
    outDir: "dist",

    // 静态资源目录
    assetsDir: "assets",

    // Source Map
    sourcemap: false,

    // 压缩器
    minify: "terser",

    // CSS 代码分割
    cssCodeSplit: true,

    // 库模式
    lib: false,

    // SSR 模式
    ssr: false,

    // Rollup 选项
    rollupOptions: {
      // 输入配置
      input: {
        main: "./src/main.tsx",
      },

      // 输出配置
      output: {
        // Chunk 文件名
        chunkFileNames: "assets/js/[name]-[hash].js",

        // 入口文件名
        entryFileNames: "assets/js/[name]-[hash].js",

        // 资源文件名
        assetFileNames: "assets/[ext]/[name]-[hash].[ext]",
      },
    },
  },
});
```

## 代码分割策略

### 手动分割

```typescript
export default defineConfig({
  build: {
    rollupOptions: {
      output: {
        manualChunks: {
          // 基础库
          "react-vendor": ["react", "react-dom"],

          // 路由
          router: ["react-router-dom"],

          // UI 库
          ui: ["antd"],

          // 工具库
          utils: ["lodash", "axios", "dayjs"],
        },
      },
    },
  },
});
```

### 动态分割函数

```typescript
export default defineConfig({
  build: {
    rollupOptions: {
      output: {
        manualChunks(id) {
          // node_modules 中的包
          if (id.includes("node_modules")) {
            // 按包名分割
            const match = id.match(/node_modules\/([^/]+)/);

            if (match) {
              const packageName = match[1];

              // 大型库单独分割
              const largePackages = ["react", "react-dom", "antd", "lodash"];
              if (largePackages.includes(packageName)) {
                return `vendor-${packageName}`;
              }

              // 其他小包合并
              return "vendor";
            }
          }

          // 业务代码分割
          if (id.includes("/src/pages/")) {
            return "pages";
          }

          if (id.includes("/src/components/")) {
            return "components";
          }
        },
      },
    },
  },
});
```

### 路由懒加载

```typescript
// src/App.tsx
import { lazy, Suspense } from 'react';
import { BrowserRouter, Routes, Route } from 'react-router-dom';

// 懒加载页面组件
const Home = lazy(() => import('./pages/Home'));
const About = lazy(() => import('./pages/About'));

function App() {
  return (
    <BrowserRouter>
      <Suspense fallback={<div>Loading...</div>}>
        <Routes>
          <Route path="/" element={<Home />} />
          <Route path="/about" element={<About />} />
          <Route path="/dashboard" element={<Dashboard />} />
        </Routes>
      </Suspense>
    </BrowserRouter>
  );
}
```

### 组件懒加载

```typescript
// src/components/HeavyComponent.tsx
```

## Tree Shaking

### 原理

Tree Shaking 通过静态分析移除未使用的代码。

```javascript
// utils.js
export function used() {
  console.log("used");
}

export function unused() {
  console.log("unused");
}

// main.js
import { used } from "./utils";

used();

// Tree Shaking 后
function used() {
  console.log("used");
}

used();
// unused 被移除 ✅
```

### 配置 Tree Shaking

```typescript
export default defineConfig({
  build: {
    rollupOptions: {
      output: {
        // Tree Shaking 配置
        treeshake: {
          // 模块副作用（默认自动检测）
          moduleSideEffects: false,

          // 属性访问
          propertyReadSideEffects: true,

          // try-catch 优化
          tryCatchDeoptimization: false,
        },
      },
    },
  },
});
```

### package.json sideEffects

```json
// package.json
{
  "name": "my-app",
  "sideEffects": false,

  // 或者指定有副作用的文件
  "sideEffects": ["*.css", "*.scss", "./src/polyfills.js"]
}
```

### 避免副作用

```typescript
// ❌ 有副作用（Tree Shaking 失效）
export const config = {
  init: () => console.log("init"),
};

config.init(); // 立即执行

// ✅ 无副作用（Tree Shaking 生效）
export function initConfig() {
  console.log("init");
}
```

## 压缩配置

### Terser 配置

```typescript
export default defineConfig({
  build: {
    minify: "terser",

    terserOptions: {
      compress: {
        // 移除 console
        drop_console: true,
        drop_debugger: true,

        // 移除特定函数调用
        pure_funcs: ["console.log", "console.info"],

        // 死码消除
        dead_code: true,

        // 内联函数
        inline: 2,
      },
      format: {
        // 移除注释
        comments: false,

        // 美化代码（调试用）
        beautify: false,
      },
      mangle: {
        // 混淆属性名
        properties: {
          regex: /^_/,
        },
      },
    },
  },
});
```

### esbuild 压缩

```typescript
export default defineConfig({
  build: {
    minify: "esbuild", // 更快但功能较少

    // esbuild 不支持复杂的压缩选项
    // 但速度比 Terser 快 20-40 倍
  },
});
```

### 保留特定代码

```typescript
export default defineConfig({
  build: {
    minify: "terser",
    terserOptions: {
      compress: {
        // 保留特定调用
        pure_funcs: [], // 不移除任何函数

        // 自定义保留
        pure_funcs: ["console.warn", "console.error"],
      },
    },
  },
});
```

## 资源优化

### CSS 优化

```typescript
export default defineConfig({
  build: {
    // CSS 代码分割
    cssCodeSplit: true,

    // CSS 压缩
    cssMinify: true,

    rollupOptions: {
      output: {
        // CSS chunk 文件名
        assetFileNames: (assetInfo) => {
          if (assetInfo.name?.endsWith(".css")) {
            return "assets/css/[name]-[hash][extname]";
          }
          return "assets/[ext]/[name]-[hash][extname]";
        },
      },
    },
  },
});
```

### 图片优化

### 字体优化

```typescript
export default defineConfig({
  build: {
    rollupOptions: {
      output: {
        assetFileNames: (assetInfo) => {
          // 字体文件单独处理
          if (/\.(woff2?|eot|ttf|otf)$/.test(assetInfo.name || "")) {
            return "assets/fonts/[name]-[hash][extname]";
          }
          return "assets/[ext]/[name]-[hash][extname]";
        },
      },
    },
  },
});
```

## 常见问题

### Q1: Chunk 体积过大？

A: 使用更细粒度的代码分割：

```typescript
manualChunks(id) {
  if (id.includes('node_modules')) {
    const match = id.match(/node_modules\/([^/]+)/);
    return match ? `vendor-${match[1]}` : 'vendor';
  }
}
```

### Q2: Tree Shaking 不生效？

A: 检查以下几点：

1. package.json 设置 `"sideEffects": false`
2. 使用 ESM 格式（非 CommonJS）
3. 避免副作用代码

### Q3: 构建时间过长？

A: 优化策略：

1. 使用 esbuild 压缩
2. 减少代码分割粒度
3. 关闭 Source Map
4. 增加 Node.js 内存

## 总结

Vite 生产构建优化要点：

1. **代码分割**：合理的分包策略
2. **Tree Shaking**：移除未使用代码
3. **压缩**：Terser/esbuild 压缩
4. **资源优化**：图片、字体、CSS
5. **分析工具**：打包体积分析
6. **性能平衡**：构建速度 vs 产物质量
