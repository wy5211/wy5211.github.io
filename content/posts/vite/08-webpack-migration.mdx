前7篇我们用 Vite 从零搭建了 SparkUI 组件库。但你手头可能有老的 Webpack 项目——启动要 30 秒，HMR 要 5 秒，每次改代码都要等。

今天我们把一个 Create React App（CRA）项目迁移到 Vite，解决常见的问题。

## 迁移前的评估

不是所有项目都适合迁移。先评估：

| 你的项目              | 适合迁移？  |
| --------------------- | ----------- |
| 纯前端（React/Vue）   | ✅ 是       |
| TypeScript/JSX        | ✅ 是       |
| 需要支持 IE11         | ❌ 否       |
| 复杂的 Webpack Loader | ⚠️ 需要改造 |
| 大量 `require()`      | ⚠️ 需要改造 |

如果符合 ✅ 或 ⚠️，可以继续。

## 开始迁移

### 1. 备份项目

```bash
# 备份
git checkout -b vite-migration
cp package.json package.json.bak
```

### 2. 卸载 CRA 依赖

```bash
npm uninstall react-scripts @testing-library/react @testing-library/jest-dom @testing-library/user-event web-vitals
```

### 3. 安装 Vite

```bash
npm install -D vite @vitejs/plugin-react
npm install -D tailwindcss postcss autoprefixer  # 如果用 Tailwind
```

### 4. 修改 package.json 脚本

```json
{
  "scripts": {
    "dev": "vite",
    "build": "vite build",
    "preview": "vite preview",
    "lint": "eslint src --ext js,jsx,ts,tsx",
    "test": "vitest"
  }
}
```

### 5. 移动 index.html

CRA 的 `index.html` 在 `public/` 目录，Vite 要放在根目录：

```bash
mv public/index.html .
```

修改 `index.html`：

```html
<!-- CRA -->
<script defer="defer" src="./index.js"></script>

<!-- Vite -->
<script type="module" src="/src/main.tsx"></script>
```

### 6. 删除 CRA 配置文件

```bash
rm -f craco.config.js
rm -f .env.production
rm -f .env.development
```

## 配置 Vite

```typescript
// vite.config.ts
import { defineConfig } from "vite";
import react from "@vitejs/plugin-react";

export default defineConfig({
  plugins: [react()],
  server: {
    port: 3000,
    open: true,
  },
  build: {
    outDir: "build",
  },
});
```

## 处理环境变量

CRA 的环境变量自动以 `REACT_APP_` 开头，Vite 需要手动配置。

### 旧代码

```javascript
// CRA
const apiUrl = process.env.REACT_APP_API_URL;
```

### 新代码

```typescript
// Vite
const apiUrl = import.meta.env.VITE_API_URL;
```

批量替换 `.env` 文件：

```bash
# .env.development
REACT_APP_API_URL=http://localhost:8080/api
# 改成
VITE_API_URL=http://localhost:8080/api
```

## 处理 require()

Vite 不支持 `require()`，需要改成 `import`。

### 旧代码

```javascript
const utils = require("./utils");
const { debounce } = require("lodash");
```

### 新代码

```typescript
import * as utils from "./utils";
import { debounce } from "lodash";
```

### 全局替换

```bash
# 用 sed 或编辑器批量替换
# require('./x') → import('./x')
# require('x') → import 'x'
```

## 处理动态导入

CRA 用 `require()` 动态加载，Vite 用 `import()`。

### 旧代码

```javascript
const module = require(`./languages/${language}.json`);
```

### 新代码

```typescript
const module = await import(`./languages/${language}.json`);
```

注意：Vite 的动态导入有限制——路径必须是相对路径或绝对路径，不能完全是变量。

## 处理 Polyfill

CRA 自带大量 Polyfill，Vite 需要手动配置。

### 安装 Polyfill

```bash
npm install @vitejs/plugin-legacy terser
```

### 配置

```typescript
// vite.config.ts
import legacy from "@vitejs/plugin-legacy";

export default defineConfig({
  plugins: [
    react(),
    legacy({
      targets: ["defaults", "not IE 11"],
    }),
  ],
});
```

如果需要支持 IE11，考虑用 `@vitejs/plugin-legacy` 的现代模式：

```typescript
legacy({
  targets: ["IE 11"],
  additionalLegacyPolyfills: ["regenerator-runtime/runtime"],
  modernPolyfills: true,
});
```

## 处理 CSS Modules

CRA 用 `[name]__[local]___[hash:base64:5]` 命名，Vite 用 `[name]_[local]_[hash:base64:5]`。

如果要保持一致，配置：

```typescript
// vite.config.ts
export default defineConfig({
  css: {
    modules: {
      generateScopedName: "[name]__[local]___[hash:base64:5]",
    },
  },
});
```

## 处理路径别名

CRA 的路径别名在 `jsconfig.json`，Vite 要在 `vite.config.ts` 配置。

### 旧配置

```json
// jsconfig.json
{
  "compilerOptions": {
    "baseUrl": "src",
    "paths": {
      "@/*": ["*"],
      "@components/*": ["components/*"]
    }
  }
}
```

### 新配置

```typescript
// vite.config.ts
import path from "path";

export default defineConfig({
  resolve: {
    alias: {
      "@": path.resolve(__dirname, "./src"),
      "@components": path.resolve(__dirname, "./src/components"),
    },
  },
});
```

`tsconfig.json` 也要同步：

```json
{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "@/*": ["./src/*"],
      "@components/*": ["./src/components/*"]
    }
  }
}
```

| -------- | ----- | ------- |
| 启动时间 | 15s   | 1s      |

| 构建时间 | 45s   | 20s     |
| 产物体积 | 350KB | 280KB   |

## 常见问题

### 1. 找不到模块？

检查路径别名是否在 `vite.config.ts` 和 `tsconfig.json` 都配置了。

### 2. 环境变量 undefined？

确保变量名以 `VITE_` 开头。

### 3. 构建后白屏？

检查 `index.html` 的 `script src` 是否正确。

### 4. 某些 Loader 找不到等价物？

可以用 `vite-plugin-commonjs` 转换 CommonJS 模块，或者寻找社区替代插件。

## 渐进式迁移

如果不敢一次性迁移，可以先用 Vite 开发，保持 Webpack 构建：

```bash
# 开发用 Vite
npm run dev  # vite

# 构建用 Webpack
npm run build  # webpack
```

这样可以提前体验 Vite 的开发速度，等稳定后再迁移构建。
