## 前言

性能优化是工程化永恒的主题。Vite 本身已经很快，但在大型项目中，我们仍然需要进一步优化。本文将系统介绍 Vite 的性能优化策略。

## 开发环境优化

### 依赖预构建优化

```typescript
// vite.config.ts
export default defineConfig({
  optimizeDeps: {
    // 明确指定需要预构建的依赖
    include: [
      "react",
      "react-dom",
      "react-router-dom",
      "lodash",
      "axios",
      "antd",
    ],

    // 排除不需要预构建的包
    exclude: ["your-local-package", "already-esm-package"],

    // 强制重新预构建（开发调试用）
    // force: true
  },
});
```

### 文件监听优化

```typescript
export default defineConfig({
  server: {
    watch: {
      // 使用轮询（Docker/WSL 环境）
      usePolling: false,

      // 轮询间隔
      interval: 100,

      // 忽略大型目录
      ignored: [
        "**/node_modules/**",
        "**/.git/**",
        "**/dist/**",
        "**/build/**",
        "**/.vscode/**",
        "**/.idea/**",
        "**/coverage/**",
      ],
    },
  },
});
```

### 禁用不必要的功能

```typescript
export default defineConfig({
  server: {
    // 禁用 HMR（某些场景）
    hmr: false,

    // 禁用文件监听
    watch: null,
  },

  // 禁用 Source Map
  build: {
    sourcemap: false,
  },
});
```

### 使用 esbuild 替代 Babel

```typescript
export default defineConfig({
  esbuild: {
    // JSX 自动运行时
    jsx: "automatic",
    jsxImportSource: "react",

    // 自定义 JSX

```

## 生产构建优化

### 构建速度优化

```typescript
export default defineConfig({
  build: {
    // 使用 esbuild 压缩（更快）
    minify: "esbuild",

    // 禁用 Source Map
    sourcemap: false,

    // 关闭 CSS 代码分割（减少构建时间）
    cssCodeSplit: false,

    // 减少并行构建
    parallel: true,

    // 限制 Chunk 大小警告
    chunkSizeWarningLimit: 1000,
  },
});
```

### Terser 优化

```typescript
export default defineConfig({
  build: {
    minify: "terser",
    terserOptions: {
      compress: {
        // 减少优化时间
        sequences: false,
        join_vars: false,

        // 其他优化
        drop_console: true,
        dead_code: true,
      },
      format: {
        comments: false,
      },
    },
  },
});
```

### 代码分割优化

```typescript
export default defineConfig({
  build: {
    rollupOptions: {
      output: {
        manualChunks: {
          // 基础库
          "react-vendor": ["react", "react-dom"],

          // 大型 UI 库
          ui: ["antd"],

          // 路由
          router: ["react-router-dom"],
        },
```

## 运行时性能优化

### 懒加载策略

```typescript
// 路由懒加载
import { lazy, Suspense } from 'react';

const Home = lazy(() => import('./pages/Home'));
const About = lazy(() => import('./pages/About'));

function App() {
  return (
    <Suspense fallback={<div>Loading...</div>}>
      <Routes>
        <Route path="/" element={<Home />} />
        <Route path="/about" element={<About />} />
      </Routes>
    </Suspense>
  );
}
```

### 组件懒加载

```typescript
// 条件加载
function Dashboard() {
  const [showChart, setShowChart] = useState(false);

  return (
    <div>
      <button onClick={() => setShowChart(true)}>
        Show Chart
      </button>

      {showChart && (
        <Suspense fallback={<div>Loading chart...</div>}>
          <Chart />
        </Suspense>
      )}
    </div>
  );
}

// 动态导入
const Chart = lazy(() =>
  import('./Chart').then(module => ({
    default: module.Chart
  }))
);
```

```typescript

export default defineConfig({
```

```typescript

// src/utils/monitor.ts
export const reportWebVitals = (metric) => {
  const { name, value, id } = metric;

  // 上报到分析平台
  if (window.gtag) {
    window.gtag("event", name, {
      event_category: "Web Vitals",
      event_label: id,
      value: Math.round(name === "CLS" ? value * 1000 : value),
      non_interaction: true,
    });
  }

  // 或发送到自己的 API
  fetch("/api/analytics", {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ name, value, id }),
  });
};

// 使用
import { getCLS, getFID, getFCP, getLCP, getTTFB } from "web-vitals";

getCLS(reportWebVitals);
getFID(reportWebVitals);
getFCP(reportWebVitals);
getLCP(reportWebVitals);
getTTFB(reportWebVitals);
```

## 常见问题

### Q1: 开发环境还是慢？

A: 尝试以下优化：

1. 清除缓存：`rm -rf node_modules/.vite`
2. 减少监听文件
3. 使用 esbuild 替代 Babel
4. 升级到最新版本

### Q2: 构建产物过大？

A: 优化策略：

1. 分析打包体积
2. 代码分割
3. Tree Shaking
4. 移除未使用依赖

### Q3: 首屏加载慢？

A: 优化方案：

1. 路由懒加载
2. 组件懒加载
3. 预加载关键资源
4. CDN 加速

## 总结

Vite 性能优化要点：

1. **开发环境**：依赖预构建、文件监听优化
2. **生产构建**：压缩、代码分割、Tree Shaking
3. **运行时**：懒加载、预加载策略
4. **依赖优化**：减少体积、按需导入
5. **监控分析**：构建分析、运行时监控
6. **大型项目**：Monorepo、多入口优化
