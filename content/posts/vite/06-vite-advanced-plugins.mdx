## 前言

Vite 的强大不仅在于其核心能力，更在于丰富的插件生态。本文将介绍 Vite 的官方插件和社区插件，帮助你构建功能完善的应用。

## 官方插件

### React 插件

```typescript
// vite.config.ts
import { defineConfig } from "vite";
import react from "@vitejs/plugin-react";

export default defineConfig({
  plugins: [
    react({
      // 使用 JSX 自动运行时
      jsxImportSource: "react",

      // Babel 配置
      babel: {
        plugins: ["babel-plugin-styled-components", "babel-plugin-macros"],
      },
    }),
  ],
});
```

### Vue 插件

```typescript
import vue from "@vitejs/plugin-vue";
import vueJsx from "@vitejs/plugin-vue-jsx";
```

## 社区插件推荐

### 1. 自动导入组件

```typescript
import Components from "unplugin-vue-components/vite";
import { AntDesignVueResolver } from "unplugin-vue-components/resolvers";

export default defineConfig({
  plugins: [
    Components({
      // 组件目录
      dirs: ["src/components"],

      // 组件有效文件扩展名

      // 解析器

      // 生成类型声明文件
```

```typescript
});
```

## 插件冲突处理

### 1. 执行顺序冲突

```typescript
// 使用 enforce 控制顺序
export default defineConfig({
  plugins: [
    {
      name: "plugin-a",
      enforce: "pre", // 先执行
      transform(code, id) {
        /* ... */
      },
    },
    {
      name: "plugin-b",
      enforce: "post", // 后执行
      transform(code, id) {
        /* ... */
      },
    },
  ],
});
```

### 2. 虚拟模块冲突

```typescript
// 使用唯一的前缀
{
  name: 'my-plugin',
  resolveId(source) {
    if (source.startsWith('my-virtual:')) {
      return source;
    }
    return null;
  }
}
```

### 3. 转换冲突

```typescript
// 检查是否已处理
{
  name: 'my-plugin',
  transform(code, id) {
    // 检查代码标记
    if (code.includes('__PROCESSED__')) {
      return null;
    }

    // 处理后添加标记
    return {
      code: `// __PROCESSED__\n${processedCode}`,
      map: null
    };
  }
}
```

### 2. 条件加载

```typescript
// 根据环境加载插件
export default defineConfig(({ mode }) => {
  const isDev = mode === "development";

  return {
    plugins: [
      react(),
      isDev && mockPlugin(), // 只在开发环境
      isDev && vitestConfig(), // 只在开发环境
      mode === "production" && compressionPlugin(),
    ].filter(Boolean),
  };
});
```

### 3. 缓存优化

```typescript
{
  name: 'cached-plugin',
  cache: new Map(),

  transform(code, id) {
    const cached = this.cache.get(id);

    if (cached && cached.code === code) {
      return cached.result;
    }

    const result = transformCode(code, id);
    this.cache.set(id, { code, result });

    return result;
  }
}
```

## 常见问题

### Q1: 插件不生效？

A: 检查以下几点：

1. 插件是否正确安装
2. 是否添加到 plugins 数组
3. enforce 顺序是否正确
4. 是否在正确的模式下运行

### Q2: 多个插件冲突？

A: 调整插件顺序：

```typescript
plugins: [
  { name: "a", enforce: "pre" },
  { name: "b" },
  { name: "c", enforce: "post" },
];
```

### Q3: 如何开发跨平台插件？

A: 使用 unplugin：

```typescript
import { createUnplugin } from "unplugin";

export const unplugin = createUnplugin((options) => {
  return {
    name: "my-plugin",
    // 插件逻辑
  };
});

// Vite
export const vitePlugin = unplugin.vite;

// Webpack
export const webpackPlugin = unplugin.webpack;
```
