## 前言

微前端是大型应用架构的解决方案。Vite 对微前端提供了良好的支持。本文将深入探讨 Vite 的微前端实现方案。

## 微前端基础

### 什么是微前端

微前端是将前端应用分解为更小、更简单的块，每个块可以独立开发、测试和部署。

```
主应用（Shell）
    ├─ 子应用 A（React）
    ├─ 子应用 B（Vue）
    └─ 子应用 C（Angular）
```

### 优势

- **独立开发**：团队独立开发
- **独立部署**：按需部署
- **技术栈无关**：可以使用不同框架
- **增量升级**：逐步迁移

## 模块联邦

### 配置模块联邦

```typescript
// vite.config.ts
import federation from "@originjs/vite-plugin-federation";

export default defineConfig({
  plugins: [
    federation({
      name: "host_app",
      remotes: {
        remote_app: {
          type: "module",
          name: "remote_app",
          entry: "http://localhost:5001/assets/remoteEntry.js",
          entryGlobalName: "remote_app",
          shareScope: "default",
        },
      },
      shared: ["react", "react-dom", "react-router-dom"],
    }),
  ],
});
```

### 使用远程模块

```typescript
// src/App.tsx
import { lazy, Suspense } from 'react';

// 懒加载远程组件
const RemoteComponent = lazy(() =>
  import('remote_app/Component')
);

function App() {
  return (
    <div>
      <h1>Host App</h1>

      <Suspense fallback={<div>Loading...</div>}>
        <RemoteComponent />
      </Suspense>
    </div>
  );
}
```

### 暴露模块

```typescript
// remote/vite.config.ts
export default defineConfig({
  plugins: [
    federation({
      name: "remote_app",
      filename: "remoteEntry.js",
      exposes: {
        "./Component": "./src/components/Component",
      },
      shared: ["react", "react-dom"],
    }),
  ],
});
```

## 通信方案

### 事件总线

```typescript
// src/utils/eventBus.ts
class EventBus {
  private events: Record<string, Function[]> = {};

  on(event: string, callback: Function) {
    if (!this.events[event]) {
      this.events[event] = [];
    }
    this.events[event].push(callback);
  }

  emit(event: string, data?: any) {
    if (this.events[event]) {
      this.events[event].forEach((callback) => callback(data));
    }
  }

  off(event: string, callback: Function) {
    if (this.events[event]) {
      this.events[event] = this.events[event].filter((cb) => cb !== callback);
    }
  }
}

export const eventBus = new EventBus();
```

### Props 传递

```typescript
// 主应用
import { loadMicroApp } from "qiankun";

const app = loadMicroApp({
  name: "app1",
  entry: "//localhost:7100",
  container: "#container",
  props: {
    data: { user: "Alice" },
    onData: (data) => console.log("Received:", data),
  },
});

// 子应用
export function mount(props) {
  const { data, onData } = props;
  console.log("Received data:", data);

  // 发送数据
  onData({ result: "success" });
}
```

```typescript

export const SUPPORTED_VERSIONS = {
  app1: "^1.0.0",
  app2: "^2.0.0",

// 加载前检查版本
function checkVersion(name: string, version: string) {
  const supported = SUPPORTED_VERSIONS[name];
  return satisfies(version, supported);
}
```

### 3. 错误处理

```typescript
// 全局错误处理
```

## 常见问题

### Q1: 路由冲突？

A: 使用不同的路由前缀：

```typescript
// 主应用
<Routes>
  <Route path="/app1/*" element={<MicroApp1 />} />
  <Route path="/app2/*" element={<MicroApp2 />} />
</Routes>

// 子应用使用相对路由
<Routes>
  <Route path="/page1" element={<Page1 />} />
</Routes>
```

### Q2: 状态共享复杂？

A: 使用事件总线或状态管理：

```typescript
// 简单场景：事件总线
eventBus.emit("user:update", user);

// 复杂场景：状态管理
import { sharedStore } from "sharedStore";
```

### Q3: 性能问题？

A: 优化策略：

1. 按需加载子应用
2. 预加载关键资源
3. 使用缓存策略
