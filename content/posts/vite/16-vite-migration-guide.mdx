## 前言

从 Webpack 迁移到 Vite 可以获得更快的开发体验。本文将提供详细的迁移指南，帮助你平滑过渡到 Vite。

## 迁移评估

### 适合迁移的场景

- ✅ 新项目或小型项目
- ✅ 使用现代框架（React 18、Vue 3）
- ✅ 追求开发体验
- ✅ 团队愿意学习新技术

### 不适合迁移的场景

- ❌ 大型复杂项目（风险高）
- ❌ 大量自定义 Webpack 配置
- ❌ 依赖旧浏览器（IE）
- ❌ 紧急上线时间

### 迁移成本评估

| 项目规模           | 迁移时间 | 风险等级 |
| ------------------ | -------- | -------- |
| 小型（< 10 配置）  | 1-2 天   | 低       |
| 中型（10-30 配置） | 3-5 天   | 中       |
| 大型（> 30 配置）  | 1-2 周   | 高       |

## 配置映射

### 入口文件

```javascript
// Webpack
module.exports = {
  entry: './src/index.js',
  output: {
    path: path.resolve(__dirname, 'dist'),
    filename: 'bundle.js'
  }
};

// Vite
// 入口：index.html 中指定
<!DOCTYPE html>
<html>
  <body>
    <script type="module" src="/src/main.js"></script>
  </body>
</html>

// 输出：自动到 dist
export default defineConfig({
  build: {
    outDir: 'dist'
  }
});
```

### 环境变量

```javascript
// Webpack
const env = process.env.NODE_ENV;
const apiUrl = process.env.API_URL;

// Vite
const env = import.meta.env.MODE;
const apiUrl = import.meta.env.VITE_API_URL;
```

### 路径别名

```javascript
// Webpack
module.exports = {
  resolve: {
    alias: {
      "@": path.resolve(__dirname, "src"),
    },
  },
};

// Vite
export default defineConfig({
  resolve: {
    alias: {
      "@": "/src",
    },
  },
});
```

### 加载器

```javascript
// Webpack
module.exports = {
  module: {
    rules: [
      {
        test: /\.js$/,
        use: "babel-loader",
      },
      {
        test: /\.css$/,
        use: ["style-loader", "css-loader"],
      },
    ],
  },
};

// Vite（内置支持）
// 无需配置，开箱即用
```

### 插件映射

| Webpack Plugin       | Vite 对应方案              |
| -------------------- | -------------------------- |
| HtmlWebpackPlugin    | 内置（index.html）         |
| MiniCssExtractPlugin | build.cssCodeSplit         |
| DefinePlugin         | build.rollupOptions.output |
| CopyPlugin           | vite-plugin-static-copy    |
| ImageminPlugin       | vite-plugin-imagemin       |
| BundleAnalyzer       | rollup-plugin-visualizer   |

## 迁移步骤

### 第一步：安装依赖

```bash
# 安装 Vite
npm install -D vite @vitejs/plugin-react

# 移除 Webpack
npm uninstall webpack webpack-cli webpack-dev-server
```

### 第二步：创建配置文件

```typescript
// vite.config.ts
import { defineConfig } from "vite";
import react from "@vitejs/plugin-react";

export default defineConfig({
  plugins: [react()],

  resolve: {
    alias: {
      "@": "/src",
    },
  },

  server: {
    port: 3000,
    open: true,
  },
});
```

### 第三步：调整 HTML

```html
<!-- 移动 index.html 到根目录 -->
<!DOCTYPE html>
<html>
  <head>
    <meta charset="UTF-8" />
    <title>My App</title>
  </head>
  <body>
    <div id="root"></div>
    <!-- 模块化引入 -->
    <script type="module" src="/src/main.tsx"></script>
  </body>
</html>
```

### 第四步：更新脚本

```json
{
  "scripts": {
    "dev": "vite",
    "build": "vite build",
    "preview": "vite preview"
  }
}
```

### 第五步：处理特殊语法

```javascript
// Webpack: require.context
const modules = require.context("./dir", true, /\.js$/);

// Vite: import.meta.glob
const modules = import.meta.glob("./dir/**/*.js");

// 动态导入
const module = await import(`./dir/${name}.js`);
```

## 常见问题解决

### 问题 1：环境变量访问

```javascript
// ❌ Webpack 方式
const API_URL = process.env.API_URL;

// ✅ Vite 方式
const API_URL = import.meta.env.VITE_API_URL;
```

### 问题 2：动态导入

```javascript
// ❌ 静态分析失败
const modulePath = "./modules/" + moduleName;
const module = await import(modulePath);

// ✅ 使用模板字符串
const module = await import(`./modules/${moduleName}.js`);

// ✅ 或使用 glob
const modules = import.meta.glob("./modules/*.js");
```

### 问题 3：全局变量

```javascript
// Webpack
new webpack.DefinePlugin({
  __API__: JSON.stringify("https://api.example.com"),
});

// Vite
export default defineConfig({
  define: {
    __API__: JSON.stringify("https://api.example.com"),
  },
});
```

### 问题 4：CSS 模块

```javascript
// Webpack
import styles from "./App.module.css";

// Vite（相同）
import styles from "./App.module.css";
```

### 问题 5：Polyfills

```typescript
// vite.config.ts
import legacy from "@vitejs/plugin-legacy";

export default defineConfig({
  plugins: [
    legacy({
      targets: ["defaults", "not IE 11"],
    }),
  ],
});
```

## 复杂场景迁移

### Monorepo 迁移

```typescript
// vite.config.ts
export default defineConfig({
  resolve: {
    alias: {
      "@pkg/ui": "/packages/ui/src",
    },
  },

  optimizeDeps: {
    include: ["@pkg/ui", "@pkg/utils"],
  },
});
```

### 多入口迁移

```typescript
// Webpack
module.exports = {
  entry: {
    main: "./src/main.js",
    admin: "./src/admin.js",
  },
};

// Vite
export default defineConfig({
  build: {
    rollupOptions: {
      input: {
        main: "./src/main.html",
        admin: "./src/admin.html",
      },
    },
  },
});
```

### SSR 项目迁移

```typescript
// 客户端入口
// src/entry-client.tsx
import { hydrateRoot } from 'react-dom/client';
hydrateRoot(document, <App />);

// 服务端入口
// src/entry-server.tsx
import { renderToString } from 'react-dom/server';
export const render = (url) => renderToString(<App />);
```

## 渐进式迁移

### 并存模式

```json
// 保留 Webpack，添加 Vite
{
  "scripts": {
    "dev:webpack": "webpack serve",
    "dev:vite": "vite",
    "build:webpack": "webpack",
    "build:vite": "vite build"
  }
}
```

### 逐步替换

```typescript
// 1. 先替换开发环境
npm run dev:vite

// 2. 测试通过后替换生产环境
npm run build:vite

// 3. 移除 Webpack 配置
rm webpack.config.js
```

## 常见问题

A: 配置相同的输出：

```typescript
export default defineConfig({
  build: {
    rollupOptions: {

        chunkFileNames: "[name].js",
        assetFileNames: "[name].[ext]",
      },
    },
  },
});
```

A: 查找 Vite 替代方案：

```typescript
// Webpack: html-webpack-plugin
// Vite: index.html（内置）

// Webpack: copy-webpack-plugin
// Vite: vite-plugin-static-copy
```
