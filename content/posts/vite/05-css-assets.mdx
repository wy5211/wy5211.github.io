上篇优化了 JS 构建，体积从 2MB 降到 180KB。但 SparkUI 的样式和资源还没处理——CSS Modules 怎么配置？图片怎么优化？图标字体怎么加载？

## CSS Modules

Vite 原生支持 CSS Modules，`.module.css` 文件会被处理成：

```typescript
// Button.module.css
.button {
  background-color: #1890ff;
}
```

```typescript
// Button.tsx
import styles from './Button.module.css';

export const Button = () => {
  return <button className={styles.button}>Click</button>;
};
```

编译后：

```css
.Button_button_abc123 {
  background-color: #1890ff;
}
```

### 自定义命名模式

默认的 `[name]_[local]_[hash]` 可能太长，可以自定义：

```typescript
// vite.config.ts
export default defineConfig({
  css: {
    modules: {
      generateScopedName: "sparkui_[local]_[hash:base64:5]",
    },
  },
});
```

### 全局 CSS

非 `.module.css` 文件不会被处理，保持原样：

```css
/* global.css */
.btn {
  padding: 4px 15px;
}
```

## CSS 预处理器

Vite 原生支持 SCSS、Less、Stylus，不需要额外配置。

```bash
# 安装预处理器
npm install -D sass
```

```typescript
// vite.config.ts
export default defineConfig({
  css: {
    preprocessorOptions: {
      scss: {
        // 全局变量
        additionalData: `@import "@sparkui/theme/variables.scss";`,
      },
    },
  },
});
```

现在每个 SCSS 文件都能用 `$primary-color` 等变量了。

## Tailwind CSS

SparkUI 用 Tailwind 做基础样式：

```bash
npm install -D tailwindcss postcss autoprefixer
npx tailwindcss init -p
```

```javascript
// tailwind.config.js
module.exports = {
  content: [
    "./packages/components/**/*.{js,ts,jsx,tsx}",
    "./apps/docs/**/*.{js,ts,jsx,tsx}",
  ],
  theme: {
    extend: {},
  },
  plugins: [],
};
```

```css
/* index.css */
@tailwind base;
@tailwind components;
@tailwind utilities;
```

```typescript
// Button.tsx
export const Button = () => {
  return (
    <button className="px-4 py-2 bg-blue-500 text-white rounded">
      Click
    </button>
  );
};
```

Vite 会自动处理 Tailwind 的指令，生成最终的 CSS。

## 图片导入

Vite 支持多种图片导入方式：

```typescript
// 1. 默认：URL 导入
import logoUrl from './logo.png';
<img src={logoUrl} alt="Logo" />

// 2. 内联：base64
import logoInline from './logo.png?url'; // 等同于默认
<img src={logoInline} alt="Logo" />

// 3. 内联：原始内容
import logoRaw from './logo.png?raw';
console.log(logoRaw); // 图片的二进制数据

// 4. Worker：Web Worker
import worker from './worker.js?worker';
new Worker(worker);
```

### 图片优化

用 `vite-plugin-imagemin` 自动压缩：

```bash
npm install -D vite-plugin-imagemin
```

```typescript
// vite.config.ts
import imagemin from "vite-plugin-imagemin";

export default defineConfig({
  plugins: [
    imagemin({
      gifsicle: { optimizationLevel: 7 },
      optipng: { optimizationLevel: 7 },
      mozjpeg: { quality: 80 },
      svgo: {
        plugins: [
          { name: "removeViewBox", active: false },
          { name: "removeEmptyAttrs", active: false },
        ],
      },
    }),
  ],
});
```

构建时，所有图片会被自动压缩——JPEG 质量 80%，PNG 最大压缩，SVG 移除注释。

### WebP 转换

自动生成 WebP 版本：

```typescript
// vite.config.ts
import { defineConfig } from "vite";
import { webp } from "vite-plugin-webp";

export default defineConfig({
  plugins: [webp()],
});
```

访问 `image.png?webp` 会自动生成 WebP 版本。

## 字体处理

```typescript
// 导入字体文件
import fontUrl from "./font.woff2";

const style = `
  @font-face {
    font-family: 'CustomFont';
    src: url(${fontUrl}) format('woff2');
  }
`;
```

Vite 会把字体文件复制到 `dist/assets/` 并更新引用。

## SVG 处理

### 作为 URL

```typescript
import logoUrl from './logo.svg';
<img src={logoUrl} alt="Logo" />
```

### 作为组件

用 `vite-plugin-svgr` 把 SVG 转成 React 组件：

```bash
npm install -D vite-plugin-svgr
```

```typescript
// vite.config.ts
import svgr from "vite-plugin-svgr";

export default defineConfig({
  plugins: [
    svgr({
      svgrOptions: {
        icon: true,
      },
    }),
  ],
});
```

```typescript
// App.tsx
import Logo from './logo.svg?react';

export const App = () => {
  return <Logo width={100} height={100} />;
};
```

## CDN 上传

生产环境把静态资源上传到 CDN：

```typescript
// vite.config.ts
import { defineConfig } from "vite";
import { viteStaticCopy } from "vite-plugin-static-copy";

export default defineConfig({
  plugins: [
    viteStaticCopy({
      targets: [
        {
          src: "public/images",
          dest: "images",
        },
      ],
    }),
  ],
  build: {
    // CDN 基础路径
    assetsDir: "https://cdn.sparkui.com/assets",
  },
});
```

构建后，所有资源 URL 会变成 `https://cdn.sparkui.com/assets/xxx.png`。

## 资源命名策略

Vite 默认用 hash 命名：

```
assets/logo-abc123.png
assets/button-def456.css
```

好处是内容变了，hash 才变，缓存更有效。

自定义命名：

```typescript
// vite.config.ts
export default defineConfig({
  build: {
    rollupOptions: {
      output: {
        // JS 文件
        chunkFileNames: "js/[name]-[hash].js",
        entryFileNames: "js/[name]-[hash].js",
        // CSS 和其他资源
        assetFileNames: "[ext]/[name]-[hash].[ext]",
      },
    },
  },
});
```

输出结构：

```
dist/
├── js/
│   ├── index-abc123.js
│   └── vendor-def456.js
├── css/
│   └── index-ghi789.css
└── png/
    └── logo-jkl012.png
```
