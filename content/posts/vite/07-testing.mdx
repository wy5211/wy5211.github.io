上篇开发了自动生成文档的插件，SparkUI 的开发效率提升了。但组件写好了，怎么保证没有 bug？

今天我们集成测试：Vitest 做单元测试、Testing Library 做组件测试、Playwright 做 E2E 测试。

## 为什么选择 Vitest

Vitest 是专为 Vite 设计的测试框架，相比 Jest：

| 特性       | Jest     | Vitest                   |
| ---------- | -------- | ------------------------ |
| 启动速度   | 5-10s    | < 1s                     |
| HMR        | 不支持   | 支持                     |
| 配置       | 复杂     | 零配置（复用 Vite 配置） |
| ESM        | 需要配置 | 原生支持                 |
| Watch 模式 | 较慢     | 极快                     |

最重要的是：**Vitest 可以直接用你的 Vite 配置**，不需要重复配置。

## 安装 Vitest

```bash
npm install -D vitest @vitest/ui
```

## 配置 Vitest

```typescript
// vite.config.ts
import { defineConfig } from "vite";
import react from "@vitejs/plugin-react";

export default defineConfig({
  plugins: [react()],
  test: {
    globals: true,
    environment: "jsdom",
    setupFiles: "./src/test/setup.ts",
    coverage: {
      provider: "v8",
      reporter: ["text", "json", "html"],
    },
  },
});
```

```typescript
// src/test/setup.ts
import { expect, afterEach } from "vitest";
import { cleanup } from "@testing-library/react";
import "@testing-library/jest-dom";

// 每个测试后清理
afterEach(() => {
  cleanup();
});
```

## 单元测试

测试工具函数：

```typescript
// src/utils/format.test.ts
import { describe, it, expect } from "vitest";
import { formatCurrency, formatDate } from "./format";

describe("formatCurrency", () => {
  it("formats numbers as currency", () => {
    expect(formatCurrency(1234.56)).toBe("¥1,234.56");
  });

  it("handles zero", () => {
    expect(formatCurrency(0)).toBe("¥0.00");
  });

  it("handles negative numbers", () => {
    expect(formatCurrency(-100)).toBe("-¥100.00");
  });
});

describe("formatDate", () => {
  it("formats dates correctly", () => {
    const date = new Date("2024-01-15");
    expect(formatDate(date)).toBe("2024-01-15");
  });
});
```

运行：

```bash
npx vitest
```

## 组件测试

用 Testing Library 测试 React 组件：

```bash
npm install -D @testing-library/react @testing-library/jest-dom
```

```typescript
// src/components/Button/Button.test.tsx
import { describe, it, expect, vi } from 'vitest';
import { render, screen, fireEvent } from '@testing-library/react';
import { Button } from './Button';

describe('Button', () => {
  it('renders children correctly', () => {
    render(<Button>Click me</Button>);
    expect(screen.getByText('Click me')).toBeInTheDocument();
  });

  it('calls onClick when clicked', () => {
    const handleClick = vi.fn();
    render(<Button onClick={handleClick}>Click me</Button>);

    fireEvent.click(screen.getByText('Click me'));
    expect(handleClick).toHaveBeenCalledTimes(1);
  });

  it('is disabled when disabled prop is true', () => {
    render(<Button disabled>Click me</Button>);
    expect(screen.getByRole('button')).toBeDisabled();
  });

  it('applies correct classes for type prop', () => {
    const { container } = render(<Button type="primary">Click</Button>);
    expect(container.firstChild).toHaveClass('primary');
  });
});
```

运行 UI：

```bash
npx vitest --ui
```

## Hook 测试

用 `@testing-library/react-hooks` 测试自定义 Hook：

```bash
npm install -D @testing-library/react
```

```typescript
// src/hooks/useCounter.test.ts
import { describe, it, expect } from "vitest";
import { renderHook, act } from "@testing-library/react";
import { useCounter } from "./useCounter";

describe("useCounter", () => {
  it("initializes with default value", () => {
    const { result } = renderHook(() => useCounter());
    expect(result.current.count).toBe(0);
  });

  it("increments correctly", () => {
    const { result } = renderHook(() => useCounter());

    act(() => {
      result.current.increment();
    });

    expect(result.current.count).toBe(1);
  });

  it("decrements correctly", () => {
    const { result } = renderHook(() => useCounter(5));

    act(() => {
      result.current.decrement();
    });

    expect(result.current.count).toBe(4);
  });
});
```

## E2E 测试

用 Playwright 做端到端测试：

```bash
npm install -D @playwright/test
npx playwright install
```

```typescript
// e2e/button.spec.ts
import { test, expect } from "@playwright/test";

test("button click triggers action", async ({ page }) => {
  await page.goto("http://localhost:3000");

  const button = page.getByRole("button", { name: /click me/i });
  await button.click();

  const result = page.getByText("Count: 1");
  await expect(result).toBeVisible();
});

test("button is disabled when prop is set", async ({ page }) => {
  await page.goto("http://localhost:3000?disabled=true");

  const button = page.getByRole("button");
  await expect(button).toBeDisabled();
});
```

运行：

```bash
npx playwright test
```

## CI 集成

在 GitHub Actions 中自动运行测试：

```yaml
# .github/workflows/test.yml
name: Test

on: [push, pull_request]

jobs:
  test:
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: "20"

      - name: Install pnpm
        uses: pnpm/action-setup@v2
        with:
          version: 8

      - name: Install dependencies
        run: pnpm install

      - name: Run unit tests
        run: pnpm test

      - name: Run E2E tests
        run: pnpm test:e2e

      - name: Upload coverage
        uses: codecov/codecov-action@v3
        with:
          files: ./coverage/coverage-final.json
```

现在每次 push 或 PR，都会自动运行测试。

## 测试最佳实践

1. **测试用户行为，不是实现细节**：用 Testing Library，不用 `querySelector`
2. **避免测试 CSS**：样式会变，测试不应该依赖它
3. **Mock 外部依赖**：API 调用用 Mock，不要真的请求
4. **保持测试简单**：一个测试只测一件事
5. **测试命名清晰**：`should...` 比 `test...` 更好
