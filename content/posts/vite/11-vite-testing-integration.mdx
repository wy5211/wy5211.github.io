## 前言

测试是保证代码质量的重要手段。Vitest 是专为 Vite 设计的测试框架，与 Vite 深度集成。本文将深入探讨 Vite 的测试集成方案。

## Vitest 配置

### 基础配置

```typescript
// vitest.config.ts
import { defineConfig } from "vitest/config";
import react from "@vitejs/plugin-react";

export default defineConfig({
  plugins: [react()],

  test: {
    // 全局变量
    globals: true,

    // 测试环境
    environment: "jsdom", // 'jsdom' | 'node' | 'happy-dom' | 'edge-runtime'

    // 设置文件
    setupFiles: ["./src/test/setup.ts"],

    // 覆盖率
    coverage: {
      provider: "v8", // 'v8' | 'istanbul'
      reporter: ["text", "json", "html", "lcov"],
      exclude: ["node_modules/", "src/test/"],
    },

    // 包含文件
    include: ["**/*.{test,spec}.{js,jsx,ts,tsx}"],

    // 排除文件
    exclude: ["node_modules", "dist", ".idea", ".git", ".cache"],

    // 监听模式
    watch: true,

    // 并发
    threads: true,
    maxThreads: 4,
    minThreads: 1,

    // 超时
    testTimeout: 10000,
    hookTimeout: 10000,

    // 隔立
    isolate: true,
  },
});
```

### 设置文件

```typescript
// src/test/setup.ts
import { expect, beforeEach, afterEach, vi } from "vitest";
import { cleanup } from "@testing-library/react";
import "@testing-library/jest-dom";

// 每个测试后清理
afterEach(() => {
  cleanup();
});

// 全局 mock
global.fetch = vi.fn();

// 环境变量
process.env.NODE_ENV = "test";
```

### 脚本配置

```json
// package.json
{
  "scripts": {
    "test": "vitest",
    "test:ui": "vitest --ui",
    "test:run": "vitest run",
    "test:coverage": "vitest run --coverage"
  }
}
```

## 单元测试

### 基础测试

```typescript
// src/utils/math.test.ts
import { describe, it, expect } from "vitest";
import { add, multiply } from "./math";

describe("Math Utils", () => {
  describe("add", () => {
    it("should add two numbers", () => {
      expect(add(1, 2)).toBe(3);
    });

    it("should handle negative numbers", () => {
      expect(add(-1, -2)).toBe(-3);
    });
  });

  describe("multiply", () => {
    it("should multiply two numbers", () => {
      expect(multiply(2, 3)).toBe(6);
    });
  });
});
```

### 异步测试

```typescript
// src/utils/api.test.ts
import { describe, it, expect, vi } from "vitest";
import { fetchData } from "./api";

describe("API Utils", () => {
  it("should fetch data", async () => {
    // Mock fetch
    global.fetch = vi.fn(() =>
      Promise.resolve({
        json: () => Promise.resolve({ data: "test" }),
      })
    ) as any;

    const data = await fetchData("/api/data");

    expect(data).toEqual({ data: "test" });
    expect(global.fetch).toHaveBeenCalledWith("/api/data");
  });
});
```

### Mock 测试

```typescript
// src/utils/user.test.ts
import { describe, it, expect, vi } from "vitest";
import { getUser } from "./user";
import * as api from "./api";

describe("User Utils", () => {
  it("should get user", async () => {
    // Mock 模块
    vi.spyOn(api, "fetchData").mockResolvedValue({
      name: "Alice",
      age: 30,
    });

    const user = await getUser(1);

    expect(user).toEqual({ name: "Alice", age: 30 });
    expect(api.fetchData).toHaveBeenCalledWith("/users/1");
  });

  afterEach(() => {
    vi.restoreAllMocks();
  });
});
```

## 组件测试

### Testing Library

```bash
npm install -D @testing-library/react
npm install -D @testing-library/jest-dom
npm install -D @testing-library/user-event
```

### 基础组件测试

```typescript
// src/components/Button.test.tsx
import { describe, it, expect, vi } from 'vitest';
import { render, screen } from '@testing-library/react';
import userEvent from '@testing-library/user-event';
import { Button } from './Button';

describe('Button', () => {
  it('should render text', () => {
    render(<Button>Click me</Button>);

    expect(screen.getByText('Click me')).toBeInTheDocument();
  });

  it('should call onClick', async () => {
    const handleClick = vi.fn();
    const user = userEvent.setup();

    render(<Button onClick={handleClick}>Click me</Button>);

    await user.click(screen.getByText('Click me'));

    expect(handleClick).toHaveBeenCalledTimes(1);
  });

  it('should be disabled', () => {
    render(<Button disabled>Click me</Button>);

    expect(screen.getByText('Click me')).toBeDisabled();
  });
});
```

### 表单测试

```typescript
// src/components/LoginForm.test.tsx
import { describe, it, expect, vi } from 'vitest';
import { render, screen } from '@testing-library/react';
import userEvent from '@testing-library/user-event';
import { LoginForm } from './LoginForm';

describe('LoginForm', () => {
  it('should submit form', async () => {
    const handleSubmit = vi.fn();
    const user = userEvent.setup();

    render(<LoginForm onSubmit={handleSubmit} />);

    // 填写表单
    await user.type(screen.getByLabelText(/email/i), 'test@example.com');
    await user.type(screen.getByLabelText(/password/i), 'password123');

    // 提交
    await user.click(screen.getByRole('button', { name: /login/i }));

    expect(handleSubmit).toHaveBeenCalledWith({
      email: 'test@example.com',
      password: 'password123'
    });
  });

  it('should show validation error', async () => {
    const user = userEvent.setup();

    render(<LoginForm onSubmit={vi.fn()} />);

    // 直接提交（不填写）
    await user.click(screen.getByRole('button', { name: /login/i }));

    expect(screen.getByText(/email is required/i)).toBeInTheDocument();
  });
});
```

### 异步组件测试

```typescript
// src/components/UserProfile.test.tsx
import { describe, it, expect, vi } from 'vitest';
import { render, screen, waitFor } from '@testing-library/react';
import { UserProfile } from './UserProfile';
import * as api from '@/api';

describe('UserProfile', () => {
  it('should show loading state', () => {
    vi.spyOn(api, 'fetchUser').mockImplementation(() => new Promise(() => {}));

    render(<UserProfile userId={1} />);

    expect(screen.getByText(/loading/i)).toBeInTheDocument();
  });

  it('should show user data', async () => {
    vi.spyOn(api, 'fetchUser').mockResolvedValue({
      name: 'Alice',
      email: 'alice@example.com'
    });

    render(<UserProfile userId={1} />);

    await waitFor(() => {
      expect(screen.getByText('Alice')).toBeInTheDocument();
      expect(screen.getByText('alice@example.com')).toBeInTheDocument();
    });
  });

  it('should show error', async () => {
    vi.spyOn(api, 'fetchUser').mockRejectedValue(new Error('Failed to fetch'));

    render(<UserProfile userId={1} />);

    await waitFor(() => {
      expect(screen.getByText(/failed to fetch/i)).toBeInTheDocument();
    });
  });
});
```

## E2E 测试

### Playwright 配置

```bash
npm install -D @playwright/test
npx playwright install
```

```typescript
// playwright.config.ts
import { defineConfig, devices } from "@playwright/test";

export default defineConfig({
  testDir: "./e2e",

  fullyParallel: true,

  forbidOnly: !!process.env.CI,

  retries: process.env.CI ? 2 : 0,

  workers: process.env.CI ? 1 : undefined,

  reporter: "html",

  use: {
    baseURL: "http://localhost:3000",
    trace: "on-first-retry",
    screenshot: "only-on-failure",
  },

  projects: [
    {
      name: "chromium",
      use: { ...devices["Desktop Chrome"] },
    },
    {
      name: "firefox",
      use: { ...devices["Desktop Firefox"] },
    },
    {
      name: "webkit",
      use: { ...devices["Desktop Safari"] },
    },
  ],

  webServer: {
    command: "npm run dev",
    url: "http://localhost:3000",
    reuseExistingServer: !process.env.CI,
  },
});
```

### E2E 测试示例

```typescript
// e2e/example.spec.ts
import { test, expect } from "@playwright/test";

test("homepage has title", async ({ page }) => {
  await page.goto("/");

  await expect(page).toHaveTitle(/Vite App/);
});

test("navigation works", async ({ page }) => {
  await page.goto("/");

  await page.click("text=About");

  await expect(page).toHaveURL(/.*about/);
});

test("login flow", async ({ page }) => {
  await page.goto("/login");

  await page.fill('[name="email"]', "test@example.com");
  await page.fill('[name="password"]', "password123");

  await page.click('button[type="submit"]');

  await expect(page).toHaveURL(/.*dashboard/);
  await expect(page.locator("text=Welcome")).toBeVisible();
});
```

```typescript
// vitest.config.ts
export default defineConfig({
  test: {
    coverage: {
      provider: "v8",

      // 覆盖率阈值

        lines: 80,
        functions: 80,
        branches: 80,
        statements: 80,

      // 报告格式
      reporter: ["text", "json", "html", "lcov"],

      // 排除文件

        "node_modules/",
        "src/test/",
        "**/*.d.ts",
        "**/*.config.*",
        "**/mockData",

      // 所有文件
      all: true,

      // 包含文件
```

## 测试最佳实践

### 1. 测试结构

```
src/
  components/
    Button/
      Button.tsx
      Button.test.tsx
  utils/
    math.ts
    math.test.ts
```

### 2. AAA 模式

```typescript
it("should add numbers", () => {
  // Arrange（准备）
  const a = 1;
  const b = 2;

  // Act（执行）
  const result = add(a, b);

  // Assert（断言）
  expect(result).toBe(3);
});
```

### 3. 描述性测试

```typescript
// ❌ 不好的测试名称
it("works", () => {});

// ✅ 好的测试名称
it("should return 404 when user not found", () => {});
```

### 4. 避免测试实现细节

```typescript
// ❌ 测试实现细节
it('should have a button class', () => {
  render(<Button />);
  expect(screen.getByRole('button')).toHaveClass('button');
});

// ✅ 测试行为
it('should call onClick when clicked', async () => {
  const handleClick = vi.fn();
  render(<Button onClick={handleClick} />);

  await userEvent.click(screen.getByRole('button'));

  expect(handleClick).toHaveBeenCalled();
});
```

## 常见问题

### Q1: 测试找不到环境变量？

A: 在测试中模拟环境变量：

```typescript
// src/test/setup.ts
import { vi } from "vitest";

vi.stubEnv("VITE_API_URL", "http://test.api.com");
```

### Q2: 如何测试路由？

A: 使用 MemoryRouter：

```typescript
render(
  <MemoryRouter initialEntries={['/about']}>
    <App />
  </MemoryRouter>
);
```

### Q3: 测试超时？

A: 增加超时时间：

```typescript
it(
  "should complete",
  async () => {
    // ...
  },
  { timeout: 30000 }
);
```

## 总结

Vite 测试集成要点：

1. **Vitest 配置**：环境、设置、覆盖率
2. **单元测试**：同步、异步、Mock
3. **组件测试**：Testing Library、用户交互
4. **E2E 测试**：Playwright 配置
5. **最佳实践**：测试结构、AAA 模式
6. **覆盖率**：阈值、报告、持续集成
