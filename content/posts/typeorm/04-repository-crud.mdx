## 前言

Repository 是 TypeORM Data Mapper 模式的核心，它充当实体与数据库之间的桥梁。通过 Repository API，我们可以用类型安全的方式完成所有 CRUD 操作。

**本篇你将学到**：

- Repository 的基本和高级操作
- `FindOptions` 详解（where、order、relations、select）
- `FindOperator` 条件构建（Like、Between、In、IsNull 等）
- 通用分页封装
- 批量操作技巧

## Repository 基础

### 获取 Repository

```typescript
import { DataSource, Repository } from "typeorm";

// 方式 1：通过 DataSource 获取
const userRepo = dataSource.getRepository(User);

// 方式 2：通过依赖注入（NestJS）
@Injectable()
export class UserService {
  constructor(
    @InjectRepository(User)
    private userRepo: Repository<User>
  ) {}
}
```

### 自定义 Repository

```typescript
import { Repository, DataSource } from "typeorm";
import { EntityRepository } from "typeorm";

// 方式 1：继承 Repository（推荐）
@EntityRepository(User)
export class UserRepository extends Repository<User> {
  // 自定义方法
  async findByName(name: string): Promise<User | null> {
    return this.findOne({ where: { name } });
  }

  async findActiveUsers(): Promise<User[]> {
    return this.find({ where: { isActive: true } });
  }
}

// 方式 2：使用 extends Repository
export class UserRepository extends Repository<User> {
  async findByName(name: string): Promise<User | null> {
    return this.createQueryBuilder("user")
      .where("user.name = :name", { name })
      .getOne();
  }
}
```

## Create 创建

### 单条创建

```typescript
const userRepo = dataSource.getRepository(User);

// 方式 1：create + save（两步操作）
const user = userRepo.create({
  name: "张三",
  email: "zhangsan@example.com",
  password: "hashed_password",
});
await userRepo.save(user);

// 方式 2：直接 save（一步操作）
const user = await userRepo.save({
  name: "张三",
  email: "zhangsan@example.com",
  password: "hashed_password",
});
// 返回完整的实体对象（包含自动生成的 id、createdAt 等）

// 方式 3：insert（只返回插入结果，不返回完整实体）
const result = await userRepo.insert({
  name: "张三",
  email: "zhangsan@example.com",
  password: "hashed_password",
});
// result.identifiers: [{ id: 1 }]
// result.raw: INSERT 返回的原始数据
```

### 批量创建

```typescript
// 方式 1：save（逐条插入，支持级联）
const users = await userRepo.save([
  { name: "张三", email: "zhangsan@example.com", password: "hash1" },
  { name: "李四", email: "lisi@example.com", password: "hash2" },
  { name: "王五", email: "wangwu@example.com", password: "hash3" },
]);
// 返回完整的实体数组

// 方式 2：insert（单条 INSERT 语句，性能更好）
await userRepo.insert([
  { name: "张三", email: "zhangsan@example.com", password: "hash1" },
  { name: "李四", email: "lisi@example.com", password: "hash2" },
]);
// 一条 INSERT INTO ... VALUES (...), (...)

// 方式 3：批量 save + 分块（大数据量）
const batchSize = 100;
for (let i = 0; i < largeArray.length; i += batchSize) {
  const batch = largeArray.slice(i, i + batchSize);
  await userRepo.save(batch);
}
```

### save vs insert 对比

| 特性       | `save()`        | `insert()`   |
| ---------- | --------------- | ------------ |
| 返回值     | 完整实体对象    | InsertResult |
| 级联操作   | ✅ 支持         | ❌ 不支持    |
| 更新已存在 | ✅ 有 id 则更新 | ❌ 报错      |
| 性能       | 较慢（逐条）    | 较快（批量） |
| 生命周期   | ✅ 触发监听器   | ❌ 不触发    |

## Read 查询

### 基本查询

```typescript
// 查询所有
const allUsers = await userRepo.find();

// 条件查询
const activeUsers = await userRepo.find({
  where: { isActive: true },
});

// 查询单条
const user = await userRepo.findOne({
  where: { id: 1 },
});

// 通过 ID 查询
const user = await userRepo.findOneBy({ id: 1 });

// 查询或抛出异常
const user = await userRepo.findOneOrFail({
  where: { id: 1 },
});
// 找不到时抛出 EntityNotFoundError

// 查询数量
const count = await userRepo.count({
  where: { isActive: true },
});

// 查询数据和总数（分页常用）
const [users, total] = await userRepo.findAndCount({
  where: { isActive: true },
  skip: 0,
  take: 10,
});
```

### FindOptions 详解

```typescript
const users = await userRepo.find({
  // 1. where：查询条件
  where: {
    isActive: true,
    role: "AUTHOR",
  },

  // 2. relations：加载关联数据
  relations: {
    posts: true,
    profile: true,
  },

  // 3. select：选择字段
  select: {
    id: true,
    name: true,
    email: true,
    posts: {
      id: true,
      title: true,
    },
  },

  // 4. order：排序
  order: {
    createdAt: "DESC",
    name: "ASC",
  },

  // 5. skip + take：分页
  skip: 0, // 跳过多少条（offset）
  take: 10, // 取多少条（limit）

  // 6. withDeleted：包含软删除
  withDeleted: false,

  // 7. cache：查询缓存
  cache: true, // 或 cache: { id: "users_list", milliseconds: 60000 }

  // 8. lock：悲观锁
  lock: { mode: "optimistic", version: 1 },

  // 9. loadEagerRelations：覆盖 eager 设置
  loadEagerRelations: false,

  // 10. join：LEFT JOIN / INNER JOIN
  join: {
    alias: "user",
    leftJoinAndSelect: {
      posts: "user.posts",
    },
  },
});
```

### 多条件查询（OR）

```typescript
import { FindOperator, Or } from "typeorm";

// 方式 1：数组条件（OR 关系）
const users = await userRepo.find({
  where: [{ name: "张三" }, { name: "李四" }, { role: "ADMIN" }],
});
// WHERE name = '张三' OR name = '李四' OR role = 'ADMIN'

// 方式 2：使用 FindOperator
const users = await userRepo.find({
  where: {
    isActive: true,
    role: In(["ADMIN", "EDITOR"]),
  },
});
// WHERE is_active = true AND role IN ('ADMIN', 'EDITOR')
```

## FindOperator 条件构建器

FindOperator 是 TypeORM 提供的条件构建工具，支持丰富的查询条件。

### 常用操作符

```typescript
import {
  Equal,
  Not,
  Like,
  ILike,
  Between,
  In,
  IsNull,
  LessThan,
  LessThanOrEqual,
  MoreThan,
  MoreThanOrEqual,
  Raw,
  Any,
} from "typeorm";

// Equal：精确匹配
await userRepo.find({
  where: { name: Equal("张三") },
});
// WHERE name = '张三'

// Not：取反
await userRepo.find({
  where: { role: Not("READER") },
});
// WHERE role != 'READER'

// Like：模糊匹配（区分大小写）
await userRepo.find({
  where: { name: Like("%张%") },
});
// WHERE name LIKE '%张%'

// ILike：模糊匹配（不区分大小写）
await userRepo.find({
  where: { email: ILike("%@GMAIL.COM") },
});
// WHERE email ILIKE '%@GMAIL.COM'

// Between：范围查询
await userRepo.find({
  where: {
    createdAt: Between(new Date("2026-01-01"), new Date("2026-12-31")),
  },
});
// WHERE created_at BETWEEN '2026-01-01' AND '2026-12-31'

// In：包含于列表
await userRepo.find({
  where: { role: In(["ADMIN", "EDITOR", "AUTHOR"]) },
});
// WHERE role IN ('ADMIN', 'EDITOR', 'AUTHOR')

// IsNull：为空
await userRepo.find({
  where: { deletedAt: IsNull() },
});
// WHERE deleted_at IS NULL

// LessThan：小于
await userRepo.find({
  where: { age: LessThan(18) },
});
// WHERE age < 18

// LessThanOrEqual：小于等于
await userRepo.find({
  where: { stock: LessThanOrEqual(0) },
});
// WHERE stock <= 0

// MoreThan：大于
await userRepo.find({
  where: { viewCount: MoreThan(1000) },
});
// WHERE view_count > 1000

// MoreThanOrEqual：大于等于
await userRepo.find({
  where: { score: MoreThanOrEqual(60) },
});
// WHERE score >= 60

// Any：匹配数组中的任意值（PostgreSQL）
await userRepo.find({
  where: { tags: Any(["TypeScript", "Node.js"]) },
});
// WHERE tags && ARRAY['TypeScript', 'Node.js']

// Raw：原始 SQL 条件
await userRepo.find({
  where: {
    name: Raw((alias) => `LENGTH(${alias}) > 3`),
  },
});
// WHERE LENGTH(name) > 3

// Raw 带参数
await userRepo.find({
  where: {
    createdAt: Raw((alias) => `${alias} > NOW() - INTERVAL '7 days'`),
  },
});
// WHERE created_at > NOW() - INTERVAL '7 days'
```

### 组合条件

```typescript
// AND：多个条件自动 AND
await userRepo.find({
  where: {
    isActive: true,
    role: In(["ADMIN", "EDITOR"]),
    createdAt: MoreThan(new Date("2026-01-01")),
  },
});
// WHERE is_active = true
//   AND role IN ('ADMIN', 'EDITOR')
//   AND created_at > '2026-01-01'

// OR：使用数组
await userRepo.find({
  where: [{ role: "ADMIN" }, { role: "EDITOR", isActive: true }],
});
// WHERE role = 'ADMIN'
//    OR (role = 'EDITOR' AND is_active = true)
```

## Update 更新

### 基本更新

```typescript
// 方式 1：查询后修改 + save
const user = await userRepo.findOneBy({ id: 1 });
if (user) {
  user.name = "新名字";
  user.bio = "新的个人简介";
  await userRepo.save(user);
  // 触发 @BeforeUpdate 监听器
}

// 方式 2：update（不查询，直接更新）
await userRepo.update(1, {
  name: "新名字",
  bio: "新的个人简介",
});
// 不触发 @BeforeUpdate 监听器

// 方式 3：条件更新
await userRepo.update({ isActive: false }, { isActive: true });
// UPDATE users SET is_active = true WHERE is_active = false
```

### save vs update 对比

| 特性       | `save()`         | `update()`   |
| ---------- | ---------------- | ------------ |
| 需要先查询 | ✅ 通常需要      | ❌ 不需要    |
| 触发监听器 | ✅ @BeforeUpdate | ❌ 不触发    |
| 级联操作   | ✅ 支持          | ❌ 不支持    |
| 返回值     | 完整实体         | UpdateResult |
| 性能       | 较慢             | 较快         |

### 批量更新

```typescript
// 更新所有匹配的记录
await userRepo.update({ role: "READER" }, { isActive: true });

// 使用 QueryBuilder 进行复杂更新
await userRepo
  .createQueryBuilder()
  .update(User)
  .set({ viewCount: () => "view_count + 1" })
  .where("id = :id", { id: 1 })
  .execute();
// UPDATE users SET view_count = view_count + 1 WHERE id = 1
```

## Delete 删除

### 基本删除

```typescript
// 方式 1：通过实体删除
const user = await userRepo.findOneBy({ id: 1 });
await userRepo.remove(user);
// 触发 @BeforeRemove 和 @AfterRemove

// 方式 2：通过 ID 删除
await userRepo.delete(1);

// 方式 3：条件删除
await userRepo.delete({ isActive: false });

// 方式 4：多个 ID
await userRepo.delete([1, 2, 3]);
```

### 软删除

```typescript
// 软删除
await userRepo.softDelete(1);
// UPDATE users SET deleted_at = NOW() WHERE id = 1

// 通过实体软删除
const user = await userRepo.findOneBy({ id: 1 });
await userRepo.softRemove(user);

// 恢复
await userRepo.restore(1);
// UPDATE users SET deleted_at = NULL WHERE id = 1
```

## 通用分页封装

分页是几乎所有应用都需要的通用功能，让我们封装一个可复用的分页工具。

### 定义分页类型

```typescript
// src/common/pagination.ts
import { FindManyOptions, FindOperator, Repository } from "typeorm";

// 分页请求参数
export interface PaginationParams {
  page: number; // 当前页码（从 1 开始）
  limit: number; // 每页条数
  orderBy?: string; // 排序字段
  orderDirection?: "ASC" | "DESC"; // 排序方向
}

// 分页响应结果
export interface PaginationResult<T> {
  data: T[]; // 数据列表
  total: number; // 总记录数
  page: number; // 当前页码
  limit: number; // 每页条数
  totalPages: number; // 总页数
  hasNextPage: boolean; // 是否有下一页
  hasPrevPage: boolean; // 是否有上一页
}
```

### 分页方法封装

```typescript
// src/common/pagination.ts
export async function paginate<T>(
  repository: Repository<T>,
  params: PaginationParams,
  findOptions?: Omit<FindManyOptions<T>, "skip" | "take" | "order">
): Promise<PaginationResult<T>> {
  const {
    page = 1,
    limit = 10,
    orderBy = "createdAt",
    orderDirection = "DESC",
  } = params;

  // 计算跳过条数
  const skip = (page - 1) * limit;

  // 执行查询
  const [data, total] = await repository.findAndCount({
    ...findOptions,
    skip,
    take: limit,
    order: {
      [orderBy]: orderDirection,
    } as any,
  });

  // 计算分页信息
  const totalPages = Math.ceil(total / limit);

  return {
    data,
    total,
    page,
    limit,
    totalPages,
    hasNextPage: page < totalPages,
    hasPrevPage: page > 1,
  };
}
```

### 使用分页

```typescript
// 在 Service 中使用
async function getPosts(params: PaginationParams) {
  return paginate(postRepo, params, {
    where: {
      status: "PUBLISHED",
    },
    relations: {
      author: true,
      tags: true,
    },
    select: {
      id: true,
      title: true,
      excerpt: true,
      publishedAt: true,
      author: {
        id: true,
        name: true,
      },
    },
  });
}

// 调用
const result = await getPosts({
  page: 1,
  limit: 10,
  orderBy: "publishedAt",
  orderDirection: "DESC",
});

console.log(result);
// {
//   data: [post1, post2, ...],
//   total: 100,
//   page: 1,
//   limit: 10,
//   totalPages: 10,
//   hasNextPage: true,
//   hasPrevPage: false
// }
```

### 带搜索的分页

```typescript
// 搜索用户
async function searchUsers(params: PaginationParams, keyword: string) {
  return paginate(userRepo, params, {
    where: [
      { name: ILike(`%${keyword}%`), isActive: true },

    ],
    select: {
      id: true,
      name: true,
      email: true,
```

## 实战：文章管理 Service

```typescript
// src/services/PostService.ts
import { Repository, Like, In, Between } from "typeorm";
import { Post, PostStatus } from "../entity/Post";

export class PostService {
  constructor(private postRepo: Repository<Post>) {}

  // 创建文章
  async create(data: {
    title: string;
    content: string;
    authorId: number;
    tagIds?: number[];
  }) {
    const post = this.postRepo.create({
      title: data.title,
      content: data.content,
      author: { id: data.authorId },
      status: PostStatus.DRAFT,
      tags: data.tagIds?.map((id) => ({ id })),
    });
    return this.postRepo.save(post);
  }

  // 更新文章
  async update(
    id: number,
    data: Partial<Pick<Post, "title" | "content" | "status">>
  ) {
    const post = await this.postRepo.findOneBy({ id });
    if (!post) throw new Error("文章不存在");

    Object.assign(post, data);

    // 如果发布，设置发布时间
    if (data.status === PostStatus.PUBLISHED && !post.publishedAt) {
      post.publishedAt = new Date();
    }

    return this.postRepo.save(post);
  }

  // 获取文章详情
  async getById(id: number) {
    return this.postRepo.findOne({
      where: { id },
      relations: {
        author: true,
        tags: true,
        comments: {
          author: true,
        },
      },
    });
  }

  // 获取文章列表（分页）
  async list(params: {
    page: number;
    limit: number;
    status?: PostStatus;
    authorId?: number;
    tagId?: number;
  }) {
    const { page, limit, status, authorId, tagId } = params;

    const qb = this.postRepo
      .createQueryBuilder("post")
      .leftJoinAndSelect("post.author", "author")
      .leftJoinAndSelect("post.tags", "tag");

    if (status) {
      qb.andWhere("post.status = :status", { status });
    }

    if (authorId) {
      qb.andWhere("post.authorId = :authorId", { authorId });
    }

    if (tagId) {
      qb.innerJoin(
        "post_tags_post",
        "pt",
        "pt.postId = post.id AND pt.tagId = :tagId",
        { tagId }
      );
    }

    qb.orderBy("post.createdAt", "DESC")
      .skip((page - 1) * limit)
      .take(limit);

    const [data, total] = await qb.getManyAndCount();

    return {
      data,
      total,
      page,
      limit,
      totalPages: Math.ceil(total / limit),
    };
  }

  // 增加浏览量
  async incrementViewCount(id: number) {
    await this.postRepo.increment({ id }, "viewCount", 1);
  }

  // 删除文章
  async delete(id: number) {
    return this.postRepo.softDelete(id);
  }
}
```

## 最佳实践

### 1. 选择合适的查询方法

```typescript
// ✅ 简单查询：使用 find/findOne
const user = await userRepo.findOneBy({ id: 1 });

// ✅ 复杂查询：使用 QueryBuilder
const posts = await postRepo
  .createQueryBuilder("post")
  .leftJoinAndSelect("post.author", "author")
  .where("post.status = :status", { status: "PUBLISHED" })
  .orderBy("post.createdAt", "DESC")
  .getMany();

// ❌ 避免：过度使用 QueryBuilder 处理简单查询
const users = await userRepo
  .createQueryBuilder("user")
  .where("user.id = :id", { id: 1 })
  .getOne();
// 使用 findOneBy 更简洁
```

### 2. 合理使用 select

```typescript
// ✅ 只查询需要的字段，减少数据传输
const users = await userRepo.find({
  select: {
    id: true,
    name: true,
    email: true,
  },
});

// ❌ 避免查询不需要的大字段
const posts = await postRepo.find();
// 如果列表页不需要 content，不要加载它
```

### 3. 使用事务处理关联操作

```typescript
// ✅ 创建文章和标签使用事务
await dataSource.transaction(async (manager) => {
  const post = manager.create(Post, {
    title: "新文章",
    content: "内容",
  });
  await manager.save(post);

  // 创建标签关联
  for (const tagId of tagIds) {
    await manager.save(PostTag, { postId: post.id, tagId });
  }
});
```

## 总结

### 核心要点

1. **Repository API**
   - `find` / `findOne` / `findOneBy`：查询
   - `save` / `insert`：创建
   - `update`：更新
   - `remove` / `delete` / `softDelete`：删除
   - `count` / `findAndCount`：计数

2. **FindOptions**
   - `where`：查询条件
   - `relations`：加载关联
   - `select`：选择字段
   - `order`：排序
   - `skip` / `take`：分页

3. **FindOperator**
   - `Like` / `ILike`：模糊匹配
   - `Between`：范围查询
   - `In`：包含于列表
   - `IsNull`：空值判断
   - `Raw`：原始 SQL

4. **分页封装**
   - 统一的 `PaginationParams` 和 `PaginationResult`
   - 通用的 `paginate()` 函数
   - 支持条件查询和关联加载
