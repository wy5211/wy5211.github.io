## 前言

在 Node.js 后端开发中，ORM（Object-Relational Mapping）框架是连接应用代码与数据库的桥梁。**TypeORM** 作为 NestJS 官方推荐的 ORM 框架，凭借**装饰器驱动**、**TypeScript 优先**的设计，成为了 TypeScript 后端开发的主流选择。

**为什么选择 TypeORM？**

- 🏆 **NestJS 官方推荐**：与 NestJS 深度集成，开箱即用
- 🎯 **装饰器驱动**：使用 TypeScript 装饰器定义实体，直观优雅
- 📊 **两种模式**：支持 Active Record 和 Data Mapper 两种设计模式
- 🔌 **多数据库支持**：PostgreSQL、MySQL、SQLite、SQL Server 等
- 🛠️ **成熟稳定**：经过大量生产验证，社区活跃
- 📈 **功能全面**：关系映射、QueryBuilder、迁移、事务、监听器等一应俱全

今天，我们将从零开始搭建 TypeORM 环境，理解核心概念，并完成第一个 CRUD 项目。

## TypeORM 概述

### 什么是 TypeORM？

**TypeORM** 是一个为 TypeScript 和 JavaScript（ES5+）设计的 ORM 框架，灵感来自 Java 的 Hibernate 和 .NET 的 Entity Framework。它使用**装饰器**来定义数据模型，并提供完整的类型安全支持。

```typescript
// TypeORM 的核心特点：

1. TypeScript First（TypeScript 优先）
   - 装饰器定义实体
   - 完整的类型推断
   - 编译时类型检查

2. Dual Pattern（双模式）
   - Active Record 模式（简单直接）
   - Data Mapper 模式（职责分离）

3. Feature Rich（功能丰富）
   - 关系映射（一对一、一对多、多对多）
   - QueryBuilder（灵活查询）
   - 数据库迁移（版本管理）
   - 事务支持（数据一致性）
   - 实体监听器（生命周期钩子）
```

## Active Record vs Data Mapper

TypeORM 支持两种设计模式，理解它们的区别是入门的关键。

### Active Record 模式

Active Record 模式中，**实体本身**包含数据操作的方法。实体继承 `BaseEntity`：

```typescript
// Active Record 模式
import { Entity, PrimaryGeneratedColumn, Column, BaseEntity } from "typeorm";

@Entity()
export class User extends BaseEntity {
  @PrimaryGeneratedColumn()
  id: number;

  @Column()
  name: string;

  @Column({ unique: true })
  email: string;
}

// 使用：直接在实体上操作
const user = new User();
user.name = "张三";
user.email = "zhangsan@example.com";
await user.save(); // 保存到数据库

const foundUser = await User.findOneBy({ id: 1 }); // 查询
await foundUser.remove(); // 删除
```

**特点**：

- ✅ 简单直观，适合小型项目
- ✅ 代码量少，快速上手
- ❌ 实体和数据库操作耦合在一起
- ❌ 不利于单元测试（难以 Mock）

### Data Mapper 模式

Data Mapper 模式中，**实体只定义数据结构**，操作通过 Repository 进行：

```typescript
// Data Mapper 模式
import { Entity, PrimaryGeneratedColumn, Column } from "typeorm";

// 实体只定义数据结构
@Entity()
export class User {
  @PrimaryGeneratedColumn()
  id: number;

  @Column()
  name: string;

  @Column({ unique: true })
  email: string;
}

// 操作通过 Repository 进行
const userRepository = dataSource.getRepository(User);

// 创建
const user = userRepository.create({
  name: "张三",
  email: "zhangsan@example.com",
});
await userRepository.save(user);

// 查询
const foundUser = await userRepository.findOneBy({ id: 1 });

// 删除
await userRepository.remove(foundUser);
```

**特点**：

- ✅ 职责分离，实体纯粹
- ✅ 易于测试，Repository 可 Mock
- ✅ 适合中大型项目
- ❌ 代码量稍多

### 如何选择？

```typescript
// Active Record：适合小型项目、快速原型
// - 代码简洁，开发速度快
// - 适合 CRUD 为主的简单应用

// Data Mapper：适合中大型项目、团队协作（本系列推荐）
// - 架构清晰，职责分离
// - 易于测试和维护
// - NestJS 官方推荐的模式
```

**本系列将使用 Data Mapper 模式**，因为它更契合 NestJS 的模块化架构，也更有利于编写可测试的代码。

## 环境搭建

### 步骤 1：创建项目

```bash
# 创建项目目录
mkdir typeorm-demo && cd typeorm-demo

# 初始化项目
npm init -y

# 安装 TypeScript
npm install typescript tsx @types/node -D

# 初始化 TypeScript 配置
npx tsc --init
```

### 步骤 2：安装 TypeORM 和数据库驱动

```bash
# 安装 TypeORM
npm install typeorm reflect-metadata

# 安装 PostgreSQL 驱动（推荐）
npm install pg

# 或安装 MySQL 驱动
# npm install mysql2

# 或安装 SQLite 驱动（学习和测试用）
# npm install better-sqlite3
```

### 步骤 3：配置 TypeScript

```json
// tsconfig.json
{
  "compilerOptions": {
    "target": "ES2022",
    "module": "commonjs",
    "lib": ["ES2022"],
    "outDir": "./dist",
    "rootDir": "./src",
    "strict": true,
    "esModuleInterop": true,
    "skipLibCheck": true,
    "forceConsistentCasingInFileNames": true,
    "experimentalDecorators": true,
    "emitDecoratorMetadata": true,
    "strictPropertyInitialization": false
  },
  "include": ["src/**/*"],
  "exclude": ["node_modules"]
}
```

**关键配置说明**：

- `experimentalDecorators`：启用装饰器支持（TypeORM 必需）
- `emitDecoratorMetadata`：启用装饰器元数据（TypeORM 必需）
- `strictPropertyInitialization`：设为 `false`，因为实体的属性可能由 TypeORM 自动填充

### 步骤 4：启动 PostgreSQL

使用 Docker 快速启动一个 PostgreSQL 实例：

```bash
# 使用 Docker 启动 PostgreSQL
docker run -d \
  --name typeorm-postgres \
  -e POSTGRES_USER=postgres \
  -e POSTGRES_PASSWORD=postgres \
  -e POSTGRES_DB=typeorm_demo \
  -p 5432:5432 \
  postgres:16
```

## 第一个 CRUD 项目

让我们创建一个完整的用户管理 CRUD 项目。

### 项目结构

```
src/
├── data-source.ts       # 数据源配置
├── entity/
│   └── User.ts          # 用户实体
├── repository/
│   └── UserRepository.ts # 用户仓储（可选）
└── index.ts             # 入口文件
```

### 步骤 1：配置数据源

```typescript
// src/data-source.ts
import "reflect-metadata";
import { DataSource } from "typeorm";
import { User } from "./entity/User";

// 创建数据源
export const AppDataSource = new DataSource({
  type: "postgres",
  host: process.env.DB_HOST || "localhost",
  port: Number(process.env.DB_PORT) || 5432,
  username: process.env.DB_USERNAME || "postgres",
  password: process.env.DB_PASSWORD || "postgres",
  database: process.env.DB_DATABASE || "typeorm_demo",
  synchronize: true, // 开发环境自动同步表结构（生产环境禁用）
  logging: true, // 打印 SQL 日志
  entities: [User], // 注册实体
});
```

**配置说明**：

| 配置项        | 说明                                       |
| ------------- | ------------------------------------------ |
| `type`        | 数据库类型：postgres、mysql、sqlite 等     |
| `synchronize` | 自动同步实体定义到表结构（仅开发环境使用） |
| `logging`     | 打印 SQL 日志，便于调试                    |
| `entities`    | 注册所有实体类                             |

### 步骤 2：定义实体

```typescript
// src/entity/User.ts
import {
  Entity,
  PrimaryGeneratedColumn,
  Column,
  CreateDateColumn,
  UpdateDateColumn,
} from "typeorm";

@Entity("users") // 指定表名
export class User {
  @PrimaryGeneratedColumn() // 自增主键
  id: number;

  @Column({ length: 100 }) // 定义列
  name: string;

  @Column({ unique: true, length: 255 })
  email: string;

  @Column({ length: 255, select: false }) // select: false 查询时不返回
  password: string;

  @Column({ default: true })
  isActive: boolean;

  @Column({ type: "text", nullable: true })
  bio: string;

  @CreateDateColumn({ name: "created_at" }) // 自动记录创建时间
  createdAt: Date;

  @UpdateDateColumn({ name: "updated_at" }) // 自动记录更新时间
  updatedAt: Date;
}
```

### 步骤 3：实现 CRUD 操作

```typescript
// src/index.ts
import "reflect-metadata";
import { AppDataSource } from "./data-source";
import { User } from "./entity/User";

async function main() {
  // 初始化数据源
  await AppDataSource.initialize();
  console.log("数据源已初始化");

  const userRepo = AppDataSource.getRepository(User);

  // ========== Create（创建）==========
  console.log("\n=== 创建用户 ===");

  // 方式 1：create + save
  const user1 = userRepo.create({
    name: "张三",
    email: "zhangsan@example.com",
    password: "hashed_password_1",
  });
  await userRepo.save(user1);
  console.log("创建用户：", user1);

  // 方式 2：直接 save
  const user2 = await userRepo.save({
    name: "李四",
    email: "lisi@example.com",
    password: "hashed_password_2",
  });
  console.log("创建用户：", user2);

  // ========== Read（查询）==========
  console.log("\n=== 查询用户 ===");

  // 查询所有
  const allUsers = await userRepo.find();
  console.log("所有用户：", allUsers);

  // 条件查询
  const activeUsers = await userRepo.find({
    where: { isActive: true },
    select: ["id", "name", "email"], // 只选择部分字段
  });
  console.log("活跃用户：", activeUsers);

  // 查询单个
  const foundUser = await userRepo.findOneBy({ id: 1 });
  console.log("ID 为 1 的用户：", foundUser);

  // ========== Update（更新）==========
  console.log("\n=== 更新用户 ===");

  // 方式 1：修改实体后 save
  if (foundUser) {
    foundUser.name = "张三（已更新）";
    foundUser.bio = "这是一段个人简介";
    await userRepo.save(foundUser);
    console.log("更新后：", foundUser);
  }

  // 方式 2：使用 update 方法
  await userRepo.update(2, { bio: "李四的个人简介" });
  console.log("更新 ID 为 2 的用户");

  // ========== Delete（删除）==========
  console.log("\n=== 删除用户 ===");

  // 方式 1：通过实体删除
  const userToDelete = await userRepo.findOneBy({ id: 2 });
  if (userToDelete) {
    await userRepo.remove(userToDelete);
    console.log("已删除用户：", userToDelete.name);
  }

  // 方式 2：通过 ID 删除
  // await userRepo.delete(2);

  // 最终结果
  console.log("\n=== 最终用户列表 ===");
  const finalUsers = await userRepo.find();
  console.log(finalUsers);
}

main().catch((error) => {
  console.error("程序出错：", error);
  process.exit(1);
});
```


## DataSource 详解

DataSource 是 TypeORM 的核心，负责管理数据库连接。

### 连接配置

```typescript
import { DataSource } from "typeorm";

// PostgreSQL 配置
const pgDataSource = new DataSource({
  type: "postgres",
  host: "localhost",
  port: 5432,
  username: "postgres",
  password: "postgres",
  database: "mydb",
  entities: ["src/entity/**/*.ts"],
  migrations: ["src/migration/**/*.ts"],
  subscribers: ["src/subscriber/**/*.ts"],
});

// MySQL 配置
const mysqlDataSource = new DataSource({
  type: "mysql",
  host: "localhost",
  port: 3306,
  username: "root",
  password: "root",
  database: "mydb",
  entities: ["src/entity/**/*.ts"],
});

// SQLite 配置（适合学习和测试）
const sqliteDataSource = new DataSource({
  type: "better-sqlite3",
  database: "database.sqlite",
  entities: ["src/entity/**/*.ts"],
});
```

### 连接池配置

```typescript
const dataSource = new DataSource({
  type: "postgres",
  // ...基本配置
  poolSize: 10, // 连接池大小
  extra: {
    // 额外的连接池配置
    max: 20, // 最大连接数
    idleTimeoutMillis: 30000, // 空闲超时
    connectionTimeoutMillis: 5000, // 连接超时
  },
});
```

### 数据源生命周期

```typescript
// 1. 初始化连接
await dataSource.initialize();
console.log("数据库已连接");

// 2. 检查连接状态
if (dataSource.isInitialized) {
  console.log("数据源已初始化");
}

// 3. 关闭连接
await dataSource.destroy();
console.log("数据库连接已关闭");
```

## 常见问题

### Q1：TypeORM 报错 "EntityMetadataNotFoundError" 怎么办？

```typescript
// 原因：实体没有被 DataSource 识别
// 解决方案：确保实体已正确注册

// 方式 1：在 DataSource 配置中添加
entities: [User],

// 方式 2：使用通配符
entities: ["src/entity/**/*.ts"],

// 方式 3：使用实体数组
entities: [__dirname + "/entity/**/*.ts"],
```

### Q2：装饰器不生效怎么办？

```typescript
// 确保以下两个条件满足：

// 1. tsconfig.json 中启用装饰器
{
  "compilerOptions": {
    "experimentalDecorators": true,
    "emitDecoratorMetadata": true
  }
}

// 2. 在入口文件顶部引入 reflect-metadata
import "reflect-metadata";
```

### Q3：synchronize 和 migration 该用哪个？

```typescript
// 开发阶段：使用 synchronize（方便快速迭代）
synchronize: true;

// 生产环境：使用 migration（安全可控）
synchronize: false;
// 通过 migration:run 管理表结构变更
```

## 总结

### 核心要点

1. **TypeORM 定位**
   - NestJS 官方推荐的 ORM 框架
   - 装饰器驱动、TypeScript 优先
   - 支持 Active Record 和 Data Mapper 两种模式

2. **两种模式**
   - Active Record：实体继承 BaseEntity，适合小型项目
   - Data Mapper：通过 Repository 操作，适合中大型项目

3. **环境搭建**
   - 安装 typeorm、reflect-metadata、数据库驱动
   - 配置 tsconfig.json 启用装饰器
   - 配置 DataSource 连接数据库

4. **基本 CRUD**
   - `save()` / `create()`：创建和保存
   - `find()` / `findOneBy()`：查询
   - `update()`：更新
   - `remove()` / `delete()`：删除
