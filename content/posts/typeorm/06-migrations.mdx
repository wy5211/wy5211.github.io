## 前言

在生产环境中，`synchronize: true` 是危险的——它可能意外删除列或修改表结构，导致数据丢失。**数据库迁移**（Migration）是安全地管理数据库结构变更的标准方案。

**本篇你将学到**：

- TypeORM CLI 配置（`data-source.ts`）
- 迁移文件的创建和编写
- 生成、执行、回滚迁移
- 数据种子（seeding）
- 生产迁移策略

---

## 为什么需要迁移？

### synchronize 的局限

```typescript
// ❌ synchronize 的问题：
const dataSource = new DataSource({
  synchronize: true, // 自动同步实体定义到表结构
});

// 问题 1：不可控
// - 不知道具体做了哪些变更
// - 没有变更历史记录

// 问题 2：不安全
// - 可能删除已有的列
// - 可能修改列类型导致数据丢失

// 问题 3：不可回滚
// - 变更出错后无法撤销
// - 没有回滚机制

// 问题 4：不可审计
// - 无法追溯谁在什么时候做了什么变更
// - 没有变更记录
```

### 迁移的优势

```typescript
// ✅ 迁移的优势：

// 1. 可控
// - 每次变更都有明确的迁移文件
// - 可以 code review

// 2. 安全
// - 只执行文件中定义的操作
// - 不会意外删除数据

// 3. 可回滚
// - 每个迁移都有 up 和 down 方法
// - 出错时可以回滚

// 4. 可审计
// - 迁移文件纳入版本控制
// - 变更历史清晰可追溯
```

## CLI 配置

### 创建数据源文件

```typescript
// src/data-source.ts
import "reflect-metadata";
import { DataSource } from "typeorm";
import { config } from "dotenv";

config(); // 加载 .env 文件

export const AppDataSource = new DataSource({
  type: "postgres",
  host: process.env.DB_HOST || "localhost",
  port: Number(process.env.DB_PORT) || 5432,
  username: process.env.DB_USERNAME || "postgres",
  password: process.env.DB_PASSWORD || "postgres",
  database: process.env.DB_DATABASE || "typeorm_demo",
  // 迁移配置
  migrations: ["src/migration/**/*.ts"],
  migrationsTableName: "migrations", // 迁移记录表名
  // 实体配置（用于生成迁移）
  entities: ["src/entity/**/*.ts"],
  // 生产环境关闭 synchronize
  synchronize: false,
  logging: true,
});
```

### 配置 package.json scripts

```json
{
  "scripts": {
    "typeorm": "typeorm-ts-node-commonjs",
    "migration:generate": "npm run typeorm -- -d src/data-source.ts migration:generate",
    "migration:create": "npm run typeorm -- migration:create",
    "migration:run": "npm run typeorm -- -d src/data-source.ts migration:run",
    "migration:revert": "npm run typeorm -- -d src/data-source.ts migration:revert",
    "migration:show": "npm run typeorm -- -d src/data-source.ts migration:show"
  }
}
```

## 迁移文件结构

### 自动生成的迁移文件

```typescript
// src/migration/1709000000000-CreateUsers.ts

import { MigrationInterface, QueryRunner } from "typeorm";

export class CreateUsers1709000000000 implements MigrationInterface {
  // up：执行迁移
  public async up(queryRunner: QueryRunner): Promise<void> {
    await queryRunner.createTable(
      new Table({
        name: "users",
        columns: [
          {
            name: "id",
            type: "serial",
            isPrimary: true,
          },
          {
            name: "name",
            type: "varchar",
            length: "100",
          },
          {
            name: "email",
            type: "varchar",
            length: "255",
            isUnique: true,
          },
          {
            name: "password_hash",
            type: "varchar",
            length: "255",
          },
          {
            name: "is_active",
            type: "boolean",
            default: true,
          },
          {
            name: "created_at",
            type: "timestamp",
            default: "now()",
          },
          {
            name: "updated_at",
            type: "timestamp",
            default: "now()",
          },
        ],
      }),
      true // 如果表存在则跳过
    );
  }

  // down：回滚迁移
  public async down(queryRunner: QueryRunner): Promise<void> {
    await queryRunner.dropTable("users");
  }
}
```

### 手动创建迁移文件

```bash
# 创建空迁移文件
npx typeorm migration:create src/migration/AddUserBio
```

```typescript
// src/migration/AddUserBio.ts

import { MigrationInterface, QueryRunner } from "typeorm";

export class AddUserBio1709000000001 implements MigrationInterface {
  public async up(queryRunner: QueryRunner): Promise<void> {
    // 添加 bio 列
    await queryRunner.addColumn(
      "users",
      new TableColumn({
        name: "bio",
        type: "text",
        isNullable: true,
      })
    );
  }

  public async down(queryRunner: QueryRunner): Promise<void> {
    // 回滚：删除 bio 列
    await queryRunner.dropColumn("users", "bio");
  }
}
```

## 迁移操作

### 生成迁移

```bash
# 根据实体变更自动生成迁移文件
npx typeorm -d src/data-source.ts migration:generate src/migration/AddUserBio
```

TypeORM 会对比实体定义和数据库结构，自动生成迁移文件。

### 执行迁移

```bash
# 执行所有未执行的迁移
npx typeorm -d src/data-source.ts migration:run
```

执行的迁移会记录在 `migrations` 表中：

```
 migrations 表：
 | id | timestamp    | name                          |
 |----|------------- |-------------------------------|
 | 1  | 1709000000000 | CreateUsers1709000000000      |
 | 2  | 1709000000001 | AddUserBio1709000000001       |
```

### 回滚迁移

```bash
# 回滚最近一次迁移
npx typeorm -d src/data-source.ts migration:revert
```

回滚会执行迁移文件的 `down` 方法，并从 `migrations` 表中删除记录。

### 查看迁移状态

```bash
# 查看已执行和未执行的迁移
npx typeorm -d src/data-source.ts migration:show
```

## QueryRunner API

QueryRunner 是迁移中操作数据库的核心工具。

### 表操作

```typescript
public async up(queryRunner: QueryRunner): Promise<void> {
  // 创建表
  await queryRunner.createTable(
    new Table({
      name: "posts",
      columns: [
        { name: "id", type: "serial", isPrimary: true },
        { name: "title", type: "varchar", length: "255" },
        { name: "content", type: "text" },
        { name: "author_id", type: "integer" },
        { name: "status", type: "enum", enum: ["DRAFT", "PUBLISHED", "ARCHIVED"], default: "'DRAFT'" },
        { name: "created_at", type: "timestamp", default: "now()" },
      ],
    }),
    true
  );

  // 重命名表
  await queryRunner.renameTable("old_name", "new_name");

  // 删除表
  await queryRunner.dropTable("posts");
}
```

### 列操作

```typescript
public async up(queryRunner: QueryRunner): Promise<void> {
  // 添加列
  await queryRunner.addColumn("users", new TableColumn({
    name: "avatar",
    type: "varchar",
    length: "500",
    isNullable: true,
  }));

  // 修改列
  await queryRunner.changeColumn("users", "name",
    new TableColumn({ name: "name", type: "varchar", length: "200" })
  );

  // 重命名列
  await queryRunner.renameColumn("users", "name", "full_name");

  // 删除列
  await queryRunner.dropColumn("users", "avatar");
}
```

### 索引操作

```typescript
public async up(queryRunner: QueryRunner): Promise<void> {
  // 创建索引
  await queryRunner.createIndex("users", new TableIndex({
    name: "idx_users_email",
    columnNames: ["email"],
    isUnique: true,
  }));

  // 创建复合索引
  await queryRunner.createIndex("posts", new TableIndex({
    name: "idx_posts_author_status",
    columnNames: ["author_id", "status"],
  }));

  // 删除索引
  await queryRunner.dropIndex("users", "idx_users_email");
}
```

### 外键操作

```typescript
public async up(queryRunner: QueryRunner): Promise<void> {
  // 添加外键
  await queryRunner.createForeignKey("posts", new TableForeignKey({
    name: "fk_posts_author",
    columnNames: ["author_id"],
    referencedTableName: "users",
    referencedColumnNames: ["id"],
    onDelete: "CASCADE",
  }));

  // 删除外键
  await queryRunner.dropForeignKey("posts", "fk_posts_author");
}
```

### 执行原始 SQL

```typescript
public async up(queryRunner: QueryRunner): Promise<void> {
  // 执行自定义 SQL
  await queryRunner.query(`
    ALTER TABLE users
    ADD COLUMN IF NOT EXISTS search_vector tsvector
  `);

  await queryRunner.query(`
    CREATE INDEX IF NOT EXISTS idx_users_search
    ON users USING gin(search_vector)
  `);

  // 更新数据
  await queryRunner.query(`
    UPDATE users
    SET search_vector = to_tsvector('simple', name || ' ' || email)
  `);
}
```

## 数据种子（Seeding）

种子数据用于初始化系统必需的数据，如管理员账号、默认配置等。

### 创建种子文件

```typescript
// src/seed/initial-data.ts

import { DataSource } from "typeorm";
import { User, UserRole } from "../entity/User";
import { Category } from "../entity/Category";
import * as bcrypt from "bcrypt";

export async function seedInitialData(dataSource: DataSource) {
  const userRepo = dataSource.getRepository(User);
  const categoryRepo = dataSource.getRepository(Category);

  // 检查是否已有数据
  const userCount = await userRepo.count();
  if (userCount > 0) {
    console.log("数据已存在，跳过种子");
    return;
  }

  // 创建管理员
  const admin = userRepo.create({
    name: "管理员",
    username: "admin",
    email: "admin@example.com",
    passwordHash: await bcrypt.hash("admin123", 10),
    role: UserRole.ADMIN,
    isActive: true,
  });
  await userRepo.save(admin);

  // 创建默认分类
  const categories = categoryRepo.create([
    { name: "技术", slug: "tech" },
    { name: "生活", slug: "life" },
    { name: "随笔", slug: "essay" },
  ]);
  await categoryRepo.save(categories);

  console.log("种子数据创建完成");
}
```

### 创建种子脚本

```typescript
// src/seed/index.ts

import "reflect-metadata";
import { AppDataSource } from "../data-source";
import { seedInitialData } from "./initial-data";

async function main() {
  await AppDataSource.initialize();
  console.log("数据源已连接");

  await seedInitialData(AppDataSource);

  await AppDataSource.destroy();
  console.log("种子完成");
}

main().catch(console.error);
```

```json
// package.json
{
  "scripts": {
    "seed": "tsx src/seed/index.ts",
    "seed:prod": "node dist/seed/index.js"
  }
}
```

### 将种子集成到迁移中

```typescript
// 也可以在迁移中执行种子
export class SeedAdminUser1709000000002 implements MigrationInterface {
  public async up(queryRunner: QueryRunner): Promise<void> {
    await queryRunner.query(`
      INSERT INTO users (name, username, email, password_hash, role, is_active)
      VALUES ('管理员', 'admin', 'admin@example.com', '$2b$10$...', 'ADMIN', true)
    `);
  }

  public async down(queryRunner: QueryRunner): Promise<void> {
    await queryRunner.query(`
      DELETE FROM users WHERE username = 'admin'
    `);
  }
}
```

## 生产迁移策略

### 1. CI/CD 自动迁移

```yaml
# .github/workflows/migrate.yml
name: Database Migration

on:
  push:
    branches: [main]
    paths:
      - "src/migration/**"
      - "src/entity/**"

jobs:
  migrate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: "20"

      - name: Install dependencies
        run: npm ci

      - name: Run migrations
        run: npm run migration:run
        env:
          DB_HOST: ${{ secrets.DB_HOST }}
          DB_PORT: ${{ secrets.DB_PORT }}
          DB_USERNAME: ${{ secrets.DB_USERNAME }}
          DB_PASSWORD: ${{ secrets.DB_PASSWORD }}
          DB_DATABASE: ${{ secrets.DB_DATABASE }}
```

### 2. 迁移最佳实践

```typescript
// ✅ 1. 每个迁移只做一件事
// 好：一个迁移创建 users 表，另一个迁移添加 bio 列
// 坏：一个迁移同时创建多个表和添加多个列

// ✅ 2. 迁移必须可回滚
// 每个 up 都要有对应的 down

// ✅ 3. 数据迁移和结构迁移分开
export class AddUserBioColumn1709000000003 implements MigrationInterface {
  public async up(queryRunner: QueryRunner): Promise<void> {
    // 步骤 1：添加可空列
    await queryRunner.addColumn(
      "users",
      new TableColumn({
        name: "bio",
        type: "text",
        isNullable: true, // 先允许为空
      })
    );

    // 步骤 2：迁移数据（如果需要）
    await queryRunner.query(`
      UPDATE users SET bio = '暂无简介' WHERE bio IS NULL
    `);
  }

  public async down(queryRunner: QueryRunner): Promise<void> {
    await queryRunner.dropColumn("users", "bio");
  }
}

// ✅ 4. 使用事务
export class SafeMigration1709000000004 implements MigrationInterface {
  public async up(queryRunner: QueryRunner): Promise<void> {
    // QueryRunner 默认在事务中运行
    // 如果出错，所有变更都会回滚
    await queryRunner.createTable(/* ... */);
    await queryRunner.createIndex(/* ... */);
  }
}

// ✅ 5. 幂等性（可选）
export async function up(queryRunner: QueryRunner): Promise<void> {
  // 使用 IF NOT EXISTS 确保幂等
  await queryRunner.query(`
    CREATE TABLE IF NOT EXISTS users (...)
  `);
}
```

## 常见迁移场景

### 添加带默认值的新列

```typescript
export class AddUserStatus1709000000005 implements MigrationInterface {
  public async up(queryRunner: QueryRunner): Promise<void> {
    // 步骤 1：添加可空列
    await queryRunner.addColumn(
      "users",
      new TableColumn({
        name: "status",
        type: "varchar",
        length: "20",
        isNullable: true,
      })
    );

    // 步骤 2：设置默认值
    await queryRunner.query(`UPDATE users SET status = 'ACTIVE'`);

    // 步骤 3：改为 NOT NULL
    await queryRunner.changeColumn(
      "users",
      "status",
      new TableColumn({
        name: "status",
        type: "varchar",
        length: "20",
        isNullable: false,
        default: "'ACTIVE'",
      })
    );
  }

  public async down(queryRunner: QueryRunner): Promise<void> {
    await queryRunner.dropColumn("users", "status");
  }
}
```

### 重命名列（安全方式）

```typescript
export class RenameUserName1709000000006 implements MigrationInterface {
  public async up(queryRunner: QueryRunner): Promise<void> {
    await queryRunner.renameColumn("users", "name", "display_name");
  }

  public async down(queryRunner: QueryRunner): Promise<void> {
    await queryRunner.renameColumn("users", "display_name", "name");
  }
}
```

### 迁移数据到新表

```typescript
export class SplitUserAddress1709000000007 implements MigrationInterface {
  public async up(queryRunner: QueryRunner): Promise<void> {
    // 1. 创建新表
    await queryRunner.createTable(
      new Table({
        name: "addresses",
        columns: [
          { name: "id", type: "serial", isPrimary: true },
          { name: "user_id", type: "integer" },
          { name: "street", type: "varchar", length: "255" },
          { name: "city", type: "varchar", length: "100" },
        ],
      })
    );

    // 2. 迁移数据
    await queryRunner.query(`
      INSERT INTO addresses (user_id, street, city)
      SELECT id, street, city FROM users
      WHERE street IS NOT NULL
    `);

    // 3. 删除旧列
    await queryRunner.dropColumn("users", "street");
    await queryRunner.dropColumn("users", "city");
  }

  public async down(queryRunner: QueryRunner): Promise<void> {
    // 恢复旧列
    await queryRunner.addColumn(
      "users",
      new TableColumn({
        name: "street",
        type: "varchar",
        length: "255",
        isNullable: true,
      })
    );
    await queryRunner.addColumn(
      "users",
      new TableColumn({
        name: "city",
        type: "varchar",
        length: "100",
        isNullable: true,
      })
    );

    // 恢复数据
    await queryRunner.query(`
      UPDATE users SET
        street = a.street,
        city = a.city
      FROM addresses a
      WHERE users.id = a.user_id
    `);

    // 删除新表
    await queryRunner.dropTable("addresses");
  }
}
```

## 总结

### 核心要点

1. **迁移 vs synchronize**
   - 生产环境必须使用迁移
   - synchronize 仅用于开发环境快速原型

2. **迁移流程**
   - 修改实体 → 生成迁移 → 检查 → 执行
   - 每个迁移包含 `up`（执行）和 `down`（回滚）

3. **QueryRunner API**
   - `createTable` / `dropTable`
   - `addColumn` / `changeColumn` / `dropColumn`
   - `createIndex` / `createForeignKey`
   - `query`（原始 SQL）

4. **种子数据**
   - 初始化必要数据（管理员、默认配置）
   - 独立于迁移管理

5. **生产策略**
   - CI/CD 自动执行迁移
   - 数据迁移和结构迁移分开
   - 确保迁移可回滚
