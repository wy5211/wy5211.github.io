## 前言

在 TypeORM 中，**实体（Entity）**是数据库表的 TypeScript 映射。通过装饰器，我们可以用优雅的方式定义表结构、字段类型、约束和索引。

**本篇你将学到**：

- `@Entity` 和 `@Column` 装饰器的完整用法
- TypeScript 与 SQL 的数据类型映射
- 主键策略（自增、UUID）
- 索引和约束的定义
- 软删除的实现方式
- 实体监听器（生命周期钩子）
- 嵌入式实体

---

## @Entity 装饰器

`@Entity` 装饰器用于将一个 TypeScript 类标记为数据库表。

### 基本用法

```typescript
import { Entity } from "typeorm";

// 表名默认为类名的小写：user
@Entity()
export class User {}

// 指定表名
@Entity("users")
export class User {}

// 指定 schema
@Entity("users", { schema: "public" })
export class User {}

// 指定数据库（多数据库场景）
@Entity("users", { database: "secondary" })
export class User {}
```

### 高级选项

```typescript
@Entity("users", {
  // 表引擎（MySQL）
  engine: "InnoDB",

  // 数据库视图（只读）
  // typeorm 会将此实体视为视图

  // 自定义表名同步选项
  synchronize: true, // 是否自动同步此表

  // 表注释
  comment: "用户信息表",
})
export class User {}
```

## @Column 装饰器详解

`@Column` 装饰器用于定义表中的列，是最常用的装饰器。

### 基本用法

```typescript
import { Entity, PrimaryGeneratedColumn, Column } from "typeorm";

@Entity("users")
export class User {
  @PrimaryGeneratedColumn()
  id: number;

  // 基本列
  @Column()
  name: string;

  // 指定类型和长度
  @Column({ type: "varchar", length: 255 })
  email: string;

  // 可空字段
  @Column({ nullable: true })
  bio: string;

  // 带默认值
  @Column({ default: true })
  isActive: boolean;

  // 唯一约束
  @Column({ unique: true })
  email: string;

  // 查询时默认不返回（如密码字段）
  @Column({ select: false })
  password: string;

  // 列注释
  @Column({ comment: "用户邮箱，唯一标识" })
  email: string;
}
```

### 数据类型映射

TypeORM 将 TypeScript 类型自动映射为 SQL 类型：

| TypeScript 类型 | SQL 类型（PostgreSQL） | 说明              |
| --------------- | ---------------------- | ----------------- |
| `string`        | `character varying`    | 默认 varchar(255) |
| `number`        | `integer`              | 整数              |
| `boolean`       | `boolean`              | 布尔              |
| `Date`          | `timestamp`            | 时间戳            |
| `Buffer`        | `bytea`                | 二进制            |
| `Object`        | `json`                 | JSON              |
| `any[]`         | `text[]`               | 数组              |

### 指定列类型

```typescript
@Entity("products")
export class Product {
  @PrimaryGeneratedColumn()
  id: number;

  // 字符串类型
  @Column({ type: "varchar", length: 100 })
  name: string;

  @Column({ type: "text" })
  description: string;

  @Column({ type: "char", length: 2 })
  countryCode: string;

  // 数值类型
  @Column({ type: "integer" })
  stock: number;

  @Column({ type: "bigint" })
  views: string; // bigint 在 JS 中是 string

  @Column({ type: "float" })
  weight: number;

  @Column({ type: "decimal", precision: 10, scale: 2 })
  price: string; // decimal 在 JS 中是 string

  @Column({ type: "real" })
  rating: number;

  // 布尔类型
  @Column({ type: "boolean", default: true })
  isActive: boolean;

  // 日期类型
  @Column({ type: "date" })
  publishDate: Date;

  @Column({ type: "time" })
  openTime: string;

  @Column({ type: "timestamp" })
  createdAt: Date;

  @Column({ type: "timestamptz" }) // 带时区
  updatedAt: Date;

  // JSON 类型
  @Column({ type: "json" })
  metadata: object;

  @Column({ type: "jsonb" }) // PostgreSQL 特有，查询更快
  settings: object;

  // 数组类型（PostgreSQL 特有）
  @Column({ type: "text", array: true })
  tags: string[];

  @Column({ type: "integer", array: true })
  scores: number[];

  // 二进制类型
  @Column({ type: "bytea" })
  avatar: Buffer;

  // 枚举类型
  @Column({
    type: "enum",
    enum: ["DRAFT", "PUBLISHED", "ARCHIVED"],
    default: "DRAFT",
  })
  status: "DRAFT" | "PUBLISHED" | "ARCHIVED";

  // 简单 JSON 类型
  @Column({ type: "simple-json" }) // 存储为字符串，自动序列化
  preferences: { theme: string; language: string };

  // 简单数组类型
  @Column({ type: "simple-array" }) // 存储为逗号分隔字符串
  categories: string[];

  // UUID 类型
  @Column({ type: "uuid" })
  uuid: string;
}
```

## 主键策略

### 自增主键

```typescript
import { PrimaryGeneratedColumn } from "typeorm";

// 整数自增（最常用）
@Entity()
export class User {
  @PrimaryGeneratedColumn()
  id: number; // SERIAL PRIMARY KEY
}

// 指定自增策略
@Entity()
export class User {
  @PrimaryGeneratedColumn({ type: "bigint" })
  id: string; // BIGSERIAL PRIMARY KEY
}
```

### UUID 主键

```typescript
import { PrimaryGeneratedColumn } from "typeorm";

// 自动生成 UUID v4
@Entity()
export class User {
  @PrimaryGeneratedColumn("uuid")
  id: string; // UUID PRIMARY KEY DEFAULT gen_random_uuid()
}

// 自定义 UUID 生成策略
@Entity()
export class User {
  @PrimaryGeneratedColumn("uuid", {
    generated: "uuid", // 或 "identity"
  })
  id: string;
}
```

### 复合主键

```typescript
import { PrimaryColumn } from "typeorm";

// 手动指定主键列
@Entity()
export class PostTag {
  @PrimaryColumn()
  postId: number;

  @PrimaryColumn()
  tagId: number;
}

// 复合主键 + 自动生成
@Entity()
export class User {
  @PrimaryColumn({ type: "uuid", generated: "uuid" })
  id: string;

  @PrimaryColumn()
  tenantId: number;
}
```

### 无自增主键

```typescript
import { PrimaryColumn } from "typeorm";

// 手动管理主键值
@Entity()
export class Country {
  @PrimaryColumn({ type: "char", length: 2 })
  code: string; // 如 "CN"、"US"
}
```

## 时间戳列

TypeORM 提供了三个便捷的时间戳装饰器：

```typescript
import { CreateDateColumn, UpdateDateColumn, DeleteDateColumn } from "typeorm";

@Entity("users")
export class User {
  @PrimaryGeneratedColumn()
  id: number;

  // 自动记录创建时间（只在 insert 时设置）
  @CreateDateColumn({ name: "created_at" })
  createdAt: Date;

  // 自动记录更新时间（每次 update 时更新）
  @UpdateDateColumn({ name: "updated_at" })
  updatedAt: Date;

  // 软删除时间（设置为 null 以外的值表示已删除）
  @DeleteDateColumn({ name: "deleted_at" })
  deletedAt: Date;
}
```

**生成的 SQL**：

```sql
CREATE TABLE users (
  id SERIAL PRIMARY KEY,
  created_at TIMESTAMP NOT NULL DEFAULT now(),
  updated_at TIMESTAMP NOT NULL DEFAULT now(),
  deleted_at TIMESTAMP NULL
);
```

## 索引

### 列级索引

```typescript
import { Column, Index } from "typeorm";

@Entity("users")
export class User {
  @Column()
  @Index() // 单列索引
  email: string;

  @Column()
  @Index({ unique: true }) // 唯一索引
  username: string;
}
```

### 表级索引

```typescript
import { Entity, Index } from "typeorm";

@Entity("users")
@Index(["firstName", "lastName"]) // 复合索引
@Index(["email"], { unique: true }) // 唯一索引
export class User {
  @Column()
  firstName: string;

  @Column()
  lastName: string;

  @Column()
  email: string;
}
```

### 高级索引选项

```typescript
@Entity("posts")
@Index(["title", "content"], {
  name: "post_fulltext_idx", // 自定义索引名
  fulltext: true, // 全文索引（MySQL）
  // PostgreSQL 使用 GIN 索引实现全文搜索
})
export class Post {
  @Column()
  title: string;

  @Column()
  content: string;
}

// 条件索引（PostgreSQL）
@Entity("users")
@Index(["email"], {
  where: "isActive = true", // 只索引活跃用户
})
export class User {
  @Column()
  email: string;

  @Column()
  isActive: boolean;
}

// 使用 synchronize: false 禁用特定索引的自动同步
@Entity("users")
@Index(["email"], { synchronize: false })
export class User {
  @Column()
  email: string;
}
```

## 约束

### 唯一约束

```typescript
import { Column, Unique } from "typeorm";

// 列级唯一约束
@Entity("users")
export class User {
  @Column({ unique: true })
  email: string;
}

// 表级唯一约束
@Entity("users")
@Unique(["email"])
export class User {
  @Column()
  email: string;
}

// 复合唯一约束
@Entity("user_roles")
@Unique("UQ_user_role", ["userId", "roleId"])
export class UserRole {
  @Column()
  userId: number;

  @Column()
  roleId: number;
}
```

### 检查约束

```typescript
import { Check } from "typeorm";

@Entity("products")
@Check(`"price" > 0`) // 价格必须大于 0
@Check(`"stock" >= 0`) // 库存不能为负
export class Product {
  @Column({ type: "decimal" })
  price: string;

  @Column()
  stock: number;
}

// 命名检查约束
@Entity("users")
@Check("age_check", `"age" >= 0 AND "age" <= 150`)
export class User {
  @Column()
  age: number;
}
```

### 外键约束

```typescript
import { ManyToOne } from "typeorm";

@Entity("posts")
export class Post {
  @ManyToOne(() => User, (user) => user.posts, {
    onDelete: "CASCADE", // 删除用户时，同时删除文章
    onUpdate: "CASCADE", // 更新用户 ID 时，同时更新文章
  })
  author: User;
}
```

外键行为选项：

| 选项          | 说明                          |
| ------------- | ----------------------------- |
| `CASCADE`     | 级联操作（删除/更新关联记录） |
| `SET NULL`    | 设为 NULL                     |
| `RESTRICT`    | 拒绝操作（有关联记录时）      |
| `NO ACTION`   | 类似 RESTRICT                 |
| `SET DEFAULT` | 设为默认值                    |

## 软删除

软删除不会真正从数据库删除记录，而是标记为"已删除"。

### 使用 @DeleteDateColumn

```typescript
import { DeleteDateColumn } from "typeorm";

@Entity("users")
export class User {
  @PrimaryGeneratedColumn()
  id: number;

  @Column()
  name: string;

  @DeleteDateColumn({ name: "deleted_at" })
  deletedAt: Date;
}
```

### 软删除操作

```typescript
const userRepo = dataSource.getRepository(User);

// 软删除（设置 deletedAt 字段）
await userRepo.softDelete(1);
// 或
await userRepo.softRemove(user);

// 恢复（清除 deletedAt 字段）
await userRepo.restore(1);
// 或
await userRepo.recover(user);

// 查询：默认排除软删除的记录
const activeUsers = await userRepo.find();
// SELECT * FROM users WHERE deleted_at IS NULL

// 查询包含软删除的记录
const allUsers = await userRepo.find({
  withDeleted: true,
});
// SELECT * FROM users
```

### 使用 @Version 实现乐观锁

```typescript
import { VersionColumn } from "typeorm";

@Entity("products")
export class Product {
  @PrimaryGeneratedColumn()
  id: number;

  @Column()
  name: string;

  @Column({ type: "decimal" })
  price: string;

  @VersionColumn()
  version: number; // 每次更新自动 +1
}

// 使用乐观锁防止并发冲突
const product = await productRepo.findOneBy({ id: 1 });
product.price = "99.99";
await productRepo.save(product);
// UPDATE products SET price = '99.99', version = version + 1
// WHERE id = 1 AND version = 旧版本号
// 如果版本号不匹配，说明被其他操作修改过，会抛出 OptimisticLockVersionMismatchError
```

## 实体监听器

实体监听器是 TypeORM 提供的生命周期钩子，可以在特定事件发生时执行自定义逻辑。

### 可用的监听器

```typescript
import {
  Entity,
  PrimaryGeneratedColumn,
  Column,
  BeforeInsert,
  AfterInsert,
  BeforeUpdate,
  AfterUpdate,
  BeforeRemove,
  AfterRemove,
  BeforeSoftRemove,
  AfterSoftRemove,
  AfterLoad,
} from "typeorm";

@Entity("users")
export class User {
  @PrimaryGeneratedColumn()
  id: number;

  @Column()
  name: string;

  @Column()
  email: string;

  @Column({ nullable: true })
  passwordHash: string;

  password: string; // 临时字段，不存储到数据库

  // ========== Insert 监听器 ==========

  @BeforeInsert()
  beforeInsert() {
    console.log(`即将插入用户：${this.name}`);
    // 在这里可以设置默认值、加密密码等
    if (this.password) {
      this.passwordHash = hashPassword(this.password);
    }
  }

  @AfterInsert()
  afterInsert() {
    console.log(`用户已创建，ID：${this.id}`);
    // 可以发送欢迎邮件、记录日志等
  }

  // ========== Update 监听器 ==========

  @BeforeUpdate()
  beforeUpdate() {
    console.log(`即将更新用户：${this.id}`);
    if (this.password) {
      this.passwordHash = hashPassword(this.password);
    }
  }

  @AfterUpdate()
  afterUpdate() {
    console.log(`用户已更新：${this.id}`);
  }

  // ========== Remove 监听器 ==========

  @BeforeRemove()
  beforeRemove() {
    console.log(`即将删除用户：${this.id}`);
  }

  @AfterRemove()
  afterRemove() {
    console.log(`用户已删除：${this.id}`);
    // 清理关联资源
  }

  // ========== Soft Remove 监听器 ==========

  @BeforeSoftRemove()
  beforeSoftRemove() {
    console.log(`即将软删除用户：${this.id}`);
  }

  @AfterSoftRemove()
  afterSoftRemove() {
    console.log(`用户已软删除：${this.id}`);
  }

  // ========== Load 监听器 ==========

  @AfterLoad()
  afterLoad() {
    // 每次从数据库加载后执行
    // 可以计算虚拟属性、格式化数据等
    this.name = this.name?.trim();
  }
}

function hashPassword(password: string): string {
  // 实际项目中使用 bcrypt
  return `hashed_${password}`;
}
```

### 实战：自动密码加密

```typescript
import * as bcrypt from "bcrypt";

@Entity("users")
export class User {
  @Column({ select: false })
  passwordHash: string;

  // 临时属性，不映射到数据库
  private _password: string;

  set password(value: string) {
    this._password = value;
  }

  @BeforeInsert()
  @BeforeUpdate()
  async hashPassword() {
    if (this._password) {
      this.passwordHash = await bcrypt.hash(this._password, 10);
    }
  }

  // 验证密码
  async validatePassword(password: string): Promise<boolean> {
    return bcrypt.compare(password, this.passwordHash);
  }
}
```

## 嵌入式实体

当一个数据结构在多个实体中重复使用时，可以使用嵌入式实体来复用。

### 定义嵌入式实体

```typescript
import { Column, CreateDateColumn, UpdateDateColumn } from "typeorm";

// 嵌入式实体：用户地址
export class Address {
  @Column({ name: "street" })
  street: string;

  @Column({ name: "city" })
  city: string;

  @Column({ name: "state" })
  state: string;

  @Column({ name: "zip_code" })
  zipCode: string;

  @Column({ name: "country" })
  country: string;
}

// 嵌入式实体：审计字段
export class AuditFields {
  @Column({ name: "created_by", nullable: true })
  createdBy: string;

  @Column({ name: "updated_by", nullable: true })
  updatedBy: string;

  @CreateDateColumn({ name: "created_at" })
  createdAt: Date;

  @UpdateDateColumn({ name: "updated_at" })
  updatedAt: Date;
}
```

### 使用嵌入式实体

```typescript
import { Entity, PrimaryGeneratedColumn, Column } from "typeorm";
import { Embedded } from "typeorm";
import { Address, AuditFields } from "./embeddables";

@Entity("users")
export class User {
  @PrimaryGeneratedColumn()
  id: number;

  @Column()
  name: string;

  // 嵌入地址
  @Embedded(() => Address)
  address: Address;

  // 嵌入审计字段
  @Embedded(() => AuditFields)
  audit: AuditFields;
}

@Entity("companies")
export class Company {
  @PrimaryGeneratedColumn()
  id: number;

  @Column()
  name: string;

  // 嵌入地址（复用）
  @Embedded(() => Address)
  address: Address;

  // 嵌入审计字段（复用）
  @Embedded(() => AuditFields)
  audit: AuditFields;
}
```

### 嵌入式实体的前缀

```typescript
// 默认：列名为嵌入属性名 + 嵌入列名
// 如：addressStreet, addressCity

// 自定义前缀
@Embedded(() => Address, { prefix: "addr_" })
address: Address;
// 列名：addr_street, addr_city

// 无前缀
@Embedded(() => Address, { prefix: false })
address: Address;
// 列名：street, city
```

### 1. 实体文件组织

```typescript
// ✅ 推荐：每个实体一个文件
// src/entity/User.ts
// src/entity/Post.ts
// src/entity/Comment.ts

// ✅ 推荐：使用 index.ts 统一导出
// src/entity/index.ts
export { User } from "./User";
export { Post } from "./Post";

```

### 2. 命名规范

```typescript
// ✅ 推荐
@Entity("users") // 表名：复数、snake_case
export class User {
  // 类名：单数、PascalCase
  @Column({ name: "first_name" }) // 列名：snake_case
  firstName: string; // 属性名：camelCase
}

// ❌ 不推荐
@Entity("User") // 表名不应该用 PascalCase
export class user {
  // 类名不应该用小写
  @Column({ name: "FirstName" }) // 列名不应该用 PascalCase
  FirstName: string;
}
```

### 3. 敏感字段保护

```typescript
@Entity("users")
export class User {
  // 密码不返回给前端
  @Column({ select: false })
  passwordHash: string;

  // 使用 DTO 控制返回数据
  // 不要直接返回实体对象
}
```

### 4. 枚举使用

```typescript
// ✅ 推荐：使用 TypeScript 枚举
export enum PostStatus {
  DRAFT = "DRAFT",
  PUBLISHED = "PUBLISHED",
  ARCHIVED = "ARCHIVED",
}

@Column({ type: "enum", enum: PostStatus, default: PostStatus.DRAFT })
status: PostStatus;

// ❌ 不推荐：使用联合类型字符串
@Column({ type: "enum", enum: ["DRAFT", "PUBLISHED", "ARCHIVED"] })
status: string;
```

## 总结

### 核心要点

1. **装饰器体系**
   - `@Entity`：定义表
   - `@Column`：定义列
   - `@PrimaryGeneratedColumn`：定义主键
   - `@CreateDateColumn` / `@UpdateDateColumn`：时间戳

2. **数据类型映射**
   - TypeScript 类型自动映射到 SQL 类型
   - 可以通过 `type` 选项显式指定
   - 支持数组、JSON、枚举等 PostgreSQL 特有类型

3. **主键策略**
   - 自增：`@PrimaryGeneratedColumn()`
   - UUID：`@PrimaryGeneratedColumn("uuid")`
   - 复合主键：多个 `@PrimaryColumn()`

4. **高级特性**
   - 软删除：`@DeleteDateColumn`
   - 乐观锁：`@VersionColumn`
   - 实体监听器：`@BeforeInsert`、`@AfterLoad` 等
   - 嵌入式实体：`@Embedded`
