在 Next.js 13 引入 App Router 后，Server Actions 成为了最受关注的新特性之一。它彻底改变了我们处理表单提交、数据变更和服务端逻辑的方式。本文将深入探讨 Server Actions 的核心概念、实战技巧和最佳实践。

## 什么是 Server Actions？

Server Actions 是 Next.js 提供的一种在服务端执行异步函数的机制。它允许你直接在组件中定义服务端逻辑，无需创建额外的 API 路由。

### 核心优势

1. **类型安全**：与 TypeScript 完美集成，端到端类型推导
2. **简化架构**：无需手动创建 API 路由
3. **自动重新验证**：数据变更后自动更新 UI
4. **渐进增强**：即使 JavaScript 禁用也能工作
5. **安全性**：敏感逻辑保持在服务端

## 基础用法

### 定义 Server Action

在 Next.js 中，使用 `"use server"` 指令标记服务端函数：

```typescript
// app/actions.ts
"use server";

import { revalidatePath } from "next/cache";
import { db } from "@/lib/db";

export async function createPost(formData: FormData) {
  const title = formData.get("title") as string;
  const content = formData.get("content") as string;

  await db.post.create({
    data: { title, content },
  });

  revalidatePath("/posts");
}
```

### 在组件中使用

Server Actions 可以直接在表单中使用：

```typescript
// app/posts/new/page.tsx
import { createPost } from "../actions";

export default function NewPostPage() {
  return (
    <form action={createPost}>
      <input name="title" type="text" placeholder="标题" required />
      <textarea name="content" placeholder="内容" required />
      <button type="submit">发布文章</button>
    </form>
  );
}
```

## 表单状态管理

### 使用 useFormStatus

跟踪表单提交状态：

### 使用 useFormState

处理表单返回状态和错误：

```typescript
"use client";

import { useFormState } from "react-dom";
import { createPost } from "./actions";

const initialState = {
  message: "",
  success: false,
};

export default function NewPostForm() {
  const [state, formAction] = useFormState(createPost, initialState);

  return (
    <form action={formAction}>
      <input name="title" type="text" required />
      <textarea name="content" required />

      {!state.success && state.message && (
        <p className="error">{state.message}</p>
      )}

      <button type="submit">发布</button>
    </form>
  );
}
```

对应的 Server Action：

```typescript
"use server";

import { revalidatePath } from "next/cache";

export async function createPost(
  prevState: { message: string; success: boolean },
  formData: FormData
) {
  try {
    const title = formData.get("title") as string;
    const content = formData.get("content") as string;

    if (!title || title.length &lt; 3) {
      return { message: "标题至少需要 3 个字符", success: false };
    }

    await db.post.create({
      data: { title, content },
    });

    revalidatePath("/posts");
    return { message: "发布成功！", success: true };
  } catch (error) {
    return { message: "发布失败，请重试", success: false };
  }
}
```

## 数据验证

### 使用 Zod 进行类型安全验证

```typescript
"use server";

import { z } from "zod";
import { revalidatePath } from "next/cache";

const postSchema = z.object({
  title: z.string().min(3).max(100),
  content: z.string().min(10),
  tags: z.array(z.string()).optional(),
});

export async function createPost(formData: FormData) {
  const rawData = {
    title: formData.get("title"),
    content: formData.get("content"),
    tags: formData.getAll("tags"),
  };

  const validatedFields = postSchema.safeParse(rawData);

  if (!validatedFields.success) {
    return {
      errors: validatedFields.error.flatten().fieldErrors,
      message: "验证失败",
    };
  }

  const { title, content, tags } = validatedFields.data;

  await db.post.create({
    data: { title, content, tags },
  });

  revalidatePath("/posts");
}
```

## 乐观更新

使用 `useOptimistic` 实现即时 UI 反馈：

```typescript
"use client";

import { useOptimistic } from "react";
import { createPost } from "./actions";

type Post = {
  id: string;
  title: string;
  content: string;
};

export function PostList({ posts }: { posts: Post[] }) {
  const [optimisticPosts, addOptimisticPost] = useOptimistic(
    posts,
    (state, newPost: Post) => [...state, newPost]
  );

  async function handleSubmit(formData: FormData) {
    const tempId = `temp-${Date.now()}`;

    addOptimisticPost({
      id: tempId,
      title: formData.get("title") as string,
      content: formData.get("content") as string,
    });

    await createPost(formData);
  }

  return (
    <>
      <ul>
        {optimisticPosts.map((post) => (
          <li key={post.id}>
            <h3>{post.title}</h3>
            <p>{post.content}</p>
          </li>
        ))}
      </ul>

      <form action={handleSubmit}>
        <input name="title" type="text" required />
        <textarea name="content" required />
        <button type="submit">发布</button>
      </form>
    </>
  );
}
```

## 缓存重新验证

### revalidatePath vs revalidateTag

```typescript
"use server";

import { revalidatePath, revalidateTag } from "next/cache";

// 重新验证特定路径
export async function updatePost(id: string, formData: FormData) {
  await db.post.update({
    where: { id },
    data: { title: formData.get("title") },
  });

  revalidatePath("/posts");
  revalidatePath(`/posts/${id}`);
}

// 使用标签重新验证
export async function deletePost(id: string) {
  await db.post.delete({ where: { id } });

  // 重新验证所有标记为 "posts" 的缓存
  revalidateTag("posts");
}
```

在数据获取时添加标签：

## 安全最佳实践

### 身份验证检查

```typescript
"use server";

import { auth } from "@/lib/auth";
import { redirect } from "next/navigation";

export async function createPost(formData: FormData) {
  const session = await auth();

  if (!session?.user) {
    redirect("/login");
  }

  // 执行创建操作
  await db.post.create({
    data: {
      title: formData.get("title") as string,
      content: formData.get("content") as string,
      authorId: session.user.id,
    },
  });

  revalidatePath("/posts");
}
```

### 速率限制

```typescript
// lib/rate-limit.ts
import { headers } from "next/headers";

const rateLimit = new Map<string, { count: number; resetAt: number }>();

export async function checkRateLimit(
  identifier: string,
  limit: number = 10,
  windowMs: number = 60000
) {
  const now = Date.now();
  const record = rateLimit.get(identifier);

  if (!record || now &gt; record.resetAt) {
    rateLimit.set(identifier, { count: 1, resetAt: now + windowMs });
    return true;
  }

  if (record.count &gt;= limit) {
    return false;
  }

  record.count++;
  return true;
}
```

在 Server Action 中使用：

```typescript
"use server";

import { checkRateLimit } from "@/lib/rate-limit";
import { headers } from "next/headers";

export async function submitContact(formData: FormData) {
  const headersList = headers();
  const ip = headersList.get("x-forwarded-for") || "unknown";

  if (!(await checkRateLimit(ip, 5, 60000))) {
    return { message: "请求过于频繁，请稍后再试", success: false };
  }

  // 处理表单提交
}
```

## 性能优化技巧

### 1. 并行执行多个 Server Actions

```typescript
"use server";

export async function initializeUser(userId: string) {
  // 并行执行多个操作
  const [profile, preferences, notifications] = await Promise.all([
    createUserProfile(userId),
    setupUserPreferences(userId),
    subscribeToNotifications(userId),
  ]);

  revalidateTag(`user-${userId}`);

  return { profile, preferences, notifications };
}
```

### 2. 减少重新验证范围

```typescript
// ❌ 不好：重新验证整个页面
revalidatePath("/dashboard");

// ✅ 更好：只重新验证特定路径
revalidatePath("/dashboard/stats");

// ✅ 最好：使用标签精确控制
revalidateTag("dashboard-stats");
```

### 3. 使用 Server Actions 进行数据预取

```typescript
// app/layout.tsx
import { getGlobalData } from "./actions";

export default async function RootLayout({
  children,
}: {
  children: React.ReactNode;
}) {

  const globalData = await getGlobalData();

  return (
    <html lang="zh">
      <body>
        <GlobalContext.Provider value={globalData}>
          {children}
        </GlobalContext.Provider>
```

## 实战案例：完整的 CRUD 操作

```typescript
// app/todos/actions.ts
"use server";

import { revalidatePath } from "next/cache";
import { z } from "zod";
import { auth } from "@/lib/auth";
import { db } from "@/lib/db";

const todoSchema = z.object({
  title: z.string().min(1).max(100),
  completed: z.boolean().optional(),
});

export async function getTodos() {
  const session = await auth();
  if (!session?.user) return [];

  return db.todo.findMany({
    where: { userId: session.user.id },
    orderBy: { createdAt: "desc" },
  });
}

export async function createTodo(formData: FormData) {
  const session = await auth();
  if (!session?.user) {
    return { error: "未授权" };
  }

  const validatedFields = todoSchema.safeParse({
    title: formData.get("title"),
  });

  if (!validatedFields.success) {
    return { error: "标题无效" };
  }

  await db.todo.create({
    data: {
      title: validatedFields.data.title,
      userId: session.user.id,
    },
  });

  revalidatePath("/todos");
  return { success: true };
}

export async function toggleTodo(id: string, completed: boolean) {
  await db.todo.update({
    where: { id },
    data: { completed },
  });

  revalidatePath("/todos");
}

export async function deleteTodo(id: string) {
  await db.todo.delete({
    where: { id },
  });

  revalidatePath("/todos");
}
```

## 常见陷阱与解决方案

### 陷阱 1：忘记添加 "use server"

```typescript
// ❌ 错误：缺少指令
export async function myAction() {
  // 这个函数会在客户端执行！
}

// ✅ 正确
("use server");
export async function myAction() {
  // 现在确保在服务端执行
}
```

### 陷阱 2：在 Server Action 中使用客户端钩子

```typescript
// ❌ 错误
"use server";
import { useState } from "react"; // 不能在 Server Action 中使用

export async function myAction() {
  const [count, setCount] = useState(0); // 报错！
}

// ✅ 正确：将状态管理移到客户端组件
```

### 陷阱 3：序列化问题

Server Actions 只能返回可序列化的数据：

```typescript
// ❌ 错误：返回不可序列化的对象
export async function myAction() {
  return {
    date: new Date(), // Date 对象不能直接序列化
    fn: () => {}, // 函数不能序列化
  };
}

// ✅ 正确
export async function myAction() {
  return {
    date: new Date().toISOString(), // 转换为字符串
    data: { key: "value" }, // 纯对象可以
  };
}
```

## 与传统 API 路由对比

| 特性     | Server Actions    | API Routes         |
| -------- | ----------------- | ------------------ |
| 类型安全 | ✅ 端到端类型推导 | ⚠️ 需手动定义类型  |
| 代码组织 | ✅ 与组件放在一起 | ⚠️ 需单独文件      |
| 表单处理 | ✅ 原生支持       | ⚠️ 需手动解析      |
| 渐进增强 | ✅ 自动支持       | ❌ 依赖 JavaScript |
| 缓存控制 | ✅ 自动重新验证   | ⚠️ 需手动管理      |
| 适用场景 | 数据变更、表单    | 公开 API、Webhook  |

## 总结

Server Actions 代表了 Next.js 全栈开发的未来方向。它通过简化数据交互流程、提供类型安全和自动缓存管理，大幅提升了开发效率。主要收获：

1. **优先使用 Server Actions 处理数据变更**，保留 API Routes 用于公开接口
2. **结合 Zod 进行数据验证**，确保类型安全
3. **使用乐观更新提升用户体验**，即时反馈操作结果
4. **注意安全最佳实践**，包括身份验证、CSRF 保护和速率限制
5. **精确控制缓存重新验证**，避免不必要的性能开销

随着 Next.js 的持续演进，Server Actions 将成为构建现代 Web 应用的标准方案。现在就是掌握这项技术的最佳时机！
