# 读书笔记系列 — 重写计划

> **For Hermes:** 按 writing-plans 方法论重写全部6篇。每篇的 bite-sized tasks 见下方。

**目标：** 把6篇AI味的"伪个人笔记"重写成真正的个人读书笔记——像写给自己看的日记，不是写给读者的文章。

**核心问题：** 现有文章质量不差，但"太完美了"——太工整、太全面、太像一篇"优秀的读后感范文"，反而失去了个人笔记应有的真实感。

---

## 一、问题诊断

### 总体问题

1. **结构千篇一律** — 每篇都是"读之前我以为...读之后我意识到..." → 3-4个加粗金句 → "今天你可以做的一件事" → trade-off结尾。6篇读完，结构记忆点为零。
2. **个人经历"太正确"** — 每个个人故事都有一个干净的教训。真实生活不是这样的——有时候你读完一本书什么都没改变，有时候过了半年才突然理解某个观点。
3. **缺少脆弱和矛盾** — 没有不确定、没有困惑、没有"我可能想错了"。所有观点都表达得非常笃定。
4. **加粗金句太多太密** — 每篇4-5个加粗总结句，像是AI在帮你"提炼要点"。真正的个人笔记不会这样。
5. **面面俱到** — 每本书都覆盖了3-5个核心概念，但每个都只触到表面。个人笔记应该只深入1-2个真正触动你的点。
6. **"前端开发者"人设太刻意** — 几乎每篇都要硬插一段"作为一个程序员/前端开发"的类比，有些自然有些勉强。
7. **AI痕迹明显** — 排比句式、工整的过渡、"这个道理在XX也适用"的万能桥段。

### 每篇具体问题

**01 明朝那些事儿**

- 问题：朱元璋→崇祯→于谦，三个人物平铺直叙，缺乏主线。像在复述书的内容。
- 亮点："崇祯的无力感"那段有真实的共鸣。
- 关键缺失：没有回答"为什么是这本书成为系列第一篇"。

**02 投资最重要的事**

- 问题：第二层思维→不确定性→周期→风险控制，4个概念平分秋色，缺乏聚焦。疫情卖基金的例子是唯一有真实感的部分。
- 亮点：开头的"然后我就亏了"很真实。
- 关键缺失：没有写出"被市场教育"的具体痛感，太轻描淡写了。

**03 一句顶一万句**

- 问题：全书最好的文学解读，但"微信好友上千人说得上话的不超过十个"这个观察太普通了，任何人都说得出来。
- 亮点：对杨百顺和牛爱国的解读很精准。
- 关键缺失：缺少个人的孤独体验。你什么时候感到过"说不上话"？

**04 非暴力沟通**

- 问题：最像"书评"的一篇。四个步骤逐个讲解，像在教NVC。个人笔记不应该教别人怎么沟通。
- 亮点：和产品经理的冲突那段是全文最有价值的部分。
- 关键缺失：应该只聚焦"观察vs评判"这一个转变，不要四个步骤都讲。

**05 认知觉醒**

- 问题：三脑模型+元认知+舒适区边缘+手机依赖，4个概念又是平铺。本能脑/情绪脑/理智脑的解释太科普了，像在写教科书。
- 亮点：手机放到另一个房间的策略很具体实用。
- 关键缺失：没有写"改变的过程"——只写了"知道"和"做到"，缺少中间那个笨拙的、反复的、失败的过程。

**06 纳瓦尔宝典**

- 问题：判断力→专属知识→财富vs金钱→杠杆→幸福，5个概念像流水账。"编程+心理学+理财=专属知识"这个自我分析太理想化了。
- 亮点：开头的播客故事简洁有力。
- 关键缺失：没有写Naval那些让你不舒服的观点——比如"你不应该为了地位工作"，读到这里你真的服气吗？

## 二、重写原则

### 不是"怎么写好读后感"，而是"怎么写真正的个人笔记"

1. **只写真正触动你的1-2个点**，不要试图覆盖全书精华。
2. **允许不完整** — 可以只写一个片段、一个想法、一个困惑。
3. **写过程，不写结论** — "我读到这一段的时候，第一反应是拒绝的...后来某天突然想通了..." 比直接给结论好。
4. **保留矛盾** — "我觉得他说得对，但同时又觉得哪里不对..."
5. **砍掉80%的加粗金句** — 最多留1-2个，或者完全不加粗。
6. **不要每篇都"作为一个程序员"** — 只在自然联想到的时候提，不要硬插。
7. **不同的书用不同的结构** — 小说可以只写一个人物的感受；投资书可以写一个具体的决策反思；心理学书可以写一次尝试和失败。
8. **写得短一点** — 1500-2500字足够。比现在短，但更密集。
9. **结尾不要总结这本书适合谁** — 你是在写日记，不是在写豆瓣评分。
10. **开头不要解释"为什么要写读书笔记"** — 第1篇说一次就够了，2-6篇直接进入。

### 结构差异化方案

| 篇目      | 建议结构                                               | 理由                              |
| --------- | ------------------------------------------------------ | --------------------------------- |
| 01 明朝   | 时间线式：买书→读到一半→读完全部→半年后的回响          | 让读者跟着你的阅读过程走          |
| 02 投资   | 聚焦式：只写"第二层思维"这一个概念，用投资亏损经历贯穿 | 一篇只深挖一个点                  |
| 03 一句   | 对话式：用书中一段对话触发自己的回忆                   | 文学作品适合情感驱动              |
| 04 非暴力 | 反思式：写一次失败的沟通尝试 + 一次成功的              | 对比比讲解更有力                  |
| 05 认知   | 日记式：用一周的时间线写尝试元认知的过程               | 用过程代替科普                    |
| 06 纳瓦尔 | 对抗式：写你同意和不同意的观点                         | Naval的观点本身就有争议性，利用它 |

## 三、每篇重写方案

### Task 1: 重写 01-明朝那些事儿

**原文问题：** 三个历史人物平铺，缺乏个人阅读历程。

**保留：** 崇祯的无力感（这是真正有共鸣的部分）
**砍掉：** 于谦的部分（和程序员类比太勉强）、"历史对普通人的用处"这个总结段
**新增：** 你的阅读过程——7本书怎么啃下来的？什么时候差点放弃？哪一段读哭了？

**建议开头：**

> 这套书7本，摞起来有半个人高。我买的时候犹豫了很久，因为封面上写着"当年明月"，我以为是那种网文改的书。结果从翻开第一页到读完最后一页，用了两个月，中间有好几次读到凌晨两点不想睡。

**建议结构：**

1. 买书的犹豫 → 翻开第一页的惊喜
2. 朱元璋：读的时候一直在想的那个问题
3. 崇祯：整本书最难受的部分（聚焦这一段）
4. 最后一段话（当年明月的结尾）给你的震撼
5. 结尾：这本书改变了什么（简短）

**Frontmatter不变。**

---

### Task 2: 重写 02-投资最重要的事

**原文问题：** 4个概念平分，缺乏痛感。

**保留：** "然后我就亏了"这个开头、疫情卖基金的例子
**砍掉：** 周期思维的泛泛而谈、风险控制的教科书式讲解、"作为一个程序员"的技术选型类比
**新增：** 亏钱的具体感受——亏了多少？当时什么心情？什么时候意识到自己错了？

**建议开头：** 保留原文的"然后我就亏了"，但往下写具体亏了多少、当时的真实感受。

**建议结构（聚焦式）：**

1. 开头：亏钱的经历（真实、具体）
2. 读完书之后的反思：亏钱的真正原因不是"买错了"，是"以为自己对"
3. "第二层思维"：这一个概念怎么改变了我看市场的方式
4. 一个具体的投资决策：读之前会怎么做 vs 读之后会怎么做
5. 结尾：简短，不讲大道理

---

### Task 3: 重写 03-一句顶一万句

**原文问题：** 书评味重，个人孤独体验太普通。

**保留：** 对杨百顺/牛爱国"说不上话"的解读、"精确到有点残忍"这个评价
**砍掉：** "微信好友上千人"这个烂大街的观察、"为什么通讯工具越来越发达"的泛泛议论
**新增：** 一个你自己的"说不上话"的真实时刻——不是泛泛的"有时觉得孤独"，而是某个具体的晚上/某次对话/某条发出去没人回复的消息

**建议开头：**

> 刘震云写完这本书，得了茅盾文学奖。但我读的时候根本没在想"文学"，我想的是某天晚上——具体哪天记不清了——我坐在工位上加班到九点多，发了一条朋友圈，过了半小时只有三个赞，没有一个人评论。删了又重发，还是一样。

**建议结构（情感驱动）：**

1. 一个"说不上话"的个人时刻
2. 然后读这本书：杨百顺的经历怎么精确地戳中了你
3. 书中某个具体的段落（引用原文）让你停下来想了很久
4. "说得上话"到底是什么？不是聊天，是...
5. 结尾：简短，甚至可以是一个未完成的疑问

---

### Task 4: 重写 04-非暴力沟通

**原文问题：** 四个步骤逐个讲解，像在教NVC，不像个人笔记。

**保留：** 和产品经理冲突的那段（最真实的部分）、观察vs评判的区分
**砍掉：** 四步骤的完整讲解（只保留"观察vs评判"和"感受vs想法"）、"请求vs命令"整段、和同事提优化需求的例子
**新增：** 尝试NVC之后的失败经历（不是每次都成功的）、读这本书之前你以为自己沟通能力还行，读完才发现一直在"暴力沟通"的震惊

**建议开头：**

> 读完这本书的第一反应不是"我学到了"，是"卧槽，我一直都是这样说话的吗"。

**建议结构（对比式）：**

1. 震惊：原来我的日常表达全是评判
2. 一个具体的例子：写出你以前会怎么说 vs NVC会怎么说（用你和家人/同事的真实对话）
3. 第一次尝试：结果怎么样？（写一次成功的，也写一次失败的）
4. 最大的改变不是话术，是...
5. 结尾：简短

---

### Task 5: 重写 05-认知觉醒

**原文问题：** 三脑模型太科普，缺乏"改变的过程"。

**保留：** 手机放到另一个房间的具体策略
**砍掉：** 三脑模型的详细解释（太长了，像科普文）、"本能脑3.6亿年、情绪脑2亿年"这些数字、舒适区边缘的完整讲解
**新增：** 读完书之后的一周里，尝试用元认知的真实过程——什么时候成功了、什么时候失败了、什么时候忘了这回事

**建议开头：**

> 读完这本书的那天晚上，我照例刷手机到一点半。

（然后写：明明刚读完一本教你控制自己的书，你还是控制不了自己。这个反差就是最好的开头。）

**建议结构（过程式）：**

1. 读完书的当晚：照旧刷手机到一点半（反差感）
2. 但第二天有点不一样了：某个瞬间突然"看到"了自己在干嘛
3. 一周的尝试：用元认知观察自己的记录（可以是几条简短的日记）
4. 最大的收获不是"控制了自己"，而是"不再骂自己了"
5. 结尾：简短

---

### Task 6: 重写 06-纳瓦尔宝典

**原文问题：** 5个概念流水账，"编程+心理学+理财"的自我分析太理想化。

**保留：** 播客开头（简洁有力）、杠杆=代码和媒体的部分
**砍掉：** 判断力的长篇解释、专属知识的"自我分析"段落、财富vs金钱vs地位的教科书式区分
**新增：** Naval让你不舒服的观点——你不完全同意的那些部分；读这本书时的"等等，真的吗？"时刻

**建议开头：** 保留原文的播客开头。

**建议结构（对抗式）：**

1. 播客开头（保留）
2. 让你拍案叫绝的观点：杠杆=代码+媒体（作为程序员这太直观了）
3. 让你犹豫的观点："幸福是可以练习的技能"——真的吗？你试了吗？
4. 让你不同意的地方（写具体）
5. 读完全书之后你在想的一个问题
6. 结尾：第06篇作为系列收尾，可以简短回顾这6本书——但不要总结"它们都指向同一个问题"，太假了。写一个真实的困惑就好。

## 五、质量标准

重写后的文章必须通过以下检查：

- [ ] 拿掉书名，文章仍然有辨识度（不像任何人都能写出来的读后感）
- [ ] 有至少一个具体的、有细节的个人故事（不是"我有个朋友"式的万能模板）
- [ ] 允许有不确定、矛盾、或者"我不知道"的时刻
- [ ] 加粗金句不超过2个（最好是0-1个）
- [ ] 没有面面俱到地复述书的内容
- [ ] "作为一个程序员/前端开发"最多出现1次，或者不出现
- [ ] 结尾没有"适合谁读/不适合谁读"的评价
- [ ] 6篇文章的结构各不相同
- [ ] 总字数1500-2500字（比现在短）
