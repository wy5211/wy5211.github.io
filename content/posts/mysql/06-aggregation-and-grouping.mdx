## 前言

在实际业务中，我们经常需要对数据进行统计分析：计算总销售额、平均客单价、用户增长趋势等。这些都需要用到聚合函数和分组查询。

本文将带你掌握 MySQL 的数据统计功能，让你能轻松处理各种统计需求。

## 聚合函数

MySQL 提供了 5 个核心聚合函数：

| 函数  | 说明   | 示例        |
| ----- | ------ | ----------- |
| COUNT | 计数   | COUNT(\*)   |
| SUM   | 求和   | SUM(amount) |
| AVG   | 平均值 | AVG(price)  |
| MAX   | 最大值 | MAX(score)  |
| MIN   | 最小值 | MIN(score)  |

### COUNT - 计数

```sql
-- COUNT(*)：统计所有行（包括 NULL）
SELECT COUNT(*) FROM users;              -- 总用户数

-- COUNT(字段)：统计非 NULL 值
SELECT COUNT(email) FROM users;          -- 有邮箱的用户数

-- COUNT(DISTINCT)：去重计数
SELECT COUNT(DISTINCT city) FROM users;  -- 有多少个城市
SELECT COUNT(DISTINCT user_id) FROM orders;  -- 有多少人下过单

-- 配合 GROUP BY 使用
SELECT
  city,
  COUNT(*) AS user_count
FROM users
GROUP BY city;
```

### SUM - 求和

```sql
-- 基本用法
SELECT SUM(total_amount) FROM orders;     -- 总销售额

-- 配合 WHERE 条件
SELECT SUM(total_amount) FROM orders WHERE status = 'completed';

-- 配合 GROUP BY
SELECT
  user_id,
  SUM(total_amount) AS total_spent
FROM orders
GROUP BY user_id;

-- SUM 会忽略 NULL 值
SELECT SUM(score) FROM students;  -- 自动跳过 NULL
```

### AVG - 平均值

```sql
-- 平均客单价
SELECT AVG(total_amount) FROM orders;

-- 各城市平均消费
SELECT
  u.city,
  AVG(o.total_amount) AS avg_amount
FROM users u
INNER JOIN orders o ON u.id = o.user_id
GROUP BY u.city;

-- 保留小数位
SELECT
  ROUND(AVG(price), 2) AS avg_price
FROM products;
```

### MAX/MIN - 最大/最小值

```sql
-- 最高订单金额
SELECT MAX(total_amount) FROM orders;

-- 最便宜的商品
SELECT MIN(price) FROM products;

-- 各用户最大单笔消费
SELECT
  user_id,
  MAX(total_amount) AS max_order_amount
FROM orders
GROUP BY user_id;

-- 查找最大值的对应记录（子查询）
SELECT * FROM orders
WHERE total_amount = (SELECT MAX(total_amount) FROM orders);
```

## GROUP BY - 分组

### 基本语法

```sql
-- 语法
SELECT 字段1, 聚合函数 FROM 表名 GROUP BY 字段1;

-- 示例：按城市统计用户数
SELECT
  city,
  COUNT(*) AS user_count
FROM users
GROUP BY city;

-- 结果：
-- +--------+------------+
-- | city   | user_count |
-- +--------+------------+
-- | 北京   |         10 |
-- | 上海   |         15 |
-- | 深圳   |          8 |
-- +--------+------------+
```

### 多字段分组

```sql
-- 按城市和状态分组
SELECT
  city,
  status,
  COUNT(*) AS user_count
FROM users
GROUP BY city, status;

-- 逻辑：先按 city 分组，再按 status 分组
-- 等价于：GROUP BY city, status
```

### GROUP BY 别名

```sql
-- MySQL 5.7.5+ 支持别名
SELECT
  DATE(created_at) AS order_date,
  COUNT(*) AS order_count,
  SUM(total_amount) AS total_amount
FROM orders
GROUP BY order_date;  -- 使用别名

-- MySQL 5.7.5 之前需要重复表达式
SELECT
  DATE(created_at) AS order_date,
  COUNT(*) AS order_count
FROM orders
GROUP BY DATE(created_at);
```

## HAVING - 过滤分组

HAVING 用于过滤 GROUP BY 后的结果，类似 WHERE，但作用于分组。

```sql
-- WHERE vs HAVING
-- WHERE：分组前过滤（不能用聚合函数）
-- HAVING：分组后过滤（可以用聚合函数）

-- 示例 1：查找消费总额 > 10000 的用户
SELECT
  user_id,
  SUM(total_amount) AS total_spent
FROM orders
GROUP BY user_id
HAVING total_spent > 10000;

-- 示例 2：订单数 > 5 的用户
SELECT
  user_id,
  COUNT(*) AS order_count
FROM orders
GROUP BY user_id
HAVING order_count > 5;

-- 示例 3：WHERE + HAVING 组合
SELECT
  user_id,
  COUNT(*) AS order_count,
  SUM(total_amount) AS total_amount
FROM orders
WHERE status = 'completed'  -- 先过滤已完成的订单
GROUP BY user_id
HAVING total_amount > 1000;  -- 再过滤金额 > 1000 的用户
```

## GROUP_CONCAT - 分组合并

将分组内的值合并成字符串。

```sql
-- 基本用法
SELECT
  user_id,
  GROUP_CONCAT(order_no) AS order_list
FROM orders
GROUP BY user_id;

-- 结果：user_id=1, order_list='ORD001,ORD002,ORD003'

-- 自定义分隔符
SELECT
  user_id,
  GROUP_CONCAT(order_no SEPARATOR ' | ') AS order_list
FROM orders
GROUP BY user_id;

-- 排序后再合并
SELECT
  user_id,
  GROUP_CONCAT(order_no ORDER BY created_at DESC) AS order_list
FROM orders
GROUP BY user_id;

-- 限制长度
SELECT
  user_id,
  GROUP_CONCAT(order_no ORDER BY created_at DESC SEPARATOR '|' LIMIT 3) AS recent_orders
FROM orders
GROUP BY user_id;
```

## 实战案例

### 销售统计

```sql
-- 1. 每日销售额统计
SELECT
  DATE(created_at) AS order_date,
  COUNT(*) AS order_count,
  SUM(total_amount) AS daily_sales,
  AVG(total_amount) AS avg_order_value
FROM orders
WHERE status = 'completed'
GROUP BY DATE(created_at)
ORDER BY order_date DESC;

-- 2. 各城市销售排名
SELECT
  u.city,
  COUNT(DISTINCT o.user_id) AS customer_count,
  COUNT(o.id) AS order_count,
  SUM(o.total_amount) AS total_sales
FROM users u
INNER JOIN orders o ON u.id = o.user_id
WHERE o.status = 'completed'
GROUP BY u.city
ORDER BY total_sales DESC;

-- 3. 用户消费排行（Top 10）
SELECT
  u.id,
  u.username,
  COUNT(o.id) AS order_count,
  SUM(o.total_amount) AS total_spent
FROM users u
INNER JOIN orders o ON u.id = o.user_id
WHERE o.status = 'completed'
GROUP BY u.id, u.username
ORDER BY total_spent DESC
LIMIT 10;
```

### 用户分析

```sql
-- 1. 用户注册趋势（按月）
SELECT
  DATE_FORMAT(created_at, '%Y-%m') AS month,
  COUNT(*) AS new_users
FROM users
GROUP BY DATE_FORMAT(created_at, '%Y-%m')
ORDER BY month DESC;

-- 2. 用户状态分布
SELECT
  CASE
    WHEN status = 1 THEN '正常'
    WHEN status = 0 THEN '禁用'
    ELSE '未知'
  END AS status_text,
  COUNT(*) AS user_count
FROM users
GROUP BY status
ORDER BY user_count DESC;

-- 3. 用户分层（RFM 模型简化版）
SELECT
  CASE
    WHEN last_order_days <= 30 AND order_count >= 5 THEN '核心用户'
    WHEN last_order_days <= 90 AND order_count >= 3 THEN '活跃用户'
    WHEN last_order_days > 90 THEN '流失用户'
    ELSE '新用户'
  END AS user_segment,
  COUNT(*) AS user_count
FROM (
  SELECT
    u.id,
    DATEDIFF(NOW(), MAX(o.created_at)) AS last_order_days,
    COUNT(o.id) AS order_count
  FROM users u
  LEFT JOIN orders o ON u.id = o.user_id
  GROUP BY u.id
) t
GROUP BY user_segment;
```

## WITH ROLLUP - 超级聚合

生成汇总行。

```sql
-- 基本用法
SELECT
  city,
  status,
  COUNT(*) AS user_count
FROM users
GROUP BY city, status WITH ROLLUP;

-- 结果：
-- +--------+--------+------------+
-- | city   | status | user_count |
-- +--------+--------+------------+
-- | 北京   |      1 |         10 |
-- | 北京   |   NULL |         10 |  ← 北京汇总
-- | 上海   |      1 |         15 |
-- | 上海   |   NULL |         15 |  ← 上海汇总
-- | NULL   |   NULL |         25 |  ← 总汇总
-- +--------+--------+------------+

-- 实际应用：销售报表
SELECT
  COALESCE(city, '总计') AS city,
  COALESCE(product_name, '小计') AS product_name,
  SUM(sales_amount) AS total_amount
FROM sales_report
GROUP BY city, product_name WITH ROLLUP;
```

## 窗口函数（MySQL 8.0+）

窗口函数在不减少行数的情况下进行聚合计算。

```sql
-- 语法
函数名 OVER (
  PARTITION BY 分组字段
  ORDER BY 排序字段
  窗口范围
)

-- 1. ROW_NUMBER：行号
SELECT
  user_id,
  order_no,
  total_amount,
  ROW_NUMBER() OVER (PARTITION BY user_id ORDER BY total_amount DESC) AS rank_in_user
FROM orders;

-- 2. RANK：排名（相同值排名相同，跳跃）
SELECT
  user_id,
  total_amount,
  RANK() OVER (ORDER BY total_amount DESC) AS amount_rank
FROM orders;

-- 3. DENSE_RANK：排名（相同值排名相同，不跳跃）
SELECT
  user_id,
  total_amount,
  DENSE_RANK() OVER (ORDER BY total_amount DESC) AS amount_rank
FROM orders;

-- 4. 聚合窗口函数
SELECT
  order_date,
  daily_sales,
  SUM(daily_sales) OVER (
    ORDER BY order_date
    ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW
  ) AS cumulative_sales
FROM daily_sales_report;

-- 5. LAG/LEAD：访问前/后行
SELECT
  order_date,
  daily_sales,
  LAG(daily_sales, 1) OVER (ORDER BY order_date) AS yesterday_sales,
  LEAD(daily_sales, 1) OVER (ORDER BY order_date) AS tomorrow_sales
FROM daily_sales_report;
```

## 最佳实践

### 1. GROUP BY 字段原则

```sql
-- ✅ SELECT 中的非聚合字段必须出现在 GROUP BY 中
SELECT city, status, COUNT(*)
FROM users
GROUP BY city, status;

-- ❌ 错误：username 不在 GROUP BY 中
SELECT city, username, COUNT(*)
FROM users
GROUP BY city;
```

### 2. 索引优化 GROUP BY

```sql
-- ✅ 为 GROUP BY 字段创建索引
CREATE INDEX idx_city_status ON users(city, status);

-- 查询会使用索引
SELECT city, status, COUNT(*)
FROM users
GROUP BY city, status;
```

### 3. WHERE 优先于 HAVING

```sql
-- ✅ 推荐：先用 WHERE 过滤
SELECT user_id, SUM(total_amount)
FROM orders
WHERE status = 'completed'  -- 先过滤
GROUP BY user_id
HAVING SUM(total_amount) > 1000;  -- 再过滤聚合结果

-- ❌ 避免：全用 HAVING
SELECT user_id, SUM(total_amount)
FROM orders
GROUP BY user_id
HAVING status = 'completed' AND SUM(total_amount) > 1000;  -- status 无法使用 HAVING
```

### 4. 避免 SELECT \*

```sql
-- ✅ 推荐：只查询需要的字段
SELECT
  city,
  COUNT(*) AS user_count
FROM users
GROUP BY city;

-- ❌ 避免：SELECT * + GROUP BY
SELECT * FROM users GROUP BY city;
-- 结果不确定，city 以外的字段随机取值
```

## 常见问题

### Q1: WHERE 和 HAVING 有什么区别？

**A:** WHERE 过滤行，HAVING 过滤分组。

```sql
-- ✅ WHERE：分组前过滤
SELECT city, COUNT(*)
FROM users
WHERE age >= 18  -- 先过滤未成年
GROUP BY city;

-- ✅ HAVING：分组后过滤
SELECT city, COUNT(*)
FROM users
GROUP BY city
HAVING COUNT(*) > 10;  -- 再过滤人数 < 10 的城市
```

### Q2: GROUP BY 性能优化？

**A:** 创建索引，减少数据量。

```sql
-- 1. 为 GROUP BY 字段建索引
CREATE INDEX idx_city ON users(city);

-- 2. 先用 WHERE 减少数据量
SELECT city, COUNT(*)
FROM users
WHERE status = 1  -- 先过滤
GROUP BY city;

-- 3. 使用 EXPLAIN 分析
EXPLAIN SELECT city, COUNT(*) FROM users GROUP BY city;
-- 查看是否使用了索引
```
