## 前言

选择合适的数据类型是数据库设计的核心环节。合理的数据类型可以：

- 节省存储空间
- 提升查询性能
- 保证数据完整性
- 避免数据溢出

本文将全面介绍 MySQL 的各种数据类型，帮助你根据业务场景选择最合适的类型。

## 整数类型

### 类型对比

| 类型      | 字节 | 最小值               | 最大值              | 适用场景           |
| --------- | ---- | -------------------- | ------------------- | ------------------ |
| TINYINT   | 1    | -128                 | 127                 | 年龄、状态标识     |
| SMALLINT  | 2    | -32768               | 32767               | 端口号、小范围计数 |
| MEDIUMINT | 3    | -8388608             | 8388607             | 中等范围整数       |
| INT       | 4    | -2147483648          | 2147483647          | 主键ID、数量       |
| BIGINT    | 8    | -9223372036854775808 | 9223372036854775807 | 大ID、高精度计数   |

### UNSIGNED 属性

```sql
-- 有符号（默认）
age TINYINT                    -- 范围：-128 到 127

-- 无符号（0 开始，正数范围更大）
age TINYINT UNSIGNED           -- 范围：0 到 255

-- 实际应用示例
CREATE TABLE users (
  id BIGINT UNSIGNED PRIMARY KEY AUTO_INCREMENT,  -- 永远是正数
  age TINYINT UNSIGNED DEFAULT 0,                 -- 年龄不能为负
  status TINYINT UNSIGNED DEFAULT 1 COMMENT '1:正常 0:禁用',
  balance DECIMAL(10, 2) DEFAULT 0.00             -- 余额可以为负
);
```

### 实战应用

```sql
-- 用户 ID：使用 BIGINT UNSIGNED
CREATE TABLE users (
  id BIGINT UNSIGNED PRIMARY KEY AUTO_INCREMENT COMMENT '用户ID',
  -- 理由：ID 永远为正，BIGINT 确保长期使用不会溢出
  -- 无符号 BIGINT 最大值：18446744073709551615
);

-- 年龄：使用 TINYINT UNSIGNED
CREATE TABLE users (
  age TINYINT UNSIGNED DEFAULT 0 COMMENT '年龄',
  -- 理由：年龄 0-255 足够，节省空间（1字节）
);

-- 商品库存：使用 INT UNSIGNED
CREATE TABLE products (
  stock INT UNSIGNED DEFAULT 0 COMMENT '库存',
  -- 理由：库存 0-4294967295 范围够用
);

-- 文章浏览量：使用 INT UNSIGNED
CREATE TABLE posts (
  views INT UNSIGNED DEFAULT 0 COMMENT '浏览量',
  -- 理由：浏览量可能很大，INT 范围足够
);

-- 状态标识：使用 TINYINT UNSIGNED
CREATE TABLE orders (
  status TINYINT UNSIGNED DEFAULT 1 COMMENT '1待支付 2已支付 3已发货 4已完成 5已取消',
  -- 理由：状态码通常很小，TINYINT 足够
);
```

### ZEROFILL 填充

```sql
-- 数值填充（已废弃，MySQL 8.0.17+ 不推荐）
CREATE TABLE test (
  id INT ZEROFILL     -- 插入 1 显示为 0000000001
);

-- 不推荐使用，应用层处理格式化更灵活
```

## 浮点类型

### 类型对比

| 类型         | 字节 | 精度     | 说明               |
| ------------ | ---- | -------- | ------------------ |
| FLOAT        | 4    | 单精度   | 约 7 位有效数字    |
| DOUBLE       | 8    | 双精度   | 约 15 位有效数字   |
| DECIMAL(M,D) | 变长 | 精确数值 | M 位总数，D 位小数 |

### 使用场景

```sql
-- FLOAT：用于科学计算、坐标、不需要精确值的场景
CREATE TABLE locations (
  latitude FLOAT(10, 6) COMMENT '纬度',
  longitude FLOAT(10, 6) COMMENT '经度',
  -- 理由：坐标不需要绝对精确，FLOAT 足够
  temp FLOAT COMMENT '温度'
);

-- DOUBLE：用于需要较高精度但非货币的场景
CREATE TABLE statistics (
  average_score DOUBLE DEFAULT 0 COMMENT '平均分',
  conversion_rate DOUBLE DEFAULT 0 COMMENT '转化率'
);

-- DECIMAL：用于货币、金额等需要精确计算的场景
CREATE TABLE orders (
  total_amount DECIMAL(10, 2) NOT NULL COMMENT '订单总额',
  -- 10 位总数，2 位小数：范围 -99999999.99 到 99999999.99
  discount DECIMAL(5, 2) DEFAULT 0.00 COMMENT '折扣',
  -- 5 位总数，2 位小数：范围 -999.99 到 999.99
  actual_amount DECIMAL(10, 2) NOT NULL COMMENT '实付金额'
);

-- DECIMAL 精度计算示例
CREATE TABLE products (
  price DECIMAL(10, 2) COMMENT '商品价格',
  quantity INT UNSIGNED DEFAULT 0 COMMENT '数量',
  subtotal DECIMAL(12, 2) COMMENT '小计'
  -- 小计位数要多一些，避免溢出
);
```

### ⚠️ 浮点数精度问题

```sql
-- 演示精度问题
CREATE TABLE float_test (
  f FLOAT,
  d DOUBLE,
  dec DECIMAL(10, 2)
);

INSERT INTO float_test VALUES
  (0.1 + 0.2, 0.1 + 0.2, 0.1 + 0.2);

-- 查询结果
SELECT f, d, dec FROM float_test;
-- f:  0.30000001192092896  (不精确)
-- d:  0.30000000000000004  (不精确)
-- dec: 0.30                (精确)

-- 结论：金额永远使用 DECIMAL！
```

## 字符串类型

### 类型对比

| 类型       | 最大长度 | 存储特点             | 适用场景            |
| ---------- | -------- | -------------------- | ------------------- |
| CHAR(n)    | 255      | 固定长度，不足补空格 | 手机号、身份证、MD5 |
| VARCHAR(n) | 65535    | 变长，按实际存储     | 用户名、标题        |
| TINYTEXT   | 255      | 短文本               | 简短描述            |
| TEXT       | 65535    | 长文本               | 文章内容            |
| MEDIUMTEXT | 16M      | 中等文本             | 长文章              |
| LONGTEXT   | 4G       | 超长文本             | 书籍内容            |

### CHAR vs VARCHAR

```sql
-- CHAR：固定长度，查询快，占用固定空间
CREATE TABLE users (
  phone CHAR(11) COMMENT '手机号',
  -- 理由：手机号固定 11 位，CHAR 查询更快

  id_card CHAR(18) COMMENT '身份证号',
  -- 理由：身份证固定 18 位

  password_hash CHAR(64) COMMENT '密码哈希',
  -- 理由：SHA256 哈希固定 64 位

  status CHAR(1) DEFAULT '1' COMMENT '状态'
  -- 理由：单字符状态标识
);

-- VARCHAR：变长，节省空间
CREATE TABLE users (
  username VARCHAR(50) COMMENT '用户名',
  -- 理由：用户名长度不一，VARCHAR 节省空间

  email VARCHAR(100) COMMENT '邮箱',
  -- 理由：邮箱长度不一

  nickname VARCHAR(50) COMMENT '昵称',
  address VARCHAR(200) COMMENT '地址'
);

-- 性能对比
CREATE TABLE char_test (
  char_col CHAR(10),
  varchar_col VARCHAR(10)
);

-- 存储相同数据时，CHAR 占用固定空间，VARCHAR 占用实际长度+1字节长度前缀
```

### TEXT 系列

```sql
-- TINYTEXT：短文本（255字节）
CREATE TABLE products (
  short_desc TINYTEXT COMMENT '简短描述'
);

-- TEXT：一般长文本（64KB）
CREATE TABLE posts (
  summary TEXT COMMENT '文章摘要',
  content TEXT COMMENT '文章内容'
);

-- MEDIUMTEXT：中等长文本（16MB）
CREATE TABLE chapters (
  content MEDIUMTEXT COMMENT '章节内容'
);

-- LONGTEXT：超长文本（4GB）
CREATE TABLE books (
  full_text LONGTEXT COMMENT '完整书籍'
);
```

### ⚠️ VARCHAR 长度限制

```sql
-- VARCHAR 最大长度受多个因素限制
-- 1. 行大小限制（65535 字节）
-- 2. 字符集影响（utf8mb4 一个字符最多 4 字节）

-- 示例：utf8mb4 字符集下的 VARCHAR
CREATE TABLE test (
  v1 VARCHAR(65535)  -- 错误！超过行大小限制
);

-- 正确做法
CREATE TABLE test (
  v1 VARCHAR(100) CHARSET latin1,    -- latin1 1字符=1字节
  v2 VARCHAR(16000) CHARSET utf8mb4  -- utf8mb4 1字符最多4字节，16000*4=64000
);

-- 实际建议：单字段不要超过 1000，长文本用 TEXT
```

## 日期时间类型

### 类型对比

| 类型      | 字节 | 格式                | 范围                    | 时区     |
| --------- | ---- | ------------------- | ----------------------- | -------- |
| DATE      | 3    | YYYY-MM-DD          | 1000-9999               | 无       |
| TIME      | 3    | HH:MM:SS            | -838:59:59 到 838:59:59 | 无       |
| DATETIME  | 8    | YYYY-MM-DD HH:MM:SS | 1000-9999               | 无       |
| TIMESTAMP | 4    | YYYY-MM-DD HH:MM:SS | 1970-2038               | 自动转换 |
| YEAR      | 1    | YYYY                | 1901-2155               | 无       |

### DATE - 日期

```sql
CREATE TABLE users (
  birthday DATE COMMENT '生日',
  hire_date DATE COMMENT '入职日期',
  graduation_date DATE COMMENT '毕业日期'
);

-- 插入数据
INSERT INTO users (birthday) VALUES ('1990-01-01');
INSERT INTO users (birthday) VALUES (CURRENT_DATE);
INSERT INTO users (birthday) VALUES ('19900101');  -- 也支持

-- 日期函数
SELECT CURRENT_DATE;        -- 当前日期
SELECT DATE('2024-01-01 12:30:00');  -- 提取日期部分
SELECT DATE_ADD(CURRENT_DATE, INTERVAL 7 DAY);  -- 7天后
```

### TIME - 时间

```sql
CREATE TABLE schedules (
  start_time TIME COMMENT '开始时间',
  duration TIME COMMENT '持续时间'
);

-- 插入数据
INSERT INTO schedules (start_time) VALUES ('09:00:00');
INSERT INTO schedules (duration) VALUES ('02:30:00');

-- TIME 可以表示负数（时间差）
INSERT INTO schedules (duration) VALUES ('-12:30:00');
```

### DATETIME - 日期时间

```sql
-- DATETIME：不自动转换时区，适合固定时间
CREATE TABLE posts (
  published_at DATETIME COMMENT '发布时间',
  event_start DATETIME COMMENT '活动开始时间',
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间'
);

-- 插入数据
INSERT INTO posts (published_at) VALUES ('2024-01-01 12:00:00');
INSERT INTO posts (published_at) VALUES ('20240101120000');  -- 也支持

-- 自动更新时间（需要使用 TIMESTAMP，见下文）
ALTER TABLE posts ADD COLUMN updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP;
```

### TIMESTAMP - 时间戳（推荐）

```sql
-- TIMESTAMP：自动转换时区，范围较小（1970-2038），但功能强大
CREATE TABLE users (
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  last_login_at TIMESTAMP NULL COMMENT '最后登录时间'
);

-- 自动记录创建和更新时间
INSERT INTO users (id) VALUES (1);
-- created_at 和 updated_at 自动设置为当前时间

UPDATE users SET id = 1 WHERE id = 1;
-- updated_at 自动更新为当前时间

-- NULL 值处理
UPDATE users SET last_login_at = CURRENT_TIMESTAMP WHERE id = 1;
UPDATE users SET last_login_at = NULL WHERE id = 1;  -- 可以设为 NULL
```

### YEAR - 年份

```sql
CREATE TABLE cars (
  make VARCHAR(50),
  model VARCHAR(50),
  year YEAR COMMENT '年份'
);

-- 插入数据
INSERT INTO cars VALUES ('Toyota', 'Camry', 2024);
INSERT INTO cars VALUES ('Honda', 'Civic', '24');  -- 两位数也支持
```

### 日期时间选择建议

```sql
-- ✅ 推荐：TIMESTAMP 用于创建和更新时间
CREATE TABLE all_tables (
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

-- ✅ 推荐：DATETIME 用于固定时间、活动时间
CREATE TABLE events (
  event_name VARCHAR(100),
  start_time DATETIME COMMENT '活动开始时间',
  end_time DATETIME COMMENT '活动结束时间'
  -- 理由：活动时间是固定的，不需要时区转换
);

-- ✅ 推荐：DATE 用于生日、纪念日
CREATE TABLE users (
  birthday DATE COMMENT '生日'
);

-- ✅ 推荐：TIME 用于营业时间
CREATE TABLE shops (
  open_time TIME COMMENT '开门时间',
  close_time TIME COMMENT '关门时间'
);
```

## JSON 类型

MySQL 5.7.8+ 支持 JSON 类型，适合存储结构化数据。

### 基本使用

```sql
-- 创建 JSON 字段
CREATE TABLE products (
  id BIGINT UNSIGNED PRIMARY KEY AUTO_INCREMENT,
  name VARCHAR(200),
  price DECIMAL(10, 2),
  attributes JSON COMMENT '商品属性',
  tags JSON COMMENT '标签列表',
  images JSON COMMENT '图片列表'
);

-- 插入 JSON 数据
INSERT INTO products (name, price, attributes, tags, images) VALUES
  ('iPhone 15 Pro', 7999.00,
   '{"color": "深空黑", "storage": "256GB", "weight": "221g"}',
   '["旗舰", "5G", "Pro"]',
   '["img1.jpg", "img2.jpg"]'
  );

-- 查询 JSON 数据
SELECT
  name,
  price,
  attributes->>"$.color" AS color,         -- 提取 color 字段
  attributes->>"$.storage" AS storage,
  JSON_EXTRACT(attributes, "$.weight") AS weight
FROM products;

-- 数组操作
SELECT
  name,
  JSON_CONTAINS(tags, '"5G"') AS has_5g,    -- 判断是否包含
  JSON_LENGTH(tags) AS tag_count,           -- 数组长度
  JSON_EXTRACT(tags, "$[0]") AS first_tag   -- 第一个元素
FROM products;
```

### JSON 函数

```sql
-- 创建测试数据
CREATE TABLE users (
  id INT PRIMARY KEY,
  profile JSON
);

INSERT INTO users VALUES
  (1, '{"name": "张三", "age": 25, "hobbies": ["篮球", "游戏"]}'),
  (2, '{"name": "李四", "age": 30, "hobbies": ["阅读", "旅游"]}');

-- JSON_EXTRACT：提取值
SELECT profile->>"$.name" FROM users WHERE id = 1;
-- 结果：张三

-- JSON_SET：设置值
UPDATE users
SET profile = JSON_SET(profile, "$.age", 26)
WHERE id = 1;

-- JSON_INSERT：插入值（不存在时插入）
UPDATE users
SET profile = JSON_INSERT(profile, "$.city", "北京")
WHERE id = 1;

-- JSON_REPLACE：替换值（存在时替换）
UPDATE users
SET profile = JSON_REPLACE(profile, "$.age", 27)
WHERE id = 1;

-- JSON_REMOVE：删除值
UPDATE users
SET profile = JSON_REMOVE(profile, "$.city")
WHERE id = 1;

-- JSON_ARRAY：创建数组
SELECT JSON_ARRAY(1, 2, 3);
-- 结果：[1, 2, 3]

-- JSON_OBJECT：创建对象
SELECT JSON_OBJECT('name', '张三', 'age', 25);
-- 结果：{"name": "张三", "age": 25}

-- JSON_CONTAINS：判断包含
SELECT * FROM users
WHERE JSON_CONTAINS(profile->>"$.hobbies", '"篮球"');

-- JSON_KEYS：获取所有键
SELECT JSON_KEYS(profile) FROM users WHERE id = 1;
-- 结果：["name", "age", "hobbies"]

-- JSON_LENGTH：获取长度
SELECT JSON_LENGTH(profile->>"$.hobbies") FROM users WHERE id = 1;
-- 结果：2
```

### JSON 索引（MySQL 8.0.17+）

```sql
-- 为 JSON 字段创建虚拟列，然后建立索引
CREATE TABLE products (
  id INT PRIMARY KEY,
  data JSON,
  brand VARCHAR(50) AS (data->>"$.brand") STORED,  -- 虚拟列
  INDEX idx_brand (brand)  -- 为虚拟列建索引
);

INSERT INTO products (id, data) VALUES
  (1, '{"brand": "Apple", "model": "iPhone"}'),
  (2, '{"brand": "Samsung", "model": "Galaxy"}');

-- 查询会使用索引
SELECT * FROM products WHERE brand = 'Apple';
```

## ENUM 和 SET 类型

### ENUM - 枚举

```sql
-- ENUM：只能选择列出的一个值
CREATE TABLE users (
  id INT PRIMARY KEY,
  gender ENUM('male', 'female', 'other') DEFAULT 'other',
  status ENUM('active', 'inactive', 'banned') DEFAULT 'active'
);

-- 插入数据
INSERT INTO users (id, gender) VALUES (1, 'male');      -- 正确
INSERT INTO users (id, gender) VALUES (2, 'female');    -- 正确
INSERT INTO users (id, gender) VALUES (3, 'unknown');   -- 错误！不在枚举中

-- 查询
SELECT * FROM users WHERE gender = 'male';

-- ENUM 存储的是索引（1, 2, 3...），而不是字符串
-- 查询索引值
SELECT gender + 0 FROM users;
```

### SET - 集合

```sql
-- SET：可以选择多个值（最多64个）
CREATE TABLE posts (
  id INT PRIMARY KEY,
  title VARCHAR(200),
  tags SET('tech', 'life', 'news', 'sports', 'entertainment')
);

-- 插入数据（可以选多个）
INSERT INTO posts VALUES
  (1, '科技新闻', 'tech,news'),
  (2, '生活分享', 'life'),
  (3, '体育赛事', 'sports,news');

-- 查询包含特定标签的文章
SELECT * FROM posts WHERE FIND_IN_SET('tech', tags) > 0;

-- SET 使用位运算存储，查找速度快
```

### ENUM vs SET

```sql
-- ENUM：单选
CREATE TABLE users (
  gender ENUM('male', 'female', 'other')
  -- 只能选择其中一个
);

-- SET：多选
CREATE TABLE users (
  interests SET('reading', 'sports', 'music', 'travel')
  -- 可以选择多个：'reading,sports'
);

-- ⚠️ 注意：ENUM 和 SET 都不推荐大量使用
-- 理由：修改需要 ALTER TABLE，扩展性差
-- 建议：使用关联表或 JSON
```

## 字段属性

### NULL vs NOT NULL

```sql
-- NOT NULL：不允许为空（推荐）
CREATE TABLE users (
  id INT PRIMARY KEY,
  username VARCHAR(50) NOT NULL,      -- 必须提供值
  email VARCHAR(100) NOT NULL,        -- 必须提供值
  nickname VARCHAR(50)                -- 可以为空
);

-- NULL：允许为空
CREATE TABLE posts (
  id INT PRIMARY KEY,
  title VARCHAR(200) NOT NULL,
  summary VARCHAR(500),               -- 可以为空
  cover_image VARCHAR(500)            -- 可以为空
);

-- 查询 NULL 值（不能使用 = 或 !=）
SELECT * FROM posts WHERE summary IS NULL;
SELECT * FROM posts WHERE summary IS NOT NULL;
SELECT * FROM posts WHERE cover_image IS NULL;

-- ⚠️ NULL 参与运算时结果为 NULL
SELECT NULL + 1;      -- 结果：NULL
SELECT NULL || 'test'; -- 结果：NULL
```

### DEFAULT - 默认值

```sql
-- 设置默认值
CREATE TABLE users (
  id INT PRIMARY KEY,
  username VARCHAR(50) NOT NULL,
  status TINYINT DEFAULT 1,                    -- 数字默认值
  email_verified BOOLEAN DEFAULT FALSE,        -- 布尔默认值
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,  -- 时间默认值
  balance DECIMAL(10, 2) DEFAULT 0.00          -- 金额默认值
);

-- INSERT 时可以省略有默认值的字段
INSERT INTO users (id, username) VALUES (1, 'zhangsan');
-- status, email_verified, created_at, balance 自动使用默认值

-- 不同数据类型的默认值
CREATE TABLE test (
  status INT DEFAULT 1,                        -- 数字
  name VARCHAR(50) DEFAULT '匿名',              -- 字符串
  active BOOLEAN DEFAULT TRUE,                 -- 布尔（1=TRUE, 0=FALSE）
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  count INT DEFAULT 0
);
```

### AUTO_INCREMENT - 自增

```sql
-- 自增主键（最常用）
CREATE TABLE users (
  id BIGINT UNSIGNED PRIMARY KEY AUTO_INCREMENT,
  username VARCHAR(50)
);

-- 插入时不需要指定 id
INSERT INTO users (username) VALUES ('user1'), ('user2'), ('user3');
-- id 自动生成：1, 2, 3

-- 设置自增起始值
CREATE TABLE orders (
  id BIGINT UNSIGNED PRIMARY KEY AUTO_INCREMENT,
  order_no VARCHAR(50)
) AUTO_INCREMENT = 1001;

INSERT INTO orders (order_no) VALUES ('ORD001');
-- id = 1001

-- 修改自增起始值
ALTER TABLE users AUTO_INCREMENT = 1000;

-- ⚠️ 注意：自增ID可能不连续（删除、回滚）
INSERT INTO users (username) VALUES ('test');
ROLLBACK;  -- 回滚
INSERT INTO users (username) VALUES ('test2');
-- ID 会跳过回滚的那个值
```

### UNIQUE - 唯一约束

```sql
-- 唯一约束：保证字段值不重复
CREATE TABLE users (
  id INT PRIMARY KEY,
  username VARCHAR(50) UNIQUE,          -- 用户名唯一
  email VARCHAR(100) NOT NULL UNIQUE,   -- 邮箱唯一且不能为空
  phone VARCHAR(20) UNIQUE              -- 手机号唯一
);

-- 组合唯一约束：多个字段的组合必须唯一
CREATE TABLE user_follows (
  follower_id INT,
  following_id INT,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  UNIQUE KEY uk_follow (follower_id, following_id)
  -- 同一个用户不能重复关注另一个用户
);

-- 允许 NULL 的唯一约束（可以有多个 NULL）
CREATE TABLE users (
  id INT PRIMARY KEY,
  phone VARCHAR(20) UNIQUE
);

INSERT INTO users VALUES (1, NULL);     -- 允许
INSERT INTO users VALUES (2, NULL);     -- 允许（多个 NULL）
INSERT INTO users VALUES (3, '123');    -- 允许
INSERT INTO users VALUES (4, '123');    -- 错误！重复
```

### COMMENT - 注释

```sql
-- 为表和字段添加注释（强烈推荐！）
CREATE TABLE users (
  id BIGINT UNSIGNED PRIMARY KEY AUTO_INCREMENT COMMENT '用户ID',
  username VARCHAR(50) NOT NULL COMMENT '用户名',
  email VARCHAR(100) NOT NULL COMMENT '邮箱地址',
```

## 最佳实践

### 1. 优先使用合适的最小类型

```sql
-- ✅ 推荐：使用合适的类型
status TINYINT UNSIGNED      -- 0-255
age TINYINT UNSIGNED         -- 0-255

-- ❌ 不推荐：过度使用大类型
status INT                   -- 浪费空间
age BIGINT                   -- 完全没必要
```

### 2. 金额永远用 DECIMAL

```sql
-- ✅ 推荐
price DECIMAL(10, 2)

-- ❌ 不推荐
price FLOAT                  -- 精度问题
price DOUBLE                 -- 精度问题
```

### 3. 字符串长度要合适

```sql
-- ✅ 推荐
username VARCHAR(50)
email VARCHAR(100)
phone VARCHAR(20)

-- ❌ 不推荐：设置过大
username VARCHAR(1000)
email VARCHAR(500)

-- ✅ 长文本使用 TEXT
description TEXT
content LONGTEXT
```

### 4. 时间类型选择

```sql
-- ✅ 创建和更新时间用 TIMESTAMP
created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP

-- ✅ 固定时间用 DATETIME
published_at DATETIME
event_start DATETIME

-- ✅ 日期用 DATE
birthday DATE
```

### 5. 必须添加注释

```sql
-- ✅ 推荐：每个字段都有注释
CREATE TABLE users (
  id BIGINT UNSIGNED PRIMARY KEY AUTO_INCREMENT COMMENT '用户ID',
  username VARCHAR(50) NOT NULL COMMENT '用户名'
) COMMENT='用户表';

-- ❌ 不推荐：没有注释
CREATE TABLE users (
  id BIGINT UNSIGNED PRIMARY KEY AUTO_INCREMENT,
  username VARCHAR(50) NOT NULL
);
```

### 6. 合理使用 NOT NULL

```sql
-- ✅ 推荐：关键字段设置为 NOT NULL
CREATE TABLE users (
  username VARCHAR(50) NOT NULL,
  email VARCHAR(100) NOT NULL,
  created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
);

-- ⚠️ 可选字段可以为 NULL
nickname VARCHAR(50),
avatar VARCHAR(500),
phone VARCHAR(20)
```

### 7. 索引字段类型选择

```sql
-- ✅ 索引字段尽量使用小类型
CREATE TABLE users (
  id BIGINT UNSIGNED PRIMARY KEY,        -- 主键
  status TINYINT UNSIGNED,               -- 索引字段用小类型
  INDEX idx_status (status)
);

-- ❌ 索引字段避免使用大类型
CREATE TABLE users (
  status VARCHAR(50),                     -- 不要用字符串做状态
  INDEX idx_status (status)
);
```

## 常见问题

### Q1: INT(11) 中的 11 是什么意思？

**A:** 这个数字是**显示宽度**，不影响存储范围和值。

```sql
CREATE TABLE test (
  id INT(11),      -- 显示宽度 11
  count INT(5)     -- 显示宽度 5
);

-- 实际存储范围完全相同
-- 只在 ZEROFILL 时有意义（已废弃）
-- 建议：直接写 INT，不要指定宽度
```

### Q2: VARCHAR 设置多大影响性能吗？

**A:** 查询性能影响不大，主要是存储空间。

```sql
-- VARCHAR(50) 和 VARCHAR(500) 查询性能差异不大
-- 主要区别是：
-- 1. 存储空间：VARCHAR(500) 可能占用更多空间
-- 2. 行大小限制：所有字段总和不能超过 65535 字节
-- 3. 内存使用：排序、临时表可能受影响

-- 建议：根据实际需求设置，留适当余量
username VARCHAR(50)   -- 够用
title VARCHAR(200)     -- 够用
content TEXT           -- 长文本用 TEXT
```

### Q3: TEXT 字段可以建索引吗？

**A:** 可以，但有前缀长度限制。

```sql
-- ❌ 不能为整个 TEXT 字段建索引
CREATE INDEX idx_content ON posts(content);

-- ✅ 只能为前缀建索引
CREATE INDEX idx_content ON posts(content(100));

-- ✅ FULLTEXT 索引（全文搜索）
CREATE FULLTEXT INDEX idx_content ON posts(content, title);
```

### Q4: 为什么要用 utf8mb4 而不是 utf8？

**A:** utf8mb4 才是完整的 UTF-8。

```sql
-- utf8：最多 3 字节，无法存储 emoji 和部分生僻字
-- utf8mb4：最多 4 字节，完整的 UTF-8 支持

CREATE DATABASE mydb CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- 测试
CREATE TABLE test (content TEXT);
INSERT INTO test VALUES ('😀');  -- emoji
-- utf8: 存储失败或乱码
-- utf8mb4: 正常存储
```

### Q5: JSON 字段和 TEXT 字段哪个更好？

**A:** 各有适用场景。

```sql
-- JSON：需要查询内部字段时使用
CREATE TABLE users (
  preferences JSON COMMENT '用户偏好设置'
);
-- 可以查询：preferences->>"$.theme"

-- TEXT：只是存储，不需要查询内部
CREATE TABLE posts (
  content TEXT COMMENT '文章内容'
);
-- 只存储和读取整体内容

-- 建议：
-- - 需要查询内部字段 → JSON
-- - 只存储整体内容 → TEXT
-- - 结构化数据 → 关联表
```

## 总结

本文全面介绍了 MySQL 的数据类型，主要内容包括：

**整数类型**：

- ✅ TINYINT、SMALLINT、INT、BIGINT
- ✅ UNSIGNED 扩展正数范围
- ✅ 根据实际需求选择最小类型

**浮点类型**：

- ✅ FLOAT、DOUBLE：用于科学计算
- ✅ DECIMAL：金额必须用，保证精度

**字符串类型**：

- ✅ CHAR：固定长度，查询快
- ✅ VARCHAR：变长，节省空间
- ✅ TEXT 系列：长文本

**日期时间**：

- ✅ DATE：日期
- ✅ DATETIME：固定时间
- ✅ TIMESTAMP：自动记录时间（推荐）
- ✅ TIME：时间

**特殊类型**：

- ✅ JSON：结构化数据
- ✅ ENUM：单选枚举
- ✅ SET：多选集合

**字段属性**：

- ✅ NULL vs NOT NULL
- ✅ DEFAULT 默认值
- ✅ AUTO_INCREMENT 自增
- ✅ UNIQUE 唯一约束
- ✅ COMMENT 注释

**选择原则**：

- ✅ 足够用：选择满足需求的最小类型
- ✅ 节省空间：能用小类型不用大类型
- ✅ 性能优先：索引字段用小类型
- ✅ 数据完整：金额用 DECIMAL，时间用 TIMESTAMP
- ✅ 必加注释：所有表和字段都要有注释
