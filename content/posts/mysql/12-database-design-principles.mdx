## 前言

良好的数据库设计是系统成功的基石。合理的设计可以避免数据冗余、保证数据完整性、提升查询性能。

本文将系统讲解数据库设计的核心原则和实战技巧。

## 数据库设计三范式

### 第一范式（1NF）

**定义**：每个字段都是不可分割的最小数据单位。

```sql
-- ❌ 违反 1NF
CREATE TABLE users (
  id INT,
  name VARCHAR(100),  -- 包含姓和名
  address VARCHAR(200)  -- 包含省市区街道
);

-- ✅ 符合 1NF
CREATE TABLE users (
  id INT,
  first_name VARCHAR(50),
  last_name VARCHAR(50),
  province VARCHAR(50),
  city VARCHAR(50),
  district VARCHAR(50),
  street VARCHAR(100)
);
```

### 第二范式（2NF）

**定义**：在满足 1NF 的基础上，非主键字段完全依赖于主键。

```sql
-- ❌ 违反 2NF（部分依赖）
CREATE TABLE order_items (
  order_id INT,
  product_id INT,
  product_name VARCHAR(100),  -- 依赖 product_id
  price DECIMAL(10, 2),       -- 依赖 product_id
  quantity INT,
  PRIMARY KEY (order_id, product_id)
);

-- ✅ 符合 2NF（拆分表）
CREATE TABLE order_items (
  order_id INT,
  product_id INT,
  quantity INT,
  PRIMARY KEY (order_id, product_id)
);

CREATE TABLE products (
  product_id INT PRIMARY KEY,
  product_name VARCHAR(100),
  price DECIMAL(10, 2)
);
```

### 第三范式（3NF）

**定义**：在满足 2NF 的基础上，非主键字段不传递依赖于主键。

```sql
-- ❌ 违反 3NF（传递依赖）
CREATE TABLE orders (
  order_id INT PRIMARY KEY,
  user_id INT,
  user_name VARCHAR(50),    -- 依赖 user_id
  user_email VARCHAR(100),  -- 依赖 user_id
  total_amount DECIMAL(10, 2)
);

-- ✅ 符合 3NF（拆分表）
CREATE TABLE orders (
  order_id INT PRIMARY KEY,
  user_id INT,
  total_amount DECIMAL(10, 2)
);

CREATE TABLE users (
  user_id INT PRIMARY KEY,
  user_name VARCHAR(50),
  user_email VARCHAR(100)
);
```

## 表关系设计

### 一对一（1:1）

```sql
-- 一个用户对应一个详细资料
CREATE TABLE users (
  user_id INT PRIMARY KEY,
  username VARCHAR(50)
);

CREATE TABLE user_profiles (
  profile_id INT PRIMARY KEY,
  user_id INT UNIQUE,  -- 唯一约束
  real_name VARCHAR(50),
  id_card VARCHAR(18),
  phone VARCHAR(20),
  FOREIGN KEY (user_id) REFERENCES users(user_id)
);

-- 或者在主表中添加外键
CREATE TABLE users (
  user_id INT PRIMARY KEY,
  username VARCHAR(50),
  profile_id INT UNIQUE,
  FOREIGN KEY (profile_id) REFERENCES user_profiles(profile_id)
);
```

### 一对多（1:N）

```sql
-- 一个用户有多个订单
CREATE TABLE users (
  user_id INT PRIMARY KEY,
  username VARCHAR(50)
);

CREATE TABLE orders (
  order_id INT PRIMARY KEY,
  user_id INT,  -- 外键
  order_no VARCHAR(50),
  total_amount DECIMAL(10, 2),
  FOREIGN KEY (user_id) REFERENCES users(user_id)
);

-- 查询用户的所有订单
SELECT u.*, o.*
FROM users u
LEFT JOIN orders o ON u.user_id = o.user_id;
```

### 多对多（M:N）

```sql
-- 一个学生可以选多门课，一门课有多个学生
CREATE TABLE students (
  student_id INT PRIMARY KEY,
  student_name VARCHAR(50)
);

CREATE TABLE courses (
  course_id INT PRIMARY KEY,
  course_name VARCHAR(100)
);

-- 中间表（关联表）
CREATE TABLE student_courses (
  student_id INT,
  course_id INT,
  enroll_date DATE,
  score DECIMAL(5, 2),
  PRIMARY KEY (student_id, course_id),  -- 联合主键
  FOREIGN KEY (student_id) REFERENCES students(student_id),
  FOREIGN KEY (course_id) REFERENCES courses(course_id)
);

-- 查询学生的选课
SELECT s.student_name, c.course_name, sc.score
FROM students s
INNER JOIN student_courses sc ON s.student_id = sc.student_id
INNER JOIN courses c ON sc.course_id = c.course_id;
```

## 外键约束

```sql
-- 创建外键
CREATE TABLE orders (
  order_id INT PRIMARY KEY,
  user_id INT,
  total_amount DECIMAL(10, 2),
  FOREIGN KEY (user_id)
    REFERENCES users(user_id)
    ON DELETE RESTRICT  -- 删除限制
    ON UPDATE CASCADE   -- 更新级联
);

-- 级联操作选项：
-- ON DELETE RESTRICT  - 有子记录时不允许删除（默认）
-- ON DELETE CASCADE   - 删除父记录时，子记录也删除
-- ON DELETE SET NULL  - 删除父记录时，子记录的外键设为 NULL
-- ON DELETE NO ACTION - 与 RESTRICT 相同

-- 实际应用示例
CREATE TABLE orders (
  order_id INT PRIMARY KEY,
  user_id INT,
  product_id INT,
  FOREIGN KEY (user_id) REFERENCES users(user_id) ON DELETE CASCADE,
  FOREIGN KEY (product_id) REFERENCES products(product_id) ON DELETE RESTRICT
);
-- 删除用户时，订单也删除
-- 有订单时，不允许删除商品
```

## 反规范化

为了性能而适度违反范式。

### 何时反规范化

```sql
-- ✅ 适合反规范化的场景：
-- 1. 读多写少
-- 2. 查询频繁
-- 3. 性能要求高

-- 示例：在订单表中冗余用户名
-- 规范化：需要 JOIN
SELECT o.order_no, u.username
FROM orders o
INNER JOIN users u ON o.user_id = u.user_id;

-- 反规范化：添加冗余字段
CREATE TABLE orders (
  order_id INT PRIMARY KEY,
  user_id INT,
  username VARCHAR(50),  -- 冗余字段
  order_no VARCHAR(50),
  total_amount DECIMAL(10, 2)
);

-- 查询时无需 JOIN
SELECT order_no, username FROM orders;
-- 性能提升，但需要维护冗余字段一致性
```

### 冗余字段设计

```sql
-- 示例：订单表冗余商品信息
CREATE TABLE order_items (
  item_id INT PRIMARY KEY,
  order_id INT,
  product_id INT,
  product_name VARCHAR(100),   -- 冗余
  price DECIMAL(10, 2),         -- 冗余（快照）
  quantity INT,
  subtotal DECIMAL(10, 2)
);

-- 好处：
-- 1. 查询订单明细时无需 JOIN products 表
-- 2. 保存历史价格（商品价格可能变化）

-- 代价：
-- 需要在插入时复制数据
```

### 计数器表

```sql
-- ❌ 差：每次查询 COUNT
SELECT COUNT(*) FROM comments WHERE post_id = 1;

-- ✅ 好：冗余计数字段
CREATE TABLE posts (
  post_id INT PRIMARY KEY,
  title VARCHAR(200),
  comment_count INT DEFAULT 0  -- 冗余
);

-- 插入评论时更新
INSERT INTO comments (post_id, content) VALUES (1, '...');
UPDATE posts SET comment_count = comment_count + 1 WHERE post_id = 1;
```

## 电商数据库设计

### 核心表

```sql
-- 用户表
CREATE TABLE users (
  user_id BIGINT UNSIGNED PRIMARY KEY AUTO_INCREMENT,
  username VARCHAR(50) NOT NULL UNIQUE,
  email VARCHAR(100) NOT NULL UNIQUE,
  password_hash VARCHAR(255) NOT NULL,
  status TINYINT UNSIGNED DEFAULT 1,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX idx_email (email),
  INDEX idx_status (status)
) ENGINE=InnoDB;

-- 分类表
CREATE TABLE categories (
  category_id BIGINT UNSIGNED PRIMARY KEY AUTO_INCREMENT,
  parent_id BIGINT UNSIGNED,
  name VARCHAR(100) NOT NULL,
  level TINYINT UNSIGNED DEFAULT 1,
  sort_order INT UNSIGNED DEFAULT 0,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (parent_id) REFERENCES categories(category_id) ON DELETE SET NULL,
  INDEX idx_parent (parent_id),
  INDEX idx_level (level)
) ENGINE=InnoDB;

-- 商品表
CREATE TABLE products (
  product_id BIGINT UNSIGNED PRIMARY KEY AUTO_INCREMENT,
  category_id BIGINT UNSIGNED NOT NULL,
  name VARCHAR(200) NOT NULL,
  description TEXT,
  price DECIMAL(10, 2) NOT NULL,
  stock INT UNSIGNED DEFAULT 0,
  sales INT UNSIGNED DEFAULT 0,
  status TINYINT UNSIGNED DEFAULT 1,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (category_id) REFERENCES categories(category_id),
  INDEX idx_category (category_id),
  INDEX idx_price (price),
  INDEX idx_status_sales (status, sales DESC)
) ENGINE=InnoDB;

-- 订单表
CREATE TABLE orders (
  order_id BIGINT UNSIGNED PRIMARY KEY AUTO_INCREMENT,
  user_id BIGINT UNSIGNED NOT NULL,
  order_no VARCHAR(50) NOT NULL UNIQUE,
  total_amount DECIMAL(10, 2) NOT NULL,
  status ENUM('pending', 'paid', 'shipped', 'completed', 'cancelled') DEFAULT 'pending',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES users(user_id),
  INDEX idx_user (user_id),
  INDEX idx_order_no (order_no),
  INDEX idx_status (status),
  INDEX idx_created (created_at DESC)
) ENGINE=InnoDB;

-- 订单明细表
CREATE TABLE order_items (
  item_id BIGINT UNSIGNED PRIMARY KEY AUTO_INCREMENT,
  order_id BIGINT UNSIGNED NOT NULL,
  product_id BIGINT UNSIGNED NOT NULL,
  product_name VARCHAR(200),   -- 冗余
  price DECIMAL(10, 2),         -- 冗余
  quantity INT UNSIGNED NOT NULL,
  subtotal DECIMAL(10, 2),
  FOREIGN KEY (order_id) REFERENCES orders(order_id) ON DELETE CASCADE,
  FOREIGN KEY (product_id) REFERENCES products(product_id),
  INDEX idx_order (order_id)
) ENGINE=InnoDB;
```

## 最佳实践

### 1. 适度范式化

```sql
-- ✅ 推荐：核心业务表遵循 3NF
-- ✅ 推荐：报表、统计表适当反规范化
-- ✅ 推荐：根据查询频率决定是否冗余
```

### 2. 命名规范

```sql
-- ✅ 表名：小写、下划线、复数
users, orders, order_items

-- ✅ 主键：表名_id
user_id, order_id

-- ✅ 外键：关联表_id
user_id, product_id

-- ✅ 索引：idx_字段名
idx_email, idx_status_created

-- ✅ 唯一索引：uk_字段名
uk_username, uk_order_no
```

### 3. 字段设计

```sql
-- ✅ 必须有主键
PRIMARY KEY (id)

-- ✅ 关键字段 NOT NULL
username VARCHAR(50) NOT NULL

-- ✅ 添加默认值
status TINYINT UNSIGNED DEFAULT 1

-- ✅ 添加注释
COMMENT '用户表'
```

## 总结

本文介绍了数据库设计原则，主要内容：

**三范式**：

- ✅ 1NF：字段不可分割
- ✅ 2NF：完全依赖主键
- ✅ 3NF：无传递依赖

**表关系**：

- ✅ 一对一、一对多、多对多
- ✅ 外键约束

**反规范化**：

- ✅ 性能优先时适度冗余
- ✅ 读多写少的场景

**实战设计**：

- ✅ ER 图设计
- ✅ 电商数据库案例
