## 前言

MySQL 是世界上最流行的开源关系型数据库管理系统，被广泛应用于互联网应用、企业系统、数据分析等领域。无论是个人博客、电商平台，还是大型社交网络，MySQL 都能提供稳定可靠的数据存储服务。

**为什么选择 MySQL？**

- 🌟 **市场占有率第一**：全球最流行的开源数据库
- 🎯 **易于学习**：语法简洁，文档丰富，社区活跃
- ⚡ **性能优秀**：支持高并发，读写速度快
- 🛡️ **稳定可靠**：历经几十年考验，久经沙场
- 💰 **免费开源**：无版权费用，降低成本
- 🔧 **生态完善**：工具丰富，框架支持广泛

## 什么是 MySQL

### 发展历史

MySQL 诞生于 **1995 年**，由瑞典的 Michael "Monty" Widenius 创建。MySQL 这个名字是 Monty 和女儿 My 的名字组合。

```
1995  - MySQL 1.0 发布
2000  - 采用 GPL 开源协议
2008  - Sun 收购 MySQL
2009  - Oracle 收购 Sun，MySQL 归属 Oracle
2016  - MySQL 8.0 发布（重大版本更新）
```

### 核心特性

MySQL 作为关系型数据库，具备以下核心特性：

**1. 关系模型**

使用表格（Table）存储数据，表与表之间通过外键建立关系。

**2. SQL 语言支持**

使用结构化查询语言（SQL）进行数据操作，包括：

- DDL（数据定义语言）：CREATE、DROP、ALTER
- DML（数据操作语言）：INSERT、UPDATE、DELETE
- DQL（数据查询语言）：SELECT
- DCL（数据控制语言）：GRANT、REVOKE

**3. ACID 特性**

支持事务的原子性、一致性、隔离性和持久性。

**4. 多存储引擎**

- **InnoDB**：默认引擎，支持事务、外键、行级锁
- **MyISAM**：表级锁，查询快，不支持事务
- **Memory**：数据存储在内存中，速度极快

**5. 跨平台**

支持 Linux、Windows、macOS 等多种操作系统。

### MySQL vs 其他数据库

| 特性     | MySQL          | PostgreSQL         | MongoDB              |
| -------- | -------------- | ------------------ | -------------------- |
| 类型     | 关系型         | 关系型             | 文档型               |
| 性能     | 优秀           | 优秀               | 优秀                 |
| 功能     | 丰富           | 最丰富             | 适中                 |
| 学习曲线 | 简单           | 中等               | 简单                 |
| 适用场景 | Web 应用、电商 | 复杂查询、数据分析 | 文档存储、灵活schema |

**选择建议**：

- **选 MySQL**：Web 应用、电商系统、快速开发
- **选 PostgreSQL**：复杂查询、数据分析、高可靠性要求
- **选 MongoDB**：文档存储、快速原型、灵活的数据结构

## 环境搭建

### 方案一：Docker 安装（推荐）

Docker 是最简单快捷的安装方式，适合开发环境。

**步骤 1：安装 Docker**

如果你还没有安装 Docker，请访问 [docker.com](https://www.docker.com) 下载并安装。

**步骤 2：创建 docker-compose.yml**

创建项目目录并创建 `docker-compose.yml` 文件：

```yaml
version: "3.8"

services:
  mysql:
    image: mysql:8.0
    container_name: mysql-dev
    restart: always
    environment:
      MYSQL_ROOT_PASSWORD: root123 # root 用户密码
      MYSQL_DATABASE: shop # 默认创建的数据库
      MYSQL_USER: dev_user # 创建普通用户
      MYSQL_PASSWORD: dev123 # 普通用户密码
      TZ: Asia/Shanghai # 时区设置
    ports:
      - "3306:3306" # 端口映射
    volumes:
      - mysql-data:/var/lib/mysql # 数据持久化
      - ./init:/docker-entrypoint-initdb.d # 初始化脚本目录
    command:
      - --character-set-server=utf8mb4 # 字符集
      - --collation-server=utf8mb4_unicode_ci
      - --default-authentication-plugin=mysql_native_password

volumes:
  mysql-data: # 声明数据卷
```

**代码解释**：

- `image: mysql:8.0`：使用 MySQL 8.0 官方镜像
- `MYSQL_ROOT_PASSWORD`：必须设置，root 用户密码
- `MYSQL_DATABASE`：可选，启动时自动创建数据库
- `volumes`：数据持久化，容器删除后数据不丢失
- `character-set-server=utf8mb4`：使用 utf8mb4 字符集，支持中文和 emoji

**步骤 3：启动 MySQL**

```bash
# 启动容器
docker-compose up -d

# 查看容器状态
docker-compose ps

# 查看日志
docker-compose logs -f mysql
```

**步骤 4：连接测试**

```bash
# 使用 docker exec 进入容器
docker exec -it mysql-dev mysql -uroot -proot123

# 或者从主机连接（需要先安装 mysql 客户端）
mysql -h127.0.0.1 -P3306 -uroot -proot123
```

### 方案二：本地安装

**macOS（使用 Homebrew）**

```bash
# 安装 MySQL
brew install mysql

# 启动 MySQL 服务
brew services start mysql

# 设置 root 密码
mysql_secure_installation
```

**Ubuntu/Debian**

```bash
# 更新包列表
sudo apt update

# 安装 MySQL Server
sudo apt install mysql-server -y

# 安全配置
sudo mysql_secure_installation

# 启动服务
sudo systemctl start mysql
sudo systemctl enable mysql
```

**Windows**

1. 访问 [MySQL 官网](https://dev.mysql.com/downloads/mysql/)
2. 下载 MySQL Installer
3. 运行安装程序，选择默认配置
4. 设置 root 密码

## 第一个数据库操作

让我们通过实际操作来熟悉 MySQL。

### 创建数据库

```sql
-- 创建数据库
CREATE DATABASE my_first_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- 查看所有数据库
SHOW DATABASES;

-- 选择数据库
USE my_first_db;
```

**代码解释**：

- `CHARACTER SET utf8mb4`：指定字符集为 utf8mb4，支持中文和 emoji
- `COLLATE utf8mb4_unicode_ci`：排序规则，ci 表示 case-insensitive（不区分大小写）

### 创建表

```sql
-- 创建用户表
CREATE TABLE users (
  id INT PRIMARY KEY AUTO_INCREMENT,
  username VARCHAR(50) NOT NULL,
  email VARCHAR(100) NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 查看表结构
DESC users;

-- 查看建表语句
SHOW CREATE TABLE users;
```

**代码解释**：

- `PRIMARY KEY`：主键，唯一标识每条记录
- `AUTO_INCREMENT`：自动增长，无需手动指定 id
- `NOT NULL`：不允许为空
- `TIMESTAMP DEFAULT CURRENT_TIMESTAMP`：默认值为当前时间

### 插入数据

```sql
-- 插入单条数据
INSERT INTO users (username, email) VALUES ('张三', 'zhangsan@example.com');

-- 插入多条数据
INSERT INTO users (username, email) VALUES
  ('李四', 'lisi@example.com'),
  ('王五', 'wangwu@example.com'),
  ('赵六', 'zhaoliu@example.com');
```

### 查询数据

```sql
-- 查询所有数据
SELECT * FROM users;

-- 查询特定列
SELECT username, email FROM users;

-- 带条件查询
SELECT * FROM users WHERE username = '张三';

-- 排序
SELECT * FROM users ORDER BY id DESC;

-- 限制结果数量
SELECT * FROM users LIMIT 2;
```

### 更新数据

```sql
-- 更新数据（注意要有 WHERE 条件！）
UPDATE users SET email = 'newemail@example.com' WHERE username = '张三';
```

### 删除数据

```sql
-- 删除数据（注意要有 WHERE 条件！）
DELETE FROM users WHERE username = '赵六';

-- 清空表
DELETE FROM users;
-- 或
TRUNC TABLE users;
```

## 最佳实践

### 1. 数据库命名规范

```sql
-- ✅ 推荐：使用小写和下划线
CREATE DATABASE my_app_db;
CREATE TABLE user_profiles;

-- ❌ 不推荐：使用驼峰或空格
CREATE DATABASE myAppDB;  -- 大小写敏感问题
CREATE DATABASE my app;   -- 错误语法
```

### 2. 字符集选择

```sql
-- ✅ 推荐：使用 utf8mb4
CREATE DATABASE mydb CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- ❌ 不推荐：使用 utf8（无法存储 emoji）
CREATE DATABASE mydb CHARACTER SET utf8;
```

**为什么选择 utf8mb4？**

- `utf8`：最多 3 字节，部分 emoji 和生僻字无法存储
- `utf8mb4`：最多 4 字节，完整的 UTF-8 支持

### 3. 密码安全

```sql
-- ✅ 推荐：设置强密码
CREATE USER 'app_user'@'%' IDENTIFIED BY 'Str0ng!Pass@word';

-- ❌ 不推荐：使用简单密码
CREATE USER 'app_user'@'%' IDENTIFIED BY '123456';
```

### 4. 备份策略

```bash
# 定期备份数据库
mysqldump -uroot -proot123 my_first_db > backup_$(date +%Y%m%d).sql

# 使用自动化脚本
# 配合 cron 定时任务
```

### 5. Docker 数据持久化

```yaml
# ✅ 推荐：使用卷持久化数据
volumes:
  - mysql-data:/var/lib/mysql

# ❌ 不推荐：不使用卷，容器删除后数据丢失
```

## 常见问题

### Q1: 忘记 root 密码怎么办？

**A:** 可以通过以下步骤重置：

```bash
# 1. 停止 MySQL 服务
docker-compose down

# 2. 以安全模式启动
docker run --rm -it mysql:8.0 --skip-grant-tables

# 3. 在另一个终端连接并重置密码
docker exec -it mysql-dev mysql
mysql> USE mysql;
mysql> UPDATE user SET authentication_string='' WHERE User='root';
mysql> FLUSH PRIVILEGES;

# 4. 重启容器并设置新密码
docker-compose restart
```

### Q2: 如何远程连接 MySQL？

**A:** 需要注意以下几点：

```sql
-- 1. 创建允许远程连接的用户
CREATE USER 'remote_user'@'%' IDENTIFIED BY 'password';
GRANT ALL PRIVILEGES ON *.* TO 'remote_user'@'%';
FLUSH PRIVILEGES;

-- 2. 确保 MySQL 绑定到 0.0.0.0（Docker 默认已配置）
-- 3. 检查防火墙是否开放 3306 端口
```

### Q3: 中文乱码如何解决？

**A:** 统一使用 utf8mb4 字符集：

```sql
-- 创建数据库时指定
CREATE DATABASE mydb CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- 创建表时指定
CREATE TABLE users (
  ...
) CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- 连接时指定
mysql -uroot -p --default-character-set=utf8mb4
```

### Q4: Docker 容器重启后数据丢失？

**A:** 确保使用了卷持久化：

```yaml
volumes:
  - mysql-data:/var/lib/mysql # 这行很关键
```

检查数据卷是否创建：

```bash
docker volume ls
docker volume inspect mysql_mysql-data
```

### Q5: 如何查看 MySQL 运行状态？

**A:** 使用以下命令：

```sql
-- 查看版本
SELECT VERSION();

-- 查看当前状态
SHOW STATUS;

-- 查看配置
SHOW VARIABLES;

-- 查看进程列表
SHOW PROCESSLIST;

-- 查看连接数
SHOW STATUS LIKE 'Threads_connected';
```
