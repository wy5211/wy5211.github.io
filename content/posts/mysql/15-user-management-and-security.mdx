## 前言

数据库安全是运维工作的重中之重。合理配置用户权限可以有效防止数据泄露和误操作。

本文将系统讲解 MySQL 用户管理和安全策略。

## 用户管理

### 创建用户

```sql
-- 基本语法
CREATE USER 'username'@'host' IDENTIFIED BY 'password';

-- 示例
CREATE USER 'app_user'@'localhost' IDENTIFIED BY 'SecurePass123!';
CREATE USER 'app_user'@'%' IDENTIFIED BY 'SecurePass123!';
CREATE USER 'app_user'@'192.168.1.%' IDENTIFIED BY 'SecurePass123!';

-- host 选项：
-- 'localhost'  - 只允许本地连接
-- '%'          - 允许任意主机连接
-- '192.168.1.%' - 允许特定网段
-- 'domain.com' - 允许特定域名
```

### 查看用户

```sql
-- 查看所有用户
SELECT user, host FROM mysql.user;

-- 查看当前用户
SELECT USER();
SELECT CURRENT_USER();

-- 查看用户权限
SHOW GRANTS FOR 'app_user'@'localhost';
```

### 修改用户

```sql
-- 重命名用户
RENAME USER 'old_name'@'localhost' TO 'new_name'@'localhost';

-- 修改密码
ALTER USER 'app_user'@'localhost' IDENTIFIED BY 'NewSecurePass456!';
-- MySQL 5.7.6 及以下
SET PASSWORD FOR 'app_user'@'localhost' = PASSWORD('NewSecurePass456!');

-- 修改用户属性
ALTER USER 'app_user'@'localhost'
  IDENTIFIED BY 'NewSecurePass456!'
  PASSWORD EXPIRE INTERVAL 90 DAY
  ACCOUNT LOCK;
```

### 删除用户

```sql
-- 删除用户
DROP USER 'app_user'@'localhost';
DROP USER IF EXISTS 'old_user'@'localhost';

-- 注意：会同时撤销该用户的所有权限
```

### 锁定/解锁用户

```sql
-- 锁定用户
ALTER USER 'app_user'@'localhost' ACCOUNT LOCK;

-- 解锁用户
ALTER USER 'app_user'@'localhost' ACCOUNT UNLOCK;

-- 查看用户状态
SELECT user, host, account_locked FROM mysql.user;
```

## 权限管理

### 授予权限（GRANT）

```sql
-- 基本语法
GRANT privileges ON database.table TO 'user'@'host';

-- 全局权限
GRANT ALL PRIVILEGES ON *.* TO 'admin'@'localhost' WITH GRANT OPTION;

-- 数据库权限
GRANT ALL PRIVILEGES ON shop.* TO 'app_user'@'localhost';

-- 表权限
GRANT SELECT, INSERT, UPDATE, DELETE ON shop.users TO 'app_user'@'localhost';

-- 列权限
GRANT SELECT(id, username), UPDATE(username) ON shop.users TO 'app_user'@'localhost';

-- 刷新权限
FLUSH PRIVILEGES;
```

### 常用权限

| 权限         | 说明          |
| ------------ | ------------- |
| ALL          | 所有权限      |
| CREATE       | 创建表/数据库 |
| DROP         | 删除表/数据库 |
| SELECT       | 查询          |
| INSERT       | 插入          |
| UPDATE       | 更新          |
| DELETE       | 删除          |
| EXECUTE      | 执行存储过程  |
| INDEX        | 创建/删除索引 |
| ALTER        | 修改表结构    |
| GRANT OPTION | 授权权限      |

```sql
-- 只读权限
GRANT SELECT ON shop.* TO 'readonly_user'@'%';

-- 读写权限
GRANT SELECT, INSERT, UPDATE, DELETE ON shop.* TO 'app_user'@'%';

-- 开发权限
GRANT SELECT, INSERT, UPDATE, DELETE, CREATE, ALTER, EXECUTE ON shop.* TO 'dev_user'@'%';
```

### 撤销权限（REVOKE）

```sql
-- 基本语法
REVOKE privileges ON database.table FROM 'user'@'host';

-- 撤销所有权限
REVOKE ALL PRIVILEGES ON *.* FROM 'user'@'localhost';

-- 撤销特定权限
REVOKE DELETE, DROP ON shop.* FROM 'app_user'@'localhost';

-- 撤销授权权限
REVOKE GRANT OPTION ON *.* FROM 'admin'@'localhost';
```

### 查看权限

```sql
-- 查看当前用户权限
SHOW GRANTS;
SHOW GRANTS FOR CURRENT_USER();

-- 查看指定用户权限
SHOW GRANTS FOR 'app_user'@'localhost';
```

## 角色管理（MySQL 8.0+）

角色简化了权限管理。

### 创建角色

```sql
-- 创建角色
CREATE ROLE 'app_read', 'app_write', 'app_admin';

-- 授予权限
GRANT SELECT ON shop.* TO 'app_read';
GRANT SELECT, INSERT, UPDATE ON shop.* TO 'app_write';
GRANT ALL PRIVILEGES ON shop.* TO 'app_admin';
```

### 分配角色

```sql
-- 将角色授予用户
GRANT 'app_read' TO 'readonly_user'@'%';
GRANT 'app_write' TO 'app_user'@'%';
GRANT 'app_admin' TO 'admin'@'%';

-- 设置默认角色
SET DEFAULT ROLE ALL TO 'app_user'@'%';
```

### 管理角色

```sql
-- 查看角色
SELECT * FROM mysql.role_edges;

-- 撤销角色
REVOKE 'app_write' FROM 'app_user'@'%';

-- 删除角色
DROP ROLE 'app_read';
```

## 密码策略

### 密码过期

```sql
-- 设置密码过期
ALTER USER 'app_user'@'localhost' PASSWORD EXPIRE;

-- 设置密码过期时间
ALTER USER 'app_user'@'localhost' PASSWORD EXPIRE INTERVAL 90 DAY;

-- 永不过期
ALTER USER 'app_user'@'localhost' PASSWORD EXPIRE NEVER;

-- 首次登录强制修改密码
ALTER USER 'new_user'@'localhost' PASSWORD EXPIRE;
```

### 密码重用

```sql
-- 禁止重用最近 5 次的密码
ALTER USER 'app_user'@'localhost'
  PASSWORD HISTORY 5;

-- 禁止重用最近 365 天的密码
ALTER USER 'app_user'@'localhost'
  PASSWORD REUSE INTERVAL 365 DAY;
```

### 密码验证

```sql
-- 密码复杂度策略
INSTALL COMPONENT 'file://component_validate_password';

-- 查看密码策略
SHOW VARIABLES LIKE 'validate_password%';

-- 配置密码策略
SET GLOBAL validate_password.policy = STRONG;
SET GLOBAL validate_password.length = 12;
```

## 连接安全

### SSL 连接

```sql
-- 创建要求 SSL 的用户
CREATE USER 'secure_user'@'%' REQUIRE SSL;

-- 创建指定证书的用户
CREATE USER 'secure_user'@'%'
REQUIRE X509
AND SUBJECT '/CN=client'
AND ISSUER '/CN=CA';

-- 查看用户 SSL 要求
SELECT user, host, ssl_type FROM mysql.user;
```

### 限制连接数

```sql
-- 限制用户连接数
GRANT USAGE ON *.* TO 'app_user'@'localhost' WITH MAX_CONNECTIONS_PER_HOUR 100;
```

## 安全最佳实践

### 1. 最小权限原则

```sql
-- ✅ 推荐：只授予必要权限
GRANT SELECT ON shop.* TO 'readonly_user'@'%';

-- ❌ 避免：授予过多权限
GRANT ALL PRIVILEGES ON *.* TO 'app_user'@'%';
```

### 2. 限制远程访问

```sql
-- ✅ 推荐：限制特定 IP
CREATE USER 'app_user'@'192.168.1.100' IDENTIFIED BY 'password';
GRANT SELECT ON shop.* TO 'app_user'@'192.168.1.100';

-- ❌ 避免：允许任意 IP
CREATE USER 'app_user'@'%' IDENTIFIED BY 'password';
```

### 3. 强密码策略

```sql
-- ✅ 推荐：强密码
CREATE USER 'app_user'@'localhost'
IDENTIFIED BY 'Str0ng!Pass@word';

-- ❌ 避免：弱密码
CREATE USER 'app_user'@'localhost'
IDENTIFIED BY '123456';
```

### 4. 定期审查权限

```sql
-- 定期检查用户权限
SELECT user, host FROM mysql.user;
SHOW GRANTS FOR 'app_user'@'localhost';

-- 删除不需要的用户
DROP USER IF EXISTS 'old_user'@'localhost';
```

## 实战应用

### 应用用户配置

```sql
-- 1. 只读用户（报表查询）
CREATE USER 'report_user'@'%' IDENTIFIED BY 'ReportPass123!';
GRANT SELECT ON shop.* TO 'report_user'@'%';

-- 2. 应用用户（增删改查）
CREATE USER 'app_user'@'%' IDENTIFIED BY 'AppPass123!';
GRANT SELECT, INSERT, UPDATE, DELETE ON shop.* TO 'app_user'@'%';
GRANT EXECUTE ON PROCEDURE shop.* TO 'app_user'@'%';

-- 3. 管理员用户（全部权限）
CREATE USER 'admin'@'localhost' IDENTIFIED BY 'AdminPass123!';
GRANT ALL PRIVILEGES ON *.* TO 'admin'@'localhost' WITH GRANT OPTION;
```
