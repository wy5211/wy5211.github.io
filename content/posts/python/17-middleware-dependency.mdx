# 中间件与依赖注入

## 前言

中间件（Middleware）和依赖注入（Dependency Injection）是 FastAPI 架构设计的两大支柱。中间件负责请求/响应的拦截处理（如日志、认证、限流），依赖注入则让代码解耦、可测试、可复用。

理解这两大机制，是写出清晰、可维护的 FastAPI 应用的重要一步。

**本文将带你：**

1. 理解 FastAPI 请求处理流程和中间件执行顺序
2. 实现自定义中间件（请求日志、耗时统计、请求 ID）
3. 掌握依赖注入的高级用法（yield 依赖、子依赖、作用域）
4. 使用应用生命周期管理资源

---

## 核心概念

### 请求处理流程

FastAPI（基于 Starlette）的请求处理流程如下：

```
客户端请求
    │
    ▼
┌─────────────────────────────────┐
│  ASGI Server (Uvicorn)          │
└──────────┬──────────────────────┘
           │
           ▼
┌─────────────────────────────────┐
│  Middleware 1 (最外层)           │  ← 请求：从外到内
│  ┌───────────────────────────┐  │
│  │  Middleware 2             │  │
│  │  ┌─────────────────────┐  │  │
│  │  │  Middleware 3       │  │  │
│  │  │  ┌───────────────┐  │  │  │
│  │  │  │  路由处理函数   │  │  │  │  ← 核心：执行业务逻辑
│  │  │  └───────────────┘  │  │  │
│  │  └─────────────────────┘  │  │
│  └───────────────────────────┘  │
└──────────┬──────────────────────┘
           │
           ▼
客户端响应                           ← 响应：从内到外
```

中间件按**注册顺序**执行请求阶段，按**逆序**执行响应阶段。

## 实战演练

### 示例 1：请求日志与耗时中间件

```python
"""
博客系统 - 自定义中间件
"""
import time
import uuid
import logging
from starlette.middleware.base import BaseHTTPMiddleware, RequestResponseEndpoint
from starlette.requests import Request
from starlette.responses import Response, JSONResponse
from fastapi import FastAPI

logger = logging.getLogger(__name__)

# ========== 请求日志中间件 ==========

class RequestLoggingMiddleware(BaseHTTPMiddleware):
    """记录每个请求的详细信息"""

    async def dispatch(self, request: Request, call_next: RequestResponseEndpoint) -> Response:
        # 生成请求唯一 ID
        request_id = str(uuid.uuid4())[:8]
        request.state.request_id = request_id

        # 记录请求开始时间
        start_time = time.time()

        # 记录请求信息
        logger.info(
            f"[{request_id}] {request.method} {request.url.path} "
            f"- 开始处理"
        )

        # 调用下一个中间件/路由
        try:
            response = await call_next(request)

            # 计算耗时
            process_time = time.time() - start_time

            # 在响应头中添加耗时和请求 ID
            response.headers["X-Process-Time"] = f"{process_time:.3f}"
            response.headers["X-Request-ID"] = request_id

            # 记录响应信息
            logger.info(
                f"[{request_id}] {request.method} {request.url.path} "
                f"- {response.status_code} - {process_time:.3f}s"
            )

            return response

        except Exception as e:
            process_time = time.time() - start_time
            logger.error(
                f"[{request_id}] {request.method} {request.url.path} "
                f"- 500 - {process_time:.3f}s - 错误: {e}"
            )
            return JSONResponse(
                status_code=500,
                content={"detail": "Internal Server Error", "request_id": request_id},
            )


# ========== 注册中间件 ==========

app = FastAPI(title="博客系统 API")

# 中间件的注册顺序很重要！
# 先注册的中间件在最外层
app.add_middleware(RequestLoggingMiddleware)
```

### 示例 2：限流中间件

```python
import time
from collections import defaultdict
from starlette.middleware.base import BaseHTTPMiddleware
from starlette.requests import Request
from starlette.responses import JSONResponse

class RateLimitMiddleware(BaseHTTPMiddleware):
    """基于滑动窗口的简单限流中间件"""

    def __init__(self, app, max_requests: int = 100, window_seconds: int = 60):
        super().__init__(app)
        self.max_requests = max_requests
        self.window_seconds = window_seconds
        # 存储每个 IP 的请求记录: {ip: [timestamp1, timestamp2, ...]}
        self.request_records: dict[str, list[float]] = defaultdict(list)

    def _get_client_ip(self, request: Request) -> str:
        """获取客户端 IP（支持代理）"""
        forwarded = request.headers.get("X-Forwarded-For")
        if forwarded:
            return forwarded.split(",")[0].strip()
        return request.client.host if request.client else "unknown"

    def _is_rate_limited(self, client_ip: str) -> bool:
        """检查是否超过限流阈值"""
        now = time.time()
        window_start = now - self.window_seconds

        # 清理过期记录
        records = self.request_records[client_ip]
        self.request_records[client_ip] = [
            ts for ts in records if ts > window_start
        ]

        # 检查是否超过限制
        return len(self.request_records[client_ip]) >= self.max_requests

    async def dispatch(self, request: Request, call_next):
        client_ip = self._get_client_ip(request)

        if self._is_rate_limited(client_ip):
            return JSONResponse(
                status_code=429,
                content={
                    "detail": "请求过于频繁，请稍后再试",
                    "retry_after": self.window_seconds,
                },
            )

        # 记录本次请求
        self.request_records[client_ip].append(time.time())

        response = await call_next(request)

        # 添加限流信息到响应头
        remaining = self.max_requests - len(self.request_records[client_ip])
        response.headers["X-RateLimit-Remaining"] = str(remaining)
        response.headers["X-RateLimit-Limit"] = str(self.max_requests)

        return response

# 使用
# app.add_middleware(RateLimitMiddleware, max_requests=100, window_seconds=60)
```

### 示例 3：依赖注入基础与进阶

```python
"""
博客系统 - 依赖注入系统
"""
from typing import Annotated, Generator, AsyncGenerator
from fastapi import Depends, HTTPException, Query, Header
from fastapi.security import OAuth2PasswordBearer
from sqlalchemy.ext.asyncio import AsyncSession
from pydantic import BaseModel


# ========== 简单依赖 ==========

def get_pagination(
    page: int = Query(1, ge=1, description="页码"),
    page_size: int = Query(10, ge=1, le=100, description="每页数量"),
) -> dict:
    """分页参数依赖"""
    return {
        "offset": (page - 1) * page_size,
        "limit": page_size,
        "page": page,
        "page_size": page_size,
    }


# ========== yield 依赖（资源清理）==========

async def get_db_session() -> AsyncGenerator[AsyncSession, None]:
    """数据库会话依赖（自动提交和回滚）"""
    async with async_session() as session:
        try:
            yield session
            await session.commit()
        except Exception:
            await session.rollback()
            raise


# ========== 子依赖链 ==========

oauth2_scheme = OAuth2PasswordBearer(tokenUrl="/auth/login")

async def get_current_user_id(token: str = Depends(oauth2_scheme)) -> int:
    """从 Token 中提取用户 ID"""
    # 实际项目中这里应该验证 JWT
    if token == "invalid":
        raise HTTPException(status_code=401, detail="无效的 Token")
    return 1  # 模拟返回用户 ID

async def get_current_user(
    user_id: int = Depends(get_current_user_id),
    session: AsyncSession = Depends(get_db_session),
) -> dict:
    """获取当前用户完整信息（依赖 get_current_user_id 和 get_db_session）"""
    # 从数据库获取用户信息
    user = {"id": user_id, "username": "admin", "role": "admin"}
    return user

# ========== 基于角色的权限依赖 ==========

def require_role(*roles: str):
    """角色权限依赖工厂"""
    async def check_permission(
        current_user: dict = Depends(get_current_user),
    ) -> dict:
        if current_user["role"] not in roles:
            raise HTTPException(
                status_code=403,
                detail=f"权限不足，需要以下角色之一: {', '.join(roles)}",
            )
        return current_user
    return check_permission


# ========== 在路由中使用依赖 ==========

from fastapi import FastAPI, APIRouter

router = APIRouter(prefix="/posts", tags=["文章"])

@router.get("/")
async def list_posts(
    pagination: dict = Depends(get_pagination),
    session: AsyncSession = Depends(get_db_session),
):
    """获取文章列表（使用分页依赖和数据库依赖）"""
    # 使用 pagination["offset"] 和 pagination["limit"]
    return {"page": pagination["page"], "page_size": pagination["page_size"]}

@router.post("/")
async def create_post(
    current_user: dict = Depends(require_role("admin", "editor")),
    session: AsyncSession = Depends(get_db_session),
):
    """创建文章（需要 admin 或 editor 角色）"""
    return {"message": f"用户 {current_user['username']} 创建文章成功"}

@router.get("/me")
async def get_my_profile(
    current_user: dict = Depends(get_current_user),
):
    """获取当前用户信息"""
    return current_user

app = FastAPI()
app.include_router(router)
```

### 示例 4：依赖作用域与缓存

```python
"""
依赖的作用域控制缓存行为
"""
from fastapi import Depends
from functools import lru_cache

# ========== 应用级依赖（全局单例）==========

@lru_cache()
def get_settings():
    """配置依赖 - 使用 lru_cache 确保全局单例"""
    return {
        "database_url": "postgresql://localhost/blog",
        "redis_url": "redis://localhost:6379",
        "jwt_secret": "your-secret-key",
        "debug": False,
    }

# ========== 使用 Annotated 简化依赖签名 ==========

from typing import Annotated

# 定义类型别名，避免重复写 Depends(...)
DbSession = Annotated[AsyncSession, Depends(get_db_session)]
PaginationParams = Annotated[dict, Depends(get_pagination)]
CurrentUser = Annotated[dict, Depends(get_current_user)]
Settings = Annotated[dict, Depends(get_settings)]

@router.get("/settings-demo")
async def settings_demo(settings: Settings):
    """使用类型别名注入配置"""
    return {"database_url": settings["database_url"]}

@router.get("/user-posts")
async def get_user_posts(
    user: CurrentUser,
    pagination: PaginationParams,
    session: DbSession,
):
    """使用类型别名简化依赖注入"""
    return {
        "user": user["username"],
        "page": pagination["page"],
    }
```

### 示例 5：应用生命周期管理

```python
"""
博客系统 - 应用生命周期管理
使用 FastAPI 的 lifespan context manager 替代 @app.on_event
"""
from contextlib import asynccontextmanager
from fastapi import FastAPI
from sqlalchemy.ext.asyncio import create_async_engine, AsyncSession, async_sessionmaker

# 全局数据库引擎和会话工厂
engine = None
async_session_factory = None


@asynccontextmanager
async def lifespan(app: FastAPI):
    """应用生命周期管理"""
    global engine, async_session_factory

    # ===== 启动阶段 =====
    print("博客系统正在启动...")

    # 初始化数据库连接
    engine = create_async_engine(
        "postgresql+asyncpg://user:pass@localhost:5432/blog",
        pool_size=10,
        max_overflow=20,
        pool_pre_ping=True,
    )
    async_session_factory = async_sessionmaker(
        engine, class_=AsyncSession, expire_on_commit=False
    )

    # 初始化 Redis 连接
    print("  [Redis] 连接成功")

    # 预热缓存
    print("  [Cache] 缓存预热完成")

    print("博客系统启动完成!")

    # yield 之前是启动阶段，之后是关闭阶段
    yield

    # ===== 关闭阶段 =====
    print("\n博客系统正在关闭...")

    # 关闭数据库连接池
    if engine:
        await engine.dispose()
        print("  [Database] 连接池已关闭")

    # 关闭 Redis 连接
    print("  [Redis] 连接已关闭")

    print("博客系统已安全关闭")


# 创建 FastAPI 应用
app = FastAPI(
    title="博客系统 API",
    version="1.0.0",
    lifespan=lifespan,
)
```

### 示例 6：内置中间件配置

```python
from fastapi import FastAPI
from fastapi.middleware.cors import CORSMiddleware
from fastapi.middleware.gzip import GZipMiddleware
from fastapi.middleware.trustedhost import TrustedHostMiddleware

app = FastAPI()

# CORS 中间件（跨域资源共享）
app.add_middleware(
    CORSMiddleware,
    allow_origins=[
        "http://localhost:3000",   # 开发环境前端
        "https://myblog.com",       # 生产环境前端
    ],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
    expose_headers=["X-Request-ID", "X-Process-Time"],  # 暴露给前端的自定义头
)

# GZip 压缩中间件（响应体大于阈值时自动压缩）
app.add_middleware(GZipMiddleware, minimum_size=1000)

# 受信任主机中间件（防止 Host 头攻击）
app.add_middleware(
    TrustedHostMiddleware,
    allowed_hosts=[
        "localhost",
        "127.0.0.1",
        "myblog.com",
        "*.myblog.com",
    ],
)
```

## 最佳实践

### 1. 中间件注册顺序

```python
app = FastAPI()

# 正确的中间件顺序（从外到内）：
# 1. 安全相关（限流、信任主机）
# 2. 日志记录
# 3. CORS
# 4. GZip 压缩
app.add_middleware(RateLimitMiddleware)
app.add_middleware(RequestLoggingMiddleware)
app.add_middleware(CORSMiddleware, ...)
app.add_middleware(GZipMiddleware, ...)
```

### 2. 依赖注入 vs 中间件的选择

```python
# 使用中间件的场景：
# - 需要处理所有请求（日志、CORS、压缩）
# - 需要修改请求或响应对象
# - 不关心具体路由逻辑

# 使用依赖注入的场景：
# - 只需要在特定路由中使用（认证、数据库会话）
# - 需要返回值供路由使用
# - 需要精细控制作用域
```

### 3. 使用 Annotated 简化代码

```python
from typing import Annotated
from fastapi import Depends

# 推荐：使用类型别名
DbSession = Annotated[AsyncSession, Depends(get_db_session)]

@router.get("/posts")
async def list_posts(session: DbSession):  # 简洁清晰
    ...

# 不推荐：每次都写 Depends
@router.get("/posts")
async def list_posts(session: AsyncSession = Depends(get_db_session)):  # 冗长
    ...
```

## 常见问题

**Q: 中间件中的异常会被 FastAPI 的异常处理器捕获吗？**

A: 不会。`BaseHTTPMiddleware` 中抛出的异常不会被 FastAPI 的 `exception_handler` 捕获。你需要在中间件内部自行处理异常并返回适当的响应。

**Q: yield 依赖中的异常会怎样？**

A: 如果 `yield` 之后的代码（清理阶段）抛出异常，FastAPI 会记录日志但不会影响已发送的响应。`yield` 之前的代码抛出异常时，yield 之后的清理代码不会执行，需要用 `try/finally` 包裹。

**Q: 全局依赖和路由依赖有什么区别？**

A: 全局依赖（`app = FastAPI(dependencies=[...])`）会应用到所有路由，路由依赖（`@router.get("/", dependencies=[...])`）只应用到特定路由。全局依赖适合做认证检查，路由依赖适合做特定权限控制。

**Q: 依赖的缓存机制是怎样的？**

A: 默认情况下，同一个请求中相同的依赖只会执行一次（缓存结果）。可以使用 `Depends(..., use_cache=False)` 禁用缓存。跨请求不会缓存，除非使用 `lru_cache` 等工具。
