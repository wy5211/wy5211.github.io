# 环境搭建与第一个项目

## 前言

如果你已经熟悉 Java、TypeScript 或 Go，学习 Python 的第一步不是语法，而是搭建一个**规范、可复现**的开发环境。Python 的生态与 Node.js 和 Go 有相似之处，但也有自己独特的工具链。本文将带你从零搭建环境，并完成第一个命令行项目。

---

## Python 版本选择

当前推荐使用 **Python 3.12+**，它已经非常成熟，且性能相比 3.11 有显著提升（快 5%-10%）。

| 特性           | Python 3.12 | Python 3.11    | Python 3.10 |
| -------------- | ----------- | -------------- | ----------- |
| 性能           | 基准        | 比 3.10 快 25% | 基准        |
| match-case     | 支持        | 支持           | 支持        |
| 更好的错误提示 | 大幅改进    | 改进           | 基础        |
| 参数注解语法   | 支持        | 不支持         | 不支持      |
| 推荐度         | 首选        | 可用           | 最低要求    |

> 如果你使用的框架（如某些旧版 Django）还不支持 3.12，退回到 3.11 即可。

## 安装方式

### 方式一：系统安装

从 [python.org](https://www.python.org/downloads/) 下载安装包，macOS 和 Windows 都提供图形化安装器。

```bash
# macOS (Homebrew)
brew install python@3.12

# Ubuntu/Debian
sudo apt update && sudo apt install python3.12 python3.12-venv

# 验证安装
python3 --version
# Python 3.12.x
```

### 方式二：pyenv（推荐）

pyenv 允许你在同一台机器上安装和切换多个 Python 版本，类似于 Java 开发者使用 SDKMAN 或 Node.js 开发者使用 nvm。

```bash
# macOS 安装 pyenv
brew install pyenv

# 安装 Python 3.12
pyenv install 3.12.4

# 设置全局默认版本
pyenv global 3.12.4

# 验证
python --version
# Python 3.12.4
```

安装完成后，需要将 pyenv 配置写入 shell 配置文件（`~/.zshrc`）：

```bash
export PYENV_ROOT="$HOME/.pyenv"
export PATH="$PYENV_ROOT/bin:$PATH"
eval "$(pyenv init --path)"
eval "$(pyenv init -)"
```

| 工具     | Python   | Node.js      | Java           | Go         |
| -------- | -------- | ------------ | -------------- | ---------- |
| 版本管理 | pyenv    | nvm / fnm    | SDKMAN         | Go 自管理  |

## 虚拟环境

Python 的虚拟环境（Virtual Environment）是**隔离项目依赖**的标准方式。每个项目应该有自己独立的虚拟环境，避免依赖冲突。

```bash
# 创建虚拟环境（在项目根目录）
python3 -m venv .venv

# 激活虚拟环境
# macOS / Linux
source .venv/bin/activate

# Windows
.venv\Scripts\activate

# 激活后，终端提示符前会出现 (.venv)
(.venv) $

# 退出虚拟环境
deactivate
```

> **为什么需要虚拟环境？** 和 Node.js 的 `node_modules` 不同，Python 全局安装的包会影响所有项目。虚拟环境本质上是在项目目录下创建了一个独立的 Python 解释器副本。

将 `.venv` 加入 `.gitignore`：

```gitignore
# Python
.venv/
__pycache__/
*.pyc
dist/
build/
*.egg-info/
```

## 包管理工具

### pip — 经典工具

pip 是 Python 自带的包管理器，类似 npm。

```bash
# 安装包
pip install requests

# 生成依赖文件
pip freeze > requirements.txt

# 从依赖文件安装
pip install -r requirements.txt

# 卸载包
pip uninstall requests

# 查看已安装的包
pip list
```

### uv — 新一代工具（强烈推荐）

[uv](https://github.com/astral-sh/uv) 由 Ruff 的开发团队 Astral 打造，用 Rust 编写，速度比 pip 快 **10-100 倍**。它同时替代了 pip、virtualenv、pip-tools 等多个工具。

```bash
# macOS / Linux 安装 uv
curl -LsSf https://astral.sh/uv/install.sh | sh

# 或者通过 Homebrew
brew install uv
```

## 使用 uv 创建第一个项目

uv 提供了一个类似 `npm init` / `cargo new` 的项目脚手架功能：

```bash
# 创建新项目
uv init my-cli-tool
cd my-cli-tool

# 项目结构
# my-cli-tool/
# ├── hello.py          # 入口文件
# ├── pyproject.toml    # 项目配置（类似 package.json）
# └── README.md
```

生成的 `pyproject.toml` 类似 Node.js 的 `package.json`：

```toml
[project]
name = "my-cli-tool"
version = "0.1.0"
description = "My first Python CLI tool"
requires-python = ">=3.12"
dependencies = []

[build-system]
requires = ["hatchling"]
build-backend = "hatchling.build"
```

添加依赖：

```bash
# 添加依赖（自动更新 pyproject.toml 并安装）
uv add requests
uv add click

# 同步依赖（类似 npm install）
uv sync
```

## 第一个命令行工具

下面用 Python 标准库 `argparse` 创建一个实用的命令行工具——文件批量重命名工具。

```python
#!/usr/bin/env python3
"""
batch_rename.py - 文件批量重命名工具
用法: python batch_rename.py <目录> --prefix <前缀> --ext <新扩展名>
"""

import argparse
import os
import sys
from pathlib import Path


def parse_args():
    """解析命令行参数"""
    parser = argparse.ArgumentParser(
        description="批量重命名文件工具",
        formatter_class=argparse.RawDescriptionHelpFormatter,
        epilog="""
示例:
  python batch_rename.py ./photos --prefix vacation_
  python batch_rename.py ./docs --ext .md
  python batch_rename.py ./downloads --dry-run
        """
    )
    parser.add_argument(
        "directory",
        type=str,
        help="目标目录路径"
    )
    parser.add_argument(
        "--prefix",
        type=str,
        default=None,
        help="文件名前缀"
    )
    parser.add_argument(
        "--ext",
        type=str,
        default=None,
        help="新的文件扩展名（如 .md、.txt）"
    )
    parser.add_argument(
        "--dry-run",
        action="store_true",
        help="仅预览重命名结果，不实际执行"
    )

    return parser.parse_args()


def batch_rename(directory: str, prefix: str = None, ext: str = None, dry_run: bool = False):
    """批量重命名目录中的文件"""
    target_dir = Path(directory)

    if not target_dir.exists():
        print(f"错误: 目录 '{directory}' 不存在")
        sys.exit(1)

    if not target_dir.is_dir():
        print(f"错误: '{directory}' 不是目录")
        sys.exit(1)

    files = sorted(target_dir.iterdir())
    renamed_count = 0

    print(f"目标目录: {target_dir.resolve()}")
    print(f"文件数量: {len(files)}")
    print("-" * 50)

    for file_path in files:
        if not file_path.is_file():
            continue

        # 构建新文件名
        new_name = file_path.name

        if prefix:
            new_name = prefix + new_name

        if ext:
            new_name = file_path.stem + ext

        if new_name == file_path.name:
            continue  # 文件名无变化，跳过

        new_path = target_dir / new_name
        renamed_count += 1

        # 显示重命名结果
        action = "预览" if dry_run else "重命名"
        print(f"  [{action}] {file_path.name} -> {new_name}")

        if not dry_run:
            file_path.rename(new_path)

    print("-" * 50)
    action = "将" if dry_run else "已"
    print(f"共 {action}重命名 {renamed_count} 个文件")


if __name__ == "__main__":
    args = parse_args()
    batch_rename(args.directory, args.prefix, args.ext, args.dry_run)
```

运行效果：

```bash
# 预览重命名
python batch_rename.py ./photos --prefix 2024_ --dry-run

# 实际执行
python batch_rename.py ./photos --prefix 2024_

# 修改扩展名
python batch_rename.py ./notes --ext .md
```

## 交互式编程

Python 的 REPL（Read-Eval-Print Loop）是一个强大的交互式编程环境，类似 Node.js 的 REPL 或浏览器的开发者控制台。

```bash
# 启动 Python REPL
python

# 或使用增强版 IPython（需安装: uv add ipython）
ipython
```

```python
>>> # 快速计算
>>> sum(range(1, 101))
5050

>>> # 快速测试
>>> import json
>>> json.dumps({"name": "Alice", "age": 30}, ensure_ascii=False)
'{"name": "Alice", "age": 30}'

>>> # 查看帮助
>>> help(str.split)
>>> dir(list)  # 查看对象所有方法和属性
```

> **IPython 的优势**：语法高亮、自动补全、魔术命令（`%timeit` 测性能、`%debug` 调试）、更好的错误提示。推荐日常开发中使用。

## Python 与其他语言的差异速览

| 特性       | Python            | Java               | TypeScript                 | Go                              |
| ---------- | ----------------- | ------------------ | -------------------------- | ------------------------------- |
| 类型系统   | 动态 + 类型注解   | 静态强类型         | 静态（编译期检查）         | 静态强类型                      |
| 变量声明   | `x = 10`          | `int x = 10;`      | `let x: number = 10;`      | `x := 10` 或 `var x int = 10`   |
| 代码块     | 缩进              | `{}`               | `{}`                       | `{}`                            |
| 分号       | 不需要            | 需要               | 可选                       | 需要                            |
| 字符串插值 | `f"Hello {name}"` | `"Hello " + name`  | `` `Hello ${name}` ``      | `fmt.Sprintf("Hello %s", name)` |
| 数组       | `list`            | `ArrayList<>`      | `Array<T>`                 | `[]T` (slice)                   |
| 哈希表     | `dict`            | `HashMap<K,V>`     | `Map<K,V>` / `Record<K,V>` | `map[K]V`                       |
| 空值       | `None`            | `null`             | `null` / `undefined`       | `nil`                           |
| 异常处理   | `try/except`      | `try/catch`        | `try/catch`                | `error` 返回值                  |

## 最佳实践

1. **始终使用虚拟环境**：每个项目独立隔离依赖，避免全局污染
2. **优先使用 uv**：比 pip 快 10-100 倍，且集成了项目管理和虚拟环境管理
3. **使用 `pyproject.toml`**：这是现代 Python 项目的标准配置文件，取代了 `setup.py` 和 `requirements.txt`
4. **使用 `if __name__ == "__main__":`**：确保模块被导入时不会执行主逻辑
5. **版本控制忽略 `.venv` 和 `__pycache__`**：虚拟环境和编译缓存不应该提交到 Git

## 常见问题

**Q: Python 2 和 Python 3 我该学哪个？**

A: 只学 Python 3。Python 2 已于 2020 年 1 月 1 日正式停止维护，所有新项目都应该使用 Python 3。

**Q: venv、virtualenv、conda 有什么区别？**

A: `venv` 是 Python 标准库内置的虚拟环境工具，足以应对大多数场景。`virtualenv` 是第三方工具，功能更多但需要额外安装。`conda` 是 Anaconda 发行版的一部分，主要用于数据科学，可以同时管理 Python 和非 Python 依赖（如 CUDA）。一般开发推荐 `venv`。

**Q: uv 能完全替代 pip 吗？**

A: 可以。uv 兼容 pip 的所有接口，且速度更快。你可以通过 `uv pip install` 命令使用，也可以通过 `uv add` 使用更现代的项目管理方式。对于新项目，推荐使用 `uv init` + `uv add` 的工作流。

**Q: Python 的包管理比 Node.js 复杂吗？**

A: 概念上不复杂，但历史包袱较重。Python 有多种包管理工具（pip、pip-tools、Poetry、uv、conda），不像 Node.js 基本统一在 npm/pnpm 上。建议新项目直接使用 uv，避免纠结工具选择。
