## 核心概念

### 为什么选择 FastAPI？

FastAPI 是现代 Python Web 框架中增长最快的项目之一。来看看它与其他框架的对比：

| 特性         | FastAPI            | Flask           | Django          |
| ------------ | ------------------ | --------------- | --------------- |
| **异步支持** | 原生 async/await   | 需要扩展        | 从 3.1 开始支持 |
| **类型注解** | 深度集成           | 无内置支持      | 部分支持        |
| **自动文档** | Swagger UI + ReDoc | 需要扩展        | 需要扩展        |
| **数据验证** | Pydantic v2 内置   | 无内置          | 内置 Form 验证  |
| **依赖注入** | 内置 Depends       | 无内置          | 有限支持        |
| **性能**     | 接近 Go/Node.js    | 中等            | 中等            |
| **学习曲线** | 低                 | 很低            | 中等            |
| **适用场景** | API 服务           | 小型应用/微服务 | 全栈 Web 应用   |

**类比理解**：

- **NestJS 开发者**：FastAPI 相当于去掉装饰器负担、用函数路由替代 Controller 的轻量版 NestJS
- **Spring Boot 开发者**：FastAPI 的 `Depends` 机制类似于 Spring 的 `@Autowired`，Pydantic 类似于 Java 的 DTO 校验
- **Express 开发者**：FastAPI 是 Express + TypeScript 类型检查 + 自动 Swagger 文档的超集

### 安装与项目创建

```bash
# 创建项目目录
mkdir blog-api && cd blog-api

# 使用 uv 创建虚拟环境并安装依赖
uv init
uv add fastapi uvicorn[standard]

# 或者使用 pip
python -m venv venv
source venv/bin/activate
pip install fastapi "uvicorn[standard]"
```

创建最简单的 FastAPI 应用：

```python
# main.py
from fastapi import FastAPI

app = FastAPI(title="博客系统 API", version="1.0.0")


@app.get("/")
async def root():
    return {"message": "Hello, FastAPI!"}


@app.get("/health")
async def health_check():
    return {"status": "ok"}
```

启动服务器：

```bash
# 开发模式（自动重载）
uvicorn main:app --reload

# 生产模式
uvicorn main:app --host 0.0.0.0 --port 8000 --workers 4
```

> **类比 NestJS**：`uvicorn main:app` 类似于 `nest start`，`main:app` 表示从 `main.py` 中导入 `app` 对象。

## 实战演练

### 示例 1：路由定义与 HTTP 方法

FastAPI 使用装饰器定义路由，支持所有 HTTP 方法：

```python
# routes/todos.py
from fastapi import APIRouter, HTTPException

router = APIRouter(prefix="/todos", tags=["Todos"])

# 模拟数据库
todos_db: list[dict] = []
todo_id_counter = 0


@router.post("/", status_code=201)
async def create_todo(title: str, description: str = ""):
    """创建一个 Todo"""
    global todo_id_counter
    todo_id_counter += 1
    todo = {
        "id": todo_id_counter,
        "title": title,
        "description": description,
        "completed": False,
    }
    todos_db.append(todo)
    return todo


@router.get("/")
async def list_todos():
    """获取所有 Todo"""
    return todos_db


@router.get("/{todo_id}")
async def get_todo(todo_id: int):
    """根据 ID 获取 Todo"""
    for todo in todos_db:
        if todo["id"] == todo_id:
            return todo
    raise HTTPException(status_code=404, detail="Todo not found")


@router.put("/{todo_id}")
async def update_todo(todo_id: int, title: str | None = None, completed: bool | None = None):
    """更新 Todo"""
    for todo in todos_db:
        if todo["id"] == todo_id:
            if title is not None:
                todo["title"] = title
            if completed is not None:
                todo["completed"] = completed
            return todo
    raise HTTPException(status_code=404, detail="Todo not found")


@router.delete("/{todo_id}", status_code=204)
async def delete_todo(todo_id: int):
    """删除 Todo"""
    for i, todo in enumerate(todos_db):
        if todo["id"] == todo_id:
            todos_db.pop(i)
            return
    raise HTTPException(status_code=404, detail="Todo not found")
```

**代码解释**：

- `APIRouter` 类似于 NestJS 的 `@Controller`，用于组织路由
- `prefix="/todos"` 为所有路由添加统一前缀
- `tags=["Todos"]` 在 Swagger 文档中对路由进行分组
- `{todo_id}` 是路径参数，FastAPI 会自动将其转换为 `int` 类型
- `HTTPException` 类似于 NestJS 的 `NotFoundException`

### 示例 2：路径参数与查询参数

FastAPI 通过函数签名自动区分路径参数、查询参数和请求体：

```python
# routes/params_demo.py
from fastapi import APIRouter, Query, Path

router = APIRouter(prefix="/demo", tags=["参数演示"])


@router.get("/items/{item_id}")
async def read_item(
    item_id: int = Path(..., description="物品 ID", ge=1),  # 路径参数
    q: str | None = Query(None, max_length=50),              # 查询参数
    skip: int = Query(0, ge=0),                              # 带默认值的查询参数
    limit: int = Query(10, ge=1, le=100),                    # 带约束的查询参数
):
    """
    演示不同类型的参数

    - item_id: 路径参数，必须 >= 1
    - q: 可选的搜索关键词
    - skip/limit: 分页参数
    """
    return {
        "item_id": item_id,
        "q": q,
        "skip": skip,
        "limit": limit,
    }


@router.get("/users/{user_id}/posts/{post_id}")
async def read_user_post(user_id: int, post_id: int):
    """多路径参数"""
    return {"user_id": user_id, "post_id": post_id}
```

**参数识别规则**：

| 参数类型     | 判断条件                             | 示例                |
| ------------ | ------------------------------------ | ------------------- |
| **路径参数** | 在路径中声明 `{param}`               | `/{item_id}`        |
| **查询参数** | 非路径参数、非请求体、有默认值或可选 | `?q=keyword&skip=0` |
| **请求体**   | 使用 Pydantic 模型声明               | `body: ItemCreate`  |

### 示例 3：请求体与 Pydantic 模型

这是 FastAPI 的核心特性 -- 使用 Pydantic 模型定义和验证请求体：

```python
# schemas/todo.py
from datetime import datetime
from pydantic import BaseModel, Field


class TodoCreate(BaseModel):
    """创建 Todo 的请求模型"""
    title: str = Field(..., min_length=1, max_length=200, description="标题")
    description: str = Field(default="", max_length=2000, description="描述")


class TodoUpdate(BaseModel):
    """更新 Todo 的请求模型 -- 所有字段可选"""
    title: str | None = Field(None, min_length=1, max_length=200)
    description: str | None = Field(None, max_length=2000)
    completed: bool | None = None


class TodoResponse(BaseModel):
    """Todo 的响应模型"""
    id: int
    title: str
    description: str
    completed: bool
    created_at: datetime

    model_config = {"from_attributes": True}
```

在路由中使用这些模型：

```python
# routes/todos_v2.py
from datetime import datetime
from fastapi import APIRouter, HTTPException
from schemas.todo import TodoCreate, TodoUpdate, TodoResponse

router = APIRouter(prefix="/v2/todos", tags=["Todos v2"])

todos_db: list[dict] = []
todo_id_counter = 0


@router.post("/", response_model=TodoResponse, status_code=201)
async def create_todo(todo: TodoCreate):
    """
    创建 Todo

    - 请求体使用 TodoCreate 模型验证
    - 响应使用 TodoResponse 模型格式化
    """
    global todo_id_counter
    todo_id_counter += 1
    new_todo = {
        "id": todo_id_counter,
        "title": todo.title,
        "description": todo.description,
        "completed": False,
        "created_at": datetime.now(),
    }
    todos_db.append(new_todo)
    return new_todo


@router.put("/{todo_id}", response_model=TodoResponse)
async def update_todo(todo_id: int, todo: TodoUpdate):
    """
    更新 Todo

    - TodoUpdate 中所有字段可选，只更新传入的字段
    - 类似于 NestJS 中的 PartialType(CreateDto)
    """
    for item in todos_db:
        if item["id"] == todo_id:
            update_data = todo.model_dump(exclude_unset=True)
            item.update(update_data)
            return item
    raise HTTPException(status_code=404, detail="Todo not found")
```

**代码解释**：

- `response_model=TodoResponse` 指定响应的数据格式，FastAPI 会自动过滤掉不在模型中的字段
- `model_dump(exclude_unset=True)` 只获取用户实际传入的字段，实现部分更新
- `model_config = {"from_attributes": True}` 允许从 ORM 对象创建 Pydantic 模型（替代旧版的 `orm_mode = True`）

> **类比 NestJS**：`response_model` 类似于 `@ApiResponse({ type: TodoResponse })`，`TodoCreate` 类似于 `CreateTodoDto`。

### 示例 4：依赖注入系统（Depends）

FastAPI 的依赖注入是其最强大的特性之一：

```python
# dependencies.py
from fastapi import Depends, Header, HTTPException


async def get_api_key(api_key: str = Header(..., alias="X-API-Key")):
    """验证 API Key 的依赖"""
    if api_key != "secret-key":
        raise HTTPException(status_code=403, detail="Invalid API Key")
    return api_key


def common_parameters(
    skip: int = Query(0, ge=0),
    limit: int = Query(10, ge=1, le=100),
):
    """可复用的分页参数"""
    return {"skip": skip, "limit": limit}
```

在路由中使用依赖：

```python
# routes/protected.py
from fastapi import APIRouter, Depends
from dependencies import get_api_key, common_parameters

router = APIRouter(tags=["受保护的路由"])


@router.get("/items/")
async def read_items(commons: dict = Depends(common_parameters)):
    """使用公共分页参数"""
    return commons


@router.get("/protected/")
async def protected_route(api_key: str = Depends(get_api_key)):
    """需要 API Key 的受保护路由"""
    return {"message": "Access granted", "api_key": api_key}
```

> **类比 Spring Boot**：`Depends(get_api_key)` 类似于 `@Autowired` 注入一个 Bean，FastAPI 的依赖注入更灵活，可以是函数、类或生成器。

### 示例 5：完整的 Todo API

将所有知识整合成一个完整的应用：

```python
# main.py
from contextlib import asynccontextmanager
from datetime import datetime
from fastapi import FastAPI, HTTPException, Query
from fastapi.middleware.cors import CORSMiddleware

# --- 模拟数据库 ---
todos_db: list[dict] = []
todo_id_counter = 0


# --- Pydantic 模型 ---
class TodoCreate:
    pass  # 前面已定义，此处省略


class TodoCreate(BaseModel):
    title: str = Field(..., min_length=1, max_length=200)
    description: str = Field(default="", max_length=2000)


class TodoUpdate(BaseModel):
    title: str | None = Field(None, min_length=1, max_length=200)
    description: str | None = Field(None, max_length=2000)
    completed: bool | None = None


class TodoResponse(BaseModel):
    id: int
    title: str
    description: str
    completed: bool
    created_at: datetime

    model_config = {"from_attributes": True}


# --- 应用生命周期 ---
@asynccontextmanager
async def lifespan(app: FastAPI):
    """应用启动和关闭时的操作"""
    # 启动时执行
    print("Blog API starting...")
    yield
    # 关闭时执行
    print("Blog API shutting down...")
    todos_db.clear()


# --- 创建 FastAPI 应用 ---
app = FastAPI(
    title="博客系统 API",
    description="基于 FastAPI 的博客系统",
    version="1.0.0",
    lifespan=lifespan,
)

# --- CORS 配置 ---
app.add_middleware(
    CORSMiddleware,
    allow_origins=["http://localhost:3000"],  # 生产环境应配置具体域名
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)


# --- 路由 ---
@app.get("/", tags=["Health"])
async def root():
    return {"message": "Blog API is running"}


@app.post("/todos", response_model=TodoResponse, status_code=201, tags=["Todos"])
async def create_todo(todo: TodoCreate):
    """创建 Todo"""
    global todo_id_counter
    todo_id_counter += 1
    new_todo = {
        "id": todo_id_counter,
        "title": todo.title,
        "description": todo.description,
        "completed": False,
        "created_at": datetime.now(),
    }
    todos_db.append(new_todo)
    return new_todo


@app.get("/todos", response_model=list[TodoResponse], tags=["Todos"])
async def list_todos(
    skip: int = Query(0, ge=0, description="跳过数量"),
    limit: int = Query(10, ge=1, le=100, description="每页数量"),
    completed: bool | None = Query(None, description="按完成状态过滤"),
):
    """获取 Todo 列表，支持分页和过滤"""
    result = todos_db
    if completed is not None:
        result = [t for t in result if t["completed"] == completed]
    return result[skip : skip + limit]


@app.get("/todos/{todo_id}", response_model=TodoResponse, tags=["Todos"])
async def get_todo(todo_id: int):
    """获取单个 Todo"""
    for todo in todos_db:
        if todo["id"] == todo_id:
            return todo
    raise HTTPException(status_code=404, detail="Todo not found")


@app.put("/todos/{todo_id}", response_model=TodoResponse, tags=["Todos"])
async def update_todo(todo_id: int, todo: TodoUpdate):
    """更新 Todo"""
    for item in todos_db:
        if item["id"] == todo_id:
            update_data = todo.model_dump(exclude_unset=True)
            item.update(update_data)
            return item
    raise HTTPException(status_code=404, detail="Todo not found")


@app.delete("/todos/{todo_id}", status_code=204, tags=["Todos"])
async def delete_todo(todo_id: int):
    """删除 Todo"""
    for i, item in enumerate(todos_db):
        if item["id"] == todo_id:
            todos_db.pop(i)
            return
    raise HTTPException(status_code=404, detail="Todo not found")
```

### 自动文档

FastAPI 最令人印象深刻的功能之一是自动生成的 API 文档。启动服务后访问：

- **Swagger UI**：`http://localhost:8000/docs`
- **ReDoc**：`http://localhost:8000/redoc`
- **OpenAPI JSON**：`http://localhost:8000/openapi.json`

这些文档会根据你的路由、参数、Pydantic 模型自动生成，无需任何额外配置。这在 NestJS 中需要安装 `@nestjs/swagger` 并手动添加装饰器才能实现。

### 1. 使用 APIRouter 组织路由

```python
# routes/todos.py
router = APIRouter(prefix="/todos", tags=["Todos"])

router = APIRouter(prefix="/users", tags=["Users"])

from routes import todos, users
```

### 2. 合理使用 async

```python
# I/O 密集型操作使用 async
@app.get("/external-api")
async def call_external():
    async with httpx.AsyncClient() as client:
        response = await client.get("https://api.example.com/data")
        return response.json()

# CPU 密集型或同步库使用普通函数
@router.get("/sync-compute")
def heavy_compute():
    # numpy、pandas 等同步库不需要 async
    result = numpy_large_computation()
    return {"result": result}
```

### 3. 使用 Pydantic 模型统一接口

```python
# 始终为请求和响应定义模型
# 好的做法
@app.post("/todos", response_model=TodoResponse)
async def create(todo: TodoCreate): ...

# 不好的做法 -- 缺少类型约束
@app.post("/todos")
async def create(title: str, desc: str): ...
```

```python

from pydantic_settings import BaseSettings


class Settings(BaseSettings):

    debug: bool = False
    database_url: str = "postgresql://localhost/blog"

    model_config = {"env_file": ".env"}


settings = Settings()
```

## 常见问题

**Q: FastAPI 和 Flask 应该怎么选？**

A: 如果你只需要一个简单的 API 或微服务，Flask 足够。但如果你需要自动文档、数据验证、类型提示和异步支持，FastAPI 是更好的选择。新项目建议直接用 FastAPI。

**Q: 必须使用 async 吗？**

A: 不是。FastAPI 会自动处理同步和异步函数。如果你使用的库是同步的（如 SQLAlchemy 的同步模式），直接写普通函数即可。FastAPI 会将其放到线程池中执行，不会阻塞事件循环。

**Q: `Depends` 和中间件有什么区别？**

A: `Depends` 是路由级别的依赖注入，适合特定路由的鉴权、参数复用等场景。中间件是全局的，适合日志记录、CORS、请求追踪等横切关注点。

**Q: FastAPI 的性能如何？**

A: 根据 TechEmpower 基准测试，FastAPI（基于 Starlette）是 Python Web 框架中最快的之一，性能接近 Node.js 和 Go。但在极端性能场景下，仍建议配合缓存（Redis）和异步数据库驱动。
