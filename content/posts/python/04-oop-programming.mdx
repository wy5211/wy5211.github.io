# 面向对象编程

## 前言

如果你来自 Java 或 TypeScript 的世界，Python 的 OOP 会让你感到既熟悉又陌生。Python 支持类、继承、封装等经典 OOP 概念，但实现方式有很大不同——Python 没有 `public`/`private` 关键字，支持多重继承，还有一套独特的"魔术方法"（Magic Methods）系统。本文将从对比的角度出发，帮你快速建立 Python OOP 的心智模型。

---

## 类定义与实例化

```python
class User:
    """用户类"""

    # 类属性 — 所有实例共享
    species = "Homo sapiens"

    def __init__(self, name: str, age: int, email: str = ""):
        """构造函数 — self 相当于 this"""
        # 实例属性 — 每个实例独立
        self.name = name
        self.age = age
        self.email = email
        self._login_count = 0  # 约定私有：单下划线
        self.__secret = "hidden"  # 名称改写：双下划线

    def greet(self) -> str:
        """实例方法"""
        return f"Hi, I'm {self.name}, {self.age} years old."

    @classmethod
    def from_dict(cls, data: dict) -> "User":
        """类方法 — 工厂方法，cls 指向当前类"""
        return cls(**data)

    @staticmethod
    def is_adult(age: int) -> bool:
        """静态方法 — 不需要 self 或 cls"""
        return age >= 18


# 创建实例
user = User("Alice", 30, "alice@example.com")
print(user.greet())           # Hi, I'm Alice, 30 years old.
print(User.is_adult(30))      # True

# 工厂方法
data = {"name": "Bob", "age": 25, "email": "bob@example.com"}
user2 = User.from_dict(data)

# 类属性
print(User.species)   # Homo sapiens
print(user.species)   # Homo sapiens
User.species = "Human"
print(user.species)   # Human — 实例读取类属性
```

### 与 Java/TypeScript 的对比

| 特性     | Python                      | Java           | TypeScript       |
| -------- | --------------------------- | -------------- | ---------------- |
| 构造函数 | `def __init__(self)`        | 构造函数同名   | `constructor()`  |
| this     | `self`（显式参数）          | `this`（隐式） | `this`（隐式）   |
| 类属性   | 直接定义在类体中            | `static` 字段  | `static` 属性    |
| 实例属性 | `self.x = ...`              | 字段声明       | 属性声明         |
| 私有     | `_x`（约定）/ `__x`（改写） | `private`      | `#x`（私有字段） |
| 工厂方法 | `@classmethod`              | 静态工厂方法   | 静态方法         |

## `__init__` 与 `__new__`

```python
class Singleton:
    """单例模式实现"""
    _instance = None

    def __new__(cls, *args, **kwargs):
        """__new__ 是真正的构造方法，负责创建实例"""
        if cls._instance is None:
            cls._instance = super().__new__(cls)
        return cls._instance

    def __init__(self):
        """__init__ 是初始化方法，负责设置属性"""
        if not hasattr(self, "initialized"):
            self.value = 0
            self.initialized = True


a = Singleton()
b = Singleton()
print(a is b)  # True — 同一个实例
```

> **关键区别**：`__new__` 负责创建对象（分配内存），`__init__` 负责初始化对象（设置属性）。日常开发中 99% 的情况只需要重写 `__init__`，只有在实现单例、继承不可变类型等特殊场景才需要 `__new__`。

## 实例属性 vs 类属性

```python
class Dog:
    # 类属性 — 所有实例共享
    count = 0

    def __init__(self, name):
        self.name = name  # 实例属性 — 每个实例独立
        Dog.count += 1

d1 = Dog("Buddy")
d2 = Dog("Max")

print(Dog.count)  # 2
print(d1.count)   # 2

# 警惕：通过实例修改类属性会创建实例属性（不会影响类属性）
d1.count = 10     # 创建了 d1.count 实例属性
print(Dog.count)  # 2 — 类属性不变
print(d1.count)   # 10 — 实例属性
print(d2.count)   # 2 — 仍然读取类属性
```

## 继承与 MRO

### 基本继承

```python
class Animal:
    def __init__(self, name: str):
        self.name = name

    def speak(self) -> str:
        return "..."

    def __str__(self):
        return f"Animal({self.name})"


class Dog(Animal):
    def __init__(self, name: str, breed: str):
        super().__init__(name)  # 调用父类 __init__
        self.breed = breed

    def speak(self) -> str:
        return "Woof!"

    def __str__(self):
        return f"Dog({self.name}, {self.breed})"


class Cat(Animal):
    def speak(self) -> str:
        return "Meow!"


# 多态
animals = [Dog("Buddy", "Golden"), Cat("Whiskers")]
for animal in animals:
    print(f"{animal} says {animal.speak()}")
# Dog(Buddy, Golden) says Woof!
# Animal(Whiskers) says Meow!
```

### MRO（方法解析顺序）

Python 使用 **C3 线性化算法**来确定多重继承中的方法查找顺序：

```python
class A:
    def greet(self):
        return "A"

class B(A):
    def greet(self):
        return "B"

class C(A):
    def greet(self):
        return "C"

class D(B, C):
    pass

d = D()
print(d.greet())       # B — 先查找 B，再查找 C，最后查找 A
print(D.__mro__)
# (<class 'D'>, <class 'B'>, <class 'C'>, <class 'A'>, <class 'object'>)
```

## 多重继承与 Mixin

多重继承在 Java 中不存在（通过接口实现），在 TypeScript 中也不直接支持。Python 的多重继承配合 Mixin 模式是一种非常实用的代码复用方式。

```python
# Mixin — 一种通过组合来复用代码的模式
class JsonMixin:
    """提供 JSON 序列化能力的 Mixin"""
    def to_json(self) -> str:
        import json
        # 获取实例的 __dict__（所有实例属性）
        return json.dumps(self.__dict__, ensure_ascii=False, indent=2)


class LogMixin:
    """提供日志能力的 Mixin"""
    def log(self, message: str) -> None:
        print(f"[{self.__class__.__name__}] {message}")


class ValidationMixin:
    """提供数据验证能力的 Mixin"""
    def validate(self) -> list[str]:
        errors = []
        if not hasattr(self, "name") or not self.name:
            errors.append("name is required")
        if hasattr(self, "age") and self.age < 0:
            errors.append("age must be non-negative")
        return errors


# 通过多重继承组合功能
class BaseModel(JsonMixin, LogMixin, ValidationMixin):
    def __init__(self, **kwargs):
        for key, value in kwargs.items():
            setattr(self, key, value)


class User(BaseModel):
    def __init__(self, name: str, age: int, **kwargs):
        super().__init__(name=name, age=age, **kwargs)


# 使用
user = User(name="Alice", age=30)
user.log("User created")                  # [User] User created
print(user.to_json())                     # JSON 格式输出
print(user.validate())                    # [] — 无错误

invalid_user = User(name="", age=-1)
print(invalid_user.validate())            # ['name is required', 'age must be non-negative']
```

> **Mixin 的设计原则**：Mixin 应该是"无状态"的——不定义 `__init__`，只提供方法。它们像插件一样为类注入能力，而不是定义类的核心行为。

## 魔术方法（Magic Methods）

魔术方法是 Python 中以双下划线开头和结尾的特殊方法，它们让你自定义类的行为，使类可以与 Python 内置操作（运算符、比较、迭代等）无缝集成。

```python
class Vector:
    """二维向量"""

    def __init__(self, x: float, y: float):
        self.x = x
        self.y = y

    # 字符串表示
    def __repr__(self) -> str:
        """开发者调试用 — repr(vector)"""
        return f"Vector({self.x}, {self.y})"

    def __str__(self) -> str:
        """用户友好 — str(vector)"""
        return f"({self.x}, {self.y})"

    # 运算符重载
    def __add__(self, other: "Vector") -> "Vector":
        return Vector(self.x + other.x, self.y + other.y)

    def __mul__(self, scalar: float) -> "Vector":
        return Vector(self.x * scalar, self.y * scalar)

    def __eq__(self, other: object) -> bool:
        if not isinstance(other, Vector):
            return NotImplemented
        return self.x == other.x and self.y == other.y

    def __hash__(self) -> int:
        return hash((self.x, self.y))

    def __abs__(self) -> float:
        return (self.x ** 2 + self.y ** 2) ** 0.5

    def __len__(self) -> int:
        return 2

    # 比较运算
    def __lt__(self, other: "Vector") -> bool:
        return abs(self) < abs(other)

    def __bool__(self) -> bool:
        return self.x != 0 or self.y != 0


# 使用
v1 = Vector(3, 4)
v2 = Vector(1, 2)

print(v1)                  # (3, 4) — 调用 __str__
print(repr(v1))            # Vector(3, 4) — 调用 __repr__
print(v1 + v2)             # (4, 6) — 调用 __add__
print(v1 * 2)              # (6, 8) — 调用 __mul__
print(abs(v1))             # 5.0 — 调用 __abs__
print(v1 == Vector(3, 4))  # True — 调用 __eq__
print(bool(v1))            # True — 调用 __bool__
print(bool(Vector(0, 0)))  # False
```

常用的魔术方法速查：

| 类别     | 方法                                             | 触发条件                        |
| -------- | ------------------------------------------------ | ------------------------------- |
| 字符串   | `__str__`, `__repr__`                            | `str()`, `repr()`               |
| 比较     | `__eq__`, `__lt__`, `__le__`, `__gt__`, `__ge__` | `==`, `<`, `<=`, `>`, `>=`      |
| 算术     | `__add__`, `__sub__`, `__mul__`, `__truediv__`   | `+`, `-`, `*`, `/`              |
| 类型转换 | `__bool__`, `__int__`, `__float__`               | `bool()`, `int()`, `float()`    |
| 容器     | `__len__`, `__getitem__`, `__contains__`         | `len()`, `obj[key]`, `x in obj` |
| 可调用   | `__call__`                                       | `obj()`                         |

## @property 装饰器

`@property` 让你可以用属性访问的语法来调用方法，类似于 Java 的 getter/setter 或 TypeScript 的 getter/setter：

```python
class Circle:
    def __init__(self, radius: float):
        self._radius = radius  # 用下划线表示"内部属性"

    @property
    def radius(self) -> float:
        """Getter — 读取时自动调用"""
        return self._radius

    @radius.setter
    def radius(self, value: float):
        """Setter — 赋值时自动调用"""
        if value < 0:
            raise ValueError("半径不能为负数")
        self._radius = value

    @property
    def area(self) -> float:
        """只读计算属性"""
        return 3.14159 * self._radius ** 2

    @property
    def diameter(self) -> float:
        """只读计算属性"""
        return self._radius * 2


c = Circle(5)
print(c.radius)    # 5 — 调用 getter
print(c.area)      # 78.54 — 调用计算属性
print(c.diameter)  # 10

c.radius = 10      # 调用 setter
print(c.area)      # 314.159

try:
    c.radius = -5  # 触发 ValueError
except ValueError as e:
    print(f"错误: {e}")  # 错误: 半径不能为负数

# c.area = 100    # AttributeError — 只读属性不能赋值
```

## dataclass 数据类

`dataclass` 是 Python 3.7+ 引入的语法糖，自动生成 `__init__`、`__repr__`、`__eq__` 等方法，非常适合定义数据模型。

```python
from dataclasses import dataclass, field, asdict, astuple
from typing import Optional


@dataclass
class Address:
    city: str
    street: str
    zip_code: str


@dataclass
class User:
    """用户数据模型 — dataclass 自动生成 __init__、__repr__、__eq__"""
    name: str
    age: int
    email: str = ""                                    # 默认值
    address: Optional[Address] = None                  # 可选字段
    tags: list[str] = field(default_factory=list)       # 可变默认值必须用 factory
    active: bool = True

    def __post_init__(self):
        """__init__ 之后自动调用，用于数据验证"""
        if self.age < 0:
            raise ValueError("年龄不能为负数")
        if self.email and "@" not in self.email:
            raise ValueError("邮箱格式不正确")


# 创建实例
user = User(name="Alice", age=30, email="alice@example.com")
print(user)
# User(name='Alice', age=30, email='alice@example.com', address=None, tags=[], active=True)

# 自动生成的 __eq__
user2 = User(name="Alice", age=30, email="alice@example.com")
print(user == user2)  # True

# 转换
print(asdict(user))   # 转为字典
print(astuple(user))  # 转为元组

# frozen=True — 不可变数据类
@dataclass(frozen=True)
class Point:
    x: float
    y: float

p = Point(3, 4)
# p.x = 5  # FrozenInstanceError — 不能修改

# frozen=True 的 dataclass 可以作为字典的 key
points = {Point(0, 0): "origin", Point(1, 1): "diagonal"}

# 排序
@dataclass(order=True)
class Score:
    value: int
    name: str = field(compare=False)  # 不参与比较

scores = [Score(85, "Alice"), Score(92, "Bob"), Score(78, "Charlie")]
scores.sort()
print(scores)  # [Score(value=78, ...), Score(value=85, ...), Score(value=92, ...)]
```

### dataclass vs 其他方案

| 特性            | dataclass          | 普通类       | NamedTuple   | Pydantic BaseModel |
| --------------- | ------------------ | ------------ | ------------ | ------------------ |
| 自动 `__init__` | 是                 | 否           | 是           | 是                 |
| 可变性          | 可配置             | 默认可变     | 不可变       | 默认可变           |
| 类型验证        | 无                 | 无           | 无           | 有                 |
| 默认值          | 支持               | 手动         | 支持         | 支持               |
| 继承            | 支持               | 支持         | 支持         | 支持               |
| 序列化          | `asdict`/`astuple` | 手动         | `_asdict`    | `.model_dump()`    |
| 使用场景        | 内部数据模型       | 需要复杂逻辑 | 简单数据载体 | API 数据验证       |

## Enum 枚举

```python
from enum import Enum, auto, IntEnum


class Status(Enum):
    """字符串枚举"""
    PENDING = "pending"
    ACTIVE = "active"
    INACTIVE = "inactive"
    DELETED = "deleted"


class Priority(IntEnum):
    """整数枚举 — 支持比较"""
    LOW = auto()       # 自动赋值 1
    MEDIUM = auto()    # 2
    HIGH = auto()      # 3
    CRITICAL = auto()  # 4


# 使用
status = Status.ACTIVE
print(status.value)        # "active"
print(status.name)         # "ACTIVE"
print(Status["PENDING"])   # Status.PENDING — 通过名称访问
print(Status("active"))    # Status.ACTIVE — 通过值访问

# 比较
print(Priority.HIGH > Priority.LOW)  # True — IntEnum 支持比较
print(Status.ACTIVE == Status.ACTIVE)  # True

# 遍历
for s in Status:
    print(f"{s.name}: {s.value}")
```

## 抽象基类（ABC）

```python
from abc import ABC, abstractmethod


class Repository(ABC):
    """数据仓库抽象基类"""

    @abstractmethod
    def get(self, id: int) -> dict:
        """根据 ID 获取数据"""
        ...

    @abstractmethod
    def save(self, data: dict) -> int:
        """保存数据，返回 ID"""
        ...

    @abstractmethod
    def delete(self, id: int) -> bool:
        """删除数据"""
        ...

    def get_or_create(self, id: int, default: dict) -> dict:
        """非抽象方法 — 子类可直接使用"""
        data = self.get(id)
        if data is None:
            new_id = self.save(default)
            return self.get(new_id)
        return data


class MemoryRepository(Repository):
    """内存实现"""

    def __init__(self):
        self._store: dict[int, dict] = {}
        self._next_id = 1

    def get(self, id: int) -> dict | None:
        return self._store.get(id)

    def save(self, data: dict) -> int:
        id = self._next_id
        self._next_id += 1
        self._store[id] = data
        return id

    def delete(self, id: int) -> bool:
        return self._store.pop(id, None) is not None


# repo = Repository()  # TypeError — 不能实例化抽象类
repo = MemoryRepository()
id = repo.save({"name": "Alice"})
print(repo.get(id))  # {'name': 'Alice'}
```

## `__slots__` 优化内存

默认情况下，Python 实例使用 `__dict__` 存储属性，这在创建大量实例时会消耗较多内存。`__slots__` 可以显著减少内存使用：

```python
import sys

class RegularPoint:
    def __init__(self, x, y):
        self.x = x
        self.y = y


class SlottedPoint:
    __slots__ = ("x", "y")  # 声明允许的属性

    def __init__(self, x, y):
        self.x = x
        self.y = y


# 内存对比
regular = RegularPoint(1, 2)
slotted = SlottedPoint(1, 2)

print(f"Regular: {sys.getsizeof(regular.__dict__)} bytes")  # 约 112 bytes
print(f"Slotted: {sys.getsizeof(slotted)} bytes")            # 约 48 bytes

# __slots__ 的限制
# slotted.z = 3  # AttributeError — 不能添加未声明的属性
```

> **何时使用 `__slots__`**：当需要创建大量实例（成千上万）时，如 ORM 模型、游戏实体等。一般情况下不需要使用。

## 最佳实践

1. **用 dataclass 替代手写 `__init__`**：减少样板代码，提高可读性
2. **用 `@property` 封装属性访问**：在需要验证或计算时，用 property 代替直接暴露属性
3. **优先组合而非继承**：Python 支持多重继承，但 Mixin + 组合通常比深层继承更灵活
4. **用 ABC 定义接口契约**：当需要强制子类实现特定方法时，使用抽象基类
5. **用 `__slots__` 优化大量对象的内存**：仅在性能敏感场景下使用

## 常见问题

**Q: Python 的私有属性真的私有吗？**

A: 不是。单下划线 `_x` 只是**约定**，告诉其他开发者"这是内部属性，请勿直接访问"。双下划线 `__x` 会触发**名称改写**（Name Mangling），变为 `_ClassName__x`，但仍然可以通过改写后的名称访问。Python 的理念是"我们都是成年人"，信任开发者的判断。

**Q: 什么时候用 dataclass，什么时候用 Pydantic BaseModel？**

A: 如果只是内部数据传递，用 dataclass（零依赖，轻量）。如果需要数据验证（如 API 输入）、序列化/反序列化（如 JSON），用 Pydantic BaseModel。在 Web API 场景下，Pydantic 是更好的选择。

**Q: Python 有接口吗？**

A: Python 没有专门的 `interface` 关键字（Java 风格）。可以用 ABC + `@abstractmethod` 来模拟接口，或者使用 `typing.Protocol`（Python 3.8+）进行结构化子类型（类似 Go 的接口——鸭子类型）。
