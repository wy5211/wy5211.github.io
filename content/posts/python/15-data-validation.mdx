## 前言

在前面的文章中，我们已经多次使用 Pydantic 定义请求和响应模型。但 Pydantic v2 的能力远不止于此 -- 它是一个强大的数据验证和序列化库，可以处理各种复杂场景。

如果你有以下框架经验：

- **NestJS 开发者**：Pydantic 类似于 `class-validator` + `class-transformer` 的组合，但更简洁高效
- **Java 开发者**：Pydantic 类似于 Jakarta Validation（`@NotNull`、`@Size` 等），但使用 Python 的声明式风格
- **TypeScript 开发者**：Pydantic 类似于 `zod`，提供运行时类型验证

**本文将带你：**

1. 了解 Pydantic v1 与 v2 的关键差异
2. 掌握 Field 验证器的各种用法
3. 实现自定义验证器（field_validator、model_validator）
4. 使用嵌套模型和泛型模型
5. 设计通用的分页和响应模型
6. 实现 SQLAlchemy 模型与 Pydantic 模型的转换

## 核心概念

### Pydantic v1 vs v2 关键差异

| 特性             | v1                              | v2                                                |
| ---------------- | ------------------------------- | ------------------------------------------------- |
| **验证器装饰器** | `@validator`                    | `@field_validator` / `@model_validator`           |
| **ORM 模式**     | `class Config: orm_mode = True` | `model_config = ConfigDict(from_attributes=True)` |
| **序列化**       | `.dict()` / `.json()`           | `.model_dump()` / `.model_dump_json()`            |
| **解析**         | `.parse_obj()` / `.parse_raw()` | `.model_validate()` / `.model_validate_json()`    |
| **排除未设置**   | `exclude_unset=True`            | `model_dump(exclude_unset=True)`                  |
| **核心引擎**     | 纯 Python                       | Rust 内核（快 5-50 倍）                           |
| **别名**         | `alias` 参数                    | `Field(alias=...)`                                |

> **重要**：如果你看到使用 `.dict()` 或 `@validator` 的代码，那可能是 v1 风格。新项目应使用 v2 API。

## 实战演练

### 示例 1：Field 验证器

`Field` 是 Pydantic 最常用的验证工具，提供了丰富的约束参数：

```python
# app/schemas/field_demo.py
from datetime import date
from pydantic import BaseModel, Field, EmailStr, HttpUrl, constr, conint


class UserProfile(BaseModel):
    """用户资料 -- 展示各种 Field 验证"""
    # 字符串验证
    username: str = Field(
        ...,
        min_length=3,
        max_length=50,
        pattern=r"^[a-zA-Z0-9_]+$",
        description="用户名，只允许字母、数字和下划线",
        examples=["alice_2024"],
    )

    # 邮箱验证（自动验证格式）
    email: EmailStr = Field(
        ...,
        description="邮箱地址",
    )

    # 年龄验证（数值范围）
    age: int = Field(
        ...,
        ge=0,    # >= 0
        le=150,  # <= 150
        description="年龄",
    )

    # 手机号（自定义正则）
    phone: str = Field(
        ...,
        pattern=r"^1[3-9]\d{9}$",
        description="中国大陆手机号",
    )

    # 网站 URL
    website: HttpUrl | None = Field(
        None,
        description="个人网站",
    )

    # 枚举约束
    gender: str = Field(
        ...,
        pattern=r"^(male|female|other)$",
        description="性别",
    )

    # 生日（日期不能是未来）
    birthday: date = Field(
        ...,
        description="出生日期",
        le=date.today(),  # 不能超过今天（注意：此处是静态值，创建时确定）
    )

    # 个人简介
    bio: str = Field(
        default="",
        max_length=500,
        description="个人简介，最多 500 字",
    )

    # 别名（API 字段名与 Python 属性名不同）
    avatarUrl: str | None = Field(
        None,
        alias="avatar_url",  # JSON 中使用 snake_case
        description="头像 URL",
    )

    model_config = {"from_attributes": True, "populate_by_name": True}
```

**Field 常用参数速查**：

| 参数                | 类型      | 说明                 | 示例                                   |
| ------------------- | --------- | -------------------- | -------------------------------------- |
| `default`           | Any       | 默认值               | `default=""`                           |
| `default_factory`   | Callable  | 默认值工厂           | `default_factory=list`                 |
| `alias`             | str       | 字段别名             | `alias="avatar_url"`                   |
| `title`             | str       | 字段标题（文档用）   | `title="Username"`                     |
| `description`       | str       | 字段描述（文档用）   | `description="用户名"`                 |
| `examples`          | list      | 示例值（文档用）     | `examples=["alice"]`                   |
| `ge`                | int/float | 大于等于             | `ge=0`                                 |
| `gt`                | int/float | 大于                 | `gt=0`                                 |
| `le`                | int/float | 小于等于             | `le=100`                               |
| `lt`                | int/float | 小于                 | `lt=100`                               |
| `min_length`        | int       | 最小长度             | `min_length=1`                         |
| `max_length`        | int       | 最大长度             | `max_length=200`                       |
| `pattern`           | str       | 正则表达式           | `pattern=r"^\d+$"`                     |
| `json_schema_extra` | dict      | JSON Schema 额外配置 | `json_schema_extra={"example": "xxx"}` |

### 示例 2：field_validator 自定义验证

当 `Field` 内置验证不够用时，使用 `@field_validator` 实现自定义验证逻辑：

```python
# app/schemas/registration.py
from pydantic import BaseModel, Field, field_validator, model_validator, model_config, ConfigDict
import re


class RegistrationForm(BaseModel):
    """用户注册表单 -- 展示自定义验证器"""
    username: str = Field(..., min_length=3, max_length=50)
    email: str = Field(..., min_length=5, max_length=200)
    password: str = Field(..., min_length=8, max_length=100)
    confirm_password: str = Field(..., min_length=8, max_length=100)
    invite_code: str | None = Field(None, max_length=20)

    # --- field_validator：验证单个字段 ---

    @field_validator("username")
    @classmethod
    def validate_username(cls, v: str) -> str:
        """用户名验证：只允许字母、数字和下划线"""
        if not re.match(r"^[a-zA-Z0-9_]+$", v):
            raise ValueError("用户名只能包含字母、数字和下划线")
        if v.startswith("_") or v.endswith("_"):
            raise ValueError("用户名不能以下划线开头或结尾")
        return v.lower()  # 统一转为小写

    @field_validator("email")
    @classmethod
    def validate_email(cls, v: str) -> str:
        """邮箱验证：简单的格式检查"""
        if "@" not in v or "." not in v.split("@")[-1]:
            raise ValueError("邮箱格式不正确")
        return v.lower()

    @field_validator("password")
    @classmethod
    def validate_password_strength(cls, v: str) -> str:
        """
        密码强度验证：
        - 至少 8 个字符
        - 包含大写字母
        - 包含小写字母
        - 包含数字
        - 包含特殊字符
        """
        if len(v) < 8:
            raise ValueError("密码至少 8 个字符")
        if not re.search(r"[A-Z]", v):
            raise ValueError("密码必须包含大写字母")
        if not re.search(r"[a-z]", v):
            raise ValueError("密码必须包含小写字母")
        if not re.search(r"\d", v):
            raise ValueError("密码必须包含数字")
        if not re.search(r'[!@#$%^&*(),.?":{}|<>]', v):
            raise ValueError("密码必须包含特殊字符")
        return v

    @field_validator("invite_code")
    @classmethod
    def validate_invite_code(cls, v: str | None) -> str | None:
        """邀请码验证：如果提供则必须是 6 位数字"""
        if v is not None and not re.match(r"^\d{6}$", v):
            raise ValueError("邀请码必须是 6 位数字")
        return v

    # --- model_validator：验证多个字段的关联 ---

    @model_validator(mode="after")
    def validate_passwords_match(self) -> "RegistrationForm":
        """验证两次输入的密码是否一致"""
        if self.password != self.confirm_password:
            raise ValueError("两次输入的密码不一致")
        # 验证通过后清除 confirm_password（不需要存入数据库）
        self.confirm_password = ""
        return self
```

**使用效果**：


### 示例 4：泛型模型（Generic Models）

泛型模型是构建可复用组件的利器：

```python
# app/schemas/common.py
from typing import Generic, TypeVar, List, Optional
from pydantic import BaseModel, Field


T = TypeVar("T")


class ResponseBase(BaseModel, Generic[T]):
    """统一 API 响应格式"""
    code: int = Field(default=200, description="状态码")
    message: str = Field(default="success", description="消息")
    data: Optional[T] = Field(default=None, description="数据")


class PaginatedResponse(BaseModel, Generic[T]):
    """通用分页响应"""
    items: List[T] = Field(description="数据列表")
    total: int = Field(description="总数")
    page: int = Field(ge=1, description="当前页码")
    page_size: int = Field(ge=1, le=100, description="每页数量")
    total_pages: int = Field(ge=0, description="总页数")


class ErrorDetail(BaseModel):
    """错误详情"""
    field: str = Field(..., description="字段名")
    message: str = Field(..., description="错误信息")
    code: str = Field(default="invalid_value", description="错误码")


class ErrorResponse(BaseModel):
    """统一错误响应"""
    code: int = Field(..., description="HTTP 状态码")
    message: str = Field(..., description="错误消息")
    details: List[ErrorDetail] = Field(default_factory=list, description="详细错误")
```

**在 FastAPI 路由中使用泛型模型**：

```python
from fastapi import APIRouter
from app.schemas.common import ResponseBase, PaginatedResponse
from app.schemas.user import UserResponse

router = APIRouter(prefix="/users")


@router.get("/", response_model=ResponseBase[PaginatedResponse[UserResponse]])
async def list_users():
    """获取用户列表"""
    return ResponseBase(data=PaginatedResponse(
        items=[],
        total=0,
        page=1,
        page_size=10,
        total_pages=0,
    ))


@router.get("/{user_id}", response_model=ResponseBase[UserResponse])
async def get_user(user_id: int):
    """获取用户详情"""
    return ResponseBase(data=UserResponse(
        id=1, username="alice", email="alice@example.com",
        is_active=True, created_at=datetime.now(),
    ))
```

### 示例 5：分页参数通用模型

```python
# app/schemas/pagination.py
from pydantic import BaseModel, Field, computed_field


class PaginationParams(BaseModel):
    """通用分页查询参数"""
    page: int = Field(default=1, ge=1, description="页码，从 1 开始")
    page_size: int = Field(default=10, ge=1, le=100, description="每页数量")

    @computed_field  # 自动计算的属性
    @property
    def offset(self) -> int:
        """计算 SQL OFFSET"""
        return (self.page - 1) * self.page_size


class SortParams(BaseModel):
    """通用排序参数"""
    sort_by: str = Field(default="created_at", description="排序字段")
    sort_order: str = Field(default="desc", pattern=r"^(asc|desc)$", description="排序方向")


class SearchParams(BaseModel):
    """通用搜索参数"""
    keyword: str | None = Field(None, max_length=100, description="搜索关键词")
    start_date: str | None = Field(None, description="开始日期 (YYYY-MM-DD)")
    end_date: str | None = Field(None, description="结束日期 (YYYY-MM-DD)")
```

在 FastAPI 中将查询参数绑定到 Pydantic 模型：

```python
from fastapi import Query, Depends
from pydantic import BaseModel, Field


def get_pagination_params(
    page: int = Query(1, ge=1),
    page_size: int = Query(10, ge=1, le=100),
) -> PaginationParams:
    """从查询参数创建分页对象"""
    return PaginationParams(page=page, page_size=page_size)


@router.get("/posts")
async def list_posts(pagination: PaginationParams = Depends(get_pagination_params)):
    """使用分页参数"""
    offset = pagination.offset  # 自动计算 offset
    # 查询数据库: SELECT * FROM posts OFFSET {offset} LIMIT {pagination.page_size}
    return {"page": pagination.page, "page_size": pagination.page_size, "offset": offset}
```

### 示例 6：SQLAlchemy 模型与 Pydantic 模型转换

```python
# app/schemas/conversion.py
from datetime import datetime
from typing import Any
from pydantic import BaseModel, ConfigDict, model_validator


class UserCreate(BaseModel):
    """创建用户 -- API 输入"""
    username: str = Field(..., min_length=3, max_length=50)
    email: str = Field(..., min_length=5, max_length=200)
    password: str = Field(..., min_length=8)


class UserUpdate(BaseModel):
    """更新用户 -- API 输入（部分更新）"""
    username: str | None = None
    email: str | None = None


class UserResponse(BaseModel):
    """用户响应 -- API 输出"""
    id: int
    username: str
    email: str
    is_active: bool
    role: str
    created_at: datetime

    model_config = ConfigDict(from_attributes=True)
    # from_attributes=True 允许从 ORM 对象创建 Pydantic 模型


class PostCreate(BaseModel):
    """创建文章"""
    title: str = Field(..., min_length=1, max_length=200)
    content: str = Field(..., min_length=1)
    is_published: bool = False
    tag_ids: list[int] = Field(default_factory=list)

    @model_validator(mode="after")
    def validate_tags_count(self) -> "PostCreate":
        """验证标签数量不超过 5 个"""
        if len(self.tag_ids) > 5:
            raise ValueError("一篇文章最多关联 5 个标签")
        return self


# --- 转换示例 ---
# 从 ORM 对象创建 Pydantic 模型
def user_to_response(user: "User") -> UserResponse:
    """将 SQLAlchemy User 对象转为 UserResponse"""
    return UserResponse.model_validate(user)

# 从 Pydantic 模型创建 ORM 对象
def create_to_user(data: UserCreate, hashed_password: str) -> "User":
    """将 UserCreate 转为 SQLAlchemy User 对象"""
    return User(
        username=data.username,
        email=data.email,
        hashed_password=hashed_password,
    )

# 批量转换
def users_to_response(users: list["User"]) -> list[UserResponse]:
    """批量转换"""
    return [UserResponse.model_validate(u) for u in users]
```

### 示例 8：自定义数据类型与 @dataclass 集成

```python
# app/schemas/custom_types.py
from pydantic import BaseModel, field_validator
from pydantic.dataclasses import dataclass
from datetime import datetime
from enum import Enum


# --- 枚举类型 ---
class PostStatus(str, Enum):
    """文章状态枚举"""
    DRAFT = "draft"
    PUBLISHED = "published"
    ARCHIVED = "archived"


class ArticleCreate(BaseModel):
    title: str
    status: PostStatus = PostStatus.DRAFT


# --- @dataclass 集成 ---
@dataclass
class Address:
    """地址 -- 使用 pydantic dataclass"""
    province: str
    city: str
    district: str
    street: str
    postal_code: str

    @field_validator("postal_code")
    @classmethod
    def validate_postal_code(cls, v: str) -> str:
        if not v.isdigit() or len(v) != 6:
            raise ValueError("邮政编码必须是 6 位数字")
        return v


class ShippingInfo(BaseModel):
    """收货信息 -- 包嵌套 dataclass"""
    recipient_name: str
    phone: str
    address: Address  # 嵌套 pydantic dataclass


# --- 自定义验证器返回复杂类型 ---
class SearchFilter(BaseModel):
    """搜索过滤"""
    min_price: float | None = None
    max_price: float | None = None
    category: str | None = None
    sort_by: str = "created_at"
    sort_order: str = "desc"

    @field_validator("max_price")
    @classmethod
    def validate_price_range(cls, v: float | None, info) -> float | None:
        """验证价格范围"""
        if v is not None and "min_price" in info.data:
            min_price = info.data["min_price"]
            if min_price is not None and v < min_price:
                raise ValueError("max_price 不能小于 min_price")
        return v

    def to_sql_where(self) -> str:
        """转换为 SQL WHERE 条件"""
        conditions = []
        if self.min_price is not None:
            conditions.append(f"price >= {self.min_price}")
        if self.max_price is not None:
            conditions.append(f"price <= {self.max_price}")
        if self.category is not None:
            conditions.append(f"category = '{self.category}'")
        return " AND ".join(conditions) if conditions else "1=1"
```

## 最佳实践

### 1. 分离 Create / Update / Response 模型

```python
# Create -- 创建时需要密码
class UserCreate(BaseModel):
    username: str
    email: str
    password: str  # 创建时需要

# Update -- 更新时所有字段可选
class UserUpdate(BaseModel):
    username: str | None = None
    email: str | None = None
    # 不包含 password（密码更新应该有单独的接口）

# Response -- 不包含敏感信息
class UserResponse(BaseModel):
    id: int
    username: str
    email: str
    # 不包含 password / hashed_password
```

### 2. 使用 computed_field 计算派生字段

```python
from pydantic import BaseModel, computed_field


class OrderItem(BaseModel):
    price: float
    quantity: int

    @computed_field  # Pydantic v2 的计算字段
    @property
    def total(self) -> float:
        return round(self.price * self.quantity, 2)

    # total 会被包含在 model_dump() 和 model_dump_json() 的输出中
```

### 3. 统一错误处理

```python
# app/exceptions.py
from fastapi import Request
from fastapi.responses import JSONResponse
from fastapi.exceptions import RequestValidationError
from pydantic import ValidationError


async def validation_exception_handler(request: Request, exc: RequestValidationError):
    """自定义 Pydantic 验证错误格式"""
    errors = []
    for error in exc.errors():
        errors.append({
            "field": ".".join(str(loc) for loc in error["loc"]),
            "message": error["msg"],
            "type": error["type"],
        })
    return JSONResponse(
        status_code=422,
        content={
            "code": 422,
            "message": "Validation error",
            "details": errors,
        },
    )


# 在 main.py 中注册
from fastapi import FastAPI
from fastapi.exceptions import RequestValidationError

app = FastAPI()
app.add_exception_handler(RequestValidationError, validation_exception_handler)
```

## 常见问题

**Q: Pydantic v2 的 `field_validator` 和 v1 的 `@validator` 有什么区别？**

A: v2 的 `@field_validator` 需要添加 `@classmethod` 装饰器，返回类型注解也更严格。此外，v2 的 `mode` 参数控制验证时机：`mode="before"` 在类型转换前验证，`mode="after"` 在类型转换后验证（默认）。

**Q: `model_config` 中的 `extra="forbid"` 和 `extra="ignore"` 怎么选？**

A: API 输入（Create/Update）建议用 `extra="forbid"`，防止客户端传入未定义的字段。内部模型可以使用 `extra="ignore"` 或默认值。

**Q: 如何处理可选的嵌套模型？**

A: 使用 `| None` 标记为可选。验证时如果传入 `None` 或不传该字段，都会跳过嵌套验证。

```python
class PostSchema(BaseModel):
    author: AuthorSchema | None = None  # 可选嵌套
```

**Q: `from_attributes=True` 有什么用？**

A: 允许 Pydantic 模型从非字典对象（如 SQLAlchemy ORM 对象、dataclass 等）创建。旧版中对应 `orm_mode = True`。在 FastAPI 中与 SQLAlchemy 配合时必须开启。

## 总结

本文我们深入学习了 Pydantic v2 的高级功能：

1. **v1 vs v2 差异**：新的验证器 API、序列化方法、Rust 内核带来的性能提升
2. **Field 验证器**：字符串、数值、正则等丰富的内置约束
3. **自定义验证器**：`@field_validator` 验证单个字段，`@model_validator` 验证字段间关系
4. **嵌套模型**：构建复杂的数据结构，如文章详情包含作者和评论
5. **泛型模型**：`Generic[T]` 设计可复用的 `ResponseBase[T]` 和 `PaginatedResponse[T]`
6. **序列化**：`model_dump()`、`model_dump_json()`、`model_validate()` 全套转换工具
7. **SQLAlchemy 转换**：`from_attributes=True` 实现 ORM 对象与 Pydantic 模型的无缝转换

Pydantic 是 FastAPI 生态的基石，掌握它的高级用法能让你构建更健壮、更优雅的 API。
