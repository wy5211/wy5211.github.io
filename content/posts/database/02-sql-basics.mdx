## 前言

SQL（Structured Query Language，结构化查询语言）是与关系型数据库交互的标准语言。无论你使用 PostgreSQL、MySQL、SQLite 还是其他关系型数据库，SQL 都是必备技能。

本文将系统介绍 SQL 的核心语法，从创建表到复杂查询，帮助你快速掌握数据库操作的基本功。

---

## 核心概念

### SQL 是什么？

**SQL** 是一种**声明式语言**，你告诉数据库**要什么数据**，而不是**如何获取**。

**特点**：

- 🎯 **声明式**：描述结果，不关心过程
- 📊 **集合导向**：操作的是数据集合，而非单条记录
- 🔧 **功能全面**：查询、定义、控制一应俱全

---

### SQL 的五大分类

| 分类    | 全称                         | 功能         | 关键字                        |
| ------- | ---------------------------- | ------------ | ----------------------------- |
| **DDL** | Data Definition Language     | 数据定义语言 | CREATE、ALTER、DROP、TRUNCATE |
| **DML** | Data Manipulation Language   | 数据操作语言 | INSERT、UPDATE、DELETE、MERGE |
| **DQL** | Data Query Language          | 数据查询语言 | SELECT                        |
| **DCL** | Data Control Language        | 数据控制语言 | GRANT、REVOKE、DENY           |
| **TCL** | Transaction Control Language | 事务控制语言 | COMMIT、ROLLBACK、SAVEPOINT   |

### 一、DDL - 数据定义语言

DDL 用于定义数据库结构（表、视图、索引等）。

#### 1. CREATE - 创建表

```sql
-- 用户表
CREATE TABLE users (
  id SERIAL PRIMARY KEY,                    -- 自增主键
  email VARCHAR(255) UNIQUE NOT NULL,       -- 唯一邮箱
  password_hash VARCHAR(255) NOT NULL,      -- 密码哈希
  full_name VARCHAR(100),                   -- 姓名
  phone VARCHAR(20),                        -- 电话
  is_verified BOOLEAN DEFAULT FALSE,        -- 是否验证
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 商品分类表
CREATE TABLE categories (
  id SERIAL PRIMARY KEY,
  name VARCHAR(100) NOT NULL,
  slug VARCHAR(100) UNIQUE NOT NULL,        -- URL 友好标识
  description TEXT,
  parent_id INTEGER REFERENCES categories(id), -- 自关联外键
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 商品表
CREATE TABLE products (
  id SERIAL PRIMARY KEY,
  name VARCHAR(255) NOT NULL,
  slug VARCHAR(255) UNIQUE NOT NULL,
  description TEXT,
  price NUMERIC(10, 2) NOT NULL,            -- 精确的金额类型
  stock INTEGER DEFAULT 0,
  category_id INTEGER REFERENCES categories(id) ON DELETE SET NULL,
  is_active BOOLEAN DEFAULT TRUE,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 订单表
CREATE TABLE orders (
  id SERIAL PRIMARY KEY,
  user_id INTEGER NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  order_number VARCHAR(50) UNIQUE NOT NULL,
  status VARCHAR(20) DEFAULT 'pending',      -- pending, paid, shipped, delivered, cancelled
  subtotal NUMERIC(10, 2) NOT NULL,
  tax NUMERIC(10, 2) DEFAULT 0,
  discount NUMERIC(10, 2) DEFAULT 0,
  total NUMERIC(10, 2) NOT NULL,
  notes TEXT,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 订单项表（多对多关系）
CREATE TABLE order_items (
  id SERIAL PRIMARY KEY,
  order_id INTEGER NOT NULL REFERENCES orders(id) ON DELETE CASCADE,
  product_id INTEGER NOT NULL REFERENCES products(id) ON DELETE RESTRICT,
  quantity INTEGER NOT NULL CHECK (quantity > 0),
  unit_price NUMERIC(10, 2) NOT NULL,
  subtotal NUMERIC(10, 2) NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

**代码解释**：

- `SERIAL`：自增整数（PostgreSQL 特有）
- `PRIMARY KEY`：主键，唯一标识每一行
- `UNIQUE`：唯一约束，不能重复
- `NOT NULL`：非空约束
- `DEFAULT`：默认值
- `REFERENCES`：外键约束
- `ON DELETE CASCADE`：删除父记录时，子记录也删除
- `ON DELETE SET NULL`：删除父记录时，子记录的外键设为 NULL
- `ON DELETE RESTRICT`：有子记录时，禁止删除父记录
- `CHECK`：检查约束（如 quantity > 0）

#### 2. ALTER - 修改表结构

```sql
-- 添加列
ALTER TABLE users ADD COLUMN avatar_url TEXT;
ALTER TABLE users ADD COLUMN last_login_at TIMESTAMP;

-- 修改列类型
ALTER TABLE products ALTER COLUMN price TYPE NUMERIC(12, 2);

-- 重命名列
ALTER TABLE users RENAME COLUMN full_name TO name;

-- 删除列
ALTER TABLE users DROP COLUMN phone;

-- 添加约束
ALTER TABLE users ADD CONSTRAINT check_email_format
  CHECK (email ~* '^[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Za-z]{2,}$');

-- 添加索引
CREATE INDEX idx_users_email ON users(email);
CREATE INDEX idx_products_category ON products(category_id);
CREATE INDEX idx_orders_user_status ON orders(user_id, status);
```

#### 3. DROP - 删除表

```sql
-- 删除表（会删除表结构和数据）
DROP TABLE IF EXISTS order_items;

-- 删除表及其依赖的对象（索引、约束等）
DROP TABLE IF EXISTS orders CASCADE;

-- 只删除数据，保留表结构
TRUNCATE TABLE products RESTART IDENTITY CASCADE;  -- RESTART IDENTITY 重置自增序列
```

### 二、DML - 数据操作语言

DML 用于操作表中的数据。

#### 1. INSERT - 插入数据

```sql
-- 插入单行（指定列）
INSERT INTO users (email, password_hash, name)
VALUES ('zhangsan@example.com', 'hashed_password_123', '张三');

-- 插入多行（批量）
INSERT INTO users (email, password_hash, name) VALUES
  ('lisi@example.com', 'hashed_password_456', '李四'),
  ('wangwu@example.com', 'hashed_password_789', '王五');

-- 插入所有列（省略列名，不推荐）
INSERT INTO users VALUES
  (DEFAULT, 'user6@example.com', 'hash', '赵六', NULL, FALSE, CURRENT_TIMESTAMP, CURRENT_TIMESTAMP);

-- 从查询结果插入
INSERT INTO archived_users (email, name)
SELECT email, name FROM users WHERE created_at < '2024-01-01';

-- 插入并返回数据
INSERT INTO products (name, slug, price, stock)
VALUES ('iPhone 15', 'iphone-15', 7999.00, 100)
RETURNING *;
```

#### 2. UPDATE - 更新数据

```sql
-- 更新单行
UPDATE users
SET name = '张三丰', is_verified = TRUE
WHERE id = 1;

-- 更新多列
UPDATE users
SET
  last_login_at = CURRENT_TIMESTAMP,
  is_verified = TRUE
WHERE email = 'zhangsan@example.com';

-- 基于条件更新
UPDATE products
SET stock = stock - 1
WHERE id = 1 AND stock > 0;  -- 防止库存变负数

-- 批量更新
UPDATE orders
SET status = 'cancelled'
WHERE created_at < '2024-01-01' AND status = 'pending';

-- 从其他表更新
UPDATE order_items oi
SET unit_price = p.price
FROM products p
WHERE oi.product_id = p.id;

-- 更新并返回数据
UPDATE users
SET login_count = login_count + 1
WHERE id = 1
RETURNING id, email, login_count;
```

#### 3. DELETE - 删除数据

```sql
-- 删除单行
DELETE FROM users WHERE id = 1;

-- 基于条件删除
DELETE FROM orders
WHERE status = 'cancelled' AND created_at < '2024-01-01';

-- 删除并返回数据
DELETE FROM products
WHERE stock = 0 AND is_active = FALSE
RETURNING *;

-- 使用子查询删除
DELETE FROM order_items
WHERE order_id IN (SELECT id FROM orders WHERE status = 'cancelled');
```

**⚠️ 重要**：DELETE 会删除整行数据，且需要提交事务。谨慎使用 WHERE 条件！

### 三、DQL - 数据查询语言

DQL 是 SQL 中最常用、最强大的部分。

#### 1. SELECT 基础查询

```sql
-- 查询所有列
SELECT * FROM users;

-- 查询指定列
SELECT id, email, name FROM users;

-- 使用别名（AS）
SELECT
  id AS user_id,
  email AS user_email,
  name AS user_name
FROM users;

-- 去重（DISTINCT）
SELECT DISTINCT status FROM orders;
SELECT COUNT(DISTINCT user_id) FROM orders;

-- 限制结果（LIMIT、OFFSET）
SELECT * FROM orders
ORDER BY created_at DESC
LIMIT 10 OFFSET 20;  -- 跳过前 20 条，取 10 条（第 21-30 条）
```

#### 2. WHERE 条件过滤

```sql
-- 比较运算符：=, !=, <>, <, >, <=, >=
SELECT * FROM products WHERE price > 1000;

-- 逻辑运算符：AND, OR, NOT
SELECT * FROM users
WHERE is_verified = TRUE AND created_at >= '2024-01-01';

-- 范围查询（BETWEEN）
SELECT * FROM orders
WHERE created_at BETWEEN '2024-01-01' AND '2024-12-31';

-- 集合查询（IN、NOT IN）
SELECT * FROM orders
WHERE status IN ('paid', 'shipped', 'delivered');

-- 空值判断（IS NULL、IS NOT NULL）
SELECT * FROM users WHERE phone IS NULL;

-- 模糊匹配（LIKE、ILIKE）
SELECT * FROM products
WHERE name LIKE '%iPhone%';        -- % 匹配任意字符
SELECT * FROM users
WHERE email ILIKE 'zhang%@example.com';  -- ILIKE 不区分大小写

-- 正则表达式匹配
SELECT * FROM users
WHERE email ~* '^[A-Za-z0-9._%+-]+@example\.com$';
```

#### 3. ORDER BY 排序

```sql
-- 升序排序（默认）
SELECT * FROM products ORDER BY price ASC;

-- 降序排序
SELECT * FROM products ORDER BY price DESC;

-- 多列排序
SELECT * FROM orders
ORDER BY status ASC, created_at DESC;

-- 按表达式排序
SELECT
  name,
  price,
  stock,
  price * stock AS total_value
FROM products
ORDER BY total_value DESC;

-- 使用 NULLS FIRST/LAST
SELECT * FROM users
ORDER BY last_login_at DESC NULLS LAST;  -- 未登录的排在最后
```

#### 4. 聚合函数

```sql
-- COUNT：计数
SELECT COUNT(*) FROM users;
SELECT COUNT(email) FROM users;  -- 忽略 NULL
SELECT COUNT(DISTINCT category_id) FROM products;

-- SUM：求和
SELECT SUM(total) FROM orders WHERE status = 'paid';

-- AVG：平均值
SELECT AVG(price) FROM products;

-- MIN、MAX：最小值、最大值
SELECT MIN(price), MAX(price) FROM products;

-- 多个聚合
SELECT
  COUNT(*) as total_orders,
  SUM(total) as total_revenue,
  AVG(total) as avg_order_value,
  MIN(total) as min_order,
  MAX(total) as max_order
FROM orders
WHERE status = 'paid';
```

#### 5. GROUP BY 分组

```sql
-- 按状态分组统计
SELECT
  status,
  COUNT(*) as order_count,
  SUM(total) as total_revenue
FROM orders
GROUP BY status
ORDER BY total_revenue DESC;

-- 按多个字段分组
SELECT
  DATE(created_at) as order_date,
  status,
  COUNT(*) as order_count
FROM orders
GROUP BY DATE(created_at), status
ORDER BY order_date DESC, status;

-- 使用 HAVING 过滤分组
SELECT
  user_id,
  COUNT(*) as order_count,
  SUM(total) as total_spent
FROM orders
WHERE status = 'paid'
GROUP BY user_id
HAVING SUM(total) > 10000  -- 分组后的条件
ORDER BY total_spent DESC;
```

**WHERE vs HAVING**：

- `WHERE`：分组前过滤（不能使用聚合函数）
- `HAVING`：分组后过滤（可以使用聚合函数）

#### 6. JOIN - 连接查询

```sql
-- INNER JOIN：内连接（只匹配两边都有的）
SELECT
  o.id,
  o.order_number,
  o.total,
  u.email,
  u.name
FROM orders o
INNER JOIN users u ON o.user_id = u.id;

-- LEFT JOIN：左连接（保留左表所有行）
SELECT
  u.id,
  u.email,
  COUNT(o.id) as order_count
FROM users u
LEFT JOIN orders o ON u.id = o.user_id
GROUP BY u.id;

-- RIGHT JOIN：右连接（保留右表所有行）
SELECT *
FROM orders o
RIGHT JOIN users u ON o.user_id = u.id;

-- FULL JOIN：全连接（保留两边所有行）
SELECT *
FROM orders o
FULL JOIN users u ON o.user_id = u.id;

-- 多表 JOIN
SELECT
  o.order_number,
  p.name as product_name,
  oi.quantity,
  oi.unit_price
FROM orders o
INNER JOIN order_items oi ON o.id = oi.order_id
INNER JOIN products p ON oi.product_id = p.id
WHERE o.status = 'paid';
```

#### 7. UNION - 合并查询

```sql
-- UNION：去重合并
SELECT email, name FROM users
UNION
SELECT email, name FROM admins;

-- UNION ALL：不去重合并（更快）
SELECT email, name FROM users
UNION ALL
SELECT email, name FROM admins;
```

#### 8. 子查询

```sql
-- 标量子查询（返回单行单列）
SELECT * FROM products
WHERE price > (SELECT AVG(price) FROM products);

-- 列表子查询（返回单列多行）
SELECT * FROM products
WHERE category_id IN (SELECT id FROM categories WHERE name LIKE '电子%');

-- 行子查询（返回多列）
SELECT * FROM users
WHERE (email, name) IN (
  SELECT email, name FROM archived_users
);

-- EXISTS 子查询（判断是否存在）
SELECT * FROM users u
WHERE EXISTS (
  SELECT 1 FROM orders o
  WHERE o.user_id = u.id AND o.total > 1000
);

-- 派生表（FROM 子查询）
SELECT
  user_stats.user_id,
  user_stats.order_count,
  u.email
FROM (
  SELECT
    user_id,
    COUNT(*) as order_count
  FROM orders
  GROUP BY user_id
) user_stats
JOIN users u ON user_stats.user_id = u.id;
```

### 四、SQL 语句执行顺序

理解 SQL 的执行顺序对编写复杂查询至关重要。

```sql
-- 语法顺序（我们写的顺序）
SELECT DISTINCT column1, aggregate_func(column2)
FROM table1
JOIN table2 ON condition
WHERE condition
GROUP BY column1
HAVING condition
ORDER BY column1
LIMIT 10;

-- 实际执行顺序（数据库执行顺序）
1. FROM      -- 确定数据来源
2. JOIN      -- 连接表
3. WHERE     -- 过滤行
4. GROUP BY  -- 分组
5. HAVING    -- 过滤分组
6. SELECT    -- 选择列
7. DISTINCT  -- 去重
8. ORDER BY  -- 排序
9. LIMIT     -- 限制结果
```

**实例分析**：

```sql
-- 这条查询如何执行？
SELECT
  u.name,
  COUNT(o.id) as order_count
FROM users u
LEFT JOIN orders o ON u.id = o.user_id
WHERE u.is_verified = TRUE
GROUP BY u.id, u.name
HAVING COUNT(o.id) > 0
ORDER BY order_count DESC
LIMIT 10;

-- 执行顺序：
-- 1. FROM users：获取 users 表
-- 2. LEFT JOIN orders：连接 orders 表
-- 3. WHERE u.is_verified = TRUE：过滤未验证用户
-- 4. GROUP BY u.id, u.name：按用户分组
-- 5. HAVING COUNT(o.id) > 0：过滤无订单用户
-- 6. SELECT u.name, COUNT(o.id)：选择显示列
-- 7. ORDER BY order_count DESC：按订单数降序
-- 8. LIMIT 10：只取前 10 名
```

## 命名规范与最佳实践

### 命名规范

| 类型   | 规范                       | 示例                       |
| ------ | -------------------------- | -------------------------- |
| 表名   | 小写+下划线，复数形式      | `users`, `order_items`     |
| 列名   | 小写+下划线                | `user_id`, `created_at`    |
| 主键   | 表名 + `_id` 或 `id`       | `id`, `user_id`            |
| 外键   | 关联表名 + `_id`           | `user_id`, `product_id`    |
| 索引   | `idx_` + 表名 + `_` + 列名 | `idx_users_email`          |
| 布尔列 | `is_` 前缀                 | `is_active`, `is_verified` |
| 时间列 | `_at` 后缀                 | `created_at`, `updated_at` |

### SQL 编写建议

```sql
-- ✅ 好的写法
SELECT
  u.id,
  u.email,
  u.name,
  COUNT(o.id) as order_count
FROM users u
INNER JOIN orders o ON u.id = o.user_id
WHERE u.is_verified = TRUE
  AND o.created_at >= '2024-01-01'
GROUP BY u.id, u.email, u.name
ORDER BY order_count DESC
LIMIT 10;

-- ❌ 不好的写法
SELECT * FROM users,orders WHERE users.id=orders.user_id;
```

**建议**：

1. ✅ 使用大写写 SQL 关键字
2. ✅ 每行一个列名，便于阅读
3. ✅ 使用表别名简化代码
4. ✅ 明确列出列名，避免使用 `SELECT *`
5. ✅ 添加注释解释复杂逻辑
6. ✅ 使用格式化工具（如 SQLFormat）

## 实战案例：电商系统常用查询

### 1. 用户订单统计

```sql
-- 查询用户的订单统计
SELECT
  u.id,
  u.email,
  u.name,
  COUNT(o.id) as total_orders,
  COALESCE(SUM(o.total), 0) as total_spent,
  COALESCE(AVG(o.total), 0) as avg_order_value
FROM users u
LEFT JOIN orders o ON u.id = o.user_id
WHERE u.is_verified = TRUE
GROUP BY u.id, u.email, u.name
ORDER BY total_spent DESC
LIMIT 20;
```

### 2. 热销商品排行榜

```sql
-- 查询热销商品 TOP 10
SELECT
  p.id,
  p.name,
  p.price,
  SUM(oi.quantity) as total_sold,
  SUM(oi.subtotal) as total_revenue,
  COUNT(DISTINCT oi.order_id) as order_count
FROM products p
INNER JOIN order_items oi ON p.id = oi.product_id
INNER JOIN orders o ON oi.order_id = o.id
WHERE o.status IN ('paid', 'shipped', 'delivered')
  AND o.created_at >= CURRENT_DATE - INTERVAL '30 days'
GROUP BY p.id, p.name, p.price
ORDER BY total_sold DESC
LIMIT 10;
```

### 3. 每日订单趋势

```sql
-- 查询最近 30 天的每日订单统计
SELECT
  DATE(created_at) as order_date,
  COUNT(*) as total_orders,
  COUNT(DISTINCT user_id) as unique_customers,
  SUM(total) as total_revenue,
  AVG(total) as avg_order_value
FROM orders
WHERE created_at >= CURRENT_DATE - INTERVAL '30 days'
  AND status = 'paid'
GROUP BY DATE(created_at)
ORDER BY order_date DESC;
```

### 4. 库存预警

```sql
-- 查询库存不足的商品
SELECT
  p.id,
  p.name,
  p.stock,
  c.name as category_name,
  (
    SELECT SUM(quantity)
    FROM order_items oi
    JOIN orders o ON oi.order_id = o.id
    WHERE oi.product_id = p.id
      AND o.status IN ('pending', 'paid')
  ) as pending_orders
FROM products p
LEFT JOIN categories c ON p.category_id = c.id
WHERE p.stock < 10
  AND p.is_active = TRUE
ORDER BY p.stock ASC;
```

## 常见问题

### Q1: SELECT \* 会影响性能吗？

**是的**。`SELECT *` 会：

- 查询不必要的列，增加网络传输
- 可能导致索引失效
- 增加内存占用

**建议**：明确列出需要的列。

### Q2: UNION 和 UNION ALL 有什么区别？

- `UNION`：去重，需要排序，较慢
- `UNION ALL`：不去重，不需要排序，较快

如果确定没有重复数据，优先使用 `UNION ALL`。

### Q3: WHERE 和 HAVING 的区别？

- `WHERE`：在分组前过滤，不能使用聚合函数
- `HAVING`：在分组后过滤，可以使用聚合函数

```sql
-- ✅ 正确
SELECT status, COUNT(*)
FROM orders
WHERE created_at >= '2024-01-01'
GROUP BY status
HAVING COUNT(*) > 10;

-- ❌ 错误（WHERE 不能用聚合函数）
SELECT status, COUNT(*)
FROM orders
WHERE COUNT(*) > 10
GROUP BY status;
```

### Q4: 如何优化慢查询？

1. 使用 `EXPLAIN ANALYZE` 分析执行计划
2. 添加适当的索引
3. 避免 `SELECT *`
4. 使用 `LIMIT` 限制结果
5. 优化 JOIN 顺序（小表驱动大表）
