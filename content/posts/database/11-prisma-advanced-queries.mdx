## 核心概念

### 关系查询

Prisma 提供了强大的关系查询能力。

**include vs select**：

- `include`：包含关联数据
- `select`：精确控制返回字段

## 实战演练

### 1. 关系查询（避免 N+1）

```typescript
// ❌ N+1 问题
const users = await prisma.user.findMany();
for (const user of users) {
  const orders = await prisma.order.findMany({
    where: { userId: user.id },
  });
}

// ✅ 使用 include
const usersWithOrders = await prisma.user.findMany({
  include: {
    orders: {
      select: {
        id: true,
        orderNumber: true,
        total: true,
        createdAt: true,
      },
      where: { status: "PAID" },
      orderBy: { createdAt: "desc" },
      take: 5,
    },
  },
});
```

### 2. 复杂过滤

### 3. 全文搜索

```typescript
// PostgreSQL 全文搜索
const users = await prisma.user.findMany({
  where: {
    OR: [
      { email: { contains: query, mode: "insensitive" } },
      { fullName: { contains: query, mode: "insensitive" } },
    ],
  },
});

// 使用原生 SQL（更强大的全文搜索）
const users = await prisma.$queryRaw`
  SELECT * FROM users
  WHERE to_tsvector('chinese', email || ' ' || full_name) @@ to_tsquery('chinese', ${query})
`;
```

### 5. Cursor 分页

```typescript
// 第一页
const firstPage = await prisma.order.findMany({
  orderBy: { createdAt: "desc" },
  take: 20,
});

// 下一页
const lastItem = firstPage[firstPage.length - 1];
const nextPage = await prisma.order.findMany({
  cursor: { id: lastItem.id },
  orderBy: { createdAt: "desc" },
  take: 20,
  skip: 1,
});

// API 实现
app.get("/orders", async (req, res) => {
  const { cursor, limit = 20 } = req.query;

  const orders = await prisma.order.findMany({
    take: Number(limit),
    ...(cursor && {
      skip: 1,
      cursor: { id: String(cursor) },
    }),
    orderBy: { createdAt: "desc" },
  });

  res.json({
    orders,
    nextCursor: orders[orders.length - 1]?.id,
  });
});
```

### 6. 原生 SQL

```typescript
// $queryRaw（查询）
const result = await prisma.$queryRaw`
  SELECT
    u.email,
    COUNT(o.id) as order_count,
    COALESCE(SUM(o.total), 0) as total_spent
  FROM users u
  LEFT JOIN orders o ON u.id = o.user_id
  WHERE u.is_verified = true
  GROUP BY u.id, u.email
  ORDER BY total_spent DESC
  LIMIT 10
`;

// $executeRaw（执行）
await prisma.$executeRaw`UPDATE users SET last_login_at = NOW() WHERE id = ${userId}`;
```

## 性能优化

### 1. select 只查询需要的字段

```typescript
// ❌ 查询所有字段
const users = await prisma.user.findMany();

// ✅ 只查询需要的字段
const users = await prisma.user.findMany({
  select: {
    id: true,
    email: true,
    fullName: true,
  },
});
```

---

### 2. 避免 N+1 查询

```typescript
// ❌ N+1 问题
const users = await prisma.user.findMany();
for (const user of users) {
  const orders = await prisma.order.findMany({
    where: { userId: user.id },
  });
}

// ✅ 使用 include
const users = await prisma.user.findMany({
  include: { orders: true },
});
```

---

### 3. 使用 take 限制结果

```typescript
const users = await prisma.user.findMany({
  take: 20,
  orderBy: { createdAt: "desc" },
});
```

## 常见问题

### Q1: 如何处理循环依赖？

使用 `select` 而不是 `include`，精确控制返回字段。

---

### Q2: 如何优化大数据量查询？

使用 cursor 分页，而不是 offset 分页。
