## PostgreSQL 监控

### 1. 查看当前连接数

```sql
SELECT count(*) FROM pg_stat_activity;

SELECT
  datname,
  count(*) as connections
FROM pg_stat_activity
GROUP BY datname;
```

---

### 2. 查看慢查询

```sql
-- 启用 pg_stat_statements
CREATE EXTENSION pg_stat_statements;

-- 查看最慢的查询
SELECT
  query,
  calls,
  total_exec_time / 1000 as total_seconds,
  mean_exec_time / 1000 as avg_seconds
FROM pg_stat_statements
ORDER BY mean_exec_time DESC
LIMIT 10;
```

---

### 3. 查看表大小

```sql
SELECT
  relname AS table_name,
  pg_size_pretty(pg_total_relation_size(relid)) AS total_size
FROM pg_catalog.pg_statio_user_tables
ORDER BY pg_total_relation_size(relid) DESC;
```

---

### 4. 查看索引使用情况

```sql
SELECT
  schemaname,
  tablename,
  indexname,
  idx_scan,
  idx_tup_read,
  idx_tup_fetch
FROM pg_stat_user_indexes
ORDER BY idx_scan ASC;
```

---

### 5. 缓存命中率

```sql
SELECT
  sum(heap_blks_read) as heap_read,
  sum(heap_blks_hit) as heap_hit,
  sum(heap_blks_hit) / (sum(heap_blks_hit) + sum(heap_blks_read)) AS cache_hit_ratio
FROM pg_statio_user_tables;
```

## 性能调优

### 1. 配置优化

```bash
# postgresql.conf

# 内存设置
shared_buffers = 256MB              # 内存的 25%
effective_cache_size = 1GB          # 内存的 50-75%
work_mem = 16MB                     # 每个查询操作的内存
maintenance_work_mem = 128MB        # 维护操作内存

# WAL 设置
wal_buffers = 16MB
checkpoint_completion_target = 0.9

# 查询优化
random_page_cost = 1.1              # SSD 优化
effective_io_concurrency = 200      # SSD 优化

# 连接设置
max_connections = 100
```

---

### 2. 查询优化

```sql
-- 使用 EXPLAIN ANALYZE
EXPLAIN ANALYZE SELECT * FROM orders WHERE user_id = 'user-1';

-- 创建索引
CREATE INDEX idx_orders_user_id ON orders(user_id);

-- 更新统计信息
ANALYZE orders;
```

---

### 3. 连接池优化

```yaml
# PgBouncer 配置
pool_mode = transaction
max_client_conn = 200
default_pool_size = 25
```

## 分库分表

### 1. 水平分片

```yaml
# 按用户 ID 分片
shards:
  - name: shard_0
    database: shard_0
    id_range: [0, 999999]

  - name: shard_1
    database: shard_1
    id_range: [1000000, 1999999]

  - name: shard_2
    database: shard_2
    id_range: [2000000, 2999999]
```

---

### 2. 应用层路由

```typescript
function getShard(userId: string) {
  const id = parseInt(userId);
  const shardIndex = Math.floor(id / 1000000);
  return shards[shardIndex];
}
```

## 读写分离

### 1. 主从复制

```bash
# postgresql.conf (主库)
wal_level = replica
max_wal_senders = 5
wal_keep_size = 1GB

# pg_hba.conf (主库)
host replication replica 192.168.1.0/24 md5
```

---

### 2. 应用层实现

```typescript
// 写操作：主库
await masterDB.query("INSERT INTO users ...");

// 读操作：从库
await slaveDB.query("SELECT * FROM users ...");
```

## 高可用架构

### 1. 主从复制

```
Master (主库)
  ↓ 复制
Slave1 (从库1)
Slave2 (从库2)
```

---

### 2. 哨兵模式（Redis）

```
Master (主库)
  ↓ 复制
Slave (从库)
  ↑ 监控
Sentinel (哨兵)
```

---

### 3. 集群模式

```
Master1 (主库1)  Master2 (主库2)  Master3 (主库3)
    ↓               ↓               ↓
Slave1 (从库1)    Slave2 (从库2)  Slave3 (从库3)
```
