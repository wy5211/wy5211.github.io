## 前言

当数据结构灵活多变、需要快速迭代时，关系型数据库可能不够灵活。MongoDB 作为最流行的文档数据库，提供了 schema-less 的设计，让开发更加灵活。

本文将系统讲解 MongoDB 的核心概念和基本操作，帮你掌握文档数据库的使用。

---

## 核心概念

### 为什么选择 MongoDB？

**MongoDB 优势**：

- ✅ 灵活的文档模型（JSON）
- ✅ Schema-less（无需预定义结构）
- ✅ 高查询性能
- ✅ 水平扩展能力
- ✅ 丰富的查询语言

**vs PostgreSQL**：

- PostgreSQL：结构化数据、事务、复杂查询
- MongoDB：灵活 schema、快速迭代、嵌入式文档

---

## MongoDB 核心概念

**1. Database（数据库）**

- 数据的逻辑容器

**2. Collection（集合）**

- 类似关系型数据库的表
- 但不需要预定义结构

**3. Document（文档）**

- JSON 格式的数据记录
- 类似关系型数据库的行

**4. Field（字段）**

- 文档中的键值对
- 类似关系型数据库的列

**5. \_id（主键）**

- 每个文档的唯一标识
- 默认自动生成 ObjectId

## CRUD 操作

### 1. Create（插入）

```javascript
// 插入单个文档
await db.collection("users").insertOne({
  email: "user@example.com",
  fullName: "张三",
  createdAt: new Date(),
});

// 插入多个文档
await db.collection("users").insertMany([
  { email: "user1@example.com", fullName: "李四" },
  { email: "user2@example.com", fullName: "王五" },
]);
```

---

### 2. Read（查询）

```javascript
// 查询单个文档
const user = await db.collection("users").findOne({
  email: "user@example.com",
});

// 查询多个文档
const users = await db
  .collection("users")
  .find({
    fullName: "张三",
  })
  .toArray();

// 条件查询
const users = await db
  .collection("users")
  .find({
    age: { $gte: 18 },
  })
  .toArray();
```

---

### 3. Update（更新）

```javascript
// 更新单个文档
await db
  .collection("users")
  .updateOne({ email: "user@example.com" }, { $set: { fullName: "李四" } });

// 更新多个文档
await db
  .collection("users")
  .updateMany({ age: { $lt: 18 } }, { $set: { isMinor: true } });

// 增加数值
await db
  .collection("products")
  .updateOne({ _id: productId }, { $inc: { stock: -1 } });
```

---

### 4. Delete（删除）

```javascript
// 删除单个文档
await db.collection("users").deleteOne({
  email: "user@example.com",
});

// 删除多个文档
await db.collection("users").deleteMany({
  createdAt: { $lt: new Date("2024-01-01") },
});
```

## 数据建模

### 嵌入式文档（推荐）

```javascript
// 订单（嵌入订单项）
const order = await db.collection("orders").insertOne({
  orderNumber: "ORD-001",
  userId: "user-123",
  items: [
    {
      productId: "prod-1",
      name: "商品A",
      quantity: 2,
      price: 100,
    },
    {
      productId: "prod-2",
      name: "商品B",
      quantity: 1,
      price: 50,
    },
  ],
  total: 250,
  status: "paid",
  createdAt: new Date(),
});
```

**优点**：

- ✅ 查询快（一次查询获取所有数据）
- ✅ 适合 1:N 关系
- ✅ 原子更新

---

### 引用式文档

```javascript
// 用户（引用订单）
const user = await db.collection("users").insertOne({
  email: "user@example.com",
  fullName: "张三",
  orderIds: ["order-1", "order-2"],
});

// 查询时需要 $lookup
const userWithOrders = await db
  .collection("users")
  .aggregate([
    { $match: { email: "user@example.com" } },
    {
      $lookup: {
        from: "orders",
        localField: "orderIds",
        foreignField: "_id",
        as: "orders",
      },
    },
  ])
  .toArray();
```

**适用场景**：

- ✅ M:N 关系
- ✅ 子文档数量不确定
- ✅ 需要独立管理子文档

---

## 索引

```javascript
// 单字段索引
await db.collection("users").createIndex({ email: 1 }, { unique: true });

// 复合索引
await db.collection("orders").createIndex({ userId: 1, createdAt: -1 });

// 文本索引
await db
  .collection("products")
  .createIndex({ name: "text", description: "text" });

// 地理空间索引
await db.collection("locations").createIndex({ location: "2dsphere" });

// 查看索引
await db.collection("users").indexes();

// 删除索引
await db.collection("users").dropIndex("email_1");
```

## 常见问题

### Q1: MongoDB vs PostgreSQL？

- 结构化数据 → PostgreSQL
- 灵活 schema → MongoDB
- 事务要求高 → PostgreSQL
- 快速迭代 → MongoDB
