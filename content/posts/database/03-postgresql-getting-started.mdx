## 核心概念

### 为什么选择 PostgreSQL？

**PostgreSQL** 是世界上最先进的开源关系型数据库系统，具有以下核心优势：

**1. 完整的 ACID 支持**

- 原子性（Atomicity）：事务要么全部成功，要么全部失败
- 一致性（Consistency）：事务前后数据保持一致
- 隔离性（Isolation）：并发事务互不干扰
- 持久性（Durability）：提交后数据永久保存

**2. 丰富的数据类型**

```sql
-- PostgreSQL 独有或独特的类型
JSONB           -- 二进制 JSON（查询性能更好）
ARRAY           -- 数组类型
UUID            -- 全球唯一标识符
CIDR            -- IP 地址范围
HSTORE          -- 键值对存储
```

**3. 强大的扩展性**

- **PostGIS**：地理空间数据扩展（GIS 应用首选）
- **pgcrypto**：加密函数库
- **full text search**：全文搜索
- **foreign data wrapper**：访问外部数据源

**4. MVCC 并发控制**

- 多版本并发控制，读写不阻塞
- 适合高并发场景

**5. 高度标准兼容**

- 完全支持 SQL:2016 标准
- 跨平台支持（Linux、Windows、macOS）

**PostgreSQL vs MySQL**：

| 特性     | PostgreSQL | MySQL      |
| -------- | ---------- | ---------- |
| 复杂查询 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐   |
| 数据类型 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐     |
| 并发性能 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐   |
| 扩展性   | ⭐⭐⭐⭐⭐ | ⭐⭐⭐     |
| 易用性   | ⭐⭐⭐⭐   | ⭐⭐⭐⭐⭐ |
| 学习曲线 | 较陡       | 较平       |

**选择建议**：

- ✅ 选择 PostgreSQL：复杂业务、数据分析、地理信息、高并发
- ✅ 选择 MySQL：简单 Web 应用、快速开发、广为人知

### PostgreSQL 核心概念

**1. 数据库集群（Cluster）**

- 一个 PostgreSQL 服务器实例管理多个数据库
- 物理上共享同一数据目录

**2. 数据库（Database）**

- 逻辑上的数据隔离
- 不同数据库之间表名可以重复

**3. Schema（模式）**

- 数据库内部的命名空间
- 默认 schema：`public`
- 同一数据库内不同 schema 的表名可以重复

**4. 表（Table）**

- 数据的实际存储单元
- 属于某个 schema

**层次关系**：

```
Cluster (服务器实例)
  └─ Database (数据库)
      └─ Schema (模式)
          └─ Table (表)
```

## 实战演练

### 一、环境搭建

我们使用 Docker Compose 搭建开发环境，这是最简单、最推荐的方式。

#### 1. 创建项目目录

```bash
mkdir postgres-learning
cd postgres-learning
```

#### 2. 创建 Docker Compose 配置

```yaml
# docker-compose.yml
version: "3.8"

services:
  postgres:
    image: postgres:16-alpine
    container_name: postgres_dev
    environment:
      POSTGRES_USER: dev # 超级用户名
      POSTGRES_PASSWORD: dev123 # 密码
      POSTGRES_DB: dev_db # 默认数据库
    ports:
      - "5432:5432" # 主机端口:容器端口
    volumes:
      # 数据持久化
      - postgres_data:/var/lib/postgresql/data
      # 初始化脚本
      - ./init.sql:/docker-entrypoint-initdb.d/init.sql
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U dev"]
      interval: 10s
      timeout: 5s
      retries: 5

  pgadmin:
    image: dpage/pgadmin4:latest
    container_name: pgadmin_dev
    environment:
      PGADMIN_DEFAULT_EMAIL: admin@example.com
      PGADMIN_DEFAULT_PASSWORD: admin
    ports:
      - "5050:80"
    depends_on:
      - postgres

volumes:
  postgres_data:
```

#### 3. 创建初始化脚本

```sql
-- init.sql
-- 这个脚本会在容器首次启动时自动执行

-- 创建示例数据库
CREATE DATABASE ecommerce;

-- 连接到 ecommerce 数据库
\c ecommerce;

-- 创建用户表
CREATE TABLE users (
  id SERIAL PRIMARY KEY,
  email VARCHAR(255) UNIQUE NOT NULL,
  password_hash VARCHAR(255) NOT NULL,
  full_name VARCHAR(100),
  is_verified BOOLEAN DEFAULT FALSE,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 插入示例数据
INSERT INTO users (email, password_hash, full_name) VALUES
  ('user1@example.com', 'hash1', '张三'),
  ('user2@example.com', 'hash2', '李四'),
  ('user3@example.com', 'hash3', '王五');

-- 授权
GRANT ALL PRIVILEGES ON DATABASE ecommerce TO dev;
```

#### 4. 启动服务

```bash
# 启动服务
docker-compose up -d

# 查看日志
docker-compose logs -f postgres

# 查看运行状态
docker-compose ps
```

#### 5. 连接数据库

**方式 1：使用 psql 命令行工具**

```bash
# 进入容器
docker exec -it postgres_dev psql -U dev -d dev_db

# 或者直接从主机连接（需要安装 psql 客户端）
psql -h localhost -p 5432 -U dev -d dev_db
```

**方式 2：使用 pgAdmin（可视化工具）**

1. 打开浏览器访问 http://localhost:5050
2. 使用 `admin@example.com` / `admin` 登录
3. 添加服务器连接：
   - Host：`postgres`
   - Port：`5432`
   - Username：`dev`
   - Password：`dev123`

### 二、基本操作

#### 1. psql 常用命令

```bash
# 连接数据库
psql -h localhost -U dev -d dev_db

# \? - 查看所有 psql 命令
\?

# \l - 列出所有数据库
\l

# \c database_name - 切换数据库
\c ecommerce

# \dt - 列出当前数据库的所有表
\dt

# \d table_name - 查看表结构
\d users

# \du - 列出所有用户
\du

# \conninfo - 显示当前连接信息
\conninfo

# \i file.sql - 执行 SQL 文件
\i /path/to/script.sql

# \o file.txt - 将查询结果输出到文件
\o output.txt
SELECT * FROM users;
\o

# \q - 退出
\q
```

#### 2. 数据库操作

```sql
-- 创建数据库
CREATE DATABASE myapp;

-- 切换数据库
\c myapp

-- 查看数据库大小
SELECT
  pg_database.datname,
  pg_size_pretty(pg_database_size(pg_database.datname)) AS size
FROM pg_database
ORDER BY pg_database_size(pg_database.datname) DESC;

-- 删除数据库（不能删除当前连接的数据库）
DROP DATABASE myapp;

-- 重命名数据库
ALTER DATABASE myapp RENAME TO newname;
```

#### 3. 表操作

```sql
-- 创建表
CREATE TABLE products (
  id SERIAL PRIMARY KEY,
  name VARCHAR(255) NOT NULL,
  price NUMERIC(10, 2) NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 查看表结构
\d products

-- 重命名表
ALTER TABLE products RENAME TO items;

-- 删除表
DROP TABLE products;

-- 删除表（如果存在）
DROP TABLE IF EXISTS products;
```

#### 4. 数据操作

```sql
-- 插入数据
INSERT INTO users (email, password_hash, full_name)
VALUES ('newuser@example.com', 'hash', '赵六');

-- 查询数据
SELECT * FROM users;

-- 查询特定列
SELECT id, email, full_name FROM users;

-- 条件查询
SELECT * FROM users WHERE is_verified = TRUE;

-- 更新数据
UPDATE users
SET is_verified = TRUE
WHERE id = 1;

-- 删除数据
DELETE FROM users WHERE id = 1;
```

### 三、用户权限管理

PostgreSQL 的权限系统非常灵活。

#### 1. 创建用户

```sql
-- 创建用户（CREATE USER 和 CREATE ROLE 相同，USER 默认可登录）
CREATE USER app_user WITH PASSWORD 'secure_password';

-- 创建只读用户
CREATE USER readonly WITH PASSWORD 'readonly_pass';

-- 创建超级用户
CREATE USER admin WITH PASSWORD 'admin_pass' SUPERUSER;
```

#### 2. 授权

```sql
-- 授予数据库连接权限
GRANT CONNECT ON DATABASE ecommerce TO app_user;

-- 授予 schema 使用权限
GRANT USAGE ON SCHEMA public TO app_user;

-- 授予表权限
GRANT SELECT, INSERT, UPDATE, DELETE ON ALL TABLES IN SCHEMA public TO app_user;

-- 授予只读权限
GRANT SELECT ON ALL TABLES IN SCHEMA public TO readonly;

-- 授予序列权限（SERIAL 自增需要）
GRANT USAGE, SELECT ON ALL SEQUENCES IN SCHEMA public TO app_user;

-- 默认权限（未来创建的表自动授权）
ALTER DEFAULT PRIVILEGES IN SCHEMA public
  GRANT SELECT, INSERT, UPDATE, DELETE ON TABLES TO app_user;
```

#### 3. 撤销权限

```sql
-- 撤销权限
REVOKE DELETE ON ALL TABLES IN SCHEMA public FROM app_user;

-- 撤销超级用户权限
ALTER USER admin WITH NOSUPERUSER;
```

#### 4. 角色管理

```sql
-- 创建角色
CREATE ROLE developer;

-- 将用户添加到角色
GRANT developer TO app_user;

-- 为角色授权
GRANT ALL PRIVILEGES ON DATABASE ecommerce TO developer;

-- 查看角色权限
\du
```

### 四、配置文件详解

PostgreSQL 的配置文件位于容器的 `/var/lib/postgresql/data` 目录。

#### 1. postgresql.conf（主配置文件）

```bash
# 进入容器查看配置
docker exec -it postgres_dev cat /var/lib/postgresql/data/postgresql.conf

# 常用配置项

# 连接设置
listen_addresses = '*'                  # 监听所有 IP
max_connections = 100                   # 最大连接数

# 内存设置（根据服务器内存调整）
shared_buffers = 256MB                  # 共享缓冲区（内存的 25%）
effective_cache_size = 1GB              # 有效缓存大小（内存的 50-75%）
work_mem = 16MB                         # 每个查询操作的内存
maintenance_work_mem = 128MB            # 维护操作内存

# WAL 设置
wal_buffers = 16MB                      # WAL 缓冲区
checkpoint_completion_target = 0.9      # 检查点完成目标

# 日志设置
logging_collector = on                  # 启用日志收集
log_directory = 'log'                   # 日志目录
log_filename = 'postgresql-%Y-%m-%d.log' # 日志文件名
log_statement = 'mod'                   # 记录 SQL（none, ddl, mod, all）
log_duration = on                       # 记录查询执行时间
```

#### 2. pg_hba.conf（认证配置文件）

```bash
# 查看认证配置
docker exec -it postgres_dev cat /var/lib/postgresql/data/pg_hba.conf

# 配置格式
# TYPE  DATABASE  USER  ADDRESS  METHOD

# 允许本地所有用户使用 trust 认证（开发环境）
local   all   all   trust

# 允许本地所有用户使用密码认证
local   all   all   md5

# 允许 IPv4 连接
host    all   all   0.0.0.0/0   md5

# 允许特定 IP 段
host    ecommerce  app_user  192.168.1.0/24  md5
```

#### 3. 修改配置（需要重启）

```bash
# 创建自定义配置文件
cat > custom_postgresql.conf <<EOF
shared_buffers = 512MB
effective_cache_size = 2GB
work_mem = 32MB
EOF

# 挂载到容器（修改 docker-compose.yml）
volumes:
  - ./custom_postgresql.conf:/etc/postgresql/postgresql.conf

# 重启服务
docker-compose restart postgres
```

### 五、连接池（PgBouncer）

在高并发场景下，直接创建大量数据库连接会消耗很多资源。这时需要使用连接池。

#### 1. 添加 PgBouncer 服务

```yaml
# docker-compose.yml
services:
  pgbouncer:
    image: edoburu/pgbouncer:latest
    container_name: pgbouncer_dev
    environment:
      DB_HOST: postgres
      DB_PORT: 5432
      DB_USER: dev
      DB_PASSWORD: dev123
      DATABASES: dev_db
      POOL_MODE: transaction # 连接池模式
      MAX_CLIENT_CONN: 200 # 最大客户端连接数
      DEFAULT_POOL_SIZE: 25 # 每个数据库的连接池大小
    ports:
      - "6432:6432"
    depends_on:
      - postgres
```

#### 2. 连接 PgBouncer

```bash
# 应用连接 PgBouncer 而不是直接连接 PostgreSQL
psql -h localhost -p 6432 -U dev -d dev_db
```

### 3. 监控与日志

**启用慢查询日志**：

```sql
-- 修改 postgresql.conf
log_min_duration_statement = 1000  -- 记录超过 1 秒的查询

-- 重启后生效
```

**查看当前活动连接**：

```sql
SELECT
  pid,
  usename,
  application_name,
  client_addr,
  state,
  query_start,
  state_change,
  query
FROM pg_stat_activity
WHERE state != 'idle';
```

## 常见问题

### Q1: 忘记密码怎么办？

```bash
# 1. 停止容器
docker-compose stop postgres

# 2. 以单用户模式启动
docker run -it --rm \
  -v postgres_data:/var/lib/postgresql/data \
  postgres:16-alpine bash

# 3. 在容器内修改密码
psql -U dev
ALTER USER dev WITH PASSWORD 'new_password';
\q

# 4. 正常启动容器
docker-compose start postgres
```

### Q2: 如何查看 PostgreSQL 版本？

```sql
-- 方式 1
SELECT version();

-- 方式 2（在命令行）
psql --version

SHOW server_version;
```

### Q3: 数据库连接失败怎么办？

**检查清单**：

```bash
# 1. 检查容器是否运行
docker-compose ps

# 2. 检查端口是否被占用
lsof -i :5432

# 3. 查看日志
docker-compose logs postgres

# 4. 测试连接
docker exec -it postgres_dev pg_isready -U dev

# 5. 检查防火墙
sudo ufw status
```

---

### Q4: 如何清理测试数据？

```sql
-- 方式 1：删除所有数据（保留表结构）
TRUNCATE TABLE users, orders, products RESTART IDENTITY CASCADE;

-- 方式 2：删除并重建表
DROP TABLE IF EXISTS users CASCADE;
CREATE TABLE users (...);

-- 方式 3：删除整个 schema
DROP SCHEMA public CASCADE;
CREATE SCHEMA public;
GRANT ALL ON SCHEMA public TO dev;
```

---

### Q5: 如何导出/导入数据？

```bash
# 导出数据库
pg_dump -h localhost -U dev -d dev_db > dev_db.sql

# 导出特定表
pg_dump -h localhost -U dev -d dev_db -t users > users.sql

# 导入数据
psql -h localhost -U dev -d dev_db < dev_db.sql

# 使用 Docker
docker exec -i postgres_dev pg_dump -U dev dev_db > backup.sql
docker exec -i postgres_dev psql -U dev dev_db < backup.sql
```
