# Terraform 基础设施即代码实战指南

在现代云原生时代，手动管理基础设施已经成为过去式。Terraform 作为基础设施即代码（Infrastructure as Code，IaC）领域的王者，让我们能够用代码定义、部署和管理云资源。本文将带你从零开始掌握 Terraform 的核心概念和实战技巧。

## 什么是基础设施即代码？

基础设施即代码（IaC）是一种通过代码来管理和配置计算基础设施的实践，而不是通过手动流程。

### 传统方式 vs IaC

| 对比项   | 传统手动方式 | IaC 方式       |
| -------- | ------------ | -------------- |
| 部署速度 | 慢，逐个操作 | 快，批量自动化 |
| 一致性   | 容易出错     | 高度一致       |
| 可重复性 | 难以复现     | 一键复制       |
| 版本控制 | 无           | Git 管理       |
| 回滚能力 | 困难         | 轻松回滚       |
| 团队协作 | 文档沟通     | 代码审查       |

### 为什么选择 Terraform？

Terraform 由 HashiCorp 开发，是目前最流行的 IaC 工具之一：

- **多云支持**：AWS、Azure、GCP、阿里云等 100+ 云服务商
- **声明式语法**：描述期望状态，而非执行步骤
- **执行计划**：变更前预览所有改动
- **状态管理**：追踪资源状态，增量更新
- **模块化**：复用和组合基础设施代码

## 快速开始

### 安装 Terraform

**macOS：**

```bash
brew install terraform
```

**Linux：**

```bash
# 下载并安装
wget https://releases.hashicorp.com/terraform/1.7.0/terraform_1.7.0_linux_amd64.zip
unzip terraform_1.7.0_linux_amd64.zip
sudo mv terraform /usr/local/bin/
```

**Windows：**

```powershell
choco install terraform
```

验证安装：

```bash
terraform version
# Terraform v1.7.0
```

### 第一个 Terraform 项目

创建项目目录：

```bash
mkdir terraform-demo
cd terraform-demo
```

创建 `main.tf` 文件：

```hcl
# 声明使用的 Provider
terraform {
  required_providers {
    docker = {
      source  = "kreuzwerker/docker"
      version = "~&gt; 3.0.0"
    }
  }
}

# 配置 Docker Provider
provider "docker" {}

# 创建一个 Docker 镜像
resource "docker_image" "nginx" {
  name         = "nginx:latest"
  keep_locally = false
}

# 创建一个 Docker 容器
resource "docker_container" "nginx" {
  image = docker_image.nginx.image_id
  name  = "tutorial-nginx"
  ports {
    internal = 80
    external = 8000
  }
}
```

### Terraform 核心命令

```bash
# 初始化项目（下载 Provider）
terraform init

# 格式化代码
terraform fmt

# 验证语法
terraform validate

# 生成执行计划
terraform plan

# 应用变更
terraform apply

# 销毁所有资源
terraform destroy
```

执行流程：

```bash
$ terraform init
Initializing the backend...
Initializing provider plugins...
- Finding latest version of kreuzwerker/docker...
- Installing kreuzwerker/docker v3.0.2...

$ terraform apply
docker_image.nginx: Creating...
docker_image.nginx: Creation complete after 30s
docker_container.nginx: Creating...
docker_container.nginx: Creation complete after 2s

Apply complete! Resources: 2 added, 0 changed, 0 destroyed.
```

## 核心概念详解

### Provider（提供者）

Provider 是 Terraform 与云服务 API 交互的插件：

```hcl
# AWS Provider
provider "aws" {
  region = "ap-northeast-1"
}

# 阿里云 Provider
provider "alicloud" {
  region = "cn-hangzhou"
}

# Kubernetes Provider
provider "kubernetes" {
  config_path = "~/.kube/config"
}
```

### Resource（资源）

Resource 是基础设施的基本单元，代表一个具体的云资源：

```hcl
# 创建 VPC
resource "aws_vpc" "main" {
  cidr_block = "10.0.0.0/16"

  tags = {
    Name = "main-vpc"
  }
}

# 创建子网
resource "aws_subnet" "public" {
  vpc_id     = aws_vpc.main.id
  cidr_block = "10.0.1.0/24"

  tags = {
    Name = "public-subnet"
  }
}
```

### Data Source（数据源）

Data Source 用于查询已存在的资源信息：

```hcl
# 查询最新的 Amazon Linux 2 AMI
data "aws_ami" "amazon_linux" {
  most_recent = true
  owners      = ["amazon"]

  filter {
    name   = "name"
    values = ["amzn2-ami-hvm-*-x86_64-gp2"]
  }
}

# 使用查询结果
resource "aws_instance" "web" {
  ami           = data.aws_ami.amazon_linux.id
  instance_type = "t3.micro"
}
```

### Variable（变量）

变量让配置更加灵活和可复用：

```hcl
# variables.tf
variable "region" {
  description = "AWS region"
  type        = string
  default     = "ap-northeast-1"
}

variable "instance_type" {
  description = "EC2 instance type"
  type        = string
  default     = "t3.micro"
}

variable "tags" {
  description = "Resource tags"
  type        = map(string)
  default = {
    Environment = "dev"
    Project     = "terraform-demo"
  }
}

# 使用变量
provider "aws" {
  region = var.region
}

resource "aws_instance" "web" {
  instance_type = var.instance_type
  tags          = var.tags
}
```

变量可以通过多种方式传入：

```bash
# 命令行参数
terraform apply -var="region=us-west-2"

# 环境变量
export TF_VAR_region=us-west-2

# 变量文件 terraform.tfvars
region = "us-west-2"
instance_type = "t3.small"
```

### Output（输出）

Output 用于输出重要信息：

```hcl
# outputs.tf
output "instance_public_ip" {
  description = "Public IP of the EC2 instance"
  value       = aws_instance.web.public_ip
}

output "instance_dns" {
  value = aws_instance.web.public_dns
}
```

### Local（局部值）

Local 用于定义重复使用的表达式：

```hcl
locals {
  common_tags = {
    Environment = var.environment
    Project     = var.project_name
    ManagedBy   = "terraform"
  }

  name_prefix = "${var.project_name}-${var.environment}"
}

resource "aws_instance" "web" {
  tags = local.common_tags
}
```

## 实战：部署完整 Web 应用

让我们创建一个完整的 AWS 基础设施，包括 VPC、EC2、RDS 和 ALB。

### 项目结构

```
terraform-webapp/
├── main.tf          # 主配置
├── variables.tf     # 变量定义
├── outputs.tf       # 输出定义
├── vpc.tf           # 网络配置
├── ec2.tf           # 计算资源
├── rds.tf           # 数据库
├── alb.tf           # 负载均衡
└── terraform.tfvars # 变量值
```

### VPC 网络配置

```hcl
# vpc.tf
resource "aws_vpc" "main" {
  cidr_block           = var.vpc_cidr
  enable_dns_hostnames = true
  enable_dns_support   = true

  tags = merge(local.common_tags, {
    Name = "${local.name_prefix}-vpc"
  })
}

# 公有子网
resource "aws_subnet" "public" {
  count                   = length(var.public_subnet_cidrs)
  vpc_id                  = aws_vpc.main.id
  cidr_block              = var.public_subnet_cidrs[count.index]
  availability_zone       = var.availability_zones[count.index]
  map_public_ip_on_launch = true

  tags = merge(local.common_tags, {
    Name = "${local.name_prefix}-public-${count.index + 1}"
  })
}

# 私有子网
resource "aws_subnet" "private" {
  count             = length(var.private_subnet_cidrs)
  vpc_id            = aws_vpc.main.id
  cidr_block        = var.private_subnet_cidrs[count.index]
  availability_zone = var.availability_zones[count.index]

  tags = merge(local.common_tags, {
    Name = "${local.name_prefix}-private-${count.index + 1}"
  })
}

# Internet Gateway
resource "aws_internet_gateway" "main" {
  vpc_id = aws_vpc.main.id

  tags = merge(local.common_tags, {
    Name = "${local.name_prefix}-igw"
  })
}

# NAT Gateway
resource "aws_eip" "nat" {
  count  = length(var.public_subnet_cidrs)
  domain = "vpc"
}

resource "aws_nat_gateway" "main" {
  count         = length(var.public_subnet_cidrs)
  allocation_id = aws_eip.nat[count.index].id
  subnet_id     = aws_subnet.public[count.index].id

  tags = merge(local.common_tags, {
    Name = "${local.name_prefix}-nat-${count.index + 1}"
  })

  depends_on = [aws_internet_gateway.main]
}
```

### EC2 实例配置

```hcl
# ec2.tf
# 创建 EC2 实例
resource "aws_instance" "web" {
  count         = var.instance_count
  ami           = data.aws_ami.amazon_linux.id
  instance_type = var.instance_type
  subnet_id     = aws_subnet.public[count.index % length(aws_subnet.public)].id

  vpc_security_group_ids = [aws_security_group.web.id]

  user_data = base64encode(templatefile("${path.module}/user_data.sh", {
    db_endpoint = aws_db_instance.main.endpoint
    db_name     = var.db_name
    db_user     = var.db_username
    db_password = var.db_password
  }))

  tags = merge(local.common_tags, {
    Name = "${local.name_prefix}-web-${count.index + 1}"
  })
}

# 安全组
resource "aws_security_group" "web" {
  name        = "${local.name_prefix}-web-sg"
  description = "Security group for web servers"
  vpc_id      = aws_vpc.main.id

  ingress {
    from_port   = 80
    to_port     = 80
    protocol    = "tcp"
    cidr_blocks = ["0.0.0.0/0"]
  }

  ingress {
    from_port   = 443
    to_port     = 443
    protocol    = "tcp"
    cidr_blocks = ["0.0.0.0/0"]
  }

  egress {
    from_port   = 0
    to_port     = 0
    protocol    = "-1"
    cidr_blocks = ["0.0.0.0/0"]
  }

  tags = local.common_tags
}
```

### RDS 数据库配置

```hcl
# rds.tf
# 数据库子网组
resource "aws_db_subnet_group" "main" {
  name       = "${local.name_prefix}-db-subnet"
  subnet_ids = aws_subnet.private[*].id

  tags = local.common_tags
}

# 数据库安全组
resource "aws_security_group" "db" {
  name        = "${local.name_prefix}-db-sg"
  description = "Security group for database"
  vpc_id      = aws_vpc.main.id

  ingress {
    from_port       = 3306
    to_port         = 3306
    protocol        = "tcp"
    security_groups = [aws_security_group.web.id]
  }

  tags = local.common_tags
}

# RDS 实例
resource "aws_db_instance" "main" {
  identifier           = "${local.name_prefix}-db"
  engine               = "mysql"
  engine_version       = "8.0"
  instance_class       = var.db_instance_class
  allocated_storage    = 20
  storage_encrypted    = true

  db_name  = var.db_name
  username = var.db_username
  password = var.db_password

  vpc_security_group_ids = [aws_security_group.db.id]
  db_subnet_group_name   = aws_db_subnet_group.main.name

  backup_retention_period = 7
  skip_final_snapshot     = false
  final_snapshot_identifier = "${local.name_prefix}-final-snapshot"

  tags = local.common_tags
}
```

### 应用负载均衡器

```hcl
# alb.tf
resource "aws_lb" "main" {
  name               = "${local.name_prefix}-alb"
  internal           = false
  load_balancer_type = "application"
  security_groups    = [aws_security_group.alb.id]
  subnets            = aws_subnet.public[*].id

  tags = local.common_tags
}

resource "aws_lb_target_group" "web" {
  name     = "${local.name_prefix}-tg"
  port     = 80
  protocol = "HTTP"
  vpc_id   = aws_vpc.main.id

  health_check {
    enabled             = true
    healthy_threshold   = 2
    interval            = 30
    matcher             = "200"
    path                = "/health"
    timeout             = 5
    unhealthy_threshold = 2
  }
}

resource "aws_lb_target_group_attachment" "web" {
  count            = var.instance_count
  target_group_arn = aws_lb_target_group.web.arn
  target_id        = aws_instance.web[count.index].id
  port             = 80
}

resource "aws_lb_listener" "http" {
  load_balancer_arn = aws_lb.main.arn
  port              = "80"
  protocol          = "HTTP"

  default_action {
    type             = "forward"
    target_group_arn = aws_lb_target_group.web.arn
  }
}
```

## 模块化最佳实践

### 模块结构

```
modules/
└── vpc/
    ├── main.tf
    ├── variables.tf
    └── outputs.tf
```

```hcl
# modules/vpc/main.tf
resource "aws_vpc" "this" {
  cidr_block = var.cidr_block

  tags = merge({
    Name = var.name
  }, var.tags)
}

# modules/vpc/variables.tf
variable "cidr_block" {
  type = string
}

variable "name" {
  type = string
}

variable "tags" {
  type    = map(string)
  default = {}
}

# modules/vpc/outputs.tf
output "vpc_id" {
  value = aws_vpc.this.id
}
```

### 使用模块

```hcl
# main.tf
module "vpc" {
  source     = "./modules/vpc"
  cidr_block = "10.0.0.0/16"
  name       = "main-vpc"
  tags = {
    Environment = "prod"
  }
}

# 引用模块输出
resource "aws_subnet" "main" {
  vpc_id     = module.vpc.vpc_id
  cidr_block = "10.0.1.0/24"
}
```

## 状态管理

### 远程状态存储

```hcl
# backend.tf
terraform {
  backend "s3" {
    bucket         = "my-terraform-state"
    key            = "prod/terraform.tfstate"
    region         = "ap-northeast-1"
    encrypt        = true
    dynamodb_table = "terraform-locks"
  }
}
```

### 状态锁定

使用 DynamoDB 防止并发操作：

```hcl
resource "aws_dynamodb_table" "terraform_locks" {
  name         = "terraform-locks"
  billing_mode = "PAY_PER_REQUEST"
  hash_key     = "LockID"

  attribute {
    name = "LockID"
    type = "S"
  }
}
```

### 状态命令

```bash
# 查看当前状态
terraform state list

# 查看特定资源状态
terraform state show aws_vpc.main

# 移动资源
terraform state mv aws_vpc.main aws_vpc.primary

# 导入已存在的资源
terraform import aws_vpc.main vpc-12345678

# 删除状态中的资源
terraform state rm aws_vpc.main
```

## CI/CD 集成

### GitHub Actions 示例

```yaml
# .github/workflows/terraform.yml
name: Terraform CI/CD

on:
  push:
    branches: [main]
  pull_request:

jobs:
  terraform:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Setup Terraform
        uses: hashicorp/setup-terraform@v3
        with:
          terraform_version: 1.7.0

      - name: Configure AWS credentials
        uses: aws-actions/configure-aws-credentials@v4
        with:
          aws-access-key-id: ${{ secrets.AWS_ACCESS_KEY_ID }}
          aws-secret-access-key: ${{ secrets.AWS_SECRET_ACCESS_KEY }}
          aws-region: ap-northeast-1

      - name: Terraform Init
        run: terraform init

      - name: Terraform Format
        run: terraform fmt -check

      - name: Terraform Plan
        run: terraform plan -out=tfplan

      - name: Terraform Apply
        if: github.ref == 'refs/heads/main'
        run: terraform apply -auto-approve tfplan
```

## 最佳实践总结

### 1. 代码组织

- 按环境分离：dev、staging、prod
- 使用模块提高复用性
- 统一命名规范

### 2. 安全实践

- 敏感数据使用 `sensitive = true`
- 状态文件加密存储
- 使用 IAM 最小权限原则
- 定期轮换密钥

### 3. 版本控制

```hcl
terraform {
  required_version = "&gt;= 1.7.0"

  required_providers {
    aws = {
      source  = "hashicorp/aws"
      version = "~&gt; 5.0"
    }
  }
}
```

### 4. 变更管理

- 始终先 `terraform plan`
- 代码审查变更内容
- 使用工作区隔离环境

### 5. 监控和告警

```hcl
# CloudWatch 告警示例
resource "aws_cloudwatch_metric_alarm" "cpu" {
  alarm_name          = "${local.name_prefix}-high-cpu"
  comparison_operator = "GreaterThanThreshold"
  evaluation_periods  = "2"
  metric_name         = "CPUUtilization"
  namespace           = "AWS/EC2"
  period              = "120"
  statistic           = "Average"
  threshold           = "80"

  dimensions = {
    AutoScalingGroupName = aws_autoscaling_group.web.name
  }
}
```

## 常见问题与解决

### 1. 状态漂移

当实际资源与状态文件不一致时：

```bash
# 刷新状态
terraform refresh

# 或者使用 plan -refresh-only
terraform plan -refresh-only
```

### 2. 依赖问题

显式声明依赖：

```hcl
resource "aws_instance" "web" {
  # ...

  depends_on = [
    aws_db_instance.main,
    aws_s3_bucket.assets
  ]
}
```

### 3. 循环依赖

使用 `terraform graph` 分析依赖关系：

```bash
terraform graph | dot -Tpng &gt; graph.png
```
