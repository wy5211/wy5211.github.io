# Linux 服务器安全加固实战指南

在云时代，服务器安全是每个运维工程师和开发者的必修课。一台未加固的服务器可能在几分钟内被扫描器发现并攻破。本文将系统性地介绍 Linux 服务器安全加固的最佳实践。

## 🔒 SSH 安全加固

SSH 是服务器管理的主要入口，也是攻击者的首要目标。

### 禁用 Root 登录

> **⚠️ 永远不要允许 root 用户直接 SSH 登录！**

```bash
# 编辑 SSH 配置
sudo vim /etc/ssh/sshd_config

# 禁用 root 登录
PermitRootLogin no

# 禁用密码登录（强制使用密钥）
PasswordAuthentication no
PubkeyAuthentication yes

# 修改默认端口（避免自动化扫描）
Port 2222

# 限制登录用户
AllowUsers your-username

# 重启 SSH 服务
sudo systemctl restart sshd
```

### 使用 SSH 密钥认证

```bash
# 本地生成 ED25519 密钥（比 RSA 更安全更快）
ssh-keygen -t ed25519 -C "your-email@example.com"

# 将公钥复制到服务器
ssh-copy-id -p 2222 username@server-ip

# 或手动添加
cat ~/.ssh/id_ed25519.pub | ssh username@server-ip "mkdir -p ~/.ssh && cat >> ~/.ssh/authorized_keys"
```

### 配置 SSH 跳板机

对于生产环境，建议使用跳板机（Bastion Host）：

```bash
# 本地 SSH 配置 ~/.ssh/config
Host bastion
    HostName bastion.example.com
    Port 2222
    User admin
    IdentityFile ~/.ssh/id_ed25519

Host production-*
    ProxyJump bastion
    User deploy
    IdentityFile ~/.ssh/id_ed25519
```

## 🛡️ 防火墙配置

### 使用 UFW（Ubuntu/Debian）

```bash
# 安装并启用 UFW
sudo apt install ufw
sudo ufw default deny incoming
sudo ufw default allow outgoing

# 允许 SSH（使用自定义端口）
sudo ufw allow 2222/tcp comment 'SSH'

# 允许 HTTP/HTTPS
sudo ufw allow 80/tcp comment 'HTTP'
sudo ufw allow 443/tcp comment 'HTTPS'

# 限制连接速率（防暴力破解）
sudo ufw limit 2222/tcp

# 启用防火墙
sudo ufw enable

# 查看状态
sudo ufw status verbose
```

### 使用 firewalld（CentOS/RHEL）

```bash
# 启动并启用 firewalld
sudo systemctl start firewalld
sudo systemctl enable firewalld

# 开放端口
sudo firewall-cmd --permanent --add-port=2222/tcp
sudo firewall-cmd --permanent --add-port=80/tcp
sudo firewall-cmd --permanent --add-port=443/tcp

# 重新加载配置
sudo firewall-cmd --reload

# 查看所有规则
sudo firewall-cmd --list-all
```

### iptables 高级规则

```bash
# 限制 SSH 连接速率
iptables -A INPUT -p tcp --dport 2222 -m state --state NEW -m recent --set
iptables -A INPUT -p tcp --dport 2222 -m state --state NEW -m recent --update --seconds 60 --hitcount 4 -j DROP

# 防止 SYN 洪水攻击
iptables -A INPUT -p tcp --syn -m limit --limit 1/s --limit-burst 3 -j ACCEPT

# 保存规则
iptables-save > /etc/iptables/rules.v4
```

## 👤 用户权限管理

### 创建受限用户

```bash
# 创建用户
sudo adduser deploy

# 添加到 sudo 组（按需）
sudo usermod -aG sudo deploy

# 或创建只能执行特定命令的用户
sudo visudo
# 添加：deploy ALL=(ALL) NOPASSWD: /usr/bin/systemctl restart nginx
```

### 最小权限原则

```bash
# 文件权限示例
chmod 700 ~/.ssh              # 只有所有者可访问
chmod 600 ~/.ssh/authorized_keys  # 只有所有者可读写
chmod 600 ~/.ssh/id_ed25519   # 私钥必须严格保护

# 检查可疑的 SUID 文件
find / -perm -4000 -type f 2>/dev/null
```

### 使用 sudo 审计

```bash
# 编辑 sudoers 配置
sudo visudo

# 添加日志记录
Defaults logfile="/var/log/sudo.log"
Defaults log_input, log_output
Defaults!/usr/bin/sudoreplay !log_output

# 所有 sudo 操作都会被记录
```

## 🔄 自动更新与补丁

### 配置自动安全更新

> **💡 自动更新只应用安全补丁，不会升级大版本，适合生产环境。**

**Ubuntu/Debian:**

```bash
# 安装 unattended-upgrades
sudo apt install unattended-upgrades apt-listchanges

# 配置自动更新
sudo dpkg-reconfigure -plow unattended-upgrades

# 手动配置
sudo vim /etc/apt/apt.conf.d/20auto-upgrades
```

```conf
APT::Periodic::Update-Package-Lists "1";
APT::Periodic::Unattended-Upgrade "1";
APT::Periodic::Download-Upgradeable-Packages "1";
APT::Periodic::AutocleanInterval "7";
```

**CentOS/RHEL:**

```bash
# 安装 dnf-automatic
sudo dnf install dnf-automatic

# 配置自动更新
sudo vim /etc/dnf/automatic.conf
```

```conf
[commands]
upgrade_type = security
download_updates = yes
apply_updates = yes

[emitters]
emit_via = stdio
```

```bash
# 启用定时器
sudo systemctl enable --now dnf-automatic.timer
```

## 🕵️ 入侵检测与日志监控

### 安装 Fail2Ban

```bash
# 安装
sudo apt install fail2ban

# 创建本地配置
sudo cp /etc/fail2ban/jail.conf /etc/fail2ban/jail.local
sudo vim /etc/fail2ban/jail.local
```

```ini
[DEFAULT]
bantime = 3600
findtime = 600
maxretry = 3
ignoreip = 127.0.0.1/8 你的本地IP

[sshd]
enabled = true
port = 2222
maxretry = 3
bantime = 86400
```

```bash
# 启动服务
sudo systemctl enable fail2ban
sudo systemctl start fail2ban

# 查看状态
sudo fail2ban-client status sshd
```

### 配置日志审计

```bash
# 安装 auditd
sudo apt install auditd

# 监控关键文件修改
sudo auditctl -w /etc/passwd -p wa -k passwd_changes
sudo auditctl -w /etc/ssh/sshd_config -p wa -k ssh_config
sudo auditctl -w /var/log/auth.log -p wa -k logins

# 持久化配置
sudo vim /etc/audit/rules.d/audit.rules
```

```
-w /etc/passwd -p wa -k passwd_changes
-w /etc/shadow -p wa -k shadow_changes
-w /etc/ssh/sshd_config -p wa -k ssh_config
-w /var/log/auth.log -p wa -k logins
-a always,exit -F arch=b64 -S chmod,chown -F auid>=1000 -F auid!=4294967295 -k perm_mod
```

### 检查可疑活动

```bash
# 查看失败登录
sudo grep "Failed password" /var/log/auth.log

# 查看成功登录
sudo grep "Accepted password" /var/log/auth.log

# 检查当前登录用户
w
who

# 检查最近登录
last -n 20

# 检查异常进程
ps aux --sort=-%cpu | head -20
ps aux --sort=-%mem | head -20

# 检查网络连接
sudo netstat -tulpn
sudo ss -tulpn
```

## 🔐 系统加固配置

### 内核参数优化

```bash
# 编辑 sysctl 配置
sudo vim /etc/sysctl.conf
```

```conf
# 防止 SYN 洪水攻击
net.ipv4.tcp_syncookies = 1
net.ipv4.tcp_max_syn_backlog = 2048
net.ipv4.tcp_synack_retries = 2

# 禁用 ICMP 重定向
net.ipv4.conf.all.accept_redirects = 0
net.ipv4.conf.default.accept_redirects = 0

# 禁用源路由
net.ipv4.conf.all.accept_source_route = 0
net.ipv4.conf.default.accept_source_route = 0

# 启用反向路径过滤
net.ipv4.conf.all.rp_filter = 1
net.ipv4.conf.default.rp_filter = 1

# 忽略 ICMP ping 请求（可选）
net.ipv4.icmp_echo_ignore_all = 1
```

```bash
# 应用配置
sudo sysctl -p
```

### 禁用不必要的服务

```bash
# 列出所有监听端口
sudo ss -tulpn

# 停止并禁用不必要的服务
sudo systemctl stop bluetooth
sudo systemctl disable bluetooth

# 检查开机自启服务
systemctl list-unit-files --type=service | grep enabled
```

### 文件系统安全

```bash
# 查找无主文件
find / -nouser -o -nogroup 2>/dev/null

# 查找全局可写文件
find / -perm -0002 -type f 2>/dev/null

# 挂载选项（编辑 /etc/fstab）
/tmp     none    defaults,noexec,nosuid,nodev    0 0
/var/tmp none    defaults,noexec,nosuid,nodev    0 0
```

## 📊 安全扫描工具

### 使用 Lynis 进行安全审计

```bash
# 安装 Lynis
sudo apt install lynis

# 运行完整审计
sudo lynis audit system

# 查看报告
sudo cat /var/log/lynis.log
```

### 使用 RKhunter 检查 Rootkit

```bash
# 安装
sudo apt install rkhunter

# 更新数据库
sudo rkhunter --update

# 扫描系统
sudo rkhunter --check

# 查看报告
sudo cat /var/log/rkhunter.log
```

### 使用 ClamAV 扫描恶意软件

```bash
# 安装
sudo apt install clamav clamav-daemon

# 更新病毒库
sudo freshclam

# 扫描系统
sudo clamscan -r --bell -i /

# 后台守护进程
sudo systemctl enable clamav-daemon
sudo systemctl start clamav-daemon
```

## 🎯 最佳实践总结

1. **最小权限原则**：只给予用户和服务必要的权限
2. **纵深防御**：多层安全措施，不依赖单一防护
3. **监控与审计**：持续监控，及时发现异常
4. **自动化更新**：及时应用安全补丁
5. **备份验证**：定期备份并测试恢复
6. **文档化**：记录所有安全配置和变更

> **⚠️ 安全是一个持续的过程，不是一次性的任务。定期审计和更新你的安全策略！**
