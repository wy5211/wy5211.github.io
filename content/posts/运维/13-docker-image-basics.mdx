## 前言

镜像是 Docker 的三大核心概念之一，是容器运行的基础。理解镜像的工作原理和掌握镜像的管理操作，是使用 Docker 的必备技能。本篇文章将详细介绍 Docker 镜像的分层存储原理，以及如何搜索、拉取、查看、删除和导出镜像。

**学习目标**：

- 理解镜像的分层存储机制
- 掌握镜像搜索和拉取
- 学会查看和管理本地镜像
- 了解镜像标签和命名规范

## 镜像的本质

### 什么是镜像

镜像是一个**只读**的模板，包含了运行应用所需的一切：

```
镜像 = 应用代码 + 运行时 + 系统库 + 配置文件 + 依赖项
```

**类比理解**：

- 📦 镜像就像一个**应用程序安装包**
- 📋 镜像就像一个**虚拟机快照**
- 🍳 镜像就像一个**菜谱**（定义了所有原料和步骤）

### 镜像的分层存储

Docker 镜像由多个只读层组成，采用**联合文件系统（UnionFS）**：

```
┌─────────────────────────┐
│  容器可写层（Container）  │  ← 容器创建时添加
├─────────────────────────┤
│  应用代码层              │  ← Dockerfile ADD/COPY
├─────────────────────────┤
│  运行时依赖层            │  ← Dockerfile RUN
├─────────────────────────┤
│  基础镜像层              │  ← Dockerfile FROM
├─────────────────────────┤
│  scratch 空白层          │  ← 最底层（无文件）
└─────────────────────────┘
```

**分层优势**：

1. 📦 **共享基础层**：多个镜像可共享相同的基础层
2. ⚡ **快速构建**：只重建变更的层
3. 💾 **节省空间**：相同的层只存储一次
4. 🔄 **版本管理**：每一层都有唯一 ID

**查看镜像分层**：

```bash
# 查看镜像的层历史
docker history nginx:latest

# 输出示例：
# IMAGE          CREATED      CREATED BY                                      SIZE
# 8f336738...    2 weeks ago  /bin/sh -c #(nop)  CMD ["nginx" "-g" "daemon...  0B
# <missing>      2 weeks ago  /bin/sh -c #(nop)  STOPSIGNAL SIGQUIT           0B
# <missing>      2 weeks ago  /bin/sh -c #(nop)  EXPOSE 80                   0B
# f8b8b...       2 weeks ago  /bin/sh -c #(nop) COPY file:3a83...             6.39MB
# <missing>      2 weeks ago  /bin/sh -c set -x     && addgroup --system ...   60.6MB
# <missing>      2 weeks ago  /bin/sh -c #(nop)  CMD ["/bin/sh"]              0B
# <missing>      2 weeks ago  /bin/sh -c #(nop) ADD file:8b...  in /         7.84MB
```

## 搜索镜像

### docker search 命令

在拉取镜像之前，可以先在 Docker Hub 上搜索。

```bash
# 基本搜索
docker search nginx

# 输出示例：
# NAME                    DESCRIPTION         STARS  OFFICIAL  AUTOMATED
# nginx                   Official Nginx...   18000  [OK]
# bitnami/nginx           Bitnami nginx...   150    [OK]
# ubuntu/nginx           Nginx...            50     [OK]
```

**字段说明**：

- `NAME`：镜像名称
- `DESCRIPTION`：镜像描述
- `STARS`：收藏数（类似 GitHub stars）
- `OFFICIAL`：[OK] 表示官方镜像
- `AUTOMATED`：[OK] 表示自动构建

### 高级搜索

```bash
# 1. 过滤官方镜像
docker search --filter is-official=true nginx

# 2. 过滤自动化构建
docker search --filter is-automated=true nginx

# 3. 限制结果数量
docker search --limit 5 nginx

# 4. 搜索超过 100 个 stars 的镜像
docker search --filter stars=100 node

# 5. 官方网页搜索（更强大）
# 访问：https://hub.docker.com/search?q=nginx&type=image
```

### 搜索技巧

**官方 vs 社区镜像**：

```bash
# ✅ 推荐：官方镜像
# 命名规则：单一单词
nginx
node
postgres
redis

# ⚠️ 谨慎使用：社区镜像
# 命名规则：用户名/镜像名
username/nginx
username/custom-app
```

**选择镜像的标准**：

1. ✅ 优先选择官方镜像（[OK] 标记）
2. ✅ 选择 stars 数量多的
3. ✅ 查看最近更新时间
4. ✅ 阅读用户评价和问题反馈

## 拉取镜像

### docker pull 命令

```bash
# 基本用法
docker pull nginx

# 等同于
docker pull nginx:latest
```

**拉取过程**：

```
latest: Pulling from library/nginx
8f336738...: Pulling fs layer
3b4f2ec...: Pulling fs layer
...
8f336738...: Pull complete
3b4f2ec...: Pull complete
Digest: sha256:abcd...
Status: Downloaded newer image for nginx:latest
docker.io/library/nginx:latest
```

### 拉取指定版本

```bash
# 拉取特定标签
docker pull nginx:1.24
docker pull nginx:1.24-alpine
docker pull nginx:stable-perl

# 查看所有可用标签
# 访问：https://hub.docker.com/_/nginx/tags
```

### 常用标签说明

**语义化版本**：

```bash
nginx:1.24        # 特定版本
nginx:1.24.0      # 完整版本号
nginx:1           # 主版本（自动使用 1.x.x 最新版）
```

**变体标签**：

```bash
nginx:alpine      # Alpine Linux 基础镜像（更小）
nginx:slim        # 精简版（删除文档等）
nginx:full        # 完整版（包含所有工具）
nginx:perl        # 带 Perl 支持
```

**特殊标签**：

```bash
nginx:latest      # 最新版本（默认）
nginx:stable      # 稳定版本
```

### 拉取私有仓库镜像

```bash
# 从私有仓库拉取
docker pull myregistry.com/myapp:1.0

# 从阿里云容器镜像服务拉取
docker pull registry.cn-hangzhou.aliyuncs.com/myapp/nginx:latest
```

### 登录仓库

```bash
# 登录 Docker Hub
docker login

# 输入用户名和密码后，可以拉取私有镜像
docker pull username/private-repo:latest

# 登录私有仓库
docker login myregistry.com

# 登出
docker logout
```

## 查看镜像

### docker images 命令

```bash
# 列出所有本地镜像
docker images

# 输出示例：
# REPOSITORY   TAG       IMAGE ID       CREATED        SIZE
# nginx        latest    8f336738...     2 weeks ago    142MB
# node         18-alpine 1a2b3c4d...     3 weeks ago    120MB
# postgres     15        e5f6g7h8...     1 month ago    378MB
```

**字段说明**：

- `REPOSITORY`：仓库名称
- `TAG`：标签（默认 latest）
- `IMAGE ID`：镜像唯一 ID（前 12 位）
- `CREATED`：创建时间
- `SIZE`：镜像大小

### 高级查看选项

```bash
# 1. 显示镜像摘要（包含 Digest）
docker images --digests

# 2. 只显示镜像 ID
docker images -q

# 3. 格式化输出
docker images --format "table {{.Repository}}\t{{.Size}}\t{{.Tag}}"

# 4. 过滤显示
docker images --filter "reference=nginx*"
docker images --filter "before=node:18"  # 在某个镜像之前
docker images --filter "since=node:18"   # 在某个镜像之后

# 5. 显示悬空镜像（无标签）
docker images --filter "dangling=true"
```

### 查看镜像详情

```bash
# 查看镜像的详细信息（JSON 格式）
docker inspect nginx:latest

# 查看特定字段
docker inspect --format='{{.Config.Image}}' nginx:latest
docker inspect --format='{{.Architecture}}' nginx:latest
docker inspect --format='{{.Size}}' nginx:latest
```

**常用格式化选项**：

```bash
# 查看镜像创建时间
docker inspect --format='{{.Created}}' nginx:latest

# 查看镜像环境变量
docker inspect --format='{{range .Config.Env}}{{.}} {{end}}' nginx:latest

# 查看镜像暴露的端口
docker inspect --format='{{range $p, $conf := .Config.ExposedPorts}}{{$p}} {{end}}' nginx:latest

# 查看镜像卷配置
docker inspect --format='{{range $p, $conf := .Config.Volumes}}{{$p}} {{end}}' postgres:15
```

## 删除镜像

### docker rmi 命令

```bash
# 删除单个镜像
docker rmi nginx:latest

# 删除多个镜像
docker rmi nginx:1.24 node:18-alpine

# 强制删除（即使有容器在使用）
docker rmi -f nginx:latest

# 通过镜像 ID 删除
docker rmi 8f336738...
```

### 删除悬空镜像

```bash
# 查看悬空镜像（<none> 标签）
docker images --filter "dangling=true"

# 删除所有悬空镜像
docker image prune

# 强制删除（未使用的镜像）
docker image prune -a
```

### 批量删除

```bash
# 删除所有镜像（危险操作）
docker rmi $(docker images -q)

# 删除特定名称的所有镜像
docker rmi $(docker images --filter=reference='nginx:*' -q)

# 删除所有 none 标签的镜像
docker rmi $(docker images --filter "dangling=true" -q)
```

### 清理镜像

```bash
# 删除未使用的镜像
docker image prune

# 删除所有未使用的镜像（包括无容器使用的）
docker image prune -a

# 清理所有未使用的 Docker 资源
docker system prune -a --volumes
```

## 导入导出镜像

### 导出镜像

**场景**：离线环境、备份、迁移

```bash
# 方式 1：导出镜像到 tar 文件
docker save -o nginx.tar nginx:latest

# 或使用重定向
docker save nginx:latest > nginx.tar

# 导出多个镜像
docker save -o images.tar nginx:latest node:18-alpine

# 使用 gzip 压缩（减小文件大小）
docker save nginx:latest | gzip > nginx.tar.gz
```

### 导入镜像

```bash
# 从 tar 文件导入
docker load -i nginx.tar

# 或使用重定向
docker load < nginx.tar

# 导入压缩的镜像
gunzip -c nginx.tar.gz | docker load
```

### 容器导出与导入

**区别**：

- `docker export`：导出容器的文件系统（当前状态）
- `docker save`：导出镜像（包含所有层和历史）

```bash
# 导出容器为 tar 文件
docker export my-container > container.tar

# 导入为新的镜像
docker import container.tar myapp:v1
```

## 镜像命名规范

### 完整格式

```
[仓库地址/][用户名/]镜像名[:标签]
```

**示例解析**：

| 镜像名称                      | 说明                             |
| ----------------------------- | -------------------------------- |
| `nginx`                       | Docker Hub 官方镜像，latest 标签 |
| `nginx:1.24`                  | Docker Hub 官方镜像，指定版本    |
| `nginx:alpine`                | Docker Hub 官方镜像，alpine 变体 |
| `username/nginx`              | 用户的镜像，latest 标签          |
| `username/nginx:custom`       | 用户的镜像，自定义标签           |
| `registry.com/nginx`          | 私有仓库镜像                     |
| `registry.com:5000/nginx:1.0` | 私有仓库，指定端口和版本         |
| `localhost:5000/myapp:v1`     | 本地私有仓库                     |

### 官方镜像命名

**特征**：

- ✅ 单一单词：`nginx`, `node`, `postgres`
- ✅ 由官方维护：有 [OK] 标记
- ✅ 可靠性和安全性有保障

**常见官方镜像**：

```bash
# Web 服务器
nginx
apache

# 编程语言
node
python
golang
java
openjdk

# 数据库
postgres
mysql
mongo
redis

# 消息队列
rabbitmq
kafka
```

### 标签最佳实践

```bash
# ✅ 生产环境：使用具体版本
docker pull nginx:1.24.0

# ✅ 开发环境：使用主版本
docker pull nginx:1.24

# ⚠️ 避免使用 latest
docker pull nginx:latest
# 原因：可能在不同时间拉取到不同版本
```

## 实战演练

### 场景 1：准备开发环境

```bash
# 1. 搜索 Node.js 镜像
docker search node --filter is-official=true

# 2. 拉取 Node.js 18 Alpine 版本
docker pull node:18-alpine

# 3. 验证镜像
docker images node:18-alpine

# 4. 查看镜像详情
docker inspect node:18-alpine

# 5. 运行测试容器
docker run --rm node:18-alpine node --version
# 输出：v18.x.x
```

### 场景 2：离线部署

```bash
# 在有网络的机器上
# 1. 拉取所需镜像
docker pull nginx:1.24
docker pull postgres:15
docker pull redis:7-alpine

# 2. 导出镜像
docker save -o images.tar \
  nginx:1.24 \
  postgres:15 \
  redis:7-alpine

# 3. 压缩（可选）
gzip images.tar

# 4. 传输到目标机器
scp images.tar.gz user@target-server:/tmp/

# 在离线机器上
# 5. 解压并导入
gunzip -c /tmp/images.tar.gz | docker load

# 6. 验证
docker images
```

### 场景 3：版本管理

```bash
# 1. 拉取应用镜像的多个版本
docker pull myapp:1.0.0
docker pull myapp:1.1.0
docker pull myapp:latest

# 2. 查看所有版本
docker images myapp

# 3. 切换使用的版本
docker run -d --name myapp myapp:1.0.0
docker stop myapp
docker rm myapp
docker run -d --name myapp myapp:1.1.0

# 4. 清理旧版本
docker rmi myapp:1.0.0
```

### 场景 4：镜像迁移

```bash
# 从旧服务器导出
old-server$ docker save -o app.tar mycompany/myapp:2.0

# 传输
old-server$ scp app.tar user@new-server:/tmp/

# 在新服务器导入
new-server$ docker load -i /tmp/app.tar
new-server$ docker run -d mycompany/myapp:2.0
```

## 最佳实践

### 1. 选择合适的基础镜像

```bash
# ✅ 推荐：使用 alpine 变体（更小、更安全）
FROM python:3.11-alpine
# 大小：~50MB

# ❌ 避免：使用默认变体（较大）
FROM python:3.11
# 大小：~900MB
```

### 2. 使用具体版本标签

```dockerfile
# ✅ 推荐
FROM node:18.17.0-alpine

# ❌ 避免
FROM node:latest  # 可能导致不一致
```

### 3. 定期清理镜像

```bash
# 每周清理一次
docker image prune -a --filter "until=72h"
```

### 4. 扫描镜像漏洞

```bash
# 使用 Docker Scout 扫描
docker scout quickscan nginx:latest

# 或使用第三方工具
trivy image nginx:latest
```

## 常见问题

### Q1: 拉取镜像速度慢怎么办？

**答**：配置镜像加速器（参考第 12 篇文章）

### Q2: 如何查看镜像的大小？

**答**：

```bash
docker images --format "table {{.Repository}}\t{{.Size}}"

# 查看所有层的大小
docker history nginx:latest
```

### Q3: 镜像和容器的关系？

**答**：

- 镜像是模板（只读）
- 容器是镜像的运行实例（可写）
- 一个镜像可以创建多个容器

### Q4: 如何判断镜像是否官方？

**答**：

```bash
docker search nginx --filter is-official=true

# 或查看镜像名称
# 官方镜像：nginx, node, postgres
# 用户镜像：username/nginx
```

### Q5: 悬空镜像是什么？

**答**：没有标签的镜像（`<none>`），通常是：

- 构建过程中产生的中间镜像
- 被新版本替代的旧镜像

可以安全删除：

```bash
docker image prune
```
