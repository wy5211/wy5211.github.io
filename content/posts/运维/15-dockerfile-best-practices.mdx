## 前言

Dockerfile 是构建 Docker 镜像的脚本，它定义了镜像的每一层。编写高质量的 Dockerfile 不仅能减小镜像体积、提高构建速度，还能增强安全性和可维护性。本篇文章将详细介绍 Dockerfile 的语法、指令和最佳实践。

**学习目标**：

- 掌握 Dockerfile 的基本语法和常用指令
- 学会使用多阶段构建优化镜像
- 理解镜像缓存机制和优化策略
- 掌握安全配置和生产环境最佳实践

## Dockerfile 基础

### 什么是 Dockerfile

Dockerfile 是一个文本文件，包含了构建 Docker 镜像的所有指令。

**基本结构**：

```dockerfile
# 注释
INSTRUCTION arguments
```

**示例**：

```dockerfile
FROM node:18-alpine
WORKDIR /app
COPY package*.json ./
RUN npm ci --only=production
COPY . .
CMD ["node", "server.js"]
```

### 常用指令

#### FROM - 基础镜像

```dockerfile
# 指定基础镜像
FROM node:18-alpine

# 多阶段构建
FROM node:18-alpine AS builder
FROM node:18-alpine
```

#### RUN - 执行命令

```dockerfile
# Shell 形式
RUN apt-get update && apt-get install -y curl

# Exec 形式
RUN ["apt-get", "update"]

# 链接命令减少层数
RUN apt-get update && \
    apt-get install -y \
        curl \
        wget \
        vim && \
    rm -rf /var/lib/apt/lists/*
```

#### COPY 和 ADD - 复制文件

```dockerfile
# COPY: 复制文件/目录
COPY package.json /app/
COPY . /app/
COPY --from=builder /app/dist ./dist

# ADD: 支持自动解压和 URL
ADD app.tar.gz /opt/
ADD https://example.com/file.tar.gz /opt/
```

#### CMD - 默认命令

```dockerfile
# Exec 形式（推荐）
CMD ["node", "server.js"]

# Shell 形式
CMD node server.js

# 只能有一个 CMD，多个会覆盖
```

#### ENTRYPOINT - 入口点

```dockerfile
#ENTRYPOINT + CMD 组合
ENTRYPOINT ["nginx"]
CMD ["-g", "daemon off;"]
# 实际执行: nginx -g daemon off;

# 覆盖 ENTRYPOINT
docker run --entrypoint /bin/bash nginx
```

#### WORKDIR - 工作目录

```dockerfile
# 设置工作目录
WORKDIR /app

# 自动创建目录
WORKDIR /app/data

# 环境变量展开
WORKDIR ${APP_HOME}
```

#### ENV - 环境变量

```dockerfile
# 设置环境变量
ENV NODE_ENV=production
ENV PATH="/app/node_modules/.bin:${PATH}"
ENV APP_VERSION=1.0.0 \
    APP_PORT=3000
```

#### EXPOSE - 暴露端口

```dockerfile
# 声明容器监听的端口
EXPOSE 80 443

# 只是声明，不会实际映射端口
```

#### VOLUME - 数据卷

```dockerfile
# 创建数据卷挂载点
VOLUME ["/var/log/app"]
VOLUME ["/data", "/config"]

# 防止数据丢失
```

#### USER - 用户

```dockerfile
# 切换用户
USER node
USER 1000:1000

# 创建用户并切换
RUN addgroup -g 1001 -S nodejs && \
    adduser -S nodejs -u 1001 && \
    USER nodejs
```

## 完整示例

### Node.js 应用

```dockerfile
# 1. 选择基础镜像
FROM node:18-alpine AS builder

# 2. 设置工作目录
WORKDIR /app

# 3. 复制依赖文件
COPY package*.json ./

# 4. 安装依赖
RUN npm ci --only=production

# 5. 复制应用代码
COPY . .

# 6. 设置非 root 用户
RUN addgroup -g 1001 -S nodejs && \
    adduser -S nodejs -u 1001
USER nodejs

# 7. 暴露端口
EXPOSE 3000

# 8. 健康检查
HEALTHCHECK --interval=30s --timeout=3s \
  CMD node healthcheck.js || exit 1

# 9. 启动命令
CMD ["node", "server.js"]
```

### Python 应用

```dockerfile
FROM python:3.11-slim

WORKDIR /app

# 安装系统依赖
RUN apt-get update && \
    apt-get install -y --no-install-recommends \
        gcc \
        && rm -rf /var/lib/apt/lists/*

# 复制依赖文件
COPY requirements.txt .

# 安装 Python 依赖
RUN pip install --no-cache-dir -r requirements.txt

# 复制应用代码
COPY . .

# 创建非 root 用户
RUN useradd -m -u 1000 appuser && \
    chown -R appuser:appuser /app
USER appuser

EXPOSE 8000

CMD ["python", "app.py"]
```

## 多阶段构建

### 为什么需要多阶段构建

**问题**：构建镜像包含大量构建工具，导致镜像体积大。

**解决**：使用多阶段构建，只保留运行时需要的文件。

### 基本语法

```dockerfile
# 构建阶段
FROM golang:1.21 AS builder
WORKDIR /src
COPY . .
RUN CGO_ENABLED=0 go build -o app

# 运行阶段
FROM alpine:3.19
WORKDIR /app
COPY --from=builder /src/app .
CMD ["./app"]
```

### Go 应用示例

```dockerfile
# 构建阶段
FROM golang:1.21-alpine AS builder

WORKDIR /app

# 安装依赖
COPY go.mod go.sum ./
RUN go mod download

# 复制源代码
COPY . .

# 构建应用
RUN CGO_ENABLED=0 GOOS=linux go build -o main .

# 运行阶段
FROM alpine:3.19

# 安装 ca 证书（如果需要 HTTPS）
RUN apk --no-cache add ca-certificates

WORKDIR /app

# 从构建阶段复制二进制文件
COPY --from=builder /app/main .

# 非特权用户
RUN addgroup -g 1000 appuser && \
    adduser -D -u 1000 -G appuser appuser
USER appuser

EXPOSE 8080

CMD ["./main"]
```

### React 应用示例

```dockerfile
# 依赖阶段
FROM node:18-alpine AS deps
WORKDIR /app
COPY package*.json ./
RUN npm ci

# 构建阶段
FROM node:18-alpine AS builder
WORKDIR /app
COPY --from=deps /app/node_modules ./node_modules
COPY . .
RUN npm run build

# 生产阶段
FROM nginx:alpine
COPY --from=builder /app/dist /usr/share/nginx/html
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
```

## 镜像缓存优化

### 缓存机制

Docker 会缓存镜像的每一层，如果指令和文件没变化，就使用缓存。

**最佳实践**：

```dockerfile
# ❌ 不好的做法：复制所有文件
COPY . .
RUN npm install

# ✅ 好的做法：先复制依赖文件
COPY package*.json ./
RUN npm install
COPY . .
```

### 优化技巧

```dockerfile
# 1. 利用缓存：先复制不变的文件
FROM node:18-alpine
WORKDIR /app

# 依赖文件很少变化
COPY package*.json ./
RUN npm ci --only=production

# 源代码经常变化
COPY . .

# 2. 减少层数：合并命令
RUN apt-get update && \
    apt-get install -y curl wget && \
    rm -rf /var/lib/apt/lists/*

# 3. 排序文件：把很少变化的文件放前面
COPY package.json ./
COPY package-lock.json ./
COPY tsconfig.json ./
COPY src/ ./src/
```

## 镜像瘦身

### 选择合适的基础镜像

```dockerfile
# ❌ 体积大：940MB
FROM node:18

# ✅ 体积小：120MB
FROM node:18-alpine

# ✅ 更小：使用 distroless
FROM gcr.io/distroless/nodejs18-debian11
```

### 清理不需要的文件

```dockerfile
# 安装后清理缓存
RUN apt-get update && \
    apt-get install -y python3 && \
    apt-get clean && \
    rm -rf /var/lib/apt/lists/*

# npm 清理缓存
RUN npm ci --only=production && \
    npm cache clean --force
```

### 使用 .dockerignore

```dockerfile
# .dockerignore 文件
node_modules
npm-debug.log
.git
.env
.DS_Store
*.md
coverage
.vscode
```

## 安全配置

### 非 root 用户

```dockerfile
FROM node:18-alpine

# 创建用户
RUN addgroup -g 1001 -S nodejs && \
    adduser -S nodejs -u 1001

# 设置工作目录并授权
WORKDIR /app
RUN chown -R nodejs:nodejs /app

# 切换用户
USER nodejs

CMD ["node", "server.js"]
```

### 最小化权限

```dockerfile
# 只保留必要的 Linux 能力
FROM node:18-alpine
RUN addgroup -g 1001 -S nodejs && \
    adduser -S nodejs -u 1001
USER nodejs

# 移除所有能力，只添加网络绑定
RUN apk add --no-cache libcap && \
    setcapcap_net_bind_service=+ep /usr/bin/node

EXPOSE 3000
CMD ["node", "server.js"]
```

### 扫描漏洞

```bash
# 构建后扫描
docker build -t myapp:1.0 .
docker scout quickscan myapp:1.0

# 或使用 Trivy
trivy image myapp:1.0
```

## 最佳实践总结

### 1. 使用明确版本标签

```dockerfile
# ✅ 推荐
FROM node:18.17.0-alpine

# ❌ 避免
FROM node:latest
FROM node:alpine
```

### 2. 合并 RUN 指令

```dockerfile
# ✅ 推荐
RUN apk add --no-cache \
        curl \
        wget \
        vim

# ❌ 避免
RUN apk add --no-cache curl
RUN apk add --no-cache wget
RUN apk add --no-cache vim
```

### 3. 利用构建缓存

```dockerfile
# ✅ 推荐
COPY package*.json ./
RUN npm ci --only=production
COPY . .

# ❌ 避免
COPY . .
RUN npm install
```

### 4. 使用多阶段构建

```dockerfile
# ✅ 推荐
FROM golang:1.21 AS builder
RUN CGO_ENABLED=0 go build -o app
FROM alpine:3.19
COPY --from=builder /app/app .
CMD ["./app"]
```

### 5. 设置健康检查

```dockerfile
HEALTHCHECK --interval=30s --timeout=3s \
  CMD curl -f http://localhost:3000/health || exit 1
```

### 6. 使用 .dockerignore

```
node_modules
npm-debug.log
.git
.env.local
```

## 常见错误

### 1. COPY 路径错误

```dockerfile
# ❌ 错误：路径不存在
COPY ../package.json /app/

# ✅ 正确：从构建上下文复制
COPY package.json /app/
```

### 2. RUN 命令换行

```dockerfile
# ❌ 错误：缺少续行符
RUN apt-get update
    apt-get install -y curl

# ✅ 正确：使用续行符
RUN apt-get update && \
    apt-get install -y curl
```

### 3. 忘记暴露端口

```dockerfile
# ❌ 忘记暴露
CMD ["nginx", "-g", "daemon off;"]

# ✅ 添加暴露
EXPOSE 80 443
CMD ["nginx", "-g", "daemon off;"]
```

## 总结

本篇文章我们学习了 Dockerfile 的编写和优化：

**核心指令**：

- `FROM`：基础镜像
- `RUN`：执行命令
- `COPY/ADD`：复制文件
- `CMD/ENTRYPOINT`：启动命令
- `WORKDIR`：工作目录
- `ENV`：环境变量
- `EXPOSE`：暴露端口
- `VOLUME`：数据卷
- `USER`：用户

**优化技巧**：

1. 使用多阶段构建减小镜像体积
2. 利用缓存机制加快构建速度
3. 选择合适的基础镜像（alpine）
4. 合并 RUN 指令减少层数
5. 使用 .dockerignore 排除不需要的文件

**安全实践**：

1. 使用非 root 用户运行
2. 扫描镜像漏洞
3. 设置资源限制
4. 使用明确版本标签
