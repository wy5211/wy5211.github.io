# Next.js 全栈开发深入实践

Next.js 已经从一个 React 框架演变为完整的全栈开发平台。随着 App Router 的推出，Next.js 提供了前所未有的服务器端渲染、数据获取和 API 开发能力，真正实现了"一次学习，全栈开发"。

本文将深入探讨 Next.js App Router 的全栈开发能力，帮助你充分利用这个强大的平台。

## App Router 核心原理

Next.js App Router（`app` 目录）是 Next.js 13 引入的新一代路由系统，它基于文件系统路由，集成了 React Server Components、Streaming、Layouts 等现代特性。

### 文件系统路由

```
app/
├── (auth)/                    # 路由组
│   ├── login/
│   │   └── page.tsx          # /login
│   └── register/
│       └── page.tsx          # /register
├── (dashboard)/               # 路由组
│   ├── dashboard/
│   │   ├── page.tsx          # /dashboard
│   │   ├── settings/
│   │   │   └── page.tsx      # /dashboard/settings
│   │   └── layout.tsx        # /dashboard 布局
├── api/                      # API Routes
│   ├── users/
│   │   ├── route.ts          # GET /api/users
│   │   └── [id]/
│   │       └── route.ts      # GET/POST /api/users/:id
│   └── auth/
│       └── route.ts          # POST /api/auth
├── layout.tsx                # 根布局
├── page.tsx                  # 首页 /
├── loading.tsx               # 全局加载状态
├── error.tsx                # 全局错误边界
└── not-found.tsx            # 全局 404 页面
```

## Server Components 最佳实践

Server Components（服务器组件）是 React 18 引入的新特性，Next.js App Router 默认使用 Server Components。

### Server vs Client Components

**Server Components**：
- 在服务器上渲染
- 直接访问数据库、文件系统
- 默认所有组件都是 Server Components
- 不能使用 `useState`、`useEffect` 等 React Hooks

**Client Components**：
- 在浏览器中渲染
- 可以使用所有 React Hooks
- 需要添加 `'use client'` 指令

### 何时使用 Server Components

```typescript
// ✅ Server Components：适合的场景

// 1. 数据获取
export default async function UserList() {
  const users = await prisma.user.findMany();

  return (
    <ul>
      {users.map(user => (
        <li key={user.id}>{user.name}</li>
      ))}
    </ul>
  );
}

// 2. 访问敏感数据（API keys、数据库连接）
export async function Dashboard() {
  const data = await fetchFromDatabase();
  // 不会暴露 API keys 到客户端

  return <div>{/* ... */}</div>;
}

// 3. 大型组件（减少 JavaScript bundle）
export default function HeavyComponent() {
  // 在服务器上渲染，减少发送到客户端的 JavaScript
  return <div>Large content...</div>;
}

// 4. 使用服务器端 API
export async function Post({ id }: { id: string }) {
  const post = await fetch(`https://api.example.com/posts/${id}`).then(r => r.json());

  return <article>{post.content}</article>;
}
```

## Server Actions vs API Routes

Next.js 14 引入了 Server Actions，提供了一种新的服务器端数据修改方式。

### API Routes（传统方式）

```typescript
// app/api/users/route.ts
import { NextRequest, NextResponse } from 'next/server';
import { prisma } from '@/lib/db';

export async function GET(request: NextRequest) {
  const users = await prisma.user.findMany();
  return NextResponse.json({ users });
}

export async function POST(request: NextRequest) {
  const body = await request.json();
  const user = await prisma.user.create({ data: body });
  return NextResponse.json({ user }, { status: 201 });
}
```

**客户端使用**：

```typescript
'use client';

export default function CreateUserForm() {
  const handleSubmit = async (e: React.FormEvent) => {
    e.preventDefault();

    const formData = new FormData(e.target as HTMLFormElement);

    await fetch('/api/users', {
      method: 'POST',
      body: JSON.stringify(Object.fromEntries(formData))
    });
  };

  return (
    <form onSubmit={handleSubmit}>
      <input name="name" placeholder="Name" />
      <input name="email" placeholder="Email" />
      <button type="submit">Create</button>
    </form>
  );
}
```

### Server Actions（新方式）

```typescript
// app/actions.ts
'use server';

import { revalidatePath } from 'next/cache';
import { prisma } from '@/lib/db';
import { z } from 'zod';

const createUserSchema = z.object({
  name: z.string().min(2),
  email: z.string().email(),
  role: z.enum(['admin', 'user']).default('user')
});

export async function createUser(formData: FormData) {
  // 验证数据
  const data = createUserSchema.parse({
    name: formData.get('name'),
    email: formData.get('email'),
    role: formData.get('role')
  });

  // 创建用户
  const user = await prisma.user.create({ data });

  // 重新验证缓存
  revalidatePath('/users');

  return { success: true, user };
}

export async function updateUser(id: number, formData: FormData) {
  const data = createUserSchema.parse({
    name: formData.get('name'),
    email: formData.get('email'),
    role: formData.get('role')
  });

  const user = await prisma.user.update({
    where: { id },
    data
  });

  revalidatePath('/users');
  revalidatePath(`/users/${id}`);

  return { success: true, user };
}

export async function deleteUser(id: number) {
  await prisma.user.delete({ where: { id } });
  revalidatePath('/users');

  return { success: true };
}
```

**客户端使用**：


### Server Actions 优势

1. **类型安全**：直接传递 FormData 或对象
2. **减少代码**：无需手动处理 fetch
3. **内置错误处理**：自动处理错误和重试
4. **优化性能**：减少网络往返
5. **表单集成**：原生支持 HTML 表单

```typescript
// app/actions/users.ts
'use server';

import { z } from 'zod';

// 使用 Zod 验证
const createUserSchema = z.object({
  name: z.string().min(2, 'Name must be at least 2 characters'),
  email: z.string().email('Invalid email address'),
  role: z.enum(['admin', 'user']).default('user')
});

export async function createUser(prevState: any, formData: FormData) {
  try {
    // 验证
    const data = createUserSchema.parse({
      name: formData.get('name'),
      email: formData.get('email'),
      role: formData.get('role') || 'user'
    });

    // 创建用户
    const user = await prisma.user.create({ data });

    revalidatePath('/users');

    return { success: true, user };
  } catch (error) {
    if (error instanceof z.ZodError) {
      return { success: false, errors: error.errors };
    }
    return { success: false, error: 'Failed to create user' };
  }
}

// 客户端使用
'use client';
import { useFormState, useFormStatus } from 'react-dom';
import { createUser } from '@/app/actions/users';

export default function CreateUserForm() {
  const [state, formAction] = useFormState(createUser, null);
  const { pending } = useFormStatus();

  return (
    <form action={formAction} className="space-y-4">
      <div>
        <label htmlFor="name">Name</label>
        <input
          id="name"
          name="name"
          className="border rounded p-2 w-full"
        />
        {state?.errors?.find(e => e.path[0] === 'name') && (
          <p className="text-red-500 text-sm">
            {state.errors.find(e => e.path[0] === 'name')?.message}
          </p>
        )}
      </div>

      <div>
        <label htmlFor="email">Email</label>
        <input
          id="email"
          name="email"
          type="email"
          className="border rounded p-2 w-full"
        />
        {state?.errors?.find(e => e.path[0] === 'email') && (
          <p className="text-red-500 text-sm">
            {state.errors.find(e => e.path[0] === 'email')?.message}
          </p>
        )}
      </div>

      <button
        type="submit"
        disabled={pending}
        className="bg-blue-500 text-white px-4 py-2 rounded disabled:opacity-50"
      >
        {pending ? 'Creating...' : 'Create User'}
      </button>

      {state?.error && (
        <p className="text-red-500">{state.error}</p>
      )}
    </form>
  );
}
```

## 缓存策略深度解析

Next.js 提供了强大的缓存系统，理解缓存策略对性能优化至关重要。

### Next.js 缓存层次

```
┌─────────────────────────────────────────────────────────────┐
│                  浏览器缓存                              │
│  - HTTP Cache headers (Cache-Control, ETag)             │
└─────────────────────────────────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────────────┐
│                  Next.js 缓存                             │
│  - Router Cache (导航缓存)                               │
│  - Full Route Cache (静态页面缓存)                         │
│  - Data Cache (fetch 请求缓存)                             │
└─────────────────────────────────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────────────┐
│                  数据源缓存                               │
│  - 数据库查询缓存                                        │
│  - CDN 缓存                                              │
└─────────────────────────────────────────────────────────────┘
```

### fetch 缓存配置

```typescript
// 默认：使用 Next.js 缓存
const data = await fetch('https://api.example.com/data');

// 禁用缓存（每次都重新获取）
const data = await fetch('https://api.example.com/data', {
  cache: 'no-store'
});

// 强制缓存（长期缓存）
const data = await fetch('https://api.example.com/data', {
  cache: 'force-cache'
});

// 设置重新验证时间（ISR）
const data = await fetch('https://api.example.com/data', {
  next: { revalidate: 3600 } // 1 小时
});

// 基于标签的重新验证
const data = await fetch('https://api.example.com/data', {
  next: { tags: ['users'] }
});

// 手动重新验证
import { revalidateTag } from 'next/cache';
revalidateTag('users');
```

### On-Demand ISR（按需 ISR）

```typescript
// app/api/revalidate/route.ts
import { NextRequest, NextResponse } from 'next/server';
import { revalidateTag, revalidatePath } from 'next/cache';

export async function POST(request: NextRequest) {
  const { path, tag } = await request.json();

  if (path) {
    revalidatePath(path);
  } else if (tag) {
    revalidateTag(tag);
  }

  return NextResponse.json({ revalidated: true });
}

// 使用：Webhook 触发重新验证
// POST /api/revalidate { "tag": "users" }
```

```typescript
// app/(dashboard)/users/actions.ts
'use server';

import { prisma } from '@/lib/db';
import { revalidatePath } from 'next/cache';
import { z } from 'zod';

const userSchema = z.object({
  name: z.string().min(2),
  email: z.string().email(),
  role: z.enum(['admin', 'user']).default('user')
});

export async function createUser(formData: FormData) {
  const data = userSchema.parse({
    name: formData.get('name'),
    email: formData.get('email'),
    role: formData.get('role') || 'user'
  });

  const user = await prisma.user.create({ data });

  revalidatePath('/users');
  return { success: true, user };
}

export async function updateUser(id: number, formData: FormData) {
  const data = userSchema.parse({
    name: formData.get('name'),
    email: formData.get('email'),
    role: formData.get('role')
  });

  const user = await prisma.user.update({
    where: { id },
    data
  });

  revalidatePath('/users');
  revalidatePath(`/users/${id}`);
  return { success: true, user };
}

export async function deleteUser(id: number) {
  await prisma.user.delete({ where: { id } });
  revalidatePath('/users');
  return { success: true };
}
```

```typescript
// app/(dashboard)/users/UserList.tsx
'use client';

import { useState } from 'react';
import { deleteUser, updateUser } from './actions';
import type { User } from '@prisma/client';

interface UserListProps {
  initialUsers: User[];
}

export default function UserList({ initialUsers }: UserListProps) {
  const [users, setUsers] = useState(initialUsers);
  const [editingId, setEditingId] = useState<number | null>(null);

  const handleDelete = async (id: number) => {
    if (!confirm('Are you sure?')) return;

    await deleteUser(id);
    setUsers(users.filter(u => u.id !== id));
  };

  const handleUpdate = async (id: number, formData: FormData) => {
    const result = await updateUser(id, formData);

    if (result.success) {
      setUsers(users.map(u => u.id === id ? result.user! : u));
      setEditingId(null);
    }
  };

  return (
    <table className="w-full border-collapse">
      <thead>
        <tr className="border-b">
          <th className="p-3 text-left">Name</th>
          <th className="p-3 text-left">Email</th>
          <th className="p-3 text-left">Role</th>
          <th className="p-3 text-left">Actions</th>
        </tr>
      </thead>
      <tbody>
        {users.map(user => (
          <tr key={user.id} className="border-b">
            {editingId === user.id ? (
              <UserForm
                mode="edit"
                user={user}
                onSubmit={(formData) => handleUpdate(user.id, formData)}
                onCancel={() => setEditingId(null)}
              />
            ) : (
              <>
                <td className="p-3">{user.name}</td>
                <td className="p-3">{user.email}</td>
                <td className="p-3">
                  <span className={`px-2 py-1 rounded ${
                    user.role === 'admin' ? 'bg-red-100' : 'bg-green-100'
                  }`}>
                    {user.role}
                  </span>
                </td>
                <td className="p-3 space-x-2">
                  <button
                    onClick={() => setEditingId(user.id)}
                    className="px-3 py-1 bg-blue-500 text-white rounded"
                  >
                    Edit
                  </button>
                  <button
                    onClick={() => handleDelete(user.id)}
                    className="px-3 py-1 bg-red-500 text-white rounded"
                  >
                    Delete
                  </button>
                </td>
              </>
            )}
          </tr>
        ))}
      </tbody>
    </table>
  );
}
```

## 总结

Next.js App Router 提供了完整的全栈开发能力，通过 Server Components、Server Actions 和强大的缓存系统，你可以构建高性能、类型安全的现代 Web 应用。

### 核心要点

1. **Server Components**：默认使用，减少客户端 JavaScript
2. **Client Components**：只在必要时使用（交互、浏览器 API）
3. **Server Actions**：简化表单处理和数据修改
4. **缓存策略**：合理使用 fetch 缓存、ISR 和 revalidate
5. **性能优化**：并行数据获取、代码分割、图片和字体优化

### 最佳实践

- 优先使用 Server Components，减少客户端 JavaScript
- 使用 Server Actions 处理表单和数据修改
- 利用 Next.js 缓存系统提升性能
- 使用 Suspense 优化加载体验
- 合理代码分割，按需加载

Next.js 的全栈能力仍在快速发展，持续关注新特性和最佳实践，将帮助你构建更好的应用。
