````
## 四、常见问题修复

### 4.1 内存泄漏

```typescript
// ❌ 问题代码：内存泄漏
class UserService {
  private listeners: Function[] = [];

  onUserUpdate(callback: Function) {
    this.listeners.push(callback);  // 永远不清理
  }

  notifyListeners(user: User) {
    this.listeners.forEach(fn => fn(user));
  }
}

// 使用 Claude Code 诊断
> " UserService 有内存泄漏，每次注册监听器都会添加到数组但从不移除"

🔍 内存泄漏分析:

问题识别:
  - listeners 数组持续增长
  - 即使组件卸载，监听器仍然存在
  - 最终导致内存耗尽

影响评估:
  - 长时间运行的应用会崩溃
  - 性能逐渐下降
  - 内存占用持续增长

修复方案:

  1. 添加清理方法
     ```typescript
     class UserService {
       private listeners = new Map<string, Function>();

       onUserUpdate(id: string, callback: Function) {
         this.listeners.set(id, callback);
       }

       offUserUpdate(id: string) {
         this.listeners.delete(id);
       }

       // 或返回取消订阅函数
       onUserUpdate(callback: Function): () => void {
         const id = generateId();
         this.listeners.set(id, callback);
         return () => this.listeners.delete(id);
       }
     }
     ```

  2. 使用 WeakMap
     ```typescript
     class UserService {
       private listeners = new WeakMap<object, Function>();

       onUserUpdate(component: object, callback: Function) {
         this.listeners.set(component, callback);
       }
       // 当组件被垃圾回收时，监听器自动移除
     }
     ```

  3. 使用事件发射器
     ```typescript
     import { EventEmitter } from 'events';

     class UserService extends EventEmitter {
       onUserUpdate(callback: Function) {
         this.on('update', callback);
         return () => this.off('update', callback);
       }
     }
     ```

验证方法:
  1. 使用 Chrome DevTools Memory profiler
  2. 使用 heapdump 比较前后状态
  3. 使用 clinic.js 或 node --inspect
````

### 4.2 并发问题

````typescript
// ❌ 问题：竞态条件
class InventoryService {
  async reserveItem(productId: string, quantity: number) {
    const item = await this.getItem(productId);
    // ❌ 竞态窗口：其他事务可能在此期间修改库存
    if (item.stock < quantity) {
      throw new Error('Insufficient stock');
    }
    item.stock -= quantity;
    await this.saveItem(item);
  }
}

// 诊断
> "InventoryService.reserveItem 有竞态条件，可能导致超卖"

🔍 并发问题分析:

场景重现:
  时刻 T0: 用户 A 和用户 B 同时查看商品（库存 10）
  时刻 T1: 用户 A 请求购买 8 个
  时刻 T2: 用户 B 请求购买 8 个
  时刻 T3: A 检查库存 (10 - 8 = 2 ✅)
  时刻 T4: B 检查库存 (10 - 8 = 2 ✅)  ❌ 应该失败
  时刻 T5: A 扣减库存 (库存 = 2)
  时刻 T6: B 扣减库存 (库存 = -6) ❌ 超卖！

修复方案:

  1. 使用数据库事务
     ```typescript
     async reserveItem(productId: string, quantity: number) {
       await this.transaction(async (tx) => {
         const item = await tx.item.findUnique({ where: { id: productId } });
         if (!item || item.stock < quantity) {
           throw new Error('Insufficient stock');
         }
         await tx.item.update({
           where: { id: productId },
           data: { stock: { decrement: quantity } }
         });
       });
     }
     ```

  2. 使用乐观锁
     ```typescript
     async reserveItem(productId: string, quantity: number) {
       const result = await this.item.updateMany({
         where: {
           id: productId,
           stock: { gte: quantity }  // 条件更新
         },
         data: { stock: { decrement: quantity } }
       });

       if (result.count === 0) {
         throw new Error('Insufficient stock');
       }
     }
     ```

  3. 使用悲观锁
     ```typescript
     async reserveItem(productId: string, quantity: number) {
       return this.transaction(async (tx) => {
         // SELECT ... FOR UPDATE
         const item = await tx.$queryRaw`
           SELECT * FROM "Item" WHERE "id" = ${productId}
           FOR UPDATE
         `;
         // ... 检查和扣减
       });
     }
     ```

推荐: 乐观锁（性能好）或事务（可靠性高）
````

### 4.3 异步错误处理

````typescript
// ❌ 问题：未捕获的异步错误
class OrderService {
  async processOrder(orderId: string) {
    const order = await this.getOrder(orderId);

    // ❌ Promise 错误未捕获
    this.sendConfirmationEmail(order.email)
      .then(() => console.log('Email sent'))
      // 缺少 .catch()

    // ❌ async/await 错误未捕获
    try {
      await this.processPayment(order);
    } catch (error) {
      console.error('Payment failed', error);
      // ❌ 没有重新抛出，继续执行
    }

    await this.updateOrderStatus(orderId, 'completed');
    // ❌ 即使支付失败也标记为完成
  }
}

// 诊断
> "OrderService.processOrder 的异步错误没有被正确处理"

🔍 异步错误分析:

问题识别:
  1. sendConfirmationEmail 的 Promise 没有 .catch()
  2. processPayment 的错误被吞噬
  3. 即使失败也继续执行

影响:
  - 用户可能收不到邮件
  - 支付失败但订单被标记为完成
  - 错误被静默忽略

修复方案:

  1. 正确处理 Promise
     ```typescript
     async processOrder(orderId: string) {
       const order = await this.getOrder(orderId);

       // ✅ 添加错误处理
       await this.sendConfirmationEmail(order.email)
         .catch(error => {
           this.logger.error('Failed to send email', { orderId, error });
           // 降级：重试或进入队列
         });

       // ✅ 重新抛出错误
       try {
         await this.processPayment(order);
       } catch (error) {
         this.logger.error('Payment failed', { orderId, error });
         await this.updateOrderStatus(orderId, 'failed');
         throw error;  // ✅ 重新抛出
       }

       await this.updateOrderStatus(orderId, 'completed');
     }
     ```

  2. 使用 async/await 统一风格
     ```typescript
     async processOrder(orderId: string) {
       const order = await this.getOrder(orderId);

       try {
         await this.sendConfirmationEmail(order.email);
       } catch (error) {
         this.logger.error('Email failed', error);
         // 继续执行，邮件不是关键路径
       }

       try {
         await this.processPayment(order);
       } catch (error) {
         await this.updateOrderStatus(orderId, 'failed');
         throw error;  // 支付失败是致命错误
       }

       await this.updateOrderStatus(orderId, 'completed');
     }
     ```

  3. 添加全局错误处理
     ```typescript
     // app.module.ts
     app.useGlobalFilters(new AllExceptionsFilter());

     // all-exceptions.filter.ts
     @Catch()
     export class AllExceptionsFilter implements ExceptionFilter {
       catch(exception: unknown, host: ArgumentsHost) {
         // 记录所有未捕获的错误
         this.logger.error('Uncaught error', exception);
         // 返回友好响应
       }
     }
     ```
````

````
            url: request.url,
            duration: `${duration}ms`
          });
        }
      })
    );
  }
}

// 数据库查询监控
export class QueryPerformanceMonitor {
  static monitor(queryRunner: QueryRunner, query: string) {
    const startTime = Date.now();
    return () => {
      const duration = Date.now() - startTime;
      if (duration > 100) {
        console.warn(`Slow query (${duration}ms):`, query);
      }
    };
  }
}
````

```typescript
// React 错误边界
class ErrorBoundary extends React.Component<
  { children: React.ReactNode },
  { hasError: boolean; error?: Error }
> {
  constructor(props: any) {
    super(props);
    this.state = { hasError: false };
  }

  static getDerivedStateFromError(error: Error) {
    return { hasError: true, error };
  }

  componentDidCatch(error: Error, errorInfo: React.ErrorInfo) {
    console.error('Error caught:', error, errorInfo);
    // 发送到错误追踪服务
    Sentry.captureException(error);
  }

  render() {
    if (this.state.hasError) {
      return <h1>Something went wrong.</h1>;
    }
    return this.props.children;
  }
}

// NestJS 异常过滤器
@Catch()
export class AllExceptionsFilter implements ExceptionFilter {
  catch(exception: unknown, host: ArgumentsHost) {
    const ctx = host.switchToHttp();
    const response = ctx.getResponse();
    const request = ctx.getRequest();

    const status =
      exception instanceof HttpException
        ? exception.getStatus()
        : 500;

    const message =
      exception instanceof HttpException
        ? exception.getResponse()
        : 'Internal server error';

    // 记录错误
    console.error({
      status,
      message,
      timestamp: new Date().toISOString(),
      path: request.url,
    });

    response.status(status).json({
      statusCode: status,
      timestamp: new Date().toISOString(),
      path: request.url,
      message,
    });
  }
```

## 总结

高效的调试需要：

1. ✅ **系统化流程**：收集信息 → 分析根因 → 制定方案 → 实施修复
2. ✅ **工具支持**：日志、调试器、性能分析器
3. ✅ **错误处理**：完善的错误边界和异常处理
4. ✅ **监控告警**：主动发现问题而非被动响应
5. ✅ **文档记录**：记录问题和解决方案，形成知识库

**调试原则**：

- 先理解问题再动手修复
- 一次只修改一个变量
- 保留复现步骤
- 添加测试防止回归
- 分享经验给团队

掌握这些技巧，你就能快速定位和解决任何问题。
