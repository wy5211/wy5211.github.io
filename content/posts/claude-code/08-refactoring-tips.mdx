# 代码重构实战技巧

代码重构是提升代码质量、可维护性和性能的关键活动。但重构也是高风险的——稍有不慎就可能引入 Bug。Claude Code 的 Plan Mode 和强大代码理解能力，让它成为理想的"重构伙伴"。本文将展示如何安全高效地进行各种类型的代码重构。

## 一、重构前的准备

```typescript
// ✅ After: 提取函数
async function createUser(data: CreateUserDto): Promise<User> {
  validateUserData(data);
  const hashedPassword = await hashPassword(data.password);
  const user = await saveUserToDatabase(data.email, hashedPassword);

// 提取的小函数
function validateUserData(data: CreateUserDto): void {

  if (!data.password) throw new Error("Password required");

async function hashPassword(password: string): Promise<string> {
  return bcrypt.hash(password, 10);
}

async function saveUserToDatabase(
  email: string,
  password: string

  return db.user.create({ email, password });

  await sendEmail({
    to: email,
    subject: "Welcome",
    template: "welcome",
  });
```

### 3.3 策略模式替代条件语句

```typescript
// ❌ Before: 复杂的条件逻辑
function calculateDiscount(order: Order): number {
  if (order.type === "regular") {
    if (order.quantity > 10) {
      return order.total * 0.1;
    } else if (order.quantity > 5) {
      return order.total * 0.05;
    }
  } else if (order.type === "vip") {
    if (order.quantity > 10) {
      return order.total * 0.2;
    } else if (order.quantity > 5) {
      return order.total * 0.15;
    } else {
      return order.total * 0.1;
    }
  } else if (order.type === "wholesale") {
    // ... 更多条件
  }
  return 0;
}

// ✅ After: 策略模式
interface DiscountStrategy {
  calculate(order: Order): number;
}

class RegularDiscountStrategy implements DiscountStrategy {
  calculate(order: Order): number {
    if (order.quantity > 10) return order.total * 0.1;
    if (order.quantity > 5) return order.total * 0.05;
    return 0;
  }
}

class VipDiscountStrategy implements DiscountStrategy {
  calculate(order: Order): number {
    if (order.quantity > 10) return order.total * 0.2;
    if (order.quantity > 5) return order.total * 0.15;
    return order.total * 0.1;
  }
}

class WholesaleDiscountStrategy implements DiscountStrategy {
  calculate(order: Order): number {
    // wholesale 逻辑
  }
}

// 使用策略
class DiscountCalculator {
  private strategies = new Map<string, DiscountStrategy>();

  register(type: string, strategy: DiscountStrategy) {
    this.strategies.set(type, strategy);
  }

  calculate(order: Order): number {
    const strategy = this.strategies.get(order.type);
    if (!strategy) {
      throw new Error(`No strategy for type: ${order.type}`);
    }
    return strategy.calculate(order);
  }
}

// 优势：
// - 每个策略独立
// - 易于添加新策略
// - 符合开闭原则
```

### 3.4 状态机替代状态变量

```typescript
// ❌ Before: 复杂的状态判断
class OrderProcessor {
  async processOrder(order: Order) {
    if (order.status === "pending") {
      // 处理 pending 逻辑
      order.status = "processing";
    } else if (order.status === "processing") {
      // 处理 processing 逻辑
      if (order.paymentStatus === "paid") {
        order.status = "shipped";
      } else if (order.paymentStatus === "failed") {
        order.status = "cancelled";
      }
    } else if (order.status === "shipped") {
      // 处理 shipped 逻辑
      if (order.delivered) {
        order.status = "completed";
      }
    }
    // ... 更多状态判断
  }
}

// ✅ After: 状态机
type OrderState =
  | "pending"
  | "processing"
  | "shipped"
  | "completed"
  | "cancelled";

abstract class OrderState {
  abstract process(order: Order): OrderState;
}

class PendingState extends OrderState {
  process(order: Order): OrderState {
    // pending 逻辑
    return new ProcessingState();
  }
}

class ProcessingState extends OrderState {
  process(order: Order): OrderState {
    if (order.paymentStatus === "paid") {
      return new ShippedState();
    } else if (order.paymentStatus === "failed") {
      return new CancelledState();
    }
    return this; // 保持当前状态
  }
}

class ShippedState extends OrderState {
  process(order: Order): OrderState {
    if (order.delivered) {
      return new CompletedState();
    }
    return this;
  }
}

class CompletedState extends OrderState {
  process(order: Order): OrderState {
    return this; // 终态
  }
}

class CancelledState extends OrderState {
  process(order: Order): OrderState {
    return this; // 终态
  }
}

// 使用状态机
class OrderProcessor {
  private state: OrderState = new PendingState();

  async processOrder(order: Order) {
    this.state = this.state.process(order);
    order.status = this.state.constructor.name;
  }
}
```

## 四、性能优化重构

### 4.1 N+1 查询优化

```typescript
// ❌ Before: N+1 查询
async function getAllUsersWithOrders(): Promise<UserWithOrders[]> {
  const users = await db.user.findMany();

  const result = [];
  for (const user of users) {
    // N+1 问题：每个用户都触发一次查询
    const orders = await db.order.findMany({
      where: { userId: user.id },
    });
    result.push({ ...user, orders });
  }

  return result;
}

// ✅ After: 使用 include
async function getAllUsersWithOrders(): Promise<UserWithOrders[]> {
  const users = await db.user.findMany({
    include: {
      orders: true, // 一次查询加载所有数据
    },
  });

  return users;
}

// 性能提升：
// - Before: 1 + N 次查询 (N = 用户数)
// - After: 1 次查询
// - 100 个用户: 101 次 → 1 次 (100x 提升)
```

### 4.2 缓存优化

```typescript
// ❌ Before: 每次都查询
class UserService {
  async findById(id: string): Promise<User> {
    return db.user.findUnique({ where: { id } });
  }
}

// ✅ After: 添加缓存
class UserService {
  private cache = new Map<string, { user: User; expiry: number }>();
  private readonly CACHE_TTL = 60000; // 1 分钟

  async findById(id: string): Promise<User | null> {
    // 检查缓存
    const cached = this.cache.get(id);
    if (cached && cached.expiry > Date.now()) {
      return cached.user;
    }

    // 查询数据库
    const user = await db.user.findUnique({ where: { id } });
    if (!user) return null;

    // 写入缓存
    this.cache.set(id, {
      user,
      expiry: Date.now() + this.CACHE_TTL,
    });

    return user;
  }

  // 更新后清除缓存
  async update(id: string, data: UpdateUserDto): Promise<User> {
    const user = await db.user.update({
      where: { id },
      data,
    });
    this.cache.delete(id); // 清除缓存
    return user;
  }
}
```

### 4.3 批量操作优化

```typescript
// ❌ Before: 逐个插入
```

## 七、常见陷阱

### 7.1 过度重构

```typescript
// ❌ 过度重构：为 3 行代码创建 5 个函数
function process(a: number, b: number): number {
  return add(multiply(a, 2), divide(b, 3));
}

function add(x: number, y: number): number {
  return x + y;
}

function multiply(x: number, y: number): number {
  return x * y;
}

function divide(x: number, y: number): number {
  return x / y;
}

// ✅ 适度重构：保持简洁
function process(a: number, b: number): number {
  return a * 2 + b / 3;
}
```

### 7.2 忽略边界情况

```typescript
// ❌ 重构时丢失边界处理
function getUser(id: string): User {
  const user = db.user.find(id); // 可能返回 undefined
  return user; // 类型不匹配
}

// ✅ 保留边界处理
function getUser(id: string): User | null {
  const user = db.user.find(id);
  return user ?? null;
}
```

### 7.3 破坏 API

```typescript
// ❌ 破坏性变更
// Before
function getUsers(): User[] {
  return db.user.findMany();
}

// After (破坏性！)
async function getUsers(): Promise<User[]> {
  return await db.user.findMany();
}

// ✅ 向后兼容
// 保留旧 API，标记废弃
/**
 * @deprecated 使用 getUsersAsync 替代
 */
function getUsers(callback: (users: User[]) => void): void {
  getUsersAsync().then(callback);
}

async function getUsersAsync(): Promise<User[]> {
  return db.user.findMany();
}
```

## 总结

成功的代码重构需要：

1. ✅ **充分测试**：重构前必须有测试保护
2. ✅ **小步前进**：每次修改都要能验证
3. ✅ **持续集成**：频繁运行测试套件
4. ✅ **版本控制**：随时可以回滚
5. ✅ **性能监控**：确保重构没有降低性能
6. ✅ **文档更新**：保持文档与代码同步
7. ✅ **团队沟通**：让团队知道重构计划

**重构原则**：

- 不要为了重构而重构
- 保持功能等价
- 一次只做一件事
- 测试驱动

掌握这些技巧，你就能安全高效地改进任何代码库。
