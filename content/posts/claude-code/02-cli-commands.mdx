# CLI 命令与快捷操作

掌握 Claude Code 的 CLI 命令就像掌握 Vim 的快捷键一样，一旦熟练，你的开发效率将呈指数级提升。本文将系统讲解所有核心命令，帮助你从"会用"进阶到"精通"。

## 一、命令体系概览

Claude Code 的命令分为以下几类：

```
Claude Code 命令体系
│
├── 基础命令 (Basic)
│   ├── /help      - 查看帮助
│   ├── /clear     - 清除对话
│   └── /exit      - 退出程序
│
├── 文件操作 (File)
│   ├── /read      - 读取文件
│   ├── /write     - 写入文件
│   ├── /edit      - 编辑文件
│   └── /find      - 查找文件
│
├── 搜索命令 (Search)
│   ├── /grep      - 内容搜索
│   ├── /glob      - 模式匹配
│   └── /search    - 综合搜索
│
├── Git 集成 (Git)
│   ├── /commit    - 提交代码
│   ├── /pr        - PR 管理
│   ├── /review    - 代码审查
│   └── /diff      - 查看变更
│
├── 测试命令 (Test)
│   ├── /test      - 运行测试
│   └── /lint      - 代码检查
│
└── 高级命令 (Advanced)
    ├── /plan      - 规划模式
    ├── /agent     - 启动 Agent
    ├── /remember  - 记忆功能
    └── /skill     - Skill 管理
```

## 二、基础命令详解

### 2.1 帮助系统

```bash
# 查看所有可用命令
/help

# 查看特定命令的帮助
/help read
/help commit

# 查看命令分类
/help --category git
/help --category file
```

**输出示例**：

```
Claude Code v1.2.3 - 命令帮助

基础命令:
  /help [command]     显示帮助信息
  /clear              清除当前对话历史
  /exit               退出 Claude Code

文件操作:
  /read <path>        读取文件内容
  /write <path>       写入文件
  /edit <path>        编辑文件

更多命令: /help --category <category>
```

### 2.2 清除与退出

```bash
# 清除对话历史（保留配置）
/clear

# 清除包括配置在内的所有内容
/clear --all

# 退出程序
/exit

# 退出并保存对话历史
/exit --save
```

**使用场景**：

```bash
# 场景 1: 对话太长，重新开始
> /clear
✅ 对话历史已清除，配置已保留

# 场景 2: 切换到完全不同的任务
> /clear --all
✅ 已重置为初始状态

# 场景 3: 完成工作，退出并保存
> /exit --save
💾 对话已保存到 ~/.claude/history/session_20250321.json
```

## 三、文件操作命令

### 3.1 读取文件

```bash
# 读取单个文件
/read src/utils/api.ts

# 读取多个文件
/read src/utils/api.ts src/utils/auth.ts

# 使用通配符
/read src/**/*.ts

# 读取并指定行数范围
/read src/components/Button.tsx:1-50

# 读取文件的特定部分
/read README.md:introduction
```

**实际应用**：

```bash
# 场景 1: 理解一个模块
> /read src/auth/**/*.ts
✅ 已读取 8 个文件
   - auth.controller.ts (120 行)
   - auth.service.ts (95 行)
   - auth.guard.ts (45 行)
   ...

# 场景 2: 查看配置文件
> /read tsconfig.json package.json
✅ 已读取 2 个文件

# 场景 3: 快速查看文件开头
> /read src/main.ts:1-30
✅ 显示前 30 行
```

### 3.2 写入文件

```bash
# 创建新文件
/write src/utils/helpers.ts
# 然后输入内容，按 Ctrl+D 结束

# 覆盖现有文件
/write src/utils/helpers.ts --force

# 追加内容
/write src/utils/helpers.ts --append

# 从模板创建
/write src/components/Button.tsx --template component
```

**实战示例**：

```typescript
// 创建新的工具函数
> /write src/utils/date.ts

export function formatDate(date: Date): string {
  return new Intl.DateTimeFormat('zh-CN').format(date);
}

export function parseDate(str: string): Date {
  return new Date(str);
}

// Ctrl+D 结束输入
✅ 已创建 src/utils/date.ts (8 行)
```

### 3.3 编辑文件

```bash
# 打开编辑器（使用 $EDITOR）
/edit src/utils/api.ts

# 使用指定编辑器
/edit src/utils/api.ts --editor code
/edit src/utils/api.ts --editor vim

# 编辑特定行
/edit src/utils/api.ts:10-30

# 编辑后自动格式化
/edit src/utils/api.ts --format
```

**编辑工作流**：

```bash
# 1. 读取文件
> /read src/utils/api.ts

# 2. 讨论需要的修改
> "请帮我优化错误处理"

# 3. 应用编辑
> /edit src/utils/api.ts

# 4. 验证修改
> /test src/utils/api.test.ts
```

### 3.4 查找文件

```bash
# 按名称查找
/find "*.test.ts"

# 按路径查找
/find "src/components/**/*.tsx"

# 查找并显示详情
/find "*.ts" --details

# 限制查找深度
/find "*.tsx" --max-depth 2
```

**高级用法**：

```bash
# 查找最近修改的文件
> /find "*.ts" --sort=modified --limit=10

# 查找大文件
> /find "*" --min-size=1MB

# 查找空文件
> /find "*" --empty
```

## 四、搜索命令

### 4.1 Grep 内容搜索

```bash
# 基本搜索
/grep "TODO"

# 在特定目录搜索
/grep "API_KEY" src/

# 使用正则表达式
/grep "interface.*\{.*\}" src/**/*.ts

# 显示行号
/grep "export function" --line-number

# 显示上下文
/grep "error" --context=3
```

**实战场景**：

```bash
# 场景 1: 查找所有 TODO
> /grep "TODO" src/
src/utils/api.ts:45: // TODO: 添加重试逻辑
src/services/auth.ts:78: // TODO: 实现 token 刷新

# 场景 2: 查找特定的导入
> /grep "from 'react'" src/**/*.tsx
✅ 找到 23 个匹配

# 场景 3: 查找调试语句
> /grep "console\.(log|debug)" src/ --line-number
src/components/User.tsx:23: console.log('render')
src/utils/api.ts:56: console.debug(data)
```

### 4.2 Glob 模式匹配

```bash
# 匹配所有 TypeScript 文件
/glob "src/**/*.ts"

# 多模式匹配
/glob "src/**/*.ts" "tests/**/*.spec.ts"

# 排除模式
/glob "src/**/*.ts" --exclude "node_modules/**"

# 统计信息
/glob "**/*.tsx" --stats
```

**实用技巧**：

```bash
# 查找所有组件
> /glob "src/components/**/*.{ts,tsx}"
✅ 找到 45 个文件

# 查找所有测试文件
> /glob "**/*.test.ts" "**/*.spec.ts"
✅ 找到 32 个文件

# 查找配置文件
> /glob "*.{json,yaml,yml,config.*}"
✅ 找到 12 个文件
```

### 4.3 综合搜索

```bash
# 搜索文件名和内容
/search "Button" --type both

# 只搜索文件名
/search "User" --type name

# 只搜索内容
/search "useState" --type content

# 搜索并打开
/search "AuthService" --open
```

## 五、Git 集成命令

### 5.1 智能提交

```bash
# 基本提交
/commit

# 自定义提交信息
/commit "feat: 添加用户认证功能"

# 查看即将提交的变更
/commit --dry-run

# 提交所有更改（包括未跟踪的文件）
/commit --all

# 自动修正提交信息
/commit --amend
```

**智能提交工作流**：

```bash
# 1. 查看状态
> /git status

# 2. 智能提交
> /commit
🤔 正在分析变更...
✨ 生成提交信息:
   feat(auth): implement JWT authentication

   - Add login/register endpoints
   - Implement JWT token generation
   - Add authentication guards
   - Include unit tests

确认提交? [Y/n] Y
✅ 提交成功: abc123f
```

### 5.2 Pull Request 管理

```bash
# 创建 PR
/pr create

# 查看当前分支的 PR
/pr view

# 列出所有 PR
/pr list

# 更新 PR 描述
/pr update --description "新描述"

# 请求审查
/pr review --request @john
```

**PR 创建流程**：

```bash
# 1. 准备功能分支
> /git checkout -b feature/user-auth

# 2. 完成开发
> [开发过程...]

# 3. 创建 PR
> /pr create
📝 正在创建 PR...
   标题: feat: 添加用户认证功能
   描述: [自动生成的描述]
   审查者: @team-leads

确认创建? [Y/n] Y
✅ PR 已创建: https://github.com/user/repo/pull/42
```

### 5.3 代码审查

```bash
# 审查当前 PR
/review

# 审查指定的 PR
/review --pr 42

# 生成审查报告
/review --report review.md

# 只审查特定文件
/review --files src/auth/**/*.ts

# 自动修复问题
/review --fix
```

**审查示例**：

````bash
> /review --pr 42

🔍 正在审查 PR #42...

## 审查结果

### 总体评价：良好

### 主要问题 (2)

1. **src/auth/login.ts:23**
   ```typescript
   const user = await db.user.findUnique({ where: { email } });
````

⚠️ 缺少错误处理。建议：

```typescript
try {
  const user = await db.user.findUnique({ where: { email } });
} catch (error) {
  throw new InternalServerErrorException("Database error");
}
```

2. **src/auth/login.ts:45**
   密码比较使用 === 不安全，应使用 bcrypt.compare()

### 改进建议 (3)

1. 添加请求日志
2. 实现 rate limiting
3. 添加单元测试

### 优点

✅ 代码结构清晰
✅ 使用 DTO 验证
✅ 错误信息友好

````

### 5.4 查看变更

```bash
# 查看未暂存的变更
/diff

# 查看已暂存的变更
/diff --staged

# 查看特定文件的变更
/diff src/utils/api.ts

# 对比两个分支
/diff main...feature-branch

# 查看统计信息
/diff --stat
````

## 六、测试命令

### 6.1 运行测试

```bash
# 运行所有测试
/test

# 运行特定测试文件
/test src/auth/auth.test.ts

# 运行匹配模式的测试
/test --grep "auth"

/test --watch
```

**测试工作流**：

```bash

# 2. 查看失败详情
> /test --reporter verbose

# 3. 只运行失败的测试
> /test --only-failures

> /test src/auth/auth.test.ts

```

### 6.2 代码检查

```bash

# 只检查错误，不检查警告

```

## 七、高级命令

**Plan Mode 详解**将在下一篇文章中深入讲解。

```bash

# 指定任务

# 后台运行
/agent "分析代码质量" --background
```

### 7.3 记忆功能

```bash
# 保存信息到记忆

/memory forget "项目使用 PostgreSQL 16"

# 清空所有记忆
/memory clear

# 导出记忆
/memory export memory.json
```

**使用场景**：

```bash
# 保存项目配置
> /remember "API 基础路径: https://api.example.com/v1"
✅ 已保存到记忆

# 保存团队约定

✅ 已保存到记忆

# 后续对话中自动应用
> "创建一个 API 调用"
✅ 将使用 https://api.example.com/v1 作为基础路径
```

```bash
/skill update --all

# 安装新 Skill
/skill install owner/repo@skill-name
```

## 八、命令组合与工作流

### 8.1 常用命令组合

```bash
# 场景 1: 快速修复 bug
> /read src/auth/login.ts
> /grep "error" src/auth/
> /edit src/auth/login.ts
> /test src/auth/login.test.ts
> /commit "fix: 修复登录错误处理"

# 场景 2: 新功能开发
> /plan "实现用户注册功能"
> /write src/auth/register.ts
> /test src/auth/register.test.ts
> /lint src/auth/register.ts
> /commit "feat: 添加用户注册"

# 场景 3: 代码重构
> /glob "src/**/*.ts"
> /read [选中的文件]
> /agent plan "重构认证模块"
> [执行重构]
> /test
> /review
> /commit "refactor: 重构认证模块"
```

### 8.2 创建自定义命令别名

```bash
# ~/.claude/aliases.json
{
  "review": "/pr review --detailed",
  "test-all": "/test --coverage",
  "quick-commit": "/commit --all --message 'chore: quick commit'",
  "check": "/lint && /test",
  "clean": "/git clean -fd"
}
```

**使用别名**：

```bash
# 使用别名
> review
# 等同于 /pr review --detailed

> check
# 等同于 /lint && /test
```

### 8.3 自动化脚本

```bash
# ~/.claude/scripts/pre-commit.sh
#!/bin/bash
echo "🔍 Running pre-commit checks..."

# 运行 linter
echo "1. Linting..."
claude lint --fix

# 运行测试
echo "2. Testing..."
claude test --coverage --threshold 80

# 检查类型
echo "3. Type checking..."
npm run type-check

echo "✅ All checks passed!"
```

**使用脚本**：

```bash
# Git hook 中使用
# .git/hooks/pre-commit
~/.claude/scripts/pre-commit.sh

# 手动运行
> ~/.claude/scripts/pre-commit.sh
```
