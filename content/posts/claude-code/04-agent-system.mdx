# Agent System 详解

Claude Code 的 Agent System 是一个精巧的架构设计，它将不同的能力分配给专门的 Agent，每个 Agent 都有自己擅长的领域。理解并正确选择 Agent，就像在游戏中选择合适的英雄一样，能让任务完成得更高效、更准确。

## 一、什么是 Agent System

### 1.1 核心概念

**Agent** 是具有特定能力和工具集的 AI 实体：

```
传统 AI 助手:
┌─────────────────────┐
│                     │
│   单一 AI 实体       │
│   - 什么都会一点     │
│   - 什么都不精通     │
│   - 工具集有限       │
│                     │
└─────────────────────┘

Agent System:
┌──────────────────────────────────────────┐
│                                          │
│  General Purpose Agent (通用)           │
│  - 处理复杂的多步骤任务                  │
│  - 完整的工具访问权限                    │
│                                          │
│  Explore Agent (探索)                   │
│  - 快速理解代码库                        │
│  - 搜索和分析能力                        │
│                                          │
│  Plan Agent (规划)                      │
│  - 任务分解和规划                        │
│  - 架构设计能力                          │
│                                          │
└──────────────────────────────────────────┘
```

### 1.2 为什么需要多个 Agent

```typescript
// 场景 1: 快速理解代码库
你的请求: "找出所有 API 端点"

❌ 使用 General Purpose Agent:
   - 可能尝试修改代码
   - 花时间理解不相关的部分
   - 响应慢，token 消耗大

✅ 使用 Explore Agent:
   - 专注于搜索和识别
   - 快速遍历文件
   - 只返回关键信息

// 场景 2: 设计新功能
你的请求: "设计一个插件系统架构"

❌ 使用 Explore Agent:
   - 只会搜索现有代码
   - 不会进行设计思考

✅ 使用 Plan Agent:
   - 分析现有架构
   - 设计新架构方案
   - 考虑扩展性和兼容性

// 场景 3: 实现功能
你的请求: "实现用户注册功能"

❌ 使用 Plan Agent:
   - 只会给出设计方案
   - 不会实际编码

✅ 使用 General Purpose Agent:
   - 设计 + 实现
   - 生成完整代码
   - 编写测试
```

```typescript
// Explore Agent 的工具集
interface ExploreAgentTools {
  // 搜索能力
  grep: GrepTool;              // ✅ 强大的正则搜索
  glob: GlobTool;              // ✅ 高效的文件匹配
  find: FindTool;              // ✅ 快速文件查找

  // 读取能力
  read: ReadTool;              // ✅ 批量读取
  quickRead: QuickReadTool;    // ✅ 只读关键部分

  // 分析能力
  analyze: AnalyzeTool;        // ✅ 代码结构分析
  dependency: DependencyTool;  // ✅ 依赖分析

  // ❌ 不包含
  // write: WriteTool;         // 不修改文件
  // edit: EditTool;           // 不编辑代码
}

// Plan Agent 的工具集
interface PlanAgentTools extends ExploreAgentTools {
  // 额外的规划能力
  architect: ArchitectTool;    // ✅ 架构设计
  design: DesignTool;          // ✅ 详细设计
  estimate: EstimateTool;      // ✅ 工作量估算

  // ❌ 不包含
  // write: WriteTool;         // 不直接编码
}

// General Purpose Agent 的工具集
interface GeneralPurposeAgentTools {
  // 继承所有工具
  ...ExploreAgentTools;
  ...PlanAgentTools;

  // 额外的执行能力
  write: WriteTool;            // ✅ 写文件
  edit: EditTool;              // ✅ 编辑代码
  execute: ExecuteTool;        // ✅ 运行命令
  test: TestTool;              // ✅ 运行测试
}
```

## 二、Explore Agent：代码库探索专家

### 2.1 核心能力

Explore Agent 专注于快速理解和分析代码库：

```
Explore Agent 擅长:

✅ 搜索代码模式
   - 查找特定的函数调用
   - 识别代码约定
   - 发现潜在的 bug 模式

✅ 分析代码结构
   - 理解项目组织方式
   - 识别模块边界
   - 映射依赖关系

✅ 快速遍历
   - 高效处理大量文件
   - 只读取必要的信息
   - 快速返回结果

❌ 不适合:
   - 修改代码
   - 设计新功能
   - 编写实现
```

## 总结

选择合适的 Agent 是高效使用 Claude Code 的关键：

1. ✅ **Explore Agent**：快速理解和搜索代码库
2. ✅ **Plan Agent**：设计架构和规划任务
3. ✅ **General Purpose Agent**：端到端的完整实现

**决策原则**：

- 只读操作 → Explore Agent
- 只需设计 → Plan Agent
- 需要实现 → General Purpose Agent

选择正确的 Agent，不仅能节省时间和 tokens，还能获得更准确的结果。
