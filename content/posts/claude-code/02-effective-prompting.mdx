上篇我们用 Claude Code 搭建了一个 Express API。但如果你给它的需求不够清晰，它生成的代码可能完全不符合你的预期。

比如你说："给我写一个分页接口"。

它可能生成：

```typescript
// 没有参数校验
// 没有错误处理
// 没有类型定义
// 分页逻辑可能有 bug
```

然后你来来回回改十轮，还是没达到你想要的效果。

## 有效 Prompt 的结构

好的 Prompt 有四个要素：

**1. 明确的目标**：要实现什么功能
**2. 约束条件**：技术栈、代码风格、性能要求
**3. 上下文**：相关的文件、现有的实现
**4. 示例**：期望的输出格式、代码风格

### 差的 Prompt

```
帮我写一个用户列表的分页接口
```

问题：

- 不知道用什么框架
- 不知道要返回哪些字段
- 不知道分页逻辑是怎样的
- 不知道是否需要权限控制

### 好的 Prompt

```
在 Express + TypeORM 项目中创建用户列表分页接口：

要求：
- 路径：GET /api/users?page=1&limit=10&keyword=xxx
- 返回字段：id, username, email, createdAt, total
- 支持按 username 或 email 模糊搜索
- 使用 TypeORM 的 repository 模式
- 添加错误处理和类型定义

参考 src/routes/post.ts 中的现有分页实现
```

这样的 Prompt 让 Claude Code 第一次就给出正确的结果。

## @ 引用：精确提供上下文

Claude Code 支持用 `@` 引用文件和目录：

```bash
# 引用单个文件
参考 @src/routes/user.ts 中的用户定义

# 引用目录
分析 @src/modules/auth/ 下的所有认证相关代码

# 引用搜索结果
@src/**/*.ts 中使用 TypeORM 的所有查询
```

### 什么时候该用 @

**用 @ 的场景**：

- 有现有代码需要参考
- 需要理解某个模块的实现
- 要保持代码风格一致

**不用 @ 的场景**：

- 创建全新的文件
- 简单的独立功能

## 分步指令 vs 一步到位

### 一步到位（适合简单任务）

```
创建一个用户注册接口，包含邮箱验证和密码加密
```

### 分步指令（适合复杂任务）

```
第一步：创建用户注册的基础接口（email, password）
第二步：添加邮箱格式验证
第三步：添加密码 bcrypt 加密
第四步：添加 JWT token 生成
```

判断标准：**如果你自己写都需要分步，那让 Claude Code 也分步**。

## 常见沟通陷阱

### 陷阱 1：过于模糊

```
❌ 优化一下这个函数
✅ 这个函数在数据量大时很慢（>1秒），用缓存优化
```

### 陷阱 2：过于冗长

```
❌ 我想要你帮我写一个接口，这个接口是用来获取用户信息的，但是要注意不要返回密码，而且要检查用户是否有权限，如果没权限就返回 403，还有要处理错误情况...

✅ 创建 GET /api/users/:id 接口，返回用户信息（排除密码），验证 JWT token，未认证返回 401
```

### 陷阱 3：自相矛盾

```
❌ 用异步方式实现，但要同步返回结果
✅ 用 async/await 实现，返回 Promise<User>
```

## 让它"问你"而不是"猜你"

Claude Code 可以用 `AskUserQuestion` 主动向你询问：

```
在实现注册接口前，我需要确认：
1. 邮箱验证是用正则还是第三方服务？
2. 密码强度要求是什么？
3. 是否需要发送欢迎邮件？
```

你可以在 CLAUDE.md 中配置：

```markdown
## 交互偏好

遇到不确定的地方，主动询问我，不要自作主张。
```

## 代码规范的传达

与其每次对话都重复规范，不如写进 CLAUDE.md：

```markdown
## 代码规范

### 命名约定

- 类名：PascalCase (UserService)
- 函数名：camelCase (getUserById)
- 常量：UPPER_SNAKE_CASE (API_BASE_URL)

### 错误处理

- 统一返回 { code, message, data }
- 使用 NestJS 的 HttpException
- 记录错误日志
```

这样 Claude Code 会自动遵循这些规范，不需要你每次都强调。

到这里，你知道怎么和 Claude Code 有效沟通了。但如果你让它"重构认证模块"，它可能二话不说就开始改代码，改了 30 个文件结果方向全错了。复杂任务需要先规划。下一篇，我们讲 Plan Mode。
