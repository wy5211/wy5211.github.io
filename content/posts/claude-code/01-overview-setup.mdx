# Claude Code 概览与环境配置

Claude Code 是 Anthropic 官方推出的 AI 编程助手，它不仅是代码生成工具，更是一个能理解复杂项目架构、参与设计决策、自动化开发工作流的智能伙伴。对于已经有 AI 编程工具使用经验的开发者来说，Claude Code 提供了更强大的上下文理解能力和更灵活的扩展机制。

本文将带你全面了解 Claude Code 的核心优势，并完成开发环境的最佳配置。

## 一、为什么选择 Claude Code

在 AI 编程工具竞争激烈的今天，Claude Code 凭借几个核心优势脱颖而出。

### 1.1 超长上下文窗口

**200K tokens 的上下文能力**意味着什么？

```
✅ 可以理解整个中小型项目的代码库
✅ 不会因为对话过长而"忘记"之前的上下文
✅ 一次性分析多个文件的依赖关系
✅ 处理复杂的跨文件重构任务
```

**实际场景对比**：

```bash
# 其他 AI 工具
❌ "对不起，上下文太长，请重新开始"
❌ "我无法记住 10 轮对话之前的内容"

# Claude Code
✅ 轻松处理 50+ 文件的代码库
✅ 保持长达数小时的连续对话上下文
✅ 理解项目的历史演进和设计决策
```

### 1.2 强大的规划能力

**Plan Mode** 是 Claude Code 的杀手锏功能。与其他 AI 工具直接动手修改代码不同，Claude Code 会：

```
1. 深入探索你的代码库
2. 理解现有架构和依赖关系
3. 制定详细的实施计划
4. 识别潜在的风险点
5. 与你确认后再执行
```

**场景示例**：

```
用户请求："重构认证系统，支持 OAuth 2.0"

传统 AI 工具：
❌ 直接开始写代码，可能破坏现有功能
❌ 不理解你复杂的业务逻辑
❌ 生成与现有架构冲突的代码

Claude Code (Plan Mode):
✅ 先探索现有的认证实现
✅ 分析所有依赖认证的模块
✅ 设计向后兼容的迁移方案
✅ 分步骤实施，每步都可以验证
```

### 1.3 可扩展的 Skills 生态系统

**Skills** 让你可以创建可重用的 AI 能力包：

```bash
# 安装社区贡献的 Skills
npx skills add owner/repo@skill-name

# 查找特定功能的 Skills
npx skills find react
npx skills find testing

# 创建自己的 Skills
npx skills init my-custom-skill
```

**实际应用**：

```
场景：团队内部有特殊的代码规范要求

解决方案：创建团队专属 Skill
- 定义代码审查规则
- 封装项目特定的工具链
- 统一团队的工作流程
- 新人快速上手指南
```

### 1.4 优秀的代码理解能力

Claude Code 不仅会"写代码"，更重要的是"理解代码"：

```typescript
// 你展示这段代码
class UserService {
  constructor(private repo: UserRepository) {}

  async createUser(dto: CreateUserDto) {
    const user = await this.repo.findByEmail(dto.email);
    if (user) throw new ConflictException();
    return this.repo.save(dto);
  }
}

// Claude Code 的理解
✅ 识别这是 NestJS 的服务模式
✅ 理解依赖注入的使用
✅ 知道这是检查邮箱唯一性的逻辑
✅ 能提出优化建议（如添加日志、事务等）
```

## 二、安装与基础配置

### 2.1 系统要求

```bash
# 基础环境
Node.js >= 18.x
npm >= 9.x 或 pnpm >= 8.x

# 推荐环境
Node.js = 20.x LTS
npm = 10.x
```

### 2.2 安装 Claude Code

```bash
# 使用 npm 安装（全局）
npm install -g @anthropic-ai/claude-code

# 使用 pnpm 安装（更快）
pnpm add -g @anthropic-ai/claude-code

# 验证安装
claude --version
```

**安装位置**：

```bash
# 全局安装路径
~/.npm-global/lib/node_modules/@anthropic-ai/claude-code

# 配置文件目录
~/.claude/
├── config.json          # 主配置文件
├── skills/              # 全局 Skills
├── memory/              # 持久化记忆
└── logs/                # 日志文件
```

### 2.3 API 密钥配置

Claude Code 需要 Anthropic API 密钥：

```bash
# 方式 1: 环境变量（推荐）
export ANTHROPIC_API_KEY="your-api-key-here"

# 添加到 ~/.zshrc 或 ~/.bashrc
echo 'export ANTHROPIC_API_KEY="your-api-key-here"' >> ~/.zshrc
source ~/.zshrc

# 方式 2: 配置文件
claude config set api_key "your-api-key-here"

# 验证配置
claude auth test
```

**获取 API 密钥**：

1. 访问 https://console.anthropic.com/
2. 注册/登录账号
3. 进入 API Keys 页面
4. 创建新的 API 密钥
5. 复制密钥（只显示一次）

### 2.4 初始化项目配置

在项目根目录初始化 Claude Code：

```bash
cd /path/to/your/project

# 初始化配置
claude init

# 生成默认配置文件
```

**生成的文件结构**：

```
your-project/
```

## 三、CLAUDE.md：项目的"第二份文档"

`CLAUDE.md` 是 Claude Code 理解你项目的关键文件，相当于给 AI 的"项目说明书"。

### 3.1 CLAUDE.md 的作用

```markdown
没有 CLAUDE.md:
❌ Claude 需要反复询问项目规范
❌ 生成的代码风格不一致
❌ 不理解项目的特殊约定
❌ 无法继承团队知识

有 CLAUDE.md:
✅ 自动遵循项目规范
✅ 代码风格保持一致
✅ 理解业务逻辑和架构决策
✅ 新人快速融入项目
```

### 3.2 CLAUDE.md 最佳实践

以下是完整的 CLAUDE.md 模板：

```markdown
# 项目名称

## 项目概述

[简要描述项目的目标和主要功能]

## 技术栈

- **框架**: Next.js 14 (App Router)
- **语言**: TypeScript 5.x
- **样式**: Tailwind CSS
- **状态管理**: Zustand
- **数据库**: PostgreSQL + Prisma
- **测试**: Jest + Testing Library

## 代码规范

### 命名约定

- 组件：PascalCase (UserProfile.tsx)
- 工具函数：camelCase (formatDate.ts)
- 常量：UPPER_SNAKE_CASE (API_BASE_URL)
- 类型/接口：PascalCase (UserData)

### 文件组织
```

src/
├── components/ # 可复用组件
├── app/ # App Router 页面
├── lib/ # 工具函数
├── hooks/ # 自定义 Hooks
├── types/ # TypeScript 类型
└── styles/ # 全局样式

````

### 编码风格

```typescript
// ✅ 好的实践
export async function getUser(id: string): Promise<User> {
  const user = await db.user.findUnique({ where: { id } });
  if (!user) {
    throw new NotFoundException('User not found');
  }
  return user;
}

// ❌ 避免
async function getUser(id: any) {
  return await db.user.findUnique({ where: { id } });
}
````

## 架构原则

1. **组件设计**
   - 优先使用 Server Components
   - 客户端组件标记 `'use client'`
   - 保持组件单一职责

2. **数据获取**
   - 使用 Server Actions 处理表单
   - 使用 React Query 缓存客户端数据
   - 避免在组件中直接调用数据库

3. **错误处理**
   - 使用 error.tsx 处理路由错误
   - 统一使用 ErrorBoundary
   - 所有 API 调用都要处理错误

## 四、环境变量与权限设置

### 4.1 权限模式配置

Claude Code 支持不同的权限模式，平衡安全性与便利性：

```bash
# ~/.claude/config.json
{
  "permissionMode": "auto",      // 自动批准常见操作
  "dangerousCommands": [         // 需要确认的命令
    "rm -rf",
    "git clean -fd",
    "npm install"
  ],
  "safeCommands": [             // 自动执行的命令
    "git status",
    "git diff",
    "ls -la"
  ]
}
```

**权限模式说明**：

| 模式     | 说明             | 适用场景           |
| -------- | ---------------- | ------------------ |
| `auto`   | 自动批准安全操作 | 个人项目，快速开发 |
| `prompt` | 每次操作都询问   | 生产环境，谨慎操作 |
| `strict` | 只允许只读操作   | 代码审查，学习模式 |

### 4.2 环境变量最佳实践

```bash
# ~/.zshrc 或 ~/.bashrc

# Claude Code 配置
export ANTHROPIC_API_KEY="sk-ant-xxx"
export CLAUDE_DEFAULT_MODEL="claude-sonnet-4-6"  # 默认模型
export CLAUDE_TIMEOUT=120000                      # 请求超时 (ms)
export CLAUDE_MAX_TOKENS=200000                   # 最大 tokens

# 项目特定配置（在项目 .envrc 中）
export CLAUDE_PROJECT_TYPE="nextjs"
export CLAUDE_TEST_FRAMEWORK="jest"
```

````
## 七、常见问题解决

### Q1: API 配额不足怎么办？

```bash
# 查看当前使用情况
claude usage

# 优化建议：
# 1. 使用更小的模型 (claude-haiku-4-5)
export CLAUDE_DEFAULT_MODEL="claude-haiku-4-5"

# 2. 减少 token 使用
# - 只包含相关文件
# - 定期清理对话历史
# - 使用 /clear 重置上下文
````

### Q2: 响应速度慢？

```bash
# 检查网络连接
claude ping

# 优化建议：
# 1. 使用更快的 API endpoint
export CLAUDE_API_BASE_URL="https://api.anthropic.com"

# 2. 减少上下文大小
# - 避免一次性读取太多文件
# - 使用 Glob 模式精确匹配
```

### Q3: 无法理解项目？

```
解决方案：
1. 完善 CLAUDE.md 文件
2. 使用 /remember 记录关键信息
3. 提供 @文件引用 代替复制粘贴
4. 使用 Plan Mode 让 Claude 先探索
```

## 八、性能优化建议

### 8.1 减少响应延迟

```bash
# 使用更快的模型（适合简单任务）
claude chat --model claude-haiku-4-5

# 限制每次请求的 token 数
export CLAUDE_MAX_TOKENS=100000

# 启用缓存
export CLAUDE_CACHE_ENABLED=true
```

### 8.2 优化 Token 使用

```typescript
// ❌ 不好：提供过多上下文
"这是我的整个项目代码：[10000 行代码]";

// ✅ 好：精确引用相关文件
"参考 @src/auth/login.ts 中的登录逻辑";

// ✅ 更好：使用 Glob 模式
"分析 @src/auth/**/*.ts 中的所有认证相关代码";
```

### 8.3 批量操作优化

```bash
# ❌ 慢：逐个文件处理
for file in src/**/*.ts; do
  claude lint $file
done

# ✅ 快：批量处理
claude lint src/**/*.ts --parallel
```
