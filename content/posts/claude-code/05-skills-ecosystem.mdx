# Skills 生态系统

Skills 是 Claude Code 的插件系统，它让 AI 助手的能力可以被无限扩展。想象一下，如果每个开发者都能将自己领域知识封装成 Skill，那么整个社区共享的就是一个庞大的专业能力库。本文将带你深入了解这个生态系统，掌握利用 Skills 提升效率的方法。

## 一、什么是 Skills

### 1.1 核心概念

**Skill** 是一个包含指令和知识的文件，用于指导 Claude Code 完成特定任务：

```typescript
// 传统方式：每次都要解释
你: "帮我写一个 React 组件，要用 TypeScript，
     使用函数式组件，添加 props 类型，
     使用 hooks，处理加载状态..."

你: "再写一个，要求一样..."

你: "又来一个..."

// 使用 Skills：一次性定义，无限复用
你: 创建 react-component Skill
    定义: 所有 React 组件规范

你: "用 react-component skill 创建一个按钮"
你: "用 react-component skill 创建一个表单"
你: "用 react-component skill 创建一个卡片"

// 结果：规范统一，效率提升 10x
```

### 1.2 Skill 的结构

一个完整的 Skill 包含以下部分：

```markdown
---
name: skill-name
description: 简短描述
---

# Skill 名称

## 何时使用

描述使用场景

## 工作流程

1. 步骤 1
2. 步骤 2
   ...

## 指令

具体的执行指令

## 示例

用户输入 → Agent 应该做什么

## 最佳实践

使用建议和注意事项
```

### 1.3 Skills vs 其他扩展机制

```
Claude Code 扩展方式对比:

1. 内置命令
   - 固定功能
   - 无法自定义
   ❌ 不灵活

2. 配置文件 (CLAUDE.md)
   - 项目级规范
   - 通用指导
   ⚠️ 缺乏针对性

3. Skills
   - 可定制
   - 可复用
   - 可分享
   ✅ 最佳选择
```

## 三、常用 Skills 介绍

### 3.1 commit - 智能提交助手

**功能**：自动生成符合规范的提交信息

```bash
# 安装
npx skills add anthropic/skills@commit -g

# 使用
/commit

# 自动分析变更
🤔 分析变更...
   - 修改了 3 个文件
   - 添加了新功能
   - 修复了 bug

✨ 生成提交信息:
   feat(auth): add OAuth2.0 support

   - Implement authorization code flow
   - Add token management
   - Include error handling

   Co-Authored-By: Claude Sonnet 4.6

确认提交? [Y/n]
```

**支持的规范**：

- Conventional Commits
- Angular Convention
- Custom Templates

### 3.2 review-pr - PR 审查助手

**功能**：自动审查 Pull Requests

````bash
# 安装
npx skills add anthropic/skills@review-pr -g

# 使用
/review

🔍 审查 PR #42...

## 审查报告

### 总体评价：良好 ⭐⭐⭐⭐

### 主要问题 (1)

1. **安全性：密码明文传输**
   文件: src/auth/login.ts:23
   ```typescript
   const response = await fetch('/api/login', {
     body: JSON.stringify({ password })  // ❌ 应该用 HTTPS
   })
````

建议: 使用 HTTPS 并考虑添加加密

### 改进建议 (3)

1. 性能：可以添加请求缓存
2. 测试：缺少边界情况测试
3. 文档：API 变更需要更新文档

### 优点

✅ 代码结构清晰
✅ 错误处理完善
✅ 类型定义准确


## 四、发现和安装 Skills

### 4.1 搜索 Skills

```bash
# 方式 1: 使用 CLI
npx skills find <keyword>

# 示例
npx skills find react
npx skills find testing
npx skills find docker

# 方式 2: 浏览网站
open https://skills.sh

# 方式 3: 使用 find-skills Skill
/skill find-skills
```

### 4.2 安装 Skills

```bash
# 全局安装（所有项目可用）
npx skills add <owner/repo>@skill-name -g

# 本地安装（仅当前项目）
npx skills add <owner/repo>@skill-name

# 示例
npx skills add anthropic/skills@commit -g
npx skills add vercel-labs/agent-skills@react-best-practices -g

# 从 URL 安装
npx skills add https://example.com/skill/SKILL.md
```

### 4.3 管理已安装的 Skills

```bash
# 列出已安装的 Skills
npx skills list

# 查看 Skill 详情
npx skills info commit

# 启用/禁用 Skill
npx skills enable commit
npx skills disable commit

# 更新 Skills
npx skills update <skill-name>

# 更新所有 Skills
npx skills update --all

# 卸载 Skill
npx skills remove commit
```

## 八、故障排除

### 8.1 常见问题

**问题 1: Skill 无法加载**

```bash
# 检查 Skill 语法
npx skills validate skill-name

# 查看详细日志
npx skills --verbose load skill-name

# 重新安装
npx skills remove skill-name
npx skills add skill-name
```

**问题 2: Skill 行为不符合预期**

```bash
# 查看 Skill 定义
npx skills info skill-name --full

# 检查版本
npx skills list --versions

# 更新到最新版
npx skills update skill-name
```

**问题 3: Skills 冲突**

```bash
# 查看冲突
npx skills conflicts

# 禁用冲突的 Skill
npx skills disable conflicting-skill

# 或设置优先级
npx skills priority preferred-skill 1
```

### 8.2 调试 Skills

```bash
# 启用调试模式
export CLAUDE_DEBUG=1

# 测试 Skill
npx skills test skill-name

# 查看执行日志
~/.claude/logs/skills.log
```
