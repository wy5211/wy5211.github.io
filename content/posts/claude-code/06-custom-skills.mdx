# 自定义 Skills 开发

自定义 Skills 是将专业知识、团队规范和最佳实践封装成可复用 AI 能力的最佳方式。通过开发 Skills，你可以让 Claude Code 精通特定的技术栈、遵循团队规范、处理特定场景，成为真正的团队专家。

## 一、何时需要创建 Skill

### 1.1 创建 Skill 的信号

```
✅ 应该创建 Skill 的情况:

1. 重复性任务
   - 每次都要解释相同的规范
   - 频繁执行相似的代码生成
   - 团队成员都问同样的问题

2. 复杂的专业知识
   - 特定领域的专业流程
   - 复杂的架构模式
   - 需要多步骤的操作

3. 团队规范
   - 代码风格要求
   - 审查标准
   - 部署流程

4. 特定场景
   - 公司特定的工具链
   - 特殊的配置要求
   - 定制的工作流

❌ 不需要创建 Skill 的情况:

1. 一次性任务
2. 简单的代码片段
3. 已经有现成的 Skill
4. 变化频繁的规范
```

### 1.2 Skill vs 配置文件

```markdown
# CLAUDE.md (项目配置)

- 通用的项目规范
- 架构说明
- 技术栈信息
- 适用于整个项目

# Skill (特定能力)

- 专门的流程指导
- 特定任务的步骤
- 可选的插件
- 适用于特定场景

# 何时使用哪个?

使用 CLAUDE.md:
✅ 项目整体规范
✅ 技术栈说明
✅ 目录结构
✅ 命名约定

使用 Skills:
✅ 创建 React 组件
✅ 运行测试套件
✅ 生成 API 文档
✅ 部署到生产环境
```

````markdown

name: react-component
description: 生成符合最佳实践的 React 组件

# React 组件生成器

## 何时使用

需要创建 React 组件时，包括：

- 页面组件
- UI 组件
- 布局组件
- 容器组件

## 前置条件

- [ ] 项目已安装 React 和 TypeScript
- [ ] 使用函数式组件 + Hooks
- [ ] 已配置 ESLint 和 Prettier

## 工作流程

### 步骤 1：询问组件信息

询问用户以下信息：

1. **组件名称**（PascalCase）
2. **组件类型**
   - Page (页面组件)
   - UI (UI 组件)
   - Layout (布局组件)
   - Container (容器组件)

3. **是否需要 props**（是/否）
4. **是否使用 Context**（是/否）
5. **是否需要样式文件**（是/否）

### 步骤 2：生成组件代码

根据信息生成组件：

```typescript
// 组件模板
import React from 'react';

interface {{ComponentName}}Props {
  // props 定义
}

export const {{ComponentName}}: React.FC<{{ComponentName}}Props> = ({
  // props
}) => {
  return (
    <div>
      {/* 组件内容 */}
    </div>
  );
};

export default {{ComponentName}};
```
````

### 步骤 3：生成测试文件

```typescript
import { render, screen } from '@testing-library/react';
import {{ComponentName}} from './{{ComponentName}}';

describe('{{ComponentName}}', () => {
  it('should render successfully', () => {
    const { baseElement } = render(<{{ComponentName}} />);
    expect(baseElement).toBeTruthy();
  });
});
```

### 步骤 4：生成样式文件（如果需要）

```css
.{{componentName}} {
  /* 样式定义 */
}
```

### 步骤 5：创建目录结构

```
src/components/{{ComponentName}}/
├── {{ComponentName}}.tsx
├── {{ComponentName}}.test.tsx
├── {{ComponentName}}.module.css (可选)
└── index.ts (导出)
```

## 代码规范

### TypeScript

- 使用 strict 模式
- 所有 props 必须定义接口
- 使用类型别名，不要用 interface（除非需要 extends）

### Hooks

- 优先使用函数式组件
- 遵循 Hooks 规则
- 自定义 Hooks 以 "use" 开头

### 样式

- 优先使用 CSS Modules
- 避免内联样式
- 使用 Tailwind CSS 类名

### 性能

- 使用 React.memo 优化重渲染
- 大列表使用虚拟化
- 懒加载路由组件

## 常见问题

### Q: 如何处理组件状态？

A: 使用 useState 管理简单状态，复杂状态考虑 useReducer 或状态管理库。

### Q: 何时使用 useMemo 和 useCallback?

A:

- **useMemo**: 缓存昂贵的计算结果
- **useCallback**: 缓存传递给子组件的函数

### Q: 如何处理表单？

A: 推荐使用 React Hook Form 或 Formik。

### 5.2 示例：团队代码规范 Skill

````markdown
---
name: team-standards
description: 团队编码规范和最佳实践
---

# 团队编码规范

## 通用规范

### 命名约定

- **变量和函数**: camelCase
  ```typescript
  const userName = "John";
  function getUserData() {}
  ```
````

- **类和接口**: PascalCase

  ```typescript
  class UserService {}
  interface UserData {}
  ```

- **常量**: UPPER_SNAKE_CASE

  ```typescript
  const MAX_RETRY_COUNT = 3;
  ```

- **文件名**: kebab-case
  ```
  user-service.ts
  api-client.ts
  ```

### 注释规范

```typescript
/**
 * 用户服务类
 * 负责用户相关的业务逻辑
 */
export class UserService {
  /**
   * 根据 ID 获取用户
   * @param id - 用户 ID
   * @returns 用户对象或 null
   * @throws {NotFoundException} 用户不存在
   */
  async findById(id: string): Promise<User | null> {
    // 实现
  }
}
```

### 错误处理

```typescript
// ✅ 好的做法
try {
  const user = await this.userService.findById(id);
  if (!user) {
    throw new NotFoundException('User not found');
  }
  return user;
} catch (error) {
  this.logger.error('Failed to find user', { id, error });
  throw error;
}

// ❌ 避免
async findById(id: string) {
  return db.user.find(id);  // 没有错误处理
}
```

### TypeScript 规范

```typescript
// ✅ 使用类型别名
type UserId = string;
type UserStatus = 'active' | 'inactive' | 'suspended';

// ✅ 明确的返回类型
async getUser(id: UserId): Promise<User> {
  // 实现
}

// ❌ 避免
async getUser(id: any) {
  // 实现
}
```

## 前端特定规范

### React 组件

```typescript
// ✅ 好的组件
interface ButtonProps {

export const Button: React.FC<ButtonProps> = ({
```

### Hooks 使用

```typescript
// ✅ 自定义 Hook
function useUserData(userId: string) {
  const [user, setUser] = useState<User | null>(null);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    fetchUser(userId)
      .then(setUser)
      .finally(() => {
        setLoading(false);
      });
  }, [userId]);

  return { user, loading };
}

// ✅ 使用依赖数组
useEffect(() => {
  // effect
}, [dependency1, dependency2]); // 明确列出依赖

// ❌ 缺少依赖

  // effect
}, []); // 如果使用了外部变量，应该添加到依赖数组
```

## 测试规范

### 单元测试

```typescript
describe("UserService", () => {
  let service: UserService;
  let repository: MockProxy<UserRepository>;

  beforeEach(() => {
    repository = mock<UserRepository>();
    service = new UserService(repository);
  });

  describe("findById", () => {
    it("should return user when found", async () => {
```
