你用 GitHub Copilot 写代码，它很擅长自动补全——你写 `function getU`，它补全 `getUserById`。你用 Cursor，它能在编辑器里对话改代码。但当你遇到一个复杂任务——"重构认证模块支持 OAuth 2.0"，它们就无能为力了。

Claude Code 不是更好的代码补全工具，而是能理解整个项目、规划复杂任务、自动执行的开发环境。

## Claude Code 的独特价值

### 200K 上下文窗口

大多数 AI 工具的上下文窗口很小，对话几轮就"忘记"前面的内容。Claude Code 有 200K tokens 的上下文能力——可以理解整个中小型项目的代码库。

这对实际开发意味着：

- 接手新项目时，不需要一个个文件贴给它
- 长对话中它能记住你 1 小时前说的约束
- 能分析多文件之间的依赖关系

### Plan Mode：先规划再执行

这是 Claude Code 的杀手锏。你让其他 AI "重构认证模块"，它二话不说就开始改代码，改了 30 个文件结果方向全错了。

Plan Mode 的流程：

1. 深入探索你的代码库
2. 理解现有架构和依赖
3. 制定详细的实施计划
4. 识别潜在风险
5. 你确认后再执行

### Skills：封装团队规范

你的团队有代码规范、Git 工作流、测试要求。其他 AI 工具每次都要你重复这些规范。Claude Code 的 Skills 让你封装这些规则，自动应用。

## 安装与配置

```bash
# 安装
npm install -g @anthropic-ai/claude-code

# 配置 API Key
export ANTHROPIC_API_KEY="your-key-here"

# 验证
claude --version
```

## CLAUDE.md：项目的"说明书"

`CLAUDE.md` 是 Claude Code 理解你项目的关键文件：

```markdown
# 项目概述

在线教育平台的 API 服务

# 技术栈

- NestJS 10.x
- TypeORM
- PostgreSQL

# 代码规范

- 类名用 PascalCase
- 函数名用 camelCase
- 所有 API 要有错误处理

# 常见任务

## 添加新接口

1. 在 src/modules/{module}/ 下创建 controller
2. 添加对应的 service
3. 编写单元测试
```

有了 CLAUDE.md，Claude Code 会自动遵循这些规范，不需要你每次都重复。

## 第一个项目：用 Claude Code 搭建 Express API

让我们用 Claude Code 从零搭建一个 Express API，体验完整工作流：

```bash
# 创建项目目录
mkdir my-express-api && cd my-express-api

# 初始化
claude init
```

Claude Code 会生成基础配置。然后：

```bash
# 启动对话模式
claude chat

# 给它一个需求
> 创建一个 Express API，包含用户注册、登录、获取用户信息三个接口
> 使用 TypeScript
> 密码要 bcrypt 加密
> 用 JWT 做认证
```

Claude Code 会：

1. 创建 `package.json`
2. 创建 `src/index.ts`（入口文件）
3. 创建 `src/routes/user.ts`（路由）
4. 创建 `src/services/auth.ts`（认证服务）
5. 创建 `src/middleware/auth.ts`（中间件）

你可以实时看到它创建文件，有问题随时打断。

## 什么时候用 Claude Code，什么时候 IDE 插件就够了

| 任务         | Copilot/Cursor | Claude Code    |
| ------------ | -------------- | -------------- |
| 自动补全函数 | ✅ 更快        | ❌             |
| 改一个小 bug | ✅ 方便        | ✅ 也可以      |
| 重构整个模块 | ❌ 做不到      | ✅ 擅长        |
| 理解新项目   | ❌ 上下文小    | ✅ 200K 上下文 |
| 多文件修改   | ❌ 容易出错    | ✅ 自动规划    |

简单来说：**补全用 Copilot，复杂任务用 Claude Code**。
