上篇我们用 Plan Mode 安全地规划了复杂任务。但规划阶段需要探索 86 个文件、理解架构、分析依赖——这些手动做太慢了。

Claude Code 的 Agent System 和 Skills 能帮你自动化这些工作。

## Agent System：让 AI 帮你探索代码

### Explore Agent：快速搜索

```bash
# 在对话中
> 用 Explore Agent 分析 @src/modules/course/ 的代码结构
```

Claude Code 会：

```
🔍 正在探索课程模块...
✓ 分析了 15 个文件
✓ 找到了 Controller、Service、Entity、DTO
✓ 绘制了模块依赖图
✓ 识别了设计模式（Repository 模式）

📊 架构概览：
course.controller.ts → course.service.ts → course.repository.ts
                              ↓
                        course.entity.ts
```

### 什么时候用 Agent

**适合**：

- 搜索代码库（"哪些文件用了这个函数？"）
- 分析模块结构（"这个模块依赖哪些其他模块？"）
- 理解不熟悉的代码（"这个项目是怎么组织的？"）

**不适合**：

- 你已经很熟悉的代码
- 简单的单文件任务

## Skills：封装重复性工作流

### 什么是 Skills

Skills 是可复用的指令模板。比如你每次写博客都要手动排版，这可以封装成一个 Skill：

```markdown
---
name: blog-formatter
description: 格式化博客文章
---

# 博客格式化

## 何时使用

当你写完博客文章需要格式化时

## 格式规范

1. 标题层级：一标题用 ##，二标题用 ###
2. 代码块：指定语言（typescript、bash 等）
3. 表格：使用 Markdown 表格格式
4. 图片：添加 alt 文本
```

### 使用社区 Skills

```bash
# 查找可用的 Skills
npx skills find react
npx skills find testing
npx skills find commit

# 安装 Skill
npx skills add owner/repo@skill-name
```

### 创建自定义 Skill

```bash
# 初始化
npx skills init code-reviewer

# 生成的文件结构
~/.claude/skills/code-reviewer/
├── SKILL.md      # Skill 定义
└── examples/     # 示例文件
```

### 实战：代码审查 Skill

```markdown
---
name: code-reviewer
description: 代码审查助手
---

# 代码审查助手

## 何时使用

需要审查 PR 或代码变更时

## 审查要点

1. 代码风格是否符合项目规范
2. 是否有潜在的 bug
3. 性能是否有改进空间
4. 测试是否充分

## 输出格式

## 总体评价

- 代码质量：[优秀/良好/需改进]
- 主要问题：
  1. [问题1]
  2. [问题2]

## 改进建议

1. [建议1]
2. [建议2]
```

使用：

```
> 用 code-reviewer 审查最近的变更
```

## Agent vs Skill 的区别

| 特性   | Agent              | Skill              |
| ------ | ------------------ | ------------------ |
| 用途   | 探索和执行         | 遵循特定流程       |
| 隔离性 | 可用 worktree 隔离 | 在当前环境执行     |
| 适用性 | 搜索类、多步骤任务 | 重复性、规范类任务 |

简单说：**Agent 是"帮我做这件事"，Skill 是"按这个规范做"**。

到这里，你可以用 Agent 快速探索代码，用 Skills 封装规范。但接手的项目里有大量重复代码、测试覆盖率只有 18%，怎么安全地重构？下一篇，我们讲代码重构与测试驱动。
