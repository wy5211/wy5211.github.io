写到第五篇，你可能已经注意到了一个问题：TeamFlow 里的按钮到处复制粘贴同一堆类名。改个按钮圆角要搜遍整个项目，每处都要改。

```html
<!-- 这种代码在项目里出现了 20 次 -->
<button
  className="px-4 py-2 bg-blue-500 text-white rounded-md hover:bg-blue-600 transition-colors"
>
  保存
</button>
```

这就是 Tailwind 的工程化挑战：**类名组合确实强大，但复用起来没有传统 CSS 类那么直观**。

好消息是，在 React 组件化的开发模式中，这个问题有成熟的解决方案。这篇我们来建立一套组件模式，让 Tailwind 代码既能保持声明式的优点，又能优雅地复用。

## 三层复用策略

不是所有重复都需要抽象。Tailwind 项目的代码组织遵循三层策略：

### 第一层：模板局部（默认选择）

大多数时候，类名直接写在模板里就好。如果一段样式只出现一两次，不需要抽象：

```tsx
// 只在某个页面用一次的布局，不需要抽成组件
<div className="flex items-center gap-4 p-6 bg-gradient-to-r from-blue-50 to-sky-50 rounded-xl">
  <h2 className="text-lg font-semibold">项目概览</h2>
</div>
```

别过度抽象。两三个相似的地方先忍着，等出现第三次再考虑抽象。

### 第二层：React 组件 + Props（主要手段）

当 UI 模式反复出现时，抽成 React 组件。通过 props 控制变体：

```tsx
// 一个基本的按钮组件
function Button({
  children,
  variant = "primary",
  size = "md",
  className,
  ...props
}) {
  // 根据变体组合不同的类名
  const baseStyles =
    "font-medium rounded-md transition-colors focus-visible:ring-2 focus-visible:ring-offset-2";

  const variants = {
    primary: "bg-blue-500 text-white hover:bg-blue-600",
    secondary: "border border-slate-300 text-slate-700 hover:bg-slate-50",
    danger: "bg-red-500 text-white hover:bg-red-600",
    ghost: "text-slate-600 hover:bg-slate-100",
  };

  const sizes = {
    sm: "px-3 py-1.5 text-sm",
    md: "px-4 py-2 text-sm",
    lg: "px-6 py-3 text-base",
  };

  return (
    <button
      className={`${baseStyles} ${variants[variant]} ${sizes[size]} ${className || ""}`}
      {...props}
    >
      {children}
    </button>
  );
}
```

这个方案能用，但有一个问题：手动拼接类名字符串很原始，而且不支持 Tailwind 的类名覆盖（比如外部传 `className="px-8"` 无法覆盖内部的 `px-4`）。

### 第三层：cva + cn()（推荐方案）

这是社区目前最成熟的方案，也是 shadcn/ui 等项目使用的方式。

## cva：声明式组件变体

[cva](https://cva.style/)（class-variance-authority）是一个专门为组件变体设计的工具库：

```bash
npm install class-variance-authority
```

用它重写上面的按钮：

```tsx
// src/components/ui/button.tsx
import { cva, type VariantProps } from "class-variance-authority";
import { cn } from "@/lib/utils";

const buttonVariants = cva(
  // 基础样式：所有变体共享
  "inline-flex items-center justify-center font-medium rounded-md transition-colors focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-offset-2 disabled:pointer-events-none disabled:opacity-50",
  {
    variants: {
      // 外观变体
      variant: {
        primary:
          "bg-blue-500 text-white hover:bg-blue-600 focus-visible:ring-blue-500",
        secondary:
          "border border-slate-300 bg-white text-slate-700 hover:bg-slate-50",
        danger:
          "bg-red-500 text-white hover:bg-red-600 focus-visible:ring-red-500",
        ghost: "text-slate-600 hover:bg-slate-100 hover:text-slate-900",
      },
      // 尺寸变体
      size: {
        sm: "h-8 px-3 text-sm gap-1.5",
        md: "h-9 px-4 text-sm gap-2",
        lg: "h-10 px-6 text-base gap-2",
        icon: "h-9 w-9",
      },
    },
    defaultVariants: {
      variant: "primary",
      size: "md",
    },
  }
);

function Button({
  className,
  variant,
  size,
  ...props
}: React.ButtonHTMLAttributes<HTMLButtonElement> &
  VariantProps<typeof buttonVariants>) {
  return (
    <button
      className={cn(buttonVariants({ variant, size, className }))}
      {...props}
    />
  );
}
```

cva 的优势在于它把变体定义和组件实现分开了。`buttonVariants` 是一个纯粹的样式配置，可以导出给其他组件使用：

```tsx
// 在其他组件中复用按钮样式
function LinkButton({ href, ...props }) {
  return (
    <a
      href={href}
      className={cn(buttonVariants({ variant: "primary", size: "md" }))}
    >
      {/* ... */}
    </a>
  );
}
```

## cn()：智能合并类名

[cn()](https://ui.shadcn.com/docs/installation) 是 `clsx` + `tailwind-merge` 的组合，解决两个问题：

1. **条件类名**：根据 props 动态添加类名
2. **类名冲突**：当外部传入的类名和内部类名冲突时，外部优先

```bash
npm install clsx tailwind-merge
```

```tsx
// src/lib/utils.ts
import { type ClassValue, clsx } from "clsx";
import { twMerge } from "tailwind-merge";

export function cn(...inputs: ClassValue[]) {
  return twMerge(clsx(inputs));
}
```

为什么需要 `tailwind-merge`？看这个例子：

```tsx
// 没有 tailwind-merge
cn("px-4", "px-8"); // → "px-4 px-8"（冲突！浏览器取 px-8）
// 但有些 CSS 属性顺序不固定，可能取 px-4

// 有 tailwind-merge
cn("px-4", "px-8"); // → "px-8"（自动去重，后面的覆盖前面的）
```

这让组件的使用者可以用 `className` 覆盖内部样式：

```tsx
// 组件内部定义了 px-4，外部用 px-8 覆盖
<Button className="px-8 w-full">全宽大按钮</Button>
// 最终结果：内部的 px-4 被外部的 px-8 替代
```

## 构建 TeamFlow 的组件库

用 cva + cn() 构建一套 TeamFlow 常用的 UI 组件。

### Badge 标签

```tsx
// src/components/ui/badge.tsx
const badgeVariants = cva(
  "inline-flex items-center rounded-full px-2 py-0.5 text-xs font-medium transition-colors",
  {
    variants: {
      variant: {
        default: "bg-slate-100 text-slate-600",
        success: "bg-green-50 text-green-600",
        warning: "bg-yellow-50 text-yellow-600",
        danger: "bg-red-50 text-red-600",
        info: "bg-blue-50 text-blue-600",
        outline: "border border-slate-300 text-slate-600",
      },
    },
    defaultVariants: {
      variant: "default",
    },
  }
);

function Badge({ className, variant, ...props }: BadgeProps) {
  return (
    <span className={cn(badgeVariants({ variant }), className)} {...props} />
  );
}
```

使用：

```tsx
<Badge variant="success">已完成</Badge>
<Badge variant="danger">紧急</Badge>
<Badge variant="info" className="bg-sky-50 text-sky-600">自定义颜色覆盖</Badge>
```

### Input 输入框

```tsx
// src/components/ui/input.tsx
const inputVariants = cva(
  "flex w-full rounded-md border bg-white px-3 py-2 text-sm transition-colors file:border-0 file:bg-transparent file:text-sm file:font-medium placeholder:text-slate-400 focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-offset-2 disabled:cursor-not-allowed disabled:opacity-50",
  {
    variants: {
      variant: {
        default:
          "border-slate-300 focus-visible:border-blue-500 focus-visible:ring-blue-200",
        error:
          "border-red-500 focus-visible:border-red-500 focus-visible:ring-red-200",
      },
      size: {
        sm: "h-8 text-sm",
        md: "h-9 text-sm",
        lg: "h-10 text-base",
      },
    },
    defaultVariants: {
      variant: "default",
      size: "md",
    },
  }
);

function Input({ className, variant, size, error, ...props }: InputProps) {
  return (
    <div className="space-y-1">
      <input
        className={cn(
          inputVariants({ variant: error ? "error" : variant, size, className })
        )}
        aria-invalid={!!error}
        {...props}
      />
      {error && <p className="text-xs text-red-500">{error}</p>}
    </div>
  );
}
```

注意 `aria-invalid`——当表单验证失败时，辅助技术需要知道这个字段有问题。这是可访问性的一部分。

### Card 卡片

```tsx
// src/components/ui/card.tsx
function Card({ className, ...props }: React.HTMLAttributes<HTMLDivElement>) {
  return (
    <div
      className={cn(
        "bg-white rounded-lg border border-slate-200 shadow-sm",
        className
      )}
      {...props}
    />
  );
}

function CardHeader({
  className,
  ...props
}: React.HTMLAttributes<HTMLDivElement>) {
  return (
    <div
      className={cn("flex flex-col space-y-1.5 p-6", className)}
      {...props}
    />
  );
}

function CardTitle({
  className,
  ...props
}: React.HTMLAttributes<HTMLHeadingElement>) {
  return (
    <h3
      className={cn("text-lg font-semibold text-slate-900", className)}
      {...props}
    />
  );
}

function CardContent({
  className,
  ...props
}: React.HTMLAttributes<HTMLDivElement>) {
  return <div className={cn("p-6 pt-0", className)} {...props} />;
}
```

组合式组件（Compound Components）让卡片的使用很灵活：

```tsx
<Card>
  <CardHeader>
    <CardTitle>项目统计</CardTitle>
  </CardHeader>
  <CardContent>
    <StatsGrid />
  </CardContent>
</Card>
```

### Select 下拉框

```tsx
// src/components/ui/select.tsx
const selectVariants = cva(
  "flex w-full items-center justify-between rounded-md border bg-white px-3 py-2 text-sm transition-colors focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-offset-2 disabled:cursor-not-allowed disabled:opacity-50",
  {
    variants: {
      variant: {
        default:
          "border-slate-300 focus-visible:border-blue-500 focus-visible:ring-blue-200",
        error:
          "border-red-500 focus-visible:border-red-500 focus-visible:ring-red-200",
      },
    },
    defaultVariants: {
      variant: "default",
    },
  }
);

function Select({
  className,
  variant,
  options,
  placeholder,
  value,
  onChange,
  ...props
}: SelectProps) {
  return (
    <select
      className={cn(selectVariants({ variant }), className)}
      value={value}
      onChange={onChange}
      {...props}
    >
      {placeholder && (
        <option value="" disabled>
          {placeholder}
        </option>
      )}
      {options.map((option) => (
        <option key={option.value} value={option.value}>
          {option.label}
        </option>
      ))}
    </select>
  );
}
```

## @apply：什么时候用，什么时候不用

Tailwind 提供了 `@apply` 指令，可以把工具类"提取"到 CSS 里：

```css
/* 可以这样做，但大多数情况下不推荐 */
.btn-primary {
  @apply px-4 py-2 bg-blue-500 text-white rounded-md hover:bg-blue-600 transition-colors;
}
```

什么时候该用 `@apply`？

**不该用的场景**（大部分情况）：你已经有 React 组件了。`@apply` 是给非组件化的 HTML 用的（比如 Markdown 渲染的内容、第三方库生成的 DOM）。在 React 项目中，用组件封装比 `@apply` 更灵活。

**应该用的场景**：

1. **Markdown 内容样式**：你无法控制 Markdown 渲染出的 HTML 标签，但想给它们加 Tailwind 样式：

```css
.prose h2 {
  @apply text-xl font-semibold text-slate-900 mb-4;
}
.prose p {
  @apply text-slate-600 leading-relaxed mb-4;
}
.prose code {
  @apply bg-slate-100 text-sm px-1.5 py-0.5 rounded font-mono;
}
```

2. **第三方库的 DOM**：比如给 TipTap 编辑器的节点加样式。

3. **极高频的全局基础样式**：比如 `body` 的基础样式。

如果你发现自己在组件里频繁想用 `@apply`，说明你应该把它抽成一个 React 组件而不是一个 CSS 类。

## 可访问性（a11y）基础

Tailwind 项目中容易被忽略但很重要的一些可访问性实践：

### 焦点管理

```tsx
// 所有可交互元素都应该有可见的焦点样式
<button className="focus-visible:ring-2 focus-visible:ring-blue-500 focus-visible:ring-offset-2">
  按钮
</button>

// 如果用了自定义焦点样式，别忘了 outline-none
<button className="outline-none focus-visible:ring-2 focus-visible:ring-blue-500">
  自定义焦点
</button>
```

### 语义化 HTML

```html
<!-- 不要这样 -->
<div className="cursor-pointer" onClick="{handleClick}">点击</div>

<!-- 应该这样 -->
<button className="text-left" onClick="{handleClick}">点击</button>
```

### 屏幕阅读器辅助

```html
<!-- 对屏幕阅读器隐藏装饰性元素 -->
<span className="sr-only">辅助说明文字</span>

<!-- 只对屏幕阅读器可见的文字 -->
<span className="sr-only">关闭</span>
<span aria-hidden="true">×</span>
```

`sr-only` 让元素在视觉上不可见，但屏幕阅读器能读到。反过来，`aria-hidden="true"` 让装饰性图标对屏幕阅读器隐藏。

## 组件目录结构

随着组件增多，一个好的目录结构能帮你快速找到代码：

```
src/
├── components/
│   ├── ui/              # 基础 UI 组件（Button、Badge、Input...）
│   │   ├── button.tsx
│   │   ├── badge.tsx
│   │   ├── input.tsx
│   │   ├── select.tsx
│   │   ├── card.tsx
│   │   └── toast.tsx
│   ├── layout/          # 布局组件（Sidebar、Header...）
│   │   ├── sidebar.tsx
│   │   └── header.tsx
│   └── features/        # 业务组件（TaskCard、KanbanColumn...）
│       ├── task-card.tsx
│       ├── kanban-column.tsx
│       └── project-list.tsx
├── lib/
│   └── utils.ts         # cn() 工具函数
└── styles/
    └── globals.css      # Tailwind 入口 + 自定义动画
```

`ui/` 放纯 UI 组件（不包含业务逻辑），`features/` 放业务组件。这个划分让你在重构 UI 时不用关心业务逻辑，改业务时也不用翻 UI 代码。

## 实战：用组件库重构任务卡片

用我们刚建的组件库重构之前写的任务卡片，看看代码变得多清爽：

```tsx
import { Badge } from "@/components/ui/badge";
import { cn } from "@/lib/utils";

function TaskCard({ task }: { task: Task }) {
  return (
    <Card
      className={cn(
        "cursor-grab active:cursor-grabbing transition-all duration-200",
        "hover:shadow-md hover:border-slate-300"
      )}
    >
      <div className="p-3 space-y-2">
        {/* 优先级 + 标题 */}
        <div className="flex items-start gap-2">
          <div
            className={cn(
              "w-2 h-2 rounded-full mt-1.5 shrink-0",
              task.priority === "urgent"
                ? "bg-red-500"
                : task.priority === "high"
                  ? "bg-orange-500"
                  : "bg-blue-500"
            )}
          />
          <p className="text-sm font-medium text-slate-900">{task.title}</p>
        </div>

        {/* 标签 */}
        <div className="flex flex-wrap gap-1">
          {task.labels.map((label) => (
            <Badge key={label} variant="default">
              {label}
            </Badge>
          ))}
        </div>

        {/* 底部信息 */}
        <div className="flex items-center justify-between pt-2 border-t border-slate-100">
          <span className="text-xs text-slate-400">{task.dueDate}</span>
          <div className="w-6 h-6 rounded-full bg-blue-100 text-blue-600 flex items-center justify-center text-xs font-medium">
            {task.assigneeName[0]}
          </div>
        </div>
      </div>
    </Card>
  );
}
```

对比之前每个 Badge 样式都要手写的版本，现在只需要 `<Badge variant="default">` 一行。如果将来要改 Badge 的样式，改 `badge.tsx` 一个文件就行。
