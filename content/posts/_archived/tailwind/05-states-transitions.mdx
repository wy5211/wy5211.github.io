你有没有遇到过这种情况：点击一个按钮，什么反馈都没有——不确定点没点上，不确定请求有没有发出去。等了两秒，页面突然跳转了。

这种"没有反馈"的界面让人觉得不可靠。用户需要通过视觉信号确认自己的操作被系统接收了。按钮按下变暗、输入框获得焦点出现边框、卡片悬停浮起——这些微交互看起来是小事，但它们构成了产品的"质感"。

Tailwind 的状态变体和动画工具类让添加这些反馈变得简单。你不需要写 CSS 选择器，直接在工具类前面加一个前缀就行。

## 状态变体全览

### 鼠标相关

```html
<!-- hover：鼠标悬停 -->
<button className="bg-blue-500 hover:bg-blue-600">保存</button>

<!-- active：按下瞬间 -->
<button className="bg-blue-500 active:bg-blue-700">点击</button>
```

### 焦点相关

三个焦点变体有不同的使用场景：

```html
<!-- focus：元素获得焦点（Tab 或点击） -->
<input
  className="border border-slate-300 focus:border-blue-500 focus:ring-2 focus:ring-blue-200"
/>

<!-- focus-visible：通过键盘 Tab 获得焦点（不包括点击） -->
<button
  className="focus-visible:ring-2 focus-visible:ring-blue-500 focus-visible:outline-none"
>
  按钮
</button>

<!-- focus-within：子元素获得焦点时，父元素应用样式 -->
<div
  className="border border-slate-300 rounded-lg p-4 focus-within:border-blue-500 focus-within:ring-2 focus-within:ring-blue-200"
>
  <input className="w-full outline-none" placeholder="搜索任务..." />
</div>
```

`focus` 和 `focus-visible` 的区别很重要。`focus` 在点击和 Tab 时都会触发，而 `focus-visible` 只在 Tab 时触发。对于按钮，用 `focus-visible` 更好——点击按钮时不需要显示焦点环（用户知道自己点了什么），但键盘用户需要焦点环来定位。

### 表单状态

```html
<!-- disabled -->
<button
  className="bg-blue-500 disabled:bg-slate-200 disabled:text-slate-400"
  disabled
>
  提交中...
</button>

<!-- invalid：表单验证失败 -->
<input
  className="border border-slate-300 invalid:border-red-500 invalid:text-red-500"
  required
  type="email"
/>

<!-- placeholder-shown：输入框有占位符（即没有内容）时 -->
<input className="border placeholder-shown:border-slate-300 border-slate-500" />
```

### 条件组合

变体可以和响应式前缀组合使用：

```html
<!-- 移动端 hover 无效（触摸屏），但桌面端有 hover 效果 -->
<button className="bg-blue-500 md:hover:bg-blue-600">保存</button>

<!-- hover 状态下改变多个属性 -->
<div
  className="bg-white hover:bg-slate-50 hover:shadow-md hover:border-slate-300 transition-all"
>
  卡片
</div>
```

## Transition：让变化平滑

### 基本用法

`transition-*` 让属性变化有过渡动画，而不是瞬间跳变：

```html
<!-- 颜色过渡 -->
<button
  className="bg-blue-500 hover:bg-blue-600 transition-colors duration-200"
>
  保存
</button>

<!-- 阴影过渡 -->
<div className="shadow-sm hover:shadow-md transition-shadow duration-300">
  卡片
</div>

<!-- 所有属性都过渡 -->
<div
  className="opacity-50 hover:opacity-100 hover:scale-105 transition-all duration-300"
>
  悬停放大
</div>
```

### 选择过渡的属性

`transition-all` 虽然方便但不够精确。好的做法是只过渡真正需要的属性：

| 工具类                 | 过渡的属性                            | 适用场景         |
| ---------------------- | ------------------------------------- | ---------------- |
| `transition-colors`    | color, background-color, border-color | 按钮、链接       |
| `transition-opacity`   | opacity                               | 显示/隐藏        |
| `transition-shadow`    | box-shadow                            | 卡片浮起         |
| `transition-transform` | transform                             | 移动、缩放、旋转 |
| `transition-all`       | 所有可动画属性                        | 不确定时用这个   |

为什么不用 `transition-all`？因为它的性能开销更大——浏览器需要监听所有属性的变化。对于简单的颜色变化，`transition-colors` 足够了，而且更精确。

### 时长控制

```html
<button className="transition-colors duration-150">快（150ms）</button>
<button className="transition-colors duration-300">标准（300ms）</button>
<button className="transition-colors duration-500">慢（500ms）</button>
```

大多数交互反馈用 150-300ms 就好。太快感觉不到，太慢觉得卡。300ms 是一个比较安全的默认值。

### 缓动函数

```html
<button className="transition-colors ease-in-out">先慢后快再慢</button>
<button className="transition-colors ease-out">快速开始，慢慢停止</button>
```

Tailwind 默认使用 `ease`（`cubic-bezier(0.4, 0, 0.2, 1)`），这对大多数场景来说已经够好了。`ease-out` 适合"出现"动画（元素进入视图），`ease-in-out` 适合来回切换的状态。

## Transform：位移、缩放、旋转

### 常用 Transform

```html
<!-- 悬停放大 -->
<div className="hover:scale-105 transition-transform duration-200">
  产品卡片
</div>

<!-- 按下缩小（模拟物理按压感） -->
<button className="active:scale-95 transition-transform duration-100">
  点击我
</button>

<!-- 微小的上浮效果 -->
<div className="hover:-translate-y-1 transition-transform duration-200">
  悬停上浮
</div>
```

### 组合使用

hover 状态下同时改变多个属性，用 `group` 来简化：

```tsx
function ProjectCard({ project }: { project: Project }) {
  return (
    <div className="group bg-white rounded-xl border border-slate-200 overflow-hidden hover:shadow-lg transition-all duration-300">
      {/* 封面图：hover 时微微放大 */}
      <div className="h-40 bg-slate-200 overflow-hidden">
        <img
          src={project.coverUrl}
          alt=""
          className="w-full h-full object-cover group-hover:scale-105 transition-transform duration-500"
        />
      </div>

      {/* 内容区：hover 时出现操作按钮 */}
      <div className="p-4">
        <h3 className="font-medium text-slate-900 mb-1">{project.name}</h3>
        <p className="text-sm text-slate-500 line-clamp-2">
          {project.description}
        </p>

        <div className="mt-3 flex items-center justify-between opacity-0 group-hover:opacity-100 transition-opacity duration-200">
          <span className="text-xs text-slate-400">
            {project.memberCount} 位成员
          </span>
          <button className="text-sm text-blue-500 hover:text-blue-600">
            查看详情 →
          </button>
        </div>
      </div>
    </div>
  );
}
```

`group` 的用法：在父元素上加 `group` 类名，在子元素上用 `group-hover:` 前缀。这样当鼠标悬停在整个卡片上时，子元素也会触发对应的样式变化。

如果同一层有多个 `group`，可以用命名区分：

```html
<div className="group/card">
  <div className="group-hover/card:opacity-100">...</div>
</div>
```

## 内置动画

Tailwind 提供了几个常用的内置动画：

```html
<!-- 加载旋转 -->
<svg className="animate-spin h-5 w-5 text-blue-500" viewBox="0 0 24 24">
  <circle
    className="opacity-25"
    cx="12"
    cy="12"
    r="10"
    stroke="currentColor"
    strokeWidth="4"
    fill="none"
  />
  <path
    className="opacity-75"
    fill="currentColor"
    d="M4 12a8 8 0 018-8V0C5.373 0 0 5.373 0 12h4z"
  />
</svg>

<!-- 脉冲闪烁（加载占位） -->
<div className="animate-pulse bg-slate-200 rounded h-4 w-3/4" />

<!-- 弹跳 -->
<div className="animate-bounce">↓</div>

<!-- 通知红点 -->
<span className="relative">
  通知
  <span
    className="absolute -top-1 -right-1 w-2 h-2 bg-red-500 rounded-full animate-ping"
  />
</span>
```

`animate-pulse` 做骨架屏加载状态特别好用：

```tsx
function CardSkeleton() {
  return (
    <div className="bg-white rounded-lg p-4 border border-slate-200 animate-pulse">
      <div className="h-4 bg-slate-200 rounded w-3/4 mb-3" />
      <div className="h-3 bg-slate-200 rounded w-full mb-2" />
      <div className="h-3 bg-slate-200 rounded w-2/3 mb-4" />
      <div className="flex items-center justify-between">
        <div className="h-3 bg-slate-200 rounded w-20" />
        <div className="h-6 w-6 bg-slate-200 rounded-full" />
      </div>
    </div>
  );
}
```

骨架屏给用户"内容正在加载"的暗示，比白屏或转圈更友好。

## 自定义动画

当内置动画不够用时，可以在 CSS 中定义自定义关键帧：

```css
/* src/index.css */
@import "tailwindcss";

@theme {
  --animate-fade-in: fade-in 0.3s ease-out;
  --animate-slide-up: slide-up 0.3s ease-out;
  --animate-slide-down: slide-down 0.3s ease-out;
  --animate-scale-in: scale-in 0.2s ease-out;
}

@keyframes fade-in {
  from {
    opacity: 0;
  }
  to {
    opacity: 1;
  }
}

@keyframes slide-up {
  from {
    opacity: 0;
    transform: translateY(8px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}

@keyframes slide-down {
  from {
    opacity: 0;
    transform: translateY(-8px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}

@keyframes scale-in {
  from {
    opacity: 0;
    transform: scale(0.95);
  }
  to {
    opacity: 1;
    transform: scale(1);
  }
}
```

然后就可以直接用 `animate-fade-in` 这些工具类了：

```tsx
// Toast 通知的出场动画
function Toast({ message, type }: ToastProps) {
  return (
    <div className="animate-slide-up bg-white rounded-lg shadow-lg border border-slate-200 px-4 py-3 flex items-center gap-3">
      <div
        className={`w-5 h-5 rounded-full flex items-center justify-center text-white text-xs ${
          type === "success"
            ? "bg-green-500"
            : type === "error"
              ? "bg-red-500"
              : "bg-blue-500"
        }`}
      >
        {type === "success" ? "✓" : type === "error" ? "✕" : "i"}
      </div>
      <p className="text-sm text-slate-700">{message}</p>
    </div>
  );
}
```

## 暗色模式

Tailwind 的暗色模式用 `dark:` 变体实现。有两种切换策略：

### 策略一：跟随系统

自动跟随操作系统的深色模式设置，不需要手动切换：

```css
/* src/index.css */
@import "tailwindcss";

@custom-variant dark (&:where(.dark, .dark *));
```

这种方式不需要 JS 逻辑，但用户无法手动控制。

### 策略二：class 切换

通过在 `<html>` 或 `<body>` 上添加 `dark` 类名来切换：

```tsx
function ThemeToggle() {
  const [dark, setDark] = useState(false);

  useEffect(() => {
    document.documentElement.classList.toggle("dark", dark);
  }, [dark]);

  return (
    <button
      onClick={() => setDark(!dark)}
      className="p-2 rounded-lg hover:bg-slate-100 dark:hover:bg-slate-800 transition-colors"
    >
      {dark ? "☀️" : "🌙"}
    </button>
  );
}
```

CSS 配置保持不变，`dark:` 变体会自动生效。

### 暗色模式的颜色搭配

暗色模式不是简单地把白色背景换成黑色。你需要调整整个颜色层级：

```tsx
// 一个在亮色和暗色模式下都好看的卡片
<div className="bg-white dark:bg-slate-800 rounded-lg border border-slate-200 dark:border-slate-700 p-4 shadow-sm">
  <h3 className="text-slate-900 dark:text-slate-100 font-medium mb-1">
    任务标题
  </h3>
  <p className="text-slate-500 dark:text-slate-400 text-sm">任务描述</p>
  <span className="text-xs bg-slate-100 dark:bg-slate-700 text-slate-600 dark:text-slate-300 px-2 py-0.5 rounded">
    标签
  </span>
</div>
```

规律是：暗色模式下每个颜色都比亮色模式"亮"一个层级——背景更深，文字更亮，边框更深，背景色更深。

可以这样记忆：

```
亮色 → 暗色
bg-white         → bg-slate-800
bg-slate-50      → bg-slate-900
bg-slate-100     → bg-slate-700
text-slate-900   → text-slate-100
text-slate-500   → text-slate-400
border-slate-200 → border-slate-700
```

## 实战：TeamFlow 的交互反馈

把交互状态和动画应用到 TeamFlow 的几个关键场景。

### 任务卡片的完整交互

```tsx
function TaskCard({ task, onDragStart }: TaskCardProps) {
  const [isDragging, setIsDragging] = useState(false);

  return (
    <div
      draggable
      onDragStart={(e) => {
        setIsDragging(true);
        onDragStart(task.id);
      }}
      onDragEnd={() => setIsDragging(false)}
      className={`
        bg-white rounded-lg p-3 border cursor-grab active:cursor-grabbing
        transition-all duration-200
        ${
          isDragging
            ? "shadow-xl scale-105 opacity-75 border-blue-300 rotate-2"
            : "shadow-sm border-slate-200 hover:shadow-md hover:border-slate-300"
        }
      `}
    >
      <div className="flex items-start gap-2 mb-2">
        <div
          className={`w-2 h-2 rounded-full mt-1.5 shrink-0 ${
            task.priority === "urgent"
              ? "bg-red-500"
              : task.priority === "high"
                ? "bg-orange-500"
                : "bg-blue-500"
          }`}
        />
        <p className="text-sm font-medium text-slate-900">{task.title}</p>
      </div>

      {task.labels.length > 0 && (
        <div className="flex flex-wrap gap-1 mb-2">
          {task.labels.map((label) => (
            <span
              key={label}
              className="text-xs text-slate-500 bg-slate-100 px-1.5 py-0.5 rounded"
            >
              {label}
            </span>
          ))}
        </div>
      )}

      <div className="flex items-center justify-between pt-2 border-t border-slate-100">
        <span className="text-xs text-slate-400">{task.dueDate}</span>
        <div className="w-6 h-6 rounded-full bg-blue-100 text-blue-600 flex items-center justify-center text-xs font-medium">
          {task.assigneeName[0]}
        </div>
      </div>
    </div>
  );
}
```

拖拽中的状态用 `shadow-xl scale-105 opacity-75 rotate-2` 营造"拿起"的感觉——阴影更大、微微放大、半透明、略微旋转。这些微妙的组合让拖拽操作有了物理感。

### 按钮的状态流转

```tsx
function SubmitButton() {
  const [status, setStatus] = useState<
    "idle" | "loading" | "success" | "error"
  >("idle");

  return (
    <button
      onClick={async () => {
        setStatus("loading");
        try {
          await submitTask();
          setStatus("success");
          setTimeout(() => setStatus("idle"), 2000);
        } catch {
          setStatus("error");
          setTimeout(() => setStatus("idle"), 2000);
        }
      }}
      disabled={status === "loading"}
      className={`
        px-4 py-2 rounded-lg text-sm font-medium transition-all duration-200
        focus-visible:ring-2 focus-visible:ring-blue-500 focus-visible:ring-offset-2
        ${status === "idle" && "bg-blue-500 hover:bg-blue-600 active:bg-blue-700 text-white"}
        ${status === "loading" && "bg-blue-500 text-white/80 cursor-wait"}
        ${status === "success" && "bg-green-500 text-white"}
        ${status === "error" && "bg-red-500 text-white"}
      `}
    >
      {status === "idle" && "提交任务"}
      {status === "loading" && (
        <span className="flex items-center gap-2">
          <svg className="animate-spin h-4 w-4" viewBox="0 0 24 24">
            <circle
              className="opacity-25"
              cx="12"
              cy="12"
              r="10"
              stroke="currentColor"
              strokeWidth="4"
              fill="none"
            />
            <path
              className="opacity-75"
              fill="currentColor"
              d="M4 12a8 8 0 018-8V0C5.373 0 0 5.373 0 12h4z"
            />
          </svg>
          提交中...
        </span>
      )}
      {status === "success" && "✓ 已提交"}
      {status === "error" && "✕ 提交失败"}
    </button>
  );
}
```

按钮的四种状态有各自的视觉表现：idle（蓝色可点击）、loading（蓝色半透明 + 旋转图标）、success（绿色 + 对勾）、error（红色 + 叉号）。`transition-all duration-200` 让状态切换有平滑过渡。

## 性能提示

动画虽好，但要注意性能。几个实用建议：

1. **优先动画 `transform` 和 `opacity`**：这两个属性由 GPU 加速，不会触发重排（reflow）。`transition-transform` 和 `transition-opacity` 几乎没有性能开销。

2. **避免动画 `box-shadow`**：阴影变化会触发重绘（repaint）。如果需要，可以用伪元素 + `opacity` 模拟：给元素加一个有阴影的 `::after` 伪元素，通过控制 opacity 来"显示"阴影。不过大部分场景下直接 `transition-shadow` 的性能开销是可以接受的。

3. **`will-change` 谨慎使用**：Tailwind 有 `will-change-transform` 等工具类，告诉浏览器"这个属性将要变化，提前优化"。但不要到处加——每个 `will-change` 都会消耗内存。只在确实会频繁变化的元素上使用。

4. **`duration` 别太长**：超过 500ms 的动画用户会觉得卡。交互反馈类动画控制在 200-300ms，页面级过渡控制在 300-500ms。

好的动画是用户注意不到的——它让操作感觉自然流畅，而不是刻意炫技。
