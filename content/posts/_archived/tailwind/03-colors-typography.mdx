上一篇我们搭好了 TeamFlow 的骨架——侧边栏、顶部栏、看板区域，布局结构没问题了。但你打开页面一看，总觉得"差点意思"。

差在哪？通常是两件事没做好：**颜色没有体系**和**排版没有节奏**。

随意用几个灰色、字号忽大忽小、间距随心所欲——这种界面能用，但不好看。好消息是，Tailwind 内置了完整的设计令牌系统，你不需要从零设计颜色和字号。理解了这套系统的逻辑，就能用它搭出有设计感的界面。

## 颜色系统：不只是选一个颜色

### 五种灰度，五种性格

Tailwind 提供了五种灰色调，每种都有不同的色彩倾向：

| 灰色系列  | 色调倾向   | 适合场景                      |
| --------- | ---------- | ----------------------------- |
| `slate`   | 偏蓝的灰   | 科技感、专业感——SaaS 产品首选 |
| `gray`    | 纯中性灰   | 通用，没有明显倾向            |
| `zinc`    | 偏暖的灰   | 现代、干净——Apple 风格的产品  |
| `neutral` | 无色彩的灰 | 最纯粹的灰，没有色彩干扰      |
| `stone`   | 偏棕的灰   | 温暖、有机——内容型网站        |

同一个界面，用 `slate` 和用 `stone` 会给人完全不同的感觉。TeamFlow 是一个项目管理工具，需要专业和清晰的视觉印象，所以用 `slate` 系列。

怎么选？去看看你喜欢的产品的用色，大部分 SaaS 产品用的是 `slate` 或 `zinc`。内容型网站和博客可能更适合 `stone`。

### 色阶：10 个亮度层级

每种颜色都有从 50（最浅）到 950（最深）的色阶：

```
slate-50   → 几乎是白色的背景
slate-100  → 非常浅的灰色背景
slate-200  → 边框、分割线
slate-300  → 禁用状态的文字
slate-400  → 占位符文字
slate-500  → 图标、辅助文字
slate-600  → 次要文字
slate-700  → 正文文字
slate-800  → 标题文字
slate-900  → 最深的文字
slate-950  → 深色背景
```

不需要记住每个色阶的具体值。掌握一个规律：**50-200 用于背景，300-500 用于弱化元素，600-900 用于文字，950 用于深色背景**。

### 颜色使用位置

同一个颜色可以用在不同属性上：

```html
<!-- bg：背景色 -->
<div className="bg-blue-50">浅蓝色背景</div>

<!-- text：文字颜色 -->
<p className="text-blue-600">蓝色文字</p>

<!-- border：边框颜色 -->
<div className="border border-blue-200">蓝色边框</div>

<!-- ring：聚焦环颜色 -->
<input className="focus:ring-2 focus:ring-blue-500" />

<!-- divide：分割线颜色 -->
<div className="divide-y divide-slate-200">
  <div>项目 1</div>
  <div>项目 2</div>
</div>
```

### 透明度修饰符

任何颜色都可以加透明度，用 `/` 语法：

```html
<!-- 蓝色 10% 透明度 → 非常浅的蓝色背景 -->
<div className="bg-blue-500/10">淡蓝色提示背景</div>

<!-- 红色 80% 透明度 → 稍微透明的红色 -->
<span className="text-red-500/80">错误提示</span>

<!-- 黑色 50% 透明度 → 半透明遮罩 -->
<div className="bg-black/50 fixed inset-0">模态框遮罩</div>
```

这个功能在做徽章、提示背景、遮罩层时特别好用。比如"高优先级"标签，不需要单独找一个浅红色，用 `bg-red-500/10 text-red-600` 就行。

### 语义化颜色

Tailwind 预设的颜色是按色相命名的（blue、red、green...），但在实际项目中，你需要建立自己的语义映射：

```
品牌主色     → blue-500
成功/安全    → green-500
警告/注意    → amber-500
错误/危险    → red-500
信息/提示    → sky-500
```

在 TeamFlow 中，我们这样用语义色：

```tsx
// 任务优先级
const priorityConfig = {
  urgent: {
    bg: "bg-red-500/10",
    text: "text-red-600",
    dot: "bg-red-500",
    label: "紧急",
  },
  high: {
    bg: "bg-orange-500/10",
    text: "text-orange-600",
    dot: "bg-orange-500",
    label: "高",
  },
  medium: {
    bg: "bg-yellow-500/10",
    text: "text-yellow-600",
    dot: "bg-yellow-500",
    label: "中",
  },
  low: {
    bg: "bg-blue-500/10",
    text: "text-blue-600",
    dot: "bg-blue-500",
    label: "低",
  },
};

// 任务状态
const statusConfig = {
  todo: { bg: "bg-slate-500/10", text: "text-slate-600", label: "待办" },
  in_progress: { bg: "bg-blue-500/10", text: "text-blue-600", label: "进行中" },
  review: { bg: "bg-purple-500/10", text: "text-purple-600", label: "审核中" },
  done: { bg: "bg-green-500/10", text: "text-green-600", label: "已完成" },
};
```

把颜色配置集中管理而不是散落在各处，改颜色时只需要改一个地方。

## 排版系统：让文字有节奏

### 字体族

Tailwind 默认使用系统字体栈，这在大多数情况下就够了：

```html
<!-- sans（默认）：系统无衬线字体栈 -->
<p className="font-sans">这段文字用系统字体渲染</p>

<!-- mono：等宽字体，代码和数字用 -->
<code className="font-mono text-sm">const x = 42;</code>
```

如果你需要自定义字体，在 CSS 里覆盖默认值：

```css
@import "tailwindcss";

@theme {
  --font-sans: "Inter", "system-ui", "sans-serif";
  --font-mono: "JetBrains Mono", "Menlo", "monospace";
}
```

### 字号阶梯

Tailwind 的字号工具类对应一组字号和行高的配对：

| 工具类      | 字号            | 行高    | 使用场景           |
| ----------- | --------------- | ------- | ------------------ |
| `text-xs`   | 0.75rem (12px)  | 1rem    | 标签、时间戳       |
| `text-sm`   | 0.875rem (14px) | 1.25rem | 辅助文字、表格内容 |
| `text-base` | 1rem (16px)     | 1.5rem  | 正文（默认值）     |
| `text-lg`   | 1.125rem (18px) | 1.75rem | 小标题             |
| `text-xl`   | 1.25rem (20px)  | 1.75rem | 区块标题           |
| `text-2xl`  | 1.5rem (24px)   | 2rem    | 页面标题           |
| `text-3xl`  | 1.875rem (30px) | 2.25rem | 大标题             |
| `text-4xl`  | 2.25rem (36px)  | 2.5rem  | Hero 标题          |

注意每个字号都自带了合理的行高。这是 Tailwind 设计系统的细节——不需要你手动调行高，默认的配对已经考虑了可读性。

### 字重

字重用来建立视觉层次：

```html
<h1 className="text-2xl font-bold">项目标题</h1>
<!-- 700 -->
<h2 className="text-lg font-semibold">区块标题</h2>
<!-- 600 -->
<p className="text-base font-medium">强调文字</p>
<!-- 500 -->
<p className="text-sm">普通文字</p>
<!-- 400，默认 -->
<span className="text-xs text-slate-400">辅助说明</span>
<!-- 400 -->
```

一个常见错误是到处用 `font-bold`。好的排版只用两三种字重：`bold` 给标题，`semibold` 给小标题，`normal`（默认）给正文。偶尔用 `medium` 区分强调文字。

### 行高和字间距

大多数时候 Tailwind 的默认行高就够了。但在特殊场景下可以手动调整：

```html
<!-- 紧凑的列表项 -->
<p className="leading-tight text-sm">任务描述文本</p>

<!-- 宽松的正文阅读 -->
<p className="leading-relaxed">这是一段较长的描述文字，需要舒适的阅读体验...</p>

<!-- 全大写标签，加宽字间距 -->
<span className="text-xs uppercase tracking-wider text-slate-500">标签</span>
```

`leading-tight`（1.25）适合标题和短文本，`leading-normal`（1.5，默认）适合正文，`leading-relaxed`（1.625）适合长篇阅读。

## 文字截断：处理溢出

在卡片、列表、表格这类紧凑空间里，文字经常超出容器。Tailwind 提供了三种处理方式：

```html
<!-- 单行截断 -->
<p className="truncate">这是一个很长很长的项目名称在侧边栏里显示不下需要截断</p>

<!-- 多行截断（最多 2 行） -->
<p className="line-clamp-2">
  这是一段很长的任务描述，在卡片里只显示前两行，多余的用省略号代替。
  这样可以保持卡片高度一致，不会因为描述长短不一导致布局参差不齐。
</p>

<!-- 不换行 -->
<span className="whitespace-nowrap">2025-04-10 09:15</span>
```

`truncate` 是最常用的——在侧边栏导航项、表格单元格、下拉选项中随处可见。`line-clamp-*` 适合卡片描述这种需要显示部分内容的场景。

一个细节：`truncate` 需要元素有明确的宽度限制才会生效。在 Flex 子元素中，通常要配合 `min-w-0` 使用：

```html
<div className="flex items-center gap-3">
  <div className="shrink-0 w-8 h-8 rounded-full bg-slate-200" />
  <!-- min-w-0 允许文字区域缩小到比内容更窄，truncate 才能工作 -->
  <div className="min-w-0">
    <p className="truncate text-sm font-medium">很长的用户名会被截断</p>
    <p className="text-xs text-slate-400 truncate">
      zhang_wei_with_long_suffix@teamflow.dev
    </p>
  </div>
</div>
```

## 装饰效果

### 圆角

圆角直接影响界面的"性格"：

```html
<div className="rounded-sm">轻微圆角 → 表格行、紧凑元素</div>
<div className="rounded">中等圆角 → 默认，大多数元素</div>
<div className="rounded-lg">较大圆角 → 卡片、按钮</div>
<div className="rounded-xl">大圆角 → 弹窗、面板</div>
<div className="rounded-full">全圆角 → 头像、标签、徽章</div>
```

TeamFlow 用 `rounded-lg` 作为卡片的标准圆角，`rounded-md` 作为按钮，`rounded-full` 用于头像和标签。保持圆角层级的一致性，整个界面才会协调。

### 阴影

阴影营造层次感：

```html
<div className="shadow-sm">轻微阴影 → 卡片、列表项</div>
<div className="shadow">标准阴影 → 弹出面板</div>
<div className="shadow-md">中等阴影 → 悬浮卡片</div>
<div className="shadow-lg">明显阴影 → 模态框</div>
```

日常开发中 `shadow-sm` 用得最多。太重的阴影会显得不专业——除非你想刻意制造"浮起"的效果（比如拖拽中的卡片）。

### 边框

```html
<!-- 实线边框 -->
<div className="border border-slate-200">标准边框</div>

<!-- 不同方向 -->
<div className="border-b border-slate-200">只有底部边框</div>

<!-- 边框作为分割线 -->
<div className="divide-y divide-slate-100">
  <div className="py-3">列表项 1</div>
  <div className="py-3">列表项 2</div>
  <div className="py-3">列表项 3</div>
</div>
```

在浅色界面上，用 `border-slate-200` 作为分割线比用 `<hr>` 更灵活。`divide-y` 会自动在子元素之间添加顶部边框，省去给每个子元素加 `border-b` 的麻烦——而且最后一个元素不会有多余边框。

## 实战：TeamFlow 的任务卡片

把颜色、排版、装饰效果组合起来，构建一个有设计感的任务卡片：

```tsx
function TaskCard({ task }: { task: Task }) {
  const priorityMap = {
    urgent: {
      color: "bg-red-500",
      label: "紧急",
      badge: "bg-red-50 text-red-600",
    },
    high: {
      color: "bg-orange-500",
      label: "高",
      badge: "bg-orange-50 text-orange-600",
    },
    medium: {
      color: "bg-yellow-500",
      label: "中",
      badge: "bg-yellow-50 text-yellow-700",
    },
    low: {
      color: "bg-blue-500",
      label: "低",
      badge: "bg-blue-50 text-blue-600",
    },
  };

  const priority = priorityMap[task.priority];

  return (
    <div className="bg-white rounded-lg p-3 shadow-sm border border-slate-200 hover:shadow-md transition-shadow cursor-pointer">
      {/* 顶部：优先级指示器 + 操作按钮 */}
      <div className="flex items-start justify-between mb-2">
        <div className="flex items-center gap-2">
          <div className={`w-2 h-2 rounded-full ${priority.color}`} />
          <span
            className={`text-xs font-medium px-1.5 py-0.5 rounded ${priority.badge}`}
          >
            {priority.label}
          </span>
        </div>
        <button className="text-slate-400 hover:text-slate-600 transition-colors text-sm">
          ⋯
        </button>
      </div>

      {/* 标题 */}
      <h4 className="text-sm font-medium text-slate-900 mb-2 leading-snug">
        {task.title}
      </h4>

      {/* 描述（最多显示 2 行） */}
      {task.description && (
        <p className="text-xs text-slate-500 line-clamp-2 mb-3 leading-relaxed">
          {task.description}
        </p>
      )}

      {/* 标签列表 */}
      {task.labels.length > 0 && (
        <div className="flex flex-wrap gap-1 mb-3">
          {task.labels.map((label) => (
            <span
              key={label}
              className="text-xs text-slate-500 bg-slate-100 px-1.5 py-0.5 rounded"
            >
              {label}
            </span>
          ))}
        </div>
      )}

      {/* 底部：截止日期 + 负责人 */}
      <div className="flex items-center justify-between pt-2 border-t border-slate-100">
        <div className="flex items-center gap-1.5 text-xs text-slate-400">
          <span>📅</span>
          <span>{task.dueDate}</span>
        </div>
        <div
          className="w-6 h-6 rounded-full bg-blue-100 text-blue-600 flex items-center justify-center text-xs font-medium"
          title={task.assigneeName}
        >
          {task.assigneeName.slice(0, 1)}
        </div>
      </div>
    </div>
  );
}
```

这个卡片虽然不大，但用到了颜色系统的多种技巧：

- **优先级**用不同颜色的圆点 + 徽章区分——一目了然
- **标题**用 `text-sm font-medium text-slate-900`——够醒目但不会太重
- **描述**用 `text-xs text-slate-500 line-clamp-2`——弱化处理，最多两行
- **标签**用 `text-xs bg-slate-100 rounded`——存在感低但可识别
- **分割线**用 `border-t border-slate-100`——比 `border-200` 更浅，适合卡片内部分隔
- **头像**用 `bg-blue-100 text-blue-600`——没有图片时的占位符

注意同一个卡片里文字只有三种颜色深度：`slate-900`（标题）、`slate-500`（描述和标签）、`slate-400`（日期和弱化信息）。这种有意识的层级划分让界面看起来"有秩序"而不是"乱糟糟"。

## 深色侧边栏的颜色搭配

TeamFlow 的侧边栏用深色背景。深色界面的颜色使用有一些特别的讲究：

```tsx
function Sidebar() {
  return (
    <aside className="w-64 bg-slate-900 text-slate-300 flex flex-col">
      {/* Logo：最亮的文字，白色 */}
      <div className="h-14 flex items-center px-6 border-b border-slate-800">
        <div className="w-7 h-7 bg-blue-500 rounded-lg" />
        <span className="ml-3 text-lg font-semibold text-white">TeamFlow</span>
      </div>

      {/* 导航菜单 */}
      <nav className="flex-1 px-3 py-4 space-y-1">
        <a
          href="#"
          className="flex items-center gap-3 px-3 py-2 rounded-lg bg-slate-800 text-white text-sm"
        >
          <span>📊</span>
          <span>仪表盘</span>
        </a>
        <a
          href="#"
          className="flex items-center gap-3 px-3 py-2 rounded-lg text-slate-400 hover:bg-slate-800/50 hover:text-slate-200 text-sm transition-colors"
        >
          <span>📁</span>
          <span>项目</span>
        </a>
      </nav>
    </aside>
  );
}
```

深色界面的颜色规则：

- **背景**用 `slate-900` 或 `slate-950`，不要太浅（`slate-800` 不够深）
- **正文**用 `slate-300` 或 `slate-400`，不用白色（白色文字在深色背景上太刺眼）
- **强调文字**（Logo、激活项）用白色
- **边框**用 `slate-800` 或 `slate-700`——比背景浅一到两级
- **hover 状态**用 `slate-800` 或 `slate-800/50`（半透明）

深色和浅色区域的对比让界面有了层次感——侧边栏"退后"，主内容区"向前"。

## 一个常见的坑：颜色不一致

一个常见的问题是同一个界面里混用了 `slate` 和 `gray`——看起来差不多但总觉得"不对劲"。这是因为不同灰色系列的色调有细微差异，混在一起就不协调。

解决方法很简单：**选一个灰色系列，坚持用它**。在 TeamFlow 中，我们全部用 `slate`。从头到尾保持一致，界面自然就有整体感。
