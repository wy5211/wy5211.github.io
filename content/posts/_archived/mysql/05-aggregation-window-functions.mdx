你想给博客的管理后台加一个数据仪表盘。需要这些数据：

- 每月发了多少篇文章（月度趋势）
- 哪个标签下的文章最多（标签热度）
- 阅读量 TOP 10 的文章
- 累计发文数随时间的变化

前两个用 COUNT + GROUP BY 就能搞定。但"累计发文数"需要"看前面的行"——普通 SQL 做不到。窗口函数就是来解决这类问题的。

## 聚合函数：COUNT、SUM、AVG、MAX、MIN

聚合函数把多行数据"压缩"成一个值。你之前在 [第 4 篇](/posts/mysql/04-joins-advanced-queries) 里已经用过了 `COUNT`：

```sql
-- 文章总数
SELECT COUNT(*) AS total_articles FROM articles WHERE deleted_at IS NULL;

-- 总阅读量
SELECT SUM(view_count) AS total_views FROM articles WHERE deleted_at IS NULL;

-- 平均阅读量
SELECT AVG(view_count) AS avg_views FROM articles WHERE deleted_at IS NULL AND status = 'published';

-- 最高阅读量
SELECT MAX(view_count) AS max_views FROM articles WHERE deleted_at IS NULL;
```

聚合函数可以一起用：

```sql
SELECT
  COUNT(*) AS total,
  SUM(view_count) AS total_views,
  ROUND(AVG(view_count), 1) AS avg_views,
  MAX(view_count) AS max_views
FROM articles
WHERE status = 'published' AND deleted_at IS NULL;
```

`ROUND(value, 1)` 保留一位小数。AVG 的结果可能是很长的小数，一般需要处理一下。

## GROUP BY：分组统计

聚合函数真正的威力在于配合 GROUP BY——按某个维度分组后再聚合。

### 月度发文趋势

```sql
SELECT
  DATE_FORMAT(created_at, '%Y-%m') AS month,
  COUNT(*) AS article_count,
  SUM(view_count) AS month_views
FROM articles
WHERE status = 'published' AND deleted_at IS NULL
GROUP BY DATE_FORMAT(created_at, '%Y-%m')
ORDER BY month DESC;
```

结果长这样：

```
month       | article_count | month_views
------------|---------------|------------
2025-04     | 3             | 2847
2025-03     | 5             | 5192
2025-02     | 2             | 1203
```

`DATE_FORMAT(created_at, '%Y-%m')` 把时间戳格式化成年-月，`GROUP BY` 按这个格式分组，`COUNT(*)` 统计每组的文章数。

### 标签热度排名

```sql
SELECT
  t.name AS tag_name,
  COUNT(at.article_id) AS article_count
FROM tags t
LEFT JOIN article_tags at ON t.id = at.tag_id
LEFT JOIN articles a ON at.article_id = a.id AND a.deleted_at IS NULL AND a.status = 'published'
GROUP BY t.id, t.name
ORDER BY article_count DESC;
```

这里用了 LEFT JOIN 而不是 INNER JOIN，因为有些标签可能还没有文章，你也要在结果里看到它们（article_count 为 0）。

### GROUP BY 的规则

**SELECT 里出现的非聚合列，必须出现在 GROUP BY 里**。这是 SQL 的一个硬规则：

```sql
-- 错误：user_id 不在 GROUP BY 里
SELECT user_id, title, COUNT(*)
FROM articles
GROUP BY user_id;

-- 正确：要么去掉 title，要么加到 GROUP BY 里
SELECT user_id, COUNT(*) AS article_count
FROM articles
GROUP BY user_id;
```

原因很简单：按 user_id 分组后，一个组里可能有多条记录，每条的 title 不同，MySQL 不知道该返回哪个 title。

## HAVING：过滤分组结果

WHERE 是在分组前过滤，HAVING 是在分组后过滤。

```sql
-- 找出有 3 篇以上文章的作者
SELECT user_id, COUNT(*) AS article_count
FROM articles
WHERE deleted_at IS NULL
GROUP BY user_id
HAVING article_count >= 3;
```

执行顺序：WHERE 过滤行 → GROUP BY 分组 → HAVING 过滤分组 → SELECT 返回结果。

**WHERE vs HAVING 的区别**：WHERE 过滤的是原始行（分组前），HAVING 过滤的是分组后的结果（可以用聚合函数）。

```sql
-- 错误：WHERE 里不能用聚合函数
SELECT user_id, COUNT(*) AS cnt FROM articles GROUP BY user_id WHERE cnt > 3;

-- 正确：用 HAVING
SELECT user_id, COUNT(*) AS cnt FROM articles GROUP BY user_id HAVING cnt > 3;
```

## 窗口函数：不压缩行的聚合

聚合函数（COUNT、SUM 等）会把多行"压缩"成一行。窗口函数也是做聚合，但**不压缩行**——每一行都有结果。

听起来有点抽象，看个例子就明白了。

### ROW_NUMBER：排名

阅读量 TOP 10 的文章：

```sql
SELECT
  id, title, view_count,
  ROW_NUMBER() OVER (ORDER BY view_count DESC) AS rank
FROM articles
WHERE status = 'published' AND deleted_at IS NULL
LIMIT 10;
```

`ROW_NUMBER() OVER (ORDER BY view_count DESC)` 做的事很简单：按 view_count 降序排列，给每行一个序号（1, 2, 3...）。

### RANK 和 DENSE_RANK

ROW_NUMBER 是严格的 1, 2, 3...，即使两行值相同也分配不同序号。RANK 和 DENSE_RANK 处理"并列"的情况：

```sql
SELECT
  title, view_count,
  RANK() OVER (ORDER BY view_count DESC) AS rnk,
  DENSE_RANK() OVER (ORDER BY view_count DESC) AS dense_rnk,
  ROW_NUMBER() OVER (ORDER BY view_count DESC) AS row_num
FROM articles
WHERE status = 'published' AND deleted_at IS NULL;
```

假设阅读量是 1000, 800, 800, 500：

| view_count | rnk | dense_rnk | row_num |
| ---------- | --- | --------- | ------- |
| 1000       | 1   | 1         | 1       |
| 800        | 2   | 2         | 2       |
| 800        | 2   | 2         | 3       |
| 500        | 4   | 3         | 4       |

- **RANK**：并列第 2，下一个跳到第 4（1, 2, 2, 4）
- **DENSE_RANK**：并列第 2，下一个是第 3（1, 2, 2, 3）
- **ROW_NUMBER**：不管并列，严格递增（1, 2, 3, 4）

大部分排名场景用 DENSE_RANK 更合理。

### LAG 和 LEAD：访问前一行 / 后一行

"本月发文数比上个月增长了多少？"LAG 函数让你访问前一行的数据：

```sql
SELECT
  month,
  article_count,
  LAG(article_count, 1) OVER (ORDER BY month) AS prev_month,
  article_count - LAG(article_count, 1) OVER (ORDER BY month) AS diff
FROM (
  SELECT
    DATE_FORMAT(created_at, '%Y-%m') AS month,
    COUNT(*) AS article_count
  FROM articles
  WHERE status = 'published' AND deleted_at IS NULL
  GROUP BY DATE_FORMAT(created_at, '%Y-%m')
) monthly
ORDER BY month;
```

`LAG(article_count, 1)` 取前一行的 article_count。第一行没有"前一行"，结果是 NULL。

LEAD 反过来，取下一行的数据。用法一样，把 LAG 换成 LEAD 就行。

### SUM OVER：累计值

"从开博到现在，累计发了多少篇文章？"SUM OVER 搞定：

```sql
SELECT
  month,
  article_count,
  SUM(article_count) OVER (ORDER BY month) AS cumulative_count
FROM (
  SELECT
    DATE_FORMAT(created_at, '%Y-%m') AS month,
    COUNT(*) AS article_count
  FROM articles
  WHERE status = 'published' AND deleted_at IS NULL
  GROUP BY DATE_FORMAT(created_at, '%Y-%m')
) monthly
ORDER BY month;
```

结果：

```
month    | article_count | cumulative_count
---------|---------------|----------------
2025-01  | 2             | 2
2025-02  | 3             | 5
2025-03  | 5             | 10
2025-04  | 3             | 13
```

`SUM(article_count) OVER (ORDER BY month)` 的意思是：按 month 排序，从第一行累加到当前行。

## CTE：让复杂查询更可读

上面的窗口函数例子里，有一个子查询套在外面。如果查询再复杂一点，层层嵌套会很难读。CTE（WITH 子句）能解决这个问题：

```sql
WITH monthly AS (
  -- 先算每月发文数
  SELECT
    DATE_FORMAT(created_at, '%Y-%m') AS month,
    COUNT(*) AS article_count
  FROM articles
  WHERE status = 'published' AND deleted_at IS NULL
  GROUP BY DATE_FORMAT(created_at, '%Y-%m')
)
SELECT
  month,
  article_count,
  SUM(article_count) OVER (ORDER BY month) AS cumulative_count,
  LAG(article_count, 1) OVER (ORDER BY month) AS prev_month
FROM monthly
ORDER BY month;
```

CTE 的好处是**自上而下**的阅读顺序：先定义 monthly（每月数据），再用它算累计值和环比。比嵌套子查询（从内向外读）直观得多。

CTE 还可以定义多个，用逗号分隔：

```sql
WITH monthly AS (
  SELECT
    DATE_FORMAT(created_at, '%Y-%m') AS month,
    COUNT(*) AS article_count
  FROM articles
  WHERE status = 'published' AND deleted_at IS NULL
  GROUP BY DATE_FORMAT(created_at, '%Y-%m')
),
with_prev AS (
  SELECT
    month,
    article_count,
    LAG(article_count, 1) OVER (ORDER BY month) AS prev_month
  FROM monthly
)
SELECT
  month,
  article_count,
  prev_month,
  CASE
    WHEN prev_month IS NULL THEN NULL
    WHEN prev_month = 0 THEN NULL
    ELSE ROUND((article_count - prev_month) * 100.0 / prev_month, 1)
  END AS growth_rate
FROM with_prev
ORDER BY month;
```

这个查询算出了每月发文量的环比增长率。每一步的逻辑都很清晰，不用来回翻看嵌套层级。

## 什么时候用窗口函数，什么时候用 GROUP BY

| 需求                    | 用什么                | 原因                           |
| ----------------------- | --------------------- | ------------------------------ |
| 每月发文数              | GROUP BY              | 按月分组，压缩成一行一个月     |
| 文章阅读量排名          | 窗口函数 (ROW_NUMBER) | 每篇文章都要显示，不压缩       |
| 每月发文数 + 环比增长率 | 窗口函数 (LAG)        | 需要访问前一行的数据           |
| 累计发文数              | 窗口函数 (SUM OVER)   | 需要累加前面的值               |
| 每个标签的文章数        | GROUP BY              | 按标签分组，压缩成一行一个标签 |

简单判断：**结果行数和原始行数一样** → 窗口函数；**结果行数变少（分组）** → GROUP BY。

## 聚合和窗口函数速查

| 函数 / 语法         | 作用               | 示例场景            |
| ------------------- | ------------------ | ------------------- |
| COUNT(\*)           | 计数               | 文章总数            |
| SUM(col)            | 求和               | 总阅读量            |
| AVG(col)            | 平均值             | 平均阅读量          |
| MAX(col) / MIN(col) | 最大 / 最小        | 最高阅读量          |
| GROUP BY            | 分组聚合           | 每月发文数          |
| HAVING              | 过滤分组结果       | 发文超过 3 篇的作者 |
| ROW_NUMBER() OVER   | 排名（严格递增）   | TOP 10 文章         |
| DENSE_RANK() OVER   | 排名（并列不跳号） | 阅读量排名          |
| LAG(col, n) OVER    | 访问前第 n 行      | 环比增长率          |
| SUM(col) OVER       | 累计值             | 累计发文数          |
| WITH ... AS         | CTE，让查询更可读  | 复杂的多步统计      |
