# 实战：从零构建一个智能 Agent

在前面的文章中，我们学习了使用各种框架和 SDK 来构建 Agent。这确实能快速上手，但有时候会让我们忽略底层的原理。

今天我们换个方式。不依赖 LangChain、不依赖 Vercel AI SDK，而是从零开始手写一个 Agent。通过这个过程，你会深入理解 Agent 的核心工作原理。

## Agent 的核心是什么

剥去所有框架的外衣，Agent 的核心其实很简单：一个循环，不断重复"感知-决策-行动"的过程。

```typescript
// 最简化的 Agent 循环
while (任务未完成) {
  // 1. 感知：获取当前状态
  状态 = 获取状态();

  // 2. 决策：基于状态决定下一步行动
  决策 = LLM.思考(任务, 状态, 历史记录);

  // 3. 行动：执行决策
  结果 = 执行(决策);

  // 4. 更新历史
  历史记录.添加(决策, 结果);

  // 5. 检查是否完成
  if (决策.类型 === "完成") {
    break;
  }
}
```

这就是 Agent 的本质。所有框架都是在基础上的封装和优化。理解了这个核心，使用任何框架都会得心应手。

## 从零实现：基础框架

让我们开始实现。先定义一些基础类型：

```typescript
// lib/agent/types.ts

// 消息类型
export interface Message {
  role: "system" | "user" | "assistant";
  content: string;
}

// 工具定义
export interface Tool {
  name: string;
  description: string;
  parameters: z.ZodType<any>;
  execute: (params: any) => Promise<any>;
}

// Agent 思考结果
export interface AgentThought {
  reasoning: string; // 推理过程
  action: string | null; // 要执行的工具名
  actionInput: any; // 工具参数
  isFinal: boolean; // 是否最终答案
}

// Agent 配置
export interface AgentConfig {
  llm: LLM;
  tools: Tool[];
  systemPrompt: string;
  maxIterations: number;
}

// LLM 接口（抽象）
export interface LLM {
  chat(messages: Message[]): Promise<string>;
}
```

现在实现核心的 Agent 类：

```typescript
// lib/agent/agent.ts

export class Agent {
  private llm: LLM;
  private tools: Map<string, Tool>;
  private systemPrompt: string;
  private maxIterations: number;

  constructor(config: AgentConfig) {
    this.llm = config.llm;
    this.tools = new Map(config.tools.map((t) => [t.name, t]));
    this.systemPrompt = config.systemPrompt;
    this.maxIterations = config.maxIterations;
  }

  async run(task: string): Promise<string> {
    const messages: Message[] = [
      { role: "system", content: this.systemPrompt },
      { role: "user", content: task },
    ];

    let iterations = 0;

    while (iterations < this.maxIterations) {
      // 1. 让 LLM 思考
      const thought = await this.think(messages);

      // 2. 检查是否是最终答案
      if (thought.isFinal) {
        messages.push({
          role: "assistant",
          content: thought.reasoning,
        });
        return thought.reasoning;
      }

      // 3. 执行工具
      if (thought.action && this.tools.has(thought.action)) {
        const tool = this.tools.get(thought.action)!;

        try {
          const result = await tool.execute(thought.actionInput);

          // 记录工具调用和结果
          messages.push({
            role: "assistant",
            content: `我执行了 ${thought.action}，结果是：${JSON.stringify(result)}`,
          });
        } catch (error) {
          messages.push({
            role: "assistant",
            content: `执行 ${thought.action} 时出错：${error}`,
          });
        }
      } else {
        // LLM 决定不使用工具
        messages.push({
          role: "assistant",
          content: thought.reasoning,
        });
      }

      iterations++;
    }

    // 达到最大迭代次数
    return "抱歉，我无法在规定次数内完成这个任务。";
  }

  private async think(messages: Message[]): Promise<AgentThought> {
    const prompt = this.buildThinkPrompt(messages);
    const response = await this.llm.chat([
      { role: "system", content: this.systemPrompt },
      { role: "user", content: prompt },
    ]);

    return this.parseThought(response);
  }

  private buildThinkPrompt(messages: Message[]): string {
    // 构建提示词，包含可用工具列表
    const toolsList = Array.from(this.tools.values())
      .map((t) => `- ${t.name}: ${t.description}`)
      .join("\n");

    return `当前任务：${messages[messages.length - 1].content}

可用工具：
${toolsList}

请分析当前情况，决定下一步行动。输出 JSON 格式：
{
  "reasoning": "你的推理过程",
  "action": "要使用的工具名（或 null）",
  "actionInput": {工具参数},
  "isFinal": true/false
}`;
  }

  private parseThought(response: string): AgentThought {
    try {
      const parsed = JSON.parse(response);
      return {
        reasoning: parsed.reasoning || "",
        action: parsed.action || null,
        actionInput: parsed.actionInput || {},
        isFinal: parsed.isFinal || false,
      };
    } catch {
      // 如果解析失败，返回默认思考
      return {
        reasoning: response,
        action: null,
        actionInput: {},
        isFinal: true,
      };
    }
  }
}
```

这个实现展示了 Agent 的核心逻辑。现在让我们实现一个具体的 LLM：

```typescript
// lib/llm/openai.ts

import { OpenAI } from "openai";
import { Message, LLM } from "../agent/types";

export class OpenAILLM implements LLM {
  private client: OpenAI;
  private model: string;

  constructor(apiKey: string, model: string = "gpt-4o-mini") {
    this.client = new OpenAI({ apiKey });
    this.model = model;
  }

  async chat(messages: Message[]): Promise<string> {
    const response = await this.client.chat.completions.create({
      model: this.model,
      messages: messages.map((m) => ({
        role: m.role,
        content: m.content,
      })),
      temperature: 0.7,
    });

    return response.choices[0].message.content || "";
  }
}
```

## 实战：代码仓库分析 Agent（从零版）

现在用我们的框架实现代码仓库分析 Agent。这个 Agent 能够：

- 获取仓库信息
- 分析代码结构
- 检测代码质量
- 生成分析报告

```typescript
// lib/tools/repo-tools.ts

import { Tool } from "../agent/types";
import { z } from "zod";

// GitHub API 工具
export const getRepoInfo: Tool = {
  name: "getRepoInfo",
  description: "获取 GitHub 仓库的基本信息（描述、星标数、语言等）",
  parameters: z.object({
    owner: z.string().describe("仓库所有者"),
    repo: z.string().describe("仓库名称"),
  }),
  execute: async ({ owner, repo }) => {
    const response = await fetch(
      `https://api.github.com/repos/${owner}/${repo}`
    );

    if (!response.ok) {
      throw new Error(`GitHub API 错误: ${response.status}`);
    }

    const data = await response.json();

    return {
      name: data.full_name,
      description: data.description,
      stars: data.stargazers_count,
      forks: data.forks_count,
      language: data.language,
      openIssues: data.open_issues_count,
      createdAt: data.created_at,
      updatedAt: data.updated_at,
    };
  },
};

// 获取代码结构工具
export const getCodeStructure: Tool = {
  name: "getCodeStructure",
  description: "获取仓库的文件和目录结构",
  parameters: z.object({
    owner: z.string(),
    repo: z.string(),
    path: z.string().optional().default(""),
  }),
  execute: async ({ owner, repo, path }) => {
    const response = await fetch(
      `https://api.github.com/repos/${owner}/${repo}/contents/${path}`
    );

    if (!response.ok) {
      throw new Error(`GitHub API 错误: ${response.status}`);
    }

    const data = await response.json();

    // 如果是数组，返回文件列表；如果是单个文件，返回文件信息
    return Array.isArray(data)
      ? data.map((item: any) => ({
          name: item.name,
          type: item.type,
          path: item.path,
        }))
      : [data];
  },
};

// 获取语言统计工具
export const getLanguageStats: Tool = {
  name: "getLanguageStats",
  description: "获取仓库的编程语言分布",
  parameters: z.object({
    owner: z.string(),
    repo: z.string(),
  }),
  execute: async ({ owner, repo }) => {
    const response = await fetch(
      `https://api.github.com/repos/${owner}/${repo}/languages`
    );

    if (!response.ok) {
      throw new Error(`GitHub API 错误: ${response.status}`);
    }

    const data = await response.json();

    // 转换为数组格式，方便排序
    return Object.entries(data)
      .map(([language, bytes]) => ({ language, bytes }))
      .sort((a, b) => b.bytes - a.bytes);
  },
};

// 获取 README 工具
export const getReadme: Tool = {
  name: "getReadme",
  description: "获取仓库的 README 文件内容",
  parameters: z.object({
    owner: z.string(),
    repo: z.string(),
  }),
  execute: async ({ owner, repo }) => {
    const response = await fetch(
      `https://api.github.com/repos/${owner}/${repo}/readme`
    );

    if (!response.ok) {
      return null; // 有些仓库可能没有 README
    }

    const data = await response.json();

    // GitHub API 返回的是 base64 编码的内容
    const content = atob(data.content);

    return {
      content: content.substring(0, 5000), // 限制长度
    };
  },
};
```

现在创建并运行 Agent：

```typescript
// app/api/analyze-scratch/route.ts

import { Agent } from "@/lib/agent/agent";
import { OpenAILLM } from "@/lib/llm/openai";
import {
  getRepoInfo,
  getCodeStructure,
  getLanguageStats,
  getReadme,
} from "@/lib/tools/repo-tools";

export async function POST(req: Request) {
  const { repoUrl } = await req.json();

  // 解析 URL
  const match = repoUrl.match(/github\.com\/([^/]+)\/([^/]+)/);
  if (!match) {
    return Response.json({ error: "无效的 GitHub URL" }, { status: 400 });
  }

  const [, owner, repo] = match;

  // 创建 LLM
  const llm = new OpenAILLM(process.env.OPENAI_API_KEY!, "gpt-4o");

  // 创建 Agent
  const agent = new Agent({
    llm,
    tools: [getRepoInfo, getCodeStructure, getLanguageStats, getReadme],
    systemPrompt: `你是一个专业的代码仓库分析专家。

你的任务是深入分析 GitHub 仓库，提供有价值的见解。分析时：
1. 先获取基本信息（描述、星标数等）
2. 了解代码结构
3. 分析语言分布
4. 阅读 README 了解项目背景

基于收集的信息，生成一份全面的分析报告。报告应该包括：
- 项目概览
- 技术栈分析
- 代码质量评估
- 社区活跃度
- 改进建议

请使用专业的语气，基于数据进行分析。`,
    maxIterations: 10,
  });

  try {
    // 运行 Agent
    const result = await agent.run(
      `请深入分析 ${owner}/${repo} 这个 GitHub 仓库。`
    );

    return Response.json({
      analysis: result,
      repo: `${owner}/${repo}`,
      timestamp: new Date().toISOString(),
    });
  } catch (error) {
    console.error("Agent 执行失败:", error);
    return Response.json({ error: "分析失败，请稍后重试" }, { status: 500 });
  }
}
```

## 添加记忆功能

真实的 Agent 需要记住之前的信息。让我们添加记忆功能：

```typescript
// lib/agent/memory.ts

export interface MemoryEntry {
  timestamp: number;
  key: string;
  value: any;
}

export class AgentMemory {
  private storage: Map<string, MemoryEntry[]> = new Map();
  private maxSize: number;

  constructor(maxSize: number = 100) {
    this.maxSize = maxSize;
  }

  // 存储信息
  store(sessionId: string, key: string, value: any): void {
    if (!this.storage.has(sessionId)) {
      this.storage.set(sessionId, []);
    }

    const entries = this.storage.get(sessionId)!;
    entries.push({
      timestamp: Date.now(),
      key,
      value,
    });

    // 限制大小
    if (entries.length > this.maxSize) {
      entries.shift(); // 移除最旧的条目
    }
  }

  // 检索信息
  retrieve(sessionId: string, key?: string): any[] {
    const entries = this.storage.get(sessionId) || [];

    if (key) {
      return entries.filter((e) => e.key === key).map((e) => e.value);
    }

    return entries.map((e) => e.value);
  }

  // 获取最近的信息
  getRecent(sessionId: string, count: number = 5): MemoryEntry[] {
    const entries = this.storage.get(sessionId) || [];
    return entries.slice(-count);
  }

  // 清空会话
  clear(sessionId: string): void {
    this.storage.delete(sessionId);
  }

  // 生成摘要（用于上下文窗口有限的情况）
  summarize(sessionId: string): string {
    const entries = this.storage.get(sessionId) || [];
    if (entries.length === 0) return "";

    // 按键分组
    const grouped = new Map<string, any[]>();
    entries.forEach((e) => {
      if (!grouped.has(e.key)) {
        grouped.set(e.key, []);
      }
      grouped.get(e.key)!.push(e.value);
    });

    // 生成摘要
    const summary = Array.from(grouped.entries())
      .map(([key, values]) => `${key}: ${JSON.stringify(values[0])}`)
      .join("\n");

    return summary;
  }
}
```

将记忆集成到 Agent 中：

```typescript
// 修改 Agent 类，添加记忆功能

export class AgentWithMemory extends Agent {
  private memory: AgentMemory;

  constructor(config: AgentConfig & { memory: AgentMemory }) {
    super(config);
    this.memory = config.memory;
  }

  async run(task: string, sessionId: string): Promise<string> {
    // 检查是否有相关记忆
    const relevantMemory = this.memory.retrieve(sessionId);

    // 构建包含记忆的提示
    const memoryContext =
      relevantMemory.length > 0
        ? `\n\n相关信息：\n${this.memory.summarize(sessionId)}`
        : "";

    const result = await super.run(task + memoryContext);

    // 存储这次交互
    this.memory.store(sessionId, "last_task", task);
    this.memory.store(sessionId, "last_result", result);

    return result;
  }

  // 可以添加更多智能的记忆管理方法
  async shouldUseMemory(task: string, sessionId: string): Promise<boolean> {
    // 简单的启发式：如果任务包含"之前"、"上次"等词，使用记忆
    const memoryKeywords = ["之前", "上次", "继续", "接着"];
    return memoryKeywords.some((keyword) => task.includes(keyword));
  }
}
```

## 从零实现的权衡

从零实现 Agent 让我们理解了核心原理，但也需要权衡利弊。

优势方面，你完全掌控每个细节，可以针对性地优化。没有黑盒，调试更容易。理解更深刻，使用框架时不会被抽象迷惑。

局限也很明显。需要处理的细节很多，容易出错。缺少框架提供的优化（如并行执行、缓存等）。维护成本高，需要自己处理各种边界情况。

实际项目中，建议在学习阶段从零实现一两个 Agent，理解原理。然后用成熟的框架（如 LangChain）来构建生产级应用，享受生态和工具支持。

## 核心原理总结

通过从零实现，我们理解了 Agent 的几个核心原理：

首先是循环执行。Agent 本质是一个不断"思考-行动"的循环，直到任务完成。其次是工具调用。通过提示词让 LLM 理解工具的用途和参数，然后解析其决策执行工具。再次是状态管理。Agent 需要跟踪执行过程，包括历史、中间结果等。最后是记忆机制。长期记忆让 Agent 能够跨会话利用经验。

这些原理在任何框架中都是一样的。框架只是提供了更方便的 API 和更多的功能。
