## 前言

随着业务规模的增长，单体应用逐渐暴露出扩展性差、部署困难、技术栈受限等问题。微服务架构通过将系统拆分为多个独立、自治的服务，能够有效应对这些挑战。Node.js 凭借其轻量级、高性能、生态丰富的特点，成为构建微服务的理想选择。

本文将带你从零开始构建一个完整的 Node.js 微服务系统，涵盖架构设计、服务拆分、通信机制、数据一致性、监控运维等核心环节。

## 一、微服务架构设计原则

### 1.1 什么是微服务

微服务是一种架构风格，将应用构建为一组小型、自治的服务，每个服务：

- **单一职责**：专注于一个业务能力
- **独立部署**：可以独立开发、测试、部署
- **松耦合**：服务间通过 API 通信，最小化依赖
- **技术异构**：可以使用不同的技术栈

### 1.2 何时选择微服务

微服务并非银弹，适合以下场景：

```
✅ 业务复杂度高，团队规模大
✅ 需要独立扩展不同模块
✅ 要求快速迭代和部署
✅ 技术栈需要多样化

❌ 团队规模小（&lt; 10人）
❌ 业务逻辑简单
❌ 对延迟极其敏感
❌ 运维能力不足
```

### 1.3 服务拆分策略

**按业务能力拆分**（推荐）：

```
电商系统拆分示例：
├── 用户服务（User Service）
├── 商品服务（Product Service）
├── 订单服务（Order Service）
├── 支付服务（Payment Service）
├── 库存服务（Inventory Service）
```

### 3.2 异步通信（消息队列）

**RabbitMQ 配置**：

```typescript
// shared/messaging/rabbitmq.ts
import amqp from 'amqplib';

export class MessageQueue {
  private connection: amqp.Connection;
  private channel: amqp.Channel;

  async connect() {
    this.connection = await amqp.connect(
      process.env.RABBITMQ_URL || 'amqp://localhost:5672'
    );
    this.channel = await this.connection.createChannel();

    // 设置 prefetch，避免消费者过载
    await this.channel.prefetch(10);

    console.log('Connected to RabbitMQ');
  }

  async publish(exchange: string, routingKey: string, message: any) {
    await this.channel.assertExchange(exchange, 'topic', { durable: true });

    this.channel.publish(
      exchange,
      routingKey,
      Buffer.from(JSON.stringify(message)),
      { persistent: true }
    );
  }

  async subscribe(
    exchange: string,
    queue: string,
    routingKeys: string[],
    callback: (message: any) => Promise&lt;void&gt;
  ) {
    await this.channel.assertExchange(exchange, 'topic', { durable: true });

    const q = await this.channel.assertQueue(queue, { durable: true });

    routingKeys.forEach(key => {
      this.channel.bindQueue(q.queue, exchange, key);
    });

    this.channel.consume(q.queue, async (msg) => {
      if (!msg) return;

      try {
        const content = JSON.parse(msg.content.toString());
        await callback(content);
        this.channel.ack(msg);
      } catch (error) {
        console.error('Message processing failed:', error);
        // 重试 3 次后拒绝
        if (msg.fields.redeliveryCount &gt;= 3) {
          this.channel.reject(msg, false);
        } else {
          this.channel.nack(msg, false, true);
        }
      }
    });
  }

  async close() {
    await this.channel.close();
    await this.connection.close();
  }
}
```

```typescript
import * as protoLoader from '@grpc/proto-loader';
import path from 'path';

const PROTO_PATH = path.join(__dirname, '../../proto/product.proto');

const packageDefinition = protoLoader.loadSync(PROTO_PATH, {
  keepCase: true,
  longs: String,
  enums: String,
  defaults: true,
  oneofs: true,
});

const proto = grpc.loadPackageDefinition(packageDefinition).product;

export class ProductGRPCServer {
  private server: grpc.Server;

  constructor() {
    this.server = new grpc.Server();
  }

  start(port: number) {
    this.server.addService((proto as any).ProductService.service, {
      GetProduct: this.getProduct.bind(this),
      CheckStock: this.checkStock.bind(this),
      UpdateStock: this.updateStock.bind(this),
    });

    this.server.bindAsync(
      `0.0.0.0:${port}`,
      grpc.ServerCredentials.createInsecure(),
      (err, port) => {
        if (err) {
          console.error('Failed to start gRPC server:', err);
          return;
        }
        console.log(`gRPC server running on port ${port}`);
      }
    );
  }

  private async getProduct(call: any, callback: any) {
    const { product_id } = call.request;
    const product = await ProductModel.findById(product_id);

    callback(null, {
      product_id: product.id,
      name: product.name,
      price: product.price,
      stock: product.stock,
    });
  }

  private async checkStock(call: any, callback: any) {
    const { product_id, quantity } = call.request;
    const product = await ProductModel.findById(product_id);

    callback(null, {
      in_stock: product.stock &gt;= quantity,
    });
  }

  private async updateStock(call: any, callback: any) {
    const { product_id, quantity, operation } = call.request;
    const product = await ProductModel.findById(product_id);

    if (operation === 'decrease') {
      product.stock -= quantity;
    } else {
      product.stock += quantity;
    }

    await product.save();

    callback(null, {
      success: true,
      new_stock: product.stock,
    });
  }
}
```

```typescript
// gateway/src/index.ts
import express from 'express';
import { createProxyMiddleware } from 'http-proxy-middleware';
import jwt from 'jsonwebtoken';
import rateLimit from 'express-rate-limit';

const app = express();

// 限流中间件
const limiter = rateLimit({
  windowMs: 60 * 1000,
  max: 100,
  message: 'Too many requests from this IP',
});

app.use(limiter);

// JWT 认证中间件
const authMiddleware = (req: any, res: any, next: any) => {
  const token = req.headers.authorization?.split(' ')[1];

  if (!token) {
    return res.status(401).json({ error: 'No token provided' });
  }

  try {
    const decoded = jwt.verify(token, process.env.JWT_SECRET!);
    req.user = decoded;
    next();
  } catch (error) {
    return res.status(401).json({ error: 'Invalid token' });
  }
};

// 路由配置
const routes = {
  '/api/users': {
    target: 'http://user-service:3001',
    auth: true,
  },
  '/api/products': {
    target: 'http://product-service:3002',
    auth: false,
  },
  '/api/orders': {
    target: 'http://order-service:3003',
    auth: true,
  },
};

// 设置代理
Object.entries(routes).forEach(([path, config]) =&gt; {
  const middleware = [createProxyMiddleware({
    target: config.target,
    changeOrigin: true,
    pathRewrite: { [`^${path}`]: '' },
    onProxyReq: (proxyReq, req, res) =&gt; {
      // 添加请求头
      if ((req as any).user) {
        proxyReq.setHeader('X-User-Id', (req as any).user.id);
      }
    },
  })];

  if (config.auth) {
    middleware.unshift(authMiddleware as any);
  }

  app.use(path, ...middleware);
});

app.listen(8080, () =&gt; {
  console.log('API Gateway running on port 8080');
});
```

## 五、数据一致性

### 5.1 分布式事务的挑战

微服务架构下，每个服务有独立数据库，传统 ACID 事务难以保证跨服务的数据一致性。

### 5.2 Saga 模式

**编排式 Saga**：

```typescript
// services/order-service/src/sagas/createOrderSaga.ts
export class CreateOrderSaga {
  private steps: SagaStep[] = [];
  private completedSteps: SagaStep[] = [];

  async execute(orderData: any) {
    // Step 1: 创建订单（Pending 状态）
    const order = await this.createOrder(orderData);
    this.steps.push({ name: 'createOrder', data: order });

    try {
      // Step 2: 检查库存
      const stockAvailable = await this.checkStock(order.items);
      if (!stockAvailable) {
        throw new Error('Insufficient stock');
      }
      this.steps.push({ name: 'checkStock', data: order.items });

      // Step 3: 预留库存
      await this.reserveStock(order.items);
      this.steps.push({ name: 'reserveStock', data: order.items });

      // Step 4: 扣款
      await this.processPayment(order);
      this.steps.push({ name: 'processPayment', data: order });

      // Step 5: 确认订单
      await this.confirmOrder(order.id);
      this.steps.push({ name: 'confirmOrder', data: order.id });

      return { success: true, order };
    } catch (error) {
      // 补偿事务
      await this.compensate();
      return { success: false, error: error.message };
    }
  }

  private async compensate() {
    // 按相反顺序执行补偿操作
    for (let i = this.steps.length - 1; i &gt;= 0; i--) {
      const step = this.steps[i];

      try {
        switch (step.name) {
          case 'reserveStock':
            await this.releaseStock(step.data);
            break;
          case 'processPayment':
            await this.refundPayment(step.data);
            break;
          case 'createOrder':
            await this.cancelOrder(step.data.id);
            break;
        }
      } catch (error) {
        console.error(`Compensation failed for step ${step.name}:`, error);
      }
    }
  }

  private async createOrder(data: any) {
    return await OrderModel.create({
      ...data,
      status: 'PENDING',
    });
  }

  private async checkStock(items: any[]) {
    for (const item of items) {
      const product = await ProductClient.getProduct(item.productId);
      if (product.stock &lt; item.quantity) {
        return false;
      }
    }
    return true;
  }

  private async reserveStock(items: any[]) {
    await MessageQueue.publish('inventory.commands', 'reserve.stock', {
      items,
      reservationId: generateId(),
    });
  }

  private async releaseStock(items: any[]) {
    await MessageQueue.publish('inventory.commands', 'release.stock', { items });
  }

  private async processPayment(order: any) {
    await PaymentClient.charge(order.userId, order.totalAmount);
  }

  private async refundPayment(order: any) {
    await PaymentClient.refund(order.paymentId);
  }

  private async confirmOrder(orderId: string) {
    await OrderModel.findByIdAndUpdate(orderId, { status: 'CONFIRMED' });
  }

  private async cancelOrder(orderId: string) {
    await OrderModel.findByIdAndUpdate(orderId, { status: 'CANCELLED' });
  }
}
```

```typescript
// 订单聚合根
export class OrderAggregate {
  private events: Event[] = [];
  private state: any = {};

  async create(data: any) {
    const event: Event = {
      id: generateId(),
      type: 'OrderCreated',
      aggregateId: data.id,
      data,
      timestamp: new Date(),
      version: 1,
    };

    await this.eventStore.save(event);
    this.events.push(event);
    this.apply(event);
  }

  async confirm() {
    const event: Event = {
      id: generateId(),
      type: 'OrderConfirmed',
      aggregateId: this.state.id,
      data: { confirmedAt: new Date() },
      timestamp: new Date(),
      version: this.state.version + 1,
    };

    await this.eventStore.save(event);
    this.events.push(event);
    this.apply(event);
  }

  private apply(event: Event) {
    switch (event.type) {
      case 'OrderCreated':
        this.state = {
          id: event.aggregateId,
          ...event.data,
          version: event.version,
        };
        break;
      case 'OrderConfirmed':
        this.state.status = 'CONFIRMED';
        this.state.confirmedAt = event.data.confirmedAt;
        this.state.version = event.version;
        break;
    }
  }

  async loadFromHistory(aggregateId: string) {
    const events = await this.eventStore.getEvents(aggregateId);
    events.forEach(event =&gt; this.apply(event));
```

### 6.1 Consul 集成

```typescript
// shared/serviceDiscovery/consul.ts
import Consul from 'consul';

export class ServiceRegistry {
  private consul: Consul;

  constructor() {
    this.consul = new Consul({
      host: process.env.CONSUL_HOST || 'localhost',
      port: '8500',
    });
  }

  async register(service: ServiceConfig) {
    await this.consul.agent.service.register({
      name: service.name,
      address: service.address,
      port: service.port,
      check: {
        http: `http://${service.address}:${service.port}/health`,
        interval: '10s',
        timeout: '5s',
      },
    });

    console.log(`Service ${service.name} registered`);
  }

  async deregister(serviceName: string) {
    await this.consul.agent.service.deregister(serviceName);
    console.log(`Service ${serviceName} deregistered`);
  }

  async discover(serviceName: string): Promise&lt;ServiceInstance[]&gt; {
    const services = await this.consul.catalog.service.nodes(serviceName);

    return services.map(service =&gt; ({
      address: service.ServiceAddress,
      port: service.ServicePort,
    }));
  }

  async getHealthyInstance(serviceName: string): Promise&lt;ServiceInstance&gt; {
    const instances = await this.discover(serviceName);

    if (instances.length === 0) {
      throw new Error(`No healthy instances for service ${serviceName}`);
    }

    // 简单轮询负载均衡
    const index = Math.floor(Math.random() * instances.length);
    return instances[index];
  }
}

// 使用示例
const registry = new ServiceRegistry();

// 服务启动时注册
await registry.register({
  name: 'user-service',
  address: '192.168.1.100',
  port: 3001,
});

// 调用其他服务时发现
```

### 7.1 分布式追踪（Jaeger）

```typescript
// shared/tracing/jaeger.ts
import * as jaeger from 'jaeger-client';
import { Middleware } from 'express';

export class TracingMiddleware {
  private tracer: jaeger.Tracer;

  constructor(serviceName: string) {
    const config = {
      serviceName,
      sampler: {
        type: 'const',
        param: 1,
      },
      reporter: {
        logSpans: true,
        agentHost: process.env.JAEGER_AGENT_HOST || 'localhost',
        agentPort: 6832,
      },
    };

    this.tracer = jaeger.initTracer(config, {
      logger: console,
    });
  }

  middleware(): Middleware {
    return (req: any, res: any, next: any) =&gt; {
      const span = this.tracer.startSpan(`${req.method} ${req.path}`);

      // 从请求头提取上下文
      const parentSpanContext = this.tracer.extract(
        jaeger.FORMAT_HTTP_HEADERS,
        req.headers
      );

      if (parentSpanContext) {
        span.addReference(jaeger.REFERENCES_CHILD_OF, parentSpanContext);
      }

      req.span = span;

      res.on('finish', () =&gt; {
        span.setTag('http.status_code', res.statusCode);
        span.finish();
      });

      next();
    };
  }

  trace(serviceName: string, operation: string) {
    return (target: any, propertyKey: string, descriptor: PropertyDescriptor) =&gt; {
      const originalMethod = descriptor.value;

      descriptor.value = async function (...args: any[]) {
        const span = this.tracer.startSpan(`${serviceName}.${operation}`);

        try {
          const result = await originalMethod.apply(this, args);
          span.setTag('success', true);
          return result;
        } catch (error) {
          span.setTag('error', true);
          span.log({ error: error.message });
          throw error;
        } finally {
          span.finish();
        }
      };

      return descriptor;
    };
  }
}
```

```typescript
// shared/metrics/prometheus.ts
import client from 'prom-client';
import express from 'express';

export class MetricsCollector {
  private register: client.Registry;

  constructor() {
    this.register = new client.Registry();
    client.collectDefaultMetrics({ register: this.register });
  }

  // HTTP 请求计数器
  httpRequestCounter = new client.Counter({
    name: 'http_requests_total',
    help: 'Total number of HTTP requests',
    labelNames: ['method', 'path', 'status'],
    registers: [this.register],
  });

  // HTTP 请求耗时
  httpRequestDuration = new client.Histogram({
    name: 'http_request_duration_seconds',
    help: 'Duration of HTTP requests in seconds',
    labelNames: ['method', 'path', 'status'],
    buckets: [0.1, 0.3, 0.5, 0.7, 1, 3, 5, 7, 10],
    registers: [this.register],
  });

  // 业务指标：订单数量
  orderCounter = new client.Counter({
    name: 'orders_total',
    help: 'Total number of orders',
    labelNames: ['status'],
    registers: [this.register],
  });

  // 业务指标：订单金额
  orderAmount = new client.Gauge({
    name: 'order_amount_total',
    help: 'Total amount of orders',
    registers: [this.register],
  });

  middleware() {
    return (req: any, res: any, next: any) =&gt; {
      const start = Date.now();

      res.on('finish', () =&gt; {
        const duration = (Date.now() - start) / 1000;

        this.httpRequestCounter.inc({
          method: req.method,
          path: req.path,
          status: res.statusCode,
        });

        this.httpRequestDuration.observe(
          {
            method: req.method,
            path: req.path,
            status: res.statusCode,
          },
          duration
        );
      });

      next();
    };
  }

  exposeMetrics(app: express.Application, path = '/metrics') {
    app.get(path, async (req, res) =&gt; {
      res.set('Content-Type', this.register.contentType);
      res.end(await this.register.metrics());
    });
  }
}

// 使用示例
const metrics = new MetricsCollector();

app.use(metrics.middleware());
metrics.exposeMetrics(app);

// 在业务代码中记录指标
async function createOrder(data: any) {
  const order = await OrderModel.create(data);

  metrics.orderCounter.inc({ status: 'created' });
  metrics.orderAmount.inc(order.totalAmount);

  return order;
}
```

## 九、最佳实践与避坑指南

### 9.1 服务粒度

```
❌ 过粗：一个服务包含多个业务能力
✅ 适中：一个服务对应一个业务能力
❌ 过细：每个功能一个服务

判断标准：
- 团队规模：2 Pizza Team（6-10人）维护 2-5 个服务
- 变更频率：一起变更的功能放在一个服务
- 数据耦合：强关联的数据放在一个服务
```

### 9.2 数据库策略

```
✅ 每个服务独立数据库
✅ 使用事件同步数据
✅ API 组合查询（CQRS）
❌ 共享数据库
❌ 跨服务 JOIN 查询
```

### 9.3 错误处理

### 9.4 配置管理

```typescript
// 使用环境变量 + 配置中心
import { ConsulKV } from 'consul';

export class ConfigService {
  private consul: ConsulKV;
  private cache: Map&lt;string, any&gt; = new Map();

  async get(key: string): Promise&lt;any&gt; {
    // 优先从缓存读取
    if (this.cache.has(key)) {
      return this.cache.get(key);
    }

    // 从 Consul 读取
    const result = await this.consul.get(key);
    this.cache.set(key, result.Value);

    return result.Value;
  }

  // 监听配置变更
  async watch(key: string, callback: (value: any) =&gt; void) {
    await this.consul.watch({
      method: this.consul.get,
      options: { key },
    }).on('change', (data) =&gt; {
      this.cache.set(key, data.Value);
      callback(data.Value);
    });
  }
}
```

## 十、总结

Node.js 微服务架构的核心要点：

1. **服务拆分**：按业务能力拆分，避免过细或过粗
2. **通信机制**：同步用 HTTP/gRPC，异步用消息队列
3. **数据一致性**：Saga 模式 + 最终一致性
4. **服务发现**：使用 Consul 等注册中心
5. **可观测性**：日志、指标、追踪三位一体
6. **容错机制**：熔断、限流、降级、重试
7. **容器化**：Docker + Kubernetes 标准化部署

微服务不是目的，而是手段。选择架构时，务必根据团队规模、业务复杂度、运维能力综合评估，避免过度设计。
