前三篇我们构建的 TeamFlow 界面在桌面端已经很好看了。但现实是——你的用户有一半可能在手机上打开它。

侧边栏占满了整个手机屏幕，看板列挤在一起看不清，按钮小到没法点击。这就是响应式设计要解决的问题。不是简单地"缩小一点"，而是针对不同的屏幕空间重新组织信息和交互。

Tailwind 的响应式工具类让这件事变得直接——你不需要写 `@media` 查询，直接在 HTML 里声明不同断点下的样式。

## 移动优先：先写小屏，再扩展大屏

Tailwind 的响应式系统遵循一个核心原则：**移动优先**。

这意味着：

- 不带前缀的工具类（如 `flex`、`text-sm`）默认在所有屏幕生效——它们是移动端的样式
- 带断点前缀的工具类（如 `md:flex`、`lg:text-lg`）只在对应屏幕宽度以上生效——它们是大屏的增强

```html
<!-- 移动端单列，平板双列，桌面三列 -->
<div className="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-4">
  <Card />
  <Card />
  <Card />
</div>
```

理解这个顺序很重要：你先写最小屏幕的样式（基础），然后逐步增强大屏的体验。而不是先写桌面版再"兼容"移动端——那是桌面优先，和 Tailwind 的设计理念相反。

### 断点系统

Tailwind 的断点是 `min-width` 媒体查询：

| 前缀   | 最小宽度 | 典型设备            |
| ------ | -------- | ------------------- |
| `sm:`  | 640px    | 大屏手机（横屏）    |
| `md:`  | 768px    | 平板竖屏            |
| `lg:`  | 1024px   | 平板横屏 / 小笔记本 |
| `xl:`  | 1280px   | 标准笔记本          |
| `2xl:` | 1536px   | 大屏显示器          |

一个容易混淆的点：`md:` 不是"在平板上生效"，而是"在 768px 及以上宽度生效"。所以 `md:flex-row` 意味着"从 768px 开始改为横向排列"，小于 768px 时保持默认的纵向排列。

## 响应式布局模式

### 模式一：隐藏与显示

最常见的响应式操作——某些元素只在大屏显示，某些只在小屏显示：

```tsx
// 导航栏：桌面显示完整菜单，移动端显示汉堡按钮
<nav className="flex items-center justify-between px-4 lg:px-6 h-14">
  <Logo />

  {/* 桌面菜单：移动端隐藏 */}
  <ul className="hidden md:flex items-center gap-6">
    <li>
      <a href="#">仪表盘</a>
    </li>
    <li>
      <a href="#">项目</a>
    </li>
    <li>
      <a href="#">团队</a>
    </li>
  </ul>

  {/* 汉堡按钮：只在移动端显示 */}
  <button className="md:hidden text-slate-600">
    <MenuIcon />
  </button>

  {/* 桌面操作区：移动端隐藏 */}
  <div className="hidden md:flex items-center gap-3">
    <button>新建任务</button>
    <Avatar />
  </div>
</nav>
```

`hidden md:flex` 的含义：默认 `display: none`，在 `md` 及以上断点变成 `display: flex`。反过来，`md:hidden` 表示默认显示，`md` 及以上隐藏。

### 模式二：方向切换

从纵向排列切换为横向排列：

```html
<!-- 按钮组：移动端竖排，桌面横排 -->
<div className="flex flex-col sm:flex-row gap-3">
  <button className="bg-blue-500 px-6 py-3 rounded-lg">确认提交</button>
  <button className="border px-6 py-3 rounded-lg">取消</button>
</div>
```

### 模式三：间距调整

小屏用紧凑间距，大屏放宽：

```html
<!-- 内容区域：移动端 16px 内边距，桌面 24px -->
<main className="px-4 lg:px-6 py-4 lg:py-8">
  <h1 className="text-2xl lg:text-3xl font-bold mb-4 lg:mb-6">项目列表</h1>
</main>
```

### 模式四：网格列数

根据屏幕空间自动调整列数：

```html
<!-- 项目卡片：1列 → 2列 → 3列 → 4列 -->
<div
  className="grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 xl:grid-cols-4 gap-4 lg:gap-6"
>
  {projects.map(project =>
  <ProjectCard key="{project.id}" project="{project}" />)}
</div>
```

### 模式五：表格卡片化

表格在小屏上很难阅读。一个常见的方案是：大屏显示表格，小屏显示卡片列表：

```tsx
// 大屏表格视图
<table className="hidden md:table w-full">
  <thead>
    <tr className="border-b">
      <th className="text-left py-3 text-sm font-medium text-slate-500">任务</th>
      <th className="text-left py-3 text-sm font-medium text-slate-500">状态</th>
      <th className="text-left py-3 text-sm font-medium text-slate-500">负责人</th>
      <th className="text-left py-3 text-sm font-medium text-slate-500">截止日期</th>
    </tr>
  </thead>
  <tbody>
    {tasks.map(task => (
      <tr key={task.id} className="border-b hover:bg-slate-50">
        <td className="py-3 text-sm">{task.title}</td>
        <td className="py-3 text-sm">{task.status}</td>
        <td className="py-3 text-sm">{task.assignee}</td>
        <td className="py-3 text-sm text-slate-500">{task.dueDate}</td>
      </tr>
    ))}
  </tbody>
</table>

// 小屏卡片视图
<div className="md:hidden space-y-3">
  {tasks.map(task => (
    <div key={task.id} className="bg-white rounded-lg p-4 border border-slate-200">
      <h4 className="text-sm font-medium mb-2">{task.title}</h4>
      <div className="flex items-center justify-between text-xs text-slate-500">
        <span>{task.status} · {task.assignee}</span>
        <span>{task.dueDate}</span>
      </div>
    </div>
  ))}
</div>
```

这样做有一个明显的缺点——同样的数据写了两次模板。如果团队有条件，可以用 CSS Grid 的特性让同一套 HTML 在不同屏幕下呈现不同布局。但对于快速迭代，两套模板的方案更清晰、更好维护。

## 实战：TeamFlow 的完整响应式适配

把 TeamFlow 从桌面端适配到移动端，需要解决几个核心问题。

### 侧边栏：固定 → 抽屉

桌面端侧边栏固定显示，移动端收为抽屉菜单：

```tsx
function App() {
  const [sidebarOpen, setSidebarOpen] = useState(false);

  return (
    <div className="flex h-screen bg-slate-100">
      {/* 侧边栏遮罩：移动端打开时显示 */}
      {sidebarOpen && (
        <div
          className="fixed inset-0 bg-black/50 z-40 lg:hidden"
          onClick={() => setSidebarOpen(false)}
        />
      )}

      {/* 侧边栏 */}
      <aside
        className={`fixed lg:static inset-y-0 left-0 z-50 w-64 bg-slate-900 text-slate-300 flex flex-col transition-transform duration-300 ${
          sidebarOpen ? "translate-x-0" : "-translate-x-full lg:translate-x-0"
        }`}
      >
        {/* 侧边栏内容不变 */}
        <SidebarContent onClose={() => setSidebarOpen(false)} />
      </aside>

      {/* 主内容区 */}
      <main className="flex-1 flex flex-col min-w-0">
        {/* 顶部导航栏：移动端显示汉堡按钮 */}
        <header className="h-14 flex items-center justify-between px-4 lg:px-6 bg-white border-b">
          <div className="flex items-center gap-3">
            {/* 汉堡按钮：桌面端隐藏 */}
            <button
              className="lg:hidden text-slate-600"
              onClick={() => setSidebarOpen(true)}
            >
              <MenuIcon />
            </button>
            <h1 className="text-lg font-semibold text-slate-900 truncate">
              官网改版 v3
            </h1>
          </div>
          <div className="flex items-center gap-3">
            <button className="hidden sm:block px-3 py-1.5 text-sm bg-blue-500 text-white rounded-md">
              新建任务
            </button>
            {/* 移动端简化为图标按钮 */}
            <button className="sm:hidden w-8 h-8 bg-blue-500 text-white rounded-md flex items-center justify-center">
              +
            </button>
          </div>
        </header>

        {/* 内容区域 */}
        <div className="flex-1 overflow-hidden p-4 lg:p-6">
          <KanbanBoard />
        </div>
      </main>
    </div>
  );
}
```

几个关键的响应式决策：

**侧边栏**：移动端用 `fixed` + `translate-x` 实现抽屉效果。`-translate-x-full` 让侧边栏滑出屏幕，`translate-x-0` 让它滑回来。`lg:static lg:translate-x-0` 确保桌面端始终显示。

**遮罩层**：`lg:hidden` 确保只在移动端显示。点击遮罩关闭侧边栏，这是移动端抽屉菜单的标准交互模式。

**汉堡按钮**：`lg:hidden` 只在移动端显示，桌面端不需要——侧边栏一直都在。

### 看板：多列 → 水平滚动 → 标签页

桌面端显示三列看板，平板上水平滚动，手机上可能需要更简洁的方案：

```tsx
function KanbanBoard() {
  return (
    <div className="flex gap-4 lg:gap-6 h-full overflow-x-auto pb-4">
      <KanbanColumn title="待办" count={5} tasks={todoTasks} />
      <KanbanColumn title="进行中" count={3} tasks={inProgressTasks} />
      <KanbanColumn title="审核中" count={2} tasks={reviewTasks} />
      <KanbanColumn title="已完成" count={12} tasks={doneTasks} />

      {/* 添加列按钮：移动端隐藏 */}
      <div className="hidden sm:flex w-72 shrink-0 items-start">
        <button className="w-full h-12 border-2 border-dashed border-slate-300 rounded-lg text-slate-400">
          + 添加列
        </button>
      </div>
    </div>
  );
}

function KanbanColumn({ title, count, tasks }: KanbanColumnProps) {
  return (
    <div className="w-72 shrink-0 flex flex-col">
      {/* 列标题 */}
      <div className="flex items-center justify-between mb-3">
        <div className="flex items-center gap-2">
          <h3 className="text-sm font-semibold text-slate-900">{title}</h3>
          <span className="text-xs text-slate-400 bg-slate-100 px-1.5 py-0.5 rounded">
            {count}
          </span>
        </div>
        <button className="text-slate-400 hover:text-slate-600">⋮</button>
      </div>

      {/* 任务列表 */}
      <div className="flex flex-col gap-3 flex-1 overflow-y-auto">
        {tasks.map((task) => (
          <MobileTaskCard key={task.id} task={task} />
        ))}
      </div>
    </div>
  );
}
```

### 任务卡片：信息精简

移动端的任务卡片需要精简信息——屏幕太小，显示不下那么多细节：

```tsx
function MobileTaskCard({ task }: { task: Task }) {
  return (
    <div className="bg-white rounded-lg p-3 shadow-sm border border-slate-200">
      {/* 标题行：优先级圆点 + 标题 */}
      <div className="flex items-start gap-2 mb-2">
        <div
          className={`w-2 h-2 rounded-full mt-1.5 shrink-0 ${
            task.priority === "urgent"
              ? "bg-red-500"
              : task.priority === "high"
                ? "bg-orange-500"
                : task.priority === "medium"
                  ? "bg-yellow-500"
                  : "bg-blue-500"
          }`}
        />
        <p className="text-sm font-medium text-slate-900 leading-snug">
          {task.title}
        </p>
      </div>

      {/* 底部信息：标签 + 负责人 */}
      <div className="flex items-center justify-between">
        {/* 标签：移动端最多显示 1 个 */}
        <div className="flex gap-1">
          <span className="text-xs text-slate-500 bg-slate-100 px-1.5 py-0.5 rounded">
            {task.labels[0]}
          </span>
          {task.labels.length > 1 && (
            <span className="text-xs text-slate-400">
              +{task.labels.length - 1}
            </span>
          )}
        </div>

        {/* 负责人头像 */}
        <div className="w-6 h-6 rounded-full bg-blue-100 text-blue-600 flex items-center justify-center text-xs font-medium">
          {task.assigneeName[0]}
        </div>
      </div>
    </div>
  );
}
```

### 统计面板：网格自适应

仪表盘的统计卡片根据屏幕宽度自动调整列数：

```tsx
function StatsGrid() {
  const stats = [
    { label: "活跃项目", value: "12", change: "+2", trend: "up" },
    { label: "待处理任务", value: "47", change: "-5", trend: "down" },
    { label: "团队成员", value: "23", change: "+1", trend: "up" },
    { label: "本周截止", value: "8", change: "0", trend: "neutral" },
  ];

  return (
    <div className="grid grid-cols-2 lg:grid-cols-4 gap-3 lg:gap-4 mb-6">
      {stats.map((stat) => (
        <div
          key={stat.label}
          className="bg-white rounded-lg p-4 lg:p-5 border border-slate-200"
        >
          <p className="text-xs lg:text-sm text-slate-500 mb-1">{stat.label}</p>
          <div className="flex items-baseline gap-2">
            <span className="text-xl lg:text-2xl font-bold text-slate-900">
              {stat.value}
            </span>
            <span
              className={`text-xs font-medium ${
                stat.trend === "up"
                  ? "text-green-600"
                  : stat.trend === "down"
                    ? "text-red-600"
                    : "text-slate-400"
              }`}
            >
              {stat.change}
            </span>
          </div>
        </div>
      ))}
    </div>
  );
}
```

移动端两列，桌面端四列。`text-xl lg:text-2xl` 让数字在桌面端更大——空间允许的情况下适当放大关键信息。

## Container 查询：更精确的响应式

Tailwind 4 支持容器查询（Container Queries），它比媒体查询更精确。媒体查询基于视口宽度，容器查询基于**父容器宽度**。

```tsx
// 一个侧边栏内的组件，根据侧边栏宽度调整布局
function ProjectList() {
  return (
    <div className="@container">
      <div className="flex flex-col @md:flex-row gap-2">
        <div className="@md:w-48">项目图标</div>
        <div>
          <h3 className="text-sm font-medium">项目名称</h3>
          <p className="text-xs text-slate-400 truncate">描述...</p>
        </div>
      </div>
    </div>
  );
}
```

容器查询的好处是：同一个组件放在侧边栏（窄）和主内容区（宽）中可以自动适配，不需要知道外部页面的布局。这在构建可复用组件库时特别有价值。

不过容器查询目前还比较新，浏览器支持在快速改善。如果需要兼容旧浏览器，还是用媒体查询更稳妥。

## 响应式设计的几个原则

1. **触摸目标不小于 44px**：按钮、链接、可点击元素的触摸区域至少 44x44px。`py-3`（12px 上下）+ 文字高度 ≈ 36px，再加点 padding 比较安全。

2. **不要隐藏核心内容**：移动端隐藏装饰性元素（logo、描述文字）可以，但不要隐藏功能。如果桌面端有个"导出"按钮，移动端也得有——可以换个位置或样式，但不能消失。

3. **性能同样重要**：移动端网络慢、性能弱。大图用 `srcset` 提供小尺寸版本，避免在移动端加载桌面端的大图。

4. **真机测试**：DevTools 的设备模拟不能替代真机测试。触摸手势、虚拟键盘、滚动行为在模拟器里的表现可能和真机不同。

## 响应式断点选择的实用建议

很多人写响应式时每个尺寸都调，搞了五六个断点。实际上大部分界面只需要两三个断点：

```
默认（移动端）→ md（平板/小笔记本）→ lg（桌面）
```

`sm` 和 `xl` 只在特定场景下需要。不要为了让每个尺寸都"完美"而过度使用断点——维护成本会指数级增长。

优先保证移动端和桌面端两个极端的体验，中间尺寸用自动流式布局过渡就好了。

下一篇我们让界面"活起来"——加上交互状态和动画。按钮的 hover 效果、表单的 focus 提示、弹窗的出场动画，这些细节决定了产品给人"精致"还是"粗糙"的感觉。
