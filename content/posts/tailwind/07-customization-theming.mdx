上一篇我们用 cva + cn() 搭建了 TeamFlow 的组件体系。但回头看看——所有颜色还是用的 Tailwind 默认的 `blue-500`、`slate-700`。如果产品经理说"我们的品牌色不是 Tailwind 的蓝色"，或者设计师给了一套完整的设计规范，你怎么让 Tailwind 的工具类和设计稿对齐？

Tailwind 4 引入了全新的 CSS-first 配置方式。不再需要 `tailwind.config.js` 文件，所有定制都在 CSS 里完成。这个改变不只是"换了配置位置"——它让设计令牌和样式真正统一在了一起。

## Tailwind 4 的配置方式

### 基本结构

Tailwind 4 的配置完全在 CSS 中完成：

```css
/* src/index.css */
@import "tailwindcss";

/* @theme 块定义自定义设计令牌 */
@theme {
  /* 颜色 */
  --color-brand: #4f46e5;
  --color-brand-light: #818cf8;

  /* 间距 */
  --spacing-18: 4.5rem;

  /* 字号 */
  --text-xxs: 0.625rem;

  /* 圆角 */
  --radius-4xl: 2rem;

  /* 动画 */
  --animate-fade-in: fade-in 0.3s ease-out;
}

/* 自定义关键帧 */
@keyframes fade-in {
  from {
    opacity: 0;
  }
  to {
    opacity: 1;
  }
}
```

定义之后就可以直接使用：

```html
<div className="bg-brand text-white rounded-4xl">品牌色背景</div>
<p className="text-xxs">比 xs 还小的文字</p>
```

### 和 Tailwind 3 的区别

如果你用过 v3，核心区别在于：

| 方面     | Tailwind 3                  | Tailwind 4         |
| -------- | --------------------------- | ------------------ |
| 配置文件 | `tailwind.config.js`        | CSS `@theme` 块    |
| 内容检测 | 手动配置 `content`          | 自动检测           |
| 定义颜色 | `theme.extend.colors.brand` | `--color-brand`    |
| 使用方式 | `bg-brand`                  | `bg-brand`（一样） |

在 v3 中，你可能这样配置：

```js
// tailwind.config.js (Tailwind 3)
module.exports = {
  theme: {
    extend: {
      colors: {
        brand: {
          DEFAULT: "#4f46e5",
          light: "#818cf8",
          dark: "#3730a3",
        },
      },
    },
  },
};
```

在 v4 中，等价的配置是：

```css
/* src/index.css (Tailwind 4) */
@theme {
  --color-brand: #4f46e5;
  --color-brand-light: #818cf8;
  --color-brand-dark: #3730a3;
}
```

效果一样——都能用 `bg-brand`、`text-brand-light`。但 v4 的方式更直观，而且配置就在 CSS 文件里，不用在 JS 和 CSS 之间跳来跳去。

## 为 TeamFlow 定义品牌色

### 设计令牌的组织

一个产品的颜色不只是"一个品牌色"。你需要一套完整的颜色体系：

```css
/* src/styles/globals.css */
@import "tailwindcss";

@theme {
  /* === 品牌色 === */
  --color-brand-50: #eef2ff;
  --color-brand-100: #e0e7ff;
  --color-brand-200: #c7d2fe;
  --color-brand-300: #a5b4fc;
  --color-brand-400: #818cf8;
  --color-brand-500: #6366f1; /* 主品牌色 */
  --color-brand-600: #4f46e5;
  --color-brand-700: #4338ca;
  --color-brand-800: #3730a3;
  --color-brand-900: #312e81;
  --color-brand-950: #1e1b4b;

  /* === 语义色 === */
  --color-success: #22c55e;
  --color-success-light: #f0fdf4;
  --color-warning: #f59e0b;
  --color-warning-light: #fffbeb;
  --color-danger: #ef4444;
  --color-danger-light: #fef2f2;
  --color-info: #3b82f6;
  --color-info-light: #eff6ff;
}
```

定义之后，这些颜色就可以像 Tailwind 默认颜色一样使用：

```tsx
// 品牌色
<button className="bg-brand-500 hover:bg-brand-600 text-white">主要操作</button>
<div className="bg-brand-50 border-brand-200 text-brand-700">品牌色提示</div>

// 语义色
<div className="bg-success-light text-success border border-success/20">操作成功</div>
<div className="bg-danger-light text-danger border border-danger/20">删除确认</div>
```

### 颜色选择的实用建议

Tailwind 默认提供了 Indigo 色作为品牌色的好起点。但我们来讨论一下怎么选择品牌色：

- **蓝色系**（blue/indigo/sky）：信任感、专业感——SaaS、企业工具
- **绿色系**（green/emerald）：成长、健康——金融、健康产品
- **紫色系**（violet/purple）：创意、高端——设计工具、创意平台
- **橙色系**（orange/amber）：活力、积极——社区、消费产品

选定一个主色后，用它生成 50-950 的色阶。[Tailwind 的颜色生成器](https://uicolors.app/) 可以帮你从单个颜色生成完整的色阶。

## CSS 变量与设计令牌

### 令牌分层

设计令牌通常分三层：**全局令牌**（primitive tokens）、**语义令牌**（semantic tokens）、**组件令牌**（component tokens）。

```css
@theme {
  /* 第一层：全局令牌 — 纯粹的颜色值 */
  --color-indigo-50: #eef2ff;
  --color-indigo-500: #6366f1;
  --color-indigo-700: #4338ca;

  /* 第二层：语义令牌 — 引用全局令牌，赋予含义 */
  --color-brand: var(--color-indigo-500);
  --color-brand-hover: var(--color-indigo-700);
  --color-surface: #ffffff;
  --color-on-surface: #0f172a;
  --color-muted: #64748b;
  --color-border: #e2e8f0;
}
```

```tsx
/* 第三层：组件令牌 — 在 cva 中定义 */
const buttonVariants = cva("rounded-md font-medium transition-colors", {
  variants: {
    variant: {
      primary: "bg-brand text-white hover:bg-brand-hover",
      secondary:
        "bg-surface text-on-surface border border-border hover:bg-slate-50",
    },
  },
});
```

这种分层的好处是：改品牌色只需要修改语义令牌的引用，所有组件自动更新。

### 用 CSS 变量实现动态主题

如果你需要在运行时切换主题（比如多品牌 SaaS），用 CSS 变量：

```css
/* 默认主题 */
:root {
  --color-brand: #6366f1;
  --color-brand-hover: #4f46e5;
  --color-brand-light: #eef2ff;
}

/* 客户 A 的主题 */
[data-theme="client-a"] {
  --color-brand: #059669;
  --color-brand-hover: #047857;
  --color-brand-light: #ecfdf5;
}

/* 客户 B 的主题 */
[data-theme="client-b"] {
  --color-brand: #d97706;
  --color-brand-hover: #b45309;
  --color-brand-light: #fffbeb;
}
```

然后在 Tailwind 中引用这些变量：

```css
@theme {
  --color-brand: var(--color-brand);
  --color-brand-hover: var(--color-brand-hover);
  --color-brand-light: var(--color-brand-light);
}
```

切换主题只需要改 `<html>` 上的 `data-theme` 属性：

```tsx
// 切换主题
document.documentElement.setAttribute("data-theme", "client-a");
```

## 暗色模式的完整方案

### 颜色体系

暗色模式不只是翻转黑白。你需要一套完整的暗色颜色体系：

```css
@theme {
  /* 亮色模式（默认） */
  --color-bg-primary: #ffffff;
  --color-bg-secondary: #f8fafc;
  --color-bg-tertiary: #f1f5f9;
  --color-text-primary: #0f172a;
  --color-text-secondary: #475569;
  --color-text-muted: #94a3b8;
  --color-border-primary: #e2e8f0;
  --color-border-secondary: #f1f5f9;
}

/* 暗色模式 */
@custom-variant dark (&:where(.dark, .dark *));

/* 通过 CSS 变量覆盖实现暗色模式 */
.dark {
  --color-bg-primary: #0f172a;
  --color-bg-secondary: #1e293b;
  --color-bg-tertiary: #334155;
  --color-text-primary: #f8fafc;
  --color-text-secondary: #cbd5e1;
  --color-text-muted: #64748b;
  --color-border-primary: #334155;
  --color-border-secondary: #1e293b;
}
```

使用时不需要写 `dark:` 前缀，CSS 变量自动切换：

```tsx
// 亮色和暗色模式自动适配
<div className="bg-bg-primary border border-border-primary">
  <h2 className="text-text-primary">标题</h2>
  <p className="text-text-secondary">描述文字</p>
</div>
```

这种方式比给每个元素加 `dark:` 前缀更干净——颜色逻辑集中在 CSS 变量里，组件代码不需要关心当前是什么主题。

### 两种暗色模式方案的对比

| 方案         | 优点                       | 缺点                     |
| ------------ | -------------------------- | ------------------------ |
| `dark:` 前缀 | 灵活，可以逐元素精确控制   | 组件代码膨胀，维护成本高 |
| CSS 变量     | 组件代码干净，主题集中管理 | 需要提前规划颜色命名     |

小项目用 `dark:` 前缀就够了。中大型项目推荐 CSS 变量方案——维护成本低得多。

### 暗色模式的切换和持久化

```tsx
function useTheme() {
  const [theme, setTheme] = useState<"light" | "dark">(() => {
    // 1. 先看 localStorage
    const saved = localStorage.getItem("theme");
    if (saved === "light" || saved === "dark") return saved;
    // 2. 跟随系统
    return window.matchMedia("(prefers-color-scheme: dark)").matches
      ? "dark"
      : "light";
  });

  useEffect(() => {
    document.documentElement.classList.toggle("dark", theme === "dark");
    localStorage.setItem("theme", theme);
  }, [theme]);

  const toggle = () => setTheme((t) => (t === "light" ? "dark" : "light"));

  return { theme, toggle };
}
```

初始化时检查三个来源的优先级：用户手动选择 > 系统偏好 > 默认亮色。

但这里有一个闪烁问题（FOUC）：页面先以默认主题渲染，JS 执行后才切换。解决方法是在 `<head>` 中加一段内联脚本：

```html
<!-- index.html -->
<head>
  <script>
    // 在页面渲染前就设置好主题，避免闪烁
    const theme =
      localStorage.getItem("theme") ||
      (window.matchMedia("(prefers-color-scheme: dark)").matches
        ? "dark"
        : "light");
    document.documentElement.classList.toggle("dark", theme === "dark");
  </script>
</head>
```

这段脚本在 CSS 加载之前执行，确保页面一开始就是正确的主题。

## 自定义间距和排版

### 间距扩展

如果你的设计规范使用了 Tailwind 默认间距之外的值：

```css
@theme {
  --spacing-18: 4.5rem; /* 72px */
  --spacing-88: 22rem; /* 352px */
  --spacing-128: 32rem; /* 512px */
}
```

之后就能用 `p-18`、`m-88`、`gap-128` 这些工具类了。

但实际上 Tailwind 的间距系统已经足够密集，很少需要自定义间距。通常需要的是**限制**可用的间距值，而不是扩展——这在团队协作中可以防止"随便用个 `p-[23px]`"的情况。

### 排版定制

如果你使用自定义字体：

```css
@theme {
  --font-sans: "Inter", "system-ui", "-apple-system", "sans-serif";
  --font-mono: "JetBrains Mono", "Menlo", "Consolas", "monospace";

  --text-display: 3.5rem;
  --text-display--line-height: 1.1;
}
```

```html
<h1 className="text-display font-bold text-slate-900">大标题</h1>
<code className="font-mono text-sm bg-slate-100 px-1.5 py-0.5 rounded"
  >code</code
>
```

## @source 指令：控制扫描范围

Tailwind 4 自动检测项目中的文件来生成工具类。但有些情况下你需要手动指定：

```css
@import "tailwindcss";

/* 扫描额外的路径 */
@source "../node_modules/@teamflow/ui/src";

/* 忽略某些路径 */
@source inline("../node_modules/@teamflow/ui/src") not inline("../node_modules/@teamflow/ui/src/test");
```

常见的使用场景：

- 使用了 monorepo 中其他包的组件——需要告诉 Tailwind 扫描那些包
- 使用了第三方组件库（如 shadcn/ui）——需要扫描 node_modules 中对应的文件

## 完整的 TeamFlow 主题配置

把所有定制组合成一个完整的配置：

```css
/* src/styles/globals.css */
@import "tailwindcss";

/* === 暗色模式支持 === */
@custom-variant dark (&:where(.dark, .dark *));

/* === 设计令牌 === */
@theme {
  /* 品牌色 */
  --color-brand-50: #eef2ff;
  --color-brand-100: #e0e7ff;
  --color-brand-200: #c7d2fe;
  --color-brand-300: #a5b4fc;
  --color-brand-400: #818cf8;
  --color-brand-500: #6366f1;
  --color-brand-600: #4f46e5;
  --color-brand-700: #4338ca;
  --color-brand-800: #3730a3;
  --color-brand-900: #312e81;
  --color-brand-950: #1e1b4b;

  /* 字体 */
  --font-sans: "Inter", "system-ui", "sans-serif";
  --font-mono: "JetBrains Mono", "monospace";

  /* 自定义圆角 */
  --radius-xl: 0.75rem;
  --radius-2xl: 1rem;

  /* 自定义动画 */
  --animate-fade-in: fade-in 0.3s ease-out forwards;
  --animate-slide-up: slide-up 0.3s ease-out forwards;
  --animate-slide-down: slide-down 0.2s ease-out forwards;
  --animate-scale-in: scale-in 0.2s ease-out forwards;
}

/* === 关键帧 === */
@keyframes fade-in {
  from {
    opacity: 0;
  }
  to {
    opacity: 1;
  }
}

@keyframes slide-up {
  from {
    opacity: 0;
    transform: translateY(8px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}

@keyframes slide-down {
  from {
    opacity: 0;
    transform: translateY(-8px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}

@keyframes scale-in {
  from {
    opacity: 0;
    transform: scale(0.95);
  }
  to {
    opacity: 1;
    transform: scale(1);
  }
}

/* === 语义色（通过 CSS 变量实现主题切换） === */
:root {
  --color-surface: #ffffff;
  --color-surface-hover: #f8fafc;
  --color-surface-active: #f1f5f9;
}

.dark {
  --color-surface: #1e293b;
  --color-surface-hover: #334155;
  --color-surface-active: #475569;
}

/* === 全局基础样式 === */
body {
  font-family: var(--font-sans);
  -webkit-font-smoothing: antialiased;
  -moz-osx-font-smoothing: grayscale;
}

/* === 第三方组件样式覆盖 === */
.prose {
  @apply text-slate-600 leading-relaxed;
}

.prose h2 {
  @apply text-xl font-semibold text-slate-900 mb-4 mt-8;
}

.prose h3 {
  @apply text-lg font-medium text-slate-800 mb-3 mt-6;
}

.prose p {
  @apply mb-4;
}

.prose code {
  @apply bg-slate-100 text-sm px-1.5 py-0.5 rounded font-mono;
}

.prose pre {
  @apply bg-slate-900 text-slate-300 rounded-lg p-4 overflow-x-auto mb-4;
}

.prose pre code {
  @apply bg-transparent p-0 text-inherit;
}
```

这个配置文件做了几件事：

1. **品牌色**：定义了完整的 `brand-50` 到 `brand-950` 色阶
2. **字体**：指定了 Inter 作为主字体，JetBrains Mono 作为代码字体
3. **动画**：定义了四个常用入场动画
4. **语义色**：用 CSS 变量实现了亮色/暗色主题自动切换
5. **全局样式**：字体平滑、Markdown 内容样式

这个文件是整个项目样式的"源头"——所有视觉决策都集中在这里。设计师改了品牌色？改这里。产品要支持暗色模式？改这里。组件代码不需要动。

## 团队协作建议

设计令牌配置好后，最重要的是让团队成员知道怎么用。几个实用建议：

1. **文档化颜色规范**：列一张"什么场景用什么颜色"的表格，放在团队 Wiki 里。比如"成功状态用 `success`，不用 `green`"。

2. **ESLint 规则**：用 `eslint-plugin-tailwindcss` 检查不合法的类名使用，比如使用了未定义的颜色。

3. **禁止 arbitrary values**：在 `tailwind.config` 中关闭 `matchFontFamily` 等模块的任意值支持，防止团队成员用 `text-[23px]` 这种非标准值。

4. **定期审查**：每个月花半小时看看有没有人在用 `blue-500` 而不是 `brand-500`。全局搜索 `text-blue`、`bg-blue` 就能发现。
