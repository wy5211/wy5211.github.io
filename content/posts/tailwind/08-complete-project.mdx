前七篇我们把 Tailwind 的知识点拆开了一个个讲：布局、颜色、响应式、动画、组件化、主题定制。现在是时候把它们组装起来了。

这篇不是零散示例的拼凑——我们会构建一个完整的 TeamFlow 项目管理界面，从项目结构开始，到侧边栏导航、看板视图、任务详情弹窗、新建任务表单，再到移动端适配和暗色模式。你会看到这些知识在真实项目中怎么协同工作。

```tsx
// src/components/ui/dialog.tsx
import { useEffect, useRef } from "react";
import { cn } from "@/lib/utils";

function Dialog({ open, onClose, title, children, className }: DialogProps) {
  const dialogRef = useRef<HTMLDialogElement>(null);

  useEffect(() => {
    const el = dialogRef.current;
    if (!el) return;
    if (open) {
      el.showModal();
    } else {
      el.close();
    }
  }, [open]);

  return (
    <dialog
      ref={dialogRef}
      onClose={onClose}
      className={cn(
        "bg-transparent backdrop:bg-black/50",
        "open:animate-scale-in",
        className
      )}
    >
      <div className="bg-white rounded-xl shadow-xl w-full max-w-lg max-h-[85vh] overflow-y-auto">
        {title && (
          <div className="flex items-center justify-between px-6 py-4 border-b border-slate-200">
            <h2 className="text-lg font-semibold text-slate-900">{title}</h2>
            <button
              onClick={onClose}
              className="w-8 h-8 flex items-center justify-center rounded-lg text-slate-400 hover:text-slate-600 hover:bg-slate-100 transition-colors"
            >
              ✕
            </button>
          </div>
        )}
        <div className="p-6">{children}</div>
      </div>
    </dialog>
  );
}
```

这里用了原生 `<dialog>` 元素——它自带焦点陷阱和 Escape 关闭功能，比手动实现的模态框更可靠。Tailwind 的 `backdrop:bg-black/50` 给遮罩层加半透明黑色。

```tsx
import { Badge } from "@/components/ui/badge";
```

```tsx
const priorityBadge: Record<
  Priority,
  { variant: "danger" | "warning" | "default" | "info"; label: string }
> = {

  high: { variant: "warning", label: "高" },
  medium: { variant: "default", label: "中" },
  low: { variant: "info", label: "低" },
};

function TaskDetail({
  task,
  open,
  onClose,
}: {
  task: Task | null;
  open: boolean;
  onClose: () => void;

                <Badge key={label} variant="outline">

}
```

```tsx
          <Input
            value={title}
            onChange={(e) => {
              setTitle(e.target.value);
              if (errors.title) setErrors((prev) => ({ ...prev, title: "" }));
            }}
            placeholder="输入任务标题"
            error={errors.title}
          />
          <p className="mt-1 text-xs text-slate-400">{title.length}/100</p>
        </div>

        {/* 任务描述 */}
        <div>
          <label className="block text-sm font-medium text-slate-700 mb-1.5">
            描述
          </label>
          <textarea
            value={description}
            onChange={(e) => setDescription(e.target.value)}
            placeholder="描述这个任务的具体内容..."
            rows={4}
            className="flex w-full rounded-md border border-slate-300 bg-white px-3 py-2 text-sm placeholder:text-slate-400 focus:outline-none focus:ring-2 focus:ring-brand-200 focus:border-brand-500 resize-none transition-colors"
          />
        </div>

        {/* 优先级 */}
        <div>
          <label className="block text-sm font-medium text-slate-700 mb-1.5">
            优先级
          </label>
          <div className="flex gap-2">
            {(["low", "medium", "high", "urgent"] as Priority[]).map((p) => {
              const colors = {
                low: "border-slate-300 text-slate-600 bg-white",
                medium: "border-yellow-400 text-yellow-700 bg-yellow-50",
                high: "border-orange-400 text-orange-700 bg-orange-50",
                urgent: "border-red-400 text-red-700 bg-red-50",
              };
              const labels = {
                low: "低",
                medium: "中",
                high: "高",
                urgent: "紧急",
              };
              return (
                <button
                  key={p}
                  type="button"
                  onClick={() => setPriority(p)}
                  className={`flex-1 px-3 py-2 rounded-md border text-sm font-medium transition-all duration-150 ${
                    priority === p
                      ? colors[p]
                      : "border-slate-200 text-slate-400 bg-slate-50 hover:border-slate-300"
                  }`}
                >
                  {labels[p]}
                </button>
              );
            })}
          </div>
        </div>

        {/* 截止日期 */}
        <div>
          <label className="block text-sm font-medium text-slate-700 mb-1.5">
            截止日期
          </label>
          <Input type="date" />
        </div>

        {/* 操作按钮 */}
        <div className="flex items-center justify-end gap-3 pt-4 border-t border-slate-200">
          <Button variant="secondary" onClick={onClose}>
            取消
          </Button>
          <Button onClick={handleSubmit}>创建任务</Button>
        </div>
      </form>
    </Dialog>
  );
}
```

表单中几个值得注意的 Tailwind 用法：

- `focus:ring-2 focus:ring-brand-200 focus:border-brand-500`：输入框的焦点反馈用了自定义品牌色
- `resize-none`：禁止 textarea 拖拽调整大小
- 优先级选择用按钮组而非下拉框——视觉更直观，点击更快
- `transition-all duration-150`：按钮切换有微妙的过渡

## 代码审查清单

项目上线前，用这个清单检查 Tailwind 代码质量：

**性能**：

- 确认生产构建的 CSS 文件大小合理（通常 < 50KB gzipped）
- 检查是否有大量未使用的 `@apply`（Tailwind 4 的 tree-shaking 会处理工具类，但 `@apply` 里的内容不会被自动清理）
- 大列表使用虚拟滚动，而不是一次性渲染所有任务卡片

**一致性**：

- 全局搜索 `text-blue`、`bg-blue` 等非品牌色——应该用 `brand` 替代
- 检查间距是否遵循 4px 基准（`p-[23px]` 这种任意值应该避免）
- 确认所有 `rounded-*` 使用统一的圆角层级

**可访问性**：

- 所有交互元素有 `focus-visible` 样式
- 图片有 `alt` 属性
- 弹窗有焦点陷阱（原生 `<dialog>` 自带）
- 颜色对比度满足 WCAG AA 标准（文字和背景的对比度至少 4.5:1）

**响应式**：

- 在 320px、768px、1280px 三个宽度下测试
- 触摸目标不小于 44px
- 没有水平滚动条（除了看板区域）

## 性能优化

### JIT 和 Tree-shaking

Tailwind 4 默认使用 JIT（Just-In-Time）模式——只生成你实际用到的工具类。这意味着你的 CSS 文件大小和使用量成正比，而不是和 Tailwind 的完整工具集成正比。

生产构建时，确保 CSS 被 purge 干净：

```bash
npm run build
# 检查输出的 CSS 文件大小
ls -lh dist/assets/*.css
```

正常情况下，一个中等项目的 Tailwind CSS 产物应该在 10-30KB（gzipped）。如果超过 50KB，说明可能有问题。

### 减少重复的类名组合

如果一个类名组合出现了 10 次以上，考虑用 cva 提取为组件。这不会减小 CSS 大小，但会减小 HTML/JS 的体积——因为组件只引用一次类名组合。

### 避免过度使用 arbitrary values

```html
<!-- 不推荐：每个都是唯一的工具类，不会被复用 -->
<div className="p-[23px] text-[#3a2b1c] w-[347px]">
  <!-- 推荐：用最接近的标准工具类，或者扩展 @theme -->
  <div className="p-6 text-slate-700 w-80"></div>
</div>
```

arbitrary values 会生成新的 CSS 规则，每一条都是唯一的，无法被 tree-shake。如果同一段颜色或间距反复出现，在 `@theme` 中定义它。

## 写在最后

这个系列从"为什么用 Tailwind"开始，到构建一个完整的 TeamFlow 界面结束。回顾一下走过的路：

**基础**：理解了 utility-first 的哲学，搭好了环境，学会了布局和间距工具类。

**核心**：掌握了颜色系统、排版、响应式设计——这些是视觉一致性的基础。

**进阶**：用状态变体和动画让界面有了反馈，用 cva + cn() 让代码有了组织。

**实战**：深度定制了主题，构建了完整的项目界面。

Tailwind 不是"不用写 CSS"，而是"换一种方式写 CSS"。它要求你理解 CSS 的原理（Flexbox、Grid、层叠、盒模型），然后用更高效的方式表达。如果你发现自己在复制粘贴类名而不理解它们做了什么，退一步看看生成的 CSS——`hover:bg-blue-600` 就是 `background-color: #2563eb` 的简写，仅此而已。

最后一点建议：不要试图一次学完所有工具类。Tailwind 有几千个工具类，没人全记住。掌握 20% 的常用工具类就能覆盖 80% 的场景，剩下的用到时查文档就好。VS Code 的 IntelliSense 插件会让查找工具类变得很快。

现在，去构建你自己的项目吧。
