上一篇我们搭好了 Tailwind 环境，也理解了 utility-first 的哲学。接下来要解决的问题是：一个页面的**骨架**怎么搭。

你打开任何一个 SaaS 产品——Notion、Linear、Jira——它们的布局都差不多：左侧固定导航、顶部状态栏、中间弹性内容区。这种布局看似简单，但用 CSS 实现时总有各种小坑：侧边栏被内容挤没了、高度撑不开、间距对不齐。

Tailwind 把 CSS 布局能力映射成了简洁的工具类。理解这些工具类的含义比记住它们更重要——因为它们本质就是 CSS 属性的缩写。

## 间距体系：一切从 4px 开始

Tailwind 的间距系统遵循一个简单的规则：**基准单位是 4px**（即 `0.25rem`）。

| 工具类 | 值      | 像素值 |
| ------ | ------- | ------ |
| `p-1`  | 0.25rem | 4px    |
| `p-2`  | 0.5rem  | 8px    |
| `p-3`  | 0.75rem | 12px   |
| `p-4`  | 1rem    | 16px   |
| `p-6`  | 1.5rem  | 24px   |
| `p-8`  | 2rem    | 32px   |
| `p-12` | 3rem    | 48px   |

这个 4px 的基准不是随意选择的。它和大多数设计系统的网格对齐（8px 网格），让间距自然和谐——两个 `p-2` 合起来就是 `p-4`，三个 `p-2` 是 `p-6`。

### Padding vs Margin vs Gap

三种间距工具有不同的使用场景：

**`p-*`（padding）**——元素**内部**的留白：

```html
<!-- 卡片内容和边框之间的间距 -->
<div className="bg-white rounded-lg p-6">
  <h2>项目名称</h2>
  <p>项目描述...</p>
</div>
```

**`m-*`（margin）**——元素**外部**的间距：

```html
<!-- 卡片之间的间距 -->
<div className="bg-white rounded-lg p-6 mb-4">第一个卡片</div>
<div className="bg-white rounded-lg p-6">第二个卡片</div>
```

**`gap-*`**——Flex/Grid 子元素之间的间距：

```html
<!-- 列表项之间的间距，不需要给每个子元素加 margin -->
<div className="flex flex-col gap-4">
  <div>卡片 1</div>
  <div>卡片 2</div>
  <div>卡片 3</div>
</div>
```

什么时候用哪个？一个简单的判断标准：

- 元素内部的留白 → `padding`
- 两个独立元素之间的距离 → `margin`
- 同一容器内子元素之间的等距 → `gap`

`gap` 比 `margin` 更好用的地方在于：它不会在第一个和最后一个子元素上产生多余的间距。所以列表、网格这类场景优先用 `gap`。

### 方向控制

间距可以精确控制四个方向：

```html
<!-- 方向简写：t(上) b(下) l(左) r(右) x(左右) y(上下) -->
<div className="px-6 py-3">
  <!-- 左右 24px，上下 12px -->
  <div className="mt-4 mb-8">
    <!-- 上边距 16px，下边距 32px -->
    <div className="pl-4 pr-2"><!-- 左内边距 16px，右内边距 8px --></div>
  </div>
</div>
```

一个实战技巧：`px-*` + `py-*` 是最常见的组合，因为同一方向的两个值通常相等。`p-*` 虽然四个方向都一样，但内容区域的水平内边距通常比垂直方向大，所以 `px-6 py-3` 比 `p-4` 更常见。

### 负值和 auto

`margin` 支持负值，用来做元素重叠或位置微调：

```html
<!-- 让头像向左偏移，和前一个头像重叠 -->
<div className="ml-[-8px] rounded-full border-2 border-white">
  <img src="avatar.jpg" alt="" />
</div>
```

`mx-auto` 用来水平居中块级元素：

```html
<div className="max-w-5xl mx-auto px-6">
  <!-- 内容居中，最大宽度 1024px -->
</div>
```

这个模式在页面布局中极其常见：`max-w-*` 限制宽度 + `mx-auto` 居中 + `px-*` 左右留白。

## Flexbox：一维布局的首选

Flexbox 解决的是一行或一列内元素的排列问题。Tailwind 的 Flex 工具类是 CSS Flexbox 属性的直接映射。

### 基础三件套：flex、items、justify

```html
<!-- 创建一个水平排列的 Flex 容器 -->
<div className="flex items-center justify-between">
  <div>左侧</div>
  <div>右侧</div>
</div>
```

这三个类几乎是固定搭配：

- `flex`：启用 Flex 布局（默认水平方向）
- `items-center`：交叉轴居中（垂直方向，因为主轴是水平的）
- `justify-between`：主轴两端对齐（左右贴边，中间均分）

### 方向控制

```html
<!-- 垂直排列 -->
<div className="flex flex-col gap-4">
  <div>第一项</div>
  <div>第二项</div>
</div>

<!-- 水平反转（右侧开始） -->
<div className="flex flex-row-reverse">
  <div>这会在右边</div>
  <div>这会在左边</div>
</div>
```

### 两个容易忽略但很重要的属性

**`flex-1`：让元素占满剩余空间**

```html
<div className="flex h-screen">
  <div className="w-64 bg-slate-900">侧边栏（固定 256px）</div>
  <div className="flex-1 bg-white">主内容区（自动占满剩余宽度）</div>
</div>
```

`flex-1` 等价于 `flex: 1 1 0%`，意思是：放大因子 1、缩小因子 1、初始大小 0。效果就是"有多少空间就占多少"。这是做弹性布局时最常用的工具类之一。

**`shrink-0`：阻止元素被压缩**

```html
<div className="flex items-center gap-3 min-w-0">
  <!-- 头像和图标不应被压缩 -->
  <div className="shrink-0 w-10 h-10 rounded-full bg-slate-200" />
  <!-- 文字区域可以压缩和截断 -->
  <div className="min-w-0">
    <p className="truncate">这是一个很长的项目名称可能会溢出</p>
  </div>
</div>
```

Flex 容器在空间不足时默认会压缩子元素。`shrink-0` 告诉浏览器"我不可以被压缩"。这在固定尺寸元素（头像、图标、标签）旁边有弹性文字内容时特别重要。

### 常见的 Flex 布局模式

在项目中你会反复用到这些模式：

```tsx
// 模式一：导航栏（左右对齐）
<nav className="flex items-center justify-between h-14 px-6">
  <Logo />
  <Actions />
</nav>

// 模式二：带标签的值（标签和值两端对齐）
<div className="flex items-center justify-between py-2">
  <span className="text-slate-500">负责人</span>
  <span className="font-medium">张伟</span>
</div>

// 模式三：标签列表（自动换行）
<div className="flex flex-wrap gap-2">
  <span className="px-2 py-1 bg-blue-50 text-blue-600 rounded text-sm">前端</span>
  <span className="px-2 py-1 bg-green-50 text-green-600 rounded text-sm">高优先级</span>
  <span className="px-2 py-1 bg-orange-50 text-orange-600 rounded text-sm">进行中</span>
</div>

// 模式四：居中提示（空状态）
<div className="flex flex-col items-center justify-center py-12 text-slate-400">
  <p>暂无任务</p>
</div>
```

## Grid：二维布局利器

如果说 Flexbox 是一维的（一行或一列），Grid 就是二维的（行和列同时控制）。Tailwind 的 Grid 工具类让你直接在模板里定义网格结构。

### 基础用法

```html
<!-- 定义 3 列等宽网格 -->
<div className="grid grid-cols-3 gap-6">
  <div>列 1</div>
  <div>列 2</div>
  <div>列 3</div>
</div>
```

### 自适应列数

看板视图的特点是列数不固定，每列内容独立滚动。用 `grid-cols-[repeat(auto-fill,...)]` 可以实现自适应：

```tsx
// 看板列：每列最小 280px，空间足够就自动增加列数
<div className="grid grid-cols-[repeat(auto-fill,minmax(280px,1fr))] gap-6">
  <KanbanColumn title="待办" tasks={todoTasks} />
  <KanbanColumn title="进行中" tasks={inProgressTasks} />
  <KanbanColumn title="已完成" tasks={doneTasks} />
</div>
```

### 固定列宽

更常见的做法是给看板一个固定的列宽，然后水平滚动：

```tsx
<div className="flex gap-6 overflow-x-auto pb-4">
  <div className="w-72 shrink-0">
    {/* 每列固定 288px 宽度 */}
    <KanbanColumn title="待办" tasks={todoTasks} />
  </div>
  <div className="w-72 shrink-0">
    <KanbanColumn title="进行中" tasks={inProgressTasks} />
  </div>
  <div className="w-72 shrink-0">
    <KanbanColumn title="已完成" tasks={doneTasks} />
  </div>
</div>
```

这里用 Flex 而不是 Grid，因为看板列需要水平滚动——Flex + `overflow-x-auto` 比 Grid 更适合这种场景。

### 跨行跨列

```html
<!-- 项目卡片的仪表盘网格 -->
<div className="grid grid-cols-4 gap-4">
  <!-- 统计卡片，每个占 1 列 -->
  <StatCard className="col-span-1" title="活跃项目" value="12" />
  <StatCard className="col-span-1" title="待处理任务" value="47" />
  <StatCard className="col-span-1" title="团队成员" value="23" />
  <StatCard className="col-span-1" title="本周截止" value="8" />

  <!-- 图表区域，横跨 3 列 -->
  <div className="col-span-3 bg-white rounded-lg p-6">
    <BurndownChart />
  </div>

  <!-- 活动流，占 1 列 -->
  <div className="col-span-1 bg-white rounded-lg p-6">
    <ActivityFeed />
  </div>
</div>
```

`col-span-*` 控制元素横跨几列。结合 `grid-cols-*` 定义的总列数，可以灵活地创建各种布局组合。

## 定位工具类

Tailwind 的定位工具类就是 CSS `position` 属性的映射：

```html
<!-- fixed：固定在视口顶部 -->
<nav className="fixed top-0 left-0 right-0 h-14 bg-white border-b z-50">
  导航栏
</nav>

<!-- sticky：滚动到阈值后固定 -->
<div className="sticky top-14 bg-slate-50 py-2 border-b z-10">看板列标题</div>

<!-- relative + absolute：相对定位容器内的绝对定位元素 -->
<div className="relative">
  <img src="cover.jpg" alt="" className="w-full h-48 object-cover rounded-lg" />
  <span
    className="absolute top-2 right-2 bg-black/50 text-white text-xs px-2 py-1 rounded"
  >
    封面
  </span>
</div>
```

几个注意点：

- `fixed` 脱离了文档流，下面的内容会被遮挡。记得给 `body` 或主容器加对应的 `padding-top` 或 `margin-top`。
- `sticky` 需要设定 `top-*` 等偏移值才生效，否则和 `relative` 没区别。
- `z-*` 控制层叠顺序。Tailwind 提供 `z-0` 到 `z-50`，以及 `z-auto`。别滥用 `z-50`，只在模态框、Toast 这类最上层元素使用。

```tsx
  title,
  priority,
  labels,
  assignee,

  title: string;

    high: "bg-red-500",
    medium: "bg-yellow-500",
    low: "bg-blue-500",

        </div>
      </div>
    </div>
  );
}
```

来拆解这个骨架中几个关键的布局决策：

**整体结构**：`flex h-screen` 让整个页面占满视口高度，侧边栏和主内容区水平排列。`overflow-hidden` 防止页面级滚动，滚动由内部区域自己管理。

**侧边栏**：`w-64 shrink-0` 固定宽度 256px 且不被压缩。`flex flex-col` 让 Logo、导航、用户区垂直排列。导航菜单用 `flex-1` 占满中间空间，`overflow-y-auto` 支持菜单项过多时滚动。

**主内容区**：`flex-1 flex flex-col min-w-0` 占满剩余宽度。这里的 `min-w-0` 很关键——没有它，长文字内容可能导致整个区域溢出。`min-w-0` 告诉浏览器"我可以缩小到比内容更窄"，配合子元素的 `truncate` 或 `overflow-hidden` 使用。

**看板区域**：`flex-1 overflow-x-auto` 让看板内容区域占满剩余高度，列太多时可以水平滚动。每列 `w-72 shrink-0` 固定宽度不被压缩。

**头像重叠**：`-space-x-2` 让头像向左偏移，实现多人头像叠放效果。配合 `border-2 border-white` 用白色边框分隔各头像。

这个骨架覆盖了日常布局 90% 的场景。后续几篇给它加上颜色、排版、响应式、动画，最终会变成一个完整的界面。

## 布局调试的几个技巧

布局出问题时，先别急着改代码。用这几个方法快速定位：

1. **加背景色**：给容器加 `bg-red-500/10`（红色半透明），立刻看到元素的实际尺寸和位置。调试完删掉就好。

2. **加边框**：`border border-dashed border-red-300` 用虚线边框看清布局边界。

3. **检查 overflow**：`overflow-hidden` 会让超出内容消失，如果内容不见了，先把 `overflow-hidden` 改成 `overflow-visible` 看看是不是内容跑出去了。

4. **min-w-0 检查**：Flex/Grid 子元素内容溢出时，加上 `min-w-0` 试试。这是最常见的布局修复之一。

下一篇我们进入样式系统的核心——颜色和排版。骨架搭好了，接下来给它穿上好看的"衣服"。
