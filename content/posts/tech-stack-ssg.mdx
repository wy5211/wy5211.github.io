# 本博客技术揭秘：Next.js App Router 与 SSG 实践

欢迎来到我的技术博客！在这篇文章中，我将分享构建这个博客所使用的技术栈，并深入探讨其中的核心技术——静态站点生成（Static Site Generation, SSG）。

## 技术栈概览

这个博客项目是基于最新的 Web 开发技术构建的，旨在提供极致的性能和优秀的开发体验。

- **框架**: [Next.js 14+](https://nextjs.org/) (App Router)
- **语言**: [TypeScript](https://www.typescriptlang.org/)
- **样式**: [Tailwind CSS](https://tailwindcss.com/)
- **内容管理**: [MDX](https://mdxjs.com/) (Markdown + JSX)
- **部署**: [Vercel](https://vercel.com/) / GitHub Pages

## 为什么选择 SSG (静态站点生成)？

静态站点生成（SSG）是指在构建阶段（Build Time）就将所有的页面预先渲染成 HTML 文件，而不是在用户请求时动态生成（SSR）或在客户端渲染（CSR）。

对于个人博客这种内容更新频率相对较低，但对读取性能要求极高的场景，SSG 是最完美的解决方案。它具有以下显著优势：

### 1. 极致的性能

由于页面已经是预先生成的静态 HTML，服务器只需要简单地返回文件即可。这意味着 Time To First Byte (TTFB) 极低，用户可以瞬间看到页面内容。结合 CDN (内容分发网络) 缓存，访问速度可以达到毫秒级。

### 2. 优秀的 SEO

搜索引擎爬虫可以非常容易地抓取静态 HTML 内容。相比于 CSR 需要执行 JavaScript 才能看到内容，SSG 对 SEO 更加友好，有助于文章被搜索引擎收录。

### 3. 更高的安全性

由于没有数据库连接，也不需要在服务器端运行复杂的逻辑，攻击者很难找到注入点或利用后台漏洞。静态文件是最安全的 Web 内容形式。

### 4. 低成本部署

静态网站可以托管在任何支持静态文件服务的平台上，如 GitHub Pages, Vercel, Netlify, AWS S3 等，通常都是免费的。

## Next.js App Router 中的 SSG 实现

在 Next.js 的新一代 App Router 架构中，SSG 的实现变得更加自然和简洁。默认情况下，所有的 Server Components 都是静态渲染的，除非使用了动态函数（如 `cookies()`, `headers()`）或显式指定了 `dynamic` 选项。

### 路由与参数生成：`generateStaticParams`

在 Pages Router 时代，我们需要使用 `getStaticPaths` 来定义哪些路径需要预渲染。而在 App Router 中，我们使用 `generateStaticParams` 函数。

这个函数在构建时运行，返回一个包含路由参数的对象数组。Next.js 会遍历这个数组，为每个参数组合生成一个静态页面。

```typescript
// src/app/posts/[slug]/page.tsx

import { getAllPosts } from "@/lib/posts";

// 这个函数告诉 Next.js 需要静态生成哪些 slug
export async function generateStaticParams() {
  const posts = getAllPosts();

  // 返回类似于 [{ slug: 'hello-world' }, { slug: 'tech-stack-ssg' }] 的数组
  return posts.map((post) => ({
    slug: post.slug,
  }));
}
```

### 数据获取：Server Components 的力量

在 App Router 中，页面组件本身就是 React Server Component。这意味着我们可以直接在组件内部使用 Node.js API（如 `fs` 模块）来读取文件系统，而不需要像以前那样通过 `getStaticProps` 传递数据。

```typescript
// src/app/posts/[slug]/page.tsx

import { getPostBySlug } from '@/lib/posts'
import { notFound } from 'next/navigation'

// 页面组件直接获取数据
export default async function Post({ params }: { params: { slug: string } }) {
  const { slug } = params
  const post = await getPostBySlug(slug)

  if (!post) {
    notFound()
  }

  return (
    <article>
      <h1>{post.title}</h1>
      {post.content}
    </article>
  )
}
```
