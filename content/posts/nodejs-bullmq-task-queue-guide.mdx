在现代 Web 应用中，许多操作不适合在请求-响应周期中同步执行：发送邮件、处理大文件、生成报表、调用第三方 API 等。这些耗时操作如果阻塞主线程，会严重影响用户体验和系统吞吐量。

任务队列是解决这类问题的经典方案。本文将深入讲解如何使用 **BullMQ** —— 一个基于 Redis 的高性能任务队列库，在 Node.js 中构建可靠的后台任务系统。

## 为什么需要任务队列？

### 同步处理的问题

```typescript
// ❌ 不好的做法：同步处理耗时操作
async function registerUser(req: Request, res: Response) {
  const user = await createUser(req.body);

  // 同步发送欢迎邮件，可能阻塞 2-5 秒
  await sendWelcomeEmail(user.email);

  // 同步生成用户报告，可能阻塞 10+ 秒
  await generateUserReport(user.id);

  // 同步通知第三方系统
  await notifyThirdPartySystem(user);

  res.json({ success: true, user });
}
```

这种做法的问题：

1. **响应时间长**：用户需要等待所有操作完成
2. **系统脆弱**：任何一个步骤失败都会影响整个流程
3. **资源浪费**：HTTP 连接长时间占用
4. **无法扩展**：流量高峰时系统容易崩溃

### 异步队列的优势

```typescript
// ✅ 好的做法：使用任务队列
async function registerUser(req: Request, res: Response) {
  const user = await createUser(req.body);

  // 将耗时操作放入队列，立即返回
  await emailQueue.add("welcome", { userId: user.id });
  await reportQueue.add("generate", { userId: user.id });
  await notificationQueue.add("notify", { userId: user.id });

  res.json({ success: true, user });
}
```

优势：

- **快速响应**：毫秒级返回，用户体验好
- **系统解耦**：各模块独立运行，互不影响
- **自动重试**：失败任务自动重试，保证可靠性
- **易于扩展**：可以根据负载动态调整 Worker 数量
- **可观测性**：任务状态、进度、失败原因都可追踪

## BullMQ 核心概念

在深入代码之前，先理解 BullMQ 的核心概念：

### Queue（队列）

队列是任务的容器，生产者向队列添加任务。

```typescript
import { Queue } from "bullmq";

const emailQueue = new Queue("email", {
  connection: {
    host: "localhost",
    port: 6379,
  },
});
```

### Job（任务）

任务是队列中的工作单元，包含数据和配置。

```typescript
const job = await emailQueue.add(
  "send-welcome",
  {
    email: "user@example.com",
    name: "张三",
  },
  {
    // 任务配置
    delay: 5000, // 5秒后执行
    attempts: 3, // 最多重试3次
    backoff: {
      type: "exponential",
      delay: 1000,
    },
  }
);
```

### Worker（消费者）

Worker 从队列中取出任务并执行。

```typescript
import { Worker } from "bullmq";

const worker = new Worker(
  "email",
  async (job) => {
    console.log(`Processing job ${job.id}:`, job.name);
    console.log("Data:", job.data);

    // 执行实际工作
    await sendEmail(job.data.email, job.data.name);

    // 可以更新进度
    await job.updateProgress(50);
  },
  {
    connection: {
      host: "localhost",
      port: 6379,
    },
  }
);
```

### Events（事件）

BullMQ 提供丰富的事件系统，用于监控任务状态。

```typescript
worker.on("completed", (job) => {
  console.log(`Job ${job.id} completed!`);
});

worker.on("failed", (job, err) => {
  console.error(`Job ${job?.id} failed:`, err.message);
});

worker.on("progress", (job, progress) => {
  console.log(`Job ${job.id} progress: ${progress}%`);
});
```

### 创建 Worker

```typescript
// src/workers/email.worker.ts
import { Worker, Job } from 'bullmq';
import { redisConnection } from '../config/redis';
import { EmailJobData } from '../types/queues';

// 模拟邮件发送
async function sendEmail(data: EmailJobData): Promise&lt;void&gt; {
  console.log(`Sending email to ${data.to}...`);
  console.log(`Subject: ${data.subject}`);
  console.log(`Template: ${data.template}`);

  // 模拟耗时操作
  await new Promise(resolve => setTimeout(resolve, 1000));

  // 模拟随机失败（用于测试重试）
  if (Math.random() &lt; 0.1) {
    throw new Error('Email service temporarily unavailable');
  }

  console.log(`Email sent successfully to ${data.to}`);
}

export const emailWorker = new Worker&lt;EmailJobData&gt;(
  'email',
  async (job: Job&lt;EmailJobData&gt;) => {
    const { name, data } = job;

    // 更新进度
    await job.updateProgress(10);

    switch (name) {
      case 'send-welcome':
        await sendEmail({
          ...data,
          subject: '欢迎加入！',
          template: 'welcome',
        });
        break;

      case 'send-notification':
        await sendEmail(data);
        break;

      case 'send-batch':
        // 批量发送
        const emails = data as unknown as EmailJobData[];
        for (let i = 0; i &lt; emails.length; i++) {
          await sendEmail(emails[i]);
          await job.updateProgress(((i + 1) / emails.length) * 100);
        }
        break;

      default:
        throw new Error(`Unknown job name: ${name}`);
    }

    await job.updateProgress(100);
    return { success: true, sentAt: new Date().toISOString() };
  },
  {
    connection: redisConnection,
    concurrency: 5, // 同时处理 5 个任务
    limiter: {
      max: 100,    // 每分钟最多
      duration: 60000, // 100 个任务
    },
  }
);

// 事件监听
emailWorker.on('completed', (job: Job) => {
  console.log(`✅ Job ${job.id} (${job.name}) completed`);
});

emailWorker.on('failed', (job: Job | undefined, err: Error) => {
  console.error(`❌ Job ${job?.id} (${job?.name}) failed:`, err.message);
});

emailWorker.on('error', (err: Error) => {
  console.error('Worker error:', err);
});

emailWorker.on('stalled', (jobId: string) => {
  console.warn(`Job ${jobId} has stalled`);
});
```

## 进阶功能详解

### 1. 延迟任务

延迟任务在指定时间后执行，非常适合定时通知、订单超时取消等场景。

```typescript
// 延迟 5 分钟发送提醒邮件
await emailQueue.add(
  "send-notification",
  {
    to: "user@example.com",
    subject: "您的订单即将超时",
    template: "order-reminder",
    context: { orderId: "12345" },
  },
  {
    delay: 5 * 60 * 1000, // 5分钟 = 300000ms
  }
);

// 订单 30 分钟后自动取消
await orderQueue.add(
  "auto-cancel",
  { orderId: "12345" },
  {
    delay: 30 * 60 * 1000,
    jobId: `cancel-order-12345`, // 自定义 jobId，防止重复
  }
);

// 如果用户支付成功，取消自动取消任务
await orderQueue.remove("cancel-order-12345");
```

### 2. 定时任务（Cron）

BullMQ 支持基于 Cron 表达式的重复任务。

```typescript
import { Queue } from "bullmq";

const reportQueue = new Queue("report", { connection: redisConnection });

// 每天凌晨 2 点生成日报
await reportQueue.add(
  "generate-daily-report",
  { type: "daily" },
  {
    repeat: {
      pattern: "0 2 * * *", // Cron 表达式
      tz: "Asia/Shanghai", // 时区
    },
    jobId: "daily-report",
  }
);

// 每周一早上 9 点发送周报
await emailQueue.add(
  "send-weekly-report",
  { type: "weekly" },
  {
    repeat: {
      pattern: "0 9 * * 1",
      tz: "Asia/Shanghai",
    },
  }
);

// 每月 1 号凌晨 3 点清理过期数据
await maintenanceQueue.add(
  "cleanup-expired",
  {},
  {
    repeat: {
      pattern: "0 3 1 * *",
    },
  }
);

// 查看所有重复任务
const repeatableJobs = await reportQueue.getRepeatableJobs();
console.log("Repeatable jobs:", repeatableJobs);

// 删除重复任务
await reportQueue.removeRepeatable("0 2 * * *", {
  tz: "Asia/Shanghai",
});
```

### 3. 优先级队列

当某些任务比其他任务更重要时，可以使用优先级。

```typescript
// VIP 用户的邮件优先发送
await emailQueue.add(
  "send-welcome",
  { to: "vip@example.com", isVip: true },
  { priority: 1 } // 数字越小，优先级越高
);

// 普通用户的邮件
await emailQueue.add(
  "send-welcome",
  { to: "normal@example.com" },
  { priority: 10 }
);

// 批量邮件优先级最低
await emailQueue.add("send-batch", emails, { priority: 100 });
```

### 4. 进度追踪

对于长时间运行的任务，实时报告进度非常重要。

```typescript
// Worker 中报告进度
const worker = new Worker("report", async (job) => {
  const { reportId, data } = job.data;

  await job.updateProgress(0);
  await job.log("Starting report generation...");

  // 步骤 1：收集数据
  const rawData = await collectData(data);
  await job.updateProgress(25);
  await job.log(`Collected ${rawData.length} records`);

  // 步骤 2：处理数据
  const processedData = await processData(rawData);
  await job.updateProgress(50);
  await job.log("Data processing completed");

  // 步骤 3：生成报告
  const report = await generateReport(processedData);
  await job.updateProgress(75);
  await job.log(`Report generated: ${report.path}`);

  // 步骤 4：上传到存储
  await uploadToStorage(report);
  await job.updateProgress(100);
  await job.log("Report uploaded successfully");

  return { reportUrl: report.url };
});

// 客户端查询进度
const job = await reportQueue.getJob(jobId);
console.log("Progress:", job?.progress);
console.log("Logs:", await job?.getLogs());
```

### 5. 失败重试策略

BullMQ 提供灵活的重试策略：

```typescript
// 固定间隔重试
await emailQueue.add("send-email", data, {
  attempts: 5,
  backoff: {
    type: "fixed",
    delay: 3000, // 每次重试间隔 3 秒
  },
});

// 指数退避重试（推荐）
await reportQueue.add("generate-report", data, {
  attempts: 5,
  backoff: {
    type: "exponential",
    delay: 1000, // 初始延迟 1 秒，之后：2s, 4s, 8s, 16s
  },
});

// 自定义重试逻辑
const worker = new Worker("report", async (job) => {
  try {
    return await processReport(job.data);
  } catch (error) {
    // 根据错误类型决定是否重试
    if (error.code === "UNRECOVERABLE_ERROR") {
      // 不重试，直接标记为失败
      throw new Error(error.message);
    }

    // 其他错误会自动重试
    throw error;
  }
});
```

### 6. 任务依赖（Flow）

BullMQ 支持任务依赖，可以构建复杂的工作流：

```typescript
import { FlowProducer } from "bullmq";

const flowProducer = new FlowProducer({ connection: redisConnection });

// 定义工作流：先生成数据，再发送邮件，最后更新状态
const flow = await flowProducer.add({
  name: "send-report-email",
  queueName: "email",
  data: {
    to: "user@example.com",
    subject: "您的报告已生成",
  },
  children: [
    {
      name: "generate-report",
      queueName: "report",
      data: { reportId: "12345" },
      children: [
        {
          name: "collect-data",
          queueName: "data",
          data: { source: "database" },
        },
      ],
    },
  ],
});

// 执行顺序：collect-data -&gt; generate-report -&gt; send-report-email
```

```typescript
// src/services/email.service.ts
import { emailQueue } from '../queues';
import { EmailJobData } from '../types/queues';

export class EmailService {
  /**
   * 发送欢迎邮件
   */
  static async sendWelcomeEmail(userId: string, email: string, name: string) {
    return emailQueue.add(
      'send-welcome',
      {
        to: email,
        subject: '欢迎加入！',
        template: 'welcome',
        context: { name, userId },
      },
      {
        jobId: `welcome-${userId}`,
        // 防止重复发送
        opts: { preventDuplicates: true },
      }
    );
  }

  /**
   * 发送密码重置邮件
   */
  static async sendPasswordReset(email: string, token: string) {
    return emailQueue.add(
      'send-password-reset',
      {
        to: email,
        subject: '重置您的密码',
        template: 'password-reset',
        context: { token, resetUrl: `${process.env.APP_URL}/reset?token=${token}` },
      },
      {
        delay: 0,
        attempts: 3,
      }
    );
  }

  /**
   * 发送订单确认邮件
   */
  static async sendOrderConfirmation(email: string, order: any) {
    return emailQueue.add(
      'send-order-confirmation',
      {
        to: email,
        subject: `订单确认 #${order.id}`,
        template: 'order-confirmation',
        context: { order },
      },
      {
        priority: 1, // 高优先级
      }
    );
  }

  /**
   * 批量发送营销邮件
   */
  static async sendMarketingCampaign(
    campaignId: string,
    recipients: Array&lt;{ email: string; name: string }&gt;
  ) {
    // 分批发送，每批 100 个
    const batchSize = 100;
    const batches = [];

    for (let i = 0; i &lt; recipients.length; i += batchSize) {
      const batch = recipients.slice(i, i + batchSize);
      batches.push(
        emailQueue.add(
          'send-batch',
          {
            campaignId,
            recipients: batch,
          },
          {
            priority: 100, // 低优先级
            delay: i * 1000, // 每批间隔 1 秒
          }
        )
      );
    }

    return Promise.all(batches);
  }

  /**
   * 获取任务状态
   */
  static async getEmailStatus(jobId: string) {
    const job = await emailQueue.getJob(jobId);
    if (!job) {
      return null;
    }

    const state = await job.getState();
    return {
      id: job.id,
      name: job.name,
      state,
      progress: job.progress,
      failedReason: job.failedReason,
      returnValue: job.returnvalue,
      timestamp: job.timestamp,
      processedOn: job.processedOn,
      finishedOn: job.finishedOn,
    };
  }
}
```

```typescript
// src/health.ts
import { Queue } from 'bullmq';

export async function checkQueueHealth(queue: Queue) {
  try {
    const [waiting, active, delayed, failed] = await Promise.all([
      queue.getWaitingCount(),
      queue.getActiveCount(),
      queue.getDelayedCount(),
      queue.getFailedCount(),
    ]);

    return {
      name: queue.name,
      healthy: true,
      counts: { waiting, active, delayed, failed },
    };
  } catch (error) {
    return {
      name: queue.name,
      healthy: false,
      error: error.message,
    };
  }
}
```

### 3. 优雅关闭

```typescript
// src/graceful-shutdown.ts
import { Worker, Queue } from 'bullmq';

const workers: Worker[] = [];
const queues: Queue[] = [];

export function registerWorker(worker: Worker) {
  workers.push(worker);
}

export function registerQueue(queue: Queue) {
  queues.push(queue);
}

async function gracefulShutdown(signal: string) {
  console.log(`\n${signal} received, shutting down gracefully...`);

  // 1. 停止接收新任务
  console.log('Pausing queues...');
  await Promise.all(queues.map(q =&gt; q.pause()));

  // 2. 等待当前任务完成
  console.log('Waiting for workers to finish...');
  await Promise.all(workers.map(w =&gt; w.close()));

  // 3. 关闭队列连接
  console.log('Closing queues...');
  await Promise.all(queues.map(q =&gt; q.close()));

  console.log('Graceful shutdown completed');
  process.exit(0);
}

process.on('SIGTERM', () =&gt; gracefulShutdown('SIGTERM'));
process.on('SIGINT', () =&gt; gracefulShutdown('SIGINT'));
```

### Q2: Redis 连接断开？

```typescript
const connection = new IORedis({
  host: 'localhost',
  port: 6379,
  maxRetriesPerRequest: null,
  retryStrategy: (times) =&gt; {
    if (times &gt; 10) {
      console.error('Redis connection failed after 10 retries');
      return null; // 停止重试
    }
    return Math.min(times * 100, 3000); // 最多等待 3 秒
  },
  reconnectOnError: (err) =&gt; {
    const targetError = 'READONLY';
    if (err.message.includes(targetError)) {
      return true; // 重连
    }
    return false;
  },
});
```

### Q3: 如何保证任务幂等性？

```typescript

  jobId: `welcome-${userId}`, // 相同 userId 不会重复添加

// Worker 中实现幂等性检查
const worker = new Worker("email", async (job) => {
  const { userId } = job.data;

  // 检查是否已发送

  if (alreadySent) {
    return { skipped: true, reason: "Already sent" };
  }

  await sendEmail(job.data);

  // 标记为已发送
  await redis.set(`email:sent:${userId}`, "1", "EX", 86400);
```
