## 为什么需要 Go

你已经在用 Java 或 Python 开发后端服务，但可能遇到过这些问题：Python 在高并发下性能不足，Java 部署复杂且启动缓慢。这时候你需要 Go。

Go 的设计目标是解决实际工程问题。它比 Java 简洁（没有复杂的继承体系），比 Python 快（接近 C 的性能），而且天生支持并发。Docker 和 Kubernetes 都是用 Go 写的，这不是巧合。Go 的单一可执行文件部署方式让你摆脱了复杂的环境依赖问题，这在微服务架构中特别有价值。

这个系列会带你构建一个任务队列 API，逐步学习 Go 的核心特性。不过先别急，我们得从环境搭建开始。

## 安装 Go

Go 的安装很直接，不管你用什么操作系统。

Windows 用户去 Go 官网下载 .msi 安装包，双击运行就行。macOS 用户可以用 Homebrew：

```bash
brew install go
```

Linux 用户需要下载压缩包并解压：

```bash
# 下载 Go 1.22 或更新版本
wget https://go.dev/dl/go1.22.0.linux-amd64.tar.gz

# 解压到 /usr/local
sudo tar -C /usr/local -xzf go1.22.0.linux-amd64.tar.gz

# 添加到 PATH（把这段加到 ~/.bashrc 或 ~/.zshrc）
export PATH=$PATH:/usr/local/go/bin
```

安装完成后，验证一下：

```bash
go version
```

应该能看到类似 `go version go1.22.0 darwin/amd64` 的输出。

## 配置环境变量

有几个环境变量需要配置，让 Go 用起来更顺手。

```bash
# Go Modules 代理（中国用户推荐）
export GOPROXY=https://goproxy.cn,direct

# 启用 Go Modules
export GO111MODULE=on
```

这些配置可以加到你的 shell 配置文件里（`~/.bashrc` 或 `~/.zshrc`）。

Windows 用户在系统环境变量里设置这些值就行。设置完之后，运行 `go env` 确认一下配置是否生效。

## 第一个 Go 程序

创建个新文件 `hello.go`，写这么几行：

```go
package main

import "fmt"

func main() {
    fmt.Println("Hello, World!")
}
```

这几行代码包含了 Go 程序的基本结构。

`package main` 声明了这个文件属于 main 包，这是可执行程序的入口包。`import "fmt"` 引入了格式化输出包，用来打印信息。`func main()` 是主函数，程序从这里开始执行。

运行这个程序有两种方式：

```bash
# 直接运行（编译并执行，不生成可执行文件）
go run hello.go

# 先编译再运行（生成可执行文件）
go build hello.go
./hello  # Linux/macOS，或者 hello.exe (Windows)
```

开发时用 `go run` 方便快捷，部署时用 `go build` 生成独立的可执行文件。

## Go Modules 的基本使用

现代 Go 开发都用 Go Modules 管理依赖，它比老的 GOPATH 模式灵活得多。

在项目目录初始化一个模块：

```bash
go mod init example.com/myproject
```

这会生成 `go.mod` 文件，内容大概是这样：

```go
module example.com/myproject

go 1.22
```

当你引入第三方库时，运行 `go mod tidy` 会自动下载依赖并更新 `go.mod` 和 `go.sum` 文件。Go Modules 让依赖管理变得简单，你不再需要操心 GOPATH 那套复杂的目录结构。

## 简单的实战练习

写个稍微完整点的程序，熟悉一下 Go 的基本感觉。假设我们要计算两个数的和与差：

```go
package main

import "fmt"

func main() {
    // 声明两个变量
    var a, b int
    a = 10
    b = 20

    // 计算
    sum := a + b
    difference := a - b

    // 格式化输出
    fmt.Printf("a = %d\n", a)
    fmt.Printf("b = %d\n", b)
    fmt.Printf("和 = %d\n", sum)
    fmt.Printf("差 = %d\n", difference)
}
```

这个程序展示了 Go 的几个特点：变量可以用 `var` 声明，也可以用 `:=` 自动推断类型；`fmt.Printf` 支持格式化输出；代码不需要分号结尾。

## 常用命令速查

Go 自带的工具链很强大，这几个命令最常用：

```bash
go run main.go      # 运行程序
go build main.go    # 编译程序
go test ./...       # 运行测试
go fmt .            # 格式化代码
go mod tidy         # 整理依赖
```

`go fmt` 会自动格式化你的代码，整个团队用统一的风格，不用争论代码格式问题了。

## 开发工具建议

VSCode 加上官方的 Go 扩展就够用了。它提供自动补全、跳转定义、重构等功能。如果你习惯用 JetBrains 的 IDE，GoLand 也是不错的选择，不过要付费。

配置好开发环境后，写 Go 代码的体验会流畅很多。自动补全、实时错误提示、快速跳转，这些功能能提高开发效率。

## 权衡与考虑

Go 不是银弹。它的语法简洁也意味着有时候需要写更多代码（比如没有泛型，虽然 Go 1.18 加入了泛型但还不成熟）。错误处理需要显式检查，不习惯的人会觉得啰嗦。

但 Go 的优势在于部署简单和并发性能。单一可执行文件意味着不用担心服务器上有没有装对依赖版本；goroutine 让并发编程变得相对简单。如果你的项目需要高性能网络服务、云原生应用、或者简单的命令行工具，Go 是个不错的选择。
