## 为什么需要结构体

在处理实际问题时，我们经常需要将不同类型的数据组合在一起。例如，一个用户信息包含姓名（字符串）、年龄（整数）、是否激活（布尔值）。如果使用单独的变量来存储这些信息，代码会变得难以管理。

Go 语言没有像 Java 或 C++ 那样的类，但它提供了结构体。结构体可以将不同类型的数据字段组合成一个自定义类型，这是 Go 实现面向对象编程的基础。

## 结构体基础

### 定义和使用结构体

```go
package main

import "fmt"

// 定义结构体
type User struct {
    Name     string
    Age      int
    Email    string
    IsActive bool
}

func main() {
    // 创建结构体实例的多种方式

    // 方式 1：使用字段名初始化（推荐）
    user1 := User{
        Name:     "张三",
        Age:      25,
        Email:    "zhangsan@example.com",
        IsActive: true,
    }

    // 方式 2：按顺序初始化（不推荐，容易出错）
    user2 := User{"李四", 30, "lisi@example.com", false}

    // 方式 3：使用 new 关键字，返回指针
    user3 := new(User)
    user3.Name = "王五"
    user3.Age = 28
    user3.Email = "wangwu@example.com"
    user3.IsActive = true

    // 访问结构体字段
    fmt.Printf("用户 1: %s, %d 岁\n", user1.Name, user1.Age)
    fmt.Printf("用户 2: %s, %d 岁\n", user2.Name, user2.Age)
    fmt.Printf("用户 3: %s, %d 岁\n", user3.Name, user3.Age)

    // 修改字段值
    user1.Age = 26
    fmt.Printf("%s 现在是 %d 岁\n", user1.Name, user1.Age)
}
```

### 结构体嵌套

结构体可以嵌套，这类似于其他语言的继承。通过嵌套，可以复用已有的结构体定义。

```go
package main

import "fmt"

// 地址结构体
type Address struct {
    City    string
    Street  string
    ZipCode string
}

// 用户结构体，嵌套 Address
type User struct {
    Name    string
    Age     int
    Address Address
}

// 公司结构体，也嵌套 Address
type Company struct {
    CompanyName string
    Address     Address
}

func main() {
    user := User{
        Name: "张三",
        Age:  25,
        Address: Address{
            City:    "北京",
            Street:  "长安街 1 号",
            ZipCode: "100000",
        },
    }

    company := Company{
        CompanyName: "科技公司",
        Address: Address{
            City:    "上海",
            Street:  "世纪大道 100 号",
            ZipCode: "200000",
        },
    }

    fmt.Printf("用户: %s, 城市: %s\n", user.Name, user.Address.City)
    fmt.Printf("公司: %s, 城市: %s\n", company.CompanyName, company.Address.City)
}
```

### 匿名嵌套

当嵌套的结构体是匿名的（没有字段名）时，可以直接访问嵌套结构体的字段，这被称为字段提升。

```go
package main

import "fmt"

type Contact struct {
    Email string
    Phone string
}

type Person struct {
    Name string
    Age  int
    Contact // 匿名嵌套
}

func main() {
    person := Person{
        Name: "张三",
        Age:  25,
        Contact: Contact{
            Email: "zhangsan@example.com",
            Phone: "13800138000",
        },
    }

    // 可以直接访问嵌套结构体的字段
    fmt.Printf("姓名: %s\n", person.Name)
    fmt.Printf("邮箱: %s\n", person.Email) // 字段提升
    fmt.Printf("电话: %s\n", person.Phone) // 字段提升

    // 也可以完整路径访问
    fmt.Printf("邮箱: %s\n", person.Contact.Email)
}
```

## 方法

Go 的方法是带有接收者的函数。接收者可以是结构体类型或结构体指针类型。

### 定义方法

```go
package main

import "fmt"

// 定义矩形结构体
type Rectangle struct {
    Width  float64
    Height float64
}

// 值接收者方法（计算面积）
func (r Rectangle) Area() float64 {
    return r.Width * r.Height
}

// 值接收者方法（计算周长）
func (r Rectangle) Perimeter() float64 {
    return 2 * (r.Width + r.Height)
}

// 指针接收者方法（修改宽度）
func (r *Rectangle) SetWidth(width float64) {
    r.Width = width
}

// 指针接收者方法（修改高度）
func (r *Rectangle) SetHeight(height float64) {
    r.Height = height
}

// 缩放矩形
func (r *Rectangle) Scale(factor float64) {
    r.Width *= factor
    r.Height *= factor
}

func main() {
    rect := Rectangle{Width: 3, Height: 4}

    fmt.Printf("矩形: 宽=%.2f, 高=%.2f\n", rect.Width, rect.Height)
    fmt.Printf("面积: %.2f\n", rect.Area())
    fmt.Printf("周长: %.2f\n", rect.Perimeter())

    // 使用指针接收者方法修改尺寸
    rect.SetWidth(6)
    rect.SetHeight(8)

    fmt.Printf("\n修改后:\n")
    fmt.Printf("矩形: 宽=%.2f, 高=%.2f\n", rect.Width, rect.Height)
    fmt.Printf("面积: %.2f\n", rect.Area())

    // 缩放
    rect.Scale(0.5)
    fmt.Printf("\n缩放后:\n")
    fmt.Printf("矩形: 宽=%.2f, 高=%.2f\n", rect.Width, rect.Height)
}
```

### 值接收者 vs 指针接收者

选择使用值接收者还是指针接收者是一个重要的设计决策。

```go
package main

import "fmt"

type Counter struct {
    count int
}

// 值接收者：不修改原始对象
func (c Counter) GetCount() int {
    return c.count
}

// 指针接收者：可以修改原始对象
func (c *Counter) Increment() {
    c.count++
}

// 指针接收者：可以修改原始对象
func (c *Counter) Add(n int) {
    c.count += n
}

// 值接收者：返回新对象
func (c Counter) WithIncrement() Counter {
    return Counter{count: c.count + 1}
}

func main() {
    counter := &Counter{count: 0}

    fmt.Printf("初始值: %d\n", counter.GetCount())

    counter.Increment()
    fmt.Printf("增加后: %d\n", counter.GetCount())

    counter.Add(5)
    fmt.Printf("加 5 后: %d\n", counter.GetCount())

    // 使用值接收者方法，返回新对象
    newCounter := counter.WithIncrement()
    fmt.Printf("原计数器: %d\n", counter.GetCount())
    fmt.Printf("新计数器: %d\n", newCounter.GetCount())
}
```

## 实战示例：任务队列 API

现在我们用结构体和方法来构建任务队列 API 的核心。

```go
package main

import (
    "fmt"
    "time"
)

// 任务状态常量
const (
    TaskStatusPending    = "待处理"
    TaskStatusProcessing = "处理中"
    TaskStatusCompleted  = "已完成"
    TaskStatusFailed     = "失败"
)

// Task 表示一个任务
type Task struct {
    ID          int
    Name        string
    Priority    int
    Status      string
    CreatedAt   time.Time
    CompletedAt time.Time
    Error       string
}

// NewTask 创建新任务的构造函数
func NewTask(id int, name string, priority int) *Task {
    return &Task{
        ID:        id,
        Name:      name,
        Priority:  priority,
        Status:    TaskStatusPending,
        CreatedAt: time.Now(),
    }
}

// Process 处理任务
func (t *Task) Process() {
    t.Status = TaskStatusProcessing
    fmt.Printf("处理任务: %s (优先级: %d)\n", t.Name, t.Priority)

    // 模拟任务处理时间
    time.Sleep(time.Millisecond * 100)

    // 根据优先级决定是否成功
    if t.Priority > 5 {
        t.Status = TaskStatusFailed
        t.Error = "优先级过高，任务失败"
    } else {
        t.Status = TaskStatusCompleted
        t.CompletedAt = time.Now()
    }
}

// IsCompleted 检查任务是否完成
func (t *Task) IsCompleted() bool {
    return t.Status == TaskStatusCompleted
}

// IsFailed 检查任务是否失败
func (t *Task) IsFailed() bool {
    return t.Status == TaskStatusFailed
}

// String 实现 Stringer 接口
func (t Task) String() string {
    return fmt.Sprintf("Task{ID: %d, Name: %s, Status: %s}", t.ID, t.Name, t.Status)
}

// Worker 表示工作线程
type Worker struct {
    ID        int
    TaskQueue chan *Task
}

// NewWorker 创建新的工作线程
func NewWorker(id int, taskQueue chan *Task) *Worker {
    return &Worker{
        ID:        id,
        TaskQueue: taskQueue,
    }
}

// Start 启动工作线程
func (w *Worker) Start() {
    fmt.Printf("Worker %d 启动\n", w.ID)
    go func() {
        for task := range w.TaskQueue {
            task.Process()
        }
    }()
}

// Stop 停止工作线程
func (w *Worker) Stop() {
    fmt.Printf("Worker %d 停止\n", w.ID)
}

// TaskQueue 任务队列
type TaskQueue struct {
    tasks     []*Task
    workers   []*Worker
    taskQueue chan *Task
    nextID    int
}

// NewTaskQueue 创建新的任务队列
func NewTaskQueue(workerCount int) *TaskQueue {
    taskQueue := make(chan *Task, 100)

    workers := make([]*Worker, workerCount)
    for i := 0; i < workerCount; i++ {
        workers[i] = NewWorker(i+1, taskQueue)
    }

    return &TaskQueue{
        tasks:     make([]*Task, 0),
        workers:   workers,
        taskQueue: taskQueue,
        nextID:    1,
    }
}

// Start 启动任务队列
func (tq *TaskQueue) Start() {
    fmt.Println("任务队列系统启动")
    for _, worker := range tq.workers {
        worker.Start()
    }
}

// AddTask 添加任务
func (tq *TaskQueue) AddTask(name string, priority int) *Task {
    task := NewTask(tq.nextID, name, priority)
    tq.tasks = append(tq.tasks, task)
    tq.taskQueue <- task
    tq.nextID++
    return task
}

// Stop 停止任务队列
func (tq *TaskQueue) Stop() {
    close(tq.taskQueue)
    for _, worker := range tq.workers {
        worker.Stop()
    }
}

// GetCompletedTasks 获取已完成的任务
func (tq *TaskQueue) GetCompletedTasks() []*Task {
    var completed []*Task
    for _, task := range tq.tasks {
        if task.IsCompleted() {
            completed = append(completed, task)
        }
    }
    return completed
}

// GetFailedTasks 获取失败的任务
func (tq *TaskQueue) GetFailedTasks() []*Task {
    var failed []*Task
    for _, task := range tq.tasks {
        if task.IsFailed() {
            failed = append(failed, task)
        }
    }
    return failed
}

// PrintStats 打印统计信息
func (tq *TaskQueue) PrintStats() {
    completed := tq.GetCompletedTasks()
    failed := tq.GetFailedTasks()

    fmt.Println("\n=== 任务统计 ===")
    fmt.Printf("总任务数: %d\n", len(tq.tasks))
    fmt.Printf("已完成: %d\n", len(completed))
    fmt.Printf("失败: %d\n", len(failed))

    if len(failed) > 0 {
        fmt.Println("\n失败的任务:")
        for _, task := range failed {
            fmt.Printf("  - %s: %s\n", task.Name, task.Error)
        }
    }
}

func main() {
    // 创建任务队列，3 个工作线程
    queue := NewTaskQueue(3)
    queue.Start()

    // 添加任务
    tasks := []struct {
        name     string
        priority int
    }{
        {"发送邮件", 2},
        {"生成报表", 3},
        {"备份数据库", 8},
        {"清理缓存", 1},
        {"同步数据", 6},
        {"处理图片", 4},
        {"发送通知", 3},
    }

    fmt.Println("\n=== 添加任务 ===")
    for _, t := range tasks {
        task := queue.AddTask(t.name, t.priority)
        fmt.Printf("添加任务: %s\n", task.Name)
    }

    // 等待任务完成
    time.Sleep(time.Second)

    // 打印统计信息
    queue.PrintStats()

    // 停止队列
    queue.Stop()
}
```

## 权衡与思考

### 值接收者 vs 指针接收者的选择

**使用值接收者的情况**：

- 方法不修改接收者
- 接收者较小（避免复制大结构的开销）
- 需要值语义的操作

**使用指针接收者的情况**：

- 方法需要修改接收者
- 接收者较大（避免复制开销）
- 需要保证一致性（如果有些方法用指针，全部都应该用）

**一致性原则**：
如果类型有一些方法使用指针接收者，那么该类型的所有方法都应该使用指针接收者，以保持一致性。

### 结构体设计的考虑

**小结构体 vs 大结构体**：

- 小结构体更简单，易于理解和测试
- 大结构体可能包含过多职责，考虑拆分

**字段可见性**：

- 小写字母开头的字段是包私有的
- 大写字母开头的字段是公开的
- 通常建议字段私有，通过方法访问

**构造函数**：

- 提供 New 开头的构造函数来初始化复杂结构体
- 确保结构体处于有效状态

### 方法设计的最佳实践

**方法职责单一**：

- 每个方法应该只做一件事
- 方法名应该清晰表达其功能

**避免副作用**：

- 除非必要，方法不应该修改接收者的状态
- 如果修改，应该在文档中明确说明

**错误处理**：

- 方法可能返回错误，调用方应该处理
- 对于不会失败的方法，不需要返回错误

## 总结

结构体和方法是 Go 实现面向对象编程的核心：

- 结构体将相关数据组合在一起
- 方法为结构体添加行为
- 指针接收者允许修改原始对象
- 值接收者提供不可变操作
- 通过结构体嵌套实现代码复用

在任务队列 API 中，我们使用结构体定义了任务和工作线程，使用方法实现了它们的行为。这种设计让代码更加模块化和易于理解。
