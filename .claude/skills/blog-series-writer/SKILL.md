# 博客系列写作助手

这是一个通用的博客系列写作工具，帮助你从零开始创建结构化的技术博客系列。

## 何时使用

当你需要：

- 创建一个系列技术博客（如"React 进阶"、"NestJS 实战"等）
- 制定详细的博客写作计划
- 按照统一格式规范撰写博客文章
- 自动处理文章间的链接关系

## 使用前准备

在激活此 skill 前，请准备以下信息：

1. **博客主题**（如：React、Vue、TypeScript、Node.js 等）
2. **技术栈**（可选，列出要覆盖的核心技术）
3. **实战项目**（可选，一个贯穿系列的项目示例）

## 工作流程

### 第一阶段：制定计划书

**目标**：创建详细的博客系列计划文档

**步骤**：

1. **创建计划文件**
   - 文件路径：`content/posts/{主题}/plan.md`
   - 参考格式：`content/posts/nestjs/plan.md`

2. **计划书内容结构**：

```markdown
# {主题} 渐进式学习计划

## 背景

- 说明为什么选择这个主题
- 技术栈选择标准
- 核心技术列表

## 博客目录结构

列出所有文章的文件名和标题

## 学习阶段划分

根据难度和主题，将文章分为多个阶段（基础→核心→进阶→实战→部署）

## 每篇文章大纲

为每篇文章提供：

- 学习目标
- 内容要点
- 实战示例
- 代码示例

## 写作规范

- 文章结构模板
- 代码示例要求
- 实战项目设计

## 学习路径图

可视化展示学习顺序

## 预期效果

读者学完后将具备的能力
```

3. **确认计划**
   - 展示计划书给用户确认
   - 根据反馈调整

### 第二阶段：撰写博客文章

**目标**：按计划逐篇撰写博客

**每篇文章的结构**：

```markdown
---
title: "序号 文章标题"
date: "YYYY-MM-DD"
summary: "一句话总结（100字以内）"
tags: ["标签1", "标签2"]
category: "{主题}"
cover: "封面图片URL（可选）"
draft: false
series: "{系列名称}"
seriesOrder: N
---

# 文章主标题

## 前言/背景

解释为什么需要学习这个主题，解决了什么问题

## 核心概念

用通俗易懂的语言解释核心概念

## 实战演练

### 示例 1：标题

\`\`\`typescript
// 完整可运行的代码
\`\`\`

**代码解释**：详细说明

## 最佳实践

总结 3-5 个关键要点

## 常见问题

Q&A 格式

## 总结

回顾核心内容

## 下一步

---

- **上一篇**：[文章标题](../文章链接)
- **下一篇**：[文章标题](../文章链接)（如果是最后一篇，则省略）
```

### 第三阶段：命名与链接规范

**文件命名规则**：

- 格式：`序号-kebab-case标题.mdx`
- 示例：`01-getting-started.mdx`、`02-advanced-concepts.mdx`
- 位置：`content/posts/{category}/`

**标题命名规则**：

- 格式：`序号 标题`
- 示例：`01 环境搭建与第一个项目`

**文章链接**：

- 每篇文章末尾添加"上一篇"和"下一篇"链接
- 第一篇只有"下一篇"
- 最后一篇只有"上一篇"

## 写作质量标准

### 代码示例要求

1. ✅ **完整性**：每个示例都是可独立运行的代码
2. ✅ **渐进式**：从简单到复杂，逐步深入
3. ✅ **注释**：关键代码添加中文注释
4. ✅ **错误处理**：展示正确的错误处理方式
5. ✅ **类型安全**：充分利用 TypeScript 类型系统

### 内容要求

- 通俗易懂，避免过多术语堆砌
- 结合实际应用场景
- 提供完整可运行的示例
- 总结最佳实践和常见坑点

### 格式要求

- 使用 Markdown 格式
- 代码块指定语言（`typescript、`bash 等）
- 适当使用表格、列表、引用等格式
- 添加适当的分隔线和空行，提升可读性

## 自动化任务

当用户请求写博客时，按以下步骤执行：

1. **询问主题信息**
   - 主题名称
   - 技术栈
   - 实战项目（可选）

2. **生成计划书**
   - 根据主题和学习路径，自动生成计划书
   - 参考类似主题的计划书结构
   - 划分学习阶段

3. **用户确认**
   - 展示计划书
   - 等待用户反馈和确认

4. **创建目录结构**

   ```
   content/posts/{主题}/
   ├── plan.md
   ├── 01-xxx.mdx
   ├── 02-xxx.mdx
   └── ...
   ```

5. **自动连续撰写** ⚡
   - **重要：一旦开始撰写，自动完成所有文章，不需要用户确认**
   - 按计划书顺序逐篇撰写
   - 每篇完成后立即开始下一篇
   - 自动更新上一篇/下一篇链接
   - 保存到对应目录
   - 每篇完成后简单提示进度

6. **完成提示**
   - 显示所有已完成的文章列表
   - 提示如何预览和发布

## 示例

用户输入：

```
我想写一个 React Hooks 系列博客，面向初学者
```

Agent 执行：

1. 生成 `content/posts/react/plan.md`
2. 确认计划后，创建 `01-hooks-introduction.mdx`
3. 撰写内容，添加适当的示例代码
4. 继续撰写后续文章...

## 技能输出

### 输出物

1. **计划书** (`plan.md`)
2. **系列博客文章** (`.mdx` 文件)
3. **目录结构** (按规范组织)

### 质量保证

- 所有代码示例可运行
- 文章格式统一规范
- 链接关系正确
- 内容循序渐进

## 注意事项

- ⚠️ 写作过程中随时可以暂停，用户可以要求修改
- ⚠️ 每篇文章写完后应保存，避免丢失
- ⚠️ 保持系列文章的一致性（术语、风格、难度）
- ⚠️ 代码示例要考虑兼容性（注明版本要求）
- ⚠️ 及时更新文章间的链接关系

完成基础系列后，可以：

- 创建配套的代码仓库
- 生成在线演示
- 制作视频教程
- 编写练习题和答案
