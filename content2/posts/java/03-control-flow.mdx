Java 的 switch 竟然比 JS 好用？——如果你被 JS 的 switch 穿透（fall-through）坑过，你会喜欢 Java 14+ 的 switch 表达式。

上一篇我们学了 Java 的类型系统，知道了 Java 对类型有多严格。今天来看控制流——这部分你大部分已经会了，但有几个 Java 独有的亮点值得注意。

## if-else：几乎一样

Java 的 if-else 和 JS 完全一致，语法和行为没有任何区别。唯一需要注意的是，Java 中 if 的条件表达式必须是 `boolean` 类型，不能像 JS 那样用非布尔值做隐式转换。

```java
int score = 85;

if (score >= 90) {
    System.out.println("优秀");
} else if (score >= 80) {
    System.out.println("良好");  // 输出这一行
} else if (score >= 60) {
    System.out.println("及格");
} else {
    System.out.println("不及格");
}
```

在 JS 中，`if (0)`、`if ("")`、`if (null)` 都是 falsy。但 Java 里这些写法直接编译报错——Java 编译器不接受 `int`、`String` 或 `null` 作为 if 条件。这是一个类型严格带来的好处：很多隐式 bug 在编译期就被消灭了。

## switch：Java 14+ 的表达式才是正道

### 老语法：和 JS 一样的坑

Java 的传统 switch 和 JS 几乎一样——默认穿透，需要手动 break。

```java
int dayOfWeek = 3;
String dayName;

switch (dayOfWeek) {
    case 1:
        dayName = "周一";
        break;     // 忘了 break？直接穿透到下一个 case
    case 2:
        dayName = "周二";
        break;
    case 3:
        dayName = "周三";
        break;
    default:
        dayName = "未知";
}
```

如果你在 JS 里被 switch 穿透坑过（比如 case 里忘了 break，逻辑直接滑到下一个分支），Java 的传统 switch 也有同样的陷阱。

### 新语法：switch 表达式（Java 14+）

Java 14 正式引入了 switch 表达式，用 `->` 箭头语法替代 `:` 冒号语法，**默认不穿透，不需要 break**。

```java
int dayOfWeek = 3;

// switch 表达式：直接返回值，用 -> 不穿透
String dayName = switch (dayOfWeek) {
    case 1 -> "周一";
    case 2 -> "周二";
    case 3 -> "周三";
    case 4 -> "周四";
    case 5 -> "周五";
    case 6, 7 -> "周末";   // 多个值用逗号合并
    default -> "未知";
};

System.out.println(dayName);  // 周三
```

注意几个关键点：

- **没有穿透**：匹配到一个 case 就结束，不需要 break
- **是表达式**：整个 switch 可以直接赋值给变量
- **多值合并**：`case 6, 7 ->` 代替了以前一堆 `case 6: case 7:` 的写法

如果某个分支需要多行逻辑，用代码块加 `yield` 返回值：

```java
String result = switch (score / 10) {
    case 10, 9 -> {
        System.out.println("优秀！");
        yield "A";
    }
    case 8 -> "B";
    case 7 -> "C";
    default -> "需要努力";
};
```

`yield` 是 switch 表达式里从代码块返回值的专用关键字，和 `break` 不同。JS 至今没有原生的 switch 表达式（TC39 提案还在 Stage 1），所以这是 Java 真正领先于 JS 的地方。

Java 的 switch 还能直接匹配字符串和枚举，这一点 JS 也做不到（JS 的 switch 底层就是 `===` 比较，对字符串没啥特殊处理）。

## for 循环：传统 for 和增强 for

### 传统 for

语法和 JS 完全一样，只是 Java 里没有 `for...in`（遍历数组索引的那种）。

```java
// 和 JS 一样的三段式 for
for (int i = 0; i < 5; i++) {
    System.out.println("i = " + i);
}
```

### 增强 for（for-each）

Java 没有 `for...of`，但有功能等价的增强 for 循环：

```java
int[] numbers = {10, 20, 30, 40, 50};

// Java 的 for-each：遍历数组或集合
for (int num : numbers) {
    System.out.println(num);
}
```

对比 JS：

```javascript
// JS 的 for...of
for (const num of numbers) {
  console.log(num);
}
```

两者功能一致，语法略有不同：Java 用 `:` 分隔，JS 用 `of` 关键字。Java 的增强 for 可以遍历数组和任何实现了 `Iterable` 接口的集合（List、Set 等）。

那 JS 的 `for...in`（遍历对象键名）呢？Java 没有直接等价物。如果你需要遍历 Map 的键值对，得用 `entrySet()`：

```java
Map<String, Integer> scores = Map.of("Alice", 90, "Bob", 85);

// 遍历 Map 的键值对
for (var entry : scores.entrySet()) {
    System.out.println(entry.getKey() + ": " + entry.getValue());
}
```

## while / do-while：快速过

和 JS 完全一样，没有任何区别。

```java
// while：先判断后执行
int count = 0;
while (count < 3) {
    System.out.println("count = " + count);
    count++;
}

// do-while：先执行后判断，至少执行一次
int input;
do {
    System.out.println("请输入一个正数");
    // input = scanner.nextInt();
    input = 1;  // 示例值
} while (input <= 0);
```

唯一值得注意的是：Java 的 `while (true)` 不需要额外声明变量为 `boolean`，因为 `true` 本身就是布尔字面量。

## break / continue：带标签的 break

基础的 break 和 continue 和 JS 一样。但 Java 的**带标签 break**比 JS 更常用——因为 Java 没有数组的高阶方法（如 `Array.prototype.find`），嵌套循环的场景更多。

```java
// 在二维数组中查找目标值
int[][] matrix = {
    {1, 2, 3},
    {4, 5, 6},
    {7, 8, 9}
};
int target = 5;

// 带标签的 break：直接跳出外层循环
search:
for (int i = 0; i < matrix.length; i++) {
    for (int j = 0; j < matrix[i].length; j++) {
        if (matrix[i][j] == target) {
            System.out.println("找到 " + target + "，位置：[" + i + "][" + j + "]");
            break search;  // 直接跳出外层 for，而不是内层
        }
    }
}
```

没有标签的话，`break` 只能跳出最内层的循环。加了 `search:` 标签后，`break search` 直接跳出带标签的那个循环。JS 也有标签语句，但实际项目中几乎没人用。在 Java 里，带标签的 break 在处理嵌套循环时是一个相当常见的模式。

`continue` 也支持标签——`continue search` 会跳过当前迭代，直接进入外层循环的下一次迭代。

## 三元运算符：和 JS 一样

`条件 ? 值1 : 值2`，完全一致。不过因为 Java 有 switch 表达式，很多以前需要嵌套三元运算符的场景，现在可以用 switch 表达式更清晰地表达：

```java
// 以前：嵌套三元，可读性差
String grade = score >= 90 ? "A" : score >= 80 ? "B" : score >= 70 ? "C" : "D";

// 现在：switch 表达式，更清晰
String grade = switch (score / 10) {
    case 10, 9 -> "A";
    case 8 -> "B";
    case 7 -> "C";
    default -> "D";
};
```

## JS 对比速查

| 特性             | JavaScript                 | Java                             |
| ---------------- | -------------------------- | -------------------------------- |
| if-else          | 完全一致                   | 完全一致，但条件必须是 boolean   |
| switch 穿透      | 默认穿透，需要 break       | switch 表达式用 `->` 不穿透      |
| switch 表达式    | 无原生支持（TC39 Stage 1） | Java 14+ 支持，用 `yield` 返回值 |
| for...of         | `for (const x of arr)`     | 增强 for：`for (T x : arr)`      |
| for...in         | 遍历对象键名               | 无等价物，用 `Map.entrySet()`    |
| while / do-while | 完全一致                   | 完全一致                         |
| 带标签 break     | 语法支持但极少使用         | 更常用，嵌套循环中的标准模式     |
| 三元运算符       | 完全一致                   | 完全一致                         |

## Trade-off 分析

Java 的 switch 表达式是真正比 JS 好用的特性。默认不穿透的设计消除了一个常见的 bug 来源，而且作为表达式可以直接赋值，写法更简洁。如果你用 Java 14+，没有任何理由再用传统的 switch 语句（除非维护老代码）。

带标签的 break 在 Java 中的存在感比 JS 高，这反映了两个语言生态的差异：JS 有 `Array.prototype.find`、`some`、`every` 等高阶方法来替代嵌套循环，Java 在没有 Stream API 之前只能靠带标签的 break。当然，现在有了 Stream，嵌套循环的使用场景也在减少。

控制流层面的核心差异不在语法，而在类型系统带来的约束：Java 的条件必须是 boolean，switch 的类型必须是编译期已知的（int、String、enum），这意味着很多在 JS 里运行时才暴露的逻辑错误，在 Java 里编译就过不了。
