# PostgreSQL 渐进式学习计划

## 背景

你写过 SQL，但不知道什么是执行计划。你建了表，但不知道什么时候该加索引。你用过外键，但不知道 cascade 是什么意思。你遇到过慢查询，但不知道怎么优化。

这个系列从"为什么需要数据库"出发，带你逐步掌握 PostgreSQL 的核心能力和生产实践。


**核心问题**：为什么不直接用文件存数据？数据库比文件系统好在哪里？ACID 是什么？

### 02. SQL 语言基础

- JOIN：INNER、LEFT、RIGHT、FULL
- 聚合函数：COUNT、SUM、AVG、MAX、MIN
- 子查询：WHERE、FROM、SELECT 中的子查询

**与前篇关联**：上一篇讲了数据库概念，这一篇学 SQL 语法。

**核心问题**：金额用什么类型存？电话号码用 VARCHAR 还是 INT？枚举用字符串还是 ENUM？

- 枚举类型：CREATE ENUM

**与前篇关联**：上一篇学会了 PostgreSQL 基础，这一篇设计表结构。

### 05. 关系设计（外键、关联）

**核心问题**：订单和用户是什么关系？商品和分类是什么关系？外键 ON DELETE 怎么配？

**主线案例**：设计电商系统的表关系：用户-订单、订单-商品项、商品-分类。

**内容要点**：

- 一对一：用户和用户资料

- 外键：FOREIGN KEY 约束
- 级联操作：ON DELETE CASCADE、SET NULL、RESTRICT
- JOIN 查询：如何关联查询
- 自关联：分类的父子关系

**与前篇关联**：上一篇设计了单表，这一篇把表关联起来。

### 06. 范式理论与反范式

**核心问题**：什么是第一、二、三范式？为什么有时候要故意违反范式（反范式）？

**主线案例**：分析电商订单表的设计，哪些符合范式、哪些需要反范式。

**内容要点**：

- 第一范式（1NF）：列的原子性
- 第二范式（2NF）：非主键字段完全依赖主键
- 第三范式（3NF）：消除传递依赖
- BCNF：Boyce-Codd 范式
- 什么时候要反范式：读多写少、性能优化
- 冗余设计：计算结果缓存、计数器字段

**与前篇关联**：上一篇设计了关系，这一篇评估设计是否合理。

## 性能篇：索引与优化（07-09）

### 07. 索引原理与类型

**核心问题**：索引为什么能加速查询？什么时候该加索引？什么情况下索引会失效？

**主线案例**：为电商系统的常用查询添加索引。

**内容要点**：

- 索引是什么：B-Tree 结构、查找效率
- 主键索引：PRIMARY KEY 自动创建
- 唯一索引：UNIQUE 索引
- 普通索引：CREATE INDEX
- 复合索引：多列索引、最左前缀原则
- 部分索引：带 WHERE 条件的索引
- 表达式索引：函数索引
- 索引的代价：写入性能、磁盘占用

**与前篇关联**：上一篇完成了表设计，这一篇优化查询性能。

### 08. SQL 查询优化

**核心问题**：查询很慢，怎么判断是哪一步的问题？怎么重写 SQL 让它更快？

**主线案例**：优化电商系统的慢查询。

**内容要点**：

- 慢查询定位：pg_stat_statements
- EXPLAIN ANALYZE：分析查询计划
- 常见性能问题：全表扫描、N+1 查询、隐式类型转换
- 优化技巧：覆盖索引、子查询改 JOIN、LIMIT 优化
- 分页优化：大数据量分页的几种方案
- 写入优化：批量 INSERT、COPY 命令
- VACUUM：清理死元组

**与前篇关联**：上一篇讲了索引原理，这一篇实际优化查询。

### 09. 执行计划分析

**核心问题**：EXPLAIN 输出一堆信息，哪些是关键？Seq Scan、Index Scan、Nested Loop 是什么意思？

**主线案例**：解读电商系统查询的执行计划，找出性能瓶颈。

**内容要点**：

- EXPLAIN 和 EXPLAIN ANALYZE 的区别
- 扫描方式：Seq Scan、Index Scan、Index Only Scan
- 连接方式：Nested Loop、Hash Join、Merge Join
- 成本估算：cost、rows、width
- 真实执行时间：actual time、actual rows
- 缓冲区命中：shared hit、shared read
- 如何根据执行计划优化

**与前篇关联**：上一篇讲了优化技巧，这一篇深入执行计划。

## 生产篇：备份与监控（10-11）

### 10. 备份与恢复策略

**核心问题**：数据库崩溃了怎么办？怎么定期备份？怎么恢复到某个时间点？

**主线案例**：为电商系统设计备份方案。

**内容要点**：

- 为什么需要备份：硬件故障、误操作、灾难恢复
- pg_dump：逻辑备份
- pg_dumpall：备份所有数据库
- pg_restore：恢复备份
- PITR：时间点恢复（WAL 归档）
- 连续归档：archive_mode 配置
- 恢复演练：定期验证备份可用性
- 备份策略：全量 + 增量、保留周期

**与前篇关联**：上一篇完成了性能优化，这一篇关心数据安全。

### 11. 监控、调优与高可用

**核心问题**：怎么监控 PostgreSQL 的健康状态？连接池怎么配？怎么做高可用？

**主线案例**：搭建电商系统的监控和高可用架构。

**内容要点**：

- 监控指标：连接数、TPS、命中率、锁等待
- pg_stat_activity：查看当前活动
- pg_stat_database：数据库级别统计
- 慢查询日志：log_min_duration_statement
- 连接池：PgBouncer
- 主从复制：流复制、逻辑复制
- 高可用：Patroni + etcd
- 分库分表：什么时候需要拆分

**与前篇关联**：上一篇讲了备份，这一篇讲运维和高可用。
