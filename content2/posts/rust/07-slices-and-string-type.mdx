## 问题场景

如果你使用过 Python 或 JavaScript，可能习惯了这样的字符串操作：

```javascript
// JavaScript
let s = "hello";
let first = s[0]; // 'h'
let substring = s.slice(0, 3); // 'hel'
```

但在 Rust 中，类似的代码会报错：

```rust
fn main() {
    let s = String::from("hello");
    // let first = s[0];  // 编译错误：String 不能直接索引
    // let substring = &s[0..2];  // 对中文可能 panic
}
```

更让人困惑的是，Rust 有两种字符串类型：`String` 和 `&str`。你可能遇到过这些编译错误：

- "expected `&str`, found `String`"
- "cannot borrow as mutable"
- "the trait bound `String: Index<_>` is not satisfied"

为什么 Rust 的字符串这么"复杂"？答案是：**Rust 对字符串的正确性要求更高**。在大多数语言中，字符串只是字节数组，但 Rust 强制你正确处理 UTF-8 编码。

## 切片的基本概念

切片是对集合中一段连续元素的引用，不拥有所有权。

### 字符串切片

字符串切片是最常见的切片类型：

```rust
fn main() {
    let s = String::from("hello world");

    // 创建字符串切片
    let hello = &s[0..5];    // "hello"
    let world = &s[6..11];   // "world"
    let whole = &s[..];      // 整个字符串

    println!("{} {}", hello, world);  // hello world
    println!("{}", whole);            // hello world
}
```

切片的语法 `[start..end]` 是左闭右开的区间：包含 start，不包含 end。

### 切片的语法糖

```rust
fn main() {
    let s = String::from("hello world");

    // 从开头开始，可以省略 0
    let slice1 = &s[0..5];   // 等价于 &s[..5]
    let slice2 = &s[..5];

    // 到末尾结束，可以省略末尾索引
    let slice3 = &s[6..11];  // 等价于 &s[6..]
    let slice4 = &s[6..];

    // 整个字符串
    let slice5 = &s[0..s.len()];  // 等价于 &s[..]
    let slice6 = &s[..];

    println!("{} {} {} {} {} {}", slice1, slice2, slice3, slice4, slice5, slice6);
}
```

### 数组切片

切片不仅适用于字符串，也适用于数组：

```rust
fn main() {
    let arr = [1, 2, 3, 4, 5];

    // 创建数组切片
    let slice = &arr[1..3];  // &[i32] 类型的切片
    println!("{:?}", slice);  // [2, 3]

    // 切片可以作为函数参数
    let sum = sum_slice(&arr[1..4]);
    println!("切片求和: {}", sum);  // 9
}

fn sum_slice(slice: &[i32]) -> i32 {
    let mut total = 0;
    for &item in slice {
        total += item;
    }
    total
}
```

### 切片作为函数参数

切片非常适合作为函数参数，因为它既接受数组引用，也接受向量引用：

```rust
fn main() {
    let arr = [1, 2, 3, 4, 5];
    let vec = vec![10, 20, 30];

    // 数组切片
    println!("数组求和: {}", sum(&arr));        // 15
    // 向量切片
    println!("向量求和: {}", sum(&vec));        // 60
    // 部分切片
    println!("部分求和: {}", sum(&arr[1..4]));   // 9
}

// 参数使用切片，可以同时接受数组和向量
fn sum(numbers: &[i32]) -> i32 {
    let mut total = 0;
    for &num in numbers {
        total += num;
    }
    total
}
```

## String vs &str

Rust 的字符串体系是很多初学者的痛点。理解 `String` 和 `&str` 的区别是关键。

### String：拥有的、可增长的字符串

`String` 是一个堆分配的、可增长的、可修改的字符串缓冲区：

```rust
fn main() {
    // 创建 String 的几种方式
    let s1 = String::from("hello");
    let s2 = "world".to_string();
    let s3 = String::new();
    let s4 = String::from("你好");

    // String 可以修改
    let mut s = String::from("hello");
    s.push_str(", world");
    s.push('!');
    println!("{}", s);  // hello, world!
}
```

### &str：字符串切片

`&str` 是对字符串的不可变引用：

```rust
fn main() {
    // 字符串字面量的类型就是 &str
    let s1: &str = "hello";

    // &str 也可以指向 String 的一部分
    let s2 = String::from("hello world");
    let s3: &str = &s2[0..5];  // "hello"

    println!("{} {} {}", s1, s3, s2);
}
```

### String 与 &str 的转换

```rust
fn main() {
    // String → &str（简单，自动解引用）
    let s = String::from("hello");
    let slice: &str = &s;           // 自动解引用
    let slice2: &str = &s[..];      // 使用切片语法
    let slice3: &str = s.as_str();  // 显式转换

    // &str → String
    let s2: String = String::from("hello");   // 从 &str 创建
    let s3: String = "hello".to_string();     // 调用 to_string()
    let s4: String = "hello".into();          // 使用 Into trait

    println!("{} {} {} {} {}", slice, slice2, slice3, s3, s4);
}
```

### 为什么函数参数推荐用 &str？

```rust
// 推荐：参数用 &str，更灵活
fn print_length(s: &str) {
    println!("'{}' 的长度是 {}", s, s.len());
}

// 不推荐：参数用 &String，灵活性差
fn print_length_string(s: &String) {
    println!("'{}' 的长度是 {}", s, s.len());
}

fn main() {
    let s = String::from("hello");

    // 两种方式都能接受 String
    print_length(&s);          // 可以
    print_length_string(&s);   // 可以

    // 但只有 &str 版本能接受字符串字面量
    print_length("world");          // 可以
    // print_length_string("world"); // 编译错误
}
```

原因：`String` 可以自动解引用为 `&str`，但 `&str` 不能自动变为 `&String`。所以 `&str` 更通用。

## UTF-8 与字符串索引

### 为什么不能用索引访问字符串？

在很多语言中，你可以用 `s[0]` 获取第一个字符。但 Rust 不允许这样做：

```rust
fn main() {
    let s = String::from("hello");
    // let h = s[0];  // 编译错误：String 不能直接索引
}
```

这是因为 Rust 的字符串是 UTF-8 编码的，不同字符占用的字节数不同：

```rust
fn main() {
    let ascii = String::from("hello");
    println!("ASCII 字节数: {}", ascii.len());  // 5，每个字符 1 字节

    let chinese = String::from("你好");
    println!("中文字节数: {}", chinese.len());  // 6，每个字符 3 字节

    let hindi = String::from("नमस्ते");
    println!("印地语字节数: {}", hindi.len());  // 18，6 个字符 18 字节
}
```

如果允许 `s[0]`，对于中文来说，返回的是第一个字节（228），这不是一个有效的字符。所以 Rust 直接禁止了索引访问。

### 正确的遍历方式

```rust
fn main() {
    let text = "Hello, 你好！";

    // 遍历字节
    println!("字节:");
    for b in text.bytes() {
        print!("{} ", b);
    }
    println!();

    // 遍历字符（推荐）
    println!("字符:");
    for c in text.chars() {
        println!("  '{}'", c);
    }

    // 获取字符数
    println!("字符数: {}", text.chars().count());
}
```

### 字符串切片的危险性

字符串切片以字节索引为单位，跨越字符边界会导致 panic：

```rust
fn main() {
    let s = String::from("hello");
    let slice = &s[0..2];  // 可以：都是 ASCII，每个字符 1 字节
    println!("{}", slice);  // "he"

    let chinese = String::from("你好");
    let slice2 = &chinese[0..3];  // 可以："你" 占 3 字节
    println!("{}", slice2);  // "你"

    // let slice3 = &chinese[0..1];  // panic：切到了字符中间
}
```

## 实战示例

### 示例1：提取第一个单词

```rust
fn main() {
    let sentence = String::from("hello world from rust");

    let first_word = first_word(&sentence);
    println!("第一个单词: {}", first_word);  // hello

    // 也可以直接传字符串字面量
    let word = first_word("good morning");
    println!("第一个单词: {}", word);  // good
}

// 参数用 &str，同时接受 String 和字符串字面量
fn first_word(s: &str) -> &str {
    // 将字符串转换为字节数组
    let bytes = s.as_bytes();

    // 遍历字节，查找空格
    for (i, &byte) in bytes.iter().enumerate() {
        if byte == b' ' {
            // 找到空格，返回空格前的切片
            return &s[..i];
        }
    }

    // 没有空格，返回整个字符串
    &s[..]
}
```

### 示例2：安全的字符串截取

```rust
fn main() {
    let text = String::from("Hello, 世界!");

    // 安全方式：使用 chars 获取前 N 个字符
    let first_n_chars: String = text.chars().take(5).collect();
    println!("前 5 个字符: {}", first_n_chars);  // Hello

    // 安全方式：使用 char_indices 获取字节边界
    if let Some((end_idx, _)) = text.char_indices().nth(5) {
        println!("前 5 个字符: {}", &text[..end_idx]);  // Hello,
    }

    // 查找字符位置
    if let Some(pos) = text.find('世') {
        println!("'世' 在字节位置: {}", pos);  // 7
    }
}
```

### 示例3：字符串常用操作

```rust
fn main() {
    let mut s = String::from("Hello");

    // 追加字符串
    s.push_str(", World");
    println!("{}", s);  // Hello, World

    // 追加字符
    s.push('!');
    println!("{}", s);  // Hello, World!

    // 替换
    let replaced = s.replace("World", "Rust");
    println!("替换: {}", replaced);  // Hello, Rust!

    // 分割
    let parts: Vec<&str> = "a-b-c-d".split('-').collect();
    println!("分割: {:?}", parts);  // ["a", "b", "c", "d"]

    // 裁剪
    let trimmed = "  hello  ".trim();
    println!("裁剪: '{}'", trimmed);  // 'hello'

    // 大小写转换
    println!("小写: {}", "HELLO".to_lowercase());  // hello
    println!("大写: {}", "hello".to_uppercase());  // HELLO
}
```

### 示例4：字符串格式化

```rust
fn main() {
    let name = "Rust";
    let version = 1.75;

    // format! 宏：创建格式化字符串
    let s1 = format!("{} 版本 {}", name, version);
    println!("{}", s1);  // Rust 版本 1.75

    // println! 宏：直接打印
    println!("{} 是一门{}的语言", name, "强大");

    // 格式化选项
    let num = 42;
    println!("二进制: {:b}", num);   // 101010
    println!("十六进制: {:x}", num); // 2a
    println!("宽度5: {:5}", num);    // "   42"
    println!("补零5: {:05}", num);   // "00042"
    println!("精度: {:.2}", 3.14159); // 3.14
}
```

## 常见错误与解决

### 错误1：混淆 String 和 &str

```rust
fn main() {
    let s = String::from("hello");

    // 错误：期望 &str，提供了 String
    // let result = some_function(s);

    // 解决：传引用
    let result = some_function(&s);

    // 或者使用 as_str()
    let result = some_function(s.as_str());
}

fn some_function(s: &str) -> &str {
    s
}
```

### 错误2：字符串切片跨越字符边界

```rust
fn main() {
    let s = String::from("你好");

    // 错误：'你' 占 3 字节，索引 1 在字符中间
    // let slice = &s[0..1];  // panic

    // 解决：使用 chars 正确获取字符
    let first = s.chars().next().unwrap();
    println!("第一个字符: {}", first);  // 你
}
```

### 错误3：尝试修改 &str

```rust
fn main() {
    let s = "hello";
    // s.push_str(" world");  // 错误：&str 是不可变的

    // 解决：转换为 String 再修改
    let mut s = s.to_string();
    s.push_str(" world");
    println!("{}", s);  // hello world
}
```

### 错误4：字符串拼接的类型不匹配

```rust
fn main() {
    let s1 = String::from("hello");
    let s2 = String::from(" world");

    // 错误：不能用 + 拼接两个 String
    // let s3 = s1 + s2;

    // 解决：String + &str
    let s3 = s1 + &s2;  // 注意：s1 被移动了！
    println!("{}", s3);  // hello world

    // 或者使用 format! 不消耗所有权
    let a = String::from("hello");
    let b = String::from(" world");
    let c = format!("{}{}", a, b);
    println!("{}", c);  // hello world
    println!("{} {}", a, b);  // a 和 b 仍然可用
}
```

## 字符串处理的权衡

### Rust 字符串设计的优势

1. **UTF-8 保证**：Rust 强制你正确处理 UTF-8，避免产生乱码
2. **内存安全**：编译期保证不会访问无效内存或越界
3. **明确的所有权**：清楚知道谁拥有字符串数据，避免悬垂指针
4. **零拷贝切片**：`&str` 可以引用 `String` 的一部分，无需拷贝

### Rust 字符串的复杂性

1. **学习曲线陡峭**：需要理解 UTF-8、字节索引、字符边界等概念
2. **代码冗长**：简单的字符串操作可能需要更多代码
3. **类型转换频繁**：`String` 和 `&str` 之间需要显式转换
4. **索引限制**：不能用整数索引访问字符

### 最佳实践建议

1. **函数参数优先用 `&str`**：比 `&String` 更灵活，能同时接受字符串字面量和 String
2. **大量拼接用 `String::with_capacity`**：预先分配容量，避免重复扩容
3. **处理 Unicode 用 `chars()`**：不要用字节索引来遍历字符
4. **格式化用 `format!`**：不消耗所有权，可读性好
5. **切片要确保在字符边界上**：使用 `char_indices()` 获取安全的字节位置

```rust
fn main() {
    // 预先分配容量，减少内存重新分配
    let mut s = String::with_capacity(100);
    s.push_str("hello");
    s.push_str(" world");
    println!("容量: {}, 长度: {}", s.capacity(), s.len());
}
```

## 什么时候使用哪个类型？

**使用 `String` 的场景：**

- 需要拥有字符串数据
- 需要修改字符串内容
- 需要动态构建字符串
- 结构体字段需要拥有数据

**使用 `&str` 的场景：**

- 只需要读取字符串
- 函数参数（更灵活）
- 字符串切片（引用部分数据）
- 字符串字面量

## 总结

切片和字符串类型是 Rust 中重要的概念。理解 `String` 和 `&str` 的区别，以及 UTF-8 编码对字符串操作的影响，是编写正确 Rust 代码的基础。

**核心要点：**

1. 切片是对集合中连续元素的引用，不拥有所有权
2. `String` 是拥有的、可增长的字符串；`&str` 是不可变的字符串切片
3. 函数参数优先使用 `&str`，因为它更通用
4. 字符串索引以字节为单位，可能跨越字符边界导致 panic
5. 使用 `chars()` 遍历字符，`char_indices()` 获取安全的字节位置
6. 字符串操作要考虑 UTF-8 编码，不能简单地用索引访问
