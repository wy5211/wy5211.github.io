## 为什么需要深入理解函数和控制流

在编程中，函数是组织代码的基本单元，控制流决定程序的执行路径。虽然这些概念在其他语言中也很常见，但 Rust 的实现有一些独特的特性，理解这些特性对于编写地道且高效的 Rust 代码至关重要。

Rust 的函数设计体现了几个重要的理念：

**表达式导向**：Rust 中几乎所有东西都是表达式，包括函数体、if 语句、块等。这让代码更简洁，也更容易组合。

**安全优先**：函数参数和返回值必须显式标注类型，编译器会在编译期检查类型匹配，防止类型相关的错误。

**零成本抽象**：函数调用没有额外的运行时开销，编译器可以内联小函数，提高性能。

**模式匹配**：Rust 的 match 表达式比传统的 switch 语句更强大，可以匹配复杂的模式。

理解这些特性不仅让你能写出更好的 Rust 代码，还会改变你对编程的思考方式。从命令式的"做什么"转变为声明式的"要什么"，代码会变得更加清晰和安全。

## 函数定义和使用

### 基本函数定义

在 Rust 中，使用 `fn` 关键字定义函数：

```rust
// 定义一个简单的函数
fn greet() {
    println!("Hello, world!");
}

fn main() {
    // 调用函数
    greet();
    greet();  // 可以多次调用
}
```

Rust 的函数命名遵循 snake_case 规范，这与很多其他语言（如 Java 的 camelCase）不同。遵守社区规范可以让代码更易读，也更容易被其他 Rust 开发者接受。

### 函数参数

函数参数是函数与外部交互的接口，Rust 要求必须显式标注每个参数的类型：

```rust
// 函数可以接受多个参数
fn print_labeled_measurement(value: i32, unit_label: char) {
    println!("测量值: {}{}", value, unit_label);
}

fn main() {
    // 调用函数时传递参数
    print_labeled_measurement(5, 'h');
    print_labeled_measurement(100, 'kg');

    // 参数的类型必须匹配
    // print_labeled_measurement(5.5, 'h');  // 错误：期望 i32，找到 f64
}
```

显式类型标注虽然看起来有些繁琐，但它带来了几个好处：

1. **文档作用**：函数签名清楚地说明了期望的输入类型
2. **编译期检查**：编译器可以在编译期发现类型错误
3. **更好的 IDE 支持**：IDE 可以提供准确的代码补全和类型提示

### 函数返回值

函数可以返回值，使用 `->` 指定返回值类型：

```rust
// 有返回值的函数
fn add(x: i32, y: i32) -> i32 {
    // 函数体：最后一个表达式（没有分号）作为返回值
    x + y
}

fn main() {
    let result = add(5, 3);
    println!("5 + 3 = {}", result);

    // 表达式调用也可以
    println!("2 + 4 = {}", add(2, 4));
}
```

这里需要注意 Rust 中一个重要的概念：**语句和表达式的区别**。

**语句**（Statement）执行某个动作但不返回值，通常以分号结尾：

```rust
fn main() {
    let y = 6;  // 这是一个语句，不返回值

    // let y = (let x = 6);  // 错误：let 语句不返回值
}
```

**表达式**（Expression）计算并返回一个值，不以分号结尾：

```rust
fn main() {
    // 代码块可以作为表达式
    let y = {
        let x = 3;
        x + 1  // 注意：没有分号，这是表达式
    };

    println!("y = {}", y);  // 输出: y = 4

    // 如果加了分号，就变成语句
    let z = {
        let x = 3;
        x + 1;  // 有分号，这是语句，返回 ()
    };

    println!("z = {:?}", z);  // 输出: z = ()
}
```

### 早期返回

使用 `return` 关键字可以提前从函数返回：

```rust
fn abs(x: i32) -> i32 {
    if x < 0 {
        return -x;  // 提前返回
    }
    x  // 最后的表达式作为返回值
}

fn divide(a: f64, b: f64) -> Option<f64> {
    if b == 0.0 {
        return None;  // 除以零，返回 None
    }
    Some(a / b)  // 正常情况，返回 Some
}

fn main() {
    println!("绝对值: {}", abs(-5));
    println!("绝对值: {}", abs(3));

    match divide(10.0, 2.0) {
        Some(result) => println!("结果: {}", result),
        None => println!("不能除以零"),
    }
}
```

在实际代码中，提前返回常用于错误处理和边界条件检查，可以让主要逻辑更清晰。

## 控制流结构

### if 表达式

Rust 的 `if` 是一个表达式，可以返回值：

```rust
fn main() {
    let number = 6;

    // if 表达式可以返回值
    let result = if number % 2 == 0 {
        "偶数"
    } else {
        "奇数"
    };

    println!("{} 是 {}", number, result);

    // 多个条件
    if number % 4 == 0 {
        println!("能被 4 整除");
    } else if number % 3 == 0 {
        println!("能被 3 整除");
    } else if number % 2 == 0 {
        println!("能被 2 整除");
    } else {
        println!("不能被 2、3、4 整除");
    }
}
```

需要注意的是，`if` 表达式的所有分支必须返回相同类型的值：

```rust
fn main() {
    let condition = true;

    // 正确：两个分支都返回整数
    let number = if condition { 5 } else { 6 };

    // 错误：分支返回不同类型
    // let value = if condition { 5 } else { "six" };
}
```

`if` 作为表达式使得代码更简洁，避免了三元运算符（Rust 没有类似 `condition ? a : b` 的三元运算符，因为 `if` 表达式已经提供了这个功能）。

### loop 循环

`loop` 创建一个无限循环，直到显式使用 `break` 退出：

```rust
fn main() {
    let mut counter = 0;

    // 无限循环
    loop {
        counter += 1;

        if counter > 10 {
            break;  // 退出循环
        }

        println!("计数: {}", counter);
    }

    println!("最终计数: {}", counter);
}
```

`loop` 可以返回值：

```rust
fn main() {
    let mut counter = 0;

    // loop 可以返回值
    let result = loop {
        counter += 1;

        if counter == 10 {
            break counter * 2;  // 返回 20
        }
    };

    println!("结果: {}", result);  // 输出: 结果: 20
}
```

对于嵌套循环，可以使用循环标签来指定 `break` 或 `continue` 的目标：

```rust
fn main() {
    let mut count = 0;

    // 外层循环标签 'outer
    'outer: loop {
        println!("外层循环: {}", count);

        let mut remaining = 10;

        loop {
            println!("  内层循环: {}", remaining);

            if remaining == 9 {
                break;  // 只退出内层循环
            }
            if count == 2 {
                break 'outer;  // 退出外层循环
            }
            remaining -= 1;
        }

        count += 1;
    }

    println!("循环结束");
}
```

### while 循环

`while` 循环在条件为真时持续执行：

```rust
fn main() {
    let mut number = 3;

    // while 循环
    while number != 0 {
        println!("{}!", number);
        number -= 1;
    }

    println!("发射!");
}
```

`while` 循环常用于条件不确定的循环，比如读取文件直到结束，或等待某个条件满足。

### for 循环

`for` 循环用于遍历迭代器，是最常用的循环方式：

```rust
fn main() {
    // 遍历范围
    for number in 1..4 {  // 1 到 3（不包含 4）
        println!("{}", number);
    }

    // 包含结束值
    for number in 1..=4 {  // 1 到 4（包含 4）
        println!("{}", number);
    }

    // 反向遍历
    for number in (1..=4).rev() {
        println!("倒计时: {}", number);
    }
}
```

遍历数组时，`for` 循环比 `while` 循环更安全：

```rust
fn main() {
    let a = [10, 20, 30, 40, 50];

    // 使用 for 遍历（推荐）
    for element in a {
        println!("元素: {}", element);
    }

    // 同时获取索引和元素
    for (index, element) in a.iter().enumerate() {
        println!("a[{}] = {}", index, element);
    }

    // 使用 while 索引遍历（不推荐，容易越界）
    let mut index = 0;
    while index < a.len() {
        println!("a[{}] = {}", index, a[index]);
        index += 1;
    }
}
```

`for` 循环的优势包括：

1. **更安全**：不会越界访问数组
2. **更简洁**：不需要手动管理索引
3. **更高效**：编译器可以进行更多优化

## match 表达式

`match` 是强大的模式匹配工具，比其他语言的 switch 语句更强大：

```rust
fn main() {
    let number = 13;

    // match 必须穷尽所有可能
    match number {
        // 匹配单个值
        1 => println!("一"),
        // 匹配多个值
        2 | 3 | 5 | 7 => println!("质数"),
        // 匹配范围
        13..=19 => println!("青少年"),
        // 默认分支（_ 是通配符）
        _ => println!("其他"),
    }

    // match 是表达式，可以返回值
    let description = match number {
        1 => "one",
        2 => "two",
        3..=10 => "between three and ten",
        _ => "something else",
    };

    println!("数字是: {}", description);
}
```

### 匹配元组

`match` 可以解构元组：

```rust
fn main() {
    let pair = (2, -2);

    match pair {
        (0, y) => println!("x 是零, y 是 {}", y),
        (x, 0) => println!("x 是 {}, y 是零", x),
        _ => println!("其他"),
    }

    // 使用匹配守卫（match guard）
    let pair = (2, -2);
    match pair {
        (x, y) if x == -y => println!("x 和 y 互为相反数"),
        (x, y) => println!("x = {}, y = {}", x, y),
    }
}
```

### 匹配 Option

`match` 常用于处理 `Option` 类型：

```rust
fn divide(a: f64, b: f64) -> Option<f64> {
    if b == 0.0 {
        None
    } else {
        Some(a / b)
    }
}

fn main() {
    let result = divide(10.0, 2.0);

    match result {
        Some(value) => println!("结果: {}", value),
        None => println!("除以零"),
    }
}
```

`Option` 类型是 Rust 处理可能不存在值的标准方式，比空指针更安全。我们将在后续章节详细介绍。

### if let 和 while let

对于只关心一种匹配模式的场景，`if let` 和 `while let` 提供了更简洁的语法：

```rust
fn main() {
    let some_value = Some(3);

    // 使用 match
    match some_value {
        Some(3) => println!("是三"),
        _ => (),
    }

    // 使用 if let（更简洁）
    if let Some(3) = some_value {
        println!("是三");
    }

    // 带条件的 if let
    if let Some(x) = some_value {
        println!("值是: {}", x);
    }

    // while let 循环
    let mut optional = Some(0);
    while let Some(i) = optional {
        if i > 5 {
            optional = None;
        } else {
            println!("i = {}", i);
            optional = Some(i + 1);
        }
    }
}
```

## 实际应用示例

### 斐波那契数列

让我们用函数和控制流来实现斐波那契数列：

```rust
/// 计算第 n 个斐波那契数
/// 使用迭代方法，效率更高
fn fibonacci(n: u32) -> u64 {
    if n == 0 {
        return 0;
    }
    if n == 1 {
        return 1;
    }

    let mut a: u64 = 0;
    let mut b: u64 = 1;

    for _ in 2..=n {
        let temp = a + b;
        a = b;
        b = temp;
    }

    b
}

fn main() {
    println!("斐波那契数列前 20 项:");
    for i in 0..20 {
        println!("fib({:2}) = {}", i, fibonacci(i));
    }

    // 找出第一个超过 10000 的斐波那契数
    let mut i = 0;
    loop {
        let fib = fibonacci(i);
        if fib > 10000 {
            println!("第一个超过 10000 的斐波那契数是 fib({}) = {}", i, fib);
            break;
        }
        i += 1;
    }
}
```

这个例子展示了：

1. 函数定义和类型标注
2. 参数和返回值类型
3. `for` 循环遍历范围
4. `loop` 无限循环和条件退出
5. 函数的多次调用
6. 计算和逻辑的结合

### 温度转换工具

让我们创建一个更复杂的温度转换工具：

```rust
/// 温度单位枚举
enum TemperatureUnit {
    Celsius,
    Fahrenheit,
}

/// 将温度从一个单位转换为另一个单位
fn convert_temperature(value: f64, from: TemperatureUnit) -> f64 {
    match from {
        TemperatureUnit::Celsius => value * 1.8 + 32.0,   // 摄氏度转华氏度
        TemperatureUnit::Fahrenheit => (value - 32.0) / 1.8,  // 华氏度转摄氏度
    }
}

/// 判断温度是否舒适
fn is_comfortable(celsius: f64) -> bool {
    match celsius {
        temp if temp >= 18.0 && temp <= 26.0 => true,  // 匹配守卫
        _ => false,
    }
}

fn main() {
    let test_temperatures = [
        (0.0, TemperatureUnit::Celsius),
        (100.0, TemperatureUnit::Celsius),
        (-40.0, TemperatureUnit::Celsius),
        (32.0, TemperatureUnit::Fahrenheit),
        (212.0, TemperatureUnit::Fahrenheit),
    ];

    println!("温度转换结果:");
    for (temp, unit) in test_temperatures.iter() {
        let converted = convert_temperature(*temp, unit.clone());

        match unit {
            TemperatureUnit::Celsius => {
                println!("{:.1}°C = {:.1}°F", temp, converted);
                println!("  舒适吗? {}", is_comfortable(*temp));
            }
            TemperatureUnit::Fahrenheit => {
                println!("{:.1}°F = {:.1}°C", temp, converted);
                println!("  舒适吗? {}", is_comfortable(converted));
            }
        }
    }
}
```

这个例子展示了：

1. 枚举的定义和使用
2. `match` 表达式的模式匹配
3. 匹配守卫（match guard）的使用
4. 函数的参数传递
5. 元组的解构
6. `for` 循环和迭代器

## 函数设计最佳实践

### 单一职责原则

每个函数应该只做一件事，而且做好：

```rust
// 好的设计：每个函数职责单一
fn is_even(n: i32) -> bool {
    n % 2 == 0
}

fn filter_even(numbers: &[i32]) -> Vec<i32> {
    numbers.iter()
        .copied()
        .filter(|&n| is_even(n))
        .collect()
}

// 不好的设计：一个函数做太多事情
fn process_numbers_bad(numbers: &[i32]) -> Vec<i32> {
    let mut result = Vec::new();
    for &n in numbers {
        if n % 2 == 0 {  // 逻辑混在一起
            result.push(n);
        }
    }
    result
}
```

### 有意义的函数名

函数名应该清楚地表达函数的用途：

```rust
// 好的函数名：清楚表达意图
fn calculate_total_price(items: &[f64], tax_rate: f64) -> f64 {
    let subtotal: f64 = items.iter().sum();
    subtotal * (1.0 + tax_rate)
}

// 不好的函数名：过于抽象
fn calc(items: &[f64], t: f64) -> f64 {
    let s: f64 = items.iter().sum();
    s * (1.0 + t)
}
```

### 参数设计

函数参数应该遵循以下原则：

1. **参数数量不宜过多**：超过 3-4 个参数考虑使用结构体
2. **使用合理的类型**：不要为了省事而使用过于通用的类型
3. **考虑借用**：如果只是读取数据，使用引用而不是转移所有权

```rust
// 好的设计：参数清晰，使用引用
struct Point {
    x: f64,
    y: f64,
}

fn distance(p1: &Point, p2: &Point) -> f64 {
    let dx = p2.x - p1.x;
    let dy = p2.y - p1.y;
    (dx * dx + dy * dy).sqrt()
}

// 不好的设计：参数过多
fn distance_bad(x1: f64, y1: f64, x2: f64, y2: f64) -> f64 {
    let dx = x2 - x1;
    let dy = y2 - y1;
    (dx * dx + dy * dy).sqrt()
}
```

### 错误处理

函数应该明确地处理错误，使用 `Result` 类型而不是 panic：

```rust
// 好的错误处理：返回 Result
fn divide(a: f64, b: f64) -> Result<f64, String> {
    if b == 0.0 {
        Err("除数不能为零".to_string())
    } else {
        Ok(a / b)
    }
}

// 不好的错误处理：使用 panic
fn divide_bad(a: f64, b: f64) -> f64 {
    if b == 0.0 {
        panic!("除数不能为零");
    }
    a / b
}
```

## 权衡分析

在使用函数和控制流时，需要在几个方面做出权衡：

**表达式 vs 语句**：

Rust 偏向表达式，几乎所有东西都可以返回值：

- **优势**：代码更简洁，更容易组合，函数式编程风格
- **劣势**：需要理解表达式和语句的区别，初学者可能不习惯

**循环类型选择**：

不同类型的循环适用于不同场景：

- **loop**：适合不确定迭代次数的无限循环，或者需要从循环返回值
- **while**：适合条件驱动的循环，如读取直到文件结束
- **for**：适合遍历集合和范围，是最常用和最安全的循环方式

**match vs if-else**：

`match` 表达式比 `if-else` 更强大，但：

- **match**：适合多个可能的值，编译器会检查穷尽性，更安全
- **if-else**：适合简单的布尔条件，对于两个分支的简单情况更简洁

**函数大小**：

函数应该多长？需要权衡：

- **小函数**：易于理解和测试，复用性好，但可能增加函数调用开销
- **大函数**：包含更多上下文，但难以理解和维护

在 Rust 中，由于零成本抽象，小函数的性能开销通常不是问题。编译器会内联小函数，所以应该优先考虑代码的可读性和可维护性。

## 总结

本文介绍了 Rust 的函数和控制流，主要内容包括：

**函数**：

- 使用 `fn` 定义函数，参数必须显式标注类型
- `->` 指定返回值类型，最后一个表达式作为返回值
- 理解语句和表达式的区别
- 使用 `return` 提前返回

**控制流**：

- `if` 是表达式，可以返回值
- `loop` 创建无限循环，可以返回值
- `while` 条件循环
- `for` 遍历迭代器，最常用
- `match` 强大的模式匹配
- `if let` 和 `while let` 简化语法

**最佳实践**：

- 函数应该有单一职责
- 使用有意义的函数名
- 合理设计参数
- 明确处理错误

掌握函数和控制流后，我们已经准备好学习 Rust 最核心的概念——所有权系统。这是理解 Rust 内存管理的关键，也是下一章的主题。
