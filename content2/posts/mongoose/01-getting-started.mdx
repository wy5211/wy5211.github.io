你用 Node.js 写后端，选了 MongoDB 做数据库。直接用 MongoDB 官方驱动写代码，没过多久就发现了问题：数据验证要手写，类型提示全靠猜，查询代码散落各处。同事说用 Mongoose，但你不知道从哪开始。

这个系列从"为什么需要 Mongoose"出发，带你逐步掌握 Mongoose 的核心能力。整个系列围绕一个 CMS 内容管理系统展开，我们会一起设计完整的数据层。

## 为什么需要 Mongoose

MongoDB 官方驱动确实能用，但它就是个基础的通信工具。想想看，如果你直接用它：

```javascript
// 官方驱动写的代码
const db = client.db("cms");
const result = await db.collection("users").insertOne({
  name: "张三",
  email: "zhangsan@example.com",
  // 这些字段有没有验证？完全不知道
});

// 查询时也是这样
const users = await db
  .collection("users")
  .find({
    status: "active", // 这个字段真的存在吗？谁知道呢
  })
  .toArray();
```

你看，没有任何保障。email 格式对不对？必填字段有没有漏？字段类型对不对？全靠运行时才发现问题。

Mongoose 解决的就是这些问题。它在 MongoDB 驱动之上加了一层抽象，提供了：

- Schema 验证：定义文档结构，数据不符合规则直接拒绝
- 类型安全：配合 TypeScript 有完整的类型推导
- 中间件系统：保存前自动加密密码、删除前级联删除子文档
- 查询构建：链式 API 让查询更直观
- 虚拟字段：方便地计算和组合数据

## MongoDB vs 关系型数据库

在开始之前，先快速对比一下 MongoDB 和传统关系型数据库：

| 特性     | MongoDB          | MySQL/PostgreSQL |
| -------- | ---------------- | ---------------- |
| 数据模型 | 文档型（BSON）   | 表格型（行/列）  |
| Schema   | 灵活的 Schema    | 严格的表结构     |
| 关系     | 嵌入文档 + 引用  | 外键关联         |
| 扩展性   | 水平扩展容易     | 垂直扩展为主     |
| 查询语言 | MQL（类似 JSON） | SQL              |

MongoDB 的文档型模型让它在处理嵌套数据时特别方便。比如我们 CMS 的文章数据：

```javascript
{
  "_id": "post_a1b2c3d4",
  "title": "TypeScript 高级技巧",
  "author": {
    "id": "usr_8x9k2m",
    "name": "张三"
  },
  "tags": ["TypeScript", "前端", "教程"],
  "comments": [
    {
      "content": "很棒的文章",
      "author": "usr_xxx",
      "createdAt": "2026-03-20T10:00:00Z"
    }
  ]
}
```

这种嵌套结构在关系型数据库里需要多张表和复杂的 JOIN，在 MongoDB 里就是一个文档。

## Mongoose 核心概念

Mongoose 有三个核心概念：Schema、Model、Document。它们的关系可以这样理解：

- **Schema（模式定义）**：定义文档的结构、类型、验证器、默认值
- **Model（模型）**：基于 Schema 创建的构造函数，用于创建和查询文档
- **Document（文档实例）**：Model 的实例，代表具体的数据库记录

```typescript
// 定义 Schema
const userSchema = new Schema({
  name: String,
  email: { type: String, unique: true },
});

// 创建 Model
const User = mongoose.model("User", userSchema);

// 创建 Document 实例
const user = new User({ name: "张三" });
await user.save();
```

Schema 就像是蓝图，Model 是根据蓝图创建的工厂，Document 是工厂生产出来的具体产品。

## 搭建开发环境

在开始写代码之前，我们需要先准备好 MongoDB 环境。有两种方式：

### 使用 Docker（推荐）

创建一个 `docker-compose.yml` 文件：

```yaml
version: "3.8"
services:
  mongodb:
    image: mongo:7
    container_name: mongodb_cms
    restart: always
    ports:
      - "27017:27017"
    environment:
      MONGO_INITDB_ROOT_USERNAME: admin
      MONGO_INITDB_ROOT_PASSWORD: password123
      MONGO_INITDB_DATABASE: cms
    volumes:
      - mongodb_data:/data/db

volumes:
  mongodb_data:
```

然后启动 MongoDB：

```bash
docker-compose up -d
```

这样就有了一个本地 MongoDB 实例，数据会持久化存储。

### 使用 MongoDB Atlas（云服务）

如果不想本地装 Docker，可以用 MongoDB Atlas 的免费版本：

1. 访问 https://www.mongodb.com/cloud/atlas
2. 注册并创建免费集群
3. 获取连接字符串，格式类似：
   ```
   mongodb+srv://username:password@cluster0.xxxxx.mongodb.net/cms?retryWrites=true&w=majority
   ```

## 连接 MongoDB

创建数据库连接配置文件 `src/config/database.ts`：

```typescript
import mongoose from "mongoose";

const connectDB = async () => {
  try {
    const options = {
      // 连接池配置
      maxPoolSize: 10,
      minPoolSize: 5,
      // 服务器选择超时
      serverSelectionTimeoutMS: 5000,
      // Socket 超时
      socketTimeoutMS: 45000,
    };

    const conn = await mongoose.connect(
      process.env.MONGODB_URI || "mongodb://localhost:27017/cms",
      options
    );

    console.log(`MongoDB 连接成功: ${conn.connection.host}`);

    // 监听连接事件
    mongoose.connection.on("error", (err) => {
      console.error("MongoDB 连接错误:", err);
    });

    mongoose.connection.on("disconnected", () => {
      console.log("MongoDB 连接断开");
    });

    // 优雅关闭
    process.on("SIGINT", async () => {
      await mongoose.connection.close();
      console.log("MongoDB 连接已关闭");
      process.exit(0);
    });
  } catch (error) {
    console.error("MongoDB 连接失败:", error);
    process.exit(1);
  }
};

export default connectDB;
```

这里配置了连接池参数，maxPoolSize 设置为 10 表示最多维护 10 个连接。这比每次查询都创建连接要高效得多。

## 创建第一个 Schema

我们来创建 CMS 中的用户模型。在 Mongoose 里，Schema 定义了文档的结构：

```typescript
import mongoose, { Document, Schema } from "mongoose";

// 定义 Document 接口
export interface IUser extends Document {
  name: string;
  email: string;
  age?: number;
  createdAt: Date;
  updatedAt: Date;
  getFullName(): string;
}

// 定义 Schema
const userSchema = new Schema<IUser>(
  {
    name: {
      type: String,
      required: [true, "姓名是必填项"],
      trim: true,
      maxlength: [50, "姓名不能超过50个字符"],
    },
    email: {
      type: String,
      required: [true, "邮箱是必填项"],
      unique: true,
      lowercase: true,
      trim: true,
      match: [/^\S+@\S+\.\S+$/, "请输入有效的邮箱地址"],
    },
    age: {
      type: Number,
      min: [0, "年龄不能为负数"],
      max: [150, "年龄不能超过150"],
    },
  },
  {
    timestamps: true, // 自动添加 createdAt 和 updatedAt
  }
);

// 添加实例方法
userSchema.methods.getFullName = function () {
  return this.name;
};

// 添加静态方法
userSchema.statics.findByEmail = function (email: string) {
  return this.findOne({ email });
};

// 创建 Model
const User = mongoose.model<IUser>("User", userSchema);

export default User;
```

这个 Schema 定义了用户的结构，每个字段都有类型和验证规则。比如 email 字段必须是唯一的、符合邮箱格式、自动转小写。timestamps: true 会自动添加 createdAt 和 updatedAt 两个字段，并在保存时自动更新。

## 第一个 CRUD 操作

有了 Model，我们就可以对数据库进行操作了。创建 `src/examples/crud.ts`：

```typescript
import User from "../models/User";
import connectDB from "../config/database";

// 连接数据库
await connectDB();

// 创建用户
const user = new User({
  name: "张三",
  email: "zhangsan@example.com",
  age: 25,
});
await user.save();
console.log("用户创建成功:", user.name);

// 查询用户
const foundUser = await User.findOne({ email: "zhangsan@example.com" });
console.log("查询结果:", foundUser?.name);

// 更新用户
foundUser.age = 26;
await foundUser.save();
console.log("更新年龄:", foundUser.name, foundUser.age);

// 删除用户
await foundUser.deleteOne();
console.log("用户已删除");

process.exit(0);
```

运行之前，先创建 `.env` 文件：

```env
MONGODB_URI=mongodb://localhost:27017/cms
```

然后运行：

```bash
npx ts-node src/examples/crud.ts
```

## 项目结构建议

随着项目变大，建议采用这样的目录结构：

```
mongoose-cms/
├── src/
│   ├── config/
│   │   └── database.ts          # 数据库连接配置
│   ├── models/
│   │   ├── User.ts              # 用户模型
│   │   ├── Post.ts              # 文章模型
│   │   └── Comment.ts           # 评论模型
│   ├── controllers/
│   │   └── userController.ts    # 业务逻辑
│   ├── services/
│   │   └── userService.ts       # 服务层
│   ├── routes/
│   │   └── userRoutes.ts        # 路由定义
│   ├── middleware/
│   │   └── errorHandler.ts      # 错误处理
│   ├── utils/
│   │   └── validators.ts        # 验证工具
│   ├── examples/
│   │   └── crud.ts              # 示例代码
│   └── index.ts                 # 应用入口
├── .env                         # 环境变量
├── .gitignore
├── docker-compose.yml           # Docker 配置
├── package.json
└── tsconfig.json
```

这种结构把不同职责的代码分开，便于维护和测试。

## 环境配置最佳实践

实际项目中，连接字符串不应该硬编码。使用环境变量：

```env
# .env
# 数据库配置
MONGODB_URI=mongodb://localhost:27017/cms

# 应用配置
NODE_ENV=development
PORT=3000
```

然后在代码中使用：

```typescript
const conn = await mongoose.connect(process.env.MONGODB_URI, options);
```

记得把 `.env` 加入 `.gitignore`，避免敏感信息泄露。

## 错误处理

数据库操作可能失败，需要适当的错误处理：

```typescript
try {
  const user = await User.create({
    name: "张三",
    email: "zhangsan@example.com",
  });
} catch (error: any) {
  if (error.code === 11000) {
    console.log("邮箱已存在");
  } else {
    console.log("验证错误:", error.message);
  }
}
```

错误码 11000 表示唯一索引冲突，这里是邮箱重复。

## 总结与思考

这篇文章我们搭建了 Mongoose 开发环境，完成了第一个 CRUD 操作。你可能会想：这看起来比直接用 MongoDB 驱动要复杂啊，多了 Schema 定义、接口声明这些代码。

这个 trade-off 是值得的。多写这些代码换来了：

- 数据验证保障，脏数据进不了数据库
- 类型提示，开发时就能发现错误
- 中间件机制，业务逻辑集中管理

下一篇文章，我们会深入 Schema 和 Model 的设计，为 CMS 创建完整的数据模型。你会看到虚拟字段、实例方法、静态方法这些高级特性如何让代码更简洁。
