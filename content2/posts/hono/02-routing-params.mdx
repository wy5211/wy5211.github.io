在上一篇文章中，我们创建了第一个 Hono 应用，搭建了博客 API 的基础结构。但真正的生产级 API 需要更复杂的路由设计，比如文章列表、文章详情、用户管理、评论系统等等。

如果所有路由都写在一个文件里，很快就会变得难以维护。我们需要一个好的方式来组织这些路由。

Hono 的路由系统为这些问题提供了优雅的解决方案。它不仅简单直观，还具备强大的表达能力，让你能够轻松构建结构清晰、易于维护的 API。

让我们继续完善我们的博客 API，学习如何使用 Hono 的路由系统。

## 基础路由回顾

让我们先快速回顾一下基础的路由定义方式。

```typescript
import { Hono } from "hono";

const app = new Hono();

// 基础路由
app.get("/", (c) => c.text("Home Page"));
app.get("/about", (c) => c.text("About Page"));
app.post("/users", (c) => c.json({ message: "Create User" }));
```

Hono 支持所有标准 HTTP 方法。

```typescript
app.get("/", handler);
app.post("/", handler);
app.put("/", handler);
app.patch("/", handler);
app.delete("/", handler);
app.head("/", handler);
app.options("/", handler);
```

## 路径参数

路径参数是 RESTful API 的核心，让你能够创建动态路由。

### 基础参数

```typescript
// 单个参数
app.get("/users/:id", (c) => {
  const id = c.req.param("id");
  return c.json({ userId: id });
});

// 访问 /users/123
// 响应: { "userId": "123" }
```

在博客 API 中，我们可以用路径参数来获取特定的文章。

```typescript
app.get("/posts/:id", (c) => {
  const id = c.req.param("id");
  const post = posts.find((p) => p.id === id);

  if (!post) {
    return c.json({ error: "Post not found" }, 404);
  }

  return c.json({ data: post });
});
```

### 多个参数

```typescript
// 多个参数
app.get("/users/:userId/posts/:postId", (c) => {
  const { userId, postId } = c.req.param();
  return c.json({
    userId,
    postId,
  });
});

// 访问 /users/42/posts/100
// 响应: { "userId": "42", "postId": "100" }
```

在博客 API 中，我们可以用多个参数来获取特定用户的特定文章。

```typescript
app.get("/users/:userId/posts/:postId", (c) => {
  const { userId, postId } = c.req.param();
  const post = posts.find((p) => p.id === postId && p.authorId === userId);

  if (!post) {
    return c.json({ error: "Post not found" }, 404);
  }

  return c.json({ data: post });
});
```

## 路由分组

当应用变大时，路由分组是组织代码的最佳方式。

### 使用 `route()` 方法

```typescript
import { Hono } from "hono";

const app = new Hono();

// 创建文章路由组
const posts = new Hono();

posts.get("/", (c) => {
  return c.json({ message: "Get all posts" });
});

posts.get("/:id", (c) => {
  const id = c.req.param("id");
  return c.json({ message: `Get post ${id}` });
});

posts.post("/", (c) => {
  return c.json({ message: "Create post" }, 201);
});

posts.put("/:id", (c) => {
  return c.json({ message: `Update post ${c.req.param("id")}` });
});

posts.delete("/:id", (c) => {
  return c.json({ message: `Delete post ${c.req.param("id")}` });
});

// 挂载到主应用
app.route("/posts", posts);

// 现在的完整路径是：
// GET    /posts
// GET    /posts/:id
// POST   /posts
// PUT    /posts/:id
// DELETE /posts/:id
```

### 多层嵌套

对于博客 API，我们可以设计更清晰的路由结构。

```typescript
const app = new Hono();

// API 路由组
const api = new Hono();

// v1 路由组
const v1 = new Hono();

// 文章路由组
const posts = new Hono();
posts.get("/", listPosts);
posts.get("/:id", getPost);

// 用户路由组
const users = new Hono();
users.get("/", listUsers);
users.get("/:id", getUser);

// 评论路由组
const comments = new Hono();
comments.get("/", listComments);
comments.post("/", createComment);

// 组装路由
v1.route("/posts", posts);
v1.route("/users", users);
v1.route("/comments", comments);

// 添加版本前缀
api.route("/v1", v1);

// 添加 API 前缀
app.route("/api", api);

// 最终路径：
// GET /api/v1/posts
// GET /api/v1/posts/:id
// GET /api/v1/users
// GET /api/v1/users/:id
```

### 实际项目结构

```
src/
├── routes/
│   ├── index.ts

│   └── comments.ts
└── app.ts
```

```typescript
// src/routes/posts.ts
import { Hono } from "hono";

const posts = new Hono();
```

## 通配符路由

Hono 支持通配符匹配，让你可以匹配更灵活的路径。

### 星号通配符

```typescript
// 匹配所有路径
app.get("*", (c) => {
  return c.text("This matches all paths");
});

// 匹配 /api 下的所有路径
app.get("/api/*", (c) => {
  return c.text("This matches /api/*");
});
```

在博客 API 中，我们可以用通配符来处理静态文件或管理路由。

```typescript
// 管理后台路由
app.get("/admin/*", adminAuthMiddleware, (c) => {
  return c.text("Admin Area");
});
```

### 捕获通配符内容

```typescript
app.get("/files/*", (c) => {
  // 获取通配符匹配的内容
  const wildcardPath = c.req.param("*");
  return c.json({ path: wildcardPath });
});

// 访问 /files/images/avatar.png
// 响应: { "path": "images/avatar.png" }
```

## 正则表达式路由

对于更复杂的匹配需求，Hono 支持正则表达式。

### 基础正则匹配

```typescript
// 只匹配数字 ID
app.get("/posts/:id{[0-9]+}", (c) => {
  const id = c.req.param("id");
  return c.json({ postId: parseInt(id) });
});

// /posts/123     ✅ 匹配
// /posts/abc     ❌ 不匹配
```

在博客 API 中，我们可以用正则来确保 ID 格式正确。

```typescript
// UUID 格式
app.get(
  "/posts/:id{[0-9a-f]{8}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{12}}",
  (c) => {
    const id = c.req.param("id");
    return c.json({ postId: id });
  }
);

// Slug 格式（字母、数字、连字符）
app.get("/posts/:slug{[a-z0-9-]+}", (c) => {
  const slug = c.req.param("slug");
  return c.json({ slug });
});
```

## 路由顺序与优先级

路由匹配遵循"先定义先匹配"的原则，路由的顺序很重要。

### 路由优先级示例

```typescript
const app = new Hono();

// 1. 特定路由优先
app.get("/posts/featured", (c) => {
  return c.text("Featured posts");
});

// 2. 参数路由在后
app.get("/posts/:id", (c) => {
  return c.text(`Post ${c.req.param("id")}`);
});

// 访问 /posts/featured
// 会匹配第一个路由，输出 "Featured posts"
```

在博客 API 中，我们需要特别注意路由顺序。

```typescript
// ✅ 正确的顺序
app.get("/posts/featured", getFeaturedPosts);
app.get("/posts/:id", getPostById);

// ❌ 错误的顺序
app.get("/posts/:id", getPostById);
app.get("/posts/featured", getFeaturedPosts);
// "featured" 会被当作 :id 处理
```

## 路由级中间件

中间件可以应用到特定路由，实现精细控制。

### 单路由中间件

```typescript
// 认证中间件
const authMiddleware = async (c, next) => {
  const token = c.req.header("Authorization");

  if (!token) {
    return c.json({ error: "Unauthorized" }, 401);
  }

  await next();
};

// 只应用到特定路由
app.post("/posts", authMiddleware, (c) => {
  return c.json({ message: "Create post" });
});

// 公开路由不需要认证
app.get("/posts", (c) => {
  return c.json({ message: "List posts" });
});
```

在博客 API 中，我们可以用路由级中间件来保护创建和更新文章的操作。

```typescript
// 创建文章需要认证
app.post("/posts", authMiddleware, createPost);

// 更新文章需要认证
app.put("/posts/:id", authMiddleware, updatePost);

// 删除文章需要认证
app.delete("/posts/:id", authMiddleware, deletePost);

// 获取文章列表公开
app.get("/posts", listPosts);

// 获取文章详情公开
app.get("/posts/:id", getPost);
```

### 路由组中间件

```typescript
const api = new Hono();

// 应用到整个路由组
api.use("*", authMiddleware);

api.get("/posts", (c) => c.json([]));
api.get("/users", (c) => c.json([]));
```

## 完整示例：博客 API 路由设计

让我们为博客 API 设计一个完整的路由结构。

```typescript
import { Hono } from "hono";
import { logger } from "hono/logger";
import { cors } from "hono/cors";

const app = new Hono();

// 全局中间件
app.use("*", logger());
app.use("*", cors());

// 基础路由
app.get("/", (c) => {
  return c.json({
    message: "Welcome to Blog API",
    version: "1.0.0",
    endpoints: {
      posts: "/api/v1/posts",
      users: "/api/v1/users",
      comments: "/api/v1/comments",
    },
  });
});

app.get("/health", (c) => {
  return c.json({
    status: "ok",
    timestamp: new Date().toISOString(),
  });
});

// ==================== API 路由组 ====================
const api = new Hono();

// v1 路由组
const v1 = new Hono();

// ==================== 文章路由 ====================
const posts = new Hono();

// 获取文章列表
posts.get("/", (c) => {
  const query = c.req.query();
  const page = parseInt(query.page || "1");
  const limit = parseInt(query.limit || "10");

  return c.json({
    data: [],
    meta: { page, limit, total: 0 },
  });
});

// 获取单篇文章
posts.get("/:id", (c) => {
  const id = c.req.param("id");
  return c.json({ data: { id } });
});

// 创建文章（需要认证）
posts.post("/", authMiddleware, async (c) => {
  const body = await c.req.json();
  return c.json({ message: "Post created" }, 201);
});

// 更新文章（需要认证）
posts.put("/:id", authMiddleware, async (c) => {
  const id = c.req.param("id");
  const body = await c.req.json();
  return c.json({ message: `Post ${id} updated` });
});

// 删除文章（需要认证）
posts.delete("/:id", authMiddleware, (c) => {
  const id = c.req.param("id");
  return c.json({ message: `Post ${id} deleted` });
});

// ==================== 用户路由 ====================
const users = new Hono();

users.get("/", (c) => {
  return c.json({ data: [] });
});

users.get("/:id", (c) => {
  const id = c.req.param("id");
  return c.json({ data: { id } });
});

// ==================== 评论路由 ====================
const comments = new Hono();

comments.get("/", (c) => {
  return c.json({ data: [] });
});

comments.post("/", authMiddleware, async (c) => {
  const body = await c.req.json();
  return c.json({ message: "Comment created" }, 201);
});

// ==================== 组装路由 ====================
v1.route("/posts", posts);
v1.route("/users", users);
v1.route("/comments", comments);

api.route("/v1", v1);

app.route("/api", api);

// 404 处理
app.notFound((c) => {
  return c.json(
    {
      error: "Route not found",
      path: c.req.path,
    },
    404
  );
});

export default app;
```

## 权衡与考虑

路由分组让代码组织更清晰，但也增加了文件层级。对于小型项目，可能不需要过度模块化。但对于博客 API 这样会持续增长的项目，提前规划好路由结构是值得的。

正则表达式路由提供了强大的验证能力，但过度使用可能影响性能。在博客 API 中，我们只在确实需要的地方使用正则，比如 UUID 验证。

路由级中间件提供了灵活的权限控制，但需要注意中间件的顺序和作用范围。在博客 API 中，我们用中间件来保护需要认证的操作，同时保持公开路由的可访问性。

在下一篇文章中，我们将深入探讨 Hono 的中间件系统，学习如何构建更复杂的请求处理逻辑，比如日志记录、错误处理、权限验证等。
