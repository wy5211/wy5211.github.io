在构建 Web API 时，数据验证是不可或缺的一环。无效或恶意的数据可能导致应用程序崩溃、数据不一致，甚至安全漏洞。传统的验证方式往往冗长、难以维护，而且与 TypeScript 类型系统脱节。

Zod 是一个 TypeScript-first 的数据验证库，它让你能够用简洁的语法定义 Schema，同时自动推断 TypeScript 类型。这意味着你只需要写一次验证规则，就能同时获得运行时验证和编译时类型检查。

Hono 与 Zod 的结合更是天衣无缝，通过 `@hono/zod-validator` 中间件，你可以轻松实现请求数据验证，让 API 更加健壮和安全。

让我们继续完善博客 API，学习如何使用 Zod 进行数据验证。

## 为什么需要数据验证

### 数据验证的重要性

没有验证的危险代码。

```typescript
// 没有验证的危险代码
app.post("/posts", async (c) => {
  const body = await c.req.json();

  // 直接使用用户数据，没有任何验证
  await db.insert(posts).values(body);

  return c.json({ message: "Post created" });
});
```

问题很多。缺少必填字段怎么办。title 格式不正确怎么办。content 太长怎么办。恶意用户传入多余字段怎么办。

```typescript
// 带验证的安全代码
app.post("/posts", zValidator("json", postSchema), async (c) => {
  const post = c.req.valid("json");
  await db.insert(posts).values(post);

  return c.json({ message: "Post created" });
});
```

### 传统验证 vs Zod

传统验证方式（冗长、重复）。

```typescript
// 传统验证方式
app.post("/posts", async (c) => {
  const body = await c.req.json();

  // 手动验证
  if (!body.title || typeof body.title !== "string") {
    return c.json({ error: "Title is required" }, 400);
  }

  if (body.title.length > 200) {
    return c.json({ error: "Title too long" }, 400);
  }

  if (!body.content || typeof body.content !== "string") {
    return c.json({ error: "Content is required" }, 400);
  }

  // 验证通过后，body 的类型还是 any
  await createPost(body);
});
```

Zod 验证（简洁、类型安全）。

```typescript
// Zod 验证
const postSchema = z.object({
  title: z.string().min(1).max(200),
  content: z.string().min(1),
  published: z.boolean().default(false),
});

app.post("/posts", zValidator("json", postSchema), async (c) => {
  const post = c.req.valid("json");
  // post 类型自动推断为 { title: string; content: string; published: boolean }

  await createPost(post);
});
```

### 基本类型验证

```typescript
import { z } from "zod";

// 字符串验证
const titleSchema = z.string();
titleSchema.parse("Hello"); // ✅ 通过
titleSchema.parse(123); // ❌ 抛出 ZodError

// 带验证规则的字符串
const usernameSchema = z
  .string()
  .min(3, "Username too short")
  .max(20, "Username too long")
  .regex(/^[a-zA-Z0-9_]+$/, "Invalid characters");

// 数字验证
const ageSchema = z
  .number()
  .min(18, "Must be 18 or older")
  .max(120, "Invalid age");

// 布尔值
const isActiveSchema = z.boolean();

// 枚举
const statusSchema = z.enum(["draft", "published", "archived"]);

// 可选类型
const excerptSchema = z.string().optional();

// 带默认值
const publishedSchema = z.boolean().default(false);
```

在博客 API 中，我们可以为文章字段定义验证规则。

## 完整示例：博客 API 验证

### Schema 定义

```typescript
import { z } from "zod";

// 创建文章的 Schema
const createPostSchema = z.object({
  title: z.string().min(1, "标题不能为空").max(200, "标题最多200个字符"),
  content: z.string().min(1, "内容不能为空").max(10000, "内容最多10000个字符"),
  excerpt: z.string().max(500, "摘要最多500个字符").optional(),
  published: z.boolean().default(false),
  tags: z
    .array(z.string())
    .min(1, "至少需要一个标签")
    .max(5, "最多5个标签")
    .optional(),
  categoryId: z.string().uuid("Invalid category ID"),
  coverImage: z.string().url("Invalid cover image URL").optional(),
});

// 更新文章的 Schema（所有字段可选）
const updatePostSchema = createPostSchema.partial();

// 查询参数 Schema
const postQuerySchema = z.object({
  page: z.string().transform(Number).default("1"),
  limit: z.string().transform(Number).default("10"),
  keyword: z.string().optional(),
  categoryId: z.string().uuid().optional(),
  published: z.enum(["true", "false", "all"]).default("all"),
  sortBy: z.enum(["createdAt", "updatedAt", "title"]).default("createdAt"),
  sortOrder: z.enum(["asc", "desc"]).default("desc"),
});

// 路径参数 Schema
const postParamSchema = z.object({
  id: z.string().uuid("Invalid post ID format"),
});
```

### 路由使用

```typescript
import { Hono } from "hono";
import { zValidator } from "@hono/zod-validator";

const app = new Hono();

// 错误处理
const handleValidationError = (result: z.ZodValidationResult, c: any) => {
  if (!result.success) {
    return c.json(
      {
        success: false,
        message: "Validation failed",
        errors: result.error.errors.map((err) => ({
          field: err.path.join("."),
          message: err.message,
        })),
      },
      400
    );
  }
};

// GET /posts - 获取文章列表
app.get(
  "/posts",
  zValidator("query", postQuerySchema, handleValidationError),
  async (c) => {
    const { page, limit, keyword, categoryId, published, sortBy, sortOrder } =
      c.req.valid("query");

    // 查询数据
    const posts = await getPosts({
      page,
      limit,
      keyword,
      categoryId,
      published,
      sortBy,
      sortOrder,
    });

    return c.json({
      success: true,
      data: posts.data,
      meta: posts.meta,
    });
  }
);

// GET /posts/:id - 获取文章详情
app.get(
  "/posts/:id",
  zValidator("param", postParamSchema, handleValidationError),
  async (c) => {
    const { id } = c.req.valid("param");

    const post = await getPost(id);

    if (!post) {
      return c.json(
        {
          success: false,
          message: "Post not found",
        },
        404
      );
    }

    return c.json({
      success: true,
      data: post,
    });
  }
);

// POST /posts - 创建文章
app.post(
  "/posts",
  zValidator("json", createPostSchema, handleValidationError),
  async (c) => {
    const data = c.req.valid("json");
    const currentUser = c.get("user");

    // 检查分类是否存在
    const category = await getCategory(data.categoryId);

    if (!category) {
      return c.json(
        {
          success: false,
          message: "Category not found",
        },
        404
      );
    }

    // 创建文章
    const post = await createPost({
      ...data,
      authorId: currentUser.id,
    });

    return c.json(
      {
        success: true,
        data: post,
        message: "Post created successfully",
      },
      201
    );
  }
);

// PATCH /posts/:id - 更新文章
app.patch(
  "/posts/:id",
  zValidator("param", postParamSchema, handleValidationError),
  zValidator("json", updatePostSchema, handleValidationError),
  async (c) => {
    const { id } = c.req.valid("param");
    const data = c.req.valid("json");
    const currentUser = c.get("user");

    // 检查文章是否存在
    const post = await getPost(id);

    if (!post) {
      return c.json(
        {
          success: false,
          message: "Post not found",
        },
        404
      );
    }

    // 检查权限
    if (post.authorId !== currentUser.id && currentUser.role !== "admin") {
      return c.json(
        {
          success: false,
          message: "Forbidden",
        },
        403
      );
    }

    // 更新文章
    const updated = await updatePost(id, data);

    return c.json({
      success: true,
      data: updated,
      message: "Post updated successfully",
    });
  }
);

// DELETE /posts/:id - 删除文章
app.delete(
  "/posts/:id",
  zValidator("param", postParamSchema, handleValidationError),
  async (c) => {
    const { id } = c.req.valid("param");
    const currentUser = c.get("user");

    const post = await getPost(id);

    if (!post) {
      return c.json(
        {
          success: false,
          message: "Post not found",
        },
        404
      );
    }

    if (post.authorId !== currentUser.id && currentUser.role !== "admin") {
      return c.json(
        {
          success: false,
          message: "Forbidden",
        },
        403
      );
    }

    await deletePost(id);

    return c.json({
      success: true,
      message: "Post deleted successfully",
    });
  }
);

export default app;
```

## 权衡与考虑

Zod 提供了强大的验证能力，但过度使用可能影响性能。在博客 API 中，我们只在真正需要验证的地方使用 Zod，比如用户输入和公开 API。

复杂的验证逻辑可能使 Schema 难以维护。在博客 API 中，我们保持 Schema 简单明了，复杂的业务逻辑放在应用层处理。

错误消息的友好性很重要，但自定义错误消息增加了维护成本。在博客 API 中，我们为关键字段提供友好的错误消息，保持用户体验和开发效率的平衡。

类型推断是 Zod 的一大优势，但需要团队熟悉 TypeScript。对于博客 API 这样的项目，类型安全带来的收益远大于学习成本。

在下一篇文章中，我们将学习错误处理与异常过滤器，了解如何实现全局错误处理机制、使用 HTTPException 和自定义异常、记录和监控错误、处理环境差异（开发 vs 生产）、提供友好的错误响应。
