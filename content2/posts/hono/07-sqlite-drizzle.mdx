## 前言

在上一篇文章中，我们构建了完善的错误处理机制。但无论错误处理做得多么好，如果数据不能持久化，应用就失去了价值。现在让我们为博客 API 添加数据库支持。

你可能用过 Prisma 或 TypeORM，但它们在边缘环境中运行时会有各种限制。Drizzle ORM 是一个新兴的 TypeScript-first ORM，它的设计目标就是在边缘环境中也能完美运行。

Drizzle 的优势在于它的零运行时开销。与 Prisma 不同，Drizzle 不需要在运行时维护一个查询引擎，它生成的 SQL 在编译时就确定了，执行时只是简单的参数绑定。这意味着更快的性能、更小的包体积，以及完美的边缘环境支持。

本文我们将使用 SQLite 作为数据库。虽然 SQLite 通常被认为是开发环境的数据库，但在边缘部署场景下，像 Cloudflare D1、Turso 这样的 SQLite 服务已经可以用于生产环境。

## 为什么选择 Drizzle ORM

先看看 Drizzle 与其他 ORM 的对比。

| 特性            | Drizzle  | Prisma       | TypeORM  |
| --------------- | -------- | ------------ | -------- |
| TypeScript 支持 | 原生     | 优秀         | 需装饰器 |
| 边缘运行        | 完美支持 | 有限         | 不支持   |
| 性能            | 极快     | 快           | 较慢     |
| 运行时开销      | 零开销   | 需要查询引擎 | 重量级   |
| SQL 控制        | 完全控制 | 抽象较高     | 完全控制 |

Drizzle 的 SQL-like API 让你可以像写 SQL 一样写查询，同时保持类型安全。

```typescript
const users = await db
  .select({
    id: users.id,
    name: users.name,
    postCount: sql<number>`count(${posts.id})`,
  })
  .from(users)
  .leftJoin(posts, eq(users.id, posts.authorId))
  .where(eq(users.role, "admin"))
  .groupBy(users.id);
```

## 定义数据库 Schema

让我们为博客 API 定义数据库结构。我们需要用户表、文章表、评论表、标签表和分类表。

```typescript
// src/db/schema.ts
import { sqliteTable, text, integer } from "drizzle-orm/sqlite-core";
import { sql, relations } from "drizzle-orm";

// 用户表
export const users = sqliteTable("users", {
  id: text("id").primaryKey(),
  email: text("email").notNull().unique(),
  username: text("username").notNull().unique(),
  password: text("password").notNull(),
  name: text("name"),
  bio: text("bio"),
  avatar: text("avatar"),
  role: text("role", { enum: ["user", "admin", "moderator"] })
    .notNull()
    .default("user"),
  createdAt: integer("created_at", { mode: "timestamp" })
    .notNull()
    .default(sql`CURRENT_TIMESTAMP`),
  updatedAt: integer("updated_at", { mode: "timestamp" })
    .notNull()
    .default(sql`CURRENT_TIMESTAMP`),
});

// 文章表
export const posts = sqliteTable("posts", {
  id: text("id").primaryKey(),
  title: text("title").notNull(),
  slug: text("slug").notNull().unique(),
  content: text("content").notNull(),
  excerpt: text("excerpt"),
  coverImage: text("cover_image"),
  published: integer("published", { mode: "boolean" }).notNull().default(false),
  authorId: text("author_id")
    .notNull()
    .references(() => users.id, { onDelete: "cascade" }),
  views: integer("views").notNull().default(0),
  createdAt: integer("created_at", { mode: "timestamp" })
    .notNull()
    .default(sql`CURRENT_TIMESTAMP`),
  updatedAt: integer("updated_at", { mode: "timestamp" })
    .notNull()
    .default(sql`CURRENT_TIMESTAMP`),
  publishedAt: integer("published_at", { mode: "timestamp" }),
});

// 评论表
export const comments = sqliteTable("comments", {
  id: text("id").primaryKey(),
  content: text("content").notNull(),
  postId: text("post_id")
    .notNull()
    .references(() => posts.id, { onDelete: "cascade" }),
  authorId: text("author_id")
    .notNull()
    .references(() => users.id, { onDelete: "cascade" }),
  parentId: text("parent_id").references(() => comments.id),
  approved: integer("approved", { mode: "boolean" }).notNull().default(false),
  createdAt: integer("created_at", { mode: "timestamp" })
    .notNull()
    .default(sql`CURRENT_TIMESTAMP`),
});

// 标签表
export const tags = sqliteTable("tags", {
  id: text("id").primaryKey(),
  name: text("name").notNull().unique(),
  slug: text("slug").notNull().unique(),
});

// 文章-标签关联表
export const postTags = sqliteTable("post_tags", {
  postId: text("post_id")
    .notNull()
    .references(() => posts.id, { onDelete: "cascade" }),
  tagId: text("tag_id")
    .notNull()
    .references(() => tags.id, { onDelete: "cascade" }),
});

// 关系定义
export const usersRelations = relations(users, ({ many }) => ({
  posts: many(posts),
  comments: many(comments),
}));

export const postsRelations = relations(posts, ({ one, many }) => ({
  author: one(users, {
    fields: [posts.authorId],
    references: [users.id],
  }),
  comments: many(comments),
  tags: many(postTags),
}));

export const commentsRelations = relations(comments, ({ one }) => ({
  post: one(posts, {
    fields: [comments.postId],
    references: [posts.id],
  }),
  author: one(users, {
    fields: [comments.authorId],
    references: [users.id],
  }),
}));
```

## 数据库连接

配置开发环境的数据库连接。

```typescript
// src/db/index.ts
import { drizzle } from "drizzle-orm/better-sqlite3";
import Database from "better-sqlite3";
import * as schema from "./schema";

const sqlite = new Database("./dev.db");
sqlite.pragma("foreign_keys = ON");

export const db = drizzle(sqlite, { schema });
```

## CRUD 操作

现在让我们看看如何使用 Drizzle 进行 CRUD 操作。

### 创建数据

```typescript
app.post("/api/posts", async (c) => {
  const { title, content, excerpt } = await c.req.json();
  const user = c.get("user");

  const post = await db
    .insert(schema.posts)
    .values({
      id: crypto.randomUUID(),
      title,
      slug: generateSlug(title),
      content,
      excerpt,
      authorId: user.id,
      createdAt: new Date(),
      updatedAt: new Date(),
    })
    .returning();

  return c.json({ success: true, data: post[0] }, 201);
});
```

### 读取数据

```typescript
import { eq, like, and, or, desc, asc } from "drizzle-orm";

// 查询单个文章
app.get("/api/posts/:id", async (c) => {
  const post = await db.query.posts.findFirst({
    where: eq(schema.posts.id, c.req.param("id")),
    with: {
      author: {
        columns: {
          id: true,
          name: true,
          username: true,
          avatar: true,
        },
      },
    },
  });

  if (!post) {
    throw new HTTPException(404, { message: "Post not found" });
  }

  return c.json({ success: true, data: post });
});

// 查询文章列表（带搜索和分页）
app.get("/api/posts", async (c) => {
  const keyword = c.req.query("keyword");
  const page = parseInt(c.req.query("page") || "1");
  const limit = parseInt(c.req.query("limit") || "10");

  const conditions = [];

  if (keyword) {
    conditions.push(
      or(
        like(schema.posts.title, `%${keyword}%`),
        like(schema.posts.content, `%${keyword}%`)
      )
    );
  }

  const where = conditions.length > 0 ? and(...conditions) : undefined;

  const [postsList, totalCount] = await Promise.all([
    db.query.posts.findMany({
      where,
      limit,
      offset: (page - 1) * limit,
      orderBy: [desc(schema.posts.createdAt)],
      with: {
        author: {
          columns: {
            id: true,
            name: true,
            username: true,
          },
        },
      },
    }),
    db
      .select({ count: sql<number>`count(*)` })
      .from(schema.posts)
      .where(where),
  ]);

  return c.json({
    success: true,
    data: postsList,
    meta: {
      page,
      limit,
      total: totalCount[0].count,
      totalPages: Math.ceil(totalCount[0].count / limit),
    },
  });
});
```

### 更新数据

```typescript
app.patch("/api/posts/:id", async (c) => {
  const { id } = c.req.param();
  const { title, content, excerpt, published } = await c.req.json();
  const user = c.get("user");

  const post = await db.query.posts.findFirst({
    where: eq(schema.posts.id, id),
  });

  if (!post) {
    throw new HTTPException(404, { message: "Post not found" });
  }

  if (post.authorId !== user.id) {
    throw new HTTPException(403, { message: "Forbidden" });
  }

  const updated = await db
    .update(schema.posts)
    .set({
      title,
      content,
      excerpt,
      published,
      slug: title ? generateSlug(title) : post.slug,
      updatedAt: new Date(),
    })
    .where(eq(schema.posts.id, id))
    .returning();

  return c.json({ success: true, data: updated[0] });
});
```

### 删除数据

```typescript
app.delete("/api/posts/:id", async (c) => {
  const { id } = c.req.param();
  const user = c.get("user");

  const post = await db.query.posts.findFirst({
    where: eq(schema.posts.id, id),
  });

  if (!post) {
    throw new HTTPException(404, { message: "Post not found" });
  }

  if (post.authorId !== user.id) {
    throw new HTTPException(403, { message: "Forbidden" });
  }

  await db.delete(schema.posts).where(eq(schema.posts.id, id));

  return c.json({ success: true, message: "Post deleted" });
});
```

## 关系查询

Drizzle 的关系查询非常直观。

```typescript
// 查询文章及其评论
app.get("/api/posts/:id", async (c) => {
  const post = await db.query.posts.findFirst({
    where: eq(schema.posts.id, c.req.param("id")),
    with: {
      author: true,
      comments: {
        where: eq(schema.comments.approved, true),
        with: {
          author: {
            columns: {
              id: true,
              name: true,
              avatar: true,
            },
          },
        },
      },
    },
  });

  return c.json({ success: true, data: post });
});
```

## Trade-off 分析

**Drizzle vs Prisma**

Drizzle 的优点是性能更好、包体积更小、边缘环境支持完美。Prisma 的优点是生态更成熟、开发体验更好、自动生成类型更方便。

如果你需要部署到 Cloudflare Workers 或 Deno Deploy，Drizzle 是更好的选择。如果是传统的 Node.js 部署，Prisma 可能更适合。

**SQL-like API vs Query Builder**

Drizzle 的 SQL-like API 让你保持对 SQL 的完全控制，但需要你熟悉 SQL。Prisma 的查询构建器更抽象，但复杂查询时可能不够灵活。
