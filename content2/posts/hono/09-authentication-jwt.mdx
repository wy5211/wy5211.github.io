## 前言

在上一篇文章中，我们构建了完整的博客 CRUD API。但有一个问题：任何人都可以创建、修改或删除文章，这显然不是我们想要的。我们需要用户认证和授权机制。

JWT (JSON Web Token) 是一种现代的认证方案，它不需要在服务器存储会话状态，天然适合边缘环境和微服务架构。JWT 由三部分组成：头部、负载和签名，用点号分隔。

与传统的 Session 认证相比，JWT 的优势在于它是无状态的。服务器不需要存储会话信息，只需要验证签名的有效性。这使得 JWT 非常适合分布式系统和边缘环境。

JWT 也有其局限性。一旦签发，在过期之前很难撤销。因此，通常采用 Access Token 和 Refresh Token 的组合：Access Token 有效期短（如 15 分钟），Refresh Token 有效期长（如 7 天）。当 Access Token 过期时，使用 Refresh Token 获取新的 Access Token。

## 用户注册

实现用户注册功能。

```typescript
// src/validators/auth.ts
import { z } from "zod";

export const registerSchema = z.object({
  email: z.string().min(1).email(),
  username: z
    .string()
    .min(3)
    .max(20)
    .regex(/^[a-zA-Z0-9_]+$/),
  password: z.string().min(8),
  name: z.string().min(1).max(50),
});
```

```typescript
// src/routes/auth.ts
import { Hono } from "hono";
import { zValidator } from "@hono/zod-validator";
import { hash, compare } from "bcrypt";
import { sign, verify } from "hono/jwt";
import { eq } from "drizzle-orm";
import * as schema from "../db/schema";
import { db } from "../db";
import { registerSchema, loginSchema } from "../validators/auth";

const auth = new Hono();

// POST /auth/register - 用户注册
auth.post("/register", zValidator("json", registerSchema), async (c) => {
  const { email, username, password, name } = c.req.valid("json");

  // 检查邮箱是否已注册
  const existingUser = await db.query.users.findFirst({
    where: eq(schema.users.email, email),
  });

  if (existingUser) {
    return c.json(
      {
        success: false,
        code: "EMAIL_EXISTS",
        message: "Email already registered",
      },
      409
    );
  }

  // 检查用户名是否已被占用
  const existingUsername = await db.query.users.findFirst({
    where: eq(schema.users.username, username),
  });

  if (existingUsername) {
    return c.json(
      {
        success: false,
        code: "USERNAME_EXISTS",
        message: "Username already taken",
      },
      409
    );
  }

  // 哈希密码
  const hashedPassword = await hash(password, 10);

  // 创建用户
  const user = await db
    .insert(schema.users)
    .values({
      id: crypto.randomUUID(),
      email,
      username,
      password: hashedPassword,
      name,
      role: "user",
      createdAt: new Date(),
      updatedAt: new Date(),
    })
    .returning();

  const { password: _, ...userWithoutPassword } = user[0];

  return c.json(
    {
      success: true,
      data: userWithoutPassword,
      message: "Registration successful",
    },
    201
  );
});
```

## 用户登录

实现登录功能，生成 Access Token 和 Refresh Token。

```typescript
export const loginSchema = z.object({
  email: z.string().min(1).email(),
  password: z.string().min(1),
});

export const refreshTokenSchema = z.object({
  refreshToken: z.string().min(1),
});
```

```typescript
// POST /auth/login - 用户登录
auth.post("/login", zValidator("json", loginSchema), async (c) => {
  const { email, password } = c.req.valid("json");

  // 查找用户
  const user = await db.query.users.findFirst({
    where: eq(schema.users.email, email),
  });

  if (!user) {
    return c.json(
      {
        success: false,
        code: "INVALID_CREDENTIALS",
        message: "Invalid email or password",
      },
      401
    );
  }

  // 验证密码
  const isValidPassword = await compare(password, user.password);

  if (!isValidPassword) {
    return c.json(
      {
        success: false,
        code: "INVALID_CREDENTIALS",
        message: "Invalid email or password",
      },
      401
    );
  }

  // 生成 Access Token
  const accessToken = await sign(
    {
      sub: user.id,
      email: user.email,
      role: user.role,
      type: "access",
      exp: Math.floor(Date.now() / 1000) + 60 * 15,
    },
    process.env.JWT_SECRET!
  );

  // 生成 Refresh Token
  const refreshToken = await sign(
    {
      sub: user.id,
      type: "refresh",
      exp: Math.floor(Date.now() / 1000) + 60 * 60 * 24 * 7,
    },
    process.env.JWT_REFRESH_SECRET!
  );

  const { password: _, ...userWithoutPassword } = user;

  return c.json({
    success: true,
    data: {
      user: userWithoutPassword,
      accessToken,
      refreshToken,
      expiresIn: 15 * 60,
    },
    message: "Login successful",
  });
});
```

## Token 刷新

实现 Token 刷新机制。

```typescript
// POST /auth/refresh - 刷新 Access Token
auth.post("/refresh", zValidator("json", refreshTokenSchema), async (c) => {
  const { refreshToken } = c.req.valid("json");

  try {
    const payload = await verify(refreshToken, process.env.JWT_REFRESH_SECRET!);

    if (payload.type !== "refresh") {
      return c.json(
        {
          success: false,
          code: "INVALID_TOKEN",
          message: "Invalid token type",
        },
        401
      );
    }

    const user = await db.query.users.findFirst({
      where: eq(schema.users.id, payload.sub as string),
    });

    if (!user) {
      return c.json(
        {
          success: false,
          code: "USER_NOT_FOUND",
          message: "User not found",
        },
        401
      );
    }

    const newAccessToken = await sign(
      {
        sub: user.id,
        email: user.email,
        role: user.role,
        type: "access",
        exp: Math.floor(Date.now() / 1000) + 60 * 15,
      },
      process.env.JWT_SECRET!
    );

    return c.json({
      success: true,
      data: {
        accessToken: newAccessToken,
        expiresIn: 15 * 60,
      },
      message: "Token refreshed successfully",
    });
  } catch (error) {
    return c.json(
      {
        success: false,
        code: "INVALID_TOKEN",
        message: "Invalid refresh token",
      },
      401
    );
  }
});
```

## 认证中间件

创建认证中间件来保护需要登录的路由。

```typescript
// src/middleware/auth.ts
import { Context, Next } from "hono";
import { verify } from "hono/jwt";
import { eq } from "drizzle-orm";
import * as schema from "../db/schema";
import { db } from "../db";

export const authMiddleware = async (c: Context, next: Next) => {
  const authorization = c.req.header("Authorization");

  if (!authorization) {
    return c.json(
      {
        success: false,
        message: "Authorization header required",
      },
      401
    );
  }

  if (!authorization.startsWith("Bearer ")) {
    return c.json(
      {
        success: false,
        message: "Invalid authorization format",
      },
      401
    );
  }

  const token = authorization.replace("Bearer ", "");

  try {
    const payload = await verify(token, process.env.JWT_SECRET!);

    if (payload.type !== "access") {
      return c.json(
        {
          success: false,
          message: "Invalid token type",
        },
        401
      );
    }

    const user = await db.query.users.findFirst({
      where: eq(schema.users.id, payload.sub as string),
    });

    if (!user) {
      return c.json(
        {
          success: false,
          message: "User not found",
        },
        401
      );
    }

    c.set("user", user);

    await next();
  } catch (error) {
    return c.json(
      {
        success: false,
        message: "Invalid or expired token",
      },
      401
    );
  }
};
```

## 权限控制中间件

创建权限检查中间件。

```typescript
// 角色检查中间件
export const requireRole = (...roles: string[]) => {
  return async (c: Context, next: Next) => {
    const user = c.get("user");

    if (!user) {
      return c.json(
        {
          success: false,
          message: "Authentication required",
        },
        401
      );
    }

    if (!roles.includes(user.role)) {
      return c.json(
        {
          success: false,
          message: "Insufficient permissions",
        },
        403
      );
    }

    await next();
  };
};
```

## 保护路由

使用认证中间件保护 API 端点。

```typescript
// 公开路由
app.get("/", (c) => c.text("Public route"));

// 需要认证的路由
app.get("/api/profile", authMiddleware, (c) => {
  const user = c.get("user");
  return c.json({
    success: true,
    data: {
      id: user.id,
      name: user.name,
      email: user.email,
    },
  });
});

// 需要管理员权限的路由
app.get("/api/admin/users", authMiddleware, requireRole("admin"), (c) => {
  return c.json({
    success: true,
    data: [],
  });
});

// 资源所有权检查
app.delete("/api/posts/:id", authMiddleware, async (c) => {
  const user = c.get("user");
  const post = await db.query.posts.findFirst({
    where: eq(schema.posts.id, c.req.param("id")),
  });

  if (!post) {
    return c.json(
      {
        success: false,
        message: "Post not found",
      },
      404
    );
  }

  if (post.authorId !== user.id && user.role !== "admin") {
    return c.json(
      {
        success: false,
        message: "Forbidden",
      },
      403
    );
  }

  await db.delete(schema.posts).where(eq(schema.posts.id, c.req.param("id")));

  return c.json({
    success: true,
    message: "Post deleted",
  });
});
```

## 完整的认证路由

```typescript
// GET /auth/me - 获取当前用户信息
auth.get("/me", authMiddleware, (c) => {
  const user = c.get("user");

  const { password: _, ...userWithoutPassword } = user;

  return c.json({
    success: true,
    data: userWithoutPassword,
  });
});

// PUT /auth/password - 修改密码
auth.put(
  "/password",
  authMiddleware,
  zValidator(
    "json",
    z.object({
      currentPassword: z.string().min(1),
      newPassword: z.string().min(8),
    })
  ),
  async (c) => {
    const user = c.get("user");
    const { currentPassword, newPassword } = c.req.valid("json");

    const isValidPassword = await compare(currentPassword, user.password);

    if (!isValidPassword) {
      return c.json(
        {
          success: false,
          message: "Current password is incorrect",
        },
        401
      );
    }

    const hashedPassword = await hash(newPassword, 10);

    await db
      .update(schema.users)
      .set({
        password: hashedPassword,
        updatedAt: new Date(),
      })
      .where(eq(schema.users.id, user.id));

    return c.json({
      success: true,
      message: "Password updated successfully",
    });
  }
);
```

## Trade-off 分析

**JWT vs Session**

JWT 的优点是无状态、易扩展、适合分布式系统。缺点是难以撤销、payload 较大。

Session 的优点是易于撤销、payload 小。缺点是需要在服务器存储状态、分布式场景下需要共享存储。

对于边缘环境和微服务架构，JWT 是更好的选择。对于传统的单体应用，Session 可能更简单。

**Access Token 有效期**

短期的 Access Token 更安全，但需要频繁刷新。长期的 Access Token 更方便，但一旦泄露风险更大。

建议使用 15 分钟的 Access Token 和 7 天的 Refresh Token，在安全性和便利性之间取得平衡。
