你用过 Express，但它太重了。你想试试边缘计算，Cloudflare Workers 的环境限制很多。你想要一个轻量、快速、能在边缘运行的框架。

Hono 就是为这个场景而生的。核心包只有 13KB，性能比 Express 快 3 倍，支持 Cloudflare Workers、Deno、Bun、Node.js。

在这个系列中，我们将一起构建一个博客 API，逐步学习 Hono 的各种特性。不过先别急，让我们从基础开始，搭建第一个 Hono 应用。

## 为什么选择 Hono

在深入之前，让我们先看看 Hono 在 Web 框架生态中的定位。

### 与主流框架对比

| 特性           | Hono         | Express     | NestJS      | Fastify     |
| -------------- | ------------ | ----------- | ----------- | ----------- |
| **包大小**     | ~13KB (gzip) | ~200KB      | ~500KB+     | ~180KB      |
| **TypeScript** | ✅ 原生支持  | ⚠️ 需要配置 | ✅ 原生支持 | ✅ 原生支持 |
| **边缘运行**   | ✅ 完美支持  | ❌ 不支持   | ❌ 不支持   | ⚠️ 有限支持 |
| **学习曲线**   | 🟢 平缓      | 🟢 平缓     | 🔴 陡峭     | 🟡 中等     |
| **性能**       | 🚀 极快      | 🐢 较慢     | 🐢 较慢     | 🚀 快       |

### Hono 的核心优势

Hono 核心包仅 13KB（gzip），这意味着你的应用启动更快，部署更轻松。

```typescript
// Hono 核心包仅 13KB（gzip）
import { Hono } from "hono";

const app = new Hono();
app.get("/", (c) => c.text("Hello Hono!"));
```

基于 Web Standards 设计，充分利用现代 JavaScript 引擎优化。在基准测试中，Hono 的性能表现优异。

```
Frameworks (simple route)

Express:       38,502 req/sec
Fastify:       78,952 req/sec
Hono:          143,210 req/sec
```

Hono 天生为边缘计算设计，支持 Cloudflare Workers、Deno Deploy、Bun、Node.js (20+)、Fastly Compute@Edge、AWS Lambda。这意味着你的应用可以部署在全球数千个边缘节点，为用户提供毫秒级响应。

100% TypeScript 编写，提供完整的类型推断和智能提示。

```typescript
import { Hono } from "hono";

const app = new Hono();

// 类型安全的路由
app.post("/users", async (c) => {
  const body = await c.req.json();
  // body: any

  return c.json({ message: "User created" });
});
```

如果你用过 Express，你会觉得 Hono 非常熟悉。

```typescript
// Express
app.get("/users/:id", (req, res) => {
  const id = req.params.id;
  res.json({ id });
});

// Hono
app.get("/users/:id", (c) => {
  const id = c.req.param("id");
  return c.json({ id });
});
```

## Hono 的核心概念

Hono 的设计非常简洁，主要由以下几个核心概念组成。

### 应用实例（App）

Hono 应用的核心是 `Hono` 类的实例。

```typescript
import { Hono } from "hono";

const app = new Hono();
```

### 路由（Routes）

定义 API 端点。

```typescript
app.get("/", handler);
app.post("/users", handler);
app.put("/users/:id", handler);
```

### 上下文（Context）

每个请求都有一个 `Context` 对象，包含了请求和响应的所有信息。

```typescript
app.get("/", (c) => {
  // c 是 Context 对象
  // c.req - 请求
  // c.json() - 响应方法
  return c.json({ message: "Hello" });
});
```

### 中间件（Middleware）

在请求处理前后执行逻辑。

```typescript
app.use("*", logger());
app.use("*", cors());
```

这就是 Hono 的全部核心概念，简单吧。

## 环境搭建

让我们开始搭建第一个 Hono 项目。

### 前置要求

- **Node.js** 20+ 或 **Deno** 或 **Bun**（任选其一）
- **包管理器**：pnpm（推荐）/ npm / yarn

### 使用 npm/pnpm 创建项目

```bash
# 创建项目目录
mkdir my-hono-app
cd my-hono-app

# 初始化项目
pnpm init

# 安装 Hono 和适配器
pnpm add hono
pnpm add @hono/node-server
```

为什么需要适配器。Hono 核心是运行时无关的，需要适配器才能在特定运行时上运行。

- `@hono/node-server` - Node.js
- `@hono/cloudflare-workers` - Cloudflare Workers
- Deno/Bun 无需额外适配器

### 项目结构

推荐的项目结构。

```
my-hono-app/
├── src/
│   └── index.ts
├── package.json
└── tsconfig.json
```

创建 `tsconfig.json`。

## 创建第一个 Hono 应用

### 基础版本

创建 `src/index.ts`。

```typescript
import { Hono } from "hono";
import { serve } from "@hono/node-server";

const app = new Hono();

// 定义路由
app.get("/", (c) => {
  return c.text("Hello Hono!");
});

// 启动服务器
const port = 3000;
console.log(`Server is running on http://localhost:${port}`);

serve({
  fetch: app.fetch,
  port,
});
```

### 启动应用

```bash
# 使用 tsx 运行 TypeScript
pnpm add -D tsx
npx tsx src/index.ts
```

访问 `http://localhost:3000`，你将看到。

```
Hello Hono!
```

### 添加更多路由

让我们创建一个更完整的示例。

```typescript
import { Hono } from "hono";
import { serve } from "@hono/node-server";

app.get("/", (c) => {
  return c.json({
```

## 多运行时支持

Hono 最强大的特性之一是"一次编写，到处运行"。同样的代码可以运行在多个运行时上。

### 1. Node.js

```typescript
// src/index.ts
import { Hono } from "hono";
import { serve } from "@hono/node-server";

const app = new Hono();

app.get("/", (c) => c.text("Running on Node.js!"));

export default app;

// 开发环境启动
if (import.meta.env.DEV) {
  serve({
    fetch: app.fetch,
    port: 3000,
  });
}
```

### 2. Deno

```typescript
// deno.json
{
  "tasks": {
    "dev": "deno run --watch --allow-net src/index.ts"
  }
}
```

```typescript
// src/index.ts (Deno)
import { Hono } from "https://deno.land/x/hono/mod.ts";
import { serve } from "https://deno.land/x/hono/server.ts";

const app = new Hono();

app.get("/", (c) => c.text("Running on Deno!"));

await serve(app.fetch, { port: 3000 });
```

### 3. Bun

```typescript
// src/index.ts (Bun)
import { Hono } from "hono";
import { serve } from "hono/bun";

const app = new Hono();

app.get("/", (c) => c.text("Running on Bun!"));

export default {
  port: 3000,
  fetch: app.fetch,
};
```

### 4. Cloudflare Workers

```typescript
// src/index.ts (Cloudflare Workers)
import { Hono } from "hono";

const app = new Hono();

app.get("/", (c) => c.text("Running on Cloudflare Workers!"));

export default app;
```

## 完整示例：博客 API 基础

让我们创建一个简单的博客 API 基础，这将是我们在后续文章中逐步完善的项目。

```typescript
import { Hono } from "hono";
import { logger } from "hono/logger";
import { cors } from "hono/cors";
import { serve } from "@hono/node-server";

// 类型定义
type Post = {
  id: string;
  title: string;
  content: string;
  published: boolean;
  authorId: string;
  createdAt: string;
};

type User = {
  id: string;
  name: string;
  email: string;
  createdAt: string;
};

// 模拟数据库
const users: User[] = [
  {
    id: "1",
    name: "Alice",
    email: "alice@example.com",
    createdAt: new Date().toISOString(),
  },
  {
    id: "2",
    name: "Bob",
    email: "bob@example.com",
    createdAt: new Date().toISOString(),
  },
];

const posts: Post[] = [
  {
    id: "1",
    title: "First Post",
    content: "Hello World",
    published: true,
    authorId: "1",
    createdAt: new Date().toISOString(),
  },
];

const app = new Hono();

// 中间件
app.use("*", logger());
app.use("*", cors());

// 路由
// GET / - 首页
app.get("/", (c) => {
  return c.json({
    message: "Welcome to Blog API",
    version: "1.0.0",
    endpoints: {
      posts: "/posts",
      users: "/users",
    },
  });
});

// GET /health - 健康检查
app.get("/health", (c) => {
  return c.json({
    status: "ok",
    timestamp: new Date().toISOString(),
  });
});

// GET /posts - 获取文章列表
app.get("/posts", (c) => {
  return c.json({
    data: posts,
    total: posts.length,
  });
});

// GET /posts/:id - 获取单篇文章
app.get("/posts/:id", (c) => {
  const id = c.req.param("id");
  const post = posts.find((p) => p.id === id);

  if (!post) {
    return c.json({ error: "Post not found" }, 404);
  }

  return c.json({ data: post });
});

// POST /posts - 创建文章
app.post("/posts", async (c) => {
  const body = await c.req.json();
  const { title, content, authorId } = body;

  if (!title || !content || !authorId) {
    return c.json({ error: "Title, content, and authorId are required" }, 400);
  }

  const newPost: Post = {
    id: String(posts.length + 1),
    title,
    content,
    published: false,
    authorId,
    createdAt: new Date().toISOString(),
  };

  posts.push(newPost);

  return c.json({ data: newPost }, 201);
});

// GET /users - 获取用户列表
app.get("/users", (c) => {
  return c.json({
    data: users,
    total: users.length,
  });
});

// 启动服务器
const port = 3000;
console.log(`Server is running on http://localhost:${port}`);

serve({
  fetch: app.fetch,
  port,
});
```

测试 API。

```bash
# 获取所有文章
curl http://localhost:3000/posts

# 获取单篇文章
curl http://localhost:3000/posts/1

# 创建文章
curl -X POST http://localhost:3000/posts \
  -H "Content-Type: application/json" \
  -d '{"title":"My First Post","content":"This is my post","authorId":"1"}'

# 获取用户列表
curl http://localhost:3000/users
```

## 权衡与考虑

选择 Hono 意味着你在轻量级和性能之间做了很好的平衡。13KB 的包大小让应用启动飞快，边缘部署能力让你的服务更接近用户。但需要注意的是，Hono 相比 Express 这样的成熟框架，生态系统相对较新，某些特殊需求可能需要自己实现。

多运行时支持是 Hono 的一大亮点，但也意味着你需要了解不同运行时的特性和限制。例如，Cloudflare Workers 有自己的环境约束，不能使用所有 Node.js 的 API。

对于博客 API 这样的项目，Hono 的轻量特性和边缘部署能力非常适合。但随着项目复杂度增加，你可能需要更多考虑架构设计和模块化组织。

在下一篇文章中，我们将深入探讨 Hono 的路由系统，学习如何组织更复杂的 API 结构，为我们的博客 API 添加更多功能。
