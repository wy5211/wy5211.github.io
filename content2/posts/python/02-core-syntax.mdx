# 核心语法：用 Java 的视角看 Python

Python 的缩进真的没有大括号吗？——作为一个 Java 开发者，你习惯了用花括号界定代码块。Python 告诉你：缩进就是语法。

上一篇我们搭好了 Python 环境，体验了完全不同的工具链。今天来看 Python 的核心语法——大部分概念你已经会了，但有些地方会让你"不太习惯"。

---

## 变量与动态类型

在 Java 里，声明变量必须写类型：`int x = 10;`。Python 直接砍掉了这一步：

```python
x = 10          # int
x = "hello"     # 现在是 str 了，合法，但不推荐这样做
name = "Alice"  # str
score = 98.5    # float
```

对比 Java 的写法，你少了类型声明、少了分号，也少了"编译器在你写错类型时拦住你"的安全感。Python 的类型检查发生在**运行时**——你把字符串传给一个期望整数的函数，只有调用那一刻才会报错。

这不是 Python 的设计疏忽。动态类型的核心 trade-off 是：用编译期的类型安全换取更快的代码迭代速度。写脚本、做数据处理、快速原型验证，动态类型的优势很明显；但如果项目规模变大、团队协作频繁，缺少编译期检查就会成为负担。Python 的应对方案是**类型注解**（Type Hints），我们在第 07 篇会专门讲。

| 特性     | Java          | Python       |
| -------- | ------------- | ------------ |
| 变量声明 | `int x = 10;` | `x = 10`     |
| 类型检查 | 编译期        | 运行时       |
| 类型变更 | 编译错误      | 合法但不推荐 |

## 基本数据类型

Python 的基本类型和 Java 大同小异，但有几个地方值得注意。

整数不会溢出——Python 的 int 是任意精度的，不需要像 Java 那样区分 `int` 和 `long`，更不需要 `BigInteger`：

```python
big = 10 ** 100  # 直接算，Java 里得用 BigInteger
```

布尔值首字母大写 `True` / `False`，不是 Java 的 `true` / `false`。空值是 `None`，不是 `null`：

```python
result = None
if result is None:      # 注意：用 is 比较 None，不是 ==
    print("没有结果")
```

`is` 比较的是对象身份（类似 Java 的 `==` 对引用的比较），`==` 比较的是值相等。对于 `None` 这个单例对象，Python 惯例是用 `is None` 而不是 `== None`，这和 Java 里用 `== null` 的直觉一致。

## 字符串操作：f-string 的优雅

在 Java 里拼接字符串，你可能写过各种姿势：

```java
// Java 的三种常见方式
String s1 = "Hello, " + name + "! You are " + age + " years old.";
String s2 = String.format("Hello, %s! You are %d years old.", name, age);
String s3 = """
    Hello, %s!
    You are %d years old.
    """.formatted(name, age);
```

Python 的 f-string（3.6+）一行搞定，还能嵌入表达式：

```python
name = "Alice"
age = 30

greeting = f"Hello, {name}! You are {age} years old."
pi = 3.14159265
print(f"Pi = {pi:.2f}")  # Pi = 3.14

# 调试输出 — Python 3.8+，打印变量名和值
print(f"{name=}")  # name='Alice'
```

f-string 没有等价物，它比 Java 的 `String.format` 更简洁，也比 Java 的 `+` 拼接更安全（不会忘空格）。如果你用过 Kotlin 的字符串模板 `${name}`，会发现 f-string 的思路几乎一样。

| 特性         | Java                                   | Python               |
| ------------ | -------------------------------------- | -------------------- |
| 字符串格式化 | `String.format("%s is %d", name, age)` | `f"{name} is {age}"` |
| 空值         | `null`                                 | `None`               |

## 控制流：缩进代替花括号

这是 Java 开发者刚接触 Python 时最不习惯的地方。

```python
score = 85

if score >= 90:
    grade = "A"
elif score >= 80:      # 注意：不是 else if，是 elif
    grade = "B"
elif score >= 70:
    grade = "C"
else:
    grade = "D"
```

没有花括号，没有分号，用冒号和缩进（4 个空格）界定代码块。`elif` 对应 Java 的 `else if`，Python 直接合并成了一个关键字。

这种设计的好处是消除了花括号风格之争（大括号换行还是不换行？），代码结构在视觉上更加清晰。坏处也很明显：混用 Tab 和空格会导致难以排查的 `IndentationError`。好消息是现代编辑器都会自动处理缩进，只要你团队统一用空格而不是 Tab。

三元表达式也比 Java 简洁：

```python
# Python
status = "成年" if age >= 18 else "未成年"

# Java
String status = age >= 18 ? "成年" : "未成年";
```

## match-case：Python 3.10+ 的模式匹配

如果你用过 Java 14+ 的 switch 表达式，Python 3.10 引入的 match-case 会有熟悉感，但能力更强——它支持**结构化模式匹配**：

```python
def describe(value):
    match value:
        case int():
            return f"整数: {value}"
        case str():
            return f"字符串: {value}"
        case [x, y]:           # 匹配长度为 2 的列表，并解构
            return f"二元列表: [{x}, {y}]"
        case {"name": n}:      # 匹配含 name 键的字典
            return f"对象: {n}"
        case 200 | 201 | 204:  # 合并多个值
            return "成功"
        case code if 400 <= code < 500:  # guard 条件
            return f"客户端错误: {code}"
        case _:                 # 兜底，类似 default
            return "未知"
```

Java 的 switch 能匹配常量值和枚举，Python 的 match 能匹配**数据结构**。这不是同一个级别的特性——在 Java 里实现同样的效果，你需要写一堆 `instanceof` 检查和强制类型转换。

| 特性     | Java                           | Python                 |
| -------- | ------------------------------ | ---------------------- |
| 多分支   | `switch (x) { case A -> ... }` | `match x: case A: ...` |
| 模式匹配 | 有限（switch + instanceof）    | 结构化解构             |

## for 循环：基于迭代器

Python 的 `for` 没有传统 C 风格的 `for (int i = 0; i < n; i++)`，它等价于 Java 的 for-each：

```python
fruits = ["apple", "banana", "cherry"]
for fruit in fruits:
    print(fruit)

# 需要 index 时用 enumerate，不要手动维护计数器
for i, fruit in enumerate(fruits):
    print(f"{i}: {fruit}")

# 同时遍历多个列表用 zip
prices = [3.5, 2.0, 5.0]
for fruit, price in zip(fruits, prices):
    print(f"{fruit}: ${price}")
```

`enumerate` 和 `zip` 是 Python 循环里的两个高频函数。Java 里你需要 `IntStream.range(0, list.size()).mapToObj(i -> ...)` 才能实现类似效果，Python 直接内置了。

`range()` 生成数字序列，注意它和 Java 的 `IntStream.range()` 类似但更简洁：

```python
for i in range(5):        # 0, 1, 2, 3, 4
for i in range(2, 10, 3): # 2, 5, 8（起始, 终止, 步长）
```

## while 循环

while 循环和 Java 几乎一样，快速过：

```python
count = 0
while count < 5:
    print(count)
    count += 1
```

Python 的 for/while 有一个独特的 `else` 子句——循环正常结束时（没有被 `break` 打断）执行：

```python
def find_even(numbers):
    for n in numbers:
        if n % 2 == 0:
            print(f"找到偶数: {n}")
            break
    else:
        print("没有偶数")
```

这个语法在其他语言里很少见，属于"用好了很优雅，不用也不会少什么"的特性。

## 函数定义：没有 public/private

在 Java 里，方法是类的一部分，需要 `public` / `private` 修饰。Python 的函数是**一等公民**，不需要依附于类，也没有访问修饰符：

```python
def connect(host: str, port: int = 5432, timeout: int = 30) -> str:
    return f"Connecting to {host}:{port} (timeout: {timeout}s)"

# 调用时可以用关键字参数，不用记参数顺序
connect(port=3306, host="localhost", timeout=60)
```

关键字参数（keyword arguments）是 Python 的一个重要特性。Java 方法调用只能按位置传参，Python 允许你用参数名传参，这对有很多可选参数的函数尤其友好。

关于"没有 private"这个问题：Python 的惯例是**用下划线约定**代替语言层面的访问控制。`_internal` 表示"内部使用，外部不要调用"，`__private` 会触发名称改写（name mangling）。这不是强制的，Python 选择信任开发者而不是用编译器强制约束。这种哲学差异在第 04 篇讲 OOP 时会更深入地讨论。

## \*args 和 \*\*kwargs

这是 Python 参数系统中最让 Java 开发者困惑的部分。

`*args` 接收任意数量的位置参数，对应 Java 的可变参数 `int... args`：

```python
def sum_all(*args):
    return sum(args)

print(sum_all(1, 2, 3))       # 6
print(sum_all(1, 2, 3, 4, 5)) # 15
```

`**kwargs` 接收任意数量的关键字参数，Java 没有原生等价物：

```python
def build_query(table: str, **conditions):
    where = " AND ".join(f"{k} = '{v}'" for k, v in conditions.items())
    return f"SELECT * FROM {table} WHERE {where}"

query = build_query("users", name="Alice", age="30", status="active")
# SELECT * FROM users WHERE name = 'Alice' AND age = '30' AND status = 'active'
```

它们还可以组合使用：

```python
def log_request(method: str, path: str, *args, status: int = 200, **headers):
    print(f"{method} {path} -> {status}")
    print(f"Headers: {headers}")

log_request("GET", "/api/users", status=200, content_type="application/json")
```

理解 `*args` 和 `**kwargs` 的关键是理解 `*` 的含义：在函数定义中，`*` 是**打包**（把多个参数打包成元组/字典）；在函数调用中，`*` 是**解包**（把列表/字典展开成独立参数）：

```python
numbers = [1, 2, 3]
print(*numbers)  # 等价于 print(1, 2, 3)

config = {"host": "localhost", "port": 5432}
connect(**config)  # 等价于 connect(host="localhost", port=5432)
```

| 特性       | Java          | Python             |
| ---------- | ------------- | ------------------ |
| 可变参数   | `int... args` | `*args`            |
| 关键字参数 | 无原生支持    | `**kwargs`         |
| 参数解包   | 无            | `*list` / `**dict` |

## 列表推导式：Stream 的简洁替代

在 Java 里把一个列表的偶数挑出来并平方，你大概会写：

```java
List<Integer> result = numbers.stream()
    .filter(n -> n % 2 == 0)
    .map(n -> n * n)
    .toList();
```

Python 的列表推导式用一行完成同样的操作：

```python
result = [n * n for n in numbers if n % 2 == 0]
```

语法结构是 `[表达式 for 变量 in 可迭代对象 if 条件]`，读起来几乎就是自然语言。还能嵌套——展平二维列表：

```python
matrix = [[1, 2, 3], [4, 5, 6], [7, 8, 9]]
flat = [x for row in matrix for x in row]  # [1, 2, 3, 4, 5, 6, 7, 8, 9]
```

不只是列表，字典和集合也有推导式：

```python
# 字典推导式
word_lengths = {word: len(word) for word in ["hello", "world", "python"]}
# {'hello': 5, 'world': 5, 'python': 6}

# 生成器表达式（用圆括号，惰性求值）
total = sum(x * x for x in range(1000000))  # 不会创建完整列表
```

列表推导式是 Python 最具辨识度的语法特性之一。当推导式变得复杂（嵌套超过两层、条件逻辑很长），就该退回到普通的 for 循环——可读性优先。

| 特性     | Java                                       | Python                |
| -------- | ------------------------------------------ | --------------------- |
| 列表推导 | `list.stream().map(i -> i*2).collect(...)` | `[i*2 for i in list]` |

## 语法差异总览

把今天涉及的核心差异汇总成一张表：

| 特性         | Java                                       | Python                 |
| ------------ | ------------------------------------------ | ---------------------- |
| 变量声明     | `int x = 10;`                              | `x = 10`               |
| 类型检查     | 编译期                                     | 运行时                 |
| 字符串格式化 | `String.format("%s is %d", name, age)`     | `f"{name} is {age}"`   |
| 空值         | `null`                                     | `None`                 |
| 多分支       | `switch (x) { case A -> ... }`             | `match x: case A: ...` |
| 列表推导     | `list.stream().map(i -> i*2).collect(...)` | `[i*2 for i in list]`  |
| 可变参数     | `int... args`                              | `*args`                |
| 关键字参数   | 无原生支持                                 | `**kwargs`             |
| 代码块界定   | 花括号 `{}`                                | 缩进（4 空格）         |
| 函数访问控制 | `public` / `private`                       | 下划线约定 `_` / `__`  |

## Trade-off：Python 语法的设计取舍

Python 的语法设计始终在两个目标之间做平衡：**开发速度**和**运行时安全**。

动态类型让你写代码更快，但失去了编译期的类型检查。Python 用类型注解（第 07 篇详细讲）和 mypy/pyright 等工具来弥补，但这终究是可选的——不像 Java 的类型系统是编译器强制执行的。

缩进代替花括号消除了格式争论，让代码结构一目了然。代价是你必须注意编辑器的配置，混用 Tab 和空格是 Python 新手最常见的坑之一。

`*args` / `**kwargs` 和关键字参数让函数接口非常灵活，但过度的灵活性会降低代码的可读性。当你看到一个函数签名 `def process(*args, **kwargs)` 时，你完全不知道它接受什么参数——这种情况在 Python 代码里应该避免，用类型注解把接口说清楚。

列表推导式简洁强大，但嵌套推导式和复杂条件会让代码变成一行天书。Python 的哲学是"优雅优于复杂"——推导式写得读起来费劲的时候，就该换成普通的 for 循环。

这些取舍背后是 Python 的核心设计哲学：**Explicit is better than implicit**（显式优于隐式）、**Simple is better than complex**（简单优于复杂）。理解这些哲学，比记住语法细节更重要。
